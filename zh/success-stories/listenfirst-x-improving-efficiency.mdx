<div id="situation">
  ## 情况
</div>

长期以来，数字营销人员一直缺乏对数据的统一视图。一个系统里可能是自然数据，另一个系统里是付费数据，而对话则来自不同渠道。但是，当客户和利益相关方提出诸如“我昨天获得了多少曝光？”紧接着又问“自然与付费的占比是多少？”这类问题时，如何以更精简的方式汇总并解读这些数据就显得尤为重要。

为解决这些问题，ListenFirst——一家基于 X Developer Platform 构建、深耕超过 10 年的社交分析平台——需要创建一款工具，从单一可信来源收集 X 的自然与付费数据。过去，这些自然与付费数据分散在不同的平台、API 和端点的各自孤岛中；而且它们从未被设计用于汇总、求和和/或混合在一起。

<div id="solution">
  ## 解决方案
</div>

为探索更高效的客户洞察方法，ListenFirst 采用了 X 的 v2 端点，将用户在 X 上的讨论更顺畅地映射到其平台中。ListenFirst 团队围绕一个明确目标展开工作：打造更干净的数据。

> *“对我们而言，使用 v2 是为了优化集成。如果我们在站点上需要拼接的模块更少，就能让运营更轻松，并使我们能够将产品扩展到更多品牌、更多内容以及更优的洞察。”*
> 
> - Jonathan Farb，ListenFirst 首席产品官。

促使团队优先采用 X 的 v2 API，尤其是 Tweets 查找端点的，是其互动与曝光数据的准确性。通过使用该端点，ListenFirst 能够生成更可靠的指标，与其客户在其他原生 X 分析中看到的结果保持一致。

<div id="impact">
  ## 影响
</div>

尽管仍处于早期阶段，采用 v2 已经提升了 ListenFirst 的效率。过去，ListenFirst 需要从不同的 API 集合中拼接数据，而这些数据与原生分析中的结果并不一致。他们对 v2 的使用使 ListenFirst 的数据更接近 X 的原生分析，实现了更高的一致性，从而提升了准确性与客户信任。现在，ListenFirst 能够产出可辩护且更为精炼的情报，帮助品牌在 X 上完善其营销策略。ListenFirst 认为，进一步采用 v2 将通过提升准确性与透明度来增强客户对指标的信心。
