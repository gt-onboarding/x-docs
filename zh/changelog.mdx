<Update label="Aug 3, 2022" description="X API v2">
  ### 新增过滤运算符别名

  在企业版、高级版和 v2 的搜索与过滤流端点中新增了过滤/查询运算符“别名”。在许多情况下，运算符指代的是帖子而非状态，例如，用 "in\_reply\_to\_tweet\_id" 作为 "in\_reply\_to\_status\_id" 的别名。在其他情况下，新的运算符别名能更清晰地表达匹配行为。例如，由于帖子只能包含一个视频附件，因此为 "has:videos" 提供了新的别名 "has:video\_link"。查看[v2 过滤流运算符表](https://developer.x.com/en/docs/twitter-api/tweets/filtered-stream/integrate/build-a-rule#list)以了解更多示例。
</Update>

<Update label="Dec 14, 2021" description="X API v2: Authentication">
  ### 支持 OAuth 2.0 身份验证

  从今天起，所有开发者都可以在开发者门户中选择 OAuth 2.0 作为身份验证方法进行认证。我们已在此次发布中为机密和公共客户端以及所有相关的 v2 端点添加了对此身份验证方法的支持。

  OAuth 2.0 是行业标准的授权协议，可让开发者对应用的作用域拥有更多控制，并改进跨多设备的授权流程。换言之，为 X 上的用户构建应用的开发者将能更好地控制其应用向用户请求的信息和数据，这样你只需向终端用户请求所需的数据和信息。

  这一现代授权协议将使你能够为终端用户呈现更精简的同意流程来授权你的应用，其中仅显示你向他们请求的特定作用域。

  如需了解此次发布的更多信息，请访问我们的论坛公告。
</Update>

<Update label="Nov 15, 2021" description="X API v2: Early Access">
  ### X API v2 的重大平台更新

  **重大平台更新：X API v2 现为主要版本，新增 Essential 和 Elevated 访问级别，以及政策变更**

  今天，我们宣布 X API v2 现已成为 X API 的主要版本。我们已在 X API v2 中推出了足够的端点与功能，可满足在 X API 上构建的所有现有应用中 90% 的需求。

  我们今天推出以下更改，以进一步提升 X API v2 的开发者体验：

  - 提供快速且免费的 Essential 级 API 访问；对使用场景已获批准的开发者提供免费的 Elevated 访问。关于这些访问级别的更多详情，请参阅我们的 About the X API 页面。

  - 我们正在移除开发者政策中限制你基于 X 核心功能进行构建、以及限制你的应用可支持用户数量的相关措辞。

    - 我们正在开放平台，鼓励你构建能让 X 更完善、更健康，并扩展公共对话的工具与产品。[查看我们希望你构建的解决方案列表](https://developer.x.com/en/docs/twitter-api/what-to-build)。

    - 具体而言，我们移除了限制复刻 X 体验的条款（包括 X 的核心功能），以及要求获得许可才能拥有大量用户令牌的条款。

    - 我们深知，在 X 上帮助用户的解决方案往往意味着开发者需要构建（或复刻）X 上已有的一些功能。这些对开发者政策的变更旨在为开发者生态提供清晰指引，并提供一个开放的 API 平台，使开发者更容易构建、创新，并对公共对话产生影响。

  我们在[论坛帖子](https://devcommunity.x.com/t/ushering-in-a-new-era-for-the-twitter-developer-platform-with-the-twitter-api-v2/162087)中更详细地介绍了此次发布，以及此前促成这一时刻的各项 X API 发布内容。

  除了查看更早的更新日志条目外，我们还整理了一些关键资源，供你了解迄今为止在 X API v2 中发布的全部内容：

  - [X API 端点映射](/zh/x-api/migrate/x-api-endpoint-map)

  - [关于 X API](/zh/x-api/getting-started/about-x-api)
</Update>

<Update label="May 10, 2018" description="X API: Standard v1.1">
  ### 引用负载渲染方式变更

  今天，我们对引用负载中 URL 的渲染方式进行了调整。

  我们新增一个名为 "quoted\_status\_permalink" 的实体，以确保能够引用被引用的帖子，并将从被引用帖子的 "text" 字段中移除 t.co 链接。

  在新格式中，t.co 链接将不再附加在 "quoted\_status" 对象中被引用帖子的 "text" 字段末尾。

  此外，我们将不再把被引用帖子的 t.co url、expanded\_url 和 display\_url 放在 "urls" 实体中。相反，这些将被添加到一个新的 "quoted\_status\_permalink" 对象中，包含 "url"、"expanded" 和 "display" 这三个 URL 属性。也就是说，"quoted\_status\_permalink" 对象将专用于指向被引用帖子的 t.co 链接，而 "entities.urls" 数组可用于识别原帖或引用内分享的其他链接。

  详情请阅读我们的[论坛帖子](https://devcommunity.x.com/t/updating-how-urls-are-rendered-in-the-quote-tweet-payload/105473)和[更新文档](https://devcommunity.x.com/t/upcoming-changes-to-the-developer-platform/104603)。
</Update>
