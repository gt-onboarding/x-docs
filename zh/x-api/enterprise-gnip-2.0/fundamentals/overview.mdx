<div id="overview">
  ## 概述
</div>

> 企业控制台（位于 console.gnip.com）面向已签订企业 API 访问协议的企业数据客户开放。若您目前尚非企业数据客户，但希望了解更多信息，您可在此处进一步了解[企业数据](https://developer.x.com/en/products/x-api/enterprise)。

本文概述了 X 的企业控制台仪表板。以下各部分对应控制台中的具体区域，涵盖产品详情、使用情况以及通用账户管理。如对您的账户有任何疑问，请联系您指定的客户经理，或查阅我们的[企业产品技术文档](/zh/x-api/enterprise-gnip-2.0/enterprise-gnip)。

作为与客户经理配合进行的企业试用与入门流程的一部分，企业客户将获得 [console.gnip.com](http://console.gnip.com) 的登录权限，该网站是用于查看和配置企业产品访问权限的用户界面。控制台的管理员初始访问权限由 X 在入门流程中开通，随后管理员可添加其他用户。企业控制台可帮助您管理并查看与企业产品相关的更多详细信息。 

以下视频对 console.gnip.com 仪表板的各个部分进行了概述。

<div id="products-tab">
  ## 产品选项卡
</div>

登录您的账户并进入 [console.gnip.com](http://console.gnip.com) 后，您将停留在仪表板的“产品”选项卡。此页面提供您账户当前可用企业产品的概览。

对于采用流式投递的产品（如 PowerTrack 或 Decahose），此页面会列出产品名称、流标签、当前活跃连接数、（如适用）每个流的当前活跃规则数量，以及最近 24 小时内投递的活动原始计数（例如帖子）。

对于采用 REST 投递的产品（如 Search API），此页面会列出产品名称、标签（也显示为“streams”）、通过这些端点返回的当前活动（例如帖子），以及每个端点的多种请求计数。

请注意，[Usage API](/zh/x-api/enterprise-gnip-2.0/fundamentals/usage) 可通过编程方式提供上述大部分数据。

点击名称或设置按钮可查看每个流的具体详情。

<div id="overview">
  ### 概览
</div>

在主控制台点击某个流即可进入该流的概览页面。

对于流式投递产品，此页面包括：

1. 各个具体流连接向你投递活动数量的趋势图

2. 当前该流上处于活跃状态的连接详情（连接 ID 和 IP 地址）

3. 该流的近期连接、断开连接和规则更新事件日志

请注意，可通过右上角的链接调整图表的刻度。你可以点击图表正下方图例中的相应项，切换显示单个连接和断开事件。

<div id="connections">
  ### 连接
</div>

“连接”页面提供你的流中最近连接事件的详细信息。其中包括每次连接的开始和结束时间（UTC 24 小时制）、每次连接的持续时长、发起连接的服务器 IP、用于参考的唯一连接 ID，以及当前连接状态。状态对应于该连接的最新事件——例如“客户端已连接”，或“已断开连接”（并标注断开类型）。有关连接调试的更多信息，请参阅[断连说明指南](/zh/x-api/enterprise-gnip-2.0/powertrack-api#integrating-with-powertrack)。

<div id="api-help">
  ### API 帮助
</div>

“API 帮助”页面会提供你的流对应的 API 端点 URL，以及（如适用）该流的规则 API 端点。除此之外，还包含示例 curl 命令，并说明如何连接到流端点，以及如何通过编程方式在该流的规则 API 端点添加、删除和列出规则。

<div id="rules">
  ### 规则
</div>

“规则”选项卡适用于 PowerTrack 流，提供一个通过用户界面手动输入纯文本规则的快速入门途径。请注意，该界面采用此手动方式最多仅支持添加 1000 条规则，且仅应用于初期测试。我们建议在任何生产环境中，通过 [Rules API](/zh/x-api/enterprise-gnip-2.0/powertrack-api#api-reference-index) 以编程方式管理规则。

<div id="settings">
  ### 设置
</div>

“设置”选项卡用于切换流中数据的输出格式，支持多种格式选项。要切换格式，只需使用相应的单选按钮。更改将在重新连接到流时生效。请注意，更新此设置会在下一次请求或连接时立即生效，并且新格式可能会导致你的解析器出现问题。

**请注意：** 若要获取尽可能多的数据，建议选择“保留数据的原始格式”。该选项将按[此处](/zh/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#post-object)所述返回增强的原生格式数据。Activity Streams 格式说明见[此处](/zh/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#activity-object)。

<div id="usage-tab">
  ## 使用标签页
</div>

“使用”标签页可提供不同时间段内你的流使用情况的洞察。若需以编程方式访问使用信息，请参阅[Usage API](/zh/x-api/enterprise-gnip-2.0/fundamentals/usage)。

<div id="monthly">
  ### 每月
</div>

“每月使用量”页面按产品细分显示你的流式用量。例如，对于某个数据源，覆盖类产品（如 PowerTrack、Historical PowerTrack 和 PowerTrack Replay）会被归为一组，既分别提供各自的数据，也提供合并汇总的总计数。计数包含本月至今、预计月末（基于本月迄今的使用情况及本月剩余时间）以及前两个月的计数。

需要注意，这些计数会在每个产品和流层面去重。如果你通过多个连接从同一 PowerTrack 流接收到相同的帖子（Post），则该帖子在此仅计数一次。计数将每 24 小时于 00:00 UTC 更新。

<div id="daily">
  ### 每日
</div>

“每日用量”页面按产品细分，提供当月每一天的去重计数。计数每 24 小时在协调世界时（UTC）00:00 更新。

<div id="rules">
  ### 规则
</div>

规则限制取决于您所签订的 PowerTrack 规则套餐，并在给定“类型”的所有流中统一生效。每当新增或删除规则时，计数都会更新。

<div id="account-tab">
  ## 账户标签页
</div>

<div id="my-profile">
  ### 我的个人资料
</div>

在此查看您的个人资料详情并更改密码。此外，您可以为自己的账户设置使用阈值通知。API 状态通知可在状态页面 https://api.twitterstat.us 查看。

<div id="account-settings">
  ### 账户设置
</div>

您可以添加、删除和编辑用户，并为单个用户和使用量阈值警报配置电子邮件通知。

<Frame>
  <img src="/images/account-settings.png.twimg.1920.png" />
</Frame>

请注意，用户分为三种类型：Account Admin、User 和 Email Only。

- Account Admin 可以创建、删除和编辑其他用户，并可使用用户名（电子邮件）和密码进行基本身份验证以连接企业 API
- User 无法创建或修改其他用户，但可以使用用户名（电子邮件）和密码进行基本身份验证以连接企业 API
- Email Only 用户无法访问仪表板，无权连接企业 API，且仅在“通知”部分被配置为接收时才会收到通知。

<div id="usage-thresholds">
  ### 使用阈值
</div>

为你的产品配置用量阈值。对在个人资料中启用了相关通知的用户而言，无论是每个产品的预警阈值还是关键阈值，均会触发电子邮件提醒。

<Frame>
  <img src="/images/usage-thresholds.png.twimg.1920.png" />
</Frame>

请注意，这些阈值每天在 19:30（UTC）进行一次评估，非实时评估。
