  ## 比较 X API 的关注相关端点
</div>

<div id="follows-lookup">
  ### 关注关系查询
</div>

v2 关注关系查询端点将取代标准 v1.1 的 [followers/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-followers-ids)、v1.1 的 [followers/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-followers-list)、v1.1 的 [friends/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-friends-ids) 以及 v1.1 的 [friends/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-friends-list) 端点。

下表对比了各类关注关系查询端点：

|     |     |     |
| :--- | :--- | :--- |
| **描述** | **标准 v1.1** | **X API v2** |
| 支持的 HTTP 方法 | `GET` | `GET` |
| 主机域名 | `https://api.x.com` | `https://api.x.com` |
| 端点路径 | /1.1/friends/ids.json<br /><br />/1.1/friends/list.json<br /><br />/1.1/followers/ids.json<br /><br />/1.1/followers/list.json | /2/users/:id/following<br /><br />/2/users/:id/followers |
| [身份验证](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文<br /><br />仅应用 | OAuth 1.0a 用户上下文<br /><br />OAuth 2.0（含 PKCE）的授权码流程<br /><br />仅应用 |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 每 15 分钟 15 次请求（每用户）<br /><br />每 15 分钟 15 次请求（每应用） | 每 15 分钟 15 次请求（每用户）<br /><br />每 15 分钟 15 次请求（每应用） |
| 每个响应的最大用户数 | GET friends/id 与 GET followers/id 每页最多返回 5000 个用户 ID。<br /><br />  <br /><br />GET friends/list 与 GET followers/list 每页最多返回 200 个用户对象。 | 每页 1000 个用户对象 |
| 分页 | 令牌在 next\_cursor 字段返回，然后可将其作为 cursor 参数的值传入以获取下一页结果。 | 令牌在 next\_token 字段返回，然后可将其作为 token 参数的值传入以获取下一页结果。<br /><br />v2 负载还提供 previous\_token 字段，也可与 pagination\_token 参数一并传入以获取上一页结果。 |
| JSON 格式 | 标准 v1.1 格式 | [X API v2 格式](/zh/x-api/fundamentals/data-dictionary)（由 fields 与 expansions 请求参数决定，与 v1.1 格式不向后兼容）<br /><br />要了解如何从标准 v1.1 格式迁移到 X API v2 格式，请参阅我们的[数据格式迁移指南](/zh/x-api/migrate/data-format-migration)。 |
| 是否支持选择在负载中返回哪些[字段](/zh/x-api/fundamentals/data-dictionary) |     | ✔   |
| 是否支持帖子[注释](/zh/x-api/fundamentals/post-annotations)字段 |     | ✔   |
| 是否支持请求新的[指标](/zh/x-api/fundamentals/metrics)字段 |     | ✔   |
| 是否支持 [conversation\_id](/zh/x-api/fundamentals/conversation-id) 字段 |     | ✔   |
| 是否要求使用由与[项目](/zh/resources/fundamentals/projects)关联的[开发者应用](/zh/resources/fundamentals/developer-apps)提供的凭据 |     | ✔   |

<div id="manage-follows">
  #### 管理关注
</div>

v2 的“管理关注”端点将取代标准版 v1.1 的 [POST friendships/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-create) 和 [POST friendships/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-destroy) 端点。

下表对比了标准版 v1.1 与 X API v2 的“创建关注”端点：

<div id="follow-a-user">
  #### 关注用户
</div>

| 描述 | 标准 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | POST | POST |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/friendships/create.json | /2/users/:id/following |
| [身份验证](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文 | OAuth 1.0a 用户上下文<br /><br />OAuth 2.0 授权码（含 PKCE） |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 每 15 分钟 50 次请求 | 每 15 分钟 50 次请求 |
| 单个用户每日最大操作次数 | 400 | 400 |
| 单个应用每日最大操作次数 | 1000 | 1000 |
| 必须使用与[项目](/zh/resources/fundamentals/projects)关联的[开发者应用](/zh/resources/fundamentals/developer-apps)的凭据 |     | ✔️  |

<div id="unfollow-a-user">
  #### 取消关注用户
</div>

下表比较了标准版 v1.1 与 X API v2 的取消关注端点：

| 说明 | 标准版 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | POST | DELETE |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/friendships/destroy.json | /2/users/:source\_user\_id/following/:target\_user\_id |
| [身份验证](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文 | OAuth 1.0a 用户上下文<br /><br />OAuth 2.0 授权码（含 PKCE） |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 每 15 分钟 15 次请求（每用户） | 每 15 分钟 50 次请求（每用户） |
| 每个应用每日最大操作次数 | 无 | 500 |
| 需要使用与[项目](/zh/resources/fundamentals/projects)关联的[开发者应用](/zh/resources/fundamentals/developer-apps)的凭证 |     | ✔️  |
