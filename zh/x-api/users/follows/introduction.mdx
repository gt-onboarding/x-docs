我们提供两组端点组，帮助你查询、创建和删除关注关系：follows lookup 和 manage follows。

<div id="follows-lookup">
  ### 关注查找
</div>

仅向具有[企业版访问权限](/zh/x-api/getting-started/about-x-api)的用户开放

“关注查找”端点可用于探索和分析用户之间的关系（常称为网络分析）。具体来说，有两个 REST 端点会返回[用户对象](/zh/x-api/fundamentals/data-dictionary#user)，用于表示指定用户正在关注的人，或正在关注该指定用户的人。

您可以使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication)、\[仅应用]\(https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)、或[OAuth 2.0 授权码（含 PKCE）](zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 对该端点进行身份验证。每次最多可请求 1,000 个用户；对于大结果集，将提供分页令牌以便翻页检索。
 

<div id="manage-follows">
  ### 管理关注
</div>

“管理关注”端点可用于关注或取消关注用户。

由于你是在代表某位用户发起请求，必须使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 或 [OAuth 2.0 Authorization Code with PKCE](zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)，并使用与该被代表用户关联的用户访问令牌。你可以通过 [三方 OAuth 流程](/zh/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）或使用 [含 PKCE 的授权码授予流程](/zh/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)（OAuth 2.0）生成该用户访问令牌。

代表每个已认证用户，你每天最多可执行 400 次关注操作；在你所有已认证用户的总量上，每个应用每天最多可执行 1,000 次操作。比如，如果你有五个已认证用户，你可以用其中两个用户分别每天关注 400 个用户（按用户上限），合计 800 次操作，剩余的 200 次操作（按应用上限）需要在其余三个用户之间分配。该限制不适用于取消关注端点，后者有单独的上限：每个应用每天 500 次操作。

<Note>
  **账户设置**

  要访问这些端点，你需要：

  - 一个已获批准的 [开发者账户](https://developer.x.com/en/portal/petition/essential/basic-info)。
  - 使用位于某个 [项目](/zh/resources/fundamentals/projects) 内的 [开发者应用](/zh/resources/fundamentals/developer-apps) 的密钥和令牌进行认证。
</Note>

在我们的[入门指南](/zh/x-api/getting-started/getting-access)中了解如何获取对 X API v2 端点的访问权限。
