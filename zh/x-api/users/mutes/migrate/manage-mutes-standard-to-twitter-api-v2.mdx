<div id="manage-mutes-standard-v11-compared-to-x-api-v2">
  ### 管理静音：标准版 v1.1 与 X API v2 的比较
</div>

如果你一直在使用标准版 v1.1 的 [POST mutes/users/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-create) 和 [POST mutes/users/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-destroy) 端点，本指南旨在帮助你理解标准版 v1.1 与 X API v2 在“管理静音”端点上的相同点与差异。

- **相同点**
  - OAuth 1.0a 用户上下文
- **不同点**
  - 端点 URL
  - 应用与项目要求
  - HTTP 方法
  - 请求参数
     

<div id="similarities">
  #### 相似点
</div>

**OAuth 1.0a 用户上下文身份验证方法**

这两个版本的端点都支持 [OAuth 1.0a 用户上下文](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication)。因此，如果你之前使用的是标准 v1.1 的“管理静音”端点之一，那么在迁移到 X API v2 版本时，可以继续使用相同的身份验证方法。

<div id="differences">
  #### 差异
</div>

**端点 URL**

- 标准 v1.1 端点：
  - POST https://api.x.com/1.1/mutes/users/create.json
    （将某用户设为静音）
  - POST https://api.x.com/1.1/mutes/users/destroy.json
    （取消对某用户的静音）
- X API v2 端点：
  - POST https://api.x.com/2/users/:id/muting
    （将某用户设为静音）
  - DELETE https://api.x.com/2/users/:source\_user\_id/muting/:target\_user\_id
    （取消对某用户的静音） 

**应用与项目要求**

使用 X API v2 端点进行身份验证时，必须使用与某个[项目](/zh/resources/fundamentals/projects)关联的[开发者应用](/zh/resources/fundamentals/developer-apps)的凭证。所有 X API v1.1 端点都可以使用独立应用或与项目关联的应用的凭证。

**请求参数**

以下标准 v1.1 请求参数在 X API v2 中有对应项：

| 标准 v1.1 | X API v2 |
| :--- | :--- |
| user\_id | target\_user\_id |
| screen\_name | 无对应项 |

请注意，标准 v1.1 参数通过查询参数传递，而 X API v2 参数通过请求体参数（用于 POST 端点）或路径参数（用于 DELETE 端点）传递。

另外，使用标准 v1.1 端点时，无需提供对目标用户执行静音操作的用户的 id，因为在[OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication)中传递的访问令牌会推断由哪个用户发起静音/取消静音操作。
