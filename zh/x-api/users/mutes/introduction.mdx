将某个账户设为静音可以将其帖子从你的时间线上隐藏，而无需取消关注或拉黑该账户。被静音的账户不会知道你已将其静音，你也可以随时取消静音。借助“管理静音”端点，开发者可以为 X 上的用户打造更安全的体验。一个用法示例是：应用允许你在指定时长内将可能发布特定主题内容的账户设为静音。使用“静音查找”端点，你可以查看你或某个已认证用户已设为静音的账户。这有助于评估你与这些被静音账户的互动方式。

由于你在使用静音查找会请求私人信息，并在“管理静音”中代表用户执行操作，你必须使用 [OAuth 1.0a User Context](/zh/resources/fundamentals/authentication) 或 [OAuth 2.0 Authorization Code with PKCE](zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 对这些端点进行身份验证，并使用已授权你的应用的用户所关联的用户访问令牌。该令牌可通过 [3-legged OAuth flow](/zh/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）或 [Authorization Code with PKCE grant flow](/zh/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)（OAuth 2.0）生成。

<div id="mutes-lookup">
  ### 静音查询
</div>

静音查询端点用于查看经过身份验证的用户已静音的账户。该端点的速率限制为每位用户每 15 分钟最多 15 次请求。

<div id="manage-mutes">
  ### 管理静音
</div>

“管理静音”端点允许您代表已认证用户对指定账号进行静音或取消静音。对于这些端点，提供两种方法：POST 和 DELETE。POST 方法用于将账号设为静音，DELETE 方法用于取消静音。POST 和 DELETE 端点的用户速率限制均为每 15 分钟 50 次请求。

\*\*请注意：\*\*如果用户在 [X](https://x.com/) 上执行静音操作，则每 15 分钟的请求上限为 200 次。

**账号设置**

要访问这些端点，您需要：

- 已获批准的[开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)。
- 使用位于某个[项目](/zh/resources/fundamentals/projects)内的[开发者应用](/zh/resources/fundamentals/developer-apps)提供的密钥和令牌进行认证。

在我们的[入门指南](/zh/x-api/getting-started/getting-access)中了解如何获取 X API v2 端点的访问权限。
