---
title= "帖子上限"
---

X API v2 设有帖子消费上限，用于限制每月可从特定端点检索的帖子数量。帖子上限适用于[项目](/zh/resources/fundamentals/projects)级别，这意味着在该项目中使用[开发者应用](/zh/resources/fundamentals/developer-apps)的密钥和令牌对下列端点发起的任何请求，都会计入该项目的帖子上限。

<div id="limited-v2-endpoints">
  ## 受限的 v2 端点
</div>

通过以下任一端点获取的帖子都会计入当月的帖子上限：

- [帖子查找](/zh/x-api/posts/lookup/introduction)
- [近期搜索](/zh/x-api/posts/counts/introduction)
- [全档案搜索](/zh/x-api/posts/counts/introduction)
- [过滤流](/zh/x-api/posts/filtered-stream)
- [用户帖子时间线](/zh/x-api/posts/timelines)
- [用户提及时间线](/zh/x-api/posts/timelines)
- [点赞查找 - 某用户点赞的帖子](/zh/x-api/posts/likes)
- [书签查找 - 某用户收藏的帖子](/zh/x-api/posts/bookmarks/introduction)
- [列表帖子查找](/zh/x-api/lists/list-tweets/introduction)
- [Spaces 查找](/zh/x-api/spaces/lookup/introduction)

<Note>
  **注意：** 如果同一条帖子在一天内由多个查询返回，它在帖子上限中仅计数一次；系统按日去重。
</Note>

<div id="post-cap-volumes">
  ## 帖子上限
</div>

帖子上限会根据你的访问层级而变化：

| 访问层级   | 每月帖子上限                |
|:---------------|:---------------------------------|
| **Free**      | 每月 1,500 条帖子           |
| **Basic**     | 每月 10,000 条帖子          |
| **Pro**       | 每月 1,000,000 条帖子       |
| **Enterprise**| 每月 5,000 万条以上         |

你可以在[开发者门户](/zh/resources/fundamentals/developer-portal)的[主控制台页面](https://developer.x.com/content/developer-twitter/en/portal/dashboard)查看你的每月帖子上限使用情况。在你的项目名称下方，会显示一个状态条，指示当月相对于帖子上限的使用情况，包括已拉取的帖子总数、已用比例以及用量重置日期。
