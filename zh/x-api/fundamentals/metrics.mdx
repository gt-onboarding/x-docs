<div id="overview">
  ## 概览
</div>

`metrics` 字段允许开发者访问帖子和媒体对象的公开与非公开互动指标。公开指标对任何拥有开发者账号的人可用，而非公开指标仅对自有/已授权账号（定义见下文）可用。可用指标包括每个请求中指定帖子获得的展示次数、转发、引用转发、点赞、回复、视频观看次数、视频观看分位（四分位）数据、URL 点击和个人资料链接点击等。若该帖子作为[广告](https://ads.x.com/)投放，还可选择查看在自然或推广场景下的指标拆分。

公开指标可通过 [App-only Token](/zh/resources/fundamentals/authentication#oauth-2-0) 认证请求。非公开指标仅可针对自有/已授权的帖子请求，这意味着开发者需要使用 [OAuth 2.0](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) 或 [OAuth 1.0a](/zh/resources/fundamentals/authentication#oauth-1-0a-2) 的用户上下文授权进行认证。

**非公开、自然和推广指标仅适用于过去 30 天内创建的帖子。**

<div id="terminology">
  ### 术语
</div>

- **已授权账户**：通过授予对其账户的访问权限而授权你的[X 开发者应用](/zh/resources/fundamentals/developer-apps)的 X 账户（任何[应用权限级别](/zh/resources/fundamentals/developer-apps#app-permissions)都可访问帖子指标）。
- **已关联账户**：与你的[X 开发者应用](/zh/resources/fundamentals/developer-apps)关联的 X 账户。
- **公开指标**：任何人在[X](https://x.com/home)上都可查看的汇总数据，例如点赞数和转发数。
- **非公开指标**：在[X](https://x.com/home)上不对公众展示的汇总数据，例如展示次数和视频观看四分位。需要[OAuth 2.0](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3)或[OAuth 1.0a](/zh/resources/fundamentals/authentication#oauth-1-0a-2)用户上下文认证。
- **自然指标**：归因于自然场景（以常规方式发布与查看）的公开和非公开指标集合。需要[OAuth 2.0](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3)或[OAuth 1.0a](/zh/resources/fundamentals/authentication#oauth-1-0a-2)用户上下文认证。
- **推广指标**：归因于推广场景（作为广告活动的一部分被发布或被查看）的公开和非公开指标集合。需要[OAuth 2.0](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3)或[OAuth 1.0a](/zh/resources/fundamentals/authentication#oauth-1-0a-2)用户上下文认证，且该帖子须在广告中被推广。

<div id="available-metrics">
  ## 可用指标
</div>

| 指标               | API 表示                                                                                                                                                                                                                                     | 说明                                                                                                                                                                                                                                                                                                                                                                                |
|:----------------------|:---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|:--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Impressions**      | `data.non_public_metrics.impression_count`, `data.organic_metrics.impression_count`, `data.promoted_metrics.impression_count`                                                                                                                           | 帖子被查看的次数（不按用户去重）。只要帖子的任意部分在屏幕上可见即计为一次查看。需要 OAuth 1.0a 用户上下文认证。                                                                                                               |
| **Retweets**         | `data.public_metrics.retweet_count`, `data.organic_metrics.retweet_count`, `data.promoted_metrics.retweet_count`                                                                                                                                       | 帖子被转推的次数。不包含引用推文（“带评论的转推”）。若要获得 X 客户端展示的“转推与评论”总数，请将 `retweet_count` 与 `quote_count` 相加。                                           |
| **Quote Tweets**     | `data.public_metrics.quote_count`                                                                                                                                                                                                                      | 帖子被带新评论（消息）转推的次数。付费场景下不存在引用推文，因此所有引用推文均为自然产生。                                                                                                                                           |
| **Likes**            | `data.public_metrics.like_count`, `data.organic_metrics.like_count`, `data.promoted_metrics.like_count`                                                                                                                                                 | 帖子被点赞的次数。为与 X 公开展示的计数保持一致，`public_metrics` 字段返回来自自然与付费场景的点赞总数。                                                                                                   |
| **Replies**          | `data.public_metrics.reply_count`, `data.organic_metrics.reply_count`, `data.promoted_metrics.reply_count`                                                                                                                                              | 帖子被回复的次数。`public_metrics` 字段返回来自自然与付费场景的回复总数。                                                                                                                                                   |
| **URL Link Clicks**  | `data.non_public_metrics.url_link_clicks`, `data.organic_metrics.url_link_clicks`, `data.promoted_metrics.url_link_clicks`                                                                                                                             | 用户点击帖子中的 URL 链接或 URL 预览卡片的次数。需要 OAuth 1.0a 用户上下文认证。                                                                                                                                                                  |
| **User Profile Clicks** | `data.non_public_metrics.user_profile_clicks`, `data.organic_metrics.user_profile_clicks`, `data.promoted_metrics.user_profile_clicks`                                                                                                             | 用户点击帖子中以下部分的次数：显示名称、用户名、头像。需要 OAuth 1.0a 用户上下文认证。                                                                                                                                               |
| **Video views**      | `includes.media.public_metrics.view_count`, `includes.media.organic_metrics.view_count`, `includes.media.promoted_metrics.view_count`                                                                                                                  | 帖子中所含视频被观看的次数。该数值为该视频在其出现的所有帖子中的观看次数总和。需要媒体扩展 `expansions=attachment.media_keys`。                                                                |
| **Video view quartiles** | `includes.media.non_public_metrics.playback_0_count`, `includes.media.non_public_metrics.playback_25_count`, `includes.media.non_public_metrics.playback_50_count`, `includes.media.non_public_metrics.playback_75_count`, `includes.media.non_public_metrics.playback_100_count` | 播放至视频各四分位区间的用户数量。需要 OAuth 1.0a 用户上下文认证以及媒体扩展 `expansions=attachment.media_keys`。                                                                                                 |

<div id="requesting-metrics">
  ## 请求度量指标
</div>

<div id="public-metrics">
  ### 公开指标
</div>

在以下请求中，我们将为帖子及其附带的视频获取公开指标，使用如下字段与扩展。请务必将 `$BEARER_TOKEN` 替换为你自己生成的 Bearer Token。

- `tweet.fields=public_metrics`
- `expansions=attachments.media_keys&media.fields=public_metrics`

<div id="sample-request">
  #### 示例请求
</div>

```bash
curl 'https://api.x.com/2/tweets?ids=1204084171334832128&tweet.fields=public_metrics&expansions=attachments.media_keys&media.fields=public_metrics' --header 'Authorization: Bearer $BEARER_TOKEN'
```

<div id="private-metrics-non-public-organic-metrics">
  ### 私有指标（非公开、自然流量指标）
</div>

以下请求会为帖子及其附带视频获取非公开指标，并包含有关自然流量指标的更多细节。由于这些字段为私有，且在 X 上不对公众可见，因此该请求需要使用 [OAuth 2.0](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) 或 [OAuth 1.0a](/zh/resources/fundamentals/authentication#oauth-1-0a-2) 的用户上下文认证。有关生成下方所需 OAuth 1.0a 签名，请参阅我们的[指南](/zh/resources/fundamentals/authentication#creating-a-signature)。

- `tweet.fields=non_public_metrics,organic_metrics`
- `expansions=attachments.media_keys&media.fields=non_public_metrics,organic_metrics`

<div id="sample-request">
  #### 示例请求
</div>

```bash
curl 'https://api.x.com/2/tweets/1204084171334832128?tweet.fields=non_public_metrics,organic_metrics&media.fields=non_public_metrics,organic_metrics&expansions=attachments.media_keys' --header 'authorization: OAuth oauth_consumer_key="CONSUMER_API_KEY", oauth_nonce="OAUTH_NONCE", oauth_signature="OAUTH_SIGNATURE", oauth_signature_method="HMAC-SHA1", oauth_timestamp="OAUTH_TIMESTAMP", oauth_token="ACCESS_TOKEN", oauth_version="1.0"'
```
