<div id="tweet">
  ### 推文
</div>

推文是 Twitter 上一切内容的基本构件。`Tweet` 对象包含一长串“根级”字段，例如 `id`、`text` 和 `created_at`。`Tweet` 对象也是多个子对象的“父”对象，包括 `user`、`media`、`poll` 和 `place`。在请求 `Tweet` 对象的这些根级字段时，请使用字段参数 `tweet.fields`。

可以在用户资源中找到并展开 `Tweet` 对象。与所请求推文相关的其他推文也可以在推文资源中找到并展开。要仅获取带有默认字段的该对象，可在用户资源中使用 `?expansions=pinned_tweet_id`，或在推文资源中使用 `?expansions=referenced_tweets.id` 进行展开。请求额外字段以补全该对象时，请将该展开与字段参数 `tweet.fields` 结合使用。

| 字段值                 | 类型   | 描述                                                                                                                                 | 使用方式                                                              |
|:-----------------------|:--------|:---------------------------------------------------------------------------------------------------------------------------------------|:----------------------------------------------------------------------|
| **id（默认）**      | string | 所请求 Tweet 的唯一标识符。                                                                                                         | 用于以编程方式检索特定 Tweet。                                        |
| **text（默认）**    | string | Tweet 的实际 UTF-8 文本。有关有效字符的详细信息，请参阅 [twitter-text](https://github.com/twitter/twitter-text/)。                 | 关键词提取与情感分析/分类。                                           |
| **edit\_history\_tweet\_ids（默认）** | object | 标示某条 Tweet 所有版本的唯一标识符。未被编辑的 Tweet 仅有一个 ID；有编辑历史的 Tweet 会有多个 ID。 | 使用此信息查找某条 Tweet 的编辑历史。                                 |
| **attachments**       | object | 指定此 Tweet 中存在的附件类型（如果有）。                                                                                           | 理解为所请求扩展返回的对象。                                          |
| **author\_id**         | string | 发布此 Tweet 的用户唯一标识符。                                                                                                     | 充填 User 对象，分享数据集以供同行评审。                              |
| **context\_annotations** | array | 包含该 Tweet 的上下文标注。                                                                                                         | 实体识别/抽取、主题分析。                                             |
| **conversation\_id**   | string | 该会话原始 Tweet 的 Tweet ID（包括直接回复以及对回复的回复）。                                                                      | 用于从一条 Tweet 重建整个会话。                                       |
| **created\_at**        | date (ISO 8601) | Tweet 的创建时间。                                                                                                            | 有助于时间序列分析并了解 Tweet 的创建时点。                           |
| **edit\_controls**     | object | 指示该 Tweet 仍可编辑的剩余时间及剩余可编辑次数。                                                                                   | 用于判断某条 Tweet 是否仍可编辑。                                     |
| **entities**          | object | 从 Tweet 文本中解析出的实体。请参阅 Twitter Objects 中的 entities。                                                                | 提供关于话题标签、URL、提及等的附加信息。                             |
| **in\_reply\_to\_user\_id** | string | 如果该 Tweet 是回复，此字段包含原始 Tweet 的作者 ID。                                                                              | 判断某条 Tweet 是否为对另一条 Tweet 的回复。                          |
| **lang**              | string | 若由 Twitter 检测到，则为该 Tweet 的语言。                                                                                          | 按语言对 Tweets 进行分类。                                            |
| **non\_public\_metrics** | object | 请求时该 Tweet 的非公开互动指标。需要用户上下文认证。                                                                               | 确定该 Tweet 产生的总展示次数。                                       |
| **organic\_metrics**   | object | 在自然情境中跟踪的、请求时该 Tweet 的互动指标。需要用户上下文认证。                                                                 | 衡量该 Tweet 的自然互动。                                             |
| **possibly\_sensitive** | boolean | 指示内容是否可能被识别为敏感。                                                                                                     | 研究特定类型内容的传播。                                              |
| **promoted\_metrics**  | object | 在推广情境中跟踪的、请求时该 Tweet 的互动指标。需要用户上下文认证。                                                                 | 衡量该 Tweet 在推广时的互动。                                         |
| **public\_metrics**    | object | 请求时该 Tweet 的公开互动指标。                                                                                                     | 衡量 Tweet 的互动情况。                                               |
| **referenced\_tweets** | array  | 此 Tweet 所引用的 Tweets 列表，例如转发、引用或回复。                                                                                | 理解转发等的对话属性。                                                |
| **reply\_settings**    | string | 显示谁可以回复该 Tweet。选项为“everyone”“mentioned\_users”“followers”。                                                             | 确定该 Tweet 的会话回复设置。                                         |
| **withheld**          | object | 包含[受限内容](https://help.x.com/en/rules-and-policies/tweet-withheld-by-country)的屏蔽详情。                                      |                                                                     |
| **media\_metadata**          | array | 包含该 Tweet 的媒体附件的元数据。     | 获取该 Tweet 媒体附件的 `alt_text` 等附加元数据。 |

**检索 Tweet 对象**

**示例请求**

在以下请求中，我们在 [Tweets lookup](/zh/resources/fundamentals/rate-limits) 端点请求 Tweet 的字段。请务必将 `$BEARER_TOKEN` 替换为你自己生成的 [Bearer Token](/zh/resources/fundamentals/authentication)。

```bash
curl --request GET 'https://api.x.com/2/tweets?ids=1212092628029698048&tweet.fields=attachments,author_id,context_annotations,created_at,entities,geo,id,in_reply_to_user_id,lang,possibly_sensitive,public_metrics,referenced_tweets,text,withheld&expansions=referenced_tweets.id' --header 'Authorization: Bearer $BEARER_TOKEN'
```

<div id="user">
  ### 用户
</div>

`user` 对象包含描述被引用用户的 Twitter 用户账户元数据。`user` 对象是 [users lookup](/zh/x-api/users/lookup/introduction) 端点返回的主要对象。在该端点请求更多用户字段时，只需使用字段参数 `user.fields`。

`user` 对象也可作为子对象出现在 Tweet 对象中，并可被扩展。可通过 `?expansions=author_id` 或 `?expansions=in_reply_to_user_id` 进行扩展，以获取仅包含默认字段的精简对象。若需请求更多字段以补全该对象，请将扩展与字段参数配合使用：`user.fields`。

| :--- | :--- | :--- | :--- |
| id（默认） | 字符串 | 此用户的唯一标识符。<br /><br />`"id": "2244994945"` | 用于以编程方式检索特定 Twitter 用户的信息。 |
| name（默认） | 字符串 | 用户在其个人资料中设置的名称。不一定是个人姓名。通常上限为 50 个字符，但可能会调整。<br /><br />`"name": "Twitter Dev"` |     |
| username（默认） | 字符串 | 该用户用于标识自身的 Twitter 屏幕名、账号或别名。用户名具有唯一性，但可能会更改。通常最长为 15 个字符，但一些历史账号可能更长。<br /><br />`"username": "TwitterDev"` |     |

**检索用户对象**

**示例请求**

在以下请求中，我们通过 [users lookup](/zh/x-api/users/lookup/introduction) 端点请求该用户的字段。请务必将 `$BEARER_TOKEN` 替换为你自己生成的 [Bearer Token](/zh/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)。
 

```{
  curl --request GET 'https://api.x.com/2/users?
  ids=2244994945&user.fields=created_at,description,entities,id,location,name,pinned_tweet_id,profile_image_url,protected,url,username,verified,withheld&expansions=pinned_tweet_id'
  --header 'Authorization: Bearer $BEARER_TOKEN'
  }
```

<div id="space">
  ### Space
</div>

Spaces 通过实时音频会话实现表达与互动。Space 数据字典包含有关某个 Space 的相关元数据；所有详细信息都会实时更新。

用户对象可在用户资源中查找并展开。通过在 expansions 查询参数中添加至少一个 host\_ids、creator\_id、speaker\_ids 或 mentioned\_user\_ids，这些对象即可进行扩展。

与推文不同，Space 为临时内容，在结束后或被创建者取消时将不可用。当你的应用处理 Space 数据时，你有责任返回最新信息，并且必须移除平台上已不再可用的数据。[Spaces 查询端点](/zh/x-api/spaces/lookup/introduction)可帮助你确保尊重用户的期望与意图。

| 字段值                 | 类型            | 说明                                                                                                         | 用途                                                     |
| :-------------------- | :--------------- | :----------------------------------------------------------------------------------------------------------- | :------------------------------------------------------- |
| id（默认）            | 字符串          | 所请求 Space 的唯一标识符。<br />`"id": "1zqKVXPQhvZJB"`                                                      | 在响应中唯一标识 Space。                                 |

| host\_ids             | 数组            | Space 主持人的唯一标识符。<br />`"host_ids": ["2244994945", "6253282"]`                                       | 展开用户对象，了解互动情况。                            |

<div id="direct-message-events">
  ### 私信事件
</div>

私信（DM）会话由事件构成。X API v2 当前支持三种事件类型：MessageCreate、ParticipantsJoin 和 ParticipantsLeave。

DM 事件对象由 [私信查询](/zh/x-api/direct-messages/lookup/introduction) 端点返回；当通过 [管理私信](/zh/x-api/direct-messages/lookup/introduction) 端点成功创建私信时，会生成一个 MessageCreate 事件。

请求 DM 事件时，默认包含三个事件对象属性（字段）：id、event\_type 和 text。若要获取更多事件字段，请使用 [fields](/zh/x-api/fundamentals/fields) 参数 dm\_event.fields 进行选择。其他可用字段包括：dm\_conversation\_id、created\_at、sender\_id、attachments、participant\_ids 和 referenced\_tweets。

其中有若干字段提供与该私信事件相关的其他 X 对象的 ID：

- sender\_id - 发送消息或将参与者邀请至群组会话的账户 ID
- participant\_ids - 账户 ID 的数组。对于 ParticipantsJoin 和 ParticipantsLeave 事件，此数组仅包含创建该事件的账户的一个 ID
- attachments - 提供发送方已上传至 X 的内容所对应的媒体 ID
- referenced\_tweets - 如果在 text 字段中检测到推文 URL，响应中会包含该推文的 ID

可使用 sender\_id、participant\_ids、referenced\_tweets.id 和 attachments.media\_keys 的 [expansions](/zh/x-api/fundamentals/expansions) 来展开这些 X 对象 ID。

|     |     |     |     |
| :--- | :--- | :--- | :--- |

| event\_type (default) | string | 描述事件类型。目前支持三种类型： <br /><br />\* MessageCreate<br />    <br />\* ParticipantsJoin<br />    <br />\* ParticipantsLeave<br />    <br /><br />"event\_type": "MessageCreate" | 在检索会话历史时，用于了解消息创建时间；对于群组会话，用于了解参与者加入和离开的时间。所有 GET 方法均支持使用 event\_type= 查询参数过滤特定事件类型。 . |

<div id="how-to-use-fields-and-expansions">
  ## 如何使用 fields 和 expansions
</div>

默认情况下，当你在请求中未使用 [fields](/zh/x-api/fundamentals/fields) 或 [expansions](/zh/x-api/fundamentals/expansions) 参数时，X API v2 的数据对象只会包含少量默认字段。本指南将演示如何在请求中使用 `fields` 和 `expansions` 查询参数，以在响应中获取更多对象与字段。

在本指南中，我们将基于下方的 Tweet 截图请求若干字段。
 

![此图为一条由 @X 发布的 Tweet 截图。你可以看到 Tweet 文本、用户名、发布时间与时间、来源，以及公开指标。截图中还包含一个视频。](https://cdn.cms-twdigitalassets.com/content/dam/developer-twitter/ScreenShotDataDictionaryGuide.png.twimg.1920.png)

如你在截图中所见，与该 Tweet 相关的可见信息包括 Tweet 作者、Tweet 指标、创建时间戳、视频以及视频观看次数。还有一些数据在截图中不可见，但仍可通过请求获取。 

当你向 API 发起请求时，默认响应较为精简，仅包含默认的 Tweet 字段（id 和 text）。你还只会收到所使用端点返回的主对象，而不会包含任何可能与主对象相关的关联数据对象。

这种精简性与 fields 和 expansions 参数相结合，使你能够根据你的使用场景仅请求所需字段。
