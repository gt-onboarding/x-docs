---
title: "字段"
---

X API v2 的各个端点提供一组称为“字段”的参数，允许你仅从端点响应中的每个对象中选择所需数据。

默认情况下，Post 对象仅返回 `id` 和 `text` 字段；对于 2022 年 9 月 29 日之后创建的帖子，还会返回 `edit_history_tweet_ids` 字段。如果你需要帖子的创建时间或公开指标，则需使用 `tweet.fields` 参数来请求。这种方式可根据你的用例仅请求所需字段，提升定制化程度。例如，你可以在请求中包含以下查询字符串：

`?tweet.fields=created_at,public_metrics`

每个对象都有其专用参数，用于请求与该对象关联的字段。当前可用的字段参数如下：

- [Post](/zh/x-api/fundamentals/data-dictionary#tweet) → `tweet.fields`
- [User](/zh/x-api/fundamentals/data-dictionary#user) → `user.fields`
- [Media](/zh/x-api/fundamentals/data-dictionary#media) → `media.fields`
- [Poll](/zh/x-api/fundamentals/data-dictionary#poll) → `poll.fields`
- [Place](/zh/x-api/fundamentals/data-dictionary#place) → `place.fields`

当使用主要返回某一特定对象的端点时，只需使用相应的字段参数，并将所需字段以逗号分隔的列表作为该参数的值进行指定，即可在响应中获取这些字段。

<div id="example">
  ### 示例
</div>

如果你正在使用 [GET /tweets/search/recent](/zh/x-api/posts/recent-search) 端点，你在响应中主要会收到 [Tweet 对象](/zh/x-api/fundamentals/data-dictionary#tweet)。在未指定任何字段参数的情况下，你只会收到默认字段 `id` 和 `text`。如果你希望在响应中获取返回帖子（Posts）的公共指标，需要在请求中包含 `tweet.fields` 参数，并将其值设为 `public_metrics`。

该请求如下所示。如需直接使用，请先将 `$BEARER_TOKEN` 替换为你的 [Bearer Token](/zh/resources/fundamentals/authentication)，再通过命令行工具发送。

```bash
curl --request GET \
  --url 'https://api.x.com/2/tweets/search/recent?query=from%3Atwitterdev&tweet.fields=public_metrics' \
  --header 'Authorization: Bearer $BEARER_TOKEN'
```

如果你在终端中发送此请求，则返回的每条帖子都会包含以下字段：

```
{
   "data": {
       "id": "1263150595717730305",
       "public_metrics": {
           "retweet_count": 12,
           "reply_count": 14,
           "like_count": 49,
           "quote_count": 7
       },
       "text": "Do you 👀our new Tweet settings?\n\nWe want to know how and why you’d use a feature like this in the API. Get the details and let us know what you think👇\nhttps://t.co/RtMhhfAcIB https://t.co/8wxeZ9fJER"
   }
}
```

如果你想检索与端点返回的主对象关联的次级对象的一组字段，需要额外包含 expansions 参数。比如，仍使用上述 GET search/tweets/recent 端点，若要获取作者的个人资料简介，需要在请求中传递 `expansions=author_id` 和 `user.fields=description`。下面是示例。若要尝试此请求，请在粘贴到命令行工具之前，将 `$BEARER_TOKEN` 替换为你的 [Bearer Token](/zh/resources/fundamentals/authentication)。

```bash
  curl --request GET \
  --url 'https://api.x.com/2/tweets/search/recent?query=from%3Atwitterdev&tweet.fields=public_metrics&expansions=author_id&user.fields=description' \
  --header 'Authorization: Bearer $BEARER_TOKEN'
```

如果在请求中指定了这些参数，则每条返回的帖子都会包含下列字段，且相关的 [user 对象](/zh/x-api/fundamentals/data-dictionary#user) 的默认字段和指定字段会返回在 `includes` 中。可通过匹配 `tweet.author_id` 与 `users.id` 将 user 对象映射回相应帖子。

```
{
  "data": [
    {
      "id": "1263150595717730305",
      "author_id": "2244994945",
      "text": "Do you 👀our new Tweet settings?\n\nWe want to know how and why you’d use a feature like this in the API. Get the details and let us know what you think👇\nhttps://t.co/RtMhhfAcIB https://t.co/8wxeZ9fJER",
      "public_metrics": {
        "retweet_count": 12,
        "reply_count": 13,
        "like_count": 51,
        "quote_count": 7
      }
    }
  ],
  "includes": {
    "users": [
      {
        "id": "2244994945",
        "username": "TwitterDev",
        "description": "The voice of the #TwitterDev team and your official source for updates, news, and events, related to the #TwitterAPI.",
        "name": "Twitter Dev"
      }
    ]
  }
}
```

请注意，无法请求特定子字段（例如 `public_metrics.retweet_count`）。当指定顶层字段（`public_metrics`）时，会返回所有子字段。我们在各端点的 API 参考页面的参数表中列出了可请求的所有字段。 

完整字段列表见[对象模型](/zh/x-api/fundamentals/data-dictionary#tweet)。要为非该端点主要资源的对象扩展并请求字段，请将[扩展](/zh/x-api/fundamentals/expansions)参数与 fields 搭配使用。
