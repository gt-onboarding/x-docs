X 致力于支持使用 X API 构建产品的开发者社区。作为这一承诺的一部分，我们旨在让我们的 API 对开发者开放且公平、保障 X 上用户的安全，并使整个 X 平台受益。任何离线存储 X 内容的开发者，都必须确保数据能够反映用户意图以及 X 上内容的当前状态。例如，当有人在 X 上删除一条帖子或其账户、将帖子设为受保护，或编辑一条帖子时，X 和我们的开发者都必须尊重该人的期望与意图，这一点至关重要。

合规事件的实时流为开发者提供工具，使其能够依照[X 开发者协议与政策](https://developer.x.com/en/developer-terms/policy)维护 X 数据的合规性。

共有两个合规事件流：一个用于帖子合规事件，另一个用于用户合规事件。这些流面向企业级访问开放，旨在帮助摄取海量数据的合作伙伴“监听”合规事件，例如帖子编辑事件。

这些流提供以下事件：

**帖子合规流：**

- delete - 表示该帖子已被删除。
- tweet\_edit - 表示一条帖子已被编辑，并提供更新后帖子的 ID。
- withheld - 表示该帖子已在一个或多个国家/地区被限制展示。
- drop - 表示该帖子应从公开视图中移除。
- undrop - 表示该帖子可以再次展示并视为公开内容。

**用户合规流：**

- **user\_delete** - 表示该用户账户已被删除
- **user\_undelete** - 表示该用户账户已被恢复
- **user\_protect** - 表示该用户账户变为私密
- **user\_unprotect** - 表示该用户账户变为公开
- **user\_withheld** - 表示该用户账户已在一个或多个国家/地区被限制展示。
- **user\_suspend** - 表示该用户账户已被暂停
- **user\_unsuspend** - 表示该用户账户已被解除暂停
- **user\_profile\_modification** - 表示该用户资料已更新，包括简介、名称、位置和 URL 的更新。

**账户设置**

要访问这些端点，您需要：

- 一个已获批准的[开发者账户](https://developer.x.com/en/portal/petition/essential/basic-info)。
- 使用位于某个[项目](/zh/resources/fundamentals/projects)中的[开发者应用](/zh/resources/fundamentals/developer-apps)的密钥和令牌进行身份验证。

在我们的[入门指南](/zh/x-api/getting-started/getting-access)中了解有关获取 X API v2 端点访问权限的更多信息。
