X 致力于支持使用 X API 构建产品的开发者社区。作为这一承诺的一部分，我们的目标是让 API 对开发者开放且公平、对 X 上的用户安全，并整体上有益于 X 平台。对任何离线存储 X 内容的开发者而言，确保数据能够反映用户意图以及 X 上内容的当前状态至关重要。例如，当 X 上的某人删除帖子或其账户、将其帖子设为受保护、或从其帖子中移除地理位置信息时，对 X 和我们的开发者来说，尊重该人的期望与意图至关重要。批量合规端点为开发者提供了一个便捷工具，帮助使 X 数据遵循[X 开发者协议与政策](https://developer.x.com/en/developer-terms/policy)。

这些批量合规端点允许你上传包含帖子或用户 ID 的大型数据集，以检索其合规状态，从而确定需要采取哪些操作以使你的数据集达到合规。请注意，批量合规端点的使用仅限于上述用例，任何其他用途均被禁止，且可能导致执法措施。

通常，使用此端点涉及 4 个步骤：

1. 创建合规作业
   你可以指定作业类型（取值为 tweets 或 users），以表明你要上传的数据集包含帖子 ID 还是用户 ID。任一时间，你针对每种作业类型最多可有一个并发作业。
2. 将数据集上传到 upload\_url
   接下来，将你的数据集作为纯文本文件上传到提供的 upload\_url，文件的每一行包含一个帖子 ID 或用户 ID。upload\_url 在 15 分钟后过期。
3. （可选）检查作业状态
   你可以检查合规作业的状态，以了解其是 created、in\_progress、failed 还是 complete。
4. 下载结果
   当作业完成后，你可以使用 download\_url 下载结果。download\_url 会在作业创建后一周过期。

   结果将包含一组 JSON 对象（每行一个对象）。每个对象包含一个帖子 ID、该帖子的创建日期（便于按日期定位帖子）、所需采取的操作、采取合规操作的原因，以及用户被暂停的日期。

你将在结果中收到以下合规事件类型：

- deleted - 表示帖子或用户账户已被删除
- deactivated - 表示帖子或用户账户已被停用
- scrub\_geo - 表示与帖子或用户关联的地理信息被移除
- protected - 表示发布该帖子的账户变为私密
- suspended - 表示发布该帖子的账户被暂停

**账户设置**

要访问这些端点，你需要：

- 一个已获批准的[开发者账户](https://developer.x.com/en/portal/petition/essential/basic-info)。
- 使用位于[项目](/zh/resources/fundamentals/projects)中的[开发者应用](/zh/resources/fundamentals/developer-apps)所提供的密钥与令牌进行认证。

在我们的[入门指南](/zh/x-api/getting-started/getting-access)中了解如何获得 X API v2 端点的访问权限。
