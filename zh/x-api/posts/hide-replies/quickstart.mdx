<div id="getting-started-with-the-hide-replies-endpoint">
  ## 隐藏回复端点快速上手
</div>

本快速入门指南将帮助你使用 [Postman](/zh/tutorials/postman-getting-started) 向隐藏回复端点发出首次请求。

如果你想查看不同语言的代码示例，请访问[隐藏回复 API 参考页面](/zh/x-api/posts/hide-replies#put-2-tweets-id-hidden)。

<Note>
  ### 前提条件

  完成本指南，你需要一组用于请求身份验证的[密钥和令牌](/zh/resources/fundamentals/authentication)。你可以按照以下步骤生成这些密钥和令牌：

  - [注册开发者账户](https://developer.x.com/en/apply-for-access)并通过审核。
  - 在开发者门户中创建[项目](/zh/resources/fundamentals/projects)并关联[开发者应用](/zh/resources/fundamentals/developer-apps)。
  - 前往你的 App 的“Keys and tokens”页面生成所需凭据。请确保将所有凭据妥善保管。
</Note>

<div id="steps-to-build-a-put-tweetsidhidden-request">
  ### 构建 PUT /tweets/:id/hidden 请求的步骤
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### 第一步：从工具或库入手
</div>

有多种工具、代码示例和库可用于向此端点发起请求，但为简化流程，我们将在此使用 Postman。

要将 X API v2 的 Postman 集合导入到你的环境，请点击以下按钮：

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  将 X API v2 添加到 Postman
</Button>

在你将 X API v2 集合导入到 Postman 后，前往 hide replies 端点。

<div id="step-two-authenticate-your-request">
  #### 第二步：为请求进行身份验证
</div>

要正确向 X API 发起请求，你需要先验证自己是否拥有相应权限。对于此端点，你必须使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 或 [OAuth 2.0（含 PKCE）授权码模式](zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)对请求进行身份验证。

在本示例中，我们将使用 OAuth 1.0a 用户上下文。

你需要将密钥和令牌——具体包括 API Key、API Secret Key、OAuth 1.0a 用户 Access Token，以及 OAuth 1.0a 用户 Access Token Secret——添加到 Postman 中。你可以在 Postman 右上角选择名为“X API v2”的环境，并将这些密钥和令牌填入“initial value”和“current value”字段（点击环境下拉菜单旁的眼睛图标）。

如果配置正确，这些变量会自动填充到该请求的 Authorization 选项卡中。
 

<div id="step-three-find-a-post-id-to-hide">
  #### 步骤三：找到要隐藏的帖子 ID
</div>

“隐藏回复”端点可以代表已授权用户隐藏或取消隐藏回复。由于本示例使用与你的用户资料关联的访问令牌，你可以隐藏参与你发起对话的用户所发布的回复。同样，如果你使用的是另一位已授权你应用的用户的访问令牌，你就可以管理该账户发起的任何对话中的回复。

请朋友在一条帖子下回复（提前告知你在测试隐藏回复），或使用测试账号回复你的任意帖子。点击该条回复，然后复制其 URL 中的数字部分。那就是我们将要隐藏的帖子 ID。

在本例中，我们将查看以下帖子，其 ID 为 `1232720193182412800`：

`https://x.com/TwitterDev/status/1232720193182412800`

<div id="step-four-hide-the-post">
  #### 第四步：隐藏帖子
</div>

在 Postman 中，打开 Hide replies 文件夹并选择 Hide a reply。在 Params 选项卡中，将帖子 ID 粘贴到 id 字段旁（无需替换 URL 中的 :id）。点击“Send”，你将看到成功的响应。

`{"hidden":true}`

<div id="step-five-unhide-the-post">
  #### 第五步：取消隐藏帖子
</div>

被隐藏的帖子会移至 X 应用中的单独标签页。要在 Postman 中取消隐藏帖子，打开 Hide replies 文件夹并选择 Unhide a reply。在 Params 选项卡中，将上一步使用的同一帖子 ID 粘贴到 id 字段中。点击“Send”后，你会看到成功的响应。

`{"hidden":false}`

hidden 字段表示帖子的隐藏状态。为 true 表示帖子已隐藏；为 false 表示帖子未隐藏。
