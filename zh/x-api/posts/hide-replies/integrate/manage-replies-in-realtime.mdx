<div id="manage-replies-in-realtime">
  ### 实时管理回复
</div>

借助 hide replies 端点，您可以构建一套工作流，帮助用户隐藏那些极可能不相关的回复。

实用的应用通常会结合多种技术，为用户提供更大价值。本文展示如何使用 [Perspective API](https://www.perspectiveapi.com/) 来隐藏回复。该 API 由 [Jigsaw](https://jigsaw.google.com/)（Google 的一个部门）训练，用于检测有害评论。应用逻辑如下：

1.  请求用户授权读取其帖子，并隐藏或取消隐藏其回复。
2.  使用 [Account Activity API](https://developer.x.com/en/docs/x-api/v1/accounts-and-users/subscribe-account-activity/overview) 检测新收到的回复。
3.  请求 Perspective API 返回一个“分数”（0 到 1 之间的数值），表示其算法对某条评论与其过去识别的有害评论相似的置信度。（Perspective 不会存储发送到该服务的文本。）
4.  如果算法分数非常高，则调用 hide replies。

<div id="strive-for-transparency">
  #### 力求透明
</div>

由于 Perspective 并未以实际的帖子数据进行训练，某些语言细微差别可能会导致应用隐藏用户希望保持可见的回复。无论你在设计应用时采用何种技术或方法，都应尽最大努力确保用户了解你的应用隐藏了哪些内容，并且可以进行调整。

* 最佳做法是始终信任用户，并赋予他们对自身决策的完全控制权。这意味着你的用户体验应提供控件，以便撤销你的应用代替用户所执行的任何操作。
* 在使用人工智能时，你的应用应采用非常高的置信阈值来检测并隐藏帖子。
* 并非所有人使用相同的用语，你的应用应在设计上避免任何偏见。留意被重新赋权/自我认同的用语以及俚语，它们可能导致误报。
* 如果你正在训练人工智能，考虑采用更贴近 X 上常用语言的模型。
