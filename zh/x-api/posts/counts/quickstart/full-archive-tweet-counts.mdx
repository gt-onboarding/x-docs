<div id="getting-started-with-the-full-archive-post-counts-endpoint">
  ### 开始使用全档案 Post counts 端点
</div>

本快速入门指南将帮助你使用 Postman（一款用于发起 HTTP 请求的图形化工具）向全档案 Post counts 端点发出你的第一个请求。

如果你希望查看不同编程语言的示例代码，请访问我们的 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 代码库。

<Note>
  ### 先决条件

  全档案 Post counts 端点目前仅向拥有 Pro 或 Enterprise 访问权限的用户开放。要使用此端点，你必须申请 Pro 或 [Enterprise 访问权限](https://docs.google.com/forms/d/e/1FAIpQLScO3bczKWO2jFHyVZJUSEGfdyfFaqt2MvmOfl_aJp0KxMqtDA/viewform)。

  除获得访问批准外，你还需要一组用于请求认证的 [keys and tokens](/zh/resources/fundamentals/authentication)。你可以按照以下步骤生成这些 keys 和 tokens：

  - 在开发者门户中，前往具有 Enterprise 访问权限的 [Project](/zh/resources/fundamentals/projects)，并确保该 Project 下关联了一个 [developer App](/zh/resources/fundamentals/developer-apps)。

  - 前往你的 App 的“Keys and tokens”页面以生成所需的凭据。请务必将所有凭据妥善保管。
</Note>

<div id="steps-to-build-a-full-archive-post-counts-request">
  #### 构建全档案帖子计数请求的步骤
</div>

**步骤一：选择一个工具或库**

有多种工具、代码示例和库可用于向该端点发起请求，为简化流程，这里我们使用 Postman。

要将 X API v2 的 Postman 集合加载到你的环境中，请点击以下按钮：

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  将 X API v2 添加到 Postman
</Button>

在将 X API v2 集合加载到 Postman 后，前往 Post counts > Full-archive Post counts 请求。

**步骤二：为请求进行身份验证**

要正确向 X API 发起请求，你需要确认自己具备访问权限。对于此端点，你必须使用 [OAuth 2.0 App-Only](/zh/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) 方法对请求进行身份验证。

你需要将密钥和令牌添加到 Postman，尤其是 [App Access Token](/zh/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)（亦称 App-only Bearer Token）。可在 Postman 右上角选择名为“X API v2”的环境，并将密钥与令牌填入“initial value”和“current value”字段（点击环境下拉旁的眼睛图标）完成配置。

如果配置正确，该变量会自动注入到请求的 authorization 选项卡中。
 

**步骤三：创建查询**

每个全档案帖子计数请求都需要一个 [query](/zh/x-api/posts/counts/integrate/build-a-query)。本示例使用一个查询来匹配由 @XDevelopers 账号发布的帖子。为此，我们使用 from 运算符并将其设为 XDevelopers（不区分大小写）：

`from:XDevelopers`

在 Postman 中，前往 “Params” 选项卡，将该查询字符串，或以逗号分隔的一串帖子 ID，输入到 `ids` 参数的 “Value” 列中。
 

| Key | Value | Description |
| :--- | :--- | :--- |
| `query` | `from:XDevelopers` | 提交给全档案帖子计数端点的查询 |

步骤四（可选）：指定粒度与时间范围

如果在步骤三后直接点击 “Send”，将返回默认的全档案帖子计数：过去 30 天内按小时聚合。若要按天返回结果，需要添加 granularity 参数并将其值设为 day。若要获取 30 天之前的帖子计数，需要同时指定 start\_time 和 end\_time 参数及其目标值。 

在 Postman 中，前往 “Params” 选项卡，在 “Query Params” 表中添加以下键值对：

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Description** |
| granularity | day | 帖子计数结果的粒度。可选值：day、hour、minute |
| start\_time | 2021-05-01T00:00:00Z | 返回帖子的最早 UTC 时间戳 |
| end\_time | 2021-06-01T00:00:00Z | 返回帖子的最晚 UTC 时间戳。 |

你现在应能在 “Send” 按钮旁看到以下 URL：

`https://api.x.com/2/tweets/counts/all?query=from%3AXDevelopers&start_time=2021-05-01T00:00:00Z&end_time=2021-06-01T00:00:00Z&granularity=day`

**第六步：分页获取结果**

如果响应中的 meta 对象包含 next\_token，你可以将其值传递给 next\_token 查询参数。

| Key | Value | Description |
| :--- | :--- | :--- |
| next\_token | 将上一次请求的 meta 对象中取得的 next\_token 填写在此处。 | 如果最新的请求未返回全部剩余结果，你会在 meta 对象中收到一个 next\_token。取出该字段的值，并在下一次请求中将其作为 next\_token 参数的值，其他请求参数保持不变。 |

设置好正确的 next\_token 值后，点击“Send”按钮即可获取下一页结果。
