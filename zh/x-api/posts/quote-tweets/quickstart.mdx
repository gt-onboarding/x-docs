<div id="getting-started-with-the-quote-tweets-lookup-endpoint">
  ## 开始使用“引用推文”查询端点
</div>

本快速入门指南将帮助您通过[Postman](/zh/tutorials/postman-getting-started)，使用一组指定字段向“引用推文”查询端点发出您的首个请求。

如需查看不同编程语言的示例代码，请访问我们的[X API v2 示例代码](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 代码库。 

<div id="prerequisites">
  #### 先决条件
</div>

要完成本指南，您需要一组用于对请求进行身份验证的[密钥和令牌](/zh/resources/fundamentals/authentication)。您可以按照以下步骤生成这些密钥和令牌：

- [注册开发者账号](https://developer.x.com/en/apply-for-access)并通过审批。
- 在开发者门户中创建一个[项目](/zh/resources/fundamentals/projects)以及关联的[开发者应用](/zh/resources/fundamentals/developer-apps)。
- 前往应用的“Keys and tokens”页面生成所需的凭据。请务必将所有凭据安全保存。

**构建引用推文查询请求的步骤**

**第一步：选择工具或库**

有多种工具、代码示例和库可用于向此端点发起请求，我们将在此使用 Postman 以简化流程。

要将 X API v2 的 Postman 集合加载到您的环境，请点击以下按钮：

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  将 X API v2 添加到 Postman
</Button>

加载 X API v2 集合后，在 Postman 中导航到 timeline 文件夹，找到“Quote Tweets by ID”请求。

**第二步：为请求进行身份验证**

要正确调用 X API，您需要验证访问权限。对该端点而言，必须使用以下任一方式对请求进行认证：[OAuth 2.0 App-Only](/zh/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)、[OAuth 2.0 Authorization Code with PKCE](zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 或 [OAuth 1.0a User Context](/zh/resources/fundamentals/authentication)。

为简化说明，我们将在此请求中使用 OAuth 2.0 App-Only；但若要请求私有[指标](/zh/x-api/fundamentals/metrics)或帖子，则需要使用上述其他身份验证方式之一。

您需要将密钥和令牌，尤其是 [App Access Token](/zh/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)（亦称 App-only Bearer Token），添加到 Postman。在 Postman 右上角选择名为“X API v2”的环境，并将您的密钥和令牌填入“initial value”和“current value”字段（点击环境下拉菜单旁的眼睛图标）。

如果设置正确，请求的 authorization 选项卡会自动引用该变量。

**第三步：确定并指定要检索其引用推文的帖子**

您必须在请求中通过传入帖子 ID 指定要检索其引用推文的帖子。

本示例中使用的帖子 ID 为 1409931481552543749。

在 Postman 中，进入“Params”选项卡，将该帖子 ID 填入 id 参数的“Value”列。

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 1409931481552543749 |

**第四步：确定并指定要检索的字段**

如果在完成第三步后点击“Send”按钮，响应中将返回默认的[Post object](/zh/x-api/fundamentals/data-dictionary#tweet)字段：id、text 和 edit\_history\_tweet\_ids。

若需其他字段，需在请求中通过 [field](/zh/x-api/fundamentals/fields) 和/或 [expansion](/zh/x-api/fundamentals/expansions) 参数进行指定。

在本练习中，我们将从不同对象请求三组额外字段：

1. 主对象中的额外 tweet.created\_at 字段。
2. 返回帖子之作者的用户对象默认字段：id、name 和 username
3. 关联用户对象中的额外 user.created\_at 字段。

在 Postman 中，进入“Params”选项卡，在“Query Params”表中添加以下 key:value 对：

|     |     |     |     |
| :--- | :--- | :--- | :--- |
| **键** | **值** | **返回字段** |
| `tweet.fields` | `created_at` | `tweets.created_at` |
| `expansions` | `author_id` | `includes.users.id`, `includes.users.name`, `includes.users.username` |
| `user.fields` | `created_at` | `includes.users.created_at` |     |

现在你应当在“Send”按钮旁看到以下 URL：

`https://api.x.com/2/tweets/:id/quote_tweets?expansions=author_id&tweet.fields=created_at&user.fields=created_at`

**请注意：**

在 Postman 中，URL 字段中的路径参数 :id **不会**自动更新为你在 `id` 参数字段中输入的值，因此上述 URL 保留了 `:id` 而非 1409931481552543749。

<div id="step-six-paginate-through-your-results">
  #### 第六步：分页获取结果
</div>

在上一个响应的底部，您会发现一个 metadata 对象，其中包含以下字段：

- results\_count
- next\_token

在第四步中，由于我们未指定 max\_results 参数，默认情况下只能获取 10 条帖子。要访问后续数据页，我们将从上一页结果中取出 next\_token 字段的值，并在 Postman 的 params 页面将该字符串填入 pagination\_token 参数，同时保持其他设置不变。

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `pagination_token` | avdjwk0udyx6 |

完成这些设置后，点击“Send”即可获取下一页结果。
