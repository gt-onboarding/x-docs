<div id="comparing-x-apis-likes-endpoints">
  ## 比较 X API 的点赞端点
</div>

本指南将重点介绍以下内容：

- **API 请求参数** - X API v2 端点引入了一组新的请求参数。虽然某些参数会比较熟悉，特别是对接入 Labs 的开发者而言，但仍存在许多重要差异，例如新增了 [fields](/zh/x-api/fundamentals/fields) 和 [expansions](/zh/x-api/fundamentals/expansions) 参数。

- **应用与项目要求** - 要访问 X API v2，您需要使用与某个[项目](/zh/resources/fundamentals/projects)关联的[开发者应用](/zh/resources/fundamentals/developer-apps)的凭据。

<div id="likes-lookup">
  #### 点赞查询
</div>

**点赞某条帖子的用户**

“点赞用户”端点是 v2 的新功能，可用于获取某条帖子被哪些用户点赞的信息。

| 描述 | X API v2 |
| :--- | :--- |
| 支持的 HTTP 方法 | GET |
| 主机域名 | https://api.x.com |
| 端点路径 | /2/tweets/:id/liking\_users |
| [身份验证](/zh/resources/fundamentals/authentication) | OAuth 2.0 Bearer Token<br /><br />OAuth 1.0a User Context |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 每 15 分钟 75 次请求（每个应用）<br /><br />每 15 分钟 75 次请求（每个用户） |

**某用户点赞的帖子**

下表对比了标准 v1.1 [GET favorites/list](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-favorites-list) 端点与 X API v2 的“某用户点赞的帖子”端点：

| 描述 | 标准 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | GET | GET |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/favorites/list.json | /2/users/:id/liked\_tweets |
| [身份验证](/zh/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 2.0 Bearer Token<br /><br />OAuth 1.0a User Context |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 每 15 分钟 75 次请求 | 每 15 分钟 75 次请求（每个应用）<br /><br />每 15 分钟 75 次请求（每个用户） |
| 数据格式 | 标准 v1.1 格式 | [X API v2 格式](/zh/x-api/fundamentals/data-dictionary)（由 fields 与 expansions 请求参数确定，且与 v1.1 格式不向后兼容）<br /><br />要了解如何从标准 v1.1 格式迁移到 X API v2 格式，请参阅我们的[数据格式迁移指南](/zh/x-api/migrate/data-format-migration)。 |

<div id="manage-likes">
  #### 管理点赞
</div>

v2 的点赞管理端点取代了 v1.1 的 [POST favorites/create](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-create) 和 [POST favorites/destroy](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-destroy) 端点。

下表对比了标准 v1.1 与 X API v2 的点赞管理端点：

<div id="like-a-post">
  #### 点赞帖子
</div>

| 描述 | 标准 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | POST | POST |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/favorites/create.json | /2/users/:id/likes |
| [身份验证](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文 | OAuth 1.0a 用户上下文 |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 每 24 小时 1000 次请求（每用户）<br /><br />每 24 小时 1000 次请求（每应用） | 每 15 分钟 50 次请求（每用户）<br /><br />每 24 小时 1000 次请求（每用户，与 DELETE 共享） |

<div id="unlike-a-post">
  #### 取消点赞帖子
</div>

| 描述 | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | POST | DELETE |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/favorites/destroy.json | /2/users/:id/likes/:tweet\_id |
| [身份验证](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文 | OAuth 1.0a 用户上下文 |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 每 24 小时 1000 次请求（每用户）<br /><br />每 24 小时 1000 次请求（每应用） | 每 15 分钟 50 次请求（每用户）<br /><br />每 24 小时 1000 次请求（每用户，与 POST 共享） |
