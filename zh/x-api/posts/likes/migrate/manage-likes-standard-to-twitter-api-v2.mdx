<div id="manage-likes-standard-v11-compared-to-x-api-v2">
  ### 管理点赞：标准版 v1.1 与 X API v2 对比
</div>

如果您一直在使用标准版 v1.1 的 [POST favorites/create](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-create) 和 [POST favorites/destroy](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-destroy) 端点，本指南旨在帮助您理解标准版 v1.1 与 X API v2 在管理点赞端点上的相同点与不同点。

- **相同点**
  - OAuth 1.0a 用户上下文
- **不同点**
  - 端点 URL 与 HTTP 方法
  - 应用与项目要求
  - 请求参数

<div id="similarities">
  #### 相似之处
</div>

**OAuth 1.0a 用户上下文身份验证方法**

这两个端点版本都支持 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication#oauth-1-0a-2)。因此，如果你之前使用的是标准版 v1.1 的管理收藏端点之一，迁移到 X API v2 后仍可继续使用相同的身份验证方法。

<div id="differences">
  #### 差异
</div>

**端点 URL 与 HTTP 方法**

- 标准 v1.1 端点：
  - POST https://api.x.com/1.1/favorites/create.json
    （点赞帖子）
  - POST https://api.x.com/1.1/favorites/destroy.json
    （取消点赞帖子）
- X API v2 端点：
  - POST https://api.x.com/2/tweets/:id/likes
    （点赞帖子）
  - DELETE https://api.x.com/2/tweets/:id/likes/:tweet\_id
    （取消点赞帖子）

**应用与项目要求**

在对请求进行身份验证时，X API v2 端点要求使用与某个[开发者 App](/zh/resources/fundamentals/developer-apps)关联到[项目](/zh/resources/fundamentals/projects)的凭据。所有 X API v1.1 端点均可使用独立 App 或与项目关联的 App 的凭据。

**请求参数**

以下标准 v1.1 请求参数在 X API v2 中有对应项：

| 标准 v1.1 | X API v2 |
| :--- | :--- |
| id  | id  |
| includes\_entities | 无对应项 |

请注意，标准 v1.1 的参数通过查询参数传递，而 X API v2 的参数在 POST 端点通过请求体参数传递，在 DELETE 端点通过路径参数传递。

另请注意，使用标准 v1.1 端点时，无需提供点赞用户的 id，因为通过[OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication)携带的[访问令牌](/zh/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)即可推断发起点赞/取消点赞操作的用户。
