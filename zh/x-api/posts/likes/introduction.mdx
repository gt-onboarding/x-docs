<div id="likes-lookup">
  ### 点赞查找
</div>

使用“点赞查找”组中的端点，您可以获取给某条帖子点过赞的账号列表，或某个账号点过赞的帖子列表。包含以下端点：

- 用户点过赞的帖子 - GET /2/users/:id/liked\_tweets
- 给帖子点过赞的用户 - GET /2/tweets/:id/liking\_users

您可以使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 或 [OAuth 2.0 Bearer Token](/zh/resources/fundamentals/authentication#oauth-2-0) 对这些端点进行身份验证。对于“点过赞的帖子”端点，将提供分页令牌以便遍历大量结果集。

“点过赞的用户”端点在任意时间范围内对每条帖子最多返回 100 个点赞账号。此外，“点过赞的帖子”端点还受项目级别的每月[帖子上限](/zh/x-api/fundamentals/post-cap)约束。
 

<div id="manage-likes">
  ### 管理点赞
</div>

“管理点赞”端点可让您代表已认证账户对指定帖子进行点赞或取消点赞。此端点组提供两种方法：POST 与 DELETE。POST 用于点赞，DELETE 用于取消点赞。

由于您以用户身份发起请求，必须使用[OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication)对这些端点进行身份验证，并使用与该用户关联的访问令牌；该令牌可通过[三方 OAuth 流程](/zh/resources/fundamentals/authentication#oauth-1-0a-2)/obtaining-user-access-tokens)生成。您可以从自己的账户或任一已认证用户的账户为帖子点赞。对于这两个端点，用户级速率限制为每个端点每 15 分钟 50 次请求。

要访问这些端点，您必须拥有已获批准的[开发者账户](/zh/resources/fundamentals/developer-portal)。进行身份验证时，必须使用位于某个[项目](/zh/resources/fundamentals/projects)内的[开发者应用](/zh/resources/fundamentals/developer-apps)所对应的密钥和令牌。

请在我们的[入门](/zh/x-api/getting-started/about-x-api)页面了解有关访问 X API v2 端点的更多信息。
