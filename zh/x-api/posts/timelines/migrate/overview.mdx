<div id="comparing-x-apis-timelines-endpoints">
  ## 比较 X API 的时间线端点
</div>

v2 倒序时间线、用户帖子时间线和用户提及时间线端点分别取代了 [v1.1 statuses/home\_timeine,](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-home_timeline) [v1.1 statuses/user\_timeline](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-user_timeline.html) 和 [v1.1 statuses/mentions\_timeline](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-mentions_timeline.html) 端点。如果您有使用此类端点旧版本的代码、应用或工具，并计划迁移到新版 X API v2 端点，那么本指南适合您。有关更深入的迁移指南，请参阅 [Standard v1.1 迁移到 X API v2](/zh/x-api/posts/timelines/migrate/standard-to-twitter-api-v2)。

本页包含三张对比表：

- 倒序主页时间线
- 用户帖子时间线
- 用户提及时间线

<div id="reverse-chronological-home-timeline">
  ### 按时间倒序的主页时间线
</div>

下表比较了标准 v1.1 与 X API v2 的主页时间线端点：

|     |     |     |
| :--- | :--- | :--- |
| **说明** | **Standard v1.1** | **X API v2** |
| 文档 | [API 参考](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-home_timeline) | [API 参考](https://developer.x.com/x-api/posts/timelines#user-mention-timeline-3) |
| 支持的 HTTP 方法 | `GET` | `GET` |
| 主机域名 | `https://api.x.com` | `https://api.x.com` |
| 端点路径 | `/1.1/statuses/home_timeline.json` | `/2/users/:id/timelines/reverse_chronological` |
| 必需参数 | `user_id` 或 `screen_name` | 将用户 ID 作为路径参数 :id |
| 身份验证 | OAuth 1.0a 用户上下文 | OAuth 1.0a 用户上下文<br /><br />[OAuth 2.0 授权码流程（含 PKCE）](/zh/resources/fundamentals/authenticationoauth-2-0/authorization-code) |
| 请求频率限制/量级限制 | 使用 OAuth 1.0a 用户上下文时，每 15 分钟 15 次请求<br /><br />请求上限：24 小时内 100,000 次。 | 每 15 分钟窗口内 180 次请求 <br /><br />[帖子上限](/zh/x-api/fundamentals/post-cap)：<br /><br />Essential 访问：500,000<br /><br />Elevated 访问：200 万<br /><br />Academic Research 访问：1,000 万 |
| 每次响应的默认帖子数 | 15  | 100 |
| 每次响应的最大发帖数 | 800 | 此端点会返回过去 7 天时间线中创建的所有帖子，以及不考虑创建日期的最新 800 条。 |
| 提供帖子编辑历史 | ✔   | ✔   |
| 可用历史帖子 | 最新的 800 条帖子，包含转帖 | 最新的 3,200 条帖子，包含转帖 |
| 时间线导航选项 | since\_id（不包含）用于轮询更新<br /><br />`max_id`（包含） | `start_time`<br /><br />`end_time`<br /><br />`since_id`（不包含）用于轮询更新 <br /><br />`until_id`（不包含） |
| 结果优化的可选参数 | `count`<br /><br />`exclude_replies`<br /><br />`include_rts`<br /><br />`trim_user`<br /><br />`tweet_mode`<br /><br />`since_id`<br /><br />`max_id` | `max_results`<br /><br />`exclude`（retweets,replies）<br /><br />`tweet.fields`<br /><br />`user.fields`<br /><br />`place.fields`<br /><br />`media.fields`<br /><br />`poll.fields`<br /><br />`expansions`<br /><br />`start_time`<br /><br />`end_time`<br /><br />`since_id`<br /><br />`until_id` |
| 支持请求并接收[注释](/zh/x-api/fundamentals/post-annotations) | 不适用 | 如果在 tweet.fields 中包含注释，结果将基于帖子文本附带推断的注释数据，例如“音乐流派”“民谣音乐”，或“音乐人”“Dolly Parton”。 |
| 支持请求并接收特定[帖子指标](/zh/x-api/fundamentals/metrics) | 不适用 | 如果在 `tweet.fields` 中包含注释，结果将为每条帖子附带 `public_metrics`，包括 `retweet_count`、`reply_count`、`quote_count`、`like_count`，以及 `non_public_metrics`，包括 `impression_count`、`user_profile_clicks`、`url_link_clicks`。<br /><br />另有媒体指标，如 `view_count` 和视频播放指标。<br /><br />对于推广帖子，使用用户上下文可获得额外的 `organic_metrics` 和 `promoted_metrics`。 |
| 支持请求并接收 [conversation\_id](/zh/x-api/fundamentals/conversation-id) | 不适用 | 返回 `conversation_id` 字段，其值表示回复线程中最先发布的帖子，帮助你跟踪会话。 |
| 帖子 JSON 格式 | [Standard v1.1 数据格式](https://developer.x.com/en/docs/twitter-api/v1/data-dictionary/overview.html) | [X API v2](/zh/x-api/fundamentals/data-dictionary) 格式（由 fields 和 expansions 请求参数确定，且与 v1.1 格式不向后兼容）<br /><br />如需了解如何从 Standard v1.1 格式迁移到 X API v2 格式，请参阅我们的[数据格式迁移指南](/zh/x-api/migrate/data-format-migration)。 |
| 结果顺序 | 按时间倒序 | 按时间倒序 |
| 结果分页 | 不适用，必须通过帖子 ID 导航 | 可使用 pagination\_token 向前或向后浏览结果 |
| 需要使用与[项目](/zh/resources/fundamentals/projects)关联的[开发者应用](/zh/resources/fundamentals/developer-apps)的凭证 |     | ✔   |

<div id="user-post-timeline">
  ### 用户帖子时间线

下表对标准版 v1.1 与 X API v2 的用户帖子时间线端点进行了对比：

|     |     |     |
| :--- | :--- | :--- |
| **描述** | **标准 v1.1** | **X API v2** |
| 文档 | [API 参考](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-user_timeline.html) | [API 参考](/zh/x-api/posts/user-posts-timeline-by-user-id) |
| 支持的 HTTP 方法 | GET | GET |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/statuses/user\_timeline.json | /2/users/:id/tweets |
| 必需参数 | user\_id 或 screen\_name | 将用户 ID 设为路径参数 :id |
| 身份验证 | OAuth 1.0a 用户上下文<br /><br />OAuth 2.0 应用专用 | OAuth 1.0a 用户上下文<br /><br />OAuth 2.0 应用专用<br /><br />OAuth 2.0 授权码（含 PKCE） |
| 请求速率/容量限制 | 使用 OAuth 1.0a 用户上下文时，每 15 分钟 900 次请求<br /><br />使用 OAuth 2.0 应用专用时，每 15 分钟 1500 次请求<br /><br />请求上限：24 小时内 100,000 次。 | 使用 OAuth 1.0a 用户上下文时，每 15 分钟窗口 900 次请求<br /><br />使用 OAuth 2.0 应用专用时，每 15 分钟窗口 1500 次请求<br /><br />[帖子上限](/zh/x-api/fundamentals/post-cap)：  <br />Essential 访问：500,000  <br />Elevated 访问：200 万  <br />Academic Research 访问：1000 万 |
| 每个响应的默认帖子数 | 15  | 10  |
| 每个响应的最大帖子数 | 200 | 100 |
| 可用历史帖子 | 最近 3,200 条帖子（包括转帖） | 最近 3,200 条帖子（包括转帖） |
| 时间线导航选项 | since\_id（不包含），用于增量轮询<br /><br />max\_id（包含） | start\_time<br /><br />end\_time<br /><br />since\_id（不包含），用于增量轮询<br /><br />until\_id（不包含） |
| 用于结果优化的可选参数 | count  <br />exclude\_replies  <br />include\_rts  <br />trim\_user  <br />tweet\_mode  <br />since\_id  <br />max\_id | max\_results  <br />exclude(retweets,replies)  <br />tweet.fields  <br />user.fields  <br />place.fields  <br />media.fields  <br />poll.fields  <br />expansions  <br />start\_time  <br />end\_time  <br />since\_id  <br />until\_id |
| 支持请求并接收[注释](/zh/x-api/fundamentals/post-annotations) | 不适用 | 返回基于帖子文本推断的注释数据的帖子结果，例如“音乐流派”“民间音乐”，或“音乐家”“Dolly Parton”。 |
| 支持请求并接收特定帖子[指标](/zh/x-api/fundamentals/metrics) | 不适用 | 返回每条帖子包含可用 public\_metrics 的结果，包括 retweet\_count、reply\_count、quote\_count 和 like\_count。  <br />  <br />在 OAuth 1.0a 用户上下文下可用：  <br />额外的 non\_public\_metrics，包括 impression\_count、user\_profile\_clicks、url\_link\_clicks。<br /><br />额外的媒体指标，如 view\_count 和视频播放指标。<br /><br />对投放的帖子，提供额外的 organic\_metrics 与 promoted\_metrics（需 OAuth 1.0a 用户上下文）。 |
| 支持请求并接收[conversation\_id](/zh/x-api/fundamentals/conversation-id) | 不适用 | 返回一个 conversation\_id 字段，其值表示回复线程中最早发布的帖子，帮助你跟踪会话。 |
| 帖子 JSON 格式 | [标准 v1.1 数据格式](/zh/x-api/fundamentals/data-dictionary) | [X API v2](/zh/x-api/fundamentals/data-dictionary) 格式（由 fields 和 expansions 请求参数确定；与 v1.1 格式不向后兼容）<br /><br />要了解如何从标准 v1.1 格式迁移到 X API v2 格式，请参阅我们的[数据格式迁移指南](/zh/x-api/migrate/data-format-migration)。 |
| 结果排序 | 按时间倒序 | 按时间倒序 |
| 结果分页 | 不适用，必须按帖子 ID 导航 | 可使用 pagination\_token 向前或向后查看结果 |
| 需要使用与[项目](/zh/resources/fundamentals/projects)关联的[开发者应用](/zh/resources/fundamentals/developer-apps)的凭据 |     | ✔   |
| 提供帖子编辑历史 | ✔   | ✔   |

| 用于优化结果的可选参数 | count  <br />trim\_user  <br />include\_entities  <br />tweet\_mode  <br />since\_id  <br />max\_id | max\_results  <br />tweet.fields  <br />user.fields  <br />place.fields  <br />media.fields  <br />poll.fields  <br />expansions  <br />start\_time  <br />end\_time  <br />since\_id  <br />until\_id |
