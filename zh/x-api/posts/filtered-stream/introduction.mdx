过滤流端点组使开发者能够过滤公共帖子流。该端点组包含多个端点，可用于创建和管理规则，并将这些规则应用于帖子流，以返回匹配的公共帖子。该端点组允许用户监听特定主题与事件、监控与竞赛相关的讨论、了解趋势的演变等。

开发者可以使用 REST [规则端点](/zh/x-api/posts/filtered-stream#post-2-tweets-search-stream-rules)，在无需断开连接的情况下向持久化流连接添加或移除规则。这些[规则](/zh/x-api/posts/filtered-stream#building-rules-for-filtered-stream)可通过运算符创建，以匹配帖子属性，如关键词、话题标签和 URL。运算符与规则子句可结合布尔逻辑与括号，以进一步细化过滤器的匹配行为。

添加一组规则后，你可以[建立流式连接](/zh/x-api/posts/filtered-stream#get-2-tweets-search-stream)，它将通过持久化的 HTTP 流式连接以 JSON 格式开始传递[帖子对象](/zh/x-api/fundamentals/data-dictionary#tweet)。连接至该流期间，你只会收到与规则匹配的内容。

此外，你可以将过滤流关联到 webhook，以 webhook 事件的形式进行投递。有关在过滤流中使用 webhooks 的更多详情，请参阅[过滤流 Webhooks 介绍](/zh/x-api/webhooks/stream/introduction)。

过滤搜索端点支持已编辑的帖子。该端点将投递匹配你一个或多个筛选条件的已编辑帖子，并包含其编辑历史，其中含有帖子 ID 数组。对于没有编辑历史的帖子，此数组仅包含一个 ID；对于已编辑的帖子，此数组包含多个 ID，按升序排列以反映编辑顺序，数组最后一个位置为最新版本。要了解帖子编辑的工作方式，请参阅[帖子编辑基础](/zh/x-api/fundamentals/edit-posts)页面。

过滤流端点的某些功能受[访问级别](/zh/x-api/getting-started/about-x-api)限制：

**专业版访问**

- 每个项目 1000 条规则
- 使用 POST /2/tweets/search/stream/rules 端点添加规则时，每 15 分钟最多 100 次请求
- 构建规则时可使用所有运算符
- 规则长度最多 1024 个字符

**企业版访问**

- 每个项目 25,000+ 条规则
- 构建规则时可使用所有运算符
- 规则长度最多 2048 个字符
- 申请企业版访问请点击[此处](https://developer.x.com/en/products/x-api/enterprise/enterprise-api-interest-form)

过滤流返回的帖子计入每月的[帖子上限](/zh/x-api/fundamentals/post-cap)。
