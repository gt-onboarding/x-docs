<div id="standard-v11-compared-to-x-api-v2">
  ## 标准版 v1.1 与 X API v2 的比较
</div>

如果你一直在使用标准版 v1.1 的 GET statuses/show 和 GET statuses/lookup，本指南将帮助你理解标准版与 X API v2 帖子查询端点之间的相似点与差异。

你可能也会对我们的[可视化数据格式迁移工具](/zh/x-api/migrate/data-format-migration)感兴趣，它可帮助你快速对比 [X API v1.1 数据格式](/zh/x-api/fundamentals/data-dictionary)与 [X API v2 格式](/zh/x-api/fundamentals/data-dictionary)之间的区别。

- **相似点**
  - OAuth 1.0a 用户上下文
  - 每次请求可返回的帖子数量上限
  - 支持帖子编辑历史与元数据

- **差异**
  - 端点 URL
  - 应用与项目要求
  - 响应数据格式
  - 请求参数

<div id="similarities">
  ### 相似点
</div>

<div id="oauth-10a-user-context-authentication-method">
  #### OAuth 1.0a 用户上下文身份验证方法
</div>

标准端点支持 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication)，而新的 X API v2 帖子查找端点同时支持 OAuth 1.0a 用户上下文和 [OAuth 2.0 应用专用](/zh/resources/fundamentals/authentication)。因此，如果您之前使用的是标准 v1.1 帖子查找端点之一，在迁移到 X API v2 版本时可以继续使用相同的身份验证方法。

应用专用身份验证可能是最易于入门的方式。要了解如何生成应用访问令牌，请参阅[本 OAuth 2.0 应用专用指南](/zh/resources/fundamentals/authentication)。

<div id="posts-per-request-limits">
  #### 每次请求的帖子数量限制
</div>

v1.1 的 [GET statuses/lookup](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-lookup) 端点允许你在一次请求中指定最多 100 条帖子。这同样适用于 GET /tweets 端点。若要指定完整的 100 条帖子，请将 `ids` 作为查询参数，并提供一个以逗号分隔的[帖子 ID](/zh/resources/fundamentals/x-ids)列表。

**对帖子编辑历史和元数据的支持**

两个版本都提供用于描述编辑历史的元数据。请参阅 Post lookup API 参考以及[编辑帖子基础知识](/zh/x-api/fundamentals/edit-posts)页面了解更多详情。

<div id="differences">
  ### 差别
</div>

<div id="endpoint-urls">
  #### 端点 URL
</div>

- **标准 v1.1 端点：**
  - `https://api.x.com/1.1/statuses/show`
  - `https://api.x.com/1.1/statuses/lookup`

- **X API v2 端点：**
  - `https://api.x.com/2/tweets`
  - `https://api.x.com/2/tweets/:id`

<div id="app-and-project-requirements">
  #### 应用与项目要求
</div>

X API v2 的端点需要使用与[项目](/zh/resources/fundamentals/projects)关联的[开发者应用](/zh/resources/fundamentals/developer-apps)凭证进行认证。X API v1.1 的端点可使用独立应用，或与项目关联的应用的凭证。

<div id="response-data-format">
  #### 响应数据格式
</div>

标准 v1.1 与 X API v2 端点版本之间的一个显著区别在于负载中字段的选择方式。

对于标准端点，许多响应字段默认返回，并可通过参数指定额外字段。

然而，X API v2 默认仅返回帖子（Post）的 `id` 和 `text` 字段。要获取其他字段和对象，需使用 [fields](/zh/x-api/fundamentals/fields) 和 [expansions](/zh/x-api/fundamentals/expansions) 参数。扩展得到的字段会在响应的 `includes` 对象中返回，并可通过匹配 ID 将其与主帖子对象对应起来。

有关使用 fields 和 expansions 的更多信息，请参阅[如何使用 fields 和 expansions 的指南](/zh/x-api/fundamentals/data-dictionary)。[数据格式迁移指南](/zh/x-api/fundamentals/fields)也提供了标准 v1.1 字段到新版 v2 字段的映射。

此外，X API v2 为对象（包括帖子（Post）和[用户](/zh/x-api/fundamentals/data-dictionary#user)对象）引入了新的 JSON 设计：

- 标准端点在 `statuses` 数组中返回帖子对象，而 X API v2 使用 `data` 数组。
- 在 X API v2 中，“转发”和“引用”帖子取代了“statuses”这一术语。
- 新术语如 `like` 取代了 `favorites` 和 `favourites` 等术语。
- 无值的属性（例如 `null`）不会包含在 X API v2 的负载中。

X API v2 中的帖子对象包含如下新字段：

- `conversation_id`
- 两个新的[注解](/zh/x-api/fundamentals/post-annotations)字段（`context` 和 `entities`）
- 新的[指标](/zh/x-api/fundamentals/metrics)字段
- `reply_setting` 字段，表示谁可以回复该帖子

<div id="request-parameters">
  #### 请求参数
</div>

以下标准 v1.1 请求参数在 X API v2 中有对应项：

| 标准 | X API v2 |
|:----------|:----------|
| `id`     | `ids`    |

某些标准 v1.1 参数在 X API v2 中不受支持：

| 标准            | 说明                                                                                                                      |
|:---------------------|:------------------------------------------------------------------------------------------------------------------------------|
| `tweet_mode`        | 已由 fields 和 expansions 功能取代。                                                                             |
| `trim_user`         | 已由 fields 和 expansions 取代。如需用户数据，请使用 `author_id` 扩展和 `user.fields`。                               |
| `include_my_retweet`| 对经身份验证用户转发的帖子，提供其源帖子的 ID。                                          |
| `include_entities`  | 使用 fields 和 expansions 控制负载中的实体。                                                               |
| `include_ext_alt_text` | 若存在替代文本，则在媒体实体中添加 `ext_alt_text` 字段。                                                         |
| `include_card_uri`  | 当附加广告卡片时，添加 `card_uri`。                                                                               |
| `map`               | 在 X API v2 中，对于不可用的帖子，返回帖子 ID 与错误消息；而在 v1.1 中对应字段会被置空。            |

<div id="curl-requests">
  ### CURL 请求
</div>

以下 cURL 请求展示了标准 v1.1 端点及其 v2 对应端点。将请求头中的 `ACCESS_TOKEN` 替换为你的应用访问令牌。对于 v2 端点，该令牌必须隶属于某个 [开发者 App](/zh/resources/fundamentals/developer-apps/overview)，且该 App 归属某个 [项目](/zh/resources/fundamentals/projects/overview)。

v1.1 的响应负载与 v2 不同。在 v2 中，你可以通过 [fields](/zh/x-api/fundamentals/fields) 和 [expansions](/zh/x-api/fundamentals/expansions) 参数请求不同的字段。

**标准 v1.1 `GET statuses/lookup` 与 v2 `GET /tweets` 端点**

```bash
curl --request GET \
  --url 'https://api.x.com/1.1/statuses/lookup.json?id=1460323737035677698%2C1460323743339741184' \
  --header 'Authorization: Bearer $ACCESS_TOKEN'
```

```
  curl --request GET \
  --url 'https://api.x.com/2/tweets?ids=1460323737035677698%2C1460323743339741184&tweet.fields=created_at&expansions=author_id&user.fields=created_at' \
  --header 'Authorization: Bearer $ACCESS_TOKEN'
```

**标准 v1.1 `GET statuses/show/:id` 与 v2 `GET /tweets/:id` 端点**

```
curl --request GET \
  --url 'https://api.x.com/1.1/statuses/show.json?id=1460323737035677698' \
  --header 'Authorization: Bearer $ACCESS_TOKEN'
```

```
curl --request GET \
  --url 'https://api.x.com/2/tweets/1460323737035677698?tweet.fields=created_at&expansions=author_id&user.fields=created_at' \
  --header 'Authorization: Bearer $ACCESS_TOKEN'
```
