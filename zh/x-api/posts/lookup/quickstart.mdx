<div id="getting-started-with-the-posts-lookup-endpoints">
  ## 开始使用 Posts 查找端点
</div>

本快速入门指南将帮助您使用 Postman 向 Posts 查找端点发出首个请求，并指定一组字段。

如果您想查看不同语言的示例代码，请访问我们的 GitHub 仓库：[X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code)。

<Note>
  ### 前提条件

  要完成本指南，您需要一组用于对请求进行身份验证的[密钥和令牌](/zh/resources/fundamentals/authentication)。您可以按照以下步骤生成这些密钥和令牌：

  - [注册开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)并通过审核。
  - 在开发者门户中创建一个[项目](/zh/resources/fundamentals/projects)及其关联的[开发者应用](/zh/resources/fundamentals/developer-apps)。
  - 前往应用的“Keys and tokens”页面生成所需凭据。请确保将所有凭据妥善保管于安全位置。
</Note>

<div id="steps-to-build-a-get-tweets-request">
  ### 构建 GET /tweets 请求的步骤
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### 第一步：选择一个工具或库
</div>

有多种工具、代码示例和库可用于向此端点发起请求，但这里我们将使用 Postman 来简化流程。

要将 X API v2 的 Postman 集合加载到你的环境中，请点击下方按钮：

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  将 X API v2 添加到 Postman
</Button>

在你已将 X API v2 集合加载到 Postman 后，前往 “Post Lookup > Multiple Posts” 请求。

<div id="step-two-authenticate-your-request">
  #### 步骤二：为请求进行身份验证
</div>

要正确向 X API 发出请求，你需要先确认你具备相应权限。你可以使用以下方式对请求进行身份验证：

- [OAuth 2.0 应用专用（App-Only）](zh/resources/fundamentals/authentication)
- [OAuth 2.0 授权码（含 PKCE）](zh/resources/fundamentals/authentication)
- [OAuth 1.0a 用户上下文](zh/resources/fundamentals/authentication)

为简化演示，我们将在此请求中使用 OAuth 2.0 应用专用。但如果你需要访问私有[指标](/zh/x-api/fundamentals/metrics)或帖子，则需改用其他身份验证方式之一。

使用 OAuth 2.0 应用专用时，你需要在 Postman 中添加密钥与令牌（尤其是应用访问令牌，即 App-only Bearer Token）：选择名为“X API v2”的环境（位于 Postman 右上角），然后将密钥与令牌填写到“initial value”和“current value”字段（点击环境下拉菜单旁的眼睛图标）。

如果配置正确，这些变量会自动填充到该请求的 Authorization 选项卡中。

<div id="step-three-identify-and-specify-which-posts-you-would-like-to-retrieve">
  #### 第三步：确定并指定要检索的帖子
</div>

必须在请求中指定希望接收的单个帖子或一组帖子。可通过访问 X.com，打开某条帖子后在其 URL 中查找帖子 ID。比如，以下 URL 的帖子 ID 是 `1228393702244134912`。

`https://x.com/TwitterDev/status/1228393702244134912`

在 Postman 中，打开“Params”选项卡，在 ids 参数的“Value”列中输入该 ID，或输入以逗号分隔的一串帖子 ID。

| Key  | Value |
| :---- | :----- |
| `ids` | `1228393702244134912,1227640996038684673,1199786642791452673` |

---

<div id="step-four-identify-and-specify-which-fields-you-would-like-to-retrieve">
  #### 第四步：确定并指定要检索的字段
</div>

如果你在第三步后点击“Send”按钮，响应中将返回默认的[帖子对象字段](/zh/x-api/fundamentals/data-dictionary)：`id`、`text` 和 `edit_history_tweet_ids`。

在本练习中，我们将从不同对象请求三组额外字段：

- 主体帖子对象中的额外 `tweet.created_at` 字段。
- 返回帖子所关联作者的用户对象的默认字段：`id`、`name` 和 `username`
- 关联用户对象中的额外 `user.created_at` 字段。

在 Postman 中，进入“Params”选项卡，在“Query Params”表中添加以下 `key:value` 对：
| Key            | Value               | Returned fields                          |
| :-------------- | :------------------- | :---------------------------------------- |
| `tweet.fields` | `created_at`        | `tweets.created_at`                      |
| `expansions`   | `author_id`         | `includes.users.id`, `includes.users.name`, `includes.users.username` |
| `user.fields`  | `created_at`        | `includes.users.created_at`              |

你应该会在“Send”按钮旁看到如下 URL：

```
https://api.x.com/2/tweets?ids=1228393702244134912,1227640996038684673,1199786642791452673&tweet.fields=created_at&expansions=author_id&user.fields=created_at
```
