帖子是 X 上的主要资源之一。以最简单的形式，帖子最多可包含 280 个字符，并可根据账户设置选择公开或私密发布。此外，帖子还可以附加多种对象，包括媒体、地点、投票和 URL。多数帖子在创建后的 30 分钟内可进行编辑。

尽管获取、筛选和分发帖子的方法多种多样，本组 REST 端点侧重返回由[帖子 ID](/zh/resources/fundamentals/x-ids)指定的单条或多条帖子。这些端点可获取帖子的最新详情、验证帖子是否可用，并查看其编辑历史。它们对于管理[合规事件](/zh/x-api/enterprise-gnip-2.0/fundamentals/firehouse)也至关重要。

帖子查询端点会提供已编辑帖子的元数据。自 2022 年 9 月 29 日起创建的所有帖子都会包含编辑元数据，即使从未被编辑。每次编辑都会生成新的帖子 ID，帖子的编辑历史用一个帖子 ID 数组表示，并以原始帖子的 ID 开头。

此端点始终返回最新编辑版本及其编辑历史。在 30 分钟编辑窗口过期后收集到的任何帖子都将代表其最终版本。要了解更多有关编辑帖子元数据的信息，请参阅[编辑帖子基础](/zh/x-api/fundamentals/edit-posts)页面。

这些端点使用 GET HTTP 方法，并返回一个或多个[帖子对象](/zh/x-api/fundamentals/data-dictionary)，其中包含帖子文本、创建时间戳，以及话题标签、提及和照片的列表与元数据等字段。
