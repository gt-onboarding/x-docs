## 开始使用帖子管理端点\*\*

本快速入门指南将帮助你使用 [Postman](/zh/tutorials/postman-getting-started) 向帖子管理端点发出你的第一个请求。

如果你想查看不同语言的示例代码，请访问我们的 [X API v2 示例代码](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 代码库。

<Info>
  **先决条件**

  要完成本指南，你需要一组用于对请求进行身份验证的[密钥和令牌](/zh/resources/fundamentals/authentication)。你可以按照以下步骤生成这些密钥和令牌：

  - [注册开发者账户](https://developer.x.com/en/apply-for-access)并通过审核。
  - 在开发者门户中创建一个[项目](/zh/resources/fundamentals/projects)及其关联的[开发者应用](/zh/resources/fundamentals/developer-apps)。
  - 前往你的应用的“Keys and tokens”页面生成所需的凭据。请务必将所有凭据妥善保管在安全位置。
</Info>

<div id="steps-to-building-manage-posts-requests">
  ### 构建管理 Posts 请求的步骤
</div>

**第一步：选择工具或库**

有多种工具、代码示例和库可用于向此端点发起请求。为简化流程，我们将在此使用 Postman。

要将 X API v2 的 Postman 集合加载到你的环境中，请点击以下按钮：

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  将 X API v2 添加到 Postman
</Button>

在将 X API v2 集合加载到 Postman 后，前往“Manage Posts”文件夹，选择“Create a Post”。
 

**第二步：为请求进行身份验证**

要正确地向 X API 发起请求，需要先验证权限。对于 manage Posts 端点，必须使用 [OAuth 1.0a User Context](/zh/resources/fundamentals/authentication) 或 [OAuth 2.0 Authorization Code with PKCE](zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 对请求进行身份验证。

在本示例中，我们将使用 OAuth 1.0a User Context。

你需要将密钥和令牌（具体为 API Key、API Secret Key、OAuth 1.0a user Access Token、OAuth 1.0a user Access Token Secret）添加到 Postman。方法是选择名为“X API v2”的环境（位于 Postman 右上角），并将密钥和令牌分别填入“initial value”和“current value”字段（点击环境下拉旁的“眼睛”图标） 。

如果配置正确，这些变量会自动注入到请求的 Authorization 选项卡中。

**第三步：指定 Post 的文本**

使用此端点创建新 Post 时，Post 的文本或媒体是必填的请求体参数。

在 Postman 中，切换到“Body”选项卡，将 Post 的文本作为 `text` 参数的值输入。此外，如需添加投票、回复某个 Post 的 ID，或设置回复权限等参数，也可在此处进行设置。详情参见我们的 [API 参考指南](/zh/x-api/posts/manage-tweets/introduction)。

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Parameter type** |
| `text` | Hello world! | body |

**第四步：确认并指定要检索的字段**

完成设置后，点击“Send”按钮，你将收到类似如下的响应：

```
{
  "data": {
    "id": "1445880548472328192",
    "text": "Hello world!"
  }
}
```

如果返回的响应对象包含 `id` 和该 Post 的 `text`，则说明已成功创建 Post。
 

**第五步：删除你的 Post**

要删除 Post，在 Postman 中已加载的 X API v2 集合的“Manage Posts”文件夹内选择“Delete a Post”请求。此端点需要提供要删除的 Post 的 ID。然后在“Params”选项卡中，将要删除的 Post 的 ID 填入 `id` 列对应的值。

删除成功后，你将收到类似如下的响应：

```
{
   "data": {
       "deleted" : true
   }
}
```
