<div id="authentication">
  ### 身份验证
</div>

所有 X API v2 端点都要求你使用一组凭证（也称为密钥和令牌）对请求进行身份验证。

这些特定端点需要使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication)，这意味着你必须使用一组 API 密钥和用户访问令牌才能成功发出请求。访问令牌必须与由你代表其发出请求的用户关联。如果你希望为另一位用户生成一组访问令牌，该用户必须通过 [三方 OAuth 流程](/zh/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)对你的应用进行授权或认证。

请注意，OAuth 1.0a 的使用门槛较高。如果你不熟悉这种身份验证方式，建议使用一个[库](/zh/x-api/tools-and-libraries/overview)、使用 Postman 等工具，或改用 OAuth 2.0 来对请求进行身份验证。

[OAuth 2.0 Authorization Code with PKCE](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 允许更精细地控制应用的权限范围，并支持跨设备的授权流程。OAuth 2.0 允许你选择细粒度的特定作用域，从而代表用户获取相应的具体权限。

要在你的应用中启用 OAuth 2.0，你必须在开发者门户的应用设置中的身份验证设置里将其启用。

<div id="developer-portal-projects-and-developer-apps">
  ### 开发者门户、项目与开发者应用
</div>

若要获取可用于 X API v2 端点的一组身份验证凭据，您必须拥有一个\[开发者账户]/resources/fundamentals/developer-portal)、在该账户下设置一个[项目](/zh/resources/fundamentals/projects)，并在该项目中创建一个[开发者应用](/zh/resources/fundamentals/developer-apps)。您可以随后在开发者应用中找到您的密钥和令牌。 
 

<div id="rate-limits">
  ### 速率限制
</div>

每天都有成千上万的开发者向 X API 发起请求。为管理庞大的请求量，我们在每个端点上设置了[速率限制](https://developer.x.com/x-api/fundamentals/rate-limits)，以限制你代表应用或代表已认证用户所能发起的请求次数。

这些端点采用用户级别的速率限制，这意味着你代表其发起请求的已认证用户，在任何开发者应用中对该端点的调用次数都有上限。对于 POST 方法，用户级别的速率限制为每 15 分钟 200 次请求。DELETE 方法的速率限制为每 15 分钟 50 次请求。此外，还存在[每 3 小时 300 次请求](https://blog.x.com/developer/en_us/topics/tools/2018/new-developer-requirements-to-protect-our-platform)的上限，其中包括通过 manage Posts 或 manage Retweets 创建的帖子。
 

<div id="source-labels">
  ### 来源标签
</div>

通过你的应用以编程方式创建的任何帖子，其元数据中会将你的应用名称和网站 URL 显示为[来源标签](https://help.x.com/en/using-twitter/how-to-tweet#source-labels)。如果你更改了 X 应用的使用场景，请务必在这些设置中同步更新，以确保遵守[开发者条款](https://developer.x.com/content/developer-twitter/en/developer-terms/agreement-and-policy)。
 

<div id="profile-settings">
  ### 个人资料设置
</div>

仅当你在个人资料设置中启用了地理位置（geo）时，才能为帖子添加位置。如果未启用地理位置，你仍可在请求正文中添加位置参数，但该位置不会附加到帖子上。对在图片中标记用户同样适用。如果你标记的用户未启用照片标记功能，即使帖子已成功创建，他们的名字也不会显示在已标记用户列表中。 
 

<div id="adding-media-to-a-post">
  ### 向帖子添加媒体
</div>

目前，X API v2 尚不支持在此处完整上传媒体。但你可以将先前已上传的媒体附加到帖子。你可以使用已[通过 v1 媒体端点上传](https://developer.x.com/en/docs/twitter-api/v1/media/upload-media/api-reference/post-media-upload)或 [X Media Studio](https://media.x.com/en/articles/products/2018/media-studio) 上传的媒体 ID。这些媒体 ID 必须属于你本人或属于已通过身份验证的用户。
