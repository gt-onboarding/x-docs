<div id="comparing-x-apis-manage-posts-endpoints">
  ## 比较 X API 的帖子管理端点
</div>

v2 帖子管理端点将取代标准 v1.1 的 [POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) 和 [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id) 端点。如果您有使用 v1.1 版本帖子管理端点的代码、应用或工具，并正考虑迁移到更新的 X API v2 端点，那么本系列指南适合您。

下表对比了标准 v1.1 与 X API v2 的帖子管理端点：

<div id="create-a-post">
  ### 创建帖子
</div>

| 说明 | 标准 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | POST | POST |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/statuses/update.json | /2/tweets |
| [身份验证](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文 | OAuth 1.0a 用户上下文<br /><br />OAuth 2.0 授权码（含 PKCE） |
| 默认请求[速率限制](/zh/x-api/fundamentals/rate-limits) | 无<br /><br />每位用户、每个应用每 3 小时可发起 300 次请求。与 v1.1 的 POST Retweets 端点共享。 | 每位用户每 15 分钟可发起 200 次请求<br /><br />每位用户、每个应用每 3 小时可发起 300 次请求。与 v2 的 POST Retweets 端点共享。 |
| 需要使用与[项目](/zh/resources/fundamentals/projects)关联的[开发者应用](/zh/resources/fundamentals/developer-apps)凭证 |     | ✔   |

<div id="delete-a-post">
  ### 删除帖子
</div>

| 描述 | 标准 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | POST | DELETE |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/statuses/destroy/:id.json | /2/tweets/:id |
| [身份验证](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文 | OAuth 1.0a 用户上下文<br /><br />OAuth 2.0 授权码（含 PKCE） |
| 默认请求[速率限制](/zh/x-api/fundamentals/rate-limits) | 无 | 每位用户每 15 分钟 50 次请求 |
| 需要使用与[项目](/zh/resources/fundamentals/projects)关联的[开发者应用](/zh/resources/fundamentals/developer-apps)凭据 |     | ✔   |
