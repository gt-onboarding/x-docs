<div id="standard-v11-compared-to-x-api-v2">
  ## 标准版 v1.1 与 X API v2 的比较
</div>

如果您一直在使用标准版 v1.1 的 [POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) 和 [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id) 端点，本指南旨在帮助您理解标准版与 X API v2 管理帖子端点之间的相似与差异。

- **相似点**
  - 身份验证
- **差异**
  - 端点 URL

  - 应用与项目要求

  - 请求参数

<div id="similarities">
  ### 相似之处
</div>

**身份验证**

标准 v1.1 与 X API v2 中用于管理帖子（[POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) 和 [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id)）的端点均使用 [OAuth 1.0a 用户上下文](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication)。因此，如果您此前使用过标准 v1.1 的相关端点，迁移到 X API v2 后仍可继续使用相同的身份验证方法。 

<div id="differences">
  ### 差异
</div>

**端点 URL**

- 标准 v1.1 端点：
  - https://api.x.com/1.1/statuses/update.json
    （创建帖子）
  - `https://api.x.com/1.1/statuses/destroy/:id.json`
    （删除帖子）
- X API v2 端点：
  - https://api.x.com/2/tweets
    （创建帖子）
  - https://api.x.com/2/tweets/:id
    （删除指定帖子）

<div id="app-and-project-requirements">
  ### 应用与项目要求
</div>

在对请求进行认证时，X API v2 的端点要求你使用与某个[项目](/zh/resources/fundamentals/developer-apps)关联的[开发者应用](/zh/resources/fundamentals/developer-apps)的凭据。所有 X API v1.1 端点均可使用独立应用或与项目关联的应用的凭据。

<div id="request-parameters">
  ### 请求参数
</div>

在 v1.1 中，相关端点支持两个查询参数（user\_id 或 screen\_name）。而在 X API v2 中，DELETE 端点仅接受数值类型的帖子 ID，且必须作为端点路径的一部分传递。

对于 POST 端点，需要通过请求的 JSON 正文传入其他参数。你可以在 [API 参考指南](/zh/x-api/posts/manage-tweets/introduction) 中了解可用参数的更多信息。
