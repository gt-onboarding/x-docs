使用 X API 创建和删除帖子是参与公共对话的关键。全新的“管理帖子”端点可满足这些需求，并提供更多功能。

我们提供两种用于管理帖子的 HTTP 方法：POST 和 DELETE。POST 方法支持发布投票、引用推文、设置回复权限、添加地理位置、附带媒体并标记用户、面向超级关注者发布等功能。相应地，DELETE 方法用于删除特定帖子。对于 POST 方法，你可以传入[所需参数](/zh/x-api/posts/creation-of-a-post)以进一步自定义请求。

“删除帖子”方法已更新以支持已编辑的帖子。若删除某个帖子编辑链中的任意一条，整个编辑链中的所有帖子都会被删除。关于编辑帖子的元数据，请参阅[编辑帖子的基础知识](/zh/x-api/fundamentals/edit-posts)。

POST 方法的用户速率限制为每 15 分钟 200 次请求；DELETE 方法为每 15 分钟 50 次请求。此外，还有每 3 小时 300 次请求的上限，包含通过“管理帖子”或“管理转推”创建的帖子。

由于你是代表用户调用“管理帖子”端点，必须使用 [OAuth 1.0a 用户上下文](https://en.docs/authentication/oauth-1-0a)或 [OAuth 2.0（含 PKCE）的授权码](https://en.docs/authentication/oauth-2-0/authorization-code)进行身份验证，并使用已授权你的应用所关联的用户访问令牌。使用 OAuth 1.0a 生成用户访问令牌时，可采用[三方 OAuth 流程](https://en.docs/authentication/oauth-2-0/bearer-tokens)；使用 OAuth 2.0 时，可采用[带 PKCE 的授权码授权流程](https://en.docs/authentication/oauth-2-0/user-access-token)。

<Info>
  **账户设置**

  要访问这些端点，你需要：

  - 一个已批准的[开发者账户](https://en/portal/petition/essential/basic-info)。
  - 使用位于某个[项目](/zh/resources/fundamentals/projects)中的[开发者应用](/zh/resources/fundamentals/developer-apps)的密钥和令牌进行身份验证。

  在我们的[入门指南](/zh/x-api/getting-started/getting-access)中了解如何获取对 X API v2 端点的访问权限。
</Info>
