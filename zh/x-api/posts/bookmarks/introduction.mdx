<div id="manage-bookmarks">
  ### 管理书签
</div>

我们提供两种用于管理书签的方法：POST 和 DELETE。POST 方法用于创建书签，相应地，DELETE 方法用于删除特定书签。

对于 POST 和 DELETE 方法，每位用户的速率限制为每 15 分钟 50 次请求。

由于你将以用户身份调用管理书签的端点，必须通过 OAuth 2.0 生成用户访问令牌进行身份验证。你可以使用[带 PKCE 的授权码模式](/zh/resources/fundamentals/authentication/oauth-2-0/user-access-token)。要使用此端点，你必须传入 `tweet.read`、`users.read` 和 `bookmark.write` 这几个 scope。

<div id="bookmarks-lookup">
  ### 书签查询
</div>

书签查询端点仅提供一种方法：GET。该方法允许你获取自己或已认证账号的书签。对于该端点的大量结果，将提供分页令牌以便分页检索。

GET 方法对每位用户的速率限制为每 15 分钟窗口内 180 次请求。使用此端点，你最多可取回你最近添加书签的 800 条帖子。

由于你是以用户身份调用书签查询端点，必须通过 OAuth 2.0 生成用户访问令牌进行认证。你可以使用[带 PKCE 的授权码流程](/zh/resources/fundamentals/authentication/oauth-2-0/user-access-token)。使用此端点时，必须传入权限范围 `tweet.read`、`users.read` 和 `bookmark.read`。

**账户设置**

要访问这些端点，你需要：

- 一个已获批准的[开发者账户](https://developer.x.com/en/portal/petition/essential/basic-info)。
- 使用位于某个[项目](/zh/resources/fundamentals/projects)中的[开发者应用](/zh/resources/fundamentals/developer-apps)的密钥和令牌进行认证。

在我们的[入门指南](/zh/x-api/getting-started/getting-access)中了解如何获取 X API v2 端点的访问权限。
