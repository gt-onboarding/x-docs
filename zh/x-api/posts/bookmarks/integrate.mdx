<div id="authentication">
  #### 身份验证
</div>

所有 X API v2 端点都要求你使用一组凭证（也称为密钥和令牌）来对请求进行身份验证。

以下端点必须使用 [OAuth 2.0（含 PKCE）的授权码流程](/zh/resources/fundamentals/authentication/oauth-2-0/authorization-code)，也就是说，你必须使用一组密钥和用户访问令牌才能成功发起请求。访问令牌必须与你代表其发起请求的用户关联。如果你想为其他用户生成一组访问令牌，他们必须通过 [Authorize URL](/zh/resources/fundamentals/authentication#authorize-url) 授权或对你的应用进行身份验证。

请注意，OAuth 2.0 的使用可能较为复杂。如果你不熟悉此身份验证方式，建议使用 [库](/zh/x-api/tools-and-libraries/overview) 或 Postman 等工具来正确完成请求的身份验证。

<div id="developer-portal-projects-and-developer-apps">
  #### 开发者门户、项目与开发者应用
</div>

要获取可用于 X API v2 端点的一组身份验证凭据，您需要拥有一个[开发者账号](/zh/resources/fundamentals/developer-portal)，在该账号下设置一个[项目](/zh/resources/fundamentals/projects)，并在该项目内创建一个[开发者应用](/zh/resources/fundamentals/developer-apps)。然后，您可以在开发者应用中找到密钥和令牌。 

<div id="rate-limits">
  #### 速率限制
</div>

每天都有成千上万的开发者向 X API 发起请求。为管理庞大的请求量，我们在每个端点上设置了[速率限制](/zh/resources/fundamentals/rate-limits)，以限制你代表你的应用或已通过身份验证的用户所能发起的请求次数。

这些端点按用户级别进行速率限制，这意味着你所代表的已认证用户在任何开发者 App 中对该端点的调用次数都有上限。对于 GET 方法，用户级别的速率限制为每 15 分钟 180 次请求。使用书签查询端点的 GET 方法，你最多可以取回最近的 800 条已加书签的帖子。此外，对于 POST 和 DELETE 方法，用户级别的速率限制为每 15 分钟 50 次请求。
