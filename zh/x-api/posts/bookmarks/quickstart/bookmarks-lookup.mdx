<div id="getting-started-with-the-bookmarks-lookup-endpoint">
  ### 书签查询端点入门
</div>

本快速入门指南将帮助你使用 [Postman](/zh/tutorials/postman-getting-started) 向书签查询端点发出第一个请求。

如果你想查看不同语言的示例代码，请访问我们的 [X API v2 示例代码](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 代码库。

<div id="prerequisites">
  #### 先决条件
</div>

要完成本指南，您需要一组用于对请求进行身份验证的[keys and tokens](/zh/resources/fundamentals/authentication)。您可以按照以下步骤生成这些 keys 和 tokens：

- [注册开发者账号](https://developer.x.com/en/apply-for-access)并通过审批。
- 在开发者门户中创建一个[项目](/zh/resources/fundamentals/projects)及其关联的[开发者应用](/zh/resources/fundamentals/developer-apps)。
- 前往您应用的“Keys and tokens”页面生成所需凭据。请务必将所有凭据妥善保存在安全位置。

<div id="steps-to-build-a-bookmarks-lookup-request">
  #### 构建“书签”查询请求的步骤
</div>

**步骤一：选择工具或库**

有多种工具、代码示例和库可用于向此端点发起请求，为简化流程，我们这里使用 Postman。

要将 X API v2 的 Postman 集合导入到您的环境，请点击以下按钮：

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  将 X API v2 添加到 Postman
</Button>

在 Postman 中加载好 X API v2 集合后，前往“Bookmarks”文件夹并选择“Bookmarks lookup”。

**步骤二：为请求进行身份验证**

要成功调用此端点，您需要使用 [OAuth 2.0 Authorization Code Flow with PKCE](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3)。您可以在 Postman 内生成访问令牌。

进入“Authorization”选项卡并选择“OAuth 2.0”。

在该选项卡中，请按以下步骤操作：

1. 为令牌命名

2. 将 Grant Type 设为 Authorization Code (with PKCE)

3. 更新以下参数：

   **Callback URL** - [https://www.example.com](https://www.example.com/)

   该地址应与您在开发者门户身份验证设置页中配置的回调 URL 一致。

   **Auth URL** - [https://x.com/i/oauth2/authorize](https://x.com/i/oauth2/authorize)

   **Access Token URL** - [https://api.x.com/2/oauth2/token](https://api.x.com/2/oauth2/token)

   **Client ID** - 从开发者门户复制并粘贴 OAuth 2.0 客户端 ID

   **Client Secret** - 从开发者门户复制并粘贴 OAuth 2.0 客户端密钥。仅在您的应用类型为机密客户端时需要。

4. 将 scopes 设置为：post.read users.read bookmark.read

5. 在 state 字段填写“State”

6. 点击“Generate Token”

7. 点击保存图标以保存文件夹更改。

您可能会收到未登录 X 的提示。如果出现此错误，请在 Postman 中登录将代表其发起请求的 X 账号。

**步骤三：指定用户**

此端点要求指定要在响应中返回其数据的用户 ID。例如，@XDevelopers 的用户 ID 是 2244994945。在 Postman 中，前往“Params”选项卡，将您自身或已认证用户的 ID 填入 id 参数。

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 2244994945 |

**步骤四：发起请求并查看响应**
