  ## 比较 X API 的搜索帖子端点
</div>

v2 Search Tweets 端点最终将取代[标准版 v1.1 search/posts](/zh/x-api/posts/search/introduction) 端点和[企业版 Search API](/zh/x-api/enterprise-gnip-2.0/fundamentals/search-api)。如果你有使用旧版 X 搜索端点的代码、应用或工具，并计划迁移到更新的 X API v2 端点，本指南适合你。

本页包含三张对比表：

- [近期搜索](#recent)
- [全档案搜索](#full-archive)
- [过滤运算符对比](#operators)

<div id="recent-search-comparison">
  ### 近期搜索对比
</div>

下表比较了不同类型的近期搜索端点：

| **说明** | **Standard v1.1** | **X API v2** |
| :--- | :--- | :--- |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/search/tweets.json | /2/tweets/search/recent |

| 时间戳格式 | YYYYMMDD | YYYY-MM-DDTHH:mm:ssZ<br />[ISO 8601 / RFC 3339](https://tools.ietf.org/html/rfc3339#section-5.6) |
| 返回的帖子最早时间 | 7 天 | 7 天 |
| 支持的 HTTP 方法 | GET | GET |
| 默认请求速率限制 | 使用 OAuth 1.0a 用户上下文：每 15 分钟 180 次请求<br /><br />使用 OAuth 2.0 仅应用：每 15 分钟 450 次请求 | **Basic：**<br /><br />使用 OAuth 2.0 仅应用：每 15 分钟 60 次请求<br /><br />使用 OAuth 1.0a 用户上下文：每 15 分钟 60 次请求<br /><br />使用 OAuth 2.0 授权码（含 PKCE）：每 15 分钟 60 次请求<br /><br />**Pro：**<br /><br />使用 OAuth 2.0 仅应用：每 15 分钟 450 次请求<br /><br />使用 OAuth 1.0a 用户上下文：每 15 分钟 180 次请求<br /><br />使用 OAuth 2.0 授权码（含 PKCE）：每 15 分钟 180 次请求 |
| 提供完全展开的 URL |     | ✔   |
| 每个响应的最大帖子数（默认） | 100（15） | 100（10） |
| 帖子 JSON 格式 | Standard v1.1 格式 | [X API v2 格式](/zh/x-api/fundamentals/data-dictionary)（由 fields 和 expansions 请求参数决定，与 v1.1 格式不兼容）<br /><br />要了解如何从 Standard v1.1 格式迁移到 X API v2 格式，请参阅我们的[数据格式迁移指南](/zh/x-api/migrate/data-format-migration)。 |
| 支持选择在有效负载中返回的[字段](/zh/x-api/fundamentals/fields) |     | ✔   |
| 支持请求并接收[注解](/zh/x-api/fundamentals/post-annotations) |     | ✔   |
| 支持在帖子对象内请求特定[指标](/zh/x-api/fundamentals/metrics) |     | ✔   |
| 支持 [conversation\_id](/zh/x-api/fundamentals/conversation-id) 运算符和字段 |     | ✔   |
| 提供帖子编辑历史 | ✔   | ✔   |
| 帖子数据数组的 JSON 键名 | statuses | data |
| 分页的 JSON 键名 | search\_metadata.next\_results | meta.next\_token |
| 支持按时间范围浏览归档 | ✔   | ✔   |
| 基于时间的请求的时间分辨率 | day | second |
| 时区 | UTC | UTC |
| 按时间浏览的请求参数 | until | start\_time<br />end\_time |
| 按帖子 ID 浏览的请求参数 | since\_id<br />max\_id | since\_id<br />until\_id |
| 分页的请求参数 | 提供 URL 编码的查询 | next\_token |
| 需要使用与[项目](/zh/resources/fundamentals/projects)关联的[开发者应用](/zh/resources/fundamentals/developer-apps)的凭据 |     | ✔   |

**过滤运算符对比**

搜索帖子的三个不同版本（standard、enterprise 和 v2）在可用运算符方面存在差异，且每个版本内的运算符可用性层级也不同，具体如下所述。 

Enterprise

- enterprise 运算符没有子层级

X API v2

- **Free：** 适用于任何项目
- **Basic：** 适用于任何项目
- **Pro：** 适用于某个项目
- **Enterprise：** 适用于某个项目

您可以在各自的指南中了解这些运算符集的详细信息：

- [Enterprise 运算符](/zh/x-api/enterprise-gnip-2.0/fundamentals/search-api#available-operators)
- [X API v2 运算符](/zh/x-api/posts/search/integrate/build-a-query#operators)
