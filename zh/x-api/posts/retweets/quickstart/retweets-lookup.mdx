<div id="getting-started-with-the-retweets-lookup-endpoint">
  ### 转推查询端点快速上手
</div>

本快速入门指南将帮助你使用 [Postman](/zh/tutorials/postman-getting-started) 向转推查询端点发出第一个请求。若需查看不同语言的示例代码，请访问我们的 GitHub 仓库：[X API v2 示例代码](https://github.com/xdevplatformTwitter-API-v2-sample-code)。

<div id="prerequisites">
  #### 先决条件
</div>

要完成本指南，您需要一组用于对请求进行身份验证的[keys and tokens](/zh/resources/fundamentals/authentication)。您可以按以下步骤生成这些 keys 和 tokens：

- [注册开发者账户](https://developer.x.com/en/apply-for-access)并通过审核。
- 在开发者门户中创建[项目](/zh/resources/fundamentals/projects)及其关联的[开发者应用](/zh/resources/fundamentals/developer-apps)。
- 前往您的应用的“Keys and tokens”页面生成所需的凭据。请务必将所有凭据安全保存。

<div id="steps-to-build-a-retweets-lookup-request">
  #### 构建转发查询请求的步骤
</div>

**第一步：选择一种工具或库入手**

你可以使用多种工具、代码示例和库来向该端点发起请求，但这里我们将使用 Postman 来简化流程。

要将 X API v2 的 Postman 集合导入到您的环境，请单击以下按钮：

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  将 X API v2 添加到 Postman
</Button>

在 Postman 中加载 X API v2 集合后，前往“Retweets”文件夹并选择“Retweeted by”。

**步骤二：为请求进行身份验证**

要正确向 X API 发起请求，您需要先确认自己拥有相应权限。为此，该端点要求您使用以下任一身份验证方式对请求进行认证：[OAuth 2.0 仅应用](/zh/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)、[OAuth 2.0 授权码（含 PKCE）](zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)，或 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication)。

为简化起见，我们在此请求中将使用 OAuth 2.0 的仅应用模式；但如果你需要请求私有[指标](/zh/x-api/fundamentals/metrics)或转发数据，则需改用其他身份验证方法之一。

要使用 OAuth 2.0 的 App-Only（仅应用）模式，你必须将密钥和令牌（尤其是 [App Access Token](/zh/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)，亦称 App-only Bearer Token）添加到 Postman。操作方法：在 Postman 右上角选择名为“X API v2”的环境，然后在“initial value”和“current value”字段中添加你的密钥和令牌（点击环境下拉菜单旁的眼睛图标即可编辑）。

如果您已正确完成上述步骤，这些变量会自动填入请求的“Authorization（授权）”选项卡中。

**步骤三：指定帖子**

使用此端点时，必须指定要获取其转发用户的帖子 ID。你可以在 X 上打开该帖子，并从其 URL 末尾提取数字 ID。比如，以下 URL 的帖子 ID 为 1354143047324299264。

https://x.com/TwitterDev/status/1354143047324299264

在 Postman 中，切换到“Params”选项卡，将该用户名填入 id 路径变量的“Value”列（位于本节底部），并确保用户名前后不含空格。

|     |     |
| :--- | :--- |
| **键** | **值** |
| `id` | 你想获取其转发用户列表的帖子 ID |

**第四步：确定并指明要检索的字段**

如果你在完成第三步后点击“发送”按钮，响应中将返回默认的 [user 对象](/zh/x-api/fundamentals/data-dictionary#user)字段：id、name 和 username。

如果你需要除 id、name 和 username 之外的其他字段，必须在请求中通过 [fields](/zh/x-api/fundamentals/fields) 和/或 [expansions](/zh/x-api/fundamentals/expansions) 参数明确指定。

在本练习中，我们将从不同对象中请求另外三组字段：

1. 主要用户对象中的新增字段 user.created\_at。
2. 与返回用户关联的置顶帖子对象的默认字段：id 和 text。
3. 关联的帖子对象中新增的 tweet.created\_at 字段。

在 Postman 中，前往“Params”选项卡，并在“Query Params”表中添加以下键值对：

|     |     |     |
| :--- | :--- | :--- |
| **键** | **值** | **返回字段** |
| user.fields | created\_at | user.created\_at |
| expansions | pinned\_tweet\_id | tweet.id, tweet.text |
| tweet.fields | created\_at | tweet.created\_at |

现在，你应当在“发送”按钮旁边看到以下 URL：

`https://api.x.com/2/tweets/1354143047324299264/retweeted_by?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=created_at`
