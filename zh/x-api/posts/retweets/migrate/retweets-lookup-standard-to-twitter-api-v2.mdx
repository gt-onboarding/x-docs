<div id="retweets-lookup-standard-v11-compared-to-x-api-v2">
  ### 转推查询：标准版 v1.1 与 X API v2 的比较
</div>

如果您一直在使用标准版 v1.1 的 [v1.1 GET statuses/retweets/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweets-id)、[v1.1 GET statuses/retweets/:ids](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweeters-ids)，本指南旨在帮助您理解标准版 v1.1 与 X API v2 转推查询端点之间的相似与差异。

- **相似点**
  - 身份验证
  - 每次请求的用户数量限制
- **差异**
  - 端点 URL

  - 请求限制

  - 应用与项目要求

  - 响应数据格式

  - 请求参数

<div id="similarities">
  #### 相似点
</div>

**身份验证**

标准 v1.1 和 X API v2 的转推查询端点（[v1.1 GET statuses/retweets/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweets-id) 和 [v1.1 GET statuses/retweeters/:ids](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweeters-ids)）均可使用 [OAuth 1.0a 用户上下文](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication) 或 OAuth 2.0 Bearer Token 进行访问。

**每次请求的用户数量限制**

对于 v1.1 和 v2 的 GET 端点，转推查询端点返回的用户数上限为 100。对于 v2 的转推查询端点，由于不传递分页令牌，默认返回 100 个用户，这也是可返回的最大值。

<div id="differences">
  #### 差异
</div>

**端点 URL**

- 标准 v1.1 端点：
  - https://api.x.com/1.1/statuses/retweets/:id.json
    （返回由 `id` 参数指定的帖子最近 100 条转帖的集合）
  - `https://api.x.com/1.1/statuses/retweeters/ids.json`
    （返回由 `id` 参数指定的帖子最多 100 个转帖用户的用户 ID 集合）
- X API v2 端点：
  - https://api.x.com/2/tweets/:id/retweeted\_by
    （返回转帖了某条帖子的账号列表）

**应用和项目要求**

使用 X API v2 端点进行请求认证时，必须使用与[项目](/zh/resources/fundamentals/developer-apps)关联的[开发者应用](/zh/resources/fundamentals/developer-apps)凭据。所有 X API v1.1 端点可使用独立应用或与项目关联的应用的凭据。

**响应数据格式**

标准 v1.1 与 X API v2 端点版本之间的主要差异之一是如何选择在负载中返回的字段。

对于标准端点，许多响应字段默认返回，你也可以通过参数指定负载中应返回哪些字段或字段集。

X API v2 版本默认仅返回用户的 id、name 和 username 字段。要请求其他任何字段或对象，你需要使用[fields](/zh/x-api/fundamentals/fields)和[expansions](/zh/x-api/fundamentals/expansions)参数。你从此端点请求的任何用户字段将返回在主用户对象中。任何展开的帖子对象及其字段会在响应中的 includes 对象内返回。随后，你可以通过匹配用户对象与展开的帖子对象中出现的 ID，将任何展开对象与用户对象对应起来。

我们鼓励你在各自的指南中进一步了解这些新参数，或阅读我们的[如何使用 fields 与 expansions](/zh/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)指南。

我们还提供了[数据格式迁移指南](/zh/x-api/migrate/data-format-migration#migrating-from-standard-v1-1s-data-format-to-v2)，可帮助你将标准 v1.1 字段映射到较新的 v2 字段。该指南还会提供在 v2 请求中为返回特定字段而需传递的具体 expansion 和 field 参数。
 

除了请求特定字段的方式变化外，X API v2 还为 API 返回的对象引入了新的 JSON 设计，包括[Post](/zh/x-api/fundamentals/data-dictionary#tweet)和[user](/zh/x-api/fundamentals/data-dictionary#user)对象。

- 在 JSON 根级别，标准端点在 statuses 数组中返回帖子对象，而 X API v2 在 data 数组中返回。 
- X API v2 的 JSON 不再将转帖与引用称为“statuses”，而是称为 Retweeted 和 Quoted Tweets。许多旧版和已弃用字段（如 contributors 和 user.translator\_type）将被移除。 
- 不再同时使用 favorites（帖子对象中）和 favourites（用户对象中），X API v2 统一使用术语 like。 
- X 采用的约定是：无值（例如 null）的 JSON 字段不会写入负载。帖子和用户属性仅在具有非空值时才会被包含。 
   

我们还为[帖子对象](/zh/x-api/fundamentals/data-dictionary#tweet)引入了一组新字段，包括：

- 一个 [conversation\_id](/zh/x-api/fundamentals/conversation-id) 字段
- 两个新的[annotations](/zh/x-api/fundamentals/post-annotations)字段，包括 context 和 entities
- 若干新的[metrics](/zh/x-api/fundamentals/metrics)字段
- 一个新的 reply\_setting 字段，显示谁可以回复某条帖子

**请求参数**

以下标准 v1.1 请求接受两个查询参数（user\_id 或 screen\_name）。X API v2 仅接受数值型用户 ID，且必须作为端点路径的一部分传递。
