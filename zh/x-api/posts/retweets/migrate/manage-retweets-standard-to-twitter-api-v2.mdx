<div id="manage-retweets-standard-v11-compared-to-x-api-v2">
  ### 管理转推：标准版 v1.1 与 X API v2 对比
</div>

如果您一直在使用标准版 v1.1 的 [POST statuses/retweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-retweet-id) 和 [POST statuses/unretweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-unretweet-id) 端点，本指南旨在帮助您了解标准版与 X API v2 转推端点之间的相同点与差异。

- **相同点**
  - 身份验证
- **不同点**
  - 端点 URL 和 HTTP 方法
  - 请求限制
  - 应用与项目要求
  - 请求参数

<div id="similarities">
  #### 相似之处
</div>

**身份验证**

标准 v1.1 与 X API v2 的转推管理端点（[POST statuses/retweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-retweet-id) 和 [POST statuses/unretweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-unretweet-id)）均使用 [OAuth 1.0a 用户上下文](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication)。因此，如果您此前使用的是标准 v1.1 的某个转推查询端点，迁移到 X API v2 后可继续使用相同的身份验证方法。 

<div id="differences">
  #### 差异
</div>

**端点 URL 和 HTTP 方法**

- Standard v1.1 端点：
  - https://api.x.com/1.1/statuses/retweet/:id.json
    （转推一条帖子。返回包含转推详情的原始帖子）
  - https://api.x.com/1.1/statuses/unretweet/:id.json
    （撤销转推。返回包含转推详情的原始帖子）
- X API v2 端点：
  - https://api.x.com/2/tweets/:id/retweets
    （转推指定帖子）
  - https://api.x.com/2/users/:id/retweets/:source\_tweet\_id
    （撤销对指定帖子的转推） 

**应用与项目要求**

使用 X API v2 端点时，请在请求认证时使用与某个[开发者应用](/zh/resources/fundamentals/developer-apps)关联的[项目](/zh/resources/fundamentals/projects)中的凭据。所有 X API v1.1 端点均可使用独立应用或与项目关联的应用的凭据。

**请求参数**

以下 Standard v1.1 请求接受两个查询参数（user\_id 或 screen\_name）。X API v2 仅接受数值型用户 ID，且必须作为端点路径的一部分传递。

| Standard v1.1 | X API v2 |
| :--- | :--- |
| **id** | **id** |
| **includes\_entities** | 无对应项 |

请注意，Standard v1.1 的参数通过查询参数传递；而 X API v2 的参数在 POST 端点中通过请求体参数传递，在 DELETE 端点中通过路径参数传递。

此外，使用 Standard v1.1 端点时，无需提供执行转推的用户 ID，因为通过 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 传递的[访问令牌](/zh/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)可用于推断发起或撤销转推的用户身份。
