<div id="overview">
  ## 概览
</div>

媒体对象表示一张照片、一个视频或一张动图 GIF。媒体对象被 X API 中的许多端点使用，并可出现在帖子、私信、用户资料、广告素材等处。每个媒体对象可能具有多个显示或播放变体，分辨率或格式各不相同。
---MDX_CONTENTEND---

<div id="media-types-size-restrictions">
  ## 媒体类型与大小限制
</div>

通过 API 上传的大小限制

- 图片：5 MB
- GIF：15 MB
- 视频：512 MB（使用 media_category=amplify 时）

<div id="creation">
  ## 创建
</div>

诸如推文（Tweets）、私信（Direct Messages）、用户头像、托管的广告卡片等对象可以包含一个或多个媒体对象。这些顶层对象统称为实体。相关的实体创建 API（例如 POST statuses/update）可通过唯一的 media_id 传入一个或多个媒体对象。

包含媒体对象的实体可按以下步骤创建：

1. 先上传媒体文件：推荐使用分块上传（images/GIF/video），或使用较早的简单上传（仅支持 images）。
2. 从步骤 1 获取 media_id。如果该实体允许传入多个 media_id 参数，可针对不同媒体重复此步骤。
3. 调用相应的端点创建实体，并包含 media_id 及其他必需参数。例如，使用 POST statuses/update 端点将 media_id 附加到一条推文。
