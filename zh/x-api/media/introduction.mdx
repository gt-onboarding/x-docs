媒体对象表示一张照片、一个视频或一段动画 GIF。媒体对象被 X API 中的许多端点使用，并可能出现在帖子、私信、用户档案、广告素材等位置。每个媒体对象可能具有多个用于显示或播放的变体，分辨率或格式各不相同。

<div id="media-types-size-restrictions">
  ## 媒体类型与大小限制
</div>

通过 API 上传的大小限制

- **图片**：`5 MB`
- **GIF**：`15 MB`
- **视频**：`512 MB`（使用 `media_category=amplify_video` 时）

<div id="creation">
  ## 创建
</div>

帖子、私信、用户头像、托管的广告卡片等对象都可以包含一个或多个媒体对象。这些顶级对象统称为实体。相关的实体创建 API（如 [`POST /2/tweets`](/zh/x-api/posts/creation-of-a-post)）可以通过唯一的 `media_id` 传入一个或多个媒体对象。

包含媒体对象的实体可按以下步骤创建：

1. 使用推荐的[分块](/zh/x-api-/media/quickstart/media-upload-chunked)上传（图片/GIF/视频），或较早的[简单](/zh/x-api/media-upload/media-upload)上传（仅图片）来上传媒体文件。
2. 从步骤 1 获取一个 `media_id`。如果该实体允许传入多个 `media_id` 参数，则此步骤可针对不同媒体重复进行。
3. 调用相应的端点创建实体，包含 `media_id` 和其他必需参数。例如，使用 [`POST /2/tweets`](/zh/x-api/posts/creation-of-a-post) 端点将 `media_id` 附加到帖子。
