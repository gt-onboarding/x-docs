本指南将帮助你使用 X API v2 的媒体上传端点发出首个媒体上传请求。

在本指南中，我们使用分块的 `POST /2/media/upload` 端点来上传视频。与单张图片上传相比，视频上传需要采用不同的工作流程。

对于视频或分块上传，你必须：

1. 使用 `INIT` 命令初始化上传
2. 使用 `APPEND` 命令上传每个数据分块
3. 使用 `FINALIZE` 命令完成上传

<Note>
  **注意：** 请参阅[该示例代码](https://github.com/xdevplatform/large-video-upload-python)，查看一个用 Python 编写的示例。
</Note>

<div id="authentication">
  ## 身份验证
</div>

要运行本指南中的示例，您需要使用 [OAuth 2](/zh/resources/fundamentals/authentication/oauth-2-0/overview) 进行身份验证。

您可以在开发者门户[项目与应用](https://developer.x.com/en/portal/projects-and-apps)的仪表板中，在所属项目的应用内找到您的 `CONSUMER_KEY`、`CONSUMER_SECRET`、`ACCESS_KEY`、`ACCESS_TOKEN`。

<div id="step-1-post-mediaupload-init">
  ## 步骤 1：POST media/upload（INIT）
</div>

`INIT` 命令用于初始化文件上传会话。它会返回一个 `media_id`，后续所有请求都应使用该值。在 `INIT` 命令成功返回后，下一步执行 `APPEND` 命令。

有关媒体文件的限制和要求，请参阅[最佳实践](/zh/x-api/media/quickstart/best-practices)。

**示例请求**

```bash
curl --location 'https://api.x.com/2/media/upload' \
    --header 'Authorization: Bearer <YOUR_ACCESS_TOKEN>' \
    --header 'Content-Type: multipart/form-data' \
    --form 'command=INIT' \
    --form 'media_type=video/mp4' \
    --form 'total_bytes=1024' \
    --form 'media_category=amplify_video'
```

<Note>
  **注意：** 发起原始 HTTP 请求时，请求体应使用 `multipart/form-data` 或 `application/x-www-form-urlencoded` 的 POST 格式。
</Note>

**示例响应**

```json
{
    "data":
    {
        "id":"1880028106020515840",
        "media_key":"13_1880028106020515840",
        "expires_after_secs":1295999
    }
}
```

响应提供媒体标识符 `id`（字符串）和 `media_key`（字符串）。必须在 `expires_after_secs` 指定的秒数内完成整个文件的上传。

<div id="step-2-post-mediaupload-append">
  ## 步骤 2：POST media/upload（APPEND）
</div>

`APPEND` 命令用于上传媒体文件的一个分片（连续字节范围）。例如，3 MB 的文件可以拆分为 3 个 1 MB 的分片，并通过 3 次 `APPEND` 请求上传。整个文件上传完成后，下一步调用 `FINALIZE` 命令。

<Info>
  将媒体文件按小分片上传的优势包括：

  - 在低带宽网络条件下提高可靠性和成功率
  - 上传可暂停并恢复
  - 可对单个分片进行重试
  - 可根据网络状况调整分片大小，例如在蜂窝网络环境下
</Info>

**示例请求**

```bash
curl --location 'https://api.x.com/2/media/upload' \
    --header 'Authorization: Bearer <YOUR_ACCESS_TOKEN>' \
    --header 'Content-Type: multipart/form-data' \
    --form 'command=APPEND' \
    --form 'media_id=1880028106020515840' \
    --form 'segment_index=0' \
    --form 'media=@/path/to/your/media/file.mp4'
```

`segment_index` 是文件分片的顺序索引，取值范围为 0–999（含）。第一个分片索引为 0，第二个为 1，依此类推。持续上传分片，直到全部分片上传完成。

<Note>
  **注意：** 进行原始 HTTP 请求时，需使用 `multipart/form-data` 的 POST 格式。
</Note>

<Note>
  **注意：** 成功上传会返回 2XX 状态码，且响应体为空。
</Note>

<div id="step-3-post-mediaupload-finalize">
  ## 步骤 3：POST media/upload（FINALIZE）
</div>

在使用 `APPEND` 命令上传完整个媒体文件后，应调用 `FINALIZE` 命令。仅当（且仅当）`FINALIZE` 命令的响应包含 `processing_info` 字段时，才可能需要使用 `STATUS` 命令，并在其返回成功后再继续创建帖子。

**示例请求**

```bash
curl --location --request POST 'https://api.x.com/2/media/upload' \
    --header 'Authorization: Bearer <YOUR_ACCESS_TOKEN>' \
    --header 'Content-Type: multipart/form-data' \
    --form 'command=FINALIZE' \
    --form 'media_id=1880028106020515840'
```

<Note>
  **注意：** 当发起原始 HTTP 请求时，请求应为 `multipart/form-data` 或 `application/x-www-form-urlencoded` 的 POST 格式。
</Note>

**示例响应**

```json
{
    "data": {
        "id": "1880028106020515840",
        "media_key": "13_1880028106020515840",
        "size": 1024,
        "expires_after_secs": 86400,
        "processing_info": {
            "state": "pending",
            "check_after_secs": 1
        }
    }
}
```

响应在 `media_id`（64 位整数）和 `media_id_string`（字符串）字段中提供了媒体标识符。对于 JavaScript 以及其他无法准确表示长整型的语言，请使用 API 响应中提供的 `media_id_string`。

返回的 `media_id` 仅在 `expires_after_secs` 秒内有效。在该时间段之后尝试在其他 API 调用中使用 `media_id` 将导致错误请求（HTTP 4xx）响应。

如果响应包含 `processing_info` 字段，则使用 `STATUS` 命令轮询 `FINALIZE` 操作的状态。异步 finalize 适用于需要更长处理时间的媒体。在未来，所有视频和动画 GIF 的处理将仅支持异步 finalize。若上传会话在使用 `media_category` 参数[初始化](###INIT)，且媒体类型为视频或动画 GIF，则会启用此行为。

<Note>
  **注意：**
  如果响应中未返回 `processing_info` 字段，则 `media_id` 可直接用于其他 API 端点。
</Note>

<div id="step-4-get-mediaupload-status">
  ## **步骤 4：GET media/upload（STATUS）**
</div>

`STATUS` 命令用于定期轮询媒体处理进度。当 `STATUS` 命令的响应为 `succeeded` 时，可继续下一步，通常是使用 `media_id` 创建帖子。

<Note>
  **注意：**
  仅当且仅当 `FINALIZE` 命令或先前的 `STATUS` 命令的响应包含 `processing_info` 字段时，才应使用 `STATUS` 命令。
</Note>

<div id="example-request">
  ## **示例请求**
</div>

```bash
curl --location --request GET 'https://api.x.com/2/media/upload?command=STATUS&media_id=1880028106020515840' \
    --header 'Authorization: Bearer <YOUR_ACCESS_TOKEN>'
```

<div id="example-response">
  ## **示例响应**
</div>

```json
{
    "data":{
        "id":"1880028106020515840",
        "media_key":"13_1880028106020515840",
        "processing_info":{
            "state":"uploading" // 状态流转为 pending -> in_progress -> [failed | succeeded]
        }
    }
}
```

响应正文包含 `processing_info` 字段，该字段提供媒体处理操作的当前状态信息。它包含一个 `state` 字段，其状态流转为：`pending` -> `in_progress` -> \[`failed` | `succeeded`]。在 `state` 字段变为 `succeeded` 之前，不能使用 `media_id` 创建帖子或其他实体。

<div id="step-5-post-tweet-with-media">
  ## 步骤 5：发布含媒体的帖子
</div>

使用 `POST /2/tweets` 端点代表已认证用户创建帖子，并在请求中包含 `media_id`。

**示例请求**

```bash
curl --location 'https://api.x.com/2/tweets' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer <YOUR_ACCESS_TOKEN>'
--data '{
    "text": "I made a post with media!",
     "media": {
       "media_ids": [
            "1880028106020515840"
        ]
    }
}'

```

**示例响应**

```bash
{
    "data": {
        "edit_history_tweet_ids": [
            "1880028106020515840"
        ],
        "id": "1880028106020515840",
        "text": "I made a post with media! https://t.co/DqNyXX"
    }
}
```
