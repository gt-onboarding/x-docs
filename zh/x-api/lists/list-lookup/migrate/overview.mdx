<div id="comparing-x-apis-list-lookup-endpoints">
  ## 比较 X API 的列表查找端点
</div>

v2 列表查找端点组将取代标准 v1.1 的 [GET lists/show](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-show) 和 [GET lists/ownership](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-ownerships) 端点。如果你的代码、应用或工具使用了这些列表查找端点中的任一版本，并计划迁移到更新的 X API v2 端点，那么本系列指南适合你。

下表比较了标准 v1.1 与 X API v2 的列表端点：

**按 ID 查找列表**

|     |     |     |
| :--- | :--- | :--- |
| 描述 | 标准 v1.1 | X API v2 |
| 支持的 HTTP 方法 | `GET` | `GET` |
| 主机域名 | `https://api.x.com` | `https://api.x.com` |
| 端点路径 | `/1.1/lists/show.json` | `/2/lists/:id` |
| [身份验证](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文<br /><br />仅限应用 | OAuth 1.0a 用户上下文<br /><br />OAuth 2.0 授权码（含 PKCE）<br /><br />仅限应用 |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 使用 OAuth 1.0a：每 15 分钟 75 次请求<br /><br />使用 OAuth 2.0：每 15 分钟 75 次请求<br /><br />仅限应用：每 15 分钟 75 次请求 | 使用 OAuth 1.0a：每 15 分钟 75 次请求<br /><br />使用 OAuth 2.0：每 15 分钟 75 次请求<br /><br />仅限应用：每 15 分钟 75 次请求 |

**用户拥有的列表查找**

|     |     |     |
| :--- | :--- | :--- |
| 描述 | 标准 v1.1 | X API v2 |
| 支持的 HTTP 方法 | `GET` | `GET` |
| 主机域名 | `https://api.x.com` | `https://api.x.com` |
| 端点路径 | `/1.1/lists/ownerships.json` | `/2/users/:id/owned_lists` |
| [身份验证](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文<br /><br />仅限应用 | OAuth 1.0a 用户上下文<br /><br />OAuth 2.0 授权码（含 PKCE）<br /><br />仅限应用 |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 使用 OAuth 1.0a：每 15 分钟 15 次请求<br /><br />仅限应用：每 15 分钟 15 次请求 | 使用 OAuth 1.0a：每 15 分钟 15 次请求<br /><br />使用 OAuth 2.0：每 15 分钟 15 次请求<br /><br />仅限应用：每 15 分钟 15 次请求 |

要访问 X API v2 端点，你必须[注册开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)。进行身份验证时，必须使用来自[开发者应用](/zh/resources/fundamentals/developer-apps)的密钥和令牌，该应用位于一个[项目](/zh/resources/fundamentals/projects)中。

在我们的[入门](/zh/x-api/getting-started/getting-access)页面了解更多关于获取 X API v2 端点访问权限的信息。
