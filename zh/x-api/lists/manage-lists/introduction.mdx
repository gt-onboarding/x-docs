[X 列表](https://help.x.com/en/using-twitter/twitter-lists) 让用户可以自定义、整理并优先查看时间线中的帖子。借助 Lists 端点，您可以构建解决方案，使人们能够根据偏好、兴趣、群组或主题来策划和管理帖子。

由于您代表用户使用管理型 List 端点发起请求，必须使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 或 [OAuth 2.0 授权码（含 PKCE）](zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 对这些端点进行身份验证，并使用与已授权您应用的用户关联的访问令牌。您可以通过 [三方 OAuth 流程](/zh/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）或 [含 PKCE 的授权码授权流程](/zh/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)（OAuth 2.0）生成这些令牌。

<div id="manage-lists">
  ### 管理列表
</div>

“管理列表”端点允许你代表经过身份验证的用户创建、删除和更新列表。对于这些端点，可用三种方法：POST、DELETE 和 PUT。POST 用于创建列表，DELETE 用于删除列表，PUT 用于更新列表的元数据。这三个端点的用户速率限制均为每 15 分钟 300 次请求。

请注意，每个账号最多可创建 1000 个列表。

**账号设置**

要访问这些端点，你需要：

- 已获批准的[开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)。
- 使用位于[项目](/zh/resources/fundamentals/projects)内的[开发者应用](/zh/resources/fundamentals/developer-apps)的密钥和令牌进行身份验证。

在我们的[入门指南](/zh/x-api/getting-started/getting-access)中了解有关获取 X API v2 端点访问权限的更多信息。
