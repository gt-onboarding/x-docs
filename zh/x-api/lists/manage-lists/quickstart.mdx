<div id="getting-started-with-the-manage-lists-endpoint-group">
  ## 快速开始：管理 Lists 端点组
</div>

本快速概览将帮助你使用 [Postman](/zh/tutorials/postman-getting-started) 向管理 List 端点发出你的第一个请求。

如果你想查看不同语言的示例代码，请访问我们的 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 仓库。

\*\*注意：\*\*在本示例中，我们将调用 _Create a List_ 端点，但你也可以将本快速入门中的方法应用于其他管理类请求。

<Note>
  **先决条件**

  要完成本指南，你需要一组用于对请求进行身份验证的 [keys and tokens](/zh/resources/fundamentals/authentication)。你可以按照以下步骤生成这些密钥和令牌：

  - [注册开发者账号](https://developer.x.com/en/apply-for-access) 并获得批准。
  - 在开发者门户中创建一个 [Project](/zh/resources/fundamentals/projects) 以及关联的 [developer App](/zh/resources/fundamentals/developer-apps)。
  - 前往你的 App 的“Keys and tokens”页面以生成所需凭据。请确保将所有凭据保存在安全位置。
</Note>

<div id="steps-to-build-a-manage-list-request">
  ### 构建管理 List 请求的步骤
</div>

**第一步：选择工具或库**

有多种工具、代码示例和库可用于向此端点发起请求，但为简化流程，这里我们将使用 Postman。

要将 X API v2 的 Postman 集合加载到你的环境中，请点击以下按钮：

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  将 X API v2 添加到 Postman
</Button>

在将 X API v2 集合加载到 Postman 后，导航到“List”文件夹，选择子文件夹“Manage List”，然后选择“Create a List”。

**第二步：为请求进行身份验证**

要正确向 X API 发起请求，你需要确认自己拥有相应权限。对于管理帖子相关的端点，你必须使用 [OAuth 1.0a User Context](/zh/resources/fundamentals/authentication) 或 [OAuth 2.0 Authorization Code with PKCE](zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 对请求进行身份验证。

在此示例中，我们将使用 OAuth 1.0a User Context。

你需要将密钥和令牌（具体为 API Key、API Secret Key、OAuth 1.0a 用户 Access Token 和 OAuth 1.0a 用户 Access Token Secret）添加到 Postman。你可以选择右上角名为“X API v2”的环境，并在“initial value”和“current value”字段中填入这些密钥和令牌（点击环境下拉菜单旁的“眼睛”图标即可编辑）。

如果配置正确，这些变量会自动填充到请求的 authorization 选项卡中。
 
**第三步：为新建的 List 指定名称**

使用此端点创建新 List 时，List 的名称是必填的请求体参数。你还可以选择提供描述，并指定该 List 是否为私密。

在 Postman 中，前往“Body”选项卡，将 List 的名称作为 name 参数的值输入。如果你希望添加描述，以与 name 相同的方式新增一个名为 description 的键，并将描述作为其值。将 List 设为私密遵循相同模式，但该参数仅接受 true 或 false。

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Parameter type** |
| `name` | 列表名称（必填） | body |
| description | 列表描述（可选） | body |
| private | true 或 false（可选） | body |

此时你应能在“Send”按钮旁看到类似的 URL：

`https://api.x.com/2/lists`

第四步：发送请求并查看响应

完成设置后，点击“Send”按钮，你将收到类似于以下的响应：

```
{
  "data": {
    "id": "1441162269824405510",
    "name": "New list created from Postman"
  }
}
```

如果返回的响应对象包含 id 和你的 List 名称，则表示已成功创建该 List。

要删除一个 List，请在已加载到 Postman 的 X API v2 集合中，前往“Lists”文件夹并选择“Delete a List”请求。此端点需要你要删除的 List 的 ID。在“Params”选项卡中，将该 List 的 ID 填入 id 列的值。

删除请求成功后，你将收到类似于以下的响应：

```
{
  "data": {
    "deleted": true
  }
}
```
