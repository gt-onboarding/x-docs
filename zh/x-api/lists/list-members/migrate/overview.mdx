<div id="comparing-x-apis-list-members-endpoints">
  ## 比较 X API 的列表成员端点
</div>

v2 列表成员端点组将取代标准 v1.1 的 [GET lists/members](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-members)、[GET lists/memberships](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-memberships)、[POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create) 和 [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy) 端点。若您的代码、应用或工具使用了上述任一列表成员端点，且正考虑迁移到更新的 X API v2 端点，那么本系列指南适合您。 

<div id="list-members-lookup">
  ### 列表成员查询
</div>

v2 列表成员查询端点将取代标准的 [GET lists/members](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-members)、[GET lists/memberships](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-memberships) 端点。

下表比较了标准 v1.1 与 X API v2 的列表端点：

**列表成员查询**

|     |     |     |
| :--- | :--- | :--- |
| 描述 | 标准 v1.1 | X API v2 |
| 支持的 HTTP 方法 | `GET` | `GET` |
| 主机域名 | `https://api.x.com` | `https://api.x.com` |
| 端点路径 | `/1.1/lists/members.json` | `/2/lists/:id/members` |
| [身份验证](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文<br /><br />仅应用 | OAuth 1.0a 用户上下文<br /><br />OAuth 2.0 含 PKCE 的授权码模式<br /><br />仅应用 |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 使用 OAuth 1.0a：每 15 分钟 900 次请求<br /><br />仅应用：每 15 分钟 75 次请求 | 使用 OAuth 1.0a：每 15 分钟 900 次请求<br /><br />使用 OAuth 2.0：每 15 分钟 900 次请求<br /><br />仅应用：每 15 分钟 900 次请求 |

**列表成员关系查询**

|     |     |     |
| :--- | :--- | :--- |
| 描述 | 标准 v1.1 | X API v2 |
| 支持的 HTTP 方法 | `GET` | `GET` |
| 主机域名 | `https://api.x.com` | `https://api.x.com` |
| 端点路径 | `/1.1/lists/memberships.json` | `/2/users/:id/list_memberships` |
| [身份验证](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文<br /><br />仅应用 | OAuth 1.0a 用户上下文<br /><br />OAuth 2.0 含 PKCE 的授权码模式<br /><br />仅应用 |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 使用 OAuth 1.0a：每 15 分钟 75 次请求<br /><br />仅应用：每 15 分钟 75 次请求 | 使用 OAuth 1.0a：每 15 分钟 75 次请求<br /><br />使用 OAuth 2.0：每 15 分钟 75 次请求<br /><br />仅应用：每 15 分钟 75 次请求 |

<div id="manage-list-members">
  ### 管理列表成员
</div>

用于管理列表成员的 v2 端点将取代标准的 [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create) 和 [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy) 端点。

下表对比了标准 v1.1 与 X API v2 的列表端点：

**添加成员**

| 描述 | 标准 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | POST | POST |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/lists/members/create.json | /2/lists/:id/members |
| [身份验证](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文 | OAuth 1.0a 用户上下文<br /><br />OAuth 2.0 授权码（含 PKCE） |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 无 | 每 15 分钟 300 次请求（每用户） |

**移除成员**

| 描述 | 标准 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 支持的 HTTP 方法 | POST | DELETE |
| 主机域名 | https://api.x.com | https://api.x.com |
| 端点路径 | /1.1/lists/members/destroy.json | /2/lists/:id/:user\_id |
| [身份验证](/zh/resources/fundamentals/authentication) | OAuth 1.0a 用户上下文 | OAuth 1.0a 用户上下文<br /><br />OAuth 2.0 授权码（含 PKCE） |
| 默认请求[速率限制](/zh/resources/fundamentals/rate-limits) | 无 | 每 15 分钟 300 次请求（每用户） |

要访问 X API v2 端点，您必须[注册开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)。进行身份验证时，必须使用位于[项目](/zh/resources/fundamentals/projects)中的[开发者应用](/zh/resources/fundamentals/developer-apps)所提供的密钥和令牌。

请在我们的[入门](/zh/x-api/getting-started/getting-access)页面了解更多关于获取 X API v2 端点访问权限的信息。
