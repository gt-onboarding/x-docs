<div id="manage-list-members-standard-v11-compared-to-x-api-v2">
  ### 管理列表成员：标准版 v1.1 与 X API v2 对比
</div>

如果您一直在使用标准版 v1.1 的 [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create) 和 [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy) 端点，本指南旨在帮助您理解标准版 v1.1 与 X API v2 在管理列表成员端点上的相同点与差异。

- **相同点**
  - 身份验证
- **差异**
  - 端点 URL
  - 应用与项目要求
  - HTTP 方法
  - 速率限制
  - 请求参数

<div id="similarities">
  #### 相似之处
</div>

**身份验证**

两个端点版本均支持 [OAuth 1.0a 用户上下文](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication)。因此，如果你之前使用的是标准 v1.1 的某个“管理列表成员”端点，那么在迁移到 X API v2 后，仍可继续使用相同的身份验证方法。

<div id="differences">
  #### 差异
</div>

**端点 URL**

- 标准 v1.1 端点：
  - POST https://api.x.com/1.1/lists/members/create.json
    （将成员添加到指定列表）
  - POST https://api.x.com/1.1/lists/members/destroy.json
    （从指定列表移除成员）
- X API v2 端点：
  - POST https://api.x.com/2/lists/:id/members
    （将成员添加到指定列表）

  - DELETE https://api.x.com/2/lists/:id/members/:user\_id
    （从指定列表移除成员）

**速率限制**

| **标准 v1.1** | **X API v2** |
| :--- | :--- |
| /1.1/lists/members/create.json<br /><br />无 | /2/lists/:id/members<br /><br />每 15 分钟 300 次请求（OAuth 1.0a 用户上下文）<br /><br />每 15 分钟 300 次请求（OAuth 2.0 授权码含 PKCE） |
| /1.1/lists/members/destroy.json<br /><br />无 | /2/lists/:id/members/:user\_id<br /><br />每 15 分钟 300 次请求（OAuth 1.0a 用户上下文）<br /><br />每 15 分钟 300 次请求（OAuth 2.0 授权码含 PKCE） |

**应用与项目要求**

在对请求进行身份验证时，X API v2 端点要求使用与某个[项目](/zh/resources/fundamentals/projects)关联的[开发者应用](/zh/resources/fundamentals/developer-apps)的凭据。所有 X API v1.1 端点可以使用独立应用或与项目关联应用的凭据。

**请求参数**

以下标准 v1.1 请求参数在 X API v2 中有对应项：

| **标准 v1.1** | **X API v2** |
| :--- | :--- |
| list\_id | id  |
| slug | 无对应项 |
| screen\_name | 无对应项 |
| owner\_screen\_name | 无对应项 |
| owner\_id | 无对应项 |
