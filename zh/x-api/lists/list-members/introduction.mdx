<div id="list-members-lookup">
  ### 列表成员查找
</div>

“成员查找”分组提供两个端点。您可以获取指定列表的成员详情，并查看某个用户加入了哪些列表。这些端点可用于基于成员信息帮助用户策划和组织新的列表。

查询成员详情的速率限制为每 15 分钟 900 次请求；查询用户成员关系的速率限制为每 15 分钟 75 次请求。

您可以使用 [OAuth 1.0a User Context](/zh/resources/fundamentals/authentication)、[App only](https://developer.x.com/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) 或 [OAuth 2.0 Authorization Code with PKCE](zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 来对该端点的请求进行身份验证。 
 

<div id="manage-list-members">
  ### 管理列表成员
</div>

“管理列表成员”端点允许你代表已认证用户向列表添加或移除成员。对于这些端点，可用的 HTTP 方法有两种：POST 和 DELETE。POST 用于向列表添加成员，DELETE 用于从列表移除成员。两个端点的用户速率限制均为每 15 分钟 300 次请求。

请注意，列表的成员数最多为 5,000。

由于你将通过这些端点代表用户发起请求，必须使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 或 [OAuth 2.0 Authorization Code with PKCE](zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)，并使用已授权你的应用的该用户所对应的用户访问令牌。你可以通过 [三方（3-legged）OAuth 流程](/zh/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）或 [Authorization Code with PKCE grant flow](/zh/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)（OAuth 2.0）生成这些令牌。

<Note>
  **账户设置**

  要访问这些端点，你需要：

  - 一个已获批准的 [开发者账号](https://developer.x.com/en/portal/petition/essential/basic-info)。
  - 使用位于某个 [项目](/zh/resources/fundamentals/projects) 内的 [开发者应用](/zh/resources/fundamentals/developer-apps) 的密钥与令牌进行认证。

  在我们的[入门指南](/zh/x-api/getting-started/getting-access)中了解如何获取 X API v2 端点的访问权限。
</Note>
