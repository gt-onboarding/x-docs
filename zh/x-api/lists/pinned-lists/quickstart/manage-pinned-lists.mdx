<div id="steps-to-build-a-manage-pinned-list-request">
  ### 构建管理置顶列表请求的步骤
</div>

**第一步：在 Postman 中选择 List 端点集合**

在将 X API v2 集合加载到 Postman 后，导航到“List”文件夹，选择其中的“Pinned Lists”文件夹，然后选择“Pin a List”。

**第二步：指定要置顶的列表**

管理置顶列表的端点需要两个 ID：一个是用户 ID（用于将列表置顶的已认证用户），另一个是目标列表 ID（要置顶的列表）。用户 ID 必须与用于认证的用户 ID 相匹配，这意味着在为请求执行认证时，必须传递与该用户 ID 关联的访问令牌。本例中，你可以指定你自己的用户 ID。你可以通过以下两种方式找到你的 ID：

1. 使用按用户名的 [users lookup](/zh/x-api/users/lookup/introduction) 端点，传入用户名即可获取 id 字段。
2. 查看你的访问令牌，其中的数字部分就是你的用户 ID。

目标列表 ID 可以是任何有效的列表。在 Postman 中，转到“Params”选项卡，将用户 ID 填入 id 路径变量的“Value”列。然后转到“Body”选项卡，将你希望置顶的列表的 ID 填入 list\_id 参数的值。请确保任何 ID 前后都不要包含空格。

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Parameter type** |
| `id` | 用户 ID | path |
| list\_id | 要置顶的目标列表 ID | body |

现在你应当能在“Send”按钮旁看到类似的 URL：

```
      https://api.x.com/2/users/2244994945/pinned_lists
```

第三步：发起请求并查看响应

当一切设置完成后，点击“Send”按钮，你会收到类似于以下示例的响应：

```
{
  "data": {
    "pinned": true
  }
}
```

如果返回的响应对象中键 pinned 的值为 true，则表示你已成功置顶目标列表。

要取消置顶列表，请在加载到 Postman 的 X API v2 集合中的“Lists”文件夹里选择“Unpin List”请求。该端点需要已认证的用户 ID 以及要取消置顶的列表 ID。在“Params”选项卡中，将用户 ID 填入 id 列的值，将要取消置顶的列表 ID 填入 list\_id 列的值。

当删除请求成功时，你将收到类似于以下示例的响应：

```
{
  "data": {
    "pinned": false
  }
}
```
