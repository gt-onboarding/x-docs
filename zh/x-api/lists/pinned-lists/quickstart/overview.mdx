<div id="getting-started-with-the-pinned-list-endpoint-group">
  ## 开始使用置顶列表端点组
</div>

本快速概览将帮助您使用 [Postman](/zh/tutorials/postman-getting-started) 向置顶列表端点发出首个请求。

如果您想查看不同语言的示例代码，请访问我们的 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 代码库。

<Note>
  ### 先决条件

  要完成本指南，您需要一组用于请求身份验证的 [keys and tokens](/zh/resources/fundamentals/authentication)。您可以按以下步骤生成这些密钥和令牌：

  - [注册开发者账号](https://developer.x.com/en/apply-for-access) 并通过审核。
  - 在开发者门户中创建一个 [Project](/zh/resources/fundamentals/projects) 以及关联的 [developer App](/zh/resources/fundamentals/developer-apps)。
  - 前往您的 App 的 “Keys and tokens” 页面生成所需凭据。请务必将所有凭据安全保存。
</Note>

加载 X API v2 的 Postman 集合

要将 X API v2 的 Postman 集合加载到您的工作区，请点击以下按钮：

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  将 X API v2 添加到 Postman
</Button>

 

<div id="authenticate-your-request">
  #### 对请求进行身份验证
</div>

要成功调用 **lookup** 端点，您可以使用 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication)、[OAuth 2.0 仅应用](/zh/resources/fundamentals/authentication#oauth-2-0)或 [OAuth 2.0（含 PKCE）的授权码流程](zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)。但对于 **manage** 端点，只能使用 OAuth 1.0a 用户上下文或 OAuth 2.0（含 PKCE）的授权码流程进行身份验证。

在使用 Postman 时，只要您已正确配置环境，默认的身份验证密钥和令牌会自动填充到您的请求中。

您可以在 Postman 右上角选择名为“X API v2”的环境，并将您的密钥和令牌添加到 “initial value” 和 “current value” 字段中（点击环境下拉菜单旁的眼睛图标）。这些密钥包括：

- API Key（亦称 Consumer Key）
- API Secret Key（亦称 Consumer Secret）
- OAuth 1.0a 用户 Access Token
- OAuth 1.0a 用户 Access Token Secret
- OAuth 2.0 应用 Access Token
- OAuth 2.0 客户端 Key（仅当您在应用的设置中配置了 OAuth 2.0 用户身份验证设置时可用）
- OAuth 2.0 客户端 Secret（仅当您在应用的设置中配置了 OAuth 2.0 用户身份验证设置时可用）
