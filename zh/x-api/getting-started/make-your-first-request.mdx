本指南将引导你完成发出首次请求的步骤。一旦你注册了 X 账户，这将是一个非常有用的参考。

如果你想使用代码示例、更技术性的指南，或 Postman 等图形化工具，请参考以下内容来完成你的首次请求：

- [快速开始：向帖子端点发出首次请求](/zh/x-api/posts/manage-tweets/quickstart)
- [Postman 入门](/zh/tutorials/postman-getting-started) - 最适合初学者
- [X API v2 代码示例](https://github.com/xdevplatform/Twitter-API-v2-sample-code)
- [帖子查询 API 参考](/zh/x-api/posts/lookup/introduction)

本指南假设你已获取 [API key 和 secret](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret)、[用户 Access Token 和 Secret](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)、[应用访问令牌（App Access Token）](/zh/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)，并已将它们安全存储。你可以按照[获取 X API 访问权限](/zh/x-api/getting-started/getting-access)指南中的步骤了解如何完成这些操作。

<div id="step-1-identify-which-endpoint-you-would-like-to-use">
  ### 步骤 1. 确定你要使用的端点
</div>

X API 允许你通过代码执行多种操作，这些操作也可以在 X 的网站或移动应用中完成。

我们在 [API 参考索引](/zh/x-api/introduction) 中提供了所有可用端点的完整列表，但为简化起见，建议优先从以下选项之一开始：

- [管理帖子 > 发布推文](/zh/x-api/posts/manage-tweets/introduction)
- [搜索帖子](/zh/x-api/posts/search/introduction)
- [用户查询](/zh/x-api/users/lookup/introduction)

<div id="step-2-choose-a-tool-to-make-your-request">
  ### 步骤 2. 选择用于发起请求的工具
</div>

有些请求较为简单，但也有一些构建起来较为棘手。因此，我们出色的开发者社区开发了工具，以屏蔽一部分复杂性。

以下是一些推荐的工具及其使用方法的详细说明：

<div id="postman">
  #### Postman
</div>

Postman 是一款用于向 REST 端点发送请求的可视化工具。我们围绕 Postman 提供了一些优质资料，帮助你上手并探索通过 X API 提供的各类端点。

建议先阅读我们的“[使用 Postman 入门](/zh/tutorials/postman-getting-started)”教程，了解如何添加密钥与令牌，并发出你的第一个请求。

我们还为各个 X API v2 端点制作了快速入门指南，其中大多使用 Postman。你可以在相应端点的章节中找到这些指南，以下是其中的一些链接：

- 快速入门：发布推文
- 快速入门：搜索帖子
- 快速入门：查询用户

请注意，无法使用 Postman 向流式端点发送请求。请参阅[过滤流](/zh/x-api/posts/filtered-stream/quickstart)或[1% 抽样流](/zh/x-api/posts/sample-stream)的快速入门指南，了解如何使用这些端点。

如果你更偏好更为简洁的图形化工具，也可以考虑使用 [Insomnia](https://insomnia.rest)。

<div id="sample-code">
  #### 示例代码
</div>

如果你想用一些简洁的代码来发起请求，我们为每个 X API v2 端点都提供了多种语言的示例代码。

你可以在我们的 GitHub 仓库 [X-API-v2-sample-code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) 中找到这些示例代码。该仓库还包含 README 文件，你可以借此了解如何设置凭据以正确配合请求使用。

例如，下面是用户查询端点的一个 cURL 示例。你只需将 $ACCESS\_TOKEN 和 $USERNAME 替换为你的 [App Access Token](/zh/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) 和 X 用户名。然后将这段代码复制到命令行工具中并按下 Enter（或 Return）键。

```bash
curl "https://api.x.com/2/users/by/username/$USERNAME" -H "Authorization: Bearer $ACCESS_TOKEN"
```

<div id="libraries">
  #### 库
</div>

活跃的 X 开发者社区还用多种编程语言构建了库，可用于向 X API 发起请求。

我们创建了“[工具与库](/zh/x-api/tools-and-libraries/overview)”页面，列出了我们所知的全部社区库。每个库都应提供一个 README 文件，可用于了解如何在本地设置该代码库并发出你的首个请求。

> **请注意：** 如果你遇到任何问题，请查阅你所请求端点的开发者文档、我们的[支持页面](https://developer.x.com/en/support)，或在论坛向社区寻求帮助。我们会协助你完成一次成功的请求！

<div id="step-3-review-the-response">
  ### 步骤 3. 检查响应
</div>

成功发起请求后，你会收到一个包含与该请求相关元数据的负载。

如果你使用的是 GET HTTP 方法的端点，你会以 JSON 格式收到与你请求的资源（帖子、用户、列表、Space 等）相关的元数据。查看返回的各字段，并尝试将所请求的信息对应到 X 上的内容。

如果你使用的是 POST、PUT 或 DELETE HTTP 方法的端点，则表示你在 X 上执行了某项操作。前往 X.com 或移动端应用，查看是否能找到该操作的结果。

<div id="step-4-adjust-your-request-using-parameters">
  ### 第 4 步。使用参数调整请求
</div>

每个端点都提供不同的参数集，可用于调整请求。例如，在使用 GET 端点时，可以通过 `fields` 和 `expansions` 参数请求更多元数据字段。你还可以在 [搜索帖子](/zh/x-api/posts/search/introduction)、[帖子计数](/zh/x-api/posts/counts/introduction) 和 [过滤流](/zh/x-api/posts/filtered-stream/introduction) 等端点中尝试多种过滤工具，将返回的数据精确到你感兴趣的帖子。

你可以在所使用端点的 API 参考中查找该端点的完整请求参数和字段列表；我们的集成指南和基础知识页面还提供了大量其他有用的集成信息。

你可以通过我们的[重要资源](/zh/x-api/getting-started/important-resources)指南进一步了解我们为你提供的各类学习资料。
