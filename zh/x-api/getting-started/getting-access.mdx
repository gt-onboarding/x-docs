<div id="step-one-sign-up-for-a-developer-account">
  ### 第一步：注册开发者账号
</div>

注册开发者账号既快捷又简单！只需点击下方按钮并回答几个问题，即可通过[基础访问](https://developer.x.com/en/portal/products/basic)开始探索并基于 X API v2 开发。

接下来，您将在入门流程中创建一个项目与其关联的开发者应用，这将为您提供一组凭据，您将使用这些凭据对所有发往 API 的请求进行身份验证。

[注册](https://developer.x.com/en/portal/products/basic)

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### 步骤二：保存你的应用密钥与令牌并确保其安全
</div>

在你获得访问权限并创建了项目与应用后，你可以在开发者应用中找到或生成以下凭据：

- **[API Key 和 Secret](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret)：** 相当于你的应用的用户名和密码。你将使用它们来对需要 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 的请求进行身份验证，或生成其他令牌，例如用户 Access Token 或 App Access Token。

- **[Access Token 和 Secret](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)：** 通常，Access Token 代表你代其发起请求的用户。你通过开发者门户生成的令牌代表该应用的拥有者。你将使用它们来对需要 [OAuth 1.0a 用户上下文](/zh/resources/fundamentals/authentication) 的请求进行身份验证。如果你希望代表其他用户发起请求，需要使用三方 OAuth 流，让对方授权你。

- **[Client ID 和 Client Secret](/zh/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)：** 这些凭据用于通过 OAuth 2.0 获取用户 Access Token。与 OAuth 1.0a 类似，用户 Access Token 用于对访问私有用户账户信息或代表其他账户执行操作的请求进行身份验证，但可通过细粒度的权限范围更精细地控制客户端应用对用户的访问。

- **[仅应用 Access Token](/zh/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)：** 当你调用仅返回 X 上公开可用信息的端点时，使用此令牌。

由于这些密钥和令牌在未重新生成之前不会过期，我们建议在收到凭据后，将它们保存在安全位置，例如密码管理器。

> **请注意：你的密钥与令牌只会在[开发者门户](/zh/resources/fundamentals/developer-portal)中显示一次，因此在生成后务必立即将这些凭据存入你的密码管理系统。**
>
> 如果你丢失或遗忘了密钥与令牌，你需要重新生成它们，这将创建新的凭据并使旧的凭据失效。这意味着你必须更新任何基于先前凭据所做的集成。
>
> 了解更多我们的[身份验证最佳实践](/zh/resources/fundamentals/authentication#authentication-best-practices)。
