<div id="introduction">
  ### 介绍
</div>

X 开发者门户提供一套自助工具，供开发者管理对 X API 和 X 广告 API 的访问。

在门户中，你可以：

- 创建并管理你的 X [项目](/zh/resources/fundamentals/projects) 与 [应用](/zh/resources/fundamentals/developer-apps)（以及它们提供的身份验证密钥和令牌）。
- 管理你对 X API 标准 v1.1 与 v2 端点的访问级别与集成。
- 了解更多可用的端点与功能。

<div id="ready-to-get-access">
  ### 准备好获取访问权限了吗？
</div>

您可以通过注册账户开始使用 X API。

如果您需要更多功能或更高的[推文上限（帖子上限）](/zh/x-api/fundamentals/post-cap)，可在开发者门户购买 Basic 或 Pro。若您对 Enterprise 感兴趣，请在[此处](https://docs.google.com/forms/d/e/1FAIpQLScO3bczKWO2jFHyVZJUSEGfdyfFaqt2MvmOfl_aJp0KxMqtDA/viewform)提交申请。

  ## 开发者门户内容预览

<div id="onboarding">
  ### 入门引导
</div>

入门向导将引导您完成设置首个[项目](/zh/resources/fundamentals/projects)和[应用](/zh/resources/fundamentals/developer-apps)的流程。您需要创建项目与应用，以获取用于对您的 API 请求进行身份验证的凭据。如果这是您首次访问开发者账户，您将看到该向导。

在此过程中，您将获得一组身份验证密钥和令牌，您可在我们的[应用概览页面](/zh/resources/fundamentals/developer-apps#overview)了解更多信息。要进一步了解与 X API 进行身份验证所需内容，请参阅[身份验证部分](/zh/resources/fundamentals/authentication)。

**请注意：** 您需要将[密钥和令牌安全存储](/zh/resources/fundamentals/authentication/guides/authentication-best-practices)，以便日后访问。不重新生成，这些凭据将无法再次查看。

<div id="project-and-app-management">
  ### 项目与应用管理
</div>

开发者账户的主要职责之一是让你可以[管理你的项目与应用](/zh/resources/fundamentals/developer-apps#app-management)。开发者可在开发者门户的控制面板中创建和管理 X 的项目与应用。你可以在此找到你的 App ID，编辑应用的设置、权限和回调 URL，并生成或吊销密钥与令牌。

<div id="learn-more-about-whats-available-with-x-api-v2">
  ### 进一步了解 X API v2 的可用功能
</div>

开发者门户设有“产品”栏目，您可以在此了解 X API 的不同版本与访问级别。

X API v2 的产品栏目包含关于 Free、Basic、Pro 和 Enterprise 访问层级的重要信息。该页面提供项目级应用限制、推文上限（帖子上限）与费用的详细说明，以及各端点的速率限制与特殊属性。您还可以对比不同访问级别，并在可用时申请额外访问权限。

查看[所有 v2 访问级别](/zh/x-api/introduction#access-levels)。

<div id="team-management">
  ## 团队管理
</div>

<Note>
  **仅适用于 Enterprise v2 帐户。**
</Note>

**[在开发者门户查看“团队”页面](https://developer.twitter.com/en/portal/teams)。**

<div id="why-use-team-functionality">
  ### 为什么使用团队功能？
</div>

团队功能有助于在 X 开发者平台内协同管理和开发“项目与应用”。团队中通常由不同成员分别负责访问控制、结算与支付；通过该功能，你可以邀请这些成员参与并为你的项目做出贡献。

<div id="inviting-team-members">
  ### 邀请团队成员
</div>

要邀请他人加入团队，管理员可以通过对方的 X 账号（handle）发出邀请。对方会收到一封电子邮件，并可通过该邮件接受邀请。接受后，他们需要同意《开发者协议与政策》，随后即可访问主账号的团队页面。

**请注意：** 团队管理目前不会根据 [App](/zh/resources/fundamentals/developer-apps) 凭证来授予或限制 API 访问权限。无法在团队中共享 App 管理权限。非所有者无法编辑、创建或删除 App（密钥/令牌）。

<div id="team-dashboard">
  ### 团队仪表板
</div>

在团队仪表板的“成员”选项卡中，您可以查看所有成员及其角色。若您是团队管理员，您可以管理开发者的访问权限并编辑每位成员的角色。

管理员还可以访问团队仪表板中的“待处理”选项卡。在此处，管理员可以查看每个已发送邀请的详情并进行管理。

<div id="team-roles">
  ### 团队角色
</div>

**管理员角色**

- 可管理团队项目与应用
- 可管理所有应用环境
- 可选择/升级订阅
- 可更新账单/支付方式
- 可添加/移除团队成员
- 可编辑团队成员角色

**开发者角色**

- 可管理自己的项目与应用
- 对团队项目与应用具有只读权限
- 可离开团队
