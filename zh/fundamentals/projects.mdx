<div id="introduction">
  ## 简介
</div>

项目可根据你计划如何使用 X API 来组织你的工作，从而更高效地管理对 API 的访问并监控使用情况。每个项目可包含一个或多个[应用](/zh/resources/fundamentals/developer-apps)，具体取决于你的访问级别（本页后文会说明）。你将使用这些应用生成[身份验证](/zh/resources/fundamentals/authentication)凭据，例如 [API Key 和 Secret](/zh/resources/fundamentals/authentication#api-key-and-secret)、用户[访问令牌](/zh/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)以及[应用访问令牌](/zh/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)。

虽然你可以使用任一应用中的这些密钥和令牌访问 X API 或 X 广告 API（使用广告 API 必须[申请额外访问权限](/zh/x-ads-api/getting-started/step-by-step-guide)），但要调用 [X API v2](/zh/x-api/introduction) 的端点，必须使用与某个项目关联的应用所生成的密钥和令牌。

如果你拥有开发者账号，可以在开发者门户的 [Projects & Apps](https://developer.x.com/en/portal/projects-and-apps) 页面查看并管理你的项目。若尚无账号，请[注册](https://developer.x.com/en/portal/petition/essential/basic-info)。

<div id="projects-and-x-api-access-levels">
  ### 项目与 X API 访问级别
</div>

目前，在项目级别共有 4 个不同的访问层级：

- 免费
- 基础
- 专业
- 企业

要了解这些访问级别各自提供的功能，请访问 [关于 X API](/zh/x-api/getting-started/about-x-api) 页面。

在免费、基础或专业层级中，您一次只能拥有一个项目。

以下将为您说明一些项目层面的差异：

<div id="free">
  #### 免费
</div>

此访问层对所有已[注册](https://developer.twitter.com/en/portal/petition/essential/basic-info)开发者账户的用户开放。

该项目内的应用数量：1
[推文上限（帖子上限）](/zh/x-api/fundamentals/post-cap)：每月 1,500 条推文

认证方式：

- [OAuth 2.0（含 PKCE）](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
- 仅限应用

<div id="basic">
  #### Basic
</div>

此访问层级面向所有已通过开发者门户订阅 Basic 访问的用户。

该项目内的应用数量：2
[推文上限（帖子上限）](/zh/x-api/fundamentals/post-cap)：每月 10,000 条推文
认证方式：

- [OAuth 2.0（含 PKCE）](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
- [OAuth 1.0a](/zh/resources/fundamentals/authentication#oauth-1-0a-2)
- 仅限应用

<div id="pro">
  #### Pro
</div>

此访问层适用于通过开发者门户订阅 Pro 访问的任何人。

该项目内可创建的应用数量：3
[推文上限（帖子上限）](/zh/x-api/fundamentals/post-cap)：每月 100 万条推文

认证方式：

- [OAuth 2.0（含 PKCE）](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
- [OAuth 1.0a](/zh/resources/fundamentals/authentication#oauth-1-0a-2)
- 仅限应用

<div id="enterprise">
  #### Enterprise
</div>

此访问层级适用于已申请 Enterprise、通过我们团队审批，并在 X 客户经理协助下签署按月自动续约合同的用户。[点击申请](https://docs.google.com/forms/d/e/1FAIpQLScO3bczKWO2jFHyVZJUSEGfdyfFaqt2MvmOfl_aJp0KxMqtDA/viewform)。

该项目下的应用数量：3+
[推文上限](/zh/x-api/fundamentals/post-cap)：每月 5,000 万+ 条推文

身份验证方式：

- [OAuth 2.0（含 PKCE）](/zh/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
- [OAuth 1.0a](/zh/resources/fundamentals/authentication#oauth-1-0a-2)
- 仅应用

<div id="tweet-caps-and-projects">
  ### 推文上限与项目
</div>

推文上限适用于项目级别，在每个自然月内有效限制你可通过某些 X API v2 端点检索的推文数量。

了解更多[推文上限](/zh/x-api/fundamentals/post-cap)。

<div id="configuring-projects">
  ### 项目配置
</div>

<div id="creating-a-project">
  #### 创建项目
</div>

要创建项目，请在你的[仪表板](https://developer.twitter.com/en/portal/dashboard)或开发者门户中的[项目与应用](https://developer.twitter.com/en/portal/projects-and-apps)页面点击“New Project”。只有当你尚未创建任何项目时，才会看到此选项。系统会提示你填写项目名称、描述和使用场景。同时，你还需要创建一个新应用或关联一个现有的独立应用。

<div id="creating-or-connecting-an-app-for-your-project">
  #### 为你的项目创建或连接应用
</div>

如果你的项目尚未包含应用，你可以在控制台中点击项目名称以添加应用。随后，你可以创建一个新应用，或选择一个现有的独立应用并将其连接到你的项目。应用是生成本指南开头所列身份验证密钥和令牌的地方。

<div id="editing-a-project">
  #### 编辑项目
</div>

要编辑项目，请在开发者门户的仪表板或“项目与应用”页面中点击你的项目名称。随后你将看到项目详情，并可选择“编辑”进行修改。

<div id="standalone-apps">
  ### 独立应用
</div>

独立应用是指不隶属于“项目”结构的应用。与这些独立应用关联的身份验证凭据可用于成功调用 X 的 API（标准版 v1.1、高级版 v1.1、企业版）或 X 广告 API。除非将它们连接到某个项目，否则独立应用在尝试请求 X API v2 的端点时会失败。

如果你在 2020 年 8 月之前创建了应用，它们会显示在开发者门户 [Projects & Apps](https://developer.twitter.com/en/portal/projects-and-apps) 下的 “Standalone Apps” 部分。你的应用总数上限为 10 个，其中也包含已连接到你项目的应用。

如果你属于团队账户，你会在“独立应用”下看到你名下的应用。如果某位团队成员拥有一个属于某个项目的应用，你可以看到该应用的名称和所有者信息，但无法更改其设置、访问其密钥和令牌，或重新生成其密钥和令牌。若需变更其应用，请联系该应用的所有者。
