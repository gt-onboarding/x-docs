X API 处理海量数据。我们通过身份验证来确保这些数据对开发者和用户同样安全。身份验证有多种方式，概述如下。

大多数开发者无需直接处理身份验证的复杂性，因为客户端库会自动代为处理。

你可以在我们的[工具与库](/zh/resources/tools-and-libraries)页面找到可用的客户端库列表。

<div id="authentication-methods">
  ## 认证方式
</div>

<CardGroup cols={2}>
  <Card title="OAuth 1.0a 用户上下文" href="/zh/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret">
    OAuth 1.0a 允许获授权的 X 开发者应用访问私有账户信息，或代表某个 X 账户执行操作。

    <br />

    [**了解更多**](/zh/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret)
  </Card>

  <Card title="仅应用" href="/zh/resources/fundamentals/authentication/oauth-2-0/overview">
    仅应用访问令牌允许 X 开发者应用访问 X 上公开可用的信息。

    <br />

    [**了解更多**](/zh/resources/fundamentals/authentication/oauth-2-0/overview)
  </Card>

  <Card title="基本身份验证" href="/zh/resources/fundamentals/authentication/basic-auth">
    许多 X 的企业级 API 需要使用 HTTP 基本身份验证。

    <br />

    [**了解更多**](/zh/resources/fundamentals/authentication/basic-auth)
  </Card>

  <Card title="OAuth 2.0 授权码流程（含 PKCE）" href="/zh/resources/fundamentals/authentication/oauth-2-0/authorization-code">
    OAuth 2.0 用户上下文允许你代表其他账户进行认证，更精细地控制应用的权限范围，并在多设备间完成授权流程。

    <br />

    [**了解更多**](/zh/resources/fundamentals/authentication/oauth-2-0/authorization-code)
  </Card>
</CardGroup>

<Note>
  **注意：**
  你的应用的 API Key 与仅应用访问令牌，以及你的个人访问令牌与访问令牌密钥，可在[开发者门户](/zh/resources/fundamentals/developer-portal)中的 [X developer Apps](/zh/resources/fundamentals/developer-apps) 部分获取。

  **如果你需要代表其他用户发起请求**，请使用[三方 OAuth 流程](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)为该用户生成一套单独的访问令牌，并在使用 OAuth 1.0a 用户上下文或 OAuth 2.0 用户上下文发起请求时，携带该用户的令牌。
</Note>
