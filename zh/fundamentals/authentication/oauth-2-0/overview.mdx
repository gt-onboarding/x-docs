<div id="bearer-token-also-known-as-app-only">
  ### Bearer Token（也称为仅应用）
</div>

OAuth 2.0 Bearer Token 代表你的[开发者应用](/zh/resources/fundamentals/developer-apps)对请求进行身份验证。由于此方法特定于应用，不涉及任何用户。该方法通常适用于只需要对公开信息进行只读访问的开发者。 

此身份验证方法要求你在请求中携带 Bearer Token，你可以在开发者应用的“Keys and tokens”部分生成它。以下是一个带有模拟 Bearer Token 的请求示例：

```json
  curl "https://api.x.com/2/tweets?ids=1261326399320715264,1278347468690915330" \
  -H "Authorization: Bearer AAAAAAAAAAAAAAAAAAAAAFnz2wAAAAAAxTmQbp%2BIHDtAhTBbyNJon%2BA72K4%3DeIaigY0QBrv6Rp8KZQQLOTpo9ubw5Jt?WRE8avbi"
```

使用仅应用身份验证的 API 调用会在应用级按端点[受速率限制](/zh/resources/fundamentals/rate-limits)。

要使用此方法，你需要一个 Bearer Token。你可以通过[POST oauth2/token](/zh/resources/fundamentals/authentication/api-reference#post-oauth2-token)端点传入你的 API Key 和 Secret 来生成，或在[开发者门户](/zh/resources/fundamentals/developer-portal)中应用设置的“Keys and tokens”部分生成。

如果你想撤销 Bearer Token，可以使用[POST oauth2/invalidate\_token](/zh/resources/fundamentals/authentication/api-reference#post-oauth2-invalidate-token)端点，或在应用设置的“Keys and tokens”部分点击 Bearer Token 旁边的“revoke”。

<div id="oauth-20-authorization-code-flow-with-pkce">
  ### 使用 OAuth 2.0（含 PKCE）的授权码流程
</div>

使用 OAuth 2.0（含 PKCE）的授权码流程，可以代表其他用户进行身份验证，更加精细地控制应用的权限范围（scopes），并改进跨设备的授权体验。换言之，为 X 用户构建应用的开发者将能更好地控制其应用向用户请求的信息，只需向终端用户请求所需的数据和信息。

这一现代授权协议可为终端用户提供更为精简的应用授权同意流程，仅展示你向其请求的特定权限范围。这不仅能降低你的数据处理负担，也有助于提升终端用户的信任。
