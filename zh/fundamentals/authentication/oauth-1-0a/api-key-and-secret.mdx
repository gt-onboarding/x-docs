<div id="api-key-and-secret">
  ### API Key 和 Secret
</div>

API Key 和 Secret（也称为 Consumer Key 和 Secret）是访问 X API 所需的最基础凭据。这些凭据相当于你的 X 应用的用户名和密码，X API 会据此识别请求来自哪一个应用。

这些凭据可由[身份验证端点](/zh/resources/fundamentals/authentication/api-reference)用于生成其他凭据，例如[用户 Access Token 和 Secret](/zh/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret)以及[Bearer Token](/zh/resources/fundamentals/authentication/oauth-2-0/bearer-tokens)。当你需要使用 OAuth 1.0a 用户上下文身份验证时，也需要将这些凭据与 Access Token 及其他授权参数一起使用，以[授权请求](/zh/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request)。 

<div id="how-to-acquire-an-api-key-and-secret">
  #### 如何获取 API Key 和 Secret
</div>

要获取 X 的 API Key 和 Secret，请按以下步骤操作：

1. [注册 X 开发者帐号](https://developer.x.com/en/apply-for-access)
2. 在[开发者门户](/zh/resources/fundamentals/developer-portal)中创建一个 [X App](/zh/resources/fundamentals/developer-apps)。请注意，如果你希望使用 [X API v2](/zh/x-api/introduction)，必须将你的 X App 添加到一个[Project](/zh/resources/fundamentals/projects)。
    

创建 X App 后，系统会向你展示 API Key、Secret，以及一个 Bearer Token。请注意，这些凭据仅会显示一次，请务必将它们保存到你的密码管理器或其他安全位置。

关于如何处理密钥和令牌的更多建议，请参阅我们的[身份验证最佳实践](/zh/resources/fundamentals/authentication/guides/authentication-best-practices)页面，其中包括在凭据遭到泄露时应采取的措施。
 

<div id="how-to-find-and-regenerate-your-api-key-and-secret-after-app-creation">
  #### 在创建应用后如何查找并重新生成 API Key 和 Secret
</div>

如果你已创建应用，需要查找或重新生成你的 API Key 和 Secret，请按以下步骤操作：

1. 进入开发者门户
2. 展开侧边导航中的“项目与应用”下拉菜单
3. 打开与目标 API Key 和 Secret 关联的应用
4. 转到“Keys and tokens”选项卡

在此页面，你可以查看与该应用关联的所有凭据。 
 

<div id="how-to-use-your-api-key-and-secret">
  #### 如何使用你的 API Key 和 Secret
</div>

如果你正在了解 X Developer Platform，我们建议使用[工具或库](/zh/resources/tools-and-libraries)来探索平台可用功能。这些工具能高效处理身份验证，为你节省大量时间与精力。对于初学者，我们特别推荐[从 Postman 开始](/zh/tutorials/postman-getting-started)或使用 [Insomnia](https://insomnia.rest/)。

如果你希望从头构建请求，请阅读我们的指南：[为 OAuth 1.0a 请求授权](/zh/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request)。
