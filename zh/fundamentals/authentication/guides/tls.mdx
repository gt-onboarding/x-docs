要访问 X API 端点，必须使用 TLS 连接。通过 TLS 通信可在数据经由公共互联网传输时保护用户与 X API 之间的信息，从而维护用户隐私和安全。连接到 X API 必须使用 TLS 1.2。


<div id="verification">
  ## 验证
</div>

<div id="use-an-up-to-date-root-store">
  ### 使用最新的根证书存储
</div>

在验证 X 证书时，确保你的应用或库使用可信且最新的根证书存储至关重要。在可能的情况下，采用操作系统提供的根证书存储通常是最简便的方法。或者，[Mozilla（NSS）根证书存储](https://www.mozilla.org/en-US/about/governance/policies/security-group/certs/)以公开透明的方式得到良好维护。curl 还提供了[该存储的 PEM 格式版本](https://curl.haxx.se/docs/caextract.html)。

X 目前的大多数证书由 [DigiCert High Assurance EV Root CA](https://www.digicert.com/digicert-root-certificates.htm) 签发，但这并不适用于 100% 的 X 相关证书，且未来也可能发生变化，因此仅信任当前使用的 DigiCert 根证书，可能会在将来导致你的应用出现问题。

<div id="check-crls-and-the-ocsp-status">
  ### 检查 CRL 和 OCSP 状态[](#check-crls-and-the-ocsp-status "Permalink to this headline")
</div>

许多应用不会检查返回证书的证书吊销列表，或仅依赖操作系统代为检查。请确保在接受 X 的证书之前，将你的应用或 TLS 库配置为强制执行 CRL 和 OCSP（在线证书状态协议）验证。

<div id="cdns">
  ### CDN[](#cdns "Permalink to this headline")
</div>

在展示包含媒体的推文时，请使用 `media_url_https` 属性来获取用于显示图像的 HTTPS URL。未来，通过 API 端点提供的所有 URL 都将提供 HTTPS 路径。

<div id="provide-an-indication-of-security-status">
  ## 提供安全状态指示
</div>

在可行的情况下，您应显示应用与 X 之间当前安全状态的指示。一些网页浏览器会以锁形图标体现，另一些则通过描述性消息显示当前连接状态。
