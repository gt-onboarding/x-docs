<div id="general">
  ## 常见问题
</div>

<AccordionGroup>
  <Accordion title="什么是 OAuth？">
    OAuth 是一种认证协议，允许用户在不共享密码的情况下授权应用代表其执行操作。更多信息请参见 [oauth.net](http://oauth.net/)。
  </Accordion>

  <Accordion title="如何生成访问令牌？">
    您必须拥有一个 [X 应用](/zh/resources/fundamentals/developer-apps)才能生成访问令牌。请在[此处](/zh/resources/fundamentals/authentication#oauth-1-0a-2)了解更多有关访问令牌的信息。
  </Accordion>

  <Accordion title="如何创建一个 X 应用？">
    您必须拥有一个[开发者账户](/zh/resources/fundamentals/developer-portal)才能创建 [X 应用](/zh/resources/fundamentals/developer-apps)。您可以在[这里](https://developer.x.com/en/portal/petition/essential/basic-info)注册。
  </Accordion>

  <Accordion title="如果我已经有一个 X 应用，如何查看并编辑该应用？">
    如果您已在 developer.x.com 登录您的 X 账号，可以在 [X 应用控制台](https://developer.x.com/content/developer-twitter/en/apps)查看并编辑您的应用。
  </Accordion>
</AccordionGroup>

<div id="technical">
  ## 技术
</div>

<AccordionGroup>
  <Accordion title="访问令牌的有效期是多久？">
    访问令牌没有明确的过期时间。如果用户在其 X 账号设置中主动撤销某个应用，或 X 暂停了某个应用，则该访问令牌将失效。若应用被暂停，[X 应用](/zh/resources/fundamentals/developer-apps) 仪表板中会有一条说明，表明该应用已被暂停。
  </Accordion>

  <Accordion title="如果访问令牌失效怎么办？">
    应假设用户的访问令牌随时可能失效。如果发生这种情况，请提示用户重新授权该应用。妥善处理这一情况对良好的用户体验至关重要。
  </Accordion>

  <Accordion title="应用注册页面会询问读/写权限。什么算写入？">
    许多用户信任应用读取他们的信息，但未必愿意让其更改用户资料或发布新帖子。通过 X API 更新信息——无论是名称、位置，还是新增帖子——都需要使用 HTTP POST。任何需要 HTTP POST 的 API 方法都被视为写入方法，并需要读写权限。
  </Accordion>
</AccordionGroup>
