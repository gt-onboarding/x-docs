<div id="how-to-get-access-to-the-ads-api">
  ## 如何获取 Ads API 访问权限
</div>

1. 注册[开发者账号](https://developer.x.com/en/apply-for-access)。
2. 创建[开发者应用](https://developer.x.com/app)，并妥善保管你的令牌。
3. 访问 [ads.x.com/help](https://ads.x.com/help)，为你的每个开发者应用提交 Ads API 访问申请。

<Note>**注意：**如果你已在使用 X 开发者平台且拥有开发者账号，请直接跳至第三步。</Note>

<div id="step-one-signup-for-a-developer-account">
  ### 第一步：注册开发者账号
</div>

要调用任何 X 的 API 产品，您必须先注册开发者账号。在开发者门户中创建一个项目与开发者应用。这将为您提供一组凭据，您将使用这些凭据对所有发送到 API 的请求进行身份验证。

<Button href="https://developer.x.com/en/apply-for-access">
  注册开发者账号
</Button>

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### 步骤二：保存应用的密钥与令牌，并确保其安全
</div>

在你的开发者应用中，你将获得一组 API 密钥（亦称 Consumer Keys）。你还可以生成一组访问令牌，用于代表你的个人 X 账号发起请求，以及一个 Bearer Token，用于对需要 OAuth 2.0 Bearer Token 的端点进行认证。由于这些密钥与令牌除非重新生成，否则不会过期，建议将其配置为环境变量，或使用安全的密码管理器进行保存。

请同时记录你的 App ID，可在应用详情页的 URL 中找到。这将方便你在下一步申请访问 X 广告 API 时使用。

<Note>
**注意：** 密钥与令牌在开发者门户中只会显示一次，因此生成后请立即将这些凭据保存到你的密码管理系统。

如果你丢失或忘记了这些密钥与令牌，需要重新生成，这将创建新的密钥与令牌并使旧的失效。届时你需要更新所有使用先前凭据配置的集成。

了解更多[身份验证最佳实践](https://developer.x.com/en/docs/authentication/guides/authentication-best-practices)。
</Note>

<div id="step-three-apply-for-access-to-the-ads-api">
  ### 第三步：申请 Ads API 访问权限
</div>

此时，你已拥有[对 X API 的基础访问权限](https://developer.x.com/en/docs/twitter-api/v1)，但尚未具备 X 广告 API 的相关功能。接下来，你需要提交申请并通过审核以获得 Ads API 访问权限。请访问 [ads.x.com/help](https://ads.x.com/help)，为你的每个开发者应用提交 Ads API 访问申请。

<div id="access-tiers">
  ### 访问层级
</div>

作为申请流程的一部分，您需要说明所需的访问级别。进一步了解[应用级与广告账户级权限](https://developer.x.com/en/docs/twitter-ads-api/obtaining-ads-account-access#levels-of-access)。

<div id="conversion-only">
  #### 仅限转化
</div>

可读写访问移动端与网页端的转化端点。

<div id="standard-access">
  #### 标准访问
</div>

可对 Analytics、Campaign Management、Creatives、Custom Audiences 和 Conversion 端点进行读写访问。

<Note>**注意：**应用获批开通 Ads API 访问后，你需要[重新生成所有用户访问令牌](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)，以便正确发起[已认证的 Ads API 请求](https://developer.x.com/en/docs/twitter-ads-api/making-authenticated-requests)。</Note>

<Note>**注意：**在 2023 年 7 月之前申请访问的 Ads API 开发者，其访问级别和权限可能不同，且可能被限制为最多 5 个 OAuth 令牌。请参阅我们的[提高访问级别](https://developer.x.com/en/docs/twitter-ads-api/increasing-access)指南，以获取更多端点访问，或为现有应用取消令牌数量限制。</Note>

<div id="make-your-first-request">
  ## 发起你的第一次请求
</div>

要测试你对 X 广告 API 的访问权限，请向 [GET accounts](https://developer.x.com/en/docs/twitter-ads-api/campaign-management/api-reference/accounts) 端点发起请求。该端点会返回当前已授权用户可访问的广告账户。在后续的 API 请求中使用此次返回的广告账户 ID，以读取和写入特定广告账户的数据。在命令行中使用 [Twurl](https://developer.x.com/en/docs/tutorials/using-twurl) 的请求如下所示。

**示例请求**

```bash
twurl -H ads-api.x.com "/11/accounts"
```

**示例响应**

