<div id="requesting-additional-app-level-permissions">
  ### 申请额外的应用级权限
</div>

如果你的应用目前仅具备下列任一应用级权限，且希望访问更多端点，请联系你的 X 代表申请**标准访问**。

在 2023 年 7 月之前申请 Ads API 访问的开发者，可能曾申请过以下任一应用级权限，这些权限现已包含在标准访问级别中：

- **Analytics**（只读）：用户可使用 [Analytics](https://developer.x.com/en/docs/twitter-ads-api/analytics/overview) 端点并具有[只读](https://developer.x.com/en/docs/apps/app-permissions)访问权限。用户无法访问 Campaign Management、Creative 或 Custom Audiences 端点，但可以在 Analytics 端点的请求参数中传入创意实体来拉取指标。若要以 media_id 或 media_name 的粒度拉取数据，用户应用需要具备 Ads API 写入权限。

- **Campaign Management & Creative**（读写）：用户可使用 [Campaign Management](https://developer.x.com/en/docs/twitter-ads-api/campaign-management/overview) 和 [Creative](https://developer.x.com/en/docs/twitter-ads-api/creatives/overview) 端点并具有[读写](https://developer.x.com/en/docs/apps/app-permissions)访问权限。用户也可以访问 Analytics 端点。

- **Custom Audiences**（读写）：用户可使用 [Custom Audiences](https://developer.x.com/en/docs/twitter-ads-api/audiences/overview) 端点，并具有[读写](https://developer.x.com/en/docs/apps/app-permissions)访问权限。用户也可以访问 Campaign Management、Creative 和 Analytics 端点。

- **Conversion**（读写）：用户可使用 [Mobile](https://developer.x.com/en/docs/twitter-ads-api/measurement/mobile-conversions/overview) 和 [Web Conversion](https://developer.x.com/en/docs/twitter-ads-api/measurement/web-conversions/overview) 端点，并具有[读写](https://developer.x.com/en/docs/apps/app-permissions)访问权限。用户可能拥有或不拥有对其他 Ads API 端点的访问权限。 

<div id="requesting-an-increase-to-user-oauth-token-limits">
  ### 申请提高用户 OAuth 令牌上限
</div>

在 2023 年 7 月之前申请了 Ads API 访问权限的开发者，其应用的用户 OAuth 令牌上限可能为 5 个。如果你需要提高开发者应用的令牌上限，请联系你的 X 客户代表。

<Note>**注意：**我们不会提高任何 Ads API 端点的[速率限制](https://developer.x.com/en/docs/twitter-ads-api/rate-limiting)。请参阅我们的[最佳实践](https://developer.x.com/en/docs/twitter-ads-api/rate-limiting)，了解如何在速率限制范围内运行。</Note>
