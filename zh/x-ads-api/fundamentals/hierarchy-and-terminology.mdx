---
title: "X 广告 API 的层级结构与术语"
---

要在 X 广告 API 上取得成功，理解 X 广告中各实体之间的相互关系至关重要。本教程将介绍 X 广告对象层级结构的基础知识。

![X Ads API hierarchy](/images/adsapi-heirarchy.png.twimg.1920.png)

<div id="ad-group-line-items">
  ## 广告组（line items）
</div>

广告组在 X 广告 API 中称为 [line items](https://developer.x.com/content/developer-twitter/en/docs/tutorials/ads-api-hierarchy-terminology#line-items)，隶属于广告活动，用于针对一组 X 用户进行定向和出价。广告主可通过将帖子或媒体（例如作为 In-stream 广告投放的推广视频）与某个 line item 关联来进行推广。另请参阅“广告组”。

<div id="account-media">
  ## 账户媒体
</div>

作为[账户媒体](/zh/x-ads-api/creatives#account-media)上传到广告账户的一组创意资产，可包含可推广的视频[片头贴片素材](/zh/x-ads-api/campaign-management#video-views-preroll-objective)（亦称流内广告），以及可在[X Audience Platform](https://developer.x.com/content/developer-twitter/en/docs/ads/measurement/overview/twitter-audience-platform)上投放的图片。

<div id="ads-accounts">
  ## 广告账户
</div>

在 X 广告体系的最上层是广告账户。一个广告账户可以由多个 X 用户管理。@Furni（我们的家具商店演示账号）就是一个管理广告账户的用户示例。我们通过 account\_id 来标识广告账户，它是一个 [base36](https://en.wikipedia.org/wiki/Base36) 的字母数字字符串。

在每个账户下，我们会存储多种可在广告系列间复用的实体类型，例如用于定向的自定义受众、账户媒体（视频或图像）以及卡片（可添加到帖子的扩展创意）。

<div id="amplify-video">
  ## Amplify 视频
</div>

在上传视频素材时，上传方可以为视频指定一个类别（例如：Tweet Video、Tweet Gif、Tweet Image 和 Amplify Video）。当要上传用于流内广告投放的视频时，必须选择 Amplify 视频类别。此类视频需要特殊处理以确保可投放，因此必须使用 Amplify Video 媒体类别。

<div id="analytics">
  ## 分析
</div>

分析指标帮助合作伙伴和广告主了解其在 X 上推广内容的效果。这包括展示次数、点击量、视频观看量和花费等信息。此外，合作伙伴和广告主还可以获取其触达受众的各类细分维度的详细指标。

X 广告 API 支持两种获取详细投放表现指标的方式：同步和异步。

使用[同步分析](/zh/x-ads-api/analytics#synchronous-analytics)请求时，所请求的指标会直接在响应中返回。同步端点支持较短的时间范围，非常适合查询实时投放表现。

使用[异步分析](/zh/x-ads-api/analytics#asynchronous-analytics)端点时，所请求的指标会在关联的“作业”处理完成后，通过可下载的结果文件提供。异步端点支持跨更长时间范围请求指标、可返回分段数据，且适用于获取更大量的数据——非常适合生成报告或进行历史回填。

<div id="campaigns">
  ## Campaigns
</div>

每个广告主投放的广告系列都必须在现有的资金工具下创建。广告系列可用于定义广告活动的名称、排期和预算等内容。延续 Furni 的示例，我们可以创建一个名为“Furni 家居装饰”的广告系列。

需要注意的是，一个广告账户最多可拥有 200 个活跃的 Campaign；该上限可由广告主的客户经理提升。Campaign 在达到结束时间或预算耗尽之前均视为活跃。已暂停的广告系列在计算广告系列数量上限时仍计为活跃。

<div id="cards">
  ## 卡片
</div>

卡片是一种可视化扩展推文的方式，可在推文中加入图片、视频或 UI 元素。X 广告 API 支持多种可用于推文的卡片类型，随后可在广告活动中进行推广。

<div id="creatives">
  ## 创意素材
</div>

广告主可以在包含创意素材（如图片、GIF 和视频）的广告系列中推广推文。这些创意素材可与 X 卡片关联，使推文能够为推文附加更丰富的照片、视频和媒体体验。创意素材也可以直接附加到推文上。例如，你可以包含文本和一个视频（或最多 3 张图片），创建一条推广推文。

每个广告分组在投放前至少需要关联一个创意素材。我们提供多种创意素材选项，包括用于获取新粉丝的 Promoted Accounts、适用于 TX Audience Platform 上各广告分组的 Media Creatives（如图片或视频），以及 Promoted Posts。

<div id="funding-instruments">
  ## 资金工具
</div>

在广告账户下还有资金工具，指广告主为在 X 广告中使用而设置的不同资金来源。这些来源既包括与 X 签署的插播订单（insertion order），也包括绑定到账户的信用卡。

<div id="line-items">
  ## 广告组项
</div>

在一个广告系列中，您可以创建一个或多个广告组，在 X 广告 API 中也称为广告组项。广告组项可视为广告系列的不同分段，每个分段都有其独立的定向设置和创意。包含多个广告组项的广告系列通常被称为“广告组型广告系列”。

在单个广告系列中使用多个广告组项的一个好处是，您可以组合出更丰富多样的定向与创意搭配。

在我们的“Furni 家居装饰”广告系列下，我们可能想创建两个广告组项：一个是“旧金山的购房者”，另一个是“伦敦的购房者”。

每个广告系列最多允许 100 个广告组项，一个广告账户在所有广告系列中最多可拥有 4,000 个活跃广告组项。

<div id="mobile-app-promotion-map">
  ## 移动应用推广（MAP）
</div>

移动应用推广（MAP）是一套产品，帮助广告主通过 X 推广移动应用，涵盖应用安装、转化和再互动等目标。

<div id="media-library">
  ## 媒体库
</div>

媒体库端点用于管理 X 广告账户中的图片、GIF 和视频。库中的媒体资产可用于帖子和创建卡片，也可在多个创意中复用，从而无需重复上传相同资产。

<div id="promoted-accounts">
  ## 推广账号
</div>

推广账号（粉丝广告系列）属于“推荐关注”，该功能会向用户推荐他们尚未关注但可能感兴趣的账号。推广账号有助于让更多人发现他们可能喜欢的各类账号。

推广账号使广告主能够推广其账号并在 X 上扩大受众。粉丝广告系列会在 X 平台的多个位置展示——包括首页时间线、推荐关注和搜索结果——且带有“推广”标识，以区别于其他推荐账号。

<div id="promotable-users">
  ## 可推广用户
</div>

每个 X 广告账户都可以包含一个或多个可推广用户，即可通过该广告账户进行推广的用户。Furni 的 @Furni 账号是 Furni 广告账户下 FULL 可推广用户的一个示例，这意味着我们可以为 @Furni 及其推文投放所有类型的广告活动。每个广告账户只能有一个 FULL 可推广用户。

另请注意，广告账户下还可以有其他类型的可推广用户。例如，如果我们想推广 @Jack 针对 Furni 产品发布的推文，他将在该账户中被设为仅限转推的可推广用户，标记为 RETWEETS\_ONLY。广告主应联系其 X 客户经理，了解将仅限转推用户添加到其广告账户的相关事宜。

<div id="promoted-video">
  ## 推广视频
</div>

推广视频是指包含在推广推文中的视频，由广告主付费投放。它们会出现在首页时间线、X 的搜索结果顶部，以及平台的其他位置，并会清晰标注为“推广”。

<div id="publisher-network">
  ## 发布商网络
</div>

参见 [X Audience Platform](/zh/x-ads-api/audiences)。

<div id="scheduled-tweets">
  ## 预定推文
</div>

预定推文允许广告主或用户创建一条推文，并安排在未来某一时间发布。除了创建和管理这些推文之外，API 还支持将它们关联到某个投放单元（line item），以便在推文上线后进行推广。这样，广告主就能先行准备原生推文，并在关键节点前提前规划广告创意。例如，预先准备一条推文创意，在新产品发布公告一出就立即上线。

<div id="segmentation">
  ## 细分
</div>

细分功能使合作伙伴和广告主能够按特定的定向维度（如年龄、设备或语言）拆分并获取指标；该功能仅可通过我们的异步分析端点使用。要请求细分指标，请使用 segmentation\_type 请求参数。有关细分选项的更多信息，请参阅 Metrics and Segmentation。

<div id="tailored-audience-list">
  ## 定向受众 - 列表
</div>

基于列表的定向受众是三种定向受众类型之一，通过上传包含您自有用户与客户数据的文件创建。系统会将您的列表与 X 上的用户进行匹配，便于您在广告系列中直接定向触达他们。请查阅这些限制。

<div id="tailored-audience-web">
  ## 定向受众 - Web
</div>

来自 Web 的定向受众（Tailored Audience from web）是三种定向受众类型之一，由访问过你网站的 X 用户组成。要使用来自 Web 的定向受众，你必须在你的网站上安装我们的 X 像素，或使用我们认可的标签管理工具合作伙伴之一。请查看相关限制。

<div id="targeting-criteria">
  ## 定向条件
</div>

如前所述，定向条件与广告行项目相关联。您可以使用多种定向方式，从位置、关键词、性别到更细粒度的属性（如用户行为和兴趣）。对于“Home Buyers in San Francisco”这一行项目，我们可以将“home buying”行为与“San Francisco”位置组合进行定向。

有关可用定向类型的更多信息，请访问我们开发者网站上的 Targeting Overview 页面。

<div id="twitter-object-nest-ton">
  ## Twitter Object Nest（TON）
</div>

Twitter Object Nest（TON）用于存储通过 Media Upload 端点上传的媒体和各类资产，供作为创意素材使用。
