---
title: 分页
---

为支持更高的广告系列上限并高效检索与账户关联的所有实体，Advertiser API 现已在多个 GET 端点提供分页功能。该分页机制易于使用，与 REST API 的基于游标的分页非常相似，详见[使用游标浏览集合](https://developer.x.com/en/docs/x-api/v1/pagination)。

<div id="getting-started">
  ## 入门
</div>

对于特定的 GET 请求，我们现在接受可选的 `cursor` 和 `count` 查询参数。如果响应中的实体数量超过 `count`，这些端点将返回前 `count` 个实体，并在响应 JSON 中包含一个 `next_cursor` 键。

`GET https://ads-api.x.com/5/accounts/abc1/campaigns?count=50`

```json
{
  "data": \[...\],
  "next_cursor": "c-3yvu1pzhd3i7",
  "request": {...}
}
```

要获取下一页结果，请在查询参数中添加 `cursor`：

`GET https://ads-api.x.com/5/accounts/abc1/campaigns?cursor=c-3yvu1pzhd3i7&count=50`

```json
{
  "data": \[...\],
  "next_cursor": "c-3w3zdyg8ywan",
  "request": {...}
}
```

<div id="going-deeper">
  ## 更深入了解
</div>

对于大多数端点，`count` 的最大值为 **1,000**，最小值为 **1**，默认值为 **200**。

`next_cursor` 返回的值始终为字符串，应视为不透明；其实现可能会发生变化。如果当前结果页返回的实体少于 `count`，则 `next_cursor` 的值为 `null`。

**注意：** 分析端点不支持此类分页。统计端点的分页需通过指定时间区间来实现。更多信息请参阅各对应端点的文档。
