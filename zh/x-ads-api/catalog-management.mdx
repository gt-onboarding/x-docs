<div id="overview">
  ## 概述
</div>

Catalog API 是一项电商解决方案，使广告主能够设置产品数据源（feed）、将产品分组为集合，并对目录产品进行全局管理。Catalog API 是实现目录程序化管理的核心工具，可让广告主更好地掌控目录的导入与更新方式。

目录包含与用户产品及产品集相关的全部信息，并与该用户的账号（handle）关联。目前，**每位用户只能创建一个目录**。 

Catalog API 支持两种产品导入方式：

1.  定时数据源：用户可以添加数据源 URL（CSV、TSV 和 XML），系统将定期抓取（X 目前支持最大 8GB 的文件）

2.  批量产品 API：用户可以通过批量（JSON）请求查看、创建、更新和删除产品属性

X 商家可基于筛选规则创建产品集。这样即可为产品附加额外元数据，并创建特殊产品集，通过有机电商功能或动态商品广告进行展示。Catalog API 与 X 购物管理器支持两种类型的产品集。 

1.  手动：用户最多可选择 50 个产品，并设置名称和描述

2.  筛选：用户最多可添加 30 个筛选条件，自动生成满足条件的产品集。筛选类型包括价格、Google 产品类别、产品类型、库存、促销价和自定义字段。

Catalog API 提供 4 个端点用于管理目录：

* [Product Catalogs](https://developer.x.com/en/docs/x-ads-api/catalog-management/api-reference/product-catalogs) \- 创建产品信息容器
* [Products](https://developer.x.com/en/docs/x-ads-api/catalog-management/api-reference/products) -  同步添加、更新、查看并从目录中移除产品
* [Product Sets](https://developer.x.com/en/docs/x-ads-api/catalog-management/api-reference/product-sets) -  基于产品信息将产品分组
* [Schedule Feeds](https://developer.x.com/en/docs/twitter-ads-api/catalog-management/api-reference/scheduled-feeds) \- 配置将基于文件的产品数据异步导入到目录中

<div id="prerequisites-for-the-catalog-api">
  ### Catalog API 的先决条件
</div>

Catalog API 端点目前仅在早期访问阶段提供。要申请访问，请联系您的 X 客户代表。

您需要通过 [X Shopping Manager](https://ads.x.com/shopping_manager) 接受服务条款。

有关 Shopping Manager 和产品规范的详细信息，请参阅 [产品规范指南](https://business.x.com/en/help/shopping-specs.html)。

### 频率限制 

下表列出了各端点的频率限制。

<div id="product-catalog">
  #### 产品目录
</div>

| HTTP 方法 | 速率限制 |
| :--- | :--- |
| GET /product_catalogs | 每 15 分钟 1,000 次 |
| POST /product_catalogs | 每 15 分钟 20 次 |
| PUT /product_catalogs/:product_catalog_id | 每 15 分钟 20 次 |
| DELETE /product_catalogs/:product_catalog_id | 每 15 分钟 20 次 |

<div id="batch-products-api">
  #### 批量产品 API
</div>

| HTTP 方法 | 速率限制 |
| :--- | :--- |
| GET /product\_catalogs/:product\_catalog_id/products | 每 15 分钟 1,000 次 |
| PUT /product\_catalogs/:product\_catalog_id/products | 每 15 分钟 600 次 |
| DELETE /product\_catalogs/:product\_catalog_id/products | 每 15 分钟 150 次 |

<div id="product-sets">
  #### 产品集
</div>

| HTTP 方法 | 速率限制 |
| :--- | :--- |
| GET /product\_catalogs/:product\_catalog\_id/product\_sets | 每 15 分钟 2,000 次 |
| POST /product\_catalogs/:product\_catalog\_id/product\_sets | 每 15 分钟 100 次 |
| PUT /product\_catalogs/:product\_catalog\_id/product\_sets/:product\_set\_id | 每 15 分钟 500 次 |
| DELETE /product\_catalogs/:product\_catalog\_id/product\_sets/:product\_set\_id | 每 15 分钟 100 次 |

<div id="product-feeds">
  #### 产品数据源
</div>

| HTTP 方法 | 速率限制 |
| :--- | :--- |

| PUT /product\_catalogs/:product\_catalog\_id/product\_feeds/:product\_feed\_id | 每 15 分钟 20 次 |
