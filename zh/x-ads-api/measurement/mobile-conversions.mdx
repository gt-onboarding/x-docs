<div id="mact-overview">
  ## MACT 概览
</div>

X 的[移动应用推广](https://biz.x.com/mobile-app-promotion)衡量功能使广告主能够跟踪在 X 上旨在推动安装或其他应用内转化的广告活动效果。X 的移动归因测量合作伙伴（MMP）为广告主提供管理能力，可指定他们希望从在 X 上推广的应用中追踪哪些转化事件。广告主一旦设置好要跟踪的事件类型和时间窗口，移动归因测量合作伙伴便会将这些事件全部发送至 X，并即时接收归因数据，标明促成转化的用户曾接触的广告活动与创意。

<div id="conversion-events">
  ### 转化事件
</div>

要向 X 报告[转化事件](https://developer.x.com/en/docs/x-ads-api/measurement/api-reference/conversion-event)并获取归因数据，合作伙伴必须发送移动应用 ID、转化类型与时间戳，以及广告/设备 ID 的哈希值。提供这些数据将会在 X 平台上记录该转化事件，并在适用时返回相应的归因结果。此外，广告主还可选择提供该转化事件的额外元数据，例如购买金额、产品 ID 或注册方式。

#### POST conversion\_event

记录一次移动端测量的转化事件。响应将指明 X 或 X Audience Platform（TAP）的归因。

这与 [GET conversion\_attribution](https://developer.x.com/en/docs/x-ads-api/measurement/api-reference/conversion-attribution#get-conversion-attribution) 端点相关。

响应中将声明 X、TAP 或无归因。当没有 X 归因时，`twitter_attribution` 节点会始终存在并取值为 `null`（见下方示例响应）。如果声明了 TAP 归因，则会出现 `tpn_attribution` 节点并据此填充。更多信息请参阅 [TAP 概览](https://developer.x.com/en/docs/x-ads-api/measurement/overview/twitter-audience-platform)。

提供若干可选参数用于设置与每次转化事件关联的元数据。此元数据不影响归因计算。

<div id="resource-url">
  ##### 资源 URL
</div>

`https://ads-api.x.com/12/conversion_event`

<div id="parameters">
  ##### 参数
</div>

| 名称 | 描述 |
| :--- | :--- |
| app\_id  <br />_必填_ | 与相应应用商店对应的唯一标识符。<br /><br />类型：int，string<br /><br />示例：`333903271`、`com.vine.android` |
| conversion\_time  <br />_必填_ | 转化事件发生的时间，采用 ISO-8601 时间戳格式（含毫秒）。<br /><br />类型：string<br /><br />示例：`2014-05-22T02:38:28.103Z` |
| conversion\_type  <br />_必填_ | 转化事件的类型。<br /><br />类型：enum<br /><br />可能取值：`PURCHASE`, `SIGN_UP`, `INSTALL`, `RE_ENGAGE`, `UPDATE`, `TUTORIAL_COMPLETE`, `RESERVATION`, `ADD_TO_CART`, `ADD_TO_WISHLIST`, `LOGIN`, `CHECKOUT_INITIATED`, `SEARCH`, `LEVEL_ACHIEVED`, `ACHIEVEMENT_UNLOCKED`, `CONTENT_VIEW`, `SHARE`, `INVITE`, `ADDED_PAYMENT_INFO`, `SPENT_CREDITS`, `RATED` |
| hashed\_device\_id  <br />_必填_ | 经 HMAC\_SHA-256 哈希的 IDFA 或 AdID。<br /><br />类型：string<br /><br />示例：`ABCD1234XYZ` |
| os\_type  <br />_必填_ | 应用的操作系统类型。<br /><br />类型：enum<br /><br />可能取值：`IOS`, `ANDROID` |
| click\_window  <br />_可选_ | 此事件的点击回溯窗口（天）。<br /><br />类型：int<br /><br />注意：`click_window` 必须大于或等于 view\_through\_window。<br /><br />默认值：`14`  <br />可能取值：`1`, `7`, `14`, `30` |
| device\_ip\_address  <br />_可选_ | 转化事件发生时设备的 IPv4 或 IPv6 地址。<br /><br />类型：string<br /><br />示例：`192.133.78.1` |
| level  <br />_可选_ | 与此事件关联的等级/关卡。<br /><br />类型：int<br /><br />示例：`2` |
| non\_twitter\_engagement\_time  <br />_可选_ | 转化前最近一次非 Twitter 互动的时间。<br /><br />类型：string<br /><br />示例：`2014-05-22T02:38:28.103Z` |
| non\_twitter\_engagement\_type  <br />_可选_ | 转化事件前的非 Twitter 互动类型。<br /><br />类型：enum<br /><br />可能取值：`CLICK`, `VIEW` |
| number\_items  <br />_可选_ | 与此事件关联的商品/项目数量。<br /><br />类型：int<br /><br />示例：`2` |
| price\_currency  <br />_可选_ | 预计为 ISO 4217 代码，表示与此事件关联的货币。<br /><br />类型：string<br /><br />示例：`EUR`、`USD`、`JPY` |
| price\_micro  <br />_可选_ | 与此事件关联的价格（以微单位计）。<br /><br />类型：int<br /><br />示例：`123450000` |
| user\_payment\_info  <br />_可选_ | 布尔值，指示与此事件关联的应用中是否存储了用户的支付信息。<br /><br />类型：bool<br /><br />可能取值：`true` 或 `false` |
| view\_through\_window  <br />_可选_ | 此事件的曝光回溯窗口（天）。<br /><br />注意：<br /><br />`click_window` 必须大于或等于 view\_through\_window。<br /><br />类型：int<br /><br />默认值：`1`  <br />可能取值：`0`, `1`, `7`, `14`, `30` |

这与 [POST conversion\_event](https://developer.x.com/en/docs/x-ads-api/measurement/api-reference/conversion-event#post-conversion-event) 端点相关。

响应中将声明 X、TAP 或无归因。`twitter_attribution` 节点始终存在；当不存在 X 归因时其值为 `null`，存在时会如下面的示例响应所示被填充。若声明了 TAP 归因，则会存在 `tpn_attribution` 节点并据此填充。更多信息请参阅 [TAP 概览](https://developer.x.com/en/docs/twitter-ads-api/measurement/twitter-audience-platform/overview)。
