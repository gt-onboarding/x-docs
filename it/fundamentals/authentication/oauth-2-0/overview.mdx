<div id="bearer-token-also-known-as-app-only">
  ### Bearer Token (noto anche come app-only)
</div>

Il Bearer Token OAuth 2.0 autentica le richieste per conto della tua [App sviluppatore](/it/resources/fundamentals/developer-apps). Poiché questo metodo è specifico dell’App, non coinvolge alcun utente. È in genere usato da sviluppatori che necessitano di accesso in sola lettura a informazioni pubbliche.

Questo metodo di autenticazione richiede di includere un Bearer Token nella richiesta, che puoi generare nella sezione Keys and tokens delle tue App sviluppatore. Ecco un esempio di richiesta con un Bearer Token fittizio:

```json
  curl "https://api.x.com/2/tweets?ids=1261326399320715264,1278347468690915330" \
  -H "Authorization: Bearer AAAAAAAAAAAAAAAAAAAAAFnz2wAAAAAAxTmQbp%2BIHDtAhTBbyNJon%2BA72K4%3DeIaigY0QBrv6Rp8KZQQLOTpo9ubw5Jt?WRE8avbi"
```

Le chiamate API che utilizzano l’autenticazione app-only sono soggette a [limiti di frequenza](/it/resources/fundamentals/rate-limits) per endpoint a livello di App.

Per utilizzare questo metodo, ti servirà un Bearer Token, che puoi generare passando la tua API Key e Secret all’endpoint [POST oauth2/token](/it/resources/fundamentals/authentication/api-reference#post-oauth2-token), oppure generandolo nella sezione "keys and tokens" delle impostazioni della tua App nel [developer portal](/it/resources/fundamentals/developer-portal).

Se desideri revocare un Bearer Token, puoi utilizzare l’endpoint [POST oauth2/invalidate\_token](/it/resources/fundamentals/authentication/api-reference#post-oauth2-invalidate-token), oppure fare clic su "revoke" accanto al Bearer Token nella sezione "keys and tokens" delle impostazioni della tua App.

<div id="oauth-20-authorization-code-flow-with-pkce">
  ### Flusso OAuth 2.0 Authorization Code con PKCE
</div>

Il flusso OAuth 2.0 Authorization Code con PKCE consente di autenticarsi per conto di un altro utente con un controllo più granulare sugli scope di un’applicazione e migliora i flussi di autorizzazione su più dispositivi. In altre parole, gli sviluppatori che creano applicazioni per le persone su X avranno maggiore controllo sulle informazioni che la loro app richiede agli utenti, così da dover chiedere agli utenti finali solo i dati strettamente necessari.

Questo moderno protocollo di autorizzazione ti permette di offrire agli utenti finali un flusso di consenso più snello per autorizzare la tua app, mostrando solo gli scope specifici richiesti. Questo non solo riduce l’onere in termini di dati, ma può anche aumentare la fiducia da parte degli utenti finali.
