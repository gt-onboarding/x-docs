<div id="using-and-generating-an-app-only-bearer-token">
  ### Utilizzo e generazione di un Bearer Token per app
</div>

Un Bearer Token offre agli sviluppatori un punto di accesso più sicuro per utilizzare le API di X ed è una delle funzionalità fondamentali di OAuth 2.0.

L’autenticazione che utilizza un Bearer Token è nota anche come autenticazione solo applicazione (application-only). Un Bearer Token è un array di byte di formato non specificato che si genera usando uno script, ad esempio un comando curl. È anche possibile ottenere un Bearer Token dal portale per sviluppatori, nella sezione chiavi e token delle impostazioni della tua App. Ulteriori informazioni su questa funzionalità sono disponibili nella [documentazione ufficiale di OAuth](https://oauth.net/2/bearer-tokens/).

<div id="when-are-they-used">
  #### Quando vengono utilizzati?
</div>

I prodotti che richiedono l’uso di un Bearer Token sono i seguenti:

- [Engagement API](/it/x-api/enterprise-gnip-2.0/fundamentals/engagement-api)
- [Account Activity API](/it/x-api/enterprise-gnip-2.0/fundamentals/account-activity)
- Altre API che utilizzano l’autenticazione con Bearer Token OAuth 2.0, come gli endpoint v2 e Labs.

<div id="prerequisites">
  #### Prerequisiti
</div>

Devi [registrarti per un account sviluppatore](https://developer.x.com/en/portal/petition/essential/basic-info) e aver creato una [X App](/it/resources/fundamentals/developer-apps). Una volta fatto, dovrai anche ottenere le chiavi API nel [portale sviluppatori](/it/resources/fundamentals/developer-portal). Segui i passaggi seguenti:

1. Accedi al tuo account X su developer.x.com.
2. Vai alla [dashboard della X App](https://developer.x.com/content/developer-twitter/en/apps) e apri la X App per cui desideri generare i token di accesso.
3. Vai alla pagina "keys and tokens".
4. In questa pagina troverai le chiavi API, gli access token utente e il bearer token.

<div id="how-to-generate-a-bearer-token">
  ### Come generare un Bearer Token
</div>

Puoi trovare il Bearer Token della tua App insieme alle altre “Keys and Tokens”.

Copia la seguente richiesta cURL nel tuo terminale dopo aver sostituito le chiavi API consumer ottenute in precedenza dalla tua [X App](/it/resources/fundamentals/developer-apps). Nota che le chiavi API consumer usate in questa pagina sono state dismesse e non funzioneranno per richieste reali.

- **API key** `<API key>` ad es. `xvz1evFS4wEEPTGEFPHBog`
- **API secret key** `<API secret key>` ad es. `L8qq9PZyRg6ieKGEKhZolGC0vJWLw8iEJ88DRdyOg`

  ```bash
  curl -u "$API_KEY:$API_SECRET_KEY" \
  --data 'grant_type=client_credentials' \
  'https://api.x.com/oauth2/token'
  ```

Ecco un esempio di come dovrebbe apparire la richiesta cURL con le tue chiavi API inserite:

```bash
   curl -u 'xvz1evFS4wEEPTGEFPHBog:L8qq9PZyRg6ieKGEKhZolGC0vJWLw8iEJ88DRdyOg' \
  --data 'grant_type=client_credentials' \
  'https://api.x.com/oauth2/token'
```

Ecco come apparirebbe la risposta. Nota che questo è un Bearer Token dismesso:

`{"token_type":"bearer","access_token":"AAAAAAAAAAAAAAAAAAAAAMLheAAAAAAA0%2BuSeid%2BULvsea4JtiGRiSDSJSI%3DEUifiRBkKG5E2XzMDjRfl76ZC9Ub0wnz4XsNiRVBChTYbJcE3F"}`

Il Bearer Token usato per autenticarsi alle risorse con OAuth 2.0 sarebbe:

`AAAAAAAAAAAAAAAAAAAAAMLheAAAAAAA0%2BuSeid%2BULvsea4JtiGRiSDSJSI%3DEUifiRBkKG5E2XzMDjRfl76ZC9Ub0wnz4XsNiRVBChTYbJcE3F`
