### POST oauth/request\_token

Consente a un’applicazione Consumer di ottenere un OAuth Request Token per richiedere l’autorizzazione dell’utente. Questo metodo soddisfa la [sezione 6.1](https://oauth.net/core/1.0/#auth_step1) del [flusso di autenticazione OAuth 1.0](http://oauth.net/core/1.0/#anchor9).

**Richiediamo l’uso di HTTPS per tutti i passaggi di autorizzazione OAuth.**

**Nota d’uso:** Per `oauth_nonce` sono accettati solo valori ASCII

**URL della risorsa[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth/request_token`

**Informazioni sulla risorsa[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Formati di risposta | JSON |
| Richiede autenticazione? | No |
| Soggetta a limitazione di frequenza? | Sì |

**Parametri[](#parameters "Permalink to this headline")**

| Nome | Obbligatorio | Descrizione | Esempio |
| :--- | :--- | :--- | :--- |
| oauth\_callback | obbligatorio | Per la conformità a OAuth 1.0a questo parametro è **obbligatorio**. Il valore specificato qui sarà utilizzato come URL a cui l’utente viene reindirizzato qualora approvi l’accesso della tua applicazione al suo account. Impostalo su `oob` per la modalità PIN out‑of‑band. Questo è anche il modo per specificare callback personalizzate da usare in applicazioni desktop/mobile. Invia sempre un `oauth_callback` in questo passaggio, indipendentemente da una callback preregistrata.<br /><br />Richiediamo che qualsiasi URL di callback usato con questo endpoint sia configurato nelle impostazioni dell’app su developer.x.com\* | `http://themattharris.local/auth.php` `twitterclient://callback` |
| x\_auth\_access\_type | facoltativo | Sovrascrive il livello di accesso che un’applicazione richiede all’account di un utente. I valori supportati sono `read` o `write`. Questo parametro consente a uno sviluppatore di registrare un’applicazione con permessi di lettura/scrittura e, quando opportuno, richiedere l’accesso in sola lettura. |     |

Scopri di più su come approvare i tuoi URL di callback in [questa pagina](/it/resources/fundamentals/developer-apps#callback-urls).

**Nota bene** - Puoi visualizzare e modificare le tue [app X](/it/resources/fundamentals/developer-apps) esistenti tramite la [dashboard delle app X](https://developer.x.com/en/apps) se hai effettuato l’accesso al tuo account X su developer.x.com.

**Esempio di richiesta[](#example-request "Permalink to this headline")**

URL della richiesta: `POST https://api.x.com/oauth/request_token`

Corpo POST della richiesta: _N/D_

Intestazione Authorization: `OAuth oauth_nonce="K7ny27JTpKVsTgdyLdDfmQQWVLERj2zAK5BslRsqyw", oauth_callback="http%3A%2F%2Fmyapp.com%3A3005%2Ftwitter%2Fprocess_callback", oauth_signature_method="HMAC-SHA1", oauth_timestamp="1300228849", oauth_consumer_key="OqEqJeafRSF11jBMStrZz", oauth_signature="Pc%2BMLdv028fxCErFyi8KXFM%2BddU%3D", oauth_version="1.0"`

Risposta: `oauth_token=Z6eEdO8MOmk394WozF5oKyuAv855l4Mlqo7hhlSLik&oauth_token_secret=Kd75W4OQfb2oJTV0vzGzeXftVAwgMnEK9MumzYcM&oauth_callback_confirmed=true`

<div id="get-oauthauthorize">
  ### GET oauth/authorize
</div>

Consente a un’applicazione Consumer di utilizzare un OAuth Request Token per richiedere l’autorizzazione dell’utente. Questo metodo soddisfa la [sezione 6.2](http://oauth.net/core/1.0/#auth_step2) del [flusso di autenticazione OAuth 1.0](http://oauth.net/core/1.0/#anchor9). Le applicazioni desktop devono utilizzare questo metodo (e non possono usare [GET oauth/authenticate](/it/resources/fundamentals/authentication/api-reference#get-oauth-authenticate)).

Nota d’uso: un `oauth_callback` non viene mai inviato a questo metodo; fornitelo invece a [POST oauth/request\_token](/it/resources/fundamentals/authentication/api-reference#post-oauth-request-token).

**URL della risorsa[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth/authorize`

**Informazioni sulla risorsa[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Formati di risposta | JSON |
| Richiede autenticazione? | Sì |
| Soggetto a rate limit? | Sì |

**Parametri[](#parameters "Permalink to this headline")**

|     |     |     |     |     |
| :--- | :--- | :--- | :--- | :--- |
| Nome | Obbligatorio | Descrizione | Valore predefinito | Esempio |
| force\_login | facoltativo | Obbliga l’utente a inserire le credenziali per assicurare che venga autorizzato l’account corretto. |     |     |
| screen\_name | facoltativo | Precompila il campo del nome utente nella schermata di accesso OAuth con il valore fornito. |     |     |

**Esempio di richiesta[](#example-request "Permalink to this headline")**

Inviare l’utente al passaggio `oauth/authorize` in un browser web, includendo un parametro oauth\_token:
`https://api.x.com/oauth/authorize?oauth_token=Z6eEdO8MOmk394WozF5oKyuAv855l4Mlqo7hhlSLik`

<div id="get-oauthauthenticate">
  ### GET oauth/authenticate
</div>

Consente a un’applicazione Consumer di utilizzare un `request_token` OAuth per richiedere l’autorizzazione dell’utente.

Questo metodo sostituisce la [Sezione 6.2](http://oauth.net/core/1.0/#auth_step2) del [flusso di autenticazione OAuth 1.0](http://oauth.net/core/1.0/#anchor9) per le applicazioni che utilizzano il flusso di autenticazione con callback. Il metodo utilizzerà l’utente attualmente connesso come account per l’autorizzazione all’accesso, a meno che il parametro `force_login` non sia impostato su `true`.

Questo metodo differisce da [GET oauth/authorize](/it/resources/fundamentals/authentication/api-reference#get-oauth-authorize) in quanto, se l’utente ha già concesso l’autorizzazione all’applicazione, il reindirizzamento avverrà senza che l’utente debba approvare nuovamente l’applicazione. Per ottenere questo comportamento, è necessario abilitare l’impostazione _Usa Accedi con X_ nel [record dell’applicazione](https://developer.x.com/apps).

**Resource URL[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth/authenticate`

**Resource Information[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Formati di risposta | JSON |
| Richiede autenticazione? | Sì |
| Soggetto a rate limit? | Sì |

**Parameters[](#parameters "Permalink to this headline")**

|     |     |     |     |     |
| :--- | :--- | :--- | :--- | :--- |
| Nome | Obbligatorio | Descrizione | Valore predefinito | Esempio |
| force\_login | facoltativo | Impone all’utente di inserire le credenziali per garantire che sia autorizzato l’account corretto. |     | _true_ |
| screen\_name | facoltativo | Precompila il campo del nome utente nella schermata di accesso OAuth con il valore indicato. |     |     |

**Example request[](#example-request "Permalink to this headline")**

Inviare l’utente al passaggio `oauth/authenticate` in un browser web, includendo un parametro oauth\_token:
`https://api.x.com/oauth/authenticate?oauth_token=Z6eEdO8MOmk394WozF5oKyuAv855l4Mlqo7hhlSLik`

### POST oauth/access\_token

Consente a un’applicazione Consumer di scambiare l’OAuth Request Token con un OAuth Access Token. Questo metodo soddisfa la [sezione 6.3](http://oauth.net/core/1.0/#auth_step3) del [flusso di autenticazione OAuth 1.0](http://oauth.net/core/1.0/#anchor9).

**URL della risorsa[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth/access_token`

**Informazioni sulla risorsa[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Formati di risposta | JSON |
| Richiede autenticazione? | Sì |
| Soggetto a rate limit? | Sì |

**Parametri[](#parameters "Permalink to this headline")**

|     |     |     |     |     |
| :--- | :--- | :--- | :--- | :--- |
| Nome | Obbligatorio | Descrizione | Valore predefinito | Esempio |
| oauth\_token | obbligatorio | L’oauth\_token qui deve essere lo stesso dell’oauth\_token restituito nello step request\_token. |     |     |
| oauth\_verifier | obbligatorio | Se si utilizza il flusso web di OAuth, impostare questo parametro al valore di _oauth\_verifier_ restituito nell’URL di callback. Se si utilizza OAuth out-of-band, impostare questo valore al PIN. Per la conformità a OAuth 1.0a questo parametro è **obbligatorio**. OAuth 1.0a è applicato rigorosamente e le applicazioni che non utilizzano _oauth\_verifier_ non riusciranno a completare il flusso OAuth. |     |     |

**Esempio di richiesta[](#example-request "Permalink to this headline")**

### POST oauth/invalidate\_token

Consente a un'applicazione registrata di revocare un OAuth `access_token` emesso presentando le proprie credenziali client. Una volta che un `access_token` è stato invalidato, nuovi tentativi di creazione produrranno un token di accesso diverso e l'uso del token invalidato non sarà più consentito.

**URL della risorsa[](#resource-url "Permalink to this headline")**

`https://api.x.com/1.1/oauth/invalidate_token`

**Informazioni sulla risorsa[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Formati di risposta | JSON |
| Richiede autenticazione? | Sì - contesto utente con i token di accesso che desideri invalidare |
| Limitato in frequenza? | Sì |

**Esempio di richiesta[](#example-request "Permalink to this headline")**

```bash
        curl --request POST
          --url 'https://api.x.com/1.1/oauth/invalidate_token.json'
          --header 'authorization: OAuth oauth_consumer_key="CLIENT_KEY",
         oauth_nonce="AUTO_GENERATED_NONCE", oauth_signature="AUTO_GENERATED_SIGNATURE",
         oauth_signature_method="HMAC-SHA1", oauth_timestamp="AUTO_GENERATED_TIMESTAMP",
         oauth_token="ACCESS_TOKEN", oauth_version="1.0"'
```

**Esempio di risposta[](#example-response "Permalink to this headline")**

```bash
        HTTP/1.1 200 OK
        Content-Type: application/json; charset=utf-8
        Content-Length: 127
        ...

        {"access_token":"ACCESS_TOKEN"}
```

**Esempio di risposta di errore dopo l'invalidazione del token[](#example-error-response-after-token-has-been-invalidated "Permalink to this headline")**

```bash
        HTTP/1.1 401 Authorization Required
        ...

        {"errors": [{
          "code": 89,
          "message": "Invalid or expired token."}
        ]}
```

<div id="post-oauth2token">
  ### POST oauth2/token
</div>

Consente a un’applicazione registrata di ottenere un token Bearer OAuth 2, utilizzabile per effettuare richieste all’API per conto dell’applicazione stessa, senza un contesto utente. Questo è detto [autenticazione solo applicazione](/it/resources/fundamentals/authentication/oauth-2-0/application-only).

Un token Bearer può essere invalidato tramite oauth2/invalidate\_token. Una volta invalidato, i nuovi tentativi di creazione restituiranno un token Bearer diverso e l’uso del token precedente non sarà più consentito.

Può esistere un solo token Bearer attivo per un’applicazione e richieste ripetute a questo metodo restituiranno lo stesso token già esistente finché non verrà invalidato.

Le risposte con esito positivo includono una struttura JSON che descrive il token Bearer emesso.

I token ricevuti con questo metodo dovrebbero essere memorizzati in cache. Se richiesto troppo frequentemente, le richieste verranno rifiutate con HTTP 403 e codice 99.

**URL della risorsa[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth2/token`

**Informazioni sulla risorsa[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Formati di risposta | JSON |
| Richiede autenticazione? | Sì - Autenticazione Basic con la tua chiave API come nome utente e il segreto della chiave API come password |
| Soggetta a rate limit? | Sì |

**Parametri[](#parameters "Permalink to this headline")**

|     |     |     |     |     |
| :--- | :--- | :--- | :--- | :--- |
| Nome | Obbligatorio | Descrizione | Valore predefinito | Esempio |
| grant\_type | obbligatorio | Specifica il tipo di grant richiesto dall’applicazione. Al momento è consentito solo _client\_credentials_. Vedi [autenticazione solo applicazione](/it/resources/fundamentals/authentication/oauth-2-0/application-only) per ulteriori informazioni. |     | _client\_credentials_ |

**Esempio di richiesta[](#example-request "Permalink to this headline")**

```bash
    POST /oauth2/token HTTP/1.1
    Host: api.x.com
    User-Agent: My X App v1.0.23
    Authorization: Basic eHZ6MWV2R ... o4OERSZHlPZw==
    Content-Type: application/x-www-form-urlencoded;charset=UTF-8
    Content-Length: 29
    Accept-Encoding: gzip

    grant_type=client_credentials
```

**Esempio di risposta:**

```bash
    HTTP/1.1 200 OK
    Status: 200 OK
    Content-Type: application/json; charset=utf-8
    ...
    Content-Encoding: gzip
    Content-Length: 140

    {"token_type":"bearer","access_token":"AAAA%2FAAA%3DAAAAAAAA"}
```

### POST oauth2/invalidate\_token

Consente a un’applicazione registrata di revocare un Bearer Token OAuth 2.0 emesso presentando le proprie credenziali client. Una volta che un Bearer Token è stato invalidato, i nuovi tentativi di creazione restituiranno un Bearer Token diverso e l’uso del token invalidato non sarà più consentito.

Le risposte con esito positivo includono una struttura JSON che descrive il Bearer Token revocato.

**URL della risorsa[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth2/invalidate_token`

**Informazioni sulla risorsa[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Formati di risposta | JSON |
| Richiede autenticazione? | Sì - [OAuth 1.0a](/it/resources/fundamentals/authentication/oauth-1-0a) con le chiavi API consumer dell’applicazione e l’access token e access token secret del proprietario dell’applicazione |
| Soggetto a limitazione di frequenza? | Sì |

**Parametri[](#parameters "Permalink to this headline")**

| Nome | Obbligatorio | Descrizione |
| :--- | :--- | :--- |
| access\_token | obbligatorio | Il valore del Bearer Token che desideri invalidare |

**Esempio di richiesta[](#example-request "Permalink to this headline")**

```
        curl --request POST
          --url 'https://api.x.com/oauth2/invalidate_token?access_token=AAAA%2FAAA%3DAAAAAAAA'
          --header 'authorization: OAuth oauth_consumer_key="CLIENT_KEY",
         oauth_nonce="AUTO_GENERATED_NONCE", oauth_signature="AUTO_GENERATED_SIGNATURE",
         oauth_signature_method="HMAC-SHA1", oauth_timestamp="AUTO_GENERATED_TIMESTAMP",
         oauth_token="ACCESS_TOKEN", oauth_version="1.0"'
```

**Esempio di risposta[](#example-response "Permalink to this headline")**

```
         Status: 200 OK
         Content-Type: application/json; charset=utf-8
         Content-Length: 135
         ...
       {
        "access_token": "AAAA%2FAAA%3DAAAAAAAA"
        }
```
