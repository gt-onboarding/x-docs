<div id="api-key-and-secret">
  ### Chiave API e Secret
</div>

La Chiave API e il Secret (noti anche come Consumer Key e Secret) sono le credenziali di base necessarie per accedere all’API X. Queste credenziali fungono da nome utente e password per la tua App X e sono utilizzate dall’API X per identificare da quale App provengono le richieste.

Queste credenziali possono essere utilizzate dagli [endpoint di autenticazione](/it/resources/fundamentals/authentication/api-reference) per generare credenziali aggiuntive, come [Access Token e Secret utente](/it/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret) e [Bearer Token](/it/resources/fundamentals/authentication/oauth-2-0/bearer-tokens). Devi anche utilizzare queste credenziali insieme agli Access Token e ad altri parametri di autorizzazione per [autorizzare le richieste](/it/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request) che richiedono l’autenticazione OAuth 1.0a in contesto utente. 

<div id="how-to-acquire-an-api-key-and-secret">
  #### Come ottenere una API Key e una Secret
</div>

Per ottenere una X API Key e una Secret, segui questi passaggi:

1. [Crea un account sviluppatore X](https://developer.x.com/en/apply-for-access)
2. Crea una [X App](/it/resources/fundamentals/developer-apps) all'interno del [developer portal](/it/resources/fundamentals/developer-portal). Nota che, se desideri utilizzare l’[API X v2](/it/x-api/introduction), devi aggiungere la tua X App a un [Project](/it/resources/fundamentals/projects).
    

Quando crei la tua X App, ti verranno mostrati la tua API Key e la Secret, insieme a un Bearer Token. Tieni presente che mostriamo queste credenziali una sola volta, quindi assicurati di salvarle nel tuo password manager o in un luogo sicuro.

Trovi ulteriori raccomandazioni su come gestire chiavi e token nella nostra pagina sulle [best practice di autenticazione](/it/resources/fundamentals/authentication/guides/authentication-best-practices), inclusi i dettagli su cosa fare se le tue credenziali sono state compromesse.
 

<div id="how-to-find-and-regenerate-your-api-key-and-secret-after-app-creation">
  #### Come trovare e rigenerare la tua API Key e Secret dopo la creazione dell'App
</div>

Se hai già creato un'App e hai bisogno di trovare o rigenerare la tua API Key e Secret, segui questi passaggi:

1. Vai al Developer Portal
2. Espandi il menu a discesa "Projects and Apps" nella barra laterale
3. Apri l'App associata all'API Key e alla Secret che desideri trovare o rigenerare
4. Vai alla scheda Keys and tokens

Da lì troverai tutte le credenziali associate alla tua App. 
 

<div id="how-to-use-your-api-key-and-secret">
  #### Come usare la tua API Key e Secret
</div>

Se stai semplicemente esplorando la X Developer Platform, ti consigliamo di usare un [tool o una libreria](/it/resources/tools-and-libraries) per scoprire le funzionalità disponibili sulla piattaforma. Questi strumenti gestiscono l’autenticazione in modo trasparente e possono farti risparmiare molto tempo e frustrazione. In particolare, consigliamo di [iniziare con Postman](/it/tutorials/postman-getting-started) o [Insomnia](https://insomnia.rest/) per gli sviluppatori alle prime armi. 

Se sei interessato a costruire una richiesta da zero, leggi la nostra guida su come [autorizzare una richiesta OAuth 1.0a](/it/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request).
