Le API X gestiscono enormi volumi di dati. Garantiamo la sicurezza di questi dati per sviluppatori e utenti tramite l’autenticazione. Esistono diversi metodi di autenticazione, elencati di seguito.

La maggior parte degli sviluppatori non dovrà occuparsi delle complessità legate all’autenticazione, poiché le librerie client le gestiscono automaticamente.

Puoi trovare l’elenco delle librerie client disponibili nella nostra pagina [Strumenti e librerie](/it/resources/tools-and-libraries).

<div id="authentication-methods">
  ## Metodi di autenticazione
</div>

<CardGroup cols={2}>
  <Card title="OAuth 1.0a User Context" href="/it/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret">
    OAuth 1.0a consente a un’app sviluppatore X autorizzata di accedere alle informazioni private di un account o di eseguire azioni su X per conto di un account X.

    <br />

    [**Scopri di più**](/it/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret)
  </Card>

  <Card title="App only" href="/it/resources/fundamentals/authentication/oauth-2-0/overview">
    Il token di accesso “solo app” consente a un’app sviluppatore X di accedere alle informazioni pubblicamente disponibili su X.

    <br />

    [**Scopri di più**](/it/resources/fundamentals/authentication/oauth-2-0/overview)
  </Card>

  <Card title="Basic authentication" href="/it/resources/fundamentals/authentication/basic-auth">
    Molte API enterprise di X richiedono l’uso dell’autenticazione HTTP Basic.

    <br />

    [**Scopri di più**](/it/resources/fundamentals/authentication/basic-auth)
  </Card>

  <Card title="OAuth 2.0 Authorization Code Flow with PKCE" href="/it/resources/fundamentals/authentication/oauth-2-0/authorization-code">
    Il contesto utente OAuth 2.0 consente di autenticarsi per conto di un altro account con un controllo più granulare sull’ambito dell’applicazione e con flussi di autorizzazione su più dispositivi.

    <br />

    [**Scopri di più**](/it/resources/fundamentals/authentication/oauth-2-0/authorization-code)
  </Card>
</CardGroup>

<Note>
  **Nota:**
  Le API Key della tua app e il token di accesso “solo app”, così come il tuo Access Token personale e l’Access Token Secret, sono disponibili nella sezione [X developer Apps](/it/resources/fundamentals/developer-apps) del [developer portal](/it/resources/fundamentals/developer-portal).

  **Se desideri effettuare richieste per conto di un altro utente**, dovrai generare un set separato di Access Token per quell’utente utilizzando il [flusso OAuth a 3 passaggi](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens) e inviare i token di quell’utente insieme alle tue richieste con OAuth 1.0a User Context o con il contesto utente OAuth 2.0.
</Note>
