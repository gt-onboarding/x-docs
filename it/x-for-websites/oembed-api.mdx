Puoi utilizzare l'API oEmbed per ottenere in modo programmatico contenuti incorporati, come [post](https://developer.x.com/en/docs/twitter-for-websites/embedded-tweets/overview) e [timeline](https://developer.x.com/en/docs/twitter-for-websites/timelines/overview).

La risposta dell'API oEmbed restituisce un frammento di HTML che verrà riconosciuto automaticamente quando [il JavaScript dei widget di X è incluso nella pagina](https://developer.x.com/web/javascript/loading).

Tieni presente che l'API è consigliata per attività in blocco; consigliamo di utilizzare il nostro strumento avanzato [publish.x.com](https://publish.x.com/#) per incorporare contenuti.

<Tabs>
  <Tab title="Embedded timelines">
    Il frammento HTML restituito sarà riconosciuto automaticamente come [timeline incorporata](https://developer.x.com/en/docs/twitter-for-websites/timelines/overview) quando [il JavaScript del widget di X è incluso nella pagina](https://developer.x.com/web/javascript/loading).

    L'endpoint oEmbed consente di personalizzare l'aspetto finale di una timeline incorporata impostando le relative proprietà nel markup HTML, che verranno interpretate dal JavaScript di X incluso per impostazione predefinita nella risposta HTML. Il formato del markup restituito può cambiare nel tempo man mano che X aggiunge nuove funzionalità o modifica la rappresentazione della timeline.

    Per una timeline di X specificata dall'URL della timeline, in un formato JSON compatibile con [oEmbed](https://oembed.com/). Sono supportate le timeline di utenti e liste. Il markup della timeline è pensato per essere memorizzato nella cache sui tuoi server fino al tempo di cache suggerito specificato dalla proprietà cache\_age.

    ## URL della risorsa

    _https://publish.x.com/oembed_

    ## Informazioni sulla risorsa

    |     |     |
    | :--- | :--- |
    | Formati di risposta | JSON |
    | Richiede autenticazione? | No  |
    | Soggetta a rate limit | No  |

    ## Parametri

    | Nome | Descrizione | Esempio |
    | :--- | :--- | :--- |
    | **url** | L'URL della timeline di X da incorporare | \*   https://x.com/TwitterDev<br /> \*https://x.com/TwitterDev/lists/national-parks
    | limit | Mostra fino a N elementi, dove N è un valore compreso tra 1 e 20 inclusi | 6   |
    | maxwidth | Imposta la larghezza massima del widget. Deve essere compresa tra 180 e 1200 inclusi | 300 |
    | maxheight | Imposta l'altezza massima del widget. Deve essere maggiore di 200 | 400 |
    | omit\_script | Non includere un elemento script nella risposta | 1   |
    | lang | Un [codice della lingua](/it/en/docs/twitter-for-websites/twitter-for-websites-supported-languages/overview "Twitter language code") supportato da X | es  |
    | theme | Se impostato su dark, la timeline viene visualizzata con testo chiaro su sfondo scuro | dark |
    | chrome | Rimuovi un componente dell'interfaccia della timeline con token separati da spazi<br /><br />\*   noheader - nasconde l'intestazione<br />\*   nofooter - nasconde il piè di pagina, se visibile<br />\*   noborders - rimuove tutti i bordi: attorno al widget, tra i Tweet e all'interno di un Tweet<br />\*   noscrollbar - ritaglia e nasconde la barra di scorrimento della timeline, se visibile<br />\*   transparent - rimuove il colore di sfondo | noheader%20nofooter |
    | aria\_polite | Imposta un valore assertive per la [cortesia delle regioni live ARIA](https://www.w3.org/TR/wai-aria/states_and_properties#aria-live) per i Tweet aggiunti a una timeline | assertive |
    | dnt | Se impostato su true, la timeline e la pagina incorporata nel tuo sito non vengono utilizzate per scopi che includono [suggerimenti personalizzati](https://support.x.com/articles/20169421) e [annunci personalizzati](https://support.x.com/articles/20170405) | true |

    ## Esempi di richieste

    ```bash
    curl --request GET --url 'https://publish.x.com/oembed?url=https%3A%2F%2Ftwitter.com%2FInterior%2Fstatus%2F507185938620219395'
    twurl -H publish.x.com "/oembed?url=https://x.com/Interior/status/463440424141459456"
    ```

    ## Esempio di risposta

    ```json

    {
      "url": "https://x.com/TwitterDev",
      "title": "",
      "html": "<a class=\"twitter-timeline\" href=\"https://x.com/TwitterDev\">Tweets by TwitterDev</a>\n<script async src=\"//platform.x.com/widgets.js\" charset=\"utf-8\"></script>",
      "width": null,
      "height": null,
      "type": "rich",
      "cache_age": "3153600000",
      "provider_name": "Twitter",
      "provider_url": "https://x.com",
      "version": "1.0"
    }
    ```

  <Tab title="Embedded Tweets">
    Lo snippet HTML restituito verrà riconosciuto automaticamente come un [Tweet incorporato](https://developer.x.com/web/embedded-tweets) quando [il JavaScript del widget di X è incluso nella pagina](https://developer.x.com/web/javascript/loading).

    L’endpoint oEmbed consente di personalizzare l’aspetto finale di un Tweet incorporato impostando le proprietà corrispondenti nel markup HTML, che verrà interpretato dal JavaScript di X incluso per impostazione predefinita nella risposta HTML. Il formato del markup restituito può cambiare nel tempo man mano che X aggiunge nuove funzionalità o modifica la rappresentazione del Tweet.

    Il markup di fallback del Post va memorizzato nella cache sui tuoi server per un periodo non superiore alla durata consigliata indicata dalla proprietà `cache_age`.

    ## URL risorsa

    _https://publish.x.com/oembed_

    ## Informazioni sulla risorsa

    |     |     |
    | :--- | :--- |
    | Formato di risposta | JSON |
    | Richiede autenticazione? | No  |
    | Soggetto a limitazioni di rate | No  |

    ## Parametri

    | Nome | Predefinito | Descrizione |
    | :--- | :--- | :--- |
    | `url` obbligatorio  <br />Stringa |     | L’URL del Post da incorporare |
    | `maxwidth`  <br />Int `[220..550]` | `325` | La larghezza massima di un Post renderizzato in pixel interi. Un valore fornito al di sotto o al di sopra dell’intervallo consentito verrà restituito come larghezza minima o massima supportata, rispettivamente; il valore di larghezza ricalcolato sarà riportato nella proprietà `width` restituita. Nota: X non supporta il parametro oEmbed `maxheight`. I Post sono essenzialmente testo e quindi hanno un’altezza imprevedibile che non può essere scalata come un’immagine o un video. Di conseguenza, la risposta oEmbed non fornirà un valore per `height`. Le implementazioni che necessitano altezze uniformi per i Post dovrebbero fare riferimento ai parametri `hide_thread` e `hide_media` di seguito. |
    | `hide_media`  <br />Boolean, String o Int | `false` | Se impostato su `true`, `"t"` o `1`, i link in un Post non vengono espansi in anteprime di foto, video o link. |
    | `hide_thread`  <br />Boolean, String o Int | `false` | Se impostato su `true`, `"t"` o `1`, una versione compressa del Post precedente nel thread di conversazione non verrà visualizzata quando il Post richiesto è una risposta a un altro Post. |
    | `omit_script`  <br />Boolean, String o Int | `false` | Se impostato su `true`, `"t"` o `1`, il `<script>` responsabile del caricamento di `widgets.js` non verrà restituito. Le tue pagine web dovrebbero includere un proprio riferimento a `widgets.js` da utilizzare in tutti i widget di X, inclusi i [Post incorporati](https://developer.x.com/web/embedded-tweets). |
    | `align`  <br />Enum `{left,right,center,none}` | `none` | Specifica se il Post incorporato deve essere allineato a sinistra, a destra o al centro nella pagina rispetto all’elemento padre. |
    | `lang`  <br />Enum([Lingua](https://developer.x.com/en/docs/twitter-for-websites/twitter-for-websites-supported-languages/overview)) | `en` | Richiede l’HTML restituito e un Post renderizzato nella [lingua di X supportata dai Post incorporati](https://developer.x.com/web/overview/languages) specificata. |
    | `theme`  <br />Enum `{light, dark}` | `light` | Se impostato su `dark`, il Post viene visualizzato con testo chiaro su sfondo scuro. |
    | `dnt`  <br />Boolean | `false` | Se impostato su `true`, il Post e la relativa pagina incorporata sul tuo sito non vengono utilizzati per finalità che includono [suggerimenti personalizzati](https://support.x.com/articles/20169421) e [annunci personalizzati](https://support.x.com/articles/20170405). |

    ```bash
    twurl -H publish.x.com "/oembed?url=https://x.com/TwitterDev"
    ```
