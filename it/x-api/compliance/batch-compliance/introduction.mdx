X è impegnata nei confronti della nostra community di sviluppatori che utilizzano l’API X. Come parte di questo impegno, puntiamo a rendere la nostra API aperta ed equa per gli sviluppatori, sicura per le persone su X e vantaggiosa per l’intera piattaforma X. È fondamentale che qualsiasi sviluppatore che archivia contenuti di X offline si assicuri che i dati riflettano l’intento degli utenti e lo stato attuale dei contenuti su X. Ad esempio, quando qualcuno su X elimina un Post o il proprio account, protegge i propri Post o rimuove le informazioni geografiche dai propri Post, è fondamentale sia per X sia per i nostri sviluppatori rispettare le aspettative e l’intento di quella persona. Gli endpoint di conformità batch forniscono agli sviluppatori uno strumento semplice per contribuire a mantenere i dati di X conformi al [Contratto e alle Norme per sviluppatori di X](https://developer.x.com/en/developer-terms/policy). 

Questi endpoint di conformità batch consentono di caricare grandi dataset di ID di Post o di utenti per recuperarne lo stato di conformità, al fine di determinare quali dati richiedono azioni per riportare i dataset in conformità. Si noti che l’uso degli endpoint di conformità batch è limitato ai suddetti casi d’uso e qualsiasi altro scopo è vietato e può comportare misure di enforcement.

In genere, lavorare con questo endpoint prevede 4 passaggi:

1. **Crea un job di conformità**
   È possibile specificare il tipo di job (con il valore tweets o users) per indicare se il dataset che si desidera caricare contiene ID di Post o ID di utenti. È possibile avere un job in parallelo per tipo di job in qualsiasi momento.
2. **Carica il tuo dataset su upload\_url**
   Successivamente, carichi il tuo dataset come file di testo semplice sull’upload\_url fornito, con ogni riga del file contenente un singolo ID di Post o ID di utente. L’upload\_url scade dopo 15 minuti.
3. **(Opzionale) Verifica lo stato del job**
   Puoi controllare lo stato del tuo job di conformità per vedere se è created, in\_progress, failed o complete.
4. **Scarica i risultati**
   Quando il tuo job è completo, puoi scaricare i risultati utilizzando il download\_url. Il download\_url scade dopo una settimana (dalla creazione del job).

   Questo risultato conterrà un insieme di oggetti JSON (un oggetto per riga). Ogni oggetto conterrà un ID di Post, la data di creazione del Post (utile per individuare i Post organizzati per data), l’azione richiesta, il motivo dell’azione di conformità e la data in cui l’utente è stato sospeso.

Riceverai i seguenti tipi di eventi di conformità nei risultati:

- deleted - indica che il Post o l’account utente è stato eliminato
- deactivated - indica che il Post o l’account utente è stato disattivato
- scrub\_geo - indica che le informazioni geografiche associate al Post o all’utente sono state rimosse
- protected - indica che l’account che ha creato il Post è diventato privato
- suspended - indica che l’account che ha creato il Post è stato sospeso

**Configurazione dell’account**

Per accedere a questi endpoint, avrai bisogno di:

- Un [account sviluppatore](https://developer.x.com/en/portal/petition/essential/basic-info) approvato.
- Di autenticarti utilizzando le chiavi e i token di una [developer App](/it/resources/fundamentals/developer-apps) che si trova all’interno di un [Project](/it/resources/fundamentals/projects). 

Scopri di più su come ottenere l’accesso agli endpoint dell’API X v2 nella nostra [guida introduttiva](/it/x-api/getting-started/getting-access).
