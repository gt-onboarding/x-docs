<div id="authentication">
  #### Autenticazione
</div>

Tutti gli endpoint dell’API X v2 richiedono di autenticare le richieste con un set di credenziali, anche note come chiavi e token. Puoi utilizzare OAuth 1.0a in contesto utente per autenticare le richieste a questo endpoint.

[OAuth 1.0a in contesto utente](/it/resources/fundamentals/authentication) significa che devi utilizzare un set di chiavi API e token di accesso dell’utente per effettuare una richiesta riuscita. I token di accesso devono essere associati all’utente per conto del quale stai effettuando la richiesta. Se desideri generare un set di token di accesso per un altro utente, quest’ultimo deve autorizzare la tua app utilizzando il [flusso OAuth a 3 passaggi](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens).

Tieni presente che OAuth 1.0a può essere complesso da utilizzare. Se non conosci questo metodo di autenticazione, ti consigliamo di usare una [libreria](/it/x-api/tools-and-libraries/overview) o uno strumento come Postman.

<div id="developer-portal-projects-and-developer-apps">
  #### Portale sviluppatori, Progetti e App per sviluppatori
</div>

Per ottenere un set di credenziali di autenticazione compatibili con gli endpoint dell’API X v2, devi [registrarti per un account sviluppatore](https://developer.x.com/en/portal/petition/essential/basic-info), configurare un [Progetto](/it/resources/fundamentals/projects) all’interno di tale account e creare un’[App per sviluppatori](/it/resources/fundamentals/developer-apps) all’interno di quel Progetto. Potrai quindi trovare le tue chiavi e i tuoi token nella tua App per sviluppatori.
 

<div id="rate-limits">
  #### Limiti di frequenza
</div>

Ogni giorno, molte migliaia di sviluppatori inviano richieste all’API X. Per gestire l’elevato volume di tali richieste, su ciascun endpoint vengono applicati [limiti di frequenza](/it/x-api/fundamentals/rate-limits) che determinano quante richieste puoi effettuare per conto della tua app o di un utente autenticato.

Questi endpoint sono soggetti a limiti a livello di utente, il che significa che l’utente autenticato per conto del quale stai effettuando la richiesta può chiamare l’endpoint solo un certo numero di volte attraverso qualsiasi App per sviluppatori.

La tabella seguente mostra i limiti di frequenza per ciascun endpoint.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Metodo HTTP** | **Limite di frequenza** |
| /2/users/:id/pinned\_lists | POST | 50 richieste ogni 15 minuti |
| /2/users/:id/pinned\_lists/:list\_id | DELETE | 50 richieste ogni 15 minuti |
| /2/users/:id/pinned\_lists | GET | 15 richieste ogni 15 minuti |

<div id="fields-and-expansions">
  #### Campi ed espansioni
</div>

L’endpoint GET dell’API X v2 consente di selezionare con precisione quali dati ottenere dall’API utilizzando strumenti chiamati `fields` ed `expansions`. Il parametro `expansions` permette di espandere gli oggetti referenziati nel payload. Ad esempio, recuperare le Liste appuntate consente di utilizzare le seguenti [expansions](/it/x-api/fundamentals/expansions):

- `owner_id`

Il parametro `fields` consente di scegliere esattamente quali [fields](/it/x-api/fundamentals/fields) dei vari oggetti dati si desidera ricevere. Questo endpoint restituisce principalmente oggetti utente. Per impostazione predefinita, l’oggetto List restituisce i campi `id` e `name`. Per ottenere campi aggiuntivi come `list.created_at` o `list.description`, è necessario richiederli esplicitamente tramite il parametro fields.

Abbiamo aggiunto una guida su come usare insieme [fields ed expansions](/it/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) nella nostra [data dictionary dell’API X v2](/it/x-api/fundamentals/data-dictionary).
