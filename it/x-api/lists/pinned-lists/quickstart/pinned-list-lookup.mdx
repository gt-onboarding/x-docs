<div id="steps-to-build-a-pinned-list-lookup-request">
  ### Passaggi per creare una richiesta di ricerca per una Lista appuntata
</div>

**Primo passaggio: scegli la raccolta di endpoint List in Postman**

Una volta caricata la raccolta API X v2 in Postman, vai alla cartella “List”, seleziona la cartella “Pinned Lists”, quindi scegli "User's pinned Lists".
 

**Secondo passaggio: identifica e specifica l'utente**

Per recuperare la Lista appuntata di un utente, devi specificare il suo ID utente all'interno della richiesta. L'ID utente deve corrispondere all'ID dell'utente autenticato, il che significa che devi passare gli Access Token associati a quell'ID quando autentichi la richiesta. In questo esempio, puoi specificare l'ID del tuo stesso account. Puoi trovare il tuo ID in due modi:

1. Utilizzando l'endpoint [users lookup](/it/x-api/users/lookup/introduction) by username, puoi passare uno username e ricevere il campo id.
2. Esaminando il tuo Access Token, noterai che la parte numerica corrisponde al tuo ID utente.

In Postman, vai alla scheda "Params" e inserisci questo ID utente nella colonna "Value" del parametro `id`.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 2244994945 (ID utente) |

**Terzo passaggio: identifica e specifica quali campi desideri recuperare**

Se fai clic sul pulsante "Send" dopo il terzo passaggio, riceverai nella risposta i campi predefiniti dell'[oggetto List](/it/x-api/fundamentals/data-dictionary#list): `id` e `name`.

Se desideri ricevere campi aggiuntivi oltre a `id` e `name`, dovrai specificarli nella richiesta con i parametri [`field`](/it/x-api/fundamentals/fields) e/o [`expansion`](/it/x-api/fundamentals/expansions).

Per questo esercizio, richiederemo tre set aggiuntivi di campi da oggetti diversi:

- Il campo aggiuntivo `follower_count` nell'oggetto List principale.
- L'[oggetto user](/it/x-api/fundamentals/data-dictionary#user) completo utilizzando il parametro di espansione.
- Il campo aggiuntivo `tweet.created_at` nell'oggetto user associato.

In Postman, vai alla scheda "Params" e aggiungi la seguente coppia chiave:valore alla tabella "Query Params":

|     |     |     |
| :--- | :--- | :--- |
| Key | Value | Returned fields |
| list.fields | follower\_count | `follower_count` |
| expansions | owner\_id | `includes.users.id,  <br/>includes.users.name,  <br/>includes.users.username` |
| user.fields | created\_at | `includes.users.created_at` |

Ora dovresti vedere un URL simile accanto al pulsante “Send”:

`https://api.x.com/2/users/2244994945/pinned_lists?expansions=owner_id&list.fields=follower_count&user.fields=created_at`

**Quarto passaggio: invia la richiesta e verifica la risposta**

Una volta impostato tutto, premi il pulsante "Send" e riceverai una risposta simile al seguente esempio:

```
{
  "data": [
    {
      "follower_count": 0,
      "id": "1454155907651158017",
      "name": "test list",
      "owner_id": "2244994945"
    }
  ],
  "includes": {
    "users": [
      {
        "username": "TwitterDev",
        "id": "2244994945",
        "created_at": "2013-12-14T04:35:55.000Z",
        "name": "Twitter Dev"
      }
    ]
  },
  "meta": {
    "result_count": 1
  }
}
```
