<div id="getting-started-with-the-pinned-list-endpoint-group">
  ## Guida introduttiva al gruppo di endpoint delle Liste fissate
</div>

Questa breve panoramica ti aiuterà a inviare la tua prima richiesta agli endpoint delle Liste fissate utilizzando [Postman](/it/tutorials/postman-getting-started).

Visita il nostro repository GitHub [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) se vuoi vedere esempi di codice in diverse lingue.

<Note>
  ### Prerequisiti

  Per completare questa guida, avrai bisogno di un set di [keys and tokens](/it/resources/fundamentals/authentication) per autenticare la tua richiesta. Puoi generare queste chiavi e token seguendo questi passaggi:

  - [Registrati per un account sviluppatore](https://developer.x.com/en/apply-for-access) e attendi l’approvazione.
  - Crea un [Project](/it/resources/fundamentals/projects) e una [developer App](/it/resources/fundamentals/developer-apps) associata nel developer portal.
  - Vai alla pagina “Keys and tokens” della tua App per generare le credenziali necessarie. Assicurati di salvare tutte le credenziali in un luogo sicuro.
</Note>

Carica la collection Postman dell’API X v2

Per aggiungere la collection Postman dell’API X v2 al tuo workspace, fai clic sul pulsante seguente:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Aggiungi API X v2 a Postman
</Button>

 

<div id="authenticate-your-request">
  #### Autentica la tua richiesta
</div>

Per inviare correttamente una richiesta agli endpoint di **lookup**, puoi usare [OAuth 1.0a User Context](/it/resources/fundamentals/authentication), [OAuth 2.0 App-Only](/it/resources/fundamentals/authentication#oauth-2-0) o [OAuth 2.0 Authorization Code with PKCE](it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). Con gli endpoint di **manage**, invece, puoi autenticarti solo con OAuth 1.0a User Context o OAuth 2.0 Authorization Code with PKCE.

In ogni caso, quando utilizzi Postman, le chiavi e i token di autenticazione predefiniti verranno compilati automaticamente nelle richieste, purché tu abbia configurato correttamente l’ambiente.

Puoi farlo selezionando l’ambiente denominato “API X v2” (nell’angolo in alto a destra di Postman) e aggiungendo le tue chiavi e i tuoi token nei campi “initial value” e “current value” (facendo clic sull’icona a forma di occhio accanto al menu a discesa dell’ambiente). Queste chiavi includono:

- API Key (nota anche come Consumer Key)
- API Secret Key (nota anche come Consumer Secret)
- OAuth 1.0a User Access Token
- OAuth 1.0a User Access Token Secret
- OAuth 2.0 App Access Token
- OAuth 2.0 Client Key (disponibile solo se hai configurato le impostazioni di OAuth 2.0 User Authentication nell’App)
- OAuth 2.0 Client Secret (disponibile solo se hai configurato le impostazioni di OAuth 2.0 User Authentication nell’App)
