Fissare le liste consente agli utenti di X di accedere più facilmente alle liste a cui si sono iscritti o che hanno creato.

Poiché con questi endpoint stai effettuando richieste per conto di un utente, devi autenticare tale utente con [OAuth 1.0a User Context](/it/resources/fundamentals/authentication) o con [OAuth 2.0 Authorization Code con PKCE](it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) e utilizzare i token di accesso dell’utente associati a un utente che ha autorizzato la tua App. Tali token possono essere generati usando il [flusso OAuth a 3 passaggi](/it/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) oppure il [flusso di concessione Authorization Code con PKCE](/it/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0).

<div id="pinned-list-lookup">
  ### Ricerca delle Liste fissate
</div>

La ricerca delle Liste fissate dispone di un unico endpoint che consente di recuperare le Liste fissate dell’utente autenticato. Per questo endpoint è previsto un limite di 15 richieste ogni 15 minuti.
 

<div id="manage-pinned-lists">
  ### Gestire le Liste fissate
</div>

Gli endpoint per gestire le Liste fissate consentono di fissare e rimuovere la fissazione di una Lista per conto di un utente autenticato. Per questi endpoint sono disponibili due metodi: POST e DELETE. Il metodo POST consente di fissare una Lista, mentre il metodo DELETE consente di rimuovere la fissazione di una Lista. È previsto un limite di 50 richieste per utente ogni 15 minuti per entrambi gli endpoint.

<Note>
  **Configurazione dell’account**

  Per accedere a questi endpoint, sono necessari:

  - Un [account sviluppatore](https://developer.x.com/en/portal/petition/essential/basic-info) approvato.
  - L’autenticazione utilizzando le chiavi e i token di una [App sviluppatore](/it/resources/fundamentals/developer-apps) che si trova all’interno di un [Progetto](/it/resources/fundamentals/projects). 

  Scopri di più su come ottenere l’accesso agli endpoint dell’API X v2 nella nostra [guida introduttiva](/it/x-api/getting-started/getting-access).
</Note>
