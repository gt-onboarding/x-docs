<div id="authentication">
  #### Autenticazione
</div>

Tutti gli endpoint dell’API X v2 richiedono di autenticare le richieste con un set di credenziali, note anche come chiavi e token. Puoi utilizzare OAuth 1.0a User Context, App only oppure OAuth 2.0 Authorization Code con PKCE per autenticare le tue richieste a questi endpoint.

[OAuth 1.0a User Context](/it/resources/fundamentals/authentication#oauth-1-0a-user-context) richiede l’uso di un set di API Key e di Access Token utente per effettuare correttamente una richiesta. Gli access token devono essere associati all’utente per conto del quale stai effettuando la richiesta. Se desideri generare un set di Access Token per un altro utente, quest’ultimo deve autorizzare la tua App utilizzando il [flusso OAuth a 3 passaggi](/it/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow).

Tieni presente che OAuth 1.0a può risultare complesso. Se non conosci questo metodo di autenticazione, ti consigliamo di usare una [libreria](/it/x-api/tools-and-libraries/overview), uno strumento come Postman oppure OAuth 2.0 per autenticare le richieste.

[OAuth 2.0 Authorization Code con PKCE](/it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) consente un controllo più granulare sull’ambito di un’applicazione e flussi di autorizzazione su più dispositivi. OAuth 2.0 permette di scegliere ambiti specifici e di dettaglio che ti concedono permessi puntuali per conto di un utente.

Per abilitare OAuth 2.0 nella tua App, attivalo nelle impostazioni di autenticazione della tua App, presenti nella sezione Impostazioni dell’App del developer portal.

[App only](/it/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) richiede semplicemente di includere un [Access Token App only](/it/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) nella richiesta. Puoi generare un token App only direttamente all’interno di una developer App oppure generarne uno utilizzando l’endpoint [POST oauth2/token](/it/resources/fundamentals/authentication#post-oauth2-token).

<div id="developer-portal-projects-and-developer-apps">
  #### Portale per sviluppatori, Progetti e App per sviluppatori
</div>

Per ottenere un set di credenziali di autenticazione compatibili con gli endpoint dell’API X v2, devi [iscriverti a un account sviluppatore](https://developer.x.com/en/portal/petition/essential/basic-info), configurare un [Progetto](/it/resources/fundamentals/projects) all’interno di tale account e creare un’[App per sviluppatori](/it/resources/fundamentals/developer-apps) all’interno di quel Progetto. Potrai quindi trovare le tue chiavi e i tuoi token all’interno della tua App per sviluppatori.
 

<div id="rate-limits">
  #### Limiti di velocità
</div>

Ogni giorno, molte migliaia di sviluppatori inviano richieste all’API X. Per gestire l’enorme volume di queste richieste, sui [limiti di velocità](/it/x-api/fundamentals/rate-limits) vengono applicati vincoli a ciascun endpoint che limitano il numero di richieste che puoi effettuare per conto della tua app o di un utente autenticato. 

Questi endpoint sono soggetti a limiti sia a livello di App sia a livello di utente. Il limite a livello di App significa che tu, sviluppatore, puoi effettuare solo un certo numero di richieste a questo endpoint in un determinato periodo di tempo da qualsiasi App (autenticata usando l’API Key e l’API Secret Key, oppure l’App-only Access Token). Il limite a livello di utente significa che l’utente autenticato per conto del quale stai effettuando la richiesta può eseguire una ricerca di Liste solo un certo numero di volte tramite qualsiasi App per sviluppatori.

La tabella seguente mostra i limiti di velocità per ciascun endpoint.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Metodo HTTP** | **Limite di velocità** |
| /2/lists/:id | GET | 75 richieste per 15 minuti |
| /2/users/:id/owned\_lists | GET | 15 richieste per 15 minuti |

Campi ed espansioni

L’endpoint GET dell’API X v2 consente di selezionare esattamente quali dati ottenere dall’API utilizzando un set di strumenti chiamati `fields` ed `expansions`. Il parametro `expansions` consente di espandere gli oggetti referenziati nel payload. Ad esempio, cercando una Lista per ID è possibile recuperare le seguenti [espansioni](/it/x-api/fundamentals/expansions):

- `owner_id`

Il parametro `fields` consente di selezionare esattamente quali [campi](/it/x-api/fundamentals/fields) all’interno dei diversi oggetti dati desideri ricevere. Questi endpoint restituiscono principalmente oggetti List. Per impostazione predefinita, l’oggetto List restituisce i campi `id` e `name`. Per ricevere campi aggiuntivi come `list.created_at` o `list.follower_count`, dovrai richiederli in modo esplicito utilizzando il parametro list.fields. 

Abbiamo aggiunto una guida sull’uso congiunto di [fields ed expansions](/it/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) al nostro [dizionario dati dell’API X v2](/it/x-api/fundamentals/data-dictionary).

 La tabella seguente mostra i campi e le espansioni disponibili per questo gruppo di endpoint:

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Campi** | **Espansioni** |
| /2/lists/:id | `list.fields`<br /><br />`user.fields` | `owner_id` |
| /2/users/:id/owned\_lists | list.fields<br /><br />user.fields | owner\_id |

<div id="pagination">
  #### Paginazione
</div>

La consultazione delle Liste di proprietà di un utente può restituire molti dati. Per garantire risultati coerenti e performanti in ogni momento, utilizziamo la paginazione. La paginazione è una funzionalità degli endpoint dell’API X v2 che restituiscono più risultati di quanti se ne possano includere in una singola risposta. Quando ciò accade, i dati vengono restituiti in una serie di “pagine”. Scopri di più su come [paginare i risultati.](/it/x-api/fundamentals/pagination)
