<div id="getting-started-with-the-list-lookup-endpoint">
  ## Guida introduttiva all’endpoint di ricerca Liste
</div>

Questa guida rapida ti aiuterà a inviare la tua prima richiesta all’endpoint di ricerca Liste usando Postman.

Visita il nostro repository GitHub [API X v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) se vuoi vedere esempi di codice in diverse lingue.

**Nota:** in questo esempio invieremo una richiesta all’endpoint _Ricerca Lista per ID_, ma puoi applicare quanto imparato in questa guida rapida anche ad altre richieste di lookup.

<Note>
  ### Prerequisiti

  Per completare questa guida, ti servirà un set di [keys and tokens](/it/resources/fundamentals/authentication) per autenticare la richiesta. Puoi generare queste chiavi e token seguendo questi passaggi:

  - [Richiedi un account sviluppatore](https://developer.x.com/en/apply-for-access) e attendi l’approvazione.
  - Crea un [Project](/it/resources/fundamentals/projects) e la relativa [developer App](/it/resources/fundamentals/developer-apps) nel portale per sviluppatori.
  - Vai alla pagina “Keys and tokens” della tua App per generare le credenziali necessarie. Assicurati di salvare tutte le credenziali in un luogo sicuro.
</Note>

<div id="steps-to-build-a-list-lookup-request">
  ### Passaggi per creare una richiesta di lookup delle Liste
</div>

**Primo passaggio: inizia con uno strumento o una libreria**

Esistono diversi strumenti, esempi di codice e librerie che puoi utilizzare per effettuare una richiesta a questo endpoint; qui useremo Postman per semplificare il processo.

Per caricare la collection Postman dell’API X v2 nel tuo ambiente, fai clic sul seguente pulsante:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Aggiungi API X v2 a Postman
</Button>

Una volta caricata la collection dell’API X v2 in Postman, vai alla cartella “List”, seleziona la sottocartella “List lookup” e quindi scegli "List by ID".
 

**Secondo passaggio: autentica la tua richiesta**

Per effettuare correttamente una richiesta all’API X, devi verificare di avere le autorizzazioni. Per farlo con questo endpoint, devi autenticare la richiesta con uno dei metodi: [App only](/it/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), [OAuth 2.0 Authorization Code con PKCE](it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) oppure [OAuth 1.0a User Context](/it/resources/fundamentals/authentication).

Per semplicità, in questa richiesta useremo App only; se però desideri richiedere [metriche](/it/x-api/fundamentals/metrics) private o Liste private, dovrai utilizzare uno degli altri metodi di autenticazione. 

Per utilizzare App only, devi aggiungere le tue chiavi e i tuoi token (in particolare l’[App only Access Token](/it/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)) a Postman selezionando l’ambiente “X API v2” (nell’angolo in alto a destra di Postman) e inserendo chiavi e token nei campi "initial value" e "current value" (facendo clic sull’icona a forma di occhio accanto al menu a discesa dell’ambiente).

Se lo hai fatto correttamente, queste variabili verranno compilate automaticamente nella scheda di autorizzazione della richiesta.
 

**Terzo passaggio: identifica e specifica quale Lista desideri recuperare**

Devi indicare la Lista che vuoi recuperare nella richiesta. Puoi trovare l’ID Lista navigando su x.com, facendo clic su una Lista e poi guardando l’URL. Ad esempio, l’ID Lista nel seguente URL è 84839422.

https://x.com/i/lists/84839422

L’ID di destinazione può essere qualsiasi ID Lista valido. In Postman, vai alla scheda "Params" e inserisci il tuo ID nella colonna "Value" della path variable id. Assicurati di non includere spazi prima o dopo l’ID.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| id  | 84839422 (L’ID della Lista) |

**Quarto passaggio: identifica e specifica quali campi desideri recuperare**

Se fai clic sul pulsante "Send" dopo il terzo passaggio, nella risposta riceverai i campi predefiniti del [List object](/it/x-api/fundamentals/data-dictionary#list): id, name.

Se desideri ricevere campi aggiuntivi, devi specificarli nella richiesta con i parametri list.fields e/o [expansions](/it/x-api/fundamentals/expansions).

Per questo esercizio, richiederemo tre set aggiuntivi di campi da oggetti diversi:

- Il campo aggiuntivo created\_at nell’oggetto principale List.

- L’[oggetto utente](/it/x-api/fundamentals/data-dictionary#user) completo utilizzando il parametro expansions

- Il campo aggiuntivo user.created\_at nell’oggetto utente associato.

In Postman, vai alla scheda "Params" e aggiungi la seguente coppia chiave:valore alla tabella "Query Params":

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| list.fields | created\_at | created\_at |
| expansions | owner\_id | includes.users.id,  <br />includes.users.name,  <br />includes.users.username |
| user.fields | created\_at | includes.users.created\_at |

Ora dovresti vedere un URL simile accanto al pulsante “Send”:

`https://api.x.com/2/lists/84839422?list.fields=owner_id&expansions=owner_id&user.fields=created_at`

**Quinto passaggio: invia la richiesta e verifica la risposta**

Quando hai configurato tutto, premi "Send" e riceverai una risposta simile al seguente esempio:

```
{
  "data": {
    "id": "84839422",
    "name": "Official Twitter Accounts",
    "owner_id": "783214"
  },
  "includes": {
    "users": [
      {
        "name": "Twitter",
        "created_at": "2007-02-20T14:35:54.000Z",
        "username": "Twitter",
        "id": "783214"
      }
    ]
  }
}
```
