<div id="authentication">
  #### Autenticazione
</div>

Tutti gli endpoint dell’API X v2 richiedono di autenticare le richieste con un set di credenziali, note anche come chiavi e token. Puoi utilizzare OAuth 1.0a User Context, App only oppure OAuth 2.0 Authorization Code con PKCE per autenticare le tue richieste a questo endpoint.

[OAuth 1.0a User Context](/it/resources/fundamentals/authentication#oauth-1-0a-2), il che significa che devi utilizzare un set di API Key e token di accesso utente per effettuare correttamente una richiesta. I token di accesso devono essere associati all’utente per conto del quale stai effettuando la richiesta. Se desideri generare un set di token di accesso per un altro utente, quest’ultimo deve autorizzare la tua App utilizzando il [flusso OAuth a 3 passaggi](/it/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow).

Tieni presente che OAuth 1.0a può essere complesso da usare. Se non conosci questo metodo di autenticazione, ti consigliamo di utilizzare una [libreria](/it/x-api/tools-and-libraries/overview), uno strumento come Postman, oppure di usare OAuth 2.0 o App only per autenticare le tue richieste.

[OAuth 2.0 Authorization Code con PKCE](/it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) consente un maggiore controllo sull’ambito di un’applicazione e flussi di autorizzazione su più dispositivi. OAuth 2.0 ti consente di scegliere ambiti specifici e granulari che ti conferiscono permessi specifici per conto di un utente.

Per abilitare OAuth 2.0 nella tua App, devi abilitarlo nelle impostazioni di autenticazione della tua App, nella sezione Impostazioni dell’App del developer portal.

[App only](/it/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) richiede semplicemente di includere un [token di accesso App only](/it/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) nella richiesta. Puoi generare un token di accesso App only direttamente all’interno di un’App per sviluppatori oppure generarne uno utilizzando l’endpoint [POST oauth2/token](/it/resources/fundamentals/authentication#post-oauth2-token).

<div id="developer-portal-projects-and-developer-apps">
  #### Portale per sviluppatori, Progetti e App per sviluppatori
</div>

Per ottenere un set di credenziali di autenticazione compatibili con gli endpoint dell’API X v2, devi [registrarti per un account sviluppatore](https://developer.x.com/en/portal/petition/essential/basic-info), configurare un [Progetto](/it/resources/fundamentals/projects) all’interno di tale account e creare un’[App per sviluppatori](/it/resources/fundamentals/developer-apps) all’interno di quel Progetto. Potrai quindi trovare le tue chiavi e i tuoi token all’interno della tua App per sviluppatori.
 

<div id="rate-limits">
  #### Limiti di frequenza
</div>

Ogni giorno, molte migliaia di sviluppatori inviano richieste all’API X. Per gestire l’elevato volume di queste richieste, su ciascun endpoint vengono applicati [limiti di frequenza](https://developer.x.com/x-api/fundamentals/rate-limits) che limitano il numero di richieste che puoi effettuare per conto della tua app o di un utente autenticato. 

Questo endpoint è soggetto a limiti sia a livello di App sia a livello di utente. Il limite a livello di App significa che tu, sviluppatore, puoi effettuare solo un certo numero di richieste a questo endpoint in un determinato periodo di tempo da qualsiasi App (utilizzando la API Key e la API Secret Key, oppure il Bearer Token). Il limite a livello di utente significa che l’utente autenticato per conto del quale stai effettuando la richiesta può eseguire una ricerca di Post in una Lista solo un numero limitato di volte, indipendentemente dall’App per sviluppatori utilizzata.

La tabella seguente mostra i limiti di frequenza per ciascun endpoint.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Metodo HTTP** | **Limite di frequenza** |
| /2/lists/:id/tweets | GET | 900 richieste ogni 15 minuti |

Campi ed espansioni

L’endpoint GET dell’API X v2 consente di selezionare con precisione quali dati restituire dall’API tramite un insieme di strumenti chiamati `fields` ed `expansions`. Il parametro `expansions` consente di espandere gli oggetti referenziati nel payload. Ad esempio, il lookup dei Post in una Lista consente di recuperare le seguenti [espansioni](/it/x-api/fundamentals/expansions):

- `author_id`

Il parametro `fields` consente di selezionare esattamente quali [campi](/it/x-api/fundamentals/fields) all’interno dei diversi oggetti dati desideri ricevere. Questo endpoint restituisce principalmente oggetti Post. Per impostazione predefinita, l’oggetto Post restituisce i campi `id` e `text`. Per ricevere campi aggiuntivi come `tweet.created_at` o `tweet.lang`, devi richiederli esplicitamente tramite un parametro fields. 

Abbiamo aggiunto una guida sull’uso congiunto di [fields ed expansions](/it/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) nella nostra [data dictionary dell’API X v2](/it/x-api/fundamentals/data-dictionary).

 La tabella seguente mostra i campi e le espansioni disponibili per l’endpoint di lookup:

|     |     |     |

| **Endpoint** | **Campi** | **Espansioni** |
| /2/lists/:id/tweets | `tweet.fields`<br /><br />`user.fields` | `author_id` |

Paginazione

Il lookup dei Post in una Lista può restituire grandi volumi di dati. Per garantire risultati coerenti e ad alte prestazioni in qualsiasi momento, utilizziamo la paginazione. La paginazione è una funzionalità degli endpoint dell’API X v2 che restituiscono più risultati di quanti possano essere inclusi in una singola risposta. Quando ciò accade, i dati vengono restituiti in una serie di “pagine”. Scopri di più su come [paginare i risultati.](/it/x-api/fundamentals/pagination)
