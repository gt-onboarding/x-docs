<div id="authentication">
  #### Autenticazione
</div>

Tutti gli endpoint dell’API X v2 richiedono di autenticare le richieste con un set di credenziali, note anche come chiavi e token. Per gli endpoint di **lookup** delle Liste puoi utilizzare OAuth 1.0a User Context, OAuth 2.0 Authorization Code con PKCE o App only per autenticare le richieste. Tuttavia, per gli endpoint di **manage** delle Liste devi autenticarti con OAuth 1.0a User Context o OAuth 2.0.

[OAuth 1.0a User Context](/it/resources/fundamentals/authentication) implica l’uso di un set di API Key e Access Token utente per effettuare una richiesta con esito positivo. Gli access token devono essere associati all’utente per conto del quale stai effettuando la richiesta. Se desideri generare un set di Access Token per un altro utente, quest’ultimo deve autorizzare la tua App utilizzando il [flusso OAuth a 3 passaggi](/it/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow).

Tieni presente che OAuth 1.0a può essere complesso da usare. Se non conosci questo metodo di autenticazione, ti consigliamo di usare una [libreria](/it/x-api/tools-and-libraries/overview), uno strumento come Postman o di utilizzare OAuth 2.0 o App only per autenticare le richieste.

[OAuth 2.0 Authorization Code con PKCE](/it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) consente un maggiore controllo sugli scope di un’applicazione e flussi di autorizzazione su più dispositivi. OAuth 2.0 ti permette di scegliere scope specifici e granulare che concedono permessi mirati per conto di un utente.

Per abilitare OAuth 2.0 nella tua App, devi abilitarlo nelle impostazioni di autenticazione della tua App, nella sezione delle impostazioni dell’App del portale sviluppatori.

[App only](/it/resources/fundamentals/authentication#oauth-2-0) richiede semplicemente di includere un [App only Access Token](/it/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) nella richiesta. Puoi generare un App only Access Token direttamente all’interno di un’App per sviluppatori oppure generarlo utilizzando l’endpoint [POST oauth2/token](/it/resources/fundamentals/authentication#post-oauth2-token).

[App only](/it/resources/fundamentals/authentication#oauth-2-0) richiede semplicemente di includere un [App only Access Token](/it/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) nella richiesta. Puoi generare un App only Access Token direttamente all’interno di un’App per sviluppatori oppure generarlo utilizzando l’endpoint [POST oauth2/token](/it/resources/fundamentals/authentication#post-oauth2-token).

<div id="developer-portal-projects-and-developer-apps">
  #### Portale sviluppatori, Progetti e App per sviluppatori
</div>

Per ottenere un set di credenziali di autenticazione compatibili con gli endpoint dell’API X v2, devi [registrarti per un account sviluppatore](https://developer.x.com/en/portal/petition/essential/basic-info), configurare un [Progetto](/it/resources/fundamentals/projects) all’interno di tale account e creare un’[App per sviluppatori](/it/resources/fundamentals/developer-apps) all’interno di quel Progetto. Potrai quindi trovare le tue chiavi e i tuoi token all’interno della tua App per sviluppatori.
 

<div id="rate-limits">
  #### Limiti di frequenza
</div>

Ogni giorno, molte migliaia di sviluppatori inviano richieste all’API X. Per gestire l’elevato volume di tali richieste, su ciascun endpoint vengono applicati [limiti di frequenza](/it/x-api/fundamentals/rate-limits) che limitano il numero di richieste che puoi effettuare per conto della tua app o di un utente autenticato.

Gli endpoint di consultazione (GET) sono soggetti a limiti sia a livello di App sia a livello utente, mentre gli endpoint di gestione (POST/DELETE) sono limitati a livello utente. Il limite a livello di App significa che tu, sviluppatore, puoi effettuare solo un certo numero di richieste a questo endpoint in un determinato periodo da qualsiasi App (utilizzando la API Key e la API Secret Key o l’App-only Access Token). Il limite a livello utente significa che l’utente autenticato per conto del quale stai effettuando la richiesta può eseguire l’operazione di lookup della Lista solo un numero limitato di volte attraverso qualsiasi App per sviluppatori.

La tabella seguente mostra i limiti di frequenza per ciascun endpoint.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Metodo HTTP** | **Limite di frequenza** |
| /2/lists/:id/members | GET | 900 richieste ogni 15 minuti |
| /2/users/:id/list\_memberships | GET | 75 richieste ogni 15 minuti |
| /2/lists/:id/members | POST | 300 richieste ogni 15 minuti |
| /2/lists/:id/members/:user\_id | DELETE | 300 richieste ogni 15 minuti |

<div id="fields-and-expansions">
  #### Campi ed espansioni
</div>

L’endpoint GET dell’API X v2 consente di selezionare con precisione quali dati ottenere dall’API utilizzando due strumenti: `fields` ed `expansions`. Il parametro `expansions` consente di includere nel risultato gli oggetti referenziati nel payload. Ad esempio, l’interrogazione dei membri di una Lista permette di richiedere le seguenti [expansions](/it/x-api/fundamentals/expansions):

- `pinned_tweet_id`

Il parametro `fields` consente di specificare esattamente quali [fields](/it/x-api/fundamentals/fields) dei vari oggetti dati si desidera ricevere. L’interrogazione dei membri della Lista restituisce principalmente oggetti utente. Per impostazione predefinita, l’oggetto utente include i campi id, name e username. Per ricevere campi aggiuntivi come `user.created_at` o `user.description`, è necessario richiederli esplicitamente utilizzando il parametro user.fields. 

Abbiamo aggiunto una guida all’uso di [fields and expansions](/it/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions).

 La tabella seguente mostra i campi e le espansioni disponibili per ciascun endpoint di lookup:

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Fields** | **Expansions** |
| /2/lists/:id/members | `user.fields`<br /><br />`tweet.fields` | `pinned_tweet_id` |
| /2/users/:id/list\_memberships | `list.fields`<br /><br />`user.fields` | `owner_id` |

<div id="pagination">
  #### Paginazione
</div>

La consultazione di membership/members può restituire una grande quantità di dati. Per garantire risultati coerenti e performanti in qualsiasi momento, utilizziamo la paginazione. La paginazione è una funzionalità degli endpoint dell’API X v2 che restituiscono più risultati di quanti ne possano essere inclusi in una singola risposta. In questi casi, i dati vengono forniti in una serie di “pagine”. Scopri di più su come [paginare i risultati.](/it/x-api/fundamentals/pagination)
