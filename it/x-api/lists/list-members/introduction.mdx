<div id="list-members-lookup">
  ### Ricerca dei membri della Lista
</div>

Il gruppo di endpoint per la ricerca dei membri offre due endpoint. È possibile ottenere i dettagli sui membri di una specifica Lista e vedere a quali Liste appartiene un utente. Questi endpoint possono essere usati per aiutare le persone a curare e organizzare nuove Liste in base alle informazioni di appartenenza.

È previsto un limite di 900 richieste ogni 15 minuti per la ricerca dei dettagli sui membri e un limite di 75 richieste ogni 15 minuti per la ricerca delle appartenenze degli utenti.

Puoi utilizzare [OAuth 1.0a User Context](/it/resources/fundamentals/authentication), [App only](https://developer.x.com/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) o [OAuth 2.0 Authorization Code with PKCE](it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) per autenticare le richieste a questo endpoint. 
 

<div id="manage-list-members">
  ### Gestire i membri della Lista
</div>

Gli endpoint per la gestione dei membri della Lista consentono di aggiungere e rimuovere membri da una Lista per conto di un utente autenticato. Per questi endpoint sono disponibili due metodi: POST e DELETE. Il metodo POST consente di aggiungere un membro a una Lista, mentre il metodo DELETE consente di rimuoverlo. È previsto un limite per utente di 300 richieste ogni 15 minuti per entrambi gli endpoint.

Nota: le Liste non possono avere più di 5.000 membri.

Poiché effettui richieste per conto di un utente con questi endpoint, devi autenticare l’utente con [OAuth 1.0a User Context](/it/resources/fundamentals/authentication) oppure con [OAuth 2.0 Authorization Code con PKCE](it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) e utilizzare i token di accesso associati a un utente che ha autorizzato la tua App, generabili tramite il [flusso OAuth a 3 passaggi](/it/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) o il [flusso Authorization Code con PKCE](/it/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0).

<Note>
  **Configurazione dell’account**

  Per accedere a questi endpoint, ti serviranno:

  - Un [account sviluppatore](https://developer.x.com/en/portal/petition/essential/basic-info) approvato.
  - Autenticarti utilizzando le chiavi e i token di una [developer App](/it/resources/fundamentals/developer-apps) che si trova all’interno di un [Project](/it/resources/fundamentals/projects).

  Per saperne di più su come ottenere l’accesso agli endpoint dell’API X v2, consulta la nostra [guida introduttiva](/it/x-api/getting-started/getting-access).
</Note>
