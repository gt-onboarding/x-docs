<div id="manage-list-members-standard-v11-compared-to-x-api-v2">
  ### Gestione dei membri delle Liste: Standard v1.1 rispetto a API X v2
</div>

Se hai utilizzato gli endpoint standard v1.1 [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create) e [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy), l'obiettivo di questa guida è aiutarti a comprendere somiglianze e differenze tra gli endpoint dello standard v1.1 e dell'API X v2 per la gestione dei membri delle Liste.

- **Somiglianze**
  - Autenticazione
- **Differenze**
  - URL degli endpoint
  - Requisiti di App e Progetto
  - Metodi HTTP
  - Limiti di richiesta
  - Parametri della richiesta

<div id="similarities">
  #### Somiglianze
</div>

**Autenticazione**

Entrambe le versioni dell’endpoint supportano [OAuth 1.0a User Context](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Pertanto, se in precedenza utilizzavi uno degli endpoint standard v1.1 per gestire i membri delle liste, puoi continuare a utilizzare lo stesso metodo di autenticazione passando alla versione API X v2.

<div id="differences">
  #### Differenze
</div>

**URL degli endpoint**

- Endpoint standard v1.1:
  - POST https://api.x.com/1.1/lists/members/create.json
    (Aggiunge un membro a una Lista specifica)
  - POST https://api.x.com/1.1/lists/members/destroy.json
    (Rimuove un membro da una Lista specifica)
- Endpoint API X v2:
  - POST https://api.x.com/2/lists/:id/members
    (Aggiunge un membro a una Lista specifica)

  - DELETE https://api.x.com/2/lists/:id/members/:user\_id
    (Rimuove un membro da una Lista specifica)

**Limiti di frequenza**

| **Standard v1.1** | **API X v2** |
| :--- | :--- |
| /1.1/lists/members/create.json<br /><br />nessuno | /2/lists/:id/members<br /><br />300 richieste per finestra di 15 minuti con OAuth 1.0a User Context<br /><br />300 richieste per finestra di 15 minuti con OAuth 2.0 Authorization Code con PKCE |
| /1.1/lists/members/destroy.json<br /><br />nessuno | /2/lists/:id/members/:user\_id<br /><br />300 richieste per finestra di 15 minuti con OAuth 1.0a User Context<br /><br />300 richieste per finestra di 15 minuti con OAuth 2.0 Authorization Code con PKCE |

**Requisiti per App e Progetti**

Gli endpoint dell’API X v2 richiedono l’uso di credenziali di un’[App per sviluppatori](/it/resources/fundamentals/developer-apps) associata a un [Progetto](/it/resources/fundamentals/projects) quando si autenticano le richieste. Tutti gli endpoint dell’API X v1.1 possono utilizzare credenziali di App autonome o di App collegate a un progetto.

**Parametri della richiesta**

I seguenti parametri di richiesta standard v1.1 hanno equivalenti nell’API X v2:

| **Standard v1.1** | **API X v2** |
| :--- | :--- |
| list\_id | id  |
| slug | Nessun equivalente |
| screen\_name | Nessun equivalente |
| owner\_screen\_name | Nessun equivalente |
| owner\_id | Nessun equivalente |
