<div id="comparing-x-apis-list-members-endpoints">
  ## Confronto degli endpoint dei membri delle Liste dell’API X
</div>

Il gruppo di endpoint v2 per i membri delle Liste sostituirà gli endpoint standard v1.1 [GET lists/members](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-members), [GET lists/memberships](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-memberships), [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create) e [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy). Se utilizzi codice, app o strumenti che si basano su una di queste versioni degli endpoint dei membri delle Liste e stai valutando la migrazione al nuovo endpoint dell’API X v2, questa serie di guide fa per te. 

<div id="list-members-lookup">
  ### Ricerca dei membri della lista
</div>

Gli endpoint v2 per la ricerca dei membri della lista sostituiranno gli endpoint standard [GET lists/members](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-members), [GET lists/memberships](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-memberships).

Le tabelle seguenti confrontano gli endpoint standard v1.1 e dell’API X v2 per le liste:

**Ricerca dei membri della lista**

|     |     |     |
| :--- | :--- | :--- |
| Descrizione | Standard v1.1 | API X v2 |
| Metodi HTTP supportati | `GET` | `GET` |
| Dominio host | `https://api.x.com` | `https://api.x.com` |
| Percorso endpoint | `/1.1/lists/members.json` | `/2/lists/:id/members` |
| [Autenticazione](/it/resources/fundamentals/authentication) | OAuth 1.0a (User Context)<br /><br />Solo app | OAuth 1.0a (User Context)<br /><br />OAuth 2.0 Authorization Code con PKCE<br /><br />Solo app |
| [Limiti di frequenza](/it/resources/fundamentals/rate-limits) predefiniti per le richieste | 900 richieste ogni 15 min con OAuth 1.0a<br /><br />75 richieste ogni 15 min con solo app | 900 richieste ogni 15 min con OAuth 1.0a<br /><br />900 richieste ogni 15 min con OAuth 2.0<br /><br />900 richieste ogni 15 min con solo app |

**Ricerca delle appartenenze alla lista**

|     |     |     |
| :--- | :--- | :--- |
| Descrizione | Standard v1.1 | API X v2 |
| Metodi HTTP supportati | `GET` | `GET` |
| Dominio host | `https://api.x.com` | `https://api.x.com` |
| Percorso endpoint | `/1.1/lists/memberships.json` | `/2/users/:id/list_memberships` |
| [Autenticazione](/it/resources/fundamentals/authentication) | OAuth 1.0a (User Context)<br /><br />Solo app | OAuth 1.0a (User Context)<br /><br />OAuth 2.0 Authorization Code con PKCE<br /><br />Solo app |
| [Limiti di frequenza](/it/resources/fundamentals/rate-limits) predefiniti per le richieste | 75 richieste ogni 15 min con OAuth 1.0a<br /><br />75 richieste ogni 15 min con solo app | 75 richieste ogni 15 min con OAuth 1.0a<br /><br />75 richieste ogni 15 min con OAuth 2.0<br /><br />75 richieste ogni 15 min con solo app |

<div id="manage-list-members">
  ### Gestione dei membri della lista
</div>

Gli endpoint v2 per la gestione dei membri della lista sostituiranno gli endpoint standard [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create), [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy).

Le tabelle seguenti confrontano gli endpoint delle liste tra lo standard v1.1 e l'API X v2:

**Aggiungi membro**

| Descrizione | Standard v1.1 | API X v2 |
| :--- | :--- | :--- |
| Metodi HTTP supportati | POST | POST |
| Dominio host | https://api.x.com | https://api.x.com |
| Percorso dell'endpoint | /1.1/lists/members/create.json | /2/lists/:id/members |
| [Autenticazione](/it/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code with PKCE |
| [Limiti di frequenza](/it/resources/fundamentals/rate-limits) predefiniti per le richieste | Nessuno | 300 richieste ogni 15 min (per utente) |

**Rimuovi membro**

| Descrizione | Standard v1.1 | API X v2 |
| :--- | :--- | :--- |
| Metodi HTTP supportati | POST | DELETE |
| Dominio host | https://api.x.com | https://api.x.com |
| Percorso dell'endpoint | /1.1/lists/members/destroy.json | /2/lists/:id/:user\_id |
| [Autenticazione](/it/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code with PKCE |
| [Limiti di frequenza](/it/resources/fundamentals/rate-limits) predefiniti per le richieste | Nessuno | 300 richieste ogni 15 min (per utente) |

Per accedere agli endpoint dell'API X v2, devi [registrarti per un account sviluppatore](https://developer.x.com/en/portal/petition/essential/basic-info). Durante l'autenticazione, devi usare chiavi e token di una [App per sviluppatori](/it/resources/fundamentals/developer-apps) che si trova all'interno di un [Progetto](/it/resources/fundamentals/projects).

Scopri di più su come ottenere l'accesso agli endpoint dell'API X v2 nella nostra pagina [Guida introduttiva](/it/x-api/getting-started/getting-access).
