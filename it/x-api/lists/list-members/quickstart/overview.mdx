<div id="getting-started-with-the-list-members-endpoint-group">
  ## Introduzione al gruppo di endpoint dei membri della lista
</div>

Questa breve panoramica ti aiuterà a inviare la tua prima richiesta agli endpoint dei membri della lista usando [Postman](/it/tutorials/postman-getting-started).

Se desideri vedere esempi di codice in diverse lingue, visita il nostro repository GitHub [API X v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code).

<Note>
  **Prerequisiti**

  Per completare questa guida, ti servirà un set di [chiavi e token](/it/resources/fundamentals/authentication) per autenticare la richiesta. Puoi generare queste chiavi e token seguendo questi passaggi:

  - [Registrati per un account sviluppatore](https://developer.x.com/en/apply-for-access) e attendi l'approvazione.
  - Crea un [Progetto](/it/resources/fundamentals/projects) e un'[App per sviluppatori](/it/resources/fundamentals/developer-apps) associata nel portale per sviluppatori.
  - Vai alla pagina “Keys and tokens” della tua App per generare le credenziali necessarie. Assicurati di salvare tutte le credenziali in un luogo sicuro.
</Note>

<div id="load-the-x-api-v2-postman-collection">
  #### Carica la raccolta Postman dell’API X v2
</div>

Per aggiungere la raccolta Postman dell’API X v2 al tuo workspace, fai clic sul pulsante seguente:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Aggiungi l’API X v2 a Postman
</Button>

<div id="authenticate-your-request">
  #### Autentica la tua richiesta
</div>

Per inviare correttamente una richiesta agli endpoint di **lookup**, puoi usare [OAuth 1.0a User Context](/it/resources/fundamentals/authentication#oauth-1-0a-2), [App only](/it/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) o [OAuth 2.0 Authorization Code with PKCE](it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). Tuttavia, con gli endpoint di **manage** puoi autenticarti solo con OAuth 1.0a User Context o OAuth 2.0 Authorization Code with PKCE.

In ogni caso, quando utilizzi Postman, le chiavi e i token di autenticazione predefiniti verranno inseriti automaticamente nelle richieste, purché tu abbia configurato correttamente l’ambiente.

Per farlo, seleziona l’ambiente denominato “API X v2” (nell’angolo in alto a destra di Postman) e aggiungi le tue chiavi e i tuoi token ai campi “initial value” e “current value” (facendo clic sull’icona a forma di occhio accanto al menu a discesa dell’ambiente). Queste chiavi includono:

- API Key (nota anche come Consumer Key)
- API Secret Key (nota anche come Consumer Secret)
- OAuth 1.0a user Access Token
- OAuth 1.0a user Access Token Secret
- App only Access Token
- OAuth 2.0 Client Key (disponibile solo se hai configurato le impostazioni di OAuth 2.0 User Authentication nell’App)
- OAuth 2.0 Client Secret (disponibile solo se hai configurato le impostazioni di OAuth 2.0 User Authentication nell’App)
