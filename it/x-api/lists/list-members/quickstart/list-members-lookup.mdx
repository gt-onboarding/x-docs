<div id="steps-to-build-a-list-members-lookup-request">
  ### Passaggi per creare una richiesta di ricerca dei membri di una Lista
</div>

**Primo passaggio: scegli la raccolta di endpoint List in Postman**

Una volta caricata in Postman la raccolta dell'API X v2, vai alla cartella “List”, seleziona la sottocartella “List members”, quindi scegli "Members lookup".
 

**Secondo passaggio: identifica e specifica da quale Lista desideri recuperare i membri**

Devi specificare la Lista da cui desideri recuperare i membri. Puoi trovare l'ID della Lista visitando x.com, aprendo una Lista e poi controllando l'URL. Ad esempio, l'ID della Lista nel seguente URL è 84839422.

https://x.com/i/lists/84839422

L'ID di destinazione può essere qualsiasi ID Lista valido. In Postman, vai alla scheda "Params" e inserisci il tuo ID nella colonna "Value" della variabile di percorso id. Assicurati di non includere spazi prima o dopo l'ID.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 84839422 (ID Lista) |

**Terzo passaggio: identifica e specifica quali campi desideri recuperare**

Se fai clic sul pulsante "Send" dopo il terzo passaggio, nella risposta riceverai i campi predefiniti del [user object](/it/x-api/fundamentals/data-dictionary#user): id, name e username.

Se desideri ricevere campi aggiuntivi oltre a id, name e username, dovrai specificarli nella richiesta con i parametri [field](/it/x-api/fundamentals/fields) e/o [expansion](/it/x-api/fundamentals/expansions).

Per questo esercizio, richiederemo tre insiemi di campi aggiuntivi da oggetti diversi:

- Il campo aggiuntivo user.created\_at negli oggetti utente primari.
- Il [Post object](/it/x-api/fundamentals/data-dictionary#tweet) completo utilizzando il parametro expansion.
- Il campo aggiuntivo tweet.created\_at negli oggetti Post associati.

In Postman, vai alla scheda "Params" e aggiungi la seguente coppia chiave:valore nella tabella "Query Params":

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| user.fields | created\_at | `created_at` |
| expansions | pinned\_tweet\_id | `includes.tweets.id,  <br/>includes.tweets.text  <br/>` |
| tweet.fields | created\_at | `includes.tweets.created_at` |

Ora dovresti vedere un URL simile accanto al pulsante “Send”:

`https://api.x.com/2/lists/84839422/members?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=created_at`
