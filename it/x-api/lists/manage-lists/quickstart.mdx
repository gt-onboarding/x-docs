<div id="getting-started-with-the-manage-lists-endpoint-group">
  ## Introduzione al gruppo di endpoint per la gestione delle Liste
</div>

Questa breve panoramica ti aiuterà a inviare la tua prima richiesta agli endpoint di gestione delle Liste utilizzando [Postman](/it/tutorials/postman-getting-started).

Se desideri consultare esempi di codice in diversi linguaggi, visita il nostro repository GitHub [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code).

**Nota:** In questo esempio invieremo una richiesta all’endpoint _Create a List_, ma quanto illustrato in questo quick start è applicabile anche ad altre richieste di gestione.

<Note>
  **Prerequisiti**

  Per completare questa guida, ti servirà un set di [keys and tokens](/it/resources/fundamentals/authentication) per autenticare la tua richiesta. Puoi generare queste chiavi e token seguendo questi passaggi:

  - [Richiedi un account sviluppatore](https://developer.x.com/en/apply-for-access) e attendi l’approvazione.
  - Crea un [Project](/it/resources/fundamentals/projects) e una [developer App](/it/resources/fundamentals/developer-apps) associata nel portale per sviluppatori.
  - Vai alla pagina “Keys and tokens” della tua App per generare le credenziali necessarie. Assicurati di salvare tutte le credenziali in un luogo sicuro.
</Note>

<div id="steps-to-build-a-manage-list-request">
  ### Passaggi per creare una richiesta di gestione di una Lista
</div>

**Primo passaggio: iniziare con uno strumento o una libreria**

Esistono diversi strumenti, esempi di codice e librerie che puoi utilizzare per effettuare una richiesta a questo endpoint, ma qui useremo Postman per semplificare il processo.

Per caricare la raccolta Postman dell’API X v2 nel tuo ambiente, fai clic sul seguente pulsante:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Aggiungi API X v2 a Postman
</Button>

Una volta caricata la raccolta API X v2 in Postman, vai alla cartella “List”, seleziona la cartella “Manage List”, quindi scegli “Create a List”.

**Secondo passaggio: autenticare la richiesta**

Per effettuare correttamente una richiesta all’API X, devi verificare di avere l’autorizzazione necessaria. Per farlo con gli endpoint di gestione dei Post, devi autenticare la richiesta usando [OAuth 1.0a User Context](/it/resources/fundamentals/authentication) oppure [OAuth 2.0 Authorization Code with PKCE](it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

In questo esempio useremo OAuth 1.0a User Context.

Devi aggiungere le tue chiavi e i tuoi token (in particolare API Key, API Secret Key, OAuth 1.0a user Access Token e OAuth 1.0a user Access Token Secret) a Postman. Puoi farlo selezionando l’ambiente denominato “X API v2” (nell’angolo in alto a destra di Postman) e aggiungendo chiavi e token nei campi “initial value” e “current value” (facendo clic sull’icona a forma di occhio accanto al menu a discesa dell’ambiente).

Se l’hai fatto correttamente, queste variabili verranno automaticamente inserite nella scheda di autorizzazione della richiesta.
 
**Terzo passaggio: specificare il nome della nuova Lista**

Quando crei una nuova Lista con questo endpoint, il nome della Lista è un parametro obbligatorio nel body. Facoltativamente puoi fornire una descrizione e specificare se la Lista è privata.

In Postman, vai alla scheda “Body” e inserisci il nome della Lista come valore del parametro name. Inoltre, se desideri aggiungere una descrizione della Lista, aggiungi una nuova chiave denominata description allo stesso modo di name, seguita dal testo della descrizione come valore. Rendere una Lista privata segue lo stesso schema, ma per questo parametro sono accettati solo i valori true o false. 

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Parameter type** |
| `name` | Nome della lista (obbligatorio) | body |
| description | Descrizione della lista (facoltativo) | body |
| private | true o false (facoltativo) | body |

Ora dovresti vedere un URL simile accanto al pulsante “Send”:

`https://api.x.com/2/lists`

Quarto passaggio: invia la richiesta e verifica la risposta

Una volta impostato tutto, premi il pulsante “Send” e riceverai una risposta simile al seguente esempio:

```
{
  "data": {
    "id": "1441162269824405510",
    "name": "New list created from Postman"
  }
}
```

Se l’oggetto di risposta contiene un id e il nome della tua Lista, la creazione è andata a buon fine. 

Per eliminare una Lista, seleziona la richiesta “Delete a List” che si trova nella cartella “Lists” della raccolta API X v2 caricata in Postman. Questo endpoint richiede l’ID della Lista che desideri eliminare. Nella scheda “Params”, inserisci l’ID della Lista da eliminare come valore nella colonna id. 

In caso di richiesta di eliminazione riuscita, riceverai una risposta simile al seguente esempio: 

```
{
  "data": {
    "deleted": true
  }
}
```
