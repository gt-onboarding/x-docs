<div id="authentication">
  #### Autenticazione
</div>

Tutti gli endpoint dell’API X v2 richiedono di autenticare le richieste con un set di credenziali, anche note come chiavi e token.

Questi endpoint richiedono l’uso di [OAuth 1.0a User Context](/it/resources/fundamentals/authentication), il che significa che devi utilizzare un set di chiavi API e di Access Token utente per effettuare correttamente una richiesta. Gli Access Token devono essere associati all’utente per conto del quale stai effettuando la richiesta. Se desideri generare un set di Access Token per un altro utente, quest’ultimo deve autorizzare o autenticare la tua app utilizzando il [flusso OAuth a 3 passaggi](/it/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow).

Tieni presente che OAuth 1.0a può risultare complesso da utilizzare. Se non conosci questo metodo di autenticazione, ti consigliamo di usare una [libreria](/it/x-api/tools-and-libraries/overview) o uno strumento come Postman per autenticare correttamente le richieste.
 

<div id="developer-portal-projects-and-developer-apps">
  #### Portale sviluppatori, Progetti e App per sviluppatori
</div>

Per ottenere un set di credenziali di autenticazione compatibili con gli endpoint dell’API X v2, devi [richiedere un account sviluppatore](https://developer.x.com/en/portal/petition/essential/basic-info), configurare un [Progetto](/it/resources/fundamentals/projects) all’interno di tale account e creare un’[App per sviluppatori](/it/resources/fundamentals/developer-apps) all’interno di quel Progetto. Potrai quindi trovare le tue chiavi e i tuoi token nella tua App per sviluppatori.
 

<div id="rate-limits">
  #### Limiti di frequenza
</div>

Ogni giorno, molte migliaia di sviluppatori inviano richieste all’API X. Per gestire l’elevato volume di richieste, su ogni endpoint vengono applicati [limiti di frequenza](/it/x-api/fundamentals/rate-limits) che determinano quante richieste puoi effettuare per conto della tua app o di un utente autenticato.

Questi endpoint sono soggetti a limiti a livello di utente, il che significa che l’utente autenticato per conto del quale invii la richiesta può chiamare l’endpoint solo un certo numero di volte, indipendentemente dall’App per sviluppatori utilizzata.

Il prospetto seguente mostra i limiti di frequenza per ciascun endpoint.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Metodo HTTP** | **Limite di frequenza** |
| /2/lists | POST | 300 richieste ogni 15 minuti |
| /2/lists/:id | DELETE / PUT | 300 richieste ogni 15 minuti |
