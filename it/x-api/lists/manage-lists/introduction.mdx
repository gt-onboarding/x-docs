[X Lists](https://help.x.com/en/using-twitter/twitter-lists) consente agli utenti di personalizzare, organizzare e dare priorità ai Post che vedono nella loro timeline. Con gli endpoint Lists puoi creare soluzioni che consentono alle persone di curare e organizzare i Post in base a preferenze, interessi, gruppi o argomenti.

Poiché effettui richieste per conto di un utente con gli endpoint di gestione delle Liste, devi autenticare questi endpoint con [OAuth 1.0a User Context](/it/resources/fundamentals/authentication) oppure [OAuth 2.0 Authorization Code with PKCE](it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) e utilizzare gli Access Token associati a un utente che ha autorizzato la tua App. Tali token possono essere generati tramite il [flusso OAuth a 3 gambe](/it/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) o il [flusso Authorization Code con PKCE](/it/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0).

<div id="manage-lists">
  ### Gestione delle Liste
</div>

Gli endpoint per la gestione delle Liste consentono di creare, eliminare e aggiornare Liste per conto di un utente autenticato. Per questi endpoint sono disponibili tre metodi: POST, DELETE e PUT. Il metodo POST consente di creare una Lista, il metodo DELETE di eliminarla e il metodo PUT di aggiornarne i metadati. È previsto un limite per utente di 300 richieste ogni 15 minuti per tutti e tre gli endpoint.

Nota: puoi creare fino a 1000 Liste per account.

**Configurazione dell’account**

Per accedere a questi endpoint, ti serviranno:

- Un [account sviluppatore](https://developer.x.com/en/portal/petition/essential/basic-info) approvato.
- L’autenticazione tramite le chiavi e i token di una [App sviluppatore](/it/resources/fundamentals/developer-apps) all’interno di un [Progetto](/it/resources/fundamentals/projects).

Scopri di più su come ottenere l’accesso agli endpoint dell’API X v2 nella nostra [guida introduttiva](/it/x-api/getting-started/getting-access).
