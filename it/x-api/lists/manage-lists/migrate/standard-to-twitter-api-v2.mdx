<div id="manage-lists-standard-v11-compared-to-x-api-v2">
  ### Gestione delle Liste: Standard v1.1 rispetto all’API X v2
</div>

Se hai lavorato con gli endpoint dello standard v1.1 [POST lists/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-create), [POST lists/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-destroy) e [POST lists/update](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-update), l’obiettivo di questa guida è aiutarti a comprendere le similitudini e le differenze tra gli endpoint di gestione delle Liste dello standard v1.1 e dell’API X v2.

- **Somiglianze**
  - Autenticazione
- **Differenze**
  - URL degli endpoint
  - Requisiti di App e Progetti
  - Metodi HTTP
  - Limiti di velocità
  - Parametri della richiesta

<div id="similarities">
  #### Somiglianze
</div>

**Autenticazione**

Entrambe le versioni dell’endpoint supportano [OAuth 1.0a User Context](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Pertanto, se in precedenza utilizzavi uno degli endpoint standard v1.1 per la gestione delle liste, puoi continuare a usare lo stesso metodo di autenticazione passando alla versione dell’API X v2.

<div id="differences">
  #### Differenze
</div>

**URL degli endpoint**

- Endpoint Standard v1.1:
  - POST https://api.x.com/1.1/lists/create.json
    (Crea una lista)
  - POST https://api.x.com/1.1/lists/destroy.json
    (Elimina una lista)
  - POST https://api.x.com/1.1/lists/update.json
    (Aggiorna una lista)
- Endpoint API X v2:
  - POST https://api.x.com/2/lists
    (Crea una lista)

  - DELETE https://api.x.com/2/lists/:id
    (Elimina una lista)

  - PUT https://api.x.com/2/lists/:id
    (Aggiorna una lista)

**Limiti di frequenza**

| **Standard v1.1** | **API X v2** |
| :--- | :--- |
| /1.1/lists/create.json<br /><br />nessuno | /2/lists<br /><br />300 richieste per finestra di 15 minuti con OAuth 1.0a User Context |
| /1.1/lists/destroy.json<br /><br />nessuno | /2/lists/:id<br /><br />300 richieste per finestra di 15 minuti con OAuth 1.0a User Context |
| /1.1/lists/update.json<br /><br />nessuno | /2/lists/:id<br /><br />300 richieste per finestra di 15 minuti con OAuth 1.0a User Context |

**Requisiti di App e Progetto**

Gli endpoint dell’API X v2 richiedono di utilizzare le credenziali di una [App per sviluppatori](/it/resources/fundamentals/developer-apps) associata a un [Progetto](/it/resources/fundamentals/projects) per autenticare le richieste. Tutti gli endpoint dell’API X v1.1 possono utilizzare credenziali di App autonome o di App associate a un progetto.

**Parametri della richiesta**

I seguenti parametri di richiesta dello standard v1.1 hanno equivalenti nell’API X v2:

**Creare una lista**

| **Standard** | **API X v2** |
| :--- | :--- |
| name | name |
| mode | private |
| description | description |

**Eliminare/Aggiornare una lista**

| **Standard** | **API X v2** |
| :--- | :--- |
| owner\_screen\_name | Nessun equivalente |
| owner\_id | Nessun equivalente |
| list\_id | id  |
| slug | Nessun equivalente |

<Note>
  **Nota:** I parametri dello standard v1.1 vengono passati come query parameter, mentre i parametri dell’API X v2 vengono passati nel body (per l’endpoint POST) o come path parameter (per gli endpoint DELETE e PUT).
</Note>
