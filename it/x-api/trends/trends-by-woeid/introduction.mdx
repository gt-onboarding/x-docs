L’endpoint di ricerca dei Trend consente agli sviluppatori di ottenere i Trend per una località, specificata utilizzando il where-on-earth ID (WOEID).

<Note>
  **Nota**: WOEID è un identificatore legacy creato da Yahoo ed è stato deprecato. L’API X utilizza il valore numerico per identificare le località dei trend a livello di città e paese. Consulta il nostro [post sul blog legacy](https://blog.x.com/engineering/en_us/a/2010/woeids-in-twitters-trends.html) o i [dati archiviati](https://archive.org/details/geoplanet_data_7.10.0.zip0.)
</Note>

Il `tweet_count` delle ultime 24 ore viene restituito per molti trend, se disponibile.

Questo endpoint supporta l’autenticazione app-auth e prevede un limite di 75 richieste per ogni finestra di 15 minuti.

<div id="getting-started">
  ### Per iniziare
</div>

Per utilizzare questo endpoint, ti serve un \[bearer token]\(https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) from the [developer portal](https://developer.x.com/en/portal/dashboard). Una volta ottenuto il bearer token, puoi chiamare l’API di utilizzo come mostrato di seguito:

```
      curl 'https://api.x.com/2/trends/by/woeid/26062' --header 'Authorization: Bearer XXXXX'
```

Se la richiesta ha esito positivo, vedrai la risposta JSON come mostrato di seguito:

**Configurazione dell’account**

Per accedere a questi endpoint, ti serviranno:

- Un approvato [developer account](https://developer.x.com/en/portal/petition/essential/basic-info).
- L’autenticazione utilizzando le chiavi e i token di una [developer App](/it/resources/fundamentals/developer-apps) che si trova all’interno di un [Project](/it/resources/fundamentals/projects).

Scopri di più su come ottenere l’accesso agli endpoint dell’API X v2 nella nostra [guida introduttiva](/it/x-api/getting-started/getting-access).
