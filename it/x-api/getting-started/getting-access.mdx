<div id="step-one-sign-up-for-a-developer-account">
  ### Primo passaggio: crea un account sviluppatore
</div>

Creare un account sviluppatore è rapido e semplice! Fai clic sul pulsante qui sotto, rispondi a poche domande e potrai iniziare a esplorare e sviluppare con l’API X v2 utilizzando l’[accesso Basic](https://developer.x.com/en/portal/products/basic).

Nel passo successivo, durante l’onboarding, creerai un Progetto e un’App per sviluppatori associata, che ti forniranno un set di credenziali da utilizzare per autenticare tutte le richieste all’API.

[Registrati](https://developer.x.com/en/portal/products/basic)

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### Passaggio due: salva la chiave e i token della tua App e mantienili al sicuro
</div>

Una volta ottenuto l’accesso e creati un Progetto e un’App, potrai trovare o generare le seguenti credenziali all’interno della tua App per sviluppatori:

- **[API Key e Secret](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret):** In pratica, il nome utente e la password della tua App. Le userai per autenticare le richieste che richiedono [OAuth 1.0a User Context](/it/resources/fundamentals/authentication) o per generare altri token come gli Access Token utente o l’App Access Token.

- **[Access Token e Secret](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens):** In generale, gli Access Token rappresentano l’utente per conto del quale stai effettuando la richiesta. Quelli che puoi generare tramite il portale sviluppatori rappresentano l’utente proprietario dell’App. Li userai per autenticare le richieste che richiedono [OAuth 1.0a User Context](/it/resources/fundamentals/authentication). Se desideri effettuare richieste per conto di un altro utente, dovrai utilizzare il flusso OAuth a 3 fasi affinché ti autorizzi.

- **[Client ID e Client Secret](/it/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce):** Queste credenziali vengono utilizzate per ottenere un Access Token utente con autenticazione OAuth 2.0. Analogamente a OAuth 1.0a, gli Access Token utente vengono utilizzati per autenticare richieste che forniscono informazioni private dell’account utente o eseguono azioni per conto di un altro account, ma con ambiti a granularità fine per un maggiore controllo sugli accessi concessi all’applicazione client.

- **[App only Access Token](/it/resources/fundamentals/authentication#bearer-token-also-known-as-app-only):** Userai questo token quando effettui richieste a endpoint che restituiscono informazioni pubblicamente disponibili su X.

Poiché queste chiavi e questi token non scadono a meno che non vengano rigenerati, ti consigliamo di salvarli in un luogo sicuro, ad esempio un password manager, non appena ricevi le credenziali.

> **Nota: le tue chiavi e i tuoi token verranno mostrati una sola volta nel [developer portal](/it/resources/fundamentals/developer-portal), quindi è importante memorizzarli nel tuo sistema di gestione delle password non appena li generi.**
>
> Se smarrisci o dimentichi chiavi e token, dovrai rigenerarli, creando nuove credenziali e invalidando le precedenti. Questo significa che dovrai aggiornare eventuali integrazioni configurate con le credenziali precedenti.
>
> Scopri di più sulle nostre [best practice di autenticazione](/it/resources/fundamentals/authentication#authentication-best-practices).
