<div id="comparing-x-apis-post-counts-endpoints">
  ## Confronto tra gli endpoint per il conteggio dei Post dell’API X
</div>

L’endpoint di conteggio dei Post della v2 sostituirà progressivamente l’[endpoint di conteggio della Search API enterprise](/it/x-api/enterprise-gnip-2.0/fundamentals/search-api#counts-requests-post-count). Se disponi di codice, app o strumenti che utilizzano una versione precedente di un endpoint di conteggio dei Post e stai valutando la migrazione ai nuovi endpoint dell’API X v2, questa guida fa al caso tuo.

Questa pagina contiene tabelle di confronto su:

- [Conteggi dei Post recenti](#recent)
- [Conteggi dei Post sull’archivio completo](#full-archive)
- [Confronto degli operatori di filtro](#operator)

<div id="recent-post-counts-comparison">
  ### Confronto dei conteggi dei Post recenti
</div>

La versione enterprise degli endpoint per il conteggio dei Post consente di ottenere conteggi per 30 giorni o dall’archivio completo. Pertanto, l’endpoint v2 per il conteggio dei Post recenti, che considera un periodo di 7 giorni, non è un sostituto diretto di nessuno dei due endpoint sopra menzionati.

Tuttavia, per agevolare i confronti, esamineremo come l’endpoint v2 per il conteggio dei Post recenti si confronta con l’endpoint enterprise a 30 giorni.

La tabella seguente confronta i vari tipi di endpoint per il conteggio dei Post recenti:

| **Descrizione** | **Enterprise** | **API X v2** |
| :--- | :--- | :--- |
| Dominio host | https://gnip-api.x.com | https://api.x.com |
| Percorso endpoint | /search/30day/accounts/:account\_name/:label/counts.json | /2/tweets/counts/recent |
| [Autenticazione](/it/resources/fundamentals/authentication) | Autenticazione di base | Token Bearer OAuth 2.0 |
| Formato timestamp | YYYYMMDDhhmm | YYYY-MM-DDTHH:mm:ssZ  <br />[ISO 8601 / RFC 3339](https://tools.ietf.org/html/rfc3339#section-5.6) |
| Restituisce conteggi di Post non più vecchi di | 31 giorni | 7 giorni |
| Metodi HTTP supportati | GET | GET |
| Limiti di frequenza predefiniti delle richieste | 20 richieste per 1 sec, aggregate tra dati di ricerca e richieste di conteggio  <br />Il limite per minuto varierà in base al partner, come specificato nel contratto. | 180 richieste per 15 min per utente  <br />450 richieste per 15 min per App |
| Supporta il filtraggio tramite [annotations](/it/x-api/fundamentals/post-annotations) |     | ✔   |
| Supporta il filtraggio tramite [conversation\_id](/it/x-api/fundamentals/conversation-id) |     | ✔   |
| Nome della chiave JSON per l’array di dati dei Post | results | data |
| Granularità temporale | Giorno, ora o minuto | Giorno, ora o minuto |
| Fuso orario | UTC | UTC |
| Parametri di richiesta per selezionare il periodo di tempo | fromDate  <br />toDate | start\_time  <br />end\_time |
| Parametri di richiesta per navigare per ID del Post |     | since\_id  <br />until\_id |
| Richiede l’uso di credenziali da una [developer App](/it/resources/fundamentals/developer-apps) associata a un [project](/it/resources/fundamentals/projects) |     | ✔   |

<div id="filtering-operator-comparison">
  ### Confronto degli operatori di filtro
</div>

Le due diverse versioni (Enterprise e v2) dei conteggi dei post differiscono per gli operatori disponibili e presentano anche livelli variabili di disponibilità degli operatori all'interno di ciascuna versione, come spiegato di seguito.

Enterprise

- Non esistono sottolivelli di operatori Enterprise. Tutti gli operatori Enterprise sono disponibili per tutti gli utenti Enterprise.

API X v2

- **Core:** Questi operatori sono disponibili per qualsiasi utente v2.
- **Advanced:** Questi operatori sono disponibili solo per gli utenti approvati per l’accesso Academic Research.

Puoi approfondire ciascun set di operatori nelle rispettive guide:

- [Operatori Enterprise](/it/x-api/enterprise-gnip-2.0/fundamentals/search-api#building-search-queries)
- [Operatori API X v2](/it/x-api/posts/search/integrate/build-a-query)
