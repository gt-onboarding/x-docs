<div id="comparing-x-apis-manage-posts-endpoints">
  ## Confronto tra gli endpoint di gestione dei post dell’API X
</div>

Gli endpoint v2 per la gestione dei post sostituiranno gli endpoint standard v1.1 [POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) e [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id). Se disponi di codice, app o strumenti che utilizzano la versione v1.1 degli endpoint di gestione dei post e stai valutando la migrazione al nuovo endpoint dell’API X v2, questa serie di guide fa al caso tuo.

Le tabelle seguenti confrontano gli endpoint standard v1.1 e quelli dell’API X v2 per la gestione dei post:

<div id="create-a-post">
  ### Creare un post
</div>

| Descrizione | Standard v1.1 | API X v2 |
| :--- | :--- | :--- |
| Metodi HTTP supportati | POST | POST |
| Dominio host | https://api.x.com | https://api.x.com |
| Percorso endpoint | /1.1/statuses/update.json | /2/tweets |
| [Autenticazione](/it/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code con PKCE |
| [Limiti di richieste](/it/x-api/fundamentals/rate-limits) predefiniti | Nessuno<br /><br />300 richieste per finestra di 3 ore per utente, per app. Condiviso con l’endpoint POST Retweets v1.1. | 200 richieste per 15 minuti per utente<br /><br />300 richieste per finestra di 3 ore per utente, per app. Condiviso con l’endpoint POST Retweets v2. |
| Richiede l’uso di credenziali di un’[App sviluppatore](/it/resources/fundamentals/developer-apps) associata a un [Progetto](/it/resources/fundamentals/projects) |     | ✔   |

<div id="delete-a-post">
  ### Eliminare un post
</div>

| Descrizione | Standard v1.1 | API X v2 |
| :--- | :--- | :--- |
| Metodi HTTP supportati | POST | DELETE |
| Dominio host | https://api.x.com | https://api.x.com |
| Percorso endpoint | /1.1/statuses/destroy/:id.json | /2/tweets/:id |
| [Autenticazione](/it/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code con PKCE |
| [Limiti di frequenza](/it/x-api/fundamentals/rate-limits) predefiniti per le richieste | Nessuno | 50 richieste ogni 15 min per utente |
| Richiede l'uso di credenziali di un'[App per sviluppatori](/it/resources/fundamentals/developer-apps) associata a un [Progetto](/it/resources/fundamentals/projects) |     | ✔   |
