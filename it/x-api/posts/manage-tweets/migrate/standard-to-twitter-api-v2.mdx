<div id="standard-v11-compared-to-x-api-v2">
  ## Standard v1.1 rispetto all’API X v2
</div>

Se hai lavorato con gli endpoint standard v1.1 [POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) e [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id), l’obiettivo di questa guida è aiutarti a comprendere somiglianze e differenze tra gli endpoint standard e gli endpoint di gestione dei post dell’API X v2.

- **Somiglianze**
  - Autenticazione
- **Differenze**
  - URL degli endpoint

  - Requisiti di App e Progetti

  - Parametri della richiesta

<div id="similarities">
  ### Somiglianze
</div>

**Autenticazione**

Sia gli endpoint dei Post dello standard v1.1 ([POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) e [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id)) sia quelli dell’API X v2 utilizzano [OAuth 1.0a User Context](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Pertanto, se in precedenza utilizzavi uno degli endpoint standard v1.1, puoi continuare a usare lo stesso metodo di autenticazione quando effettui la migrazione alla versione dell’API X v2. 

<div id="differences">
  ### Differenze
</div>

**URL degli endpoint**

- Endpoint standard v1.1:
  - https://api.x.com/1.1/statuses/update.json
    (Crea un post)
  - `https://api.x.com/1.1/statuses/destroy/:id.json`
    (Elimina un post)
- Endpoint dell'API X v2:
  - https://api.x.com/2/tweets
    (Crea un post)
  - https://api.x.com/2/tweets/:id
    (Elimina uno specifico post)

<div id="app-and-project-requirements">
  ### Requisiti per App e Progetti
</div>

Gli endpoint dell’API X v2 richiedono l’uso di credenziali di un’[App per sviluppatori](/it/resources/fundamentals/developer-apps) associata a un [Progetto](/it/resources/fundamentals/developer-apps) per autenticare le richieste. Tutti gli endpoint dell’API X v1.1 possono utilizzare credenziali di App autonome o di App associate a un progetto.

<div id="request-parameters">
  ### Parametri della richiesta
</div>

I seguenti parametri standard della richiesta v1.1 accettavano due parametri di query (user\_id o screen\_name). L’API X v2 accetta solo l’ID numerico del Post per l’endpoint DELETE, che deve essere incluso nel percorso dell’endpoint.

Per l’endpoint POST, parametri aggiuntivi devono essere forniti nel corpo JSON della richiesta. Puoi sapere di più sui parametri disponibili nella [guida di riferimento dell’API](/it/x-api/posts/manage-tweets/introduction).
