<div id="authentication">
  ### Autenticazione
</div>

Tutti gli endpoint dell'API X v2 richiedono di autenticare le richieste con un set di credenziali, note anche come chiavi e token.

Questi endpoint specifici richiedono l’uso di [OAuth 1.0a User Context](/it/resources/fundamentals/authentication), il che significa che devi utilizzare un set di chiavi API e Access Token utente per effettuare correttamente una richiesta. Gli Access Token devono essere associati all’utente per conto del quale stai effettuando la richiesta. Se desideri generare un set di Access Token per un altro utente, quest’ultimo deve autorizzare o autenticare la tua App utilizzando il [flusso OAuth a 3 passaggi](/it/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow).

Tieni presente che OAuth 1.0a può essere complesso da usare. Se non hai familiarità con questo metodo di autenticazione, ti consigliamo di utilizzare una [libreria](/it/x-api/tools-and-libraries/overview), uno strumento come Postman oppure OAuth 2.0 per autenticare le richieste.

[OAuth 2.0 Authorization Code with PKCE](/it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) consente un maggiore controllo sull’ambito di un’applicazione e flussi di autorizzazione su più dispositivi. OAuth 2.0 consente di scegliere ambiti specifici e granulari che ti concedono permessi specifici per conto di un utente.

Per abilitare OAuth 2.0 nella tua App, attivalo nelle impostazioni di autenticazione della tua App, disponibili nella sezione Impostazioni dell’App del portale per sviluppatori.

<div id="developer-portal-projects-and-developer-apps">
  ### Limiti di frequenza
</div>

Ogni giorno, molte migliaia di sviluppatori inviano richieste all’API X. Per gestire l’elevato volume di traffico, vengono applicati [limiti di frequenza](https://developer.x.com/x-api/fundamentals/rate-limits) a ciascun endpoint, che restringono il numero di richieste che puoi effettuare per conto della tua app o di un utente autenticato.

Questi endpoint sono limitati a livello utente, il che significa che l’utente autenticato per conto del quale stai effettuando la richiesta può chiamare l’endpoint solo un certo numero di volte su qualsiasi App per sviluppatori. Il limite di frequenza per utente è di 200 richieste ogni 15 minuti per il metodo POST. Il metodo DELETE ha un limite di 50 richieste ogni 15 minuti. Inoltre, è previsto un limite di [300 richieste ogni 3 ore](https://blog.x.com/developer/en_us/topics/tools/2018/new-developer-requirements-to-protect-our-platform), incluse le pubblicazioni create tramite gestione Post o gestione Retweet.
 

<div id="rate-limits">
  ### Etichette di origine
</div>

Il nome della tua App e l’URL del sito web verranno mostrati come [etichetta di origine](https://help.x.com/en/using-twitter/how-to-tweet#source-labels) nei metadati di qualsiasi Post creato in modo programmatico dalla tua applicazione. Se modifichi il caso d’uso di un’App X, assicurati di aggiornarlo in queste impostazioni per garantire la conformità ai [Termini per sviluppatori](https://developer.x.com/content/developer-twitter/en/developer-terms/agreement-and-policy).
 

<div id="source-labels">
  ### Impostazioni del profilo
</div>

Puoi aggiungere una località ai Post solo se hai la geolocalizzazione abilitata nelle impostazioni del profilo. Se non hai la geolocalizzazione abilitata, puoi comunque includere un parametro di località nel body della richiesta, ma non verrà associato al tuo Post. Lo stesso vale per il tag degli utenti nelle immagini. Se l’utente che stai taggando non ha abilitato il tag delle foto, il suo nome non apparirà nell’elenco degli utenti taggati, anche se il Post è stato creato correttamente. 
 

<div id="profile-settings">
  ### Aggiunta di media a un Post
</div>

Al momento non è possibile caricare completamente i media con la versione v2 dell’API X. Tuttavia, puoi allegare a un Post media caricati in precedenza. Puoi usare gli ID dei media già [caricati con l’endpoint media v1](https://developer.x.com/en/docs/twitter-api/v1/media/upload-media/api-reference/post-media-upload) o con [X Media Studio](https://media.x.com/en/articles/products/2018/media-studio). Questi ID dei media devono essere tuoi o appartenere a un utente autenticato.
