La creazione e l’eliminazione dei Post tramite l’API X sono fondamentali per partecipare alla conversazione pubblica. I nuovi endpoint per la gestione dei Post consentono di fare questo e molto altro.

Sono disponibili due metodi per gestire i Post: POST e DELETE. Il metodo POST consente di pubblicare sondaggi, citare Tweet, impostare le opzioni di risposta, aggiungere informazioni geografiche, allegare contenuti multimediali e taggare utenti, nonché pubblicare per i Super Follower, oltre ad altre funzionalità. Il metodo DELETE consente invece di eliminare un Post specifico. Con il metodo POST, puoi passare [i parametri](/it/x-api/posts/creation-of-a-post) desiderati per personalizzare ulteriormente la richiesta.

Il metodo di eliminazione dei Post è stato aggiornato per supportare i Post modificati. Quando viene eliminato un Post all’interno di una catena di modifiche, tutti i Post in quella catena vengono eliminati. Per saperne di più sui metadati della modifica dei Post, consulta la pagina [Fondamenti dei Post modificati](/it/x-api/fundamentals/edit-posts).

È previsto un rate limit utente di 200 richieste ogni 15 minuti per il metodo POST. Il metodo DELETE ha un limite di 50 richieste ogni 15 minuti. Inoltre, è previsto un limite di 300 richieste ogni 3 ore, che include i Post creati tramite gestione dei Post o gestione dei Retweet.

Poiché stai effettuando richieste per conto di un utente con gli endpoint di gestione dei Post, devi autenticarti con [OAuth 1.0a User Context](https://en.docs/authentication/oauth-1-0a) o [OAuth 2.0 Authorization Code con PKCE](https://en.docs/authentication/oauth-2-0/authorization-code) e utilizzare token di accesso utente associati a un utente che ha autorizzato la tua App. Per generare questo token di accesso utente con OAuth 1.0a, puoi utilizzare il [flusso OAuth a 3 fasi](https://en.docs/authentication/oauth-2-0/bearer-tokens). Per generare un token di accesso utente con OAuth 2.0, puoi utilizzare il [flusso Authorization Code con PKCE](https://en.docs/authentication/oauth-2-0/user-access-token).

<Info>
  **Configurazione dell’account**

  Per accedere a questi endpoint, avrai bisogno di:

  - Un [account sviluppatore](https://en/portal/petition/essential/basic-info) approvato.
  - Autenticarti utilizzando le chiavi e i token di una [App sviluppatore](/it/resources/fundamentals/developer-apps) che si trova all’interno di un [Progetto](/it/resources/fundamentals/projects).

  Scopri di più su come ottenere l’accesso agli endpoint dell’API X v2 nella nostra [guida introduttiva](/it/x-api/getting-started/getting-access).
</Info>
