## Introduzione agli endpoint per la gestione dei Post\*\*

Questa guida rapida ti aiuterà a inviare la tua prima richiesta agli endpoint per la gestione dei Post usando [Postman](/it/tutorials/postman-getting-started).

Visita il repository GitHub [API X v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) se vuoi vedere esempi di codice in diverse lingue.

<Info>
  **Prerequisiti**

  Per completare questa guida, ti servirà un set di [chiavi e token](/it/resources/fundamentals/authentication) per autenticare la tua richiesta. Puoi generarli seguendo questi passaggi:

  - [Richiedi un account sviluppatore](https://developer.x.com/en/apply-for-access) e attendi l’approvazione.
  - Crea un [Progetto](/it/resources/fundamentals/projects) e una [App sviluppatore](/it/resources/fundamentals/developer-apps) associata nel portale per sviluppatori.
  - Vai alla pagina “Keys and tokens” della tua App per generare le credenziali necessarie. Assicurati di salvare tutte le credenziali in un luogo sicuro.
</Info>

<div id="steps-to-building-manage-posts-requests">
  ### Passaggi per creare richieste di gestione dei Post
</div>

**Primo passaggio: inizia con uno strumento o una libreria**

Esistono diversi strumenti, esempi di codice e librerie che puoi usare per effettuare una richiesta a questo endpoint, ma qui utilizzeremo Postman per semplificare il processo.

Per caricare la collection Postman dell'API X v2 nel tuo ambiente, fai clic sul seguente pulsante:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Aggiungi API X v2 a Postman
</Button>

Una volta caricata la collection API X v2 in Postman, vai alla cartella “Manage Posts” e seleziona “Create a Post”.
 

**Secondo passaggio: autentica la richiesta**

Per effettuare correttamente una richiesta all’API X, devi verificare di avere l’autorizzazione a farlo. Per gli endpoint di gestione dei Post, devi autenticare la richiesta usando [OAuth 1.0a User Context](/it/resources/fundamentals/authentication) oppure [OAuth 2.0 Authorization Code with PKCE](it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

In questo esempio utilizzeremo OAuth 1.0a User Context.

Devi aggiungere le tue chiavi e i tuoi token (in particolare API Key, API Secret Key, OAuth 1.0a user Access Token e OAuth 1.0a user Access Token Secret) in Postman. Puoi farlo selezionando l’ambiente denominato “X API v2” (nell’angolo in alto a destra di Postman) e aggiungendo chiavi e token nei campi "initial value" e "current value" (facendo clic sull’icona a forma di occhio accanto al menu a discesa dell’ambiente).

Se lo hai fatto correttamente, queste variabili verranno caricate automaticamente nella scheda di autorizzazione della richiesta.

**Terzo passaggio: specifica il testo del Post**

Quando crei un nuovo Post con questo endpoint, il testo o i media del Post sono parametri obbligatori del body.

In Postman, vai alla scheda “Body” e inserisci il testo del Post come valore del parametro `text`. Inoltre, se desideri aggiungere parametri per elementi come sondaggi, risposta a un ID di Post o impostazioni delle risposte, puoi farlo qui. Puoi saperne di più su ciò che è disponibile nella nostra [guida di riferimento all’API](/it/x-api/posts/manage-tweets/introduction).

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Parameter type** |
| `text` | Hello world! | body |

**Quarto passaggio: identifica e specifica quali campi desideri recuperare**

Una volta impostato tutto, premi il pulsante "Send" e riceverai una risposta simile al seguente esempio:

```
{
  "data": {
    "id": "1445880548472328192",
    "text": "Hello world!"
  }
}
```

Se l’oggetto di risposta contiene un `id` e il `text` del tuo Post, hai creato correttamente un Post.
 

**Quinto passaggio: elimina il tuo Post**

Per eliminare un Post, seleziona la richiesta “Delete a Post” presente nella cartella “Manage Posts” della collection API X v2 caricata in Postman. Questo endpoint richiede l’ID del Post che desideri eliminare. Quindi, nella scheda “Params”, inserisci l’ID del Post che desideri eliminare come valore della colonna `id`. 

Se la richiesta di eliminazione va a buon fine, riceverai una risposta simile al seguente esempio:

```
{
   "data": {
       "deleted" : true
   }
}
```
