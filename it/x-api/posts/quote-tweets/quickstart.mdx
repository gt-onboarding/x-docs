<div id="getting-started-with-the-quote-tweets-lookup-endpoint">
  ## Guida introduttiva all’endpoint di ricerca dei Quote Tweet
</div>

Questa guida rapida ti aiuterà a effettuare la prima richiesta agli endpoint di ricerca dei Quote Tweet con un set di campi specifici utilizzando [Postman](/it/tutorials/postman-getting-started).

Se desideri consultare esempi di codice in diverse lingue, visita il nostro repository GitHub [API X v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code). 

<div id="prerequisites">
  #### Prerequisites
</div>

Per completare questa guida, ti servirà un set di [key e token](/it/resources/fundamentals/authentication) per autenticare la richiesta. Puoi generarli seguendo questi passaggi:

- [Richiedi un account sviluppatore](https://developer.x.com/en/apply-for-access) e attendi l’approvazione.
- Crea un [Progetto](/it/resources/fundamentals/projects) e un’[App per sviluppatori](/it/resources/fundamentals/developer-apps) associata nel portale per sviluppatori.
- Vai alla pagina “Keys and tokens” della tua App per generare le credenziali necessarie. Assicurati di salvarle tutte in un luogo sicuro.

**Passaggi per creare una richiesta di recupero dei Quote Tweet**

**Primo passaggio: inizia con uno strumento o una libreria**

Esistono diversi strumenti, esempi di codice e librerie che puoi usare per inviare una richiesta a questo endpoint; qui utilizzeremo Postman per semplificare il processo.

Per caricare la collection Postman dell’API X v2 nel tuo ambiente, fai clic sul seguente pulsante:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Aggiungi API X v2 a Postman
</Button>

Una volta caricata in Postman la collection dell’API X v2, vai alla cartella timeline e trova la richiesta “Quote Tweets by ID”.

**Secondo passaggio: autentica la tua richiesta**

Per effettuare correttamente una richiesta all’API X, devi verificare di avere le autorizzazioni necessarie. Per questo endpoint, devi autenticare la richiesta con [OAuth 2.0 App-Only](/it/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), [OAuth 2.0 Authorization Code with PKCE](it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) o [OAuth 1.0a User Context](/it/resources/fundamentals/authentication).

Per semplicità, utilizzeremo OAuth 2.0 App-Only per questa richiesta, ma dovrai usare uno degli altri metodi di autenticazione se desideri richiedere [metriche](/it/x-api/fundamentals/metrics) o Post privati.

Devi aggiungere le tue key e i tuoi token, in particolare l’[App Access Token](/it/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token) (noto anche come App-only Bearer Token), in Postman. Puoi farlo selezionando l’ambiente denominato “X API v2” nell’angolo in alto a destra di Postman e aggiungendo key e token nei campi “initial value” e “current value” (facendo clic sull’icona a forma di occhio accanto al menu a discesa dell’ambiente).

Questa variabile verrà inserita automaticamente nella scheda “Authorization” della richiesta se hai eseguito correttamente questa operazione.

**Terzo passaggio: identifica e specifica il Post per cui desideri recuperare i Quote Tweet**

Devi specificare, nella richiesta, il Post di cui vuoi recuperare i Quote Tweet passando il Post ID.

In questo esempio, il Post ID che useremo è 1409931481552543749.

In Postman, vai alla scheda “Params” e inserisci questo Post ID nella colonna “Value” del parametro id.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 1409931481552543749 |

**Quarto passaggio: identifica e specifica quali campi desideri recuperare**

Se fai clic sul pulsante “Send” dopo il terzo passaggio, riceverai nella risposta i campi predefiniti del [Post object](/it/x-api/fundamentals/data-dictionary#tweet): id, text ed edit\_history\_tweet\_ids.

Se desideri ricevere campi aggiuntivi, devi specificarli nella richiesta tramite i parametri [field](/it/x-api/fundamentals/fields) e/o [expansion](/it/x-api/fundamentals/expansions).

Per questo esercizio, richiederemo tre set aggiuntivi di campi da oggetti diversi:

1. Il campo aggiuntivo tweet.created\_at negli oggetti post principali.
2. I campi predefiniti dell’oggetto utente degli autori associati per i Post restituiti: id, name e username
3. Il campo aggiuntivo user.created\_at negli oggetti utente associati.

In Postman, vai alla scheda “Params” e aggiungi la seguente coppia chiave:valore alla tabella “Query Params”:

|     |     |     |     |
| :--- | :--- | :--- | :--- |
| **Chiave** | **Valore** | **Campi restituiti** |
| `tweet.fields` | `created_at` | `tweets.created_at` |
| `expansions` | `author_id` | `includes.users.id`, `includes.users.name`, `includes.users.username` |
| `user.fields` | `created_at` | `includes.users.created_at` |     |

Dovresti ora vedere il seguente URL accanto al pulsante "Send":

`https://api.x.com/2/tweets/:id/quote_tweets?expansions=author_id&tweet.fields=created_at&user.fields=created_at`

**Nota bene:**

In Postman, il parametro di percorso :id nel campo URL **non** si aggiorna automaticamente al valore che inserisci nel campo params `id`. Per questo motivo, l’URL sopra include `:id` e non 1409931481552543749.

<div id="step-six-paginate-through-your-results">
  #### Passaggio sei: esegui la paginazione dei risultati
</div>

Nella risposta precedente, in fondo troverai un oggetto di meta dati che include i seguenti campi:

- results\_count
- next\_token

Al passaggio quattro, poiché non abbiamo specificato il parametro max\_results, per impostazione predefinita otterremo solo 10 post. Per accedere alle pagine aggiuntive di dati, prendi il valore del campo next\_token dagli ultimi risultati e aggiungi quella stringa come valore del parametro pagination\_token nella pagina dei parametri di Postman, mantenendo tutto il resto invariato.

|     |     |
| :--- | :--- |
| **Chiave** | **Valore** |
| `pagination_token` | avdjwk0udyx6 |

Una volta configurato tutto, puoi fare clic di nuovo su "Send" e dovresti ricevere la pagina successiva dei risultati.

Abbiamo preparato una guida sulla [paginazione](/it/x-api/fundamentals/pagination) per approfondire questo concetto.
