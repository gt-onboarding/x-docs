- Stream con campionamento all’1%.
- Stream con campionamento al 10%. Comunemente chiamato “Decahose”.

Sono detti “stream di volume” perché entrambi forniscono grandi volumi di dati. Anche lo stream all’1% può emettere diverse decine di Post ogni secondo. Con questi stream puoi identificare e monitorare trend, analizzare il sentiment generale, seguire eventi globali e molto altro.

Questi endpoint di streaming inviano [oggetti Post](/it/x-api/fundamentals/data-dictionary#tweet) tramite una connessione HTTP GET persistente e utilizzano l’autenticazione [OAuth 2.0 App-Only](/it/resources/fundamentals/authentication#oauth-2-0). Con l’accesso Essential puoi avere una connessione alla volta. Con tutti i livelli di accesso, le richieste di connessione possono essere effettuate fino a 50 volte per finestra di 15 minuti.

Questi endpoint degli stream di volume supportano i Post modificati. Forniranno i Post modificati insieme alla relativa cronologia delle modifiche, inclusa una array di ID Post. Per i Post senza cronologia delle modifiche, questa array conterrà un singolo ID. Per i Post che sono stati modificati, questa array contiene più ID, disposti in ordine crescente che riflette l’ordine delle modifiche, con la versione più recente nell’ultima posizione. Per saperne di più su come funzionano le modifiche ai Post, consulta la pagina [Nozioni fondamentali sulle modifiche ai Post](/it/x-api/fundamentals/edit-posts).

_Per utilizzare queste API, devi prima configurare un account con il nostro team vendite enterprise._

<Note>
  **Configurazione dell’account**

  Per accedere a questi endpoint, avrai bisogno di:

  - Un [account sviluppatore](https://developer.x.com/en/portal/petition/essential/basic-info) approvato.
  - Autenticarti utilizzando le chiavi e i token di una [App sviluppatore](/it/resources/fundamentals/developer-apps) che si trova all’interno di un [Progetto](/it/resources/fundamentals/projects).

  Scopri di più su come ottenere l’accesso agli endpoint dell’API X v2 nella nostra [guida introduttiva](/it/x-api/getting-started/getting-access).
</Note>
