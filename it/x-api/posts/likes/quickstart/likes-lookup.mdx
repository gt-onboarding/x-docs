</div>

**Primo passaggio: iniziare con uno strumento o una libreria**

Esistono diversi strumenti, esempi di codice e librerie che puoi utilizzare per effettuare una richiesta a questo endpoint, ma qui useremo lo strumento Postman per semplificare il processo.

Per caricare la raccolta Postman dell'API X v2 nel tuo ambiente, fai clic sul seguente pulsante:

[![](https://cdn.cms-twdigitalassets.com/content/dam/developer-twitter/m1_vnext/carat.svg)<br />Aggiungi API X v2 a Postman](https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400)

Una volta caricata la raccolta API X v2 in Postman, vai alla cartella "Likes" e seleziona "Liking users".¬†
¬†

**Secondo passaggio: autentica la tua richiesta**

Per effettuare una richiesta corretta a questo endpoint, dovrai utilizzare l'autenticazione [OAuth 1.0a User Context](/it/resources/fundamentals/authentication) o [OAuth 2.0 Bearer Token](/it/resources/fundamentals/authentication#oauth-2-0). Per farlo, devi aggiungere le seguenti chiavi e i seguenti token a Postman selezionando l'ambiente denominato "X API v2" e aggiungendo le seguenti variabili nei campi Initial value e Current value:

- consumer\_key con la tua API Key
- consumer\_secret con la tua API Key Secret
- access\_token con il tuo Access Token
- token\_secret con il tuo Access Token Secret
  ¬†

**Terzo passaggio: specifica un Tweet**

Con questo endpoint devi specificare l'ID del Tweet per il quale vuoi ottenere gli utenti che hanno messo Mi piace. Puoi trovare l'ID di un Tweet aprendo quel Tweet su X e prendendo il codice numerico alla fine dell'URL. Ad esempio, l'ID del Tweet del seguente URL √® 1354143047324299264.

https://x.com/TwitterDev/status/1354143047324299264

In Postman, vai alla scheda "Params" e inserisci questo ID nella colonna "Value" della variabile di percorso tweet\_id (in fondo alla sezione), assicurandoti di non includere spazi prima o dopo.¬†

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | L'ID del Tweet di cui vuoi ottenere gli utenti che hanno messo Mi piace |

**Quarto passaggio: identifica e specifica quali campi desideri recuperare**

Se fai clic sul pulsante "Send" dopo il terzo passaggio, nella risposta riceverai i campi predefiniti dell'[oggetto user](/it/x-api/fundamentals/data-dictionary#user): id, name e username.

Se desideri ricevere campi aggiuntivi oltre a id, name e username, dovrai specificare tali campi nella richiesta con i parametri [fields](/it/x-api/fundamentals/fields) e/o [expansions](/it/x-api/fundamentals/expansions).

Per questo esercizio richiederemo tre set aggiuntivi di campi da oggetti diversi:

1. Il campo aggiuntivo user.created\_at negli oggetti user principali.
2. I campi predefiniti dell'oggetto dei Tweet fissati in alto associati per gli utenti restituiti: id e text.
3. Il campo aggiuntivo tweet.created\_at negli oggetti Tweet associati.

In Postman, vai alla scheda "Params" e aggiungi la seguente coppia chiave:valore alla tabella "Query Params":

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| user.fields | created\_at | user.created\_at |
| expansions | pinned\_tweet\_id | tweet.id, tweet.text |
| tweet.fields | created\_at | tweet.created\_at |

Dovresti ora vedere il seguente URL accanto al pulsante "Send":

`https://api.x.com/2/tweets/1354143047324299264/liking_users?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=created_at`
