<div id="getting-started-with-the-manage-likes-endpoints">
  ### Guida introduttiva agli endpoint per la gestione dei Like
</div>

Questa guida rapida ti aiuterà a inviare la tua prima richiesta agli endpoint per la gestione dei Like utilizzando [Postman](/it/tutorials/postman-getting-started).

Se desideri consultare esempi di codice in diverse lingue, visita il nostro repository GitHub [API X v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code). 

<div id="prerequisites">
  #### Prerequisiti
</div>

Per completare questa guida, devi disporre di un set di [chiavi e token](/it/resources/fundamentals/authentication), che puoi generare seguendo questi passaggi:

1. [Richiedi un account sviluppatore](https://developer.x.com/en/apply-for-access) e attendi l’approvazione.
2. Crea un [Progetto](/it/resources/fundamentals/projects) e un’[App per sviluppatori](/it/resources/fundamentals/developer-apps) associata nel portale degli sviluppatori.
3. Vai alla pagina “Keys and tokens” della tua app e salva le API Key, gli Access Token e il Bearer Token nel tuo password manager.

<div id="steps-to-build-a-manage-likes-request">
  #### Passaggi per creare una richiesta per gestire i Mi piace
</div>

**Primo passaggio: inizia con uno strumento o una libreria**

Esistono diversi strumenti, esempi di codice e librerie che puoi utilizzare per effettuare una richiesta a questo endpoint, ma qui useremo Postman per semplificare il processo.

Per caricare la collection Postman dell'API X v2 nel tuo ambiente, fai clic sul seguente pulsante:

[![](https://cdn.cms-twdigitalassets.com/content/dam/developer-twitter/m1_vnext/carat.svg)<br />Aggiungi API X v2 a Postman](https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400)

Una volta caricata la collection API X v2 in Postman, vai alla cartella “Likes” e seleziona “Like a Post”.

**Secondo passaggio: autentica la tua richiesta**

Per effettuare correttamente una richiesta a questo endpoint, devi utilizzare [OAuth 1.0a User Context](/it/resources/fundamentals/authentication). Per farlo, aggiungi le seguenti chiavi e token a Postman selezionando l'ambiente denominato “X API v2” e inserendo le seguenti variabili nei campi Initial value e Current value:

- consumer\_key con la tua API Key
- consumer\_secret con la tua API Key Secret
- access\_token con il tuo Access Token
- token\_secret con il tuo Access Token Secret
   

**Terzo passaggio: specifica quale Post intendi mettere Mi piace**

Gli endpoint per la gestione dei Mi piace richiedono due ID: uno per l’utente (l’utente che desidera mettere Mi piace a un Post) e l’altro che rappresenta l’ID del Post che l’utente sta cercando di mettere o togliere Mi piace.

L’ID utente deve corrispondere all’ID dell’utente autenticato, il che significa che devi passare i [token di accesso](/it/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) associati all’ID utente quando autentichi la richiesta. In questo caso, puoi specificare l’ID del tuo stesso account. Puoi trovare il tuo ID in due modi:

1. Utilizzando l’endpoint [user lookup by username](/it/x-api/users/user-lookup-by-id), puoi passare un nome utente e ricevere il campo id.
2. Esaminando il tuo Access Token: la parte numerica è il tuo ID utente.
    

Devi anche specificare un Post a cui vuoi mettere Mi piace. Puoi trovare l’ID del Post visitando x.com, cliccando su un Post e poi controllando l’URL. Ad esempio, l’ID del Post nel seguente URL è 1228393702244134912.

https://x.com/TwitterDev/status/1228393702244134912

In Postman, vai alla scheda "Params" e inserisci il tuo ID nella colonna "Value" della variabile di percorso id. Vai alla scheda “Body” e inserisci l’ID del Post a cui desideri mettere Mi piace come valore per il parametro tweet\_id. Assicurati di non includere spazi prima o dopo alcun ID.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | (il tuo ID utente) |
| tweet\_id | (l’ID del Post a cui desideri mettere Mi piace) |

Se clicchi sul pulsante "Send", riceverai un oggetto di risposta contenente lo stato della relazione:

- Se ricevi "liked": true, allora l’id ha messo correttamente Mi piace al tweet\_id.
   

**Quarto passaggio: invia la richiesta e verifica la risposta**

Una volta configurato tutto, premi il pulsante "Send" e riceverai la seguente risposta:

```
{
    "data": {
        "liked": true
    }
}
```

Se desideri togliere il Mi piace allo stesso Post, puoi usare la richiesta “Unlike a Post”, che si trova anche nella cartella “Likes” della collection API X v2 in Postman. L’id e il tweet\_id vanno usati nello stesso modo dell’esempio precedente. Per togliere il Mi piace a un Post, non devi aggiungere un body JSON; assicurati quindi di includere i parametri di query richiesti per id e tweet\_id.
