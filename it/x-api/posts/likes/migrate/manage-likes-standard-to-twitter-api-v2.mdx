<div id="manage-likes-standard-v11-compared-to-x-api-v2">
  ### Gestione dei Mi piace: standard v1.1 rispetto all’API X v2
</div>

Se hai lavorato con gli endpoint dello standard v1.1 [POST favorites/create](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-create) e [POST favorites/destroy](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-destroy), l’obiettivo di questa guida è aiutarti a comprendere somiglianze e differenze tra gli endpoint di gestione dei Mi piace dello standard v1.1 e dell’API X v2.

- **Somiglianze**
  - Contesto utente OAuth 1.0a
- **Differenze**
  - URL degli endpoint e metodi HTTP
  - Requisiti di App e Progetti
  - Parametri della richiesta

<div id="similarities">
  #### Somiglianze
</div>

**Metodo di autenticazione OAuth 1.0a in contesto utente**

Entrambe le versioni dell’endpoint supportano [OAuth 1.0a User Context](/it/resources/fundamentals/authentication#oauth-1-0a-2). Pertanto, se in precedenza utilizzavi uno degli endpoint standard v1.1 per la gestione dei preferiti, puoi continuare a usare lo stesso metodo di autenticazione se esegui la migrazione alla versione dell’API X v2.

<div id="differences">
  #### Differenze
</div>

**URL degli endpoint e metodi HTTP**

- Endpoint standard v1.1:
  - POST https://api.x.com/1.1/favorites/create.json
    (mettere Mi piace a un Post)
  - POST https://api.x.com/1.1/favorites/destroy.json
    (togliere Mi piace a un Post)
- Endpoint API X v2:
  - POST https://api.x.com/2/tweets/:id/likes
    (mettere Mi piace a un Post)
  - DELETE https://api.x.com/2/tweets/:id/likes/:tweet\_id
    (togliere Mi piace a un Post)

**Requisiti di App e Progetto**

Gli endpoint dell’API X v2 richiedono l’uso di credenziali di una [developer App](/it/resources/fundamentals/developer-apps) associata a un [Project](/it/resources/fundamentals/projects) quando si autenticano le richieste. Tutti gli endpoint dell’API X v1.1 possono usare credenziali di App autonome o di App associate a un Project.

**Parametri della richiesta**

I seguenti parametri di richiesta standard v1.1 hanno equivalenti nell’API X v2:

| Standard v1.1 | API X v2 |
| :--- | :--- |
| id  | id  |
| includes\_entities | Nessun equivalente |

Si noti che i parametri standard v1.1 vengono passati come parametri di query, mentre i parametri dell’API X v2 vengono passati come parametri del body per l’endpoint POST o come parametri di percorso per l’endpoint DELETE.

Inoltre, l’id dell’utente che mette Mi piace a un Post non è richiesto quando si usano gli endpoint standard v1.1, poiché i [token di accesso](/it/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) passati con [OAuth 1.0a User Context](/it/resources/fundamentals/authentication) consentono di dedurre quale utente sta eseguendo l’azione di mettere/togliere Mi piace.
