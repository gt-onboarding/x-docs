<div id="comparing-x-apis-likes-endpoints">
  ## Confronto degli endpoint dei Mi piace dell’API X
</div>

Queste guide si concentreranno sui seguenti aspetti:

- **Parametri di richiesta dell’API** - L’endpoint dell’API X v2 introduce un nuovo insieme di parametri di richiesta. Alcuni parametri saranno familiari, soprattutto a chi ha integrato Labs, ma ci sono differenze importanti, come l’introduzione dei parametri [fields](/it/x-api/fundamentals/fields) ed [expansions](/it/x-api/fundamentals/expansions).

- **Requisiti per App e Progetti** - Per accedere all’API X v2, è necessario utilizzare le credenziali di una [developer App](/it/resources/fundamentals/developer-apps) associata a un [Project](/it/resources/fundamentals/projects)

<div id="likes-lookup">
  #### Ricerca dei Mi piace
</div>

**Utenti che hanno messo Mi piace a un Post**

L'endpoint degli utenti che hanno messo Mi piace è una nuova funzionalità della v2 che consente di ottenere informazioni sugli utenti che hanno messo Mi piace a un Post.

| Descrizione | API X v2 |
| :--- | :--- |
| Metodi HTTP supportati | GET |
| Dominio host | https://api.x.com |
| Percorso dell'endpoint | /2/tweets/:id/liking\_users |
| [Autenticazione](/it/resources/fundamentals/authentication) | OAuth 2.0 Bearer Token<br /><br />OAuth 1.0a User Context |
| [Limiti di frequenza](/it/resources/fundamentals/rate-limits) predefiniti per le richieste | 75 richieste ogni 15 min (per App)<br /><br />75 richieste ogni 15 min (per utente) |

**Post a cui un utente ha messo Mi piace**

Le tabelle seguenti confrontano l'endpoint Standard v1.1 [GET favorites/list](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-favorites-list) e gli endpoint dell’API X v2 per i Post a cui è stato messo Mi piace:

| Descrizione | Standard v1.1 | API X v2 |
| :--- | :--- | :--- |
| Metodi HTTP supportati | GET | GET |
| Dominio host | https://api.x.com | https://api.x.com |
| Percorso dell'endpoint | /1.1/favorites/list.json | /2/users/:id/liked\_tweets |
| [Autenticazione](/it/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 2.0 Bearer Token<br /><br />OAuth 1.0a User Context |
| [Limiti di frequenza](/it/resources/fundamentals/rate-limits) predefiniti per le richieste | 75 richieste ogni 15 min | 75 richieste ogni 15 min (per App)<br /><br />75 richieste ogni 15 min (per utente) |
| Formati dei dati | Formato Standard v1.1 | [Formato API X v2](/it/x-api/fundamentals/data-dictionary) (determinato dai parametri di richiesta fields ed expansions, non retrocompatibile con i formati v1.1)<br /><br />Per ulteriori informazioni su come migrare dal formato Standard v1.1 al formato API X v2, consulta la nostra [guida alla migrazione dei formati di dati](/it/x-api/migrate/data-format-migration). |

<div id="manage-likes">
  #### Gestione dei Mi piace
</div>

Gli endpoint v2 per la gestione dei Mi piace sostituiscono gli endpoint v1.1 [POST favorites/create](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-create) e [POST favorites/destroy](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-destroy).

Le tabelle seguenti confrontano gli endpoint standard v1.1 e quelli dell’API X v2 per la gestione dei Mi piace:

<div id="like-a-post">
  #### Mettere Mi piace a un post
</div>

| Descrizione | Standard v1.1 | API X v2 |
| :--- | :--- | :--- |
| Metodi HTTP supportati | POST | POST |
| Dominio host | https://api.x.com | https://api.x.com |
| Percorso endpoint | /1.1/favorites/create.json | /2/users/:id/likes |
| [Autenticazione](/it/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context |
| [Limiti di frequenza](/it/resources/fundamentals/rate-limits) predefiniti per le richieste | 1000 richieste ogni 24 ore (per utente)<br /><br />1000 richieste ogni 24 ore (per app) | 50 richieste ogni 15 min (per utente)<br /><br />1000 richieste ogni 24 ore (per utente, condivise con DELETE) |

<div id="unlike-a-post">
  #### Rimuovere il Mi piace da un Post
</div>

| Descrizione | Standard v1.1 | API X v2 |
| :--- | :--- | :--- |
| Metodi HTTP supportati | POST | DELETE |
| Dominio host | https://api.x.com | https://api.x.com |
| Percorso endpoint | /1.1/favorites/destroy.json | /2/users/:id/likes/:tweet\_id |
| [Autenticazione](/it/resources/fundamentals/authentication) | Contesto utente OAuth 1.0a | Contesto utente OAuth 1.0a |
| [Limiti di frequenza](/it/resources/fundamentals/rate-limits) predefiniti | 1000 richieste ogni 24 ore (per utente)<br /><br />1000 richieste ogni 24 ore (per app) | 50 richieste ogni 15 minuti (per utente)<br /><br />1000 richieste ogni 24 ore (per utente, condivise con POST) |
