<div id="manage-bookmarks">
  ### Gestire i segnalibri
</div>

Sono disponibili due metodi per gestire i segnalibri: POST e DELETE. Il metodo POST consente di creare segnalibri. Analogamente, il metodo DELETE consente di eliminare un segnalibro specifico.

È previsto un limite di 50 richieste per utente ogni 15 minuti per i metodi POST e DELETE.

Poiché stai effettuando richieste per conto di un utente con gli endpoint per la gestione dei segnalibri, devi autenticarti generando un token di accesso utente con OAuth 2.0. Puoi utilizzare il [flusso Authorization Code con PKCE](/it/resources/fundamentals/authentication/oauth-2-0/user-access-token) per farlo. Per utilizzare questo endpoint devi includere gli scope `tweet.read`, `users.read` e `bookmark.write`.

<div id="bookmarks-lookup">
  ### Ricerca dei segnalibri
</div>

L’endpoint Bookmarks lookup dispone di un solo metodo, GET. Questo metodo consente di recuperare i segnalibri dal tuo account o da un account autenticato. Per questo endpoint verranno forniti token di paginazione per scorrere insiemi di risultati di grandi dimensioni.

È previsto un rate limit per utente di 180 richieste ogni finestra di 15 minuti per il metodo GET. Con questo endpoint otterrai 800 dei tuoi Post più recenti aggiunti ai segnalibri.

Poiché stai effettuando richieste per conto di un utente con gli endpoint Bookmarks lookup, devi autenticarti generando un token di accesso utente con OAuth 2.0. Puoi utilizzare il [flusso Authorization Code con PKCE](/it/resources/fundamentals/authentication/oauth-2-0/user-access-token) per farlo. Per utilizzare questo endpoint devi fornire gli scope `tweet.read`, `users.read` e `bookmark.read`.

**Configurazione dell’account**

Per accedere a questi endpoint, ti serviranno:

- Un [account sviluppatore](https://developer.x.com/en/portal/petition/essential/basic-info) approvato.
- L’autenticazione utilizzando le chiavi e i token di una [App per sviluppatori](/it/resources/fundamentals/developer-apps) che si trova all’interno di un [Progetto](/it/resources/fundamentals/projects).

Scopri di più su come ottenere l’accesso agli endpoint dell’API X v2 nella nostra [guida introduttiva](/it/x-api/getting-started/getting-access).
