<div id="getting-started-with-the-posts-lookup-endpoints">
  ## Guida introduttiva agli endpoint di ricerca dei Post
</div>

Questa guida rapida ti aiuterà a effettuare la prima richiesta agli endpoint di ricerca dei Post con un set di campi specifici utilizzando Postman.

Se desideri vedere esempi di codice in lingue diverse, visita il nostro repository GitHub [API X v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code).

<Note>
  ### Prerequisiti

  Per completare questa guida, ti servirà un set di [chiavi e token](/it/resources/fundamentals/authentication) per autenticare la richiesta. Puoi generare queste chiavi e questi token seguendo questi passaggi:

  - [Registrati per un account sviluppatore](https://developer.x.com/en/portal/petition/essential/basic-info) e attendi l’approvazione.
  - Crea un [Progetto](/it/resources/fundamentals/projects) e un’[App per sviluppatori](/it/resources/fundamentals/developer-apps) associata nel portale per sviluppatori.
  - Vai alla pagina “Keys and tokens” della tua App per generare le credenziali richieste. Assicurati di salvare tutte le credenziali in un luogo sicuro.
</Note>

<div id="steps-to-build-a-get-tweets-request">
  ### Passaggi per creare una richiesta GET /tweets
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### Primo passaggio: iniziare con uno strumento o una libreria
</div>

Esistono diversi strumenti, esempi di codice e librerie che puoi utilizzare per inviare una richiesta a questo endpoint, ma qui useremo Postman per semplificare il processo.

Per importare la collection Postman dell'API X v2 nel tuo ambiente, fai clic sul pulsante qui sotto:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Aggiungi API X v2 a Postman
</Button>

Dopo aver importato la collection dell'API X v2 in Postman, vai alla richiesta "Post Lookup > Multiple Posts".

<div id="step-two-authenticate-your-request">
  #### Passaggio due: Autentica la tua richiesta
</div>

Per effettuare correttamente una richiesta all’API X, devi verificare di avere l’autorizzazione. Puoi autenticare la tua richiesta con:

- [OAuth 2.0 App-Only](it/resources/fundamentals/authentication)
- [OAuth 2.0 Authorization Code con PKCE](it/resources/fundamentals/authentication)
- [OAuth 1.0a User Context](it/resources/fundamentals/authentication)

Per semplicità useremo OAuth 2.0 App-Only per questa richiesta, ma se desideri richiedere [metriche](/it/x-api/fundamentals/metrics) private o Post, dovrai usare uno degli altri metodi di autenticazione.

Per utilizzare OAuth 2.0 App-Only, devi aggiungere le tue chiavi e i tuoi token (in particolare l’App Access Token, noto anche come App-only Bearer Token) a Postman selezionando l’ambiente denominato “X API v2” (nell’angolo in alto a destra di Postman) e inserendo chiavi e token nei campi “initial value” e “current value” (cliccando sull’icona a forma di occhio accanto al menu a discesa dell’ambiente).

Se hai fatto tutto correttamente, queste variabili verranno importate automaticamente nella scheda di autorizzazione della richiesta.

<div id="step-three-identify-and-specify-which-posts-you-would-like-to-retrieve">
  #### Passaggio tre: identifica e specifica quali post desideri recuperare
</div>

Devi specificare un post o un insieme di post che desideri ricevere nella richiesta. Puoi trovare l’ID del post andando su X.com, cliccando su un post e poi guardando l’URL. Ad esempio, l’ID del post del seguente URL è `1228393702244134912`.

`https://x.com/TwitterDev/status/1228393702244134912`

In Postman, vai alla scheda "Params" e inserisci questo ID, oppure una stringa di ID di post separati da virgole, nella colonna "Value" del parametro ids.

| Key  | Value |
| :---- | :----- |
| `ids` | `1228393702244134912,1227640996038684673,1199786642791452673` |

---

<div id="step-four-identify-and-specify-which-fields-you-would-like-to-retrieve">
  #### Passaggio quattro: identifica e specifica quali campi desideri recuperare
</div>

Se fai clic sul pulsante "Send" dopo il passaggio tre, nella risposta riceverai i [campi oggetto Post](/it/x-api/fundamentals/data-dictionary) predefiniti: `id`, `text` ed `edit_history_tweet_ids`.

Per questo esercizio, richiederemo tre insiemi aggiuntivi di campi da oggetti diversi:

- Il campo aggiuntivo `tweet.created_at` negli oggetti Post.
- I campi predefiniti dell’oggetto utente degli autori associati per i Post restituiti: `id`, `name` e `username`
- Il campo aggiuntivo `user.created_at` negli oggetti utente associati.

In Postman, vai alla scheda "Params" e aggiungi la seguente coppia `key:value` alla tabella "Query Params":
| Key            | Value               | Returned fields                          |
| :-------------- | :------------------- | :---------------------------------------- |
| `tweet.fields` | `created_at`        | `tweets.created_at`                      |
| `expansions`   | `author_id`         | `includes.users.id`, `includes.users.name`, `includes.users.username` |
| `user.fields`  | `created_at`        | `includes.users.created_at`              |

Dovresti vedere questo URL accanto al pulsante "Send":

```
https://api.x.com/2/tweets?ids=1228393702244134912,1227640996038684673,1199786642791452673&tweet.fields=created_at&expansions=author_id&user.fields=created_at
```
