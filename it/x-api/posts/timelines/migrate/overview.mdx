<div id="comparing-x-apis-timelines-endpoints">
  ## Confronto degli endpoint delle timeline dell’API X
</div>

Gli endpoint della timeline inversa in ordine cronologico della v2, della timeline dei post dell’utente e della timeline delle menzioni dell’utente sostituiscono rispettivamente gli endpoint [v1.1 statuses/home\_timeline](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-home_timeline), [v1.1 statuses/user\_timeline](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-user_timeline.html) e [v1.1 statuses/mentions\_timeline](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-mentions_timeline.html). Se hai codice, app o strumenti che usano una versione precedente di questo endpoint e stai valutando la migrazione al nuovo endpoint dell’API X v2, questa guida fa per te. Per una guida alla migrazione più approfondita, consulta [Migrazione dallo Standard v1.1 all’API X v2](/it/x-api/posts/timelines/migrate/standard-to-twitter-api-v2).

Questa pagina contiene tre tabelle di confronto:

- Timeline della home inversa in ordine cronologico
- Timeline dei post dell’utente
- Timeline delle menzioni dell’utente

<div id="reverse-chronological-home-timeline">
  ### Timeline Home in ordine cronologico inverso
</div>

Le tabelle seguenti confrontano gli endpoint della timeline Home standard v1.1 e dell’API X v2:

|     |     |     |
| :--- | :--- | :--- |
| **Descrizione** | **Standard v1.1** | **API X v2** |
| Documentazione | [Riferimento API](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-home_timeline) | [Riferimento API](https://developer.x.com/x-api/posts/timelines#user-mention-timeline-3) |
| Metodi HTTP supportati | `GET` | `GET` |
| Dominio host | `https://api.x.com` | `https://api.x.com` |
| Percorsi endpoint | `/1.1/statuses/home_timeline.json` | `/2/users/:id/timelines/reverse_chronological` |
| Parametri richiesti | `user_id` o `screen_name` | ID utente impostato come parametro di percorso :id |
| Autenticazione | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />[Flusso con codice di autorizzazione OAuth 2.0 con PKCE](/it/resources/fundamentals/authenticationoauth-2-0/authorization-code) |
| Limiti di frequenza/limiti di volume delle richieste | 15 richieste ogni 15 minuti con OAuth 1.0a User Context<br /><br />Limite richieste: 100.000 in un periodo di 24 ore. | 180 richieste ogni finestra di 15 minuti <br /><br />[Limiti di post](/it/x-api/fundamentals/post-cap):<br /><br />500.000 con accesso Essential<br /><br />2 milioni con accesso Elevated<br /><br />10 milioni con accesso Academic Research |
| Post predefiniti per risposta | 15  | 100 |
| Numero massimo di Post per risposta | 800 | Questo endpoint restituisce tutti i Post creati su una timeline negli ultimi 7 giorni, oltre agli 800 più recenti indipendentemente dalla data di creazione. |
| Fornisce la cronologia delle modifiche del Post | ✔   | ✔   |
| Post storici disponibili | Gli 800 Post più recenti, inclusi i Retweet | I 3.200 Post più recenti, inclusi i Retweet |
| Opzioni di navigazione della timeline | since\_id (esclusivo) usato per il polling degli aggiornamenti<br /><br />`max_id` (inclusivo) | `start_time`<br /><br />`end_time`<br /><br />`since_id` (esclusivo) usato per il polling degli aggiornamenti <br /><br />`until_id` (esclusivo) |
| Parametri opzionali per il perfezionamento dei risultati | `count`<br /><br />`exclude_replies`<br /><br />`include_rts`<br /><br />`trim_user`<br /><br />`tweet_mode`<br /><br />`since_id`<br /><br />`max_id` | `max_results`<br /><br />`exclude` (retweets,replies)<br /><br />`tweet.fields`<br /><br />`user.fields`<br /><br />`place.fields`<br /><br />`media.fields`<br /><br />`poll.fields`<br /><br />`expansions`<br /><br />`start_time`<br /><br />`end_time`<br /><br />`since_id`<br /><br />`until_id` |
| Supporta la richiesta e la ricezione di [annotazioni](/it/x-api/fundamentals/post-annotations) | N/D | Se le annotazioni sono incluse in `tweet.fields`, i risultati saranno arricchiti con dati di annotazione inferiti dal testo del Post, ad esempio "Genere musicale" e "Folk" oppure "Musicista" e "Dolly Parton". |
| Supporta la richiesta e la ricezione di specifiche [metriche](/it/x-api/fundamentals/metrics) del Post | N/D | Se le metriche sono incluse in `tweet.fields`, i risultati includeranno `public_metrics` per Post, tra cui `retweet_count`, `reply_count`, `quote_count` e `like_count`, e `non_public_metrics`, tra cui `impression_count`, `user_profile_clicks`, `url_link_clicks`.<br /><br />Metriche multimediali aggiuntive come `view_count` e metriche di riproduzione video.<br /><br />Ulteriori `organic_metrics` e `promoted_metrics` disponibili con User Context per i Post promossi. |
| Supporta la richiesta e la ricezione di [conversation\_id](/it/x-api/fundamentals/conversation-id) | N/D | Restituisce un campo `conversation_id` il cui valore rappresenta il primo Post pubblicato in un thread di risposte, per aiutarti a tracciare le conversazioni. |
| Formato JSON dei post | [Formato dati Standard v1.1](https://developer.x.com/en/docs/twitter-api/v1/data-dictionary/overview.html) | Formato [API X v2](/it/x-api/fundamentals/data-dictionary) (determinato dai parametri di richiesta fields ed expansions, non compatibile con le versioni precedenti v1.1)<br /><br />Per ulteriori informazioni sulla migrazione dal formato Standard v1.1 al formato API X v2, consulta la nostra [guida alla migrazione dei formati dati](/it/x-api/migrate/data-format-migration). |
| Ordine dei risultati | In ordine cronologico inverso | In ordine cronologico inverso |
| Paginazione dei risultati | N/D: è necessario usare la navigazione per ID del post | I risultati possono essere scorsi in avanti o indietro usando un pagination\_token |
| Richiede l’uso di credenziali di una [App per sviluppatori](/it/resources/fundamentals/developer-apps) associata a un [Progetto](/it/resources/fundamentals/projects) |     | ✔   |
