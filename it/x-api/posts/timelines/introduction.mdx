Questi tre endpoint delle timeline supportano i post modificati. Questi endpoint restituiscono sempre l’ultima versione, insieme alla cronologia delle modifiche. Qualsiasi post acquisito dopo la finestra di modifica di 30 minuti rappresenterà la sua versione finale. I metadati di modifica includono un array di ID per tutti i post nella sua cronologia. Per i post senza cronologia di modifiche, questo array conterrà un singolo ID. Per i post che sono stati modificati, questo array contiene più ID, disposti in ordine crescente in base all’ordine delle modifiche, con la versione più recente nell’ultima posizione dell’array. Per ulteriori informazioni su come funzionano le modifiche ai post, vedi la pagina [Nozioni fondamentali sulle modifiche ai post](/it/x-api/fundamentals/edit-posts).

<div id="reverse-chronological-home-timeline">
  ## Timeline home in ordine cronologico inverso
</div>

Questo endpoint consente di recuperare i Post, i Retweet e le risposte più recenti pubblicati dall’utente autenticato e dagli account che segue.

Poiché stai effettuando richieste per conto di un utente, devi autenticare questi endpoint utilizzando il [flusso di autorizzazione OAuth 2.0 con PKCE](/it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) oppure il [contesto utente OAuth 1.0a](/it/resources/fundamentals/authentication). Questo endpoint ha un limite per utente di 180 richieste per finestra di 15 minuti. Può restituire tutti i Post creati su una timeline negli ultimi 7 giorni, nonché gli 800 più recenti indipendentemente dalla data di creazione.

<div id="user-post-timeline">
  ## Timeline dei Post dell'utente
</div>

L'endpoint della timeline dei Post dell'utente fornisce accesso ai Post pubblicati da uno specifico account X. Recuperare i Post di un utente consente di creare esperienze come mostrare una timeline in un'interfaccia utente, analizzare i Post di un utente per comprenderne meglio i contenuti o creare, in modo programmatico, flussi di lavoro di coinvolgimento basati sui suoi Post. Questo endpoint offre accesso ai Post, ai Retweet, alle risposte e ai Quote Tweet più recenti di un singolo account X, in modo analogo a quanto visibile nella timeline del profilo di un utente.

L'endpoint della timeline dei Post dell'utente è un endpoint REST che riceve un singolo parametro di percorso per indicare l'utente desiderato (tramite ID utente). L'endpoint può restituire fino a 3.200 tra i Post, i Retweet, le risposte e i Quote Tweet più recenti pubblicati dall'utente.

I Post sono restituiti in ordine cronologico inverso, a partire dal più recente. I risultati sono [paginati](/it/x-api/fundamentals/pagination) fino a 100 Post per pagina. Vengono forniti token di paginazione per scorrere insiemi di Post di grandi dimensioni. Gli ID dei Post più recenti e più vecchi inclusi nella pagina specificata sono forniti anche come metadati e possono essere utilizzati per effettuare il polling delle timeline alla ricerca di Post recenti. La timeline dei Post dell'utente supporta inoltre i parametri start\_time ed end\_time per ricevere Post creati all'interno di una determinata finestra temporale.

L'endpoint della timeline dei Post dell'utente supporta i parametri [fields](/it/x-api/fundamentals/fields) e [expansions](/it/x-api/fundamentals/expansions) e restituisce il [nuovo formato dati JSON](/it/x-api/fundamentals/data-dictionary).

Per effettuare correttamente una richiesta a questo endpoint, devi autorizzarla utilizzando i metodi di autenticazione [OAuth 1.0a User Context](/it/resources/fundamentals/authentication), [OAuth 2.0 Authorization Code with PKCE](/it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) o [OAuth 2.0 App-Only](/it/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token). Devi utilizzare OAuth 1.0a User Context o OAuth 2.0 Authorization Code with PKCE quando richiedi metriche non pubbliche, metriche promosse o la timeline di un utente protetto.

L'endpoint della timeline dei Post dell'utente è progettato per supportare due modelli d'uso comuni:

- "Ottenere i Post storici di un utente": richieste alla timeline dei Post dell'utente per ricevere i Post creati dall'utente di interesse in ordine cronologico su un intervallo di tempo recente specifico. L'intervallo temporale può essere impostato utilizzando start\_time ed end\_time e scorrendo (paginando) tutti i risultati. In alcuni casi, è possibile recuperare l'intera cronologia dei Post di un utente se ha pubblicato fino a un massimo di 3.200 Post sul proprio account. I Post inclusi dipendono dalla disponibilità pubblica e dall'autenticazione utilizzata per le richieste.

- "Polling per nuovi Post": richieste alla timeline dei Post dell'utente effettuate con continuità per recuperare nuovi Post creati da un utente specifico. L'ID dell'ultimo Post ricevuto può essere impostato come parametro per le nuove richieste successive all'ultimo Post.

<div id="user-mention-timeline">
  ## Cronologia delle menzioni dell’utente
</div>

L’endpoint della cronologia delle menzioni dell’utente consente di richiedere Post che menzionano un utente X specifico, ad esempio se un account X ha menzionato @XDevelopers all’interno di un Post. Include anche le risposte ai Post dell’utente indicato. Recuperare le menzioni di un utente consente di creare esperienze come individuare rapidamente chi risponde ai Post di un utente, chi lo menziona o creare flussi di lavoro di engagement con i suoi Post in modo programmatico. L’endpoint consente di richiedere le menzioni e le risposte più recenti di un singolo utente, in modo simile a quanto si può vedere nelle [notifiche per le menzioni](https://x.com/notifications/mentions) di un utente su X.

La cronologia delle menzioni dell’utente è un endpoint REST che riceve un singolo parametro di percorso per indicare l’utente desiderato (tramite ID utente). L’endpoint può restituire fino a 800 menzioni più recenti per quell’utente.

I Post sono restituiti in ordine cronologico inverso, a partire dal più recente. I risultati sono [paginati](/it/x-api/fundamentals/pagination) fino a 100 Post per pagina. Sono forniti token di paginazione per scorrere set di Post di grandi dimensioni. Gli ID dei Post più recente e più vecchio inclusi nella pagina sono forniti come metadati, che possono essere utilizzati anche per effettuare il polling delle timeline per i Post recenti o per navigare nella timeline in modo simile all’endpoint v1.1 [mentions\_timeline](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-mentions_timeline). L’endpoint consente inoltre di specificare i parametri start\_time ed end\_time per ricevere Post creati all’interno di una determinata finestra temporale. 

Per effettuare correttamente una richiesta a questo endpoint, devi autorizzarla utilizzando i metodi di autenticazione [OAuth 1.0a User Context](/it/resources/fundamentals/authentication), [OAuth 2.0 Authorization Code with PKCE](/it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) o [OAuth 2.0 App-Only](/it/resources/fundamentals/authentication#bearer-token-also-known-as-app-only). Devi utilizzare OAuth 1.0a User Context o OAuth 2.0 Authorization Code con PKCE quando richiedi metriche non pubbliche, metriche promosse o la timeline di un utente protetto.

L’endpoint della cronologia delle menzioni dell’utente supporta i parametri [fields](/it/x-api/fundamentals/fields) ed [expansions](/it/x-api/fundamentals/expansions) e restituisce il [nuovo formato di dati JSON](/it/x-api/fundamentals/data-dictionary).
