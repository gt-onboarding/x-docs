**Passaggi per creare una richiesta per la home timeline in ordine cronologico inverso**

_Per questo esempio, effettueremo una richiesta all'endpoint della timeline dei Post dell'utente per ID, ma puoi applicare quanto appreso in questa guida rapida anche alle richieste delle timeline delle menzioni dell'utente._

**Primo passaggio: iniziare con uno strumento o una libreria**

Esistono diversi strumenti, esempi di codice e librerie che puoi utilizzare per effettuare una richiesta a questo endpoint, ma qui useremo Postman per semplificare il processo.

Per caricare la raccolta Postman dell'API X v2 nel tuo ambiente, fai clic sul seguente pulsante:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Aggiungi API X v2 a Postman
</Button>

Una volta caricata la raccolta API X v2 in Postman, vai alla cartella "Timelines" e trova la richiesta "Reverse chronological home timeline".

**Secondo passaggio: autenticare la tua richiesta**

Per inviare correttamente una richiesta all'API X, devi verificare di avere l'autorizzazione. Per farlo con questo endpoint, devi autenticare la richiesta con [OAuth 2.0 Authorization Code with PKCE](/it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) oppure con il metodo di autenticazione [OAuth 1.0a User Context](/it/resources/fundamentals/authentication).

Devi aggiungere le tue chiavi e i tuoi token a Postman. Puoi farlo selezionando l'ambiente denominato ‚ÄúX API v2‚Äù nell'angolo in alto a destra di Postman e aggiungendo chiavi e token nei campi "initial value" e "current value" (cliccando sull'icona a forma di occhio accanto al menu a discesa dell'ambiente).

Se hai eseguito correttamente questa operazione, queste variabili verranno inserite automaticamente nella scheda di autorizzazione della richiesta.

**Terzo passaggio: identificare e specificare l'utente per cui desideri recuperare la home timeline**

Devi specificare un utente per cui desideri recuperare i Post recenti nella richiesta. In questo esempio, passeremo un singolo ID utente.

Gli ID utente sono semplicemente il valore numerico che rappresenta un handle di account, reperibile nell'URL del profilo di un account. Ad esempio, il nome utente del seguente account √® `XDevelopers`.

`https://x.com/XDevelopers`

Per convertire questo nome utente nell'ID utente, utilizza l'[endpoint users lookup](/it/x-api/users/lookup/introduction) con il nome utente e individua l'ID numerico nel payload. Nel caso di @XDevelopers, l'ID utente √® 2244994945.

In Postman, vai alla scheda "Params" e inserisci questo ID utente nella colonna "Value" del parametro id.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 2244994945 |

**Quarto passaggio: identificare e specificare quali campi desideri recuperare**

Se fai clic sul pulsante "Send" dopo il terzo passaggio, riceverai nella risposta i campi predefiniti del [Post object](/it/x-api/fundamentals/data-dictionary#tweet): id, text ed edit\_history\_tweet\_ids.

Se desideri ricevere campi aggiuntivi, dovrai specificarli nella richiesta con i parametri [field](/it/x-api/fundamentals/fields) e/o [expansion](/it/x-api/fundamentals/expansions).

Per questo esercizio, richiederemo tre set aggiuntivi di campi provenienti da oggetti differenti:

1. Il campo aggiuntivo tweet.created\_at negli oggetti utente principali.

2. I campi predefiniti dell‚Äôoggetto utente degli autori associati per i post restituiti: id, name e username

3. Il campo aggiuntivo user.created\_at negli oggetti utente associati.
   ¬†

In Postman, vai alla scheda "Params" e aggiungi la seguente coppia chiave:valore alla tabella "Query Params":

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| `tweet.fields` | `created_at` | `tweets.created_at` |
| `expansions` | `author_id` | `includes.users.id`, `includes.users.name`, `includes.users.username` |
| `user.fields` | `created_at` | `includes.users.created_at` |
| max\_results | 5   |     |

Ora dovresti vedere il seguente URL accanto al pulsante "Send":

`https://api.x.com/2/users/:id/timelines/reverse_chronological?tweet.fields=created_at&expansions=author_id&user.fields=created_at&max_results=5`

<Info>
  **Nota:**

  In Postman, il parametro di percorso :id nel campo URL **non** si aggiorna automaticamente al valore che inserisci nel campo dei parametri `id`, motivo per cui l‚ÄôURL sopra include `:id` e non 2244994945.
</Info>

**Passo sei: esegui la paginazione dei risultati**
