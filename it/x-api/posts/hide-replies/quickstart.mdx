<div id="getting-started-with-the-hide-replies-endpoint">
  ## Introduzione all’endpoint per nascondere le risposte
</div>

Questa guida rapida ti aiuterà a inviare la tua prima richiesta all’endpoint per nascondere le risposte usando [Postman](/it/tutorials/postman-getting-started).

Se vuoi vedere degli snippet di codice in varie lingue, visita la [pagina di riferimento dell’API per nascondere le risposte](/it/x-api/posts/hide-replies#put-2-tweets-id-hidden).

<Note>
  ### Prerequisiti

  Per completare questa guida, ti servirà un set di [keys and tokens](/it/resources/fundamentals/authentication) per autenticare la richiesta. Puoi generare queste chiavi e token seguendo questi passaggi:

  - [Richiedi un account sviluppatore](https://developer.x.com/en/apply-for-access) e attendi l’approvazione.
  - Crea un [Project](/it/resources/fundamentals/projects) e una [developer App](/it/resources/fundamentals/developer-apps) associata nel portale per sviluppatori.
  - Vai alla pagina “Keys and tokens” della tua App per generare le credenziali necessarie. Assicurati di salvare tutte le credenziali in un luogo sicuro.
</Note>

<div id="steps-to-build-a-put-tweetsidhidden-request">
  ### Procedura per creare una richiesta PUT /tweets/:id/hidden
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### Passaggio uno: iniziare con uno strumento o una libreria
</div>

Esistono diversi strumenti, esempi di codice e librerie che puoi utilizzare per effettuare una richiesta a questo endpoint, ma qui useremo Postman per semplificare il processo.

Per importare la raccolta API X v2 in Postman nel tuo ambiente, fai clic sul seguente pulsante:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Aggiungi API X v2 a Postman
</Button>

Una volta importata la raccolta API X v2 in Postman, vai all’endpoint “hide replies”.

<div id="step-two-authenticate-your-request">
  #### Passaggio due: autentica la tua richiesta
</div>

Per inviare correttamente una richiesta all’API X, devi verificare di avere le autorizzazioni. Per farlo con questo endpoint, devi autenticare la richiesta usando [OAuth 1.0a User Context](/it/resources/fundamentals/authentication) oppure [OAuth 2.0 Authorization Code with PKCE](it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

In questo esempio useremo OAuth 1.0a User Context.

Devi aggiungere le tue chiavi e i tuoi token — in particolare API Key, API Secret Key, OAuth 1.0a user Access Token e OAuth 1.0a user Access Token Secret — in Postman. Puoi farlo selezionando l’ambiente denominato “API X v2” nell’angolo in alto a destra di Postman e aggiungendo chiavi e token nei campi “initial value” e “current value” (facendo clic sull’icona a forma di occhio accanto al menu a discesa dell’ambiente).

Queste variabili verranno automaticamente riportate nella scheda Authorization della richiesta se hai eseguito correttamente l’operazione.
 

<div id="step-three-find-a-post-id-to-hide">
  #### Terzo passaggio: trova un ID di Post da nascondere
</div>

L’endpoint per nascondere le risposte può nascondere o ripristinare la visibilità delle risposte per conto di un utente autorizzato. Poiché in questo esempio utilizziamo i token di accesso associati al tuo profilo utente, potrai nascondere le risposte degli utenti che partecipano a una conversazione iniziata da te. Allo stesso modo, se utilizzassi token di accesso appartenenti a un altro utente che ha autorizzato la tua app, potresti moderare le risposte a qualsiasi conversazione iniziata da quell’account.

Chiedi a un amico di rispondere a un Post (informandolo che stai testando la funzione di nascondere le risposte) oppure rispondi a uno qualsiasi dei tuoi Post da un account di test. Fai clic su quella risposta, quindi copia la parte numerica del suo URL. Quello sarà l’ID del Post che nasconderemo.

In questo caso, esamineremo il seguente Post, che ha l’ID `1232720193182412800`:

`https://x.com/TwitterDev/status/1232720193182412800`

<div id="step-four-hide-the-post">
  #### Quarto passaggio: Nascondere il post
</div>

In Postman, apri la cartella Hide replies e seleziona Hide a reply. Nella scheda Params, incolla l’ID del post accanto al campo id (non è necessario sostituire :id nell’URL). Fai clic su "Send" e vedrai una risposta positiva.

`{"hidden":true}`

<div id="step-five-unhide-the-post">
  #### Passaggio cinque: mostrare nuovamente il Post
</div>

I Post nascosti vengono spostati in una scheda separata nell’app X. Per rendere nuovamente visibile un Post in Postman, apri la cartella Hide replies e seleziona Unhide a reply. Nella scheda Params, incolla lo stesso ID del Post usato nel passaggio precedente nel campo id. Fai clic su “Send” e vedrai una risposta positiva.

`{"hidden":false}`

Il campo hidden rappresenta lo stato di visibilità del Post. Uno stato hidden pari a true significa che il Post è nascosto. Allo stesso modo, false significa che il Post non è nascosto.
