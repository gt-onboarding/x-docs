<div id="manage-retweets-standard-v11-compared-to-x-api-v2">
  ### Gestione dei Retweet: Standard v1.1 rispetto all’API X v2
</div>

Se hai lavorato con gli endpoint dello standard v1.1 [POST statuses/retweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-retweet-id) e [POST statuses/unretweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-unretweet-id), l’obiettivo di questa guida è aiutarti a comprendere somiglianze e differenze tra gli endpoint dei Retweet dello standard e dell’API X v2.

- **Somiglianze**
  - Autenticazione
- **Differenze**
  - URL degli endpoint e metodi HTTP
  - Limitazioni delle richieste
  - Requisiti di App e Progetti
  - Parametri della richiesta

<div id="similarities">
  #### Somiglianze
</div>

**Autenticazione**

Sia gli endpoint di gestione dei Retweet dello standard v1.1 ([POST statuses/retweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-retweet-id) e [POST statuses/unretweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-unretweet-id)) sia quelli dell’API X v2 utilizzano [OAuth 1.0a User Context](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Pertanto, se in precedenza utilizzavi uno degli endpoint di lookup dei Retweet dello standard v1.1, puoi continuare a usare lo stesso metodo di autenticazione se esegui la migrazione alla versione per l’API X v2. 

<div id="differences">
  #### Differenze
</div>

**URL degli endpoint e metodi HTTP**

- Endpoint Standard v1.1:
  - https://api.x.com/1.1/statuses/retweet/:id.json
    (Esegue un Retweet di un Post. Restituisce il Post originale con i dettagli del Retweet incorporati)
  - https://api.x.com/1.1/statuses/unretweet/:id.json
    (Annulla un Retweet. Restituisce il Post originale con i dettagli del Retweet incorporati)
- Endpoint API X v2:
  - https://api.x.com/2/tweets/:id/retweets
    (Esegue il Retweet di un determinato Post)
  - https://api.x.com/2/users/:id/retweets/:source\_tweet\_id
    (Annulla il Retweet di un determinato Post)

**Requisiti di App e Progetto**

Gli endpoint dell’API X v2 richiedono l’uso di credenziali di una [App sviluppatore](/it/resources/fundamentals/developer-apps) associata a un [Progetto](/it/resources/fundamentals/projects) quando autentichi le richieste. Tutti gli endpoint dell’API X v1.1 possono utilizzare credenziali di App autonome o di App associate a un Progetto.

**Parametri della richiesta**

I seguenti endpoint standard v1.1 accettavano due parametri di query (user\_id o screen\_name). L’API X v2 accetta solo l’ID utente numerico, che deve essere passato come parte del path dell’endpoint.

| Standard v1.1 | API X v2 |
| :--- | :--- |
| **id** | **id** |
| **includes\_entities** | Nessun equivalente |

Nota: i parametri dello Standard v1.1 vengono passati come parametri di query, mentre i parametri dell’API X v2 vengono passati come parametri nel body per l’endpoint POST o come parametri nel path per l’endpoint DELETE.

Inoltre, non è necessario specificare l’ID dell’utente che esegue il Retweet di un Post quando si utilizzano gli endpoint standard v1.1, poiché i [token di accesso](/it/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) passati con il [contesto utente OAuth 1.0a](/it/resources/fundamentals/authentication) determinano quale utente sta avviando o annullando il Retweet.
