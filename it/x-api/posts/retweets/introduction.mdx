<div id="retweets-lookup">
  ### Ricerca dei Retweet
</div>

Con l’endpoint di lookup dei Retweet puoi ottenere l’elenco degli account che hanno fatto Retweet di un Post. Per questo endpoint vengono forniti token di paginazione per scorrere grandi insiemi di risultati in batch fino a 100 utenti.

Puoi autenticare questi endpoint con [OAuth 1.0a User Context](/it/resources/fundamentals/authentication), \[OAuth 2.0 App-Only]\(https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), o [OAuth 2.0 Authorization Code con PKCE](/it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3). 

<div id="retweets-of-me">
  ### Repost dei miei Post
</div>

La ricerca “Repost dei miei Post” consente di vedere quali tuoi Post sono stati condivisi da altri.

Questo offre agli utenti informazioni su quali dei loro Post vengono condivisi e risuonano con il proprio pubblico, aiutando a favorire interazioni più mirate e ad apportare modifiche alla strategia dei contenuti.

Puoi autenticare questi endpoint con [OAuth 1.0a User Context](/it/resources/fundamentals/authentication) o con [OAuth 2.0 Authorization Code with PKCE](/it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

<div id="manage-retweets">
  #### Gestisci i Retweet
</div>

Gli endpoint per la gestione dei Retweet consentono di eseguire un Retweet o annullare un Retweet di un Post specifico per conto di un account autenticato. Per questo gruppo di endpoint sono disponibili due metodi: POST e DELETE. Il metodo POST consente di fare Retweet di un Post, mentre il metodo DELETE consente di annullare il Retweet di un determinato Post.

Poiché stai effettuando richieste per conto di un utente, devi autenticare questi endpoint con [OAuth 1.0a User Context](/it/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code con PKCE](it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) e utilizzare i token di accesso utente associati all’utente per conto del quale stai effettuando la richiesta. Puoi generare questo token di accesso utente utilizzando il [flusso OAuth a 3 passaggi](/it/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) o il [flusso Authorization Code con PKCE](/it/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0). Puoi fare Retweet di un Post dal tuo account o dall’account di un utente autenticato. Con entrambi gli endpoint è previsto un limite per utente di 50 richieste ogni 15 minuti per endpoint.
