<div id="authentication">
  #### Autenticazione
</div>

Tutti gli endpoint dell’API X v2 richiedono di autenticare le richieste con un set di credenziali, note anche come chiavi e token.

Per autenticare le richieste all’endpoint di ricerca dei Retweet puoi usare OAuth 1.0a User Context oppure un OAuth 2.0 Bearer Token.

Gli endpoint per la gestione dei Retweet richiedono l’uso di OAuth 1.0a User Context; ciò significa che devi utilizzare un set di chiavi API e token di accesso utente per completare correttamente la richiesta. I token di accesso devono essere associati all’utente per conto del quale stai effettuando la richiesta. Se desideri generare un set di token di accesso per un altro utente, quest’ultimo deve autorizzare o autenticare la tua app tramite il flusso OAuth a 3 passaggi (3-legged OAuth).

Tieni presente che OAuth 1.0a può risultare complesso da utilizzare. Se non conosci questo metodo di autenticazione, ti consigliamo di usare una [libreria](/it/x-api/tools-and-libraries/overview) per autenticare correttamente le tue richieste.

**Nota bene**

Se richiedi i seguenti campi, è obbligatorio usare OAuth 1.0a User Context:

- tweet.fields.non\_public\_metrics
- tweet.fields.promoted\_metrics
- tweet.fields.organic\_metrics
- media.fields.non\_public\_metrics
- media.fields.promoted\_metrics
- media.fields.organic\_metrics

<div id="developer-portal-projects-and-developer-apps">
  #### Portale sviluppatori, Progetti e App sviluppatore
</div>

Per ottenere un set di credenziali di autenticazione compatibili con gli endpoint dell’API X v2, devi [registrarti per un account sviluppatore](https://developer.x.com/en/portal/petition/essential/basic-info), configurare un [Progetto](/it/resources/fundamentals/projects) all’interno di tale account e creare un’[App sviluppatore](/it/resources/fundamentals/developer-apps) all’interno di quel Progetto. Potrai quindi trovare le tue chiavi e i tuoi token all’interno della tua App sviluppatore. 
 

<div id="rate-limits">
  #### Limiti di frequenza
</div>

Ogni giorno, molte migliaia di sviluppatori inviano richieste all’API X. Per gestire l’elevato volume di queste richieste, su ciascun endpoint vengono applicati [limiti di frequenza](/it/x-api/fundamentals/rate-limits) che restringono il numero di richieste effettuabili per conto della tua app o di un utente autenticato.

Gli endpoint per la gestione dei Retweet sono limitati a 50 richieste ogni 15 minuti (per utente). Inoltre, per l’endpoint POST, il limite è di 300 richieste in una finestra di 3 ore (per utente, per app).

Per l’endpoint di lookup dei Retweet, il limite è di 75 richieste per finestra di 15 minuti. Inoltre, questo endpoint restituisce solo i 100 utenti che hanno retwittato più di recente.

<div id="fields-and-expansions">
  #### Campi ed espansioni
</div>

L’API X v2 consente di selezionare con precisione quali dati ottenere dall’API tramite un set di strumenti chiamati campi ed espansioni. Il parametro expansion consente di includere nel risultato gli oggetti referenziati nel payload. Ad esempio, questo endpoint consente di richiedere le seguenti [espansioni](/it/x-api/fundamentals/expansions):

- attachments.poll\_ids
- attachments.media\_keys
- author\_id, entities.mentions.username
- geo.place\_id
- in\_reply\_to\_user\_id,
- referenced\_tweets.id,
- referenced\_tweets.id.author\_id

Il parametro fields consente di scegliere esattamente quali [campi](/it/x-api/fundamentals/fields) dei vari oggetti dati si desidera ricevere. Questi endpoint restituiscono principalmente oggetti Post. Per impostazione predefinita, l’oggetto Post include i campi id e text. Per ottenere campi aggiuntivi come tweet.created\_at o tweet.entities, è necessario richiederli esplicitamente tramite il parametro fields. Alcuni campi importanti da considerare per l’integrazione sono i dati dei sondaggi, le metriche, le annotazioni dei Post e i campi conversation ID.

Abbiamo aggiunto al nostro [dizionario dati dell’API X v2](/it/x-api/fundamentals/data-dictionary) una guida su come [usare insieme campi ed espansioni](/it/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions).
