La ricerca dei Post è una funzione importante per far emergere le conversazioni su X relative a uno specifico argomento o evento. Sebbene questa funzionalità esista in X, questi endpoint offrono maggiore flessibilità e potenza nel filtrare e acquisire Post, così da trovare più facilmente i dati pertinenti per la tua ricerca; creare applicazioni di “ascolto” quasi in tempo reale; oppure, in generale, esplorare, analizzare e/o agire sui Post legati a un tema di interesse. 

Offriamo due endpoint che consentono di cercare Post: ricerca recente e ricerca dell’archivio completo. Entrambi questi endpoint REST condividono progettazione e funzionalità comuni, incluso l’uso di un’unica query di ricerca per filtrare i Post su uno specifico argomento. Queste query di ricerca si creano con un set di operatori che confrontano attributi dei Post e degli utenti, come parole chiave del messaggio, hashtag e URL. Gli operatori possono essere combinati in query con logica booleana e parentesi per affinare il comportamento di corrispondenza. 

Sia la ricerca recente sia la ricerca dell’archivio completo forniscono metadati relativi ai Post modificati. Tutti gli oggetti dei Post creati dal 29 settembre 2022 includono metadati di modifica del Post, anche se il Post non è mai stato modificato. Ogni volta che un Post viene modificato, viene creato un nuovo ID del Post. La cronologia delle modifiche di un Post è documentata da un array di ID del Post, a partire dall’ID originale.

Questi endpoint restituiscono sempre l’ultima modifica insieme all’eventuale cronologia delle modifiche. Qualsiasi Post raccolto dopo la finestra di modifica di 30 minuti rappresenterà la sua versione finale. Per saperne di più sui metadati di modifica dei Post, consulta la pagina [Nozioni di base sui Post modificati](/it/x-api/fundamentals/edit-posts).

Una volta impostata la query e avviata la ricezione dei Post, questi endpoint consentono di navigare i risultati sia per intervalli temporali sia per intervalli di ID dei Post. Questo è pensato per supportare due casi d’uso comuni: 

- **Storico**: le richieste riguardano un periodo di interesse, senza concentrarsi sulla natura in tempo reale dei dati. Si effettua una singola richiesta e tutti i dati corrispondenti vengono restituiti utilizzando la paginazione, se necessario. Questa è la modalità predefinita per la ricerca dei Post.
- **Polling** o **ascolto**: le richieste vengono effettuate in modalità “ci sono nuovi Post dalla mia ultima richiesta?”. Le richieste sono continue e in genere il caso d’uso è focalizzato su un “ascolto” quasi in tempo reale dei Post di interesse.

Molti operatori e limiti di query sono esclusivi per l’[accesso Enterprise](/it/x-api/getting-started/about-x-api), il che significa che devi utilizzare chiavi e token di un’App all’interno di un [Project](/it/resources/fundamentals/projects) con accesso Enterprise per sfruttare le funzionalità aggiuntive. Puoi saperne di più nelle sezioni degli endpoint riportate di seguito.

Sia la ricerca recente sia la ricerca dell’archivio completo contribuiscono ai [limiti di post](/it/x-api/fundamentals/post-cap) mensili.

**Configurazione dell’account**

Per accedere a questi endpoint, ti serviranno:

- Un [account sviluppatore](https://developer.x.com/en/portal/petition/essential/basic-info) approvato.
- L’autenticazione utilizzando le chiavi e i token di una [developer App](/it/resources/fundamentals/developer-apps) che si trova all’interno di un [Project](/it/resources/fundamentals/projects). 

Scopri di più su come ottenere l’accesso agli endpoint dell’API X v2 nella nostra [guida introduttiva](/it/x-api/getting-started/getting-access).

<div id="recent-search">
  ## Ricerca recente
</div>

L’endpoint di ricerca recente consente di accedere in modo programmatico a Post pubblici filtrati pubblicati nell’ultima settimana ed è disponibile per tutti gli sviluppatori che dispongono di un account sviluppatore e utilizzano chiavi e token di un’[App](/it/resources/fundamentals/developer-apps) all’interno di un [Project](/it/resources/fundamentals/projects).

Puoi autenticare le tue richieste con [OAuth 1.0a User Context](/it/resources/fundamentals/authentication), [OAuth 2.0 App-Only](/it/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) o [OAuth 2.0 Authorization Code with PKCE](it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). Tuttavia, se desideri ricevere metriche private o una ripartizione tra metriche organiche e promosse nei risultati dei tuoi Post, dovrai usare OAuth 1.0a User Context oppure OAuth 2.0 Authorization Code with PKCE e passare Access Token utente associati all’utente che ha pubblicato il contenuto in questione.

Questo endpoint può restituire fino a 100 Post per richiesta in ordine cronologico inverso e fornisce token di [paginazione](/it/x-api/posts/search/integrate/paginate) per scorrere insiemi di Post corrispondenti di grandi dimensioni.

Quando utilizzi un Project con accesso standard, puoi usare il set di base di [operatori](/it/x-api/posts/search/integrate/build-a-query) ed effettuare query fino a 512 caratteri. Quando utilizzi un Project con accesso Enterprise, hai a disposizione operatori aggiuntivi. I Project con accesso Enterprise possono effettuare query fino a 4096 caratteri.

Scopri di più sui [livelli di accesso](/it/x-api/getting-started/about-x-api).

<div id="full-archive-search">
  ## Ricerca dell’archivio completo
</div>

L’endpoint di ricerca dell’archivio completo v2 è disponibile solo per i Progetti con accesso [Pro](/it/x-api/getting-started/about-x-api "Pro") e [Enterprise](/it/x-api/getting-started/about-x-api). L’endpoint consente di accedere in modo programmatico ai Post pubblici dall’archivio completo, risalendo al primo Post del marzo 2006, in base alla query di ricerca.

Puoi autenticare le richieste a questo endpoint utilizzando [OAuth 2.0 App-Only](/it/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) e il [token di accesso dell’App](/it/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token) deve provenire da un’App all’interno di un Progetto con accesso Pro o Enterprise. Poiché con questo endpoint non puoi effettuare richieste per conto di altri utenti (OAuth 1.0a User Context o OAuth 2.0 Authorization Code con PKCE), non potrai recuperare [metriche](/it/x-api/fundamentals/metrics) private.

Questo endpoint può restituire fino a 500 Post per richiesta in ordine cronologico inverso e fornisce token di [paginazione](/it/x-api/posts/search/integrate/paginate) per scorrere set di Post corrispondenti di grandi dimensioni.

**Nota:** Se richiedi [annotazioni](/it/x-api/fundamentals/post-annotations) tramite il parametro tweet.fields, il parametro max\_results è attualmente limitato a un valore massimo di 100. Questo potrebbe cambiare in futuro; tieni presente questa limitazione.

Poiché questo endpoint è disponibile solo per chi è stato approvato per l’accesso Pro ed Enterprise, hai accesso all’intero set di [operatori](/it/x-api/posts/search/integrate/build-a-query) di ricerca e puoi eseguire query fino a 1024 caratteri.
