I Messaggi Diretti consentono conversazioni private su X. Sono una delle funzionalità più popolari di X, con un'ampia varietà di casi d'uso: dalle chat di gruppo tra amici all'assistenza clienti per brand di tutto il mondo. Le nuove versioni v2 degli endpoint dei Messaggi Diretti verranno introdotte a fasi; questa prima fase include endpoint fondamentali per creare Messaggi Diretti e per elencare gli eventi delle conversazioni. Per la prima volta, l'API X v2 supporta conversazioni di gruppo.

Questo rilascio iniziale di Gestione Messaggi Diretti include tre metodi POST per creare Messaggi Diretti:

- **POST /2/dm\_conversations/with/:participant\_id/messages** - Crea un Messaggio Diretto uno a uno. Questo metodo crea una nuova conversazione 1-1 oppure recupera la conversazione corrente e vi aggiunge il Messaggio Diretto. Il parametro di percorso :participant\_id è l'ID utente dell'account che riceve il messaggio.
- **POST /2/dm\_conversations** - Crea una nuova conversazione di gruppo e vi aggiunge un Messaggio Diretto. Queste richieste richiedono un elenco di partecipanti alla conversazione. Nota che puoi creare più conversazioni con lo stesso elenco di partecipanti. Queste richieste restituiranno sempre un nuovo ID conversazione.
- **POST /2/dm\_conversations/:dm\_conversation\_id/messages** - Crea un Messaggio Diretto e lo aggiunge a una conversazione esistente. Il parametro di percorso :dm\_conversation\_id è l'ID della conversazione a cui verrà aggiunto il messaggio.

Nota che gli ID degli eventi dei Messaggi Diretti sono comuni tra v1.1 e v2 (così come nell'app X), quindi i metodi v1.1 per nascondere/eliminare i Messaggi Diretti possono essere usati insieme a questo nuovo endpoint v2. Nota inoltre che le Enterprise e Premium Account Activity API supportano i messaggi uno a uno v2, ma non supportano ancora le conversazioni di gruppo.

È previsto un rate limit utente di 200 richieste ogni 15 minuti per il metodo POST. È inoltre previsto un rate limit di 1000 richieste ogni 24 ore per utente. Inoltre, è previsto un rate limit di 15000 richieste ogni 24 ore. Nota che questi rate limit sono condivisi tra questi endpoint POST.

Poiché stai effettuando richieste per conto di un utente con gli endpoint per la gestione dei Post, devi autenticarti con [OAuth 1.0a User Context](https://developer-staging.x.com/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code con PKCE](it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) e usare un access token utente associato a un utente che ha autorizzato la tua app. Per generare questo access token utente con OAuth 1.0a, puoi usare il [flusso OAuth a 3 passaggi](/it/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token). Per generare un access token utente con OAuth 2.0, puoi usare il [flusso Authorization Code con PKCE](/it/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce).

<Note>
  **Configurazione dell'account**

  Per accedere a questi endpoint, ti serviranno:

  - Un [account sviluppatore](https://developer.x.com/en/portal/petition/essential/basic-info) approvato.
  - Autenticarti usando le chiavi e i token di una [developer app](/it/resources/fundamentals/developer-apps) che si trova all'interno di un [progetto](/it/resources/fundamentals/projects).

  Scopri di più su come ottenere l'accesso agli endpoint dell'API X v2 nella nostra [guida introduttiva](/it/x-api/getting-started/getting-access).
</Note>
