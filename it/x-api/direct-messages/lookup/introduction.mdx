Questa versione iniziale del lookup dei Messaggi Diretti include tre metodi GET:

- **GET /2/dm\_conversations/with/:participant\_id/dm\_events** - Recupera gli eventi dei Messaggi Diretti associati a una conversazione uno a uno. Il parametro di percorso :participant\_id è l’ID utente dell’account che sta conversando con l’utente autenticato che effettua questa richiesta.
- **GET /2/dm\_conversations/:dm\_conversation\_id/dm\_events** - Recupera gli eventi dei Messaggi Diretti associati a uno specifico ID conversazione, come indicato dal parametro di percorso :dm\_conversation\_id.
- **GET /2/dm\_events** - Recupera gli eventi dei Messaggi Diretti associati a un utente, incluse sia le conversazioni uno a uno sia quelle di gruppo. Sono disponibili eventi fino a 30 giorni prima.

Si noti che gli ID degli eventi dei Messaggi Diretti sono comuni tra la v1.1 e la v2 (così come nell’App X), quindi il metodo v1.1 per elencare un singolo evento può essere utilizzato insieme a questi nuovi endpoint v2. Si noti anche che le Enterprise e Premium Account Activity API supportano i messaggi uno a uno v2, ma non supportano ancora le conversazioni di gruppo.

Con questa versione, sono supportati tre tipi di evento e questi endpoint supportano parametri di query per filtrarli:

- **MessageCreate** - È stato creato un messaggio.
- **ParticipantsJoin** - Un nuovo partecipante è entrato in una conversazione.
- **ParticipantsLeave** - Un partecipante ha lasciato una conversazione.

È previsto un rate limit per utente di 300 richieste ogni 15 minuti per i metodi GET. Questo rate limit è condiviso tra questi endpoint GET.

Poiché stai effettuando richieste per conto di un utente con gli endpoint v2 dei Messaggi Diretti, devi autenticarti con [OAuth 1.0a User Context](/it/resources/fundamentals/authentication) oppure con [OAuth 2.0 Authorization Code with PKCE](/it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3), utilizzando Access Token associati agli utenti che hanno autorizzato la tua App X. Per generare questi Access Token con OAuth 1.0a, puoi utilizzare il [flusso OAuth a 3 passaggi](/it/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow). Per generare Access Token utente con OAuth 2.0, puoi utilizzare il [flusso Authorization Code con PKCE](/it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

**Configurazione dell’account**

Per accedere a questi endpoint, ti serviranno:

- Un [account sviluppatore](https://developer.x.com/en/portal/petition/essential/basic-info) approvato.
- Autenticarti utilizzando le chiavi e i token di una [developer App](/it/resources/fundamentals/developer-apps) che si trova all’interno di un [Project](/it/resources/fundamentals/projects).

Scopri di più su come ottenere l’accesso agli endpoint dell’API X v2 nella nostra [guida introduttiva](/it/x-api/getting-started/getting-access).
