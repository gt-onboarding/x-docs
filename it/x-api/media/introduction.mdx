Un oggetto multimediale rappresenta una singola foto, un video o una GIF animata. Gli oggetti multimediali sono utilizzati da molti endpoint dell’API X e possono essere inclusi in post, messaggi diretti, profili utente, creatività pubblicitarie e altro ancora. Ogni oggetto multimediale può avere più varianti di visualizzazione o riproduzione, con risoluzioni o formati diversi.

<div id="media-types-size-restrictions">
  ## Tipi di media e limiti di dimensione
</div>

Limiti di dimensione per il caricamento tramite API

- **Immagine**: `5 MB`
- **GIF**: `15 MB`
- **Video**: `512 MB` (quando si usa `media_category=amplify_video`)

<div id="creation">
  ## Creazione
</div>

Oggetti come Post, Messaggi Diretti, foto profilo utente, card Ads ospitate, ecc. possono contenere uno o più oggetti multimediali. Questi oggetti di primo livello sono collettivamente denominati entità. All’API di creazione dell’entità pertinente (ad es. [`POST /2/tweets`](/it/x-api/posts/creation-of-a-post)) è possibile passare uno o più oggetti multimediali utilizzando un `media_id` univoco.

Un’entità che contiene oggetti multimediali può essere creata seguendo questi passaggi:

1. Carica i file multimediali utilizzando il caricamento [a blocchi](/it/x-api-/media/quickstart/media-upload-chunked) consigliato (immagini/GIF/video) oppure il caricamento [semplice](/it/x-api/media-upload/media-upload) precedente (solo immagini).
2. Ricevi un `media_id` dal passaggio 1. Questo passaggio può essere ripetuto più volte con media diversi se l’entità consente di passare più parametri `media_id`.
3. Crea l’entità chiamando l’endpoint appropriato, includendo il `media_id` e gli altri parametri richiesti. Ad esempio, allega un `media_id` a un Post utilizzando l’endpoint [`POST /2/tweets`](/it/x-api/posts/creation-of-a-post).
