**Guida introduttiva**

<div id="authentication">
  ### Autenticazione
</div>

Puoi autenticare questo endpoint con [OAuth 1.0a User Context](/it/resources/fundamentals/authentication), \[OAuth 2.0 App-Only]\(https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), o [OAuth 2.0 Authorization Code con PKCE](it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).
 

<div id="making-a-request">
  ### Effettuare una richiesta
</div>

Puoi chiamare l’endpoint Communities Lookup fornendo un ID, come mostrato di seguito (utilizzando l’ID `1758747817642700922`):

```
      curl --location 'https://api.x.com/2/communities/1758747817642700922' --header 'Authorization: ••••••'
```

Se la richiesta va a buon fine, vedrai una risposta JSON come la seguente:

```
{
    "data": {
        "access": "Public",
        "description": "Welcome to the Anime Community! Where anime fans gather to share their favorite shows and discuss everything anime-related.",
        "join_policy": "Open",
        "name": "Anime Community"
    }
}
```

<div id="community-fields">
  ### Campi Community
</div>

Puoi recuperare dati aggiuntivi associati a una community specificando i Community Fields.

Sono disponibili i seguenti Community Fields:

| Valore del campo | Tipo | Descrizione |     |
| :--- | :--- | :--- | :--- |
| created\_at | date (ISO 8601) | Data e ora di creazione della Community. |     |
| id  | string | L’identificatore univoco della Community. |     |
| name | string | Il nome della Community. |     |
| description | string | Il testo della descrizione della Community, se presente. |     |
| access | string | Il livello di accesso della Community.  <br />  <br />Può essere uno tra:<br /><br />\* `Public`<br />\* `Closed` |     |
| join\_policy | string | La policy di adesione alla Community.  <br />  <br />Può essere una tra:<br /><br />\* `Open`<br />\* `RestrictedJoinRequestsDisabled`<br />\* `RestrictedJoinRequestsRequireAdminApproval`<br />\* `RestrictedJoinRequestsRequireModeratorApproval`<br />\* `SuperFollowRequired` |     |
| member\_count | integer | Il numero di membri che hanno aderito alla Community. |     |

<div id="example-request">
  #### Esempio di richiesta
</div>

Puoi chiamare l’endpoint Communities Lookup con campi aggiuntivi della community come mostrato di seguito (utilizzando l’ID `1758747817642700922`):

```
curl --location 'https://api.x.com/2/communities/1758747817642700922?community.fields=access,created_at,description,id,join_policy,member_count,name' --header 'Authorization: ••••••'
```

**Esempio di risposta**

Se la richiesta va a buon fine, vedrai una risposta JSON come quella riportata di seguito:

```
{
    "data": {
        "description": "Welcome to the Anime Community! Where anime fans gather to share their favorite shows and discuss everything anime-related.",
        "access": "Public",
        "member_count": 40369,
        "created_at": "2024-02-17T06:58:50.000Z",
        "join_policy": "Open",
        "name": "Anime Community",
        "id": "Q29tbXVuaXR5OjE3NTg3NDc4MTc2NDI3MDA5MjI="
    }
}
```
