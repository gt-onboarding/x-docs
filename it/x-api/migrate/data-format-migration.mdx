<div id="updated-json-design">
  #### Progettazione JSON aggiornata
</div>

Oltre alle modifiche al modo in cui si richiedono determinati campi, l'API X v2 introduce anche nuove progettazioni JSON per gli oggetti restituiti dalle API, inclusi gli oggetti [Post](/it/x-api/fundamentals/data-dictionary#tweet) e [user](/it/x-api/fundamentals/data-dictionary#user).

- Al livello radice del JSON, gli endpoint standard restituiscono oggetti Post in un array **statuses**, mentre l'API X v2 restituisce un array **data**.
- Invece di fare riferimento agli "statuses" Retweeted e Quoted, il JSON dell'API X v2 fa riferimento ai Post Retweeted e Quoted. Molti campi legacy e deprecati, come **contributors** e **user.translator\_type**, vengono rimossi.
- Invece di usare sia **favorites** (nell'oggetto Post) sia **favourites** (nell'oggetto user), l'API X v2 usa il termine like.
- X adotta la convenzione secondo cui i valori JSON privi di valore (ad esempio, null) non vengono scritti nel payload. Gli attributi di Post e user sono inclusi solo se hanno valori non nulli.¬†

<div id="migrating-from-native-enriched-data-format-to-v2">
  ### Migrazione dal formato di dati Native Enriched a v2
</div>

Il formato di dati Native Enriched √® utilizzato dai nostri prodotti [enterprise](/it/x-api/enterprise-gnip-2.0/enterprise-gnip).

Il formato di dati Native Enriched √® stato aggiornato per fornire i metadati dei Tweet modificati. Per ulteriori informazioni sui metadati di Modifica Tweet, consulta la pagina [Fondamenti di Modifica Tweet](/it/x-api/enterprise-gnip-2.0/fundamentals/edit-tweets).

Se stai utilizzando gli endpoint standard v1.1, fai riferimento alla [guida dalla standard v1.1 a v2](/it/x-api/migrate/data-format-migration#migrating-from-standard-v1-1s-data-format-to-v2). Se stai utilizzando i prodotti enterprise con Activity Streams, abbiamo anche una guida [da Activity Streams a v2](/it/x-api/migrate/data-format-migration#migrating-from-activity-streams-data-format-to-v2).

API X v2 introduce nuovi design JSON per gli oggetti [Tweet](/it/x-api/fundamentals/data-dictionary#tweet) e [user](/it/x-api/fundamentals/data-dictionary#user).

- A livello root del JSON, il formato Native Enriched restituisce oggetti Tweet in un array results, mentre API X v2 restituisce un array data.
- Invece di utilizzare sia favorites (nell‚Äôoggetto Tweet) sia favourites (nell‚Äôoggetto user), API X v2 utilizza il termine like.
- X adotta la convenzione secondo cui i campi JSON senza valore (ad esempio null) non vengono inclusi nel payload. Gli attributi di Tweet e user sono presenti solo se hanno valori non null.
- Tutti i campi id in v2 saranno in formato stringa

Oltre alle modifiche apportate al nuovo formato JSON, abbiamo introdotto anche un nuovo set di campi nell‚Äôoggetto Tweet, tra cui:

- conversation\_id
- reply\_settings
- alt\_text sui media
- Due nuovi campi di [annotazioni](/it/x-api/fundamentals/post-annotations), context ed entities
- Diversi nuovi campi di [metriche](/it/x-api/fundamentals/metrics)
- Diversi nuovi campi di [polls](/it/x-api/fundamentals/data-dictionary#poll)

Molti campi legacy e deprecati vengono rimossi:

- contributors
- Alcuni campi entities.media e extended\_entities.media
- filter\_level
- timestamp\_ms

<div id="migrating-from-activity-streams-data-format-to-v2">
  ### Migrazione dal formato dati Activity Streams alla v2
</div>

Il formato dati Activity Streams √® disponibile con i nostri prodotti [enterprise](/it/x-api/enterprise-gnip-2.0/enterprise-gnip).

Il formato dati Activity Streams √® stato aggiornato per fornire i metadati dei Tweet modificati. Per saperne di pi√π sui metadati di Modifica dei Tweet, consulta la pagina [Nozioni di base sulla modifica dei Tweet](/it/x-api/enterprise-gnip-2.0/fundamentals/edit-tweets).

Se utilizzi gli endpoint standard v1.1, consulta la [guida alla migrazione da standard v1.1 a v2](/it/x-api/migrate/overview). Se utilizzi gli endpoint premium o il formato Native Enriched per enterprise, consulta la [guida alla migrazione da Native Enriched a v2](/it/x-api/migrate/data-format-migration#migrating-from-native-enriched-data-format-to-v2).

L‚ÄôAPI X v2 introduce nuovi modelli JSON per gli oggetti [Post](/it/x-api/fundamentals/data-dictionary#tweet) e [user](/it/x-api/fundamentals/data-dictionary#user).

- Al livello radice del JSON, il formato Activity Streams restituisce oggetti Tweet in un array results, mentre l‚ÄôAPI X v2 restituisce un array data.
- Invece di fare riferimento alle ‚Äúattivit√†‚Äù Retweeted e Quoted, il JSON dell‚ÄôAPI X v2 fa riferimento ai Tweet retweeted e quoted.
- Invece di usare sia favorites (nell‚Äôoggetto Tweet) sia favourites (nell‚Äôoggetto user), l‚ÄôAPI X v2 usa il termine like.
- Twitter adotta la convenzione secondo cui i valori JSON privi di valore (ad esempio null) non vengono inclusi nel payload. Gli attributi di Tweet e user sono inclusi solo se hanno valori non null.
- Tutti i campi id nella v2 sono in formato stringa.

Oltre alle modifiche apportate al nuovo formato JSON, abbiamo introdotto anche un nuovo set di campi nell‚Äôoggetto Tweet, tra cui:

- conversation\_id
- reply\_settings
- alt\_text sui media
- Due nuovi campi di [annotations](/it/x-api/fundamentals/post-annotations), inclusi context ed entities
- Diversi nuovi campi di [metrics](/it/x-api/fundamentals/metrics)
- Diversi nuovi campi di [polls](/it/x-api/fundamentals/data-dictionary#poll)

Molti campi legacy e deprecati vengono rimossi o sostituiti:

- display\_text\_range
- generator
- gnip
- link
- objectType
- provider
- twitter\_entities.symbols sostituito con data.entities.cashtags
- Alcuni campi twitter\_extended\_entities.media e twitter\_entities.media
- twitter\_filter\_level
- twitterTimeZone
- verb
