<div id="tweet">
  ### Tweet
</div>

I Tweet sono il blocco fondamentale di tutto ciò che riguarda Twitter. L’oggetto Tweet include un lungo elenco di campi a livello “root”, come `id`, `text` e `created_at`. Gli oggetti Tweet sono anche l’oggetto “genitore” di diversi oggetti figli, tra cui `user`, `media`, `poll` e `place`. Usa il parametro di campo `tweet.fields` quando richiedi questi campi a livello root sull’oggetto Tweet.

L’oggetto Tweet può essere trovato ed espanso nella risorsa utente. Ulteriori Tweet correlati al Tweet richiesto possono essere trovati ed espansi anche nella risorsa Tweet. L’oggetto è disponibile per l’espansione con `?expansions=pinned_tweet_id` nella risorsa utente o `?expansions=referenced_tweets.id` nella risorsa Tweet per ottenere l’oggetto con i soli campi predefiniti. Usa l’espansione insieme al parametro di campo `tweet.fields` quando richiedi campi aggiuntivi per completare l’oggetto.

| Valore del campo      | Tipo   | Descrizione                                                                                                                           | Come può essere utilizzato                                           |
|:-----------------------|:--------|:---------------------------------------------------------------------------------------------------------------------------------------|:----------------------------------------------------------------------|
| **id (predefinito)**      | string | L'identificatore univoco del Tweet richiesto.                                                                                        | Usalo per recuperare in modo programmatico uno specifico Tweet.      |
| **text (predefinito)**    | string | Il testo UTF-8 effettivo del Tweet. Vedi [twitter-text](https://github.com/twitter/twitter-text/) per i dettagli sui caratteri validi.   | Estrazione di parole chiave e analisi/classificazione del sentiment. |
| **edit\_history\_tweet\_ids (predefinito)** | object | Identificatori univoci che indicano tutte le versioni di un Tweet. Per i Tweet senza modifiche, ci sarà un solo ID. Per i Tweet con una cronologia di modifiche, ci saranno più ID. | Usa queste informazioni per trovare la cronologia delle modifiche di un Tweet. |
| **attachments**       | object | Specifica il tipo di allegati (se presenti) in questo Tweet.                                                                    | Comprendere gli oggetti restituiti per le espansioni richieste.      |
| **author\_id**         | string | L'identificatore univoco dell'utente che ha pubblicato questo Tweet.                                                                | Ricostruzione dell'oggetto User, condivisione del dataset per peer review. |
| **context\_annotations** | array | Contiene le annotazioni di contesto per il Tweet.                                                                                          | Riconoscimento/estrazione di entità, analisi dei temi.               |
| **conversation\_id**   | string | L'ID del Tweet originale della conversazione (che include risposte dirette e risposte alle risposte).                         | Usalo per ricostruire la conversazione a partire da un Tweet.        |
| **created\_at**        | date (ISO 8601) | Ora di creazione del Tweet.                                                                                                      | Utile per analisi delle serie temporali e per capire quando è stato creato un Tweet. |
| **edit\_controls**     | object | Indica per quanto tempo ancora il Tweet può essere modificato e il numero di modifiche rimanenti.                                                | Usalo per determinare se un Tweet è idoneo alla modifica.            |
| **entities**          | object | Le entità estratte dal testo del Tweet. Vedi le entità in Twitter Objects.                                                       | Fornisce informazioni aggiuntive su hashtag, URL, menzioni, ecc.     |
| **in\_reply\_to\_user\_id** | string | Se il Tweet rappresentato è una risposta, questo campo conterrà l'ID dell'autore del Tweet originale.                                       | Determina se un Tweet era in risposta a un altro Tweet.              |
| **lang**              | string | Lingua del Tweet, se rilevata da Twitter.                                                                                      | Classifica i Tweet per lingua.                                       |
| **non\_public\_metrics** | object | Metriche di engagement non pubbliche per il Tweet al momento della richiesta. Richiede autenticazione con contesto utente.                       | Determina le impression totali generate dal Tweet.                   |
| **organic\_metrics**   | object | Metriche di engagement, tracciate in contesto organico, per il Tweet al momento della richiesta. Richiede autenticazione con contesto utente.  | Misura l'engagement organico del Tweet.                              |
| **possibly\_sensitive** | boolean | Indica se il contenuto potrebbe essere riconosciuto come sensibile.                                                                           | Studia la circolazione di determinati tipi di contenuto.             |
| **promoted\_metrics**  | object | Metriche di engagement, tracciate in contesto sponsorizzato, per il Tweet al momento della richiesta. Richiede autenticazione con contesto utente. | Misura l'engagement del Tweet quando è stato sponsorizzato.          |
| **public\_metrics**    | object | Metriche di engagement pubbliche per il Tweet al momento della richiesta.                                                                 | Misura l'engagement del Tweet.                                       |
| **referenced\_tweets** | array  | Un elenco di Tweet a cui questo Tweet fa riferimento, come Retweet, Tweet citati o risposte.                                                | Comprendere gli aspetti conversazionali dei retweet, ecc.            |
| **reply\_settings**    | string | Indica chi può rispondere a un determinato Tweet. Le opzioni sono "everyone", "mentioned\_users" e "followers".                                  | Determina le impostazioni di risposta alla conversazione per il Tweet.                 |
| **withheld**          | object | Contiene i dettagli relativi ai [contenuti trattenuti](https://help.x.com/en/rules-and-policies/tweet-withheld-by-country).     |                                                                     |
| **media\_metadata**          | array | Contiene i metadati degli allegati multimediali del Tweet.     | Ottieni metadati aggiuntivi come l'`alt_text` dell'allegato multimediale di un Tweet. |

**Recupero di un oggetto Tweet**

**Esempio di richiesta**

Nella seguente richiesta, richiediamo i campi per il Tweet sull'endpoint [Tweets lookup](/it/resources/fundamentals/rate-limits). Assicurati di sostituire `$BEARER_TOKEN` con il tuo [Bearer Token](/it/resources/fundamentals/authentication) generato.

```bash
curl --request GET 'https://api.x.com/2/tweets?ids=1212092628029698048&tweet.fields=attachments,author_id,context_annotations,created_at,entities,geo,id,in_reply_to_user_id,lang,possibly_sensitive,public_metrics,referenced_tweets,text,withheld&expansions=referenced_tweets.id' --header 'Authorization: Bearer $BEARER_TOKEN'
```

<div id="user">
  ### Utente
</div>

L’oggetto utente contiene i metadati dell’account utente di Twitter che descrivono l’utente di riferimento. L’oggetto utente è l’oggetto principale restituito dall’endpoint [users lookup](/it/x-api/users/lookup/introduction). Per richiedere campi utente aggiuntivi su questo endpoint, utilizzare il parametro dei campi `user.fields`.

L’oggetto utente può anche comparire come oggetto figlio ed essere espanso all’interno dell’oggetto Tweet. È possibile ottenerlo in forma condensata, con i soli campi predefiniti, tramite `?expansions=author_id` o `?expansions=in_reply_to_user_id`. Per richiedere campi aggiuntivi e completare l’oggetto, usare l’espansione insieme al parametro dei campi `user.fields`.
 

| Valore del campo | Tipo | Descrizione | Come può essere utilizzato |
| :--- | :--- | :--- | :--- |
| id (predefinito) | stringa | L'identificatore univoco di questo utente.<br /><br />`"id": "2244994945"` | Usalo per recuperare in modo programmatico informazioni su uno specifico utente di Twitter. |
| name (predefinito) | stringa | Il nome dell'utente, così come definito nel suo profilo. Non necessariamente un nome proprio. In genere limitato a 50 caratteri, ma soggetto a modifiche.<br /><br />`"name": "Twitter Dev"` |     |
| username (predefinito) | stringa | Il nome visualizzato, handle o alias con cui questo utente si identifica su Twitter. I nomi utente sono univoci ma possono cambiare. In genere con una lunghezza massima di 15 caratteri, ma potrebbero esistere account storici con nomi più lunghi.<br /><br />`"username": "TwitterDev"` |     |
| confirmed\_email | stringa | L'email confermata dell'utente autenticato. | |
| connection\_status | array | Fornisce un elenco della relazione tra l'utente autenticato e l'utente oggetto della ricerca, ad esempio: following, followed, follow request sent, follow request received, blocking, muting<br /><br />"connection\_status": \[  <br />           "follow\_request\_received",  <br />           "follow\_request\_sent",  <br />           "blocking",  <br />           "followed\_by",  <br />           "following",  <br />           "muting"  <br />] | Può essere usato per determinare lo stato della connessione tra l'utente autenticato e l'utente oggetto della ricerca. |

| verified | boolean | Indica se questo utente è verificato su X.<br /><br />`"verified": true` | Indica se questo utente di X ha un account verificato. Un account verificato segnala che un account di interesse pubblico è autentico. |
| withheld | object | Contiene i dettagli sull’oscuramento per i [contenuti limitati](https://help.x.com/en/rules-and-policies/tweet-withheld-by-country), se applicabile. |     |

```{
  curl --request GET 'https://api.x.com/2/users?
  ids=2244994945&user.fields=created_at,description,entities,id,location,name,pinned_tweet_id,profile_image_url,protected,url,username,verified,withheld&expansions=pinned_tweet_id'
  --header 'Authorization: Bearer $BEARER_TOKEN'
  }
```

Gli Spaces consentono di esprimersi e interagire tramite conversazioni audio in diretta. Il dizionario dei dati di uno Space contiene i metadati rilevanti su quello Space; tutti i dettagli vengono aggiornati in tempo reale.

Gli oggetti utente sono disponibili ed espandibili nella risorsa utente. È possibile espanderli aggiungendo almeno uno tra `host_ids`, `creator_id`, `speaker_ids`, `mentioned_user_ids` al parametro di query `expansions`.

A differenza dei post, gli Space sono temporanei e non sono più disponibili dopo la conclusione o se vengono annullati dal creatore. Quando la tua app gestisce i dati degli Space, sei responsabile di fornire le informazioni più aggiornate e devi rimuovere i dati che non sono più disponibili sulla piattaforma. Gli [endpoint di lookup degli Space](/it/x-api/spaces/lookup/introduction) possono aiutarti a rispettare le aspettative e l’intento degli utenti.

<div id="direct-message-events">
  ### Eventi dei Messaggi Diretti
</div>

Le conversazioni di Messaggi Diretti (DM) sono composte da eventi. L'API X v2 attualmente supporta tre tipi di evento: MessageCreate, ParticipantsJoin e ParticipantsLeave.

Gli oggetti evento DM sono restituiti dagli endpoint di [Direct Message lookup](/it/x-api/direct-messages/lookup/introduction) e viene creato un evento MessageCreate quando i Messaggi Diretti vengono creati correttamente con gli endpoint di [Manage Direct Messages](/it/x-api/direct-messages/lookup/introduction).

Quando si richiedono eventi DM, sono inclusi tre attributi predefiniti dell'oggetto evento, o campi: id, event\_type e text. Per ricevere campi evento aggiuntivi, utilizzare il parametro [fields](/it/x-api/fundamentals/fields) dm\_event.fields per selezionare altri campi. Altri campi evento disponibili includono: dm\_conversation\_id, created\_at, sender\_id, attachments, participant\_ids e referenced\_tweets.

Diversi di questi campi forniscono gli ID di altri oggetti X correlati all'evento di Messaggio Diretto:

- sender\_id - L'ID dell'account che ha inviato il messaggio o che ha invitato un partecipante a una conversazione di gruppo
- participant\_ids - Un array di ID account. Per gli eventi ParticipantsJoin e ParticipantsLeave questo array conterrà un singolo ID dell'account che ha creato l'evento
- attachments - Fornisce i media ID per i contenuti caricati su X dal mittente
- referenced\_tweets - Se nel campo text viene rilevato un URL di Post, l'ID di quel Post è incluso nella risposta

Le [expansions](/it/x-api/fundamentals/expansions) sender\_id, participant\_ids, referenced\_tweets.id e attachments.media\_keys sono disponibili per espandere questi ID di oggetti X.

**Recupero di un oggetto evento di Messaggio Diretto**

**Esempio di richiesta**

In questo esempio, creeremo una richiesta che recupera gli eventi associati a una conversazione uno a uno. La richiesta restituirà i campi fondamentali dell’evento di Messaggio Diretto, insieme a campi aggiuntivi per i Tweet a cui si fa riferimento e i loro autori. Creiamo una query che richieda:

- Attributi fondamentali dell’evento, come quando è stato creato e a quale conversazione appartiene (dm\_conversation).
- L’ID dell’account e la descrizione di chi ha inviato il Messaggio Diretto.
- Il testo di qualsiasi Tweet di riferimento e quando è stato pubblicato.
- L’ID dell’account e la descrizione dell’autore di qualsiasi Tweet di riferimento.

Per restituire tali attributi, la query della richiesta includerà quanto segue:

`?dm_event.fields=id,sender_id,text,created_at,dm_conversation_id&expansions=sender_id,referenced_tweets.id&tweet.fields=created_at,text,author_id&user.fields=description`

```bash
      curl --request GET 'https://api.x.com/2/dm_conversations/with/:participant_id/dm_events?tweet.fields=created_at,text,author_id&user.fields=description&expansions=sender_id,participant_ids,referenced_tweets.id&dm_event.fields=id,sender_id,text,participant_ids,created_at,'
    --header 'Authorization: Bearer $BEARER_TOKEN'
```

Assicurati di sostituire $BEARER\_TOKEN con il tuo [Bearer Token](/it/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token) generato.

<div id="how-to-use-fields-and-expansions">
  ## Come utilizzare fields ed expansions
</div>

Per impostazione predefinita, gli oggetti dati dell’API X v2 includono un numero ridotto di campi quando si invia una richiesta senza utilizzare i parametri [fields](/it/x-api/fundamentals/fields) o [expansions](/it/x-api/fundamentals/expansions). Questa guida illustra come utilizzare i parametri di query `fields` ed `expansions` nella richiesta per ricevere oggetti e campi aggiuntivi nella risposta.

In questa guida, richiederemo diversi campi per il seguente screenshot del Tweet.
 

![Questa immagine mostra uno screenshot di un Tweet pubblicato da @X. Sono visibili il testo del Tweet, il nome utente, la data e l’ora di pubblicazione, la fonte e le metriche pubbliche. Include anche un video. ](https://cdn.cms-twdigitalassets.com/content/dam/developer-twitter/ScreenShotDataDictionaryGuide.png.twimg.1920.png)

Come si può vedere nello screenshot, sono presenti diverse informazioni visibili relative al Tweet, tra cui l’autore del Tweet, le metriche del Tweet, il timestamp di creazione, il video e il conteggio delle visualizzazioni del video. Sono inoltre disponibili diversi dati non visibili nello screenshot, ma comunque richiedibili.

Quando si invia una richiesta all’API, la risposta predefinita è minimale e contiene solo i campi predefiniti del Tweet (id e text). Si riceve inoltre soltanto l’oggetto principale restituito dall’endpoint utilizzato e non gli eventuali oggetti di dati associati che potrebbero essere correlati all’oggetto principale.

Questa semplicità, insieme ai parametri fields ed expansions, consente di richiedere solo i campi necessari, in base al caso d’uso.
