<div id="conversation-threads-using-the-x-api">
  ### Thread di conversazione con l'API X
</div>

Osservando come si evolvono le conversazioni su X, un singolo Post può generare diversi thread di conversazione, ognuno dei quali può aumentare in lunghezza e complessità man mano che più persone intervengono. L’identificazione delle relazioni tra i Post e la comprensione dei thread di conversazione sono funzionalità del payload e delle capacità di ricerca dell'API X v2. Quando i Post vengono pubblicati in risposta a un Post (nota come risposta), o in risposta a una risposta, su ogni risposta è ora presente un `conversation_id` definito, che corrisponde all'ID del Post originale che ha avviato la conversazione.

Le risposte a un determinato Post, così come le risposte a quelle risposte, sono tutte incluse nella conversazione avviata dal singolo Post originale. Indipendentemente dal numero di thread di risposta generati, condivideranno tutti lo stesso `conversation_id` del Post originale che ha avviato la conversazione. Utilizzando l'API X v2, puoi recuperare e ricostruire l’intero thread di conversazione, per comprendere cosa viene detto e come si evolvono conversazioni e idee.

Recuperare `conversation_id` come parametro `tweet.fields`
Per richiedere il `conversation_id` per tutti i Post restituiti da un endpoint v2, aggiungere il campo tweet.fields=conversation\_id ai parametri della richiesta. Il campo conversation\_id corrisponde sempre all’ID del Post originale nel thread di risposte della conversazione. Tutti i Post all’interno dello stesso thread di risposte, inclusi quelli generati da thread precedenti, presenteranno lo stesso conversation\_id.

<div id="request-with-conversation_id-parameter">
  ### Richiesta con il parametro `conversation_id`
</div>

```
curl --request GET \
  --url 'https://api.x.com/2/tweets?ids=1225917697675886593&tweet.fields=author_id,conversation_id,created_at,in_reply_to_user_id,referenced_tweets&expansions=author_id,in_reply_to_user_id,referenced_tweets.id&user.fields=name,username' \
  --header 'Authorization: Bearer $ACCESS_TOKEN'
```

<div id="using-conversation_id-as-a-filter-operator">
  ### Uso di `conversation_id` come operatore di filtro
</div>

Il `conversation_id` può essere usato come parametro di query nelle ricerche recenti oppure come operatore all’interno di una regola per il [flusso filtrato](/it/x-api/posts/filtered-stream/introduction). In questo modo otterrai l’intero thread della conversazione dei Post, in tempo reale tramite flusso filtrato oppure in ordine cronologico inverso tramite [search Tweets](/it/x-api/posts/search/introduction). Puoi anche ottenere il conteggio dei Post in una conversazione usando questo operatore con [Posts counts](/it/x-api/posts/counts/introduction).

<div id="request-to-query-by-conversation_id">
  ## Richiesta per eseguire una query per `conversation_id`
</div>

```
curl --request GET \
  --url 'https://api.x.com/2/tweets/search/recent?query=conversation_id:1279940000004973111&tweet.fields=in_reply_to_user_id,author_id,created_at,conversation_id' \
  --header 'Authorization: Bearer $ACCESS_TOKEN'
```
