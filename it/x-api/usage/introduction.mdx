L'API Usage nell'API X v2 consente agli sviluppatori di recuperare in modo programmatico l'utilizzo del proprio progetto. Con questo endpoint, gli sviluppatori possono tenere traccia e monitorare il numero di post che hanno estratto per un determinato ciclo di fatturazione.

Gli sviluppatori possono utilizzare l'endpoint GET per ottenere l'utilizzo giornaliero del progetto fino agli ultimi 90 giorni. L'utilizzo può anche essere aggregato per singola app client connessa al tuo progetto.

È previsto un rate limit per app di 50 richieste ogni 15 minuti per questo endpoint GET.

<div id="getting-started">
  ### Per iniziare
</div>

Per utilizzare questo endpoint, ti serve un \[bearer token]\(https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) dal [portale per sviluppatori](https://developer.x.com/en/portal/dashboard). Una volta ottenuto il bearer token,  puoi chiamare la Usage API come mostrato di seguito:

```
      curl 'https://api.x.com/2/usage/tweets' --header 'Authorization: Bearer XXXXX'
```

Se la richiesta va a buon fine, vedrai la risposta JSON come mostrata di seguito:

```
{
    "data": {
        "cap_reset_day": 10,
        "project_cap": "5000000000",
        "project_id": "1369785403853424",
        "project_usage": "43435"
    }
}
```

<Note>
  **Configurazione dell'account**

  Per accedere a questi endpoint, ti serviranno:

  - Un [account sviluppatore](https://developer.x.com/en/portal/petition/essential/basic-info) approvato.
  - L’autenticazione utilizzando le chiavi e i token di una [developer App](/it/resources/fundamentals/developer-apps) che si trova all'interno di un [Project](/it/resources/fundamentals/projects). 

  Scopri di più su come ottenere l’accesso agli endpoint dell’API X v2 nella nostra [guida introduttiva](/it/x-api/getting-started/getting-access).
</Note>
