<div id="comparing-x-apis-mutes-endpoints">
  ## Confronto degli endpoint di silenziamento dell’API X
</div>

<div id="mutes-lookup">
  #### Ricerca delle disattivazioni
</div>

L’endpoint v2 per la ricerca delle disattivazioni sostituirà gli endpoint standard [v1.1 GET mutes/users/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/get-mutes-users-ids) e [GET mutes/users/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/get-mutes-users-list).

Le tabelle seguenti confrontano gli endpoint di disattivazione Standard v1.1 e API X v2:
 

| Descrizione | Standard v1.1 | API X v2 |
| :--- | :--- | :--- |
| Metodi HTTP supportati | ******GET****** | ******GET****** |
| Dominio host | ******https://api.x.com****** | ******https://api.x.com****** |
| Percorso dell’endpoint | ******/1.1/mutes/users/ids.json******<br /><br />/1.1/mutes/users/list.json | ******/2/users/:id/muting****** |
| [Autenticazione](/it/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code con PKCE |
| [Limiti di velocità](/it/resources/fundamentals/rate-limits) predefiniti per le richieste | 15 richieste ogni 15 min (per utente) | 15 richieste ogni 15 min (per utente) |
| Formati dei dati | Formato Standard v1.1 | [Formato API X v2](/it/x-api/fundamentals/data-dictionary) (determinato dai parametri di richiesta fields ed expansions; non retrocompatibile con i formati v1.1)<br /><br />Per ulteriori informazioni su come migrare dal formato Standard v1.1 al formato API X v2, consulta la nostra [guida alla migrazione dei formati di dati](/it/x-api/migrate/data-format-migration). |
| Richiede l’uso di credenziali di una [developer App](/it/resources/fundamentals/developer-apps) associata a un [Project](/it/resources/fundamentals/projects) |     | ✔️  |

<div id="manage-mutes">
  #### Gestire i silenzi
</div>

Gli endpoint v2 per la gestione dei silenzi sostituiranno gli endpoint standard v1.1 [POST mutes/users/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-create) e [POST mutes/users/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-destroy).

Le tabelle seguenti confrontano gli endpoint di silenziamento standard v1.1 e dell’API X v2:

**Silenziare un utente**

| Descrizione | Standard v1.1 | API X v2 |
| :--- | :--- | :--- |
| Metodi HTTP supportati | POST | POST |
| Dominio host | https://api.x.com | https://api.x.com |
| Percorso endpoint | /1.1/mutes/users/create.json | /2/users/:id/muting |
| [Autenticazione](/it/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code con PKCE |
| [Limiti di velocità](/it/resources/fundamentals/rate-limits) predefiniti per le richieste | 50 richieste ogni 15 min | 50 richieste ogni 15 min |
| Richiede l’uso di credenziali di una [developer App](/it/resources/fundamentals/developer-apps) associata a un [Project](/it/resources/fundamentals/projects) |     | ✔️  |

**Riattivare un utente**

Le tabelle seguenti confrontano gli endpoint di riattivazione standard v1.1 e dell’API X v2:

| Descrizione | Standard v1.1 | API X v2 |
| :--- | :--- | :--- |
| Metodi HTTP supportati | POST | DELETE |
| Dominio host | https://api.x.com | https://api.x.com |
| Percorso endpoint | /1.1/mutes/users/destroy.json | /2/users/:source\_user\_id/muting/:target\_user\_id |
| [Autenticazione](/it/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code con PKCE |
| [Limiti di velocità](/it/resources/fundamentals/rate-limits) predefiniti per le richieste | 50 richieste ogni 15 min | 50 richieste ogni 15 min |
| Richiede l’uso di credenziali di una [developer App](/it/resources/fundamentals/developer-apps) associata a un [Project](/it/resources/fundamentals/projects) |     | ✔️  |
