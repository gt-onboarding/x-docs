<div id="manage-mutes-standard-v11-compared-to-x-api-v2">
  ### Gestione dei silenzi: Standard v1.1 rispetto a API X v2
</div>

Se hai lavorato con gli endpoint standard v1.1 [POST mutes/users/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-create) e [POST mutes/users/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-destroy), l'obiettivo di questa guida è aiutarti a comprendere somiglianze e differenze tra gli endpoint per la gestione dei silenzi nello standard v1.1 e nell’API X v2.

- **Somiglianze**
  - Contesto utente OAuth 1.0a
- **Differenze**
  - URL degli endpoint
  - Requisiti di App e Progetti
  - Metodi HTTP
  - Parametri della richiesta
     

<div id="similarities">
  #### Somiglianze
</div>

**Metodo di autenticazione OAuth 1.0a User Context**

Entrambe le versioni dell’endpoint supportano [OAuth 1.0a User Context](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Pertanto, se in precedenza utilizzavi uno degli endpoint standard v1.1 per gestire i silenziamenti, puoi continuare a usare lo stesso metodo di autenticazione se esegui la migrazione alla versione dell’API X v2.

<div id="differences">
  #### Differenze
</div>

**URL degli endpoint**

- Endpoint Standard v1.1:
  - POST https://api.x.com/1.1/mutes/users/create.json
    (silenzia un utente)
  - POST https://api.x.com/1.1/mutes/users/destroy.json
    (riattiva un utente)
- Endpoint API X v2:
  - POST https://api.x.com/2/users/:id/muting
    (silenzia un utente)
  - DELETE https://api.x.com/2/users/:source\_user\_id/muting/:target\_user\_id
    (riattiva un utente) 

**Requisiti di App e Progetto**

Gli endpoint dell’API X v2 richiedono l’uso delle credenziali di una [App per sviluppatori](/it/resources/fundamentals/developer-apps) associata a un [Progetto](/it/resources/fundamentals/projects) per autenticare le richieste. Tutti gli endpoint dell’API X v1.1 possono utilizzare credenziali di App autonome o di App associate a un progetto.

**Parametri della richiesta**

I seguenti parametri di richiesta Standard v1.1 hanno equivalenti nell’API X v2:

| Standard v1.1 | API X v2 |
| :--- | :--- |
| user\_id | target\_user\_id |
| screen\_name | Nessun equivalente |

Si noti che i parametri Standard v1.1 vengono passati come query parameter, mentre i parametri dell’API X v2 vengono passati come body parameter (per l’endpoint POST) o path parameter (per l’endpoint DELETE).

Inoltre, l’id dell’utente che mette in silenzio un altro utente non è richiesto quando si utilizzano gli endpoint Standard v1.1, poiché i token di accesso passati con [OAuth 1.0a User Context](/it/resources/fundamentals/authentication) consentivano di dedurre quale utente stesse avviando l’operazione di silenziamento/riattivazione.
