<div id="authentication">
  #### Autenticazione
</div>

Tutti gli endpoint dell’API X v2 richiedono di autenticare le richieste con un set di credenziali, note anche come chiavi e token. È possibile utilizzare OAuth 1.0a User Context oppure OAuth 2.0 Authorization Code con PKCE per autenticare le richieste a questi endpoint.

[OAuth 1.0a User Context](/it/resources/fundamentals/authentication) richiede di utilizzare le API Key, gli Access Token dell’utente e alcuni altri parametri per [creare un’intestazione di autorizzazione](/it/resources/fundamentals/authentication#authorizing-a-request), che andrà poi aggiunta alla richiesta. Gli Access Token devono essere associati all’utente per conto del quale si sta effettuando la richiesta. Se si desidera generare un set di Access Token per un altro utente, quest’ultimo deve autorizzare la tua App utilizzando il [flusso OAuth a 3 attori (3-legged)](https://developer-staging.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens).

Nota: OAuth 1.0a può risultare complesso da utilizzare. Se non si ha familiarità con questo metodo di autenticazione, si consiglia di usare una [libreria](/it/x-api/tools-and-libraries/overview), uno strumento come Postman oppure OAuth 2.0 per autenticare le richieste.

[OAuth 2.0 Authorization Code con PKCE](/it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) consente un controllo più granulare sull’ambito (scope) di un’applicazione e flussi di autorizzazione multi-dispositivo. OAuth 2.0 permette di selezionare scope specifici e di dettaglio che conferiscono permessi precisi per conto di un utente.

Per abilitare OAuth 2.0 nella tua App, attivalo nelle impostazioni di autenticazione della tua App, disponibili nella sezione Impostazioni dell’App del developer portal.

<div id="developer-portal-projects-and-developer-apps">
  #### Portale sviluppatori, Progetti e App per sviluppatori
</div>

Per ottenere un set di credenziali di autenticazione compatibili con gli endpoint dell’API X v2, devi [registrarti per un account sviluppatore](https://developer.x.com/en/portal/petition/essential/basic-info), configurare un [Project](/it/resources/fundamentals/projects) all’interno di tale account e creare un’[App per sviluppatori](/it/resources/fundamentals/developer-apps) all’interno di quel Project. Potrai quindi trovare le tue chiavi e i tuoi token all’interno della tua App per sviluppatori.
 

<div id="rate-limits">
  #### Limiti di frequenza
</div>

Ogni giorno, molte migliaia di sviluppatori inviano richieste all’API X. Per gestire l’enorme volume di queste richieste, su ciascun endpoint vengono applicati [limiti di frequenza](/it/x-api/fundamentals/rate-limits) che determinano quante richieste puoi effettuare per conto della tua app o di un utente autenticato.

Questi endpoint sono soggetti a limiti a livello utente: l’utente autenticato per conto del quale stai effettuando la richiesta può chiamare l’endpoint solo un certo numero di volte, indipendentemente dall’App per sviluppatori utilizzata.

È previsto un limite per utente di 50 richieste ogni 15 minuti per endpoint con i metodi POST e DELETE. Con il metodo GET, invece, il limite è di 15 richieste ogni 15 minuti.
 

<div id="fields-and-expansions">
  #### Campi ed espansioni
</div>

L’endpoint GET dell’API X v2 consente di selezionare con precisione quali dati restituire dall’API utilizzando un insieme di strumenti chiamati campi ed espansioni. Il parametro expansions consente di espandere gli oggetti referenziati nel payload. Ad esempio, questo endpoint consente di recuperare le seguenti [expansions](/it/x-api/fundamentals/expansions):

- pinned\_tweet\_id

Il parametro **fields** consente di selezionare esattamente quali [campi](/it/x-api/fundamentals/fields) all’interno dei diversi oggetti dati si desidera ricevere. Questo endpoint restituisce principalmente oggetti User. Per impostazione predefinita, l’oggetto User restituisce i campi **id**, **name** e **username**. Per ricevere campi aggiuntivi come **user.created\_at** o **user.entities**, è necessario richiederli esplicitamente utilizzando il parametro **fields**.

Abbiamo aggiunto alla nostra [data dictionary dell’API X v2](/it/x-api/fundamentals/data-dictionary) una guida sull’uso congiunto di [fields ed expansions](/it/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions).

<div id="pagination">
  #### Paginazione
</div>

L'endpoint dei silenziamenti può restituire una grande quantità di dati. Per garantire risultati coerenti e performanti in ogni momento, utilizziamo la paginazione. La paginazione è una funzionalità degli endpoint dell'API X v2 che restituiscono più risultati di quanti se ne possano includere in una singola risposta. In tal caso, i dati vengono restituiti in una serie di “pagine”. Scopri di più su come [navigare tra i risultati.](/it/x-api/fundamentals/pagination)

<Note>
  **Nota:** Se un utente applica il silenziamento da [X](https://x.com/), esiste un limite di 200 richieste ogni 15 minuti.
</Note>
