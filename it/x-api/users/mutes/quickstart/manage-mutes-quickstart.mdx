<div id="getting-started-with-the-manage-mutes-endpoints">
  ## Guida introduttiva agli endpoint per la gestione dei mute
</div>

Questa guida rapida ti aiuterà a inviare la prima richiesta agli endpoint per la gestione dei mute utilizzando [Postman](/it/tutorials/postman-getting-started).

Se desideri consultare codice di esempio in diverse lingue, visita il nostro repository GitHub [API X v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code).

<Note>
  **Prerequisiti**

  Per completare questa guida, avrai bisogno di un set di [chiavi e token](/it/resources/fundamentals/authentication) per autenticare la tua richiesta. Puoi generare queste chiavi e questi token seguendo questi passaggi:

  - [Richiedi un account sviluppatore](https://developer.x.com/en/apply-for-access) e attendi l’approvazione.
  - Crea un [Progetto](/it/resources/fundamentals/projects) e una [App per sviluppatori](/it/resources/fundamentals/developer-apps) associata nel portale sviluppatori.
  - Vai alla pagina “Keys and tokens” della tua App per generare le credenziali necessarie. Assicurati di salvare tutte le credenziali in un luogo sicuro.
</Note>

<div id="steps-to-build-a-manage-mutes-request">
  ### Passaggi per creare una richiesta per gestire i silenziamenti
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### Passaggio uno: inizia con uno strumento o una libreria
</div>

Esistono diversi strumenti, esempi di codice e librerie che puoi utilizzare per inviare una richiesta a questo endpoint, ma qui useremo Postman per semplificare il processo.

Per caricare la raccolta Postman dell’API X v2 nel tuo ambiente, fai clic sul pulsante seguente:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Aggiungi API X v2 a Postman
</Button>

Una volta caricata la raccolta dell’API X v2 in Postman, vai alla cartella “Mutes” e seleziona “Mute a user’s ID”.
 

<div id="step-two-authenticate-your-request">
  #### Passaggio due: autentica la tua richiesta
</div>

Per effettuare correttamente una richiesta all’API X, devi verificare di avere l’autorizzazione. Per farlo con questo endpoint, devi autenticare la richiesta utilizzando [OAuth 1.0a User Context](/it/resources/fundamentals/authentication) oppure [OAuth 2.0 Authorization Code with PKCE](it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

In questo esempio useremo OAuth 1.0a User Context.

Devi aggiungere le tue chiavi e i tuoi token — in particolare la tua API Key, API Secret Key, OAuth 1.0a user Access Token e OAuth 1.0a user Access Token Secret — in Postman. Puoi farlo selezionando l’ambiente denominato “API X v2” nell’angolo in alto a destra di Postman e aggiungendo chiavi e token nei campi “initial value” e “current value” (cliccando sull’icona a forma di occhio accanto al menu a discesa dell’ambiente).

Se hai eseguito correttamente questi passaggi, queste variabili verranno richiamate automaticamente nella scheda di autorizzazione della richiesta.
 

<div id="step-three-specify-who-is-going-to-mute-whom">
  #### Passaggio tre: specificare chi silenzia chi
</div>

Gli endpoint per la gestione dei silenziamenti richiedono due ID: uno per l’utente (l’utente che desidera silenziare o riattivare l’audio di un altro utente) e uno per l’utente di destinazione (l’utente che verrà silenziato o riattivato). L’ID dell’utente deve corrispondere all’ID dell’utente autenticato, il che significa che devi inviare i token di accesso associati a tale ID utente quando autentichi la richiesta. In questo caso, puoi specificare l’ID del tuo account. Puoi trovare il tuo ID in due modi:

1. Usando l’endpoint [user lookup](/it/x-api/users/lookup/introduction) per nome utente, puoi passare un nome utente e ottenere il campo id.
2. Controllando il tuo token di accesso, noterai che la parte numerica corrisponde al tuo ID utente.

L’ID di destinazione può essere qualsiasi ID utente valido. In Postman, vai alla scheda "Params" e inserisci il tuo ID nella colonna "Value" della variabile di percorso `id`. Vai alla scheda “Body” e inserisci l’ID dell’utente che desideri silenziare come valore per il parametro target\_user\_id. Assicurati di non includere spazi prima o dopo gli ID.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | ID utente autenticato |
| target\_user\_id | l’ID dell’utente che desideri silenziare |

<div id="step-four-make-your-request-and-review-your-response">
  #### Passaggio quattro: invia la richiesta e verifica la risposta
</div>

Una volta configurato tutto, fai clic sul pulsante "Send" e riceverai una risposta simile al seguente esempio:

```
      { "data": { "muting": true } }
```

Se ricevi "muting": true, significa che l'id sta silenziando correttamente il target\_user\_id.

Per riattivare lo stesso utente, puoi utilizzare la richiesta denominata "Unmute a user ID", che si trova anche nella cartella "Mutes" della raccolta API X v2 caricata in Postman. Il source\_user\_id deve essere il tuo ID utente e il target\_user\_id deve essere l'ID utente da riattivare. Non dovrai aggiungerlo come corpo JSON, quindi assicurati di includere i parametri di query richiesti per source\_user\_id e target\_user\_id.

In caso di riattivazione riuscita, riceverai una risposta simile al seguente esempio:

```
      { "data": { "muting": false } }
```
