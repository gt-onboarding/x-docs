<div id="authentication">
  #### Autenticazione
</div>

Tutti gli endpoint dell’API X v2 richiedono che le richieste siano [autenticate](/it/resources/fundamentals/authentication) con un set di credenziali, noto anche come chiavi e token. Puoi utilizzare OAuth 1.0a User Context, App only oppure OAuth 2.0 Authorization Code con PKCE per autenticare le richieste a questi endpoint.

[OAuth 1.0a User Context](/it/resources/fundamentals/authentication#oauth-1-0a-2) richiede di utilizzare le tue API keys, i token di accesso utente e alcuni altri parametri per [creare un’intestazione di autorizzazione](/it/resources/fundamentals/authentication#authorizing-a-request), che poi allegherai alla tua richiesta. I token di accesso devono essere associati all’utente per conto del quale stai effettuando la richiesta. Se desideri generare un set di token di accesso per un altro utente, quest’ultimo deve autorizzare la tua App utilizzando il [flusso OAuth a 3 passaggi](https://developer-staging.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens).

Tieni presente che OAuth 1.0a può essere complesso da utilizzare. Se non conosci questo metodo di autenticazione, ti consigliamo di usare una [libreria](/it/x-api/tools-and-libraries/overview), uno strumento come Postman oppure OAuth 2.0 per autenticare le tue richieste. Se desideri recuperare un Post o metriche private da questi endpoint, dovrai utilizzare OAuth 1.0a User Context oppure OAuth 2.0 Authorization Code con PKCE, che garantiranno di disporre delle autorizzazioni appropriate dall’utente proprietario di quel contenuto.

[App only](/it/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) richiede semplicemente di inviare un [token di accesso App only](/it/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) insieme alla tua richiesta. Puoi generare un token di accesso App only direttamente all’interno di un’App per sviluppatori oppure generarlo utilizzando l’endpoint [POST oauth2/token](/it/resources/fundamentals/authentication#post-oauth2-token).

[OAuth 2.0 Authorization Code con PKCE](/it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) consente un controllo più granulare sull’ambito di un’applicazione e flussi di autorizzazione su più dispositivi. OAuth 2.0 consente di scegliere scope specifici e dettagliati che ti conferiscono autorizzazioni puntuali per conto di un utente.

Per abilitare OAuth 2.0 nella tua App, devi abilitarlo nelle impostazioni di autenticazione della tua App, disponibili nella sezione delle impostazioni dell’App nel portale per sviluppatori.

**Nota bene**

Se stai richiedendo i seguenti campi, è richiesto OAuth 1.0a User Context oppure OAuth 2.0 Authorization Code:

- tweet.fields.non\_public\_metrics
- tweet.fields.promoted\_metrics
- tweet.fields.organic\_metrics

<div id="developer-portal-projects-and-developer-apps">
  #### Portale sviluppatori, Progetti e App per sviluppatori
</div>

Per ottenere un set di credenziali di autenticazione compatibili con gli endpoint dell’API X v2, devi [registrarti per un account sviluppatore](https://developer.x.com/en/portal/petition/essential/basic-info), configurare un [Progetto](/it/resources/fundamentals/projects) all’interno di tale account e creare un’[App per sviluppatori](/it/resources/fundamentals/developer-apps) all’interno di quel Progetto. Potrai quindi trovare le tue chiavi e i tuoi token all’interno della tua App per sviluppatori. 
 

<div id="rate-limits">
  #### Limiti di velocità
</div>

Ogni giorno, molte migliaia di sviluppatori inviano richieste all’API X. Per gestire l’elevato volume di queste richieste, su ciascun endpoint vengono applicati [limiti di velocità](/it/x-api/fundamentals/rate-limits) che restringono il numero di richieste eseguibili per conto della tua App o di un utente autenticato.

Gli endpoint di ricerca utente sono soggetti a limiti di velocità sia a livello di App sia a livello di Utente. Tuttavia, l’endpoint di ricerca dell’utente autenticato è soggetto a limiti di velocità solo a livello di Utente.

Il limite a livello di App significa che tu, sviluppatore, puoi effettuare solo un certo numero di richieste a questo endpoint in un dato intervallo di tempo da qualsiasi App specifica (come determinato dalle chiavi e dai token che stai utilizzando). Il limite a livello di Utente significa che l’utente autenticato per conto del quale stai effettuando la richiesta può eseguire l’operazione solo un numero limitato di volte su qualsiasi App di uno sviluppatore.

La tabella seguente mostra i limiti di velocità per ciascun endpoint.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Metodo HTTP** | **Limite di velocità / Livello** |
| /2/users | GET | 900 richieste ogni 15 minuti / App e Utente |
| /2/users/:id | GET | 900 richieste ogni 15 minuti / App e Utente |
| /2/users/by | GET | 900 richieste ogni 15 minuti / App e Utente |
| /2/users/by/username/:username | GET | 900 richieste ogni 15 minuti / App e Utente |
| /2/users/me | GET | 75 richieste ogni 15 minuti / Utente |

<div id="fields-and-expansions">
  #### Campi ed espansioni
</div>

L’API X v2 consente di selezionare con precisione quali dati ottenere dall’API utilizzando un insieme di strumenti chiamati campi ed espansioni. Il parametro expansions consente di includere gli oggetti referenziati nel payload. Ad esempio, questo endpoint consente di utilizzare l’espansione pinned\_tweet\_id.

Il parametro fields consente di scegliere esattamente quali [campi](/it/x-api/fundamentals/fields) dei diversi oggetti dati si desidera ricevere. Questi endpoint restituiscono principalmente oggetti utente. Per impostazione predefinita, l’oggetto utente restituisce i campi id, name e username. Per ricevere campi aggiuntivi come user.created\_at o user.location, è necessario richiederli esplicitamente tramite il parametro fields. Alcuni campi importanti da considerare per la tua integrazione includono i dati dei sondaggi nei Post, le metriche, le annotazioni e i campi conversation\_id.

Abbiamo aggiunto una guida su come [utilizzare campi ed espansioni](/it/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) nel nostro [dizionario dei dati dell’API X v2](/it/x-api/fundamentals/data-dictionary).
