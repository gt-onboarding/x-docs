  ## Confronto degli endpoint di ricerca utenti dell’API X
</div>

Gli endpoint di ricerca utenti v2 sostituiranno gli endpoint standard v1.1 [GET users/lookup](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-users-lookup.html) e [GET users/show](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-users-show.html). Se hai codice, app o strumenti che utilizzano una di queste versioni degli endpoint di ricerca utenti e stai valutando la migrazione ai più recenti endpoint dell’API X v2, questa serie di guide fa per te. 
 

Le tabelle seguenti confrontano i vari tipi di endpoint di ricerca utenti:
 

|     |     |     |
| :--- | :--- | :--- |
| **Descrizione** | **Standard v1.1** | **API X v2** |
| Metodi HTTP supportati | `GET` | `GET` |
| Dominio host | `https://api.x.com` | `https://api.x.com` |
| Percorso endpoint | `/1.1/users/show.json` `/1.1/users/lookup.json` | `/2/users`<br /><br />`/2/users/:id`<br /><br />`/2/users/by`<br /><br />`/2/users/by/:username` |
| [Autenticazione](/it/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />Solo app<br /><br />OAuth 2.0 con PKCE |
| [Limiti di velocità](/it/resources/fundamentals/rate-limits) predefiniti per le richieste | 900 richieste per 15 min (per utente)<br /><br />/show - 900 richieste per 15 min (per app)  <br />/lookup - 300 richieste per 15 min (per app) | 900 richieste per 15 min (per utente)<br /><br />300 richieste per 15 min (per app) |
| Utenti massimi per risposta | /show -  1<br /><br />/lookup - 100 | 100 |
| Formato dell’oggetto di risposta JSON | Formato Standard v1.1 | [Formato API X v2](/it/x-api/fundamentals/data-dictionary) (determinato dai parametri di richiesta fields ed expansions, non retrocompatibile con i formati v1.1)<br /><br />Per saperne di più su come eseguire la migrazione dal formato Standard v1.1 al formato API X v2, visita la nostra [guida alla migrazione dei formati dei dati](/it/x-api/migrate/data-format-migration). |
| Supporta la selezione di quali [campi](/it/x-api/fundamentals/data-dictionary) restituire nel payload |     | ✔   |
| Supporta i campi [annotations](/it/x-api/fundamentals/post-annotations) (sul Post fissato) |     | ✔   |
| Supporta la richiesta di nuovi campi [metrics](/it/x-api/fundamentals/metrics) (sul Post fissato) |     | ✔   |
| Supporta il campo [conversation\_id](/it/x-api/fundamentals/conversation-id) (sul Post fissato) |     | ✔   |
| Richiede l’uso di credenziali di una [App sviluppatore](/it/resources/fundamentals/developer-apps) associata a un [progetto](/it/resources/fundamentals/projects) |     | ✔   |
