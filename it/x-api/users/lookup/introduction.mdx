Puoi autenticare la richiesta a tutti gli endpoint di ricerca utenti, ad eccezione di quello per l’utente autenticato, con [OAuth 1.0a User Context](/it/resources/fundamentals/authentication), [App only](/it/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), o [OAuth 2.0 Authorization code with PKCE](it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). Tuttavia, se desideri accedere a metriche private o a dati di utenti con profilo privato, dovrai utilizzare OAuth 1.0a User Context o OAuth 2.0 Authorization Code con PKCE e includere nella richiesta l’access token dell’utente autenticato.    

Questo endpoint è comunemente utilizzato per ottenere dettagli aggiornati su un utente, verificare l’esistenza di un utente o aggiornare i dettagli archiviati in seguito a un evento di conformità.

<div id="authenticated-user-lookup">
  ### Ricerca dell’utente autenticato
</div>

Se effettui l’autenticazione per conto di altri utenti, è fondamentale poter vedere per chi puoi inviare una richiesta.

Questo endpoint richiede l’uso di [OAuth 1.0a User Context](/it/resources/fundamentals/authentication) o di [OAuth 2.0 Authorization Code con PKCE](it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). Restituisce informazioni sull’utente autorizzato, ovvero l’utente associato ai token di accesso utente che invii con la richiesta.

Puoi accedere a questo endpoint con il metodo GET. Per questo endpoint è previsto un limite di 75 richieste per utente ogni 15 minuti.

<Note>
  **Configurazione dell’account**

  Per accedere a questi endpoint, ti serviranno:

  - Un [account sviluppatore](https://developer.x.com/en/portal/petition/essential/basic-info) approvato.
  - Autenticarti utilizzando le chiavi e i token di una [App per sviluppatori](/it/resources/fundamentals/developer-apps) che si trova all’interno di un [Project](/it/resources/fundamentals/projects).

  Scopri di più su come ottenere l’accesso agli endpoint dell’API X v2 nella nostra [guida introduttiva](/it/x-api/getting-started/getting-access).
</Note>
