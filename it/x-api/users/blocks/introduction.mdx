<div id="blocks-lookup">
  ### Ricerca dei blocchi
</div>

L’endpoint GET di ricerca dei blocchi consente di vedere quali account hai bloccato per conto di un utente autorizzato. Questo endpoint ha un limite di 15 richieste ogni 15 minuti per utente.

Poiché con la ricerca dei blocchi stai richiedendo informazioni private e operi per conto di un utente con permessi di gestione dei blocchi, devi autenticare questi endpoint con [OAuth 1.0a User Context](/it/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code con PKCE](it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) e utilizzare i token di accesso dell’utente associati a un utente che ha autorizzato la tua App, generabili tramite il [flusso OAuth a 3 passaggi](/it/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) o il [flusso di concessione Authorization Code con PKCE](/it/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0).

<Note>
  **Configurazione dell’account**

  Per accedere a questi endpoint, ti serviranno:

  - Un [account sviluppatore](https://developer.x.com/en/portal/petition/essential/basic-info) approvato.
  - L’autenticazione utilizzando le chiavi e i token di una [App sviluppatore](/it/resources/fundamentals/developer-apps) che si trova all’interno di un [Project](/it/resources/fundamentals/projects).

  Scopri di più su come ottenere l’accesso agli endpoint dell’API X v2 nella nostra [guida introduttiva](/it/x-api/getting-started/getting-access).
</Note>
