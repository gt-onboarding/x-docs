<div id="authentication">
  #### Autenticazione
</div>

Tutti gli endpoint dell’API X v2 richiedono di autenticare le richieste con un set di credenziali, anche note come chiavi e token. Puoi utilizzare OAuth 1.0a User Context oppure OAuth 2.0 Authorization Code con PKCE per autenticare le richieste a questi endpoint.

[OAuth 1.0a User Context](/it/resources/fundamentals/authentication) richiede l’uso delle tue API Keys, dei token di accesso utente (Access Tokens) e di alcuni altri parametri per [creare un’intestazione di autorizzazione](/it/resources/fundamentals/authentication#authorizing-a-request), che poi includerai nella richiesta. Gli Access Tokens devono essere associati all’utente per conto del quale stai effettuando la richiesta. Se desideri generare un set di Access Tokens per un altro utente, quest’ultimo deve autorizzare la tua App utilizzando il [flusso OAuth a 3 attori (3-legged)](https://developer-staging.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens).

Tieni presente che OAuth 1.0a può essere complesso da usare. Se non conosci questo metodo di autenticazione, consigliamo di utilizzare una [libreria](/it/x-api/tools-and-libraries/overview), uno strumento come Postman oppure OAuth 2.0 per autenticare le richieste.

[OAuth 2.0 Authorization Code con PKCE](/it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) consente un controllo più preciso sull’ambito di un’applicazione e flussi di autorizzazione su più dispositivi. OAuth 2.0 consente di scegliere ambiti specifici e granulari che ti concedono permessi specifici per conto di un utente.

Per abilitare OAuth 2.0 nella tua App, devi attivarlo nelle impostazioni di autenticazione della tua App, disponibili nella sezione delle impostazioni dell’App del portale per sviluppatori.

<div id="developer-portal-projects-and-developer-apps">
  #### Portale per sviluppatori, Progetti e App per sviluppatori
</div>

Per ottenere un set di credenziali di autenticazione compatibili con gli endpoint dell’API X v2, devi [registrarti per un account sviluppatore](https://developer.x.com/en/portal/petition/essential/basic-info), creare un [Progetto](/it/resources/fundamentals/projects) all’interno di quell’account e creare un’[App per sviluppatori](/it/resources/fundamentals/developer-apps) all’interno di quel Progetto. Potrai quindi trovare chiavi e token all’interno della tua App per sviluppatori.
 

<div id="rate-limits">
  #### Limiti di frequenza
</div>

Ogni giorno, molte migliaia di sviluppatori inviano richieste all’API X. Per gestire l’elevato volume di traffico, [i limiti di frequenza](/it/x-api/fundamentals/rate-limits) vengono applicati a ciascun endpoint e limitano il numero di richieste che puoi effettuare per conto della tua app o di un utente autenticato.

Questi endpoint sono soggetti a limiti di frequenza a livello utente, il che significa che l’utente autenticato per conto del quale stai effettuando la richiesta può chiamare l’endpoint solo un numero definito di volte, indipendentemente dall’App per sviluppatori utilizzata. Il limite per i metodi POST e DELETE è di 50 richieste per endpoint ogni 15 minuti. Con il metodo GET, invece, il limite è di 15 richieste ogni 15 minuti.
 

<div id="fields-and-expansions">
  #### Campi ed espansioni
</div>

Gli endpoint GET dell’API X v2 consentono di selezionare con precisione quali dati recuperare dall’API utilizzando strumenti chiamati campi ed espansioni. Il parametro expansions consente di espandere gli oggetti referenziati nel payload. Ad esempio, questo endpoint consente di ottenere le seguenti [espansioni](/it/x-api/fundamentals/expansions):

- pinned\_tweet\_id

Il parametro **fields** consente di selezionare esattamente quali [campi](/it/x-api/fundamentals/fields) all’interno dei diversi oggetti dati si desidera ricevere. Questi endpoint restituiscono principalmente oggetti Post. Per impostazione predefinita, l’oggetto Post restituisce i campi **id** e **text**. Per ricevere campi aggiuntivi come **tweet.created\_at** o **tweet.entities**, è necessario richiederli esplicitamente utilizzando il parametro **fields**. Alcuni campi importanti da considerare per l’integrazione sono i dati dei sondaggi, le metriche, le annotazioni dei Post e i campi conversation\_id.

Abbiamo aggiunto al nostro [dizionario dati dell’API X v2](/it/x-api/fundamentals/data-dictionary) una guida su come [usare insieme campi ed espansioni](/it/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions).
