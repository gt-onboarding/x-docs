<div id="getting-started-with-the-manage-follows-endpoints">
  ## Guida introduttiva agli endpoint per la gestione dei follow
</div>

Questa guida rapida ti aiuterà a inviare la tua prima richiesta agli endpoint per la gestione dei follow usando [Postman](/it/tutorials/postman-getting-started).

Se vuoi consultare esempi di codice in diverse lingue, visita il nostro repository GitHub [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code).

<Note>
  ### Prerequisiti

  Per completare questa guida, ti servirà un set di [keys and tokens](/it/resources/fundamentals/authentication) per autenticare la tua richiesta. Puoi generare queste chiavi e questi token seguendo questi passaggi:

  - [Richiedi un account sviluppatore](https://developer.x.com/en/apply-for-access) e attendi l’approvazione.
  - Crea un [Project](/it/resources/fundamentals/projects) e una [developer App](/it/resources/fundamentals/developer-apps) associata nel portale per sviluppatori.
  - Vai alla pagina “Keys and tokens” della tua App per generare le credenziali necessarie. Assicurati di salvare tutte le credenziali in un luogo sicuro.
</Note>

<div id="steps-to-build-a-manage-follows-request">
  ### Passaggi per creare una richiesta per gestire i follow
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### Primo passaggio: iniziare con uno strumento o una libreria
</div>

Esistono diversi strumenti, esempi di codice e librerie che puoi utilizzare per inviare una richiesta a questo endpoint, ma qui useremo Postman per semplificare il processo.

Per caricare la raccolta Postman dell’API X v2 nel tuo ambiente, fai clic sul seguente pulsante:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Aggiungi API X v2 a Postman
</Button>

Una volta caricata la raccolta dell’API X v2 in Postman, vai alla cartella “Follows” e seleziona “Follow a user ID”.
 

<div id="step-two-authenticate-your-request">
  #### Passaggio due: autentica la tua richiesta
</div>

Per inviare correttamente una richiesta all’API X, devi verificare di avere le autorizzazioni necessarie. Per farlo con questo endpoint, devi autenticare la richiesta usando [OAuth 1.0a User Context](/it/resources/fundamentals/authentication) oppure [OAuth 2.0 Authorization Code with PKCE](it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

In questo esempio useremo OAuth 1.0a User Context.

Devi aggiungere le tue chiavi e i tuoi token – in particolare API Key, API Secret Key, OAuth 1.0a user Access Token e OAuth 1.0a user Access Token Secret – in Postman. Puoi farlo selezionando l’ambiente denominato “API X v2” nell’angolo in alto a destra di Postman e aggiungendo chiavi e token nei campi “initial value” e “current value” (facendo clic sull’icona a forma di occhio accanto al menu a discesa dell’ambiente).

Queste variabili verranno richiamate automaticamente nella scheda Authorization della richiesta se hai eseguito correttamente questa operazione.
 

<div id="step-three-specify-who-is-going-to-follow-whom">
  #### Terzo passaggio: specificare chi segue chi
</div>

Gli endpoint per la gestione dei follow accettano due ID: uno per l’utente di origine (l’utente che desidera seguire o smettere di seguire un altro utente) e uno per l’utente di destinazione (l’utente che sarà seguito o non più seguito). L’ID dell’utente di origine deve corrispondere all’ID dell’utente autenticato. In questo caso, puoi indicare l’ID del tuo account. Puoi trovare il tuo ID in due modi:

1. Utilizzando l’endpoint [user lookup by username](/it/x-api/users/user-lookup-by-id), puoi passare un nome utente e ottenere il campo id.
2. Consultando il tuo Access Token, noterai che la parte numerica corrisponde al tuo ID utente.

L’ID di destinazione può essere qualsiasi ID utente valido. Ad esempio, l’ID utente di @XDevelopers è 2244994945.

In Postman, vai alla scheda "Params" e inserisci il tuo ID nella colonna "Value" della variabile di percorso id. Vai alla scheda "Body" e inserisci 2244994945 (l’ID utente di @XDevelopers) come valore del parametro target\_user\_id. Assicurati di non includere spazi prima o dopo alcun ID.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | (il tuo ID utente) |
| target\_user\_id | 2244994945 |

Facendo clic sul pulsante "Send", riceverai un oggetto di risposta contenente lo stato della relazione:

- Se ricevi "following": true, allora l’ID sta seguendo correttamente il target\_user\_id.
- Se ricevi "pending": true, allora il target\_user\_id è protetto e deve accettare la tua richiesta di follow.

####

Quarto passaggio: invia la richiesta e verifica la risposta

Una volta configurato tutto, fai clic sul pulsante "Send" e riceverai la seguente risposta:

```{
    "data": {
        "following": true,
        "pending_follow": false
    }
}
```

Allo stesso modo, se volessi smettere di seguire un utente, utilizzeresti la richiesta "Unfollow a user ID" all'interno della stessa raccolta Postman. Tuttavia, i parametri source\_user\_id e target\_user\_id devono essere passati come variabili di percorso utilizzando l'endpoint di unfollow.
