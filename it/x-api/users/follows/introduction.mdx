Offriamo due gruppi di endpoint per aiutarti a cercare, creare ed eliminare le relazioni di follow: ricerca follow e gestione follow.

<div id="follows-lookup">
  ### Ricerca dei follow
</div>

Disponibile solo per chi ha l’[accesso Enterprise](/it/x-api/getting-started/about-x-api)

Gli endpoint di lookup dei follow consentono di esplorare e analizzare le relazioni tra gli utenti, talvolta chiamata analisi di rete. In particolare, sono disponibili due endpoint REST che restituiscono [oggetti utente](/it/x-api/fundamentals/data-dictionary#user) che rappresentano chi un utente specificato segue o chi segue l’utente specificato.

Puoi autenticare questo endpoint con [OAuth 1.0a User Context](/it/resources/fundamentals/authentication), \[App only]\(https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), oppure [OAuth 2.0 Authorization Code with PKCE](it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). Puoi richiedere fino a 1.000 utenti per richiesta e verranno forniti token di paginazione per scorrere set di risultati di grandi dimensioni.
 

<div id="manage-follows">
  ### Gestire i follow
</div>

Gli endpoint per la gestione dei follow consentono di seguire o smettere di seguire utenti.

Poiché stai effettuando richieste per conto di un utente, devi autenticare questi endpoint con [OAuth 1.0a User Context](/it/resources/fundamentals/authentication) oppure [OAuth 2.0 Authorization Code with PKCE](it/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) e utilizzare i token di accesso dell’utente associati all’utente per conto del quale stai effettuando la richiesta. Puoi generare questo token di accesso utente utilizzando il [flusso OAuth a 3 passaggi](/it/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) oppure il [flusso Authorization Code with PKCE grant](/it/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0).

Puoi effettuare fino a 400 azioni di follow al giorno per ciascun utente autenticato e fino a 1.000 azioni al giorno per App complessive su tutti i tuoi utenti autenticati. Ad esempio, se hai cinque utenti autenticati, puoi seguire 400 utenti al giorno (limite per utente) con due di questi utenti per un totale di 800 azioni e dovrai suddividere le restanti 200 azioni (limite per app) tra i tre utenti rimanenti. Questo limite non si applica all’endpoint di unfollow, che ha un limite separato di 500 azioni al giorno (per app).

<Note>
  **Configurazione dell’account**

  Per accedere a questi endpoint, ti serviranno:

  - Un [account sviluppatore](https://developer.x.com/en/portal/petition/essential/basic-info) approvato.
  - L’autenticazione utilizzando le chiavi e i token di una [developer App](/it/resources/fundamentals/developer-apps) che si trova all’interno di un [Project](/it/resources/fundamentals/projects).
</Note>

Scopri di più su come ottenere l’accesso agli endpoint dell’API X v2 nella nostra [guida introduttiva](/it/x-api/getting-started/getting-access).
