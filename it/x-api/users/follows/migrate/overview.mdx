  ## Confronto degli endpoint dei follower dell’API X
</div>

<div id="follows-lookup">
  ### Ricerca dei follow
</div>

Gli endpoint di ricerca dei follow della v2 sostituiranno gli endpoint standard v1.1 [followers/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-followers-ids), v1.1 [followers/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-followers-list), v1.1 [friends/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-friends-ids) e v1.1 [friends/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-friends-list).

Le seguenti tabelle confrontano i vari tipi di endpoint di ricerca dei follow:

|     |     |     |
| :--- | :--- | :--- |
| **Descrizione** | **Standard v1.1** | **API X v2** |
| Metodi HTTP supportati | `GET` | `GET` |
| Dominio host | `https://api.x.com` | `https://api.x.com` |
| Percorso dell’endpoint | /1.1/friends/ids.json<br /><br />/1.1/friends/list.json<br /><br />/1.1/followers/ids.json<br /><br />/1.1/followers/list.json | /2/users/:id/following<br /><br />/2/users/:id/followers |
| [Autenticazione](/it/resources/fundamentals/authentication) | OAuth 1.0a User Context<br /><br />App only | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code con PKCE<br /><br />App only |
| [Limiti di frequenza](/it/resources/fundamentals/rate-limits) predefiniti per le richieste | 15 richieste ogni 15 min (per utente)<br /><br />15 richieste ogni 15 min (per app) | 15 richieste ogni 15 min (per utente)<br /><br />15 richieste ogni 15 min (per app) |
| Numero massimo di utenti per risposta | GET friends/ids e GET followers/ids restituiscono un massimo di 5000 ID utente per pagina.<br /><br />  <br /><br />GET friends/list e GET followers/list restituiscono un massimo di 200 oggetti utente per pagina. | 1000 oggetti utente per pagina |
| Paginazione | Il token è restituito in un campo next\_cursor, che può poi essere passato come valore al parametro cursor per ottenere la pagina successiva dei risultati. | Il token è restituito in un campo next\_token, che può poi essere passato come valore al parametro token per ottenere la pagina successiva dei risultati.<br /><br />Il payload v2 fornisce anche un campo previous\_token, che può essere passato con il parametro pagination\_token per ottenere la pagina precedente dei risultati. |
| Formato JSON | Formato Standard v1.1 | [Formato API X v2](/it/x-api/fundamentals/data-dictionary) (determinato dai parametri di richiesta fields ed expansions, non retrocompatibile con i formati v1.1)<br /><br />Per ulteriori informazioni sulla migrazione dal formato Standard v1.1 al formato API X v2, consulta la nostra [guida alla migrazione dei formati dati](/it/x-api/migrate/data-format-migration). |
| Supporta la selezione di quali [fields](/it/x-api/fundamentals/data-dictionary) restituire nel payload |     | ✔   |
| Supporta i campi delle [annotazioni dei post](/it/x-api/fundamentals/post-annotations) |     | ✔   |
| Supporta la richiesta di nuovi campi delle [metriche](/it/x-api/fundamentals/metrics) |     | ✔   |
| Supporta il campo [conversation\_id](/it/x-api/fundamentals/conversation-id) |     | ✔   |
| Richiede l’uso di credenziali di una [developer App](/it/resources/fundamentals/developer-apps) associata a un [project](/it/resources/fundamentals/projects) |     | ✔   |

<div id="manage-follows">
  #### Gestione dei follow
</div>

Gli endpoint v2 per la gestione dei follow sostituiranno gli endpoint standard v1.1 [POST friendships/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-create) e [POST friendships/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-destroy).

Le tabelle seguenti confrontano gli endpoint standard v1.1 e dell’API X v2 per la creazione dei follow:

<div id="follow-a-user">
  #### Seguire un utente
</div>

| Descrizione | Standard v1.1 | API X v2 |
| :--- | :--- | :--- |
| Metodi HTTP supportati | POST | POST |
| Dominio host | https://api.x.com | https://api.x.com |
| Percorso endpoint | /1.1/friendships/create.json | /2/users/:id/following |
| [Autenticazione](/it/resources/fundamentals/authentication) | Contesto utente OAuth 1.0a | Contesto utente OAuth 1.0a<br /><br />OAuth 2.0 con PKCE |
| [Limiti di frequenza](/it/resources/fundamentals/rate-limits) predefiniti per le richieste | 50 richieste ogni 15 min | 50 richieste ogni 15 min |
| Operazioni massime giornaliere per utente | 400 | 400 |
| Operazioni massime giornaliere per app | 1000 | 1000 |
| Richiede l'uso di credenziali di un'[App per sviluppatori](/it/resources/fundamentals/developer-apps) associata a un [Progetto](/it/resources/fundamentals/projects) |     | ✔️  |

<div id="unfollow-a-user">
  #### Smettere di seguire un utente
</div>

Le seguenti tabelle confrontano gli endpoint per annullare un follow tra Standard v1.1 e API X v2:

| Descrizione | Standard v1.1 | API X v2 |
| :--- | :--- | :--- |
| Metodi HTTP supportati | POST | DELETE |
| Dominio host | https://api.x.com | https://api.x.com |
| Percorso endpoint | /1.1/friendships/destroy.json | /2/users/:source\_user\_id/following/:target\_user\_id |
| [Autenticazione](/it/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code con PKCE |
| [Limiti di velocità](/it/resources/fundamentals/rate-limits) predefiniti per le richieste | 15 richieste ogni 15 min (per utente) | 50 richieste ogni 15 min (per utente) |
| Numero massimo di operazioni giornaliere per app | Nessuno | 500 |
| Richiede l’uso di credenziali da una [App per sviluppatori](/it/resources/fundamentals/developer-apps) associata a un [Progetto](/it/resources/fundamentals/projects) |     | ✔️  |
