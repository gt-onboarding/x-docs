<div id="manage-follows-standard-v11-compared-to-x-api-v2">
  #### Gestione dei follow: Standard v1.1 rispetto ad API X v2
</div>

Se hai utilizzato gli endpoint standard v1.1 [POST friendships/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-create) e [POST friendships/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-destroy), l’obiettivo di questa guida è aiutarti a comprendere somiglianze e differenze tra gli endpoint standard e quelli per la gestione dei follow dell’API X v2.

- **Somiglianze**
  - Contesto utente OAuth 1.0a
- **Differenze**
  - URL degli endpoint
  - Requisiti di App e Progetti
  - Metodi HTTP
  - Parametri della richiesta

     

<div id="similarities">
  #### Somiglianze
</div>

**Metodo di autenticazione OAuth 1.0a in contesto utente**

Entrambe le versioni dell’endpoint supportano [OAuth 1.0a User Context](/it/resources/fundamentals/authentication#oauth-1-0a-2). Pertanto, se in precedenza utilizzavi uno degli endpoint standard v1.1 per la gestione dei follow, puoi continuare a usare lo stesso metodo di autenticazione se esegui la migrazione alla versione dell’API X v2. 

<div id="differences">
  #### Differenze
</div>

**URL degli endpoint**

- Endpoint Standard v1.1:
  - POST https://api.x.com/1.1/friendships/create.json
    (seguire un utente)
  - POST https://api.x.com/1.1/friendships/destroy.json
    (smettere di seguire un utente)
- Endpoint API X v2:
  - POST https://api.x.com/2/users/:id/following
    (seguire un utente)
  - DELETE https://api.x.com/2/users/:source\_user\_id/following/:target\_user\_id
    (smettere di seguire un utente) 

**Requisiti di App e Progetti**

Gli endpoint dell’API X v2 richiedono l’uso di credenziali di una [developer App](/it/resources/fundamentals/developer-apps) associata a un [Project](/it/resources/fundamentals/projects) durante l’autenticazione delle richieste. Tutti gli endpoint dell’API X v1.1 possono utilizzare credenziali di App autonome o di App associate a un progetto.

**Parametri della richiesta**

I seguenti parametri di richiesta dello Standard v1.1 hanno equivalenti nell’API X v2:

| Standard v1.1 | API X v2 |
| :--- | :--- |
| Nessun equivalente | id (POST), source\_user\_id (DELETE) |
| user\_id | target\_user\_id |
| screen\_name | Nessun equivalente |

Si noti che i parametri dello Standard v1.1 vengono passati come query parameter, mentre i parametri dell’API X v2 vengono passati come body parameter (per l’endpoint POST) o path parameter (per l’endpoint DELETE).

Inoltre, i parametri v2 id e source\_user\_id non sono necessari quando si utilizzano gli endpoint Standard v1.1 poiché i token di accesso passati con OAuth 1.0a User Context consentivano di dedurre quale utente stesse avviando l’azione di follow/unfollow.
