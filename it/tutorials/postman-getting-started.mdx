<div id="introduction">
  ## Introduzione
</div>

Postman è un'applicazione desktop e web che consente di inviare richieste a un'API tramite un'interfaccia grafica. Consigliamo di usare Postman con l’API X, l’API X Ads e gli endpoint Labs sia per esplorare le funzionalità delle API sia per la risoluzione dei problemi della propria applicazione.

Attualmente sono disponibili due raccolte Postman:

<div id="prerequisites">
  ### Prerequisiti
</div>

Prima di iniziare a usare le raccolte Postman di X, assicurati di disporre dell’accesso e delle credenziali necessari per lo strumento della piattaforma per sviluppatori di X che preferisci. Puoi trovare ulteriori informazioni su come ottenere l’accesso nella nostra [pagina “Guida introduttiva”](/it/resources/platform-overview).

Una volta che disponi di un [account sviluppatore](https://developer.x.com/en/portal/petition/essential/basic-info), hai configurato un’[App per sviluppatori](/it/resources/fundamentals/developer-apps), hai un set di chiavi e token di [autenticazione](/it/resources/fundamentals/authentication) e hai configurato correttamente l’ambiente per inviare richieste all’API specifica che intendi usare, puoi seguire i passaggi seguenti per iniziare.
 

<div id="getting-started-with-xs-postman-collections">
  ## Introduzione alle collection Postman di X
</div>

<div id="step-one-add-one-of-the-x-postman-collections-to-your-account">
  ### Primo passaggio: aggiungi una delle raccolte Postman di X al tuo account
</div>

Anche se potresti creare gli endpoint specifici che desideri utilizzare in Postman, abbiamo già fatto il lavoro per te e preparato una raccolta pronta all’uso di API pertinenti. Ti basta fare clic su uno dei link nella sezione precedente “Raccolte Postman” e una raccolta con tutti gli endpoint associati all’API selezionata verrà aggiunta alla tua app Postman. Queste raccolte sono disponibili anche nel [Postman API Network](https://explore.postman.com/). Ogni endpoint includerà automaticamente i parametri disponibili, le risposte di esempio e il tipo di autenticazione già impostato, quindi dovrai solo aggiungere le tue credenziali e i valori dei parametri per iniziare a esplorarne le funzionalità.

In questo esempio, lavoreremo con la [raccolta API X v2](https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400). 
 

<div id="step-two-add-your-keys-and-tokens-as-environmental-variables">
  ### Passaggio due: aggiungi le tue chiavi e i tuoi token come variabili d'ambiente
</div>

Quando aggiungi la collection alla tua istanza di Postman, verrà creato automaticamente un ambiente chiamato “API X v2”, al quale dovrai aggiungere le tue chiavi e i tuoi token. In questo passaggio ti guideremo nell’aggiunta delle chiavi e dei token dalla tua App per sviluppatori all’ambiente “API X v2”.

Per aggiungere le tue chiavi e i tuoi token all’ambiente “API X v2”, fai clic sul pulsante “manage environments” nell’angolo in alto a destra di Postman; in alternativa, fai clic sul pulsante delle impostazioni nello stesso angolo.

  <img src="/images/using-postman-1.png.twimg.1920.png" alt="Questa immagine mostra il pulsante “manage environments” evidenziato nella console di Postman." />
</Frame>

Dall’elenco degli ambienti, fai clic su “API X v2”.

Quindi aggiungi una variabile per ciascuna delle diverse chiavi e dei token che hai generato tramite la dashboard delle App nella tabella. Ecco un esempio di come dovrebbe apparire la tua tabella:

| consumer\_secret | `AJX560A2Omgwyjr6Mml2esedujnZLHXXXXXX` | `AJX560A2Omgwyjr6Mml2esedujnZLHXXXXXX` |
| access\_token | `1995XXXXX-0NGqVhk3s96IX6SgT3H2bbjOPjcyQXXXXXXX` | `1995XXXXX-0NGqVhk3s96IX6SgT3H2bbjOPjcyQXXXXXXX` |
| token\_secret | `rHVuh7dgDuJCOGeoe4tndtjKwWiDjBZHLaZXXXXXX` | `rHVuh7dgDuJCOGeoe4tndtjKwWiDjBZHLaZXXXXXX` |

Nota: le chiavi e i token riportati nella tabella sono di esempio e non funzioneranno se usati in una richiesta.

Una volta aggiunte le credenziali come variabili e verificato che l’ambiente “API X v2” sia selezionato, sarai pronto a inviare richieste alla collection “API X v2”. Questo perché le schede di autorizzazione di ciascun endpoint sono configurate per ereditare automaticamente le variabili da questo ambiente.

Per usare Postman con User Access Tokens, salta a XXXX per ulteriori dettagli su come procedere.

### Terzo passaggio: seleziona un endpoint 

Il passaggio successivo è scegliere un endpoint dalla raccolta e iniziare a creare la richiesta. Puoi selezionare un endpoint dalla navigazione sul lato destro. Ecco come si presenta:

<Frame>
  <img src="/images/using-postman-2.png.twimg.1920.png" alt="Questa immagine mostra la richiesta «Single Posts» selezionata nel menu a discesa «Post Lookup» nella sezione «API X v2»." />
</Frame>

Per questo esempio, utilizzeremo l'endpoint API X v2 > Post Lookup > Single Post. 

<div id="step-four-add-values-to-the-params-tab">
  #### Passaggio quattro: aggiungi valori alla scheda Params
</div>

Nel prossimo passaggio, vai alla scheda Params. Dovresti vedere un insieme di parametri inattivi con descrizioni che spiegano la funzione di ciascun parametro e un elenco di tutti i possibili valori che puoi inviare con la tua richiesta.

In questo esempio, attiveremo i parametri di query expansions e tweet.fields e aggiungeremo i seguenti valori:

|     |     |
| :--- | :--- |
| **Chiave** | **Valore** |
| `tweet.fields` | `created_at,attachments` |
| expansions | author\_id |

Oltre ad aggiungere i parametri di query, dobbiamo aggiungere la Path Variable obbligatoria, id. Poiché questo endpoint restituisce Post, dobbiamo aggiungere un ID Post valido come valore.

Puoi trovare l’ID del Post andando su x.com, cliccando su un Post e poi guardando l’URL. Ad esempio, l’ID del Post del seguente URL è `1228393702244134912`:

`https://x.com/XDevelopers/status/1228393702244134912`

Nella scheda Params, scorri oltre tutti i parametri di query fino alla sezione “Path Variables”. Aggiungeremo l’ID del Post che desideri utilizzare come valore per la chiave id.

Se hai inserito correttamente tutto in questo passaggio, la scheda Params dovrebbe apparire come segue:

  <img src="/images/using-postman-3.png.twimg.1920.png" alt="Questa immagine mostra la tabella &#x22;Params&#x22; compilata in base alle istruzioni incluse in precedenza nella pagina." />

<div id="step-five-send-your-request-and-review-your-response">
  #### Passaggio cinque: invia la tua richiesta e verifica la risposta
</div>

Ora che la tua richiesta è pronta, puoi fare clic sul pulsante “Send”.

Se tutto è stato configurato correttamente, dovresti ricevere il seguente payload:

```json
{
    "data": {
        "author_id": "2244994945",
        "text": "What did the developer write in their Valentine’s card?\n  \nwhile(true) {\n    I = Love(You);  \n}",
        "id": "1228393702244134912",
        "created_at": "2020-02-14T19:00:55.000Z"
    },
    "includes": {
        "users": [
            {
                "username": "XDevelopers",
                "name": "Developers",
                "id": "2244994945"
            }
        ]
    }
}
```

<div id="generating-a-user-access-token-with-postman">
  ### Generazione di un token di accesso dell’utente con Postman:
</div>

<div id="using-oauth-10a-to-generate-a-user-access-token">
  #### Utilizzare OAuth 1.0a per generare un token di accesso utente
</div>

Rivedi il processo in tre passaggi utilizzato nella [raccolta di test del flusso OAuth 1.0a](https://www.postman.com/xapidevelopers/workspace/twitter-s-public-workspace/collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400?ctx=documentation).

<div id="using-oauth-20-to-generate-a-user-access-token">
  #### Utilizzo di OAuth 2.0 per generare un token di accesso utente
</div>

Se desideri generare un token di accesso utente OAuth 2.0 in Postman, puoi creare token di accesso OAuth 2.0 da usare insieme alla [raccolta Postman dell’API X v2](https://www.postman.com/xapidevelopers/workspace/twitter-s-public-workspace/collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400?ctx=documentation). 

Fai clic sulla raccolta nel tuo workspace, vai alla scheda “Auth” e seleziona come tipo “OAuth 2.0”. Da lì, sotto l’intestazione “Configure New Token”, apri “Configuration Options”. Puoi impostare “Grant Type” su “Authorization Code (With PKCE)”.

Aggiorna la tua Callback URL in modo che corrisponda alla callback URL associata alla tua applicazione. Inoltre, aggiorna i seguenti parametri:

- Auth URL - https://x.com/i/oauth2/authorize
- Access Token URL - https://api.x.com/2/oauth2/token
- Client ID - Il tuo client ID OAuth 2.0 dal Dev Portal
- Client Secret - Se stai utilizzando un client riservato
- Update Scope - Scope da far corrispondere agli endpoint a cui vuoi connetterti. Esempio “tweet.read users.read”
- La tua callback URL (nota anche come redirect URL). Deve corrispondere a quanto impostato nelle impostazioni di autenticazione della tua App.
- State - state

Quando sei pronto, fai clic su “Get New Access Token” per generare un token di accesso. Se visualizzi una finestra di dialogo che indica che qualcosa è andato storto, potresti dover premere il pulsante Indietro per accedere. Dovrai autorizzare la tua app ad accedere al tuo account facendo clic su “Authorize app” nella finestra di dialogo.

Dopo aver autorizzato la tua app, verrai reindirizzato a Postman, dove potrai vedere il tuo token e selezionare il pulsante “Use Token” per iniziare a effettuare richieste per conto di un utente autorizzato.

Ora sei pronto per utilizzare la raccolta Postman.

<div id="whats-next">
  ## Cosa succede dopo
</div>

Se fai clic sul pulsante “Code” in Postman, puoi convertire la richiesta appena creata nel linguaggio che preferisci, ad esempio Python, Node o Ruby, per aiutarti ad avviare lo sviluppo. Postman offre [un’ottima documentazione](https://learning.getpostman.com/) che potrebbe esserti utile. Disponiamo anche di [codice di esempio su GitHub](https://github.com/xdevplatform) che può aiutarti a integrarti più rapidamente con gli endpoint.
