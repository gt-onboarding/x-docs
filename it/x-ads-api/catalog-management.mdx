<div id="overview">
  ## Panoramica
</div>

La Catalog API è una soluzione per il commerce che consente agli inserzionisti di configurare feed di prodotti, raggruppare i prodotti in set e gestire in modo completo i prodotti del catalogo. La Catalog API sarà uno strumento fondamentale che abilita la gestione programmatica dei cataloghi e offre agli inserzionisti un maggiore controllo su come i loro cataloghi vengono acquisiti e aggiornati.

Un catalogo contiene tutto ciò che riguarda i prodotti e i set di prodotti dell’utente ed è associato all’handle dell’utente. Attualmente, **un utente può creare un solo catalogo**.

La Catalog API supporta 2 modalità per acquisire i prodotti:

1.  Scheduled Feed: l’utente può aggiungere un URL del feed (CSV, TSV e XML) che verrà recuperato periodicamente (X supporta attualmente file fino a 8 GB)

2.  Batch Products API: l’utente può visualizzare, creare, aggiornare ed eliminare attributi di prodotto con richieste batch (JSON)

I merchant su X possono creare set di prodotti basati su regole di filtro. In questo modo potranno associare metadati aggiuntivi ai loro prodotti e creare set di prodotti speciali per mettere in evidenza tali prodotti tramite funzionalità di commerce organico o annunci dinamici di prodotto. La Catalog API e lo Shopping Manager di X supportano due tipi di set di prodotti.

1.  Manual: l’utente può selezionare fino a 50 prodotti e impostare un nome e una descrizione

2.  Filter: l’utente può aggiungere fino a 30 filtri per generare automaticamente set con prodotti che soddisfano tali filtri. Tipi di filtri includono prezzo, categoria di prodotto Google, tipo di prodotto, inventario, prezzo scontato e campi personalizzati.

La Catalog API include 4 endpoint per gestire il catalogo:

* [Product Catalogs](https://developer.x.com/en/docs/x-ads-api/catalog-management/api-reference/product-catalogs) - Creazione di un contenitore per le informazioni sui prodotti
* [Products](https://developer.x.com/en/docs/x-ads-api/catalog-management/api-reference/products) - Aggiunta sincrona, aggiornamento, visualizzazione e rimozione di prodotti dai cataloghi
* [Product Sets](https://developer.x.com/en/docs/x-ads-api/catalog-management/api-reference/product-sets) - Raggruppamento dei prodotti in base alle informazioni di prodotto
* [Schedule Feeds](https://developer.x.com/en/docs/twitter-ads-api/catalog-management/api-reference/scheduled-feeds) - Configurazione dell’acquisizione asincrona di dati di prodotto basati su file nei cataloghi

<div id="prerequisites-for-the-catalog-api">
  ### Prerequisiti per la Catalog API
</div>

Gli endpoint della Catalog API sono attualmente disponibili solo in early access. Per richiedere l’accesso, contatta il tuo referente X.

È necessario accettare i termini di servizio tramite [X Shopping Manager](https://ads.x.com/shopping_manager).

Per maggiori dettagli su Shopping Manager e le specifiche dei prodotti, consulta la [guida alle specifiche dei prodotti](https://business.x.com/en/help/shopping-specs.html).

### Limiti di frequenza 

La tabella seguente elenca i limiti di frequenza per ciascun endpoint.

<div id="product-catalog">
  #### Catalogo prodotti
</div>

| Metodo HTTP | Limite di frequenza |
| :--- | :--- |
| GET /product_catalogs | 1.000 ogni 15 minuti |
| POST /product_catalogs | 20 ogni 15 minuti |
| PUT /product\_catalogs/:product\_catalog_id | 20 ogni 15 minuti |
| DELETE /product\_catalogs/:product\_catalog_id | 20 ogni 15 minuti |

<div id="batch-products-api">
  #### API Batch dei prodotti
</div>

| Metodo HTTP | Limite di frequenza |
| :--- | :--- |
| GET /product\_catalogs/:product\_catalog_id/products | 1.000 ogni 15 minuti |
| PUT /product\_catalogs/:product\_catalog_id/products | 600 ogni 15 minuti |
| DELETE /product\_catalogs/:product\_catalog_id/products | 150 ogni 15 minuti |
