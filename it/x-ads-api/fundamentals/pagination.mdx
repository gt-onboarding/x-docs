---
title: Paginazione
---

Per supportare un numero più elevato di campagne e il recupero efficiente di tutte le entità associate a un account, l'API Advertiser ora supporta la paginazione su molti endpoint GET. Il meccanismo di paginazione è facile da usare ed è molto simile alla paginazione basata su cursori dell’API REST, come descritto in [Using cursors to navigate collections](https://developer.x.com/en/docs/x-api/v1/pagination).

<div id="getting-started">
  ## Per iniziare
</div>

Per alcune richieste GET, ora accettiamo i parametri di query `cursor` e `count`, entrambi opzionali. Se una risposta contiene più di `count` entità, questi endpoint restituiranno le prime `count` entità e una chiave `next_cursor` nel JSON di risposta.

`GET https://ads-api.x.com/5/accounts/abc1/campaigns?count=50`

```json
{
  "data": \[...\],
  "next_cursor": "c-3yvu1pzhd3i7",
  "request": {...}
}
```

Per ottenere la pagina successiva di risultati, aggiungi `cursor` ai parametri di query:

`GET https://ads-api.x.com/5/accounts/abc1/campaigns?cursor=c-3yvu1pzhd3i7&count=50`

```json
{
  "data": \[...\],
  "next_cursor": "c-3w3zdyg8ywan",
  "request": {...}
}
```

<div id="going-deeper">
  ## Approfondimenti
</div>

Per la maggior parte degli endpoint, il valore massimo di `count` è **1.000**, il minimo è **1** e il valore predefinito è **200**.

Il valore restituito da `next_cursor` è sempre una stringa e va considerato opaco; l’implementazione può cambiare. Se nella pagina corrente del set di risultati vengono restituite meno di `count` entità, il valore di `next_cursor` sarà `null`.

**Nota:** gli endpoint di analytics non supportano questo tipo di paginazione. Il paging sugli endpoint delle statistiche è supportato specificando intervalli temporali. Consulta la documentazione dei singoli endpoint per ulteriori informazioni.
