<div id="how-to-get-access-to-the-ads-api">
  ## Come ottenere l’accesso all’API X Ads
</div>

1. Registra un [account sviluppatore](https://developer.x.com/en/apply-for-access).
2. Crea un’[App per sviluppatori](https://developer.x.com/app) e proteggi il tuo token.
3. Visita [ads.x.com/help](https://ads.x.com/help) per inviare una richiesta di accesso all’API X Ads per ciascuna delle tue App per sviluppatori.

<Note>**Nota:** Se stai già sviluppando sulla Piattaforma per sviluppatori di X e hai un account sviluppatore, passa al punto tre.</Note>

<div id="step-one-signup-for-a-developer-account">
  ### Passaggio uno: registrati per un account sviluppatore
</div>

Per inviare richieste a qualsiasi prodotto API di X, devi prima creare un account sviluppatore. Nel portale per sviluppatori, crea un Progetto e un'App per sviluppatori. Questo ti fornirà un set di credenziali da usare per autenticare tutte le richieste all'API.

<Button href="https://developer.x.com/en/apply-for-access">
  Crea un account sviluppatore
</Button>

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### Secondo passaggio: salva le chiavi e i token della tua App e mantienili al sicuro
</div>

All'interno della tua App per sviluppatori ti verrà fornito un set di API keys (note anche come Consumer keys). Avrai anche la possibilità di generare un set di access tokens che possono essere usati per effettuare richieste per conto del tuo account X personale e un bearer token che può essere usato per autenticare gli endpoint che richiedono l’OAuth 2.0 bearer token. Poiché queste chiavi e questi token non scadono a meno che non vengano rigenerati, ti suggeriamo di creare variabili d’ambiente o di utilizzare un gestore di password sicuro.

Prendi nota anche dell’ID della tua App, reperibile nell’URL della pagina dei dettagli della tua App. Questo renderà più semplice richiedere l’accesso all’API X Ads nel prossimo passaggio.

<Note>
**Nota:** le tue chiavi e i tuoi token verranno mostrati una sola volta nel portale per sviluppatori, quindi è importante archiviare queste credenziali nel tuo sistema di gestione delle password non appena li generi.

Se perdi o dimentichi le chiavi e i token, dovrai rigenerarli, creando nuove chiavi e nuovi token e invalidando quelli vecchi. Ciò significa che dovrai aggiornare eventuali integrazioni che potresti aver configurato con le credenziali precedenti.

Scopri di più sulle nostre [best practice di autenticazione](https://developer.x.com/en/docs/authentication/guides/authentication-best-practices).
</Note>

<div id="step-three-apply-for-access-to-the-ads-api">
  ### Passaggio tre: richiedere l’accesso all’API X Ads
</div>

A questo punto avrai [accesso di base all’API X](https://developer.x.com/en/docs/twitter-api/v1), ma non alle funzionalità specifiche dell’API X Ads. Successivamente dovrai richiedere l’accesso e ottenere l’approvazione per l’API X Ads. Visita [ads.x.com/help](https://ads.x.com/help) per inviare una richiesta di accesso all’API X Ads per ciascuna delle tue app per sviluppatori.

<div id="access-tiers">
  ### Livelli di accesso
</div>

Nel processo di candidatura, dovrai indicare il livello di accesso richiesto. Scopri di più sulle [autorizzazioni a livello di App e di account pubblicitario](https://developer.x.com/en/docs/twitter-ads-api/obtaining-ads-account-access#levels-of-access).

<div id="conversion-only">
  #### Solo conversioni
</div>

Accesso agli endpoint di conversione per dispositivi mobili e web con permessi di lettura e scrittura.

<div id="standard-access">
  #### Accesso standard
</div>

Accesso agli endpoint Analytics, Gestione campagne, Creatività, Pubblici personalizzati e Conversion con permessi di lettura e scrittura.

<Note>**Nota:** Dopo l’approvazione della tua app per l’accesso all’API X Ads, dovrai [rigenerare eventuali token di accesso utente](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens) per inviare correttamente [richieste autenticate all’API X Ads](https://developer.x.com/en/docs/twitter-ads-api/making-authenticated-requests).</Note>

<Note>**Nota:** Gli sviluppatori dell’API X Ads che hanno richiesto l’accesso prima di luglio 2023 potrebbero avere livelli di accesso e autorizzazioni diversi e potrebbero essere limitati a cinque token OAuth. Consulta la nostra guida su [aumentare l’accesso](https://developer.x.com/en/docs/twitter-ads-api/increasing-access) per ottenere l’accesso a endpoint aggiuntivi o rimuovere i limiti di token per le applicazioni esistenti.</Note>

<div id="make-your-first-request">
  ## Effettua la tua prima richiesta
</div>

Per testare l’accesso all’API X Ads, effettua una richiesta all’endpoint [GET accounts](https://developer.x.com/en/docs/twitter-ads-api/campaign-management/api-reference/accounts). Questo endpoint restituisce gli account pubblicitari a cui l’utente attualmente autorizzato ha accesso. Usa gli ID degli account pubblicitari restituiti in questa risposta nelle successive richieste API per leggere e scrivere dati per uno specifico account pubblicitario. Utilizzando [Twurl](https://developer.x.com/en/docs/tutorials/using-twurl) dalla riga di comando, la richiesta sarà la seguente.

**Esempio di richiesta**

```bash
twurl -H ads-api.x.com "/11/accounts"
```

**Esempio di risposta**

