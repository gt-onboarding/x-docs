<div id="how-to-get-access-to-the-ads-api">
  ## So erhalten Sie Zugriff auf die Ads-API
</div>

1. Registrieren Sie sich für ein [Entwicklerkonto](https://developer.x.com/en/apply-for-access).
2. Erstellen Sie eine [Entwickler-App](https://developer.x.com/app) und sichern Sie Ihr Token.
3. Besuchen Sie [ads.x.com/help](https://ads.x.com/help), um für jede Ihrer Entwickler-Apps einen Antrag auf Ads-API-Zugriff zu stellen.

<Note>**Hinweis:** Wenn Sie bereits auf der X Developer Platform entwickeln und ein Entwicklerkonto haben, fahren Sie mit Schritt drei fort.</Note>

<div id="step-one-signup-for-a-developer-account">
  ### Schritt eins: Registrieren Sie sich für ein Entwicklerkonto
</div>

Um Anfragen an eines der X-API‑Produkte zu stellen, müssen Sie sich zunächst für ein Entwicklerkonto registrieren. Erstellen Sie im Developer-Portal ein Projekt und eine Entwickler-App. Dadurch erhalten Sie Zugangsdaten, mit denen Sie alle Anfragen an die API authentifizieren.

<Button href="https://developer.x.com/en/apply-for-access">
  Registrieren Sie sich für ein Entwicklerkonto
</Button>

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### Schritt zwei: Speichern Sie die Schlüssel und Token Ihrer App und halten Sie sie sicher
</div>

In Ihrer Entwickler-App erhalten Sie einen Satz API-Schlüssel (auch Consumer Keys genannt). Sie haben außerdem die Möglichkeit, einen Satz Zugriffstoken zu generieren, mit denen Sie Anfragen im Namen Ihres persönlichen X-Kontos stellen können, sowie ein Bearer-Token, das zur Authentifizierung von Endpunkten verwendet werden kann, die ein OAuth 2.0 Bearer Token erfordern. Da diese Schlüssel und Token nicht ablaufen, es sei denn, sie werden neu generiert, empfehlen wir, Umgebungsvariablen zu verwenden oder einen sicheren Passwortmanager einzusetzen.

Bitte notieren Sie sich auch Ihre App-ID; diese finden Sie in der URL Ihrer App-Details. Das erleichtert im nächsten Schritt die Beantragung des Zugriffs auf die X-Ads-API.

<Note>
**Hinweis:** Ihre Schlüssel und Token werden im Entwicklerportal nur einmal angezeigt. Es ist daher wichtig, dass Sie diese Zugangsdaten unmittelbar nach ihrer Generierung in Ihrem Passwortmanager speichern.

Wenn Sie die Schlüssel und Token verlegen oder vergessen, müssen Sie sie neu generieren. Dadurch werden neue Schlüssel und Token erstellt und die alten ungültig. Das bedeutet, dass Sie alle Integrationen, die Sie mit Ihren bisherigen Zugangsdaten eingerichtet haben, aktualisieren müssen.

Erfahren Sie mehr über unsere [Best Practices zur Authentifizierung](https://developer.x.com/en/docs/authentication/guides/authentication-best-practices).
</Note>

<div id="step-three-apply-for-access-to-the-ads-api">
  ### Schritt drei: Zugriff auf die X-Ads-API beantragen
</div>

An diesem Punkt verfügen Sie über [Basiszugriff auf die X-API](https://developer.x.com/en/docs/twitter-api/v1), jedoch nicht über spezifische Funktionen der X-Ads-API. Als Nächstes müssen Sie Zugriff beantragen und eine Genehmigung für den Zugriff auf die X-Ads-API erhalten. Besuchen Sie [ads.x.com/help](https://ads.x.com/help), um für jede Ihrer Entwickler-Apps eine Anfrage auf Zugriff auf die X-Ads-API einzureichen.

<div id="access-tiers">
  ### Zugriffsstufen
</div>

Im Rahmen des Antragsprozesses müssen Sie angeben, welche Art von Zugriff Sie benötigen. Weitere Informationen zu [Berechtigungen auf App- und Anzeigenkontoebene](https://developer.x.com/en/docs/twitter-ads-api/obtaining-ads-account-access#levels-of-access).

<div id="conversion-only">
  #### Nur Conversions
</div>

Zugriff auf Mobile- und Web-Conversion-Endpunkte mit Lese- und Schreibrechten.

<div id="standard-access">
  #### Standardzugang
</div>

Zugriff auf Endpunkte für Analytics, Kampagnenverwaltung, Creatives, Benutzerdefinierte Zielgruppen und Conversions mit Lese- und Schreibrechten.

<Note>**Hinweis:** Nachdem Ihre App für den Zugriff auf die X-Ads-API genehmigt wurde, müssen Sie [Benutzerzugriffstoken neu generieren](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens), um korrekt [authentifizierte Anfragen an die X-Ads-API](https://developer.x.com/en/docs/twitter-ads-api/making-authenticated-requests) zu stellen.</Note>

<Note>**Hinweis:** Entwickler der X-Ads-API, die vor Juli 2023 Zugriff beantragt haben, können über abweichende Zugriffs- und Berechtigungsstufen verfügen und sind möglicherweise auf fünf OAuth-Token beschränkt. Lesen Sie unseren Leitfaden zum [Erhöhen des Zugriffs](https://developer.x.com/en/docs/twitter-ads-api/increasing-access), um zusätzliche Endpunkte freizuschalten oder Token-Limits für bestehende Anwendungen aufzuheben.</Note>

<div id="make-your-first-request">
  ## Führen Sie Ihre erste Anfrage aus
</div>

Um Ihren Zugriff auf die X-Ads-API zu testen, senden Sie eine Anfrage an den Endpunkt [GET accounts](https://developer.x.com/en/docs/twitter-ads-api/campaign-management/api-reference/accounts). Dieser Endpunkt gibt die Werbekonten zurück, auf die der aktuell autorisierte Nutzer Zugriff hat. Verwenden Sie die Werbekonto-IDs aus dieser Anfrage in nachfolgenden API-Anfragen, um Daten für ein bestimmtes Werbekonto zu lesen und zu schreiben. Mit [Twurl](https://developer.x.com/en/docs/tutorials/using-twurl) auf der Kommandozeile würde die Anfrage wie folgt aussehen.

**Beispielanfrage**

```bash
twurl -H ads-api.x.com "/11/accounts"
```

**Beispielantwort**
