<div id="timezones-accounts-and-billing">
  ## Zeitzonen, Konten und Abrechnung
</div>

Datum‑Uhrzeit‑Werte werden stets in UTC zurückgegeben (erkennbar am „Z“ am Ende des Werts). Datums-/Uhrzeitangaben können in POST- oder PUT-Anfragen in beliebigen Zeitzonen angegeben werden, unter Verwendung des ISO‑8601‑Standardformats für Zeitzonen. Die Zeit wird gemäß einer Teilmenge von [ISO‑8601](http://en.wikipedia.org/wiki/ISO_8601) dargestellt. Konkret lautet die strptime-Zeichenfolge für unser Datumsformat `%Y-%m-%dT%l:%M:%S%z`. Die Zeitzone des Werbetreibendenkontos bestimmt den Zeitpunkt, zu dem die offiziellen Abrechnungszahlen festgeschrieben werden.

Beim Abfragen der API auf Kontoebene ([GET accounts](/de/x-ads-api/campaign-management#accounts)) erhalten Sie Zeitzoneninformationen in folgendem Format:

```json
    {
      "request": {
        "params": {
          "account_id": "18ce54d4x5t"
        }
      },
      "data": {
        "name": "API McTestface",
        "business_name": null,
        "timezone": "America/Los_Angeles",
        "timezone_switch_at": "2016-07-21T07:00:00Z",
        "id": "18ce54d4x5t",
        "created_at": "2016-07-21T22:42:09Z",
        "salt": "54cb7b5a34183f77d82dd6d0f4329691",
        "updated_at": "2017-09-09T06:42:14Z",
        "business_id": null,
        "approval_status": "ACCEPTED",
        "deleted": false
      }
    }
```

Dies umfasst zwei Zeitzonenwerte: `timezone` (siehe [Zeitzonen in Wikipedia](https://en.wikipedia.org/wiki/Tz_database#Names_of_time_zones) für einen Überblick) und `timezone_switch_at`. Beachten Sie, dass `timezone_switch_at` in UTC (+00:00) angegeben ist, aber stets Mitternacht in der betreffenden Zeitzone repräsentiert.

Die Zeitzone des Werbetreibenden ist über die API nicht änderbar. Dieses Attribut wird auf Vertrags-/Abrechnungsebene von der/dem Account-Manager/in des Werbetreibenden bei X festgelegt.

Bitte berücksichtigen Sie den Wert `timezone_switch_at` beim Erstellen von Berichten und beim Abfragen unserer Analytics-Endpunkte, da es an dem Tag, an dem das Konto von der Zeitzone America/Los\_Angeles auf den neuen lokalen Wert wechselt, zu einer Lücke kommen kann.

<div id="specifying-datetime-values-with-timezone">
  ### Angeben von Datetime-Werten mit Zeitzone
</div>

Datetime-Werte werden stets in UTC zurückgegeben (erkennbar am `Z` am Ende des Datetime-Werts). In POST- oder PUT-Befehlen können Datetimes in beliebigen Zeitzonen gemäß dem [ISO-8601-Standard](http://en.wikipedia.org/wiki/ISO_8601) für Zeitzonen angegeben werden. Zum Beispiel ist `2017-07-10T08:00:00-0800` ein zulässiger Eingabewert und wird automatisch in den UTC-Wert `2017-07-10T16:00:00Z` umgerechnet.

Bei Verwendung der Analytics-Endpunkte mit `granularity` von `DAY` oder `TOTAL` muss `start_time` auf Mitternacht des gewünschten Tages in der lokalen Zeitzone des Kontoinhabers gesetzt werden. Der zu verwendende Zeitzonen-Offset entspricht dem Offset des aktuellen Tages, nicht dem des betreffenden Tages. Beispielsweise beträgt für ein Ads-Konto in America/Los\_Angeles während der Pacific Daylight Time der UTC-Offset -0700. In einer Analytics-Anfrage sollte die Zeit daher wie folgt angegeben werden: `start_time=2017-05-21T07:00:00Z` oder `start_time=2017-05-21T00:00:00-0700`. Befände sich das Ads-Konto in Asia/Tokyo, wo der Offset stets +09:00 beträgt, würden die Werte wie folgt angegeben: `start_time=2017-05-20T15:00:00Z` oder `start_time=2017-05-21T00:00:00+0900`.

<div id="accepted-utc-offset-formats">
  ### Akzeptierte UTC-Offset-Formate
</div>

Siehe [ISO 8601-Zeitzonenbezeichner](http://en.wikipedia.org/wiki/ISO_8601#Time_zone_designators).

Unterstützt: `Z`, `-HHMM`, `+HHMM`, `-HH:MM`, `+HH:MM`, `-HH`, `+HH`
