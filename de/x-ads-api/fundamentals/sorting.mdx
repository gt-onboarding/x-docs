---
title: Sortierung
---

Für die meisten X-Ads-API-Collection-Endpunkte (die eine Liste von Objekten zurückgeben) ist eine Sortierung verfügbar. Je nach Endpunkt stehen verschiedene Parameter für die Sortierung zur Verfügung. Die meisten durch diese Parameter zurückgegebenen Felder (außer IDs und ENUMs) sind sortierbar.

| API-Collection | Sortierbare Parameter |
| :--- | :--- |
| [Accounts](/de/x-ads-api/campaign-management#accounts) | `created_at`, `updated_at`, `deleted`, `name` |
| [Funding Instruments](/de/x-ads-api/campaign-management#funding-instruments) | `created_at`, `updated_at`, `deleted`, `funded_amount_local_micro`, `start_time`, `end_time` |
| [Campaigns](/de/x-ads-api/campaign-management#campaigns) | `created_at`, `updated_at`, `deleted`, `name`, `start_time`, `end_time`, `daily_budget_amount_local_micro`, `total_budget_amount_local_micro`, `standard_delivery` |
| [Line Items](/de/x-ads-api/campaign-management#line-items) | `created_at`, `updated_at`, `deleted`, `bid_amount_local_micro` |
| [Cards](https://developer.x.com/en/docs/ads/creatives/api-reference/image-app-download) | `created_at`, `updated_at`, `deleted`, `name` |
| [Promoted Accounts](/de/x-ads-api/campaign-management#advertiser-api) | `created_at`, `updated_at`, `deleted`, `paused` |
| [Promoted Tweets](/de/x-ads-api/campaign-management#promoted-tweets) | `created_at`, `updated_at`, `deleted`, `paused` |
| [App Event Tags](/de/x-ads-api/measurement/mobile-conversions#app-event-tags) | `created_at`, `updated_at`, `deleted`, `post_view_attribution_window`, `post_engagement_attribution_window`, `assisted_conversion`, `provider_app_event_name` |

<div id="getting-started">
  ## Erste Schritte
</div>

Hängen Sie in Ihrer API-Anfrage einfach `sort_by=[attribute name]-[asc/desc]` an, wobei das Attribut ein gültiges Attribut ist, das im JSON Ihrer GET-Anfrage zurückgegeben wird.

Sie können zum Beispiel `?name-asc` verwenden, um alphabetisch nach Namen zu sortieren, oder `?name-desc`, um in umgekehrter Reihenfolge zu sortieren.
