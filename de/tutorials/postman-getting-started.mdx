<div id="introduction">
  ## Einführung
</div>

Postman ist eine Desktop- und Webanwendung, mit der Sie über eine grafische Benutzeroberfläche API-Anfragen senden können. Wir empfehlen, Postman mit der X-API, der X-Ads-API und den Labs-Endpunkten zu verwenden – sowohl zum Erkunden der API-Funktionen als auch zur Fehlerbehebung in Ihrer Anwendung.

Derzeit stehen zwei Postman-Collections zur Verfügung:

<div id="prerequisites">
  ### Voraussetzungen
</div>

Bevor Sie mit den Postman-Collections von X beginnen, stellen Sie sicher, dass Sie über den entsprechenden Zugriff und die Anmeldedaten für das X-Entwicklerplattform-Tool Ihrer Wahl verfügen. Weitere Informationen zum Erlangen von Zugriff finden Sie auf unserer Seite [„Erste Schritte“](/de/resources/platform-overview).

Sobald Sie ein [Entwicklerkonto](https://developer.x.com/en/portal/petition/essential/basic-info) besitzen, eine [Developer App](/de/resources/fundamentals/developer-apps) eingerichtet und einen Satz [Authentifizierungs-](/de/resources/fundamentals/authentication)schlüssel und -Tokens erhalten haben und Ihre Umgebung korrekt eingerichtet ist, um Anfragen an die spezifische API zu stellen, die Sie verwenden möchten, können Sie den folgenden Schritten zum Einstieg folgen.
 

<div id="getting-started-with-xs-postman-collections">
  ## Einstieg in die Postman-Sammlungen von X
</div>

<div id="step-one-add-one-of-the-x-postman-collections-to-your-account">
  ### Schritt eins: Fügen Sie eine der X Postman-Collections zu Ihrem Konto hinzu
</div>

Zwar könnten Sie die spezifischen Endpunkte, die Sie in Postman verwenden möchten, selbst zusammenstellen, aber wir haben die ganze Vorarbeit für Sie erledigt und eine sofort einsatzbereite Sammlung relevanter APIs gebaut. Klicken Sie einfach auf einen der Links im vorherigen Abschnitt „Postman-Collections“, und eine Collection mit allen zu der ausgewählten API gehörenden Endpunkten wird zu Ihrer Postman-App hinzugefügt. Diese Collections sind auch im [Postman API Network](https://explore.postman.com/) verfügbar. Jeder Endpunkt enthält automatisch die verfügbaren Parameter, Beispielantworten und den vorkonfigurierten Authentifizierungstyp, sodass Sie nur noch Ihre Zugangsdaten und Parameterwerte hinzufügen müssen, um die Funktionen zu erkunden.

In diesem Beispiel arbeiten wir mit der X [API v2 Collection](https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400). 
 

<div id="step-two-add-your-keys-and-tokens-as-environmental-variables">
  ### Schritt zwei: Fügen Sie Ihre Keys und Tokens als Umgebungsvariablen hinzu
</div>

Wenn Sie die Collection zu Ihrer Postman-Instanz hinzufügen, wird automatisch eine Umgebung mit dem Namen „X-API v2“ erstellt, der Sie Ihre Keys und Tokens hinzufügen müssen. In diesem Schritt führen wir Sie durch den Prozess, Ihre Keys und Tokens aus Ihrer Entwickler-App in die Umgebung „X-API v2“ einzutragen. 

Um Ihre Keys und Tokens zur Umgebung „X-API v2“ hinzuzufügen, klicken Sie oben rechts in Postman auf die Schaltfläche „Manage Environments“. Alternativ können Sie auch über die Einstellungen oben rechts dorthin gelangen. 
 

<Frame>
  <img src="/images/using-postman-1.png.twimg.1920.png" alt="Dieses Bild zeigt die in der Postman-Konsole hervorgehobene Schaltfläche „Manage Environments“." />
</Frame>

Klicken Sie in der Liste der Umgebungen auf „X-API v2“. 

Fügen Sie anschließend für alle unterschiedlichen Keys und Tokens, die Sie über das Apps-Dashboard generiert haben, jeweils eine Variable hinzu und tragen Sie diese in die Tabelle ein. So könnte Ihre Tabelle aussehen:

| VARIABLE | INITIAL VALUE | CURRENT VALUE |
| :--- | :--- | :--- |
| consumer\_key | `QAktM6W6DF6F7XXXXXX` | `QAktM6W6DF6F7XXXXXX` |
| consumer\_secret | `AJX560A2Omgwyjr6Mml2esedujnZLHXXXXXX` | `AJX560A2Omgwyjr6Mml2esedujnZLHXXXXXX` |
| access\_token | `1995XXXXX-0NGqVhk3s96IX6SgT3H2bbjOPjcyQXXXXXXX` | `1995XXXXX-0NGqVhk3s96IX6SgT3H2bbjOPjcyQXXXXXXX` |
| token\_secret | `rHVuh7dgDuJCOGeoe4tndtjKwWiDjBZHLaZXXXXXX` | `rHVuh7dgDuJCOGeoe4tndtjKwWiDjBZHLaZXXXXXX` |
| bearer\_token | `AAAAAAAAAAAAAAAAAAAAAL9v6AAAAAAA99t03huuqRYg0mpYAAFRbPR3XXXXXXX` | `AAAAAAAAAAAAAAAAAAAAAL9v6AAAAAAA99t03huuqRYg0mpYAAFRbPR3XXXXXXX` |

Beachten Sie, dass die in der obigen Tabelle verwendeten Keys und Tokens frei erfunden sind und in Anfragen nicht funktionieren. 

Sobald Sie Ihre Zugangsdaten als Variablen hinzugefügt und sichergestellt haben, dass die Umgebung „X-API v2“ ausgewählt ist, können Sie Anfragen an die „X-API v2“-Collection stellen. Die Autorisierungs-Tabs der einzelnen Endpunkte sind so konfiguriert, dass sie die Variablen automatisch aus dieser Umgebung übernehmen. 

Wenn Sie Postman mit User Access Tokens verwenden möchten, überspringen Sie XXXX zu den zusätzlichen Details, wie dies funktioniert.

### Schritt drei: Einen Endpoint auswählen 

Als Nächstes wählen Sie einen Endpoint aus der Sammlung und beginnen mit dem Erstellen Ihrer Anfrage. Sie können einen Endpoint über die Navigation auf der rechten Seite auswählen. So sieht das aus:

<Frame>
  <img src="/images/using-postman-2.png.twimg.1920.png" alt="Dieses Bild zeigt die ausgewählte Anfrage „Single Posts“ unter dem Dropdown „Post Lookup“ im Abschnitt „X-API v2“." />
</Frame>

Für dieses Beispiel verwenden wir den Endpoint X-API v2 > Post Lookup > Single Post. 

<div id="step-four-add-values-to-the-params-tab">
  #### Schritt vier: Werte auf der Registerkarte „Params“ hinzufügen
</div>

Als Nächstes wechseln Sie zur Registerkarte „Params“. Sie sollten eine Reihe inaktiver Parameter mit Beschreibungen sehen, die erläutern, wofür der jeweilige Parameter steht, sowie eine Liste aller möglichen Werte, die Sie mit Ihrer Anfrage übergeben können.

In diesem Beispiel aktivieren wir die Query-Parameter expansions und tweet.fields und fügen die folgenden Werte hinzu:

|     |     |
| :--- | :--- |
| **Schlüssel** | **Wert** |
| `tweet.fields` | `created_at,attachments` |
| expansions | author\_id |

Zusätzlich zu den Query-Parametern müssen wir die erforderliche Pfadvariable id hinzufügen. Da dieser Endpoint Posts zurückgibt, müssen wir als Wert eine gültige Post-ID angeben.

Sie können die Post-ID ermitteln, indem Sie zu x.com navigieren, auf einen Post klicken und dann in der URL nachsehen. Beispielsweise lautet die Post-ID der folgenden URL `1228393702244134912`:

`https://x.com/XDevelopers/status/1228393702244134912`

Scrollen Sie auf der Registerkarte „Params“ an allen Query-Parametern vorbei, bis der Abschnitt „Path Variables“ angezeigt wird. Dort fügen wir die Post-ID ein, die Sie als Wert für den Schlüssel id verwenden möchten.

Wenn Sie in diesem Schritt alles korrekt eingegeben haben, sollte die Registerkarte „Params“ wie folgt aussehen:

</Frame>

<div id="step-five-send-your-request-and-review-your-response">
  #### Schritt fünf: Senden Sie Ihre Anfrage und prüfen Sie die Antwort
</div>

Jetzt, da in Ihrer Anfrage alles eingerichtet ist, können Sie auf die Schaltfläche „Send“ klicken.

Wenn alles korrekt eingerichtet wurde, sollten Sie die folgende Payload erhalten:

```json
{
    "data": {
        "author_id": "2244994945",
        "text": "What did the developer write in their Valentine’s card?\n  \nwhile(true) {\n    I = Love(You);  \n}",
        "id": "1228393702244134912",
        "created_at": "2020-02-14T19:00:55.000Z"
    },
    "includes": {
        "users": [
            {
                "username": "XDevelopers",
                "name": "Developers",
                "id": "2244994945"
            }
        ]
    }
}
```

<div id="generating-a-user-access-token-with-postman">
  ### Generieren eines Benutzerzugangstokens mit Postman:
</div>

<div id="using-oauth-10a-to-generate-a-user-access-token">
  #### Verwenden von OAuth 1.0a zum Erzeugen eines Benutzerzugriffstokens
</div>

Sehen Sie sich den dreistufigen Prozess in der [OAuth 1.0a Flow-Testsammlung](https://www.postman.com/xapidevelopers/workspace/twitter-s-public-workspace/collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400?ctx=documentation) an.

<div id="using-oauth-20-to-generate-a-user-access-token">
  #### Verwenden von OAuth 2.0 zum Generieren eines Benutzerzugriffstokens
</div>

Wenn Sie in Postman ein OAuth-2.0-Benutzerzugriffstoken generieren möchten, können Sie OAuth-2.0-Zugriffstoken erstellen und zusammen mit der X‑[API v2 Postman Collection](https://www.postman.com/xapidevelopers/workspace/twitter-s-public-workspace/collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400?ctx=documentation) verwenden.

Klicken Sie in Ihrem Workspace auf die Collection, wechseln Sie zum Tab „Auth“ und wählen Sie als Typ „OAuth 2.0“ aus. Gehen Sie anschließend unter der Überschrift „Configure New Token“ zu „Configuration Options“. Dort können Sie den „Grant Type“ auf „Authorization Code (with PKCE)“ setzen.

Aktualisieren Sie Ihre Callback-URL so, dass sie der Callback-URL entspricht, die Ihrer Anwendung zugeordnet ist. Zusätzlich sollten Sie die folgenden Parameter anpassen:

- Auth URL – https://x.com/i/oauth2/authorize
- Access Token URL – https://api.x.com/2/oauth2/token
- Client ID – Ihre OAuth-2.0-Client-ID aus dem Dev-Portal
- Client Secret – falls Sie einen vertraulichen Client verwenden
- Scope – Scopes, die zu den Endpunkten passen, die Sie aufrufen möchten. Beispiel: „tweet.read users.read“
- Ihre Callback-URL (auch Redirect-URL genannt). Diese muss mit den Authentifizierungseinstellungen Ihrer App übereinstimmen.
- State – state

Wenn Sie bereit sind, klicken Sie auf „Get New Access Token“, um ein Zugriffstoken zu generieren. Wenn ein Dialogfeld erscheint, das meldet, dass etwas schiefgelaufen ist, müssen Sie möglicherweise die Zurück-Taste drücken, um sich anzumelden. Autorisieren Sie anschließend Ihre App, indem Sie im Dialogfeld auf „Authorize app“ klicken.

Nach der Autorisierung werden Sie zu Postman zurückgeleitet, wo Sie Ihr Token sehen und die Schaltfläche „Use Token“ auswählen können, um im Namen eines autorisierten Nutzers Anfragen zu stellen.

Sie können die Postman-Collection nun verwenden.

<div id="whats-next">
  ## Wie geht es weiter
</div>

Wenn Sie in Postman auf die Schaltfläche „Code“ klicken, können Sie die soeben erstellte Anfrage in eine Sprache Ihrer Wahl wie Python, Node oder Ruby umwandeln, um den Einstieg zu erleichtern. Postman verfügt über [ausführliche Dokumentation](https://learning.getpostman.com/), die hilfreich sein kann. Außerdem haben wir [Beispielcode auf GitHub](https://github.com/xdevplatform), der Ihnen hilft, sich schneller mit den Endpunkten zu integrieren.
