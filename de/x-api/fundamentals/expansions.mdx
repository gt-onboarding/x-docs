---
title: Erweiterungen
---

Mit Erweiterungen können Entwickler Objekte einbinden, auf die im Payload verwiesen wird. Erweiterbare Objekte werden über ihre ID referenziert. So können die Felder `referenced_tweets.id` und `author_id`, die im Payload der [Posts-Abfrage](/de/x-api/posts/lookup/introduction) zurückgegeben werden, zu vollständigen Objekten erweitert werden. Wenn Sie Felder zum Nutzer anfordern möchten, der den Post veröffentlicht hat, oder zu Medien, Umfragen (Polls) oder Orten, die in diesem Post enthalten sind, müssen Sie den entsprechenden Erweiterungs-Query-Parameter in Ihrer Anfrage übergeben, um diese Daten in der Antwort zu erhalten. Derzeit unterstützen v2-Endpunkte, die Posts, Users, Lists, Spaces und Direct-Message-Event-Objekte zurückgeben, alle Erweiterungen (siehe Beispiele unten).

Wenn Sie eine Erweiterung in Ihre Anfrage aufnehmen, fügen wir die Standardfelder des erweiterten Objekts in derselben Antwort bei. So erhalten Sie zusätzliche Daten in einer Antwort, ohne separate Anfragen stellen zu müssen. Wenn Sie zusätzliche [Felder](/de/x-api/fundamentals/fields) anfordern möchten, die sich auf das erweiterte Objekt beziehen, können Sie den entsprechenden Feldparameter für dieses erweiterte Objekt angeben, zusammen mit einer kommagetrennten Liste der Felder, die Sie in der Antwort erhalten möchten. Bitte beachten Sie, dass Felder nicht immer in der gleichen Reihenfolge zurückgegeben werden, in der sie in der Abfrage angefordert wurden.

Der Post-Payload unten enthält Referenz-IDs für ergänzende Objekte, die wir erweitern können, darunter die `author_id` des Nutzers, der den Post veröffentlicht hat, die `id` eines referenzierten Posts und ein `media_key` für einen Medienanhang.

```json
{
    "data": {
        "attachments": {
            "media_keys": ["16_1211797899316740096"]
        },
        "author_id": "2244994945",
        "id": "1212092628029698048",
        "referenced_tweets": [
            {
                "type": "replied_to",
                "id": "1212092627178287104"
            }
        ],
        "text": "We believe the best future version of our API will come from building it with YOU. Here’s to another great year with everyone who builds on the Twitter platform. We can’t wait to continue working with you in the new year. https://t.co/yvxdK6aOo2"
    }
}
```

<div id="available-expansions-for-post-payloads">
  ### Verfügbare Erweiterungen für Post-Payloads
</div>

| Erweiterung                    | Beschreibung                                                                        |
|:------------------------------|:-------------------------------------------------------------------------------------|
| `author_id`                    | Gibt ein Benutzerobjekt zurück, das den Autor des Posts repräsentiert               |
| `referenced_tweets.id`         | Gibt ein Post-Objekt zurück, auf das sich dieser Post bezieht (als Retweet, zitierter Tweet oder Antwort) |
| `edit_history_tweet_ids`       | Gibt Post-Objekte zurück, die Teil der Bearbeitungshistorie eines Posts sind        |
| `in_reply_to_user_id`          | Gibt ein Benutzerobjekt zurück, das den Autor des Posts repräsentiert, auf den sich der angeforderte Post als Antwort bezieht |
| `attachments.media_keys`       | Gibt ein Medienobjekt zurück, das die im Post enthaltenen Bilder, Videos und GIFs repräsentiert |
| `attachments.poll_ids`         | Gibt ein Umfrageobjekt zurück, das Metadaten zur im Post enthaltenen Umfrage enthält |
| `geo.place_id`                 | Gibt ein Ortsobjekt zurück, das Metadaten zum im Post markierten Standort enthält   |
| `entities.mentions.username`   | Gibt ein Benutzerobjekt für den im Post erwähnten Benutzer zurück                   |
| `referenced_tweets.id.author_id` | Gibt ein Benutzerobjekt für den Autor des referenzierten Posts zurück             |

<div id="available-expansion-for-user-payloads">
  ### Verfügbare Erweiterungen für Benutzer-Payloads
</div>

| Erweiterung          | Beschreibung                                               |
|:--------------------|:-----------------------------------------------------------|
| `pinned_tweet_id`  | Gibt ein Post-Objekt zurück, das den am Anfang des Benutzerprofils angehefteten Post repräsentiert |

<div id="available-expansions-for-direct-message-event-payloads">
  ### Verfügbare Expansions für Direct-Message-Event-Payloads
</div>

| Expansion                | Beschreibung                                                              |
|:--------------------------|:--------------------------------------------------------------------------|
| `attachments.media_keys` | Gibt ein Medienobjekt zurück, das an eine Direct Message angehängt wurde |
| `referenced_tweets.id`   | Gibt ein Post-Objekt zurück, das in einer Direct Message referenziert wurde |
| `sender_id`              | Gibt ein User-Objekt zurück, das den Autor einer Direct Message darstellt und angibt, wer einen Teilnehmer zu einem Gespräch eingeladen hat |
| `participant_ids`        | Gibt ein User-Objekt zurück, das einen Teilnehmer darstellt, der einem Gespräch beigetreten ist oder es verlassen hat |

<div id="available-expansions-for-spaces-payloads">
  ### Verfügbare Expansions für Spaces-Payloads
</div>

| Expansion          | Beschreibung                                               |
|:--------------------|:-----------------------------------------------------------|
| `invited_user_ids` | Gibt Benutzerobjekte zurück, die die eingeladenen Accounts repräsentieren |
| `speaker_ids`      | Gibt Benutzerobjekte zurück, die die Accounts repräsentieren, die während eines Space gesprochen haben |
| `creator_id`       | Gibt ein Benutzerobjekt zurück, das den Account repräsentiert, der den Space erstellt hat |
| `host_ids`         | Gibt Benutzerobjekte zurück, die die Accounts repräsentieren, die als Hosts festgelegt wurden |
| `topics_ids`       | Gibt Themenbeschreibungen zurück, die vom Ersteller festgelegt wurden |

<div id="available-expansion-for-lists-payloads">
  ### Verfügbare Erweiterungen für Listen-Payloads
</div>

| Erweiterung | Beschreibung                                                    |
|:-------------|:----------------------------------------------------------------|
| `owner_id`  | Gibt ein Benutzerobjekt zurück, das angibt, welches Konto die Liste erstellt und verwaltet |

<div id="expanding-the-media-object">
  ## Erweiterung des Media-Objekts
</div>

In der folgenden Anfrage verlangen wir die Expansion `geo.place_id`, damit sie neben den standardmäßigen Post-Feldern enthalten ist:

**Beispielanfrage**

```
{`curl 'https://api.x.com/2/tweets/:ID?expansions=geo.place_id' --header 'Authorization: Bearer $ACCESS_TOKEN'`}
```

**Beispielantwort**

```
{`{
  "data": {
      "geo": {
          "place_id": "01a9a39529b27f36"
      },
      "id": "ID",
      "text": "Test"
  },
  "includes": {
      "places": [
          {
              "full_name": "Manhattan, NY",
              "id": "01a9a39529b27f36"
          }
      ]
  }
}`}

```

<div id="expanding-the-poll-object">
  ## Erweiterung des Poll-Objekts
</div>

In der folgenden Anfrage fordern wir die Erweiterung `attachments.poll_ids` an, damit sie zusammen mit den Standard-Post-Feldern zurückgegeben wird:

**Beispielanfrage**

<CodeBlock language="bash">
  {`curl 'https://api.x.com/2/tweets/1199786642791452673?expansions=attachments.poll_ids' --header 'Authorization: Bearer $ACCESS_TOKEN'`}
</CodeBlock>

**Beispielantwort**

```
{`{
  "data": {
      "attachments": {
          "poll_ids": ["1199786642468413448"]
      },
      "id": "1199786642791452673",
      "text": "C#"
  },
  "includes": {
      "polls": [
          {
              "id": "1199786642468413448",
              "options": [
                  {
                      "position": 1,
                      "label": "„C Sharp“",
                      "votes": 795
                  },
                  {
                      "position": 2,
                      "label": "„C Hashtag“",
                      "votes": 156
                  }
              ]
          }
      ]
  }
}`}
```
