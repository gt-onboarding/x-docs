<div id="conversation-threads-using-the-x-api">
  ### Konversations-Threads mit der X-API
</div>

Wenn Sie betrachten, wie sich Konversationen auf X entwickeln, kann ein Post mehrere Konversations-Threads auslösen, von denen jeder an Länge und Komplexität zunimmt, wenn sich mehr Personen beteiligen. Das Erkennen von Beziehungen zwischen Posts und das Verständnis von Konversations-Threads ist eine Funktion der X-API-v2-Payloads und Suchfunktionen. Wenn Posts als Antwort auf einen Post (Reply) oder als Antwort auf eine Reply veröffentlicht werden, ist auf jeder Reply eine `conversation_id` gesetzt, die mit der Post-ID des ursprünglichen Posts übereinstimmt, der die Konversation gestartet hat.

Antworten auf einen bestimmten Post sowie Antworten auf diese Antworten sind alle Teil der Konversation, die von dem einzelnen ursprünglichen Post ausgeht. Unabhängig davon, wie viele Reply-Threads entstehen, teilen sie alle eine gemeinsame `conversation_id` mit dem ursprünglichen Post, der die Konversation ausgelöst hat. Mit der X-API v2 können Sie einen gesamten Konversations-Thread abrufen und rekonstruieren, sodass Sie nachvollziehen können, was gesagt wird und wie sich Konversationen und Ideen entwickeln.

Abrufen von `conversation_id` als `tweet.fields`-Parameter
Um die `conversation_id` für alle Posts anzufordern, die von einem v2-Endpunkt zurückgegeben werden, fügen Sie den Parameter tweet.fields=conversation\_id zur Anfrage hinzu. Das Feld conversation\_id ist stets die Post-ID des ursprünglichen Posts im Antwortthread der Unterhaltung. Alle Posts innerhalb desselben Antwortthreads, einschließlich solcher, die aus früheren Antwortthreads hervorgehen, weisen dieselbe conversation\_id auf.

<div id="request-with-conversation_id-parameter">
  ### Anfrage mit dem Parameter `conversation_id`
</div>

```
curl --request GET \
  --url 'https://api.x.com/2/tweets?ids=1225917697675886593&tweet.fields=author_id,conversation_id,created_at,in_reply_to_user_id,referenced_tweets&expansions=author_id,in_reply_to_user_id,referenced_tweets.id&user.fields=name,username' \
  --header 'Authorization: Bearer $ACCESS_TOKEN'
```

<div id="using-conversation_id-as-a-filter-operator">
  ### Verwendung von `conversation_id` als Filteroperator
</div>

Die `conversation_id` kann als Suchparameter in der jüngsten Suche oder als Operator innerhalb einer Regel für den [Gefilterten Stream](/de/x-api/posts/filtered-stream/introduction) verwendet werden. Dadurch wird der gesamte Konversations-Thread von Posts entweder in Echtzeit über den Gefilterten Stream oder in umgekehrt chronologischer Reihenfolge über [Search Tweets](/de/x-api/posts/search/introduction) zurückgegeben. Sie können außerdem die Anzahl der Posts in einer Konversation mit diesem Operator über [Posts counts](/de/x-api/posts/counts/introduction) abrufen.

<div id="request-to-query-by-conversation_id">
  ## Anfrage, um nach `conversation_id` zu filtern
</div>

```
curl --request GET \
  --url 'https://api.x.com/2/tweets/search/recent?query=conversation_id:1279940000004973111&tweet.fields=in_reply_to_user_id,author_id,created_at,conversation_id' \
  --header 'Authorization: Bearer $ACCESS_TOKEN'
```
