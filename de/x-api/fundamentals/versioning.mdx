<p>Zusammen mit der X-API v2 haben wir eine neue Versionierungsstrategie eingeführt, die Entwicklern dabei hilft, besser zu verstehen, wann Änderungen an den öffentlichen APIs von X zu erwarten sind und wann eine Migration auf neue Versionen erforderlich ist. </p>
<p>Entwickler werden über Außerdienststellungen, endgültige Stilllegungen, Änderungen und Erweiterungen der X-API über unsere <a href="https://developer.x.com/en/updates/stay-informed">Kommunikationskanäle</a> benachrichtigt, damit sie diese Änderungen in ihrer Entwicklungsroadmap entsprechend einplanen können. Alle Änderungen an der API werden im Changelog dokumentiert.</p>
<p>Die X-API hat derzeit drei verschiedene Versionen. Wir empfehlen Nutzern nachdrücklich, bei der Planung ihrer Integration die X-API v2 zu verwenden, es sei denn, die für Ihren Anwendungsfall erforderliche Funktionalität ist in v2 noch nicht verfügbar. </p>
<p>Weitere Informationen zu den einzelnen Versionen finden Sie auf den folgenden Seiten:</p>

<ul>
  <li><a href="/de/x-api/introduction">X-API v2</a></li>
  <li>Gnip 2.0 (<a href="/de/x-api/enterprise-gnip-2.0/enterprise-gnip">Enterprise</a>)</li>
</ul>

<div id="versioning-strategy">
  ## Versionierungsstrategie
</div>

<p>Die Versionierung der X-API wird durch Versionsnummern dargestellt, die im Routenpfad unserer Endpunkte angegeben sind:</p>
<p>`https://api.x.com/2/tweets`</p>
<p>Wir beabsichtigen, Hauptversionen der X-API nach Bedarf, jedoch nicht häufiger als alle 12 Monate, zu veröffentlichen. Eine Hauptversion wird veröffentlicht, wenn in der API Breaking Changes eingeführt werden. Beim Start einer neuen Hauptversion veröffentlichen wir Migrationsleitfäden, um Entwickler beim Umstieg auf die neue Version zu unterstützen. </p>
<p>Ein Breaking Change erfordert, dass Entwickler ihren Code ändern, um die bestehende Funktionalität in ihrer App beizubehalten. Nicht-breaking Änderungen sind additive Erweiterungen und werden bei Verfügbarkeit in die aktuelle Version ausgerollt, ohne dass Arbeiten auf Seiten der Entwickler erforderlich sind, es sei denn, Sie möchten die neue Funktionalität nutzen.</p>
<p>Wenn ein Breaking Change während eines Zyklus ausgerollt werden muss (aus Sicherheits- oder Datenschutzgründen), wird diese Änderung in der aktuellen Version vorgenommen.</p>

<div id="breaking-changes">
  ### Breaking Changes
</div>

<p>Diese Änderungen erfordern, dass Entwickler ihren Code anpassen, um die bestehende Funktionalität ihrer Anwendung beizubehalten.</p>

<ul>
  <li>Hinzufügen eines neuen erforderlichen Parameters</li>
  <li>Entfernung eines vorhandenen Endpoints</li>
  <li>Entfernung eines beliebigen Felds in der Antwort (ob erforderlich oder optional)</li>
  <li>Entfernung eines Query-Parameters</li>
  <li>Umstrukturierung des Eingabe- oder Ausgabeformats (zum Beispiel das Verschieben eines Top-Level-Felds in ein Unterfeld oder das Inline-Verschieben von Fehlermeldungen)</li>
  <li>Änderung des Namens oder Datentyps eines vorhandenen Eingabeparameters oder Ausgabewerts</li>
  <li>Änderung des Namens eines Felds</li>
  <li>Änderung des Ressourcennamens</li>
  <li>Änderung eines Antwortcodes</li>
  <li>Änderung von Fehlertypen</li>
  <li>Änderungen an bestehenden Autorisierungs-Scopes</li>
</ul>

<div id="non-breaking-changes">
  ### Rückwärtskompatible Änderungen
</div>

<ul>
  <li>Hinzufügen eines neuen Endpunkts</li>
  <li>Hinzufügen eines neuen optionalen Parameters</li>
  <li>Hinzufügen eines neuen Antwortfelds</li>
  <li>Änderung von Fehlermeldungstexten</li>
  <li>Verfügbarkeit neuer Scopes</li>
  <li>„Nullen“ von Feldern (Setzen des Werts eines Felds auf null aus Gründen des Datenschutzes/der Sicherheit als Alternative zur vollständigen Entfernung)</li>
</ul>

<div id="deprecation-and-retirement">
  ### Außerkraftsetzung und Außerdienststellung
</div>

<p>Zunächst einmal unsere Definition dessen, was Außerkraftsetzung und Außerdienststellung für die X-API bedeuten:</p>

<ul>
  <li><b>Außerkraftsetzung:</b> Die Funktion wird vom Team nicht mehr unterstützt. Es wird keine neue Funktionalität für diese Funktion veröffentlicht, und falls es Fehler oder Probleme mit dem Produkt gibt, ist die Wahrscheinlichkeit, dass wir eine Korrektur in Betracht ziehen, äußerst gering. </li>
  <li><b>Außerdienststellung:</b> Die Funktion ist nicht mehr zugänglich.</li>
</ul>

<p>In den meisten Fällen wird, sobald eine neue Version veröffentlicht ist, die vorherige Version als außer Kraft gesetzt markiert. Versionen verbleiben für einen bestimmten Zeitraum in diesem Status und werden anschließend außer Dienst gestellt. </p>
<p>Bitte <a href="https://developer.x.com/en/updates/stay-informed">bleiben Sie informiert</a>, um mehr über zukünftige Außerkraftsetzungen und Außerdienststellungen zu erfahren.</p>
