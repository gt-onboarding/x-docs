<div id="comparing-x-apis-manage-posts-endpoints">
  ## Vergleich der „Posts verwalten“-Endpunkte der X-API
</div>

Die v2-Endpunkte zum Verwalten von Posts ersetzen die Standard-v1.1-Endpunkte [POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) und [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id). Wenn Sie Code, Apps oder Tools haben, die die v1.1-Version dieser Endpunkte verwenden und eine Migration auf den neueren X-API-v2-Endpunkt in Erwägung ziehen, sind diese Leitfäden für Sie gedacht.

Die folgenden Tabellen vergleichen die Standard-v1.1-Endpunkte mit den X-API-v2-Endpunkten zum Verwalten von Posts:

<div id="create-a-post">
  ### Einen Post erstellen
</div>

| Beschreibung | Standard v1.1 | X-API v2 |
| :--- | :--- | :--- |
| Unterstützte HTTP-Methoden | POST | POST |
| Host-Domain | https://api.x.com | https://api.x.com |
| Endpoint-Pfad | /1.1/statuses/update.json | /2/tweets |
| [Authentifizierung](/de/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Autorisierungscode mit PKCE |
| Standard-[Rate-Limits](/de/x-api/fundamentals/rate-limits) für Anfragen | Keine<br /><br />300 Anfragen pro 3‑Stunden‑Fenster pro Nutzer und App. Gemeinsam mit dem v1.1‑POST‑Retweets‑Endpoint. | 200 Anfragen pro 15 Minuten pro Nutzer<br /><br />300 Anfragen pro 3‑Stunden‑Fenster pro Nutzer und App. Gemeinsam mit dem v2‑POST‑Retweets‑Endpoint. |
| Erfordert die Verwendung von Anmeldedaten aus einer [Developer App](/de/resources/fundamentals/developer-apps), die einem [Project](/de/resources/fundamentals/projects) zugeordnet ist |     | ✔   |

<div id="delete-a-post">
  ### Einen Post löschen
</div>

| Beschreibung | Standard v1.1 | X-API v2 |
| :--- | :--- | :--- |
| Unterstützte HTTP-Methoden | POST | DELETE |
| Host-Domain | https://api.x.com | https://api.x.com |
| Endpunkt-Pfad | /1.1/statuses/destroy/:id.json | /2/tweets/:id |
| [Authentifizierung](/de/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code mit PKCE |
| Standardmäßige [Rate Limits](/de/x-api/fundamentals/rate-limits) für Anfragen | Keine | 50 Anfragen pro 15 Min. pro Nutzer |
| Erfordert die Verwendung von Zugangsdaten aus einer [Developer App](/de/resources/fundamentals/developer-apps), die einem [Project](/de/resources/fundamentals/projects) zugeordnet ist |     | ✔   |
