<div id="standard-v11-compared-to-x-api-v2">
  ## Standard v1.1 im Vergleich zur X-API v2
</div>

Wenn Sie mit den Standard-v1.1-Endpunkten [POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) und [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id) gearbeitet haben, soll Ihnen dieser Leitfaden helfen, die Gemeinsamkeiten und Unterschiede zwischen den Standard- und den X-API-v2-Endpunkten zum Verwalten von Posts zu verstehen.

- **Gemeinsamkeiten**
  - Authentifizierung
- **Unterschiede**
  - Endpoint-URLs

  - Anforderungen für Apps und Projekte

  - Anfrageparameter

<div id="similarities">
  ### Gemeinsamkeiten
</div>

**Authentifizierung**

Sowohl die Standard-v1.1- als auch die X‑API‑v2‑Endpunkte zur Verwaltung von Posts ([POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) und [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id)) verwenden [OAuth 1.0a User Context](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Wenn Sie zuvor einen der Standard‑v1.1‑Endpunkte verwendet haben, können Sie bei der Migration auf die X‑API‑v2‑Version weiterhin dieselbe Authentifizierungsmethode verwenden. 

<div id="differences">
  ### Unterschiede
</div>

**Endpoint-URLs**

- Standard-v1.1-Endpunkte:
  - https://api.x.com/1.1/statuses/update.json
    (Erstellt einen Post)
  - `https://api.x.com/1.1/statuses/destroy/:id.json`
    (Löscht einen Post)
- X-API-v2-Endpoint:
  - https://api.x.com/2/tweets
    (Erstellt einen Post)
  - https://api.x.com/2/tweets/:id
    (Löscht einen bestimmten Post)

<div id="app-and-project-requirements">
  ### Anforderungen an Apps und Projekte
</div>

Die X-API-v2-Endpunkte erfordern bei der Authentifizierung Ihrer Anfragen die Verwendung von Anmeldedaten aus einer [Developer-App](/de/resources/fundamentals/developer-apps), die einem [Projekt](/de/resources/fundamentals/developer-apps) zugeordnet ist. Alle X-API-v1.1-Endpunkte können Anmeldedaten aus eigenständigen Apps oder aus Apps verwenden, die einem Projekt zugeordnet sind.

<div id="request-parameters">
  ### Anfrageparameter
</div>

Die folgenden Standardparameter in v1.1 akzeptierten zwei Abfrageparameter (user\_id oder screen\_name). Die X-API v2 akzeptiert für den DELETE-Endpunkt nur die numerische Post-ID, die als Teil des Endpunktpfads übergeben werden muss.

Für den POST-Endpunkt müssen zusätzliche Parameter im JSON-Body der Anfrage übergeben werden. Weitere Informationen zu den verfügbaren Parametern finden Sie im [API-Referenzhandbuch](/de/x-api/posts/manage-tweets/introduction).
