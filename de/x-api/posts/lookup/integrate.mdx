<div id="helpful-tools">
  ## Nützliche Tools
</div>

Bevor wir in einige grundlegende Konzepte einsteigen, empfehlen wir, sich mit den folgenden Tools vertraut zu machen:

**Postman**\
Postman ist ein hervorragendes Tool, um Endpunkte zu testen – einschließlich aller Pfad- und Body-Parameter –, damit Sie verstehen, welche Optionen verfügbar sind. Lesen Sie unseren [Leitfaden für den Einstieg mit Postman](/de/tutorials/postman-getting-started), um mehr zu erfahren.

**Codebeispiele**\
Finden Sie Codebeispiele für Ihre bevorzugte Programmiersprache auf unserer [GitHub-Seite](https://github.com/xdevplatform/Twitter-API-v2-sample-code).

**Bibliotheken von Drittanbietern**\
Nutzen Sie Community-gestützte [Bibliotheken von Drittanbietern](/de/resources/tools-and-libraries), die mit v2-Endpunkten kompatibel sind.

<div id="authentication">
  ### Authentifizierung
</div>

Alle X-API v2-Endpunkte erfordern authentifizierte Anfragen. Sie können sich authentifizieren mit:

- [OAuth 1.0a User Context](/de/resources/fundamentals/authentication) unter Verwendung von API-Schlüsseln, Zugriffstokens und zusätzlichen Parametern, um [einen Authorization-Header zu erstellen](/de/resources/fundamentals/authentication#authorizing-a-request).
- [OAuth 2.0 App-Only](/de/resources/fundamentals/authentication#oauth-2-0), indem Sie ein [App Access Token](/de/resources/fundamentals/authentication) mit Ihrer Anfrage übermitteln.
- [OAuth 2.0 Authorization Code with PKCE](/de/resources/fundamentals/authentication) für mehr Kontrolle über den App-Scope und die Autorisierung über mehrere Geräte.

<Note>
  OAuth 1.0a kann in der Implementierung herausfordernd sein. Falls Sie damit nicht vertraut sind, ziehen Sie die Verwendung einer [Bibliothek](/de/resources/tools-and-libraries) oder von OAuth 2.0 für Anfragen in Betracht. Für private Metriken oder Posts verwenden Sie OAuth 1.0a User Context oder OAuth 2.0 Authorization Code with PKCE.
</Note>

<Tip>
  **Bitte beachten**

  > Wenn Sie die folgenden Felder anfordern, ist OAuth 1.0a User Context oder OAuth 2.0 Authorization Code erforderlich:
  >
  > - `tweet.fields.non_public_metrics`
  > - `tweet.fields.promoted_metrics`
  > - `tweet.fields.organic_metrics`
  > - `media.fields.non_public_metrics`
  > - `media.fields.promoted_metrics`
  > - `media.fields.organic_metrics`
</Tip>

<div id="developer-portal-projects-and-apps">
  ### Developer-Portal, Projekte und Apps
</div>

Um Zugangsdaten für die X-API v2 zu erhalten, benötigen Sie:

1. Ein genehmigtes [Entwicklerkonto](/de/resources/fundamentals/developer-portal).
2. Ein [Projekt](/de/resources/fundamentals/projects) innerhalb dieses Entwicklerkontos.
3. Eine [Entwickler-App](/de/resources/fundamentals/developer-apps) innerhalb dieses Projekts, in der Schlüssel und Token zu finden sind.

<div id="rate-limits">
  ### Rate Limits
</div>

Anfragen an die X-API unterliegen [Rate Limits](/de/resources/fundamentals/rate-limits), um das Volumen zu steuern. Limits gelten sowohl auf App- als auch auf Benutzer-Ebene:

- **App-Ebene**: Begrenzt die Anzahl der Anfragen pro Zeitraum, die von einer beliebigen App gestellt werden.
- **Benutzer-Ebene**: Begrenzt, wie häufig ein authentifizierter Benutzer Post-Abfragen über Entwickler-Apps durchführen kann.

<div id="fields-and-expansions">
  ### Felder und Erweiterungen
</div>

Die X-API v2 ermöglicht die Auswahl bestimmter Datenfelder mit `fields` und `expansions`:

- **Expansions**: Ermöglichen das Abrufen zusätzlicher, verknüpfter Objekte. Unterstützte Expansions sind:
  - `edit_history_tweet_ids`
  - `attachments.poll_ids`
  - `attachments.media_keys`
  - `author_id`
  - `entities.mentions.username`
  - `geo.place_id`
  - `in_reply_to_user_id`
  - `referenced_tweets.id`
  - `referenced_tweets.id.author_id`

- **Fields**: Geben Datenfelder innerhalb von Objekten an, um zusätzliche Informationen zurückzugeben. Das Post-Objekt umfasst standardmäßig `id`, `text` und `edit_history_tweet_ids`. Weitere Felder wie `tweet.created_at` und `tweet.entities` müssen explizit angefordert werden.

Weitere Informationen finden Sie im Leitfaden zu Feldern und Erweiterungen im [X-API-v2-Datenwörterbuch](/de/x-api/fundamentals/data-dictionary).

<div id="post-edits">
  ### Bearbeitung von Posts
</div>

Berechtigte Posts können bis zu fünfmal innerhalb von 30 Minuten nach der Veröffentlichung bearbeitet werden. Der Posts-Lookup-Endpunkt liefert stets die neueste Version eines Posts. Für nahezu Echtzeit‑Anwendungsfälle beachten Sie dieses Zeitfenster. Weitere Details finden Sie unter [Grundlagen zur Bearbeitung von Posts](/de/x-api/fundamentals/edit-posts).

<div id="edge-cases">
  ### Randfälle
</div>

- **Beworbene Metriken**: Das Anfordern beworbener Metriken für nicht beworbene Posts liefert eine leere Antwort.
- **Abgeschnittener Text**: Post-Text wird bei Retweets abgeschnitten. Um den vollständigen Text abzurufen, erweitern Sie den referenzierten Post.
