<div id="getting-started-with-the-posts-lookup-endpoints">
  ## Einstieg in die Endpunkte zur Beitragsabfrage
</div>

Diese Schnellstartanleitung hilft Ihnen, Ihre erste Anfrage mit angegebenen Feldern an die Endpunkte zur Beitragsabfrage mit Postman zu stellen.

Wenn Sie Beispielcode in verschiedenen Sprachen sehen möchten, besuchen Sie bitte unser GitHub-Repository mit [Beispielcode für die X-API v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code).

<Note>
  ### Voraussetzungen

  Um diese Anleitung abzuschließen, benötigen Sie einen Satz von [Keys und Tokens](/de/resources/fundamentals/authentication), um Ihre Anfrage zu authentifizieren. Sie können diese Keys und Tokens generieren, indem Sie die folgenden Schritte ausführen:

  - [Registrieren Sie sich für ein Entwicklerkonto](https://developer.x.com/en/portal/petition/essential/basic-info) und lassen Sie es genehmigen.
  - Erstellen Sie ein [Projekt](/de/resources/fundamentals/projects) und eine zugehörige [Developer-App](/de/resources/fundamentals/developer-apps) im Developer-Portal.
  - Navigieren Sie zur Seite „Keys and Tokens“ Ihrer App, um die erforderlichen Zugangsdaten zu generieren. Stellen Sie sicher, dass Sie alle Zugangsdaten an einem sicheren Ort aufbewahren.
</Note>

<div id="steps-to-build-a-get-tweets-request">
  ### Schritte zum Erstellen einer GET-/tweets-Anforderung
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### Schritt eins: Beginnen Sie mit einem Tool oder einer Bibliothek
</div>

Es gibt verschiedene Tools, Codebeispiele und Bibliotheken, mit denen Sie eine Anfrage an diesen Endpoint stellen können. Wir verwenden hier Postman, um den Prozess zu vereinfachen.

Um die Postman-Collection für die X-API v2 in Ihre Umgebung zu laden, klicken Sie auf die Schaltfläche unten:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X-API v2 zu Postman hinzufügen
</Button>

Sobald Sie die X-API-v2-Collection in Postman geladen haben, navigieren Sie zur Anfrage „Post Lookup > Multiple Posts“.

<div id="step-two-authenticate-your-request">
  #### Schritt zwei: Authentifizieren Sie Ihre Anfrage
</div>

Um eine Anfrage an die X-API korrekt zu stellen, müssen Sie nachweisen, dass Sie berechtigt sind. Sie können Ihre Anfrage authentifizieren mit:

- [OAuth 2.0 App-Only](de/resources/fundamentals/authentication)
- [OAuth 2.0 Authorization Code mit PKCE](de/resources/fundamentals/authentication)
- [OAuth 1.0a User Context](de/resources/fundamentals/authentication)

Der Einfachheit halber verwenden wir für diese Anfrage OAuth 2.0 App-Only. Wenn Sie jedoch private [Metriken](/de/x-api/fundamentals/metrics) oder Posts anfordern möchten, müssen Sie eine der anderen Authentifizierungsmethoden nutzen.

Um OAuth 2.0 App-Only zu nutzen, müssen Sie Ihre Keys und Tokens (insbesondere das App Access Token, auch App-only Bearer Token genannt) in Postman hinzufügen, indem Sie die Umgebung mit dem Namen „X API v2“ (oben rechts in Postman) auswählen und Ihre Keys und Tokens in die Felder „initial value“ und „current value“ eintragen (klicken Sie dazu auf das Augensymbol neben der Umgebungs-Dropdown-Liste).

Wenn Sie dies korrekt eingerichtet haben, werden diese Variablen automatisch in den Autorisierungs-Tab der Anfrage übernommen.

<div id="step-three-identify-and-specify-which-posts-you-would-like-to-retrieve">
  #### Schritt drei: Festlegen, welche Posts Sie abrufen möchten
</div>

Sie müssen in der Anfrage einen Post oder eine Gruppe von Posts angeben, die Sie abrufen möchten. Sie finden die Post-ID, indem Sie zu X.com navigieren, auf einen Post klicken und dann in der URL nachsehen. In der folgenden URL ist die Post-ID zum Beispiel `1228393702244134912`.

`https://x.com/TwitterDev/status/1228393702244134912`

Navigieren Sie in Postman zur Registerkarte „Params“ und geben Sie diese ID oder eine durch Kommas getrennte Liste von Post-IDs in die Spalte „Value“ des Parameters ids ein.

| Key  | Value |
| :---- | :----- |
| `ids` | `1228393702244134912,1227640996038684673,1199786642791452673` |

---

<div id="step-four-identify-and-specify-which-fields-you-would-like-to-retrieve">
  #### Schritt vier: Festlegen, welche Felder Sie abrufen möchten
</div>

Wenn Sie nach Schritt drei auf die Schaltfläche „Send“ klicken, erhalten Sie die Standard-[Post-Objektfelder](/de/x-api/fundamentals/data-dictionary) in Ihrer Antwort: `id`, `text` und `edit_history_tweet_ids`.

Für diese Übung fordern wir drei zusätzliche, unterschiedliche Feldsätze aus verschiedenen Objekten an:

- Das zusätzliche Feld `tweet.created_at` in den primären Post-Objekten.
- Die Standardfelder der Benutzerobjekte der zugehörigen Autoren für die zurückgegebenen Posts: `id`, `name` und `username`
- Das zusätzliche Feld `user.created_at` in den zugehörigen Benutzerobjekten.

Navigieren Sie in Postman zur Registerkarte „Params“ und fügen Sie das folgende `key:value`-Paar zur Tabelle „Query Params“ hinzu:
| Key            | Value               | Returned fields                          |
| :-------------- | :------------------- | :---------------------------------------- |
| `tweet.fields` | `created_at`        | `tweets.created_at`                      |
| `expansions`   | `author_id`         | `includes.users.id`, `includes.users.name`, `includes.users.username` |
| `user.fields`  | `created_at`        | `includes.users.created_at`              |

Neben der Schaltfläche „Send“ sollten Sie diese URL sehen:

```
https://api.x.com/2/tweets?ids=1228393702244134912,1227640996038684673,1199786642791452673&tweet.fields=created_at&expansions=author_id&user.fields=created_at
```
