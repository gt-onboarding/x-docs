<div id="postman">
  #### Postman
</div>

Postman ist ein hervorragendes Tool, um einen Endpoint zu testen. Jede Postman-Anfrage enthält sämtliche Pfad- und Body-Parameter, sodass Sie schnell verstehen, welche Optionen Ihnen zur Verfügung stehen. Weitere Informationen zu unseren Postman-Collections finden Sie auf der Seite ["Using Postman"](/de/tutorials/postman-getting-started).

<div id="authentication">
  #### Authentifizierung
</div>

Alle Endpunkte der X-API v2 erfordern, dass Sie Ihre Anfragen mit einem Satz von Zugangsdaten authentifizieren, auch bekannt als Keys und Tokens.

Diese Endpunkte erfordern die Verwendung von [OAuth 2.0 mit PKCE (Authorization Code Flow)](/de/resources/fundamentals/authentication/oauth-2-0/authorization-code). Das bedeutet, dass Sie einen Satz von Keys und Benutzer-Access-Tokens verwenden müssen, um eine Anfrage erfolgreich auszuführen. Die Access-Tokens müssen dem Nutzer zugeordnet sein, in dessen Namen Sie die Anfrage stellen. Wenn Sie Access-Tokens für einen anderen Nutzer generieren möchten, muss dieser Ihre App über eine [Authorize-URL](/de/resources/fundamentals/authentication#authorize-url) autorisieren bzw. sich authentifizieren.

Bitte beachten Sie, dass OAuth 2.0 in der Praxis knifflig sein kann. Wenn Sie mit dieser Authentifizierungsmethode nicht vertraut sind, empfehlen wir die Verwendung einer [Library](/de/x-api/tools-and-libraries/overview) oder eines Tools wie Postman, um Ihre Anfragen korrekt zu authentifizieren.

<div id="developer-portal-projects-and-developer-apps">
  #### Entwicklerportal, Projekte und Entwickler-Apps
</div>

Um einen Satz von Anmeldedaten zu erhalten, der mit den Endpunkten der X-API v2 funktioniert, müssen Sie ein [Entwicklerkonto](/de/resources/fundamentals/developer-portal) besitzen, darin ein [Projekt](/de/resources/fundamentals/projects) einrichten und innerhalb dieses Projekts eine [Entwickler-App](/de/resources/fundamentals/developer-apps) erstellen. Anschließend finden Sie Ihre Schlüssel und Token in Ihrer Entwickler-App. 

<div id="rate-limits">
  #### Rate Limits
</div>

Jeden Tag stellen viele Tausend Entwickler Anfragen an die X-API. Um das hohe Anfrageaufkommen zu steuern, gelten für jeden Endpoint [Rate Limits](/de/resources/fundamentals/rate-limits), die die Anzahl der Anfragen begrenzen, die Sie im Namen Ihrer App oder eines authentifizierten Nutzers stellen können.

Diese Endpoints sind auf Nutzerebene begrenzt. Das bedeutet, dass der authentifizierte Nutzer, in dessen Namen Sie die Anfrage stellen, den Endpoint nur eine bestimmte Anzahl Mal über beliebige Developer-Apps aufrufen kann. Es gibt ein Nutzer-Rate-Limit von 180 Anfragen pro 15‑Minuten‑Fenster für die GET-Methode. Mit der GET-Methode des Bookmarks-Lookup-Endpoints erhalten Sie bis zu 800 Ihrer zuletzt mit Lesezeichen versehenen Posts. Zusätzlich gibt es ein Nutzer-Rate-Limit von 50 Anfragen pro 15 Minuten für die Methoden POST und DELETE.
