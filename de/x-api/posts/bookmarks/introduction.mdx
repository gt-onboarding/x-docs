<div id="manage-bookmarks">
  ### Lesezeichen verwalten
</div>

Wir bieten zwei Methoden zum Verwalten von Lesezeichen an: POST und DELETE. Mit der POST-Methode können Sie Lesezeichen erstellen. Mit der DELETE-Methode können Sie ein bestimmtes Lesezeichen löschen.

Für die Methoden POST und DELETE gilt ein nutzerbezogenes Ratenlimit von 50 Anfragen pro 15 Minuten.

Da Sie Anfragen im Namen eines Nutzers an die Endpunkte zum Verwalten von Lesezeichen stellen, müssen Sie sich authentifizieren, indem Sie ein Nutzerzugriffstoken mit OAuth 2.0 generieren. Sie können dazu den [Authorization Code with PKCE grant flow](/de/resources/fundamentals/authentication/oauth-2-0/user-access-token) verwenden. Um diesen Endpunkt zu nutzen, müssen Sie die Scopes `tweet.read`, `users.read` und `bookmark.write` übergeben.

<div id="bookmarks-lookup">
  ### Lesezeichenabfrage
</div>

Der Endpunkt für die Lesezeichenabfrage stellt eine Methode bereit: GET. Mit dieser Methode können Sie Lesezeichen von sich selbst oder einem authentifizierten Konto abrufen. Für das Durchblättern großer Ergebnismengen werden Paginierungs-Token bereitgestellt.

Für die GET-Methode gilt ein nutzerbasiertes Ratenlimit von 180 Anfragen pro 15-Minuten-Fenster. Über diesen Endpunkt erhalten Sie bis zu 800 Ihrer zuletzt mit Lesezeichen versehenen Posts.

Da Sie bei den Lesezeichenabfrage-Endpunkten im Namen eines Nutzers Anfragen stellen, müssen Sie sich authentifizieren, indem Sie mit OAuth 2.0 ein Benutzerzugriffstoken generieren. Sie können dafür den [Authorization Code with PKCE Grant Flow](/de/resources/fundamentals/authentication/oauth-2-0/user-access-token) verwenden. Um diesen Endpunkt zu nutzen, müssen Sie die Scopes `tweet.read`, `users.read` und `bookmark.read` übergeben.

**Kontoeinrichtung**

Um auf diese Endpunkte zuzugreifen, benötigen Sie:

- Ein genehmigtes [Developer-Konto](https://developer.x.com/en/portal/petition/essential/basic-info).
- Eine Authentifizierung mit den Keys und Tokens aus einer [Developer-App](/de/resources/fundamentals/developer-apps), die sich innerhalb eines [Projekts](/de/resources/fundamentals/projects) befindet.

Erfahren Sie mehr über den Zugriff auf die Endpunkte der X-API v2 in unserem [Getting-Started-Leitfaden](/de/x-api/getting-started/getting-access).
