<div id="getting-started-with-the-manage-bookmarks-endpoints">
  ## Erste Schritte mit den Endpunkten zum Verwalten von Lesezeichen
</div>

Diese Schnellstartanleitung hilft Ihnen, Ihre erste Anfrage an die Endpunkte zum Verwalten von Lesezeichen mit [Postman](/de/tutorials/postman-getting-started) zu stellen.

Wenn Sie Beispielcode in verschiedenen Sprachen sehen möchten, besuchen Sie unser GitHub-Repository mit [Beispielcode für die X-API v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code).

<Note>
  **Voraussetzungen**

  Um diese Anleitung abzuschließen, benötigen Sie einen Satz von [Keys und Tokens](/de/resources/fundamentals/authentication), um Ihre Anfrage zu authentifizieren. Sie können diese Keys und Tokens generieren, indem Sie die folgenden Schritte ausführen:

  - [Registrieren Sie sich für ein Entwicklerkonto](https://developer.x.com/en/apply-for-access) und lassen Sie es genehmigen.
  - Erstellen Sie ein [Projekt](/de/resources/fundamentals/projects) und eine zugehörige [Developer App](/de/resources/fundamentals/developer-apps) im Developer-Portal.
  - Navigieren Sie zur Seite „Keys and tokens“ Ihrer App, um die erforderlichen Zugangsdaten zu generieren. Speichern Sie alle Zugangsdaten an einem sicheren Ort.
</Note>

<div id="steps-to-build-a-manage-bookmarks-request">
  ### Schritte zum Erstellen einer Anfrage zum Verwalten von Lesezeichen
</div>

**Schritt eins: Beginnen Sie mit einem Tool oder einer Bibliothek**

Es gibt verschiedene Tools, Codebeispiele und Bibliotheken, mit denen Sie eine Anfrage an diesen Endpunkt stellen können. Wir verwenden hier Postman, um den Prozess zu vereinfachen.

Um die X-API v2 Postman-Collection in Ihre Umgebung zu laden, klicken Sie bitte auf die folgende Schaltfläche:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X-API v2 zu Postman hinzufügen
</Button>

Sobald Sie die X-API v2 Collection in Postman geladen haben, navigieren Sie zum Ordner „Bookmarks“ und wählen Sie „Create a Bookmark“.

**Schritt zwei: Authentifizieren Sie Ihre Anfrage**

Für eine erfolgreiche Anfrage an diesen Endpunkt müssen Sie [OAuth 2.0 Authorization Code Flow with PKCE](/de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) verwenden. Sie können in Postman ein Zugriffstoken generieren.

Gehen Sie zur Registerkarte „Authorization“ und wählen Sie „OAuth 2.0“ aus.

Befolgen Sie in dieser Registerkarte diese Schritte:

1. Benennen Sie Ihr Token.

2. Wählen Sie als Grant Type „Authorization Code (with PKCE)“.

3. Aktualisieren Sie die Parameter:

   **Callback URL** – [https://www.example.com](https://www.example.com/)

   Diese sollte der Callback-URL entsprechen, die Sie auf Ihrer Seite mit den Auth-Einstellungen im Developer Portal festgelegt haben.

   **Auth URL** – [https://x.com/i/oauth2/authorize](https://x.com/i/oauth2/authorize)

   **Access Token URL** – [https://api.x.com/2/oauth2/token](https://api.x.com/2/oauth2/token)

   **Client ID** – Kopieren Sie die OAuth 2.0 Client-ID aus dem Developer Portal.

   **Client Secret** – Kopieren Sie das OAuth 2.0 Client Secret aus dem Developer Portal. Sie benötigen dies nur, wenn Sie einen App-Typ verwenden, der ein Confidential Client ist.

4. Aktualisieren Sie die Scopes mit den folgenden Werten: post.read users.read bookmark.write

5. Füllen Sie das Feld state mit „State“.

6. Klicken Sie auf „Generate Token“.

7. Klicken Sie auf das Speichersymbol, um die Ordneränderungen zu speichern.

Möglicherweise erhalten Sie eine Meldung, dass Sie nicht bei X angemeldet sind. Wenn dieser Fehler auftritt, melden Sie sich in dem X-Konto innerhalb von Postman an, in dessen Namen Sie posten möchten.

**Schritt drei: Geben Sie einen Nutzer an**

Bei diesem Endpunkt müssen Sie die Nutzer-ID angeben, deren Follower Sie in der Antwort erhalten möchten. Beispielsweise lautet die Nutzer-ID für @XDevelopers `2244994945`.

Navigieren Sie in Postman zur Registerkarte „Params“ und geben Sie Ihre eigene ID oder die eines authentifizierten Nutzers als Wert für den Parameter „id“ ein.

|     |     |     |
| :--- | :--- | :--- |
|     | Key | Value |
|     | id  | `2244994945` |

**Schritt vier: Geben Sie einen Post an, den Sie mit einem Lesezeichen versehen möchten**

Wechseln Sie zur Registerkarte „Body“ und stellen Sie sicher, dass die Post-ID auf den Beitrag gesetzt ist, den Sie in Ihren Lesezeichen speichern möchten. Die JSON-Nutzlast sollte ähnlich wie unten aussehen.

```
      {"tweet_id": "1460323737035677698"}
```

**Schritt fünf: Senden Sie Ihre Anfrage und überprüfen Sie Ihre Antwort**

Sobald alles eingerichtet ist, klicken Sie auf die Schaltfläche „Send“, und Sie erhalten eine ähnliche Antwort wie im folgenden Beispiel:

```
{
   "data": {
       "bookmarked": true
   }
}
```

Um einen Post zu löschen, wählen Sie die Anfrage „Remove a Bookmark“, die sich ebenfalls im Ordner „Bookmarks“ der in Postman geladenen X-API v2 Collection befindet. Zuerst geben Sie die Nutzer-ID des Nutzers an, in dessen Namen Sie die Anfrage stellen, als Wert für die Spalte „id“. Dieser Endpunkt erfordert außerdem die ID des Posts, den Sie löschen möchten. Geben Sie dann in der Registerkarte „Params“ die ID des Posts, den Sie löschen möchten, als Wert für die Spalte „tweet\_id“ ein.

Wenn Ihre Löschanfrage erfolgreich ist, erhalten Sie eine Antwort ähnlich dem folgenden Beispiel:

```
{
   "data": {
       "bookmarked": false
   }
}
```
