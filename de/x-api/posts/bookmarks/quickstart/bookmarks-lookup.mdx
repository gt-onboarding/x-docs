<div id="getting-started-with-the-bookmarks-lookup-endpoint">
  ### Erste Schritte mit dem Bookmarks-Lookup-Endpunkt
</div>

Diese Schnellstartanleitung hilft Ihnen, Ihre erste Anfrage an den Bookmarks-Lookup-Endpunkt mit [Postman](/de/tutorials/postman-getting-started) zu senden.

Besuchen Sie unser GitHub-Repository mit [X-API v2-Beispielcode](https://github.com/xdevplatform/Twitter-API-v2-sample-code), wenn Sie Beispielcode in verschiedenen Sprachen sehen möchten.

<div id="prerequisites">
  #### Voraussetzungen
</div>

Um diese Anleitung abzuschließen, benötigen Sie einen Satz [Keys und Tokens](/de/resources/fundamentals/authentication), um Ihre Anfrage zu authentifizieren. Sie können diese Keys und Tokens wie folgt erstellen:

- [Registrieren Sie sich für ein Entwicklerkonto](https://developer.x.com/en/apply-for-access) und lassen Sie es freigeben.
- Erstellen Sie ein [Projekt](/de/resources/fundamentals/projects) und eine zugehörige [Developer-App](/de/resources/fundamentals/developer-apps) im Developer-Portal.
- Navigieren Sie zur Seite „Keys and tokens“ Ihrer App, um die erforderlichen Zugangsdaten zu generieren. Speichern Sie alle Zugangsdaten an einem sicheren Ort.

<div id="steps-to-build-a-bookmarks-lookup-request">
  #### Schritte zum Erstellen einer Bookmarks-Lookup-Anfrage
</div>

**Schritt eins: Mit einem Tool oder einer Bibliothek beginnen**

Es gibt verschiedene Tools, Codebeispiele und Bibliotheken, mit denen Sie eine Anfrage an diesen Endpunkt stellen können. Wir verwenden hier Postman, um den Prozess zu vereinfachen.

Um die X-API v2 Postman-Sammlung in Ihre Umgebung zu laden, klicken Sie bitte auf die folgende Schaltfläche:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X-API v2 zu Postman hinzufügen
</Button>

Sobald Sie die X-API v2-Sammlung in Postman geladen haben, navigieren Sie zum Ordner „Bookmarks“ und wählen Sie „Bookmarks lookup“.

**Schritt zwei: Authentifizieren Sie Ihre Anfrage**

Für eine erfolgreiche Anfrage an diesen Endpunkt müssen Sie den [OAuth 2.0 Authorization Code Flow mit PKCE](/de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) verwenden. Sie können ein Zugriffstoken direkt in Postman generieren.

Gehen Sie zur Registerkarte „Authorization“ und wählen Sie „OAuth 2.0“.

Befolgen Sie in dieser Registerkarte diese Schritte:

1. Benennen Sie Ihr Token.

2. Wählen Sie als Grant Type „Authorization Code (with PKCE)“.

3. Aktualisieren Sie die Parameter:

   **Callback URL** – [https://www.example.com](https://www.example.com/)

   Diese muss mit der Callback-URL übereinstimmen, die Sie auf der Auth-Einstellungsseite im Developer Portal festgelegt haben.

   **Auth URL** – [https://x.com/i/oauth2/authorize](https://x.com/i/oauth2/authorize)

   **Access Token URL** – [https://api.x.com/2/oauth2/token](https://api.x.com/2/oauth2/token)

   **Client ID** – Kopieren Sie die OAuth 2.0-Client-ID aus dem Developer Portal ein.

   **Client Secret** – Kopieren Sie das OAuth 2.0-Client-Secret aus dem Developer Portal ein. Dies benötigen Sie nur, wenn Ihr App-Typ ein vertraulicher Client ist.

4. Aktualisieren Sie die Scopes mit folgenden Werten: post.read users.read bookmark.read

5. Füllen Sie das Feld state mit „State“.

6. Klicken Sie auf „Generate Token“.

7. Klicken Sie auf das Speichersymbol, um die Ordneränderungen zu sichern.

Möglicherweise erhalten Sie die Meldung, dass Sie nicht bei X angemeldet sind. In diesem Fall müssen Sie sich in dem X-Konto innerhalb von Postman anmelden, in dessen Namen Sie posten möchten.

**Schritt drei: Geben Sie einen Nutzer an**

Bei diesem Endpunkt müssen Sie die Nutzer-ID angeben, deren Lesezeichen Sie in der Antwort abrufen möchten. Beispielsweise lautet die Nutzer-ID für @XDevelopers 2244994945. Navigieren Sie in Postman zur Registerkarte „Params“ und tragen Sie die ID von sich selbst oder eines authentifizierten Nutzers als Wert für den Parameter id ein.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 2244994945 |

**Schritt vier: Senden Sie Ihre Anfrage und prüfen Sie Ihre Antwort**

Sobald alles eingerichtet ist, klicken Sie auf die Schaltfläche „Send“ und Sie erhalten eine Antwort ähnlich dem folgenden Beispiel:
