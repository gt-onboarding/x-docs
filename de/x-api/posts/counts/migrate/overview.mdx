<div id="comparing-x-apis-post-counts-endpoints">
  ## Vergleich der Endpunkte für Beitragszählungen der X-API
</div>

Der Endpunkt für Beitragszählungen in v2 wird schließlich den [Counts-Endpunkt der Enterprise Search API](/de/x-api/enterprise-gnip-2.0/fundamentals/search-api#counts-requests-post-count) ersetzen. Wenn Sie Code, Apps oder Tools verwenden, die eine ältere Version eines Endpunkts für Beitragszählungen nutzen und eine Migration auf die neueren Endpunkte der X-API v2 erwägen, ist dieser Leitfaden für Sie.

Diese Seite enthält Vergleichstabellen zu:

- [Aktuelle Beitragszählungen](#recent)
- [Vollarchiv-Beitragszählungen](#full-archive)
- [Vergleich der Filteroperatoren](#operator)

<div id="recent-post-counts-comparison">
  ### Vergleich der aktuellen Post-Zählungen
</div>

Die Enterprise-Version der Post-Count-Endpunkte ermöglicht es, Zählungen entweder für 30 Tage oder aus dem Vollarchiv abzurufen. Daher ist der v2-Endpunkt für aktuelle Post-Zählungen, der einen Zeitraum von 7 Tagen betrachtet, kein direkter Ersatz für einen der vorgenannten Endpunkte.

Um Vergleiche zu erleichtern, betrachten wir, wie sich der v2-Endpunkt für aktuelle Post-Zählungen im Vergleich zum Enterprise-30-Tage-Endpunkt verhält.

Die folgende Tabelle vergleicht die verschiedenen Typen aktueller Post-Count-Endpunkte:

| **Beschreibung** | **Enterprise** | **X-API v2** |
| :--- | :--- | :--- |
| Host-Domain | https://gnip-api.x.com | https://api.x.com |
| Endpunktpfad | /search/30day/accounts/:account\_name/:label/counts.json | /2/tweets/counts/recent |
| [Authentifizierung](/de/resources/fundamentals/authentication) | Basic Authentication | OAuth 2.0 Bearer Token |
| Zeitstempelformat | YYYYMMDDhhmm | YYYY-MM-DDTHH:mm:ssZ  <br />[ISO 8601 / RFC 3339](https://tools.ietf.org/html/rfc3339#section-5.6) |
| Gibt Zählungen von Posts zurück, die nicht älter sind als | 31 Tage | 7 Tage |
| Unterstützte HTTP-Methoden | GET | GET |
| Standardmäßige Anfragelimits | 20 Anfragen pro 1 Sek., aggregiert über Such- und Zählanfragen  <br />Das minutenbasierte Limit variiert je nach Partner, wie in Ihrem Vertrag festgelegt. | 180 Anfragen pro 15 Min. pro Benutzer  <br />450 Anfragen pro 15 Min. pro App |
| Unterstützt Filterung mittels [annotations](/de/x-api/fundamentals/post-annotations) |     | ✔   |
| Unterstützt Filterung mittels [conversation\_id](/de/x-api/fundamentals/conversation-id) |     | ✔   |
| JSON-Schlüsselname für das Post-Datenarray | results | data |
| Zeitliche Granularität | Tag, Stunde oder Minute | Tag, Stunde oder Minute |
| Zeitzone | UTC | UTC |
| Anfrageparameter zur Auswahl des Zeitraums | fromDate  <br />toDate | start\_time  <br />end\_time |
| Anfrageparameter zur Navigation per Post-ID |     | since\_id  <br />until\_id |
| Erfordert die Verwendung von Zugangsdaten aus einer [Developer-App](/de/resources/fundamentals/developer-apps), die einem [Projekt](/de/resources/fundamentals/projects) zugeordnet ist |     | ✔   |

<div id="filtering-operator-comparison">
  ### Vergleich der Filteroperatoren
</div>

Die beiden Versionen (Enterprise und v2) der Post-Zählungen unterscheiden sich darin, welche Operatoren verfügbar sind. Außerdem gibt es innerhalb jeder Version unterschiedliche Verfügbarkeitsstufen, die unten erläutert werden.

Enterprise

- Es gibt keine Unterstufen bei den Enterprise-Operatoren. Alle Enterprise-Operatoren stehen allen Enterprise-Nutzern zur Verfügung.

X API v2

- **Core:** Diese Operatoren stehen allen v2-Nutzern zur Verfügung.
- **Advanced:** Diese Operatoren sind nur für Nutzer verfügbar, die für den Zugriff im Rahmen der Academic Research zugelassen wurden.

Weitere Informationen zu diesen Operatorgruppen finden Sie in den jeweiligen Leitfäden:

- [Enterprise operators](/de/x-api/enterprise-gnip-2.0/fundamentals/search-api#building-search-queries)
- [X API v2 operators](/de/x-api/posts/search/integrate/build-a-query)
