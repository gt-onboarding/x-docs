Weitere Informationen zu unseren Postman-Collections finden Sie auf der Seite [Postman verwenden](/de/tutorials/postman-getting-started).

<div id="authentication">
  #### Authentifizierung
</div>

Alle Endpunkte der X-API v2 erfordern, dass Anfragen mit einem Satz von Anmeldedaten [authentifiziert](/de/resources/fundamentals/authentication) werden, auch bekannt als Keys und Tokens. Dieser Endpunkt erfordert die Verwendung eines [OAuth 2.0 Bearer Tokens](/de/resources/fundamentals/authentication#oauth-2-0). Das bedeutet, dass Sie ein [Bearer Token](/de/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) übermitteln müssen, um eine erfolgreiche Anfrage zu stellen. Sie können ein Bearer Token entweder direkt in einer Entwickler-App erzeugen oder über den Endpunkt [POST oauth2/token](/de/resources/fundamentals/authentication#post-oauth2-token) generieren.

<div id="developer-portal-projects-and-developer-apps">
  #### Developer-Portal, Projekte und Entwickler-Apps
</div>

Um mit beliebigen Endpunkten der X-API v2 zu arbeiten, müssen Sie ein [Entwicklerkonto](/de/resources/fundamentals/developer-portal) besitzen, innerhalb dieses Kontos ein [Projekt](/de/resources/fundamentals/projects) einrichten und innerhalb dieses Projekts eine [Entwickler-App](/de/resources/fundamentals/developer-apps) erstellen. Die Schlüssel und Token Ihrer Entwickler-App funktionieren für die Endpunkte mit aktuellen Post-Zählungen. Wenn Sie den Endpunkt für Post-Zählungen im Vollarchiv verwenden oder erweiterte Operatoren sowie längere Abfragen nutzen möchten, benötigen Sie eine Genehmigung für Enterprise-Zugriff.

Bitte besuchen Sie unseren Abschnitt zum Enterprise-Zugriff, um mehr zu erfahren.

<div id="rate-limits">
  #### Rate Limits
</div>

Täglich senden viele tausend Entwickler Anfragen an die X-API. Um das Volumen zu steuern, gelten für jeden Endpoint [Rate Limits](/de/x-api/fundamentals/rate-limits), die die Anzahl der Anfragen begrenzen, die ein Entwickler im Namen einer App oder eines authentifizierten Nutzers stellen kann.

Dieser Endpoint ist auf App-Ebene ratebegrenzt. Das bedeutet, dass Sie als Entwickler innerhalb eines bestimmten Zeitraums nur eine bestimmte Anzahl von Anfragen an diesen Endpoint von einer beliebigen App aus stellen können (basierend auf den von Ihnen verwendeten Anmeldedaten). 

<div id="building-queries">
  #### Abfragen erstellen
</div>

Die zentrale Funktion dieser Endpunkte ist die Verwendung einer einzigen Abfrage, um Posts so zu filtern, dass die für Sie relevanten Zählwerte geliefert werden. Diese Abfragen bestehen aus Operatoren, die auf Post- und Nutzerattributen matchen, etwa auf Schlüsselwörtern im Text, Hashtags und URLs. Operatoren lassen sich mit boolescher Logik und Klammern zu Abfragen kombinieren, um das Matching-Verhalten zu verfeinern.

Weitere Informationen finden Sie in unserem Leitfaden [zum Erstellen einer Abfrage](/de/x-api/posts/counts/integrate/build-a-query).

<div id="pagination">
  #### Paginierung
</div>

Bei aktuellen Post-Zählungen wird kein next\_token zurückgegeben. Unabhängig von der Granularität erhalten Sie das Post-Volumen der letzten 7 Tage in einem einzigen API-Aufruf.

Bei Post-Zählungen aus der Vollarchivsuche erhalten Sie Daten für die letzten 30 Tage. Für Daten, die älter als 30 Tage sind, erhalten Sie ein next\_token, mit dem Sie durch die Ergebnisse paginieren können, um zusätzliche Daten abzurufen. 

<div id="building-queries-for-post-counts">
  ### Abfragen für Post-Zählungen erstellen
</div>

Die Endpunkte für Post-Zählungen akzeptieren eine einzelne Abfrage per GET-Anfrage und geben eine Reihe historischer Post-Zählungen zurück, die der Abfrage entsprechen. Abfragen bestehen aus Operatoren, mit denen eine Vielzahl von Post-Attributen abgeglichen wird.
