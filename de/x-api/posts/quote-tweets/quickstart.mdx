<div id="getting-started-with-the-quote-tweets-lookup-endpoint">
  ## Einstieg in den Quote-Tweets-Lookup-Endpunkt
</div>

Diese Schnellstartanleitung hilft Ihnen, mit [Postman](/de/tutorials/postman-getting-started) Ihre erste Anfrage an die Quote‑Tweets‑Lookup‑Endpoints mit einer festgelegten Auswahl an Feldern zu stellen.

Wenn Sie Beispielcode in verschiedenen Sprachen sehen möchten, besuchen Sie bitte unser GitHub-Repository [X‑API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code). 

<div id="prerequisites">
  #### Voraussetzungen
</div>

Um diese Anleitung abzuschließen, benötigen Sie einen Satz von [Keys und Tokens](/de/resources/fundamentals/authentication), um Ihre Anfrage zu authentifizieren. Sie können diese Keys und Tokens generieren, indem Sie die folgenden Schritte ausführen:

- [Registrieren Sie sich für ein Entwicklerkonto](https://developer.x.com/en/apply-for-access) und lassen Sie es genehmigen.
- Erstellen Sie ein [Projekt](/de/resources/fundamentals/projects) und eine zugehörige [Developer-App](/de/resources/fundamentals/developer-apps) im Developer-Portal.
- Navigieren Sie zur Seite „Keys and tokens“ Ihrer App, um die erforderlichen Anmeldedaten zu generieren. Speichern Sie alle Anmeldedaten an einem sicheren Ort.

**Schritte zum Erstellen einer Anfrage für die Abfrage von Zitat-Posts (Quote Tweets)**

**Schritt eins: Beginnen Sie mit einem Tool oder einer Bibliothek**

Es gibt verschiedene Tools, Codebeispiele und Bibliotheken, mit denen Sie eine Anfrage an diesen Endpoint stellen können. Wir verwenden hier Postman, um den Prozess zu vereinfachen.

Um die X-API-v2-Postman-Collection in Ihre Umgebung zu laden, klicken Sie bitte auf die folgende Schaltfläche:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X-API v2 zu Postman hinzufügen
</Button>

Sobald Sie die X-API-v2-Collection in Postman geladen haben, navigieren Sie zum Timeline-Ordner und suchen Sie die Anfrage „Quote Tweets by ID“.

**Schritt zwei: Authentifizieren Sie Ihre Anfrage**

Um korrekt eine Anfrage an die X-API zu stellen, müssen Sie nachweisen, dass Sie berechtigt sind. Für diesen Endpoint müssen Sie Ihre Anfrage mit einer der folgenden Authentifizierungsmethoden authentifizieren: [OAuth 2.0 App-Only](/de/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), [OAuth 2.0 Authorization Code mit PKCE](de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) oder [OAuth 1.0a User Context](/de/resources/fundamentals/authentication).

Der Einfachheit halber verwenden wir für diese Anfrage OAuth 2.0 App-Only. Wenn Sie jedoch private [Metriken](/de/x-api/fundamentals/metrics) oder Posts anfragen möchten, müssen Sie eine der anderen Authentifizierungsmethoden verwenden.

Fügen Sie Ihre Keys und Tokens hinzu, insbesondere das [App Access Token](/de/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token) (auch bekannt als App-only Bearer Token), in Postman. Wählen Sie dazu oben rechts in Postman die Umgebung „X API v2“ aus und tragen Sie Ihre Keys und Tokens in die Felder „initial value“ und „current value“ ein (klicken Sie auf das Augensymbol neben der Umgebungs-Auswahlliste).

Diese Variable wird automatisch in den Authorization-Tab der Anfrage übernommen, wenn Sie dies korrekt eingerichtet haben.

**Schritt drei: Identifizieren und geben Sie den Post an, für den Sie die Zitat-Posts abrufen möchten**

Geben Sie in der Anfrage den Post an, für den Sie Zitat-Posts abrufen möchten, indem Sie die Post-ID übergeben.

In diesem Beispiel lautet die Post-ID, die wir verwenden, 1409931481552543749.

Navigieren Sie in Postman zur Registerkarte „Params“ und tragen Sie diese Post-ID in die Spalte „Value“ des Parameters id ein.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 1409931481552543749 |

**Schritt vier: Identifizieren und geben Sie die Felder an, die Sie abrufen möchten**

Wenn Sie nach Schritt drei auf die Schaltfläche „Send“ klicken, erhalten Sie in der Antwort die Standardfelder des [Post-Objekts](/de/x-api/fundamentals/data-dictionary#tweet): id, text und edit\_history\_tweet\_ids.

Wenn Sie zusätzliche Felder erhalten möchten, müssen Sie diese in Ihrer Anfrage mit den Parametern [field](/de/x-api/fundamentals/fields) und/oder [expansion](/de/x-api/fundamentals/expansions) angeben.

Für diese Übung fordern wir drei zusätzliche, unterschiedliche Feldsätze aus verschiedenen Objekten an:

1. Das zusätzliche Feld tweet.created\_at in den primären Benutzerobjekten.
2. Die Standardfelder der zugehörigen Autoren-Benutzerobjekte für die zurückgegebenen Posts: id, name und username
3. Das zusätzliche Feld user.created\_at in den zugehörigen Benutzerobjekten.

Navigieren Sie in Postman zur Registerkarte „Params“ und fügen Sie das folgende Key:Value-Paar zur Tabelle „Query Params“ hinzu:

|     |     |     |     |
| :--- | :--- | :--- | :--- |
| **Schlüssel** | **Wert** | **Zurückgegebene Felder** |
| `tweet.fields` | `created_at` | `tweets.created_at` |
| `expansions` | `author_id` | `includes.users.id`, `includes.users.name`, `includes.users.username` |
| `user.fields` | `created_at` | `includes.users.created_at` |     |

Sie sollten nun die folgende URL neben dem Button „Send“ sehen:

`https://api.x.com/2/tweets/:id/quote_tweets?expansions=author_id&tweet.fields=created_at&user.fields=created_at`

**Bitte beachten:**

In Postman wird der Pfadparameter :id im URL-Feld **nicht** automatisch auf den Wert aktualisiert, den Sie im `id`-Parameterfeld eingeben. Aus diesem Grund enthält die obige URL `:id` und nicht 1409931481552543749.

<div id="step-six-paginate-through-your-results">
  #### Schritt sechs: Paginieren Sie durch Ihre Ergebnisse
</div>

In der vorherigen Antwort finden Sie unten ein Metadatenobjekt, das die folgenden Felder enthält:

- results\_count
- next\_token

In Schritt vier, da wir keinen Parameter max\_results angegeben haben, erhalten wir standardmäßig nur 10 Posts. Um auf die zusätzlichen Datenseiten zuzugreifen, nehmen wir den Wert des Felds next\_token aus unseren letzten Ergebnissen und fügen diese Zeichenfolge als Wert des Parameters pagination\_token auf der Postman-Params-Seite hinzu, wobei alles andere unverändert bleibt.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `pagination_token` | avdjwk0udyx6 |

Sobald dies eingerichtet ist, können Sie erneut auf „Send“ klicken und sollten die nächste Ergebnisseite erhalten.

Wir haben eine Anleitung zur [Paginierung](/de/x-api/fundamentals/pagination) zusammengestellt, um dieses Konzept weiter zu erläutern.
