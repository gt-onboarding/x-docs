<div id="getting-started-with-the-likes-lookup-endpoint">
  ### Erste Schritte mit dem Likes-Lookup-Endpunkt
</div>

Diese Schnellstartanleitung hilft Ihnen, Ihre erste Anfrage an den Likes-Lookup-Endpunkt mit [Postman](/de/tutorials/postman-getting-started) zu senden.

Wenn Sie Beispielcode in verschiedenen Sprachen sehen möchten, besuchen Sie bitte unser GitHub-Repository mit [Beispielcode für die X-API v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code).
 

<div id="prerequisites">
  #### Voraussetzungen
</div>

Damit Sie diese Anleitung abschließen können, benötigen Sie einen Satz [Schlüssel und Tokens](/de/resources/fundamentals/authentication), den Sie wie folgt erstellen können:

1. [Beantragen Sie ein Entwicklerkonto](https://developer.x.com/en/apply-for-access) und lassen Sie es genehmigen.

2. Erstellen Sie im Developer-Portal ein [Projekt](/de/resources/fundamentals/projects) und eine zugehörige [Developer-App](/de/resources/fundamentals/developer-apps).

3. Navigieren Sie zur Seite „Keys and tokens“ Ihrer App und speichern Sie Ihre API-Schlüssel, Zugriffstokens und den Bearer-Token in Ihrem Passwortmanager.

<div id="steps-to-build-a-likes-lookup-request">
  #### Schritte zum Erstellen einer Likes-Lookup-Anfrage
</div>

**Schritt eins: Beginnen Sie mit einem Tool oder einer Bibliothek**

Es gibt verschiedene Tools, Codebeispiele und Bibliotheken, mit denen Sie Anfragen an diesen Endpoint stellen können. Hier verwenden wir das Tool Postman, um den Prozess zu vereinfachen.

Um die X-API-v2-Postman-Collection in Ihre Umgebung zu laden, klicken Sie bitte auf die folgende Schaltfläche:

[![](https://cdn.cms-twdigitalassets.com/content/dam/developer-twitter/m1_vnext/carat.svg)<br />X API v2 zu Postman hinzufügen](https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400)

Sobald Sie die X-API-v2-Collection in Postman geladen haben, navigieren Sie zum Ordner „Likes“ und wählen Sie „Liking users“.
 

**Schritt zwei: Authentifizieren Sie Ihre Anfrage**

Um eine erfolgreiche Anfrage an diesen Endpoint zu stellen, müssen Sie entweder die Authentifizierung [OAuth 1.0a User Context](/de/resources/fundamentals/authentication) oder [OAuth 2.0 Bearer Token](/de/resources/fundamentals/authentication#oauth-2-0) verwenden. Fügen Sie dazu die folgenden Keys und Tokens zu Postman hinzu, indem Sie die Umgebung mit dem Namen „X API v2“ auswählen und die folgenden Variablen in die Felder Initial value und Current value eintragen:

- consumer\_key mit Ihrem API Key
- consumer\_secret mit Ihrem API Key Secret
- access\_token mit Ihrem Access Token
- token\_secret mit Ihrem Access Token Secret
   

**Schritt drei: Geben Sie einen Tweet an**

Bei diesem Endpoint müssen Sie die Tweet-ID angeben, für die Sie die likenden Nutzer abrufen möchten. Sie können die ID eines Tweets finden, indem Sie zu diesem Tweet auf X navigieren und den numerischen Code am Ende der URL entnehmen. Zum Beispiel ist die Tweet-ID der folgenden URL 1354143047324299264.

https://x.com/TwitterDev/status/1354143047324299264

Navigieren Sie in Postman zur Registerkarte „Params“ und geben Sie diese ID in die Spalte „Value“ der tweet\_id-Pfadvariable (am unteren Rand des Abschnitts) ein. Achten Sie darauf, keine Leerzeichen vor oder nach der ID einzufügen. 

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | Die Tweet-ID, deren likende Nutzer Sie abrufen möchten |

**Schritt vier: Identifizieren und geben Sie an, welche Felder Sie abrufen möchten**

Wenn Sie nach Schritt drei auf die Schaltfläche „Send“ klicken, erhalten Sie in der Antwort die Standardfelder des [user object](/de/x-api/fundamentals/data-dictionary#user): id, name und username.

Wenn Sie zusätzliche Felder über id, name und username hinaus erhalten möchten, müssen Sie diese Felder in Ihrer Anfrage mit den Parametern [fields](/de/x-api/fundamentals/fields) und/oder [expansions](/de/x-api/fundamentals/expansions) angeben.

Für diese Übung fordern wir drei zusätzliche Feldsätze aus unterschiedlichen Objekten an:

1. Das zusätzliche Feld user.created\_at in den primären User-Objekten.
2. Die Standardfelder des verknüpften Objekts der angehefteten Tweets für die zurückgegebenen Nutzer: id und text.
3. Das zusätzliche Feld tweet.created\_at in den verknüpften Tweet-Objekten.

Navigieren Sie in Postman zur Registerkarte „Params“ und fügen Sie das folgende Key:Value-Paar zur Tabelle „Query Params“ hinzu:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| user.fields | created\_at | user.created\_at |
| expansions | pinned\_tweet\_id | tweet.id, tweet.text |
| tweet.fields | created\_at | tweet.created\_at |

Sie sollten nun die folgende URL neben der Schaltfläche „Send“ sehen:

`https://api.x.com/2/tweets/1354143047324299264/liking_users?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=created_at`

**Schritt sechs: „Gefällt mir“-Tweets eines Nutzers abrufen**

Möglicherweise möchten Sie auch eine Anfrage stellen, um die „Gefällt mir“-Tweets eines Nutzers abzurufen. Mit dem Likes-Lookup-Endpunkt können Sie Informationen zu den „Gefällt mir“-Tweets eines Nutzers erhalten. Navigieren Sie dazu zum Ordner „Likes“ und wählen Sie „Liked Tweets“.

Bei diesem Endpunkt müssen Sie die User-ID angeben, deren „Gefällt mir“-Tweets Sie abrufen möchten. Sie können den Endpunkt [User Lookup](/de/x-api/users/lookup/introduction) verwenden, um diese Information zu erhalten.

Navigieren Sie in Postman zur Registerkarte „Params“ und geben Sie diese User-ID in die Spalte „Value“ der id-Pfadvariable (am unteren Ende des Abschnitts) ein. Achten Sie darauf, vor oder nach der ID keine Leerzeichen einzufügen.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| id  | Die User-ID, deren „Gefällt mir“-Tweets Sie abrufen möchten |
| max\_results | 5   |

Sie sehen nun eine ähnliche URL mit Ihrer ID anstelle der von TwitterDev neben der Schaltfläche „Send“:

`https://api.x.com/2/users/2244994945/liked_tweets?max_results=5`
