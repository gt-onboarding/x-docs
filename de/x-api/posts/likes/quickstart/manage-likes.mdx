<div id="getting-started-with-the-manage-likes-endpoints">
  ### Erste Schritte mit den Endpunkten zum Verwalten von Likes
</div>

Diese Schnellstartanleitung hilft Ihnen, Ihre erste Anfrage an die Endpunkte zum Verwalten von Likes mit [Postman](/de/tutorials/postman-getting-started) zu senden.

Wenn Sie Beispielcode in verschiedenen Sprachen sehen möchten, besuchen Sie unser GitHub-Repository mit [Beispielcode für die X-API v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code). 

<div id="prerequisites">
  #### Voraussetzungen
</div>

Damit Sie diese Anleitung abschließen können, benötigen Sie einen Satz [Keys und Tokens](/de/resources/fundamentals/authentication). Diese können Sie generieren, indem Sie die folgenden Schritte ausführen:

1. [Beantragen Sie ein Entwicklerkonto](https://developer.x.com/en/apply-for-access) und lassen Sie es genehmigen.

2. Erstellen Sie ein [Projekt](/de/resources/fundamentals/projects) und eine zugehörige [Developer-App](/de/resources/fundamentals/developer-apps) im Developer-Portal.

3. Navigieren Sie zur Seite „Keys and tokens“ Ihrer App und speichern Sie Ihre API Keys, Access Tokens und den Bearer Token in Ihrem Passwort-Manager.

<div id="steps-to-build-a-manage-likes-request">
  #### Schritte zum Erstellen einer „Likes verwalten“-Anfrage
</div>

**Schritt eins: Beginnen Sie mit einem Tool oder einer Bibliothek**

Es gibt mehrere Tools, Codebeispiele und Bibliotheken, mit denen Sie eine Anfrage an diesen Endpunkt senden können. Wir verwenden hier jedoch Postman, um den Prozess zu vereinfachen.

Um die Postman-Sammlung für die X-API v2 in Ihre Umgebung zu laden, klicken Sie bitte auf die folgende Schaltfläche:

[![](https://cdn.cms-twdigitalassets.com/content/dam/developer-twitter/m1_vnext/carat.svg)<br />X-API v2 zu Postman hinzufügen](https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400)

Sobald Sie die X-API-v2-Sammlung in Postman geladen haben, navigieren Sie zum Ordner „Likes“ und wählen Sie „Like a Post“.

**Schritt zwei: Authentifizieren Sie Ihre Anfrage**

Um eine erfolgreiche Anfrage an diesen Endpunkt zu stellen, müssen Sie [OAuth 1.0a User Context](/de/resources/fundamentals/authentication) verwenden. Fügen Sie dazu die folgenden Keys und Tokens in Postman hinzu, indem Sie die Umgebung namens „X API v2“ auswählen und die folgenden Variablen in die Felder Anfangswert und aktueller Wert eintragen:

- consumer\_key mit Ihrem API Key
- consumer\_secret mit Ihrem API Key Secret
- access\_token mit Ihrem Access Token
- token\_secret mit Ihrem Access Token Secret
   

**Schritt drei: Geben Sie an, welchen Post Sie liken werden**

Die Endpunkte zum Verwalten von Likes erfordern zwei IDs: eine für den Nutzer (der Nutzer, der einen Post liken möchte) und eine für die Post-ID, die der Nutzer liken oder unliken möchte. 

Die Nutzer-ID muss der ID des authentifizierenden Nutzers entsprechen. Das bedeutet, dass Sie beim Authentifizieren Ihrer Anfrage die [Access Tokens](/de/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) übergeben müssen, die der Nutzer-ID zugeordnet sind. In diesem Fall können Sie die ID angeben, die zu Ihrem eigenen Nutzer gehört. Sie können Ihre ID auf zwei Arten finden:

1. Mit dem Endpunkt [user lookup by username](/de/x-api/users/user-lookup-by-id) können Sie einen Benutzernamen übergeben und das Feld id erhalten. 
2. In Ihrem Access Token ist der numerische Teil Ihre Nutzer-ID.
    

Sie müssen außerdem einen Post angeben, den Sie liken möchten. Sie finden die Post-ID, indem Sie zu x.com navigieren, einen Post öffnen und dann die URL prüfen. Beispielsweise ist die Post-ID der folgenden URL 1228393702244134912.

https://x.com/TwitterDev/status/1228393702244134912

Navigieren Sie in Postman zur Registerkarte „Params“ und geben Sie Ihre ID in die Spalte „Value“ der Pfadvariable id ein. Wechseln Sie zur Registerkarte „Body“ und geben Sie die ID des Posts, den Sie liken möchten, als Wert für den Parameter tweet\_id ein. Achten Sie darauf, keine Leerzeichen vor oder nach einer ID einzufügen.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | (Ihre Nutzer-ID) |
| tweet\_id | (die ID des Posts, den Sie liken möchten) |

Wenn Sie auf die Schaltfläche „Send“ klicken, erhalten Sie ein Response-Objekt, das den Status der Beziehung enthält:

- Wenn Sie „liked“: true erhalten, dann liked die id die tweet\_id erfolgreich.
   

**Schritt vier: Senden Sie Ihre Anfrage und prüfen Sie Ihre Antwort**

Sobald alles eingerichtet ist, klicken Sie auf die Schaltfläche „Send“ und Sie erhalten die folgende Antwort:

```
{
    "data": {
        "liked": true
    }
}
```

Wenn Sie denselben Post unliken möchten, können Sie die Anfrage mit dem Titel „Unlike a Post“ verwenden, die sich ebenfalls im Ordner „Likes“ der X-API-v2-Sammlung in Postman befindet. Die id und tweet\_id sollten auf die gleiche Weise wie im vorherigen Beispiel verwendet werden. Um einen Post zu unliken, müssen Sie dies nicht als JSON-Body hinzufügen. Stellen Sie daher sicher, dass Sie die erforderlichen Query-Parameter für id und tweet\_id angeben.
