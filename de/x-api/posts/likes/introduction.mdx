<div id="likes-lookup">
  ### Likes-Lookup
</div>

Mit den Endpunkten der Likes-Lookup-Gruppe können Sie eine Liste von Accounts abrufen, die einen Post geliked haben, oder eine Liste von Posts, die ein Account geliked hat. Diese Endpunkte umfassen:

- Von einem Nutzer gelikete Posts - GET /2/users/:id/liked\_tweets
- Nutzer, die einen Post geliked haben - GET /2/tweets/:id/liking\_users

Sie können diese Endpunkte entweder mit [OAuth 1.0a User Context](/de/resources/fundamentals/authentication) oder [OAuth 2.0 Bearer Token](/de/resources/fundamentals/authentication#oauth-2-0) authentifizieren. Für die Endpunkte zu geliketen Posts werden Paginierungstokens bereitgestellt, um große Ergebnismengen seitenweise abzurufen.

Der Endpunkt für liking users liefert insgesamt höchstens 100 Accounts pro Post über den gesamten Zeitraum. Zusätzlich unterliegt der Endpunkt für gelikete Posts dem monatlichen [Post-Limit](/de/x-api/fundamentals/post-cap) auf Projektebene.
 

<div id="manage-likes">
  ### Likes verwalten
</div>

Die Endpunkte zum Verwalten von Likes ermöglichen es Ihnen, im Namen eines authentifizierten Kontos einen angegebenen Post zu liken oder das Like zu entfernen. Für diese Endpunktgruppe stehen zwei Methoden zur Verfügung: POST und DELETE. Die POST-Methode ermöglicht es, einen Post zu liken, und die DELETE-Methode ermöglicht es, das Like für einen Post zu entfernen.

Da Sie Anfragen im Namen eines Nutzers stellen, müssen Sie diese Endpunkte mit [OAuth 1.0a User Context](/de/resources/fundamentals/authentication) authentifizieren und die dem Nutzer zugeordneten Access Tokens verwenden, die über den [3-legged OAuth-Flow](/de/resources/fundamentals/authentication#oauth-1-0a-2)/obtaining-user-access-tokens) generiert werden können. Sie können einen Post von Ihrem Konto oder dem Konto eines authentifizierten Nutzers liken. Für beide Endpunkte gilt ein nutzerbasiertes Ratenlimit von 50 Anfragen pro 15 Minuten und pro Endpunkt.

Um auf diese Endpunkte zuzugreifen, benötigen Sie ein genehmigtes [Developer-Konto](/de/resources/fundamentals/developer-portal). Bei der Authentifizierung müssen Sie Schlüssel und Tokens aus einer [Developer-App](/de/resources/fundamentals/developer-apps) verwenden, die sich innerhalb eines [Projekts](/de/resources/fundamentals/projects) befindet.

Erfahren Sie mehr über den Zugriff auf die X-API-v2-Endpunkte auf unserer Seite [Erste Schritte](/de/x-api/getting-started/about-x-api).
