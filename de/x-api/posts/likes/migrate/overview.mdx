<div id="comparing-x-apis-likes-endpoints">
  ## Vergleich der Likes-Endpunkte der X‑API
</div>

Diese Leitfäden konzentrieren sich auf die folgenden Bereiche:

- **API-Anfrageparameter** – Der X‑API‑v2‑Endpunkt führt einen neuen Satz von Anfrageparametern ein. Einige Parameter werden vertraut sein, insbesondere für diejenigen, die bereits mit Labs arbeiten, jedoch gibt es viele wichtige Unterschiede, etwa die Einführung der Parameter [fields](/de/x-api/fundamentals/fields) und [expansions](/de/x-api/fundamentals/expansions).

- **Anforderungen für App und Projekt** – Für den Zugriff auf die X‑API v2 müssen Sie Zugangsdaten einer [Developer App](/de/resources/fundamentals/developer-apps) verwenden, die einem [Project](/de/resources/fundamentals/projects) zugeordnet ist

<div id="likes-lookup">
  #### Likes-Abfrage
</div>

**Nutzer, die einen Post mit „Gefällt mir“ markiert haben**

Der Endpoint für Nutzer, denen ein Post gefällt, ist eine neue Funktion in v2 und ermöglicht es, Informationen über die „Liker“ eines Posts abzurufen.

| Beschreibung | X-API v2 |
| :--- | :--- |
| Unterstützte HTTP-Methoden | GET |
| Host-Domain | https://api.x.com |
| Endpoint-Pfad | /2/tweets/:id/liking\_users |
| [Authentifizierung](/de/resources/fundamentals/authentication) | OAuth 2.0 Bearer Token<br /><br />OAuth 1.0a User Context |
| Standard-[Rate-Limits](/de/resources/fundamentals/rate-limits) für Anfragen | 75 Anfragen pro 15 Min. (pro App)<br /><br />75 Anfragen pro 15 Min. (pro Nutzer) |

**Posts, die ein Nutzer geliked hat**

Die folgenden Tabellen vergleichen den Standard-v1.1-Endpoint [GET favorites/list](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-favorites-list) mit den X-API-v2-Endpoints für gelikete Posts:

| Beschreibung | Standard v1.1 | X-API v2 |
| :--- | :--- | :--- |
| Unterstützte HTTP-Methoden | GET | GET |
| Host-Domain | https://api.x.com | https://api.x.com |
| Endpoint-Pfad | /1.1/favorites/list.json | /2/users/:id/liked\_tweets |
| [Authentifizierung](/de/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 2.0 Bearer Token<br /><br />OAuth 1.0a User Context |
| Standard-[Rate-Limits](/de/resources/fundamentals/rate-limits) für Anfragen | 75 Anfragen pro 15 Min. | 75 Anfragen pro 15 Min. (pro App)<br /><br />75 Anfragen pro 15 Min. (pro Nutzer) |
| Datenformate | Standard-v1.1-Format | [X-API-v2-Format](/de/x-api/fundamentals/data-dictionary) (bestimmt durch die Request-Parameter fields und expansions; nicht abwärtskompatibel zu v1.1-Formaten)<br /><br />Weitere Informationen zur Migration vom Standard-v1.1-Format zum X-API-v2-Format finden Sie in unserem [Leitfaden zur Migration von Datenformaten](/de/x-api/migrate/data-format-migration). |

<div id="manage-likes">
  #### Likes verwalten
</div>

Die v2-Endpunkte zum Verwalten von Likes ersetzen die v1.1-Endpunkte [POST favorites/create](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-create) und [POST favorites/destroy](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-destroy).

Die folgenden Tabellen vergleichen die Standard-v1.1-Endpunkte und die Endpunkte der X-API v2 zum Verwalten von Likes:

<div id="like-a-post">
  #### Einen Post liken
</div>

| Beschreibung | Standard v1.1 | X-API v2 |
| :--- | :--- | :--- |
| Unterstützte HTTP-Methoden | POST | POST |
| Host-Domain | https://api.x.com | https://api.x.com |
| Endpoint-Pfad | /1.1/favorites/create.json | /2/users/:id/likes |
| [Authentifizierung](/de/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context |
| Standardmäßige [Rate-Limits](/de/resources/fundamentals/rate-limits) für Anfragen | 1000 Anfragen pro 24 Stunden (pro Benutzer)<br /><br />1000 Anfragen pro 24 Stunden (pro App) | 50 Anfragen pro 15 Min. (pro Benutzer)<br /><br />1000 Anfragen pro 24 Stunden (pro Benutzer, gemeinsam mit DELETE) |

<div id="unlike-a-post">
  #### „Gefällt mir“ von einem Post entfernen
</div>

| Beschreibung | Standard v1.1 | X-API v2 |
| :--- | :--- | :--- |
| Unterstützte HTTP-Methoden | POST | DELETE |
| Host-Domain | https://api.x.com | https://api.x.com |
| Endpunktpfad | /1.1/favorites/destroy.json | /2/users/:id/likes/:tweet\_id |
| [Authentifizierung](/de/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context |
| Standardmäßige [Rate Limits](/de/resources/fundamentals/rate-limits) für Anfragen | 1000 Anfragen pro 24 Stunden (pro Nutzer)<br /><br />1000 Anfragen pro 24 Stunden (pro App) | 50 Anfragen pro 15 Min (pro Nutzer)<br /><br />1000 Anfragen pro 24 Stunden (pro Nutzer, gemeinsam mit POST) |
