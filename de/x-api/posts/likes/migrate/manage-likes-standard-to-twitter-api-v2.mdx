<div id="manage-likes-standard-v11-compared-to-x-api-v2">
  ### Likes verwalten: Standard v1.1 im Vergleich zur X-API v2
</div>

Wenn Sie mit den Standard-v1.1-Endpunkten [POST favorites/create](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-create) und [POST favorites/destroy](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-destroy) gearbeitet haben, hilft Ihnen dieser Leitfaden, die Gemeinsamkeiten und Unterschiede zwischen den Standard-v1.1- und den X-API-v2-Endpunkten zur Verwaltung von Likes zu verstehen.

- **Gemeinsamkeiten**
  - OAuth 1.0a User Context
- **Unterschiede**
  - Endpunkt-URLs und HTTP-Methoden
  - Anforderungen an Apps und Projekte
  - Request-Parameter

<div id="similarities">
  #### Gemeinsamkeiten
</div>

**Authentifizierungsmethode für OAuth 1.0a User Context**

Beide Endpunkt-Versionen unterstützen [OAuth 1.0a User Context](/de/resources/fundamentals/authentication#oauth-1-0a-2). Wenn Sie also zuvor einen der Standard-Endpunkte in v1.1 zum Verwalten von Favoriten verwendet haben, können Sie bei der Migration zur X-API v2 weiterhin dieselbe Authentifizierungsmethode nutzen.

<div id="differences">
  #### Unterschiede
</div>

**Endpoint-URLs und HTTP-Methoden**

- Standard-v1.1-Endpoints:
  - POST https://api.x.com/1.1/favorites/create.json
    (einen Post liken)
  - POST https://api.x.com/1.1/favorites/destroy.json
    (ein Like von einem Post entfernen)
- X-API-v2-Endpoint:
  - POST https://api.x.com/2/tweets/:id/likes
    (einen Post liken)
  - DELETE https://api.x.com/2/tweets/:id/likes/:tweet\_id
    (ein Like von einem Post entfernen) 

**App- und Projektanforderungen**

Die X-API-v2-Endpoints erfordern, dass Sie bei der Authentifizierung Ihrer Anfragen Anmeldedaten aus einer [Developer-App](/de/resources/fundamentals/developer-apps) verwenden, die einem [Projekt](/de/resources/fundamentals/projects) zugeordnet ist. Alle X-API-v1.1-Endpoints können Anmeldedaten von eigenständigen Apps oder von Apps verwenden, die einem Projekt zugeordnet sind.

**Anfrageparameter**

Die folgenden Standard‑v1.1‑Anfrageparameter haben Entsprechungen in der X‑API v2:

| Standard v1.1 | X API v2 |
| :--- | :--- |
| id  | id  |
| includes\_entities | Keine Entsprechung |

Bitte beachten Sie, dass die Standard‑v1.1‑Parameter als Query-Parameter übergeben werden, während die X‑API‑v2‑Parameter als Body-Parameter für den POST-Endpoint bzw. als Pfadparameter für den DELETE-Endpoint übergeben werden.

Außerdem ist eine ID des Nutzers, der einen Post liked, bei Verwendung der Standard‑v1.1‑Endpoints nicht erforderlich, da die mit [OAuth 1.0a User Context](/de/resources/fundamentals/authentication) übergebenen [Access Tokens](/de/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) erkennen lassen, welcher Nutzer das Liken bzw. das Entfernen des Likes initiiert.
