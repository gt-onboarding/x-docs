<div id="getting-started-with-reverse-chronological-home-timeline">
  ## Erste Schritte mit der umgekehrt chronologischen Startseiten-Timeline
</div>

Diese Kurzanleitung hilft Ihnen, Ihre erste Anfrage an einen der Timeline-Endpunkte mit einem Satz festgelegter Felder zu stellen – unter Verwendung von [Postman](/de/tutorials/postman-getting-started).

Wenn Sie Beispielcode in verschiedenen Sprachen sehen möchten, besuchen Sie bitte unser GitHub-Repository [X-API-v2-Beispielcode](https://github.com/xdevplatform/Twitter-API-v2-sample-code).

<Note>
  **Voraussetzungen**

  Um diese Anleitung abzuschließen, benötigen Sie einen Satz [Keys und Tokens](/de/resources/fundamentals/authentication), um Ihre Anfrage zu authentifizieren. Sie können diese Keys und Tokens generieren, indem Sie die folgenden Schritte ausführen:

  - [Registrieren Sie sich für ein Entwicklerkonto](https://developer.x.com/en/apply-for-access) und lassen Sie es freischalten.
  - Erstellen Sie ein [Projekt](/de/resources/fundamentals/projects) und eine zugehörige [Developer-App](/de/resources/fundamentals/developer-apps) im Developer-Portal.
  - Wechseln Sie zur Seite „Keys and tokens“ Ihrer App, um die erforderlichen Anmeldedaten zu generieren. Speichern Sie alle Anmeldedaten an einem sicheren Ort.
</Note>

**Schritte zum Erstellen einer Anfrage für die umgekehrt chronologische Startseiten-Timeline**

_Für dieses Beispiel stellen wir eine Anfrage an den Endpunkt „User Post timeline by ID“, aber Sie können das Gelernte aus diesem Quickstart auch auf Anfragen für Mention-Timelines anwenden._

**Schritt eins: Beginnen Sie mit einem Tool oder einer Bibliothek**

Es gibt mehrere Tools, Codebeispiele und Bibliotheken, mit denen Sie eine Anfrage an diesen Endpunkt stellen können. Wir verwenden hier Postman, um den Prozess zu vereinfachen.

Um die X-API-v2-Postman-Collection in Ihre Umgebung zu laden, klicken Sie bitte auf die folgende Schaltfläche:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X-API v2 zu Postman hinzufügen
</Button>

Sobald Sie die X-API-v2-Collection in Postman geladen haben, navigieren Sie zum Ordner „Timelines“ und wählen Sie die Anfrage „Reverse chronological home timeline“ aus.

**Schritt zwei: Authentifizieren Sie Ihre Anfrage**

Um korrekt eine Anfrage an die X-API zu stellen, müssen Sie Ihre Berechtigung nachweisen. Für diesen Endpunkt müssen Sie Ihre Anfrage entweder mit [OAuth 2.0 Authorization Code mit PKCE](/de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) oder mit [OAuth 1.0a User Context](/de/resources/fundamentals/authentication) authentifizieren.

Fügen Sie Ihre Keys und Tokens in Postman hinzu. Wählen Sie dazu oben rechts die Umgebung „X API v2“ aus und tragen Sie Ihre Keys und Tokens in die Felder „initial value“ und „current value“ ein (über das Augensymbol neben dem Umgebungs-Dropdown).

Diese Variablen werden automatisch in den Reiter „Authorization“ der Anfrage übernommen, wenn alles korrekt konfiguriert ist.

**Schritt drei: Legen Sie fest, für welchen Nutzer Sie die Startseiten-Timeline abrufen möchten**

Geben Sie in der Anfrage den Nutzer an, für den Sie aktuelle Posts abrufen möchten. In diesem Beispiel übergeben wir eine einzelne User-ID.

User-IDs sind der numerische Wert, der ein Account-Handle repräsentiert und den Sie in der Profil-URL eines Accounts finden können. Beispielsweise lautet der Benutzername des folgenden Accounts `XDevelopers`.

`https://x.com/XDevelopers`

Um diesen Benutzernamen in die User-ID zu konvertieren, verwenden Sie den [Users-Lookup-Endpunkt](/de/x-api/users/lookup/introduction) mit dem Benutzernamen und entnehmen die numerische User-ID dem Payload. Im Fall von @XDevelopers lautet die User-ID 2244994945.

Navigieren Sie in Postman zum Reiter „Params“ und tragen Sie diese User-ID in die Spalte „Value“ des Parameters id ein.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 2244994945 |

**Schritt vier: Legen Sie fest, welche Felder Sie abrufen möchten**

Wenn Sie nach Schritt drei auf die Schaltfläche „Send“ klicken, erhalten Sie in der Antwort die standardmäßigen Felder des [Post-Objekts](/de/x-api/fundamentals/data-dictionary#tweet): id, text und edit\_history\_tweet\_ids.

Wenn Sie zusätzliche Felder erhalten möchten, müssen Sie diese in Ihrer Anfrage über die Parameter [fields](/de/x-api/fundamentals/fields) und/oder [expansions](/de/x-api/fundamentals/expansions) angeben.

Für diese Übung fordern wir drei zusätzliche, unterschiedliche Feldsätze aus verschiedenen Objekten an:

1. Das zusätzliche Feld tweet.created\_at in den primären Benutzerobjekten.

2. Die Standardfelder des Benutzerobjekts der zugehörigen Autoren für die zurückgegebenen Posts: id, name und username

3. Das zusätzliche Feld user.created\_at in den zugehörigen Benutzerobjekten.
    

In Postman navigieren Sie zum Tab „Params“ und fügen Sie der Tabelle „Query Params“ das folgende Key-Value-Paar hinzu:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| `tweet.fields` | `created_at` | `tweets.created_at` |
| `expansions` | `author_id` | `includes.users.id`, `includes.users.name`, `includes.users.username` |
| `user.fields` | `created_at` | `includes.users.created_at` |
| max\_results | 5   |     |

Sie sollten nun neben dem Button „Send“ die folgende URL sehen:

`https://api.x.com/2/users/:id/timelines/reverse_chronological?tweet.fields=created_at&expansions=author_id&user.fields=created_at&max_results=5`

<Info>
  **Bitte beachten:**

  In Postman wird der Pfadparameter :id im URL-Feld **nicht** automatisch auf den Wert aktualisiert, den Sie im `id`-Params-Feld eingeben. Aus diesem Grund enthält die obige URL `:id` und nicht 2244994945.
</Info>

**Schritt sechs: Durch Ihre Ergebnisse paginieren**

In der vorherigen Antwort finden Sie am Ende ein Metadatenobjekt mit folgenden Feldern:

- oldest\_id
- newest\_id
- results\_count
- next\_token
- previous\_token

In Schritt vier haben wir einen Wert von max\_results = 5 verwendet. Das bedeutet, dass jede Seite höchstens fünf Ergebnisse enthält. Um auf weitere Seiten zuzugreifen, nehmen Sie den Wert des Feldes next\_token aus den letzten Ergebnissen und tragen diesen String als Wert des Parameters pagination\_token auf der Postman-Seite „Params“ ein, wobei alles andere unverändert bleibt.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `pagination_token` | t3buvdr5pujq9g7bggsnf3ep2ha28 |

Sobald alles eingerichtet ist, klicken Sie erneut auf „Send“, und Sie sollten die nächste Ergebnisseite erhalten.

Wir haben eine Anleitung zur [Paginierung](/de/x-api/fundamentals/pagination) zusammengestellt, die dieses Konzept weiter erläutert.
