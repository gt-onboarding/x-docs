<div id="getting-started-with-the-user-post-and-mention-timeline-endpoints">
  ## Erste Schritte mit den Endpunkten für die Nutzer-Post- und Erwähnungs-Timelines
</div>

Diese Schnellstartanleitung hilft Ihnen, Ihre erste Anfrage an den Endpunkt der Nutzer-Post-Timeline mit einem Satz bestimmter Felder mithilfe von [Postman](/de/tutorials/postman-getting-started) zu stellen.

Wenn Sie Beispielcode in verschiedenen Sprachen sehen möchten, besuchen Sie bitte unser GitHub-Repository mit [Beispielcode für die X-API v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code).

<Note>
  **Voraussetzungen**

  Um diese Anleitung abzuschließen, benötigen Sie einen Satz an [Keys und Tokens](/de/resources/fundamentals/authentication), um Ihre Anfrage zu authentifizieren. Sie können diese Keys und Tokens generieren, indem Sie die folgenden Schritte ausführen:

  - [Melden Sie sich für ein Developer-Konto an](https://developer.x.com/en/apply-for-access) und lassen Sie es genehmigen.
  - Erstellen Sie ein [Project](/de/resources/fundamentals/projects) und eine zugehörige [Developer-App](/de/resources/fundamentals/developer-apps) im Developer-Portal.
  - Navigieren Sie zur Seite „Keys and tokens“ Ihrer App, um die erforderlichen Zugangsdaten zu generieren. Speichern Sie alle Zugangsdaten an einem sicheren Ort.
</Note>

**Schritte zum Erstellen einer Timeline-Anfrage**

_Für dieses Beispiel stellen wir eine Anfrage an den Endpunkt „User Post timeline by ID“, aber Sie können das in diesem Schnellstart Gelernte auch auf Anfragen an Nutzer-Erwähnungs-Timelines anwenden._

**Schritt eins: Wählen Sie ein Tool oder eine Bibliothek**

Es gibt mehrere Tools, Codebeispiele und Bibliotheken, mit denen Sie eine Anfrage an diesen Endpunkt stellen können. Wir verwenden hier Postman, um den Prozess zu vereinfachen.

Um die Postman-Collection für die X-API v2 in Ihre Umgebung zu laden, klicken Sie auf die folgende Schaltfläche:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2 zu Postman hinzufügen
</Button>

Sobald Sie die Collection der X-API v2 in Postman geladen haben, navigieren Sie zum Ordner „timeline“ und wählen Sie die Anfrage „User Post timeline by ID“ aus.
 

**Schritt zwei: Authentifizieren Sie Ihre Anfrage**

Um eine Anfrage korrekt an die X-API zu stellen, müssen Sie Ihre Berechtigung nachweisen. Bei diesem Endpunkt müssen Sie Ihre Anfrage mit einer der folgenden Methoden authentifizieren: [OAuth 2.0 App-Only](/de/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), [OAuth 2.0 Authorization Code with PKCE](de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) oder [OAuth 1.0a User Context](/de/resources/fundamentals/authentication).

Der Einfachheit halber verwenden wir für diese Anfrage OAuth 2.0 App-Only. Wenn Sie jedoch private [Metriken](/de/x-api/fundamentals/metrics) oder Posts anfordern möchten, müssen Sie eine der anderen Authentifizierungsmethoden verwenden.

Fügen Sie Ihre Keys und Tokens hinzu, insbesondere das [App Access Token](/de/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token) (auch bekannt als App-only Bearer Token), in Postman. Wählen Sie dazu oben rechts in Postman die Umgebung „X API v2“ aus und tragen Sie Ihre Keys und Tokens in die Felder „initial value“ und „current value“ ein (klicken Sie auf das Augensymbol neben dem Umgebungs-Dropdown).

Diese Variable wird automatisch in den Reiter „Authorization“ der Anfrage übernommen, wenn Sie dies korrekt eingerichtet haben.
 

**Schritt drei: Bestimmen Sie, von welchem Nutzer Sie Posts abrufen möchten**

Geben Sie in der Anfrage den Nutzer an, für den Sie aktuelle Posts abrufen möchten. In diesem Beispiel übergeben wir eine einzelne Nutzer-ID.

Nutzer-IDs sind die numerischen Werte, die ein Account-Handle repräsentieren und die Sie in der Profil-URL eines Accounts finden können. Zum Beispiel lautet der Benutzername des folgenden Accounts `XDevelopers`.

`https://x.com/XDevelopers`

Um diesen Benutzernamen in die Nutzer-ID zu konvertieren, verwenden Sie den [users lookup endpoint](/de/x-api/users/lookup/introduction) mit dem Benutzernamen und entnehmen die numerische Nutzer-ID dem Payload. Im Fall von @XDevelopers ist die Nutzer-ID 2244994945.

Navigieren Sie in Postman zum Reiter „Params“ und tragen Sie diese Nutzer-ID in die Spalte „Value“ des Parameters „id“ ein.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 2244994945 |

**Schritt vier: Bestimmen Sie, welche Felder Sie abrufen möchten**

Wenn Sie nach Schritt drei auf die Schaltfläche „Send“ klicken, erhalten Sie in Ihrer Antwort die Standardfelder des [Post-Objekts](/de/x-api/fundamentals/data-dictionary#tweet): id und text.

Wenn Sie zusätzliche Felder über id und text hinaus erhalten möchten, müssen Sie diese in Ihrer Anfrage über die Parameter [field](/de/x-api/fundamentals/fields) und/oder [expansion](/de/x-api/fundamentals/expansions) angeben.

Für diese Übung fordern wir drei zusätzliche, unterschiedliche Feldsätze aus verschiedenen Objekten an:

1. Das zusätzliche Feld tweet.created\_at in den primären User-Objekten.

2. Die Standardfelder des User-Objekts der zugehörigen Autorinnen/Autoren für die zurückgegebenen Posts: id, name und username

3. Das zusätzliche Feld user.created\_at in den zugehörigen User-Objekten.
    

Navigieren Sie in Postman zur Registerkarte „Params“ und fügen Sie der Tabelle „Query Params“ das folgende Key:Value-Paar hinzu:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| `tweet.fields` | `created_at` | `tweets.created_at` |
| `expansions` | `author_id` | `includes.users.id`, `includes.users.name`, `includes.users.username` |
| `user.fields` | `created_at` | `includes.users.created_at` |
| max\_results | 5   |     |

Neben der Schaltfläche „Send“ sollten Sie nun die folgende URL sehen:

`https://api.x.com/2/users/:id/tweets?tweet.fields=created_at&expansions=author_id&user.fields=created_at&max_results=5`

<Info>
  **Bitte beachten:**

  In Postman wird der Pfadparameter :id im URL-Feld **nicht** automatisch auf den Wert aktualisiert, den Sie im `id`-Parameterfeld eingeben. Daher enthält die obige URL `:id` und nicht 2244994945.
</Info>

**Schritt sechs: Ergebnisse paginieren**

In der vorherigen Antwort finden Sie am Ende ein Metadatenobjekt mit folgenden Feldern:

- oldest\_id
- newest\_id
- results\_count
- next\_token
- previous\_token

In Schritt vier haben wir einen max\_results-Wert von 5 übergeben. Das bedeutet, dass jede Seite höchstens fünf Ergebnisse enthält. Um auf weitere Datenseiten zuzugreifen, nehmen wir den Wert des Felds next\_token aus den letzten Ergebnissen und fügen diese Zeichenfolge als Wert des Parameters pagination\_token auf der Postman-Seite Params hinzu, wobei alles andere unverändert bleibt.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `pagination_token` | t3buvdr5pujq9g7bggsnf3ep2ha28 |

Sobald alles eingerichtet ist, können Sie erneut auf „Send“ klicken und sollten die nächste Ergebnisseite erhalten.

Wir haben einen Leitfaden zur [Paginierung](/de/x-api/fundamentals/pagination) zusammengestellt, der dieses Konzept ausführlicher erklärt.
