<div id="comparing-x-apis-timelines-endpoints">
  ## Vergleich der Timeline-Endpunkte der X-API
</div>

Die Endpunkte für die umgekehrt chronologische Timeline, die Nutzer-Posts-Timeline und die Nutzer-Erwähnungen-Timeline in v2 ersetzen die Endpunkte [v1.1 statuses/home\_timeine,](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-home_timeline) [v1.1 statuses/user\_timeline](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-user_timeline.html) und [v1.1 statuses/mentions\_timeline](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-mentions_timeline.html). Wenn Sie Code, Apps oder Tools haben, die eine ältere Version dieses Endpunkts verwenden und eine Migration zum neueren X-API-v2-Endpunkt in Erwägung ziehen, ist dieser Leitfaden für Sie. Eine ausführlichere Migrationsanleitung finden Sie unter [Standard v1.1 migration to X API v2](/de/x-api/posts/timelines/migrate/standard-to-twitter-api-v2).

Diese Seite enthält drei Vergleichstabellen:

- Umgekehrt chronologische Home-Timeline
- Nutzer-Posts-Timeline
- Nutzer-Erwähnungen-Timeline

<div id="reverse-chronological-home-timeline">
  ### Umgekehrt chronologische Home-Timeline
</div>

|     |     |     |
| :--- | :--- | :--- |
| **Beschreibung** | **Standard v1.1** | **X-API v2** |
| Dokumentation | [API-Referenz](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-home_timeline) | [API-Referenz](https://developer.x.com/x-api/posts/timelines#user-mention-timeline-3) |
| Unterstützte HTTP-Methoden | `GET` | `GET` |
| Host-Domain | `https://api.x.com` | `https://api.x.com` |
| Endpunktpfade | `/1.1/statuses/home_timeline.json` | `/2/users/:id/timelines/reverse_chronological` |
| Erforderliche Parameter | `user_id` oder `screen_name` | Benutzer-ID als Pfadparameter :id |
| Authentifizierung | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />[OAuth 2.0 mit PKCE](/de/resources/fundamentals/authenticationoauth-2-0/authorization-code) |
| Anfrageraten-/Volumenlimits | 15 Anfragen pro 15 Minuten mit OAuth 1.0a User Context<br /><br />Anfragelimit: 100.000 innerhalb von 24 Stunden. | 180 Anfragen pro 15-Minuten-Fenster <br /><br />[Tweet-Limits (Post-Limit)](/de/x-api/fundamentals/post-cap):<br /><br />500.000 bei Essential-Zugang<br /><br />2 Millionen bei Elevated-Zugang<br /><br />10 Millionen bei Academic-Research-Zugang |
| Standard-Posts pro Antwort | 15  | 100 |
| Maximale Posts pro Antwort | 800 | Dieser Endpunkt liefert alle in den letzten 7 Tagen auf einer Timeline erstellten Posts sowie die jüngsten 800, unabhängig vom Erstellungsdatum. |
| Bietet Bearbeitungshistorie von Posts | ✔   | ✔   |
| Verfügbare historische Posts | Die jüngsten 800 Posts, einschließlich Retweets | Die jüngsten 3.200 Posts, einschließlich Retweets |
| Optionen zur Timeline-Navigation | since\_id (exklusiv) für Update-Polling<br /><br />`max_id` (inklusiv) | `start_time`<br /><br />`end_time`<br /><br />`since_id` (exklusiv) für Update-Polling <br /><br />`until_id` (exklusiv) |
| Optionale Parameter zur Verfeinerung der Ergebnisse | `count`<br /><br />`exclude_replies`<br /><br />`include_rts`<br /><br />`trim_user`<br /><br />`tweet_mode`<br /><br />`since_id`<br /><br />`max_id` | `max_results`<br /><br />`exclude` (retweets,replies)<br /><br />`tweet.fields`<br /><br />`user.fields`<br /><br />`place.fields`<br /><br />`media.fields`<br /><br />`poll.fields`<br /><br />`expansions`<br /><br />`start_time`<br /><br />`end_time`<br /><br />`since_id`<br /><br />`until_id` |
| Unterstützt das Anfordern und Empfangen von [Annotationen](/de/x-api/fundamentals/post-annotations) | N. v. | Wenn Annotationen in `tweet.fields` enthalten sind, werden die Ergebnisse mit abgeleiteten Annotationsdaten auf Basis des Post-Texts versehen, z. B. „Musikgenre“ und „Folk-Musik“ oder „Musiker“ und „Dolly Parton“. |
| Unterstützt das Anfordern und Empfangen spezifischer Post-[Metriken](/de/x-api/fundamentals/metrics) | N. v. | Wenn Metrikfelder in `tweet.fields` enthalten sind, werden die Ergebnisse pro Post mit `public_metrics` versehen, einschließlich `retweet_count`, `reply_count`, `quote_count` und `like_count`, sowie mit `non_public_metrics`, einschließlich `impression_count`, `user_profile_clicks`, `url_link_clicks`.<br /><br />Zusätzliche Medienmetriken wie `view_count` und Video-Wiedergabemetriken.<br /><br />Zusätzliche `organic_metrics` und `promoted_metrics` sind mit User Context für beworbene Posts verfügbar. |
| Unterstützt das Anfordern und Empfangen von [conversation\_id](/de/x-api/fundamentals/conversation-id) | N. v. | Gibt ein `conversation_id`-Feld zurück, dessen Wert den zuerst veröffentlichten Post in einem Antwort-Thread darstellt, um die Nachverfolgung von Konversationen zu erleichtern. |
| JSON-Format für Posts | [Standard-v1.1-Datenformat](https://developer.x.com/en/docs/twitter-api/v1/data-dictionary/overview.html) | [X-API-v2](/de/x-api/fundamentals/data-dictionary)-Format (gesteuert über die Request-Parameter fields und expansions; nicht abwärtskompatibel zu v1.1-Formaten)<br /><br />Weitere Informationen zur Migration vom Standard-v1.1-Format zum X-API-v2-Format finden Sie in unserem [Migrationsleitfaden für Datenformate](/de/x-api/migrate/data-format-migration). |
| Reihenfolge der Ergebnisse | Umgekehrt chronologisch | Umgekehrt chronologisch |
| Ergebnispaginierung | Nicht verfügbar – Navigation muss über die Post-ID erfolgen | Ergebnisse können vor- oder zurückgehend mit einem pagination\_token durchlaufen werden |
| Erfordert die Verwendung von Zugangsdaten aus einer [Developer-App](/de/resources/fundamentals/developer-apps), die einem [Projekt](/de/resources/fundamentals/projects) zugeordnet ist |     | ✔   |
