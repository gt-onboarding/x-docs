<div id="retweets-lookup">
  ### Retweets lookup
</div>

Mit dem Retweets-Lookup-Endpunkt können Sie eine Liste von Accounts abrufen, die einen Post retweetet haben. Für diesen Endpunkt werden Paginierungs-Token bereitgestellt, um große Ergebnismengen in Batches von bis zu 100 Nutzern zu durchlaufen.

Sie können diese Endpunkte wahlweise mit [OAuth 1.0a User Context](/de/resources/fundamentals/authentication), \[OAuth 2.0 App-Only]\(https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) oder [OAuth 2.0 Authorization Code with PKCE](/de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) authentifizieren. 

<div id="retweets-of-me">
  ### Retweets von mir
</div>

Die Abfrage „Reposts von mir“ zeigt Ihnen, welche Ihrer eigenen Posts von anderen geteilt wurden.

Dies gibt Nutzerinnen und Nutzern Einblicke, welche ihrer Posts geteilt werden und beim Publikum Anklang finden, was gezieltere Interaktionen und Anpassungen der Content-Strategie erleichtern kann.

Sie können diese Endpunkte entweder mit [OAuth 1.0a User Context](/de/resources/fundamentals/authentication) oder mit [OAuth 2.0 Authorization Code with PKCE](/de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) authentifizieren.

<div id="manage-retweets">
  #### Retweets verwalten
</div>

Die Endpunkte zum Verwalten von Retweets ermöglichen es Ihnen, im Namen eines authentifizierten Kontos einen bestimmten Post zu retweeten oder einen Retweet rückgängig zu machen. Für diese Endpunktgruppe stehen zwei Methoden zur Verfügung: POST und DELETE. Mit der POST-Methode können Sie einen Post retweeten, und mit der DELETE-Methode können Sie den Retweet eines bestimmten Posts rückgängig machen.

Da Sie Anfragen im Namen eines Nutzers stellen, müssen Sie diese Endpunkte entweder mit [OAuth 1.0a User Context](/de/resources/fundamentals/authentication) oder [OAuth 2.0 Authorization Code with PKCE](de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) authentifizieren und die Benutzer-Access-Tokens verwenden, die dem Nutzer zugeordnet sind, in dessen Namen Sie die Anfrage stellen. Sie können dieses Benutzer-Access-Token mithilfe des [3-legged OAuth-Flows](/de/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) oder des [Authorization Code with PKCE grant flow](/de/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)) (OAuth 2.0) erzeugen. Sie können einen Post von Ihrem Konto oder vom Konto eines authentifizierten Nutzers retweeten. Für beide Endpunkte gilt ein nutzerbezogenes Ratenlimit von 50 Anfragen pro 15 Minuten und Endpunkt.
