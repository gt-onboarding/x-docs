<div id="getting-started-with-the-manage-retweets-endpoints">
  ### Einstieg in die Endpunkte zum Verwalten von Retweets
</div>

Dieser Schnellstartleitfaden hilft Ihnen, Ihre erste Anfrage an die Endpunkte zum Verwalten von Retweets mit Postman zu senden.

Wenn Sie Beispielcode in verschiedenen Sprachen sehen möchten, besuchen Sie bitte unser GitHub-Repository mit [X-API v2-Beispielcode](https://github.com/xdevplatform/Twitter-API-v2-sample-code). 

<div id="prerequisites">
  #### Voraussetzungen
</div>

Um diese Anleitung abzuschließen, benötigen Sie einen Satz von [Keys und Tokens](/de/resources/fundamentals/authentication), um Ihre Anfrage zu authentifizieren. Sie können diese Keys und Tokens generieren, indem Sie die folgenden Schritte ausführen:

- [Registrieren Sie sich für ein Entwicklerkonto](https://developer.x.com/en/apply-for-access) und lassen Sie es genehmigen.
- Erstellen Sie ein [Projekt](/de/resources/fundamentals/projects) und eine zugehörige [Developer-App](/de/resources/fundamentals/developer-apps) im Developer-Portal.
- Navigieren Sie zur Seite „Keys and Tokens“ Ihrer App, um die erforderlichen Zugangsdaten zu generieren. Stellen Sie sicher, dass Sie alle Zugangsdaten an einem sicheren Ort speichern.

<div id="steps-to-build-a-manage-retweets-request">
  #### Schritte zum Erstellen einer Anfrage zum Verwalten von Retweets
</div>

**Schritt eins: Mit einem Tool oder einer Bibliothek beginnen**

Es gibt mehrere Tools, Codebeispiele und Bibliotheken, mit denen Sie eine Anfrage an diesen Endpoint stellen können. Wir verwenden hier jedoch Postman, um den Prozess zu vereinfachen.

Um die X-API-v2-Postman-Collection in Ihre Umgebung zu laden, klicken Sie bitte auf die folgende Schaltfläche:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X-API v2 zu Postman hinzufügen
</Button>

Sobald Sie die X-API-v2-Collection in Postman geladen haben, navigieren Sie zum Ordner „Retweets“ und wählen Sie „Retweet a Post“.
 

Schritt zwei: Authentifizieren Sie Ihre Anfrage

Um eine Anfrage an die X-API korrekt zu stellen, müssen Sie sicherstellen, dass Sie die erforderlichen Berechtigungen haben. Bei diesem Endpoint müssen Sie Ihre Anfrage entweder mit [OAuth 1.0a User Context](/de/resources/fundamentals/authentication) oder [OAuth 2.0 Authorization Code with PKCE](de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) authentifizieren.

In diesem Beispiel verwenden wir OAuth 1.0a User Context.

Sie müssen Ihre Keys und Tokens – insbesondere Ihren API Key, API Secret Key, OAuth 1.0a User Access Token und OAuth 1.0a User Access Token Secret – in Postman hinzufügen. Wählen Sie dazu oben rechts in Postman die Umgebung „X API v2“ und tragen Sie Ihre Keys und Tokens in die Felder „initial value“ und „current value“ ein (klicken Sie auf das Augensymbol neben dem Umgebungs-Dropdown).

Diese Variablen werden automatisch in den Tab „Authorization“ der Anfrage übernommen, wenn Sie dies korrekt eingerichtet haben.

**Schritt drei: Geben Sie an, welchen Post Sie retweeten werden**

Endpoints zum Verwalten von Retweets erfordern zwei IDs: eine für den Nutzer, der einen Retweet durchführen oder rückgängig machen möchte, und die ID des Posts.

Die ID des Nutzers muss mit der ID des authentifizierten Nutzers übereinstimmen. Das bedeutet, dass Sie bei der Authentifizierung Ihrer Anfrage die [Access Tokens](/de/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) übermitteln müssen, die mit dieser Nutzer-ID verknüpft sind. In diesem Fall können Sie die ID Ihres eigenen Accounts angeben. Sie können Ihre ID auf zwei Arten finden:

1. Mit dem Endpoint [user lookup by username](/de/x-api/users/user-lookup-by-username) können Sie einen Benutzernamen übergeben und erhalten das Feld id. 
2. Im Access Token ist der numerische Teil Ihre Nutzer-ID.
    

Sie müssen außerdem einen Post angeben, den Sie retweeten möchten. Sie finden die Post-ID, indem Sie zu X.com navigieren, einen Post öffnen und in der URL nachsehen. Beispielsweise lautet die Post-ID der folgenden URL 1228393702244134912.

https://x.com/TwitterDev/status/1228393702244134912

Navigieren Sie in Postman zum Tab „Params“ und geben Sie Ihre ID in die Spalte „Value“ der Pfadvariablen id ein. Geben Sie außerdem die ID des Posts, den Sie retweeten möchten, als Wert für den Parameter tweet\_id ein. Achten Sie darauf, keine Leerzeichen vor oder nach einer ID einzufügen.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | (Ihre Nutzer-ID) |
| tweet\_id | (die ID des Tweets, den Sie retweeten möchten) |

Wenn Sie auf die Schaltfläche „Send“ klicken, erhalten Sie ein Response-Objekt, das den Status der Beziehung enthält:

- Wenn Sie „retweeted“: true erhalten, retweetet die id das tweet\_id erfolgreich.
   

**Schritt vier: Senden Sie Ihre Anfrage und prüfen Sie Ihre Antwort**

Sobald alles eingerichtet ist, klicken Sie auf die Schaltfläche „Send“ und Sie erhalten die folgende Antwort:

```
{
    "data": {
        "retweeted": true
    }
}
```

Wenn Sie „retweeted“: true erhalten, wurde das tweet\_id erfolgreich retweetet.

Wenn Sie einen Retweet desselben Nutzers rückgängig machen möchten, können Sie die Anfrage „Undo a Retweet“ verwenden, die sich ebenfalls im Ordner „Retweets“ der in Postman geladenen X-API-v2-Collection befindet. Die `id` sollte Ihre Nutzer-ID sein und `source_tweet_id` die Post-ID, deren Retweet Sie rückgängig machen möchten. Sie müssen dies nicht als JSON-Body hinzufügen, stellen Sie also sicher, dass Sie die erforderlichen Query-Parameter für id und `source_tweet_id` setzen. 

Bei einem erfolgreichen Rückgängigmachen eines Retweets erhalten Sie eine ähnliche Antwort wie im folgenden Beispiel:

```
{
  "data": {
    "retweeted": false
  }
}
```
