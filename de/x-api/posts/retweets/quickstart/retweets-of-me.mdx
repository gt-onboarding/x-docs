Die Abfrage „Reposts von mir“ ermöglicht es Ihnen, zu sehen, welche Ihrer eigenen Posts von anderen geteilt wurden.

Dies gibt Nutzerinnen und Nutzern Einblicke, welche ihrer Posts geteilt werden und bei ihrem Publikum Anklang finden, was gezieltere Interaktionen und Anpassungen der Content-Strategie erleichtern kann.

<div id="authentication">
  #### Authentifizierung
</div>

Da Sie Anfragen im Namen eines Nutzers stellen, müssen Sie diese Endpunkte entweder mit [OAuth 1.0a User Context](/de/resources/fundamentals/authentication) oder [OAuth 2.0 Authorization Code with PKCE](/de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) authentifizieren und die Benutzer-Access-Tokens verwenden, die dem Nutzer zugeordnet sind, in dessen Namen Sie die Anfrage stellen. Sie können dieses Benutzer-Access-Token über den [3-legged OAuth Flow](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens) (OAuth 1.0a) oder den [Authorization Code with PKCE Grant Flow](/de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) (OAuth 2.0) erzeugen.

<div id="making-a-request">
  #### Eine Anfrage senden
</div>

Sobald sich ein Nutzer bei Ihrer App authentifiziert hat, können Sie die Retweets-of-Me-API im Namen des Nutzers wie unten gezeigt aufrufen:

```bash
      curl 'https://api.x.com/2/users/reposts_of_me' --header 'Authorization: ••••••'
```
