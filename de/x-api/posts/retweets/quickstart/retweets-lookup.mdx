<div id="getting-started-with-the-retweets-lookup-endpoint">
  ### Einstieg in den Retweets-Lookup-Endpunkt
</div>

Diese Kurzanleitung hilft Ihnen, Ihre erste Anfrage an den Retweets-Lookup-Endpunkt mit [Postman](/de/tutorials/postman-getting-started) zu senden. Wenn Sie Beispielcode in verschiedenen Sprachen ansehen möchten, besuchen Sie unser GitHub-Repository mit [Beispielcode für die X-API v2](https://github.com/xdevplatformTwitter-API-v2-sample-code).

<div id="prerequisites">
  #### Voraussetzungen
</div>

Um diesen Leitfaden abzuschließen, benötigen Sie eine Reihe von [Keys und Tokens](/de/resources/fundamentals/authentication), um Ihre Anfrage zu authentifizieren. Sie können diese Keys und Tokens generieren, indem Sie die folgenden Schritte ausführen:

- [Registrieren Sie sich für ein Entwicklerkonto](https://developer.x.com/en/apply-for-access) und lassen Sie es genehmigen.
- Erstellen Sie ein [Projekt](/de/resources/fundamentals/projects) und eine zugehörige [Developer-App](/de/resources/fundamentals/developer-apps) im Developer-Portal.
- Navigieren Sie zur „Keys and tokens“-Seite Ihrer App, um die erforderlichen Zugangsdaten zu generieren. Speichern Sie alle Zugangsdaten unbedingt an einem sicheren Ort.

<div id="steps-to-build-a-retweets-lookup-request">
  #### Schritte zum Erstellen einer Retweet-Lookup-Anfrage
</div>

**Schritt 1: Starten Sie mit einem Tool oder einer Bibliothek**

Es gibt verschiedene Tools, Codebeispiele und Bibliotheken, mit denen Sie Anfragen an diesen Endpunkt senden können. Hier verwenden wir Postman, um den Prozess zu vereinfachen.

Um die Postman-Collection der X-API v2 in Ihre Umgebung zu laden, klicken Sie bitte auf die folgende Schaltfläche:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X-API v2 in Postman hinzufügen
</Button>

Sobald Sie die X-API-v2-Sammlung in Postman geladen haben, navigieren Sie zum Ordner „Retweets“ und wählen Sie „Retweeted by“ aus.

**Schritt zwei: Authentifizieren Sie Ihre Anforderung**

Um eine Anfrage an die X-API korrekt zu stellen, müssen Sie sicherstellen, dass Sie über die erforderlichen Berechtigungen verfügen. Dieser Endpunkt erfordert dafür die Authentifizierung Ihrer Anfrage mit entweder [OAuth 2.0 App-Only](/de/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), [OAuth 2.0 mit PKCE](de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) oder [OAuth 1.0a Benutzerkontext](/de/resources/fundamentals/authentication).

Der Einfachheit halber verwenden wir für diese Anfrage OAuth 2.0 App-Only. Wenn Sie jedoch private [Metriken](/de/x-api/fundamentals/metrics) oder Retweets anfordern möchten, müssen Sie eine der anderen Authentifizierungsmethoden nutzen.

Um OAuth 2.0 App-Only zu verwenden, müssen Sie Ihre Keys und Tokens hinzufügen, insbesondere den [App Access Token](/de/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token) (auch als App-only Bearer Token bekannt) in Postman. Wählen Sie dazu oben rechts in Postman die Umgebung „X-API v2“ aus und fügen Sie Ihre Keys und Tokens in die Felder „Initial value“ und „Current value“ ein (indem Sie auf das Augensymbol neben der Umgebungs-Auswahlliste klicken).

Diese Variablen werden automatisch in den Autorisierungsreiter der Anfrage übernommen, wenn Sie alles richtig eingerichtet haben.

**Schritt drei: Einen Post festlegen**

Mit diesem Endpunkt müssen Sie die Post-ID angeben, zu der Sie die Nutzer abrufen möchten, die retweetet haben. Sie finden die ID eines Posts, indem Sie den jeweiligen Post auf X öffnen und den numerischen Code am Ende der URL entnehmen. Zum Beispiel lautet die Post-ID der folgenden URL 1354143047324299264.

https://x.com/TwitterDev/status/1354143047324299264

Navigieren Sie in Postman zur Registerkarte „Params“ und geben Sie diesen Benutzernamen in die Spalte „Value“ der Pfadvariablen „id“ ein (am unteren Ende des Abschnitts). Achten Sie darauf, vor oder nach Benutzernamen keine Leerzeichen einzufügen.

|     |     |
| :--- | :--- |
| **Schlüssel** | **Wert** |
| `id` | Die Beitrags-ID, für die Sie die Nutzer abrufen möchten, die den Beitrag retweetet haben |

**Schritt vier: Bestimmen und angeben, welche Felder Sie abrufen möchten**

Wenn Sie nach Schritt drei auf die Schaltfläche „Senden“ klicken, erhalten Sie in der Antwort die Standardfelder des [Benutzerobjekts](/de/x-api/fundamentals/data-dictionary#user): id, name und username.

Wenn Sie über id, name und username hinaus zusätzliche Felder erhalten möchten, müssen Sie diese in Ihrer Anfrage über die Parameter [fields](/de/x-api/fundamentals/fields) und/oder [expansions](/de/x-api/fundamentals/expansions) anfordern.

In dieser Übung werden wir drei zusätzliche Felder-Sets aus verschiedenen Objekten anfordern:

1. Das zusätzliche Feld user.created\_at in den primären User-Objekten.
2. Die Standardfelder des zugehörigen angehefteten Post-Objekts für die zurückgegebenen Nutzer: id und text.
3. Das zusätzliche Feld tweet.created\_at in den zugehörigen Post-Objekten.

Navigieren Sie in Postman zur Registerkarte „Params“ und fügen Sie der Tabelle „Query Params“ das folgende Schlüssel-Wert-Paar hinzu:

|     |     |     |
| :--- | :--- | :--- |
| **Schlüssel** | **Wert** | **Zurückgegebene Felder** |
| user.fields | created\_at | user.created\_at |
| expansions | pinned\_tweet\_id | tweet.id, tweet.text |
| tweet.fields | created\_at | tweet.created\_at |

Sie sollten nun die folgende URL neben dem Button „Senden“ sehen:

`https://api.x.com/2/tweets/1354143047324299264/retweeted_by?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=created_at`
