<div id="authentication">
  #### Authentifizierung
</div>

Alle Endpunkte der X-API v2 erfordern, dass Sie Ihre Anfragen mit Anmeldedaten (Schlüsseln und Tokens) authentifizieren.

Für Anfragen an den Retweets-Lookup-Endpunkt können Sie entweder OAuth 1.0a User Context oder ein OAuth 2.0 Bearer Token verwenden.

Die Endpunkte zum Verwalten von Retweets erfordern OAuth 1.0a User Context. Das bedeutet, Sie müssen API-Schlüssel und Benutzerzugriffstokens verwenden, um eine erfolgreiche Anfrage zu stellen. Die Zugriffstokens müssen dem Benutzer zugeordnet sein, in dessen Namen Sie die Anfrage stellen. Wenn Sie Zugriffstokens für einen anderen Benutzer generieren möchten, muss dieser Ihre App über den 3-legged OAuth-Flow autorisieren bzw. sich authentifizieren.

Bitte beachten Sie, dass OAuth 1.0a in der Anwendung anspruchsvoll sein kann. Wenn Sie mit dieser Authentifizierungsmethode nicht vertraut sind, empfehlen wir, eine [Bibliothek](/de/x-api/tools-and-libraries/overview) zu verwenden, um Ihre Anfragen korrekt zu authentifizieren.

**Bitte beachten**

Wenn Sie die folgenden Felder anfordern, ist OAuth 1.0a User Context erforderlich:

- tweet.fields.non\_public\_metrics
- tweet.fields.promoted\_metrics
- tweet.fields.organic\_metrics
- media.fields.non\_public\_metrics
- media.fields.promoted\_metrics
- media.fields.organic\_metrics

<div id="developer-portal-projects-and-developer-apps">
  #### Developer-Portal, Projekte und Entwickler-Apps
</div>

Um einen Satz an Authentifizierungsdaten zu erhalten, der mit den Endpunkten der X-API v2 funktioniert, müssen Sie sich [für ein Entwicklerkonto registrieren](https://developer.x.com/en/portal/petition/essential/basic-info), innerhalb dieses Kontos ein [Projekt](/de/resources/fundamentals/projects) einrichten und innerhalb dieses Projekts eine [Entwickler-App](/de/resources/fundamentals/developer-apps) erstellen. Anschließend finden Sie Ihre Keys und Tokens in Ihrer Entwickler-App. 
 

<div id="rate-limits">
  #### Rate Limits
</div>

Jeden Tag stellen viele tausend Entwickler Anfragen an die X-API. Um das hohe Volumen dieser Anfragen zu verwalten, gelten für jeden Endpunkt [Rate Limits](/de/x-api/fundamentals/rate-limits), die die Anzahl der Anfragen begrenzen, die Sie im Namen Ihrer App oder eines authentifizierten Nutzers stellen können.

Die Endpunkte zum Verwalten von Retweets sind auf 50 Anfragen pro 15 Minuten (pro Nutzer) begrenzt. Zusätzlich sind Sie für den POST-Endpunkt auf 300 Anfragen pro 3-Stunden-Fenster (pro Nutzer, pro App) begrenzt. 

Beim Retweets-Lookup-Endpunkt sind Sie auf 75 Anfragen pro 15-Minuten-Fenster begrenzt. Außerdem gibt dieser Endpunkt nur die 100 zuletzt retweetenden Nutzer zurück.

<div id="fields-and-expansions">
  #### Felder und Erweiterungen
</div>

Die X-API v2 ermöglicht es Nutzerinnen und Nutzern, mit einer Reihe von Werkzeugen namens Felder und Erweiterungen genau auszuwählen, welche Daten sie von der API zurückerhalten möchten. Der Parameter expansion ermöglicht es, in der Nutzlast referenzierte Objekte aufzulösen. Beispielsweise erlaubt dieser Endpunkt das Abrufen der folgenden [Erweiterungen](/de/x-api/fundamentals/expansions):

- attachments.poll\_ids
- attachments.media\_keys
- author\_id, entities.mentions.username
- geo.place\_id
- in\_reply\_to\_user\_id,
- referenced\_tweets.id,
- referenced\_tweets.id.author\_id

Der Parameter fields ermöglicht es Ihnen, genau die [Felder](/de/x-api/fundamentals/fields) innerhalb der verschiedenen Datenobjekte auszuwählen, die Sie erhalten möchten. Diese Endpunkte liefern primär Post-Objekte. Standardmäßig gibt das Post-Objekt die Felder id und text zurück. Um zusätzliche Felder wie tweet.created\_at oder tweet.entities zu erhalten, müssen Sie diese explizit über einen fields-Parameter anfordern. Einige wichtige Felder, die Sie für Ihre Integration berücksichtigen sollten, sind unsere Umfragedaten, Metriken, Post‑Annotationen und die Konversations-ID.

Wir haben in unserem [Datenwörterbuch zur X-API v2](/de/x-api/fundamentals/data-dictionary) einen Leitfaden hinzugefügt, wie Sie [Felder und Erweiterungen verwenden](/de/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) können.
