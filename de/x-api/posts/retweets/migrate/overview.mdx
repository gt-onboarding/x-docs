<div id="comparing-x-apis-retweets-endpoints">
  ## Vergleich der Retweets-Endpunkte der X-API
</div>

**Retweets-Abfrage**

Der v2-Endpunkt für die Retweets-Abfrage ersetzt die Standard-Endpunkte [v1.1 GET statuses/retweets/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweets-id) und [v1.1 GET statuses/retweets/:ids](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweeters-ids).

Die folgenden Tabellen vergleichen die Retweets-Endpunkte von Standard v1.1 und X-API v2:

| Beschreibung | Standard v1.1 | X-API v2 |
| :--- | :--- | :--- |
| Unterstützte HTTP-Methoden | GET | GET |
| Host-Domain | https://api.x.com | https://api.x.com |
| Endpoint-Pfad | /1.1/retweeters/id.json<br /><br />`/1.1/retweets/ids.json` | /2/users/:id/retweeted\_by |
| [Authentifizierung](/de/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 2.0 Bearer Token<br /><br />OAuth 1.0a User Context |
| Standard-[Rate-Limits](/de/resources/fundamentals/rate-limits) für Anfragen | 75 Anfragen pro 15 Min. | 75 Anfragen pro 15 Min. (pro App)<br /><br />75 Anfragen pro 15 Min. (pro Nutzer) |
| Datenformat | Standard-v1.1-Format | [X-API-v2-Format](/de/x-api/fundamentals/data-dictionary) (bestimmt durch die Request-Parameter fields und expansions, nicht rückwärtskompatibel mit v1.1-Formaten)<br /><br />Weitere Informationen zur Migration vom Standard-v1.1-Format zum X-API-v2-Format finden Sie in unserem [Leitfaden zur Datenformat-Migration](/de/x-api/migrate/data-format-migration). |
| Erfordert die Verwendung von Zugangsdaten aus einer [Developer-App](/de/resources/fundamentals/developer-apps), die einem [Project](/de/resources/fundamentals/projects) zugeordnet ist |     | ✔️  |

<div id="manage-retweets">
  ### Retweets verwalten
</div>

Die folgenden Tabellen vergleichen den Standard-v1.1- und den X-API-v2-Endpunkt zum Rückgängigmachen von Retweets:

**Einen Post retweeten**

| Beschreibung | Standard v1.1 | X-API v2 |
| :--- | :--- | :--- |
| Unterstützte HTTP-Methoden | POST | POST |
| Host-Domain | https://api.x.com | https://api.x.com |
| Endpoint-Pfad | /1.1/statuses/retweet/:id.json | /2/users/:id/retweets |
| [Authentifizierung](/de/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context |
| Standardmäßige [Rate Limits](/de/resources/fundamentals/rate-limits) für Anfragen | Keine<br /><br />300 Anfragen pro 3-Stunden-Fenster (pro Nutzer, pro App). Dies wird mit dem POST-Tweet-Endpoint geteilt | 50 Anfragen pro 15 Min (pro Nutzer)<br /><br />300 Anfragen pro 3-Stunden-Fenster (pro Nutzer, pro App). Dies wird mit dem POST-Tweet-Endpoint für das Verwalten von Posts geteilt. |
| Erfordert die Verwendung von Zugangsdaten aus einer [Developer-App](/de/resources/fundamentals/developer-apps), die einem [Projekt](/de/resources/fundamentals/projects) zugeordnet ist |     | ✔️  |
