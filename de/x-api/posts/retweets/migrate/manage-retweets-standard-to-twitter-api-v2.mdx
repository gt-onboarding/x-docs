<div id="manage-retweets-standard-v11-compared-to-x-api-v2">
  ### Retweets verwalten: Standard v1.1 im Vergleich zur X-API v2
</div>

Wenn Sie mit den Endpunkten der Standardversion v1.1 [POST statuses/retweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-retweet-id) und [POST statuses/unretweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-unretweet-id) gearbeitet haben, soll Ihnen dieser Leitfaden helfen, die Gemeinsamkeiten und Unterschiede zwischen den Standard- und den X-API v2-Retweet-Endpunkten zu verstehen.

- **Gemeinsamkeiten**
  - Authentifizierung
- **Unterschiede**
  - Endpoint-URLs und HTTP-Methoden
  - Anfragelimits
  - Anforderungen an App und Projekt
  - Anfrageparameter

<div id="similarities">
  #### Gemeinsamkeiten
</div>

**Authentifizierung**

Sowohl die Endpunkte zur Verwaltung von Retweets in der Standard-API v1.1 ([POST statuses/retweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-retweet-id) und [POST statuses/unretweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-unretweet-id)) als auch in der X-API v2 verwenden [OAuth 1.0a User Context](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Wenn Sie zuvor einen der Retweet-Lookup-Endpunkte der Standard-API v1.1 verwendet haben, können Sie beim Umstieg auf die X-API v2 dieselbe Authentifizierungsmethode weiter nutzen. 

<div id="differences">
  #### Unterschiede
</div>

**Endpoint-URLs und HTTP-Methoden**

- Standard-v1.1-Endpoints:
  - https://api.x.com/1.1/statuses/retweet/:id.json
    (Erstellt einen Retweet zu einem Post. Gibt den ursprünglichen Post mit eingebetteten Retweet-Details zurück)
  - https://api.x.com/1.1/statuses/unretweet/:id.json
    (Macht einen Retweet rückgängig. Gibt den ursprünglichen Post mit eingebetteten Retweet-Details zurück)
- X-API-v2-Endpoint:
  - https://api.x.com/2/tweets/:id/retweets
    (Retweetet einen bestimmten Post)
  - https://api.x.com/2/users/:id/retweets/:source\_tweet\_id
    (Macht den Retweet eines bestimmten Posts rückgängig)

**App- und Projektanforderungen**

Die X-API-v2-Endpoints setzen voraus, dass Sie bei der Authentifizierung Ihrer Anfragen Anmeldedaten aus einer [Developer-App](/de/resources/fundamentals/developer-apps) verwenden, die einem [Projekt](/de/resources/fundamentals/projects) zugeordnet ist. Alle X-API-v1.1-Endpoints können Anmeldedaten von eigenständigen Apps oder von Apps verwenden, die einem Projekt zugeordnet sind.

**Anfrageparameter**

Die folgenden Standard-v1.1-Endpunkte akzeptierten zwei Abfrageparameter (user\_id oder screen\_name). Die X-API v2 akzeptiert nur die numerische Benutzer-ID, und diese muss als Teil des Endpoint-Pfads übergeben werden.

| Standard v1.1 | X API v2 |
| :--- | :--- |
| **id** | **id** |
| **includes\_entities** | Kein Pendant |

Bitte beachten Sie, dass die Standard-v1.1-Parameter als Query-Parameter übergeben werden, während die X-API-v2-Parameter als Body-Parameter für den POST-Endpoint bzw. als Pfadparameter für den DELETE-Endpoint übergeben werden.

Außerdem ist bei Verwendung der Standard-v1.1-Endpoints keine ID des Nutzers erforderlich, der einen Post retweetet, da die mit dem [OAuth 1.0a User Context](/de/resources/fundamentals/authentication) übergebenen [Access Tokens](/de/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) erkennen lassen, welcher Nutzer den Retweet initiiert bzw. rückgängig macht.
