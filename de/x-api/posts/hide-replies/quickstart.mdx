<div id="getting-started-with-the-hide-replies-endpoint">
  ## Einstieg in den Endpunkt zum Ausblenden von Antworten
</div>

Diese Schnellstartanleitung hilft Ihnen, Ihre erste Anfrage an den Endpunkt zum Ausblenden von Antworten mit [Postman](/de/tutorials/postman-getting-started) zu stellen.

Wenn Sie Codebeispiele in verschiedenen Sprachen sehen möchten, besuchen Sie bitte die [API-Referenzseite zum Ausblenden von Antworten](/de/x-api/posts/hide-replies#put-2-tweets-id-hidden).

<Note>
  ### Voraussetzungen

  Um diese Anleitung abzuschließen, benötigen Sie einen Satz an [Keys und Tokens](/de/resources/fundamentals/authentication), um Ihre Anfrage zu authentifizieren. Sie können diese Keys und Tokens generieren, indem Sie die folgenden Schritte ausführen:

  - [Für ein Entwicklerkonto registrieren](https://developer.x.com/en/apply-for-access) und die Genehmigung erhalten.
  - Ein [Projekt](/de/resources/fundamentals/projects) und eine zugehörige [Developer-App](/de/resources/fundamentals/developer-apps) im Developer-Portal erstellen.
  - Zur Seite „Keys and tokens“ Ihrer App navigieren, um die erforderlichen Zugangsdaten zu generieren. Stellen Sie sicher, dass Sie alle Zugangsdaten an einem sicheren Ort speichern.
</Note>

<div id="steps-to-build-a-put-tweetsidhidden-request">
  ### Schritte zum Erstellen einer PUT-/tweets/:id/hidden-Anforderung
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### Schritt eins: Beginnen Sie mit einem Tool oder einer Bibliothek
</div>

Es gibt verschiedene Tools, Codebeispiele und Bibliotheken, mit denen Sie eine Anfrage an diesen Endpoint stellen können. Wir verwenden hier Postman, um den Prozess zu vereinfachen.

Um die X-API-v2-Postman-Collection in Ihre Umgebung zu laden, klicken Sie bitte auf die folgende Schaltfläche:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X-API v2 zu Postman hinzufügen
</Button>

Sobald Sie die X-API-v2-Collection in Postman geladen haben, navigieren Sie zum Endpoint „hide replies“.

<div id="step-two-authenticate-your-request">
  #### Schritt zwei: Authentifizieren Sie Ihre Anfrage
</div>

Um eine Anfrage an die X-API korrekt zu stellen, müssen Sie nachweisen, dass Sie die erforderliche Berechtigung haben. Bei diesem Endpoint müssen Sie Ihre Anfrage entweder mit [OAuth 1.0a User Context](/de/resources/fundamentals/authentication) oder mit [OAuth 2.0 Authorization Code mit PKCE](de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) authentifizieren.

In diesem Beispiel verwenden wir OAuth 1.0a User Context.

Fügen Sie Ihre Schlüssel und Token – konkret Ihren API Key, API Secret Key, den OAuth 1.0a User Access Token und den OAuth 1.0a User Access Token Secret – in Postman hinzu. Wählen Sie dazu oben rechts in Postman die Umgebung „X API v2“ aus und tragen Sie Ihre Schlüssel und Token in die Felder „initial value“ und „current value“ ein (indem Sie auf das Augensymbol neben der Umgebungs-Dropdown-Liste klicken).

Diese Variablen werden automatisch in den Autorisierungs-Tab der Anfrage übernommen, wenn Sie dies korrekt eingerichtet haben.
 

<div id="step-three-find-a-post-id-to-hide">
  #### Schritt drei: Eine Post-ID zum Ausblenden finden
</div>

Der Endpoint zum Ausblenden von Antworten kann im Namen eines autorisierten Nutzers Antworten aus- oder wieder einblenden. Da wir in diesem Beispiel die Access Tokens verwenden, die mit deinem Nutzerprofil verknüpft sind, kannst du Antworten von Nutzern ausblenden, die an einer von dir gestarteten Unterhaltung teilnehmen. Wenn du hingegen Access Tokens eines anderen Nutzers verwenden würdest, der deine App autorisiert hat, könntest du Antworten in allen von diesem Konto gestarteten Unterhaltungen moderieren.

Bitte eine Freundin oder einen Freund, auf einen Post zu antworten (weise darauf hin, dass du das Ausblenden von Antworten testest), oder antworte von einem Testkonto auf einen deiner Posts. Klicke auf diese Antwort und kopiere dann den numerischen Teil ihrer URL. Das ist die Post-ID, die wir ausblenden werden.

In diesem Fall betrachten wir den folgenden Post mit der ID `1232720193182412800`:

`https://x.com/TwitterDev/status/1232720193182412800`

<div id="step-four-hide-the-post">
  #### Schritt vier: Beitrag ausblenden
</div>

Öffnen Sie in Postman den Ordner Hide replies und wählen Sie Hide a reply. Fügen Sie im Tab „Params“ die Beitrags-ID neben dem Feld „id“ ein (Sie müssen „:id“ in der URL nicht ersetzen). Klicken Sie auf „Send“ und Sie erhalten eine erfolgreiche Antwort.

`{"hidden":true}`

<div id="step-five-unhide-the-post">
  #### Schritt fünf: Post einblenden
</div>

Ausgeblendete Posts werden in der X App in einen separaten Tab verschoben. Um einen Post in Postman einzublenden, öffnen Sie den Ordner Hide replies und wählen Sie Unhide a reply. Fügen Sie im Tab Params dieselbe Post-ID aus dem vorherigen Schritt in das Feld id ein. Klicken Sie auf „Send“ und Sie sehen eine erfolgreiche Antwort.

`{"hidden":false}`

Das Feld hidden gibt den Ausblendstatus des Posts an. Ein Status von true bedeutet, dass der Post ausgeblendet ist. Entsprechend bedeutet false, dass der Post nicht ausgeblendet ist.
