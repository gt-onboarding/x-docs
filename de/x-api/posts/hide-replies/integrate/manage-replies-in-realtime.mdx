<div id="manage-replies-in-realtime">
  ### Antworten in Echtzeit verwalten
</div>

Mit dem Endpunkt zum Ausblenden von Antworten können Sie einen Workflow erstellen, der Ihren Nutzern hilft, Antworten mit sehr hoher Wahrscheinlichkeit auf Irrelevanz auszublenden.

Nützliche Apps kombinieren oft mehrere Technologien, um ihren Nutzern einen Mehrwert zu bieten. Diese Seite zeigt, wie man Antworten mithilfe der [Perspective API](https://www.perspectiveapi.com/) ausblendet. Diese API ist eine von [Jigsaw](https://jigsaw.google.com/), einer Einheit innerhalb von Google, trainierte künstliche Intelligenz zur Erkennung toxischer Kommentare. Die Anwendungslogik funktioniert wie folgt:

1.  Sie bittet den Nutzer um Erlaubnis, seine Posts zu lesen sowie seine Antworten auszublenden oder wieder einzublenden.
2.  Sie verwendet die [Account Activity API](https://developer.x.com/en/docs/x-api/v1/accounts-and-users/subscribe-account-activity/overview), um eingehende Antworten zu erkennen.
3.  Sie fordert von der Perspective API eine „Bewertung“ (eine Zahl zwischen 0 und 1) an, die angibt, wie sicher der Algorithmus ist, dass ein Kommentar toxischen Kommentaren ähnelt, die er in der Vergangenheit gesehen hat. (Perspective speichert den an den Dienst übermittelten Text nicht.)
4.  Sie ruft den Endpunkt zum Ausblenden von Antworten auf, wenn die Bewertung des Algorithmus sehr hoch ist.

<div id="strive-for-transparency">
  #### Streben Sie nach Transparenz
</div>

Da Perspective nicht auf tatsächlichen Posts trainiert ist, können bestimmte Sprachnuancen dazu führen, dass diese App eine Antwort ausblendet, die ein Nutzer sichtbar lassen möchte. Unabhängig von der Technologie oder dem Ansatz, den Sie beim Entwurf Ihrer App verwenden, sollten Sie stets bestmögliche Anstrengungen unternehmen, damit Ihre Nutzer verstehen, was Ihre App ausgeblendet hat, und Änderungen vornehmen können.

* Am besten ist es, dem Nutzer zu vertrauen und ihm die volle Kontrolle über seine Entscheidungen zu geben. Das bedeutet, dass Ihre User Experience Bedienelemente enthalten sollte, mit denen sich jede von Ihrer App im Namen des Nutzers vorgenommene Aktion rückgängig machen lässt.
* Beim Einsatz künstlicher Intelligenz sollte Ihre App eine sehr hohe Vertrauensschwelle verwenden, um Posts zu erkennen und auszublenden.
* Nicht alle verwenden dieselben Wörter, und Ihre App sollte so gestaltet sein, dass Vorurteile vermieden werden. Achten Sie auf reclaimte Begriffe und Slang, die zu Fehlalarmen führen können.
* Wenn Sie eine künstliche Intelligenz trainieren, sollten Sie ein Modell wählen, das die auf X häufig verwendete Sprache möglichst genau widerspiegelt.
