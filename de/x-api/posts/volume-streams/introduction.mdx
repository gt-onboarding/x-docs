- 1%-Stichprobenstream.
- 10%-Stichprobenstream. Häufig als „Decahose“ bezeichnet.

Diese werden als „Volumenstreams“ bezeichnet, da beide große Datenmengen liefern. Selbst der 1%-Stream kann pro Sekunde viele Dutzend Posts ausgeben. Mit diesen Streams können Sie Trends identifizieren und verfolgen, die allgemeine Stimmung beobachten, globale Ereignisse überwachen und vieles mehr.

Diese Streaming-Endpunkte liefern [Post-Objekte](/de/x-api/fundamentals/data-dictionary#tweet) über eine persistente HTTP-GET-Verbindung und verwenden die Authentifizierung [OAuth 2.0 App-Only](/de/resources/fundamentals/authentication#oauth-2-0). Mit Essential-Zugang können Sie jeweils nur eine Verbindung gleichzeitig halten. Bei allen Zugriffsstufen können Verbindungsanfragen bis zu 50-mal pro 15-Minuten-Fenster gestellt werden.

Diese Volumenstream-Endpunkte unterstützen bearbeitete Posts. Sie liefern bearbeitete Posts zusammen mit deren Bearbeitungsverlauf, einschließlich eines Arrays von Post-IDs. Für Posts ohne Bearbeitungsverlauf enthält dieses Array eine einzelne ID. Für bearbeitete Posts enthält dieses Array mehrere IDs in aufsteigender Reihenfolge der Bearbeitungen; die neueste Version steht an letzter Position im Array. Weitere Informationen zur Funktionsweise von Post-Bearbeitungen finden Sie auf der Seite [Grundlagen zu Post-Bearbeitungen](/de/x-api/fundamentals/edit-posts).

_Um diese APIs zu verwenden, müssen Sie zunächst ein Konto über unser Enterprise-Vertriebsteam einrichten._

<Note>
  **Kontoeinrichtung**

  Für den Zugriff auf diese Endpunkte benötigen Sie:

  - Ein genehmigtes [Entwicklerkonto](https://developer.x.com/en/portal/petition/essential/basic-info).
  - Eine Authentifizierung mit den Schlüsseln und Token aus einer [Developer-App](/de/resources/fundamentals/developer-apps), die sich innerhalb eines [Projekts](/de/resources/fundamentals/projects) befindet.

  Erfahren Sie mehr über den Zugriff auf die Endpunkte der X-API v2 in unserem [Leitfaden für den Einstieg](/de/x-api/getting-started/getting-access).
</Note>
