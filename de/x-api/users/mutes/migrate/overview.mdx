  ## Vergleich der Mute-Endpunkte der X-API
</div>

<div id="mutes-lookup">
  #### Lookup für Stummschaltungen
</div>

Der v2-Endpunkt für das Lookup von stummschaltenden Beziehungen ersetzt die Standard-Endpunkte [v1.1 GET mutes/users/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/get-mutes-users-ids) und [GET mutes/users/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/get-mutes-users-list).

Die folgenden Tabellen vergleichen die Standard-v1.1-Endpunkte und die X-API-v2-Endpunkte für Stummschaltungen:
 

| Beschreibung | Standard v1.1 | X-API v2 |
| :--- | :--- | :--- |
| Unterstützte HTTP-Methoden | ******GET****** | ******GET****** |
| Host-Domain | ******https://api.x.com****** | ******https://api.x.com****** |
| Endpunktpfad | ******/1.1/mutes/users/ids.json******<br /><br />/1.1/mutes/users/list.json | ******/2/users/:id/muting****** |
| [Authentifizierung](/de/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code mit PKCE |
| Standard-Request-[Ratenlimits](/de/resources/fundamentals/rate-limits) | 15 Requests pro 15 Min. (pro Nutzer) | 15 Requests pro 15 Min. (pro Nutzer) |
| Datenformate | Standard-v1.1-Format | [X-API-v2-Format](/de/x-api/fundamentals/data-dictionary) (gesteuert durch die Request-Parameter fields und expansions; nicht abwärtskompatibel zu v1.1-Formaten)<br /><br />Weitere Informationen zur Migration vom Standard-v1.1-Format auf das X-API-v2-Format finden Sie in unserem [Leitfaden zur Datenformat-Migration](/de/x-api/migrate/data-format-migration). |
| Erfordert die Verwendung von Zugangsdaten aus einer [Developer-App](/de/resources/fundamentals/developer-apps), die einem [Projekt](/de/resources/fundamentals/projects) zugeordnet ist |     | ✔️  |

<div id="manage-mutes">
  #### Stummschaltungen verwalten
</div>

Die v2-Endpunkte zur Verwaltung von Stummschaltungen ersetzen die Standard-v1.1-Endpunkte [POST mutes/users/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-create) und [POST mutes/users/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-destroy).

Die folgenden Tabellen vergleichen die Standard-v1.1- und die X-API-v2-Endpunkte für Stummschaltungen:

**Einen Nutzer stummschalten**

| Beschreibung | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| Unterstützte HTTP-Methoden | POST | POST |
| Hostdomain | https://api.x.com | https://api.x.com |
| Endpunktpfad | /1.1/mutes/users/create.json | /2/users/:id/muting |
| [Authentifizierung](/de/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code mit PKCE |
| Standardmäßige [Rate Limits](/de/resources/fundamentals/rate-limits) für Anfragen | 50 Anfragen pro 15 Min. | 50 Anfragen pro 15 Min. |
| Erfordert die Verwendung von Zugangsdaten aus einer [Developer-App](/de/resources/fundamentals/developer-apps), die mit einem [Projekt](/de/resources/fundamentals/projects) verknüpft ist |     | ✔️  |
