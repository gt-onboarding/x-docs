<div id="manage-mutes-standard-v11-compared-to-x-api-v2">
  ### Stummschaltungen verwalten: Standard v1.1 im Vergleich zur X-API v2
</div>

Wenn Sie mit den Standard-v1.1-Endpunkten [POST mutes/users/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-create) und [POST mutes/users/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-destroy) gearbeitet haben, hilft Ihnen dieser Leitfaden, die Gemeinsamkeiten und Unterschiede zwischen den Standard-v1.1-Endpunkten und den X-API-v2-Endpunkten zum Verwalten von Stummschaltungen zu verstehen.

- **Gemeinsamkeiten**
  - OAuth 1.0a User Context
- **Unterschiede**
  - Endpoint-URLs
  - Anforderungen an App und Projekt
  - HTTP-Methoden
  - Anfrageparameter
     

<div id="similarities">
  #### Gemeinsamkeiten
</div>

**Authentifizierungsmethode OAuth 1.0a User Context**

Beide Endpoint-Versionen unterstützen [OAuth 1.0a User Context](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Wenn Sie zuvor einen der Standardendpunkte in v1.1 zum Verwalten von Stummschaltungen verwendet haben, können Sie bei der Migration auf die X-API-v2-Version dieselbe Authentifizierungsmethode weiterhin verwenden.

<div id="differences">
  #### Unterschiede
</div>

**Endpoint-URLs**

- Standard-v1.1-Endpunkte:
  - POST https://api.x.com/1.1/mutes/users/create.json
    (einen Nutzer stummschalten)
  - POST https://api.x.com/1.1/mutes/users/destroy.json
    (einen Nutzer entstummen)
- X-API-v2-Endpunkt:
  - POST https://api.x.com/2/users/:id/muting
    (einen Nutzer stummschalten)
  - DELETE https://api.x.com/2/users/:source\_user\_id/muting/:target\_user\_id
    (einen Nutzer entstummen)

**App- und Projektanforderungen**

Die X-API-v2-Endpunkte erfordern, dass Sie für die Authentifizierung Ihrer Anfragen Anmeldedaten aus einer [Developer-App](/de/resources/fundamentals/developer-apps) verwenden, die einem [Projekt](/de/resources/fundamentals/projects) zugeordnet ist. Alle X-API-v1.1-Endpunkte können Anmeldedaten von eigenständigen Apps oder von Apps verwenden, die einem Projekt zugeordnet sind.

**Anfrageparameter**

Die folgenden Standard-v1.1-Anfrageparameter haben Entsprechungen in der X-API v2:

| Standard v1.1 | X API v2 |
| :--- | :--- |
| user\_id | target\_user\_id |
| screen\_name | Keine Entsprechung |

Beachten Sie, dass die Standard-v1.1-Parameter als Query-Parameter übergeben werden, während die X-API-v2-Parameter als Body-Parameter (für den POST-Endpunkt) bzw. Pfadparameter (für den DELETE-Endpunkt) übergeben werden.

Zudem ist bei den Standard-v1.1-Endpunkten keine id des Nutzers erforderlich, der einen Zielnutzer stummschaltet, da die mit [OAuth 1.0a User Context](/de/resources/fundamentals/authentication) übergebenen Access Tokens erkennen ließen, welcher Nutzer die Stummschaltung bzw. das Entstummen initiiert hat.
