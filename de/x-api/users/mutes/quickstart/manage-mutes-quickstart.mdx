<div id="getting-started-with-the-manage-mutes-endpoints">
  ## Erste Schritte mit den Endpunkten zum Verwalten von Stummschaltungen
</div>

Diese Schnellstartanleitung hilft Ihnen, mit [Postman](/de/tutorials/postman-getting-started) Ihre erste Anfrage an die Endpunkte zum Verwalten von Stummschaltungen zu senden.

Wenn Sie Beispielcode in verschiedenen Sprachen sehen möchten, besuchen Sie bitte unser GitHub-Repository mit [Beispielcode für die X-API v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code).

<Note>
  **Voraussetzungen**

  Für diese Anleitung benötigen Sie einen Satz [Keys und Tokens](/de/resources/fundamentals/authentication), um Ihre Anfrage zu authentifizieren. Sie können diese Keys und Tokens generieren, indem Sie die folgenden Schritte ausführen:

  - [Beantragen Sie ein Entwicklerkonto](https://developer.x.com/en/apply-for-access) und lassen Sie es genehmigen.
  - Erstellen Sie ein [Projekt](/de/resources/fundamentals/projects) und eine zugehörige [Developer-App](/de/resources/fundamentals/developer-apps) im Developer-Portal.
  - Navigieren Sie zur Seite „Keys and tokens“ Ihrer App, um die erforderlichen Zugangsdaten zu generieren. Speichern Sie alle Zugangsdaten an einem sicheren Ort.
</Note>

<div id="steps-to-build-a-manage-mutes-request">
  ### Schritte zum Erstellen einer „Stummschaltungen verwalten“-Anfrage
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### Schritt eins: Beginnen Sie mit einem Tool oder einer Bibliothek
</div>

Es gibt verschiedene Tools, Codebeispiele und Bibliotheken, mit denen Sie eine Anfrage an diesen Endpoint stellen können. Wir verwenden hier Postman, um den Prozess zu vereinfachen.

Um die X‑API‑v2-Postman‑Collection in Ihre Umgebung zu laden, klicken Sie bitte auf die folgende Schaltfläche:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X‑API v2 zu Postman hinzufügen
</Button>

Sobald Sie die X‑API‑v2‑Collection in Postman geladen haben, navigieren Sie zum Ordner „Mutes“ und wählen Sie „Mute a user’s ID“.
 

<div id="step-two-authenticate-your-request">
  #### Schritt zwei: Authentifizieren Sie Ihre Anfrage
</div>

Um eine Anfrage an die X-API ordnungsgemäß zu stellen, müssen Sie nachweisen, dass Sie berechtigt sind. Bei diesem Endpoint müssen Sie dazu Ihre Anfrage entweder mit [OAuth 1.0a User Context](/de/resources/fundamentals/authentication) oder [OAuth 2.0 Authorization Code mit PKCE](de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) authentifizieren.

In diesem Beispiel verwenden wir OAuth 1.0a User Context.

Fügen Sie Ihre Keys und Tokens – konkret Ihren API Key, API Secret Key, OAuth 1.0a User Access Token und OAuth 1.0a User Access Token Secret – in Postman hinzu. Wählen Sie dazu oben rechts in Postman die Umgebung „X API v2“ aus und tragen Sie Ihre Keys und Tokens in die Felder „initial value“ und „current value“ ein (klicken Sie dazu auf das Augensymbol neben der Umgebungs-Auswahlliste).

Diese Variablen werden automatisch im Authorization-Tab der Anfrage verwendet, wenn Sie dies korrekt eingerichtet haben.
 

<div id="step-three-specify-who-is-going-to-mute-whom">
  #### Schritt drei: Festlegen, wer wen stummschaltet
</div>

Die Endpunkte zum Verwalten von Stummschaltungen erfordern zwei IDs: eine für den Benutzer (der Benutzer, der einen anderen Benutzer stummschalten oder die Stummschaltung aufheben möchte) und den Zielbenutzer (der Benutzer, der stummgeschaltet wird oder dessen Stummschaltung aufgehoben wird). Die Benutzer-ID muss der ID des authentifizierten Benutzers entsprechen. Das bedeutet, dass Sie bei der Authentifizierung Ihrer Anfrage die zu dieser Benutzer-ID gehörenden Access Tokens übergeben müssen. In diesem Fall können Sie die ID angeben, die zu Ihrem eigenen Benutzer gehört. Ihre ID können Sie auf zwei Arten ermitteln:

1. Mit dem Endpoint [user lookup](/de/x-api/users/lookup/introduction) nach Benutzername können Sie einen Benutzernamen übergeben und erhalten das Feld id.
2. In Ihrem Access Token sehen Sie, dass der numerische Teil Ihre Benutzer-ID ist.

Die Ziel-ID kann jede gültige Benutzer-ID sein. Navigieren Sie in Postman zur Registerkarte „Params“ und geben Sie Ihre ID in die Spalte „Value“ der `id`-Pfadvariablen ein. Navigieren Sie zur Registerkarte „Body“ und geben Sie die ID des Benutzers, den Sie stummschalten möchten, als Wert für den Parameter target\_user\_id ein. Achten Sie darauf, vor oder nach einer ID keine Leerzeichen einzufügen.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | ID des authentifizierten Benutzers |
| target\_user\_id | die Benutzer-ID, die Sie stummschalten möchten |

<div id="step-four-make-your-request-and-review-your-response">
  #### Schritt vier: Senden Sie Ihre Anfrage und prüfen Sie die Antwort
</div>

Sobald alles eingerichtet ist, klicken Sie auf die Schaltfläche „Send“, und Sie erhalten eine Antwort ähnlich der folgenden Beispielantwort:

```
      { "data": { "muting": true } }
```

Wenn Sie ein „muting“: true erhalten, wird die target\_user\_id erfolgreich stummgeschaltet.

Um denselben Nutzer wieder stummzuschalten aufzuheben, können Sie die Anfrage mit dem Titel „Unmute a user ID“ verwenden, die sich ebenfalls im Ordner „Mutes“ der in Postman geladenen X-API-v2-Kollektion befindet. Die source\_user\_id sollte Ihre Benutzer-ID sein und target\_user\_id die Benutzer-ID, deren Stummschaltung aufgehoben werden soll. Sie müssen dies nicht als JSON-Body hinzufügen; stellen Sie daher sicher, dass Sie die erforderlichen Query-Parameter für source\_user\_id und target\_user\_id angeben.

Bei erfolgreicher Aufhebung der Stummschaltung erhalten Sie eine Antwort ähnlich dem folgenden Beispiel:

```
      { "data": { "muting": false } }
```
