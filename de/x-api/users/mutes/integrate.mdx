<div id="postman">
  #### Postman
</div>

Postman ist ein hervorragendes Tool, mit dem Sie einen Endpunkt testen können. Jede Postman-Anfrage enthält sämtliche Pfad- und Body-Parameter, damit Sie schnell erkennen, was verfügbar ist. Weitere Informationen zu unseren Postman Collections finden Sie auf der Seite ["Using Postman"](/de/tutorials/postman-getting-started).

<div id="authentication">
  #### Authentifizierung
</div>

Alle Endpunkte der X-API v2 erfordern, dass Sie Ihre Anfragen mit einem Satz von Zugangsdaten authentifizieren, auch bekannt als Keys und Tokens. Sie können entweder OAuth 1.0a User Context oder OAuth 2.0 Authorization Code mit PKCE verwenden, um Ihre Anfragen an diese Endpunkte zu authentifizieren.

[OAuth 1.0a User Context](/de/resources/fundamentals/authentication) erfordert die Verwendung Ihrer API-Keys, Benutzer-Access-Tokens und einiger weiterer Parameter, um einen [Authorization-Header zu erstellen](/de/resources/fundamentals/authentication#authorizing-a-request), den Sie anschließend mit Ihrer Anfrage übermitteln. Die Access-Tokens müssen dem Benutzer zugeordnet sein, in dessen Namen Sie die Anfrage stellen. Wenn Sie ein Set an Access-Tokens für einen anderen Benutzer generieren möchten, muss dieser Ihre App mithilfe des [dreistufigen OAuth-Flows](https://developer-staging.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens) autorisieren.

Bitte beachten Sie, dass OAuth 1.0a schwierig zu verwenden sein kann. Wenn Sie mit dieser Authentifizierungsmethode nicht vertraut sind, empfehlen wir, eine [Bibliothek](/de/x-api/tools-and-libraries/overview) zu verwenden, ein Tool wie Postman zu nutzen oder OAuth 2.0 für die Authentifizierung Ihrer Anfragen zu verwenden.

[OAuth 2.0 Authorization Code mit PKCE](/de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) ermöglicht eine feinere Kontrolle über den Scope einer Anwendung sowie Autorisierungsabläufe über mehrere Geräte hinweg. Mit OAuth 2.0 können Sie spezifische, fein granulare Scopes auswählen, die Ihnen bestimmte Berechtigungen im Namen eines Benutzers gewähren.

Um OAuth 2.0 in Ihrer App zu aktivieren, müssen Sie es in den Authentifizierungseinstellungen Ihrer App aktivieren, die Sie im Abschnitt „App-Einstellungen“ des Developer-Portals finden.

<div id="developer-portal-projects-and-developer-apps">
  #### Developer-Portal, Projekte und Entwickler-Apps
</div>

Um einen Satz Authentifizierungsdaten zu erhalten, der mit den Endpunkten der X-API v2 funktioniert, müssen Sie sich für ein [Entwicklerkonto anmelden](https://developer.x.com/en/portal/petition/essential/basic-info), ein [Projekt](/de/resources/fundamentals/projects) innerhalb dieses Kontos einrichten und eine [Entwickler-App](/de/resources/fundamentals/developer-apps) innerhalb dieses Projekts erstellen. Anschließend finden Sie Ihre Schlüssel und Tokens in Ihrer Entwickler-App.
 

<div id="rate-limits">
  #### Rate Limits
</div>

Jeden Tag stellen viele Tausend Entwickler Anfragen an die X-API. Um das enorme Volumen dieser Anfragen zu steuern, gelten für jeden Endpoint [Rate Limits](/de/x-api/fundamentals/rate-limits), die die Anzahl der Anfragen begrenzen, die Sie im Namen Ihrer App oder eines authentifizierten Nutzers stellen können.

Diese Endpoints sind auf Nutzerebene ratebegrenzt. Das bedeutet, dass der authentifizierte Nutzer, in dessen Namen Sie die Anfrage stellen, den Endpoint nur eine bestimmte Anzahl von Malen über alle Developer-Apps hinweg aufrufen kann.

Es gilt ein Nutzer-Rate-Limit von 50 Anfragen pro 15 Minuten und pro Endpoint für die Methoden POST und DELETE. Mit der Methode GET beträgt das Rate-Limit jedoch nur 15 Anfragen pro 15 Minuten.
 

<div id="fields-and-expansions">
  #### Felder und Erweiterungen
</div>

Der X-API v2 GET-Endpunkt ermöglicht es Nutzerinnen und Nutzern, genau festzulegen, welche Daten sie mithilfe von sogenannten Feldern und Erweiterungen aus der API zurückgeben lassen. Der Parameter expansions ermöglicht es, Objekte zu erweitern, auf die im Payload verwiesen wird. Mit diesem Endpunkt können Sie beispielsweise die folgenden [Erweiterungen](/de/x-api/fundamentals/expansions) abrufen:

- pinned\_tweet\_id

Der Parameter **fields** ermöglicht es Ihnen, genau zu bestimmen, welche [Felder](/de/x-api/fundamentals/fields) innerhalb der verschiedenen Datenobjekte Sie erhalten möchten. Dieser Endpunkt liefert in erster Linie User-Objekte. Standardmäßig gibt das User-Objekt die Felder **id**, **name** und **username** zurück. Um zusätzliche Felder wie **user.created\_at** oder **user.entities** zu erhalten, müssen Sie diese explizit über den Parameter **fields** anfordern.

Wir haben unserer [X-API v2-Datendi­rektion](/de/x-api/fundamentals/data-dictionary) eine Anleitung zur gemeinsamen Verwendung von [Feldern und Erweiterungen](/de/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) hinzugefügt.

<div id="pagination">
  #### Paginierung
</div>

Die Abfrage stummgeschalteter Nutzer kann viele Daten zurückliefern. Um jederzeit konsistente und performante Ergebnisse sicherzustellen, verwenden wir Paginierung. Paginierung ist eine Funktion in X-API-v2-Endpunkten, die mehr Ergebnisse zurückgibt, als in einer einzelnen Antwort enthalten sein können. In diesem Fall werden die Daten in mehreren „Seiten“ geliefert. Erfahren Sie mehr darüber, wie Sie [durch Ergebnisse paginieren.](/de/x-api/fundamentals/pagination)

<Note>
  **Bitte beachten:** Wenn ein Nutzer über [X](https://x.com/) stumm schaltet, gilt ein Limit von 200 Anfragen pro 15 Minuten.
</Note>
