<div id="postman">
  #### Postman
</div>

Postman ist ein hervorragendes Tool, um Endpunkte zu testen. Jede Postman-Anfrage enthält alle Pfad- und Body-Parameter, damit Sie schnell nachvollziehen können, welche Optionen Ihnen zur Verfügung stehen. Weitere Informationen zu unseren Postman-Collections finden Sie auf der Seite ["Using Postman"](/de/tutorials/postman-getting-started).

<div id="authentication">
  #### Authentifizierung
</div>

Alle Endpunkte der X-API v2 erfordern, dass Anfragen mit einem Satz von Anmeldedaten [authentifiziert](/de/resources/fundamentals/authentication) werden, auch bekannt als Keys und Tokens. Sie können entweder OAuth 1.0a User Context, App only oder OAuth 2.0 Authorization Code mit PKCE verwenden, um Anfragen an diese Endpunkte zu authentifizieren.

[OAuth 1.0a User Context](/de/resources/fundamentals/authentication#oauth-1-0a-2) erfordert die Verwendung Ihrer API Keys, Benutzer-Access Tokens und einiger weiterer Parameter, um einen [Authorization-Header zu erstellen](/de/resources/fundamentals/authentication#authorizing-a-request), den Sie dann mit Ihrer Anfrage übermitteln. Die Access Tokens müssen dem Benutzer zugeordnet sein, in dessen Auftrag Sie die Anfrage stellen. Wenn Sie einen Satz von Access Tokens für einen anderen Benutzer generieren möchten, muss dieser Ihre App mithilfe des [3-legged OAuth-Flows](https://developer-staging.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens) autorisieren.

Bitte beachten Sie, dass OAuth 1.0a schwierig sein kann. Wenn Sie mit dieser Authentifizierungsmethode nicht vertraut sind, empfehlen wir, eine [Library](/de/x-api/tools-and-libraries/overview) zu verwenden, ein Tool wie Postman einzusetzen oder OAuth 2.0 zu nutzen, um Ihre Anfragen zu authentifizieren. Wenn Sie einen Post oder private Metriken von diesen Endpunkten anfordern möchten, müssen Sie entweder OAuth 1.0a User Context oder OAuth 2.0 Authorization Code mit PKCE verwenden. So wird sichergestellt, dass Sie die erforderlichen Berechtigungen des Benutzers besitzen, dem diese Inhalte gehören.

[App only](/de/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) erfordert lediglich, dass Sie ein [App-only Access Token](/de/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) mit Ihrer Anfrage übermitteln. Sie können ein App-only Access Token entweder direkt in einer Developer-App erzeugen oder eines mithilfe des Endpunkts [POST oauth2/token](/de/resources/fundamentals/authentication#post-oauth2-token) generieren.

[OAuth 2.0 Authorization Code mit PKCE](/de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) ermöglicht eine feinere Kontrolle über den Scope einer Anwendung sowie Autorisierungsflüsse über mehrere Geräte hinweg. Mit OAuth 2.0 können Sie spezifische, fein granulare Scopes auswählen, die Ihnen bestimmte Berechtigungen im Namen eines Benutzers gewähren.

Um OAuth 2.0 in Ihrer App zu aktivieren, müssen Sie es in den Authentifizierungseinstellungen Ihrer App aktivieren, die Sie im Abschnitt „App-Einstellungen“ des Developer-Portals finden.

**Bitte beachten**

Wenn Sie die folgenden Felder anfordern, ist OAuth 1.0a User Context oder OAuth 2.0 Authorization Code erforderlich:

- tweet.fields.non\_public\_metrics
- tweet.fields.promoted\_metrics
- tweet.fields.organic\_metrics

<div id="developer-portal-projects-and-developer-apps">
  #### Entwicklerportal, Projekte und Entwickler-Apps
</div>

Um einen Satz an Authentifizierungsdaten zu erhalten, der mit den Endpunkten der X-API v2 funktioniert, müssen Sie sich für ein [Entwicklerkonto registrieren](https://developer.x.com/en/portal/petition/essential/basic-info), ein [Projekt](/de/resources/fundamentals/projects) in diesem Konto einrichten und innerhalb dieses Projekts eine [Entwickler-App](/de/resources/fundamentals/developer-apps) erstellen. Anschließend finden Sie Ihre Keys und Tokens in Ihrer Entwickler-App. 
 

<div id="rate-limits">
  #### Rate Limits
</div>

Jeden Tag stellen viele Tausend Entwickler Anfragen an die X-API. Um das hohe Volumen dieser Anfragen zu bewältigen, gelten für jeden Endpoint [Rate Limits](/de/x-api/fundamentals/rate-limits), die die Anzahl der Anfragen begrenzen, die du im Namen deiner App oder eines authentifizierten Nutzers stellen kannst. 

Die Endpoints zur Nutzerabfrage sind sowohl auf App-Ebene als auch auf Nutzer-Ebene ratebegrenzt. Der Endpoint zur Abfrage des authentifizierten Nutzers ist hingegen auf Nutzer-Ebene ratebegrenzt.

Das Rate Limit auf App-Ebene bedeutet, dass du als Entwickler innerhalb eines bestimmten Zeitraums nur eine bestimmte Anzahl von Anfragen an diesen Endpoint von einer bestimmten App aus stellen kannst (identifiziert durch die verwendeten Keys und Tokens). Das Rate Limit auf Nutzer-Ebene bedeutet, dass der authentifizierte Nutzer, in dessen Namen du die Anfrage stellst, dies über beliebige Entwickler-Apps nur eine bestimmte Anzahl von Malen ausführen kann.

Die folgende Tabelle zeigt die Rate Limits für jeden Endpoint.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **HTTP-Methode** | **Rate Limit / Ebene** |
| /2/users | GET | 900 Anfragen pro 15 Minuten / App und Nutzer |
| /2/users/:id | GET | 900 Anfragen pro 15 Minuten / App und Nutzer |
| /2/users/by | GET | 900 Anfragen pro 15 Minuten / App und Nutzer |
| /2/users/by/username/:username | GET | 900 Anfragen pro 15 Minuten / App und Nutzer |
| /2/users/me | GET | 75 Anfragen pro 15 Minuten / Nutzer |

<div id="fields-and-expansions">
  #### Felder und Erweiterungen
</div>

Die X-API v2 ermöglicht es Nutzerinnen und Nutzern, mithilfe eines Satzes von Tools namens Felder und Erweiterungen genau auszuwählen, welche Daten sie von der API zurückerhalten möchten. Der Parameter expansion ermöglicht es, Objekte zu erweitern, auf die im Payload verwiesen wird. Dieser Endpoint erlaubt Ihnen zum Beispiel, die pinned\_tweet\_id-Expansion zu verwenden.

Der Parameter fields ermöglicht es Ihnen, genau auszuwählen, welche [Felder](/de/x-api/fundamentals/fields) innerhalb der verschiedenen Datenobjekte Sie erhalten möchten. Diese Endpoints liefern in erster Linie User-Objekte. Standardmäßig gibt das User-Objekt die Felder id, name und username zurück. Um zusätzliche Felder wie user.created\_at oder user.location zu erhalten, müssen Sie diese explizit über einen fields-Parameter anfordern. Einige wichtige Felder, die Sie für Ihre Integration in Betracht ziehen sollten, sind unsere Post-Umfragedaten, Metriken, Anmerkungen und die Konversations-ID-Felder.

Wir haben einen Leitfaden dazu hinzugefügt, wie Sie [Felder und Erweiterungen verwenden](/de/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) in unserem [X-API-v2-Datenwörterbuch](/de/x-api/fundamentals/data-dictionary).
