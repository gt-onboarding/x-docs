  ## Vergleich der Endpunkte zur Benutzerabfrage der X-API
</div>

Die v2-Endpunkte zur Benutzerabfrage ersetzen die Standard-v1.1-Endpunkte [GET users/lookup](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-users-lookup.html) und [GET users/show](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-users-show.html). Wenn Sie Code, Apps oder Tools haben, die eine dieser Versionen der Benutzerabfrage-Endpunkte verwenden und eine Migration auf die neueren Endpunkte der X-API v2 in Erwägung ziehen, sind diese Leitfäden für Sie gedacht. 
 

Die folgenden Tabellen vergleichen die verschiedenen Typen von Endpunkten zur Benutzerabfrage:
 

|     |     |     |
| :--- | :--- | :--- |
| **Beschreibung** | **Standard v1.1** | **X-API v2** |
| Unterstützte HTTP-Methoden | `GET` | `GET` |
| Host-Domain | `https://api.x.com` | `https://api.x.com` |
| Endpunktpfad | `/1.1/users/show.json` `/1.1/users/lookup.json` | `/2/users`<br /><br />`/2/users/:id`<br /><br />`/2/users/by`<br /><br />`/2/users/by/:username` |
| [Authentifizierung](/de/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />Nur App<br /><br />OAuth 2.0 Authorization Code mit PKCE |
| Standard-[Ratenlimits](/de/resources/fundamentals/rate-limits) für Anfragen | 900 Anfragen pro 15 Min. (pro Benutzer)<br /><br />/show – 900 Anfragen pro 15 Min. (pro App)  <br />/lookup – 300 Anfragen pro 15 Min. (pro App) | 900 Anfragen pro 15 Min. (pro Benutzer)<br /><br />300 Anfragen pro 15 Min. (pro App) |
| Maximale Anzahl Benutzer pro Antwort | /show – 1<br /><br />/lookup – 100 | 100 |
| JSON-Antwortobjektformat | Standard-v1.1-Format | [X-API-v2-Format](/de/x-api/fundamentals/data-dictionary) (bestimmt durch die Anfrageparameter fields und expansions; nicht abwärtskompatibel mit v1.1-Formaten)<br /><br />Weitere Informationen zur Migration vom Standard-v1.1-Format zum X-API-v2-Format finden Sie in unserem [Leitfaden zur Migration von Datenformaten](/de/x-api/migrate/data-format-migration). |
| Unterstützung der Auswahl, welche [Felder](/de/x-api/fundamentals/data-dictionary) in der Payload zurückgegeben werden |     | ✔   |
| Unterstützung der Felder [annotations](/de/x-api/fundamentals/post-annotations) (bei angeheftetem Post) |     | ✔   |
| Unterstützung der Anforderung neuer [Metrik](/de/x-api/fundamentals/metrics)-Felder (bei angeheftetem Post) |     | ✔   |
| Unterstützung des Felds [conversation\_id](/de/x-api/fundamentals/conversation-id) (bei angeheftetem Post) |     | ✔   |
| Erfordert die Verwendung von Zugangsdaten aus einer [Developer-App](/de/resources/fundamentals/developer-apps), die einem [Projekt](/de/resources/fundamentals/projects) zugeordnet ist |     | ✔   |
