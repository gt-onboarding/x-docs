Sie können Ihre Anfrage an alle Nutzer-Lookup-Endpunkte außer dem Lookup des authentifizierten Nutzers mit [OAuth 1.0a User Context](/de/resources/fundamentals/authentication), [App-only](/de/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) oder [OAuth 2.0 Authorization Code mit PKCE](de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) authentifizieren. Wenn Sie jedoch auf private Metriken oder Daten von privaten Nutzern zugreifen möchten, müssen Sie OAuth 1.0a User Context oder OAuth 2.0 Authorization Code mit PKCE verwenden und das Access Token des authentifizierten Nutzers mit Ihrer Anfrage übermitteln.

Dieser Endpunkt wird häufig verwendet, um aktuelle Details zu einem Nutzer zu erhalten, zu überprüfen, ob ein Nutzer existiert, oder um Ihre gespeicherten Details nach einem Compliance-Ereignis zu aktualisieren.

<div id="authenticated-user-lookup">
  ### Authentifizierte Benutzerabfrage
</div>

Wenn Sie sich im Namen anderer Nutzer authentifizieren, ist es entscheidend, sehen zu können, für wen Sie Anfragen stellen dürfen.

Dieser Endpunkt erfordert die Verwendung von [OAuth 1.0a User Context](/de/resources/fundamentals/authentication) oder [OAuth 2.0 Authorization Code mit PKCE](de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). Er gibt Informationen über den autorisierten Nutzer zurück, also den Nutzer, der mit den User-Access-Tokens verknüpft ist, die Sie mit der Anfrage übermitteln.

Sie können auf diesen Endpunkt über die GET-Methode zugreifen. Für diesen Endpunkt gilt ein nutzerbasiertes Rate-Limit von 75 Anfragen pro 15 Minuten.

<Note>
  **Kontoeinrichtung**

  Um auf diese Endpunkte zuzugreifen, benötigen Sie:

  - Ein genehmigtes [Developer-Konto](https://developer.x.com/en/portal/petition/essential/basic-info).
  - Eine Authentifizierung mit den Keys und Tokens aus einer [Developer App](/de/resources/fundamentals/developer-apps), die sich innerhalb eines [Project](/de/resources/fundamentals/projects) befindet.

  Erfahren Sie mehr über den Zugriff auf die Endpunkte der X-API v2 in unserem [Getting-Started-Leitfaden](/de/x-api/getting-started/getting-access).
</Note>
