Wir bieten zwei Gruppen von Endpunkten an, mit denen Sie Follow-Beziehungen nachschlagen, erstellen und löschen können: follows lookup und manage follows.

<div id="follows-lookup">
  ### Follows lookup
</div>

Nur verfügbar für Nutzer mit [Enterprise-Zugriff](/de/x-api/getting-started/about-x-api)

Die Follows-Lookup-Endpunkte ermöglichen es Ihnen, Beziehungen zwischen Nutzern zu untersuchen und zu analysieren, was manchmal als Netzwerkanalyse bezeichnet wird. Konkret gibt es zwei REST-Endpunkte, die [User-Objekte](/de/x-api/fundamentals/data-dictionary#user) zurückgeben, die darstellen, wem ein angegebener Nutzer folgt oder wer einem angegebenen Nutzer folgt.

Sie können diesen Endpunkt entweder mit [OAuth 1.0a User Context](/de/resources/fundamentals/authentication), \[App only]\(https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), oder [OAuth 2.0 Authorization Code with PKCE](de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) authentifizieren. Sie können bis zu 1.000 Nutzer pro Anfrage abrufen; für das Paging durch große Ergebnismengen werden Paginierungs-Token bereitgestellt.
 

<div id="manage-follows">
  ### Follows verwalten
</div>

Die Endpunkte zum Verwalten von Follows ermöglichen es Ihnen, Nutzern zu folgen oder das Folgen zu beenden.

Da Sie Anfragen im Namen eines Nutzers stellen, müssen Sie diese Endpunkte entweder mit [OAuth 1.0a User Context](/de/resources/fundamentals/authentication) oder [OAuth 2.0 Authorization Code with PKCE](de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) authentifizieren und die Benutzer-Access-Tokens verwenden, die dem Nutzer zugeordnet sind, in dessen Namen Sie die Anfrage stellen. Sie können dieses Benutzer-Access-Token mithilfe des [3-legged OAuth Flow](/de/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) oder mithilfe des [Authorization Code with PKCE Grant Flow](/de/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)) (OAuth 2.0) generieren.

Pro authentifiziertem Nutzer sind Sie auf 400 Follow-Aktionen pro Tag beschränkt und über alle authentifizierten Nutzer Ihrer App hinweg auf 1.000 Aktionen pro Tag (pro App). Wenn Sie beispielsweise fünf authentifizierte Nutzer haben, können zwei dieser Nutzer jeweils 400 anderen Nutzern pro Tag folgen (Limit pro Nutzer) – insgesamt 800 Aktionen – und müssen die verbleibenden 200 Aktionen (pro App) auf die restlichen drei Nutzer aufteilen. Dieses Limit gilt nicht für den Unfollow-Endpunkt; dafür gilt ein separates Limit von 500 Aktionen pro Tag (pro App).

<Note>
  **Kontoeinrichtung**

  Für den Zugriff auf diese Endpunkte benötigen Sie:

  - Ein genehmigtes [Developer-Konto](https://developer.x.com/en/portal/petition/essential/basic-info).
  - Eine Authentifizierung mit den Schlüsseln und Tokens aus einer [Developer-App](/de/resources/fundamentals/developer-apps), die sich innerhalb eines [Projects](/de/resources/fundamentals/projects) befindet.
</Note>

Erfahren Sie mehr über den Zugriff auf die Endpunkte der X-API v2 in unserem [Getting-Started-Leitfaden](/de/x-api/getting-started/getting-access).
