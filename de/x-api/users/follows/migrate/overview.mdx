<div id="comparing-x-apis-follows-endpoints">
  ## Vergleich der Follows-Endpunkte der X-API
</div>

<div id="follows-lookup">
  ### Follows-Lookup
</div>

Die v2-Follows-Lookup-Endpunkte ersetzen die Standard-v1.1-Endpunkte [followers/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-followers-ids), v1.1 [followers/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-followers-list), v1.1 [friends/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-friends-ids), und v1.1 [friends/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-friends-list).

Die folgenden Tabellen vergleichen die verschiedenen Arten von Follows-Lookup-Endpunkten:

|     |     |     |
| :--- | :--- | :--- |
| **Beschreibung** | **Standard v1.1** | **X API v2** |
| Unterstützte HTTP-Methoden | `GET` | `GET` |
| Host-Domain | `https://api.x.com` | `https://api.x.com` |
| Endpunktpfad | /1.1/friends/ids.json<br /><br />/1.1/friends/list.json<br /><br />/1.1/followers/ids.json<br /><br />/1.1/followers/list.json | /2/users/:id/following<br /><br />/2/users/:id/followers |
| [Authentifizierung](/de/resources/fundamentals/authentication) | OAuth 1.0a User Context<br /><br />App only | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code mit PKCE<br /><br />App only |
| Standard-[Ratenlimits](/de/resources/fundamentals/rate-limits) für Anfragen | 15 Anfragen pro 15 min (pro Nutzer)<br /><br />15 Anfragen pro 15 min (pro App) | 15 Anfragen pro 15 min (pro Nutzer)<br /><br />15 Anfragen pro 15 min (pro App) |
| Maximale Nutzer pro Antwort | GET friends/ids & GET followers/ids liefern maximal 5000 Nutzer-IDs pro Seite.<br /><br />  <br /><br />GET friends/list & GET followers/list liefern maximal 200 Nutzerobjekte pro Seite. | 1000 Nutzerobjekte pro Seite |
| Paginierung | Token wird in einem next\_cursor-Feld zurückgegeben und kann anschließend als Wert für den cursor-Parameter übergeben werden, um die nächste Ergebnisseite zu erhalten. | Token wird in einem next\_token-Feld zurückgegeben und kann anschließend als Wert für den token-Parameter übergeben werden, um die nächste Ergebnisseite zu erhalten.<br /><br />Die v2-Nutzlast liefert außerdem ein previous\_token-Feld, das ebenfalls mit dem pagination\_token-Parameter übergeben werden kann, um die vorherige Ergebnisseite zu erhalten. |
| JSON-Format | Standard v1.1 Format | [X API v2 format](/de/x-api/fundamentals/data-dictionary) (bestimmt durch die Request-Parameter fields und expansions; nicht abwärtskompatibel mit v1.1-Formaten)<br /><br />Weitere Informationen zur Migration vom Standard v1.1 Format zum X API v2 Format finden Sie in unserem [Leitfaden zur Migration von Datenformaten](/de/x-api/migrate/data-format-migration). |
| Unterstützt die Auswahl, welche [Felder](/de/x-api/fundamentals/data-dictionary) in der Nutzlast zurückgegeben werden |     | ✔   |
| Unterstützt die Post-[annotations](/de/x-api/fundamentals/post-annotations)-Felder |     | ✔   |
| Unterstützt das Anfordern neuer [metrics](/de/x-api/fundamentals/metrics)-Felder |     | ✔   |
| Unterstützt das Feld [conversation\_id](/de/x-api/fundamentals/conversation-id) |     | ✔   |
| Erfordert die Verwendung von Zugangsdaten aus einer [developer App](/de/resources/fundamentals/developer-apps), die einem [Projekt](/de/resources/fundamentals/projects) zugeordnet ist |     | ✔   |

<div id="manage-follows">
  #### Folgen verwalten
</div>

Die v2-Endpunkte zum Verwalten von Folgen ersetzen die Standard-v1.1-Endpunkte [POST friendships/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-create) und [POST friendships/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-destroy).

Die folgenden Tabellen vergleichen die Standard-v1.1- und X-API-v2-Endpunkte zum Erstellen von Folgebeziehungen:

<div id="follow-a-user">
  #### Einem Nutzer folgen
</div>

| Beschreibung | Standard v1.1 | X-API v2 |
| :--- | :--- | :--- |
| Unterstützte HTTP-Methoden | POST | POST |
| Hostdomain | https://api.x.com | https://api.x.com |
| Endpunktpfad | /1.1/friendships/create.json | /2/users/:id/following |
| [Authentifizierung](/de/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Autorisierungscode mit PKCE |
| Standardmäßige [Rate-Limits](/de/resources/fundamentals/rate-limits) für Anfragen | 50 Anfragen pro 15 Min | 50 Anfragen pro 15 Min |
| Maximale tägliche Vorgänge pro Nutzer | 400 | 400 |
| Maximale tägliche Vorgänge pro App | 1000 | 1000 |
| Erfordert die Verwendung von Anmeldedaten aus einer [Developer-App](/de/resources/fundamentals/developer-apps), die einem [Projekt](/de/resources/fundamentals/projects) zugeordnet ist |     | ✔️  |

<div id="unfollow-a-user">
  #### Einem Nutzer nicht mehr folgen
</div>

Die folgenden Tabellen vergleichen die Endpunkte zum Beenden eines Follows in Standard v1.1 und der X-API v2:

| Beschreibung | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| Unterstützte HTTP-Methoden | POST | DELETE |
| Host-Domain | https://api.x.com | https://api.x.com |
| Endpunkt-Pfad | /1.1/friendships/destroy.json | /2/users/:source\_user\_id/following/:target\_user\_id |
| [Authentifizierung](/de/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code mit PKCE |
| Standard-[Rate-Limits](/de/resources/fundamentals/rate-limits) für Anfragen | 15 Anfragen pro 15 Min. (pro Nutzer) | 50 Anfragen pro 15 Min. (pro Nutzer) |
| Maximale tägliche Vorgänge pro App | Keine | 500 |
| Erfordert die Verwendung von Zugangsdaten einer [Developer-App](/de/resources/fundamentals/developer-apps), die mit einem [Project](/de/resources/fundamentals/projects) verknüpft ist |     | ✔️  |
