<div id="manage-follows-standard-v11-compared-to-x-api-v2">
  #### Follows verwalten: Standard v1.1 im Vergleich zur X-API v2
</div>

Wenn Sie mit den Standard-v1.1-Endpunkten [POST friendships/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-create) und [POST friendships/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-destroy) gearbeitet haben, soll Ihnen dieser Leitfaden helfen, die Gemeinsamkeiten und Unterschiede zwischen den Standard-Endpunkten und den X-API-v2-Endpunkten zur Verwaltung von Follows zu verstehen.

- **Gemeinsamkeiten**
  - OAuth 1.0a User Context
- **Unterschiede**
  - Endpoint-URLs
  - Anforderungen an App und Projekt
  - HTTP-Methoden
  - Anfrageparameter

     

<div id="similarities">
  #### Gemeinsamkeiten
</div>

**Authentifizierungsmethode OAuth 1.0a User Context**

Beide Endpunktversionen unterstützen [OAuth 1.0a User Context](/de/resources/fundamentals/authentication#oauth-1-0a-2). Wenn Sie zuvor einen der Standard-Endpunkte der v1.1 zum Verwalten von Follows verwendet haben, können Sie bei der Migration zur X-API v2 dieselbe Authentifizierungsmethode weiter nutzen. 

<div id="differences">
  #### Unterschiede
</div>

**Endpoint-URLs**

- Standard-v1.1-Endpoints:
  - POST https://api.x.com/1.1/friendships/create.json
    (einem Nutzer folgen)
  - POST https://api.x.com/1.1/friendships/destroy.json
    (einem Nutzer nicht mehr folgen)
- X-API-v2-Endpoint:
  - POST https://api.x.com/2/users/:id/following
    (einem Nutzer folgen)
  - DELETE https://api.x.com/2/users/:source\_user\_id/following/:target\_user\_id
    (einem Nutzer nicht mehr folgen)

**App- und Projektanforderungen**

Die X-API-v2-Endpoints erfordern, dass Sie bei der Authentifizierung Ihrer Anfragen Anmeldedaten aus einer [Developer App](/de/resources/fundamentals/developer-apps) verwenden, die einem [Project](/de/resources/fundamentals/projects) zugeordnet ist. Alle X-API-v1.1-Endpoints können Anmeldedaten von eigenständigen Apps oder von Apps verwenden, die einem Projekt zugeordnet sind.

**Anfrageparameter**

Die folgenden Standard-v1.1-Anfrageparameter haben Entsprechungen in der X-API v2:

| Standard v1.1 | X API v2 |
| :--- | :--- |
| Keine Entsprechung | id (POST), source\_user\_id (DELETE) |
| user\_id | target\_user\_id |
| screen\_name | Keine Entsprechung |

Bitte beachten Sie, dass die Standard-v1.1-Parameter als Query-Parameter übergeben werden, während die X-API-v2-Parameter als Body-Parameter (für den POST-Endpoint) bzw. als Pfadparameter (für den DELETE-Endpoint) übergeben werden.

Außerdem sind die v2-Parameter id und source\_user\_id nicht erforderlich, wenn die Standard-v1.1-Endpoints verwendet werden, da die mit OAuth 1.0a User Context übergebenen Access Tokens erkennen ließen, welcher Nutzer das Folgen/Entfolgen initiiert hat.
