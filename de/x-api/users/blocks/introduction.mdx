<div id="blocks-lookup">
  ### Block-Abfrage
</div>

Der GET-Endpunkt für die Block-Abfrage zeigt Ihnen, welche Accounts Sie im Namen eines autorisierten Nutzers blockiert haben. Dieser Endpunkt hat ein Rate-Limit von 15 Anfragen pro 15 Minuten und pro Nutzer.

Da Sie mit der Block-Abfrage private Informationen anfordern und im Namen eines Nutzers mit Berechtigung zur Blockverwaltung handeln, müssen Sie diese Endpunkte entweder mit [OAuth 1.0a User Context](/de/resources/fundamentals/authentication) oder [OAuth 2.0 Authorization Code mit PKCE](de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) authentifizieren und die Benutzerzugriffstoken eines Nutzers verwenden, der Ihre App autorisiert hat. Diese können über den [3-legged OAuth Flow](/de/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) oder den [Authorization Code mit PKCE Grant Flow](/de/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0) generiert werden.

<Note>
  **Kontoeinrichtung**

  Um auf diese Endpunkte zuzugreifen, benötigen Sie:

  - Ein genehmigtes [Developer-Konto](https://developer.x.com/en/portal/petition/essential/basic-info).
  - Eine Authentifizierung mit den Schlüsseln und Token aus einer [Developer-App](/de/resources/fundamentals/developer-apps), die sich innerhalb eines [Projekts](/de/resources/fundamentals/projects) befindet.

  Weitere Informationen zum Zugriff auf die Endpunkte der X-API v2 finden Sie in unserem [Einstiegshandbuch](/de/x-api/getting-started/getting-access).
</Note>
