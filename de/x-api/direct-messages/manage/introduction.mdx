Direct Messages ermöglichen private Unterhaltungen auf X. Direct Messages gehören zu den beliebtesten Funktionen von X und decken eine Vielzahl von Anwendungsfällen ab – von Gruppenchats unter Freunden bis hin zu globalem Kundensupport für Marken. Neue v2-Versionen der Direct-Message-Endpunkte werden schrittweise eingeführt; diese erste Phase umfasst grundlegende Endpunkte zum Erstellen von Direct Messages und zum Auflisten von Ereignissen in Direct-Message-Unterhaltungen. Erstmals unterstützt die X-API v2 _Gruppen_-Unterhaltungen.

Diese erste Version von Manage Direct Messages umfasst drei POST-Methoden zum Erstellen von Direct Messages:

- **POST /2/dm\_conversations/with/:participant\_id/messages** – Erstellt eine 1:1-Direct Message. Diese Methode erstellt entweder eine neue 1:1-Unterhaltung oder ruft die bestehende Unterhaltung ab und fügt die Direct Message hinzu. Der :participant\_id-Pfadparameter ist die User-ID des Kontos, das die Nachricht erhält.
- **POST /2/dm\_conversations** – Erstellt eine neue Gruppenunterhaltung und fügt ihr eine Direct Message hinzu. Diese Anfragen erfordern eine Liste von Gesprächsteilnehmenden. Beachten Sie, dass Sie mehrere Unterhaltungen mit derselben Teilnehmendenliste erstellen können. Diese Anfragen geben stets eine neue Unterhaltungs-ID zurück.
- **POST /2/dm\_conversations/:dm\_conversation\_id/messages** – Erstellt eine Direct Message und fügt sie einer bestehenden Unterhaltung hinzu. Der :dm\_conversation\_id-Pfadparameter ist die ID der Unterhaltung, der die Nachricht hinzugefügt wird.

Beachten Sie, dass Direct-Message-Ereignis-IDs in v1.1 und v2 (sowie in der X App) identisch sind, sodass die v1.1-Methoden zum Ausblenden/Löschen von Direct Messages zusammen mit diesem neuen v2-Endpunkt verwendet werden können. Beachten Sie außerdem, dass die Enterprise- und Premium-Account-Activity-APIs v2-1:1-Nachrichten unterstützen, jedoch noch keine Gruppenunterhaltungen.

Es gibt ein nutzerbasiertes Ratenlimit von 200 Anfragen pro 15 Minuten für die POST-Methode. Es gibt außerdem ein Ratenlimit von 1.000 Anfragen pro 24 Stunden pro Nutzer. Zusätzlich gibt es ein Ratenlimit von 15.000 Anfragen pro 24 Stunden. Diese Ratenlimits werden über diese POST-Endpunkte hinweg geteilt.

Da Sie im Namen eines Nutzers mit den Manage-Posts-Endpunkten Anfragen stellen, müssen Sie sich entweder mit [OAuth 1.0a User Context](https://developer-staging.x.com/resources/fundamentals/authentication) oder [OAuth 2.0 Authorization Code with PKCE](de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) authentifizieren und ein Benutzer-Access-Token verwenden, das einem Nutzer zugeordnet ist, der Ihre App autorisiert hat. Um dieses Benutzer-Access-Token mit OAuth 1.0a zu erzeugen, können Sie den [3-legged OAuth flow](/de/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token) verwenden. Um ein Benutzer-Access-Token mit OAuth 2.0 zu erzeugen, können Sie den [Authorization Code with PKCE grant flow](/de/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)) verwenden.

  - Die Authentifizierung mit den Schlüsseln und Tokens aus einer [Developer App](/de/resources/fundamentals/developer-apps), die sich innerhalb eines [Project](/de/resources/fundamentals/projects) befindet.

  Erfahren Sie mehr über den Zugriff auf die X-API-v2-Endpunkte in unserem [Leitfaden für den Einstieg](/de/x-api/getting-started/getting-access).
</Note>
