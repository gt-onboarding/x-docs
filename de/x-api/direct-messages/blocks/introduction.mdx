Die Endpunkte zum Verwalten von DM-Blockierungen ermöglichen es, im Namen eines authentifizierten Nutzers ein bestimmtes Konto zu blockieren oder die Blockierung aufzuheben. Für diese Endpunkte stehen zwei POST-Methoden zur Verfügung:

- **/2/users/:id/dm/block**: Blockiert ein Konto
- **/2/users/:id/dm/unblock**: Hebt die Blockierung eines Kontos auf

<div id="getting-started">
  ### Erste Schritte
</div>

<div id="authentication">
  ### Authentifizierung
</div>

Da Sie Anfragen im Namen eines Nutzers stellen, müssen Sie diese Endpunkte entweder mit [OAuth 1.0a User Context](/de/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) oder mit [OAuth 2.0 Authorization Code with PKCE](/de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) authentifizieren und die Benutzer-Access-Tokens verwenden, die dem Nutzer zugeordnet sind, in dessen Namen Sie die Anfrage stellen. Sie können dieses Benutzer-Access-Token über den [3-legged OAuth-Flow](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens) (OAuth 1.0a) oder über den [Authorization Code with PKCE Grant Flow](/de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) (OAuth 2.0) erzeugen.

<div id="making-a-request">
  ### Eine Anfrage stellen
</div>

Blockieren

Sobald sich ein Nutzer bei Ihrer App authentifiziert hat, können Sie im Namen des Nutzers den Block-Endpoint wie unten gezeigt aufrufen:

```
      curl --request POST 'https://api.x.com/2/users/:id/dm/block' --header 'Authorization: ••••••'
```

Wenn die Anfrage erfolgreich ist, sehen Sie die JSON-Antwort wie unten gezeigt:

```
{
   "data": {
       "blocked": true
   }
}
```

**Entsperren**

Sobald sich ein Nutzer bei Ihrer App authentifiziert hat, können Sie im Namen des Nutzers den Unblock-Endpoint wie unten gezeigt aufrufen:

```
      curl --request POST 'https://api.x.com/2/users/:id/dm/unblock' --header 'Authorization: ••••••'
```

Wenn die Anfrage erfolgreich ist, sehen Sie die JSON-Antwort wie unten gezeigt:

```
{
   "data": {
       "blocked": false
   }
}
```
