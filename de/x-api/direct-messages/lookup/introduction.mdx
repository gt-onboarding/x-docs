Diese erste Version der Direct Messages Lookup umfasst drei GET-Methoden:

- **GET /2/dm\_conversations/with/:participant\_id/dm\_events** - Ruft Direct-Message-Ereignisse ab, die einer Eins-zu-eins-Unterhaltung zugeordnet sind. Der Pfadparameter :participant\_id ist die User-ID des Kontos, das die Unterhaltung mit dem authentifizierten Benutzer führt, der diese Anfrage stellt.
- **GET /2/dm\_conversations/:dm\_conversation\_id/dm\_events** - Ruft Direct-Message-Ereignisse ab, die einer bestimmten Unterhaltungs-ID zugeordnet sind, wie durch den Pfadparameter :dm\_conversation\_id angegeben.
- **GET /2/dm\_events** - Ruft Direct-Message-Ereignisse ab, die einem Benutzer zugeordnet sind, einschließlich Eins-zu-eins- und Gruppenunterhaltungen. Ereignisse bis zu 30 Tage in der Vergangenheit sind verfügbar.

Beachten Sie, dass Direct-Message-Ereignis-IDs in v1.1 und v2 (sowie in der X App) identisch sind, sodass die v1.1-Methode zum Auflisten eines einzelnen Ereignisses zusammen mit diesen neuen v2-Endpunkten verwendet werden kann. Beachten Sie außerdem, dass die Enterprise- und Premium-Account-Activity-APIs v2-Eins-zu-eins-Nachrichten unterstützen, jedoch noch keine Gruppenunterhaltungen.

Mit dieser Version werden drei Ereignistypen unterstützt; die Endpunkte unterstützen Abfrageparameter, um nach diesen zu filtern:

- **MessageCreate** - Eine Nachricht wurde erstellt.
- **ParticipantsJoin** - Ein neuer Teilnehmer ist einer Unterhaltung beigetreten.
- **ParticipantsLeave** - Ein Teilnehmer hat eine Unterhaltung verlassen.

Es gibt ein nutzerbasiertes Rate-Limit von 300 Anfragen pro 15 Minuten für die GET-Methoden. Dieses Rate-Limit wird über diese GET-Endpunkte hinweg geteilt.

Da Sie im Namen eines Benutzers Anfragen an Direct-Message-v2-Endpunkte stellen, müssen Sie sich entweder mit [OAuth 1.0a User Context](/de/resources/fundamentals/authentication) oder [OAuth 2.0 Authorization Code with PKCE](/de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) authentifizieren und dabei Access Tokens verwenden, die mit Benutzern verknüpft sind, die Ihre X App autorisiert haben. Um diese Access Tokens mit OAuth 1.0a zu generieren, können Sie den [3-legged OAuth flow](/de/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) verwenden. Um Benutzer-Access-Tokens mit OAuth 2.0 zu generieren, können Sie den [Authorization Code with PKCE grant flow](/de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) verwenden.

**Kontoeinrichtung**

Um auf diese Endpunkte zuzugreifen, benötigen Sie:

- Ein genehmigtes [Entwicklerkonto](https://developer.x.com/en/portal/petition/essential/basic-info).
- Eine Authentifizierung mit den Schlüsseln und Tokens aus einer [Developer-App](/de/resources/fundamentals/developer-apps), die sich innerhalb eines [Projekts](/de/resources/fundamentals/projects) befindet.

Erfahren Sie mehr darüber, wie Sie Zugriff auf die Endpunkte der X-API v2 erhalten, in unserem [Leitfaden für den Einstieg](/de/x-api/getting-started/getting-access).
