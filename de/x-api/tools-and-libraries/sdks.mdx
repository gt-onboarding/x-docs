<div id="introduction">
  ## Einführung
</div>

Ein Software Development Kit (SDK) ist eine Sammlung von Softwaretools und Programmen, die für eine bestimmte Plattform oder API zugeschnitten sind. Der Zweck eines SDKs besteht darin, die Funktionalität von Anwendungen zu erstellen oder zu erweitern, indem Bibliotheken oder Codebasen bereitgestellt werden, die Entwickler effizient und mit minimalem Programmieraufwand in ihren Anwendungen nutzen können. Dies beschleunigt den Entwicklungsprozess erheblich und spart Zeit, Geld und Aufwand.

> Die X Developer Platform bietet jetzt zwei offizielle SDKs für alle, die in TypeScript/JavaScript und Java entwickeln. Diese ermöglichen es Entwicklern, effizienter zu arbeiten, da die manuelle Implementierung der Komplexität rund um die X-API v2 entfällt. Stattdessen können die vorgefertigten Funktionen für alle verfügbaren v2-Endpunkte genutzt und der Authentifizierungsprozess vereinfacht werden. Da diese SDKs vom Developer-Platform-Team entwickelt und gepflegt werden, sind sie stets mit zukünftigen Releases der X-API v2 auf dem neuesten Stand.

Da diese SDKs die X-API kapseln, benötigen Sie ein [Developer-Konto](https://developer.x.com/en/portal/dashboard), um Anfragen mit den Zugangsdaten aus einer [Developer-App](/de/resources/fundamentals/developer-apps) zu authentifizieren, die innerhalb eines [Projekts](/de/resources/fundamentals/projects) liegt.

<div id="client-basics">
  #### Grundlagen zum Client
</div>

Importieren Sie die Klassen (Java) bzw. das Paket (TypeScript) am Anfang einer Arbeitsdatei, um Zugriff auf die Authentifizierung und die Bibliotheks-Clients zu erhalten. Um die Methoden des Bibliotheks-Clients zu verwenden, müssen Anmeldedaten übergeben werden. Dies kann entweder ein Bearer-Token (nur App) sein oder eine client\_id/client\_secret-Kombination, wenn mit OAuth 2.0 im Benutzerkontext authentifiziert wird.

So könnte das aussehen:

<Tabs>
  <Tab title="Java">
    ```java
    // Import classes:
    import com.twitter.clientlib.ApiClient;
    import com.twitter.clientlib.ApiException;
    import com.twitter.clientlib.Configuration;
    import com.twitter.clientlib.auth.*;
    import com.twitter.clientlib.model.*;
    import com.twitter.clientlib.TwitterCredentialsBearer;
    import com.twitter.clientlib.api.TwitterApi;

    // Instantiate library client
    TwitterApi apiInstance = new TwitterApi();

    // Instantiate auth credentials (App-only example)
    TwitterCredentialsBearer credentials = new TwitterCredentialsBearer(System.getenv("APP-ONLY-ACCESS-TOKEN"));

    // Pass credentials to library client
    apiInstance.setTwitterCredentials(credentials);
    ```
  </Tab>

  <Tab title="TypeScript">
    ```typescript
    //Import package
    import { Client, auth } from "twitter-api-sdk";

    // Initialize auth client first
    const authClient = new auth.OAuth2User({
     client_id: process.env.CLIENT_ID as string,
     client_secret: process.env.CLIENT_SECRET as string,
     callback: "YOUR-CALLBACK",
     scopes: ["tweet.read", "users.read", "offline.access"],
    });

     // Pass auth credentials to the library client
     const twitterClient = new Client(authClient);
    ```
  </Tab>
</Tabs>

<div id="authentication-flow">
  #### Authentifizierungsablauf
</div>

Wenn Sie die Option „nur Anwendung“ zur Authentifizierung der SDKs verwenden, müssen Sie lediglich das Token bereitstellen; der Bibliotheks-Client kann die Endpoint-Methoden dann sofort nutzen. Beachten Sie, dass Tokens für „nur Anwendung“ nicht für Endpoints verwendet werden können, die eine Authentifizierung im Benutzerkontext erfordern.

Die Authentifizierung im Benutzerkontext mit OAuth 2.0 erfordert nach dem Erstellen des Auth-Clients einige zusätzliche Schritte.

- Autorisierungs-URL erzeugen
- Die Anwendung über die Autorisierungs-URL autorisieren
- Weiterleitung zur Callback-URL (diese muss mit der auf der Seite für Auth-Einstellungen im Developer-Portal konfigurierten Callback-URL übereinstimmen)
- Code Verifier auswerten, um ihn gegen ein Zugriffstoken einzulösen

Die SDKs stellen auf dem Auth-Client Methoden bereit, die diese Schritte vereinfachen. Ein vollständiges Beispiel dafür, wie eine Anfrage mit OAuth 2.0 im Benutzerkontext authentifiziert wird, finden Sie in den GitHub-Repositories.

- [TypeScript](https://github.com/xdevplatform/twitter-api-typescript-sdk/blob/main/examples/oauth2-callback_pkce_s256.ts)

- [Java](https://github.com/xdevplatform/twitter-api-java-sdk/blob/main/examples/src/main/java/com/twitter/clientlib/auth/OAuth20GetAccessToken.java)

<div id="endpoint-methods">
  #### Endpoint-Methoden
</div>

Die im Bibliotheksclient bereitgestellten Methoden sind eindeutig benannt und entsprechen jeweils einem Endpoint; alle Parameter werden als Argumente übergeben.
Hier ein Beispiel für die Beitragsabfrage per ID:

<Tabs>
  <Tab title="Java">
    ```java
    String id = "1511757922354663425"; // String | Eine einzelne Tweet-ID.
    Set<String> expansions = new HashSet<>(Arrays.asList("author_id")); // Set<String> | Eine kommaseparierte Liste von zu erweiternden Feldern.
    Set<String> tweetFields = new HashSet<>(Arrays.asList("created_at", "lang", "context_annotations")); // Set<String> | Eine kommaseparierte Liste von Tweet-Feldern zur Anzeige.
    Set<String> userFields = new HashSet<>(Arrays.asList("created_at", "description", "name")); // Set<String> | Eine kommaseparierte Liste von User-Feldern zur Anzeige.

    try {
     SingleTweetLookupResponse result = apiInstance.tweets().findTweetById(id, expansions, tweetFields, userFields, null, null, null);
     System.out.println(result);
    } catch (ApiException e) {
     System.err.println("Exception when calling TweetsApi#findTweetById");
     System.err.println("Status code: " + e.getCode());
     System.err.println("Reason: " + e.getResponseBody());
     System.err.println("Response headers: " + e.getResponseHeaders());
     e.printStackTrace();
    }

    ```
  </Tab>

  <Tab title="TypeScript">
    ```typescript
    const lookupTweetById = await client.tweets.findTweetById(
      // Tweet-ID
      "1511757922354663425",
      {
        // Optionale Parameter
        expansions: ["author_id"],
        "user.fields": ["created_at", "description", "name"],
      }
    );
    ```
  </Tab>
</Tabs>
