Der Trends lookup endpoint ermöglicht es Entwicklern, die Trends für einen Ort abzurufen, der über die where-on-earth id (WOEID) angegeben wird.

<Note>
  **Hinweis**: WOEID ist ein legacy Identifier, der von Yahoo eingeführt und inzwischen veraltet ist. X API verwendet den numerischen Wert, um Trend-Standorte auf Stadt- und Länderebene zu identifizieren. Reference unseren legacy [blog post](https://blog.x.com/engineering/en_us/a/2010/woeids-in-twitters-trends.html), oder [archived data](https://archive.org/details/geoplanet_data_7.10.0.zip0.)
</Note>

Der `tweet_count` für die letzten 24 Stunden wird ebenfalls für viele Trends zurückgegeben, sofern dieser verfügbar ist.

Dieser Endpunkt unterstützt app-auth-Authentifizierung und hat ein Rate Limit von 75 Requests pro 15-Minuten-Fenster.

<div id="getting-started">
  ### Erste Schritte
</div>

Um diesen Endpoint zu verwenden, benötigen Sie ein \[Bearer-Token]\(https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) aus dem [Developer-Portal](https://developer.x.com/en/portal/dashboard). Sobald Sie das Bearer-Token haben, können Sie die Usage-API wie unten gezeigt aufrufen:

```
      curl 'https://api.x.com/2/trends/by/woeid/26062' --header 'Authorization: Bearer XXXXX'
```

Wenn die Anfrage erfolgreich ist, sehen Sie die JSON-Antwort wie unten dargestellt:

**Kontoeinrichtung**

Um auf diese Endpoints zuzugreifen, benötigen Sie:

- Ein genehmigtes [Developer-Konto](https://developer.x.com/en/portal/petition/essential/basic-info).
- Eine Authentifizierung mit den Schlüsseln und Token aus einer [Developer App](/de/resources/fundamentals/developer-apps), die sich innerhalb eines [Projects](/de/resources/fundamentals/projects) befindet. 

Erfahren Sie mehr über den Zugriff auf die Endpoints der X-API v2 in unserem [Einstiegsleitfaden](/de/x-api/getting-started/getting-access).
