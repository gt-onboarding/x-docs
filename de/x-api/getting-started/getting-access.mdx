<div id="step-one-sign-up-for-a-developer-account">
  ### Schritt eins: Registrieren Sie sich für ein Entwicklerkonto
</div>

Die Registrierung für ein Entwicklerkonto ist schnell und einfach! Klicken Sie unten auf die Schaltfläche, beantworten Sie ein paar Fragen und beginnen Sie, die X-API v2 mit [Basic access](https://developer.x.com/en/portal/products/basic) zu erkunden und darauf aufzubauen.

Als Nächstes erstellen Sie im Rahmen des Onboardings ein Projekt und eine zugehörige Entwickler-App. Dadurch erhalten Sie Zugangsdaten, mit denen Sie alle Anfragen an die API authentifizieren.

[Registrieren](https://developer.x.com/en/portal/products/basic)

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### Schritt zwei: Speichern Sie die Schlüssel und Tokens Ihrer App und halten Sie sie sicher
</div>

Sobald Sie Zugriff haben und ein Projekt sowie eine App erstellt haben, können Sie die folgenden Anmeldeinformationen in Ihrer Entwickler-App finden oder generieren:

- **[API-Schlüssel und -Geheimnis](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret):** Im Wesentlichen der Benutzername und das Passwort Ihrer App. Sie verwenden diese, um Anfragen zu authentifizieren, die [OAuth 1.0a User Context](/de/resources/fundamentals/authentication) erfordern, oder um andere Tokens wie Benutzerzugriffstokens oder App-Zugriffstoken zu generieren.

- **[Zugriffstoken und -Geheimnis](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens):** Zugriffstokens repräsentieren in der Regel den Nutzer, in dessen Namen Sie die Anfrage stellen. Die Tokens, die Sie über das Entwicklerportal generieren können, repräsentieren den Nutzer, dem die App gehört. Sie verwenden diese, um Anfragen zu authentifizieren, die [OAuth 1.0a User Context](/de/resources/fundamentals/authentication) erfordern. Wenn Sie Anfragen im Namen eines anderen Nutzers stellen möchten, müssen Sie den 3-legged-OAuth-Flow verwenden, damit dieser Sie autorisieren kann.

- **[Client-ID und Client-Geheimnis](/de/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce):** Diese Anmeldeinformationen werden verwendet, um ein Benutzerzugriffstoken mit OAuth 2.0-Authentifizierung zu erhalten. Ähnlich wie bei OAuth 1.0a werden Benutzerzugriffstokens verwendet, um Anfragen zu authentifizieren, die private Kontoinformationen bereitstellen oder Aktionen im Namen eines anderen Kontos ausführen – jedoch mit fein abgestuften Berechtigungen für eine präzisere Kontrolle darüber, auf welche Zugriffsrechte die Client-Anwendung beim Nutzer verfügt.

- **[App-only-Zugriffstoken](/de/resources/fundamentals/authentication#bearer-token-also-known-as-app-only):** Verwenden Sie dieses Token, wenn Sie Anfragen an Endpunkte stellen, die öffentlich auf X verfügbare Informationen zurückgeben.

Da diese Schlüssel und Tokens nicht ablaufen, es sei denn, sie werden neu generiert, empfehlen wir, sie unmittelbar nach Erhalt Ihrer Anmeldeinformationen an einem sicheren Ort, z. B. in einem Passwortmanager, zu speichern.

> **Bitte beachten Sie: Ihre Schlüssel und Tokens werden im [Developer-Portal](/de/resources/fundamentals/developer-portal) nur einmal angezeigt. Daher ist es wichtig, dass Sie diese Anmeldeinformationen unmittelbar nach ihrer Generierung in Ihrem Passwortverwaltungssystem speichern.**
>
> Wenn Sie die Schlüssel und Tokens verlegen oder vergessen, müssen Sie sie neu generieren. Dadurch werden neue Anmeldeinformationen erstellt und die alten ungültig. Das bedeutet, dass Sie alle Integrationen aktualisieren müssen, die Sie möglicherweise mit Ihren vorherigen Anmeldeinformationen eingerichtet haben.
>
> Erfahren Sie mehr über unsere [Best Practices zur Authentifizierung](/de/resources/fundamentals/authentication#authentication-best-practices).
