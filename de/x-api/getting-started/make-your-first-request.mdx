Dieser Leitfaden führt Sie Schritt für Schritt durch das Senden Ihrer ersten Anfrage. Er ist eine hervorragende Ressource, sobald Sie ein X-Konto erstellt haben.

Wenn Sie mit Codebeispielen, technischen Anleitungen oder einem grafischen Tool wie Postman arbeiten möchten, verwenden Sie bitte die folgenden Leitfäden, um Ihre erste Anfrage zu senden:

- [Schnellstart: Ihre erste Anfrage an den Post-Endpunkt](/de/x-api/posts/manage-tweets/quickstart)
- [Erste Schritte mit Postman](/de/tutorials/postman-getting-started) – ideal für Einsteiger
- [X-API v2 Codebeispiele](https://github.com/xdevplatform/Twitter-API-v2-sample-code)
- [API-Referenz: Post Lookup](/de/x-api/posts/lookup/introduction)

Dieser Leitfaden setzt voraus, dass Sie Ihren [API-Schlüssel und Ihr Secret](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret), [User Access Token und Secret](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens) sowie das [App Access Token](/de/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) beschafft und sicher gespeichert haben. Wie das geht, erfahren Sie im Leitfaden [Zugriff auf die X-API erhalten](/de/x-api/getting-started/getting-access).

<div id="step-1-identify-which-endpoint-you-would-like-to-use">
  ### Schritt 1. Wählen Sie den Endpunkt aus, den Sie verwenden möchten
</div>

Die X-API ermöglicht es Ihnen, eine Vielzahl unterschiedlicher Aktionen per Code auszuführen, die Sie möglicherweise auch auf der X-Website oder in der mobilen App durchführen können.

Eine vollständige Liste der über die API verfügbaren Endpunkte finden Sie in unserem [API Reference Index](/de/x-api/introduction). Der Einfachheit halber empfehlen wir jedoch, zunächst einen der folgenden Endpunkte zu verwenden:

- [Beiträge verwalten > Einen Tweet veröffentlichen](/de/x-api/posts/manage-tweets/introduction)
- [Beiträge durchsuchen](/de/x-api/posts/search/introduction)
- [Nutzerabfrage](/de/x-api/users/lookup/introduction)

<div id="step-2-choose-a-tool-to-make-your-request">
  ### Schritt 2. Wählen Sie ein Tool, um Ihre Anfrage zu stellen
</div>

Einige Anfragen sind unkompliziert, andere hingegen anspruchsvoller in der Umsetzung. Deshalb hat die großartige Entwicklercommunity Tools geschaffen, die einen Teil der Komplexität abnehmen.

Im Folgenden finden Sie empfohlene Tools sowie Hinweise zu deren Verwendung:

<div id="postman">
  #### Postman
</div>

Postman ist ein visuelles Tool, mit dem Sie Anfragen an REST-Endpunkte senden können. Wir haben hilfreiche Materialien zu Postman erstellt, die Ihnen den Einstieg erleichtern und dabei helfen, die verschiedenen über die X-API verfügbaren Endpunkte zu erkunden.

Wir empfehlen, unser Tutorial „[Erste Schritte mit Postman](/de/tutorials/postman-getting-started)“ zu lesen, um zu lernen, wie Sie Ihre Keys und Tokens hinzufügen und Ihre erste Anfrage senden.

Wir haben außerdem für jeden unserer X-API-v2-Endpunkte eine Schnellstartanleitung erstellt, von denen die meisten Postman verwenden. Sie finden diese Anleitungen in den jeweiligen Endpunktabschnitten; hier sind einige Links:

- Schnellstart: Einen Tweet posten
- Schnellstart: Nach Posts suchen
- Schnellstart: Einen Nutzer suchen

Bitte beachten Sie, dass Sie mit Postman keine Anfragen an Streaming-Endpunkte senden können. Besuchen Sie die Schnellstartanleitung zum [Gefilterten Stream](/de/x-api/posts/filtered-stream/quickstart) oder zum [1 %-Sampled-Stream](/de/x-api/posts/sample-stream), um zu erfahren, wie Sie mit diesen Endpunkten arbeiten.

Wenn Sie ein einfacheres grafisches Tool bevorzugen, sollten Sie auch [Insomnia](https://insomnia.rest) in Betracht ziehen.

<div id="sample-code">
  #### Beispielcode
</div>

Wenn Sie einfachen Code verwenden möchten, um Ihre Anfrage zu stellen, haben wir für jeden unserer X-API-v2-Endpunkte Beispielcode in mehreren Programmiersprachen zusammengestellt.

Sie finden die Codebeispiele in unserem GitHub-Repository, [X-API-v2-sample-code](https://github.com/xdevplatform/Twitter-API-v2-sample-code). Es enthält auch eine README-Datei, die erklärt, wie Sie Ihre Zugangsdaten einrichten, damit die Anfragen korrekt funktionieren.

Hier ist zum Beispiel ein cURL-Beispiel für den User-Lookup-Endpunkt. Alles, was Sie tun müssen, ist $ACCESS\_TOKEN und $USERNAME durch Ihr [App Access Token](/de/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) und Ihren X-Handle zu ersetzen. Kopieren Sie anschließend diesen Code in Ihr Kommandozeilen-Tool und drücken Sie „Return“ oder „Enter“.

```bash
curl "https://api.x.com/2/users/by/username/$USERNAME" -H "Authorization: Bearer $ACCESS_TOKEN"
```

<div id="libraries">
  #### Bibliotheken
</div>

Die großartige X Developer Community hat zudem Bibliotheken in verschiedenen Programmiersprachen erstellt, mit denen Sie Anfragen an die X-API stellen können.

Wir haben eine Seite ["Tools and libraries"](/de/x-api/tools-and-libraries/overview) erstellt, die alle uns bekannten Community-Bibliotheken auflistet. Jede Bibliothek sollte eine README-Datei enthalten, mit der Sie erfahren, wie Sie das Repository auf Ihrem Rechner einrichten und Ihre erste Anfrage stellen.

> **Bitte beachten:** Wenn Sie auf Probleme stoßen, lesen Sie bitte die Entwicklerdokumentation zum Endpunkt, den Sie anfragen, unseren [Support-Bereich](https://developer.x.com/en/support) oder wenden Sie sich für Hilfe an die Community in unseren Foren. Wir helfen Ihnen, eine erfolgreiche Anfrage zu stellen!

<div id="step-3-review-the-response">
  ### Schritt 3. Antwort prüfen
</div>

Sobald Ihre Anfrage erfolgreich war, erhalten Sie eine Payload mit Metadaten zu dieser Anfrage.

Wenn Sie einen Endpoint mit der HTTP-Methode GET verwendet haben, erhalten Sie Metadaten zur Ressource (Post, Nutzer, Liste, Space usw.), an die Sie die Anfrage gerichtet haben, im JSON-Format. Prüfen Sie die zurückgegebenen Felder und ob Sie die angeforderten Informationen dem Inhalt auf X zuordnen können.

Wenn Sie einen Endpoint mit den HTTP-Methoden POST, PUT oder DELETE verwendet haben, haben Sie eine Aktion auf X ausgeführt. Gehen Sie zu X.com oder in die mobile App und prüfen Sie, ob Sie diese Aktion nachvollziehen können.

<div id="step-4-adjust-your-request-using-parameters">
  ### Schritt 4. Passen Sie Ihre Anfrage mithilfe von Parametern an
</div>

Jeder Endpoint verfügt über einen eigenen Satz von Parametern, mit denen Sie Ihre Anfrage verändern können. Sie können beispielsweise zusätzliche Metadatenfelder anfordern, wenn Sie GET-Endpoints mit den Parametern fields und expansions verwenden. Außerdem können Sie mit verschiedenen Filterwerkzeugen experimentieren, etwa über Endpoints wie [Posts durchsuchen](/de/x-api/posts/search/introduction), [Post-Zählungen](/de/x-api/posts/counts/introduction) und [Gefilterter Stream](/de/x-api/posts/filtered-stream/introduction), um die zurückgegebenen Daten auf genau die Posts einzugrenzen, die Sie interessieren.

Eine vollständige Liste der Anfrageparameter und Felder finden Sie in der API-Referenz für den betreffenden Endpoint sowie zahlreiche weitere hilfreiche Integrationsinformationen in unseren Integrationsleitfäden und Grundlagen-Seiten.

Mehr über alle von uns bereitgestellten Schulungsmaterialien erfahren Sie in unserem Leitfaden [Wichtige Ressourcen](/de/x-api/getting-started/important-resources).
