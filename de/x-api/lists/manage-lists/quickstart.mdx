<div id="getting-started-with-the-manage-lists-endpoint-group">
  ## Erste Schritte mit der Endpunktgruppe „Listen verwalten“
</div>

Dieser kurze Überblick hilft Ihnen, Ihre erste Anfrage an die Endpunkte zum Verwalten von Listen mit [Postman](/de/tutorials/postman-getting-started) zu senden.

Wenn Sie Beispielcode in verschiedenen Sprachen sehen möchten, besuchen Sie bitte unser GitHub-Repository mit [Beispielcode für die X-API v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code).

**Hinweis:** In diesem Beispiel senden wir eine Anfrage an den Endpunkt _Create a List_, aber Sie können die Erkenntnisse aus diesem Schnellstart auch auf andere Verwaltungsanfragen anwenden.

<Note>
  **Voraussetzungen**

  Um diese Anleitung abzuschließen, benötigen Sie ein Set an [Schlüsseln und Token](/de/resources/fundamentals/authentication), um Ihre Anfrage zu authentifizieren. Sie können diese Schlüssel und Token generieren, indem Sie die folgenden Schritte ausführen:

  - [Registrieren Sie sich für ein Entwicklerkonto](https://developer.x.com/en/apply-for-access) und lassen Sie es genehmigen.
  - Erstellen Sie ein [Projekt](/de/resources/fundamentals/projects) und eine zugehörige [Developer-App](/de/resources/fundamentals/developer-apps) im Developer-Portal.
  - Navigieren Sie zur Seite „Keys and tokens“ Ihrer App, um die erforderlichen Zugangsdaten zu generieren. Stellen Sie sicher, dass Sie alle Zugangsdaten an einem sicheren Ort speichern.
</Note>

<div id="steps-to-build-a-manage-list-request">
  ### Schritte zum Erstellen einer „Manage List“-Anfrage
</div>

**Schritt eins: Beginnen Sie mit einem Tool oder einer Bibliothek**

Es gibt verschiedene Tools, Codebeispiele und Bibliotheken, mit denen Sie eine Anfrage an diesen Endpoint stellen können. Wir verwenden hier Postman, um den Prozess zu vereinfachen.

Um die X‑API v2-Postman‑Kollektion in Ihre Umgebung zu laden, klicken Sie bitte auf die folgende Schaltfläche:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X‑API v2 zu Postman hinzufügen
</Button>

Sobald Sie die X‑API v2‑Kollektion in Postman geladen haben, navigieren Sie zum Ordner „List“, wählen Sie den Ordner „Manage List“ und dann „Create a List“.

**Schritt zwei: Authentifizieren Sie Ihre Anfrage**

Um eine Anfrage an die X‑API korrekt zu stellen, müssen Sie Ihre Berechtigung nachweisen. Für die Manage‑Posts‑Endpoints müssen Sie Ihre Anfrage entweder mit [OAuth 1.0a User Context](/de/resources/fundamentals/authentication) oder [OAuth 2.0 Authorization Code with PKCE](de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) authentifizieren.

In diesem Beispiel verwenden wir OAuth 1.0a User Context.

Sie müssen Ihre Keys und Tokens (insbesondere Ihren API Key, API Secret Key, OAuth 1.0a User Access Token und OAuth 1.0a User Access Token Secret) in Postman hinterlegen. Wählen Sie dazu die Umgebung „X API v2“ (oben rechts in Postman) und tragen Sie Ihre Keys und Tokens in die Felder „initial value“ und „current value“ ein (klicken Sie auf das Augensymbol neben der Umgebungs‑Dropdownliste).

Wenn Sie dies korrekt gemacht haben, werden diese Variablen automatisch in den Authorization‑Tab der Anfrage übernommen.
 
**Schritt drei: Geben Sie den Namen für die neue List an**

Beim Erstellen einer neuen List mit diesem Endpoint ist ein Name als Body‑Parameter erforderlich. Optional können Sie eine Beschreibung angeben und festlegen, ob die List privat ist.

Navigieren Sie in Postman zum Tab „Body“ und geben Sie den Namen der List als Wert für den Parameter name ein. Wenn Sie zusätzlich eine Beschreibung für die List hinzufügen möchten, fügen Sie einfach einen neuen Key mit der Bezeichnung description auf die gleiche Weise wie den Namen hinzu, gefolgt von der Beschreibung der List als Wert. Das Festlegen der List als privat folgt demselben Muster; für diesen Parameter werden nur true‑ oder false‑Werte akzeptiert. 

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Parameter type** |
| `name` | Name der List (erforderlich) | body |
| description | Beschreibung der List (optional) | body |
| private | true oder false (optional) | body |

Sie sollten nun neben der Schaltfläche „Send“ eine ähnliche URL sehen:

`https://api.x.com/2/lists`

Schritt vier: Senden Sie Ihre Anfrage und prüfen Sie die Antwort

Wenn alles eingerichtet ist, klicken Sie auf „Send“. Sie erhalten dann eine Antwort ähnlich dem folgenden Beispiel:

```
{
  "data": {
    "id": "1441162269824405510",
    "name": "New list created from Postman"
  }
}
```

Wenn das zurückgegebene Response‑Objekt eine id und den Namen Ihrer List enthält, wurde die List erfolgreich erstellt. 

Um eine List zu löschen, wählen Sie die Anfrage „Delete a List“, die sich ebenfalls im Ordner „Lists“ der in Postman geladenen X‑API v2‑Kollektion befindet. Dieser Endpoint erfordert die ID der List, die Sie löschen möchten. Geben Sie im Tab „Params“ die ID der zu löschenden List als Wert für die Spalte id ein. 

Bei einer erfolgreichen Löschanfrage erhalten Sie eine Antwort, die dem folgenden Beispiel ähnelt: 

```
{
  "data": {
    "deleted": true
  }
}
```
