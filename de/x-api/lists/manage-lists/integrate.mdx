  ### Nützliche Tools
</div>

Bevor wir auf einige zentrale Konzepte eingehen, die Ihnen bei der Integration dieses Endpunkts helfen, empfehlen wir, sich mit Folgendem vertraut zu machen:

<div id="postman">
  #### Postman
</div>

Postman ist ein hervorragendes Tool, mit dem Sie einen Endpunkt testen können. Jede Postman-Anfrage enthält alle Pfad- und Body-Parameter, damit Sie schnell erfassen, was Ihnen zur Verfügung steht. Weitere Informationen zu unseren Postman-Sammlungen finden Sie auf der Seite ["Postman verwenden"](/de/tutorials/postman-getting-started).

<div id="authentication">
  #### Authentifizierung
</div>

Alle X-API-v2-Endpunkte erfordern, dass Sie Ihre Anfragen mit einem Satz von Anmeldedaten authentifizieren, auch bekannt als Keys und Tokens.

Diese Endpunkte erfordern die Verwendung des [OAuth-1.0a-User-Context](/de/resources/fundamentals/authentication). Das bedeutet, Sie müssen einen Satz von API-Keys und Benutzer-Access-Tokens verwenden, um eine erfolgreiche Anfrage zu stellen. Die Access-Tokens müssen dem Nutzer zugeordnet sein, in dessen Namen Sie die Anfrage senden. Wenn Sie für einen anderen Nutzer Access-Tokens erzeugen möchten, muss dieser Ihre App über den [3-legged-OAuth-Flow](/de/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) autorisieren bzw. authentifizieren.

Bitte beachten Sie, dass OAuth 1.0a in der Nutzung anspruchsvoll sein kann. Wenn Sie mit dieser Authentifizierungsmethode nicht vertraut sind, empfehlen wir, eine [Bibliothek](/de/x-api/tools-and-libraries/overview) oder ein Tool wie Postman zu verwenden, um Ihre Anfragen korrekt zu authentifizieren.
 

<div id="developer-portal-projects-and-developer-apps">
  #### Developer-Portal, Projekte und Entwickler-Apps
</div>

Um einen Satz von Authentifizierungsdaten zu erhalten, der mit den Endpunkten der X-API v2 funktioniert, müssen Sie sich für ein [Entwicklerkonto registrieren](https://developer.x.com/en/portal/petition/essential/basic-info), ein [Projekt](/de/resources/fundamentals/projects) in diesem Konto einrichten und darin eine [Entwickler-App](/de/resources/fundamentals/developer-apps) erstellen. Anschließend finden Sie Ihre Schlüssel und Token in Ihrer Entwickler-App.
 

<div id="rate-limits">
  #### Rate Limits
</div>

Jeden Tag stellen viele Tausend Entwickler Anfragen an die X-API. Um das hohe Volumen dieser Anfragen zu verwalten, gelten für jeden Endpoint [Rate Limits](/de/x-api/fundamentals/rate-limits), die die Anzahl der Anfragen begrenzen, die Sie im Namen Ihrer App oder eines authentifizierten Nutzers stellen können. 

Diese Endpoints sind auf Nutzerebene ratebegrenzt. Das bedeutet, dass der authentifizierte Nutzer, in dessen Namen Sie die Anfrage stellen, den Endpoint nur eine bestimmte Anzahl von Malen über beliebige Developer-Apps hinweg aufrufen kann. 

Die folgende Tabelle zeigt die Rate Limits für jeden Endpoint.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **HTTP-Methode** | **Rate Limit** |
| /2/lists | POST | 300 Anfragen pro 15 Minuten |
| /2/lists/:id | DELETE / PUT | 300 Anfragen pro 15 Minuten |
