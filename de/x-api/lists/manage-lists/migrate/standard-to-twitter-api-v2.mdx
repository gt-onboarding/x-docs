<div id="manage-lists-standard-v11-compared-to-x-api-v2">
  ### Listen verwalten: Standard v1.1 im Vergleich zur X-API v2
</div>

Wenn Sie mit den Standard-v1.1-Endpunkten [POST lists/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-create), [POST lists/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-destroy) und [POST lists/update](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-update) gearbeitet haben, soll Ihnen dieser Leitfaden helfen, die Gemeinsamkeiten und Unterschiede zwischen den Standard-v1.1- und den X-API-v2-Endpunkten zur Listenverwaltung zu verstehen.

- **Gemeinsamkeiten**
  - Authentifizierung
- **Unterschiede**
  - Endpoint-URLs
  - Anforderungen für Apps und Projekte
  - HTTP-Methoden
  - Rate Limits
  - Anfrageparameter

<div id="similarities">
  #### Gemeinsamkeiten
</div>

**Authentifizierung**

Beide Endpoint-Versionen unterstützen [OAuth 1.0a User Context](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Wenn Sie zuvor einen der Standard-Endpunkte der v1.1 zum Verwalten von Listen verwendet haben, können Sie beim Umstieg auf die X-API v2 weiterhin dieselbe Authentifizierungsmethode verwenden.

<div id="differences">
  #### Unterschiede
</div>

**Endpoint-URLs**

- Standard-v1.1-Endpoints:
  - POST https://api.x.com/1.1/lists/create.json
    (Erstellt eine Liste)
  - POST https://api.x.com/1.1/lists/destroy.json
    (Löscht eine Liste)
  - POST https://api.x.com/1.1/lists/update.json
    (Aktualisiert eine Liste)
- X-API-v2-Endpoint:
  - POST https://api.x.com/2/lists
    (Erstellt eine Liste)

  - DELETE https://api.x.com/2/lists/:id
    (Löscht eine Liste)

  - PUT https://api.x.com/2/lists/:id
    (Aktualisiert eine Liste)

**Ratenlimits**

| **Standard v1.1** | **X-API v2** |
| :--- | :--- |
| /1.1/lists/create.json<br /><br />keine | /2/lists<br /><br />300 Anfragen pro 15‑Minuten‑Fenster mit OAuth 1.0a User Context |
| /1.1/lists/destroy.json<br /><br />keine | /2/lists/:id<br /><br />300 Anfragen pro 15‑Minuten‑Fenster mit OAuth 1.0a User Context |
| /1.1/lists/update.json<br /><br />keine | /2/lists/:id<br /><br />300 Anfragen pro 15‑Minuten‑Fenster mit OAuth 1.0a User Context |

**App- und Projektanforderungen**

Die X-API-v2-Endpoints erfordern, dass Sie bei der Authentifizierung Ihrer Anfragen Anmeldeinformationen aus einer [Developer App](/de/resources/fundamentals/developer-apps) verwenden, die einem [Project](/de/resources/fundamentals/projects) zugeordnet ist. Alle X-API-v1.1-Endpoints können Anmeldeinformationen von eigenständigen Apps oder von Apps verwenden, die mit einem Projekt verknüpft sind.

**Anfrageparameter**

Die folgenden Standard-v1.1-Anfrageparameter haben Entsprechungen in der X-API v2:

**Liste erstellen**

| **Standard** | **X-API v2** |
| :--- | :--- |
| name | name |
| mode | private |
| description | description |

**Liste löschen/aktualisieren**

| **Standard** | **X-API v2** |
| :--- | :--- |
| owner\_screen\_name | Keine Entsprechung |
| owner\_id | Keine Entsprechung |
| list\_id | id  |
| slug | Keine Entsprechung |

<Note>
  **Bitte beachten:** Standard-v1.1-Parameter werden als Query-Parameter übergeben, während die X-API-v2-Parameter als Body-Parameter (für den POST-Endpoint) bzw. als Pfadparameter (für die DELETE- und PUT-Endpoints) übergeben werden.
</Note>
