<div id="getting-started-with-the-pinned-list-endpoint-group">
  ## Erste Schritte mit der Endpunktgruppe für angepinnte Listen
</div>

Dieser kurze Überblick hilft Ihnen, Ihre erste Anfrage an die Endpunkte für angepinnte Listen mit [Postman](/de/tutorials/postman-getting-started) zu stellen.

Besuchen Sie unser GitHub-Repository [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code), wenn Sie Beispielcode in verschiedenen Sprachen sehen möchten.

<Note>
  ### Voraussetzungen

  Um diese Anleitung abzuschließen, benötigen Sie einen Satz an [keys and tokens](/de/resources/fundamentals/authentication), um Ihre Anfrage zu authentifizieren. Sie können diese Keys und Tokens generieren, indem Sie die folgenden Schritte ausführen:

  - [Registrieren Sie sich für ein Entwicklerkonto](https://developer.x.com/en/apply-for-access) und lassen Sie es genehmigen.
  - Erstellen Sie ein [Project](/de/resources/fundamentals/projects) und eine zugehörige [developer App](/de/resources/fundamentals/developer-apps) im Developer-Portal.
  - Navigieren Sie zur Seite „Keys and tokens“ Ihrer App, um die erforderlichen Zugangsdaten zu generieren. Stellen Sie sicher, dass Sie alle Zugangsdaten an einem sicheren Ort speichern.
</Note>

X-API-v2-Postman-Collection laden

Um die X-API-v2-Postman-Collection in Ihren Arbeitsbereich zu laden, klicken Sie bitte auf die folgende Schaltfläche:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2 zu Postman hinzufügen
</Button>

 

<div id="authenticate-your-request">
  #### Authentifizieren Sie Ihre Anfrage
</div>

Um eine erfolgreiche Anfrage an **lookup**-Endpoints zu stellen, können Sie entweder [OAuth 1.0a User Context](/de/resources/fundamentals/authentication), [OAuth 2.0 App-Only](/de/resources/fundamentals/authentication#oauth-2-0) oder [OAuth 2.0 Authorization Code with PKCE](de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) verwenden. Bei **manage**-Endpoints können Sie jedoch nur mit OAuth 1.0a User Context oder OAuth 2.0 Authorization Code with PKCE authentifizieren.

Unabhängig davon werden bei Verwendung von Postman die standardmäßigen Authentifizierungsschlüssel und -tokens automatisch in Ihren Anfragen ausgefüllt, sofern Sie Ihre Umgebung korrekt eingerichtet haben.

Wählen Sie dazu die Umgebung „X API v2“ (oben rechts in Postman) aus und tragen Sie Ihre Schlüssel und Tokens in die Felder „initial value“ und „current value“ ein (klicken Sie auf das Augensymbol neben der Umgebungs-Dropdown-Liste). Diese Schlüssel umfassen Folgendes:

- API Key (auch bekannt als Consumer Key)
- API Secret Key (auch bekannt als Consumer Secret)
- OAuth 1.0a User Access Token
- OAuth 1.0a User Access Token Secret
- OAuth 2.0 App Access Token
- OAuth 2.0 Client Key (nur verfügbar, wenn Sie in den Einstellungen Ihrer App OAuth 2.0 User Authentication eingerichtet haben)
- OAuth 2.0 Client Secret (nur verfügbar, wenn Sie in den Einstellungen Ihrer App OAuth 2.0 User Authentication eingerichtet haben)
