<div id="steps-to-build-a-manage-pinned-list-request">
  ### Schritte zum Erstellen einer Anfrage zum Verwalten angehefteter Listen
</div>

**Schritt eins: Wählen Sie die List-Endpunkt-Sammlung in Postman aus**

Sobald Sie die X-API-v2-Sammlung in Postman geladen haben, navigieren Sie zum Ordner „List“, wählen Sie den Ordner „Pinned Lists“ und dann „Pin a List“.

**Schritt zwei: Geben Sie die anzupinnende Liste an**

Endpunkte zum Verwalten angehefteter Listen erfordern zwei IDs: eine für den Benutzer (den authentifizierten Benutzer, der eine Liste anheftet) und die Zielliste (die Liste, die angeheftet werden soll). Die Benutzer-ID muss der ID des authentifizierenden Benutzers entsprechen, was bedeutet, dass Sie bei der Authentifizierung Ihrer Anfrage die Access Tokens übergeben müssen, die mit der Benutzer-ID verknüpft sind. In diesem Fall können Sie die ID angeben, die zu Ihrem Benutzer gehört. Ihre ID können Sie auf zwei Arten ermitteln:

1. Über den Endpoint [users lookup](/de/x-api/users/lookup/introduction) nach Benutzername können Sie einen Benutzernamen übergeben und das Feld id erhalten.
2. Im Access Token ist der numerische Teil Ihre Benutzer-ID.

Die Ziel-Listen-ID kann zu jeder gültigen Liste gehören. Navigieren Sie in Postman zur Registerkarte „Params“ und tragen Sie die Benutzer-ID in die Spalte „Value“ der id-Pfadvariablen ein. Wechseln Sie zur Registerkarte „Body“ und geben Sie die ID der Liste, die Sie anheften möchten, als Wert für den Parameter list\_id ein. Achten Sie darauf, vor oder nach einer ID keine Leerzeichen einzufügen.

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Parameter type** |
| `id` | Die Benutzer-ID | path |
| list\_id | Die Ziel-Listen-ID, die angeheftet werden soll | body |

Sie sollten nun neben der „Send“-Schaltfläche eine ähnliche URL sehen:

```
      https://api.x.com/2/users/2244994945/pinned_lists
```

Schritt drei: Senden Sie Ihre Anfrage und überprüfen Sie die Antwort

Sobald alles eingerichtet ist, klicken Sie auf die Schaltfläche „Send“, und Sie erhalten eine Antwort ähnlich dem folgenden Beispiel:

```
{
  "data": {
    "pinned": true
  }
}
```

Sie haben die Zielliste erfolgreich angeheftet, wenn das zurückgegebene Response-Objekt für den Schlüssel pinned den Wert true enthält.

Um eine Liste zu lösen, wählen Sie die Anfrage „Unpin List“, die sich ebenfalls im Ordner „Lists“ der in Postman geladenen X-API-v2-Sammlung befindet. Dieser Endpunkt erfordert die authentifizierte Benutzer-ID und die ID der Liste, die gelöst werden soll. Geben Sie auf der Registerkarte „Params“ die Benutzer-ID als Wert für die Spalte id und die ID der zu lösenden Liste als Wert für die Spalte list\_id ein.

Bei einer erfolgreichen Löschanfrage erhalten Sie eine Antwort ähnlich dem folgenden Beispiel:

```
{
  "data": {
    "pinned": false
  }
}
```
