<div id="postman">
  #### Postman
</div>

Postman ist ein hervorragendes Tool, mit dem Sie Endpunkte testen können. Jede Postman-Anfrage enthält alle Pfad- und Body-Parameter, damit Sie schnell nachvollziehen können, was verfügbar ist. Um mehr über unsere Postman-Collections zu erfahren, besuchen Sie bitte unsere ["Using Postman"](/de/tutorials/postman-getting-started) Seite.

<div id="authentication">
  #### Authentifizierung
</div>

Alle Endpunkte der X-API v2 erfordern, dass Sie Ihre Anfragen mit Anmeldedaten (Keys und Tokens) authentifizieren. Sie können den OAuth 1.0a User Context verwenden, um Ihre Anfragen an diesen Endpunkt zu authentifizieren. 

[OAuth 1.0a User Context](/de/resources/fundamentals/authentication) bedeutet, dass Sie einen Satz von API-Keys und Benutzer-Access-Tokens verwenden müssen, um eine erfolgreiche Anfrage zu stellen. Die Access-Tokens müssen dem Nutzer zugeordnet sein, in dessen Namen Sie die Anfrage stellen. Wenn Sie einen Satz von Access-Tokens für einen anderen Nutzer generieren möchten, muss dieser Ihre App über den [3-legged OAuth Flow](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens) autorisieren.

Bitte beachten Sie, dass OAuth 1.0a anspruchsvoll in der Anwendung sein kann. Wenn Sie mit dieser Authentifizierungsmethode nicht vertraut sind, empfehlen wir die Verwendung einer [Bibliothek](/de/x-api/tools-and-libraries/overview) oder eines Tools wie Postman.

<div id="developer-portal-projects-and-developer-apps">
  #### Entwicklerportal, Projekte und Entwickler-Apps
</div>

Um Authentifizierungsdaten zu erhalten, die mit den Endpunkten der X-API v2 funktionieren, müssen Sie sich für ein [Entwicklerkonto anmelden](https://developer.x.com/en/portal/petition/essential/basic-info), in diesem Konto ein [Projekt](/de/resources/fundamentals/projects) einrichten und in diesem Projekt eine [Entwickler-App](/de/resources/fundamentals/developer-apps) erstellen. Anschließend finden Sie Ihre Keys und Tokens in Ihrer Entwickler-App.
 

<div id="rate-limits">
  #### Rate Limits
</div>

Jeden Tag senden viele Tausend Entwickler Anfragen an die X-API. Um das große Volumen dieser Anfragen zu verwalten, werden für jeden Endpunkt [Rate Limits](/de/x-api/fundamentals/rate-limits) festgelegt, die die Anzahl der Anfragen begrenzen, die Sie im Namen Ihrer App oder eines authentifizierten Nutzers stellen können. 

Diese Endpunkte sind auf Nutzerebene ratebegrenzt, was bedeutet, dass der authentifizierte Nutzer, in dessen Namen Sie die Anfrage stellen, den Endpunkt über jede Developer-App hinweg nur eine bestimmte Anzahl von Malen aufrufen kann. 

Die folgende Tabelle zeigt die Rate Limits für jeden Endpunkt.

|     |     |     |
| :--- | :--- | :--- |
| **Endpunkt** | **HTTP-Methode** | **Rate Limit** |
| /2/users/:id/pinned\_lists | POST | 50 Anfragen pro 15 Minuten |
| /2/users/:id/pinned\_lists/:list\_id | DELETE | 50 Anfragen pro 15 Minuten |
| /2/users/:id/pinned\_lists | GET | 15 Anfragen pro 15 Minuten |

<div id="fields-and-expansions">
  #### Felder und Erweiterungen
</div>

Der GET-Endpunkt der X-API v2 ermöglicht es Nutzerinnen und Nutzern, mithilfe der Parameter `fields` und `expansions` genau festzulegen, welche Daten die API zurückgeben soll. Der Parameter `expansions` erlaubt es, im Payload referenzierte Objekte aufzulösen. Wenn Sie beispielsweise angeheftete Listen abrufen, können Sie die folgenden [Expansions](/de/x-api/fundamentals/expansions) einbeziehen:

- `owner_id`

Der Parameter `fields` ermöglicht Ihnen, genau auszuwählen, welche [Felder](/de/x-api/fundamentals/fields) innerhalb der verschiedenen Datenobjekte Sie erhalten möchten. Dieser Endpunkt liefert primär Benutzerobjekte. Standardmäßig gibt das Listenobjekt die Felder `id` und `name` zurück. Um zusätzliche Felder wie `list.created_at` oder `list.description` zu erhalten, müssen Sie diese explizit über einen `fields`-Parameter anfordern.

Wir haben unserer [X-API v2 Datendefinition](/de/x-api/fundamentals/data-dictionary) einen Leitfaden zur gemeinsamen Verwendung von [fields und expansions](/de/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) hinzugefügt.

Die folgende Tabelle zeigt die für den Lookup-Endpunkt verfügbaren Felder und Expansions:

|     |     |     |
| :--- | :--- | :--- |
| **Endpunkt** | **Felder** | **Expansions** |
| /2/users/:id/pinned\_lists | `list.fields`<br /><br />`user.fields` | `owner_id` |
