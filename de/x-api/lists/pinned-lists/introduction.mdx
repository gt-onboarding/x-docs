Das Anheften von Listen ermöglicht X Nutzerinnen und Nutzern einen leichteren Zugriff auf Listen, die sie entweder abonniert oder selbst erstellt haben.

Da Sie bei diesen Endpunkten im Namen eines Nutzers bzw. einer Nutzerin Anfragen stellen, müssen Sie ihn bzw. sie entweder mit [OAuth 1.0a User Context](/de/resources/fundamentals/authentication) oder [OAuth 2.0 Authorization Code with PKCE](de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) authentifizieren und die zugehörigen Benutzerzugriffstoken eines Nutzers bzw. einer Nutzerin verwenden, der bzw. die Ihre App autorisiert hat. Diese können über den [3-legged OAuth flow](/de/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) oder den [Authorization Code with PKCE grant flow](/de/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)) (OAuth 2.0) generiert werden.

<div id="pinned-list-lookup">
  ### Abfrage angehefteter Listen
</div>

Die Abfrage angehefteter Listen hat einen Endpoint, mit dem Sie die angehefteten Listen eines authentifizierten Nutzers abrufen können. Für diesen Endpoint gilt ein Rate-Limit von 15 Anfragen pro 15 Minuten.
 

<div id="manage-pinned-lists">
  ### Angepinnte Listen verwalten
</div>

Die Endpunkte zum Verwalten angepinnter Listen ermöglichen es Ihnen, eine Liste im Namen eines authentifizierten Nutzers anzuheften oder wieder zu lösen. Für diese Endpunkte stehen zwei Methoden zur Verfügung: POST und DELETE. Mit POST können Sie eine Liste anheften, mit DELETE lösen Sie sie. Für beide Endpunkte gilt ein nutzerbasiertes Rate-Limit von 50 Anfragen pro 15 Minuten.

<Note>
  **Kontoeinrichtung**

  Um auf diese Endpunkte zuzugreifen, benötigen Sie:

  - Ein genehmigtes [Developer-Konto](https://developer.x.com/en/portal/petition/essential/basic-info).
  - Eine Authentifizierung mit den Schlüsseln und Tokens aus einer [Developer-App](/de/resources/fundamentals/developer-apps), die innerhalb eines [Projekts](/de/resources/fundamentals/projects) liegt. 

  Erfahren Sie mehr über den Zugriff auf die Endpunkte der X-API v2 in unserem [Getting-Started-Leitfaden](/de/x-api/getting-started/getting-access).
</Note>
