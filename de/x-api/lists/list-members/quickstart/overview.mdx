<div id="getting-started-with-the-list-members-endpoint-group">
  ## Erste Schritte mit der Endpunktgruppe „List members“
</div>

Dieser kurze Überblick hilft Ihnen, Ihre erste Anfrage an die Endpunkte für List members mit [Postman](/de/tutorials/postman-getting-started) zu stellen.

Wenn Sie Beispielcode in verschiedenen Sprachen sehen möchten, besuchen Sie bitte unser GitHub-Repository [Beispielcode für X-API v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code). 

<Note>
  **Voraussetzungen**

  Um diese Anleitung abzuschließen, benötigen Sie einen Satz von [Schlüsseln und Tokens](/de/resources/fundamentals/authentication), um Ihre Anfrage zu authentifizieren. Sie können diese Schlüssel und Tokens generieren, indem Sie die folgenden Schritte ausführen:

  - [Registrieren Sie sich für ein Entwicklerkonto](https://developer.x.com/en/apply-for-access) und lassen Sie es genehmigen.
  - Erstellen Sie ein [Projekt](/de/resources/fundamentals/projects) und eine zugehörige [Developer-App](/de/resources/fundamentals/developer-apps) im Developer-Portal.
  - Navigieren Sie zur Seite „Keys and tokens“ Ihrer App, um die erforderlichen Zugangsdaten zu generieren. Stellen Sie sicher, dass Sie alle Zugangsdaten an einem sicheren Ort speichern.
</Note>

<div id="authenticate-your-request">
  #### Authentifizieren Sie Ihre Anfrage
</div>

Um eine erfolgreiche Anfrage an **lookup**-Endpoints zu senden, können Sie entweder [OAuth 1.0a User Context](/de/resources/fundamentals/authentication#oauth-1-0a-2), [App only](/de/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) oder [OAuth 2.0 Authorization Code mit PKCE](de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) verwenden. Bei **manage**-Endpoints können Sie jedoch nur mit OAuth 1.0a User Context oder OAuth 2.0 Authorization Code mit PKCE authentifizieren.

Unabhängig davon werden bei Verwendung von Postman die standardmäßigen Authentifizierungsschlüssel und -Tokens automatisch in Ihre Anfragen übernommen, sofern Sie Ihre Umgebung korrekt eingerichtet haben.

Wählen Sie dazu die Umgebung mit dem Namen „X API v2“ (oben rechts in Postman) aus und tragen Sie Ihre Schlüssel und Tokens in die Felder „initial value“ und „current value“ ein (klicken Sie auf das Augensymbol neben der Umgebungs-Dropdown-Liste). Diese Schlüssel umfassen Folgendes:

- API Key (auch bekannt als Consumer Key)
- API Secret Key (auch bekannt als Consumer Secret)
- OAuth 1.0a User Access Token
- OAuth 1.0a User Access Token Secret
- App only Access Token
- OAuth 2.0 Client Key (nur verfügbar, wenn Sie in den Einstellungen Ihrer App die OAuth 2.0 User Authentication Settings eingerichtet haben)
- OAuth 2.0 Client Secret (nur verfügbar, wenn Sie in den Einstellungen Ihrer App die OAuth 2.0 User Authentication Settings eingerichtet haben)
