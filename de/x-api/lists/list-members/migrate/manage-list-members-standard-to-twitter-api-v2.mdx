<div id="manage-list-members-standard-v11-compared-to-x-api-v2">
  ### Listenmitglieder verwalten: Standard v1.1 im Vergleich zur X-API v2
</div>

Wenn Sie mit den Standard-v1.1-Endpunkten [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create) und [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy) gearbeitet haben, soll Ihnen dieser Leitfaden helfen, die Gemeinsamkeiten und Unterschiede zwischen den Standard-v1.1-Endpunkten und den X-API-v2-Endpunkten zur Verwaltung von Listenmitgliedern zu verstehen.

- **Gemeinsamkeiten**
  - Authentifizierung
- **Unterschiede**
  - Endpoint-URLs
  - Anforderungen an App und Projekt
  - HTTP-Methoden
  - Rate Limits
  - Anfrageparameter

<div id="similarities">
  #### Gemeinsamkeiten
</div>

**Authentifizierung**

Beide Endpunktversionen unterstützen [OAuth 1.0a User Context](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Wenn Sie zuvor einen der Standard-v1.1-Endpunkte zur Verwaltung von Listenmitgliedern verwendet haben, können Sie bei der Migration zur X-API v2 weiterhin dieselbe Authentifizierungsmethode verwenden.

<div id="differences">
  #### Unterschiede
</div>

**Endpoint-URLs**

- Standard-v1.1-Endpunkte:
  - POST https://api.x.com/1.1/lists/members/create.json
    (Fügt ein Mitglied zu einer angegebenen Liste hinzu)
  - POST https://api.x.com/1.1/lists/members/destroy.json
    (Entfernt ein Mitglied aus einer angegebenen Liste)
- X-API-v2-Endpunkt:
  - POST https://api.x.com/2/lists/:id/members
    (Fügt ein Mitglied zu einer angegebenen Liste hinzu)

  - DELETE https://api.x.com/2/lists/:id/members/:user\_id
    (Entfernt ein Mitglied aus einer angegebenen Liste)

**Rate Limits**

| **Standard v1.1** | **X-API v2** |
| :--- | :--- |
| /1.1/lists/members/create.json<br /><br />keine | /2/lists/:id/members<br /><br />300 Anfragen pro 15‑Minuten‑Fenster mit OAuth 1.0a User Context<br /><br />300 Anfragen pro 15‑Minuten‑Fenster mit OAuth 2.0 Authorization Code mit PKCE |
| /1.1/lists/members/destroy.json<br /><br />keine | /2/lists/:id/members/:user\_id<br /><br />300 Anfragen pro 15‑Minuten‑Fenster mit OAuth 1.0a User Context<br /><br />300 Anfragen pro 15‑Minuten‑Fenster mit OAuth 2.0 Authorization Code mit PKCE |

**App- und Projektanforderungen**

Die X-API-v2-Endpunkte erfordern, dass Sie bei der Authentifizierung Ihrer Anfragen Zugangsdaten aus einer [Developer App](/de/resources/fundamentals/developer-apps) verwenden, die einem [Project](/de/resources/fundamentals/projects) zugeordnet ist. Alle X-API-v1.1-Endpunkte können Zugangsdaten von eigenständigen Apps oder von Apps verwenden, die zu einem Projekt gehören.

**Request-Parameter**

Die folgenden Standard-v1.1-Request-Parameter haben Entsprechungen in der X-API v2:

| **Standard v1.1** | **X-API v2** |
| :--- | :--- |
| list\_id | id  |
| slug | Keine Entsprechung |
| screen\_name | Keine Entsprechung |
| owner\_screen\_name | Keine Entsprechung |
| owner\_id | Keine Entsprechung |
