<div id="comparing-x-apis-list-members-endpoints">
  ## Vergleich der List-Mitgliederendpunkte der X-API
</div>

Die v2-Endpunktgruppe für List-Mitglieder wird die Standard-v1.1-Endpunkte [GET lists/members](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-members), [GET lists/memberships](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-memberships), [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create) und [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy) ersetzen. Wenn Sie Code, Apps oder Tools nutzen, die einen dieser List-Mitgliederendpunkte verwenden, und eine Migration auf den neueren X-API-v2-Endpunkt erwägen, ist diese Leitfadenreihe für Sie gedacht. 

<div id="list-members-lookup">
  ### Nachschlagen von Listenmitgliedern
</div>

Die v2-Endpunkte zum Nachschlagen von Listenmitgliedern ersetzen die Standardendpunkte [GET lists/members](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-members), [GET lists/memberships](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-memberships).

Die folgenden Tabellen vergleichen die Standardendpunkte von v1.1 und der X-API v2 für Listen:

**Nachschlagen von Listenmitgliedern**

|     |     |     |
| :--- | :--- | :--- |
| Beschreibung | Standard v1.1 | X-API v2 |
| Unterstützte HTTP-Methoden | `GET` | `GET` |
| Hostdomain | `https://api.x.com` | `https://api.x.com` |
| Endpunktpfad | `/1.1/lists/members.json` | `/2/lists/:id/members` |
| [Authentifizierung](/de/resources/fundamentals/authentication) | OAuth 1.0a User Context<br /><br />Nur App | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code mit PKCE<br /><br />Nur App |
| Standardmäßige [Rate-Limits](/de/resources/fundamentals/rate-limits) für Anfragen | 900 Anfragen pro 15 Min mit OAuth 1.0a<br /><br />75 Anfragen pro 15 Min nur mit App | 900 Anfragen pro 15 Min mit OAuth 1.0a<br /><br />900 Anfragen pro 15 Min mit OAuth 2.0<br /><br />900 Anfragen pro 15 Min nur mit App |

**Nachschlagen der Listenmitgliedschaften**

|     |     |     |
| :--- | :--- | :--- |
| Beschreibung | Standard v1.1 | X-API v2 |
| Unterstützte HTTP-Methoden | `GET` | `GET` |
| Hostdomain | `https://api.x.com` | `https://api.x.com` |
| Endpunktpfad | `/1.1/lists/memberships.json` | `/2/users/:id/list_memberships` |
| [Authentifizierung](/de/resources/fundamentals/authentication) | OAuth 1.0a User Context<br /><br />Nur App | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code mit PKCE<br /><br />Nur App |
| Standardmäßige [Rate-Limits](/de/resources/fundamentals/rate-limits) für Anfragen | 75 Anfragen pro 15 Min mit OAuth 1.0a<br /><br />75 Anfragen pro 15 Min nur mit App | 75 Anfragen pro 15 Min mit OAuth 1.0a<br /><br />75 Anfragen pro 15 Min mit OAuth 2.0<br /><br />75 Anfragen pro 15 Min nur mit App |

<div id="manage-list-members">
  ### Listenmitglieder verwalten
</div>

Die v2-Endpunkte zum Verwalten von Listenmitgliedern ersetzen die Standardendpunkte [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create) und [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy).

Die folgenden Tabellen vergleichen die Standardendpunkte von v1.1 und der X-API v2 für Listen:

**Mitglied hinzufügen**

| Beschreibung | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| Unterstützte HTTP-Methoden | POST | POST |
| Host-Domain | https://api.x.com | https://api.x.com |
| Endpoint-Pfad | /1.1/lists/members/create.json | /2/lists/:id/members |
| [Authentifizierung](/de/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code mit PKCE |
| Standardmäßige [Rate Limits](/de/resources/fundamentals/rate-limits) für Anfragen | Keine | 300 Anfragen pro 15 Min. (pro Nutzer) |

**Mitglied entfernen**

| Beschreibung | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| Unterstützte HTTP-Methoden | POST | DELETE |
| Host-Domain | https://api.x.com | https://api.x.com |
| Endpoint-Pfad | /1.1/lists/members/destroy.json | /2/lists/:id/:user\_id |
| [Authentifizierung](/de/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code mit PKCE |
| Standardmäßige [Rate Limits](/de/resources/fundamentals/rate-limits) für Anfragen | Keine | 300 Anfragen pro 15 Min. (pro Nutzer) |

Um auf die Endpunkte der X-API v2 zuzugreifen, müssen Sie sich für ein [Entwicklerkonto registrieren](https://developer.x.com/en/portal/petition/essential/basic-info). Bei der Authentifizierung müssen Sie Schlüssel und Token aus einer [Developer App](/de/resources/fundamentals/developer-apps) verwenden, die sich innerhalb eines [Projekts](/de/resources/fundamentals/projects) befindet.

Weitere Informationen zum Zugriff auf die Endpunkte der X-API v2 finden Sie auf unserer Seite [Erste Schritte](/de/x-api/getting-started/getting-access).
