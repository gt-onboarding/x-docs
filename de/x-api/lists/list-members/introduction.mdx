<div id="list-members-lookup">
  ### Abfrage von Listenmitgliedern
</div>

Die Gruppe „Abfrage von Listenmitgliedern“ bietet zwei Endpunkte. Sie können Details zu Mitgliedern einer angegebenen Liste abrufen und sehen, in welchen Listen ein Nutzer Mitglied ist. Diese Endpunkte ermöglichen es, neue Listen auf Grundlage der Mitgliedschaftsinformationen zu kuratieren und zu organisieren.

Für die Abfrage von Mitgliederdetails gilt ein Rate-Limit von 900 Anfragen pro 15 Minuten; für die Abfrage der Nutzermitgliedschaften gilt ein Limit von 75 Anfragen pro 15 Minuten.

Sie können [OAuth 1.0a User Context](/de/resources/fundamentals/authentication), [App only](https://developer.x.com/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) oder [OAuth 2.0 Authorization Code with PKCE](de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) verwenden, um Ihre Anfragen an diesen Endpunkt zu authentifizieren. 
 

<div id="manage-list-members">
  ### Listenmitglieder verwalten
</div>

Die Endpunkte zum Verwalten von Listenmitgliedern ermöglichen es Ihnen, im Namen eines authentifizierten Nutzers Mitglieder zu einer Liste hinzuzufügen oder daraus zu entfernen. Für diese Endpunkte stehen zwei Methoden zur Verfügung: POST und DELETE. Mit der POST-Methode können Sie ein Mitglied zu einer Liste hinzufügen, mit der DELETE-Methode können Sie ein Mitglied aus einer Liste entfernen. Für beide Endpunkte gilt ein nutzerbasiertes Limit von 300 Anfragen pro 15 Minuten.

Beachten Sie, dass Listen nicht mehr als 5.000 Mitglieder haben können.

Da Sie mit diesen Endpunkten Anfragen im Namen eines Nutzers stellen, müssen Sie diesen entweder mit [OAuth 1.0a User Context](/de/resources/fundamentals/authentication) oder [OAuth 2.0 Authorization Code with PKCE](de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) authentifizieren und die Benutzerzugriffstoken eines Nutzers verwenden, der Ihre App autorisiert hat. Diese können über den [3-legged OAuth-Flow](/de/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) oder den [Authorization Code with PKCE grant flow](/de/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0) generiert werden.

<Note>
  **Kontoeinrichtung**

  Um auf diese Endpunkte zuzugreifen, benötigen Sie:

  - Ein genehmigtes [Developer-Konto](https://developer.x.com/en/portal/petition/essential/basic-info).
  - Eine Authentifizierung mit den Schlüsseln und Tokens aus einer [Developer-App](/de/resources/fundamentals/developer-apps), die sich innerhalb eines [Projekts](/de/resources/fundamentals/projects) befindet.

  Erfahren Sie mehr über den Zugriff auf die X-API-v2-Endpunkte in unserem [Einstiegsleitfaden](/de/x-api/getting-started/getting-access).
</Note>
