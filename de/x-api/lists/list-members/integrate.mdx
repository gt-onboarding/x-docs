<div id="postman">
  #### Postman
</div>

Postman ist ein hervorragendes Tool, mit dem Sie Endpunkte testen können. Jede Postman-Anfrage enthält alle Pfad- und Body-Parameter, damit Sie schnell nachvollziehen können, welche Optionen Ihnen zur Verfügung stehen. Weitere Informationen zu unseren Postman-Collections finden Sie auf unserer Seite ["Using Postman"](/de/tutorials/postman-getting-started).

<div id="authentication">
  #### Authentifizierung
</div>

Alle Endpunkte der X-API v2 erfordern, dass Sie Ihre Anfragen mit Anmeldedaten (Keys und Tokens) authentifizieren. Für die Lists-**lookup**-Endpunkte können Sie entweder OAuth 1.0a User Context, OAuth 2.0 Authorization Code mit PKCE oder App-only verwenden. Für die **manage**-Lists-Endpunkte müssen Sie jedoch mit OAuth 1.0a User Context oder OAuth 2.0 authentifizieren.

[OAuth 1.0a User Context](/de/resources/fundamentals/authentication) bedeutet, dass Sie einen Satz aus API-Keys und Benutzer-Access-Tokens verwenden müssen, um eine erfolgreiche Anfrage zu stellen. Die Access-Tokens müssen dem Benutzer zugeordnet sein, in dessen Auftrag Sie die Anfrage stellen. Wenn Sie Access-Tokens für einen anderen Benutzer generieren möchten, muss dieser Ihre App über den [3-legged OAuth Flow](/de/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) autorisieren.

Bitte beachten Sie, dass OAuth 1.0a komplex sein kann. Wenn Sie mit dieser Authentifizierungsmethode nicht vertraut sind, empfehlen wir, eine [Library](/de/x-api/tools-and-libraries/overview) zu verwenden, ein Tool wie Postman zu nutzen oder OAuth 2.0 bzw. App-only für die Authentifizierung Ihrer Anfragen zu verwenden.

[OAuth 2.0 Authorization Code mit PKCE](/de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) ermöglicht eine feinere Kontrolle über den Scope einer Anwendung sowie Autorisierungsflüsse über mehrere Geräte hinweg. Mit OAuth 2.0 können Sie feingranulare Scopes wählen, die Ihnen bestimmte Berechtigungen im Namen eines Benutzers gewähren.

Um OAuth 2.0 in Ihrer App zu aktivieren, müssen Sie dies in den Authentifizierungseinstellungen Ihrer App im Abschnitt „App-Einstellungen“ des Developer-Portals einschalten.

[App-only](/de/resources/fundamentals/authentication#oauth-2-0) erfordert lediglich, dass Sie Ihrer Anfrage ein [App-only Access Token](/de/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) beifügen. Sie können ein App-only Access Token entweder direkt in einer Developer-App generieren oder über den Endpunkt [POST oauth2/token](/de/resources/fundamentals/authentication#post-oauth2-token) erzeugen.

[App-only](/de/resources/fundamentals/authentication#oauth-2-0) erfordert lediglich, dass Sie Ihrer Anfrage ein [App-only Access Token](/de/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) beifügen. Sie können ein App-only Access Token entweder direkt in einer Developer-App generieren oder über den Endpunkt [POST oauth2/token](/de/resources/fundamentals/authentication#post-oauth2-token) erzeugen.

<div id="developer-portal-projects-and-developer-apps">
  #### Developer-Portal, Projekte und Entwickler-Apps
</div>

Um einen Satz von Authentifizierungsdaten zu erhalten, der mit den Endpunkten der X-API v2 funktioniert, müssen Sie sich für ein [Entwicklerkonto registrieren](https://developer.x.com/en/portal/petition/essential/basic-info), ein [Projekt](/de/resources/fundamentals/projects) innerhalb dieses Kontos einrichten und eine [Entwickler-App](/de/resources/fundamentals/developer-apps) innerhalb dieses Projekts erstellen. Anschließend finden Sie Ihre Schlüssel und Token in Ihrer Entwickler-App.
 

<div id="rate-limits">
  #### Rate Limits
</div>

Jeden Tag stellen viele tausend Entwickler Anfragen an die X-API. Um das enorme Volumen dieser Anfragen zu steuern, gelten für jeden Endpoint [Rate Limits](/de/x-api/fundamentals/rate-limits), die die Anzahl der Anfragen begrenzen, die Sie im Namen Ihrer App oder eines authentifizierten Nutzers stellen können.

Lookup-(GET)-Endpoints sind sowohl auf App-Ebene als auch auf Nutzer-Ebene limitiert; Manage-(POST/DELETE)-Endpoints sind auf Nutzer-Ebene limitiert. Das App-Rate-Limit bedeutet, dass Sie als Entwickler innerhalb eines bestimmten Zeitraums nur eine bestimmte Anzahl von Anfragen an diesen Endpoint von einer beliebigen App aus stellen können (dies gilt bei Verwendung des API Key und API Secret Key oder des App-only Access Token). Das Nutzer-Rate-Limit bedeutet, dass der authentifizierte Nutzer, in dessen Namen Sie die Anfrage stellen, eine Listenabfrage nur eine bestimmte Anzahl von Malen über beliebige Entwickler-Apps hinweg durchführen kann.

Die folgende Tabelle zeigt die Rate Limits für jeden Endpoint.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **HTTP-Methode** | **Rate Limit** |
| /2/lists/:id/members | GET | 900 Anfragen pro 15 Minuten |
| /2/users/:id/list\_memberships | GET | 75 Anfragen pro 15 Minuten |
| /2/lists/:id/members | POST | 300 Anfragen pro 15 Minuten |
| /2/lists/:id/members/:user\_id | DELETE | 300 Anfragen pro 15 Minuten |

<div id="fields-and-expansions">
  #### Felder und Erweiterungen
</div>

Der GET-Endpoint der X-API v2 ermöglicht es Nutzerinnen und Nutzern, mit den Parametern `fields` und `expansions` genau festzulegen, welche Daten die API zurückliefert. Der Parameter `expansions` ermöglicht das Einbeziehen von Objekten, auf die im Payload verwiesen wird. Wenn Sie beispielsweise Listenmitglieder abfragen, können Sie die folgenden [Erweiterungen](/de/x-api/fundamentals/expansions) einbeziehen:

- `pinned_tweet_id`

Der Parameter `fields` ermöglicht es Ihnen, genau zu bestimmen, welche [Felder](/de/x-api/fundamentals/fields) innerhalb der verschiedenen Datenobjekte Sie erhalten möchten. Die Abfrage von Listenmitgliedern liefert primär Benutzerobjekte. Standardmäßig gibt das Benutzerobjekt die Felder id, name und username zurück. Um zusätzliche Felder wie `user.created_at` oder `user.description` zu erhalten, müssen Sie diese explizit über den Parameter user.fields anfordern. 

Wir haben eine Anleitung zur Verwendung von [Feldern und Erweiterungen](/de/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) hinzugefügt.

Die folgende Tabelle zeigt die verfügbaren Felder und Erweiterungen für jeden Lookup-Endpoint:

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Felder** | **Erweiterungen** |
| /2/lists/:id/members | `user.fields`<br /><br />`tweet.fields` | `pinned_tweet_id` |
| /2/users/:id/list\_memberships | `list.fields`<br /><br />`user.fields` | `owner_id` |

<div id="pagination">
  #### Paginierung
</div>

Das Abrufen von Mitgliedschaften/Mitgliedern kann sehr viele Daten liefern. Um jederzeit konsistente und performante Ergebnisse bereitzustellen, verwenden wir Paginierung. Paginierung ist eine Funktion in X-API-v2-Endpunkten, die mehr Ergebnisse zurückgeben, als in einer einzelnen Antwort enthalten sein können. In diesem Fall werden die Daten in einer Reihe von „Seiten“ zurückgegeben. Erfahren Sie mehr darüber, wie Sie [Ergebnisse paginieren.](/de/x-api/fundamentals/pagination)
