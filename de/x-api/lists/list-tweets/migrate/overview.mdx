<div id="comparing-x-apis-list-posts-lookup-endpoints">
  ## Vergleich der List-Posts-Lookup-Endpunkte der X-API
</div>

Der v2-List-Posts-Lookup-Endpunkt wird den Standard-v1.1-[GET lists/statuses](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-statuses) ersetzen. Wenn Sie Code, Apps oder Tools verwenden, die diese Version des Endpunkts nutzen, und eine Migration zum neueren X-API-v2-Endpunkt erwägen, sind diese Leitfäden für Sie.

Die folgenden Tabellen vergleichen die Standard-v1.1- und X-API-v2-Listen-Endpunkte:

**Listen-Lookup nach ID**

|     |     |     |
| :--- | :--- | :--- |
| Beschreibung | Standard v1.1 | X-API v2 |
| Unterstützte HTTP-Methoden | `GET` | `GET` |
| Host-Domain | `https://api.x.com` | `https://api.x.com` |
| Endpunktpfad | `/1.1/lists/statuses.json` | `/2/lists/:id/tweets` |
| [Authentifizierung](/de/resources/fundamentals/authentication) | OAuth 1.0a User Context<br /><br />App-only | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code mit PKCE<br /><br />App-only |
| Standardmäßige [Ratenlimits](/de/resources/fundamentals/rate-limits) für Anfragen | 900 Anfragen pro 15 Min mit OAuth 1.0a<br /><br />900 Anfragen pro 15 Min mit App-only | 900 Anfragen pro 15 Min mit OAuth 1.0a<br /><br />900 Anfragen pro 15 Min mit OAuth 2.0<br /><br />900 Anfragen pro 15 Min mit App-only |

Um auf die X-API-v2-Endpunkte zuzugreifen, müssen Sie sich für ein [Developer-Konto registrieren](https://developer.x.com/en/portal/petition/essential/basic-info). Bei der Authentifizierung müssen Sie Schlüssel und Tokens aus einer [Developer-App](/de/resources/fundamentals/developer-apps) verwenden, die einem [Projekt](/de/resources/fundamentals/projects) zugeordnet ist.

Weitere Informationen zum Zugriff auf die X-API-v2-Endpunkte finden Sie auf unserer Seite [Erste Schritte](/de/x-api/getting-started/getting-access).
