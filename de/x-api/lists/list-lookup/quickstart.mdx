<div id="getting-started-with-the-list-lookup-endpoint">
  ## Erste Schritte mit dem Listenabfrage-Endpunkt
</div>

Diese Schnellstartanleitung hilft Ihnen, Ihre erste Anfrage an den Listenabfrage-Endpunkt mit Postman zu stellen.

Bitte besuchen Sie unser GitHub-Repository mit [X-API-v2-Beispielcode](https://github.com/xdevplatform/Twitter-API-v2-sample-code), wenn Sie Beispielcode in verschiedenen Sprachen sehen möchten.

**Hinweis:** In diesem Beispiel senden wir eine Anfrage an den Endpunkt _Listenabfrage nach ID_, die Erkenntnisse aus diesem Schnellstart können Sie jedoch auch auf andere Lookup-Anfragen anwenden.

<Note>
  ### Voraussetzungen

  Um diese Anleitung abzuschließen, benötigen Sie einen Satz an [Keys und Tokens](/de/resources/fundamentals/authentication), um Ihre Anfrage zu authentifizieren. Sie können diese Keys und Tokens generieren, indem Sie die folgenden Schritte befolgen:

  - [Registrieren Sie sich für ein Entwicklerkonto](https://developer.x.com/en/apply-for-access) und warten Sie auf die Genehmigung.
  - Erstellen Sie ein [Projekt](/de/resources/fundamentals/projects) und eine zugehörige [Developer-App](/de/resources/fundamentals/developer-apps) im Developer-Portal.
  - Navigieren Sie zur Seite „Keys and tokens“ Ihrer App, um die erforderlichen Anmeldedaten zu generieren. Stellen Sie sicher, dass Sie alle Anmeldedaten an einem sicheren Ort speichern.
</Note>

<div id="steps-to-build-a-list-lookup-request">
  ### Schritte zum Erstellen einer List-Lookup-Anfrage
</div>

**Schritt eins: Beginnen Sie mit einem Tool oder einer Bibliothek**

Es gibt verschiedene Tools, Codebeispiele und Bibliotheken, mit denen Sie eine Anfrage an diesen Endpoint stellen können. Wir verwenden hier Postman, um den Prozess zu vereinfachen.

Um die X-API-v2-Postman-Collection in Ihre Umgebung zu laden, klicken Sie bitte auf den folgenden Button:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X-API v2 zu Postman hinzufügen
</Button>

Sobald Sie die X-API-v2-Collection in Postman geladen haben, navigieren Sie zum Ordner „List“, wählen Sie den Ordner „List lookup“ und dann „List by ID“.
 

**Schritt zwei: Authentifizieren Sie Ihre Anfrage**

Um korrekt eine Anfrage an die X-API zu stellen, müssen Sie sicherstellen, dass Sie über die erforderlichen Berechtigungen verfügen. Für diesen Endpoint müssen Sie Ihre Anfrage mit einer der folgenden Methoden authentifizieren: [App only](/de/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), [OAuth 2.0 Authorization Code with PKCE](de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) oder [OAuth 1.0a User Context](/de/resources/fundamentals/authentication).

Der Einfachheit halber verwenden wir hier „App only“. Wenn Sie jedoch private [Metriken](/de/x-api/fundamentals/metrics) oder Listen anfragen möchten, müssen Sie eine der anderen Authentifizierungsmethoden nutzen.

Um „App only“ zu verwenden, fügen Sie Ihre Keys und Tokens (insbesondere den [App-only-Access-Token](/de/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)) in Postman hinzu, indem Sie die Umgebung „X API v2“ auswählen (oben rechts in Postman) und Ihre Keys und Tokens in die Felder „initial value“ und „current value“ eintragen (über das Augensymbol neben dem Umgebungs-Dropdown).

Wenn Sie dies korrekt gemacht haben, werden diese Variablen automatisch in den Authorization-Tab der Anfrage übernommen.
 

**Schritt drei: Identifizieren und geben Sie an, welche Liste Sie abrufen möchten**

Sie müssen in der Anfrage eine Liste angeben, die Sie abrufen möchten. Sie finden die Listen-ID, indem Sie zu x.com navigieren, eine Liste öffnen und dann die URL prüfen. In der folgenden URL lautet die Listen-ID zum Beispiel 84839422.

https://x.com/i/lists/84839422

Die Ziel-ID kann jede gültige Listen-ID sein. Navigieren Sie in Postman zum Tab „Params“ und geben Sie Ihre ID in die Spalte „Value“ der id-Pfadvariable ein. Achten Sie darauf, keine Leerzeichen vor oder nach der ID einzufügen.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| id  | 84839422 (Listen-ID) |

**Schritt vier: Identifizieren und geben Sie an, welche Felder Sie abrufen möchten**

Wenn Sie nach Schritt drei auf den Button „Send“ klicken, erhalten Sie in der Antwort die Standardfelder des [List-Objekts](/de/x-api/fundamentals/data-dictionary#list): id, name.

Wenn Sie zusätzliche Felder erhalten möchten, müssen Sie diese in Ihrer Anfrage über list.fields und/oder [expansions](/de/x-api/fundamentals/expansions) angeben.

Für diese Übung fordern wir drei zusätzliche Feldsätze aus unterschiedlichen Objekten an:

- Das zusätzliche Feld created\_at im primären List-Objekt.

- Das vollständige [User-Objekt](/de/x-api/fundamentals/data-dictionary#user) mithilfe des expansions-Parameters

- Das zusätzliche Feld user.created\_at im zugehörigen User-Objekt.

Navigieren Sie in Postman zum Tab „Params“ und fügen Sie das folgende Key:Value-Paar zur Tabelle „Query Params“ hinzu:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| list.fields | created\_at | created\_at |
| expansions | owner\_id | includes.users.id,  <br />includes.users.name,  <br />includes.users.username |
| user.fields | created\_at | includes.users.created\_at |

Sie sollten nun eine ähnliche URL neben dem Button „Send“ sehen:

`https://api.x.com/2/lists/84839422?list.fields=owner_id&expansions=owner_id&user.fields=created_at`

**Schritt fünf: Senden Sie Ihre Anfrage und prüfen Sie die Antwort**

Sobald alles eingerichtet ist, klicken Sie auf „Send“. Sie erhalten dann eine Antwort, die dem folgenden Beispiel ähnelt:

```
{
  "data": {
    "id": "84839422",
    "name": "Offizielle Twitter Accounts",
    "owner_id": "783214"
  },
  "includes": {
    "users": [
      {
        "name": "Twitter",
        "created_at": "2007-02-20T14:35:54.000Z",
        "username": "Twitter",
        "id": "783214"
      }
    ]
  }
}
```
