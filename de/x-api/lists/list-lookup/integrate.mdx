<div id="postman">
  #### Postman
</div>

Postman ist ein hervorragendes Tool, mit dem Sie Endpunkte testen können. Jede Postman-Anfrage enthält alle Pfad- und Body-Parameter, damit Sie schnell nachvollziehen können, was verfügbar ist. Weitere Informationen zu unseren Postman-Collections finden Sie auf unserer Seite ["Postman verwenden"](/de/tutorials/postman-getting-started).

<div id="authentication">
  #### Authentifizierung
</div>

Alle Endpunkte der X-API v2 erfordern, dass Sie Ihre Anfragen mit Anmeldeinformationen (Keys und Tokens) authentifizieren. Sie können dafür entweder OAuth 1.0a User Context, App-only oder OAuth 2.0 Authorization Code mit PKCE verwenden.

[OAuth 1.0a User Context](/de/resources/fundamentals/authentication#oauth-1-0a-user-context) bedeutet, dass Sie einen Satz aus API-Keys und Benutzer-Access-Tokens verwenden müssen, um eine erfolgreiche Anfrage zu stellen. Die Access-Tokens müssen dem Benutzer zugeordnet sein, in dessen Namen Sie die Anfrage stellen. Wenn Sie Access-Tokens für einen anderen Benutzer generieren möchten, muss dieser Ihre App über den [3-legged OAuth-Flow](/de/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) autorisieren.

Bitte beachten Sie, dass OAuth 1.0a anspruchsvoll in der Anwendung sein kann. Wenn Sie mit dieser Authentifizierungsmethode nicht vertraut sind, empfehlen wir die Verwendung einer [Bibliothek](/de/x-api/tools-and-libraries/overview), eines Tools wie Postman oder die Nutzung von OAuth 2.0 zur Authentifizierung Ihrer Anfragen.

[OAuth 2.0 Authorization Code mit PKCE](/de/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) ermöglicht eine feinere Kontrolle über den Scope einer Anwendung und unterstützt Autorisierungsflüsse über mehrere Geräte hinweg. Mit OAuth 2.0 können Sie spezifische, fein granulare Scopes auswählen, die Ihnen bestimmte Berechtigungen im Namen eines Benutzers gewähren.

Um OAuth 2.0 in Ihrer App zu aktivieren, müssen Sie es in den Authentifizierungseinstellungen Ihrer App im Bereich App-Einstellungen des Developer-Portals einschalten.

[App-only](/de/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) erfordert lediglich, dass Sie Ihrer Anfrage ein [App-only-Access-Token](/de/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) beifügen. Sie können ein App-only-Token entweder direkt in einer Developer-App generieren oder über den Endpunkt [POST oauth2/token](/de/resources/fundamentals/authentication#post-oauth2-token) erstellen.

<div id="developer-portal-projects-and-developer-apps">
  #### Developer-Portal, Projekte und Entwickler-Apps
</div>

Um Authentifizierungsdaten zu erhalten, die mit den Endpunkten der X-API v2 funktionieren, müssen Sie sich für ein [Entwicklerkonto anmelden](https://developer.x.com/en/portal/petition/essential/basic-info), ein [Projekt](/de/resources/fundamentals/projects) innerhalb dieses Kontos einrichten und eine [Entwickler-App](/de/resources/fundamentals/developer-apps) innerhalb dieses Projekts erstellen. Anschließend finden Sie Ihre Schlüssel und Token in Ihrer Entwickler-App.
 

<div id="rate-limits">
  #### Ratenlimits
</div>

Jeden Tag stellen viele Tausend Entwickler Anfragen an die X-API. Um das enorme Volumen dieser Anfragen zu verwalten, gelten für jeden Endpunkt [Ratenlimits](/de/x-api/fundamentals/rate-limits), die die Anzahl der Anfragen begrenzen, die du im Namen deiner App oder eines authentifizierten Nutzers stellen kannst. 

Diese Endpunkte sind sowohl auf App- als auch auf Nutzer-Ebene ratebegrenzt. Das App-Ratenlimit bedeutet, dass du als Entwickler von einer beliebigen App aus (unter Verwendung des API Key und API Secret Key oder des App-only Access Token) innerhalb eines bestimmten Zeitraums nur eine festgelegte Anzahl von Anfragen an diesen Endpunkt senden kannst. Das Nutzer-Ratenlimit bedeutet, dass der authentifizierte Nutzer, in dessen Namen du die Anfrage stellst, eine Listenabfrage nur eine bestimmte Anzahl von Malen über alle Entwickler-Apps hinweg durchführen kann.

Die folgende Tabelle zeigt die Ratenlimits für jeden Endpunkt.

|     |     |     |
| :--- | :--- | :--- |
| **Endpunkt** | **HTTP-Methode** | **Ratenlimit** |
| /2/lists/:id | GET | 75 Anfragen pro 15 Minuten |
| /2/users/:id/owned\_lists | GET | 15 Anfragen pro 15 Minuten |

Felder und Erweiterungen

Der GET-Endpunkt der X-API v2 ermöglicht es Nutzern, mithilfe der Parameter `fields` und `expansions` genau auszuwählen, welche Daten die API zurückgeben soll. Der Parameter `expansions` ermöglicht es, im Payload referenzierte Objekte aufzulösen. Beispielsweise erlaubt die Abfrage einer Liste nach ID das Abrufen der folgenden [Expansions](/de/x-api/fundamentals/expansions):

- `owner_id`

Der Parameter `fields` ermöglicht dir, genau festzulegen, welche [Felder](/de/x-api/fundamentals/fields) innerhalb der verschiedenen Datenobjekte du erhalten möchtest. Diese Endpunkte liefern hauptsächlich Listenobjekte. Standardmäßig gibt das Listenobjekt die Felder `id` und `name` zurück. Um zusätzliche Felder wie `list.created_at` oder `list.follower_count` zu erhalten, musst du diese explizit über den Parameter list.fields anfordern. 

Wir haben unserer [X-API v2-Datendictionary](/de/x-api/fundamentals/data-dictionary) eine Anleitung zur gemeinsamen Verwendung von [fields und expansions](/de/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) hinzugefügt.

Die folgende Tabelle zeigt die für diese Endpunktgruppe verfügbaren Felder und Expansions:

|     |     |     |
| :--- | :--- | :--- |
| **Endpunkt** | **Felder** | **Expansions** |
| /2/lists/:id | `list.fields`<br /><br />`user.fields` | `owner_id` |
| /2/users/:id/owned\_lists | list.fields<br /><br />user.fields | owner\_id |

<div id="pagination">
  #### Paginierung
</div>

Das Abrufen nutzereigener Listen kann große Datenmengen liefern. Um jederzeit konsistente und performante Ergebnisse bereitzustellen, verwenden wir Paginierung. Paginierung ist eine Funktion von X-API-v2-Endpunkten, die mehr Ergebnisse zurückgeben, als in einer einzelnen Antwort enthalten sein können. In diesem Fall werden die Daten in einer Reihe von „Seiten“ zurückgegeben. Erfahren Sie mehr darüber, wie Sie [Ergebnisse paginieren.](/de/x-api/fundamentals/pagination)
