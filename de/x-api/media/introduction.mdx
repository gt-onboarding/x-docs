Ein Medienobjekt steht für ein einzelnes Foto, Video oder animiertes GIF. Medienobjekte werden von vielen Endpunkten innerhalb der X-API verwendet und können in Posts, Direct Messages, Profilen, Werbemitteln und an anderen Stellen enthalten sein. Jedes Medienobjekt kann mehrere Anzeige- oder Wiedergabevarianten mit unterschiedlichen Auflösungen oder Formaten aufweisen.

<div id="media-types-size-restrictions">
  ## Medientypen und Größenbeschränkungen
</div>

Größenbeschränkungen für das Hochladen über die API

- **Bild**: `5 MB`
- **GIF**: `15 MB`
- **Video**: `512 MB` (bei Verwendung von `media_category=amplify_video`)

<div id="creation">
  ## Erstellung
</div>

Objekte wie Posts, Direct Messages, Benutzerprofilbilder, gehostete Ads-Karten usw. können ein oder mehrere Medienobjekte enthalten. Diese Objekte der obersten Ebene werden zusammenfassend als Entities bezeichnet. Die relevante API zur Erstellung von Entities (z. B. [`POST /2/tweets`](/de/x-api/posts/creation-of-a-post)) kann ein oder mehrere Medienobjekte über eine eindeutige `media_id` entgegennehmen.

Eine Entity, die Medienobjekte enthält, kann durch folgende Schritte erstellt werden:

1. Laden Sie die Mediendateien entweder mit dem empfohlenen [Chunked-Upload](/de/x-api-/media/quickstart/media-upload-chunked) (Bilder/GIF/Video) oder dem älteren [Simple-Upload](/de/x-api/media-upload/media-upload) (nur Bilder) hoch.
2. Empfangen Sie eine `media_id` aus Schritt 1. Dieser Schritt kann mit unterschiedlichen Medien mehrfach wiederholt werden, wenn die Entity mehrere `media_id`-Parameter zulässt.
3. Erstellen Sie die Entity, indem Sie den entsprechenden Endpoint aufrufen und die `media_id` sowie andere erforderliche Parameter angeben. Beispiel: Fügen Sie einem Post eine `media_id` hinzu, indem Sie den Endpoint [`POST /2/tweets`](/de/x-api/posts/creation-of-a-post) verwenden.
