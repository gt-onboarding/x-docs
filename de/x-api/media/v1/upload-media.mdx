<div id="overview">
  ## Übersicht
</div>

Ein Medienobjekt steht für ein einzelnes Foto, Video oder animiertes GIF. Medienobjekte werden von vielen Endpunkten der X-API verwendet und können in Posts, Direct Messages, Benutzerprofilen, Werbemitteln und an anderer Stelle enthalten sein. Jedes Medienobjekt kann mehrere Anzeige- oder Wiedergabevarianten mit unterschiedlichen Auflösungen oder Formaten besitzen.

<div id="media-types-size-restrictions">
  ## Medientypen und Größenbeschränkungen
</div>

Größenbeschränkungen für Uploads über die API

- Bild: 5 MB
- GIF: 15 MB
- Video: 512 MB (bei Verwendung von media_category=amplify)

<div id="creation">
  ## Erstellung
</div>

Objekte wie Tweets, Direct Messages, Benutzerprofilbilder, gehostete Ads Cards usw. können ein oder mehrere Medienobjekte enthalten. Diese Objekte der obersten Ebene werden zusammengefasst als Entities bezeichnet. Die relevante Entity-Erstellungs-API (z. B. POST statuses/update) kann mithilfe einer eindeutigen media_id ein oder mehrere Medienobjekte entgegennehmen.

Eine Entity, die Medienobjekte enthält, kann wie folgt erstellt werden:

1. Laden Sie die Mediendatei(en) entweder mit dem empfohlenen chunked upload (Bilder/GIF/Video) oder mit dem älteren simple upload (nur Bilder) hoch.
2. Empfangen Sie eine media_id aus Schritt 1. Dieser Schritt kann mit unterschiedlichen Medien mehrfach wiederholt werden, wenn die Entity mehrere media_id-Parameter zulässt.
3. Erstellen Sie die Entity, indem Sie den entsprechenden Endpoint aufrufen und die media_id sowie andere erforderliche Parameter übermitteln. Beispielsweise können Sie eine media_id an einen Tweet anhängen, indem Sie den Endpoint POST statuses/update verwenden.

  ## Abrufen
</div>
