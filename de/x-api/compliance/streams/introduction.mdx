X ist der Community unserer Entwickler verpflichtet, die mit der X-API arbeiten. Im Rahmen dieses Engagements möchten wir unsere API für Entwickler offen und fair, für Menschen auf X sicher und für die X-Plattform insgesamt vorteilhaft gestalten. Es ist entscheidend, dass jeder Entwickler, der X-Inhalte offline speichert, sicherstellt, dass die Daten der Nutzerintention und dem aktuellen Stand der Inhalte auf X entsprechen. Wenn beispielsweise jemand auf X einen Post oder sein Konto löscht, seine Posts schützt oder einen Post bearbeitet, ist es sowohl für X als auch für unsere Entwickler unerlässlich, die Erwartungen und die Intention dieser Person zu respektieren.

Echtzeit-Streams von Compliance-Ereignissen bieten Entwicklern die Werkzeuge, um X-Daten im Einklang mit dem [X Developer Agreement and Policy](https://developer.x.com/en/developer-terms/policy) zu halten.

Es gibt zwei Compliance-Event-Streams: einen für Post-Compliance-Ereignisse und einen für User-Compliance-Ereignisse. Diese Streams sind mit Enterprise-Zugriff verfügbar und wurden entwickelt, um Partnern, die große Datenmengen ingestieren, dabei zu helfen, auf Compliance-Ereignisse wie Post-Edit-Events zu „hören“.

Diese Streams liefern die folgenden Ereignisse:

**Post-Compliance-Stream:**

- delete – zeigt an, dass der Post gelöscht wurde.

- tweet\_edit – zeigt an, dass ein Post bearbeitet wurde, und gibt die ID des aktualisierten Posts an.

- withheld – zeigt an, dass der Post in einem oder mehreren Ländern zurückgehalten wurde.

- drop – zeigt an, dass der Post aus der öffentlichen Ansicht entfernt werden sollte.

- undrop – zeigt an, dass der Post wieder angezeigt und als öffentlich behandelt werden darf.

**User-Compliance-Stream:**

- **user\_delete** – zeigt an, dass das Nutzerkonto gelöscht wurde

- **user\_undelete** – zeigt an, dass das Nutzerkonto wiederhergestellt wurde

- **user\_protect** – zeigt an, dass das Nutzerkonto privat wurde

- **user\_unprotect** – zeigt an, dass das Nutzerkonto öffentlich wurde

- **user\_withheld** – zeigt an, dass das Nutzerkonto in einem oder mehreren Ländern zurückgehalten wurde.

- **user\_suspend** – zeigt an, dass das Nutzerkonto gesperrt wurde

- **user\_unsuspend** – zeigt an, dass die Sperrung des Nutzerkontos aufgehoben wurde

- **user\_profile\_modification** – zeigt an, dass das Nutzerprofil aktualisiert wurde. Dies umfasst eine aktualisierte Beschreibung, den Namen, den Standort und die URL.

**Kontoeinrichtung**

Um auf diese Endpunkte zuzugreifen, benötigen Sie:

- Ein genehmigtes [Entwicklerkonto](https://developer.x.com/en/portal/petition/essential/basic-info).
- Eine Authentifizierung mit den Schlüsseln und Tokens aus einer [Developer App](/de/resources/fundamentals/developer-apps), die sich innerhalb eines [Projects](/de/resources/fundamentals/projects) befindet.

Erfahren Sie in unserem [Getting-Started-Leitfaden](/de/x-api/getting-started/getting-access) mehr darüber, wie Sie Zugriff auf die Endpunkte der X-API v2 erhalten.
