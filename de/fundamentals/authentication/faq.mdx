<div id="general">
  ## Allgemein
</div>

<AccordionGroup>
  <Accordion title="Was ist OAuth?">
    OAuth ist ein Authentifizierungsprotokoll, das es Nutzern ermöglicht, einer Anwendung zu gestatten, in ihrem Namen zu handeln, ohne ihr Passwort weiterzugeben. Weitere Informationen finden Sie auf [oauth.net](http://oauth.net/).
  </Accordion>

  <Accordion title="Wie generiere ich Zugriffstoken?">
    Sie benötigen eine [X-App](/de/resources/fundamentals/developer-apps), um Zugriffstoken zu generieren. Mehr über Zugriffstoken erfahren Sie [hier](/de/resources/fundamentals/authentication#oauth-1-0a-2). 
  </Accordion>

  <Accordion title="Wie erstelle ich eine X-App?">
    Sie benötigen ein [Entwicklerkonto](/de/resources/fundamentals/developer-portal), um eine [X-App](/de/resources/fundamentals/developer-apps) zu erstellen. Sie können sich [hier](https://developer.x.com/en/portal/petition/essential/basic-info) registrieren.
  </Accordion>

  <Accordion title="Wenn ich bereits eine X-App habe, wie kann ich diese App anzeigen und bearbeiten?">
    Sie können Ihre App im [X-App-Dashboard](https://developer.x.com/content/developer-twitter/en/apps) anzeigen und bearbeiten, sofern Sie auf developer.x.com in Ihrem X-Konto angemeldet sind.
  </Accordion>
</AccordionGroup>

<div id="technical">
  ## Technisch
</div>

<AccordionGroup>
  <Accordion title="Wie lange ist ein Zugriffstoken gültig?">
    Zugriffstokens laufen nicht explizit ab. Ein Zugriffstoken wird ungültig, wenn ein Nutzer eine Anwendung in seinen X-Kontoeinstellungen ausdrücklich widerruft oder wenn X eine Anwendung suspendiert. Falls eine Anwendung suspendiert wird, erscheint im [X-App](/de/resources/fundamentals/developer-apps)-Dashboard ein Hinweis, dass sie suspendiert wurde.
  </Accordion>

  <Accordion title="Was passiert, wenn ein Zugriffstoken ungültig wird?">
    Gehen Sie davon aus, dass das Zugriffstoken eines Nutzers jederzeit ungültig _werden kann_. In diesem Fall sollten Sie den Nutzer auffordern, die Anwendung erneut zu autorisieren. Eine reibungslose Handhabung dieser Situation ist wichtig für eine gute Nutzererfahrung.
  </Accordion>

  <Accordion title="Auf der Registrierungsseite für Anwendungen wird nach Lese-/Schreibzugriff gefragt. Was gilt als Schreibvorgang?">
    Viele Nutzer vertrauen einer Anwendung das Lesen ihrer Informationen an, möchten aber nicht unbedingt, dass ihre Profilinformationen geändert oder neue Beiträge veröffentlicht werden. Das Aktualisieren von Informationen über die X-API – sei es der Name, der Standort oder das Hinzufügen eines neuen Beitrags – erfordert einen HTTP POST. Jede API-Methode, die einen HTTP POST erfordert, gilt als Schreibmethode und benötigt Lese- und Schreibzugriff.
  </Accordion>
</AccordionGroup>
