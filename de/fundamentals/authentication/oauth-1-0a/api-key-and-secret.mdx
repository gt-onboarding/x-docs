<div id="api-key-and-secret">
  ### API Key und Secret
</div>

Der API Key und das Secret (auch bekannt als Consumer Key und Secret) sind die grundlegendsten Zugangsdaten, die für den Zugriff auf die X-API erforderlich sind. Diese Zugangsdaten fungieren als Benutzername und Passwort für Ihre X-App und werden von der X-API verwendet, um zu erkennen, von welcher App Anfragen stammen.

Diese Zugangsdaten können von [Authentifizierungsendpunkten](/de/resources/fundamentals/authentication/api-reference) verwendet werden, um zusätzliche Zugangsdaten zu erzeugen, wie etwa [User Access Tokens und Secrets](/de/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret) sowie [Bearer Tokens](/de/resources/fundamentals/authentication/oauth-2-0/bearer-tokens). Sie müssen diese Zugangsdaten außerdem zusammen mit Access Tokens und weiteren Autorisierungsparametern verwenden, um [Anfragen zu autorisieren](/de/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request), die eine OAuth 1.0a User Context-Authentifizierung erfordern. 

<div id="how-to-acquire-an-api-key-and-secret">
  #### So erhalten Sie einen API-Schlüssel und ein Secret
</div>

Um einen X API Key und ein Secret zu erhalten, befolgen Sie bitte diese Schritte:

1. [Registrieren Sie sich für ein X-Entwicklerkonto](https://developer.x.com/en/apply-for-access)
2. Erstellen Sie eine [X App](/de/resources/fundamentals/developer-apps) im [Developer-Portal](/de/resources/fundamentals/developer-portal). Beachten Sie, dass Sie, wenn Sie die [X-API v2](/de/x-api/introduction) nutzen möchten, Ihre X App zu einem [Project](/de/resources/fundamentals/projects) hinzufügen müssen.
    

Wenn Sie Ihre X App erstellen, werden Ihnen Ihr API Key und Secret zusammen mit einem Bearer-Token angezeigt. Bitte beachten Sie, dass wir diese Anmeldedaten nur einmal anzeigen. Stellen Sie daher sicher, dass Sie sie in Ihrem Passwortmanager oder an einem sicheren Ort speichern.

Weitere Empfehlungen zum Umgang mit Ihren Keys und Tokens finden Sie auf unserer Seite zu den [Best Practices für die Authentifizierung](/de/resources/fundamentals/authentication/guides/authentication-best-practices), einschließlich Details dazu, was Sie tun sollten, wenn Ihre Zugangsdaten kompromittiert wurden.
 

<div id="how-to-find-and-regenerate-your-api-key-and-secret-after-app-creation">
  #### So finden und regenerieren Sie Ihren API-Key und -Secret nach der Erstellung der App
</div>

Wenn Sie bereits eine App erstellt haben und Ihren API-Key und -Secret finden oder regenerieren müssen, befolgen Sie bitte diese Schritte:

1. Navigieren Sie zum Developer-Portal
2. Klappen Sie in der Seitenleiste das Dropdown „Projekte und Apps“ auf
3. Öffnen Sie die App, der der API-Key und -Secret zugeordnet ist, den bzw. das Sie finden oder regenerieren möchten
4. Wechseln Sie zum Tab „Keys and tokens“

Dort finden Sie alle zu Ihrer App gehörenden Zugangsdaten. 
 

<div id="how-to-use-your-api-key-and-secret">
  #### So verwenden Sie Ihren API-Key und Ihr Secret
</div>

Wenn Sie die X Developer Platform zunächst nur erkunden, empfehlen wir die Verwendung eines [Tools oder einer Bibliothek](/de/resources/tools-and-libraries), um einen Überblick über die verfügbaren Funktionen zu erhalten. Diese Tools übernehmen die Authentifizierung zuverlässig und ersparen Ihnen viel Zeit und Frustration. Wir empfehlen insbesondere [den Einstieg mit Postman](/de/tutorials/postman-getting-started) oder [Insomnia](https://insomnia.rest/) für Entwicklerinnen und Entwickler, die gerade beginnen.

Wenn Sie eine Anfrage von Grund auf selbst erstellen möchten, lesen Sie bitte unseren Leitfaden zum [Autorisieren einer OAuth-1.0a-Anfrage](/de/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request).
