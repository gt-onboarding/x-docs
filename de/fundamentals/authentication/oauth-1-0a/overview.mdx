<div id="oauth-10a">
  ## OAuth 1.0a
</div>

Viele Endpunkte der X Developer Platform verwenden die OAuth-1.0a-Methode, um im Namen eines X-Kontos zu handeln oder API-Anfragen zu stellen. Wenn Sie beispielsweise eine X-Developer-App haben, können Sie API-Anfragen im Namen eines beliebigen X-Kontos stellen, sofern dieser Nutzer Ihre App authentifiziert.

Bitte beachten Sie: Wenn Ihnen Konzepte wie HMAC-SHA1 und Prozent-Codierung nicht vertraut sind, empfehlen wir, den unten stehenden Abschnitt „Nützliche Tools“ zu lesen. Dort sind einige API-Clients aufgeführt, die den Authentifizierungsprozess erheblich vereinfachen.

<div id="key-concepts">
  ### Zentrale Konzepte
</div>

<div id="signing-a-request-with-keys-and-tokens">
  #### Eine Anfrage mit Schlüsseln und Tokens signieren
</div>

Sie müssen jede API-Anfrage signieren, indem Sie mehrere generierte Schlüssel und Tokens im Authorization-Header übergeben. Zu Beginn können Sie auf der Detailseite Ihrer [X-Entwickler-App](/de/resources/fundamentals/developer-apps) mehrere Schlüssel und Tokens generieren, darunter die folgenden:

|     |     |
| :--- | :--- |
| API-Schlüssel und -Geheimnis:<br /><br />`oauth_consumer_key`<br /><br />`oauth_consumer_secret` | Betrachten Sie diese als den Benutzernamen und das Passwort, die Ihre X-Entwickler-App bei API-Anfragen repräsentieren. |
| Zugriffstoken und -Geheimnis:<br /><br />`oauth_token`<br /><br />`oauth_token_secret` | Ein Zugriffstoken und ein Zugriffstoken-Geheimnis sind benutzerspezifische Anmeldedaten, die zur Authentifizierung von OAuth-1.0a-API-Anfragen verwendet werden. Sie geben das X-Konto an, in dessen Namen die Anfrage gestellt wird.<br /><br />Sie können Ihr eigenes Zugriffstoken und -Geheimnis generieren, wenn Ihre App im Namen desselben X-Kontos Anfragen stellen soll, das mit Ihrem Entwicklerkonto auf der Detailseite der X-Entwickler-App verknüpft ist.<br /><br />Wenn Sie Zugriffstoken für einen anderen Nutzer generieren möchten, siehe unten „Anfragen im Namen von Nutzern stellen“. |

<div id="making-requests-on-behalf-of-users">
  #### Anfragen im Namen von Nutzenden stellen
</div>

Beim Erstellen einer Signatur benötigen Sie ein Set an Zugriffstokens, die die Person repräsentieren, in deren Namen Sie die Anfrage stellen.

Sie können ein Set an Zugriffstokens generieren, das das X-Konto repräsentiert, dem die X-Entwickler-App gehört, über die [Detailseite der App](https://developer.x.com/content/developer-twitter/en/apps). Wenn Sie jedoch eine Anfrage im Namen eines anderen X-Kontos stellen möchten, muss die Inhaberin bzw. der Inhaber dieses Kontos Ihnen Zugriff gewähren, indem sie bzw. er sich im Rahmen des [3-legged OAuth-Flows](/de/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) in ihrem bzw. seinem Konto anmeldet. Das Ergebnis dieses Prozesses ist ein Set an Zugriffstokens (oauth\_token und oauth\_token\_secret), mit denen eine OAuth‑1.0a-Anfrage gestellt werden kann.

Sobald Sie diese Keys und Tokens haben, können Sie entweder [eine Signatur](/de/resources/fundamentals/authentication#creating-a-signature) von Grund auf erstellen. Wir empfehlen dies nur, wenn Sie genau wissen, was Sie tun, oder wenn Sie eines der unten genannten Tools verwenden, um eine Anfrage an einen Endpunkt zu stellen, der OAuth 1.0a erfordert.

Zur Referenz finden Sie hier ein Beispiel für eine signierte cURL-Anfrage, bei der alle generierten Tokens in einem Authorization-Header übergeben werden:

```bash
curl --request POST \
  --url 'https://api.x.com/1.1/statuses/update.json?status=Hello%20world' \
  --header 'authorization: OAuth oauth_consumer_key="CONSUMER_API_KEY", oauth_nonce="OAUTH_NONCE", oauth_signature="OAUTH_SIGNATURE", oauth_signature_method="HMAC-SHA1", oauth_timestamp="OAUTH_TIMESTAMP", oauth_token="ACCESS_TOKEN", oauth_version="1.0"' \
```

<Note>
  **Hinweis:** Zugriffstokens für Nutzende sind sensibel und sollten äußerst sorgfältig geschützt werden. Wenn Zugriffstokens generiert werden, vertraut die vertretene Person darauf, dass Ihre Anwendung sie sicher verwahrt. Wenn sowohl die Sicherheit der API-Schlüssel als auch der Zugriffstokens für Nutzende kompromittiert ist, könnte Ihre Anwendung potenziell Zugriff auf private Informationen und Kontofunktionen offenlegen. Wir empfehlen Ihnen, mehr über das [Sichern von Schlüsseln und Zugriffstokens](/de/resources/fundamentals/developer-apps) zu erfahren.
</Note>

<div id="useful-tools">
  ### Nützliche Tools
</div>

Das [Signieren einer Anfrage](/de/resources/fundamentals/authentication#creating-a-signature) ist komplex. Wir empfehlen, eine API-Client-Bibliothek zu verwenden, die einen Großteil der Authentifizierungstokens automatisch generiert:

|     |     |
| :--- | :--- |
| [Postman](https://www.getpostman.com/) | Ein API-Client, mit dem Sie REST-API-Anfragen erstellen und senden können. Lesen Sie unser Tutorial „[Erste Schritte mit Postman](/de/tutorials/postman-getting-started)“, um mehr über dieses Tool zu erfahren. |
| [Insomnia](https://insomnia.rest/) | Insomnia ist ein REST-API-Client mit Cookie-Verwaltung, Umgebungsvariablen, Codegenerierung und Authentifizierung für macOS, Windows und Linux. |
