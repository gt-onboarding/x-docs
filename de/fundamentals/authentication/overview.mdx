Die X-APIs verarbeiten enorme Datenmengen. Wir sichern diese Daten für Entwickler und Nutzer durch Authentifizierung. Es gibt mehrere Authentifizierungsmethoden, die unten aufgeführt sind.

Die meisten Entwickler müssen sich nicht mit den damit verbundenen Komplexitäten befassen, da Clientbibliotheken dies automatisch übernehmen.

Eine Liste verfügbarer Clientbibliotheken finden Sie auf unserer Seite [Tools and libraries](/de/resources/tools-and-libraries).

<div id="authentication-methods">
  ## Authentifizierungsmethoden
</div>

<CardGroup cols={2}>
  <Card title="OAuth 1.0a User Context" href="/de/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret">
    OAuth 1.0a ermöglicht einer autorisierten X-Entwickler-App, auf private Kontoinformationen zuzugreifen oder Aktionen im Namen eines X-Kontos auszuführen.

    <br />

    [**Mehr erfahren**](/de/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret)
  </Card>

  <Card title="App only" href="/de/resources/fundamentals/authentication/oauth-2-0/overview">
    Ein App-only-Access-Token ermöglicht einer X-Entwickler-App den Zugriff auf öffentlich verfügbare Informationen auf X.

    <br />

    [**Mehr erfahren**](/de/resources/fundamentals/authentication/oauth-2-0/overview)
  </Card>

  <Card title="Basic authentication" href="/de/resources/fundamentals/authentication/basic-auth">
    Viele Enterprise-APIs von X erfordern die Verwendung von HTTP Basic Authentication.

    <br />

    [**Mehr erfahren**](/de/resources/fundamentals/authentication/basic-auth)
  </Card>

  <Card title="OAuth 2.0 Authorization Code Flow with PKCE" href="/de/resources/fundamentals/authentication/oauth-2-0/authorization-code">
    Der OAuth-2.0-User-Context ermöglicht die Authentifizierung im Namen eines anderen Kontos, mit feinerer Kontrolle über den Scope einer Anwendung und Autorisierungsabläufen über mehrere Geräte hinweg.

    <br />

    [**Mehr erfahren**](/de/resources/fundamentals/authentication/oauth-2-0/authorization-code)
  </Card>
</CardGroup>

<Note>
  **Hinweis:**
  Die API-Schlüssel Ihrer App und das App-only-Access-Token sowie Ihr persönliches Access-Token und Access-Token-Secret können im Bereich [X developer Apps](/de/resources/fundamentals/developer-apps) im [Developer-Portal](/de/resources/fundamentals/developer-portal) abgerufen werden.

  **Wenn Sie Anfragen im Namen eines anderen Nutzers stellen möchten**, müssen Sie für diesen Nutzer mit dem [3-legged OAuth Flow](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens) einen separaten Satz von Access-Tokens generieren und die Tokens dieses Nutzers mit Ihren Anfragen im OAuth-1.0a-User-Context oder im OAuth-2.0-User-Context übermitteln.
</Note>
