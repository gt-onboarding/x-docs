### POST oauth/request\_token

Ermöglicht einer Consumer-Anwendung, ein OAuth-Request-Token zu erhalten, um die Benutzerautorisierung anzufordern. Diese Methode erfüllt [Abschnitt 6.1](https://oauth.net/core/1.0/#auth_step1) des [OAuth-1.0-Authentifizierungsablaufs](http://oauth.net/core/1.0/#anchor9).

**Wir verlangen, dass Sie für alle OAuth-Autorisierungsschritte HTTPS verwenden.**

**Hinweis zur Verwendung:** Für den `oauth_nonce` werden nur ASCII-Werte akzeptiert.

**Ressourcen-URL[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth/request_token`

**Ressourceninformationen[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Antwortformate | JSON |
| Erfordert Authentifizierung? | Nein |
| Ratenbegrenzung? | Ja |

**Parameter[](#parameters "Permalink to this headline")**

| Name | Erforderlich | Beschreibung | Beispiel |
| :--- | :--- | :--- | :--- |
| oauth\_callback | erforderlich | Für die OAuth-1.0a-Compliance ist dieser Parameter **erforderlich**. Der hier angegebene Wert wird als die URL verwendet, zu der ein Nutzer weitergeleitet wird, wenn er den Zugriff Ihrer Anwendung auf sein Konto genehmigt. Setzen Sie diesen auf `oob` für den Out-of-Band-PIN-Modus. Auf diese Weise geben Sie auch benutzerdefinierte Callbacks für Desktop-/Mobilanwendungen an. Senden Sie in diesem Schritt immer einen `oauth_callback`, unabhängig von einem vorab registrierten Callback.<br /><br />Wir verlangen, dass jede mit diesem Endpunkt verwendete Callback-URL in den Einstellungen der App auf developer.x.com konfiguriert ist.\* | `http://themattharris.local/auth.php` `twitterclient://callback` |
| x\_auth\_access\_type | optional | Überschreibt die Zugriffsebene, die eine Anwendung für das Konto eines Nutzers anfordert. Unterstützte Werte sind `read` oder `write`. Dieser Parameter ermöglicht es, eine Lese-/Schreibanwendung zu registrieren, aber bei Bedarf nur Lesezugriff anzufordern. |     |

Erfahren Sie mehr darüber, wie Sie Ihre Callback-URLs genehmigen, auf [dieser Seite](/de/resources/fundamentals/developer-apps#callback-urls).

**Bitte beachten** – Sie können Ihre vorhandenen [X-Apps](/de/resources/fundamentals/developer-apps) über das [X-App-Dashboard](https://developer.x.com/en/apps) anzeigen und bearbeiten, wenn Sie mit Ihrem X-Konto auf developer.x.com angemeldet sind.

**Beispielanfrage[](#example-request "Permalink to this headline")**

Request-URL: `POST https://api.x.com/oauth/request_token`

Request-POST-Body: _N/V_

Authorization-Header: `OAuth oauth_nonce="K7ny27JTpKVsTgdyLdDfmQQWVLERj2zAK5BslRsqyw", oauth_callback="http%3A%2F%2Fmyapp.com%3A3005%2Ftwitter%2Fprocess_callback", oauth_signature_method="HMAC-SHA1", oauth_timestamp="1300228849", oauth_consumer_key="OqEqJeafRSF11jBMStrZz", oauth_signature="Pc%2BMLdv028fxCErFyi8KXFM%2BddU%3D", oauth_version="1.0"`

Antwort: `oauth_token=Z6eEdO8MOmk394WozF5oKyuAv855l4Mlqo7hhlSLik&oauth_token_secret=Kd75W4OQfb2oJTV0vzGzeXftVAwgMnEK9MumzYcM&oauth_callback_confirmed=true`

<div id="get-oauthauthorize">
  ### GET oauth/authorize
</div>

Ermöglicht einer Consumer-Anwendung, ein OAuth-Request-Token zu verwenden, um die Autorisierung durch den Nutzer anzufordern. Diese Methode erfüllt [Abschnitt 6.2](http://oauth.net/core/1.0/#auth_step2) des [OAuth 1.0-Authentifizierungsablaufs](http://oauth.net/core/1.0/#anchor9). Desktop-Anwendungen müssen diese Methode verwenden (und können [GET oauth/authenticate](/de/resources/fundamentals/authentication/api-reference#get-oauth-authenticate) nicht verwenden).

**Hinweis zur Verwendung:** Ein `oauth_callback` wird nie an diese Methode gesendet; geben Sie ihn stattdessen bei [POST oauth/request\_token](/de/resources/fundamentals/authentication/api-reference#post-oauth-request-token) an.

**Ressourcen-URL[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth/authorize`

**Ressourceninformationen[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Antwortformat | JSON |
| Authentifizierung erforderlich? | Ja |
| Rate-Limit? | Ja |

**Parameter[](#parameters "Permalink to this headline")**

|     |     |     |     |     |
| :--- | :--- | :--- | :--- | :--- |
| Name | Erforderlich | Beschreibung | Standardwert | Beispiel |
| force\_login | optional | Erzwingt, dass der Nutzer seine Anmeldedaten eingibt, um sicherzustellen, dass das richtige Nutzerkonto autorisiert wird. |     |     |
| screen\_name | optional | Füllt das Benutzernamenfeld des OAuth-Anmeldebildschirms mit dem angegebenen Wert vor. |     |     |

**Beispielanfrage[](#example-request "Permalink to this headline")**

Leiten Sie den Nutzer im Webbrowser zum Schritt `oauth/authorize` weiter und fügen Sie einen oauth\_token-Parameter hinzu:
`https://api.x.com/oauth/authorize?oauth_token=Z6eEdO8MOmk394WozF5oKyuAv855l4Mlqo7hhlSLik`

<div id="get-oauthauthenticate">
  ### GET oauth/authenticate
</div>

Ermöglicht es einer Consumer-Anwendung, einen OAuth-`request_token` zu verwenden, um die Autorisierung des Nutzers anzufordern.

Diese Methode ersetzt [Abschnitt 6.2](http://oauth.net/core/1.0/#auth_step2) des [OAuth-1.0-Authentifizierungsablaufs](http://oauth.net/core/1.0/#anchor9) für Anwendungen, die den Callback-Authentifizierungsablauf verwenden. Die Methode verwendet den aktuell angemeldeten Nutzer als Konto für die Zugriffsautorisierung, es sei denn, der Parameter `force_login` ist auf `true` gesetzt.

Diese Methode unterscheidet sich von [GET oauth/authorize](/de/resources/fundamentals/authentication/api-reference#get-oauth-authorize) dadurch, dass, wenn der Nutzer der Anwendung bereits Berechtigungen erteilt hat, die Weiterleitung erfolgt, ohne dass der Nutzer die Anwendung erneut genehmigen muss. Um dieses Verhalten zu ermöglichen, müssen Sie die Einstellung _Mit X anmelden verwenden_ in Ihrem [App-Eintrag](https://developer.x.com/apps) aktivieren.

**Ressourcen-URL[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth/authenticate`

**Ressourceninformationen[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Antwortformate | JSON |
| Authentifizierung erforderlich? | Ja |
| Ratenbegrenzung? | Ja |

**Parameter[](#parameters "Permalink to this headline")**

|     |     |     |     |     |
| :--- | :--- | :--- | :--- | :--- |
| Name | Erforderlich | Beschreibung | Standardwert | Beispiel |
| force\_login | optional | Erzwingt, dass der Nutzer seine Anmeldedaten eingibt, um sicherzustellen, dass das richtige Nutzerkonto autorisiert ist. |     | _true_ |
| screen\_name | optional | Füllt das Benutzernamenfeld des OAuth-Anmeldebildschirms mit dem angegebenen Wert vor. |     |     |

**Beispielanforderung[](#example-request "Permalink to this headline")**

Leiten Sie den Nutzer im Webbrowser zum Schritt `oauth/authenticate` weiter und fügen Sie einen oauth\_token-Parameter hinzu:
`https://api.x.com/oauth/authenticate?oauth_token=Z6eEdO8MOmk394WozF5oKyuAv855l4Mlqo7hhlSLik`

### POST oauth/access\_token

Ermöglicht es einer Consumer-Anwendung, das OAuth-Request-Token gegen ein OAuth-Access-Token auszutauschen. Diese Methode erfüllt [Abschnitt 6.3](http://oauth.net/core/1.0/#auth_step3) des [OAuth-1.0-Authentifizierungsablaufs](http://oauth.net/core/1.0/#anchor9).

**Ressourcen-URL[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth/access_token`

**Ressourceninformationen[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Antwortformate | JSON |
| Authentifizierung erforderlich? | Ja |
| Ratenbegrenzung? | Ja |

**Parameter[](#parameters "Permalink to this headline")**

|     |     |     |     |     |
| :--- | :--- | :--- | :--- | :--- |
| Name | Erforderlich | Beschreibung | Standardwert | Beispiel |
| oauth\_token | erforderlich | Das hier verwendete oauth\_token muss mit dem im Schritt request\_token zurückgegebenen oauth\_token übereinstimmen. |     |     |
| oauth\_verifier | erforderlich | Wenn der OAuth-Webflow verwendet wird, setzen Sie diesen Parameter auf den Wert des in der Callback-URL zurückgegebenen _oauth\_verifier_. Wenn Sie Out-of-Band-OAuth verwenden, setzen Sie diesen Wert auf den PIN-Code. Für die Einhaltung von OAuth 1.0a ist dieser Parameter **erforderlich**. OAuth 1.0a wird strikt durchgesetzt, und Anwendungen, die den _oauth\_verifier_ nicht verwenden, können den OAuth-Flow nicht abschließen. |     |     |

**Beispielanforderung[](#example-request "Permalink to this headline")**

`POST https://api.x.com/oauth/access_token?oauth_token=qLBVyoAAAAAAx72QAAATZxQWU6P&oauth_verifier=ghLM8lYmAxDbaqL912RZSRjCCEXKDIzx`

Aus PIN-basiertem Flow: `POST https://api.x.com/oauth/access_token?oauth_token=9Npq8AAAAAAAx72QBRABZ4DAfY9&oauth_verifier=4868795`

**Beispielantwort[](#example-response "Permalink to this headline")**

`oauth_token=6253282-eWudHldSbIaelX7swmsiHImEL4KinwaGloHANdrY&oauth_token_secret=2EEfA6BG5ly3sR3XjE0IBSnlQu4ZrUzPiYTmrkVU&user_id=6253282&screen_name=xapi`

### POST oauth/invalidate\_token

Ermöglicht es einer registrierten Anwendung, ein ausgegebenes OAuth-Access-Token (`access_token`) anhand ihrer Client-Anmeldedaten zu widerrufen. Sobald ein `access_token` ungültig gemacht wurde, liefern neue Erstellungsversuche ein anderes Access-Token und die Verwendung des ungültig gemachten Tokens ist nicht mehr zulässig.

**Ressourcen-URL[](#resource-url "Permalink to this headline")**

`https://api.x.com/1.1/oauth/invalidate_token`

**Ressourceninformationen[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Antwortformate | JSON |
| Erfordert Authentifizierung? | Ja – Benutzerkontext mit den Access-Tokens, die Sie ungültig machen möchten |
| Ratenbegrenzung? | Ja |

**Beispielanfrage[](#example-request "Permalink to this headline")**

```bash
        curl --request POST
          --url 'https://api.x.com/1.1/oauth/invalidate_token.json'
          --header 'authorization: OAuth oauth_consumer_key="CLIENT_KEY",
         oauth_nonce="AUTO_GENERATED_NONCE", oauth_signature="AUTO_GENERATED_SIGNATURE",
         oauth_signature_method="HMAC-SHA1", oauth_timestamp="AUTO_GENERATED_TIMESTAMP",
         oauth_token="ACCESS_TOKEN", oauth_version="1.0"'
```

**Beispielantwort[](#example-response "Permalink to this headline")**

```bash
        HTTP/1.1 200 OK
        Content-Type: application/json; charset=utf-8
        Content-Length: 127
        ...

        {"access_token":"ACCESS_TOKEN"}
```

**Beispiel-Fehlerantwort nach Ungültigmachung des Tokens[](#example-error-response-after-token-has-been-invalidated "Permalink to this headline")**

```bash
        HTTP/1.1 401 Authorization Required
        ...

        {"errors": [{
          "code": 89,
          "message": "Invalid or expired token."}
        ]}
```

<div id="post-oauth2token">
  ### POST oauth2/token
</div>

Ermöglicht einer registrierten Anwendung, ein OAuth‑2-Bearer-Token zu erhalten, das verwendet werden kann, um API-Anfragen im Namen der Anwendung ohne Benutzerkontext zu stellen. Dies wird [Application-only authentication](/de/resources/fundamentals/authentication/oauth-2-0/application-only) genannt.

Ein Bearer-Token kann mit oauth2/invalidate\_token ungültig gemacht werden. Sobald ein Bearer-Token ungültig gemacht wurde, führen neue Erstellungsversuche zu einem anderen Bearer-Token, und die Verwendung des vorherigen Tokens ist nicht mehr zulässig.

Für eine Anwendung darf jeweils nur ein aktives Bearer-Token existieren. Wiederholte Anfragen an diese Methode liefern dasselbe bereits existierende Token, bis es ungültig gemacht wurde.

Erfolgreiche Antworten enthalten eine JSON-Struktur, die das ausgegebene Bearer-Token beschreibt.

Über diese Methode erhaltene Tokens sollten zwischengespeichert werden. Bei zu häufiger Anforderung werden Anfragen mit HTTP 403 und dem Code 99 abgelehnt.

**Resource URL[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth2/token`

**Resource Information[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Response formats | JSON |
| Requires authentication? | Ja – Basic Auth mit Ihrem API-Schlüssel als Benutzername und Ihrem API-Schlüsselgeheimnis als Passwort |
| Rate limited? | Ja |

**Parameters[](#parameters "Permalink to this headline")**

|     |     |     |     |     |
| :--- | :--- | :--- | :--- | :--- |
| Name | Required | Description | Default Value | Example |
| grant\_type | required | Gibt den Typ der von der Anwendung angeforderten Berechtigung (Grant) an. Derzeit ist nur _client\_credentials_ zulässig. Siehe [Application-Only Authentication](/de/resources/fundamentals/authentication/oauth-2-0/application-only) für weitere Informationen. |     | _client\_credentials_ |

**Example request[](#example-request "Permalink to this headline")**

```bash
    POST /oauth2/token HTTP/1.1
    Host: api.x.com
    User-Agent: My X App v1.0.23
    Authorization: Basic eHZ6MWV2R ... o4OERSZHlPZw==
    Content-Type: application/x-www-form-urlencoded;charset=UTF-8
    Content-Length: 29
    Accept-Encoding: gzip

    grant_type=client_credentials
```

**Example response:**

```bash
    HTTP/1.1 200 OK
    Status: 200 OK
    Content-Type: application/json; charset=utf-8
    ...
    Content-Encoding: gzip
    Content-Length: 140

    {"token_type":"bearer","access_token":"AAAA%2FAAA%3DAAAAAAAA"}
```

### POST oauth2/invalidate\_token

Ermöglicht einer registrierten Anwendung, ein ausgestelltes OAuth 2.0 Bearer-Token durch Vorlage ihrer Client-Anmeldedaten zu widerrufen. Sobald ein Bearer-Token widerrufen wurde, führt jeder neue Erstellungsvorgang zu einem anderen Bearer-Token, und die Verwendung des widerrufenen Tokens ist nicht mehr zulässig oder möglich.

Erfolgreiche Antworten enthalten eine JSON-Struktur, die das widerrufene Bearer-Token beschreibt.

**Resource URL[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth2/invalidate_token`

**Resource Information[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Response formats | JSON |
| Requires authentication? | Ja – [OAuth 1.0a](/de/resources/fundamentals/authentication/oauth-1-0a) mit den Consumer-API-Schlüsseln der Anwendung sowie dem Access Token und dem Access Token Secret des Eigentümers der Anwendung |
| Rate limited? | Ja |

**Parameters[](#parameters "Permalink to this headline")**

| Name | Required | Description |
| :--- | :--- | :--- |
| access\_token | required | Der Wert des Bearer-Tokens, das Sie widerrufen möchten |

**Example request[](#example-request "Permalink to this headline")**

```
        curl --request POST
          --url 'https://api.x.com/oauth2/invalidate_token?access_token=AAAA%2FAAA%3DAAAAAAAA'
          --header 'authorization: OAuth oauth_consumer_key="CLIENT_KEY",
         oauth_nonce="AUTO_GENERATED_NONCE", oauth_signature="AUTO_GENERATED_SIGNATURE",
         oauth_signature_method="HMAC-SHA1", oauth_timestamp="AUTO_GENERATED_TIMESTAMP",
         oauth_token="ACCESS_TOKEN", oauth_version="1.0"'
```

**Example response[](#example-response "Permalink to this headline")**

```
         Status: 200 OK
         Content-Type: application/json; charset=utf-8
         Content-Length: 135
         ...
       {
        "access_token": "AAAA%2FAAA%3DAAAAAAAA"
        }
```
