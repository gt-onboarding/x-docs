<div id="using-and-generating-an-app-only-bearer-token">
  ### Verwenden und Generieren eines App-only-Bearer-Tokens
</div>

Ein Bearer-Token bietet Entwicklerinnen und Entwicklern einen sichereren Einstiegspunkt für die Nutzung der X-APIs und ist eine der Kernfunktionen von OAuth 2.0.

Die Authentifizierung mit Bearer-Token ist auch als „application-only authentication“ bekannt. Ein Bearer-Token ist ein Byte-Array unbekannten Formats, das Sie mithilfe eines Skripts, z. B. eines curl-Befehls, generieren. Sie können ein Bearer-Token auch im Developer-Portal im Bereich „Keys and tokens“ in den Einstellungen Ihrer App abrufen. Weitere Informationen finden Sie in der [offiziellen OAuth-Dokumentation](https://oauth.net/2/bearer-tokens/).

<div id="when-are-they-used">
  #### Wann werden sie verwendet?
</div>

Die folgenden Produkte erfordern die Verwendung eines Bearer-Tokens:

- [Engagement API](/de/x-api/enterprise-gnip-2.0/fundamentals/engagement-api)
- [Account Activity API](/de/x-api/enterprise-gnip-2.0/fundamentals/account-activity)
- Andere APIs, die die OAuth-2.0-Bearer-Token-Authentifizierung nutzen, etwa v2- und Labs-Endpunkte.

<div id="prerequisites">
  #### Voraussetzungen
</div>

Sie benötigen ein [Entwicklerkonto](https://developer.x.com/en/portal/petition/essential/basic-info) und müssen eine [X App](/de/resources/fundamentals/developer-apps) erstellt haben. Sobald beides vorhanden ist, müssen Sie außerdem die API-Schlüssel im [Developer-Portal](/de/resources/fundamentals/developer-portal) abrufen. Gehen Sie wie folgt vor:

1. Melden Sie sich mit Ihrem X-Konto auf developer.x.com an.
2. Navigieren Sie zum [X App-Dashboard](https://developer.x.com/content/developer-twitter/en/apps) und öffnen Sie die X App, für die Sie Zugriffstoken generieren möchten.
3. Navigieren Sie zur Seite „Keys and Tokens“.
4. Auf dieser Seite finden Sie die API-Schlüssel, Benutzer-Zugriffstoken und das Bearer-Token.

<div id="how-to-generate-a-bearer-token">
  ### So erzeugen Sie ein Bearer-Token
</div>

Sie finden das Bearer-Token für Ihre App zusammen mit Ihren „Keys and Tokens“.

Kopieren Sie die folgende cURL-Anfrage in Ihre Kommandozeile, nachdem Sie die folgenden Consumer-API-Schlüssel, die Sie zuvor aus Ihrer [X-App](/de/resources/fundamentals/developer-apps) erhalten haben, angepasst haben. Beachten Sie, dass die auf dieser Seite verwendeten Consumer-API-Schlüssel außer Betrieb genommen wurden und für echte Anfragen nicht funktionieren.

- **API-Schlüssel** `<API key>` z. B. `xvz1evFS4wEEPTGEFPHBog`
- **API-Secret-Schlüssel** `<API secret key>` z. B. `L8qq9PZyRg6ieKGEKhZolGC0vJWLw8iEJ88DRdyOg`

  ```bash
  curl -u "$API_KEY:$API_SECRET_KEY" \
  --data 'grant_type=client_credentials' \
  'https://api.x.com/oauth2/token'
  ```

Hier ist ein Beispiel, wie die cURL-Anfrage mit Ihren API-Schlüsseln aussehen sollte:

```bash
   curl -u 'xvz1evFS4wEEPTGEFPHBog:L8qq9PZyRg6ieKGEKhZolGC0vJWLw8iEJ88DRdyOg' \
  --data 'grant_type=client_credentials' \
  'https://api.x.com/oauth2/token'
```

So würde die Antwort aussehen. Beachten Sie, dass dies ein außer Betrieb genommenes Bearer-Token ist:

`{"token_type":"bearer","access_token":"AAAAAAAAAAAAAAAAAAAAAMLheAAAAAAA0%2BuSeid%2BULvsea4JtiGRiSDSJSI%3DEUifiRBkKG5E2XzMDjRfl76ZC9Ub0wnz4XsNiRVBChTYbJcE3F"}`

Unser Bearer-Token, das zur Authentifizierung für Ressourcen mit OAuth 2.0 verwendet wird, wäre:

`AAAAAAAAAAAAAAAAAAAAAMLheAAAAAAA0%2BuSeid%2BULvsea4JtiGRiSDSJSI%3DEUifiRBkKG5E2XzMDjRfl76ZC9Ub0wnz4XsNiRVBChTYbJcE3F`
