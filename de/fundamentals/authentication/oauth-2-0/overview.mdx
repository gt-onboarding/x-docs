<div id="bearer-token-also-known-as-app-only">
  ### Bearer-Token (auch bekannt als App-only)
</div>

Das OAuth 2.0 Bearer-Token authentifiziert Anfragen im Namen Ihrer [Developer-App](/de/resources/fundamentals/developer-apps). Da diese Methode spezifisch für die App ist, sind keine Nutzer beteiligt. Sie eignet sich typischerweise für Entwickler, die schreibgeschützten Zugriff auf öffentliche Informationen benötigen.

Diese Authentifizierungsmethode erfordert, dass Sie Ihrer Anfrage ein Bearer-Token beifügen, das Sie im Bereich „Keys and tokens“ Ihrer Developer-Apps generieren können. Hier ist ein Beispiel, wie eine Anfrage mit einem fiktiven Bearer-Token aussieht:

```json
  curl "https://api.x.com/2/tweets?ids=1261326399320715264,1278347468690915330" \
  -H "Authorization: Bearer AAAAAAAAAAAAAAAAAAAAAFnz2wAAAAAAxTmQbp%2BIHDtAhTBbyNJon%2BA72K4%3DeIaigY0QBrv6Rp8KZQQLOTpo9ubw5Jt?WRE8avbi"
```

API-Aufrufe mit App-only-Authentifizierung sind pro Endpoint auf App-Ebene [rate-limitiert](/de/resources/fundamentals/rate-limits).

Um diese Methode zu verwenden, benötigen Sie ein Bearer-Token, das Sie generieren können, indem Sie Ihren API Key und Ihr Secret über den Endpoint [POST oauth2/token](/de/resources/fundamentals/authentication/api-reference#post-oauth2-token) übermitteln, oder indem Sie es im Bereich „keys and tokens“ Ihrer App-Einstellungen im [Developer-Portal](/de/resources/fundamentals/developer-portal) erzeugen.

Wenn Sie ein Bearer-Token widerrufen möchten, können Sie den Endpoint [POST oauth2/invalidate\_token](/de/resources/fundamentals/authentication/api-reference#post-oauth2-invalidate-token) verwenden oder neben dem Bearer-Token im Bereich „keys and tokens“ Ihrer App-Einstellungen auf „revoke“ klicken.

<div id="oauth-20-authorization-code-flow-with-pkce">
  ### OAuth 2.0 Authorization Code Flow mit PKCE
</div>

Der OAuth 2.0 Authorization Code Flow mit PKCE ermöglicht es Ihnen, sich im Namen eines anderen Nutzers zu authentifizieren, gibt Ihnen mehr Kontrolle über die Scopes einer Anwendung und verbessert Autorisierungsabläufe über mehrere Geräte hinweg. Anders ausgedrückt: Entwickler, die Anwendungen für Menschen auf X entwickeln, haben mehr Kontrolle darüber, welche Informationen ihre App von den Nutzern anfordert, sodass Sie Ihre Endnutzer nur um die Daten und Informationen bitten müssen, die Sie tatsächlich benötigen.

Dieses moderne Autorisierungsprotokoll ermöglicht es Ihnen, Ihren Endnutzern einen schlankeren Zustimmungsablauf zur Autorisierung Ihrer App zu präsentieren, der nur die spezifischen Scopes anzeigt, die Sie angefordert haben. Das reduziert nicht nur Ihre Datenlast, sondern kann auch zu größerem Vertrauen seitens der Endnutzer führen.
