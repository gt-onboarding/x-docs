Sie können die oEmbed-API verwenden, um programmgesteuert eingebettete Inhalte wie [Tweets](https://developer.x.com/en/docs/twitter-for-websites/embedded-tweets/overview) und [Timelines](https://developer.x.com/en/docs/twitter-for-websites/timelines/overview) abzurufen. 

Die Antwort der oEmbed-API enthält ein HTML-Snippet, das automatisch erkannt wird, wenn [das Widget-JavaScript von X auf der Seite eingebunden ist](https://developer.x.com/web/javascript/loading).

Bitte beachten Sie, dass die API für Aufgaben in großem Umfang empfohlen wird. Zum Einbetten von Inhalten raten wir zur Nutzung unseres leistungsfähigen Tools [publish.x.com](https://publish.x.com/#).

<Tabs>
  <Tab title="Embedded timelines">
    Das zurückgegebene HTML-Snippet wird automatisch als [eingebettete Timeline](https://developer.x.com/en/docs/twitter-for-websites/timelines/overview) erkannt, wenn [das Widget-JavaScript von X auf der Seite eingebunden ist](https://developer.x.com/web/javascript/loading).

    Der oEmbed-Endpunkt ermöglicht die Anpassung des endgültigen Erscheinungsbilds einer eingebetteten Timeline, indem die entsprechenden Eigenschaften im HTML-Markup gesetzt werden, die standardmäßig von dem in der HTML-Antwort enthaltenen JavaScript von X interpretiert werden. Das Format des zurückgegebenen Markups kann sich im Laufe der Zeit ändern, wenn X neue Funktionen hinzufügt oder die Darstellung der Timeline anpasst.

    Für eine X-Timeline, die über die Timeline-URL angegeben ist, in einem mit [oEmbed](https://oembed.com/) kompatiblen JSON-Format. Benutzer- und Listen-Timelines werden unterstützt. Das Timeline-Markup sollte auf Ihren Servern bis zur vorgeschlagenen Cache-Lebensdauer, die durch die Eigenschaft cache\_age angegeben ist, zwischengespeichert werden.

    ## Resource URL

    _https://publish.x.com/oembed_

    ## Resource Information

    |     |     |
    | :--- | :--- |
    | Response formats | JSON |
    | Requires authentication? | No  |
    | Rate limited | No  |

    ## Parameters

    | Name | Description | Example |
    | :--- | :--- | :--- |
    | **url** | Die URL der X-Timeline, die eingebettet werden soll | \*   https://x.com/TwitterDev<br /> _https://x.com/TwitterDev/lists/national-parks
    | limit | Bis zu N Elemente anzeigen, wobei N ein Wert zwischen 1 und 20 (einschließlich) ist | 6   |
    | maxwidth | Die maximale Breite des Widgets festlegen. Muss zwischen 180 und 1200 (einschließlich) liegen | 300 |
    | maxheight | Die maximale Höhe des Widgets festlegen. Muss größer als 200 sein | 400 |
    | omit\_script | Kein script-Element in die Antwort aufnehmen | 1   |
    | lang | Ein unterstützter X-[Sprachcode](/de/en/docs/twitter-for-websites/twitter-for-websites-supported-languages/overview "Twitter language code") | es  |
    | theme | Wenn auf dark gesetzt, wird die Timeline mit hellem Text auf dunklem Hintergrund angezeigt | dark |
    | chrome | Eine Anzeige-Komponente der Timeline mit durch Leerzeichen getrennten Tokens entfernen<br /><br />_   noheader – blendet die Kopfzeile aus<br />\*   nofooter – blendet die Fußzeile aus, falls sichtbar<br />\*   noborders – entfernt alle Rahmen: um das Widget, zwischen Tweets und innerhalb eines Tweets<br />\*   noscrollbar – die Timeline-Scrollbar zuschneiden und ausblenden, falls sichtbar<br />\*   transparent – Hintergrundfarbe entfernen | noheader%20nofooter |
    | aria\_polite | Einen assertiven [ARIA-Live-Region-Politeness](https://www.w3.org/TR/wai-aria/states_and_properties#aria-live)-Wert für zu einer Timeline hinzugefügte Tweets festlegen | assertive |
    | dnt | Wenn auf true gesetzt, werden die Timeline und ihre eingebettete Seite auf Ihrer Website nicht für Zwecke verwendet, die [personalisierte Vorschläge](https://support.x.com/articles/20169421) und [personalisierte Werbung](https://support.x.com/articles/20170405) einschließen | true |

    ## Example Requests

    ```bash
    curl --request GET --url 'https://publish.x.com/oembed?url=https%3A%2F%2Ftwitter.com%2FInterior%2Fstatus%2F507185938620219395'
    twurl -H publish.x.com "/oembed?url=https://x.com/Interior/status/463440424141459456"
    ```

    ## Example Response

    ```json

    {
      "url": "https://x.com/TwitterDev",
      "title": "",
      "html": "<a class=\"twitter-timeline\" href=\"https://x.com/TwitterDev\">Tweets by TwitterDev</a>\n<script async src=\"//platform.x.com/widgets.js\" charset=\"utf-8\"></script>",
      "width": null,
      "height": null,
      "type": "rich",
      "cache_age": "3153600000",
      "provider_name": "Twitter",
      "provider_url": "https://x.com",
      "version": "1.0"
    }
    ```

  <Tab title="Embedded Tweets">
    Das zurückgegebene HTML-Snippet wird automatisch als [eingebetteter Tweet](https://developer.x.com/web/embedded-tweets) erkannt, wenn [das Widget-JavaScript von X auf der Seite eingebunden ist](https://developer.x.com/web/javascript/loading).

    Der oEmbed-Endpunkt ermöglicht es, das endgültige Erscheinungsbild eines eingebetteten Tweets anzupassen, indem die entsprechenden Eigenschaften im HTML-Markup gesetzt werden, das standardmäßig von dem mit der HTML-Antwort ausgelieferten JavaScript von X interpretiert wird. Das Format des zurückgegebenen Markups kann sich im Laufe der Zeit ändern, wenn X neue Funktionen hinzufügt oder die Tweet-Darstellung anpasst.

    Das Tweet-Fallback-Markup sollte auf Ihren Servern bis zur in der Eigenschaft `cache_age` angegebenen empfohlenen Cache-Dauer zwischengespeichert werden.

    _https://publish.x.com/oembed_

    |     |     |
    | :--- | :--- |
    | Antwortformate | JSON |
    | Authentifizierung erforderlich? | Nein  |
    | Ratebegrenzung | Nein  |

    ## Parameter

    | Name | Standardwert | Beschreibung |
    | :--- | :--- | :--- |
    | `url` erforderlich  <br />String |     | Die URL des einzubettenden Tweets |
    | `maxwidth`  <br />Int `[220..550]` | `325` | Die maximale Breite eines gerenderten Tweets in ganzen Pixeln. Ein angegebener Wert außerhalb des zulässigen Bereichs wird entsprechend als minimale bzw. maximale unterstützte Breite zurückgegeben; der angepasste Breitenwert wird in der zurückgegebenen Eigenschaft `width` ausgewiesen. Beachten Sie, dass X den oEmbed-Parameter `maxheight` nicht unterstützt. Tweets bestehen grundsätzlich aus Text und haben daher eine unvorhersehbare Höhe, die nicht wie ein Bild oder Video skaliert werden kann. Entsprechend liefert die oEmbed-Antwort keinen Wert für `height`. Implementierungen, die konsistente Höhen für Tweets benötigen, sollten die untenstehenden Parameter `hide_thread` und `hide_media` verwenden. |
    | `hide_media`  <br />Boolean, String oder Int | `false` | Wenn auf `true`, `"t"` oder `1` gesetzt, werden Links in einem Tweet nicht zu Foto-, Video- oder Link-Vorschauen erweitert. |
    | `hide_thread`  <br />Boolean, String oder Int | `false` | Wenn auf `true`, `"t"` oder `1` gesetzt, wird eine eingeklappte Version des vorherigen Tweets im Konversations-Thread nicht angezeigt, wenn der angeforderte Tweet eine Antwort auf einen anderen Tweet ist. |
    | `omit_script`  <br />Boolean, String oder Int | `false` | Wenn auf `true`, `"t"` oder `1` gesetzt, wird das für das Laden von `widgets.js` verantwortliche `<script>` nicht zurückgegeben. Ihre Webseiten sollten eine eigene Referenz auf `widgets.js` enthalten, die für alle X-Widgets einschließlich [Embedded Tweets](https://developer.x.com/web/embedded-tweets) verwendet wird. |
    | `align`  <br />Enum `{left,right,center,none}` | `none` | Legt fest, ob der eingebettete Tweet relativ zum Elternelement innerhalb der Seite links, rechts oder zentriert ausgerichtet bzw. gefloatet werden soll. |
    | `lang`  <br />Enum([Sprache](https://developer.x.com/en/docs/twitter-for-websites/twitter-for-websites-supported-languages/overview)) | `en` | Fordert HTML und einen gerenderten Tweet in der angegebenen [von Embedded Tweets unterstützten X-Sprache](https://developer.x.com/web/overview/languages) an. |
    | `theme`  <br />Enum `{light, dark}` | `light` | Wenn auf `dark` gesetzt, wird der Tweet mit hellem Text auf dunklem Hintergrund angezeigt. |
    | `dnt`  <br />Boolean | `false` | Wenn auf `true` gesetzt, werden der Tweet und die darauf eingebettete Seite auf Ihrer Website nicht für Zwecke verwendet, die [personalisierte Vorschläge](https://support.x.com/articles/20169421) und [personalisierte Anzeigen](https://support.x.com/articles/20170405) einschließen. |

    ```bash
    twurl -H publish.x.com "/oembed?url=https://x.com/TwitterDev"
    ```
