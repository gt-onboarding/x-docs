<div id="mact-overview">
  ## Présentation de MACT
</div>

La mesure de la [promotion d’applications mobiles](https://biz.x.com/mobile-app-promotion) sur X permet aux annonceurs d’évaluer la performance des campagnes publicitaires sur X conçues pour générer des installations ou d’autres conversions in‑app. Un partenaire de mesure mobile X permet à un annonceur de définir les conversions qu’il souhaite suivre à partir des applications qu’il promeut sur X. Une fois que l’annonceur a configuré les types d’événements et les fenêtres d’attribution à suivre, le partenaire de mesure mobile enverra tous ces événements à X et recevra immédiatement des données d’attribution indiquant la campagne et le créatif avec lesquels l’utilisateur a interagi, à l’origine de la conversion.

<div id="conversion-events">
  ### Événements de conversion
</div>

Pour signaler un [événement de conversion](https://developer.x.com/en/docs/x-ads-api/measurement/api-reference/conversion-event) à X et obtenir des données d’attribution, le partenaire doit envoyer l’ID de l’application mobile, le type et l’horodatage de la conversion, ainsi qu’une valeur hachée de l’identifiant publicitaire/de l’appareil. La transmission de ces données enregistrera l’événement de conversion sur la plateforme X et renverra les informations d’attribution, le cas échéant. En outre, l’annonceur peut choisir de fournir des métadonnées supplémentaires concernant l’événement de conversion, comme la valeur d’un achat, l’ID d’un produit ou la méthode d’inscription.

#### POST conversion\_event

Enregistrer un événement de conversion de mesure mobile. La réponse indiquera l’attribution X ou X Audience Platform (TAP).

Ceci se rapporte à l’endpoint [GET conversion\_attribution](https://developer.x.com/en/docs/x-ads-api/measurement/api-reference/conversion-attribution#get-conversion-attribution).

Dans la réponse, l’attribution sera revendiquée par X, par TAP, ou aucune attribution ne sera revendiquée. Le nœud `twitter_attribution` sera toujours présent et aura la valeur `null` lorsqu’il n’y a pas d’attribution X (voir l’exemple de réponse ci-dessous). Si une attribution TAP est revendiquée, un nœud `tpn_attribution` sera présent et renseigné en conséquence. Veuillez consulter la [présentation de TAP](https://developer.x.com/en/docs/x-ads-api/measurement/overview/twitter-audience-platform) pour plus d’informations.

Plusieurs paramètres optionnels sont disponibles pour définir les métadonnées associées à chaque événement de conversion. Ces métadonnées n’ont aucun impact sur les calculs d’attribution.

<div id="resource-url">
  ##### URL de ressource
</div>

`https://ads-api.x.com/12/conversion_event`

<div id="parameters">
  ##### Paramètres
</div>

| Nom | Description |
| :--- | :--- |
| app\_id  <br />_requis_ | L’identifiant unique dans l’app store correspondant.<br /><br />Type : int, string<br /><br />Exemple : `333903271`, `com.vine.android` |
| conversion\_time  <br />_requis_ | L’heure de l’événement de conversion au format d’horodatage ISO-8601, avec les millisecondes.<br /><br />Type : string<br /><br />Exemple : `2014-05-22T02:38:28.103Z` |
| conversion\_type  <br />_requis_ | Le type d’événement de conversion.<br /><br />Type : enum<br /><br />Valeurs possibles : `PURCHASE`, `SIGN_UP`, `INSTALL`, `RE_ENGAGE`, `UPDATE`, `TUTORIAL_COMPLETE`, `RESERVATION`, `ADD_TO_CART`, `ADD_TO_WISHLIST`, `LOGIN`, `CHECKOUT_INITIATED`, `SEARCH`, `LEVEL_ACHIEVED`, `ACHIEVEMENT_UNLOCKED`, `CONTENT_VIEW`, `SHARE`, `INVITE`, `ADDED_PAYMENT_INFO`, `SPENT_CREDITS`, `RATED` |
| hashed\_device\_id  <br />_requis_ | L’IDFA ou l’AdID haché avec HMAC\_SHA-256.<br /><br />Type : string<br /><br />Exemple : `ABCD1234XYZ` |
| os\_type  <br />_requis_ | Le type d’OS de l’app.<br /><br />Type : enum<br /><br />Valeurs possibles : `IOS`, `ANDROID` |
| click\_window  <br />_optionnel_ | La fenêtre de clic pour cet événement, en jours.<br /><br />Type : int<br /><br />**Remarque** : `click_window` doit être supérieur ou égal à view\_through\_window<br /><br />Valeur par défaut : `14`  <br />Valeurs possibles : `1`, `7`, `14`, `30` |
| device\_ip\_address  <br />_optionnel_ | L’adresse IPv4 ou IPv6 de l’appareil au moment de l’événement de conversion.<br /><br />Type : string<br /><br />Exemple : `192.133.78.1` |
| level  <br />_optionnel_ | Un niveau associé à cet événement.<br /><br />Type : int<br /><br />Exemple : `2` |
| non\_twitter\_engagement\_time  <br />_optionnel_ | L’heure de la dernière interaction hors Twitter avant la conversion.<br /><br />Type : string<br /><br />Exemple : `2014-05-22T02:38:28.103Z` |
| non\_twitter\_engagement\_type  <br />_optionnel_ | Le type d’interaction hors Twitter avant l’événement de conversion.<br /><br />Type : enum<br /><br />Valeurs possibles : `CLICK`, `VIEW` |
| number\_items  <br />_optionnel_ | Le nombre d’articles associés à cet événement.<br /><br />Type : int<br /><br />Exemple : `2` |
| price\_currency  <br />_optionnel_ | Doit être un code ISO 4217 indiquant la devise associée à cet événement.<br /><br />Type : string<br /><br />Exemples : `EUR`, `USD`, `JPY` |
| price\_micro  <br />_optionnel_ | Un montant associé à cet événement, en micros.<br /><br />Type : int<br /><br />Exemple : `123450000` |
| user\_payment\_info  <br />_optionnel_ | Une valeur booléenne indiquant si les informations de paiement de l’utilisateur sont stockées dans l’app associée à cet événement.<br /><br />Type : bool<br /><br />Valeurs possibles : `true` ou `false` |
| view\_through\_window  <br />_optionnel_ | La fenêtre d’attribution par vue pour cet événement, en jours.<br /><br />**Remarque** :<br /><br />`click_window` doit être supérieur ou égal à view\_through\_window.<br /><br />Type : int<br /><br />Valeur par défaut : `1`  <br />Valeurs possibles : `0`, `1`, `7`, `14`, `30` |

#### GET accounts/:account\_id/app\_event\_tags

Récupère les détails de certaines ou de toutes les balises d’événements d’application associées au compte actuel.

Ce sont elles qui définissent les fenêtres de conversion configurées pour chaque type de conversion dans le suivi des conversions d’applications mobiles.

<div id="resource-url">

</div>

`https://ads-api.x.com/12/accounts/:account_id/app_event_tags`

<div id="parameters">
  ##### Paramètres
</div>

| Nom | Description |
| :--- | :--- |
| account\_id  <br />_requis_ | Identifiant du compte exploité. Apparaît dans le chemin de la ressource et constitue généralement un paramètre requis pour toutes les requêtes de l’API publicitaire, à l’exception de [GET accounts](/fr/x-ads-api/campaign-management#accounts). #### GET conversion\_attribution

Interrogez X pour vérifier l’attribution de conversion sans consigner d’événement de conversion. La réponse indiquera l’attribution par X.

Cela se rapporte au point de terminaison [POST conversion\_event](https://developer.x.com/en/docs/x-ads-api/measurement/api-reference/conversion-event#post-conversion-event).

L’attribution revendiquée dans la réponse sera soit X, soit TAP, soit aucune. Le nœud `twitter_attribution` sera toujours présent et aura la valeur `null` lorsqu’il n’y a pas d’attribution X, et sera renseigné comme dans l’exemple de réponse ci-dessous. Si une attribution TAP est revendiquée, un nœud `tpn_attribution` sera présent et renseigné en conséquence. Veuillez consulter la [présentation de TAP](https://developer.x.com/en/docs/twitter-ads-api/measurement/twitter-audience-platform/overview) pour plus d’informations.
