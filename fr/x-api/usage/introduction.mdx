L’API Usage dans l’API X v2 permet aux développeurs de récupérer par programmation l’utilisation de leur projet. À l’aide de cet endpoint, les développeurs peuvent suivre et contrôler le nombre de Posts qu’ils ont récupérés pour un cycle de facturation donné.

Les développeurs peuvent utiliser l’endpoint GET pour obtenir l’utilisation quotidienne du projet pour les 90 derniers jours au maximum. L’utilisation peut également être agrégée par application cliente connectée à votre projet.

Une limite de débit par application de 50 requêtes par tranche de 15 minutes s’applique à cet endpoint GET.

<div id="getting-started">
  ### Mise en route
</div>

Pour utiliser cet endpoint, vous avez besoin d’un \[jeton Bearer]\(https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) from the [developer portal](https://developer.x.com/en/portal/dashboard). Une fois que vous disposez du jeton Bearer, vous pouvez appeler l’API d’usage comme indiqué ci-dessous :

```
      curl 'https://api.x.com/2/usage/tweets' --header 'Authorization: Bearer XXXXX'
```

Si la requête aboutit, vous devriez voir la réponse JSON suivante :

```
{
    "data": {
        "cap_reset_day": 10,
        "project_cap": "5000000000",
        "project_id": "1369785403853424",
        "project_usage": "43435"
    }
}
```

<Note>
  **Configuration du compte**

  Pour accéder à ces endpoints, vous aurez besoin de :

  - D’un [compte développeur](https://developer.x.com/en/portal/petition/essential/basic-info) approuvé.
  - De vous authentifier à l’aide des clés et jetons d’une [application développeur](/fr/resources/fundamentals/developer-apps) située au sein d’un [projet](/fr/resources/fundamentals/projects). 

  Pour en savoir plus sur l’accès aux endpoints de l’API X v2, consultez notre [guide de démarrage](/fr/x-api/getting-started/getting-access).
</Note>
