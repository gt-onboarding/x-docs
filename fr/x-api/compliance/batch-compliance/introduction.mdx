X s’engage envers notre communauté de développeurs qui créent avec l’API X. Dans le cadre de cet engagement, nous visons à rendre notre API ouverte et équitable pour les développeurs, sûre pour les personnes sur X et bénéfique pour la plateforme X dans son ensemble. Il est crucial que tout développeur qui stocke du contenu X hors ligne s’assure que les données reflètent l’intention de l’utilisateur et l’état actuel du contenu sur X. Par exemple, lorsqu’une personne sur X supprime un Post ou son compte, protège ses Posts, ou supprime les informations de géolocalisation de ses Posts, il est essentiel, pour X comme pour nos développeurs, de respecter les attentes et l’intention de cette personne. Les endpoints de conformité par lot fournissent aux développeurs un outil simple pour aider à maintenir les données X en conformité avec le [Contrat et la politique du développeur X](https://developer.x.com/en/developer-terms/policy). 

Ces endpoints de conformité par lot vous permettent de téléverser de grands jeux de données d’ID de Post ou d’utilisateur afin de récupérer leur statut de conformité et de déterminer quelles données nécessitent une action pour mettre vos jeux de données en conformité. Veuillez noter que l’utilisation des endpoints de conformité par lot est restreinte aux cas d’usage susmentionnés, et toute autre finalité est interdite et peut entraîner une mesure d’application.

En général, 4 étapes sont impliquées lors de l’utilisation de cet endpoint :

1. **Créer une tâche de conformité**
   Vous pouvez spécifier le type de tâche (avec la valeur tweets ou users) pour indiquer si le jeu de données que vous souhaitez téléverser contient des ID de Posts ou des ID d’utilisateurs. Vous pouvez avoir une tâche concurrente par type de tâche à tout moment.
2. **Téléverser votre jeu de données vers l’upload\_url**
   Ensuite, vous téléversez votre jeu de données en tant que fichier texte brut vers l’upload\_url fourni, chaque ligne du fichier contenant un seul ID de Post ou un ID d’utilisateur. L’upload\_url expire après 15 minutes.
3. **(Optionnel) Vérifier l’état de la tâche**
   Vous pouvez vérifier l’état de votre tâche de conformité pour voir si elle est created, in\_progress, failed ou complete.
4. **Télécharger les résultats**
   Lorsque votre tâche est terminée, vous pouvez télécharger les résultats à l’aide du download\_url. Le download\_url expire au bout d’une semaine (à partir de la création de la tâche).

   Ce résultat contiendra un ensemble d’objets JSON (un objet par ligne). Chaque objet contiendra un ID de Post, la date de création du Post (utile pour localiser les Posts organisés par date), l’action requise, la raison de l’action de conformité et la date à laquelle l’utilisateur a été suspendu.

Vous recevrez les types d’événements de conformité suivants dans vos résultats :

- deleted - indique que le Post ou le compte utilisateur a été supprimé
- deactivated - indique que le Post ou le compte utilisateur a été désactivé
- scrub\_geo - indique que les informations de géolocalisation associées au Post ou à l’utilisateur ont été retirées
- protected - indique que le compte qui a publié le Post est devenu privé
- suspended - indique que le compte qui a publié le Post a été suspendu

**Configuration du compte**

Pour accéder à ces endpoints, vous aurez besoin de :

- D’un [compte développeur](https://developer.x.com/en/portal/petition/essential/basic-info) approuvé.
- De vous authentifier à l’aide des clés et des jetons d’une [application développeur](/fr/resources/fundamentals/developer-apps) qui se trouve dans un [Projet](/fr/resources/fundamentals/projects). 

En savoir plus sur l’accès aux endpoints de l’API X v2 dans notre [guide de démarrage](/fr/x-api/getting-started/getting-access).
