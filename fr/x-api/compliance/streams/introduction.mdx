X s’engage auprès de notre communauté de développeurs qui utilisent l’API X. Dans le cadre de cet engagement, nous visons à rendre notre API ouverte et équitable pour les développeurs, sûre pour les personnes sur X et bénéfique pour l’ensemble de la plateforme X. Il est crucial que tout développeur qui stocke du contenu X hors ligne s’assure que les données reflètent l’intention des utilisateurs et l’état actuel du contenu sur X. Par exemple, lorsque quelqu’un sur X supprime un Post ou son compte, protège ses Posts ou édite un Post, il est essentiel, tant pour X que pour nos développeurs, de respecter les attentes et l’intention de cette personne.

Des flux en temps réel d’événements de conformité offrent aux développeurs les outils nécessaires pour maintenir les données X en conformité avec le [X Developer Agreement and Policy](https://developer.x.com/en/developer-terms/policy).

Il existe deux flux d’événements de conformité, l’un pour les événements de conformité des Posts et l’autre pour les événements de conformité des Utilisateurs. Ces flux sont disponibles avec l’accès Enterprise et sont conçus pour aider les partenaires qui ingèrent de grands volumes de données à « écouter » les événements de conformité, tels que les événements d’édition de Post.

Ces flux fournissent les événements suivants :

**Flux de conformité des Posts :**

- delete - indique que le Post a été supprimé.

- tweet\_edit - indique qu’un Post a été modifié et fournit l’ID du Post mis à jour.

- withheld - indique que le Post a été restreint dans un ou plusieurs pays.

- drop - indique que le Post doit être retiré de l’affichage public.

- undrop - indique que le Post peut être à nouveau affiché et traité comme public.

**Flux de conformité des Utilisateurs :**

- **user\_delete** - indique que le compte Utilisateur a été supprimé

- **user\_undelete** - indique que le compte Utilisateur a été restauré

- **user\_protect** - indique que le compte Utilisateur est devenu privé

- **user\_unprotect** - indique que le compte Utilisateur est redevenu public

- **user\_withheld** - indique que le compte Utilisateur a été restreint dans un ou plusieurs pays.

- **user\_suspend** - indique que le compte Utilisateur a été suspendu

- **user\_unsuspend** - indique que le compte Utilisateur a été rétabli

- **user\_profile\_modification** - indique que le profil Utilisateur a été mis à jour. Cela inclut une description, un nom, une localisation et une URL mis à jour.

**Configuration du compte**

Pour accéder à ces endpoints, vous aurez besoin de :

- D’un [compte développeur](https://developer.x.com/en/portal/petition/essential/basic-info) approuvé.
- De vous authentifier à l’aide des clés et jetons d’une [developer App](/fr/resources/fundamentals/developer-apps) qui se trouve dans un [Project](/fr/resources/fundamentals/projects).

En savoir plus sur l’accès aux endpoints de l’API X v2 dans notre [guide de démarrage](/fr/x-api/getting-started/getting-access).
