<div id="standard-v11-compared-to-x-api-v2">
  ## Standard v1.1 comparé à l’API X v2
</div>

Si vous avez travaillé avec les endpoints standard v1.1 [POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) et [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id), l’objectif de ce guide est de vous aider à comprendre les similitudes et les différences entre les endpoints de gestion des posts du standard et de l’API X v2.

- **Similarités**
  - Authentification
- **Différences**
  - URL des endpoints

  - Exigences pour l’application et le projet

  - Paramètres de requête

<div id="similarities">
  ### Similarités
</div>

**Authentification**

Les deux versions, la standard v1.1 et l’API X v2, gèrent les endpoints de Posts ([POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) et [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id)) avec [OAuth 1.0a en contexte utilisateur](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Ainsi, si vous utilisiez auparavant l’un des endpoints standard v1.1, vous pouvez continuer à employer la même méthode d’authentification lors de la migration vers l’API X v2. 

<div id="differences">
  ### Différences
</div>

**URL des endpoints**

- Endpoints standard v1.1 :
  - https://api.x.com/1.1/statuses/update.json
    (Crée un post)
  - `https://api.x.com/1.1/statuses/destroy/:id.json`
    (Supprime un post)
- Endpoint de l’API X v2 :
  - https://api.x.com/2/tweets
    (Crée un post)
  - https://api.x.com/2/tweets/:id
    (Supprime un post spécifié)

<div id="app-and-project-requirements">
  ### Exigences relatives aux applications et aux projets
</div>

Les endpoints de l’API X v2 exigent que vous utilisiez des identifiants d’une [application développeur](/fr/resources/fundamentals/developer-apps) associée à un [projet](/fr/resources/fundamentals/developer-apps) lors de l’authentification de vos requêtes. Tous les endpoints de l’API X v1.1 peuvent utiliser des identifiants provenant d’applications autonomes ou d’applications associées à un projet.

<div id="request-parameters">
  ### Paramètres de requête
</div>

Dans la version 1.1, les paramètres de requête standard prenaient deux paramètres de requête (user\_id ou screen\_name). L’API X v2 n’accepte que l’ID numérique du Post pour l’endpoint DELETE, qui doit être inclus dans le chemin de l’endpoint.

Pour l’endpoint POST, des paramètres supplémentaires doivent être fournis dans le corps JSON de la requête. Vous pouvez en savoir plus sur les paramètres disponibles dans le [guide de référence de l’API](/fr/x-api/posts/manage-tweets/introduction).
