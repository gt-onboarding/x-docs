<div id="comparing-x-apis-post-counts-endpoints">
  ## Comparaison des endpoints de comptage de posts de l’API X
</div>

L’endpoint de comptage de posts de l’API X v2 finira par remplacer l’[endpoint de comptage de l’API Search Enterprise](/fr/x-api/enterprise-gnip-2.0/fundamentals/search-api#counts-requests-post-count). Si vous avez du code, des applications ou des outils qui utilisent une ancienne version d’un endpoint de comptage de posts et envisagez de migrer vers les endpoints plus récents de l’API X v2, ce guide est fait pour vous.

Cette page contient deux tableaux de comparaison :

- [Comptage des posts récents](#recent)
- [Comptage des posts sur l’archive complète](#full-archive)
- [Comparaison des opérateurs de filtrage](#operator)

<div id="recent-post-counts-comparison">
  ### Comparaison des décomptes de Posts récents
</div>

La version Entreprise des endpoints de décomptes de Posts permet d’obtenir des décomptes sur 30 jours ou sur l’archive complète. Par conséquent, l’endpoint v2 des décomptes de Posts récents, qui couvre une période de 7 jours, ne remplace directement aucun des endpoints susmentionnés.

Cependant, pour faciliter les comparaisons, nous allons examiner en quoi l’endpoint v2 des décomptes de Posts récents se compare à l’endpoint Entreprise sur 30 jours.

Le tableau suivant compare les différents types d’endpoints de décomptes de Posts récents :

| **Description** | **Entreprise** | **API X v2** |
| :--- | :--- | :--- |
| Domaine hôte | https://gnip-api.x.com | https://api.x.com |
| Chemin d’endpoint | /search/30day/accounts/:account\_name/:label/counts.json | /2/tweets/counts/recent |
| [Authentification](/fr/resources/fundamentals/authentication) | Authentification de base | Jeton Bearer OAuth 2.0 |
| Format d’horodatage | YYYYMMDDhhmm | YYYY-MM-DDTHH:mm:ssZ  <br />[ISO 8601 / RFC 3339](https://tools.ietf.org/html/rfc3339#section-5.6) |
| Renvoie les décomptes de Posts âgés d’au plus | 31 jours | 7 jours |
| Méthodes HTTP prises en charge | GET | GET |
| Limites de taux de requêtes par défaut | 20 requêtes par 1 s, agrégées entre les données de recherche et les requêtes de décomptes  <br />La limite par minute varie selon le partenaire, comme spécifié dans votre contrat. | 180 requêtes par 15 min et par utilisateur  <br />450 requêtes par 15 min et par application |
| Prend en charge le filtrage via les [annotations](/fr/x-api/fundamentals/post-annotations) |     | ✔   |
| Prend en charge le filtrage via [conversation\_id](/fr/x-api/fundamentals/conversation-id) |     | ✔   |
| Nom de clé JSON pour le tableau de données de Posts | results | data |
| Granularité temporelle | Jour, heure ou minute | Jour, heure ou minute |
| Fuseau horaire | UTC | UTC |
| Paramètres de requête pour sélectionner la période | fromDate  <br />toDate | start\_time  <br />end\_time |
| Paramètres de requête pour naviguer par ID de Post |     | since\_id  <br />until\_id |
| Nécessite l’utilisation d’identifiants d’une [application développeur](/fr/resources/fundamentals/developer-apps) associée à un [projet](/fr/resources/fundamentals/projects) |     | ✔   |

<div id="filtering-operator-comparison">
  ### Comparaison des opérateurs de filtrage
</div>

Les deux versions (Enterprise et v2) des décomptes de posts diffèrent par les opérateurs disponibles et présentent également des niveaux de disponibilité variables au sein de chaque version, comme expliqué ci-dessous.

Enterprise

- Il n’existe pas de sous-niveaux d’opérateurs Enterprise. Tous les opérateurs Enterprise sont disponibles pour tous les utilisateurs Enterprise.

API X v2

- **Core :** Ces opérateurs sont disponibles pour tout utilisateur v2.
- **Advanced :** Ces opérateurs ne sont disponibles que pour les utilisateurs approuvés pour l’accès Academic Research.

Vous pouvez en savoir plus sur chacun de ces ensembles d’opérateurs dans leurs guides respectifs :

- [Enterprise operators](/fr/x-api/enterprise-gnip-2.0/fundamentals/search-api#building-search-queries)
- [X API v2 operators](/fr/x-api/posts/search/integrate/build-a-query)
