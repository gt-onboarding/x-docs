<div id="manage-likes-standard-v11-compared-to-x-api-v2">
  ### Gérer les J’aime : Standard v1.1 vs API X v2
</div>

Si vous avez travaillé avec les endpoints standard v1.1 [POST favorites/create](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-create) et [POST favorites/destroy](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-destroy), ce guide a pour objectif de vous aider à comprendre les similitudes et les différences entre les endpoints de gestion des J’aime de la version standard v1.1 et de l’API X v2.

- **Similitudes**
  - Contexte utilisateur OAuth 1.0a
- **Différences**
  - URL des endpoints et méthodes HTTP
  - Exigences pour l’application et le projet
  - Paramètres de requête

<div id="similarities">
  #### Similarités
</div>

**Méthode d’authentification OAuth 1.0a en contexte utilisateur**

Les deux versions de l’endpoint prennent en charge [OAuth 1.0a User Context](/fr/resources/fundamentals/authentication#oauth-1-0a-2). Par conséquent, si vous utilisiez auparavant l’un des endpoints standard v1.1 pour gérer les favoris, vous pouvez continuer à utiliser la même méthode d’authentification si vous migrez vers la version de l’API X v2.

<div id="differences">
  #### Différences
</div>

**URL d’endpoint et méthodes HTTP**

- Endpoints Standard v1.1 :
  - POST https://api.x.com/1.1/favorites/create.json
    (aimer un Post)
  - POST https://api.x.com/1.1/favorites/destroy.json
    (ne plus aimer un Post)
- Endpoint API X v2 :
  - POST https://api.x.com/2/tweets/:id/likes
    (aimer un Post)
  - DELETE https://api.x.com/2/tweets/:id/likes/:tweet\_id
    (ne plus aimer un Post) 

**Exigences pour les applications et les Projets**

Les endpoints de l’API X v2 exigent d’utiliser des identifiants provenant d’une [application développeur](/fr/resources/fundamentals/developer-apps) associée à un [Projet](/fr/resources/fundamentals/projects) lors de l’authentification de vos requêtes. Tous les endpoints de l’API X v1.1 peuvent utiliser des identifiants d’applications autonomes ou d’applications associées à un Projet.

**Paramètres de requête**

Les paramètres de requête Standard v1.1 suivants ont des équivalents dans l’API X v2 :

| Standard v1.1 | API X v2 |
| :--- | :--- |
| id  | id  |
| includes\_entities | Aucun équivalent |

Veuillez noter que les paramètres Standard v1.1 sont transmis en paramètres de requête, tandis que ceux de l’API X v2 sont transmis dans le corps pour l’endpoint POST ou dans le chemin pour l’endpoint DELETE.

Par ailleurs, il n’est pas nécessaire de fournir l’id de l’utilisateur qui aime un Post lors de l’utilisation des endpoints Standard v1.1, car les [jetons d’accès](/fr/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) transmis avec le [contexte utilisateur OAuth 1.0a](/fr/resources/fundamentals/authentication) permettent de déduire quel utilisateur initie l’action d’aimer/ne plus aimer.
