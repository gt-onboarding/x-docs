<div id="getting-started-with-reverse-chronological-home-timeline">
  ## Prise en main de la timeline d’accueil antéchronologique
</div>

Ce guide de démarrage rapide vous aidera à effectuer votre première requête vers l’un des endpoints de timelines avec un ensemble de champs spécifiés à l’aide de [Postman](/fr/tutorials/postman-getting-started).

Si vous souhaitez voir des exemples de code dans différents langages, veuillez consulter notre dépôt GitHub [Exemples pour l’API X v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code).

<Note>
  **Prérequis**

  Pour suivre ce guide, vous aurez besoin d’un ensemble de [clés et tokens](/fr/resources/fundamentals/authentication) pour authentifier votre requête. Vous pouvez générer ces clés et tokens en suivant ces étapes :

  - [Inscrivez-vous pour un compte développeur](https://developer.x.com/en/apply-for-access) et obtenez l’approbation.
  - Créez un [Projet](/fr/resources/fundamentals/projects) et une [application développeur](/fr/resources/fundamentals/developer-apps) associée dans le portail développeur.
  - Accédez à la page « Keys and tokens » de votre App pour générer les identifiants requis. Veillez à conserver tous les identifiants dans un endroit sécurisé.
</Note>

**Étapes pour construire une requête de timeline d’accueil antéchronologique**

_Pour cet exemple, nous enverrons une requête à l’endpoint de timeline des Posts d’un utilisateur par ID, mais vous pouvez appliquer les enseignements de ce démarrage rapide aux requêtes de timelines de mentions d’utilisateur également._

**Première étape : Commencez avec un outil ou une bibliothèque**

Il existe plusieurs outils, exemples de code et bibliothèques que vous pouvez utiliser pour effectuer une requête vers cet endpoint, mais nous allons utiliser l’outil Postman ici pour simplifier le processus.

Pour charger la collection Postman de l’API X v2 dans votre environnement, veuillez cliquer sur le bouton suivant :

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Ajouter l’API X v2 à Postman
</Button>

Une fois la collection API X v2 chargée dans Postman, accédez au dossier « Timelines » et trouvez la requête « Reverse chronological home timeline ».

**Deuxième étape : Authentifiez votre requête**

Pour effectuer correctement une requête vers l’API X, vous devez vérifier que vous disposez des autorisations nécessaires. Pour ce faire avec cet endpoint, vous devez authentifier votre requête avec [OAuth 2.0 Authorization Code with PKCE](/fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) ou [OAuth 1.0a User Context](/fr/resources/fundamentals/authentication).

Vous devez ajouter vos clés et tokens à Postman. Pour cela, sélectionnez l’environnement nommé « X API v2 » dans le coin supérieur droit de Postman, puis ajoutez vos clés et tokens dans les champs « initial value » et « current value » (en cliquant sur l’icône en forme d’œil à côté du menu déroulant des environnements).

Ces variables seront automatiquement appliquées dans l’onglet d’autorisation de la requête si tout est correctement configuré.

**Troisième étape : Identifier et spécifier l’utilisateur dont vous souhaitez récupérer la timeline d’accueil**

Vous devez indiquer, dans la requête, l’utilisateur pour lequel vous souhaitez récupérer des Posts récents. Dans cet exemple, nous transmettrons un seul ID utilisateur.

Les IDs utilisateur sont simplement la valeur numérique représentant un handle de compte, que vous pouvez trouver dans l’URL du profil d’un compte. Par exemple, le nom d’utilisateur du compte suivant est `XDevelopers`.

`https://x.com/XDevelopers`

Pour convertir ce nom d’utilisateur en ID utilisateur, utilisez l’[endpoint de recherche d’utilisateurs](/fr/x-api/users/lookup/introduction) avec le nom d’utilisateur et trouvez l’ID utilisateur numérique dans le payload. Dans le cas de @XDevelopers, l’ID utilisateur est 2244994945.

Dans Postman, accédez à l’onglet « Params » et saisissez cet ID utilisateur dans la colonne « Value » du paramètre id.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 2244994945 |

**Quatrième étape : Identifier et spécifier les champs que vous souhaitez récupérer**

Si vous cliquez sur le bouton « Send » après l’étape trois, vous recevrez les champs par défaut de l’[objet Post](/fr/x-api/fundamentals/data-dictionary#tweet) dans votre réponse : id, text et edit\_history\_tweet\_ids.

Si vous souhaitez recevoir des champs supplémentaires, vous devrez les spécifier dans votre requête avec les paramètres [field](/fr/x-api/fundamentals/fields) et/ou [expansion](/fr/x-api/fundamentals/expansions).

Pour cet exercice, nous demanderons trois ensembles supplémentaires de champs provenant d’objets différents :

1. Le champ supplémentaire tweet.created\_at dans les objets utilisateur principaux.

2. Les champs par défaut de l’objet utilisateur des auteurs associés pour les posts renvoyés : id, name et username

3. Le champ supplémentaire user.created\_at dans les objets utilisateur associés.
    

Dans Postman, accédez à l’onglet « Params » et ajoutez la paire clé:valeur suivante au tableau « Query Params » :

|     |     |     |
| :--- | :--- | :--- |
| **Clé** | **Valeur** | **Champs renvoyés** |
| `tweet.fields` | `created_at` | `tweets.created_at` |
| `expansions` | `author_id` | `includes.users.id`, `includes.users.name`, `includes.users.username` |
| `user.fields` | `created_at` | `includes.users.created_at` |
| max\_results | 5   |     |

Vous devriez maintenant voir l’URL suivante à côté du bouton « Send » :

`https://api.x.com/2/users/:id/timelines/reverse_chronological?tweet.fields=created_at&expansions=author_id&user.fields=created_at&max_results=5`

<Info>
  **Veuillez noter :**

  Dans Postman, le paramètre de chemin :id dans le champ URL ne se mettra **pas** automatiquement à jour avec la valeur que vous saisissez dans le champ de paramètres `id`, c’est pourquoi l’URL ci-dessus inclut `:id` et non 2244994945.
</Info>

**Étape six : parcourir les résultats par pagination**

Dans la réponse précédente, vous trouverez en bas un objet de métadonnées comprenant les champs suivants :

- oldest\_id
- newest\_id
- results\_count
- next\_token
- previous\_token

À l’étape quatre, nous avons défini max\_results à 5, ce qui signifie que chaque page contiendra au maximum cinq résultats. Pour accéder aux pages de données suivantes, récupérez la valeur du champ next\_token des derniers résultats et utilisez cette chaîne comme valeur du paramètre pagination\_token dans l’onglet Params de Postman, en laissant tout le reste inchangé.

|     |     |
| :--- | :--- |
| **Clé** | **Valeur** |
| `pagination_token` | t3buvdr5pujq9g7bggsnf3ep2ha28 |

Une fois la configuration terminée, cliquez de nouveau sur « Send » et vous devriez recevoir la page de résultats suivante.

Nous avons rédigé un guide sur la [pagination](/fr/x-api/fundamentals/pagination) pour approfondir ce concept.
