<div id="getting-started-with-the-hide-replies-endpoint">
  ## Premiers pas avec l’endpoint de masquage des réponses
</div>

Ce guide de démarrage rapide vous aidera à effectuer votre première requête vers l’endpoint de masquage des réponses à l’aide de [Postman](/fr/tutorials/postman-getting-started).

Si vous souhaitez voir des extraits de code dans différents langages, consultez la [page de référence de l’API pour le masquage des réponses](/fr/x-api/posts/hide-replies#put-2-tweets-id-hidden).

<Note>
  ### Prérequis

  Pour suivre ce guide, vous aurez besoin d’un jeu de [clés et jetons](/fr/resources/fundamentals/authentication) pour authentifier votre requête. Vous pouvez générer ces clés et jetons en suivant ces étapes :

  - [Inscrivez-vous à un compte développeur](https://developer.x.com/en/apply-for-access) et obtenez l’approbation.
  - Créez un [projet](/fr/resources/fundamentals/projects) et une [application développeur](/fr/resources/fundamentals/developer-apps) associée dans le portail développeur.
  - Accédez à la page « Keys and tokens » de votre application pour générer les identifiants requis. Assurez-vous d’enregistrer tous les identifiants dans un emplacement sécurisé.
</Note>

<div id="steps-to-build-a-put-tweetsidhidden-request">
  ### Étapes pour créer une requête PUT /tweets/:id/hidden
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### Première étape : démarrer avec un outil ou une bibliothèque
</div>

Il existe plusieurs outils, exemples de code et bibliothèques que vous pouvez utiliser pour envoyer une requête à cet endpoint, mais nous allons utiliser Postman ici pour simplifier le processus.

Pour charger la collection API X v2 dans votre environnement, veuillez cliquer sur le bouton suivant :

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Ajouter l’API X v2 à Postman
</Button>

Une fois la collection API X v2 chargée dans Postman, accédez à l’endpoint « masquer les réponses ».

<div id="step-two-authenticate-your-request">
  #### Deuxième étape : authentifiez votre requête
</div>

Pour envoyer correctement une requête à l’API X, vous devez vérifier que vous disposez des autorisations nécessaires. Pour ce faire avec cet endpoint, vous devez authentifier votre requête à l’aide soit d’[OAuth 1.0a User Context](/fr/resources/fundamentals/authentication), soit d’[OAuth 2.0 Authorization Code with PKCE](fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

Dans cet exemple, nous allons utiliser OAuth 1.0a User Context.

Ajoutez vos clés et jetons — en particulier votre API Key, API Secret Key, OAuth 1.0a user Access Token et OAuth 1.0a user Access Token Secret — dans Postman. Pour cela, sélectionnez l’environnement nommé « X API v2 » dans l’angle supérieur droit de Postman, puis renseignez vos clés et jetons dans les champs « initial value » et « current value » (en cliquant sur l’icône en forme d’œil à côté de la liste déroulante des environnements).

Ces variables seront automatiquement reprises dans l’onglet Authorization de la requête si vous avez procédé correctement.
 

<div id="step-three-find-a-post-id-to-hide">
  #### Troisième étape : trouver un ID de Post à masquer
</div>

L’endpoint hide replies peut masquer ou réafficher des réponses pour le compte d’un utilisateur autorisé. Comme nous utilisons dans cet exemple les jetons d’accès associés à votre profil, vous pourrez masquer des réponses d’utilisateurs participant à une conversation que vous avez initiée. De même, si vous utilisiez des jetons d’accès appartenant à un autre utilisateur ayant autorisé votre application, vous pourriez modérer les réponses à toute conversation démarrée par ce compte.

Demandez à un ami de répondre à un Post (précisez que vous testez la fonction de masquage des réponses) ou répondez à l’un de vos Posts depuis un compte de test. Cliquez sur cette réponse, puis copiez la partie numérique de son URL. Ce sera l’ID du Post que nous allons masquer.

Dans ce cas, nous examinerons le Post suivant, dont l’ID est `1232720193182412800` :

`https://x.com/TwitterDev/status/1232720193182412800`

<div id="step-four-hide-the-post">
  #### Quatrième étape : Masquer le Post
</div>

Dans Postman, ouvrez le dossier Hide replies et sélectionnez Hide a reply. Dans l’onglet Params, collez l’ID du Post à côté du champ id (vous n’avez pas besoin de remplacer :id dans l’URL). Cliquez sur « Send » et vous verrez une réponse indiquant la réussite de l’opération.

`{"hidden":true}`

<div id="step-five-unhide-the-post">
  #### Étape cinq : Rendre le Post visible
</div>

Les Posts masqués sont déplacés vers un onglet distinct dans l’application X. Pour rendre un Post visible dans Postman, ouvrez le dossier Hide replies et sélectionnez Unhide a reply. Dans l’onglet Params, collez le même identifiant de Post utilisé à l’étape précédente dans le champ id. Cliquez sur « Send » et vous verrez une réponse indiquant la réussite de l’opération.

`{"hidden":false}`

Le champ hidden représente l’état de masquage du Post. Un état de masquage à true signifie que le Post est masqué. De même, false signifie que le Post n’est pas masqué.
