Cet endpoint vous permet de masquer ou d’afficher à nouveau des réponses de manière programmatique selon des critères que vous définissez. Comme dans l’expérience principale de X, les réponses seront masquées de la conversation principale mais resteront visibles sur une page séparée. Vous pouvez utiliser cet endpoint pour créer des applications aidant les utilisateurs à masquer les réponses qu’ils ne jugent pas pertinentes.

L’endpoint de masquage des réponses utilise l’authentification [OAuth 1.0a User Context](/fr/resources/fundamentals/authentication#oauth-1-0a-2) ou [OAuth 2.0 Authorization Code avec PKCE](fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). En cas de succès, l’endpoint masque une seule réponse publiée dans une conversation de Post initiée par un utilisateur authentifié. Chaque conversation permet de masquer jusqu’à 725 Posts.

<Note>
  **Configuration du compte**

  Pour accéder à ces endpoints, vous aurez besoin de :

  - D’un [compte développeur](https://developer.x.com/en/portal/petition/essential/basic-info) approuvé.
  - De vous authentifier à l’aide des clés et des jetons d’une [application développeur](/fr/resources/fundamentals/developer-apps) située dans un [projet](/fr/resources/fundamentals/projects).

  Pour en savoir plus sur l’accès aux endpoints de l’API X v2, consultez notre [guide de démarrage](/fr/x-api/getting-started/getting-access).
</Note>
