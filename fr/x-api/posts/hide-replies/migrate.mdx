  ## Comparaison des endpoints de masquage des réponses de l’API X
</div>

L’endpoint v2 de masquage des réponses remplace l’endpoint Labs de masquage des réponses. Si vous avez du code, des applications ou des outils qui utilisent la version Labs de cet endpoint et que vous envisagez de migrer vers le nouvel endpoint de l’API X v2, ce guide est pour vous.

Pour utiliser la nouvelle API X v2 (y compris l’endpoint de masquage des réponses), vous devrez [adhérer au nouveau portail développeur](https://developer.x.com/en/portal/opt-in), créer un [projet](/fr/resources/fundamentals/projects) et y ajouter une application. Vous pourrez ensuite utiliser les identifiants associés à cette application pour effectuer des requêtes vers l’endpoint de masquage des réponses. Ajouter la même application inscrite pour l’endpoint Labs v2 de masquage des réponses maintiendra l’authentification de vos utilisateurs.

Le tableau suivant compare les différences entre Labs et le nouvel endpoint de l’API X v2 :
 

| **Description** | **Labs v2** | **API X v2** |
| :--- | :--- | :--- |
| Domaine hôte | https://api.x.com | https://api.x.com |
| Chemin d’endpoint | /labs/2/tweets/:id/hidden | /2/tweets/:id/hidden |
| Authentification | Contexte utilisateur OAuth 1.0a | Contexte utilisateur OAuth 1.0a |
| Méthodes HTTP prises en charge | PUT | PUT |
| Limites de débit par défaut des requêtes | 10 requêtes par 15 minutes (partagées entre tous les utilisateurs authentifiés) | 50 requêtes par 15 minutes (pour chaque utilisateur authentifié) |
| Peut masquer des réponses | ✔︎  | ✔︎  |
| Peut réafficher une réponse précédemment masquée | ✔︎  | ✔︎  |
| Peut masquer ou réafficher des réponses plusieurs fois | ✔︎  | ✔︎  |
| Nécessite l’utilisation d’identifiants d’une [application développeur](/fr/resources/fundamentals/developer-apps) associée à un [projet](/fr/resources/fundamentals/projects) |     | ✔   |
