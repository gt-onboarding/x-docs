<div id="comparing-x-apis-search-posts-endpoints">
  ## Comparaison des endpoints de recherche de posts de l’API X
</div>

L’endpoint v2 Search Tweets finira par remplacer l’endpoint [standard v1.1 search/posts](/fr/x-api/posts/search/introduction) et l’[Enterprise Search API](/fr/x-api/enterprise-gnip-2.0/fundamentals/search-api). Si vous avez du code, des applications ou des outils qui utilisent une ancienne version d’un endpoint de recherche X et envisagez de migrer vers les endpoints plus récents de l’API X v2, ce guide est fait pour vous.

Cette page contient trois tableaux comparatifs :

- [Recherche récente](#recent)
- [Recherche sur l’archive complète](#full-archive)
- [Comparaison des opérateurs de filtrage](#operators)

<div id="recent-search-comparison">
  ### Comparaison de la recherche récente
</div>

**Comparaison des opérateurs de filtrage**

Les quatre versions (standard, enterprise et v2) de la recherche de posts diffèrent par les opérateurs disponibles et présentent également des niveaux variables de disponibilité des opérateurs au sein de chaque version, expliqués ci-dessous. 

Enterprise

- Il n’existe pas de sous-niveaux d’opérateurs Enterprise

API X v2

- **Free :** Disponible avec n’importe quel projet
- **Basic :** Disponible avec n’importe quel projet
- **Pro :** Disponible avec un projet
- **Enterprise :** Disponible avec un projet

Vous pouvez en savoir plus sur chacun de ces ensembles d’opérateurs dans leurs guides respectifs :

- [Opérateurs Enterprise](/fr/x-api/enterprise-gnip-2.0/fundamentals/search-api#available-operators)
- [Opérateurs de l’API X v2](/fr/x-api/posts/search/integrate/build-a-query#operators)
