  ## Comparaison des endpoints de flux filtré de l’API X
</div>

Le groupe d’endpoints de flux filtré v2 remplace [statuses/filter standard v1.1](https://developer.x.com/en/docs/x-api/v1/tweets/filter-realtime/api-reference/post-statuses-filter) et l’[API PowerTrack](/fr/x-api/enterprise-gnip-2.0/powertrack-api). Si vous avez du code, des applications ou des outils utilisant une ancienne version de l’endpoint de flux filtré et que vous envisagez de migrer vers l’endpoint plus récent de l’API X v2, cette comparaison peut vous aider à démarrer.

Consultez nos guides de migration plus détaillés :

[Migration de Standard v1.1 vers l’API X v2](fr/x-api/posts/filtered-stream#standard-v1-1-compared-to-x-api-v2)

[Migration de l’API PowerTrack vers l’API X v2](/fr/x-api/enterprise-gnip-2.0/powertrack-api#powertrack-operators)

Le tableau suivant compare les endpoints de flux filtré proposés par X :
 

| **Description** | **Standard v1.1** | **API PowerTrack** | **API X v2** |
| :--- | :--- | :--- | :--- |
| Accès | Application X | Nécessite un contrat et un compte Enterprise | Nécessite un compte développeur ([s’inscrire](https://developer.x.com/en/portal/petition/essential/basic-info)), et une [application X](/fr/resources/fundamentals/developer-apps) au sein d’un [projet](/fr/resources/fundamentals/projects) |

| [Authentification](/fr/resources/fundamentals/authentication) | Contexte utilisateur OAuth 1.0a | Authentification HTTP Basic | OAuth 2.0 App-Only |

| Connexions maximales autorisées | 2 simultanées par utilisateur autorisé | Prend en charge des connexions multiples/redondantes, déterminées par contrat | Accès Pro :  <br />1 |
| [Fonctionnalités de récupération et de redondance](/fr/x-api/posts/filtered-stream#filtered-stream-recovery-and-redundancy-features) | Aucune | Rattrapage, connexions redondantes et API Replay |     |
