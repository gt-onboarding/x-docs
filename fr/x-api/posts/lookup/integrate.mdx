<div id="authentication">
  ### Authentification
</div>

Tous les endpoints de l’API X v2 exigent des requêtes authentifiées. Vous pouvez vous authentifier via :

- [Contexte utilisateur OAuth 1.0a](/fr/resources/fundamentals/authentication) en utilisant des clés API, des jetons d’accès et des paramètres supplémentaires pour [créer un en-tête d’autorisation](/fr/resources/fundamentals/authentication#authorizing-a-request).
- [OAuth 2.0 App-Only](/fr/resources/fundamentals/authentication#oauth-2-0) en transmettant un [jeton d’accès d’application](/fr/resources/fundamentals/authentication) avec votre requête.
- [Code d’autorisation OAuth 2.0 avec PKCE](/fr/resources/fundamentals/authentication) pour un contrôle plus fin du périmètre de l’application et l’autorisation multi-appareils.

<Note>
  OAuth 1.0a peut être difficile à implémenter. Si vous n’êtes pas à l’aise, envisagez d’utiliser une [bibliothèque](/fr/resources/tools-and-libraries) ou OAuth 2.0 pour vos requêtes. Pour les métriques privées ou les Posts, utilisez le contexte utilisateur OAuth 1.0a ou le code d’autorisation OAuth 2.0 avec PKCE.
</Note>

<Tip>
  **Veuillez noter**

  > Si vous demandez les champs suivants, le contexte utilisateur OAuth 1.0a ou le code d’autorisation OAuth 2.0 est requis :
  >
  > - `tweet.fields.non_public_metrics`
  > - `tweet.fields.promoted_metrics`
  > - `tweet.fields.organic_metrics`
  > - `media.fields.non_public_metrics`
  > - `media.fields.promoted_metrics`
  > - `media.fields.organic_metrics`
</Tip>

<div id="developer-portal-projects-and-apps">
  ### Portail développeur, Projets et applications
</div>

Pour obtenir des identifiants pour l’API X v2, vous avez besoin de :

1. D’un [compte développeur](/fr/resources/fundamentals/developer-portal) approuvé.
2. D’un [projet](/fr/resources/fundamentals/projects) au sein de ce compte développeur.
3. D’une [application développeur](/fr/resources/fundamentals/developer-apps) dans ce projet, où se trouvent les clés et les jetons.

<div id="rate-limits">
  ### Limites de débit
</div>

Les requêtes de l’API X sont soumises à des [limites de débit](/fr/resources/fundamentals/rate-limits) pour gérer le volume. Les limites s’appliquent à la fois au niveau de l’application et de l’utilisateur :

- **Niveau application** : limite le nombre de requêtes qu’une application peut effectuer par période.
- **Niveau utilisateur** : limite la fréquence à laquelle un utilisateur authentifié peut effectuer des recherches de posts sur l’ensemble des applications développeur.

<div id="fields-and-expansions">
  ### Champs et expansions
</div>

L’API X v2 permet de sélectionner des champs de données spécifiques à l’aide de `fields` et `expansions` :

- **Expansions** : permettent de récupérer des objets liés supplémentaires. Les expansions prises en charge incluent :
  - `edit_history_tweet_ids`
  - `attachments.poll_ids`
  - `attachments.media_keys`
  - `author_id`
  - `entities.mentions.username`
  - `geo.place_id`
  - `in_reply_to_user_id`
  - `referenced_tweets.id`
  - `referenced_tweets.id.author_id`

- **Fields** : spécifient les champs de données au sein des objets pour renvoyer des données supplémentaires. L’objet Post comprend par défaut `id`, `text` et `edit_history_tweet_ids`. D’autres options, comme `tweet.created_at` et `tweet.entities`, doivent être demandées explicitement.

Pour en savoir plus, consultez le guide sur les champs et les expansions dans le [dictionnaire de données de l’API X v2](/fr/x-api/fundamentals/data-dictionary).

<div id="post-edits">
  ### Modifications de posts
</div>

Les posts éligibles peuvent être modifiés jusqu’à cinq fois dans les 30 minutes suivant leur publication. Le point de terminaison de consultation des posts fournit toujours la dernière version du post. Pour les cas d’usage en quasi temps réel, tenez compte de cette fenêtre temporelle. Pour en savoir plus, consultez [Principes fondamentaux de la modification des posts](/fr/x-api/fundamentals/edit-posts).

<div id="edge-cases">
  ### Cas limites
</div>

- **Mesures promues** : La demande de mesures promues pour des posts non promus renvoie une réponse vide.
- **Texte tronqué** : Le texte du post est tronqué pour les Retweets. Pour récupérer le texte complet, développez le post référencé.
