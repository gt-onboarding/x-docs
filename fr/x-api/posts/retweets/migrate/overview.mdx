<div id="comparing-x-apis-retweets-endpoints">
  ## Comparaison des endpoints Retweets de l’API X
</div>

**Recherche de Retweets**

L’endpoint de recherche de Retweets v2 remplacera les endpoints standard [v1.1 GET statuses/retweets/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweets-id) et [v1.1 GET statuses/retweets/:ids](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweeters-ids).

Les tableaux suivants comparent les endpoints Retweets standard v1.1 et de l’API X v2 :

| Description | Standard v1.1 | API X v2 |
| :--- | :--- | :--- |
| Méthodes HTTP prises en charge | GET | GET |
| Domaine hôte | https://api.x.com | https://api.x.com |
| Chemin d’endpoint | /1.1/retweeters/id.json<br /><br />`/1.1/retweets/ids.json` | /2/users/:id/retweeted\_by |
| [Authentification](/fr/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 2.0 Bearer Token<br /><br />OAuth 1.0a User Context |
| [Limites de débit](/fr/resources/fundamentals/rate-limits) par défaut | 75 requêtes par 15 min | 75 requêtes par 15 min (par application)<br /><br />75 requêtes par 15 min (par utilisateur) |
| Format des données | Format standard v1.1 | [Format API X v2](/fr/x-api/fundamentals/data-dictionary) (déterminé par les paramètres de requête fields et expansions, non rétrocompatible avec les formats v1.1)<br /><br />Pour en savoir plus sur la migration du format standard v1.1 vers le format API X v2, consultez notre [guide de migration des formats de données](/fr/x-api/migrate/data-format-migration). |
| Nécessite l’utilisation d’identifiants provenant d’une [developer App](/fr/resources/fundamentals/developer-apps) associée à un [Project](/fr/resources/fundamentals/projects) |     | ✔️  |

<div id="manage-retweets">
  ### Gérer les Retweets
</div>

Les tableaux suivants comparent l’endpoint d’annulation de Retweet standard v1.1 et de l’API X v2 :

**Retweeter un post**

| Description | Standard v1.1 | API X v2 |
| :--- | :--- | :--- |
| Méthodes HTTP prises en charge | POST | POST |
| Domaine hôte | https://api.x.com | https://api.x.com |
| Chemin d’endpoint | /1.1/statuses/retweet/:id.json | /2/users/:id/retweets |
| [Authentification](/fr/resources/fundamentals/authentication) | Contexte utilisateur OAuth 1.0a | Contexte utilisateur OAuth 1.0a |
| [Plafonds de requêtes](/fr/resources/fundamentals/rate-limits) par défaut | Aucun<br /><br />300 requêtes par fenêtre de 3 heures (par utilisateur, par application). Partagé avec l’endpoint POST Tweet | 50 requêtes par 15 min (par utilisateur)<br /><br />300 requêtes par fenêtre de 3 heures (par utilisateur, par application). Partagé avec l’endpoint POST Tweet pour la gestion des posts. |
| Nécessite l’utilisation d’identifiants provenant d’une [application développeur](/fr/resources/fundamentals/developer-apps) associée à un [projet](/fr/resources/fundamentals/projects) |     | ✔️  |
