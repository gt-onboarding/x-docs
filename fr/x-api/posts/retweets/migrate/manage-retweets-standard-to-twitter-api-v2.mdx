<div id="manage-retweets-standard-v11-compared-to-x-api-v2">
  ### Gérer les Retweets : standard v1.1 comparé à l’API X v2
</div>

Si vous avez travaillé avec les endpoints standard v1.1 [POST statuses/retweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-retweet-id) et [POST statuses/unretweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-unretweet-id), ce guide a pour objectif de vous aider à comprendre les similitudes et les différences entre les endpoints de Retweet de la version standard et de l’API X v2.

- **Similitudes**
  - Authentification
- **Différences**
  - URL des endpoints et méthodes HTTP
  - Limites des requêtes
  - Exigences pour les applications et les projets
  - Paramètres de requête

<div id="similarities">
  #### Similarités
</div>

**Authentification**

Les deux endpoints de gestion des Retweets de la version standard v1.1 ([POST statuses/retweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-retweet-id) et [POST statuses/unretweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-unretweet-id)) utilisent [OAuth 1.0a User Context](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Par conséquent, si vous utilisiez auparavant l’un des endpoints de recherche des Retweets de la version standard v1.1, vous pouvez continuer à utiliser la même méthode d’authentification en migrant vers l’API X v2. 

<div id="differences">
  #### Différences
</div>

**URL d’endpoints et méthodes HTTP**

- Endpoints Standard v1.1 :
  - https://api.x.com/1.1/statuses/retweet/:id.json
    (Retweete un Post. Renvoie le Post d’origine avec les détails du Retweet intégrés)
  - https://api.x.com/1.1/statuses/unretweet/:id.json
    (Annule un Retweet. Renvoie le Post d’origine avec les détails du Retweet intégrés)
- Endpoint de l’API X v2 :
  - https://api.x.com/2/tweets/:id/retweets
    (Retweete un Post donné)
  - https://api.x.com/2/users/:id/retweets/:source\_tweet\_id
    (Annule un Retweet d’un Post donné) 

**Exigences relatives aux applications et aux projets**

Les endpoints de l’API X v2 exigent d’utiliser des identifiants provenant d’une [application développeur](/fr/resources/fundamentals/developer-apps) associée à un [projet](/fr/resources/fundamentals/projects) lors de l’authentification de vos requêtes. Tous les endpoints de l’API X v1.1 peuvent utiliser des identifiants provenant d’applications autonomes ou d’applications associées à un projet.

**Paramètres de requête**

Les endpoints Standard v1.1 suivants acceptaient deux paramètres de requête (user\_id ou screen\_name). L’API X v2 n’accepte que l’ID utilisateur numérique, qui doit être transmis dans le chemin de l’endpoint.

| Standard v1.1 | API X v2 |
| :--- | :--- |
| **id** | **id** |
| **includes\_entities** | Aucun équivalent |

Veuillez noter que, dans Standard v1.1, les paramètres sont transmis en tant que paramètres de requête, tandis que, dans l’API X v2, ils sont transmis en tant que paramètres de corps pour l’endpoint POST ou en tant que paramètres de chemin pour l’endpoint DELETE.

De plus, l’ID de l’utilisateur qui retweete un Post n’est pas requis lors de l’utilisation des endpoints Standard v1.1, puisque les [Access Tokens](/fr/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) transmis avec le [contexte utilisateur OAuth 1.0a](/fr/resources/fundamentals/authentication) permettent de déterminer quel utilisateur initie ou annule le Retweet.
