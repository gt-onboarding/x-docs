<div id="retweets-lookup">
  ### Recherche des Retweets
</div>

Avec le point de terminaison « Retweets lookup », vous pouvez obtenir la liste des comptes qui ont retweeté un Post. Pour cet endpoint, des jetons de pagination sont fournis pour parcourir de grands ensembles de résultats par lots jusqu’à 100 utilisateurs.

Vous pouvez authentifier ces endpoints avec soit [OAuth 1.0a User Context](/fr/resources/fundamentals/authentication), \[OAuth 2.0 App-Only]\(https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), ou [OAuth 2.0 Authorization Code with PKCE](/fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3). 

<div id="retweets-of-me">
  ### Mes Retweets
</div>

La recherche « Reposts of Me » vous permet de voir lesquels de vos propres posts ont été partagés par d’autres.

Elle offre aux utilisateurs une visibilité sur les posts qui sont partagés et qui trouvent un écho auprès de leur audience, ce qui peut favoriser des interactions plus ciblées et des ajustements de la stratégie de contenu.

Vous pouvez authentifier ces endpoints soit avec [OAuth 1.0a User Context](/fr/resources/fundamentals/authentication), soit avec [OAuth 2.0 Authorization Code with PKCE](/fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

<div id="manage-retweets">
  #### Gérer les Retweets
</div>

Les endpoints de gestion des Retweets vous permettent de Retweeter un Post ou d’annuler un Retweet d’un Post spécifié pour le compte d’un utilisateur authentifié. Pour ce groupe d’endpoints, deux méthodes sont disponibles : POST et DELETE. La méthode POST permet de Retweeter un Post, et la méthode DELETE d’annuler un Retweet d’un Post donné.

Comme vous effectuez des requêtes pour le compte d’un utilisateur, vous devez authentifier ces endpoints avec [OAuth 1.0a (contexte utilisateur)](/fr/resources/fundamentals/authentication) ou [OAuth 2.0 (Authorization Code avec PKCE)](fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), et utiliser les jetons d’accès utilisateur associés à l’utilisateur au nom duquel vous faites la requête. Vous pouvez générer ce jeton d’accès utilisateur via le [flux OAuth à 3 intervenants](/fr/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) ou via le [flux Authorization Code avec PKCE](/fr/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0). Vous pouvez Retweeter un Post depuis votre compte ou depuis le compte d’un utilisateur authentifié. Pour ces deux endpoints, la limite par utilisateur est de 50 requêtes par tranche de 15 minutes et par endpoint.
