<div id="authentication">
  #### Authentification
</div>

Tous les endpoints de l’API X v2 exigent que vous authentifiiez vos requêtes avec un ensemble d’identifiants, également appelés clés et jetons.

Vous pouvez utiliser soit OAuth 1.0a en contexte utilisateur (User Context), soit un jeton d’accès OAuth 2.0 (Bearer Token) pour authentifier vos requêtes vers l’endpoint de recherche des Retweets.

Les endpoints de gestion des Retweets nécessitent l’utilisation d’OAuth 1.0a en contexte utilisateur, ce qui signifie que vous devez utiliser un ensemble de clés d’API et de jetons d’accès utilisateur pour effectuer une requête réussie. Les jetons d’accès doivent être associés à l’utilisateur pour le compte duquel vous effectuez la requête. Si vous souhaitez générer un ensemble de jetons d’accès pour un autre utilisateur, celui-ci doit autoriser ou authentifier votre application via le flux OAuth à 3 volets (3-legged OAuth).

Veuillez noter qu’OAuth 1.0a peut être délicat à utiliser. Si vous n’êtes pas familier avec cette méthode d’authentification, nous vous recommandons d’utiliser une [bibliothèque](/fr/x-api/tools-and-libraries/overview) pour authentifier correctement vos requêtes.

**Veuillez noter**

Si vous demandez les champs suivants, OAuth 1.0a en contexte utilisateur est requis :

- tweet.fields.non\_public\_metrics
- tweet.fields.promoted\_metrics
- tweet.fields.organic\_metrics
- media.fields.non\_public\_metrics
- media.fields.promoted\_metrics
- media.fields.organic\_metrics

<div id="developer-portal-projects-and-developer-apps">
  #### Portail développeur, Projets et applications développeur
</div>

Pour obtenir un ensemble d’identifiants d’authentification compatibles avec les endpoints de l’API X v2, vous devez [créer un compte développeur](https://developer.x.com/en/portal/petition/essential/basic-info), configurer un [Projet](/fr/resources/fundamentals/projects) dans ce compte, puis créer une [application développeur](/fr/resources/fundamentals/developer-apps) dans ce Projet. Vous pourrez ensuite retrouver vos clés et jetons dans votre application développeur. 
 

<div id="rate-limits">
  #### Limites de débit
</div>

Chaque jour, des milliers de développeurs envoient des requêtes à l’API X. Pour gérer le volume de ces requêtes, des [limites de débit](/fr/x-api/fundamentals/rate-limits) sont appliquées à chaque endpoint, ce qui restreint le nombre de requêtes que vous pouvez effectuer au nom de votre application ou d’un utilisateur authentifié.

Les endpoints de gestion des Retweets sont limités à 50 requêtes par tranche de 15 minutes (par utilisateur). En outre, pour l’endpoint POST, la limite est de 300 requêtes sur une fenêtre de 3 heures (par utilisateur, par application).

Avec l’endpoint de consultation des Retweets, la limite est de 75 requêtes par tranche de 15 minutes. De plus, seuls les 100 utilisateurs ayant retweeté le plus récemment seront renvoyés par cet endpoint.

<div id="fields-and-expansions">
  #### Champs et extensions
</div>

L’API X v2 permet aux utilisateurs de choisir précisément quelles données renvoyer via un ensemble d’outils appelés champs et extensions. Le paramètre d’extension permet de développer les objets référencés dans la charge utile. Par exemple, ce point de terminaison permet de récupérer les [extensions](/fr/x-api/fundamentals/expansions) suivantes :

- attachments.poll\_ids
- attachments.media\_keys
- author\_id, entities.mentions.username
- geo.place\_id
- in\_reply\_to\_user\_id,
- referenced\_tweets.id,
- referenced\_tweets.id.author\_id

Le paramètre de champs permet de sélectionner précisément quels [champs](/fr/x-api/fundamentals/fields) au sein des différents objets de données vous souhaitez recevoir. Ces points de terminaison renvoient principalement des objets Post. Par défaut, l’objet Post renvoie les champs id et text. Pour recevoir des champs supplémentaires comme tweet.created\_at ou tweet.entities, vous devez les demander explicitement à l’aide d’un paramètre de champs. Parmi les champs importants que vous pourriez envisager d’utiliser dans votre intégration figurent nos données de sondage, les mesures, les annotations de Post et l’ID de conversation.

Nous avons ajouté à notre [dictionnaire de données de l’API X v2](/fr/x-api/fundamentals/data-dictionary) un guide expliquant comment [utiliser conjointement les champs et les extensions](/fr/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions).
