Un objet média représente une photo, une vidéo ou un GIF animé. Les objets média sont utilisés par de nombreuses API de l’API X et peuvent être inclus dans les posts, les Messages privés, les profils utilisateur, les créations publicitaires, etc. Chaque objet média peut proposer plusieurs variantes d’affichage ou de lecture, avec différentes résolutions ou formats.

<div id="media-types-size-restrictions">
  ## Types de médias et limites de taille
</div>

Limites de taille pour l’envoi via l’API

- **Image** : `5 MB`
- **GIF** : `15 MB`
- **Vidéo** : `512 MB` (avec `media_category=amplify_video`)

<div id="creation">
  ## Création
</div>

Des objets tels que des posts, des Messages privés, des photos de profil, des cartes publicitaires hébergées, etc., peuvent contenir un ou plusieurs objets média. Ces objets de premier niveau sont collectivement appelés entités. L’API de création d’entité concernée (par exemple, [`POST /2/tweets`](/fr/x-api/posts/creation-of-a-post)) peut recevoir un ou plusieurs objets média au moyen d’un `media_id` unique.

Une entité contenant un ou plusieurs objets média peut être créée en procédant comme suit :

1. Téléversez le(s) fichier(s) média en utilisant soit le téléversement [segmenté](/fr/x-api-/media/quickstart/media-upload-chunked) recommandé (images/GIF/vidéo), soit l’ancien téléversement [simple](/fr/x-api/media-upload/media-upload) (images uniquement).
2. Recevez un `media_id` à l’issue de l’étape 1. Cette étape peut être répétée plusieurs fois avec des médias différents si l’entité autorise plusieurs paramètres `media_id`.
3. Créez l’entité en appelant le point de terminaison approprié, en incluant le `media_id` et les autres paramètres requis. Par exemple, associez un `media_id` à un post à l’aide du point de terminaison [`POST /2/tweets`](/fr/x-api/posts/creation-of-a-post).
