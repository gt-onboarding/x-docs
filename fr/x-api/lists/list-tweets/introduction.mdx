La consultation de Posts d’une Liste dispose d’un unique point de terminaison pour récupérer les Posts d’une Liste spécifiée. Avec ce point de terminaison, vous pouvez créer des solutions permettant aux utilisateurs de personnaliser, organiser et prioriser les Posts qu’ils voient dans leur fil.

La limite est de 900 requêtes par tranche de 15 minutes pour ce point de terminaison, et la réponse permet d’interroger jusqu’aux 800 Posts les plus récents pour une Liste donnée.

Vous pouvez utiliser [OAuth 1.0a User Context](/fr/resources/fundamentals/authentication), \[App only]\(https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), ou [OAuth 2.0 Authorization Code with PKCE](fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) pour authentifier vos requêtes vers ce point de terminaison. 

**Configuration du compte**

Pour accéder à ces points de terminaison, vous aurez besoin de :

- D’un [compte développeur](https://developer.x.com/en/portal/petition/essential/basic-info) approuvé.
- De vous authentifier à l’aide des clés et jetons d’une [developer App](/fr/resources/fundamentals/developer-apps) située au sein d’un [Project](/fr/resources/fundamentals/projects). 

En savoir plus sur l’accès aux points de terminaison de l’API X v2 dans notre [guide de démarrage](/fr/x-api/getting-started/getting-access).
