  ## Comparaison des endpoints de recherche de posts de liste de l’API X
</div>

L’endpoint v2 de recherche de posts de liste remplacera l’endpoint standard v1.1 [GET lists/statuses](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-statuses). Si vous avez du code, des applications ou des outils qui utilisent cette version de l’endpoint et envisagez de migrer vers le nouvel endpoint de l’API X v2, cette série de guides est faite pour vous.

Les tableaux suivants comparent les endpoints standard v1.1 et de l’API X v2 pour les listes :

**Recherche de liste par ID**

|     |     |     |
| :--- | :--- | :--- |
| Description | Standard v1.1 | API X v2 |
| Méthodes HTTP prises en charge | `GET` | `GET` |
| Domaine hôte | `https://api.x.com` | `https://api.x.com` |
| Chemin de l’endpoint | `/1.1/lists/statuses.json` | `/2/lists/:id/tweets` |
| [Authentification](/fr/resources/fundamentals/authentication) | OAuth 1.0a (contexte utilisateur)<br /><br />App only | OAuth 1.0a (contexte utilisateur)<br /><br />OAuth 2.0 Authorization Code avec PKCE<br /><br />App only |
| [Limites de débit](/fr/resources/fundamentals/rate-limits) par défaut | 900 requêtes par 15 min avec OAuth 1.0a<br /><br />900 requêtes par 15 min avec App only | 900 requêtes par 15 min avec OAuth 1.0a<br /><br />900 requêtes par 15 min avec OAuth 2.0<br /><br />900 requêtes par 15 min avec App only |

Pour accéder aux endpoints de l’API X v2, vous devez [créer un compte développeur](https://developer.x.com/en/portal/petition/essential/basic-info). Lors de l’authentification, vous devez utiliser des clés et des jetons d’une [application développeur](/fr/resources/fundamentals/developer-apps) située dans un [projet](/fr/resources/fundamentals/projects).

Pour en savoir plus sur l’obtention de l’accès aux endpoints de l’API X v2, consultez notre page [Bien démarrer](/fr/x-api/getting-started/getting-access).
