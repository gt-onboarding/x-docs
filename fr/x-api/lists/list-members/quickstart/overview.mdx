<div id="getting-started-with-the-list-members-endpoint-group">
  ## Prise en main du groupe d’endpoints List members
</div>

Cet aperçu rapide vous aidera à effectuer votre première requête vers les endpoints List members à l’aide de [Postman](/fr/tutorials/postman-getting-started).

Si vous souhaitez voir des exemples de code dans différents langages, veuillez consulter notre dépôt GitHub [Exemples pour l’API X v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code).

<Note>
  **Prérequis**

  Pour suivre ce guide, vous aurez besoin d’un ensemble de [clés et jetons](/fr/resources/fundamentals/authentication) pour authentifier votre requête. Vous pouvez générer ces clés et jetons en suivant ces étapes :

  - [Inscrivez-vous pour un compte développeur](https://developer.x.com/en/apply-for-access) et obtenez l’approbation.
  - Créez un [projet](/fr/resources/fundamentals/projects) et une [application développeur](/fr/resources/fundamentals/developer-apps) associée dans le portail développeur.
  - Accédez à la page « Keys and tokens » de votre application pour générer les identifiants requis. Assurez-vous d’enregistrer tous les identifiants dans un endroit sécurisé.
</Note>

<div id="authenticate-your-request">
  #### Authentifiez votre requête
</div>

Pour effectuer une requête réussie vers les points de terminaison de **lookup**, vous pouvez utiliser [OAuth 1.0a User Context](/fr/resources/fundamentals/authentication#oauth-1-0a-2), [App only](/fr/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) ou [OAuth 2.0 Authorization Code with PKCE](fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). En revanche, avec les points de terminaison de **manage**, vous ne pouvez vous authentifier qu’avec OAuth 1.0a User Context ou OAuth 2.0 Authorization Code with PKCE.

Dans tous les cas, lorsque vous utilisez Postman, les clés et jetons d’authentification par défaut seront automatiquement renseignés dans vos requêtes, à condition que vous ayez correctement configuré votre environnement.

Pour ce faire, sélectionnez l’environnement nommé « API X v2 » (dans le coin supérieur droit de Postman) et ajoutez vos clés et jetons dans les champs « initial value » et « current value » (en cliquant sur l’icône en forme d’œil à côté du menu déroulant de l’environnement). Ces clés comprennent les éléments suivants :

- API Key (également appelée Consumer Key)
- API Secret Key (également appelée Consumer Secret)
- OAuth 1.0a user Access Token
- OAuth 1.0a user Access Token Secret
- App only Access Token
- OAuth 2.0 Client Key (disponible uniquement si vous avez configuré l’authentification utilisateur OAuth 2.0 dans les paramètres de votre App)
- OAuth 2.0 Client Secret (disponible uniquement si vous avez configuré l’authentification utilisateur OAuth 2.0 dans les paramètres de votre App)
