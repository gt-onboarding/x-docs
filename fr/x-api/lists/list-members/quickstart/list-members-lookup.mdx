<div id="steps-to-build-a-list-members-lookup-request">
  ### Étapes pour créer une requête de recherche des membres d’une Liste
</div>

**Première étape : choisir la collection d’endpoints List dans Postman**

Une fois la collection de l’API X v2 chargée dans Postman, accédez au dossier « List », sélectionnez le sous-dossier « List members », puis choisissez « Members lookup ».
 

**Deuxième étape : identifier et indiquer la Liste dont vous souhaitez récupérer les membres**

Vous devez spécifier la Liste dont vous souhaitez récupérer les membres. Vous pouvez trouver l’ID de la Liste en vous rendant sur x.com, en cliquant sur une Liste, puis en regardant l’URL. Par exemple, l’ID de la Liste de l’URL suivante est 84839422.

https://x.com/i/lists/84839422

L’ID cible peut être n’importe quel ID de Liste valide. Dans Postman, accédez à l’onglet « Params » et saisissez votre ID dans la colonne « Value » de la variable de chemin id. Veillez à ne pas inclure d’espace avant ou après l’ID.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 84839422 (List ID) |

**Troisième étape : identifier et spécifier les champs que vous souhaitez récupérer**

Si vous cliquez sur le bouton « Send » après la troisième étape, vous recevrez dans la réponse les champs par défaut de l’[objet user](/fr/x-api/fundamentals/data-dictionary#user) : id, name et username.

Si vous souhaitez recevoir des champs supplémentaires au-delà de id, name et username, vous devez les spécifier dans votre requête avec les paramètres [fields](/fr/x-api/fundamentals/fields) et/ou [expansions](/fr/x-api/fundamentals/expansions).

Pour cet exercice, nous demanderons trois ensembles de champs supplémentaires provenant d’objets différents :

- Le champ supplémentaire user.created\_at dans les objets user principaux.
- L’[objet Post](/fr/x-api/fundamentals/data-dictionary#tweet) complet à l’aide du paramètre d’expansion.
- Le champ supplémentaire tweet.created\_at dans les objets Post associés.

Dans Postman, accédez à l’onglet « Params » et ajoutez la paire clé:valeur suivante au tableau « Query Params » :

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| user.fields | created\_at | `created_at` |
| expansions | pinned\_tweet\_id | `includes.tweets.id,  <br/>includes.tweets.text  <br/>` |
| tweet.fields | created\_at | `includes.tweets.created_at` |

Vous devriez maintenant voir une URL similaire à côté du bouton « Send » :

`https://api.x.com/2/lists/84839422/members?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=created_at`
