<div id="manage-list-members-standard-v11-compared-to-x-api-v2">
  ### Gérer les membres de liste : Standard v1.1 vs API X v2
</div>

Si vous avez utilisé les endpoints standard v1.1 [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create) et [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy), ce guide a pour but de vous aider à comprendre les similitudes et les différences entre les endpoints de gestion des membres de liste du standard v1.1 et de l’API X v2.

- **Similarités**
  - Authentification
- **Différences**
  - URL des endpoints
  - Exigences pour l’application et le projet
  - Méthodes HTTP
  - Limites de débit
  - Paramètres de requête

<div id="similarities">
  #### Similarités
</div>

**Authentification**

Les deux versions d’endpoint prennent en charge [OAuth 1.0a User Context](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Par conséquent, si vous utilisiez auparavant l’un des endpoints standard v1.1 pour gérer les membres d’une Liste, vous pouvez continuer à utiliser la même méthode d’authentification en migrant vers la version API X v2.

<div id="differences">
  #### Différences
</div>

**URL d’endpoint**

- Endpoints standard v1.1 :
  - POST https://api.x.com/1.1/lists/members/create.json
    (Ajoute un membre à une liste spécifiée)
  - POST https://api.x.com/1.1/lists/members/destroy.json
    (Supprime un membre d’une liste spécifiée)
- Endpoint de l’API X v2 :
  - POST https://api.x.com/2/lists/:id/members
    (Ajoute un membre à une liste spécifiée)

  - DELETE https://api.x.com/2/lists/:id/members/:user\_id
    (Supprime un membre d’une liste spécifiée)

**Limites de débit**

| **Standard v1.1** | **API X v2** |
| :--- | :--- |
| /1.1/lists/members/create.json<br /><br />aucune | /2/lists/:id/members<br /><br />300 requêtes par fenêtre de 15 minutes avec OAuth 1.0a (contexte utilisateur)<br /><br />300 requêtes par fenêtre de 15 minutes avec OAuth 2.0 (code d’autorisation avec PKCE) |
| /1.1/lists/members/destroy.json<br /><br />aucune | /2/lists/:id/members/:user\_id<br /><br />300 requêtes par fenêtre de 15 minutes avec OAuth 1.0a (contexte utilisateur)<br /><br />300 requêtes par fenêtre de 15 minutes avec OAuth 2.0 (code d’autorisation avec PKCE) |

**Exigences pour l’app et le projet**

Les endpoints de l’API X v2 exigent d’utiliser des identifiants provenant d’une [app développeur](/fr/resources/fundamentals/developer-apps) associée à un [projet](/fr/resources/fundamentals/projects) lors de l’authentification de vos requêtes. Tous les endpoints de l’API X v1.1 peuvent utiliser des identifiants provenant d’apps autonomes ou d’apps liées à un projet.

**Paramètres de requête**

Les paramètres de requête standard v1.1 suivants ont des équivalents dans l’API X v2 :

| **Standard v1.1** | **API X v2** |
| :--- | :--- |
| list\_id | id  |
| slug | Aucun équivalent |
| screen\_name | Aucun équivalent |
| owner\_screen\_name | Aucun équivalent |
| owner\_id | Aucun équivalent |
