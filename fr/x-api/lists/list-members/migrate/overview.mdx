<div id="comparing-x-apis-list-members-endpoints">
  ## Comparaison des endpoints des membres de liste de l’API X
</div>

Le groupe d’endpoints v2 pour les membres de liste remplacera les endpoints standard v1.1 [GET lists/members](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-members), [GET lists/memberships](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-memberships), [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create) et [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy). Si vous avez du code, des applications ou des outils qui utilisent l’une de ces versions des endpoints des membres de liste et que vous envisagez de migrer vers le nouvel endpoint de l’API X v2, cet ensemble de guides est fait pour vous. 

<div id="list-members-lookup">
  ### Recherche des membres d’une Liste
</div>

Les endpoints v2 de recherche des membres d’une Liste remplaceront les endpoints standard [GET lists/members](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-members), [GET lists/memberships](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-memberships).

Les tableaux suivants comparent les endpoints standard v1.1 et de l’API X v2 pour les Listes :

**Recherche des membres d’une Liste**

|     |     |     |
| :--- | :--- | :--- |
| Description | v1.1 standard | API X v2 |
| Méthodes HTTP prises en charge | `GET` | `GET` |
| Domaine hôte | `https://api.x.com` | `https://api.x.com` |
| Chemin d’endpoint | `/1.1/lists/members.json` | `/2/lists/:id/members` |
| [Authentication](/fr/resources/fundamentals/authentication) | OAuth 1.0a Contexte utilisateur<br /><br />Application uniquement | OAuth 1.0a Contexte utilisateur<br /><br />OAuth 2.0 Code d’autorisation avec PKCE<br /><br />Application uniquement |
| [Limites de débit](/fr/resources/fundamentals/rate-limits) de requêtes par défaut | 900 requêtes par 15 min avec OAuth 1.0a<br /><br />75 requêtes par 15 min avec application uniquement | 900 requêtes par 15 min avec OAuth 1.0a<br /><br />900 requêtes par 15 min avec OAuth 2.0<br /><br />900 requêtes par 15 min avec application uniquement |

**Recherche d’appartenance à une Liste**

|     |     |     |
| :--- | :--- | :--- |
| Description | v1.1 standard | API X v2 |
| Méthodes HTTP prises en charge | `GET` | `GET` |
| Domaine hôte | `https://api.x.com` | `https://api.x.com` |
| Chemin d’endpoint | `/1.1/lists/memberships.json` | `/2/users/:id/list_memberships` |
| [Authentication](/fr/resources/fundamentals/authentication) | OAuth 1.0a Contexte utilisateur<br /><br />Application uniquement | OAuth 1.0a Contexte utilisateur<br /><br />OAuth 2.0 Code d’autorisation avec PKCE<br /><br />Application uniquement |
| [Limites de débit](/fr/resources/fundamentals/rate-limits) de requêtes par défaut | 75 requêtes par 15 min avec OAuth 1.0a<br /><br />75 requêtes par 15 min avec application uniquement | 75 requêtes par 15 min avec OAuth 1.0a<br /><br />75 requêtes par 15 min avec OAuth 2.0<br /><br />75 requêtes par 15 min avec application uniquement |

<div id="manage-list-members">
  ### Gérer les membres de liste
</div>

Les endpoints v2 de gestion des membres de liste remplaceront les endpoints standard [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create), [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy).

Les tableaux suivants comparent les endpoints de liste de la version standard v1.1 et de l’API X v2 :

**Ajouter un membre**

| Description | Standard v1.1 | API X v2 |
| :--- | :--- | :--- |
| Méthodes HTTP prises en charge | POST | POST |
| Domaine hôte | https://api.x.com | https://api.x.com |
| Chemin d’endpoint | /1.1/lists/members/create.json | /2/lists/:id/members |
| [Authentication](/fr/resources/fundamentals/authentication) | Contexte utilisateur OAuth 1.0a | Contexte utilisateur OAuth 1.0a<br /><br />Code d’autorisation OAuth 2.0 avec PKCE |
| [Limites de débit](/fr/resources/fundamentals/rate-limits) par défaut pour les requêtes | Aucune | 300 requêtes par 15 min (par utilisateur) |

**Supprimer un membre**

| Description | Standard v1.1 | API X v2 |
| :--- | :--- | :--- |
| Méthodes HTTP prises en charge | POST | DELETE |
| Domaine hôte | https://api.x.com | https://api.x.com |
| Chemin d’endpoint | /1.1/lists/members/destroy.json | /2/lists/:id/:user\_id |
| [Authentication](/fr/resources/fundamentals/authentication) | Contexte utilisateur OAuth 1.0a | Contexte utilisateur OAuth 1.0a<br /><br />Code d’autorisation OAuth 2.0 avec PKCE |
| [Limites de débit](/fr/resources/fundamentals/rate-limits) par défaut pour les requêtes | Aucune | 300 requêtes par 15 min (par utilisateur) |

Pour accéder aux endpoints de l’API X v2, vous devez [vous inscrire pour obtenir un compte développeur](https://developer.x.com/en/portal/petition/essential/basic-info). Lors de l’authentification, vous devez utiliser des clés et des jetons d’une [application développeur](/fr/resources/fundamentals/developer-apps) située au sein d’un [projet](/fr/resources/fundamentals/projects).

Pour en savoir plus sur l’accès aux endpoints de l’API X v2, consultez notre page [prise en main](/fr/x-api/getting-started/getting-access).
