  ## Comparaison des endpoints de mise en sourdine de l’API X
</div>

<div id="mutes-lookup">
  #### Recherche des comptes mis en sourdine
</div>

Le point de terminaison v2 de recherche des comptes mis en sourdine remplacera les points de terminaison standard [v1.1 GET mutes/users/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/get-mutes-users-ids) et [GET mutes/users/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/get-mutes-users-list).

Les tableaux suivants comparent les points de terminaison de mise en sourdine de la version standard v1.1 et de l’API X v2 :
 

| Description | Standard v1.1 | API X v2 |
| :--- | :--- | :--- |
| Méthodes HTTP prises en charge | ******GET****** | ******GET****** |
| Domaine hôte | ******https://api.x.com****** | ******https://api.x.com****** |
| Chemin du point de terminaison | ******/1.1/mutes/users/ids.json******<br /><br />/1.1/mutes/users/list.json | ******/2/users/:id/muting****** |
| [Authentification](/fr/resources/fundamentals/authentication) | OAuth 1.0a en contexte utilisateur | OAuth 1.0a en contexte utilisateur<br /><br />OAuth 2.0 Code d’autorisation avec PKCE |
| [Limites de débit](/fr/resources/fundamentals/rate-limits) par défaut des requêtes | 15 requêtes par 15 min (par utilisateur) | 15 requêtes par 15 min (par utilisateur) |
| Formats de données | Format Standard v1.1 | [Format API X v2](/fr/x-api/fundamentals/data-dictionary) (déterminé par les paramètres de requête fields et expansions, non rétrocompatible avec les formats v1.1)<br /><br />Pour en savoir plus sur la migration du format Standard v1.1 vers le format API X v2, consultez notre [guide de migration des formats de données](/fr/x-api/migrate/data-format-migration). |
| Nécessite l’utilisation d’identifiants d’une [application développeur](/fr/resources/fundamentals/developer-apps) associée à un [projet](/fr/resources/fundamentals/projects) |     | ✔️  |

<div id="manage-mutes">
  #### Gérer les mises en sourdine
</div>

Les endpoints v2 de gestion des mises en sourdine remplaceront les endpoints standard v1.1 [POST mutes/users/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-create) et [POST mutes/users/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-destroy).

Les tableaux suivants comparent les endpoints de mise en sourdine standard v1.1 et de l’API X v2 :

**Mettre un utilisateur en sourdine**

| Description | Standard v1.1 | API X v2 |
| :--- | :--- | :--- |
| Méthodes HTTP prises en charge | POST | POST |
| Domaine hôte | https://api.x.com | https://api.x.com |
| Chemin d’endpoint | /1.1/mutes/users/create.json | /2/users/:id/muting |
| [Authentification](/fr/resources/fundamentals/authentication) | Contexte utilisateur OAuth 1.0a | Contexte utilisateur OAuth 1.0a<br /><br />OAuth 2.0 Authorization Code avec PKCE |
| [Limites de taux](/fr/resources/fundamentals/rate-limits) de requêtes par défaut | 50 requêtes par 15 min | 50 requêtes par 15 min |
| Nécessite l’utilisation d’identifiants provenant d’une [application développeur](/fr/resources/fundamentals/developer-apps) associée à un [projet](/fr/resources/fundamentals/projects) |     | ✔️  |

**Annuler la mise en sourdine d’un utilisateur**

Les tableaux suivants comparent les endpoints d’annulation de mise en sourdine standard v1.1 et de l’API X v2 :

| Description | Standard v1.1 | API X v2 |
| :--- | :--- | :--- |
| Méthodes HTTP prises en charge | POST | DELETE |
| Domaine hôte | https://api.x.com | https://api.x.com |
| Chemin d’endpoint | /1.1/mutes/users/destroy.json | /2/users/:source\_user\_id/muting/:target\_user\_id |
| [Authentification](/fr/resources/fundamentals/authentication) | Contexte utilisateur OAuth 1.0a | Contexte utilisateur OAuth 1.0a<br /><br />OAuth 2.0 Authorization Code avec PKCE |
| [Limites de taux](/fr/resources/fundamentals/rate-limits) de requêtes par défaut | 50 requêtes par 15 min | 50 requêtes par 15 min |
| Nécessite l’utilisation d’identifiants provenant d’une [application développeur](/fr/resources/fundamentals/developer-apps) associée à un [projet](/fr/resources/fundamentals/projects) |     | ✔️  |
