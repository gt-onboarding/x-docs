Vous pouvez authentifier votre requête vers tous les endpoints de recherche d’utilisateurs, à l’exception de celui pour l’utilisateur authentifié, avec [OAuth 1.0a User Context](/fr/resources/fundamentals/authentication), [App only](/fr/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) ou [OAuth 2.0 Authorization Code with PKCE](fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). Cependant, si vous souhaitez accéder à des mesures privées ou à des données d’utilisateurs privés, vous devrez utiliser OAuth 1.0a User Context ou OAuth 2.0 Authorization Code with PKCE et transmettre, avec votre requête, le jeton d’accès de l’utilisateur authentifié.

Cet endpoint est couramment utilisé pour obtenir des informations à jour sur un utilisateur, vérifier l’existence d’un utilisateur ou mettre à jour vos données stockées à la suite d’un événement de conformité.

<div id="authenticated-user-lookup">
  ### Recherche de l’utilisateur authentifié
</div>

Si vous vous authentifiez au nom d’autres utilisateurs, il est essentiel de pouvoir identifier pour quel utilisateur vous pouvez effectuer une requête.

Cet endpoint nécessite d’utiliser [OAuth 1.0a User Context](/fr/resources/fundamentals/authentication) ou [OAuth 2.0 Authorization Code with PKCE](fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). Il renvoie des informations sur l’utilisateur autorisé, c’est-à-dire l’utilisateur associé aux jetons d’accès utilisateur que vous transmettez avec la requête.

Vous pouvez accéder à cet endpoint via la méthode GET. Une limite de 75 requêtes par utilisateur, par tranche de 15 minutes, s’applique à cet endpoint.

<Note>
  **Configuration du compte**

  Pour accéder à ces endpoints, vous aurez besoin de :

  - D’un [compte développeur](https://developer.x.com/en/portal/petition/essential/basic-info) approuvé.
  - De vous authentifier à l’aide des clés et des jetons d’une [application développeur](/fr/resources/fundamentals/developer-apps) qui se trouve au sein d’un [projet](/fr/resources/fundamentals/projects). 

  En savoir plus sur l’accès aux endpoints de l’API X v2 dans notre [guide de démarrage](/fr/x-api/getting-started/getting-access).
</Note>
