<div id="blocks-lookup">
  ### Consultation des blocages
</div>

L’endpoint GET de consultation des blocages vous permet de voir quels comptes vous avez bloqués au nom d’un utilisateur autorisé. Cet endpoint est soumis à une limite de 15 requêtes par tranche de 15 minutes et par utilisateur.

Parce que vous demandez des informations privées via la consultation des blocages, et au nom d’un utilisateur disposant des droits de gestion des blocages, vous devez authentifier ces endpoints avec [OAuth 1.0a en contexte utilisateur](/fr/resources/fundamentals/authentication) ou [OAuth 2.0 Code d’autorisation avec PKCE](fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), et utiliser les jetons d’accès utilisateur associés à un utilisateur ayant autorisé votre application. Ces jetons peuvent être générés via le [flux OAuth à 3 volets](/fr/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) ou le [flux de concession Code d’autorisation avec PKCE](/fr/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0).

<Note>
  **Configuration du compte**

  Pour accéder à ces endpoints, vous aurez besoin de :

  - D’un [compte développeur](https://developer.x.com/en/portal/petition/essential/basic-info) approuvé.
  - D’authentifier en utilisant les clés et jetons d’une [application développeur](/fr/resources/fundamentals/developer-apps) située au sein d’un [projet](/fr/resources/fundamentals/projects).

  En savoir plus sur l’accès aux endpoints de l’API X v2 dans notre [guide de démarrage](/fr/x-api/getting-started/getting-access).
</Note>
