  ## Comparaison des endpoints de suivi de l’API X
</div>

<div id="follows-lookup">
  ### Recherche des relations de suivi
</div>

Les endpoints de recherche des relations de suivi v2 remplaceront les endpoints standard v1.1 [followers/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-followers-ids), v1.1 [followers/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-followers-list), v1.1 [friends/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-friends-ids) et v1.1 [friends/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-friends-list).

Les tableaux suivants comparent les différents types d’endpoints pour la recherche des relations de suivi :

|     |     |     |
| :--- | :--- | :--- |
| **Description** | **Standard v1.1** | **API X v2** |
| Méthodes HTTP prises en charge | `GET` | `GET` |
| Domaine hôte | `https://api.x.com` | `https://api.x.com` |
| Chemin d’endpoint | /1.1/friends/ids.json<br /><br />/1.1/friends/list.json<br /><br />/1.1/followers/ids.json<br /><br />/1.1/followers/list.json | /2/users/:id/following<br /><br />/2/users/:id/followers |
| [Authentification](/fr/resources/fundamentals/authentication) | OAuth 1.0a User Context<br /><br />App only | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code with PKCE<br /><br />App only |
| [Limites de débit](/fr/resources/fundamentals/rate-limits) de requête par défaut | 15 requêtes par 15 min (par utilisateur)<br /><br />15 requêtes par 15 min (par application) | 15 requêtes par 15 min (par utilisateur)<br /><br />15 requêtes par 15 min (par application) |
| Nombre maximal d’utilisateurs par réponse | GET friends/id et GET followers/id renvoient jusqu’à 5 000 identifiants d’utilisateurs par page.<br /><br />  <br /><br />GET friends/list et GET followers/list renvoient jusqu’à 200 objets utilisateur par page. | 1 000 objets utilisateur par page |
| Pagination | Un jeton est renvoyé dans un champ next\_cursor, qui peut ensuite être transmis comme valeur au paramètre cursor pour renvoyer la page suivante de résultats. | Un jeton est renvoyé dans un champ next\_token, qui peut ensuite être transmis comme valeur au paramètre token pour renvoyer la page suivante de résultats.<br /><br />La charge utile v2 fournit également un champ previous\_token, qui peut aussi être transmis avec le paramètre pagination\_token pour renvoyer la page précédente de résultats. |
| Format JSON | Format Standard v1.1 | [Format API X v2](/fr/x-api/fundamentals/data-dictionary) (déterminé par les paramètres de requête fields et expansions, non rétrocompatible avec les formats v1.1)<br /><br />Pour en savoir plus sur la migration du format Standard v1.1 vers le format API X v2, veuillez consulter notre [guide de migration des formats de données](/fr/x-api/migrate/data-format-migration). |
| Prise en charge de la sélection des [champs](/fr/x-api/fundamentals/data-dictionary) à renvoyer dans la charge utile |     | ✔   |
| Prise en charge des champs [annotations de Post](/fr/x-api/fundamentals/post-annotations) |     | ✔   |
| Prise en charge de la demande de nouveaux champs de [mesures](/fr/x-api/fundamentals/metrics) |     | ✔   |
| Prise en charge du champ [conversation\_id](/fr/x-api/fundamentals/conversation-id) |     | ✔   |
| Nécessite l’utilisation d’identifiants d’une [application développeur](/fr/resources/fundamentals/developer-apps) associée à un [projet](/fr/resources/fundamentals/projects) |     | ✔   |

<div id="manage-follows">
  #### Gérer les abonnements
</div>

Les endpoints de gestion des abonnements de la v2 remplaceront les endpoints standard de la v1.1 [POST friendships/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-create) et [POST friendships/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-destroy).

Les tableaux suivants comparent les endpoints standard de la v1.1 et de l’API X v2 pour la création d’un abonnement :

<div id="follow-a-user">
  #### Suivre un utilisateur
</div>

| Description | Standard v1.1 | API X v2 |
| :--- | :--- | :--- |
| Méthodes HTTP prises en charge | POST | POST |
| Domaine hôte | https://api.x.com | https://api.x.com |
| Chemin d’endpoint | /1.1/friendships/create.json | /2/users/:id/following |
| [Authentification](/fr/resources/fundamentals/authentication) | Contexte utilisateur OAuth 1.0a | Contexte utilisateur OAuth 1.0a<br /><br />OAuth 2.0 avec PKCE (code d’autorisation) |
| [Limites de débit](/fr/resources/fundamentals/rate-limits) de requêtes par défaut | 50 requêtes par 15 min | 50 requêtes par 15 min |
| Opérations quotidiennes maximales par utilisateur | 400 | 400 |
| Opérations quotidiennes maximales par application | 1000 | 1000 |
| Nécessite l’utilisation d’identifiants d’une [application développeur](/fr/resources/fundamentals/developer-apps) associée à un [projet](/fr/resources/fundamentals/projects) |     | ✔️  |

<div id="unfollow-a-user">
  #### Se désabonner d’un utilisateur
</div>

Les tableaux suivants comparent les endpoints de suppression de suivi de la version Standard v1.1 et de l’API X v2 :

| Description | Standard v1.1 | API X v2 |
| :--- | :--- | :--- |
| Méthodes HTTP prises en charge | POST | DELETE |
| Domaine hôte | https://api.x.com | https://api.x.com |
| Chemin d’endpoint | /1.1/friendships/destroy.json | /2/users/:source\_user\_id/following/:target\_user\_id |
| [Authentication](/fr/resources/fundamentals/authentication) | Contexte utilisateur OAuth 1.0a | Contexte utilisateur OAuth 1.0a<br /><br />OAuth 2.0 avec PKCE |
| [Limites de débit](/fr/resources/fundamentals/rate-limits) de requêtes par défaut | 15 requêtes par 15 min (par utilisateur) | 50 requêtes par 15 min (par utilisateur) |
| Opérations quotidiennes maximales par application | Aucune | 500 |
| Nécessite l’utilisation d’identifiants d’une [application développeur](/fr/resources/fundamentals/developer-apps) associée à un [projet](/fr/resources/fundamentals/projects) |     | ✔️  |
