<div id="manage-follows-standard-v11-compared-to-x-api-v2">
  #### Gérer les abonnements : v1.1 Standard comparée à l’API X v2
</div>

Si vous avez travaillé avec les endpoints v1.1 Standard [POST friendships/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-create) et [POST friendships/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-destroy), ce guide a pour objectif de vous aider à comprendre les similitudes et les différences entre les endpoints de gestion des abonnements de la v1.1 Standard et de l’API X v2.

- **Similarités**
  - Contexte utilisateur OAuth 1.0a
- **Différences**
  - URL des endpoints
  - Exigences pour l’application et le projet
  - Méthodes HTTP
  - Paramètres de requête

     

<div id="similarities">
  #### Similarités
</div>

**Méthode d’authentification OAuth 1.0a en contexte utilisateur**

Les deux versions de l’endpoint prennent en charge [OAuth 1.0a User Context](/fr/resources/fundamentals/authentication#oauth-1-0a-2). Par conséquent, si vous utilisiez auparavant l’un des endpoints standard v1.1 pour gérer les suivis, vous pouvez continuer à utiliser la même méthode d’authentification en migrant vers la version de l’API X v2. 

<div id="differences">
  #### Différences
</div>

**URLs d’endpoint**

- Endpoints Standard v1.1 :
  - POST https://api.x.com/1.1/friendships/create.json
    (suivre un utilisateur)
  - POST https://api.x.com/1.1/friendships/destroy.json
    (ne plus suivre un utilisateur)
- Endpoint de l’API X v2 :
  - POST https://api.x.com/2/users/:id/following
    (suivre un utilisateur)
  - DELETE https://api.x.com/2/users/:source\_user\_id/following/:target\_user\_id
    (ne plus suivre un utilisateur) 

**Exigences pour les applications et les projets**

Les endpoints de l’API X v2 requièrent l’utilisation d’identifiants issus d’une [application développeur](/fr/resources/fundamentals/developer-apps) associée à un [projet](/fr/resources/fundamentals/projects) lors de l’authentification de vos requêtes. Tous les endpoints de l’API X v1.1 peuvent utiliser des identifiants provenant d’applications autonomes ou d’applications associées à un projet.

**Paramètres de requête**

Les paramètres de requête Standard v1.1 suivants ont des équivalents dans l’API X v2 :

| Standard v1.1 | API X v2 |
| :--- | :--- |
| Aucun équivalent | id (POST), source\_user\_id (DELETE) |
| user\_id | target\_user\_id |
| screen\_name | Aucun équivalent |

Veuillez noter que les paramètres Standard v1.1 sont transmis en tant que paramètres de requête, tandis que les paramètres de l’API X v2 sont transmis en tant que paramètres de corps (pour l’endpoint POST) ou de chemin (pour l’endpoint DELETE).

En outre, les paramètres v2 id et source\_user\_id ne sont pas requis lors de l’utilisation des endpoints Standard v1.1, car les jetons d’accès transmis avec OAuth 1.0a en contexte utilisateur permettaient de déduire quel utilisateur initiait l’action suivre/ne plus suivre.
