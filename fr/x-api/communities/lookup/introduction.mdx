<Note>
  **Configuration du compte**

  Pour accéder à ces endpoints, vous aurez besoin de :

  - D’un [compte développeur](https://developer.x.com/en/portal/petition/essential/basic-info) approuvé.
  - De vous authentifier à l’aide des clés et jetons d’une [application développeur](/fr/resources/fundamentals/developer-apps) située au sein d’un [projet](/fr/resources/fundamentals/projects).

</Note>

**Premiers pas**

<div id="authentication">
  ### Authentification
</div>

Vous pouvez authentifier cet endpoint avec l’un des mécanismes suivants : [OAuth 1.0a en contexte utilisateur](/fr/resources/fundamentals/authentication), \[OAuth 2.0 App-Only]\(https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), ou [OAuth 2.0 avec code d’autorisation et PKCE](fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).
 

<div id="making-a-request">
  ### Effectuer une requête
</div>

Vous pouvez appeler le point de terminaison Communities Lookup en fournissant un ID comme indiqué ci-dessous (en utilisant l’ID `1758747817642700922`) :

```
      curl --location 'https://api.x.com/2/communities/1758747817642700922' --header 'Authorization: ••••••'
```

Si la requête réussit, vous devriez voir la réponse JSON suivante :

```
{
    "data": {
        "access": "Public",
        "description": "Welcome to the Anime Community! Where anime fans gather to share their favorite shows and discuss everything anime-related.",
        "join_policy": "Open",
        "name": "Anime Community"
    }
}
```

<div id="community-fields">
  ### Champs Community
</div>

Vous pouvez récupérer des données supplémentaires associées à une communauté en spécifiant des Community Fields.

Les Community Fields suivants sont disponibles :

| Valeur du champ | Type | Description |     |
| :--- | :--- | :--- | :--- |
| created\_at | date (ISO 8601) | Date et heure de création de la communauté. |     |
| id  | string | Identifiant unique de la communauté. |     |
| name | string | Nom de la communauté. |     |
| description | string | Texte de la description de la communauté, le cas échéant. |     |
| access | string | Niveau d’accès de la communauté.  <br />  <br />Peut être l’un des suivants :<br /><br />\* `Public`<br />\* `Closed` |     |
| join\_policy | string | Politique d’adhésion de la communauté.  <br />  <br />Peut être l’une des suivantes :<br /><br />\* `Open`<br />\* `RestrictedJoinRequestsDisabled`<br />\* `RestrictedJoinRequestsRequireAdminApproval`<br />\* `RestrictedJoinRequestsRequireModeratorApproval`<br />\* `SuperFollowRequired` |     |
| member\_count | integer | Nombre de membres ayant rejoint la communauté. |     |

<div id="example-request">
  #### Exemple de requête
</div>

Vous pouvez appeler le point de terminaison Communities Lookup avec des champs de communauté supplémentaires, comme ci-dessous (en utilisant l’ID `1758747817642700922`) :

```
curl --location 'https://api.x.com/2/communities/1758747817642700922?community.fields=access,created_at,description,id,join_policy,member_count,name' --header 'Authorization: ••••••'
```

**Exemple de réponse**

Si la requête aboutit, vous verrez la réponse JSON suivante :

```
{
    "data": {
        "description": "Welcome to the Anime Community! Where anime fans gather to share their favorite shows and discuss everything anime-related.",
        "access": "Public",
        "member_count": 40369,
        "created_at": "2024-02-17T06:58:50.000Z",
        "join_policy": "Open",
        "name": "Anime Community",
        "id": "Q29tbXVuaXR5OjE3NTg3NDc4MTc2NDI3MDA5MjI="
    }
}
```
