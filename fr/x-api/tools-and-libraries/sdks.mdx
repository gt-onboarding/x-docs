<div id="introduction">
  ## Introduction
</div>

Un kit de développement logiciel (SDK) est un ensemble d’outils et de programmes conçus pour une plateforme ou une API spécifique. L’objectif d’un SDK est de créer ou d’étendre les fonctionnalités d’applications en fournissant des bibliothèques ou des bases de code que les développeurs peuvent intégrer dans leurs applications, efficacement et avec un minimum de code. Cela accélère considérablement le développement, faisant gagner du temps, de l’argent et des efforts.

> La X Developer Platform propose désormais deux SDK officiels pour celles et ceux qui développent en TypeScript/JavaScript et en Java. Ils permettent de créer plus efficacement en évitant d’implémenter manuellement la complexité de l’API X v2, en tirant parti des fonctions prédéfinies pour tous les endpoints v2 disponibles et en simplifiant l’authentification. Conçus et maintenus par l’équipe de la Developer Platform, ils seront toujours à jour avec les futures versions de l’API X v2.

Comme ces SDK encapsulent l’API X, vous devez disposer d’un [compte développeur](https://developer.x.com/en/portal/dashboard) pour authentifier les requêtes à l’aide des identifiants d’une [application développeur](/fr/resources/fundamentals/developer-apps), située au sein d’un [projet](/fr/resources/fundamentals/projects).

<div id="client-basics">
  #### Notions de base du client
</div>

Importez les classes (Java) et le package (TypeScript) en haut d’un fichier source pour accéder aux clients d’authentification et de bibliothèque. Pour utiliser les méthodes du client de bibliothèque, vous devez fournir des identifiants d’authentification : soit un Bearer Token (application uniquement), soit un client\_id/client\_secret dans le cadre d’une authentification avec contexte utilisateur OAuth 2.0.

Voici des exemples de ce à quoi cela ressemble :

<Tabs>
  <Tab title="Java">
    ```java
    // Import classes:
    import com.twitter.clientlib.ApiClient;
    import com.twitter.clientlib.ApiException;
    import com.twitter.clientlib.Configuration;
    import com.twitter.clientlib.auth.*;
    import com.twitter.clientlib.model.*;
    import com.twitter.clientlib.TwitterCredentialsBearer;
    import com.twitter.clientlib.api.TwitterApi;

    // Instantiate library client
    TwitterApi apiInstance = new TwitterApi();

    // Instantiate auth credentials (App-only example)
    TwitterCredentialsBearer credentials = new TwitterCredentialsBearer(System.getenv("APP-ONLY-ACCESS-TOKEN"));

    // Pass credentials to library client
    apiInstance.setTwitterCredentials(credentials);
    ```
  </Tab>

  <Tab title="TypeScript">
    ```typescript
    //Import package
    import { Client, auth } from "twitter-api-sdk";

    // Initialize auth client first
    const authClient = new auth.OAuth2User({
     client_id: process.env.CLIENT_ID as string,
     client_secret: process.env.CLIENT_SECRET as string,
     callback: "YOUR-CALLBACK",
     scopes: ["tweet.read", "users.read", "offline.access"],
    });

     // Pass auth credentials to the library client
     const twitterClient = new Client(authClient);
    ```
  </Tab>
</Tabs>

<div id="authentication-flow">
  #### Flux d’authentification
</div>

Si vous utilisez l’option « application only » pour authentifier les SDK, vous n’avez qu’à fournir le jeton et le client de la bibliothèque sera immédiatement prêt à utiliser les méthodes d’endpoint. Notez que les jetons « application only » ne peuvent pas être utilisés sur des endpoints qui requièrent une authentification avec contexte utilisateur.

L’authentification OAuth 2.0 avec contexte utilisateur nécessite quelques étapes supplémentaires après la création du client d’authentification.

- Générer l’URL d’autorisation
- Autoriser l’application depuis l’URL d’autorisation
- Rediriger vers le callback (cela doit correspondre à l’URL de callback définie sur la page des paramètres d’authentification dans le Developer Portal)
- Analyser le code verifier pour l’échanger contre un jeton d’accès

Les SDK fournissent des méthodes sur le client d’authentification qui simplifient ces étapes. Pour un exemple complet de requête s’authentifiant avec OAuth 2.0 en contexte utilisateur, consultez les dépôts GitHub.

- [TypeScript](https://github.com/xdevplatform/twitter-api-typescript-sdk/blob/main/examples/oauth2-callback_pkce_s256.ts)

- [Java](https://github.com/xdevplatform/twitter-api-java-sdk/blob/main/examples/src/main/java/com/twitter/clientlib/auth/OAuth20GetAccessToken.java)

<div id="endpoint-methods">
  #### Méthodes d’endpoint
</div>

Les méthodes exposées par le client de la bibliothèque sont clairement nommées pour correspondre à chaque endpoint, et tous les paramètres sont passés sous forme d’arguments.
Voici un exemple de recherche de post par ID :

<Tabs>
  <Tab title="Java">
    ```java
    String id = "1511757922354663425"; // String | Un ID de Tweet unique.
    Set<String> expansions = new HashSet<>(Arrays.asList("author_id")); // Set<String> | Liste de champs à étendre, séparés par des virgules.
    Set<String> tweetFields = new HashSet<>(Arrays.asList("created_at", "lang", "context_annotations")); // Set<String> | Liste de champs de Tweet à afficher, séparés par des virgules.
    Set<String> userFields = new HashSet<>(Arrays.asList("created_at", "description", "name")); // Set<String> | Liste de champs d’utilisateur à afficher, séparés par des virgules.

    try {
     SingleTweetLookupResponse result = apiInstance.tweets().findTweetById(id, expansions, tweetFields, userFields, null, null, null);
     System.out.println(result);
    } catch (ApiException e) {
     System.err.println("Exception lors de l’appel à TweetsApi#findTweetById");
     System.err.println("Code de statut : " + e.getCode());
     System.err.println("Raison : " + e.getResponseBody());
     System.err.println("En-têtes de la réponse : " + e.getResponseHeaders());
     e.printStackTrace();
    }

    ```
  </Tab>

  <Tab title="TypeScript">
    ```typescript
    const lookupTweetById = await client.tweets.findTweetById(
      // ID du Tweet
      "1511757922354663425",
      {
        // Paramètres optionnels
        expansions: ["author_id"],
        "user.fields": ["created_at", "description", "name"],
      }
    );
    ```
  </Tab>
</Tabs>
