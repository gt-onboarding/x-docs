<div id="step-one-sign-up-for-a-developer-account">
  ### Première étape : Créer un compte développeur
</div>

La création d’un compte développeur est rapide et simple ! Cliquez sur le bouton ci-dessous, répondez à quelques questions, et vous pourrez commencer à explorer et à développer avec l’API X v2 grâce à l’[accès Basic](https://developer.x.com/en/portal/products/basic).

Vous créerez ensuite un Projet et une application développeur associée lors du processus d’intégration, ce qui vous fournira un jeu d’identifiants que vous utiliserez pour authentifier toutes les requêtes adressées à l’API.

[S’inscrire](https://developer.x.com/en/portal/products/basic)

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### Deuxième étape : Enregistrez la clé et les jetons de votre application et gardez-les en sécurité
</div>

Une fois que vous disposez de l’accès et avez créé un Projet et une Application, vous pourrez trouver ou générer les identifiants suivants dans votre application développeur :

- **[Clé API et secret](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret) :** En pratique, il s’agit du nom d’utilisateur et du mot de passe de votre application. Vous les utiliserez pour authentifier les requêtes nécessitant le [contexte utilisateur OAuth 1.0a](/fr/resources/fundamentals/authentication), ou pour générer d’autres jetons tels que des jetons d’accès utilisateur ou un jeton d’accès d’application.

- **[Jeton d’accès et secret](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens) :** En général, les jetons d’accès représentent l’utilisateur pour le compte duquel vous effectuez la requête. Ceux que vous pouvez générer via le portail développeur représentent l’utilisateur propriétaire de l’application. Vous les utiliserez pour authentifier les requêtes nécessitant le [contexte utilisateur OAuth 1.0a](/fr/resources/fundamentals/authentication). Si vous souhaitez effectuer des requêtes au nom d’un autre utilisateur, vous devrez utiliser le flux OAuth à 3 volets afin qu’il vous autorise.

- **[ID client et secret client](/fr/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) :** Ces identifiants sont utilisés pour obtenir un jeton d’accès utilisateur avec l’authentification OAuth 2.0. À l’instar d’OAuth 1.0a, les jetons d’accès utilisateur servent à authentifier des requêtes qui fournissent des informations privées de compte ou réalisent des actions au nom d’un autre compte, mais avec une granularité de portée plus fine pour mieux contrôler les accès de l’application cliente à l’utilisateur.

- **[Jeton d’accès « App only »](/fr/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) :** Vous utiliserez ce jeton pour effectuer des requêtes vers des endpoints qui renvoient des informations publiquement disponibles sur X.

Étant donné que ces clés et jetons n’expirent pas sauf s’ils sont régénérés, nous vous recommandons de les enregistrer dans un emplacement sécurisé, comme un gestionnaire de mots de passe, dès que vous avez reçu vos identifiants.

> **Veuillez noter : vos clés et jetons ne s’afficheront qu’une seule fois dans le [portail développeur](/fr/resources/fundamentals/developer-portal). Il est donc important de stocker ces identifiants dans votre gestionnaire de mots de passe dès leur génération.**
>
> Si vous égarez ou oubliez vos clés et jetons, vous devrez les régénérer, ce qui crée de nouveaux identifiants et invalide les anciens. Vous devrez alors mettre à jour toutes les intégrations que vous avez configurées avec vos identifiants précédents.
>
> En savoir plus sur nos [bonnes pratiques d’authentification](/fr/resources/fundamentals/authentication#authentication-best-practices).
