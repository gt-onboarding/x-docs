Ce guide vous présente les étapes à suivre pour effectuer votre première requête. C’est une excellente ressource une fois que vous avez créé un compte X.

Si vous souhaitez utiliser des extraits de code, des guides plus techniques, ou un outil graphique comme Postman, consultez les guides suivants pour effectuer votre première requête :

- [Guide de démarrage rapide pour effectuer votre première requête vers l’endpoint de publication](/fr/x-api/posts/manage-tweets/quickstart)
- [Premiers pas avec Postman](/fr/tutorials/postman-getting-started) - idéal pour les débutants
- [Exemples de code de l’API X v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code)
- [Référence de l’API de consultation de posts](/fr/x-api/posts/lookup/introduction)

Ce guide suppose que vous avez récupéré votre [clé et secret d’API](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret), votre [jeton d’accès utilisateur et son secret](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens), votre [jeton d’accès d’application](/fr/resources/fundamentals/authentication#bearer-token-also-known-as-app-only), et que vous les avez stockés en lieu sûr. Vous pouvez apprendre à le faire en suivant les étapes du guide [Obtenir l’accès à l’API X](/fr/x-api/getting-started/getting-access).

<div id="step-1-identify-which-endpoint-you-would-like-to-use">
  ### Étape 1. Identifiez l’endpoint que vous souhaitez utiliser
</div>

L’API X vous permet d’exécuter, par programmation, diverses actions que vous pouvez également effectuer sur le site web ou l’application mobile de X.

Nous proposons une liste complète des endpoints disponibles dans notre [Index de référence de l’API](/fr/x-api/introduction), mais nous vous recommandons, pour plus de simplicité, de vous en tenir à l’un des suivants :

- [Gérer les posts > Publier un Tweet](/fr/x-api/posts/manage-tweets/introduction)
- [Rechercher des posts](/fr/x-api/posts/search/introduction)
- [Recherche d’utilisateurs](/fr/x-api/users/lookup/introduction)

<div id="step-2-choose-a-tool-to-make-your-request">
  ### Étape 2. Choisissez un outil pour effectuer votre requête
</div>

Si certaines requêtes sont simples, d’autres sont plus difficiles à concevoir. C’est pourquoi l’excellente communauté de développeurs a créé des outils pour masquer une partie de la complexité.

Voici quelques outils recommandés et des indications sur leur utilisation :

<div id="postman">
  #### Postman
</div>

Postman est un outil visuel que vous pouvez utiliser pour envoyer des requêtes vers des endpoints REST. Nous avons créé d’excellentes ressources sur Postman pour vous aider à démarrer et à explorer les différents endpoints disponibles via l’API X.

Nous vous recommandons de lire notre tutoriel ["Prise en main de Postman"](/fr/tutorials/postman-getting-started) pour apprendre à ajouter vos clés et jetons, et effectuer votre première requête.

Nous avons également publié un guide de démarrage rapide pour chacun de nos endpoints de l’API X v2, dont la plupart utilisent Postman. Vous pouvez retrouver ces guides dans la section de chaque endpoint ; en voici quelques exemples :

- Quick start: Post a Tweet
- Quick start: Search for Posts
- Quick start: Lookup a user

Veuillez noter que vous ne pouvez pas envoyer de requêtes vers des endpoints de streaming avec Postman. Veuillez consulter le guide de démarrage rapide du [flux filtré](/fr/x-api/posts/filtered-stream/quickstart) ou du [flux échantillonné à 1 %](/fr/x-api/posts/sample-stream) pour apprendre à utiliser ces endpoints.

Si vous préférez un outil graphique plus simple, vous pouvez également envisager d’utiliser [Insomnia](https://insomnia.rest).

<div id="sample-code">
  #### Exemple de code
</div>

Si vous souhaitez utiliser un peu de code simple pour effectuer votre requête, nous avons rassemblé des exemples dans plusieurs langages pour chacun de nos endpoints de l’API X v2.

Vous pouvez retrouver ces exemples dans notre dépôt GitHub, [X-API-v2-sample-code](https://github.com/xdevplatform/Twitter-API-v2-sample-code), qui contient également un fichier README expliquant comment configurer vos identifiants afin d’exécuter correctement les requêtes.

Par exemple, voici un exemple cURL pour l’endpoint de recherche d’utilisateur. Pour utiliser cette requête, remplacez simplement $ACCESS\_TOKEN et $USERNAME par votre [jeton d’accès d’app](/fr/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) et votre handle X. Ensuite, copiez-collez ce code dans votre outil en ligne de commande et appuyez sur « Entrée ».

```bash
curl "https://api.x.com/2/users/by/username/$USERNAME" -H "Authorization: Bearer $ACCESS_TOKEN"
```

  #### Bibliothèques
</div>

L’extraordinaire communauté des développeurs X a également créé des bibliothèques dans une variété de langages de programmation qui peuvent être utilisées pour envoyer des requêtes à l’API X.

> Remarque : si vous rencontrez des problèmes, consultez la documentation pour le point de terminaison visé, notre [section d’assistance](https://developer.x.com/en/support), ou sollicitez la communauté sur nos forums. Nous vous aiderons à réussir votre requête !

<div id="step-3-review-the-response">
  ### Étape 3. Examiner la réponse
</div>

Une fois votre requête exécutée avec succès, vous recevrez un payload contenant des métadonnées liées à cette requête.

Si vous avez utilisé un endpoint avec la méthode HTTP GET, vous recevrez des métadonnées relatives à la ressource (Post, utilisateur, Liste, Space, etc.) visée par la requête, au format JSON. Passez en revue les différents champs renvoyés et vérifiez si les informations demandées correspondent au contenu sur X.

Si vous avez utilisé un endpoint avec une méthode HTTP POST, PUT ou DELETE, vous avez effectué une action sur X. Rendez-vous sur X.com ou dans l’application mobile et vérifiez que cette action est bien visible.

<div id="step-4-adjust-your-request-using-parameters">
  ### Étape 4. Ajustez votre requête à l’aide de paramètres
</div>

Chaque endpoint propose un ensemble distinct de paramètres que vous pouvez utiliser pour modifier votre requête. Par exemple, vous pouvez demander des champs de métadonnées supplémentaires lorsque vous utilisez des endpoints GET avec les paramètres fields et expansions. Vous pouvez également tester divers outils de filtrage avec des endpoints tels que [recherche de Posts](/fr/x-api/posts/search/introduction), [comptage de Posts](/fr/x-api/posts/counts/introduction) et [flux filtré](/fr/x-api/posts/filtered-stream/introduction) afin de restreindre les données reçues aux seuls Posts qui vous intéressent.

Vous trouverez la liste complète des paramètres de requête et des champs dans la référence de l’API pour l’endpoint que vous utilisez, ainsi qu’une foule d’autres informations utiles pour l’intégration dans nos guides d’intégration et nos pages consacrées aux fondamentaux.

Vous pouvez en savoir plus sur l’ensemble des ressources pédagogiques que nous mettons à votre disposition dans notre guide [Ressources importantes](/fr/x-api/getting-started/important-resources).
