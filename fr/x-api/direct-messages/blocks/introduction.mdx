Les points de terminaison de gestion des blocages de MD permettent de bloquer ou de débloquer un compte donné au nom d’un utilisateur authentifié. Pour ces points de terminaison, deux méthodes POST sont disponibles :

- **/2/users/:id/dm/block** : permet de bloquer un compte
- **/2/users/:id/dm/unblock** : permet de débloquer un compte

<div id="getting-started">
  ### Bien démarrer
</div>

<div id="authentication">
  ### Authentification
</div>

Puisque vous effectuez des requêtes au nom d’un utilisateur, vous devez authentifier ces endpoints soit avec [OAuth 1.0a en contexte utilisateur](/fr/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow), soit avec [OAuth 2.0 « Authorization Code » avec PKCE](/fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), et utiliser les jetons d’accès associés à l’utilisateur pour lequel vous effectuez la requête. Vous pouvez générer ce jeton d’accès utilisateur à l’aide du [flux OAuth à 3 étapes](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens) (OAuth 1.0a) ou du [flux d’autorisation « Authorization Code » avec PKCE](/fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) (OAuth 2.0).

<div id="making-a-request">
  ### Effectuer une requête
</div>

Blocage

Une fois qu’un utilisateur s’est authentifié auprès de votre application, vous pouvez appeler le point de terminaison Block pour le compte de l’utilisateur, comme indiqué ci-dessous :

```
      curl --request POST 'https://api.x.com/2/users/:id/dm/block' --header 'Authorization: ••••••'
```

Si la requête aboutit, vous verrez la réponse JSON suivante :

```
{
   "data": {
       "blocked": true
   }
}
```

**Déblocage**

Une fois qu’un utilisateur s’est authentifié auprès de votre application, vous pouvez appeler le point de terminaison Unblock pour le compte de l’utilisateur, comme indiqué ci-dessous :

```
      curl --request POST 'https://api.x.com/2/users/:id/dm/unblock' --header 'Authorization: ••••••'
```

Si la requête aboutit, vous verrez la réponse JSON suivante :

```
{
   "data": {
       "blocked": false
   }
}
```
