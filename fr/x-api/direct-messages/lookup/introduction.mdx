Cette version initiale de la recherche de Messages privés inclut trois méthodes GET :

- **GET /2/dm\_conversations/with/:participant\_id/dm\_events** - Récupère les événements de Messages privés associés à une conversation en tête-à-tête. Le paramètre de chemin :participant\_id correspond à l’ID utilisateur du compte qui converse avec l’utilisateur authentifié effectuant cette requête.
- **GET /2/dm\_conversations/:dm\_conversation\_id/dm\_events** - Récupère les événements de Messages privés associés à un ID de conversation spécifique, tel qu’indiqué par le paramètre de chemin :dm\_conversation\_id.
- **GET /2/dm\_events** - Récupère les événements de Messages privés associés à un utilisateur, pour les conversations en tête-à-tête et de groupe. Des événements jusqu’à 30 jours en arrière sont disponibles.

Notez que les identifiants des événements de Messages privés sont communs entre les versions v1.1 et v2 (ainsi que l’app X), de sorte que la méthode v1.1 permettant de lister un événement unique peut être utilisée avec ces nouveaux endpoints v2. Notez également que les API Enterprise et Premium Account Activity prennent en charge les messages v2 en tête-à-tête, mais pas encore les conversations de groupe.

Avec cette version, trois types d’événements sont pris en charge, et ces endpoints acceptent des paramètres de requête pour les filtrer :

- **MessageCreate** - Un message a été créé.
- **ParticipantsJoin** - Un nouveau participant a rejoint une conversation.
- **ParticipantsLeave** - Un participant a quitté une conversation.

Une limite de débit de 300 requêtes par utilisateur et par tranche de 15 minutes s’applique aux méthodes GET. Cette limite est partagée entre ces endpoints GET.

Puisque vous effectuez des requêtes au nom d’un utilisateur avec les endpoints v2 de Messages privés, vous devez vous authentifier soit avec [OAuth 1.0a User Context](/fr/resources/fundamentals/authentication), soit avec [OAuth 2.0 Authorization Code with PKCE](/fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3), en utilisant des jetons d’accès associés aux utilisateurs ayant autorisé votre application X. Pour générer ces jetons d’accès avec OAuth 1.0a, vous pouvez utiliser le [flux OAuth à 3 acteurs](/fr/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow). Pour générer des jetons d’accès utilisateur avec OAuth 2.0, vous pouvez utiliser le [flux d’autorisation Authorization Code avec PKCE](/fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

**Configuration du compte**

Pour accéder à ces endpoints, vous aurez besoin de :

- D’un [compte développeur](https://developer.x.com/en/portal/petition/essential/basic-info) approuvé.
- De vous authentifier à l’aide des clés et jetons d’une [application développeur](/fr/resources/fundamentals/developer-apps) située au sein d’un [projet](/fr/resources/fundamentals/projects).

Pour en savoir plus sur l’accès aux endpoints de l’API X v2, consultez notre [guide de prise en main](/fr/x-api/getting-started/getting-access).
