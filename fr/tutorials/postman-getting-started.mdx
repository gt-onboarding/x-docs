<div id="introduction">
  ## Introduction
</div>

Postman est une application de bureau et web qui permet d’effectuer des requêtes à une API depuis une interface graphique. Nous recommandons d’utiliser Postman avec l’API X, l’API publicitaire X et les endpoints Labs lors de l’exploration des fonctionnalités des API, ainsi que pour le diagnostic des problèmes de votre application.

Nous proposons actuellement deux collections Postman :

<div id="prerequisites">
  ### Prérequis
</div>

Avant de commencer à utiliser les collections Postman de X, assurez-vous de disposer des droits d’accès et des identifiants appropriés pour l’outil de la plateforme développeur X que vous avez choisi. Pour en savoir plus sur l’obtention de l’accès, consultez notre [page « Bien démarrer »](/fr/resources/platform-overview).

Une fois que vous disposez d’un [compte développeur](https://developer.x.com/en/portal/petition/essential/basic-info), que vous avez configuré une [application développeur](/fr/resources/fundamentals/developer-apps), que vous possédez un ensemble de clés et de jetons [d’authentification](/fr/resources/fundamentals/authentication), et que vous avez correctement configuré votre environnement pour effectuer des requêtes vers l’API spécifique que vous prévoyez d’utiliser, vous pouvez suivre les étapes ci-dessous pour commencer.
 

<div id="getting-started-with-xs-postman-collections">
  ## Bien démarrer avec les collections Postman de X
</div>

<div id="step-one-add-one-of-the-x-postman-collections-to-your-account">
  ### Première étape : ajoutez l’une des collections Postman de X à votre compte
</div>

Vous pourriez configurer manuellement les endpoints spécifiques que vous souhaitez utiliser dans Postman, mais nous avons déjà fait le gros du travail en préparant une collection d’API pertinentes prête à l’emploi. Il vous suffit de cliquer sur l’un des liens de la section précédente « Collections Postman » et une collection contenant tous les endpoints associés à l’API sélectionnée sera ajoutée à votre application Postman. Ces collections sont également disponibles dans le [réseau d’API Postman](https://explore.postman.com/). Chaque endpoint inclut automatiquement les paramètres disponibles, des exemples de réponses et le type d’authentification préconfiguré ; vous n’aurez donc qu’à ajouter vos identifiants et les valeurs de paramètres pour commencer à explorer les fonctionnalités.

Dans cet exemple, nous allons utiliser la [collection API X v2](https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400). 
 

<div id="step-two-add-your-keys-and-tokens-as-environmental-variables">
  ### Deuxième étape : Ajoutez vos clés et jetons comme variables d’environnement
</div>

Lorsque vous ajoutez la collection à votre instance Postman, un environnement nommé « API X v2 » est automatiquement créé, et vous devrez y ajouter vos clés et jetons. Dans cette étape, nous vous guidons pour ajouter les clés et jetons de votre application développeur à l’environnement « API X v2 ». 

Pour ajouter vos clés et jetons à l’environnement « API X v2 », cliquez sur le bouton « Manage Environments » dans l’angle supérieur droit de Postman. Vous pouvez aussi cliquer sur le bouton des paramètres dans l’angle supérieur droit. 
 

<Frame>
  <img src="/images/using-postman-1.png.twimg.1920.png" alt="This image shows the &#x22;manage environments&#x22; button highlighted in the Postman console." />
</Frame>

Dans la liste des environnements, cliquez sur « API X v2 ». 

Ajoutez ensuite une variable pour chacune des clés et chacun des jetons que vous avez générés via le tableau de bord Apps, dans le tableau. Voici un exemple de ce à quoi votre tableau devrait ressembler :

| consumer\_key | `QAktM6W6DF6F7XXXXXX` | `QAktM6W6DF6F7XXXXXX` |
| consumer\_secret | `AJX560A2Omgwyjr6Mml2esedujnZLHXXXXXX` | `AJX560A2Omgwyjr6Mml2esedujnZLHXXXXXX` |
| access\_token | `1995XXXXX-0NGqVhk3s96IX6SgT3H2bbjOPjcyQXXXXXXX` | `1995XXXXX-0NGqVhk3s96IX6SgT3H2bbjOPjcyQXXXXXXX` |
| token\_secret | `rHVuh7dgDuJCOGeoe4tndtjKwWiDjBZHLaZXXXXXX` | `rHVuh7dgDuJCOGeoe4tndtjKwWiDjBZHLaZXXXXXX` |
| bearer\_token | `AAAAAAAAAAAAAAAAAAAAAL9v6AAAAAAA99t03huuqRYg0mpYAAFRbPR3XXXXXXX` | `AAAAAAAAAAAAAAAAAAAAAL9v6AAAAAAA99t03huuqRYg0mpYAAFRbPR3XXXXXXX` |

Notez que les clés et jetons utilisés dans le tableau ci-dessus sont fictifs et ne fonctionneront pas s’ils sont utilisés dans une requête. 

Une fois que vous avez ajouté vos identifiants comme variables et vérifié que l’environnement « API X v2 » est sélectionné, vous pouvez envoyer des requêtes à la collection API X v2. Les onglets d’autorisation de chaque endpoint sont configurés pour hériter automatiquement des variables de cet environnement. 

Pour utiliser Postman avec des jetons d’accès utilisateur, passez à XXXX pour consulter les détails supplémentaires.

<div id="step-three-select-an-endpoint">
  ### Troisième étape : sélectionner un endpoint
</div>

L’étape suivante consiste à choisir un endpoint dans la collection et à commencer à construire votre requête. Vous pouvez sélectionner un endpoint dans la navigation à droite. Voici à quoi cela ressemble :

<Frame>
  <img src="/images/using-postman-2.png.twimg.1920.png" alt="Cette image montre la requête « Single Posts » sélectionnée sous le menu déroulant « Post Lookup » dans la section « API X v2 »." />
</Frame>

Pour cet exemple, nous allons utiliser API X v2 > Post Lookup > Single Post. 

<div id="step-four-add-values-to-the-params-tab">
  #### Quatrième étape : ajouter des valeurs à l’onglet Params
</div>

L’étape suivante consiste à accéder à l’onglet Params. Vous devriez voir un ensemble de paramètres inactifs avec des descriptions expliquant la fonction de chaque paramètre, ainsi qu’une liste de toutes les valeurs possibles que vous pouvez transmettre avec votre requête.

Dans cet exemple, nous allons activer les paramètres de requête expansions et tweet.fields et ajouter les valeurs suivantes :

|     |     |
| :--- | :--- |
| **Clé** | **Valeur** |
| `tweet.fields` | `created_at,attachments` |
| expansions | author\_id |

En plus d’ajouter les paramètres de requête, nous devons ajouter la variable de chemin requise, id. Étant donné que cet endpoint renvoie des Posts, nous devons ajouter un ID de Post valide comme valeur.

Vous pouvez trouver l’ID de Post en vous rendant sur x.com, en cliquant sur un Post, puis en regardant l’URL. Par exemple, l’ID de Post de l’URL suivante est `1228393702244134912` :

`https://x.com/XDevelopers/status/1228393702244134912`

Dans l’onglet Params, faites défiler au-delà de tous les paramètres de requête pour afficher la section « Path Variables ». Nous allons ajouter l’ID de Post que vous souhaitez utiliser comme valeur pour la clé id.

Si vous avez correctement saisi tout ce qui est indiqué dans cette étape, l’onglet Params devrait ressembler à ce qui suit :

<div id="step-five-send-your-request-and-review-your-response">
  #### Cinquième étape : envoyez votre requête et examinez la réponse
</div>

Maintenant que tout est prêt dans votre requête, vous pouvez cliquer sur le bouton « Send ».

Si tout a été configuré correctement, vous devriez recevoir la charge utile suivante :

```json
{
    "data": {
        "author_id": "2244994945",
        "text": "What did the developer write in their Valentine’s card?\n  \nwhile(true) {\n    I = Love(You);  \n}",
        "id": "1228393702244134912",
        "created_at": "2020-02-14T19:00:55.000Z"
    },
    "includes": {
        "users": [
            {
                "username": "XDevelopers",
                "name": "Developers",
                "id": "2244994945"
            }
        ]
    }
}
```

<div id="generating-a-user-access-token-with-postman">
  ### Générer un jeton d’accès utilisateur avec Postman :
</div>

<div id="using-oauth-10a-to-generate-a-user-access-token">
  #### Utiliser OAuth 1.0a pour générer un jeton d’accès utilisateur
</div>

Consultez le processus en trois étapes utilisé dans la [collection de test du flux OAuth 1.0a](https://www.postman.com/xapidevelopers/workspace/twitter-s-public-workspace/collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400?ctx=documentation).

<div id="using-oauth-20-to-generate-a-user-access-token">
  #### Utiliser OAuth 2.0 pour générer un jeton d’accès utilisateur
</div>

Si vous souhaitez générer un jeton d’accès utilisateur OAuth 2.0 dans Postman, vous pouvez créer des jetons d’accès OAuth 2.0 à utiliser avec la [collection Postman de l’API X v2](https://www.postman.com/xapidevelopers/workspace/twitter-s-public-workspace/collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400?ctx=documentation).

Cliquez sur la collection dans votre espace de travail, puis ouvrez l’onglet « Auth » et sélectionnez le type « OAuth 2.0 ». Ensuite, sous l’en-tête « Configure New Token », accédez à « Configuration Options ». Mettez « Grant Type » sur « Authorization Code (With PKCE) ».

Mettez à jour votre URL de rappel pour qu’elle corresponde à l’URL de rappel associée à votre application. Mettez également à jour les paramètres suivants :

- Auth URL - https://x.com/i/oauth2/authorize
- Access Token URL - https://api.x.com/2/oauth2/token
- Client ID - Votre identifiant client OAuth 2.0 depuis le Dev Portal
- Client Secret - Si vous utilisez un client confidentiel
- Scope - Périmètres correspondant aux endpoints que vous souhaitez utiliser. Exemple : « tweet.read users.read »
- Votre URL de rappel (également appelée URL de redirection). Elle doit correspondre à celle définie dans les paramètres d’authentification de votre application.
- State - state

Lorsque vous êtes prêt, cliquez sur « Get New Access Token » pour générer un jeton d’accès. Si une boîte de dialogue indique qu’une erreur s’est produite, vous devrez peut-être revenir en arrière pour vous connecter. Vous devrez autoriser votre application à accéder à votre compte en cliquant sur « Authorize app » dans la boîte de dialogue.

Après avoir autorisé votre application, vous serez redirigé vers Postman, où vous pourrez voir votre jeton et cliquer sur le bouton « Use Token » pour commencer à effectuer des requêtes au nom d’un utilisateur autorisé.

Vous êtes maintenant prêt à utiliser la collection Postman.

<div id="whats-next">
  ## Et ensuite
</div>

Si vous cliquez sur le bouton « Code » dans Postman, vous pouvez convertir la requête que nous venons de créer dans le langage de votre choix (Python, Node, Ruby, etc.) pour commencer plus facilement. Postman propose une [excellente documentation](https://learning.getpostman.com/) qui peut vous être utile. Nous mettons également à disposition du [code d’exemple sur GitHub](https://github.com/xdevplatform) pour vous aider à vous intégrer plus rapidement aux endpoints.
