Les connexions TLS sont requises pour accéder aux endpoints de l’API X. La communication via TLS préserve la confidentialité et la sécurité des utilisateurs en protégeant les informations échangées entre l’utilisateur et l’API X lorsqu’elles transitent sur l’Internet public. Les connexions à l’API X exigent TLS version 1.2.


<div id="verification">
  ## Vérification
</div>

<div id="use-an-up-to-date-root-store">
  ### Utiliser un magasin de certificats racine à jour
</div>

Il est important que votre application ou bibliothèque utilise un magasin de certificats racine fiable et à jour pour vérifier le certificat X. Lorsque c’est possible, utiliser le magasin racine fourni par votre système d’exploitation est souvent l’approche la plus simple. À défaut, le [magasin racine Mozilla (NSS)](https://www.mozilla.org/en-US/about/governance/policies/security-group/certs/) est maintenu publiquement et de manière transparente. cURL fournit également [une version de ce magasin au format PEM](https://curl.haxx.se/docs/caextract.html).

X émet actuellement la majorité de ses certificats à partir de la [DigiCert High Assurance EV Root CA](https://www.digicert.com/digicert-root-certificates.htm), mais ce n’est pas le cas pour 100 % des certificats liés à X et cela pourrait évoluer. Se fier uniquement aux racines DigiCert actuellement utilisées pourrait donc entraîner des problèmes avec votre application à l’avenir.

<div id="check-crls-and-the-ocsp-status">
  ### Vérifier les CRL et le statut OCSP[](#check-crls-and-the-ocsp-status "Permalien vers ce titre")
</div>

De nombreuses applications ne vérifient pas la liste de révocation des certificats pour les certificats renvoyés ou s’appuient sur le système d’exploitation pour le faire. Assurez-vous que votre application ou votre bibliothèque TLS est configurée pour imposer la vérification des CRL et d’OCSP (Online Certificate Status Protocol) avant d’accepter le certificat de X.

<div id="cdns">
  ### CDN[](#cdns "Lien permanent vers ce titre")
</div>

Pour afficher des Tweets contenant des médias, utilisez l’attribut `media_url_https`, qui fournit les URL HTTPS à utiliser pour les images. À l’avenir, toutes les URL renvoyées par les endpoints de l’API proposeront des chemins HTTPS.

<div id="provide-an-indication-of-security-status">
  ## Indiquer l’état de sécurité
</div>

Si possible, indiquez le statut actuel de la connexion entre votre application et X. Certains navigateurs web l’affichent via une icône de cadenas, tandis que d’autres le signalent par un message descriptif.
