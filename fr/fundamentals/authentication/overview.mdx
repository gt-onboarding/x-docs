Les API X traitent d’énormes volumes de données. Nous garantissons la sécurité de ces données, pour les développeurs comme pour les utilisateurs, au moyen de l’authentification. Il existe plusieurs méthodes d’authentification, présentées ci-dessous.

La plupart des développeurs n’auront pas à gérer les complexités liées à l’authentification, car les bibliothèques clientes les prennent automatiquement en charge.

Vous trouverez la liste des bibliothèques clientes disponibles sur notre page [Outils et bibliothèques](/fr/resources/tools-and-libraries).

<div id="authentication-methods">
  ## Méthodes d’authentification
</div>

<CardGroup cols={2}>
  <Card title="Contexte utilisateur OAuth 1.0a" href="/fr/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret">
    OAuth 1.0a permet à une application développeur X autorisée d’accéder à des informations de compte privées ou d’effectuer une action sur X au nom d’un compte X.

    <br />

    [**En savoir plus**](/fr/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret)
  </Card>

  <Card title="App only" href="/fr/resources/fundamentals/authentication/oauth-2-0/overview">
    Le jeton d’accès App only permet à une application développeur X d’accéder aux informations publiquement disponibles sur X.

    <br />

    [**En savoir plus**](/fr/resources/fundamentals/authentication/oauth-2-0/overview)
  </Card>

  <Card title="Authentification Basic" href="/fr/resources/fundamentals/authentication/basic-auth">
    De nombreuses API Enterprise de X nécessitent l’utilisation de l’authentification HTTP Basic.

    <br />

    [**En savoir plus**](/fr/resources/fundamentals/authentication/basic-auth)
  </Card>

  <Card title="Flux de code d’autorisation OAuth 2.0 avec PKCE" href="/fr/resources/fundamentals/authentication/oauth-2-0/authorization-code">
    Le contexte utilisateur OAuth 2.0 vous permet de vous authentifier au nom d’un autre compte, avec un contrôle plus fin de la portée de l’application, et de gérer des parcours d’autorisation sur plusieurs appareils.

    <br />

    [**En savoir plus**](/fr/resources/fundamentals/authentication/oauth-2-0/authorization-code)
  </Card>
</CardGroup>

<Note>
  **Remarque :**
  Les clés API de votre application et le jeton d’accès App only, ainsi que votre jeton d’accès personnel et votre secret de jeton d’accès, peuvent être obtenus dans la section [X developer Apps](/fr/resources/fundamentals/developer-apps) du [portail développeur](/fr/resources/fundamentals/developer-portal).

  **Si vous souhaitez effectuer des requêtes au nom d’un autre utilisateur**, vous devrez générer pour cet utilisateur un jeu distinct de jetons d’accès à l’aide du [flux OAuth à 3 volets](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens), puis transmettre ces jetons avec vos requêtes en contexte utilisateur OAuth 1.0a ou en contexte utilisateur OAuth 2.0.
</Note>
