<div id="api-key-and-secret">
  ### Clé et secret d’API
</div>

La clé et le secret d’API (également appelés clé et secret consommateur) sont les identifiants de base requis pour accéder à l’API X. Ils jouent le rôle de nom d’utilisateur et de mot de passe de votre application X et permettent à l’API X d’identifier l’application à l’origine des requêtes.

Ces identifiants peuvent être utilisés par les [endpoints d’authentification](/fr/resources/fundamentals/authentication/api-reference) pour générer d’autres identifiants, tels que les [jetons d’accès utilisateur et leurs secrets](/fr/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret) et les [jetons Bearer](/fr/resources/fundamentals/authentication/oauth-2-0/bearer-tokens). Vous devez également utiliser ces identifiants, avec des jetons d’accès et d’autres paramètres d’autorisation, pour [autoriser des requêtes](/fr/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request) nécessitant l’authentification OAuth 1.0a en contexte utilisateur. 

<div id="how-to-acquire-an-api-key-and-secret">
  #### Comment obtenir une clé et un secret d’API
</div>

Pour obtenir une clé et un secret d’API X, veuillez suivre ces étapes :

1. [Créez un compte développeur X](https://developer.x.com/en/apply-for-access)
2. Créez une [application X](/fr/resources/fundamentals/developer-apps) dans le [portail développeur](/fr/resources/fundamentals/developer-portal). Notez que si vous souhaitez utiliser l’[API X v2](/fr/x-api/introduction), vous devez ajouter votre application X à un [projet](/fr/resources/fundamentals/projects).
    

Lors de la création de votre application X, votre clé et votre secret d’API vous seront fournis, ainsi qu’un jeton Bearer. Veuillez noter que ces identifiants ne sont affichés qu’une seule fois ; veillez donc à les enregistrer dans votre gestionnaire de mots de passe ou dans un endroit sécurisé.

Nous proposons d’autres recommandations sur la gestion de vos clés et jetons sur notre page des [meilleures pratiques d’authentification](/fr/resources/fundamentals/authentication/guides/authentication-best-practices), y compris des détails sur la marche à suivre si vos identifiants ont été compromis.
 

<div id="how-to-find-and-regenerate-your-api-key-and-secret-after-app-creation">
  #### Comment trouver et régénérer votre clé API et votre secret après la création de l’App
</div>

Si vous avez déjà créé une App et que vous devez retrouver ou régénérer votre clé API et votre secret, veuillez suivre ces étapes :

1. Accédez au portail développeur
2. Développez le menu déroulant « Projets et applications » dans la barre latérale
3. Ouvrez l’App associée à la clé API et au secret que vous souhaitez retrouver ou régénérer
4. Accédez à l’onglet Keys and tokens

Vous y trouverez toutes les informations d’identification associées à votre App. 
 

<div id="how-to-use-your-api-key-and-secret">
  #### Comment utiliser votre clé et votre secret d’API
</div>

Si vous découvrez simplement la X Developer Platform, nous vous recommandons d’utiliser un [outil ou une bibliothèque](/fr/resources/tools-and-libraries) pour explorer ce que la plateforme propose. Ces outils gèrent l’authentification de manière transparente et peuvent vous faire gagner beaucoup de temps et vous éviter bien des frustrations. Nous recommandons tout particulièrement de [commencer avec Postman](/fr/tutorials/postman-getting-started) ou [Insomnia](https://insomnia.rest/) si vous débutez.

Si vous souhaitez créer une requête à partir de zéro, veuillez consulter notre guide sur [l’autorisation d’une requête OAuth 1.0a](/fr/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request).
