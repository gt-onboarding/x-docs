### POST oauth/request\_token

Permet à une application consommatrice d’obtenir un jeton de requête OAuth (Request Token) pour demander l’autorisation de l’utilisateur. Cette méthode satisfait la [section 6.1](https://oauth.net/core/1.0/#auth_step1) du [flux d’authentification OAuth 1.0](http://oauth.net/core/1.0/#anchor9).

**Nous exigeons l’utilisation de HTTPS pour toutes les étapes d’autorisation OAuth.**

**Remarque d’utilisation :** Seules les valeurs ASCII sont acceptées pour le `oauth_nonce`.

**URL de la ressource[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth/request_token`

**Informations sur la ressource[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Formats de réponse | JSON |
| Authentification requise ? | Non |
| Limitation de débit ? | Oui |

**Paramètres[](#parameters "Permalink to this headline")**

| Nom | Requis | Description | Exemple |
| :--- | :--- | :--- | :--- |
| oauth\_callback | requis | Pour la conformité avec OAuth 1.0a, ce paramètre est **requis**. La valeur spécifiée sera utilisée comme URL de redirection si l’utilisateur approuve l’accès de votre application à son compte. Définissez-la sur `oob` pour le mode code hors bande (out-of-band). C’est également ainsi que vous spécifiez des callbacks personnalisés pour les applications de bureau/mobile. Envoyez toujours un `oauth_callback` à cette étape, indépendamment d’un callback préenregistré.<br /><br />Nous exigeons que toute URL de callback utilisée avec ce point de terminaison soit configurée dans les paramètres de l’application sur developer.x.com.\* | `http://themattharris.local/auth.php` `twitterclient://callback` |
| x\_auth\_access\_type | optionnel | Remplace le niveau d’accès qu’une application demande au compte d’un utilisateur. Les valeurs prises en charge sont `read` ou `write`. Ce paramètre permet à un développeur d’enregistrer une application en lecture/écriture tout en demandant un accès en lecture seule lorsque cela est approprié. |     |

En savoir plus sur l’approbation de vos URL de callback sur [cette page](/fr/resources/fundamentals/developer-apps#callback-urls).

**Veuillez noter** — Vous pouvez afficher et modifier vos [applications X](/fr/resources/fundamentals/developer-apps) existantes via le [tableau de bord des applications X](https://developer.x.com/en/apps) si vous êtes connecté à votre compte X sur developer.x.com.

**Exemple de requête[](#example-request "Permalink to this headline")**

URL de la requête : `POST https://api.x.com/oauth/request_token`

Corps POST de la requête : _N/A_

En-tête Authorization : `OAuth oauth_nonce="K7ny27JTpKVsTgdyLdDfmQQWVLERj2zAK5BslRsqyw", oauth_callback="http%3A%2F%2Fmyapp.com%3A3005%2Ftwitter%2Fprocess_callback", oauth_signature_method="HMAC-SHA1", oauth_timestamp="1300228849", oauth_consumer_key="OqEqJeafRSF11jBMStrZz", oauth_signature="Pc%2BMLdv028fxCErFyi8KXFM%2BddU%3D", oauth_version="1.0"`

Réponse : `oauth_token=Z6eEdO8MOmk394WozF5oKyuAv855l4Mlqo7hhlSLik&oauth_token_secret=Kd75W4OQfb2oJTV0vzGzeXftVAwgMnEK9MumzYcM&oauth_callback_confirmed=true`

<div id="get-oauthauthorize">
  ### GET oauth/authorize
</div>

Permet à une application Consommateur d’utiliser un jeton de requête OAuth pour demander l’autorisation de l’utilisateur. Cette méthode répond à la [section 6.2](http://oauth.net/core/1.0/#auth_step2) du [flux d’authentification OAuth 1.0](http://oauth.net/core/1.0/#anchor9). Les applications de bureau doivent utiliser cette méthode (et ne peuvent pas utiliser [GET oauth/authenticate](/fr/resources/fundamentals/authentication/api-reference#get-oauth-authenticate)).

Note d’utilisation : un `oauth_callback` n’est jamais envoyé à cette méthode ; fournissez-le plutôt à [POST oauth/request\_token](/fr/resources/fundamentals/authentication/api-reference#post-oauth-request-token).

**URL de la ressource[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth/authorize`

**Informations sur la ressource[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Formats de réponse | JSON |
| Authentification requise ? | Oui |
| Soumis à une limite de débit ? | Oui |

**Paramètres[](#parameters "Permalink to this headline")**

|     |     |     |     |     |
| :--- | :--- | :--- | :--- | :--- |
| Nom | Obligatoire | Description | Valeur par défaut | Exemple |
| force\_login | facultatif | Oblige l’utilisateur à saisir ses identifiants afin de garantir que le bon compte utilisateur est autorisé. |     |     |
| screen\_name | facultatif | Préremplit le champ du nom d’utilisateur sur l’écran de connexion OAuth avec la valeur fournie. |     |     |

**Exemple de requête[](#example-request "Permalink to this headline")**

Redirigez l’utilisateur vers l’étape `oauth/authorize` dans un navigateur Web, en incluant un paramètre oauth\_token :
`https://api.x.com/oauth/authorize?oauth_token=Z6eEdO8MOmk394WozF5oKyuAv855l4Mlqo7hhlSLik`

<div id="get-oauthauthenticate">
  ### GET oauth/authenticate
</div>

Autorise une application consommatrice à utiliser un `request_token` OAuth pour demander l’autorisation de l’utilisateur.

Cette méthode remplace la [section 6.2](http://oauth.net/core/1.0/#auth_step2) du [flux d’authentification OAuth 1.0](http://oauth.net/core/1.0/#anchor9) pour les applications utilisant le flux d’authentification avec rappel (callback). La méthode utilisera l’utilisateur actuellement connecté comme compte pour l’autorisation d’accès, sauf si le paramètre `force_login` est défini sur `true`.

Cette méthode diffère de [GET oauth/authorize](/fr/resources/fundamentals/authentication/api-reference#get-oauth-authorize) en ce que, si l’utilisateur a déjà accordé l’autorisation à l’application, la redirection aura lieu sans que l’utilisateur ait à approuver à nouveau l’application. Pour obtenir ce comportement, vous devez activer le paramètre Use Sign in with X dans l’[enregistrement de votre application](https://developer.x.com/apps).

**URL de la ressource[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth/authenticate`

**Informations sur la ressource[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Formats de réponse | JSON |
| Authentification requise ? | Oui |
| Soumis à des limites de débit ? | Oui |

**Paramètres[](#parameters "Permalink to this headline")**

|     |     |     |     |     |
| :--- | :--- | :--- | :--- | :--- |
| Nom | Obligatoire | Description | Valeur par défaut | Exemple |
| force\_login | optionnel | Oblige l’utilisateur à saisir ses identifiants pour garantir que le bon compte utilisateur est autorisé. |     | _true_ |
| screen\_name | optionnel | Préremplit le champ du nom d’utilisateur de l’écran de connexion OAuth avec la valeur fournie. |     |     |

**Exemple de requête[](#example-request "Permalink to this headline")**

Envoyez l’utilisateur à l’étape `oauth/authenticate` dans un navigateur web, en incluant un paramètre oauth\_token :
`https://api.x.com/oauth/authenticate?oauth_token=Z6eEdO8MOmk394WozF5oKyuAv855l4Mlqo7hhlSLik`

### POST oauth/access\_token

Permet à une application consommatrice d’échanger le OAuth Request Token contre un OAuth Access Token. Cette méthode respecte la [section 6.3](http://oauth.net/core/1.0/#auth_step3) du [flux d’authentification OAuth 1.0](http://oauth.net/core/1.0/#anchor9).

**URL de la ressource[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth/access_token`

**Informations sur la ressource[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Formats de réponse | JSON |
| Authentification requise ? | Oui |
| Limitation de débit ? | Oui |

**Paramètres[](#parameters "Permalink to this headline")**

|     |     |     |     |     |
| :--- | :--- | :--- | :--- | :--- |
| Nom | Obligatoire | Description | Valeur par défaut | Exemple |
| oauth\_token | obligatoire | Le `oauth_token` ici doit être identique au `oauth_token` renvoyé à l’étape `request_token`. |     |     |
| oauth\_verifier | obligatoire | Si vous utilisez le flux web OAuth, définissez ce paramètre sur la valeur de `_oauth_verifier_` renvoyée dans l’URL de rappel (callback). Si vous utilisez OAuth hors bande, définissez cette valeur sur le code PIN. Pour la conformité à OAuth 1.0a, ce paramètre est **obligatoire**. OAuth 1.0a est strictement appliqué et les applications qui n’utilisent pas `_oauth_verifier_` ne pourront pas terminer le flux OAuth. |     |     |

**Exemple de requête[](#example-request "Permalink to this headline")**

`POST https://api.x.com/oauth/access_token?oauth_token=qLBVyoAAAAAAx72QAAATZxQWU6P&oauth_verifier=ghLM8lYmAxDbaqL912RZSRjCCEXKDIzx`

Depuis un code PIN `POST https://api.x.com/oauth/access_token?oauth_token=9Npq8AAAAAAAx72QBRABZ4DAfY9&oauth_verifier=4868795`

**Exemple de réponse[](#example-response "Permalink to this headline")**

`oauth_token=6253282-eWudHldSbIaelX7swmsiHImEL4KinwaGloHANdrY&oauth_token_secret=2EEfA6BG5ly3sR3XjE0IBSnlQu4ZrUzPiYTmrkVU&user_id=6253282&screen_name=xapi`

### POST oauth/invalidate\_token

Permet à une application enregistrée de révoquer un access\_token OAuth émis en présentant ses identifiants client. Une fois qu’un access\_token a été invalidé, toute nouvelle tentative de création produira un jeton d’accès différent et l’utilisation du jeton invalidé ne sera plus autorisée.

**URL de la ressource[](#resource-url "Permalink to this headline")**

`https://api.x.com/1.1/oauth/invalidate_token`

**Informations sur la ressource[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Formats de réponse | JSON |
| Authentification requise ? | Oui — Contexte utilisateur avec les jetons d’accès que vous souhaitez invalider |
| Limitation de débit ? | Oui |

**Exemple de requête[](#example-request "Permalink to this headline")**

```bash
        curl --request POST
          --url 'https://api.x.com/1.1/oauth/invalidate_token.json'
          --header 'authorization: OAuth oauth_consumer_key="CLIENT_KEY",
         oauth_nonce="AUTO_GENERATED_NONCE", oauth_signature="AUTO_GENERATED_SIGNATURE",
         oauth_signature_method="HMAC-SHA1", oauth_timestamp="AUTO_GENERATED_TIMESTAMP",
         oauth_token="ACCESS_TOKEN", oauth_version="1.0"'
```

**Exemple de réponse[](#example-response "Permalink to this headline")**

```bash
        HTTP/1.1 200 OK
        Content-Type: application/json; charset=utf-8
        Content-Length: 127
        ...

        {"access_token":"ACCESS_TOKEN"}
```

**Exemple de réponse d’erreur après invalidation du jeton[](#example-error-response-after-token-has-been-invalidated "Permalink to this headline")**

```bash
        HTTP/1.1 401 Authorization Required
        ...

        {"errors": [{
          "code": 89,
          "message": "Invalid or expired token."}
        ]}
```

<div id="post-oauth2token">
  ### POST oauth2/token
</div>

Permet à une application enregistrée d’obtenir un jeton Bearer OAuth 2, utilisable pour effectuer des requêtes à l’API au nom de l’application elle-même, sans contexte utilisateur. Ceci s’appelle l’[authentification « Application-only »](/fr/resources/fundamentals/authentication/oauth-2-0/application-only).

Un jeton Bearer peut être invalidé via oauth2/invalidate\_token. Une fois invalidé, toute nouvelle tentative de création produira un jeton Bearer différent et l’utilisation du jeton précédent ne sera plus autorisée.

Un seul jeton Bearer peut être actif pour une application ; des requêtes répétées vers cette méthode renverront le même jeton existant jusqu’à son invalidation.

Les réponses réussies incluent une structure JSON décrivant le jeton Bearer délivré.

Les jetons reçus par cette méthode doivent être mis en cache. En cas de tentatives trop fréquentes, les requêtes seront rejetées avec un HTTP 403 et le code 99.

**URL de la ressource[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth2/token`

**Informations sur la ressource[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Formats de réponse | JSON |
| Authentification requise ? | Oui — authentification Basic avec votre clé API comme nom d’utilisateur et le secret de votre clé API comme mot de passe |
| Limitation de débit ? | Oui |

**Paramètres[](#parameters "Permalink to this headline")**

|     |     |     |     |     |
| :--- | :--- | :--- | :--- | :--- |
| Nom | Requis | Description | Valeur par défaut | Exemple |
| grant\_type | requis | Spécifie le type d’octroi (grant) demandé par l’application. À ce jour, seul _client\_credentials_ est autorisé. Voir l’[authentification « Application-only »](/fr/resources/fundamentals/authentication/oauth-2-0/application-only) pour plus d’informations. |     | _client\_credentials_ |

**Exemple de requête[](#example-request "Permalink to this headline")**

```bash
    POST /oauth2/token HTTP/1.1
    Host: api.x.com
    User-Agent: My X App v1.0.23
    Authorization: Basic eHZ6MWV2R ... o4OERSZHlPZw==
    Content-Type: application/x-www-form-urlencoded;charset=UTF-8
    Content-Length: 29
    Accept-Encoding: gzip

    grant_type=client_credentials
```

**Exemple de réponse :**

```bash
    HTTP/1.1 200 OK
    Status: 200 OK
    Content-Type: application/json; charset=utf-8
    ...
    Content-Encoding: gzip
    Content-Length: 140

    {"token_type":"bearer","access_token":"AAAA%2FAAA%3DAAAAAAAA"}
```

### POST oauth2/invalidate\_token

Permet à une application enregistrée de révoquer un jeton Bearer OAuth 2.0 émis en présentant ses identifiants client. Une fois un jeton Bearer invalidé, toute nouvelle création fournira un jeton Bearer différent et l’utilisation du jeton invalidé ne sera plus autorisée.

Les réponses réussies incluent une structure JSON décrivant le jeton Bearer révoqué.

**URL de la ressource[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth2/invalidate_token`

**Informations sur la ressource[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Formats de réponse | JSON |
| Authentification requise ? | Oui — [OAuth 1.0a](/fr/resources/fundamentals/authentication/oauth-1-0a) avec les clés API consommateur de l’application, le jeton d’accès du propriétaire de l’application et le secret du jeton d’accès |
| Limitation de débit ? | Oui |

**Paramètres[](#parameters "Permalink to this headline")**

| Nom | Obligatoire | Description |
| :--- | :--- | :--- |
| access\_token | obligatoire | Valeur du jeton Bearer que vous souhaitez invalider |

**Exemple de requête[](#example-request "Permalink to this headline")**

```
        curl --request POST
          --url 'https://api.x.com/oauth2/invalidate_token?access_token=AAAA%2FAAA%3DAAAAAAAA'
          --header 'authorization: OAuth oauth_consumer_key="CLIENT_KEY",
         oauth_nonce="AUTO_GENERATED_NONCE", oauth_signature="AUTO_GENERATED_SIGNATURE",
         oauth_signature_method="HMAC-SHA1", oauth_timestamp="AUTO_GENERATED_TIMESTAMP",
         oauth_token="ACCESS_TOKEN", oauth_version="1.0"'
```

**Exemple de réponse[](#example-response "Permalink to this headline")**

```
         Status: 200 OK
         Content-Type: application/json; charset=utf-8
         Content-Length: 135
         ...
       {
        "access_token": "AAAA%2FAAA%3DAAAAAAAA"
        }
```
