<div id="introduction">
  ## Introduction
</div>

Les Projets vous permettent d’organiser votre travail en fonction de la manière dont vous comptez utiliser l’API X, afin de gérer efficacement votre accès à l’API et de suivre votre utilisation. Chaque Projet peut contenir une ou plusieurs [Applications](/fr/resources/fundamentals/developer-apps) selon votre niveau d’accès (décrit plus loin sur cette page). Vous utiliserez ces Applications pour générer des informations [d’authentification](/fr/resources/fundamentals/authentication) telles que des [clés et secrets d’API](/fr/resources/fundamentals/authentication#api-key-and-secret), des [jetons d’accès](/fr/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) utilisateur et un [jeton d’accès d’application](/fr/resources/fundamentals/authentication#bearer-token-also-known-as-app-only).

Bien que vous puissiez utiliser ces clés et jetons à partir de n’importe quelle Application pour accéder à l’API X ou à l’API publicitaire X (vous devez [demander un accès supplémentaire](/fr/x-ads-api/getting-started/step-by-step-guide) pour utiliser l’API publicitaire), vous devez utiliser des clés et jetons provenant d’une Application associée à un Projet pour pouvoir utiliser les endpoints de l’[API X v2](/fr/x-api/introduction).

Si vous avez un compte développeur, vous pouvez consulter et gérer vos Projets sur la page [Projects & Apps](https://developer.x.com/en/portal/projects-and-apps) dans le portail développeur. [Inscrivez-vous](https://developer.x.com/en/portal/petition/essential/basic-info) pour obtenir un compte si vous n’en avez pas encore un.

<div id="projects-and-x-api-access-levels">
  ### Projets et niveaux d’accès à l’API X
</div>

À l’heure actuelle, il existe 4 paliers d’accès différents appliqués au niveau du Projet :

- Free
- Basic
- Pro
- Enterprise

Pour en savoir plus sur ce que propose chacun de ces niveaux d’accès, veuillez consulter la page [À propos de l’API X](/fr/x-api/getting-started/about-x-api).

Vous ne pouvez avoir qu’un seul Projet avec l’un des niveaux Free, Basic ou Pro.

Nous présenterons ici quelques différences propres aux Projets :

<div id="free">
  #### Gratuit
</div>

Ce niveau d’accès est proposé à toute personne qui s’est [inscrite](https://developer.twitter.com/en/portal/petition/essential/basic-info) pour obtenir un compte développeur.

Nombre d’applications dans ce projet : 1
[Plafond de posts](/fr/x-api/fundamentals/post-cap) : 1 500 Tweets par mois

Méthodes d’authentification :

- [OAuth 2.0 avec PKCE](/fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
- Application uniquement

<div id="basic">
  #### Basic
</div>

Ce niveau d’accès est disponible pour toute personne ayant souscrit à l’offre Basic via le portail développeur.

Nombre d’applications dans ce projet : 2
[Plafond de consommation de posts](/fr/x-api/fundamentals/post-cap) : 10 000 posts par mois
Méthodes d’authentification :

- [OAuth 2.0 avec PKCE](/fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
- [OAuth 1.0a](/fr/resources/fundamentals/authentication#oauth-1-0a-2)
- Application uniquement

<div id="pro">
  #### Pro
</div>

Ce niveau d’accès est proposé à toute personne ayant souscrit à l’accès Pro via le portail développeur.

Nombre d’applications dans ce projet : 3
[Plafond de posts](/fr/x-api/fundamentals/post-cap) : 1 million de tweets par mois

Méthodes d’authentification :

- [OAuth 2.0 avec PKCE](/fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
- [OAuth 1.0a](/fr/resources/fundamentals/authentication#oauth-1-0a-2)
- App only

<div id="enterprise">
  #### Enterprise
</div>

Ce palier d’accès est proposé à toute personne ayant demandé l’accès Enterprise, ayant été approuvée par notre équipe et ayant signé un contrat mensuel à renouvellement automatique via un account manager X. [Postulez ici](https://docs.google.com/forms/d/e/1FAIpQLScO3bczKWO2jFHyVZJUSEGfdyfFaqt2MvmOfl_aJp0KxMqtDA/viewform).

Nombre d’applications au sein de ce projet : 3+
[Plafond de consommation de Tweets](/fr/x-api/fundamentals/post-cap) : plus de 50 millions de Tweets par mois

Méthodes d’authentification :

- [OAuth 2.0 avec PKCE](/fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
- [OAuth 1.0a](/fr/resources/fundamentals/authentication#oauth-1-0a-2)
- Application uniquement

<div id="configuring-projects">
  ### Configuration des projets
</div>

<div id="creating-a-project">
  #### Création d’un projet
</div>

Pour créer un projet, cliquez sur « New Project » dans votre [tableau de bord](https://developer.twitter.com/en/portal/dashboard) ou sur la page [Projects & Apps](https://developer.twitter.com/en/portal/projects-and-apps) du portail développeur. Cette option n’apparaît que si vous n’avez pas encore créé de projet. Il vous sera demandé de renseigner un nom de projet, une description et un cas d’usage. Vous devrez également créer une nouvelle application ou connecter une application autonome existante.

<div id="creating-or-connecting-an-app-for-your-project">
  #### Créer ou connecter une application pour votre projet
</div>

Si votre projet n’inclut pas d’application, vous pouvez en ajouter une en cliquant sur le nom du projet dans le tableau de bord. À partir de là, vous pouvez soit créer une nouvelle application, soit sélectionner une application autonome existante à connecter à votre projet. C’est dans l’application que vous pouvez générer les clés et jetons d’authentification mentionnés au début de ce guide.

<div id="editing-a-project">
  #### Modification d’un projet
</div>

Pour modifier un projet, cliquez sur le nom de votre projet depuis le tableau de bord ou la page Projets et applications dans le portail développeur. Vous verrez alors les détails de votre projet et pourrez sélectionner « modifier » pour apporter des modifications.

<div id="standalone-apps">
  ### Applications autonomes
</div>

Les applications autonomes sont des applications qui existent en dehors de la structure des Projets. Les identifiants d’authentification associés à ces applications autonomes peuvent effectuer des requêtes réussies vers les API standard v1.1, premium v1.1, enterprise de X, ou vers l’API publicitaire X. Les applications autonomes échoueront lorsqu’elles tenteront d’effectuer des requêtes vers les endpoints de l’API X v2, sauf si vous les connectez à un Projet.

Si vous avez créé une application avant août 2020, elle sera visible dans la section « Standalone Apps » du portail développeur sous [Projects & Apps](https://developer.twitter.com/en/portal/projects-and-apps). Vous serez limité à dix applications au total, y compris celles qui sont connectées à votre Projet.

Si vous faites partie d’un compte d’équipe, vous verrez les applications que vous possédez sous Applications autonomes. Si un membre de votre équipe possède une application qui fait partie d’un Projet, vous pourrez voir le nom de l’application et les informations sur son propriétaire, mais vous ne pourrez pas modifier ses paramètres, accéder à ses clés et jetons, ni régénérer ses clés et jetons. Vous devrez contacter le propriétaire de l’application pour apporter des modifications à son application.
