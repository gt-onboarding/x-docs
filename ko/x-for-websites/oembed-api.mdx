oEmbed API를 사용하면 [트윗](https://developer.x.com/en/docs/twitter-for-websites/embedded-tweets/overview)과 [타임라인](https://developer.x.com/en/docs/twitter-for-websites/timelines/overview) 같은 임베디드 콘텐츠를 프로그래밍 방식으로 가져올 수 있습니다. 

oEmbed API의 응답은 [페이지에 X 위젯 JavaScript가 포함되어 있는 경우](https://developer.x.com/web/javascript/loading) 자동으로 인식되는 HTML 스니펫을 반환합니다.

이 API는 대량 작업에 사용하는 것을 권장하며, 콘텐츠 임베딩에는 신뢰성 높은 [publish.x.com](https://publish.x.com/#) 도구 사용을 권장합니다.

<Tabs>
  <Tab title="Embedded timelines">
    반환된 HTML 스니펫은 [임베드 타임라인](https://developer.x.com/en/docs/twitter-for-websites/timelines/overview)으로 자동 인식되며, [페이지에 X 위젯 JavaScript가 포함되어 있으면](https://developer.x.com/web/javascript/loading) 적용됩니다.

    oEmbed 엔드포인트는 임베드된 타임라인의 최종 표시 형태를 사용자 지정할 수 있도록, HTML 응답에 기본적으로 번들되는 X의 JavaScript가 해석할 수 있는 HTML 마크업 속성을 설정할 수 있게 합니다. X가 새로운 기능을 추가하거나 타임라인 표현을 조정함에 따라 반환되는 마크업 형식은 시간이 지나면서 변경될 수 있습니다.

    타임라인 URL로 지정한 X 타임라인에 대해 [oEmbed](https://oembed.com/) 호환 JSON 형식이 제공됩니다. 사용자 타임라인과 리스트 타임라인을 지원합니다. 타임라인 마크업은 cache\_age 속성에 지정된 권장 캐시 수명 동안 귀하의 서버에서 캐시하는 용도로 제공됩니다.

    ## Resource URL

    _https://publish.x.com/oembed_

    ## Resource Information

    |     |     |
    | :--- | :--- |
    | Response formats | JSON |
    | Requires authentication? | No  |
    | Rate limited | No  |

    ## Parameters

    | Name | Description | Example |
    | :--- | :--- | :--- |
    | **url** | 임베드할 X 타임라인의 URL | \*   https://x.com/TwitterDev<br /> \*https://x.com/TwitterDev/lists/national-parks
    | limit | 표시할 항목 수 N을 지정합니다. N은 1에서 20 사이의 값(포함)이어야 합니다 | 6   |
    | maxwidth | 위젯의 최대 너비를 설정합니다. 180에서 1200 사이(포함)여야 합니다 | 300 |
    | maxheight | 위젯의 최대 높이를 설정합니다. 200보다 커야 합니다 | 400 |
    | omit\_script | 응답에 script 요소를 포함하지 않습니다 | 1   |
    | lang | 지원되는 X [언어 코드](/ko/en/docs/twitter-for-websites/twitter-for-websites-supported-languages/overview "Twitter language code") | es  |
    | theme | dark로 설정하면 어두운 배경에 밝은 텍스트로 타임라인이 표시됩니다 | dark |
    | chrome | 공백으로 구분한 토큰으로 타임라인 표시 구성 요소 제거<br /><br />\*   noheader - 헤더 숨김<br />\*   nofooter - 표시되는 경우 푸터 숨김<br />\*   noborders - 위젯 주변, 트윗 간, 트윗 내부의 모든 테두리 제거<br />\*   noscrollbar - 표시되는 경우 타임라인 스크롤바 자르고 숨김<br />\*   transparent - 배경색 제거 | noheader%20nofooter |
    | aria\_polite | 타임라인에 추가되는 트윗에 대해 assertive 수준의 [ARIA 라이브 영역 정중도](https://www.w3.org/TR/wai-aria/states_and_properties#aria-live) 값을 설정합니다 | assertive |
    | dnt | true로 설정하면 귀하의 사이트의 임베드 페이지와 타임라인은 [개인화된 제안](https://support.x.com/articles/20169421) 및 [개인화된 광고](https://support.x.com/articles/20170405) 등의 목적으로 사용되지 않습니다 | true |

    ## Example Requests

    ```bash
    curl --request GET --url 'https://publish.x.com/oembed?url=https%3A%2F%2Ftwitter.com%2FInterior%2Fstatus%2F507185938620219395'
    twurl -H publish.x.com "/oembed?url=https://x.com/Interior/status/463440424141459456"
    ```

    ## Example Response

    ```json

    {
      "url": "https://x.com/TwitterDev",
      "title": "",
      "html": "<a class=\"twitter-timeline\" href=\"https://x.com/TwitterDev\">Tweets by TwitterDev</a>\n<script async src=\"//platform.x.com/widgets.js\" charset=\"utf-8\"></script>",
      "width": null,
      "height": null,
      "type": "rich",
      "cache_age": "3153600000",
      "provider_name": "Twitter",
      "provider_url": "https://x.com",
      "version": "1.0"
    }
    ```

  <Tab title="Embedded Tweets">
    [X의 위젯 JavaScript가 페이지에 포함되어 있으면](https://developer.x.com/web/javascript/loading) 반환된 HTML 스니펫은 자동으로 [임베드된 트윗](https://developer.x.com/web/embedded-tweets)으로 인식됩니다.

    oEmbed 엔드포인트는 기본적으로 HTML 응답과 함께 제공되는 X의 JavaScript가 해석할 수 있도록, HTML 마크업에서 해당 속성을 설정해 임베디드 트윗의 최종 표시 형태를 사용자 지정할 수 있게 합니다. X가 새로운 기능을 추가하거나 트윗 표현을 조정함에 따라 반환되는 마크업의 형식은 시간이 지나며 변경될 수 있습니다.

    `cache_age` 속성에 명시된 권장 캐시 수명 동안 트윗 폴백 마크업은 귀하의 서버에 캐시하도록 되어 있습니다.

    ## 리소스 URL

    _https://publish.x.com/oembed_

    ## 리소스 정보

    |     |     |
    | :--- | :--- |
    | 응답 형식 | JSON |
    | 인증 필요 여부 | 아니요  |
    | Rate limit 적용 | 아니요  |

    ## 매개변수

    | 이름 | 기본값 | 설명 |
    | :--- | :--- | :--- |
    | `url`required  <br />문자열 |     | 임베드할 트윗의 URL |
    | `maxwidth`  <br />정수 `[220..550]` | `325` | 렌더링된 트윗의 최대 너비(픽셀 단위, 정수). 허용 범위를 벗어난 값을 제공하면 각각 지원되는 최소 또는 최대 너비로 조정되어 반환되며, 조정된 너비 값은 반환되는 `width` 속성에 반영됩니다. X는 oEmbed의 `maxheight` 매개변수를 지원하지 않습니다. 트윗은 본질적으로 텍스트이므로 이미지나 동영상처럼 스케일링할 수 없는 가변적인 높이를 갖습니다. 이와 관련하여 oEmbed 응답에는 `height` 값이 제공되지 않습니다. 트윗에 대해 일정한 높이가 필요한 구현은 아래의 `hide_thread` 및 `hide_media` 매개변수를 참고하세요. |
    | `hide_media`  <br />불리언, 문자열 또는 정수 | `false` | `true`, `"t"`, 또는 `1`로 설정하면 트윗의 링크가 사진, 동영상 또는 링크 미리보기로 확장되지 않습니다. |
    | `hide_thread`  <br />불리언, 문자열 또는 정수 | `false` | `true`, `"t"`, 또는 `1`로 설정하면 요청된 트윗이 다른 트윗에 대한 답글인 경우 대화 스레드에서 이전 트윗의 접힌 버전이 표시되지 않습니다. |
    | `omit_script`  <br />불리언, 문자열 또는 정수 | `false` | `true`, `"t"`, 또는 `1`로 설정하면 `widgets.js`를 로드하는 `<script>`가 반환되지 않습니다. 웹페이지에는 [임베디드 트윗](https://developer.x.com/web/embedded-tweets)을 포함한 모든 X 위젯에서 사용할 수 있도록 `widgets.js`에 대한 자체 참조를 포함해야 합니다. |
    | `align`  <br />열거형 `{left,right,center,none}` | `none` | 임베드된 트윗을 상위 요소를 기준으로 페이지에서 왼쪽, 오른쪽 또는 가운데로 부동 배치할지 지정합니다. |
    | `lang`  <br />열거형([언어](https://developer.x.com/en/docs/twitter-for-websites/twitter-for-websites-supported-languages/overview)) | `en` | 지정된 [임베디드 트윗이 지원하는 X 언어](https://developer.x.com/web/overview/languages)로 반환되는 HTML과 렌더링된 트윗을 요청합니다. |
    | `theme`  <br />열거형 `{light, dark}` | `light` | `dark`로 설정하면 어두운 배경에 밝은 텍스트로 트윗이 표시됩니다. |
    | `dnt`  <br />불리언 | `false` | `true`로 설정하면 귀하의 사이트에 임베드된 트윗과 해당 페이지는 [개인화된 추천](https://support.x.com/articles/20169421) 및 [개인화된 광고](https://support.x.com/articles/20170405)를 포함한 목적으로 사용되지 않습니다. |
