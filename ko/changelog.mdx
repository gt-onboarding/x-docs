<Update label="Jun 3, 2025" description="X API">
  ### DM 이벤트 동작 변경

  오늘 X API v2의 DM 이벤트 동작을 변경했습니다. 이번 변경으로 ParticipantsJoin 및 ParticipantsLeave 이벤트에만 participant\_ids 배열이 포함되며, 이벤트 발생 시점에 참여하거나 나간 사용자만 나열됩니다. MessageCreate 이벤트에는 더 이상 participant\_ids 배열이 포함되지 않습니다. 자세한 내용은 [여기](https://devcommunity.x.com/t/upcoming-change-to-x-api-dm-events-endpoints-behavior/241841)를 참고하세요.
</Update>

<Update label="Aug 3, 2022" description="X API v2">
  ### 필터링 연산자 별칭 추가

  필터링/쿼리 연산자 ‘별칭’이 엔터프라이즈, 프리미엄, v2 전반의 검색 및 필터링 스트림 엔드포인트에 추가되었습니다. 많은 경우 연산자는 statuses 대신 posts를 가리킵니다. 예: "in\_reply\_to\_status\_id"의 별칭으로서 "in\_reply\_to\_tweet\_id". 다른 경우에는 새로운 연산자 별칭이 수행되는 매칭을 더 명확히 합니다. 예를 들어 게시물은 하나의 비디오 첨부만 가질 수 있으므로 "has:videos"에 대한 새로운 "has:video\_link" 별칭이 제공됩니다. 더 많은 예시는 [v2 필터링 스트림 연산자 표](https://developer.x.com/en/docs/twitter-api/tweets/filtered-stream/integrate/build-a-rule#list)를 확인하세요.
</Update>

  ### OAuth 2.0 인증 지원

  오늘부터 모든 개발자는 개발자 포털에서 OAuth 2.0을 인증 방법으로 선택해 OAuth 2.0으로 인증할 수 있습니다. 이번 릴리스의 일환으로 기밀 클라이언트와 공개 클라이언트, 그리고 관련 v2 엔드포인트에서 이 인증 방법을 사용할 수 있도록 지원을 추가했습니다.

  이 현대적인 권한 부여 프로토콜을 통해 최종 사용자에게 앱 승인을 위한 더 간소화된 동의 흐름을 제공할 수 있으며, 여러분이 요청한 특정 스코프만 표시합니다.

  ### X API v2 주요 플랫폼 업데이트

  **주요 플랫폼 업데이트: X API v2가 기본 버전으로 전환, 새로운 Essential 및 Elevated 액세스 도입, 정책 변경**

  오늘 X API v2가 X API의 기본 버전이 되었음을 발표합니다. 우리는 X API v2에 충분한 엔드포인트와 기능을 제공하여, X API를 기반으로 구축된 기존 앱의 90%가 필요로 하는 요구사항을 충족했습니다.

  X API v2의 개발자 경험을 더욱 개선하기 위해 오늘 다음과 같은 변경을 도입합니다:

  - 빠르고 무료인 Essential 액세스, 그리고 사용 사례가 승인된 개발자를 위한 무료 Elevated 액세스를 제공합니다. 이러한 액세스 수준에 대한 자세한 내용은 About the X API 페이지에서 확인하실 수 있습니다.

  - X의 핵심 기능을 활용한 빌드를 제한하고, 앱이 지원할 수 있는 사용자 수를 제한하던 Developer Policy의 문구를 제거했습니다.

    - 우리는 플랫폼을 개방하여, X를 더 나아지게 하고 건강하게 만들며 공적 대화를 확장하는 도구와 제품을 여러분이 구축하도록 장려합니다. [여러분이 만들어 주시길 바라는 솔루션 목록을 확인해 보세요](https://developer.x.com/en/docs/twitter-api/what-to-build).

    - 구체적으로, X의 핵심 기능을 포함한 X 경험의 복제를 제한하던 조항과, 많은 수의 사용자 토큰 보유에 대해 허가를 요구하던 조항을 제거했습니다.

    - X에서 사람들을 돕는 솔루션을 구축하려면 종종 X에서 제공되는 일부 기능을 직접 구축(또는 유사하게 구현)해야 함을 우리는 잘 알고 있습니다. 이번 Developer Policy 변경은 개발자 생태계에 명확성을 제공하고, 개발자가 공적 대화에 영향을 미치는 빌드와 혁신을 더 쉽게 할 수 있도록 개방형 API 플랫폼을 제공하기 위한 것입니다.

  이번 출시와, 이 시점에 이르기까지의 X API 출시 내역에 대해서는 [포럼 게시물](https://devcommunity.x.com/t/ushering-in-a-new-era-for-the-twitter-developer-platform-with-the-twitter-api-v2/162087)에서 자세히 다루고 있습니다.

<Update label="Jan 26, 2021" description="X API v2 학술 연구 제품 트랙 소개">
  ### 학술 연구 제품 트랙 소개

  오늘 [X API v2](/ko/x-api/introduction)에 새로운 [학술 연구 제품 트랙](/ko/resources/fundamentals/projects)을 소개합니다. 이 업데이트는 학술 연구자를 위한 새로운 신청 절차를 도입하며, 승인된 연구자에게 더 넓은 액세스와 고급 기능을 제공하는 학술 연구 [프로젝트](/ko/resources/fundamentals/projects)를 제공합니다.

  새로운 학술 연구 제품 트랙을 통해 다음 기능을 이용할 수 있습니다:

  - 새로운 X API v2 [전체 아카이브 검색 엔드포인트](/ko/x-api/posts/full-archive-search). 현재는 학술 연구 제품 트랙에서만 제공됩니다.

  - [기본](/ko/resources/fundamentals/projects) [액세스 레벨](https://developer.x.com/en/products/x-api/early-access/guide#na_2)에서 제공되는 50만 건에서 상향된 1,000만 건의 [트윗 한도(게시 한도)](/ko/x-api/fundamentals/post-cap).

  - [최근 검색](/ko/x-api/posts/recent-search), [전체 아카이브 검색](/ko/x-api/posts/full-archive-search), [필터링 스트림](/ko/x-api/posts/filtered-stream)에 사용할 수 있는 새로운 필터 연산자: `$`(cashtag), `bio`(필터링 스트림 전용), `bio_name`(필터링 스트림 전용), `bio_location`(필터링 스트림 전용), `place`, `place_country`, `point_radius`, `bounding_box`, `-is:nullcast`, `has:cashtags`, `has:geo`.

  - 필터링 스트림의 규칙 한도를 동시 1,000개로 상향. 기본 액세스 레벨의 표준 제품 트랙에서 제공되는 25개에서 증가합니다.

  - 쿼리 및 규칙의 문자 한도를 1,024자로 상향. 기본 액세스 레벨의 표준 제품 트랙에서 제공되는 512자 한도에서 증가합니다.

  이번 출시에 대한 자세한 내용은 [포럼 공지](https://devcommunity.x.com/t/introducing-the-new-academic-research-product-track/148632)를 방문하세요.
</Update>

<Update label="Dec 18, 2020" description="대화 답글 설정 포스트 필드(X API v2용)">
  ### 답글 설정 필드 추가

  오늘 포스트 객체에 reply\_settings라는 새 필드를 도입합니다. 이 필드는 포스트 조회, 답글 숨기기, 최근 검색, 샘플 스트림, 필터링 스트림, 팔로우 조회, 사용자 포스트 타임라인, 사용자 멘션 타임라인 등 포스트를 반환하는 모든 X API v2 엔드포인트에서 사용할 수 있으며, 어느 엔드포인트에서든 포스트 객체를 [확장](/ko/x-api/fundamentals/expansions)하여 포함할 수도 있습니다.

  이 필드는 포스트 작성자가 다른 사용자의 답글 허용 범위를 어떻게 설정했는지를 나타냅니다. 모든 사용자, 포스트에서 언급된 사용자만, 혹은 작성자가 팔로우하는 사용자만으로 제한할 수 있습니다. `reply_settings` 필드는 `everyone`, `mentionedUsers`, `following` 값을 반환합니다. 포스트에 대화 제어가 설정되지 않은 경우에도 이 필드는 포스트 객체에 `everyone` 값으로 표시됩니다.

  - GET /users의 단일 및 다중 사용자명 [lookup](/ko/x-api/migrate/overview#username-lookup-path)을 위한 경로 및 쿼리 매개변수 조정

  - 필드 이름을 `stats`에서 `public_metrics`로 변경

  기타 변경 사항:

  - 사용자에서 `most_recent_tweet_id` 확장 제거

  - 게시물의 `source` 필드에서 HTML 태그를 제거하여 파싱 용이성 향상

  최근 검색의 핵심 검색, 페이지네이션 및 기타 기능은 동일하게 유지됩니다.

  v1에서 v2로 업데이트할 때 참고할 [마이그레이션 자료](/ko/x-api/migrate/overview)를 준비했습니다.

  이 변경 사항에 대한 자세한 내용은 [포럼 공지](https://devcommunity.x.com/t/releasing-a-new-version-of-labs-endpoints/134219)에서 확인하실 수 있습니다.

<Update label="Feb 22, 2017" description="X API: Enterprise v1.1">
  ### 새로운 보강 기능과 페이로드 변경

  새로운 보강 기능: 이제 엔터프라이즈 API를 통해 설문조사 메타데이터가 제공됩니다. 자세한 내용은 [문서](/ko/x-api/enterprise-gnip-2.0/fundamentals/account-activity#migration-introduction)를 참조하세요.

  새로운 페이로드 필드: 사용자가 게시물에 사진을 추가할 때 "Add description" 옵션을 선택하면 페이로드의 entities.media\[] 및 extended\_entities.media\[] 섹션에 이미지 대체 텍스트(설명) 필드가 포함될 수 있습니다. 이는 보강된 네이티브 형식에서만 제공됩니다.

  기본 페이로드 형식: Gnip 제품에서 새로 생성된 스트림은 이제 기본적으로 original 형식 JSON을 사용합니다.
</Update>
