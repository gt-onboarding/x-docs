  ## 개요

<div id="introduction">
  ### 소개
</div>

X 개발자 포털에는 개발자가 X API 및 X Ads API에 대한 접근을 직접 관리할 수 있는 셀프 서비스 도구 모음이 포함되어 있습니다.

포털에서 다음을 수행할 수 있습니다:

- X [프로젝트](/ko/resources/fundamentals/projects)와 [앱](/ko/resources/fundamentals/developer-apps)(및 해당 인증 키와 토큰)을 생성하고 관리합니다.
- X API standard v1.1 및 v2 엔드포인트에 대한 접근 수준과 통합을 관리합니다.
- 제공되는 다양한 엔드포인트와 기능을 살펴봅니다.

<div id="ready-to-get-access">
  ### 액세스할 준비가 되셨나요?
</div>

계정을 등록하면 X API 사용을 시작할 수 있습니다.

추가 기능이나 더 높은 [트윗 한도](/ko/x-api/fundamentals/post-cap)가 필요한 경우 개발자 포털에서 Basic 또는 Pro를 구매할 수 있습니다. 엔터프라이즈에 관심이 있으시다면 [여기](https://docs.google.com/forms/d/e/1FAIpQLScO3bczKWO2jFHyVZJUSEGfdyfFaqt2MvmOfl_aJp0KxMqtDA/viewform)에서 신청해 주세요.

<div id="onboarding">
  ### 온보딩
</div>

온보딩 마법사는 첫 [프로젝트](/ko/resources/fundamentals/projects)와 [앱](/ko/resources/fundamentals/developer-apps) 설정 과정을 단계별로 안내합니다. API 요청을 인증하는 데 필요한 자격 증명을 받으려면 프로젝트와 앱을 생성해야 합니다. 개발자 계정에 처음 접속하는 경우 이 마법사가 표시됩니다.

이 과정에서 인증 키와 토큰 세트를 받게 되며, 자세한 내용은 [앱 개요 페이지](/ko/resources/fundamentals/developer-apps#overview)에서 확인할 수 있습니다. X API로 인증하는 데 필요한 사항은 [인증 섹션](/ko/resources/fundamentals/authentication)을 참조하세요.

**참고:** 나중에 다시 사용할 수 있도록 [키와 토큰을 안전한 위치에 보관](/ko/resources/fundamentals/authentication/guides/authentication-best-practices)하세요. 재생성하지 않으면 이러한 자격 증명을 다시 확인할 방법이 없습니다.

<div id="project-and-app-management">
  ### 프로젝트와 앱 관리
</div>

개발자 계정의 주요 역할 중 하나는 [프로젝트와 앱을 관리](/ko/resources/fundamentals/developer-apps#app-management)할 수 있도록 하는 것입니다. 개발자는 개발자 포털의 대시보드에서 X 프로젝트와 앱을 생성하고 관리할 수 있습니다. 여기에서 App ID를 확인하고, 앱의 설정, 권한, 콜백 URL을 수정하며, 키와 토큰을 생성하거나 취소할 수 있습니다.

<div id="learn-more-about-whats-available-with-x-api-v2">
  ### X API v2에서 제공되는 기능 자세히 알아보기
</div>

개발자 포털에는 X API의 다양한 버전과 액세스 수준을 더 자세히 확인할 수 있는 제품 섹션이 마련되어 있습니다.

X API v2 제품 섹션에서는 Free, Basic, Pro, Enterprise 액세스 등급에 관한 중요한 정보를 제공합니다. 이 페이지에는 프로젝트 수준 앱 제한, 트윗 한도(게시 한도), 비용뿐 아니라 엔드포인트별 요청 한도 및 특수 속성에 대한 세부 정보가 포함되어 있습니다. 또한 서로 다른 액세스 수준을 비교해 볼 수 있으며, 가능하다면 추가 액세스를 신청할 수도 있습니다.

[v2의 모든 액세스 수준](/ko/x-api/introduction#access-levels)을 확인하세요.

<div id="team-management">
  ## 팀 관리
</div>

<Note>
  **Enterprise v2 계정에서만 제공됩니다.**
</Note>

**[개발자 포털의 ‘팀’ 페이지 보기](https://developer.twitter.com/en/portal/teams).**

<div id="why-use-team-functionality">
  ### 팀 기능을 왜 사용하나요?
</div>

팀 기능은 X Developer Platform에서 프로젝트와 앱을 함께 개발할 수 있도록 지원합니다. 팀에는 접근 제어, 청구/결제 등 각기 다른 책임자가 있는 경우가 많으며, 이 기능을 통해 해당 담당자를 초대해 프로젝트에 참여시키고 기여하도록 할 수 있습니다.

<div id="inviting-team-members">
  ### 팀 구성원 초대
</div>

팀에 구성원을 초대하려면 관리자가 해당 사용자의 X 핸들로 초대할 수 있습니다. 초대 대상자는 이메일을 받고, 해당 이메일을 통해 초대를 수락할 수 있습니다. 수락 후에는 Developer Agreement & Policy에 동의해야 하며, 이후 메인 계정의 팀 페이지에 접근할 수 있습니다.

**참고:** 팀 관리는 현재 [App](/ko/resources/fundamentals/developer-apps) 자격 증명에 따른 API 액세스 권한을 부여하거나 제한하지 않습니다. 팀 전체에서 App 관리를 공유할 수 없습니다. App(키/토큰)은 소유자가 아닌 사용자가 편집, 생성 또는 삭제할 수 없습니다.

<div id="team-dashboard">
  ### 팀 대시보드
</div>

팀 대시보드의 "members" 탭에서는 모든 구성원과 각자의 역할을 확인할 수 있습니다. 팀의 관리자인 경우 개발자 접근 권한을 관리하고 각 구성원의 역할을 수정할 수 있습니다.

관리자는 팀 대시보드의 "pending" 탭에도 접근할 수 있습니다. 여기에서 관리자는 발송된 각 초대의 세부 정보를 확인하고 관리할 수 있습니다.

<div id="team-roles">
  ### 팀 역할
</div>

**관리자 역할**

- 팀 프로젝트와 앱을 관리할 수 있는 권한
- 모든 앱 환경을 관리할 수 있는 권한
- 구독 선택/업그레이드 권한
- 결제 수단/청구 정보 업데이트 권한
- 팀원 추가/제거 권한
- 팀원 역할 편집 권한

**개발자 역할**

- 본인 소유의 프로젝트와 앱을 관리할 수 있는 권한
- 팀 프로젝트와 앱에 대한 읽기 전용 권한
- 팀을 탈퇴할 수 있는 권한
