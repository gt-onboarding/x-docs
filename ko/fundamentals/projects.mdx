<div id="introduction">
  ## 소개
</div>

프로젝트는 X API의 사용 목적에 맞춰 작업을 구성하여 API 접근을 효율적으로 관리하고 사용량을 모니터링할 수 있도록 해줍니다. 각 프로젝트에는 액세스 수준(이 페이지에서 뒤에 설명됨)에 따라 하나 이상의 [앱](/ko/resources/fundamentals/developer-apps)을 포함할 수 있습니다. 이러한 앱을 사용해 [API 키와 시크릿](/ko/resources/fundamentals/authentication#api-key-and-secret), 사용자 [액세스 토큰](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow), [앱 액세스 토큰](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) 등의 [인증](/ko/resources/fundamentals/authentication) 자격 증명을 생성합니다.

이러한 키와 토큰은 어떤 앱에서든 X API 또는 X Ads API에 사용할 수 있지만(Ads API를 사용하려면 [추가 액세스 신청](/ko/x-ads-api/getting-started/step-by-step-guide)이 필요함), [X API v2](/ko/x-api/introduction) 엔드포인트를 사용하려면 해당 프로젝트에 연결된 앱에서 발급된 키와 토큰을 사용해야 합니다.

개발자 계정이 있다면 개발자 포털의 [Projects & Apps](https://developer.x.com/en/portal/projects-and-apps) 페이지에서 프로젝트를 확인하고 관리할 수 있습니다. 아직 계정이 없다면 [가입](https://developer.x.com/en/portal/petition/essential/basic-info)하세요.

<div id="projects-and-x-api-access-levels">
  ### 프로젝트와 X API 액세스 수준
</div>

현재 프로젝트 수준에 적용되는 액세스 등급은 다음의 4가지입니다:

- Free
- Basic
- Pro
- Enterprise

각 액세스 수준에서 제공되는 기능에 대해 자세히 알아보려면 [about X API](/ko/x-api/getting-started/about-x-api) 페이지를 방문하세요.

Free, Basic, Pro 가운데 프로젝트는 하나만 보유할 수 있습니다.

아래에서는 프로젝트별 차이점을 몇 가지 설명합니다:

<div id="free">
  #### 무료
</div>

이 액세스 티어는 개발자 계정에 [가입](https://developer.twitter.com/en/portal/petition/essential/basic-info)한 모든 사용자에게 제공됩니다.

해당 프로젝트 내 앱 수: 1
[트윗 한도(게시 한도)](/ko/x-api/fundamentals/post-cap): 월 1,500개 트윗

인증 방식:

- [OAuth 2.0 with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
- 앱 전용

<div id="basic">
  #### Basic
</div>

이 액세스 티어는 개발자 포털에서 Basic 액세스를 구독한 모든 사용자에게 제공됩니다.

해당 프로젝트의 앱 수: 2
[트윗 한도(게시 한도)](/ko/x-api/fundamentals/post-cap): 월 10,000건
인증 방식:

- [OAuth 2.0 with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
- [OAuth 1.0a](/ko/resources/fundamentals/authentication#oauth-1-0a-2)
- 앱 전용

<div id="pro">
  #### Pro
</div>

이 액세스 등급은 개발자 포털에서 Pro 액세스를 구독한 모든 사용자에게 제공됩니다.

해당 프로젝트 내 앱 수: 3
[트윗 소비 한도](/ko/x-api/fundamentals/post-cap): 월 100만 트윗

인증 방법:

- [OAuth 2.0 with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
- [OAuth 1.0a](/ko/resources/fundamentals/authentication#oauth-1-0a-2)
- 앱 전용

<div id="enterprise">
  #### 엔터프라이즈
</div>

이 액세스 등급은 엔터프라이즈를 신청해 당사 팀의 승인을 받고 X 계정 매니저를 통해 월간 자동 갱신 계약에 서명한 분께 제공됩니다. [여기에서 신청하세요](https://docs.google.com/forms/d/e/1FAIpQLScO3bczKWO2jFHyVZJUSEGfdyfFaqt2MvmOfl_aJp0KxMqtDA/viewform).

해당 프로젝트 내 앱 수: 3개 이상
[트윗 한도(게시 한도)](/ko/x-api/fundamentals/post-cap): 월 5천만 개 이상

인증 방식:

- [OAuth 2.0 with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
- [OAuth 1.0a](/ko/resources/fundamentals/authentication#oauth-1-0a-2)
- 앱 전용

<div id="tweet-caps-and-projects">
  ### 트윗 한도와 프로젝트
</div>

트윗 소비 한도는 프로젝트 단위로 적용되며, 해당 월 동안 특정 X API v2 엔드포인트에서 조회할 수 있는 트윗 수를 사실상 제한합니다.

[트윗 한도](/ko/x-api/fundamentals/post-cap)에 대해 자세히 알아보세요.

<div id="configuring-projects">
  ### 프로젝트 구성하기
</div>

<div id="creating-a-project">
  #### 프로젝트 만들기
</div>

프로젝트를 만들려면 [dashboard](https://developer.twitter.com/en/portal/dashboard)에서 "New Project"를 클릭하거나 개발자 포털의 [Projects & Apps](https://developer.twitter.com/en/portal/projects-and-apps) 페이지로 이동해 클릭하세요. 아직 프로젝트를 만들지 않았다면에만 이 옵션이 표시됩니다. 프로젝트 이름, 설명, 사용 사례를 입력하라는 안내가 표시됩니다. 또한 새 App을 만들거나 기존 독립형 App을 연결하라는 요청을 받게 됩니다.

<div id="creating-or-connecting-an-app-for-your-project">
  #### 프로젝트에 앱 생성 또는 연결
</div>

프로젝트에 앱이 없다면 대시보드에서 프로젝트 이름을 클릭해 추가할 수 있습니다. 여기에서 새 앱을 만들거나 기존 독립형 앱을 선택해 프로젝트에 연결할 수 있습니다. 이 앱에서 이 가이드 앞부분에 나열된 인증 키와 토큰을 생성합니다.

<div id="editing-a-project">
  #### 프로젝트 편집
</div>

프로젝트를 편집하려면 개발자 포털의 대시보드 또는 프로젝트와 앱 페이지에서 프로젝트 이름을 클릭하세요. 그러면 프로젝트 상세 정보 페이지가 열리며, 변경하려면 “Edit”를 선택할 수 있습니다.

<div id="standalone-apps">
  ### 독립 실행형 앱
</div>

독립 실행형 앱은 프로젝트 구조 밖에서 존재하는 앱입니다. 이 독립 실행형 앱에 연결된 인증 자격 증명은 X의 API 표준 v1.1, 프리미엄 v1.1, 엔터프라이즈 또는 X Ads API에 정상적으로 요청할 수 있습니다. 단, 프로젝트에 연결하지 않으면 X API v2 엔드포인트에 대한 요청은 실패합니다.

2020년 8월 이전에 앱을 생성했다면 개발자 포털의 [Projects & Apps](https://developer.twitter.com/en/portal/projects-and-apps) 아래 "Standalone Apps" 섹션에서 확인할 수 있습니다. 프로젝트에 연결된 앱을 포함해 총 앱 수는 10개로 제한됩니다.

팀 계정의 구성원인 경우, 독립 실행형 앱 섹션에서 본인이 소유한 앱을 볼 수 있습니다. 팀원이 프로젝트에 속한 앱을 소유한 경우, 해당 앱의 이름과 소유자 정보는 볼 수 있지만 설정을 변경하거나 키와 토큰에 접근하거나 이를 재발급할 수는 없습니다. 해당 앱에 변경이 필요하면 앱 소유자에게 문의하세요.
