X API는 방대한 데이터를 처리합니다. 이 데이터를 개발자와 사용자 모두에게 안전하게 보호하는 방법은 인증입니다. 인증에는 여러 방식이 있으며, 아래에 나열되어 있습니다.

대부분의 개발자는 클라이언트 라이브러리가 이러한 복잡한 부분을 자동으로 처리하므로 인증과 관련된 세부 사항을 직접 다룰 필요가 없습니다.

사용 가능한 클라이언트 라이브러리 목록은 [도구와 라이브러리](/ko/resources/tools-and-libraries) 페이지에서 확인할 수 있습니다.

    <br />

    OAuth 2.0 사용자 컨텍스트는 애플리케이션 범위에 대한 더 세밀한 제어와 여러 기기에 걸친 인가 플로우를 통해 다른 계정을 대신해 인증할 수 있도록 해줍니다.

    [**자세히 알아보기**](/ko/resources/fundamentals/authentication/oauth-2-0/authorization-code)
  </Card>

<Note>
  **참고:**
  앱의 API 키와 앱 전용 액세스 토큰, 개인 액세스 토큰과 액세스 토큰 시크릿은 [developer portal](/ko/resources/fundamentals/developer-portal)의 [X developer Apps](/ko/resources/fundamentals/developer-apps) 섹션에서 확인할 수 있습니다.

  **다른 사용자를 대신해 요청하려면**, 해당 사용자를 위해 [3-legged OAuth flow](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)를 사용하여 별도의 액세스 토큰 세트를 발급받고, 그 사용자의 토큰을 OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0 사용자 컨텍스트 요청에 함께 전달해야 합니다.
</Note>
