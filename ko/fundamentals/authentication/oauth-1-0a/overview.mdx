<div id="oauth-10a">
  ## OAuth 1.0a
</div>

X 개발자 플랫폼의 많은 엔드포인트는 X 계정을 대신해 동작하거나 API 요청을 수행하기 위해 OAuth 1.0a 방식을 사용합니다. 예를 들어 X 개발자 앱이 있다면, 사용자가 앱을 인증하기만 하면 어떤 X 계정이든 그 계정을 대신해 API 요청을 보낼 수 있습니다.

참고: HMAC-SHA1 및 퍼센트 인코딩 같은 개념에 익숙하지 않다면, 인증 과정을 크게 단순화해 주는 API 클라이언트를 소개한 아래의 "유용한 도구" 섹션을 확인하시길 권장합니다.

<div id="key-concepts">
  ### 핵심 개념
</div>

<div id="signing-a-request-with-keys-and-tokens">
  #### 키와 토큰으로 요청 서명하기
</div>

각 API 요청은 생성된 여러 키와 토큰을 인증 헤더에 포함해 서명해야 합니다. 시작하려면 [X 개발자 앱](/ko/resources/fundamentals/developer-apps) 상세 페이지에서 다음을 포함한 여러 키와 토큰을 생성할 수 있습니다:

|     |     |
| :--- | :--- |
| API 키와 시크릿:<br /><br />`oauth_consumer_key`<br /><br />`oauth_consumer_secret` | API 요청 시 귀하의 X 개발자 앱을 나타내는 사용자 이름과 비밀번호에 해당한다고 생각하면 됩니다. |
| 액세스 토큰과 시크릿:<br /><br />`oauth_token`<br /><br />`oauth_token_secret` | 액세스 토큰과 액세스 토큰 시크릿은 OAuth 1.0a API 요청 인증에 사용되는 사용자별 자격 증명입니다. 요청을 어떤 X 계정을 대신해 수행하는지 지정합니다.<br /><br />앱이 개발자 계정과 연결된 동일한 X 계정을 대신해 요청을 보내도록 하려면 해당 상세 페이지에서 본인의 액세스 토큰과 토큰 시크릿을 생성할 수 있습니다.<br /><br />다른 사용자의 액세스 토큰을 생성하려면 아래의 "사용자를 대신해 요청하기"를 참조하세요. |

<div id="making-requests-on-behalf-of-users">
  #### 사용자 대신 요청 보내기
</div>

서명을 생성하려면, 대신 요청을 보낼 사용자를 대표하는 액세스 토큰 세트가 필요합니다.

[X 개발자 앱을 소유한 X 계정을 대표하는 액세스 토큰 세트는 앱의 세부 정보 페이지](https://developer.x.com/content/developer-twitter/en/apps)에서 생성할 수 있습니다. 다른 X 계정 대신 요청하려는 경우에는, 해당 계정의 소유자가 [3-legged OAuth 플로우](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)의 일부로 자신의 계정에 로그인해 여러분에게 접근 권한을 부여해야 합니다. 이 과정의 결과물은 OAuth 1.0a 요청에 사용할 수 있는 액세스 토큰 세트(oauth\_token 및 oauth\_token\_secret)입니다.

이러한 키와 토큰을 확보했다면 처음부터 [서명을 생성](/ko/resources/fundamentals/authentication#creating-a-signature)할 수 있습니다. 다만, 해당 절차에 숙련된 경우가 아니라면 권장하지 않으며, 아래에 언급된 도구 중 하나를 사용해 OAuth 1.0a가 필요한 엔드포인트에 요청하는 경우를 예외로 합니다.

참고용으로, 생성된 모든 토큰을 Authorization 헤더에 포함한 서명된 cURL 요청 예시는 다음과 같습니다:

```bash
curl --request POST \
  --url 'https://api.x.com/1.1/statuses/update.json?status=Hello%20world' \
  --header 'authorization: OAuth oauth_consumer_key="CONSUMER_API_KEY", oauth_nonce="OAUTH_NONCE", oauth_signature="OAUTH_SIGNATURE", oauth_signature_method="HMAC-SHA1", oauth_timestamp="OAUTH_TIMESTAMP", oauth_token="ACCESS_TOKEN", oauth_version="1.0"' \
```

<Note>
  **참고:** 사용자 액세스 토큰은 민감한 정보이므로 각별히 보호해야 합니다. 액세스 토큰이 생성되는 순간, 해당 사용자는 여러분의 애플리케이션이 이를 안전하게 보관할 것이라고 신뢰하는 것입니다. API 키와 사용자 액세스 토큰의 보안이 모두 침해되면, 애플리케이션을 통해 개인 정보와 계정 기능에 대한 접근이 노출될 수 있습니다. [키와 액세스 토큰 보안](/ko/resources/fundamentals/developer-apps)에 대해 자세히 알아보시길 권장합니다.
</Note>

<div id="useful-tools">
  ### 유용한 도구
</div>

[요청 서명](/ko/resources/fundamentals/authentication#creating-a-signature) 과정은 복잡합니다. 인증 토큰의 상당 부분을 자동으로 생성해 주는 API 클라이언트 라이브러리 사용을 권장합니다:

|     |     |
| :--- | :--- |
| [Postman](https://www.getpostman.com/) | REST API 요청을 구성하고 전송할 수 있는 API 클라이언트입니다. 이 도구에 대해 자세히 알아보려면 “[Postman 시작하기](/ko/tutorials/postman-getting-started)” 튜토리얼을 참조하세요. |
| [Insomnia](https://insomnia.rest/) | Insomnia는 쿠키 관리, 환경 변수, 코드 생성, 인증을 지원하는 REST API 클라이언트로, macOS, Windows, Linux에서 사용할 수 있습니다. |
