<div id="api-key-and-secret">
  ### API 키와 시크릿
</div>

API 키와 시크릿(Consumer Key와 Secret이라고도 함)은 X API에 접근하는 데 필요한 가장 기본적인 자격 증명입니다. 이 자격 증명은 X 앱의 사용자 이름과 비밀번호에 해당하며, X API는 이를 통해 어떤 앱에서 요청이 들어왔는지를 식별합니다.

이 자격 증명은 [인증 엔드포인트](/ko/resources/fundamentals/authentication/api-reference)에서 [사용자 액세스 토큰과 시크릿](/ko/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret), [베어러 토큰](/ko/resources/fundamentals/authentication/oauth-2-0/bearer-tokens)과 같은 추가 자격 증명을 생성하는 데 사용할 수 있습니다. 또한 OAuth 1.0a 사용자 컨텍스트 인증이 필요한 요청을 [인가](/ko/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request)하려면 액세스 토큰 및 기타 권한 부여 매개변수와 함께 이 자격 증명을 사용해야 합니다. 

<div id="how-to-acquire-an-api-key-and-secret">
  #### API 키와 시크릿을 얻는 방법
</div>

X API 키와 시크릿을 얻으려면 다음 단계를 따르세요:

1. [X 개발자 계정에 가입](https://developer.x.com/en/apply-for-access)
2. [개발자 포털](/ko/resources/fundamentals/developer-portal)에서 [X App](/ko/resources/fundamentals/developer-apps)을 생성하세요. [X API v2](/ko/x-api/introduction)를 사용하려면 X App을 [프로젝트](/ko/resources/fundamentals/projects)에 추가해야 합니다.
    

X App을 생성하면 API 키와 시크릿, 그리고 Bearer 토큰이 제공됩니다. 이 자격 증명은 한 번만 표시되므로 비밀번호 관리자 등 안전한 곳에 반드시 저장하세요.

자격 증명이 유출된 경우의 대처 방법을 포함해 키와 토큰을 안전하게 관리하는 방법은 [인증 모범 사례](/ko/resources/fundamentals/authentication/guides/authentication-best-practices) 페이지에서 확인할 수 있습니다.
 

<div id="how-to-find-and-regenerate-your-api-key-and-secret-after-app-creation">
  #### 앱 생성 후 API Key와 Secret을 찾고 재생성하는 방법
</div>

이미 앱을 생성했고 API Key와 Secret을 찾거나 재생성해야 하는 경우, 다음 단계를 따르세요.

1. 개발자 포털로 이동합니다.
2. 사이드 내비게이션에서 ‘프로젝트와 앱’ 드롭다운을 확장합니다.
3. 찾거나 재생성하려는 API Key와 Secret이 연결된 앱을 엽니다.
4. Keys and tokens 탭으로 이동합니다.

이 페이지에서 앱과 연계된 모든 자격 증명을 확인할 수 있습니다. 
 

<div id="how-to-use-your-api-key-and-secret">
  #### API 키와 시크릿 사용 방법
</div>

X Developer Platform을 탐색 중이라면, 플랫폼에서 제공되는 기능을 살펴보기 위해 [도구나 라이브러리](/ko/resources/tools-and-libraries)를 사용하는 것을 권장합니다. 이러한 도구는 인증을 원활하게 처리하여 많은 시간과 불편을 줄여 줍니다. 특히 초보 개발자에게는 [Postman 시작하기](/ko/tutorials/postman-getting-started) 또는 [Insomnia](https://insomnia.rest/)를 권장합니다.

처음부터 직접 요청을 만들고자 한다면 [OAuth 1.0a 요청 권한 부여](/ko/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request) 가이드를 읽어 보세요.
