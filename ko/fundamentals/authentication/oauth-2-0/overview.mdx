<div id="bearer-token-also-known-as-app-only">
  ### 베어러 토큰(앱 전용이라고도 함)
</div>

OAuth 2.0 베어러 토큰은 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 대신하여 요청을 인증합니다. 이 방법은 앱에만 해당하므로 사용자가 관여하지 않습니다. 일반적으로 공개 정보에 읽기 전용으로 접근해야 하는 개발자가 사용합니다.

이 인증 방법을 사용하려면 요청에 베어러 토큰을 포함해야 하며, 개발자 앱의 Keys and tokens 섹션에서 생성할 수 있습니다. 다음은 가짜 베어러 토큰을 사용한 요청 예시입니다:

```json
  curl "https://api.x.com/2/tweets?ids=1261326399320715264,1278347468690915330" \
  -H "Authorization: Bearer AAAAAAAAAAAAAAAAAAAAAFnz2wAAAAAAxTmQbp%2BIHDtAhTBbyNJon%2BA72K4%3DeIaigY0QBrv6Rp8KZQQLOTpo9ubw5Jt?WRE8avbi"
```

앱 전용 인증을 사용하는 API 호출은 앱 수준에서 엔드포인트별로 [요청 한도](/ko/resources/fundamentals/rate-limits)가 적용됩니다.

이 방법을 사용하려면 베어러 토큰이 필요합니다. [POST oauth2/token](/ko/resources/fundamentals/authentication/api-reference#post-oauth2-token) 엔드포인트를 통해 API 키와 시크릿을 전달하여 생성하거나, [개발자 포털](/ko/resources/fundamentals/developer-portal)의 앱 설정에 있는 "keys and tokens" 섹션에서 생성할 수 있습니다.

베어러 토큰을 폐기하려면 [POST oauth2/invalidate\_token](/ko/resources/fundamentals/authentication/api-reference#post-oauth2-invalidate-token) 엔드포인트를 사용하거나, 개발자 포털의 앱 설정 "keys and tokens" 섹션에서 베어러 토큰 옆의 "revoke"를 클릭하세요.

<div id="oauth-20-authorization-code-flow-with-pkce">
  ### OAuth 2.0 Authorization Code Flow with PKCE
</div>

OAuth 2.0 Authorization Code Flow with PKCE는 다른 사용자를 대신해 인증할 수 있도록 하며, 애플리케이션의 스코프에 대한 제어를 강화하고 여러 기기에서의 인가 흐름을 개선합니다. 즉, X를 위한 애플리케이션을 개발하는 개발자는 앱이 사용자에게 요청하는 정보 범위를 보다 정밀하게 제어할 수 있어, 최종 사용자에게 꼭 필요한 데이터와 정보만 요청하면 됩니다.

이 현대적 인가 프로토콜을 사용하면 최종 사용자에게 앱 인가를 위한 더 간소화된 동의 흐름을 제공할 수 있으며, 여기에는 여러분이 요청한 특정 스코프만 표시됩니다. 이는 데이터 수집 부담을 줄일 뿐만 아니라 최종 사용자의 신뢰 향상으로도 이어질 수 있습니다.
