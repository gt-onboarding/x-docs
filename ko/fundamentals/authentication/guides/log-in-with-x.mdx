X로 로그인(Log in with X), 또는 X로 로그인(Sign in with X)을 사용하면 사이트나 애플리케이션에 버튼을 추가하여 X 사용자가 한 번의 클릭만으로 등록 사용자 계정의 혜택을 누릴 수 있습니다. 이 기능은 웹사이트, iOS, 모바일, 데스크톱 애플리케이션에서 모두 사용할 수 있습니다.

![X로 로그인 버튼](https://cdn.cms-twdigitalassets.com/content/dam/developer-twitter/auth-docs/sign-in-with-twitter-gray.png.twimg.1920.png)

<div id="features">
  ## 기능
</div>

- 사용 편의성 - 사이트 신규 방문자는 처음 로그인할 때 버튼 두 번만 클릭하면 됩니다.
- X 연동 - Log in with X 플로우를 통해 사용자 대신 X API 사용 권한을 부여할 수 있습니다.
- OAuth 기반 - 다양한 클라이언트 라이브러리와 예제 코드가 Log in with X API와 호환됩니다.

<div id="available-for">
  ## 제공 대상
</div>

- 브라우저 - 사용자가 브라우저를 사용할 수 있다면 Log in with X를 통합할 수 있습니다. 브라우저 로그인 플로우를 확인하세요.
- 모바일 기기 - 인터넷에 연결된 모든 모바일 기기에서 Log in with X를 사용할 수 있습니다. 모바일 로그인 플로우를 확인하세요.

<div id="implementing-log-in-with-x">
  ## X로 로그인 구현
</div>

브라우저 및 모바일 웹에서의 X로 로그인 구현은 OAuth를 기반으로 합니다. 이 페이지는 로그인 플로우에서 액세스 토큰을 얻기 위해 필요한 요청을 설명합니다.

“X로 로그인” 플로우를 사용하려면 [X 앱 설정](/ko/resources/fundamentals/developer-apps)으로 이동해 _"이 앱을 X로 로그인을 위해 사용할 수 있도록 허용하시겠습니까?"_ 옵션이 활성화되어 있는지 확인하세요.

이 페이지는 독자가 OAuth 1.0a 프로토콜을 사용해 요청에 서명하는 방법을 알고 있다고 가정합니다. 요청 서명 방법을 알고 싶다면 [요청 승인](/ko/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request) 페이지를 참조하세요.

이 페이지의 요청 서명을 검증하려면 다음 컨슈머 시크릿을 사용하세요: L8qq9PZyRg6ieKGEKhZolGC0vJWLw8iEJ88DRdyOg. 이 값은 테스트용이며 실제 요청에는 작동하지 않습니다.

요청 토큰을 획득하고, 사용자를 리디렉션하며, 요청 토큰을 액세스 토큰으로 전환하는 세 단계로 X로 로그인을 구현하는 절차는 아래에 나와 있습니다.

<Tabs>
  <Tab title="Step 1">
    ### 1단계: 요청 토큰 얻기

    사인인 플로우를 시작하려면 [X 앱](/ko/resources/fundamentals/developer-apps)이 서명된 메시지를 [POST oauth/request\_token](/ko/resources/fundamentals/authentication/api-reference#post-oauth-request-token)으로 보내 요청 토큰을 받아야 합니다. 이 요청에서 유일하게 고유한 매개변수는 oauth\_callback이며, 이는 사용자가 2단계를 완료했을 때 리디렉션할 URL을 URL 인코딩한 값이어야 합니다. 나머지 매개변수는 OAuth 서명 과정에서 추가됩니다.

    <Note>
      **참고:** [POST oauth/request\_token](/ko/resources/fundamentals/authentication/api-reference#post-oauth-request-token) 엔드포인트와 함께 사용하는 모든 [콜백 URL](/ko/resources/fundamentals/developer-apps#callback-urls)은 [개발자 포털](/ko/resources/fundamentals/developer-portal)의 [X 앱 설정](/ko/resources/fundamentals/developer-apps)에 등록되어 있어야 합니다.
    </Note>

    <Frame>
      <img src="/images/auth-4.png.twimg.1920.png" alt="" />
    </Frame>

    **요청 예시(Authorization 헤더는 줄바꿈 처리됨):**

    ```
    POST /oauth/request_token HTTP/1.1
    User-Agent: themattharris' HTTP Client
    Host: api.x.com
    Accept: */*
    Authorization:
            OAuth oauth_callback="http%3A%2F%2Flocalhost%2Fsign-in-with-twitter%2F",
                  oauth_consumer_key="cChZNFj6T5R0TigYB9yd1w",
                  oauth_nonce="ea9ec8429b68d6b77cd5600adbbb0456",
                  oauth_signature="F1Li3tvehgcraF8DMJ7OyxO4w9Y%3D",
                  oauth_signature_method="HMAC-SHA1",
                  oauth_timestamp="1318467427",
                  oauth_version="1.0"
    ```

    앱은 응답의 HTTP 상태 코드를 확인해야 합니다. 200이 아닌 값은 실패를 의미합니다. 응답 본문에는 oauth\_token, oauth\_token\_secret, oauth\_callback\_confirmed 매개변수가 포함됩니다. 앱은 oauth\_callback\_confirmed가 true인지 확인하고, 나머지 두 값을 다음 단계에 사용할 수 있도록 저장해야 합니다.

    **응답 예시(응답 본문은 줄바꿈 처리됨):**

    ```
    HTTP/1.1 200 OK
    Date: Thu, 13 Oct 2011 00:57:06 GMT
    Status: 200 OK
    Content-Type: text/html; charset=utf-8
    Content-Length: 146
    Pragma: no-cache
    Expires: Tue, 31 Mar 1981 05:00:00 GMT
    Cache-Control: no-cache, no-store, must-revalidate, pre-check=0, post-check=0
    Vary: Accept-Encoding
    Server: tfe

    oauth_token=NPcudxy0yU5T3tBzho7iCotZ3cnetKwcTIRlX0iwRl0&
    oauth_token_secret=veNRnAWe6inFuo8o2u8SLLZLjolYDmDP7SzL0YfYI&
    oauth_callback_confirmed=true

    ```

  <Tab title="Step 2">
    ### 2단계: 사용자 리디렉션

    다음 단계는 사용자가 적절한 플로우를 완료할 수 있도록 아래의 브라우저 로그인 플로우에 설명된 대로 사용자를 X로 보내는 것입니다. 사용자를 [GET oauth/authenticate](/ko/resources/fundamentals/authentication/api-reference#get-oauth-authenticate)로 이동시키고, 1단계에서 획득한 요청 토큰을 oauth\_token 매개변수로 전달해야 합니다.

    웹사이트에서 이를 구현하는 가장 원활한 방법은 원래의 “로그인” 요청에 대한 응답으로 HTTP 302 리디렉트를 반환하는 것입니다. 모바일 및 데스크톱 앱은 새 브라우저 창을 열거나 임베디드 웹뷰를 통해 해당 URL을 열어야 합니다.

    **리디렉션 대상 예시 URL:**

    https://api.x.com/oauth/authenticate?oauth\_token=NPcudxy0yU5T3tBzho7iCotZ3cnetKwcTIRlX0iwRl0

    로그인 엔드포인트는 사용자의 상태에 따라 다음 세 가지 중 하나로 동작합니다.

    1. **로그인됨 및 승인됨**: 사용자가 x.com에 로그인되어 있고 호출 애플리케이션을 이미 승인한 경우, 즉시 인증되며 유효한 OAuth 요청 토큰과 함께 콜백 URL로 반환됩니다. x.com으로의 리디렉트는 사용자에게 눈에 띄지 않습니다.
    2. **로그인됨이나 미승인**: 사용자가 x.com에 로그인되어 있지만 호출 애플리케이션을 승인하지 않은 경우, 호출 애플리케이션에 대한 접근 공유 요청이 표시됩니다. 권한 부여를 수락하면 유효한 OAuth 요청 토큰과 함께 콜백 URL로 리디렉트됩니다.
    3. **미로그인**: 사용자가 x.com에 로그인되어 있지 않은 경우, 동일한 화면에서 자격 증명 입력과 애플리케이션의 정보 접근 허용을 요청받습니다. 로그인 완료 후 유효한 OAuth 요청 토큰과 함께 콜백 URL로 반환됩니다.

    인증이 성공하면, callback\_url로 oauth\_token 및 oauth\_verifier 매개변수를 포함한 요청이 전송됩니다. 애플리케이션은 이 토큰이 1단계에서 받은 요청 토큰과 일치하는지 확인해야 합니다.

    **클라이언트 리디렉트에서의 요청(쿼리스트링 매개변수 줄바꿈 처리):**

    ```
    GET /sign-in-with-twitter/?
            oauth_token=NPcudxy0yU5T3tBzho7iCotZ3cnetKwcTIRlX0iwRl0&
            oauth_verifier=uw7NjWHT6OJ1MpJOXsHfNxoAhPKpgI8BlYDhxEjIBY HTTP/1.1
    Host: localhost
    User-Agent: Mozilla/5.0 (Macintosh; Intel Mac OS X 10_6_8) AppleWebKit/535.5 (KHTML, like Gecko) Chrome/16.0.891.1 Safari/535.5
    Accept: text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8
    Referer: http://localhost/sign-in-with-twitter/
    Accept-Encoding: gzip,deflate,sdch
    Accept-Language: en-US,en;q=0.8
    Accept-Charset: ISO-8859-1,utf-8;q=0.7,*;q=0.3
    ```
  </Tab>

  <Tab title="Step 3">
    ### 3단계: 요청 토큰을 액세스 토큰으로 변환

    요청 토큰을 실제로 사용할 수 있는 액세스 토큰으로 전환하려면, 애플리케이션이 2단계에서 획득한 oauth\_verifier 값을 포함해 [POST oauth/access\_token](/ko/resources/fundamentals/authentication/api-reference#post-oauth-access-token) 엔드포인트로 요청을 보내야 합니다. 요청 토큰은 헤더의 oauth\_token 항목에도 전달되며, 이는 서명 과정에서 자동으로 추가됩니다.

    **예시 요청(Authorization 헤더 줄바꿈 적용):**

    ```
    POST /oauth/access_token HTTP/1.1
    User-Agent: themattharris' HTTP Client
    Host: api.x.com
    Accept: */*
    Authorization: OAuth oauth_consumer_key="cChZNFj6T5R0TigYB9yd1w",
                          oauth_nonce="a9900fe68e2573b27a37f10fbad6a755",
                          oauth_signature="39cipBtIOHEEnybAR4sATQTpl2I%3D",
                          oauth_signature_method="HMAC-SHA1",
                          oauth_timestamp="1318467427",
                          oauth_token="NPcudxy0yU5T3tBzho7iCotZ3cnetKwcTIRlX0iwRl0",
                          oauth_version="1.0"
    Content-Length: 57
    Content-Type: application/x-www-form-urlencoded

    oauth_verifier=uw7NjWHT6OJ1MpJOXsHfNxoAhPKpgI8BlYDhxEjIBY
    ```

    성공적인 응답에는 oauth\_token, oauth\_token\_secret 매개변수가 포함됩니다. 토큰과 토큰 시크릿은 저장해 두고, 향후 X API에 대한 인증 요청에 사용해야 합니다. 사용자의 신원을 확인하려면 [GET account/verify\_credentials](https://dev.x.com/rest/reference/get/account/verify_credentials)를 사용하십시오.

    **예시 응답(응답 본문 줄바꿈 적용):**

    ```
    HTTP/1.1 200 OK
    Date: Thu, 13 Oct 2011 00:57:08 GMT
    Status: 200 OK
    Content-Type: text/html; charset=utf-8
    Content-Length: 157
    Pragma: no-cache
    Expires: Tue, 31 Mar 1981 05:00:00 GMT
    Cache-Control: no-cache, no-store, must-revalidate, pre-check=0, post-check=0
    Vary: Accept-Encoding
    Server: tfe

    oauth_token=7588892-kagSNqWge8gB1WwE3plnFsJHAZVfxWD7Vb57p0b4&
    oauth_token_secret=PbKfYqSryyeKDWz4ebtY3o5ogNLG11WJuZBc9fQrQo
    ```

  <Tab title="Browser sign in flow">
    브라우저 로그인 플로우는 웹 브라우저를 열거나 임베드할 수 있는 웹사이트와 애플리케이션에 적합합니다. 개요는 다음과 같습니다.

    - 애플리케이션이 “X로 로그인” 링크 또는 버튼을 표시합니다.
    - 사용자가 로그인 버튼을 클릭합니다.
    - 현재 웹 브라우저가 X로 리디렉션되거나(또는 새 브라우저가 열려 X로 이동합니다).
    - 필요한 경우 사용자는 X에서 로그인 및 권한 부여 단계를 완료합니다.
    - X가 사용자에 대한 권한 정보를 포함해 애플리케이션이 제어하는 URL로 다시 리디렉션합니다.

    X는 권한 상태를 추적하므로, 이미 X.com에 로그인했고 애플리케이션을 승인한 사용자의 경우 UI가 표시되지 않고 자동으로 애플리케이션으로 리디렉션됩니다.

    ### 데스크톱 플로우

    플로우를 설명하기 위해, 위에 표시된 웹사이트(“지금까지 만들어진 최고의 웹사이트”)가 랜딩 페이지의 X로 로그인 버튼에서 볼 수 있듯이 이 API를 구현했다고 가정해 보겠습니다.

    사용자가 로그인 버튼을 클릭하면, 표시되는 페이지는 사용자가 로그인했는지와 애플리케이션에 계정 액세스를 이전에 허용했는지에 따라 달라집니다.

    사용자가 x.com에 로그인했지만 액세스를 부여하지 않은 경우, 요청된 권한 목록과 함께 로그인 및 취소 버튼이 표시됩니다.

    사용자가 x.com에 로그인하지 않은 경우 사용자 이름과 비밀번호 입력란이 표시됩니다. 사용자가 이미 애플리케이션에 액세스를 허용한 경우에도 권한 목록은 계속 표시됩니다.
