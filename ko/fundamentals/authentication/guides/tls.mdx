X API 엔드포인트에 액세스하려면 TLS 연결이 필요합니다. TLS를 통한 통신은 공개 인터넷을 거쳐 사용자와 X API 사이를 오가는 정보를 보호하여 사용자 개인정보와 보안을 유지합니다. X API에 대한 연결에는 TLS 1.2 버전이 필요합니다.

<div id="use-an-up-to-date-root-store">
  ### 최신 루트 스토어 사용
</div>

애플리케이션 또는 라이브러리에서 X 인증서를 검증할 때에는 신뢰할 수 있고 최신 상태의 루트 스토어를 사용하는 것이 중요합니다. 가능하다면 운영체제에서 제공하는 루트 스토어를 사용하는 것이 가장 간단한 방법일 수 있습니다. 대안으로, [Mozilla(NSS) 루트 스토어](https://www.mozilla.org/en-US/about/governance/policies/security-group/certs/)는 공개적이고 투명한 방식으로 잘 유지·관리되고 있습니다. curl은 또한 [PEM 형식으로 변환된 해당 스토어](https://curl.haxx.se/docs/caextract.html)를 제공합니다.

현재 X는 대부분의 인증서를 [DigiCert High Assurance EV Root CA](https://www.digicert.com/digicert-root-certificates.htm)에서 발급하고 있으나, 이는 X 관련 모든 인증서(100%)에 해당하지 않으며 앞으로도 계속 동일하다고 보장할 수 없습니다. 따라서 현재 사용 중인 DigiCert 루트만 신뢰하도록 구성하면 향후 앱에서 문제가 발생할 수 있습니다.

<div id="check-crls-and-the-ocsp-status">
  ### CRL 및 OCSP 상태 확인[](#check-crls-and-the-ocsp-status "Permalink to this headline")
</div>

많은 애플리케이션은 반환된 인증서에 대해 인증서 폐기 목록(CRL)을 확인하지 않거나, 확인을 운영체제에 맡깁니다. 애플리케이션 또는 TLS 라이브러리가 X의 인증서를 수락하기 전에 CRL 및 OCSP(Online Certificate Status Protocol) 검증을 반드시 수행하도록 구성되어 있는지 확인하십시오.

<div id="cdns">
  ### CDN[](#cdns "이 표제에 대한 고정 링크")
</div>

미디어가 포함된 트윗을 표시할 때는 이미지를 표시할 HTTPS URL로 `media_url_https` 속성을 사용하세요. 앞으로 API 엔드포인트에서 제공되는 모든 URL은 HTTPS 경로를 제공하게 됩니다.

<div id="provide-an-indication-of-security-status">
  ## 보안 상태를 표시하세요
</div>

가능하다면 애플리케이션과 X 간의 현재 보안 상태를 표시해야 합니다. 일부 웹 브라우저는 자물쇠 아이콘으로 이를 나타내며, 다른 브라우저는 설명 메시지로 현재 연결 상태를 보여줍니다.
