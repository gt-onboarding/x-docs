title: X ID

---

X의 각 객체—트윗, 다이렉트 메시지, 사용자, 리스트 등—에는 고유한 ID가 있습니다.

플랫폼 초기에는 이러한 ID가 충분히 작은 숫자였기 때문에 순차적으로 생성할 수 있었습니다. 시간이 흐르며 성장에 맞추기 위해 ID는 32비트에서 64비트로 전환되었습니다. 현재 X ID는 순차적이 아닌 시간 기반의 고유한 64비트 부호 없는 정수입니다. 전체 ID는 타임스탬프, 워커 번호, 시퀀스 번호로 구성됩니다. X는 이러한 ID를 일관되게 생성하기 위해 “Snowflake”라는 내부 서비스를 개발했습니다([자세한 내용은 X 블로그 참고](https://blog.x.com/engineering/en_us/a/2010/announcing-snowflake.html)).

64비트처럼 큰 숫자는 정수를 64비트 미만으로 표현하는 프로그래밍 언어에서 문제를 일으킬 수 있습니다. 예를 들어 JavaScript에서는 정수가 53비트로 제한됩니다. 이를 해결하기 위한 우회책으로, 초기 X API(v1/1.1) 설계에서는 ID 값을 정수와 문자열 두 가지 형식으로 반환했습니다.

```json
{"id": 10765432100123456789, "id_str": "10765432100123456789"}
```

브라우저의 JavaScript 콘솔에서 `(10765432100123456789).toString()` 명령을 실행하면 결과는 `"10765432100123458000"`이 됩니다. 64비트 정수는 변환 과정에서 정확도가 떨어집니다(이를 때때로 “munging”이라고 하며, 데이터에 대한 파괴적 변경을 의미합니다).

X API 1.1 이하 버전에서는 정확도 손실을 피하기 위해 항상 숫자의 문자열 표현을 사용해야 합니다.

신규 버전의 API에서는 모든 큰 정수 값이 기본적으로 문자열로 표현됩니다.
