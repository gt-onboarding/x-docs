<div id="tweet">
  ### Tweet
</div>

트윗은 X의 모든 것을 구성하는 기본 단위입니다. Tweet 객체에는 `id`, `text`, `created_at`과 같은 ‘루트 수준’ 필드가 있습니다. Tweet 객체는 `user`, `media`, `poll`, `place` 등을 포함한 여러 하위 객체의 ‘부모’ 객체이기도 합니다. Tweet 객체에서 이러한 루트 수준 필드를 요청할 때는 필드 매개변수 `tweet.fields`를 사용하세요.

Tweet 객체는 사용자 리소스에서 찾아 확장할 수 있습니다. 요청한 트윗과 관련된 추가 트윗은 Tweet 리소스에서도 찾아 확장할 수 있습니다. 이 객체는 사용자 리소스에서는 `?expansions=pinned_tweet_id`, Tweet 리소스에서는 `?expansions=referenced_tweets.id`를 사용해 기본 필드만 포함된 상태로 확장할 수 있습니다. 객체를 완성하기 위해 추가 필드를 요청할 때는 확장과 함께 필드 매개변수 `tweet.fields`를 사용하세요.

| Field Value           | Type   | Description                                                                                                                           | How it Can Be Used                                                  |
|:-----------------------|:--------|:---------------------------------------------------------------------------------------------------------------------------------------|:----------------------------------------------------------------------|
| **id (default)**      | string | 요청된 트윗의 고유 식별자입니다.                                                                                         | 이를 사용해 특정 트윗을 프로그래밍 방식으로 조회합니다.               |
| **text (default)**    | string | 트윗의 실제 UTF-8 텍스트입니다. 유효한 문자에 대한 자세한 내용은 [twitter-text](https://github.com/twitter/twitter-text/)를 참조하세요.   | 키워드 추출 및 감성 분석/분류.                                       |
| **edit\_history\_tweet\_ids (default)** | object | 트윗의 모든 버전을 나타내는 고유 식별자입니다. 편집이 없는 트윗은 ID가 하나이며, 편집 기록이 있는 트윗은 여러 개의 ID를 가집니다. | 이 정보를 사용해 트윗의 편집 이력을 찾습니다.                     |
| **attachments**       | object | 이 트윗에 포함된 첨부 유형(있는 경우)을 지정합니다.                                                                    | 요청한 확장에 대해 반환되는 객체를 이해하는 데 활용.                 |
| **author\_id**         | string | 이 트윗을 게시한 사용자의 고유 식별자입니다.                                                                             | 사용자 객체 하이드레이션, 동료 검토용 데이터셋 공유.                 |
| **context\_annotations** | array | 트윗의 컨텍스트 주석을 포함합니다.                                                                                          | 엔터티 인식/추출, 주제 분석.                                         |
| **conversation\_id**   | string | 대화의 원본 트윗(직접 답글 및 그 답글 포함)의 트윗 ID입니다.                         | 이를 사용해 트윗에서 대화를 재구성합니다.                             |
| **created\_at**        | date (ISO 8601) | 트윗의 생성 시각입니다.                                                                                                      | 시계열 분석 및 트윗 생성 시점 파악에 유용합니다.                      |
| **edit\_controls**     | object | 트윗을 더 얼마나 편집할 수 있는지와 남은 편집 횟수를 나타냅니다.                                                | 트윗이 편집 가능 대상인지 판단하는 데 사용합니다.                    |
| **entities**          | object | 트윗 텍스트에서 파싱된 엔터티입니다. Twitter Objects의 엔터티를 참조하세요.                                       | 해시태그, URL, 멘션 등과 같은 추가 정보를 제공합니다.                |
| **in\_reply\_to\_user\_id** | string | 해당 트윗이 답글인 경우, 이 필드에는 원본 트윗 작성자의 사용자 ID가 포함됩니다.                                       | 트윗이 다른 트윗에 대한 답글이었는지 판단합니다.                     |
| **lang**              | string | Twitter에서 감지된 경우 트윗의 언어입니다.                                                                                      | 사용 언어별로 트윗을 분류합니다.                                     |
| **non\_public\_metrics** | object | 요청 시점의 트윗 비공개 참여 지표입니다. 사용자 컨텍스트 인증이 필요합니다.                       | 트윗으로 생성된 전체 노출 수를 파악합니다.                           |
| **organic\_metrics**   | object | 요청 시점에 유기적 컨텍스트에서 추적된 트윗의 참여 지표입니다. 사용자 컨텍스트 인증이 필요합니다.  | 트윗의 유기적 참여를 측정합니다.                                     |
| **possibly\_sensitive** | boolean | 콘텐츠가 민감한 것으로 인식될 수 있는지 나타냅니다.                                                                           | 특정 유형의 콘텐츠 확산을 연구합니다.                               |
| **promoted\_metrics**  | object | 요청 시점에 프로모션 컨텍스트에서 추적된 트윗의 참여 지표입니다. 사용자 컨텍스트 인증이 필요합니다. | 트윗이 프로모션되었을 때의 참여를 측정합니다.                        |
| **public\_metrics**    | object | 요청 시점의 트윗 공개 참여 지표입니다.                                                                 | 트윗 참여도를 측정합니다.                                            |
| **referenced\_tweets** | array  | 이 트윗이 참조하는 트윗 목록으로, 리트윗, 인용 트윗, 답글 등이 포함됩니다.                                                | 리트윗 등의 대화적 측면을 파악합니다.                                |
| **reply\_settings**    | string | 이 트윗에 누가 답글할 수 있는지 표시합니다. 옵션은 "everyone", "mentioned\_users", "followers"입니다.                                  | 트윗의 대화 답글 설정을 파악합니다.                                  |
| **withheld**          | object | [보류된 콘텐츠](https://help.x.com/en/rules-and-policies/tweet-withheld-by-country)에 대한 보류 세부 정보를 포함합니다.     |                                                                     |
| **media\_metadata**          | array | 트윗의 미디어 첨부에 대한 메타데이터를 포함합니다.     | 트윗 미디어 첨부의 `alt_text` 등 추가 메타데이터를 가져옵니다. |

**트윗 객체 가져오기**

**샘플 요청**

다음 요청에서는 [Tweets lookup](/ko/resources/fundamentals/rate-limits) 엔드포인트에서 트윗 필드를 요청합니다. `$BEARER_TOKEN`을 생성한 본인의 [Bearer Token](/ko/resources/fundamentals/authentication)으로 바꾸세요.

```bash
curl --request GET 'https://api.x.com/2/tweets?ids=1212092628029698048&tweet.fields=attachments,author_id,context_annotations,created_at,entities,geo,id,in_reply_to_user_id,lang,possibly_sensitive,public_metrics,referenced_tweets,text,withheld&expansions=referenced_tweets.id' --header 'Authorization: Bearer $BEARER_TOKEN'
```

<div id="user">
  ### 사용자
</div>

user 객체에는 참조된 사용자를 설명하는 Twitter 사용자 계정 메타데이터가 포함됩니다. user 객체는 [users lookup](/ko/x-api/users/lookup/introduction) 엔드포인트에서 반환되는 기본 객체입니다. 이 엔드포인트에서 추가 사용자 필드를 요청하려면 필드 파라미터 `user.fields`를 사용하면 됩니다.

user 객체는 Tweet 객체의 하위 객체로도 포함될 수 있으며 확장할 수 있습니다. 기본 필드만 포함된 축약된 객체를 가져오려면 `?expansions=author_id` 또는 `?expansions=in_reply_to_user_id`를 사용해 확장하세요. 객체를 완성하기 위해 추가 필드를 요청할 때는 확장과 함께 필드 파라미터 `user.fields`를 사용하세요.
 

| 필드 값 | 유형 | 설명 | 사용 방법 |
| :--- | :--- | :--- | :--- |
| id (기본값) | string | 이 사용자의 고유 식별자입니다.<br /><br />`"id": "2244994945"` | 특정 Twitter 사용자의 정보를 프로그래밍 방식으로 조회하는 데 사용합니다. |
| name (기본값) | string | 사용자가 프로필에 지정한 이름입니다. 반드시 실명이 아닐 수 있습니다. 일반적으로 50자로 제한되지만 변경될 수 있습니다.<br /><br />`"name": "Twitter Dev"` |     |
| username (기본값) | string | 사용자가 자신을 식별하는 Twitter 사용자 이름(스크린 네임, 핸들 또는 별칭)입니다. 사용자 이름은 고유하지만 변경될 수 있습니다. 일반적으로 최대 15자이지만, 더 긴 이름을 가진 과거 계정이 존재할 수 있습니다.<br /><br />`"username": "TwitterDev"` |     |

  ### 스페이스
</div>

Spaces는 라이브 오디오 대화를 통해 표현하고 상호작용할 수 있게 합니다. Space 데이터 사전에는 Space에 관한 관련 메타데이터가 포함되어 있으며, 모든 세부 정보는 실시간으로 업데이트됩니다.

사용자 객체는 사용자 리소스에서 조회하고 확장할 수 있습니다. `expansions` 쿼리 매개변수에 `host_ids`, `creator_id`, `speaker_ids`, `mentioned_user_ids` 중 하나 이상을 추가하면 해당 객체를 확장할 수 있습니다.

트윗과 달리 Space는 일시적이며, 종료되거나 생성자가 취소하면 더 이상 이용할 수 없습니다. 앱이 Space 데이터를 처리할 때는 최신 정보를 반환할 책임이 있으며, 플랫폼에서 더 이상 이용할 수 없는 데이터는 반드시 제거해야 합니다. [Space 조회 엔드포인트](/ko/x-api/spaces/lookup/introduction)는 사용자의 기대와 의도를 존중하도록 하는 데 도움이 됩니다.

리스트 객체에는 참조된 리스트를 설명하는 [Twitter Lists](https://help.x.com/en/using-twitter/twitter-lists) 메타데이터가 포함됩니다. 리스트 객체는 List 조회 엔드포인트에서 반환되는 기본 객체입니다. 이 엔드포인트에서 추가 리스트 필드를 요청할 때는 필드 파라미터 `list.fields`를 사용하십시오.

리스트 객체는 다른 데이터 객체의 하위 객체로 포함되지 않습니다. 대신 사용자 객체는 사용자 리소스에서 조회 및 확장할 수 있습니다. 이러한 객체는 `expansions` 쿼리 파라미터에 `owner_id`를 추가하여 확장할 수 있습니다. 기본 리스트 객체를 완성하기 위해 추가 필드를 요청할 때는 이 확장을 `list.fields` 필드 파라미터와 함께 사용하고, 확장 객체를 완성하기 위해서는 `user.fields`를 사용하십시오.

| id (기본)      | string          | 이 리스트의 고유 식별자입니다.<br />`"id": "2244994945"`                                                                                                           | 특정 리스트에 대한 정보를 프로그램적으로 조회하는 데 사용합니다.            |
| name (기본)    | string          | 리스트 생성 시 지정된 리스트의 이름입니다.<br />`"name": "Twitter Lists"`                                                                                          |                                                                            |
| created\_at     | date (ISO 8601) | 리스트가 생성된 UTC 날짜와 시간입니다.<br />`"created_at": "2013-12-14T04:35:55.000Z"`                                                                              | 리스트가 X에서 얼마나 오래 존재했는지 확인합니다.                           |

<div id="direct-message-events">
  ### 다이렉트 메시지 이벤트
</div>

다이렉트 메시지(DM) 대화는 이벤트로 구성됩니다. X API v2는 현재 세 가지 이벤트 유형을 지원합니다: MessageCreate, ParticipantsJoin, ParticipantsLeave.

DM 이벤트 객체는 [다이렉트 메시지 조회](/ko/x-api/direct-messages/lookup/introduction) 엔드포인트에서 반환되며, [다이렉트 메시지 관리](/ko/x-api/direct-messages/lookup/introduction) 엔드포인트로 다이렉트 메시지를 성공적으로 생성하면 MessageCreate 이벤트가 생성됩니다.

DM 이벤트를 요청할 때 기본적으로 포함되는 이벤트 객체 속성(필드)은 세 가지입니다: id, event\_type, text. 추가 이벤트 필드를 받으려면 [fields](/ko/x-api/fundamentals/fields) 파라미터 dm\_event.fields를 사용해 다른 필드를 선택하세요. 사용 가능한 기타 이벤트 필드는 다음과 같습니다: dm\_conversation\_id, created\_at, sender\_id, attachments, participant\_ids, referenced\_tweets.

이러한 필드 중 일부는 다이렉트 메시지 이벤트와 관련된 다른 X 객체의 ID를 제공합니다:

- sender\_id - 메시지를 보낸 계정의 ID 또는 그룹 대화에 참가자를 초대한 계정의 ID
- participant\_ids - 계정 ID 배열입니다. ParticipantsJoin 및 ParticipantsLeave 이벤트의 경우 이 배열에는 해당 이벤트를 생성한 계정의 단일 ID가 포함됩니다
- attachments - 발신자가 X에 업로드한 콘텐츠의 미디어 ID를 제공합니다
- referenced\_tweets - text 필드에서 트윗 URL이 발견되면 해당 트윗의 ID가 응답에 포함됩니다

sender\_id, participant\_ids, referenced\_tweets.id, attachments.media\_keys [expansions](/ko/x-api/fundamentals/expansions)를 사용해 이러한 X 객체 ID를 확장할 수 있습니다.

|     |     |     |     |
| :--- | :--- | :--- | :--- |
| **필드 값** | **유형** | **설명** | **사용 방법** |
| id (기본값) | string | 이벤트의 고유 식별자입니다.<br /><br />"id": "1050118621198921728" | 이를 사용해 특정 대화 이벤트를 프로그래밍 방식으로 조회합니다(v1.1 엔드포인트에서 사용 가능). |
| event\_type (기본값) | string | 이벤트 유형을 설명합니다. 현재 세 가지 유형이 지원됩니다: <br /><br />\* MessageCreate<br />    <br />\* ParticipantsJoin<br />    <br />\* ParticipantsLeave<br />    <br /><br />"event\_type": "MessageCreate" | 대화 기록을 조회할 때 메시지 생성 시점과, 그룹 대화의 경우 참가자가 참여·퇴장한 시점을 파악합니다. 모든 GET 메서드는 event\_type= 쿼리 매개변수로 특정 이벤트 유형에 대한 필터링을 지원합니다. |

**다이렉트 메시지 이벤트 객체 가져오기**

**샘플 요청**

이 예에서는 일대일 대화와 연관된 이벤트를 가져오는 요청을 구성합니다. 이 요청은 기본 다이렉트 메시지 이벤트 필드와 함께 참조된 트윗 및 해당 작성자의 추가 필드를 반환합니다. 다음을 요청하는 쿼리를 구성해 보겠습니다:

- 생성 시점과 어떤 대화의 일부인지(dm\_conversation)와 같은 기본 이벤트 속성
- 다이렉트 메시지를 보낸 계정 ID와 설명
- 참조된 트윗의 텍스트와 게시 시점
- 참조된 트윗 작성자의 계정 ID와 설명

이러한 속성을 반환하려면 요청 쿼리에 다음을 포함합니다:

`?dm_event.fields=id,sender_id,text,created_at,dm_conversation_id&expansions=sender_id,referenced_tweets.id&tweet.fields=created_at,text,author_id&user.fields=description`

```bash
      curl --request GET 'https://api.x.com/2/dm_conversations/with/:participant_id/dm_events?tweet.fields=created_at,text,author_id&user.fields=description&expansions=sender_id,participant_ids,referenced_tweets.id&dm_event.fields=id,sender_id,text,participant_ids,created_at,'
    --header 'Authorization: Bearer $BEARER_TOKEN'
```

$BEARER\_TOKEN을 직접 생성한 [Bearer Token](/ko/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)으로 교체하세요.

<div id="how-to-use-fields-and-expansions">
  ## 필드와 확장을 사용하는 방법
</div>

기본적으로 X API v2 데이터 객체는 [fields](/ko/x-api/fundamentals/fields) 또는 [expansions](/ko/x-api/fundamentals/expansions) 매개변수를 사용하지 않고 요청할 경우 소수의 기본 필드만 포함합니다. 이 가이드는 요청에서 `fields` 및 `expansions` 쿼리 매개변수를 사용하는 방법을 보여 주어, 응답에서 추가 객체와 필드를 받을 수 있도록 합니다.

이 가이드에서는 다음 트윗 스크린샷을 예로 들어 여러 필드를 요청해 보겠습니다.
 

![이 이미지는 @X가 게시한 트윗의 스크린샷입니다. 트윗 텍스트, 사용자 이름, 게시 일시, 소스, 공개 지표를 확인할 수 있으며 비디오도 포함되어 있습니다.](https://cdn.cms-twdigitalassets.com/content/dam/developer-twitter/ScreenShotDataDictionaryGuide.png.twimg.1920.png)

스크린샷에서 볼 수 있듯이 트윗 작성자, 트윗 지표, 생성 시각, 비디오, 비디오 조회수 등 트윗과 관련된 여러 가시적 정보가 있습니다. 또한 스크린샷에는 보이지 않지만 요청할 수 있는 데이터도 여럿 있습니다. 

API에 요청할 때 기본 응답은 단순하며 기본 트윗 필드(id 및 text)만 포함합니다. 또한 사용 중인 엔드포인트에서 반환되는 기본 객체만 수신하고, 기본 객체와 연관될 수 있는 관련 데이터 객체는 수신하지 않습니다.

이러한 단순성은 fields 및 expansions 매개변수와 함께 사용 사례에 따라 필요한 필드만 요청할 수 있도록 해 줍니다. 
 

<div id="requesting-additional-fields-and-objects">

</div>

요청:

```bash
curl --request GET --url 'https://api.x.com/2/tweets?ids=1260294888811347969' \
  --header 'Authorization: Bearer $BEARER_TOKEN'
```

1. [object model](/ko/x-api/fundamentals/data-dictionary#tweet)을 참고하거나 각 엔드포인트의 API 참조 페이지에 나열된 필드 목록을 검토하여, 요청할 추가 필드를 식별하세요.

   이 경우 다음 추가 필드를 요청합니다:
   attachments, author\_id, created\_at, public\_metrics.

2. 쉼표로 구분된 목록으로 위 필드를 값에 포함해 `tweet.fields` 쿼리 매개변수를 구성하세요:
   `?tweet.fields=attachments,author_id,created_at,public_metrics`

3. 앞서 보낸 GET /tweets 요청에 쿼리 매개변수를 추가하세요.

`curl --request GET --url 'https://api.x.com/2/tweets?ids=1260294888811347969&tweet.fields=attachments,author_id,created_at,public_metrics' \   --header 'Authorization: Bearer $BEARER_TOKEN'`
