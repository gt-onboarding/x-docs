---
title: 확장
---

확장을 사용하면 개발자가 페이로드에서 참조된 객체를 완전한 객체로 확장할 수 있습니다. 확장 가능한 객체는 ID로 참조됩니다. 예를 들어 [게시물 조회](/ko/x-api/posts/lookup/introduction) 페이로드에 반환되는 `referenced_tweets.id`와 `author_id` 필드는 완전한 객체로 확장할 수 있습니다. 게시물을 작성한 사용자와 관련된 필드나 그 게시물에 포함된 미디어, 설문조사, 장소와 관련된 필드를 요청하려면 해당 확장 쿼리 매개변수를 요청에 포함해야 하며, 그러면 응답에서 해당 데이터를 받을 수 있습니다. 현재 게시물, 사용자, 리스트, 스페이스, 다이렉트 메시지 이벤트 객체를 반환하는 v2 엔드포인트는 모두 확장을 지원합니다(아래 예시 참조).

요청에 확장을 포함하면 해당 확장 객체의 기본 필드가 동일한 응답에 함께 포함됩니다. 별도의 요청 없이 하나의 응답에서 추가 데이터를 받을 수 있도록 해줍니다. 확장된 객체와 관련된 추가 [필드](/ko/x-api/fundamentals/fields)를 요청하려면, 해당 확장 객체에 대응하는 필드 매개변수를 쉼표로 구분된 필드 목록과 함께 포함하면 됩니다. 참고로, 필드는 쿼리에서 요청한 순서와 항상 동일한 순서로 반환되지는 않습니다.

아래 Post 페이로드에는 확장할 수 있는 보조 객체에 대한 참조 ID가 포함되어 있으며, 여기에는 게시물을 작성한 사용자의 `author_id`, _참조된_ 게시물의 `id`, 그리고 미디어 첨부의 `media_key`가 포함됩니다.

```json
{
    "data": {
        "attachments": {
            "media_keys": ["16_1211797899316740096"]
        },
        "author_id": "2244994945",
        "id": "1212092628029698048",
        "referenced_tweets": [
            {
                "type": "replied_to",
                "id": "1212092627178287104"
            }
        ],
        "text": "We believe the best future version of our API will come from building it with YOU. Here’s to another great year with everyone who builds on the Twitter platform. We can’t wait to continue working with you in the new year. https://t.co/yvxdK6aOo2"
    }
}
```

<div id="available-expansions-for-post-payloads">
  ### 게시물 페이로드에서 사용 가능한 확장
</div>

| 확장                          | 설명                                                                                |
|:--------------------------------|:------------------------------------------------------------------------------------|
| `author_id`                    | 게시물 작성자를 나타내는 사용자 객체를 반환합니다                                   |
| `referenced_tweets.id`         | 이 게시물이 참조하는 게시물 객체를 반환합니다(리트윗, 인용 트윗, 또는 답글)          |
| `edit_history_tweet_ids`       | 해당 게시물의 편집 기록에 포함된 게시물 객체들을 반환합니다                         |
| `in_reply_to_user_id`          | 이 요청된 게시물이 누구에게 답글인지 나타내는 게시물 작성자의 사용자 객체를 반환합니다 |
| `attachments.media_keys`       | 게시물에 포함된 이미지, 동영상, GIF를 나타내는 미디어 객체를 반환합니다             |
| `attachments.poll_ids`         | 게시물에 포함된 투표의 메타데이터를 담은 투표 객체를 반환합니다                     |
| `geo.place_id`                 | 게시물에 태그된 위치의 메타데이터를 담은 장소 객체를 반환합니다                     |
| `entities.mentions.username`   | 게시물에서 언급된 사용자의 사용자 객체를 반환합니다                                |
| `referenced_tweets.id.author_id` | 참조된 게시물의 작성자에 대한 사용자 객체를 반환합니다                             |

<div id="available-expansion-for-user-payloads">
  ### 사용자 페이로드에서 사용 가능한 확장
</div>

| 확장            | 설명                                               |
|:--------------------|:-----------------------------------------------------------|
| `pinned_tweet_id`  | 사용자 프로필 상단에 고정된 게시물을 나타내는 Post 객체를 반환합니다 |

<div id="available-expansions-for-direct-message-event-payloads">
  ### 다이렉트 메시지 이벤트 페이로드에서 사용 가능한 확장
</div>

| Expansion                | Description                                                              |
|:--------------------------|:--------------------------------------------------------------------------|
| `attachments.media_keys` | 다이렉트 메시지에 첨부된 Media 객체를 반환합니다                         |
| `referenced_tweets.id`   | 다이렉트 메시지에서 참조된 Post 객체를 반환합니다                        |
| `sender_id`              | 다이렉트 메시지의 작성자이자 대화에 참가자를 초대한 사용자를 나타내는 User 객체를 반환합니다 |
| `participant_ids`        | 대화에 참여하거나 떠난 참가자를 나타내는 User 객체를 반환합니다          |

<div id="available-expansions-for-spaces-payloads">
  ### Spaces 페이로드에서 사용 가능한 확장
</div>

| Expansion          | Description                                               |
|:--------------------|:-----------------------------------------------------------|
| `invited_user_ids` | 초대된 계정을 나타내는 사용자(User) 객체를 반환합니다 |
| `speaker_ids`      | Space에서 발언한 계정을 나타내는 사용자(User) 객체를 반환합니다 |
| `creator_id`       | Space를 생성한 계정을 나타내는 사용자(User) 객체를 반환합니다 |
| `host_ids`         | 호스트로 지정된 계정을 나타내는 사용자(User) 객체를 반환합니다 |
| `topics_ids`       | 생성자가 설정한 토픽의 설명을 반환합니다 |

<div id="available-expansion-for-lists-payloads">
  ### 리스트 페이로드에서 사용 가능한 확장
</div>

| 확장   | 설명                                                    |
|:-------------|:----------------------------------------------------------------|
| `owner_id`  | 리스트를 생성하고 관리하는 계정을 나타내는 사용자 객체를 반환합니다 |

<div id="expanding-the-media-object">
  ## 미디어 객체 확장
</div>

다음 요청에서는 기본 게시물 필드에 더해 `geo.place_id` 확장을 요청합니다:

**샘플 요청**

```
{`curl 'https://api.x.com/2/tweets/:ID?expansions=geo.place_id' --header 'Authorization: Bearer $ACCESS_TOKEN'`}
```

**샘플 응답**

```
{`{
  "data": {
      "geo": {
          "place_id": "01a9a39529b27f36"
      },
      "id": "ID",
      "text": "Test"
  },
  "includes": {
      "places": [
          {
              "full_name": "Manhattan, NY",
              "id": "01a9a39529b27f36"
          }
      ]
  }
}`}

```

<div id="expanding-the-poll-object">
  ## 투표 객체 확장
</div>

다음 요청에서는 기본 게시물 필드와 함께 포함되도록 `attachments.poll_ids` 확장을 요청합니다.

**샘플 요청**

<CodeBlock language="bash">
  {`curl 'https://api.x.com/2/tweets/1199786642791452673?expansions=attachments.poll_ids' --header 'Authorization: Bearer $ACCESS_TOKEN'`}
</CodeBlock>

**샘플 응답**

```
{`{
  "data": {
      "attachments": {
          "poll_ids": ["1199786642468413448"]
      },
      "id": "1199786642791452673",
      "text": "C#"
  },
  "includes": {
      "polls": [
          {
              "id": "1199786642468413448",
              "options": [
                  {
                      "position": 1,
                      "label": "“C Sharp”",
                      "votes": 795
                  },
                  {
                      "position": 2,
                      "label": "“C Hashtag”",
                      "votes": 156
                  }
              ]
          }
      ]
  }
}`}
```
