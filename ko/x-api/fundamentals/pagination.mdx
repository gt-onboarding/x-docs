---
title: 페이지네이션
---

<div id="introduction">
  ## 소개
</div>

페이지네이션은 단일 응답으로 반환할 수 있는 범위를 초과하는 결과를 제공하는 X API v2 엔드포인트에서 사용하는 기능입니다. 이 경우 데이터는 일련의 ‘페이지’로 나누어 전달됩니다. 페이지네이션은 전체 결과 데이터 세트를 가져오기 위해 모든 페이지를 프로그램적으로 요청하는 방법을 의미합니다. 모든 API 엔드포인트가 페이지네이션을 지원하거나 필요로 하지는 않지만, 결과 세트가 큰 경우에는 자주 사용됩니다.

<div id="use-cases-for-pagination">
  ### 페이지네이션 사용 사례
</div>

**요청의 모든 결과를 가져오기:** 페이지네이션은 특정 요청과 매개변수에 관련된 모든 데이터를 받아오기 위해 사용해야 합니다. 요청에서 일치하는 결과가 `max_results`보다 많은 경우 페이지네이션이 필요합니다. 페이지네이션 토큰으로 요청을 반복하면 모든 결과를 가져올 수 있습니다. `next_token` 값 없이 응답이 반환되면 모든 결과를 모두 페이지 처리했다고 볼 수 있습니다. 페이지네이션을 폴링 용도로 사용해서는 안 됩니다. 이전 요청 이후의 최신 결과를 가져오려면 `since_id`를 사용한 폴링을 검토하세요.

**요청 결과를 순방향/역방향으로 탐색하기:** 페이지네이션은 응답의 `next_token` 및 `previous_token` 값을 사용해 요청 결과를 탐색할 때의 이동 방향을 제공합니다. 이러한 토큰은 다음 요청에서 `pagination_token`으로 설정하여 다음 또는 이전 페이지로 이동할 수 있습니다.

<div id="pagination-token-definitions">
  ### 페이지네이션 토큰 정의
</div>

- `next_token` - 페이지네이션을 지원하는 엔드포인트의 meta 객체 응답에 포함되어 반환되는 불투명한 문자열입니다. 더 많은 결과가 있음을 나타내며, 다음 요청에서 `pagination_token` 매개변수로 사용해 다음 페이지의 결과를 가져올 수 있습니다. 마지막 페이지의 결과에는 `next_token`이 포함되지 않습니다.
  - `previous_token` - 페이지네이션을 지원하는 엔드포인트의 meta 객체 응답에 포함되어 반환되는 불투명한 문자열입니다. 이전 페이지의 결과가 있음을 나타내며, 다음 요청에서 `pagination_token` 매개변수로 설정해 이전 페이지의 결과를 가져올 수 있습니다.
  - `pagination_token` - 페이지네이션 요청에 사용되는 매개변수입니다. 다음 페이지의 결과를 가져오려면 `next_token`의 값으로 설정합니다. 이전 페이지의 결과를 가져오려면 `previous_token`의 값으로 설정합니다.

<div id="fundamentals-of-pagination">
  ### 페이지네이션의 기초
</div>

- 페이지네이션을 사용하는 엔드포인트는 초기 요청에 첫 번째 페이지의 결과를 반환하며, 추가 페이지가 있을 경우 JSON 응답의 meta 객체에 `next_token`을 제공합니다. 모든 결과를 받으려면 응답에 `next_token`이 더 이상 포함되지 않을 때까지 이 과정을 반복하세요.
  - 결과는 최신순(내림차순)으로 제공됩니다. 이는 개별 페이지 내에서도, 여러 페이지 전반에서도 동일합니다:
    - 첫 번째 응답의 첫 번째 게시물이 가장 최신입니다.
    - 마지막 응답의 마지막 게시물이 가장 오래되었습니다.
  - `max_results` 요청 매개변수를 사용하면 응답 페이지당 반환되는 게시물 수를 설정할 수 있습니다. `max_results`에는 기본값과 최대값이 있습니다.
  - 모든 페이지네이션 구현에는 이후 요청에 사용할 수 있도록 응답 페이로드에서 토큰을 파싱하는 과정이 포함됩니다.
  - 일부 상황에서는 페이지당 결과가 `max_results`보다 적을 수 있습니다. 페이지당 결과 수가 항상 `max_results` 매개변수 값과 같다고 가정하지 마세요.
  - 전체 결과를 얻기 위해 페이지네이션을 사용하는 가장 좋은 방법은 통합 코드에서 루프 로직을 사용하거나 X API v2를 지원하는 [라이브러리](/ko/x-api/tools-and-libraries/overview)를 사용하는 것입니다.

<div id="pagination-example">
  ### 페이지네이션 예시
</div>

여기서는 `max_results`가 `100`으로 설정되어 있고, 2019년 1월 1일 17:00:00 UTC부터 12월 12일 00:00:00 UTC 사이에 사용자 ID `2244994945`(@XDevelopers)가 생성한 게시물이 약 295개이므로 결과 페이지는 총 세 개입니다. 첫 번째 페이지의 첫 번째 게시물(`1337498609819021312`)이 가장 최근이며, 세 번째 페이지의 마지막 게시물(`1082718487011885056`)이 가장 오래되었습니다.

<div id="initial-request">
  #### 초기 요청
</div>

```bash
      https://api.x.com/2/users/2244994945/tweets?tweet.fields=created_at&max_results=100&start_time=2019-01-01T17:00:00Z&end_time=2020-12-12T01:00:00Z
```
