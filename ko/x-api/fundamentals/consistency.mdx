<div id="consistency-across-the-x-api-v2-endpoints">
  ## X API v2 엔드포인트 전반의 일관성
</div>

X API의 새로운 v2 버전에서 핵심적인 측면 중 하나는 엔드포인트 전반의 일관성입니다. 이는 반환되는 객체, 기능, 그리고 API 동작이 유사한 엔드포인트 간에 표준화되어 있다는 의미입니다.

다음 요소들은 모든 엔드포인트에서 동일하게 적용됩니다:

<div id="path-naming">
  ### 경로 명명
</div>

경로 명명에는 항상 엔드포인트 [버전](/ko/x-api/fundamentals/versioning)이 포함되며, 그 뒤에 **리소스**가 옵니다. 그다음 경로에는 앞선 리소스와 연관된 **ID**, 어떤 데이터를 반환할지 결정하는 데 도움이 되는 **선택 동사**(예: `search`, `sample`), 데이터가 어떻게 전달될지 결정하는 **전달 동사**(예: `stream`), 또는 기본 리소스와 **관계**가 있는 다른 리소스(예: `/user/12/tweets`)가 포함될 수 있습니다. 마지막으로 엔드포인트에 쿼리 매개변수가 있다면 끝에 **쿼리 매개변수**를 추가할 수 있습니다.

다음은 이러한 경로 및 쿼리 항목을 구성하는 예시입니다:

`/version/resource/id?param1=value&param2=value /version/resource/delivery/selection?param1=value&param2=value`

실제 요청 예:

`/2/tweets/1067094924124872705?expansions=attachments.media_keys&tweet.fields=author_id /2/users/2244994945?user.fields=created_at,description /2/tweets/search/stream /2/tweets/search/recent?query=snow`

<div id="json-schema">
  ### JSON 스키마
</div>

요청에 대한 응답은 [JSON Schema](http://json-schema.org/)를 사용해 정의됩니다. 즉, 요청은 각 요소가 ID를 가진 객체들의 집합을 배열 형태로 일관되게 반환합니다. ID를 키로 하는 맵 형태로는 반환되지 않습니다.

<div id="response-object-and-parameters">
  ### 응답 객체와 매개변수
</div>

기본적으로 반환되는 객체는 해당 객체 유형의 모든 엔드포인트에서 동일합니다:

- `id` 객체는 항상 문자열입니다.
- 매개변수와 응답 필드는 일관되게 미국식 영어 표기를 사용합니다.
- 값이 없으면 필드는 비어 있거나 반환되지 않습니다.
- `entities` 객체에는 게시물 텍스트에서 추출된 엔터티만 포함됩니다. 여기에는 `urls`, `hashtags`, `mentions`, `cashtags`가 포함됩니다.
- `media_keys` 및 `poll_ids` 필드를 포함한 모든 카드 관련 정보는 `attachments` 객체로 반환됩니다.

<div id="authentication">
  ### 인증
</div>

모든 X API v2 엔드포인트에는 인증이 필요합니다. 많은 엔드포인트는 [OAuth 2.0 Bearer Token](/ko/resources/fundamentals/authentication#oauth-2-0) 방식을 지원하며, 요청을 성공시키려면 해당 엔드포인트로의 요청에 Bearer Token을 포함해야 합니다.

다른 사용자를 대신해 데이터를 생성, 수정 또는 조회해야 하는 엔드포인트에는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication)를 사용하세요. 이를 위해서는 [개발자 앱의](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) [API 키와 토큰](/ko/resources/fundamentals/authentication#api-key-and-secret)뿐 아니라, 요청을 수행하려는 대상 사용자를 위해 발급받은 사용자 [액세스 토큰](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)도 제공해야 합니다. [3-legged OAuth flow](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)를 통해 액세스 토큰을 발급받을 수 있습니다. 이 권한 헤더를 자동으로 구성해 주는 [도구나 라이브러리](/ko/x-api/tools-and-libraries/overview)를 사용하세요.

인증에 대한 자세한 내용은 문서의 [authentication](/ko/resources/fundamentals/authentication)에서 확인할 수 있습니다.

<div id="fields">
  ### 필드
</div>

각 엔드포인트에서 반환되는 객체는 요약되어 있습니다. 개발자가 API 응답을 더 세밀하게 제어할 수 있도록, 원하는 필드를 지정하는 [`fields`](/ko/x-api/fundamentals/fields) 매개변수를 사용할 수 있습니다. 필드는 엔드포인트 간에 일관되게 유지됩니다. Post 객체가 반환되는 모든 엔드포인트에서 Post 객체는 동일한 필드를 제공합니다. 유사한 엔드포인트에서도 동일한 필드 집합을 조회할 수 있습니다.

예를 들어, 동일한 Post 필드를 [Posts lookup](/ko/x-api/posts/lookup/introduction)과 [Users lookup](/ko/x-api/users/lookup/introduction)의 확장된 고정 Post에서도 조회할 수 있습니다.

<div id="expansions">
  ### 확장
</div>

적절한 경우 [확장](/ko/x-api/fundamentals/expansions)은 모든 중첩된 id 필드(예: `author_id`와 같이 `*_id`로 명명된 모든 필드)에 사용할 수 있습니다. 또한 현재 객체의 최상위 식별자가 아닌 id를 가진 모든 필드에도 확장을 사용할 수 있습니다. 예를 들어, [게시물 조회](/ko/x-api/posts/lookup/introduction)에서 Post는 현재 객체이며, 필드 `id`가 최상위 식별자입니다. `author_id` 또는 `referenced_tweets.id` 필드는 쉼표로 구분된 값을 `expansions` 매개변수에 추가하면 전체 사용자 또는 Post 객체로 확장할 수 있습니다.

이러한 필드와 관련하여 발견한 [불일치 사항을 신고해 주세요](https://twitterdevfeedback.uservoice.com/).
