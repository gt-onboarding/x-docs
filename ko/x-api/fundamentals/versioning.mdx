<p>X API v2와 함께 개발자가 X의 공개 API에 어떤 변경이 언제 발생할지, 그리고 언제 새 버전으로 마이그레이션해야 하는지를 더 잘 파악할 수 있도록 하는 새로운 버전 관리 전략을 도입했습니다. </p>
<p>개발자는 개발 로드맵에 이러한 변경 사항을 적절히 반영할 수 있도록 당사의 <a href="https://developer.x.com/en/updates/stay-informed">커뮤니케이션 채널</a>을 통해 X API의 사용 중단, 서비스 종료, 변경 및 추가 사항에 대한 안내를 받게 됩니다. API의 모든 변경 사항은 변경 로그에 기록됩니다.</p>
<p>현재 X API에는 세 가지 버전이 있습니다. 사용 사례에 필요한 기능이 아직 v2에 제공되지 않은 경우를 제외하고, 통합을 계획할 때는 X API v2 사용을 강력히 권장합니다. </p>
<p>각 버전에 대해 자세히 알아보려면 다음 페이지를 참고하세요:</p>

<ul>
  <li><a href="/ko/x-api/introduction">X API v2</a></li>
  <li>Gnip 2.0 (<a href="/ko/x-api/enterprise-gnip-2.0/enterprise-gnip">엔터프라이즈</a>)</li>
</ul>

<div id="versioning-strategy">
  ## 버전 관리 전략
</div>

<p>X API의 버전 관리는 엔드포인트 경로에 선언된 버전 번호로 표시됩니다:</p>
<p>`https://api.x.com/2/tweets`</p>
<p>필요할 때 주요 버전을 출시하되, 12개월보다 더 짧은 주기로는 출시하지 않는 것을 목표로 합니다. API에 하위 호환성을 깨는 변경 사항이 도입될 때 주요 버전을 출시합니다. 새로운 주요 버전을 출시할 때는 개발자가 새 버전으로 이전할 수 있도록 마이그레이션 가이드를 게시합니다. </p>
<p>하위 호환성 파괴 변경은 앱의 기존 기능을 유지하려면 개발자가 코드를 수정해야 함을 의미합니다. 비파괴적 변경은 기능이 추가되는 형태이며 준비가 되면 최신 버전에 배포됩니다. 새 기능을 활용하려는 경우가 아니라면 개발자 측의 추가 작업은 필요하지 않습니다.</p>
<p>보안 또는 개인정보 보호 등의 이유로 중간 주기에 하위 호환성 파괴 변경을 적용해야 하는 경우, 해당 변경은 최신 버전에 반영됩니다.</p>

<div id="breaking-changes">
  ### 호환성을 깨는 변경 사항
</div>

<p>다음 변경 사항이 있으면 애플리케이션의 기존 기능을 유지하기 위해 개발자가 코드를 수정해야 합니다.</p>

<ul>
  <li>새 필수 매개변수 추가</li>
  <li>기존 엔드포인트 제거</li>
  <li>응답의 필드 제거(필수 또는 선택)</li>
  <li>쿼리 매개변수 제거</li>
  <li>입력 또는 출력 형식 재구성(예: 최상위 필드를 하위 필드로 내리거나, 오류 위치를 인라인으로 변경)</li>
  <li>기존 입력 매개변수 또는 출력 값의 이름이나 데이터 유형 변경</li>
  <li>필드 이름 변경</li>
  <li>리소스 이름 변경</li>
  <li>응답 코드 변경</li>
  <li>오류 유형 변경</li>
  <li>기존 권한 범위(scope) 변경</li>
</ul>

<div id="non-breaking-changes">
  ### 하위 호환성을 깨지 않는 변경
</div>

<ul>
  <li>새 엔드포인트 추가</li>
  <li>새 선택적 매개변수 추가</li>
  <li>새 응답 필드 추가</li>
  <li>오류 메시지 문구 변경</li>
  <li>새 스코프 제공</li>
  <li>필드 ‘널 처리’(완전히 제거하는 대신 개인정보 보호/보안을 위해 값을 null로 변경)</li>
</ul>

<div id="deprecation-and-retirement">
  ### 사용 중단과 폐기
</div>

<p>먼저, X API에서 사용 중단과 폐기가 무엇을 의미하는지에 대한 우리의 정의는 다음과 같습니다.</p>

<ul>
  <li><b>사용 중단(Deprecation):</b> 해당 기능은 더 이상 팀의 지원을 받지 않습니다. 해당 기능과 관련한 신규 기능은 제공되지 않으며, 제품에 버그나 문제가 발생하더라도 이를 수정할 가능성은 매우 낮습니다. </li>
  <li><b>폐기(Retired):</b> 해당 기능은 더 이상 이용할 수 없습니다.</li>
</ul>

<p>대부분의 경우 새 버전이 출시되는 즉시 이전 버전은 사용 중단으로 표시됩니다. 버전은 일정 기간 동안 사용 중단 상태로 유지되며, 이후 폐기됩니다. </p>
<p>향후 사용 중단 및 폐기에 대해 더 알아보려면 <a href="https://developer.x.com/en/updates/stay-informed">최신 정보를 확인하세요</a>.</p>
