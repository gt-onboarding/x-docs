계정을 뮤트하면 해당 계정을 언팔로우하거나 차단하지 않고도 타임라인에서 해당 계정의 게시물을 보이지 않게 할 수 있습니다. 뮤트된 계정은 자신이 뮤트되었다는 사실을 알 수 없으며, 언제든지 뮤트를 해제할 수 있습니다. manage mutes 엔드포인트를 통해 개발자는 X에서 더 안전한 사용자 경험을 구현할 수 있습니다. 예를 들어, 특정 주제에 대해 게시할 가능성이 있는 계정을 지정된 기간 동안 뮤트하도록 하는 애플리케이션을 만들 수 있습니다. mutes lookup 엔드포인트를 사용하면 본인 또는 인증된 사용자가 누구를 뮤트했는지 확인할 수 있습니다. 이는 뮤트된 계정과의 상호작용 방식을 파악하는 데 유용합니다.

mute lookup으로 비공개 정보를 요청하고, manage mutes로 사용자를 대신해 작업하므로, 해당 엔드포인트는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)로 인증해야 하며, 앱을 승인한 사용자와 연결된 사용자 액세스 토큰을 사용해야 합니다. 해당 토큰은 [3-legged OAuth flow](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) 또는 [Authorization Code with PKCE grant flow](/ko/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0)을 통해 생성할 수 있습니다.

<div id="mutes-lookup">
  ### 음소거 조회
</div>

음소거 조회 엔드포인트를 사용하면 인증된 사용자가 어떤 계정을 음소거했는지 확인할 수 있습니다. 이 엔드포인트의 요청 한도는 사용자당 15분마다 15회입니다.

<div id="manage-mutes">
  ### 음소거 관리
</div>

음소거 관리 엔드포인트를 사용하면 인증된 사용자를 대신해 지정된 계정을 음소거하거나 해제할 수 있습니다. 이 엔드포인트에는 POST와 DELETE 두 가지 메서드가 있습니다. POST는 계정을 음소거하고, DELETE는 계정의 음소거를 해제합니다. POST와 DELETE 엔드포인트 모두에 대해 사용자 기준 15분당 50건의 요청 제한이 적용됩니다.

**참고:** 사용자가 [X](https://x.com/)에서 직접 음소거하는 경우 15분당 200건의 제한이 적용됩니다.

**계정 설정**

이 엔드포인트에 액세스하려면 다음이 필요합니다:

- 승인된 [developer account](https://developer.x.com/en/portal/petition/essential/basic-info)
- [Project](/ko/resources/fundamentals/projects) 내에 있는 [developer App](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용한 인증

X API v2 엔드포인트에 대한 액세스 권한 얻기에 대해 더 알아보려면 [시작 가이드](/ko/x-api/getting-started/getting-access)를 참고하세요.
