  ### 음소거 관리: Standard v1.1과 X API v2 비교
</div>

표준 v1.1의 [POST mutes/users/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-create) 및 [POST mutes/users/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-destroy) 엔드포인트로 작업해 오셨다면, 이 가이드는 Standard v1.1과 X API v2의 음소거 관리 엔드포인트 간의 유사점과 차이점을 이해하는 데 도움을 드리는 것을 목표로 합니다.

- **유사점**
  - OAuth 1.0a 사용자 컨텍스트
- **차이점**
  - 엔드포인트 URL
  - 앱 및 프로젝트 요구사항
  - HTTP 메서드
  - 요청 매개변수
     

<div id="similarities">
  #### 유사점
</div>

**OAuth 1.0a 사용자 컨텍스트 인증 방식**

두 엔드포인트 버전 모두 [OAuth 1.0a 사용자 컨텍스트](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication)를 지원합니다. 따라서 이전에 표준 v1.1의 음소거 관리 엔드포인트를 사용했다면, X API v2로 마이그레이션하더라도 동일한 인증 방식을 계속 사용할 수 있습니다.

<div id="differences">
  #### 차이점
</div>

**엔드포인트 URL**

- Standard v1.1 엔드포인트:
  - POST https://api.x.com/1.1/mutes/users/create.json
    (사용자 음소거)
  - POST https://api.x.com/1.1/mutes/users/destroy.json
    (사용자 음소거 해제)
- X API v2 엔드포인트:
  - POST https://api.x.com/2/users/:id/muting
    (사용자 음소거)
  - DELETE https://api.x.com/2/users/:source\_user\_id/muting/:target\_user\_id
    (사용자 음소거 해제)

**앱 및 프로젝트 요구 사항**

X API v2 엔드포인트는 요청을 인증할 때 [developer App](/ko/resources/fundamentals/developer-apps)의 자격 증명을 사용해야 하며, 해당 App은 [Project](/ko/resources/fundamentals/projects)에 연결되어 있어야 합니다. 모든 X API v1.1 엔드포인트는 독립 실행형 App 또는 프로젝트에 연결된 App의 자격 증명을 사용할 수 있습니다.

**요청 매개변수**

다음 Standard v1.1 요청 매개변수는 X API v2에서 대응 항목이 있습니다:

| Standard v1.1 | X API v2 |
| :--- | :--- |
| user\_id | target\_user\_id |
| screen\_name | 해당 없음 |

Standard v1.1 매개변수는 쿼리 매개변수로 전달되는 반면, X API v2 매개변수는 본문 매개변수(POST 엔드포인트의 경우) 또는 경로 매개변수(DELETE 엔드포인트의 경우)로 전달된다는 점에 유의하세요.

또한, Standard v1.1 엔드포인트를 사용할 때는 대상 사용자를 음소거하는 사용자의 id가 필요하지 않았습니다. 이는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication)와 함께 전달된 액세스 토큰을 통해 어느 사용자가 음소거/해제를 시작했는지 유추할 수 있었기 때문입니다.
