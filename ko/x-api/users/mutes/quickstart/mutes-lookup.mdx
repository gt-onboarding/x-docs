<div id="getting-started-with-the-mutes-lookup-endpoint">
  ## mutes 조회 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 Postman을 사용해 mutes 조회 엔드포인트로 첫 요청을 보내는 방법을 안내합니다.

다양한 언어의 샘플 코드를 보고 싶다면 [X API v2 샘플 코드](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요.

<Note>
  **사전 준비 사항**

  이 가이드를 완료하려면 요청을 인증할 [키와 토큰](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계에 따라 이러한 키와 토큰을 생성할 수 있습니다:

  - [개발자 계정에 신청](https://developer.x.com/en/apply-for-access)하고 승인을 받습니다.
  - 개발자 포털에서 [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성합니다.
  - 앱의 “Keys and tokens” 페이지로 이동해 필요한 자격 증명을 생성합니다. 모든 자격 증명은 안전한 위치에 보관하세요.
</Note>

<div id="steps-to-build-a-mutes-lookup-request">
  ### 음소거 조회 요청을 구성하는 단계
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### 1단계: 도구 또는 라이브러리로 시작
</div>

이 엔드포인트에 요청을 보내는 데 사용할 수 있는 다양한 도구, 코드 예제, 라이브러리가 있지만 여기서는 과정을 단순화하기 위해 Postman을 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 불러오려면 다음 버튼을 클릭하세요.

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Postman에 X API v2 추가
</Button>

Postman에 X API v2 컬렉션을 불러온 후 ‘Mutes’ 폴더로 이동해 ‘Mutes lookup’을 선택하세요.
 

<div id="step-two-authenticate-your-request">
  #### 2단계: 요청 인증
</div>

X API에 올바르게 요청하려면 권한이 있는지 확인해야 합니다. 이 엔드포인트에서는 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용해 요청을 인증해야 합니다.

이 예제에서는 OAuth 1.0a 사용자 컨텍스트를 사용합니다.

API Key, API Secret Key, OAuth 1.0a 사용자 Access Token, OAuth 1.0a 사용자 Access Token Secret 등 키와 토큰을 Postman에 추가해야 합니다. Postman 오른쪽 상단에서 “X API v2” 환경을 선택한 뒤, 환경 드롭다운 옆 눈 아이콘을 클릭해 "initial value"와 "current value" 필드에 키와 토큰을 입력하세요.

올바르게 설정했다면 이러한 변수는 요청의 authorization 탭에 자동으로 불러와집니다.
 

<div id="step-three-specify-a-user">
  #### 3단계: 사용자 지정
</div>

이 엔드포인트에서는 자신 또는 인증된 사용자가 누구를 음소거했는지 확인하려면 자신의 사용자 ID 또는 인증된 사용자의 ID를 지정해야 합니다.

Postman에서 "Params" 탭으로 이동한 다음, 섹션 하단의 “Path Variables”에서 id의 "Value" 열에 인증된 사용자 ID를 입력하고 ID 앞뒤에 공백이 없도록 하세요.

“Path Variables” 섹션 위에는 추가할 수 있는 선택적 “Query Params”가 있습니다. 이 예에서는 변수 max\_results를 선택하고 값으로 5를 입력합니다.

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Parameter Type** |
| `id` | (your user ID) | Path |
| max\_results | 5   | Query |

<div id="step-four-identify-and-specify-which-fields-you-would-like-to-retrieve">
  #### 4단계: 가져올 필드를 식별하고 지정하기
</div>

3단계 후 "Send" 버튼을 클릭하면 응답에 기본 [user object](/ko/x-api/fundamentals/data-dictionary#user) 필드(id, name, username)가 반환됩니다.

id, name, username 외의 필드를 추가로 받으려면 요청에서 [fields](/ko/x-api/fundamentals/fields) 및/또는 [expansions](/ko/x-api/fundamentals/expansions) 매개변수로 해당 필드를 지정해야 합니다.

이번 연습에서는 서로 다른 오브젝트에서 다음 세 가지 추가 필드 세트를 요청합니다:

1. 기본 사용자 오브젝트의 추가 user.created\_at 필드
2. 반환된 사용자에 연결된 고정 게시물 오브젝트의 기본 필드: id, text
3. 연결된 게시물 오브젝트의 추가 tweet.created\_at 필드

Postman에서 "Params" 탭으로 이동한 뒤 "Query Params" 표에 다음 키:값 쌍을 추가하세요:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| user.fields | created\_at | user.created\_at |
| expansions | pinned\_tweet\_id | tweet.id, tweet.text |
| tweet.fields | created\_at | includes.tweets.created\_at |

이제 "Send" 버튼 옆에 표시되는 예시 ID URL 대신, 본인의 사용자 ID가 포함된 유사한 URL을 볼 수 있습니다:

`https://api.x.com/2/users/1324848235714736129/muting?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=created_at&max_results=5`

<div id="step-five-make-your-request-and-review-your-response">
  #### 5단계: 요청 보내기 및 응답 검토
</div>

모든 구성이 완료되면 "Send" 버튼을 클릭하세요. 그러면 아래 예시와 유사한 응답을 받게 됩니다:

```
{
  "data": [
    {
      "username": "TwitterDev",
      "created_at": "2013-12-14T04:35:55.000Z",
      "id": "2244994945",
      "name": "Twitter Dev",
      "pinned_tweet_id": "1430984356139470849"
    }
  ],
  "includes": {
    "tweets": [
      {
        "created_at": "2021-08-26T20:03:51.000Z",
        "id": "1430984356139470849",
        "text": "Help us build a better Twitter Developer Platform!\n \nTake the annual developer survey &gt;&gt;&gt; https://t.co/9yTbEKlJHH https://t.co/fYIwKPzqua"
      }
    ]
  },
  "meta": {
    "result_count": 1
  }
}
```

<div id="step-six-paginate-through-your-results">
  #### 6단계: 결과 페이지네이션
</div>

응답의 하단에는 meta 객체가 있습니다. next\_token을 받았다면 가져올 수 있는 결과의 다음 페이지가 있다는 뜻입니다. 다음 페이지의 결과를 가져오려면 next\_token 필드의 값을 읽어 추가 pagination\_token 쿼리 매개변수의 값으로 요청에 포함하세요.
 

|     |     |
| :--- | :--- |
| **키** | **값** |
| pagination\_token | 1710819323648428707 |

이 매개변수를 추가해 요청을 보내면, 3단계에서 max\_results를 5로 지정했으므로 이후 페이로드에 다음 5개의 결과가 전달됩니다. 모든 결과가 반환될 때까지 이 과정을 반복할 수 있지만, max\_results 매개변수를 사용해 요청당 최대 1000명의 사용자를 요청할 수도 있으므로 페이지네이션 빈도를 줄일 수 있습니다.
