<div id="authentication">
  #### 인증
</div>

모든 X API v2 엔드포인트는 키와 토큰으로도 알려진 자격 증명을 사용해 요청을 인증해야 합니다. 이러한 엔드포인트에 대한 요청 인증에는 OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0 Authorization Code with PKCE 중 하나를 사용할 수 있습니다.

[OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication)에서는 API 키, 사용자 액세스 토큰, 그리고 몇 가지 추가 매개변수를 사용해 [인증 헤더를 생성](/ko/resources/fundamentals/authentication#authorizing-a-request)한 후 요청에 함께 전달해야 합니다. 액세스 토큰은 요청을 수행하는 데 있어 위임받은 해당 사용자와 연결되어 있어야 합니다. 다른 사용자를 위한 액세스 토큰을 생성하려면, 그 사용자가 [3-legged OAuth 플로우](https://developer-staging.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)를 통해 귀하의 앱을 승인해야 합니다.

OAuth 1.0a는 사용이 까다로울 수 있습니다. 이 인증 방식에 익숙하지 않다면 [라이브러리](/ko/x-api/tools-and-libraries/overview)를 사용하거나 Postman과 같은 도구를 이용하거나, OAuth 2.0으로 요청을 인증할 것을 권장합니다.

[OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)는 애플리케이션의 스코프에 대한 더 세밀한 제어와 여러 기기에 걸친 인증 플로우를 제공합니다. OAuth 2.0은 사용자 대신 특정 권한을 부여하는 정교한 세분화 스코프를 선택할 수 있도록 합니다.

앱에서 OAuth 2.0을 사용하려면, 개발자 포털의 앱 설정 섹션에 있는 앱 인증 설정에서 이를 활성화해야 합니다.

<div id="developer-portal-projects-and-developer-apps">
  #### 개발자 포털, 프로젝트와 개발자 앱
</div>

X API v2 엔드포인트에서 사용할 인증 자격 증명을 받으려면 [개발자 계정에 등록](https://developer.x.com/en/portal/petition/essential/basic-info)하고 해당 계정 내에 [프로젝트](/ko/resources/fundamentals/projects)를 설정한 뒤, 그 프로젝트 내에 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성해야 합니다. 이후 개발자 앱에서 키와 토큰을 확인할 수 있습니다.
 

<div id="rate-limits">
  #### 요청 제한
</div>

매일 수천 명의 개발자가 X API에 요청을 보냅니다. 이러한 대량의 요청을 관리하기 위해 [요청 제한](/ko/x-api/fundamentals/rate-limits)이 각 엔드포인트에 적용되어, 앱 또는 인증된 사용자를 대신해 보낼 수 있는 요청 수를 제한합니다. 

이 엔드포인트들은 사용자 수준으로 제한되며, 이는 요청을 보내는 인증된 사용자가 어느 개발자 앱에서 호출하든 해당 엔드포인트를 호출할 수 있는 횟수가 정해져 있다는 뜻입니다.

POST 및 DELETE 메서드의 경우 엔드포인트당 15분에 50회까지 사용자 제한이 적용됩니다. 반면 GET 메서드는 15분에 15회로 제한됩니다.
 

<div id="fields-and-expansions">
  #### 필드와 확장
</div>

X API v2 GET 엔드포인트는 사용자가 ‘필드(fields)’와 ‘확장(expansions)’이라는 도구를 사용해 API에서 반환받을 데이터를 정확히 선택할 수 있도록 합니다. expansions 매개변수는 페이로드에서 참조되는 객체를 확장합니다. 예를 들어, 이 엔드포인트에서는 다음 [확장](/ko/x-api/fundamentals/expansions)을 호출할 수 있습니다:

- pinned\_tweet\_id

**fields** 매개변수를 사용하면 여러 데이터 객체 내에서 수신하려는 정확한 [필드](/ko/x-api/fundamentals/fields)를 선택할 수 있습니다. 이 엔드포인트는 주로 User 객체를 반환합니다. 기본적으로 User 객체는 **id**, **name**, **username** 필드를 반환합니다. **user.created\_at** 또는 **user.entities**와 같은 추가 필드를 받으려면 **fields** 매개변수로 해당 필드를 명시적으로 요청해야 합니다.

[X API v2 데이터 사전](/ko/x-api/fundamentals/data-dictionary)에 [필드와 확장](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)을 함께 사용하는 방법에 대한 가이드를 추가했습니다.

<div id="pagination">
  #### 페이지네이션
</div>

뮤트 조회는 많은 데이터를 반환할 수 있습니다. 언제든지 일관되고 성능이 안정적인 결과를 제공하기 위해 페이지네이션을 사용합니다. 페이지네이션은 단일 응답에 담을 수 있는 양을 초과하는 결과를 반환하는 X API v2 엔드포인트에서 제공하는 기능입니다. 이런 경우 데이터는 일련의 ‘페이지’로 나누어 반환됩니다. 결과를 [페이지네이션하는 방법](/ko/x-api/fundamentals/pagination)에 대해 자세히 알아보세요.

<Note>
  **참고:** 사용자가 [X](https://x.com/)에서 뮤트하는 경우, 15분당 요청 한도는 200건입니다.
</Note>
