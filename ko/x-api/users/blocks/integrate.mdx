<div id="authentication">
  #### 인증
</div>

모든 X API v2 엔드포인트는 키와 토큰으로도 불리는 자격 증명으로 요청을 인증해야 합니다. 이러한 엔드포인트에 대한 요청 인증에는 OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0 Authorization Code with PKCE를 사용할 수 있습니다.

[OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication)는 요청에 포함할 [Authorization 헤더를 생성](/ko/resources/fundamentals/authentication#authorizing-a-request)하기 위해 API 키, 사용자 액세스 토큰, 그리고 몇 가지 추가 매개변수를 사용해야 합니다. 액세스 토큰은 요청을 대행하는 해당 사용자와 연결되어 있어야 합니다. 다른 사용자의 액세스 토큰을 생성하려면 해당 사용자가 [3-legged OAuth 플로우](https://developer-staging.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)를 통해 귀하의 앱을 승인해야 합니다.

OAuth 1.0a는 사용이 까다로울 수 있습니다. 이 인증 방식에 익숙하지 않다면 [라이브러리](/ko/x-api/tools-and-libraries/overview)를 사용하거나 Postman과 같은 도구를 사용하거나, OAuth 2.0으로 요청을 인증할 것을 권장합니다.

[OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3)는 애플리케이션의 스코프에 대한 더 정교한 제어와 멀티 디바이스 전반의 인증 플로우를 지원합니다. OAuth 2.0을 사용하면 사용자 대행 권한을 부여하는 세분화된 스코프를 선택할 수 있습니다.

앱에서 OAuth 2.0을 사용하려면 개발자 포털의 앱 설정 섹션에 있는 인증 설정에서 해당 옵션을 활성화해야 합니다.

<div id="rate-limits">
  #### 레이트 한도
</div>

매일 수천 명의 개발자가 X API에 요청을 보냅니다. 이러한 대규모 요청을 관리하기 위해 [레이트 한도](/ko/x-api/fundamentals/rate-limits)가 각 엔드포인트에 적용되어, 앱 또는 인증된 사용자를 대신해 보낼 수 있는 요청 횟수를 제한합니다. 

이 엔드포인트들은 사용자 수준에서 한도가 적용됩니다. 즉, 어떤 개발자 앱에서 호출하든, 요청을 대신하는 인증된 사용자는 해당 엔드포인트를 정해진 횟수까지만 호출할 수 있습니다. POST 및 DELETE 메서드의 경우 엔드포인트별로 15분당 50회의 사용자 한도가 있습니다. 반면 GET 메서드는 15분당 15회로 더 낮은 한도가 적용됩니다.
 

<div id="fields-and-expansions">
  #### 필드와 확장
</div>

X API v2의 GET 엔드포인트는 ‘필드’와 ‘확장’이라는 도구 세트를 통해, 사용자가 API에서 반환받을 데이터를 정확하게 선택할 수 있게 합니다. expansions 매개변수는 페이로드에서 참조되는 객체를 확장합니다. 예를 들어, 이 엔드포인트에서는 다음 [확장](/ko/x-api/fundamentals/expansions)을 요청할 수 있습니다:

- pinned\_tweet\_id

**fields** 매개변수는 서로 다른 데이터 객체 내에서 수신하려는 [필드](/ko/x-api/fundamentals/fields)를 정확히 선택할 수 있게 합니다. 이들 엔드포인트는 주로 Post 객체를 반환합니다. 기본적으로 Post 객체는 **id**와 **text** 필드를 반환합니다. **tweet.created\_at** 또는 **tweet.entities**와 같은 추가 필드를 받으려면 **fields** 매개변수로 해당 필드를 명시적으로 요청해야 합니다. 통합 시 고려할 중요한 필드로는 투표 데이터, 지표, Post 주석, 대화 ID 등이 있습니다.

[fields와 expansions 사용 방법](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)에 대한 가이드를 [X API v2 데이터 사전](/ko/x-api/fundamentals/data-dictionary)에 추가했습니다.
