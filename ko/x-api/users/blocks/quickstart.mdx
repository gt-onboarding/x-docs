<div id="getting-started-with-the-blocks-lookup-endpoint">
  ## blocks 조회 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 [Postman](/ko/tutorials/postman-getting-started)을 사용해 blocks 조회 엔드포인트에 첫 요청을 보내는 방법을 안내합니다.

여러 언어의 샘플 코드를 보려면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요.

<Note>
  ### 사전 준비 사항

  이 가이드를 완료하려면 요청 인증에 사용할 [keys and tokens](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 통해 해당 키와 토큰을 생성할 수 있습니다:

  - [개발자 계정 신청](https://developer.x.com/en/apply-for-access) 후 승인을 받습니다.
  - 개발자 포털에서 [프로젝트](/ko/resources/fundamentals/projects)와 연동된 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성합니다.
  - 앱의 “Keys and tokens” 페이지로 이동해 필요한 자격 증명을 생성합니다. 모든 자격 증명은 안전한 위치에 보관하세요.
</Note>

<div id="steps-to-build-a-blocks-lookup-request">
  ### 차단 조회 요청을 구성하는 단계
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### 1단계: 도구 또는 라이브러리로 시작
</div>

이 엔드포인트에 요청을 보내는 데 사용할 수 있는 다양한 도구, 코드 예시, 라이브러리가 있지만, 여기서는 과정을 간소화하기 위해 Postman을 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 로드하려면 아래 버튼을 클릭하세요.

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Postman에 X API v2 추가
</Button>

Postman에 X API v2 컬렉션을 로드한 후 “Blocks” 폴더로 이동하여 “Blocks Lookup”을 선택하세요.
 

<div id="step-two-authenticate-your-request">
  #### 2단계: 요청 인증
</div>

X API에 올바르게 요청하려면 권한이 있는지 확인해야 합니다. 이 엔드포인트에서는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용하여 요청을 인증해야 합니다.

이 예제에서는 OAuth 1.0a User Context를 사용합니다.

Postman에 키와 토큰, 즉 API Key, API Secret Key, OAuth 1.0a 사용자 액세스 토큰, OAuth 1.0a 사용자 액세스 토큰 시크릿을 추가해야 합니다. 이를 위해 Postman 오른쪽 상단에서 “X API v2”라는 이름의 환경을 선택한 후 "initial value"와 "current value" 필드에 키와 토큰을 입력합니다(환경 드롭다운 옆 눈 아이콘 클릭).

이 변수를 올바르게 설정했다면 요청의 Authorization 탭에 자동으로 반영됩니다.
 

<div id="step-three-specify-a-user">
  #### 3단계: 사용자 지정
</div>

이 엔드포인트를 사용하려면, 차단한 사용자를 확인하기 위해 본인의 사용자 ID 또는 인증된 사용자의 사용자 ID를 지정해야 합니다.

Postman에서 "Params" 탭으로 이동한 다음, 섹션 하단에 있는 id 경로 변수의 "Value" 열에 이 사용자 ID를 입력하세요. 사용자 ID 앞뒤에 공백이 없도록 주의하세요.
 

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | (your user ID) |
| max\_results | 5   |

<div id="step-four-identify-and-specify-which-fields-you-would-like-to-retrieve">
  #### 4단계: 가져올 필드를 파악하고 지정하기
</div>

3단계 이후 "Send" 버튼을 클릭하면 응답에서 기본 [user object](/ko/x-api/fundamentals/data-dictionary#user) 필드인 id, name, username이 반환됩니다.

id, name, username 외의 필드를 더 받으려면, 요청에 [fields](/ko/x-api/fundamentals/fields) 및/또는 [expansions](/ko/x-api/fundamentals/expansions) 파라미터로 해당 필드를 명시해야 합니다.

이 연습에서는 서로 다른 오브젝트에서 다음 세 가지 추가 필드 세트를 요청합니다:

1. 기본 사용자 오브젝트의 추가 user.created\_at 필드
2. 반환된 사용자에 연관된 고정된 게시물 오브젝트의 기본 필드: id, text
3. 연관된 게시물 오브젝트의 추가 tweet.created\_at 필드

Postman에서 "Params" 탭으로 이동하여 "Query Params" 표에 아래 key:value 쌍을 추가하세요:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| user.fields | created\_at | user.created\_at |
| expansions | pinned\_tweet\_id | tweet.id, tweet.text |
| tweet.fields | created\_at | includes.tweets.created\_at |

이제 "Send" 버튼 옆의 URL에서 TwitterDev의 URL 대신 본인의 사용자 ID가 포함된 유사한 URL이 표시되어야 합니다:

```
      https://api.x.com/2/users/2244994945/blocking?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=created_at
```

<div id="step-six-paginate-through-your-results">
  #### 6단계: 결과 페이지네이션 수행
</div>

응답의 하단에 meta 객체가 포함되어 있음을 확인할 수 있습니다. next\_token을 받았다면 가져올 수 있는 다음 페이지의 결과가 있다는 뜻입니다. 다음 페이지의 결과를 가져오려면 next\_token 필드의 값을 추출해 요청에 추가 매개변수인 pagination\_token의 값으로 포함하세요.
 

|     |     |
| :--- | :--- |
| **키** | **값** |
| pagination\_token | 1D3PU6DRII9HEZZZ |

이 추가 매개변수를 포함해 요청을 보내면, 3단계에서 max\_results를 5로 지정했기 때문에 이후 페이로드에 다음 5개의 결과가 제공됩니다. 모든 결과가 반환될 때까지 이 과정을 반복할 수 있지만, max\_results 매개변수를 사용해 요청당 최대 1000명의 사용자를 요청할 수도 있으므로 페이지네이션 횟수를 줄일 수 있습니다.
