<div id="authentication">
  #### 인증
</div>

모든 X API v2 엔드포인트에는 키와 토큰으로도 불리는 자격 증명 세트를 사용한 [인증](/ko/resources/fundamentals/authentication)이 필요합니다. 이러한 엔드포인트에 대한 요청을 인증하려면 OAuth 1.0a 사용자 컨텍스트, 앱 전용, 또는 OAuth 2.0 Authorization Code with PKCE 중 하나를 사용할 수 있습니다.

[OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication#oauth-1-0a-2)는 API 키, 사용자 액세스 토큰, 그리고 몇 가지 추가 매개변수를 사용해 [Authorization 헤더를 생성](/ko/resources/fundamentals/authentication#authorizing-a-request)한 뒤, 이를 요청과 함께 전달해야 합니다. 액세스 토큰은 요청을 대신 수행하는 대상 사용자와 연결되어 있어야 합니다. 다른 사용자의 액세스 토큰 세트를 생성하려면, 해당 사용자가 [3-legged OAuth 플로우](https://developer-staging.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)를 통해 귀하의 앱을 승인해야 합니다.

OAuth 1.0a는 다소 사용이 까다로울 수 있습니다. 이 인증 방식에 익숙하지 않다면 [라이브러리](/ko/x-api/tools-and-libraries/overview)를 사용하거나 Postman 같은 도구를 활용하거나, OAuth 2.0으로 요청을 인증할 것을 권장합니다. 이러한 엔드포인트에서 게시물 또는 비공개 지표를 요청하려면, 해당 콘텐츠 소유자의 적절한 권한을 보장하기 위해 OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0 Authorization Code with PKCE를 사용해야 합니다.

[앱 전용](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)은 요청에 [앱 전용 액세스 토큰](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)만 포함하면 됩니다. 개발자 앱 내에서 직접 앱 전용 액세스 토큰을 생성하거나, [POST oauth2/token](/ko/resources/fundamentals/authentication#post-oauth2-token) 엔드포인트를 사용해 생성할 수 있습니다.

[OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)는 애플리케이션의 권한 범위에 대한 더 정교한 제어와 여러 디바이스 간의 인증 플로우를 제공합니다. OAuth 2.0에서는 사용자 대신 수행되는 권한을 세밀한 스코프로 선택할 수 있습니다.

앱에서 OAuth 2.0을 사용하려면, 개발자 포털의 앱 설정 섹션에 있는 앱 인증 설정에서 해당 옵션을 활성화해야 합니다.

**참고**

다음 필드를 요청하는 경우, OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0 Authorization Code가 필요합니다:

- tweet.fields.non\_public\_metrics
- tweet.fields.promoted\_metrics
- tweet.fields.organic\_metrics

<div id="developer-portal-projects-and-developer-apps">
  #### 개발자 포털, 프로젝트와 개발자 앱
</div>

X API v2 엔드포인트에서 사용할 인증 자격 증명을 받으려면 [개발자 계정에 가입](https://developer.x.com/en/portal/petition/essential/basic-info)한 뒤, 해당 계정 안에 [프로젝트](/ko/resources/fundamentals/projects)를 설정하고 그 프로젝트 내에 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성해야 합니다. 이후 개발자 앱에서 키와 토큰을 확인할 수 있습니다. 
 

<div id="rate-limits">
  #### 요청 한도
</div>

매일 수천 명의 개발자가 X API에 요청을 보냅니다. 이러한 대규모 트래픽을 관리하기 위해, [요청 한도](/ko/x-api/fundamentals/rate-limits)가 각 엔드포인트에 적용되어 앱 또는 인증된 사용자를 대신해 보낼 수 있는 요청 횟수를 제한합니다. 

사용자 조회 엔드포인트에는 앱 수준과 사용자 수준의 요청 한도가 모두 적용됩니다. 반면, 인증된 사용자 조회 엔드포인트는 사용자 수준 한도만 적용됩니다.

앱 수준 한도는 개발자인 여러분이 사용하는 키와 토큰으로 식별되는 특정 앱에서, 주어진 기간 동안 해당 엔드포인트로 보낼 수 있는 요청 횟수가 제한됨을 의미합니다. 사용자 수준 한도는 여러분이 대신 요청을 수행하는 인증된 사용자가, 어떤 개발자 앱을 사용하든 주어진 기간 동안 수행할 수 있는 요청 횟수가 제한됨을 의미합니다.

아래 표는 각 엔드포인트에 대한 요청 한도를 보여줍니다.

|     |     |     |
| :--- | :--- | :--- |
| **엔드포인트** | **HTTP 메서드** | **요청 한도 / 수준** |
| /2/users | GET | 15분당 900회 요청 / 앱 및 사용자 |
| /2/users/:id | GET | 15분당 900회 요청 / 앱 및 사용자 |
| /2/users/by | GET | 15분당 900회 요청 / 앱 및 사용자 |
| /2/users/by/username/:username | GET | 15분당 900회 요청 / 앱 및 사용자 |
| /2/users/me | GET | 15분당 75회 요청 / 사용자 |

<div id="fields-and-expansions">
  #### 필드와 확장
</div>

X API v2는 사용자가 ‘필드’와 ‘확장’이라는 도구 세트를 통해 API에서 반환받을 데이터를 정확하게 선택할 수 있도록 합니다. 확장 매개변수는 페이로드에서 참조된 객체를 펼쳐 포함시킬 수 있게 해줍니다. 예를 들어, 이 엔드포인트에서는 pinned\_tweet\_id 확장을 사용할 수 있습니다.

fields 매개변수를 사용하면 서로 다른 데이터 객체 내에서 수신하려는 [필드](/ko/x-api/fundamentals/fields)를 정확히 지정할 수 있습니다. 이 엔드포인트들은 주로 사용자 객체를 반환합니다. 기본적으로 사용자 객체는 id, name, username 필드를 반환합니다. user.created\_at 또는 user.location과 같은 추가 필드를 받으려면 fields 매개변수를 사용하여 해당 필드를 명시적으로 요청해야 합니다. 통합 시 고려할 만한 중요한 필드로는 게시물 투표 데이터, 지표, 주석, 그리고 대화 ID 등이 있습니다.

[X API v2 데이터 사전](/ko/x-api/fundamentals/data-dictionary)에 [필드와 확장 사용 방법](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) 가이드를 추가했습니다.
