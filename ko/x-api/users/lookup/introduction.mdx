인증된 사용자 조회를 제외한 모든 사용자 조회 엔드포인트에 대한 요청은 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication), [App only](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), 또는 [OAuth 2.0 Authorization code with PKCE](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)로 인증할 수 있습니다. 다만 비공개 지표 또는 비공개 사용자의 데이터에 접근하려면 OAuth 1.0a User Context나 OAuth 2.0 Authorization Code with PKCE를 사용하고, 요청에 인증된 사용자의 액세스 토큰을 함께 전달해야 합니다.    

이 엔드포인트는 사용자 최신 정보를 조회하거나 사용자의 존재를 확인하고, 컴플라이언스 이벤트 이후 저장된 정보를 업데이트하는 데 일반적으로 사용됩니다.

<div id="authenticated-user-lookup">
  ### 인증된 사용자 조회
</div>

다른 사용자를 대신해 인증하는 경우, 어떤 사용자를 대신해 요청을 보낼 수 있는지 확인하는 것이 중요합니다.

이 엔드포인트를 사용하려면 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용해야 합니다. 이 엔드포인트는 권한이 부여된 사용자, 즉 요청에 포함하여 전달하는 사용자 액세스 토큰과 연결된 사용자에 대한 정보를 반환합니다.

이 엔드포인트는 GET 메서드로 호출할 수 있습니다. 이 엔드포인트에는 15분당 75회의 사용자 레이트 한도가 적용됩니다.

<Note>
  **계정 설정**

  이 엔드포인트에 액세스하려면 다음이 필요합니다:

  - 승인된 [개발자 계정](https://developer.x.com/en/portal/petition/essential/basic-info)
  - [프로젝트](/ko/resources/fundamentals/projects) 내 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용한 인증

  X API v2 엔드포인트 액세스 방법에 대해서는 [시작하기 가이드](/ko/x-api/getting-started/getting-access)를 참고하세요.
</Note>
