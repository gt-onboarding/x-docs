<div id="getting-started-with-the-users-lookup-endpoints">
  ## 사용자 조회 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 [Postman](/ko/tutorials/postman-getting-started)을 사용해 지정한 필드 집합과 함께 사용자 조회 엔드포인트에 첫 요청을 보내는 방법을 안내합니다.

다양한 언어의 샘플 코드를 확인하려면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요.

<Note>
  ### 사전 준비 사항

  이 가이드를 완료하려면 요청을 인증하기 위한 [keys and tokens](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따라 이러한 keys and tokens를 생성할 수 있습니다.

  - [개발자 계정](https://developer.x.com/en/apply-for-access)에 신청하고 승인을 받습니다.
  - 개발자 포털에서 [Project](/ko/resources/fundamentals/projects)와 연결된 [developer App](/ko/resources/fundamentals/developer-apps)을 생성합니다.
  - App의 “Keys and tokens” 페이지로 이동해 필요한 자격 증명을 생성합니다. 모든 자격 증명은 안전한 위치에 보관하세요.
</Note>

<div id="steps-to-build-a-users-lookup-request">
  ### 사용자 조회 요청을 빌드하는 단계
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### 1단계: 도구 또는 라이브러리로 시작
</div>

이 엔드포인트에 요청을 보내는 방법으로 사용할 수 있는 도구, 코드 예제, 라이브러리가 여러 가지 있습니다. 여기서는 과정을 간소화하기 위해 Postman을 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 로드하려면 아래 버튼을 클릭하세요.

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Postman에 X API v2 추가
</Button>

Postman에 X API v2 컬렉션을 로드한 후 GET /users/by 엔드포인트로 이동하세요.
 

<div id="step-two-authenticate-your-request">
  #### 2단계: 요청 인증
</div>

X API에 올바르게 요청하려면 권한이 있음을 검증해야 합니다. 이를 위해 이 엔드포인트는 [App only](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), [OAuth 2.0 Authorization Code with PKCE](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), 또는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 인증 방식 중 하나로 요청을 인증해야 합니다.

간단히 하기 위해 이 예시에서는 App only를 사용하지만, 비공개 [지표](/ko/x-api/fundamentals/metrics)나 사용자 데이터를 요청하려면 다른 인증 방식 중 하나를 사용해야 합니다.

App only를 사용하려면 키와 토큰, 특히 Postman에 [App Access Token](/ko/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token) (App only Bearer Token이라고도 함)을 추가해야 합니다. 이를 위해 Postman 우측 상단에서 “X API v2” 환경을 선택한 뒤, "initial value"와 "current value" 필드에 키와 토큰을 입력하세요 (환경 드롭다운 옆 눈 아이콘 클릭).

올바르게 설정했다면 이 변수들은 요청의 Authorization 탭에 자동으로 반영됩니다.
 

<div id="step-three-identify-and-specify-which-users-you-would-like-to-retrieve">
  #### 3단계: 가져올 사용자(들) 식별 및 지정
</div>

요청에 수신하려는 단일 사용자 또는 사용자 집합을 반드시 지정해야 합니다. 사용하는 사용자 엔드포인트에 따라 사용자 ID 또는 사용자 이름을 전달할 수 있습니다. 여기서는 한 번의 요청에 여러 사용자 이름을 전달할 수 있는 [GET /users/by 엔드포인트](/ko/x-api/users/user-lookup-by-usernames)를 사용합니다(이는 [single-ID](/ko/x-api/users/user-lookup-by-id), [multi-ID](/ko/x-api/users/user-lookup-by-ids), [single-username](/ko/x-api/users/user-lookup-by-username) 엔드포인트와 대비됩니다). 또한 usernames 쿼리 매개변수를 사용해 사용자 이름 집합을 전달합니다.

사용자 이름은 계정 프로필 URL에서 확인할 수 있는 계정 핸들입니다. 예를 들어, 다음 계정의 사용자 이름은 XDevelopers입니다.

`https://x.com/XDevelopers`

Postman에서 "Params" 탭으로 이동한 다음, 이 사용자 이름 또는 쉼표로 구분된 사용자 이름 문자열을 username 매개변수의 "Value" 열에 입력하세요. 사용자 이름과 쉼표 사이에 공백이 없도록 하세요.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `username` | XDeveloeprs,API,adsapi |

<div id="step-four-identify-and-specify-which-fields-you-would-like-to-retrieve">
  #### 4단계: 가져올 필드를 식별하고 지정하기
</div>

3단계 이후 "Send" 버튼을 클릭하면 응답에서 기본 [user object](/ko/x-api/fundamentals/data-dictionary#user) 필드인 id, name, username을 받습니다.

id, name, username 외의 추가 필드를 받으려면 요청에 [field](/ko/x-api/fundamentals/fields) 및/또는 [expansion](/ko/x-api/fundamentals/expansions) 파라미터로 해당 필드를 지정해야 합니다.

이 연습에서는 서로 다른 오브젝트에서 다음과 같은 세 가지 추가 필드 세트를 요청합니다:

1. 기본 사용자 오브젝트의 추가 user.created\_at 필드
2. 반환된 사용자에 연관된 고정된 게시물 객체의 기본 필드: id 및 text
3. 연관된 게시물 객체의 추가 tweet.created\_at 필드
    

Postman에서 "Params" 탭으로 이동하여 "Query Params" 테이블에 다음 key:value 쌍을 추가하세요:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| `user.fields` | `created_at` | `user.created_at` |
| `expansions` | `author_id` | tweet.id, tweet.text |
| `tweet.fields` | `created_at` | `includes.users.created_at` |

이제 "Send" 버튼 옆에 다음 URL이 표시됩니다:

```
      https://api.x.com/2/users/by?usernames=twitterdev,twitterapi,adsapi&user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=author_id,created_at
```
