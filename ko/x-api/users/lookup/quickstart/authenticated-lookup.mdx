<div id="getting-started-with-the-authenticated-user-lookup-endpoint">
  ## 인증된 사용자 조회 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 Postman을 사용해 인증된 사용자 조회 엔드포인트에 첫 요청을 보내는 방법을 안내합니다.

다양한 언어의 샘플 코드를 확인하려면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요.

<Note>
  ### 사전 준비 사항

  이 가이드를 완료하려면 요청 인증을 위한 [keys and tokens](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따라 이러한 키와 토큰을 생성할 수 있습니다:

  - [개발자 계정 신청](https://developer.x.com/en/apply-for-access) 후 승인을 받습니다.
  - 개발자 포털에서 [Project](/ko/resources/fundamentals/projects)와 연결된 [developer App](/ko/resources/fundamentals/developer-apps)을 생성합니다.
  - App의 “Keys and tokens” 페이지로 이동해 필요한 자격 증명을 생성합니다. 모든 자격 증명은 안전한 위치에 보관하세요.
</Note>

<div id="steps-to-build-an-authenticated-user-lookup-request">
  ### 인증된 사용자 조회 요청을 구성하는 단계
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### 1단계: 도구 또는 라이브러리로 시작하기
</div>

이 엔드포인트에 요청을 보내는 데 사용할 수 있는 도구, 코드 예제, 라이브러리가 여럿 있지만, 여기서는 절차를 단순화하기 위해 Postman을 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 로드하려면 다음 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Postman에 X API v2 추가
</Button>

Postman에 X API v2 컬렉션을 로드한 뒤 “Authenticated User Lookup” 폴더로 이동하여 “Lookup an Authenticated User”를 선택하세요.

<div id="step-two-authenticate-your-request">
  #### 2단계: 요청 인증하기
</div>

X API에 올바르게 요청하려면 권한을 보유하고 있음을 확인해야 합니다. 이 엔드포인트의 경우 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용해 요청을 인증해야 합니다.

이 예제에서는 OAuth 1.0a User Context를 사용합니다.

API Key, API Secret Key, OAuth 1.0a 사용자 Access Token, OAuth 1.0a 사용자 Access Token Secret을 Postman에 추가해야 합니다. 이를 위해 Postman 우측 상단에서 “X API v2”라는 환경을 선택한 다음, 환경 드롭다운 옆 눈 아이콘을 클릭해 "initial value"와 "current value" 필드에 키와 토큰을 입력하세요.

변수를 올바르게 설정했다면 요청의 Authorization 탭에 자동으로 채워집니다.
 

<div id="step-three-determine-which-user-fields-you-want-to-retrieve">
  ##### 3단계: 가져올 사용자 필드를 결정합니다
</div>

3단계 후에 "Send" 버튼을 클릭하면 응답에서 기본 [user object](/ko/x-api/fundamentals/data-dictionary#user) 필드인 id, name, username을 받게 됩니다.

id, name, username 외의 추가 필드를 받으려면, 요청에서 [field](/ko/x-api/fundamentals/fields) 및/또는 [expansion](/ko/x-api/fundamentals/expansions) 매개변수로 해당 필드를 지정해야 합니다.

이 연습에서는 서로 다른 오브젝트에서 다음 세 가지 추가 필드 세트를 요청합니다:

1. 기본 사용자 오브젝트의 추가 user.created\_at 필드.

2. 반환된 사용자에 연결된 고정 게시물 오브젝트의 기본 필드: id 및 text.

3. 연결된 게시물 오브젝트의 추가 tweet.created\_at 필드.

Postman에서 "Params" 탭으로 이동하여 "Query Params" 표에 다음 key:value 쌍을 추가하세요:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| user.fields | created\_at | user.created\_at |
| expansions | pinned\_tweet\_id | includes.tweets.id,  <br />includes.tweets.text |
| tweet.fields | created\_at, author\_id | includes.tweets.created\_at, includes\_tweets.author\_id |

이제 “Send” 버튼 옆에 다음과 유사한 URL이 표시됩니다:

```
      https://api.x.com/2/users/me?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=author_id,created_at
```
