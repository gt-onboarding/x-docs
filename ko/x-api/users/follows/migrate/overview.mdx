  ## X API 팔로우 엔드포인트 비교
</div>

<div id="follows-lookup">
  ### 팔로우 조회
</div>

v2 팔로우 조회 엔드포인트는 표준 v1.1 [followers/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-followers-ids), v1.1 [followers/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-followers-list), v1.1 [friends/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-friends-ids), 그리고 v1.1 [friends/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-friends-list) 엔드포인트를 대체합니다.

다음 표는 다양한 유형의 팔로우 조회 엔드포인트를 비교합니다:

|     |     |     |
| :--- | :--- | :--- |
| **설명** | **Standard v1.1** | **X API v2** |
| 지원 HTTP 메서드 | `GET` | `GET` |
| 호스트 도메인 | `https://api.x.com` | `https://api.x.com` |
| 엔드포인트 경로 | /1.1/friends/ids.json<br /><br />/1.1/friends/list.json<br /><br />/1.1/followers/ids.json<br /><br />/1.1/followers/list.json | /2/users/:id/following<br /><br />/2/users/:id/followers |
| [Authentication](/ko/resources/fundamentals/authentication) | OAuth 1.0a User Context<br /><br />App only | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code with PKCE<br /><br />App only |
| 기본 요청 [요율 제한](/ko/resources/fundamentals/rate-limits) | 15분당 15회(사용자 기준)<br /><br />15분당 15회(앱 기준) | 15분당 15회(사용자 기준)<br /><br />15분당 15회(앱 기준) |
| 응답당 최대 사용자 수 | GET friends/id 및 GET followers/id는 페이지당 최대 5,000개의 사용자 ID를 반환합니다.<br /><br />  <br /><br />GET friends/list 및 GET followers/list는 페이지당 최대 200개의 사용자 오브젝트를 반환합니다. | 페이지당 1,000개의 사용자 오브젝트 |
| 페이지네이션 | 토큰이 next\_cursor 필드로 반환되며, 이를 cursor 매개변수 값으로 전달하여 다음 페이지 결과를 가져올 수 있습니다. | 토큰이 next\_token 필드로 반환되며, 이를 token 매개변수 값으로 전달하여 다음 페이지 결과를 가져올 수 있습니다.<br /><br />또한 v2 페이로드는 previous\_token 필드를 제공하며, 이를 pagination\_token 매개변수와 함께 전달하여 이전 페이지 결과를 가져올 수 있습니다. |
| JSON 형식 | Standard v1.1 형식 | [X API v2 형식](/ko/x-api/fundamentals/data-dictionary) (fields 및 expansions 요청 매개변수로 결정되며, v1.1 형식과 하위 호환되지 않음)<br /><br />Standard v1.1 형식에서 X API v2 형식으로 마이그레이션하는 방법은 [데이터 형식 마이그레이션 가이드](/ko/x-api/migrate/data-format-migration)를 참조하세요. |
| 페이로드에서 반환할 [fields](/ko/x-api/fundamentals/data-dictionary)를 선택하는 기능 지원 |     | ✔   |
| Post [annotations](/ko/x-api/fundamentals/post-annotations) 필드 지원 |     | ✔   |
| 새로운 [metrics](/ko/x-api/fundamentals/metrics) 필드 요청 지원 |     | ✔   |
| [conversation\_id](/ko/x-api/fundamentals/conversation-id) 필드 지원 |     | ✔   |
| [project](/ko/resources/fundamentals/projects)와 연결된 [developer App](/ko/resources/fundamentals/developer-apps)의 자격 증명 사용 필요 |     | ✔   |

<div id="manage-follows">
  #### 팔로우 관리
</div>

v2 팔로우 관리 엔드포인트는 표준 v1.1의 [POST friendships/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-create) 및 [POST friendships/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-destroy) 엔드포인트를 대체합니다.

다음 표는 표준 v1.1과 X API v2의 팔로우 생성 엔드포인트를 비교합니다.

<div id="follow-a-user">
  #### 사용자 팔로우
</div>

| 설명 | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| 지원되는 HTTP 메서드 | POST | POST |
| 호스트 도메인 | https://api.x.com | https://api.x.com |
| 엔드포인트 경로 | /1.1/friendships/create.json | /2/users/:id/following |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 1.0a 사용자 컨텍스트 | OAuth 1.0a 사용자 컨텍스트<br /><br />OAuth 2.0 인증 코드(PKCE 포함) |
| 기본 요청 [요율 제한](/ko/resources/fundamentals/rate-limits) | 15분당 요청 50회 | 15분당 요청 50회 |
| 사용자당 일일 최대 작업 수 | 400 | 400 |
| 앱당 일일 최대 작업 수 | 1000 | 1000 |
| [개발자 앱](/ko/resources/fundamentals/developer-apps)이 [프로젝트](/ko/resources/fundamentals/projects)와 연결된 자격 증명 사용 필요 |     | ✔️  |

<div id="unfollow-a-user">
  #### 사용자를 언팔로우하기
</div>

다음 표는 표준 v1.1과 X API v2의 팔로우 삭제 엔드포인트를 비교합니다:

| 설명 | 표준 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 지원되는 HTTP 메서드 | POST | DELETE |
| 호스트 도메인 | https://api.x.com | https://api.x.com |
| 엔드포인트 경로 | /1.1/friendships/destroy.json | /2/users/:source\_user\_id/following/:target\_user\_id |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 1.0a 사용자 컨텍스트 | OAuth 1.0a 사용자 컨텍스트<br /><br />OAuth 2.0 Authorization Code with PKCE |
| 기본 요청 [요율 제한](/ko/resources/fundamentals/rate-limits) | 15분당 15개 요청(사용자당) | 15분당 50개 요청(사용자당) |
| 앱당 일일 최대 작업 수 | 없음 | 500 |
| [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 자격 증명 사용 필요 |     | ✔️  |
