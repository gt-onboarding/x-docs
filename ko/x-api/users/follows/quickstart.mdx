<div id="getting-started-with-the-manage-follows-endpoints">
  ## 팔로우 관리 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 [Postman](/ko/tutorials/postman-getting-started)을 사용해 팔로우 관리 엔드포인트로 첫 요청을 보내는 방법을 안내합니다.

여러 언어의 샘플 코드를 보려면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요.

<Note>
  ### 사전 준비 사항

  이 가이드를 완료하려면 요청을 인증할 [키와 토큰](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따라 키와 토큰을 생성하세요:

  - [개발자 계정 신청](https://developer.x.com/en/apply-for-access) 후 승인을 받습니다.
  - 개발자 포털에서 [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성합니다.
  - 앱의 “Keys and tokens” 페이지로 이동해 필요한 자격 증명을 생성합니다. 모든 자격 증명은 안전한 위치에 보관하세요.
</Note>

<div id="steps-to-build-a-manage-follows-request">
  ### 팔로우 관리 요청을 구성하는 단계
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### 1단계: 도구 또는 라이브러리로 시작
</div>

이 엔드포인트에 요청을 보내는 데 사용할 수 있는 도구, 코드 예제, 라이브러리가 여러 가지 있지만, 여기서는 과정을 단순화하기 위해 Postman을 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 로드하려면 아래 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Postman에 X API v2 추가
</Button>

Postman에 X API v2 컬렉션을 로드한 후 “Follows” 폴더로 이동하여 “Follow a user ID”를 선택하세요.
 

<div id="step-two-authenticate-your-request">
  #### 2단계: 요청 인증
</div>

X API에 적절히 요청하려면 권한을 확인해야 합니다. 이 엔드포인트의 경우 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 중 하나로 요청을 인증해야 합니다.

이 예제에서는 OAuth 1.0a User Context를 사용합니다.

Postman에 키와 토큰—구체적으로 API Key, API Secret Key, OAuth 1.0a user Access Token, OAuth 1.0a user Access Token Secret—을 추가해야 합니다. Postman 우측 상단에서 "X API v2"라는 환경을 선택한 뒤, 환경 드롭다운 옆의 눈 아이콘을 클릭하여 "initial value" 및 "current value" 필드에 키와 토큰을 입력하세요.

올바르게 설정했다면, 이 변수들은 요청의 Authorization 탭에 자동으로 채워집니다.
 

<div id="step-three-specify-who-is-going-to-follow-whom">
  #### 3단계: 누가 누구를 팔로우할지 지정하기
</div>

Manage follows 엔드포인트는 두 개의 ID를 받습니다. 하나는 소스 사용자(다른 사용자를 팔로우하거나 언팔로우하려는 사용자)용이고, 다른 하나는 타깃 사용자(팔로우되거나 언팔로우될 사용자)용입니다. 소스 사용자의 ID는 인증 사용자(로그인한 사용자)의 사용자 ID와 일치해야 합니다. 이 경우 본인의 사용자 ID를 지정하면 됩니다. ID를 확인하는 방법은 두 가지입니다:

1. [user lookup by username](/ko/x-api/users/user-lookup-by-id) 엔드포인트를 사용해 사용자 이름을 전달하면 id 필드를 받을 수 있습니다.
2. 액세스 토큰을 확인하면 숫자 부분이 본인의 사용자 ID입니다.

타깃 ID는 유효한 어떤 사용자 ID든 가능합니다. 예를 들어 @XDevelopers의 사용자 ID는 2244994945입니다.

Postman에서 "Params" 탭으로 이동하여 id 경로 변수의 "Value" 열에 본인의 ID를 입력합니다. “Body” 탭으로 이동해 target\_user\_id 매개변수의 값으로 2244994945(@XDevelopers의 사용자 ID)를 입력합니다. ID 앞뒤에 공백이 없도록 주의하세요.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | (your user ID) |
| target\_user\_id | 2244994945 |

"Send" 버튼을 클릭하면 관계 상태를 포함하는 응답 객체를 받습니다:

- "following": true를 받으면 해당 id가 target\_user\_id를 성공적으로 팔로우한 것입니다.
- "pending": true를 받으면 target\_user\_id가 보호 계정이므로 팔로우 요청을 수락해야 합니다.

####

4단계: 요청을 보내고 응답을 확인하세요

모든 설정이 완료되면 "Send" 버튼을 눌러 다음과 같은 응답을 받게 됩니다:

```{
    "data": {
        "following": true,
        "pending_follow": false
    }
}
```

마찬가지로 사용자를 언팔로우하려는 경우에는 동일한 Postman 컬렉션에서 "Unfollow a user ID" 요청을 사용하면 됩니다. 다만, unfollow 엔드포인트를 사용할 때는 source\_user\_id와 target\_user\_id 매개변수를 경로 변수로 전달해야 합니다.
