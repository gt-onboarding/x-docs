<div id="getting-started-with-the-list-lookup-endpoint">
  ## 목록 조회 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 Postman을 사용해 목록 조회 엔드포인트로 첫 요청을 보내는 방법을 안내합니다.

여러 언어의 샘플 코드를 보고 싶다면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요.

**참고:** 이 예제에서는 _ID로 목록 조회_ 엔드포인트에 요청을 보내지만, 여기서 배운 내용을 다른 조회 요청에도 동일하게 적용할 수 있습니다.

<Note>
  ### 사전 준비 사항

  이 가이드를 완료하려면 요청을 인증하기 위한 [keys and tokens](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따라 이러한 키와 토큰을 생성할 수 있습니다.

  - [개발자 계정에 가입](https://developer.x.com/en/apply-for-access)하고 승인을 받습니다.
  - 개발자 포털에서 [프로젝트](/ko/resources/fundamentals/projects)와 연관된 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성합니다.
  - 앱의 “Keys and tokens” 페이지로 이동해 필요한 자격 증명을 생성합니다. 모든 자격 증명은 안전한 위치에 보관하세요.
</Note>

<div id="steps-to-build-a-list-lookup-request">
  ### List 조회 요청을 빌드하는 단계
</div>

**1단계: 도구 또는 라이브러리로 시작하기**

이 엔드포인트에 요청을 보내는 데 사용할 수 있는 다양한 도구, 코드 예제, 라이브러리가 있습니다. 여기서는 과정을 단순화하기 위해 Postman을 사용합니다.

X API v2 Postman 컬렉션을 환경에 로드하려면 다음 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Postman에 X API v2 추가
</Button>

Postman에 X API v2 컬렉션이 로드되면 “List” 폴더로 이동하고, “List lookup” 폴더를 선택한 다음 "List by ID"를 선택하세요.
 

**2단계: 요청 인증하기**

X API에 올바르게 요청하려면 권한이 있는지 확인해야 합니다. 이 엔드포인트에서는 [App only](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), [OAuth 2.0 Authorization Code with PKCE](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), 또는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 중 하나로 요청을 인증해야 합니다.

단순화를 위해 이 요청에서는 App only를 사용하지만, 비공개 [지표](/ko/x-api/fundamentals/metrics)나 List를 요청하려면 다른 인증 방법 중 하나를 사용해야 합니다. 

App only를 사용하려면 키와 토큰(특히 [App only Access Token](/ko/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token))을 Postman에 추가해야 합니다. Postman 우측 상단의 “X API v2” 환경을 선택하고, "initial value"와 "current value" 필드에 키와 토큰을 입력하세요(환경 드롭다운 옆 눈 아이콘 클릭).

올바르게 설정했다면 이러한 변수는 요청의 Authorization 탭에 자동으로 반영됩니다.
 

**3단계: 가져올 List 식별 및 지정하기**

요청에서 수신하려는 List를 지정해야 합니다. x.com에서 List를 클릭한 후 URL에서 List ID를 확인할 수 있습니다. 예를 들어, 다음 URL의 List ID는 84839422입니다.

https://x.com/i/lists/84839422

대상 ID는 유효한 어떤 List ID도 가능합니다. Postman에서 "Params" 탭으로 이동하여 id 경로 변수의 "Value" 열에 ID를 입력하세요. ID 앞뒤에 공백이 없도록 주의하세요.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| id  | 84839422 (List ID) |

**4단계: 가져올 필드 식별 및 지정하기**

3단계를 마친 뒤 "Send" 버튼을 클릭하면 응답에서 기본 [List object](/ko/x-api/fundamentals/data-dictionary#list) 필드(id, name)를 받게 됩니다.

추가 필드를 받으려면 요청에서 list.fields 및/또는 [expansion](/ko/x-api/fundamentals/expansions) 매개변수로 해당 필드를 지정해야 합니다.

이 연습에서는 서로 다른 오브젝트에서 다음 세 가지 추가 필드 세트를 요청합니다:

- 기본 Lists 오브젝트의 추가 created\_at 필드

- expansion 매개변수를 사용한 전체 [user object](/ko/x-api/fundamentals/data-dictionary#user)

- 연관된 user 오브젝트의 추가 user.created\_at 필드

Postman에서 "Params" 탭으로 이동하여 "Query Params" 테이블에 다음 key:value 쌍을 추가하세요:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| list.fields | created\_at | created\_at |
| expansions | owner\_id | includes.users.id,  <br />includes.users.name,  <br />includes.users.username |
| user.fields | created\_at | includes.users.created\_at |

이제 “Send” 버튼 옆에 다음과 유사한 URL이 표시됩니다:

`https://api.x.com/2/lists/84839422?list.fields=owner_id&expansions=owner_id&user.fields=created_at`

**5단계: 요청 보내고 응답 확인하기**

모든 설정을 완료했다면 "Send" 버튼을 클릭하세요. 그러면 아래 예시와 유사한 응답을 받게 됩니다:

```
{
  "data": {
    "id": "84839422",
    "name": "Official Twitter Accounts",
    "owner_id": "783214"
  },
  "includes": {
    "users": [
      {
        "name": "Twitter",
        "created_at": "2007-02-20T14:35:54.000Z",
        "username": "Twitter",
        "id": "783214"
      }
    ]
  }
}
```
