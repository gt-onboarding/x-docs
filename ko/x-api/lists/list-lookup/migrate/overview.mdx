  ## X API의 리스트 조회 엔드포인트 비교
</div>

v2 리스트 조회 엔드포인트 그룹은 표준 v1.1 [GET lists/show](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-show) 및 [GET lists/ownership](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-ownerships) 엔드포인트를 대체합니다. 리스트 조회 엔드포인트의 이 버전들 중 하나를 사용하는 코드, 앱, 또는 도구가 있고 더 최신의 X API v2 엔드포인트로 마이그레이션을 고려 중이라면, 이 가이드 모음이 도움이 됩니다.

다음 표는 표준 v1.1과 X API v2 리스트 엔드포인트를 비교합니다:

**ID로 리스트 조회**

|     |     |     |
| :--- | :--- | :--- |
| 설명 | 표준 v1.1 | X API v2 |
| 지원되는 HTTP 메서드 | `GET` | `GET` |
| 호스트 도메인 | `https://api.x.com` | `https://api.x.com` |
| 엔드포인트 경로 | `/1.1/lists/show.json` | `/2/lists/:id` |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 1.0a 사용자 컨텍스트<br /><br />앱 전용 | OAuth 1.0a 사용자 컨텍스트<br /><br />OAuth 2.0 Authorization Code with PKCE<br /><br />앱 전용 |
| 기본 요청 [레이트 리밋](/ko/resources/fundamentals/rate-limits) | OAuth 1.0a: 15분당 75회<br /><br />OAuth 2.0: 15분당 75회<br /><br />앱 전용: 15분당 75회 | OAuth 1.0a: 15분당 75회<br /><br />OAuth 2.0: 15분당 75회<br /><br />앱 전용: 15분당 75회 |

**사용자 소유 리스트 조회**

|     |     |     |
| :--- | :--- | :--- |
| 설명 | 표준 v1.1 | X API v2 |
| 지원되는 HTTP 메서드 | `GET` | `GET` |
| 호스트 도메인 | `https://api.x.com` | `https://api.x.com` |
| 엔드포인트 경로 | `/1.1/lists/ownerships.json` | `/2/users/:id/owned_lists` |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 1.0a 사용자 컨텍스트<br /><br />앱 전용 | OAuth 1.0a 사용자 컨텍스트<br /><br />OAuth 2.0 Authorization Code with PKCE<br /><br />앱 전용 |
| 기본 요청 [레이트 리밋](/ko/resources/fundamentals/rate-limits) | OAuth 1.0a: 15분당 15회<br /><br />앱 전용: 15분당 15회 | OAuth 1.0a: 15분당 15회<br /><br />OAuth 2.0: 15분당 15회<br /><br />앱 전용: 15분당 15회 |

X API v2 엔드포인트에 접근하려면 [개발자 계정을 등록](https://developer.x.com/en/portal/petition/essential/basic-info)해야 합니다. 인증 시에는 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용해야 하며, 해당 앱은 [프로젝트](/ko/resources/fundamentals/projects) 내에 있어야 합니다.

X API v2 엔드포인트 액세스에 대한 자세한 내용은 [시작하기](/ko/x-api/getting-started/getting-access) 페이지를 참고하세요.
