<div id="list-members-lookup">
  ### 리스트 멤버 조회
</div>

멤버 조회 그룹에는 사용할 수 있는 엔드포인트가 두 개 있습니다. 지정한 리스트의 멤버 세부 정보를 가져오고, 특정 사용자가 어떤 리스트의 멤버인지 확인할 수 있습니다. 이 엔드포인트는 멤버십 정보를 바탕으로 새로운 리스트를 선별하고 구성하는 데 활용할 수 있습니다.

멤버 세부 정보를 조회할 때는 15분당 900회, 사용자 멤버십을 조회할 때는 15분당 75회의 요청 한도가 적용됩니다.

이 엔드포인트에 대한 요청을 인증하려면 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication), [App only](https://developer.x.com/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), 또는 [OAuth 2.0 Authorization Code with PKCE](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용할 수 있습니다. 
 

<div id="manage-list-members">
  ### 목록 구성원 관리
</div>

목록 구성원 관리 엔드포인트를 사용하면 인증된 사용자를 대신해 목록에 구성원을 추가하거나 제거할 수 있습니다. 이 엔드포인트에는 POST와 DELETE 두 가지 메서드가 있습니다. POST는 목록에 구성원을 추가하고, DELETE는 목록에서 구성원을 제거합니다. 두 엔드포인트 모두 사용자 기준 15분당 300건의 요청 한도가 적용됩니다.

목록의 구성원 수는 5,000명을 초과할 수 없습니다.

이 엔드포인트로 사용자를 대신해 요청하므로, [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)로 해당 사용자를 인증해야 하며, 귀하의 앱을 승인한 사용자와 연결된 사용자 액세스 토큰을 사용해야 합니다. 이 토큰은 [3-legged OAuth flow](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) 또는 [Authorization Code with PKCE grant flow](/ko/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0)를 통해 발급받을 수 있습니다.

<Note>
  **계정 설정**

  이 엔드포인트에 액세스하려면 다음이 필요합니다.

  - 승인된 [developer account](https://developer.x.com/en/portal/petition/essential/basic-info)
  - [Project](/ko/resources/fundamentals/projects) 내의 [developer App](/ko/resources/fundamentals/developer-apps)에서 발급된 키와 토큰을 사용한 인증

  X API v2 엔드포인트 액세스 방법은 [getting started guide](/ko/x-api/getting-started/getting-access)를 참고하세요.
</Note>
