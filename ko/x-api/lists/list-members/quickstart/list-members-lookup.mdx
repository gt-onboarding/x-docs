<div id="steps-to-build-a-list-members-lookup-request">
  ### 목록 멤버 조회 요청을 구성하는 단계
</div>

**1단계: Postman에서 List 엔드포인트 컬렉션 선택**

Postman에 X API v2 컬렉션을 불러온 후 “List” 폴더로 이동하여 “List members” 폴더를 선택한 다음 "Members lookup"을 선택합니다.
 

**2단계: 멤버를 조회할 목록을 식별하고 지정**

멤버를 조회할 목록을 지정해야 합니다. x.com에서 목록을 클릭한 뒤 URL에서 목록 ID를 확인할 수 있습니다. 예를 들어, 다음 URL의 목록 ID는 84839422입니다.

https://x.com/i/lists/84839422

대상 ID는 유효한 목록 ID라면 어떤 값이든 가능합니다. Postman에서 "Params" 탭으로 이동하여 id 경로 변수의 "Value" 열에 ID를 입력합니다. ID 앞뒤에 공백이 없도록 주의하세요.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 84839422 (List ID) |

**3단계: 가져올 필드를 식별하고 지정**

3단계 후 "Send" 버튼을 클릭하면 응답에서 기본 [user object](/ko/x-api/fundamentals/data-dictionary#user) 필드인 id, name, username을 받게 됩니다.

id, name, username 외의 추가 필드를 받으려면 요청에 [field](/ko/x-api/fundamentals/fields) 및/또는 [expansion](/ko/x-api/fundamentals/expansions) 파라미터로 해당 필드를 지정해야 합니다.

이 연습에서는 서로 다른 오브젝트에서 다음 세 가지 추가 필드 집합을 요청합니다:

- 기본 사용자 오브젝트의 추가 user.created\_at 필드
- expansion 파라미터를 사용한 전체 [Post object](/ko/x-api/fundamentals/data-dictionary#tweet)
- 연관된 Post 오브젝트의 추가 tweet.created\_at 필드

Postman에서 "Params" 탭으로 이동하여 "Query Params" 표에 다음 key:value 쌍을 추가합니다:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| user.fields | created\_at | `created_at` |
| expansions | pinned\_tweet\_id | `includes.tweets.id,  <br/>includes.tweets.text  <br/>` |
| tweet.fields | created\_at | `includes.tweets.created_at` |

이제 “Send” 버튼 옆에 다음과 유사한 URL이 표시됩니다:

`https://api.x.com/2/lists/84839422/members?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=created_at`
