<div id="getting-started-with-the-list-members-endpoint-group">
  ## List members 엔드포인트 그룹 시작하기
</div>

이 간단한 개요는 [Postman](/ko/tutorials/postman-getting-started)을 사용해 List members 엔드포인트로 첫 요청을 보내는 데 도움이 됩니다.

다양한 언어의 샘플 코드를 보려면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요.

<Note>
  **사전 준비 사항**

  이 가이드를 완료하려면 요청 인증에 사용할 [keys and tokens](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따라 해당 키와 토큰을 생성할 수 있습니다:

  - [개발자 계정 신청](https://developer.x.com/en/apply-for-access) 후 승인을 받습니다.
  - 개발자 포털에서 [Project](/ko/resources/fundamentals/projects)와 연결된 [developer App](/ko/resources/fundamentals/developer-apps)을 생성합니다.
  - App의 “Keys and tokens” 페이지로 이동해 필요한 자격 증명을 생성합니다. 모든 자격 증명은 안전한 위치에 보관하세요.
</Note>

<div id="authenticate-your-request">
  #### 요청 인증
</div>

**lookup** 엔드포인트에 성공적으로 요청하려면 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication#oauth-1-0a-2), [앱 전용](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only), 또는 [OAuth 2.0 Authorization Code with PKCE](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용할 수 있습니다. 반면, **manage** 엔드포인트는 OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0 Authorization Code with PKCE로만 인증할 수 있습니다.

어느 방식을 사용하더라도 Postman에서 환경을 올바르게 설정했다면 기본 인증 키와 토큰이 요청에 자동으로 채워집니다.

Postman 오른쪽 상단에서 “X API v2” 환경을 선택하고, 환경 드롭다운 옆의 눈 아이콘을 클릭하여 "initial value"와 "current value" 필드에 키와 토큰을 추가합니다. 포함해야 하는 키는 다음과 같습니다:

- API Key(Consumer Key라고도 함)
- API Secret Key(Consumer Secret이라고도 함)
- OAuth 1.0a 사용자 Access Token
- OAuth 1.0a 사용자 Access Token Secret
- 앱 전용 Access Token
- OAuth 2.0 Client Key(앱 설정에서 OAuth 2.0 사용자 인증 설정을 구성한 경우에만 사용 가능)
- OAuth 2.0 Client Secret(앱 설정에서 OAuth 2.0 사용자 인증 설정을 구성한 경우에만 사용 가능)
