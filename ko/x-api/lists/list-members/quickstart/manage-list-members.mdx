<Note>

<div id="steps-to-build-a-manage-list-member-request">
  ### 리스트 멤버 관리 요청을 구성하는 단계
</div>

<div id="step-one-choose-the-list-endpoint-collection-from-postman">
  #### 1단계: Postman에서 List 엔드포인트 컬렉션 선택
</div>

Postman에서 X API v2 컬렉션을 불러온 후 “List” 폴더로 이동해 “List members” 폴더를 선택한 다음 “Add a member”를 선택하세요.
 

<div id="step-two-specify-the-user-to-add">
  #### 2단계: 추가할 사용자 지정
</div>

List 멤버 관리 엔드포인트에는 두 개의 ID가 필요합니다: List의 ID와 추가할 사용자의 ID. [users lookup](/ko/x-api/users/lookup/introduction)을 사용해 사용자의 ID를 조회하고, 사용자 이름(username)을 전달하면 id 필드를 받을 수 있습니다.

대상 ID는 유효한 사용자 ID라면 무엇이든 가능합니다. Postman에서 "Params" 탭으로 이동하여 id 경로 변수의 "Value" 열에 자신의 ID를 입력하세요. “Body” 탭으로 이동하여 추가하려는 사용자의 ID를 user\_id 매개변수의 값으로 입력하세요. 어떤 ID 앞뒤에도 공백이 없도록 주의하세요.

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Parameter type** |
| `id` | List ID | path |
| user\_id | 멤버로 추가하려는 대상 사용자 ID | body |

이제 "Send" 버튼 옆에 다음과 유사한 URL이 표시됩니다:

`https://api.x.com/2/lists/1441162269824405510/members`

3단계: 요청 전송 및 응답 확인

모든 설정을 마쳤다면 "Send" 버튼을 클릭하면, 아래 예시와 유사한 응답을 받게 됩니다:

```
{
  "data": {
    "is_member": true
  }
}
```

반환된 응답 객체에서 is\_member 키의 값이 true라면 사용자가 List의 멤버로 성공적으로 추가된 것입니다.

List에서 멤버를 제거하려면, Postman에 로드된 X API v2 컬렉션의 “Lists” 폴더에 있는 “Remove a member” 요청을 선택하세요. 이 엔드포인트에는 List의 ID와 제거하려는 멤버의 사용자 ID가 필요합니다. “Params” 탭에서 id 열의 값으로 List의 ID를, user\_id 열의 값으로 제거할 사용자의 ID를 입력하세요.

삭제 요청이 성공하면 아래 예시와 유사한 응답을 받게 됩니다:

```
{
  "data": {
    "is_member": false
  }
}
```
