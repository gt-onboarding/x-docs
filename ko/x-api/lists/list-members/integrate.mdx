<div id="authentication">
  #### 인증
</div>

모든 X API v2 엔드포인트는 키와 토큰으로도 알려진 자격 증명 세트를 사용해 요청을 인증해야 합니다. Lists **lookup(조회)** 엔드포인트에는 OAuth 1.0a 사용자 컨텍스트, OAuth 2.0 Authorization Code with PKCE, 또는 App only 중 하나를 사용할 수 있습니다. 그러나 Lists **manage(관리)** 엔드포인트에는 반드시 OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0으로 인증해야 합니다.

[OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication)는 성공적인 요청을 위해 API 키와 사용자 액세스 토큰 세트를 사용해야 함을 의미합니다. 액세스 토큰은 요청을 위임받은 해당 사용자와 연결되어 있어야 합니다. 다른 사용자의 액세스 토큰 세트를 생성하려면, 해당 사용자가 [3-legged OAuth 플로우](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)를 통해 귀하의 앱을 승인해야 합니다.

OAuth 1.0a는 다소 사용이 까다로울 수 있습니다. 이 인증 방식에 익숙하지 않다면 [라이브러리](/ko/x-api/tools-and-libraries/overview)나 Postman과 같은 도구를 사용하거나, OAuth 2.0 또는 App only 방식으로 요청을 인증하는 것을 권장합니다.

[OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)는 애플리케이션의 스코프를 더 세밀하게 제어하고, 여러 기기에 걸친 인가 플로우를 지원합니다. OAuth 2.0을 사용하면 사용자 대신 수행할 수 있는 권한을 부여하는 정교한 스코프를 선택할 수 있습니다.

앱에서 OAuth 2.0을 사용하려면, 개발자 포털의 앱 설정 섹션에 있는 인증 설정에서 이를 활성화해야 합니다.

[App only](/ko/resources/fundamentals/authentication#oauth-2-0)는 요청에 [App only Access Token](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)만 포함하면 됩니다. 개발자 앱 내에서 직접 App only Access Token을 생성하거나, [POST oauth2/token](/ko/resources/fundamentals/authentication#post-oauth2-token) 엔드포인트를 사용해 생성할 수 있습니다.

[App only](/ko/resources/fundamentals/authentication#oauth-2-0)는 요청에 [App only Access Token](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)만 포함하면 됩니다. 개발자 앱 내에서 직접 App only Access Token을 생성하거나, [POST oauth2/token](/ko/resources/fundamentals/authentication#post-oauth2-token) 엔드포인트를 사용해 생성할 수 있습니다.

<div id="developer-portal-projects-and-developer-apps">
  #### 개발자 포털, 프로젝트와 개발자 앱
</div>

X API v2 엔드포인트에서 사용할 인증 자격 증명을 받으려면 [개발자 계정에 가입](https://developer.x.com/en/portal/petition/essential/basic-info)한 뒤 해당 계정 내에 [프로젝트](/ko/resources/fundamentals/projects)를 설정하고, 그 프로젝트 내에 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성해야 합니다. 이후 개발자 앱에서 키와 토큰을 확인할 수 있습니다.
 

<div id="rate-limits">
  #### 할당량 제한
</div>

매일 수천 명의 개발자가 X API에 요청을 보냅니다. 이러한 대량의 요청을 관리하기 위해 각 엔드포인트에는 [할당량 제한](/ko/x-api/fundamentals/rate-limits)이 적용되어, 앱 또는 인증된 사용자를 대신해 보낼 수 있는 요청 수가 제한됩니다. 

조회(GET) 엔드포인트에는 앱 수준과 사용자 수준의 할당량 제한이 모두 적용되고, 관리(POST/DELETE) 엔드포인트에는 사용자 수준의 제한이 적용됩니다. 앱 할당량 제한은 개발자인 여러분이 특정 기간 동안 어떤 앱에서든(API Key와 API Secret Key 또는 App-only Access Token을 사용하는 경우) 해당 엔드포인트로 보낼 수 있는 요청 수가 제한됨을 의미합니다. 사용자 할당량 제한은 여러분이 요청을 대신 수행하는 인증된 사용자가 어떤 개발자 앱을 사용하더라도 목록 조회를 수행할 수 있는 횟수가 제한됨을 의미합니다.

아래 표는 각 엔드포인트의 할당량 제한을 보여줍니다.

|     |     |     |
| :--- | :--- | :--- |
| **엔드포인트** | **HTTP 메서드** | **할당량 제한** |
| /2/lists/:id/members | GET | 15분당 요청 900회 |
| /2/users/:id/list\_memberships | GET | 15분당 요청 75회 |
| /2/lists/:id/members | POST | 15분당 요청 300회 |
| /2/lists/:id/members/:user\_id | DELETE | 15분당 요청 300회 |

<div id="fields-and-expansions">
  #### 필드와 확장
</div>

X API v2 GET 엔드포인트를 사용하면 `fields`와 `expansions`라는 도구 세트를 통해 API에서 어떤 데이터를 반환받을지 정확히 선택할 수 있습니다. `expansions` 매개변수는 페이로드에서 참조되는 객체를 펼쳐 포함하도록 합니다. 예를 들어, 리스트 멤버 조회에서는 다음 [expansions](/ko/x-api/fundamentals/expansions)을 가져올 수 있습니다:

- `pinned_tweet_id`

`fields` 매개변수는 다양한 데이터 객체 내에서 받고자 하는 정확한 [fields](/ko/x-api/fundamentals/fields)를 선택할 수 있게 합니다. 리스트 멤버 조회는 주로 사용자 객체를 반환합니다. 기본적으로 사용자 객체는 id, name, username 필드를 반환합니다. `user.created_at` 또는 `user.description`과 같은 추가 필드를 받으려면 user.fields 매개변수를 사용해 해당 항목을 명시적으로 요청해야 합니다. 

[fields and expansions](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) 사용 방법에 관한 가이드를 추가했습니다.

아래 표는 각 조회 엔드포인트에서 사용 가능한 필드와 확장을 보여줍니다:

|     |     |     |
| :--- | :--- | :--- |
| **엔드포인트** | **필드** | **확장** |
| /2/lists/:id/members | `user.fields`<br /><br />`tweet.fields` | `pinned_tweet_id` |
| /2/users/:id/list\_memberships | list.fields<br /><br />user.fields | owner\_id |

<div id="pagination">
  #### 페이지네이션
</div>

membership/members 조회는 방대한 데이터를 반환할 수 있습니다. 언제나 일관되고 높은 성능의 결과를 제공하기 위해 페이지네이션을 사용합니다. 페이지네이션은 단일 응답에 모두 담을 수 없는 결과가 있을 때 X API v2 엔드포인트에서 추가 결과를 분할해 반환하는 기능입니다. 이 경우 데이터는 일련의 ‘페이지’로 제공됩니다. [결과를 페이지 단위로 탐색하는 방법](/ko/x-api/fundamentals/pagination)에 대해 더 알아보세요.
