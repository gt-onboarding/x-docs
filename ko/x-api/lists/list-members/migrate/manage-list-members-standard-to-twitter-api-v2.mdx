<div id="manage-list-members-standard-v11-compared-to-x-api-v2">
  ### 리스트 멤버 관리: Standard v1.1과 X API v2 비교
</div>

Standard v1.1의 [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create) 및 [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy) 엔드포인트를 사용해 오셨다면, 이 가이드는 Standard v1.1과 X API v2의 리스트 멤버 관리 엔드포인트 간 유사점과 차이점을 이해하는 데 도움을 드립니다.

- **유사점**
  - 인증
- **차이점**
  - 엔드포인트 URL
  - 앱 및 프로젝트 요건
  - HTTP 메서드
  - 호출 한도
  - 요청 파라미터

<div id="similarities">
  #### 유사점
</div>

**인증**

두 엔드포인트 버전 모두 [OAuth 1.0a 사용자 컨텍스트](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication)를 지원합니다. 따라서 이전에 표준 v1.1의 목록 멤버 관리 엔드포인트 중 하나를 사용했다면, X API v2로 마이그레이션하더라도 동일한 인증 방식을 계속 사용할 수 있습니다.

<div id="differences">
  #### 차이점
</div>

**엔드포인트 URL**

- Standard v1.1 엔드포인트:
  - POST https://api.x.com/1.1/lists/members/create.json
    (지정된 리스트에 멤버 추가)
  - POST https://api.x.com/1.1/lists/members/destroy.json
    (지정된 리스트에서 멤버 제거)
- X API v2 엔드포인트:
  - POST https://api.x.com/2/lists/:id/members
    (지정된 리스트에 멤버 추가)

  - DELETE https://api.x.com/2/lists/:id/members/:user\_id
    (지정된 리스트에서 멤버 제거)

**요청 한도**

| **Standard v1.1** | **X API v2** |
| :--- | :--- |
| /1.1/lists/members/create.json<br /><br />없음 | /2/lists/:id/members<br /><br />OAuth 1.0a 사용자 컨텍스트: 15분당 300회<br /><br />OAuth 2.0 Authorization Code with PKCE: 15분당 300회 |
| /1.1/lists/members/destroy.json<br /><br />없음 | /2/lists/:id/members/:user\_id<br /><br />OAuth 1.0a 사용자 컨텍스트: 15분당 300회<br /><br />OAuth 2.0 Authorization Code with PKCE: 15분당 300회 |

**앱 및 프로젝트 요구 사항**

X API v2 엔드포인트는 요청 인증 시 [developer App](/ko/resources/fundamentals/developer-apps)의 자격 증명을 사용해야 하며, 해당 앱은 [Project](/ko/resources/fundamentals/projects)와 연결되어 있어야 합니다. X API v1.1의 모든 엔드포인트는 독립 실행형 앱 또는 프로젝트와 연결된 앱의 자격 증명을 사용할 수 있습니다.

**요청 매개변수**

다음 Standard v1.1 요청 매개변수는 X API v2에서의 대응 항목입니다:

| **Standard v1.1** | **X API v2** |
| :--- | :--- |
| list\_id | id  |
| slug | 해당 없음 |
| screen\_name | 해당 없음 |
| owner\_screen\_name | 해당 없음 |
| owner\_id | 해당 없음 |
