<div id="list-members-lookup-standard-v11-compared-to-x-api-v2">
  ### 리스트 멤버 조회: 표준 v1.1과 X API v2 비교
</div>

표준 v1.1의 [GET lists/members](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-members) 및 [GET lists/memberships](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-memberships) 엔드포인트를 사용해 오셨다면, 이 가이드는 표준 v1.1과 X API v2의 리스트 멤버 엔드포인트 간 유사점과 차이점을 이해하는 데 도움을 드립니다.

- **유사점**
  - 인증 방법
- **차이점**
  - 엔드포인트 URL
  - 레이트 리밋
  - 앱 및 프로젝트 요건
  - 요청당 데이터 객체 한도
  - 응답 데이터 형식
  - 요청 매개변수

<div id="similarities">
  #### 유사점
</div>

**인증**

두 엔드포인트 버전 모두 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication#oauth-1-0a-2)와 [App only](/ko/resources/fundamentals/authentication#oauth-2-0)를 지원합니다. 따라서 이전에 표준 v1.1 List members 엔드포인트 중 하나를 사용했다면 X API v2로 마이그레이션해도 동일한 인증 방식을 계속 사용할 수 있습니다.

사용하는 인증 라이브러리/패키지에 따라 App only 인증이 가장 간단하게 시작할 수 있는 방법일 가능성이 높으며, 간단한 요청 헤더로 설정할 수 있습니다. App only 액세스 토큰을 생성하는 방법은 [이 App only 가이드](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)를 참고하세요.

<div id="differences">
  #### 차이점
</div>

**엔드포인트 URL**

- Standard v1.1 엔드포인트:
  - GET https://api.x.com/1.1/lists/members.json
    (지정한 리스트의 멤버 조회)
  - GET https://api.x.com/1.1/lists/memberships.json
    (사용자가 멤버로 속한 리스트 조회)
- X API v2 엔드포인트:
  - GET https://api.x.com/2/lists/:id/members
    (지정한 리스트의 멤버 조회)

  - GET https://api.x.com/2/users/:id/list\_memberships
    (사용자가 멤버로 속한 리스트 조회)

**요청 한도**

|     |     |
| :--- | :--- |
| **Standard v1.1** | **X API v2** |
| /1.1/lists/members.json<br /><br />OAuth 1.0a 사용자 컨텍스트 기준 15분당 900회<br /><br />앱 전용 15분당 15회 | /2/lists/:id/members<br /><br />OAuth 1.0a 사용자 컨텍스트 기준 15분당 900회<br /><br />OAuth 2.0 Authorization Code with PKCE 기준 15분당 900회<br /><br />앱 전용 15분당 900회 |
| /1.1/lists/memberships.json<br /><br />OAuth 1.0a 사용자 컨텍스트 기준 15분당 15회<br /><br />앱 전용 15분당 15회 | /2/users/:id/list\_memberships<br /><br />OAuth 1.0a 사용자 컨텍스트 기준 15분당 15회<br /><br />OAuth 2.0 Authorization Code with PKCE 기준 15분당 15회<br /><br />앱 전용 15분당 15회 |

**앱 및 프로젝트 요구 사항**

X API v2 엔드포인트는 요청 인증 시 [developer App](/ko/resources/fundamentals/developer-apps)의 자격 증명을 사용해야 하며, 해당 App은 [Project](/ko/resources/fundamentals/projects)에 연결되어 있어야 합니다. 모든 X API v1.1 엔드포인트는 독립 실행형 App 또는 프로젝트에 연결된 App의 자격 증명을 사용할 수 있습니다.

**요청당 데이터 오브젝트 한도**

Standard v1.1의 /1.1/lists/members 엔드포인트는 요청당 최대 5,000명의 사용자를 반환합니다. 새로운 v2 엔드포인트는 요청당 최대 100명의 사용자를 반환합니다. 기본적으로 100개의 사용자 오브젝트가 반환되며, 결과 수를 변경하려면 1~100 사이의 숫자와 함께 쿼리 매개변수 max\_results= 를 전달해야 합니다. 그런 다음 응답 페이로드에 반환된 next\_token을 다음 요청의 pagination\_token 쿼리 매개변수로 전달하면 됩니다.

또한 /1.1/lists/memberships 엔드포인트는 요청당 최대 1,000개의 리스트를 반환합니다. v2 대체 엔드포인트에서는 요청당 최대 100개의 리스트를 허용합니다. 기본적으로 100개의 리스트 오브젝트가 반환되며, 결과 수를 변경하려면 /1.1/lists/members와 동일한 방식으로 쿼리 매개변수 max\_results= 및 pagination\_token을 사용하세요.

**응답 데이터 형식**

Standard v1.1과 X API v2 엔드포인트 버전 간의 가장 큰 차이점 중 하나는 페이로드에 어떤 필드를 반환할지 선택하는 방식입니다.

Standard 엔드포인트의 경우 기본적으로 많은 응답 필드를 받으며, 추가로 어떤 필드 또는 필드 집합을 페이로드에 반환할지 매개변수로 지정할 수 있습니다.

X API v2의 /users/:id/list\_memberships 는 기본적으로 리스트 id와 name 필드를 제공합니다. 추가 필드 또는 오브젝트를 요청하려면 [fields](/ko/x-api/fundamentals/fields) 및 [expansions](/ko/x-api/fundamentals/expansions) 매개변수를 사용해야 합니다. 이 엔드포인트에서 요청한 리스트 필드는 기본 리스트 오브젝트에 포함되어 반환됩니다. 확장된 오브젝트와 필드는 응답의 includes 오브젝트로 반환됩니다. 이후 기본 및 확장 오브젝트에 공통으로 포함된 ID를 매칭하여 확장 오브젝트를 기본 리스트 오브젝트와 연결할 수 있습니다.

가능한 리스트 필드와 확장의 예는 다음과 같습니다:

- created\_at

- follower\_count

- member\_count

- owner\_id

- description

- private

|     |     |
| :--- | :--- |
| **엔드포인트** | **확장** |
| /2/lists/:id/members | pinned\_tweet\_id |
| /2/users/:id/list\_memberships | owner\_id |

이러한 신규 매개변수에 대해서는 각 가이드를 참고하시거나, [fields 및 expansions 사용 방법](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) 가이드를 읽어보시기 바랍니다. 

또한 표준 v1.1 필드를 최신 v2 필드에 매핑하는 데 도움이 되는 [데이터 형식 마이그레이션 가이드](/ko/x-api/migrate/data-format-migration)를 준비했습니다. 이 가이드는 특정 필드를 반환하기 위해 v2 요청에 포함해야 하는 구체적인 expansion 및 field 매개변수도 제공합니다. 

특정 필드의 요청 방식 변경과 더불어, X API v2는 [Post](/ko/x-api/fundamentals/data-dictionary#tweet) 및 [user](/ko/x-api/fundamentals/data-dictionary#user) 객체를 포함해 API가 반환하는 객체에 대해 새로운 JSON 설계를 도입합니다.

- JSON 루트 수준에서 표준 엔드포인트는 statuses 배열에 Post 객체를 반환하는 반면, X API v2는 data 배열을 반환합니다. 

- 예전에는 리트윗과 인용을 "statuses"로 지칭했으나, X API v2 JSON에서는 리트윗과 인용 트윗으로 표기합니다. contributors 및 user.translator\_type과 같은 많은 레거시/사용 중단 필드는 제거됩니다. 

- Post 객체의 favorites와 user 객체의 favourites를 모두 사용하던 대신, X API v2는 like라는 용어를 사용합니다. 

- X는 값이 없는 JSON 항목(예: null)은 페이로드에 기록하지 않는 규칙을 채택하고 있습니다. Post 및 user 속성은 값이 null이 아닐 때에만 포함됩니다.

**요청 매개변수**

다음 표준 v1.1 요청 매개변수는 X API v2에 해당 항목이 있습니다:

**리스트 멤버 조회**

|     |     |
| :--- | :--- |
| **Standard v1.1** | **X API v2** |
| list\_id | id  |
| slug | 해당 없음 |
| owner\_screen\_name | 해당 없음 |
| owner\_id | 해당 없음 |
| count | max\_results |
| cursor | pagination\_token |
| include\_entities | 해당 없음 |
| skip\_status | 해당 없음 |

**리스트 멤버십 조회**

|     |     |
| :--- | :--- |

| screen\_name | 해당 없음 |

| cursor | pagination\_token |
