<div id="getting-started-with-the-list-posts-lookup-endpoint">
  ## List Posts 조회 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 Postman을 사용해 List Posts 조회 엔드포인트에 첫 요청을 보내는 방법을 안내합니다.

여러 언어의 샘플 코드를 보려면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요.

<Note>
  ### 사전 준비 사항

  이 가이드를 완료하려면 요청 인증에 사용할 [keys and tokens](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따라 키와 토큰을 생성하세요:

  - [개발자 계정 신청](https://developer.x.com/en/apply-for-access) 후 승인을 받습니다.
  - 개발자 포털에서 [Project](/ko/resources/fundamentals/projects)와 연결된 [developer App](/ko/resources/fundamentals/developer-apps)을 생성합니다.
  - App의 “Keys and tokens” 페이지로 이동해 필요한 자격 증명을 생성합니다. 모든 자격 증명은 안전한 위치에 보관하세요.
</Note>

<div id="steps-to-build-a-list-posts-lookup-request">
  ### 목록 게시물 조회 요청을 빌드하는 단계
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### 1단계: 도구 또는 라이브러리로 시작
</div>

이 엔드포인트에 요청을 보내는 데 사용할 수 있는 다양한 도구, 코드 예제, 라이브러리가 있지만 여기서는 과정을 단순화하기 위해 Postman을 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 로드하려면 다음 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Postman에 X API v2 추가
</Button>

Postman에 X API v2 컬렉션이 로드되면 “List” 폴더로 이동한 다음 “List Posts” 폴더를 선택하고, 마지막으로 "List Posts lookup"을 선택하세요.
 

<div id="step-two-authenticate-your-request">
  #### 2단계: 요청 인증
</div>

X API에 올바르게 요청하려면 권한이 있는지 확인해야 합니다. 이 엔드포인트에서 이를 수행하려면 [App only](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), [OAuth 2.0 Authorization Code with PKCE](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), 또는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 인증 방식 중 하나로 요청을 인증해야 합니다.

간단히 하기 위해 이 예시에서는 App only를 사용하지만, 비공개 [지표](/ko/x-api/fundamentals/metrics) 또는 게시물을 요청하려면 다른 인증 방식 중 하나를 사용해야 합니다.

App only를 사용하려면 Postman에서 오른쪽 상단의 환경 드롭다운 옆 눈 아이콘을 클릭해 “X API v2” 환경을 선택한 뒤, 키와 토큰(특히 [App only Access Token](/ko/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token), App only Bearer Token이라고도 함)을 "initial value" 및 "current value" 필드에 추가하세요.

올바르게 설정했다면 이러한 변수는 요청의 Authorization 탭에 자동으로 반영됩니다.
 

<div id="step-three-identify-and-specify-which-list-you-would-like-to-retrieve-posts-from">
  #### 3단계: 어떤 목록에서 게시글을 가져올지 식별하고 지정하기
</div>

요청에서 수신하려는 목록을 반드시 지정해야 합니다. x.com에서 목록을 클릭한 후 URL에서 목록 ID를 확인할 수 있습니다. 예를 들어, 다음 URL의 목록 ID는 84839422입니다.

https://x.com/i/lists/84839422

대상 ID는 유효한 목록 ID라면 무엇이든 사용할 수 있습니다. Postman에서 "Params" 탭으로 이동해 id 경로 변수의 "Value" 열에 해당 ID를 입력하세요. ID 앞뒤에 공백이 포함되지 않도록 주의하세요.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| id  | 84839422 (목록 ID) |

<div id="step-four-identify-and-specify-which-fields-you-would-like-to-retrieve">
  #### 4단계: 가져올 필드 식별 및 지정
</div>

3단계 후 "Send" 버튼을 클릭하면 응답에 기본 [Post object](/ko/x-api/fundamentals/data-dictionary#tweet) 필드인 id와 text가 반환됩니다.

추가 필드를 받으려면 요청의 tweet.fields 및/또는 [expansion](/ko/x-api/fundamentals/expansions) 매개변수에 해당 필드를 지정해야 합니다.

이 연습에서는 서로 다른 오브젝트에서 다음 세 가지 추가 필드 세트를 요청합니다:

- 기본 Lists 오브젝트의 추가 created\_at 필드
- expansion 매개변수를 사용한 전체 [user object](/ko/x-api/fundamentals/data-dictionary#user)
- 연관된 user 오브젝트의 추가 user.created\_at 필드

Postman에서 "Params" 탭으로 이동하여 "Query Params" 표에 다음 키:값 쌍을 추가합니다:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| tweet.fields | created\_at | created\_at |
| expansions | author\_id | includes.users.id,  <br />includes.users.name,  <br />includes.users.username |
| user.fields | created\_at | includes.users.created\_at |

이제 “Send” 버튼 옆에 다음과 유사한 URL이 표시됩니다:

`https://api.x.com/2/lists/84839422/tweets?expansions=author_id&user.fields=created_at&max_results=1`

<div id="step-five-make-your-request-and-review-your-response">
  #### 5단계: 요청을 보내고 응답을 검토하세요
</div>

모든 설정이 완료되면 "Send" 버튼을 클릭하여 요청을 보내세요. 그러면 아래 예시와 유사한 응답을 받게 됩니다:

```
{
  "data": [
    {
      "author_id": "4172587277",
      "id": "1458172421115101189",
      "text": "A Alemanha registrou nesta semana um recorde de novos casos de Covid-19. Segundo o governo e especialistas em Saúde, pessoas não vacinadas são responsáveis pela situação \nhttps://t.co/4POyaPwMLu"
    }
  ],
  "includes": {
    "users": [
      {
        "username": "MomentsBrasil",
        "name": "Twitter Moments Brasil",
        "created_at": "2015-11-12T16:46:02.000Z",
        "id": "4172587277"
      }
    ]
  },
  "meta": {
    "result_count": 1,
    "next_token": "7140dibdnow9c7btw3z2vwioavpvutgzrzm9icis4ndix"
  }
}
```

**참고:** 이 엔드포인트의 응답은 지정된 리스트에 대해 최신 800개의 게시물을 조회하는 것을 지원합니다.
