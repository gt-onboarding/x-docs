<div id="authentication">
  #### 인증
</div>

모든 X API v2 엔드포인트는 키와 토큰으로도 불리는 자격 증명 세트로 요청을 인증해야 합니다. 이 엔드포인트에 대한 요청 인증에는 OAuth 1.0a 사용자 컨텍스트, 앱 전용, 또는 OAuth 2.0 Authorization Code with PKCE 중 하나를 사용할 수 있습니다. 

[OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication#oauth-1-0a-2)는 성공적인 요청을 위해 API 키와 사용자 액세스 토큰 세트를 사용해야 함을 의미합니다. 액세스 토큰은 요청을 수행하는 대상 사용자와 연계되어 있어야 합니다. 다른 사용자의 액세스 토큰 세트를 생성하려면, 해당 사용자가 [3-legged OAuth 플로우](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)를 통해 귀하의 앱을 승인해야 합니다.

OAuth 1.0a는 사용하기 까다로울 수 있습니다. 이 인증 방법에 익숙하지 않다면 [라이브러리](/ko/x-api/tools-and-libraries/overview)를 사용하거나 Postman과 같은 도구를 사용하되, 가능하면 OAuth 2.0 또는 앱 전용 방식을 사용해 요청을 인증할 것을 권장합니다.

[OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)는 애플리케이션 범위에 대한 더 정교한 제어와 여러 기기에 걸친 인증 플로우를 제공합니다. OAuth 2.0을 사용하면 사용자 대신 특정 권한을 부여하는 세밀한 스코프를 선택할 수 있습니다. 

앱에서 OAuth 2.0을 사용하려면, 개발자 포털의 앱 설정 섹션에 있는 앱 인증 설정에서 이를 활성화해야 합니다.

[앱 전용](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)은 요청에 [앱 전용 액세스 토큰](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)만 전달하면 됩니다. 개발자 앱 내에서 직접 앱 전용 액세스 토큰을 생성하거나, [POST oauth2/token](/ko/resources/fundamentals/authentication#post-oauth2-token) 엔드포인트로 생성할 수 있습니다.

<div id="developer-portal-projects-and-developer-apps">
  #### 개발자 포털, 프로젝트와 개발자 앱
</div>

X API v2 엔드포인트에서 사용할 인증 자격 증명을 받으려면 [개발자 계정을 신청](https://developer.x.com/en/portal/petition/essential/basic-info)하고, 해당 계정에서 [프로젝트](/ko/resources/fundamentals/projects)를 설정한 다음, 그 프로젝트 내에 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성해야 합니다. 이후 개발자 앱에서 키와 토큰을 확인할 수 있습니다.
 

<div id="rate-limits">
  #### 할당량 제한
</div>

매일 수많은 개발자가 X API에 요청을 보냅니다. 이러한 대규모 트래픽을 관리하기 위해 [할당량 제한](https://developer.x.com/x-api/fundamentals/rate-limits)이 각 엔드포인트에 적용되어, 앱 또는 인증된 사용자를 대신해 보낼 수 있는 요청 수를 제한합니다.

이 엔드포인트에는 앱 수준과 사용자 수준의 할당량 제한이 모두 적용됩니다. 앱 할당량 제한은 개발자인 여러분이 특정 기간 동안 특정 앱(API Key와 API Secret Key 또는 Bearer Token 사용을 가정)에서 이 엔드포인트로 보낼 수 있는 요청 수를 제한한다는 의미입니다. 사용자 할당량 제한은 여러분이 대신 요청을 수행하는 인증된 사용자가 모든 개발자 앱 전반에서 목록 게시물 조회를 수행할 수 있는 횟수가 제한된다는 의미입니다.

아래 표는 각 엔드포인트의 할당량 제한을 보여줍니다.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **HTTP method** | **Rate limit** |
| /2/lists/:id/tweets | GET | 15분당 900회 요청 |

필드와 확장

X API v2 GET 엔드포인트는 `fields`와 `expansions`라는 도구 세트를 사용해 API에서 반환받을 데이터를 정확히 선택할 수 있도록 합니다. `expansions` 파라미터를 사용하면 페이로드에서 참조되는 객체를 확장할 수 있습니다. 예를 들어, 목록 게시물을 조회하면 다음 [expansions](/ko/x-api/fundamentals/expansions)을 가져올 수 있습니다:

- `author_id`

`fields` 파라미터를 사용하면 서로 다른 데이터 객체 내에서 수신하려는 [fields](/ko/x-api/fundamentals/fields)를 정확히 선택할 수 있습니다. 이 엔드포인트는 주로 Post 객체를 반환합니다. 기본적으로 Post 객체는 `id`와 `text` 필드를 반환합니다. `tweet.created_at` 또는 `tweet.lang` 같은 추가 필드를 받으려면 fields 파라미터로 별도로 요청해야 합니다.

[X API v2 데이터 사전](/ko/x-api/fundamentals/data-dictionary)에 [fields와 expansions](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)를 함께 사용하는 방법에 대한 가이드를 추가했습니다.

아래 표는 조회 엔드포인트에서 사용 가능한 필드와 확장을 보여줍니다:

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Fields** | **Expansions** |
| /2/lists/:id/tweets | `tweet.fields`<br /><br />`user.fields` | `author_id` |

페이지네이션

목록 게시물 조회는 많은 데이터를 반환할 수 있습니다. 언제나 일관되고 고성능의 결과를 제공하기 위해 페이지네이션을 사용합니다. 페이지네이션은 단일 응답에 담을 수 있는 양을 초과하는 결과를 반환하는 X API v2 엔드포인트에서 제공되는 기능입니다. 이런 경우 데이터는 일련의 ‘페이지’로 나뉘어 반환됩니다. [결과를 페이지네이션하는 방법](/ko/x-api/fundamentals/pagination)에 대해 자세히 알아보세요.
