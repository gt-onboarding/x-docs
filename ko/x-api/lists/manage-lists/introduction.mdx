[X 리스트](https://help.x.com/en/using-twitter/twitter-lists)는 사용자가 타임라인에서 보는 게시물을 맞춤화하고, 정리하고, 우선순위를 지정할 수 있도록 합니다. Lists 엔드포인트를 사용하면 사용자 선호도, 관심사, 그룹 또는 주제에 따라 게시물을 큐레이션하고 정리할 수 있는 솔루션을 구축할 수 있습니다.

manage List 엔드포인트를 통해 사용자를 대신해 요청을 수행하므로, 이 엔드포인트는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)로 인증해야 하며, 앱을 승인한 사용자와 연계된 사용자 액세스 토큰을 사용해야 합니다. 이 토큰은 [3-legged OAuth flow](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) 또는 [Authorization Code with PKCE grant flow](/ko/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0)을 사용하여 생성할 수 있습니다.

<div id="manage-lists">
  ### 목록 관리
</div>

목록 관리 엔드포인트를 사용하면 인증된 사용자를 대신해 목록을 생성, 삭제, 업데이트할 수 있습니다. 이 엔드포인트에는 POST, DELETE, PUT의 세 가지 메서드가 제공됩니다. POST는 목록을 생성하고, DELETE는 목록을 삭제하며, PUT은 목록의 메타데이터를 업데이트합니다. 세 엔드포인트 모두에 대해 사용자 기준 15분당 300회의 요청 한도가 적용됩니다.

계정당 최대 1,000개의 목록을 생성할 수 있습니다.

**계정 설정**

이 엔드포인트에 액세스하려면 다음이 필요합니다:

- 승인된 [developer account](https://developer.x.com/en/portal/petition/essential/basic-info)
- [Project](/ko/resources/fundamentals/projects) 내 [developer App](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용한 인증

X API v2 엔드포인트에 대한 액세스 권한에 대해 자세히 알아보려면 [시작하기 가이드](/ko/x-api/getting-started/getting-access)를 참조하세요.
