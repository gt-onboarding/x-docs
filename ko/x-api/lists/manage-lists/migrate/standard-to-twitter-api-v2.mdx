<div id="manage-lists-standard-v11-compared-to-x-api-v2">
  ### 목록 관리: Standard v1.1과 X API v2 비교
</div>

표준 v1.1의 [POST lists/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-create), [POST lists/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-destroy), [POST lists/update](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-update) 엔드포인트를 사용해 오셨다면, 이 가이드는 표준 v1.1과 X API v2의 목록 관리 엔드포인트 간 유사점과 차이점을 이해하는 데 도움이 되도록 작성되었습니다.

- **유사점**
  - 인증
- **차이점**
  - 엔드포인트 URL
  - 앱 및 프로젝트 요건
  - HTTP 메서드
  - 할당량 제한
  - 요청 파라미터

<div id="similarities">
  #### 유사점
</div>

**인증**

두 엔드포인트 버전 모두 [OAuth 1.0a 사용자 컨텍스트](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication)를 지원합니다. 따라서 이전에 표준 v1.1의 목록 관리 엔드포인트를 사용했다면, X API v2로 마이그레이션하더라도 동일한 인증 방식을 계속 사용할 수 있습니다.

<div id="differences">
  #### 차이점
</div>

**엔드포인트 URL**

- Standard v1.1 엔드포인트:
  - POST https://api.x.com/1.1/lists/create.json
    (리스트 생성)
  - POST https://api.x.com/1.1/lists/destroy.json
    (리스트 삭제)
  - POST https://api.x.com/1.1/lists/update.json
    (리스트 업데이트)
- X API v2 엔드포인트:
  - POST https://api.x.com/2/lists
    (리스트 생성)

  - DELETE https://api.x.com/2/lists/:id
    (리스트 삭제)

  - PUT https://api.x.com/2/lists/:id
    (리스트 업데이트)

**요청 한도**

| **Standard v1.1** | **X API v2** |
| :--- | :--- |
| /1.1/lists/create.json<br /><br />없음 | /2/lists<br /><br />OAuth 1.0a 사용자 컨텍스트 기준 15분당 300회 요청 |
| /1.1/lists/destroy.json<br /><br />없음 | /2/lists/:id<br /><br />OAuth 1.0a 사용자 컨텍스트 기준 15분당 300회 요청 |
| /1.1/lists/update.json<br /><br />없음 | /2/lists/:id<br /><br />OAuth 1.0a 사용자 컨텍스트 기준 15분당 300회 요청 |

**앱 및 프로젝트 요구 사항**

X API v2 엔드포인트는 요청을 인증할 때 [developer App](/ko/resources/fundamentals/developer-apps)의 자격 증명이 [Project](/ko/resources/fundamentals/projects)와 연결되어 있어야 합니다. 모든 X API v1.1 엔드포인트는 독립 실행형 앱 또는 프로젝트에 연결된 앱의 자격 증명을 사용할 수 있습니다.

**요청 매개변수**

다음 Standard v1.1 요청 매개변수는 X API v2에 해당 항목이 있습니다:

**리스트 생성**

| **Standard** | **X API v2** |
| :--- | :--- |
| name | name |
| mode | private |
| description | description |

**리스트 삭제/업데이트**

| **Standard** | **X API v2** |
| :--- | :--- |
| owner\_screen\_name | 해당 없음 |
| owner\_id | 해당 없음 |
| list\_id | id  |
| slug | 해당 없음 |

<Note>
  **주의:** Standard v1.1 매개변수는 쿼리 매개변수로 전달되는 반면, X API v2 매개변수는 본문 매개변수(POST 엔드포인트의 경우) 또는 경로 매개변수(DELETE 및 PUT 엔드포인트의 경우)로 전달됩니다.
</Note>
