<div id="comparing-x-apis-lists-endpoints">
  ## X API의 목록 엔드포인트 비교
</div>

v2 목록 관리 엔드포인트는 궁극적으로 [POST lists/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-create), [POST lists/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-destroy), [POST lists/update](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-update)를 대체할 예정입니다. 이전 버전 엔드포인트를 사용하는 코드, 앱, 또는 도구가 있고 최신 X API v2로 마이그레이션을 고려 중이라면 이 가이드가 도움이 됩니다.

다음 표는 표준 v1.1과 X API v2의 목록 엔드포인트를 비교합니다:

**목록 생성**

| 설명 | 표준 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 지원 HTTP 메서드 | POST | POST |
| 호스트 도메인 | https://api.x.com | https://api.x.com |
| 엔드포인트 경로 | /1.1/lists/create.json | /2/lists |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 1.0a 사용자 컨텍스트 | OAuth 1.0a 사용자 컨텍스트 |
| 기본 요청 [레이트 리밋](/ko/resources/fundamentals/rate-limits) | 없음 | 15분당 300회(사용자 기준) |

**목록 삭제**

| 설명 | 표준 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 지원 HTTP 메서드 | POST | DELETE |
| 호스트 도메인 | https://api.x.com | https://api.x.com |
| 엔드포인트 경로 | /1.1/lists/destroy.json | /2/lists/:id |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 1.0a 사용자 컨텍스트 | OAuth 1.0a 사용자 컨텍스트 |
| 기본 요청 [레이트 리밋](/ko/resources/fundamentals/rate-limits) | 없음 | 15분당 300회(사용자 기준) |

**목록 업데이트**

| 설명 | 표준 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 지원 HTTP 메서드 | POST | PUT |
| 호스트 도메인 | https://api.x.com | https://api.x.com |
| 엔드포인트 경로 | /1.1/lists/update.json | /2/lists/:id |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 1.0a 사용자 컨텍스트 | OAuth 1.0a 사용자 컨텍스트 |
| 기본 요청 [레이트 리밋](/ko/resources/fundamentals/rate-limits) | 없음 | 15분당 300회(사용자 기준) |

X API v2 엔드포인트에 액세스하려면 [개발자 계정에 가입](https://developer.x.com/en/portal/petition/essential/basic-info)해야 합니다. 인증 시에는 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용해야 하며, 해당 앱은 [프로젝트](/ko/resources/fundamentals/projects) 내에 있어야 합니다.

X API v2 엔드포인트 액세스 방법은 [시작하기](/ko/x-api/getting-started/getting-access) 페이지에서 자세히 알아보세요.
