리스트 고정 기능을 사용하면 X 사용자가 자신이 구독했거나 직접 만든 리스트에 더 쉽게 접근할 수 있습니다.

이 엔드포인트에서 사용자를 대신해 요청을 수행하므로, 반드시 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)로 해당 사용자를 인증하고, 앱을 승인한 사용자의 액세스 토큰을 사용해야 합니다. 이 토큰은 [3-legged OAuth flow](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) 또는 [Authorization Code with PKCE grant flow](/ko/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0)를 통해 생성할 수 있습니다.

<div id="pinned-list-lookup">
  ### 고정 목록 조회
</div>

고정 목록 조회에는 인증된 사용자의 고정 목록을 가져올 수 있는 단일 엔드포인트가 있습니다. 이 엔드포인트의 요청 한도는 15분당 15회입니다.
 

<div id="manage-pinned-lists">
  ### 고정 목록 관리
</div>

고정 목록 관리 엔드포인트를 사용하면 인증된 사용자를 대신해 목록을 고정하거나 고정 해제할 수 있습니다. 이 엔드포인트에는 POST와 DELETE 두 가지 메서드가 있습니다. POST 메서드는 목록을 고정하고, DELETE 메서드는 고정을 해제합니다. 두 엔드포인트 모두 사용자 기준 15분당 50회의 요청 한도가 적용됩니다.

<Note>
  **계정 설정**

  이 엔드포인트에 액세스하려면 다음이 필요합니다:

  - 승인된 [developer account](https://developer.x.com/en/portal/petition/essential/basic-info).
  - [Project](/ko/resources/fundamentals/projects) 내 [developer App](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용한 인증

  [X API v2 엔드포인트 액세스 받기](/ko/x-api/getting-started/getting-access) 가이드에서 자세히 알아보세요.
</Note>
