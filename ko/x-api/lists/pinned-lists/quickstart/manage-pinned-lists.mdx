<div id="steps-to-build-a-manage-pinned-list-request">
  ### 고정된 List 관리 요청을 구성하는 단계
</div>

**1단계: Postman에서 List 엔드포인트 컬렉션 선택**

Postman에 X API v2 컬렉션을 불러온 후 “List” 폴더로 이동하여 “Pinned Lists” 폴더를 선택한 다음 "Pin a List"를 선택합니다.
 

**2단계: 고정할 List 지정**

고정된 List 관리 엔드포인트에는 두 개의 ID가 필요합니다. 하나는 사용자(해당 List를 고정할 인증된 사용자)이고, 다른 하나는 대상 List(고정할 List)입니다. 사용자 ID는 인증 사용자 ID와 일치해야 하므로, 요청을 인증할 때 해당 사용자 ID와 연결된 Access Token을 전달해야 합니다. 이 경우 본인의 사용자에 속한 ID를 지정할 수 있습니다. ID를 찾는 방법은 두 가지입니다:

1. [users lookup](/ko/x-api/users/lookup/introduction) by username 엔드포인트를 사용하여 사용자 이름을 전달하면 id 필드를 받을 수 있습니다.
2. Access Token의 숫자 부분이 사용자 ID입니다.

대상 List ID는 유효한 어떤 List도 가능합니다. Postman에서 "Params" 탭으로 이동해 id 경로 변수의 "Value" 열에 사용자 ID를 입력합니다. “Body” 탭으로 이동해 고정하려는 List의 ID를 list\_id 매개변수의 값으로 입력합니다. 어떤 ID의 앞뒤에도 공백이 없도록 하세요.

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Parameter type** |
| `id` | 사용자 ID | path |
| list\_id | 고정할 대상 List ID | body |

이제 "Send" 버튼 옆에 다음과 유사한 URL이 표시되어야 합니다:

```
      https://api.x.com/2/users/2244994945/pinned_lists
```

3단계: 요청 보내기 및 응답 검토

모든 설정이 완료되면 "Send" 버튼을 클릭하세요. 그러면 다음 예시와 유사한 응답을 받게 됩니다:

```
{
  "data": {
    "pinned": true
  }
}
```

반환된 응답 객체에서 pinned 키의 값이 true이면 대상 List가 성공적으로 고정된 것입니다. 

List의 고정을 해제하려면, Postman에 로드된 X API v2 컬렉션의 “Lists” 폴더에 있는 “Unpin List” 요청을 선택하세요. 이 엔드포인트에는 인증된 사용자 ID와 고정을 해제할 List의 ID가 필요합니다. “Params” 탭에서 id 열의 값으로 사용자 ID를 입력하고, 고정을 해제할 List의 ID를 list\_id 열의 값으로 입력합니다. 

삭제 요청이 성공하면 다음 예시와 유사한 응답을 받게 됩니다:

```
{
  "data": {
    "pinned": false
  }
}
```
