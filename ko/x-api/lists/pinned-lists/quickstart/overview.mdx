<div id="getting-started-with-the-pinned-list-endpoint-group">
  ## 고정 목록 엔드포인트 그룹 시작하기
</div>

이 간단한 개요는 [Postman](/ko/tutorials/postman-getting-started)을 사용해 고정 목록 엔드포인트로 첫 요청을 보내는 데 도움을 드립니다.

여러 언어의 샘플 코드를 보고 싶다면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 저장소를 방문하세요.

<Note>
  ### 사전 준비 사항

  이 가이드를 완료하려면 요청 인증을 위한 [keys and tokens](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따라 키와 토큰을 생성할 수 있습니다:

  - [개발자 계정에 가입](https://developer.x.com/en/apply-for-access)하고 승인을 받습니다.
  - 개발자 포털에서 [Project](/ko/resources/fundamentals/projects)와 연결된 [developer App](/ko/resources/fundamentals/developer-apps)을 생성합니다.
  - App의 “Keys and tokens” 페이지로 이동하여 필요한 자격 증명을 생성합니다. 모든 자격 증명은 안전한 위치에 보관하세요.
</Note>

<div id="authenticate-your-request">
  #### 요청 인증하기
</div>

**lookup** 엔드포인트에 성공적으로 요청하려면 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication), [OAuth 2.0 App-Only](/ko/resources/fundamentals/authentication#oauth-2-0), 또는 [OAuth 2.0 Authorization Code with PKCE](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용할 수 있습니다. 반면 **manage** 엔드포인트는 OAuth 1.0a User Context 또는 OAuth 2.0 Authorization Code with PKCE로만 인증할 수 있습니다.

Postman을 사용할 때 환경을 올바르게 설정했다면 기본 인증 키와 토큰이 요청에 자동으로 채워집니다.

Postman의 오른쪽 상단에서 "X API v2" 환경을 선택한 뒤, 환경 드롭다운 옆의 눈 아이콘을 클릭하여 "initial value"와 "current value" 필드에 키와 토큰을 추가하면 됩니다. 포함해야 할 키는 다음과 같습니다.

- API Key(Consumer Key라고도 함)
- API Secret Key(Consumer Secret이라고도 함)
- OAuth 1.0a user Access Token
- OAuth 1.0a user Access Token Secret
- OAuth 2.0 App Access Token
- OAuth 2.0 Client Key(앱 설정에서 OAuth 2.0 User Authentication 설정을 구성한 경우에만 제공)
- OAuth 2.0 Client Secret(앱 설정에서 OAuth 2.0 User Authentication 설정을 구성한 경우에만 제공)
