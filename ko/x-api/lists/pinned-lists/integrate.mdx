<div id="authentication">
  #### 인증
</div>

모든 X API v2 엔드포인트는 키와 토큰으로도 알려진 자격 증명 세트를 사용해 요청을 인증해야 합니다. 이 엔드포인트에는 OAuth 1.0a 사용자 컨텍스트를 사용할 수 있습니다. 

성공적으로 요청하려면 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication)를 사용해야 하며, 이는 API 키와 사용자 액세스 토큰 세트를 사용해야 함을 의미합니다. 액세스 토큰은 요청을 대행하는 해당 사용자와 연결되어 있어야 합니다. 다른 사용자를 위한 액세스 토큰 세트를 생성하려면, 그 사용자가 [3-legged OAuth flow](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)를 통해 귀하의 앱을 승인해야 합니다.

OAuth 1.0a는 사용하기 까다로울 수 있습니다. 이 인증 방식에 익숙하지 않다면 [라이브러리](/ko/x-api/tools-and-libraries/overview)를 사용하거나 Postman 같은 도구를 사용하는 것을 권장합니다.

<div id="developer-portal-projects-and-developer-apps">
  #### 개발자 포털, 프로젝트와 개발자 앱
</div>

X API v2 엔드포인트에서 사용할 인증 자격 증명을 받으려면 [개발자 계정을 등록](https://developer.x.com/en/portal/petition/essential/basic-info)하고 해당 계정 내에 [프로젝트](/ko/resources/fundamentals/projects)를 설정한 뒤, 그 프로젝트 내에 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성해야 합니다. 그런 다음 개발자 앱에서 키와 토큰을 확인할 수 있습니다.
 

<div id="rate-limits">
  #### 요청 제한
</div>

매일 수천 명의 개발자가 X API에 요청을 보냅니다. 이러한 대규모 트래픽을 관리하기 위해 [요청 제한](/ko/x-api/fundamentals/rate-limits)이 각 엔드포인트에 적용되어, 앱 또는 인증된 사용자를 대신해 보낼 수 있는 요청 수를 제한합니다. 

이들 엔드포인트는 사용자 수준에서 제한되므로, 여러분이 대신 요청을 수행하는 인증된 사용자는 어떤 개발자 앱이든 관계없이 해당 엔드포인트를 일정 횟수만 호출할 수 있습니다. 

아래 표는 각 엔드포인트의 요청 제한을 보여줍니다.

|     |     |     |
| :--- | :--- | :--- |
| **엔드포인트** | **HTTP 메서드** | **요청 제한** |
| /2/users/:id/pinned\_lists | POST | 15분당 50회 |
| /2/users/:id/pinned\_lists/:list\_id | DELETE | 15분당 50회 |
| /2/users/:id/pinned\_lists | GET | 15분당 15회 |

<div id="fields-and-expansions">
  #### 필드와 확장
</div>

X API v2 GET 엔드포인트는 `fields` 및 `expansions`라는 도구를 사용하여 API에서 반환받을 데이터를 정확하게 선택할 수 있도록 합니다. `expansions` 매개변수는 페이로드에서 참조된 객체를 확장합니다. 예를 들어, 고정된 리스트를 조회하면 다음 [확장](/ko/x-api/fundamentals/expansions)을 가져올 수 있습니다:

- `owner_id`

`fields` 매개변수는 서로 다른 데이터 객체에서 받고자 하는 [필드](/ko/x-api/fundamentals/fields)를 정확히 선택할 수 있도록 합니다. 이 엔드포인트는 주로 사용자 객체를 반환합니다. 기본적으로 리스트 객체는 `id`와 `name` 필드를 반환합니다. `list.created_at` 또는 `list.description`과 같은 추가 필드를 받으려면 `fields` 매개변수를 사용해 해당 필드를 명시적으로 요청해야 합니다.

[X API v2 데이터 사전](/ko/x-api/fundamentals/data-dictionary)에 [필드와 확장](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)을 함께 사용하는 가이드를 추가했습니다.

아래 표는 조회 엔드포인트에서 사용 가능한 필드와 확장을 보여줍니다:

|     |     |     |
| :--- | :--- | :--- |
| **엔드포인트** | **필드** | **확장** |
| /2/users/:id/pinned\_lists | `list.fields`<br /><br />`user.fields` | `owner_id` |
