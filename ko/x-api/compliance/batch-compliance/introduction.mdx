X는 X API로 구축하는 개발자 커뮤니티에 대한 약속을 지키기 위해 노력하고 있습니다. 이 약속의 일환으로, 우리는 API를 개발자에게 개방적이고 공정하게 제공하며, X 이용자에게 안전하고 X 플랫폼 전체에도 유익하도록 만드는 것을 목표로 합니다. X 콘텐츠를 오프라인으로 저장하는 모든 개발자는 데이터가 사용자 의도와 X의 현재 콘텐츠 상태를 반영하도록 하는 것이 중요합니다. 예를 들어, X에서 누군가가 게시물을 삭제하거나 계정을 삭제·비활성화하고, 게시물을 보호(비공개 전환)하거나, 게시물에서 위치 정보(geo)를 제거하는 경우, 그 사람의 기대와 의도를 X와 우리 개발자 모두가 존중하는 것이 매우 중요합니다. 배치 컴플라이언스 엔드포인트는 개발자가 X 데이터를 [X Developer Agreement and Policy](https://developer.x.com/en/developer-terms/policy)에 부합하도록 유지하는 데 도움이 되는 간편한 도구를 제공합니다.

이 배치 컴플라이언스 엔드포인트를 사용하면 대규모 게시물 또는 사용자 ID 데이터셋을 업로드하여 컴플라이언스 상태를 조회하고, 데이터셋을 컴플라이언스에 맞추기 위해 어떤 데이터에 조치가 필요한지 판단할 수 있습니다. 배치 컴플라이언스 엔드포인트의 사용은 앞서 언급한 사용 사례로 제한되며, 그 외의 목적은 금지되며 제재 조치로 이어질 수 있다는 점에 유의하십시오.

일반적으로 이 엔드포인트를 사용할 때는 다음 4단계를 거칩니다:

1. **컴플라이언스 작업 생성**
   업로드하려는 데이터셋에 게시물 ID 또는 사용자 ID가 있는지를 나타내기 위해 작업 유형을 지정할 수 있습니다(값은 tweets 또는 users). 각 작업 유형당 동시에 하나의 작업만 실행할 수 있습니다.
2. **upload\_url에 데이터셋 업로드**
   다음으로, 제공된 upload\_url에 데이터셋을 일반 텍스트 파일로 업로드합니다. 파일의 각 줄에는 하나의 게시물 ID 또는 사용자 ID가 포함되어야 합니다. upload\_url은 15분 후 만료됩니다.
3. **(선택 사항) 작업 상태 확인**
   컴플라이언스 작업의 상태가 created, in\_progress, failed, complete 중 무엇인지 확인할 수 있습니다.
4. **결과 다운로드**
   작업이 완료되면 download\_url을 사용해 결과를 다운로드할 수 있습니다. download\_url은(작업 생성 시점부터) 1주일 후 만료됩니다.

   결과에는 JSON 객체 집합(한 줄당 하나의 객체)이 포함됩니다. 각 객체에는 게시물 ID, 게시물 생성 날짜(날짜별로 게시물을 찾는 데 유용), 필요한 조치, 컴플라이언스 조치 사유, 사용자 정지 날짜가 포함됩니다.

결과에서 다음 컴플라이언스 이벤트 유형을 받게 됩니다:

- deleted - 게시물 또는 사용자 계정이 삭제되었음을 나타냄
- deactivated - 게시물 또는 사용자 계정이 비활성화되었음을 나타냄
- scrub\_geo - 게시물 또는 사용자와 연관된 위치 정보가 제거되었음을 나타냄
- protected - 게시물을 작성한 계정이 비공개로 전환되었음을 나타냄
- suspended - 게시물을 작성한 계정이 정지되었음을 나타냄

**계정 설정**

이 엔드포인트에 액세스하려면 다음이 필요합니다:

- 승인된 [developer account](https://developer.x.com/en/portal/petition/essential/basic-info).
- [Project](/ko/resources/fundamentals/projects) 내에 있는 [developer App](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용한 인증.
