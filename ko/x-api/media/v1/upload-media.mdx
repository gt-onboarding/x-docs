<div id="overview">
  ## 개요
</div>

미디어 객체는 단일 사진, 동영상 또는 애니메이티드 GIF를 나타냅니다. 미디어 객체는 X API의 많은 엔드포인트에서 사용되며, 게시물, 다이렉트 메시지, 사용자 프로필, 광고 크리에이티브 등 다양한 곳에 포함될 수 있습니다. 각 미디어 객체는 해상도나 형식이 다른 여러 표시 또는 재생 버전을 가질 수 있습니다.

<div id="media-types-size-restrictions">
  ## 미디어 유형 및 크기 제한
</div>

API 업로드 시 크기 제한

- 이미지 5 MB
- GIF 15 MB
- 동영상 512 MB (media_category=amplify 사용 시)

<div id="creation">
  ## 생성
</div>

트윗, 다이렉트 메시지, 사용자 프로필 사진, 호스팅된 광고 카드 등과 같은 객체에는 하나 이상의 미디어 객체가 포함될 수 있습니다. 이러한 최상위 객체를 통틀어 엔터티라고 합니다. 관련 엔터티 생성 API(예: POST statuses/update)는 고유한 media_id를 통해 하나 이상의 미디어 객체를 전달받을 수 있습니다.

미디어 객체를 포함하는 엔터티는 다음 단계로 생성할 수 있습니다.

1. 권장 방식인 청크 업로드(이미지/GIF/동영상) 또는 이전의 단순 업로드(이미지만 가능)를 사용해 미디어 파일을 업로드합니다.
2. 1단계에서 media_id를 받습니다. 엔터티가 여러 media_id 매개변수 전달을 허용하는 경우, 이 단계는 서로 다른 미디어에 대해 여러 번 반복할 수 있습니다.
3. media_id 및 기타 필요한 매개변수를 포함하여 적절한 엔드포인트를 호출해 엔터티를 생성합니다. 예를 들어 POST statuses/update 엔드포인트를 사용해 트윗에 media_id를 첨부합니다.
