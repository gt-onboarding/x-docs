**시작하기**

<div id="authentication">
  ### 인증
</div>

이 엔드포인트는 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication), \[OAuth 2.0 앱 전용]\(https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), 또는 [OAuth 2.0 권한 부여 코드(PKCE 포함)](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)로 인증할 수 있습니다.
 

<div id="making-a-request">
  ### 요청 보내기
</div>

아래와 같이 ID를 제공해 Communities Lookup 엔드포인트를 호출할 수 있습니다(예시 ID `1758747817642700922`):

```
      curl --location 'https://api.x.com/2/communities/1758747817642700922' --header 'Authorization: ••••••'
```

요청이 성공하면 아래와 같은 JSON 응답이 반환됩니다:

```
{
    "data": {
        "access": "Public",
        "description": "Welcome to the Anime Community! Where anime fans gather to share their favorite shows and discuss everything anime-related.",
        "join_policy": "Open",
        "name": "Anime Community"
    }
}
```

<div id="community-fields">
  ### 커뮤니티 필드
</div>

커뮤니티 필드를 지정하여 커뮤니티와 관련된 추가 데이터를 가져올 수 있습니다.

사용 가능한 커뮤니티 필드는 다음과 같습니다:

| 필드 값 | 타입 | 설명 |     |
| :--- | :--- | :--- | :--- |
| created\_at | date (ISO 8601) | 커뮤니티 생성 시각. |     |
| id  | string | 커뮤니티의 고유 식별자. |     |
| name | string | 커뮤니티 이름. |     |
| description | string | 제공된 경우 커뮤니티 설명 텍스트. |     |
| access | string | 커뮤니티의 접근 수준.  <br />  <br />다음 중 하나일 수 있습니다:<br /><br />\* `Public`<br />\* `Closed` |     |
| join\_policy | string | 커뮤니티의 가입 정책.  <br />  <br />다음 중 하나일 수 있습니다:<br /><br />\* `Open`<br />\* `RestrictedJoinRequestsDisabled`<br />\* `RestrictedJoinRequestsRequireAdminApproval`<br />\* `RestrictedJoinRequestsRequireModeratorApproval`<br />\* `SuperFollowRequired` |     |
| member\_count | integer | 커뮤니티 가입자 수. |     |

<div id="example-request">
  #### 예시 요청
</div>

다음과 같이 추가 커뮤니티 필드를 포함해 Communities Lookup 엔드포인트를 호출할 수 있습니다(ID `1758747817642700922` 사용):

```
curl --location 'https://api.x.com/2/communities/1758747817642700922?community.fields=access,created_at,description,id,join_policy,member_count,name' --header 'Authorization: ••••••'
```
