<div id="what-is-the-x-api-v2">
  ## X API v2란 무엇인가요?
</div>

X API v2는 이제 기본 X API이며, 제품 투자와 혁신의 핵심입니다. 다양한 개발자 커뮤니티를 더 잘 지원하기 위해 개발자들과 협력하여 차세대 X API를 구축했습니다. 개발자 피드백을 바탕으로 더 폭넓은 요구를 충족하도록 API를 재구성하고, 새로운 기능과 엔드포인트를 도입했으며, 개발자 경험을 개선했습니다.

X API v2는 이제 기본 X API이며, 제품 투자와 혁신의 핵심입니다. 지난 몇 년간 우리는 개발자들과 협력하여 더 폭넓은 요구를 충족하도록 API를 재구성하고, 새로운 기능과 엔드포인트를 도입했으며, 개발자 경험을 개선했습니다. 우리는 개방형 개발자 플랫폼을 지속적으로 구축하는 데 전념하고 있으며, 여러분이 X API v2로 무엇을 만들어낼지 기대합니다.

<div id="why-migrate">
  ## 왜 마이그레이션해야 하나요?
</div>

X API v2는 현대적이고 더 지속 가능한 기반 위에 구축되었으며, 표준 v1.1 및 엔터프라이즈 제품을 대체하는 개선된 엔드포인트뿐 아니라 완전히 새로운 기능도 포함합니다. 레거시 API(v1.1 및 엔터프라이즈) 사용자는 v2로의 마이그레이션을 시작할 것을 강력히 권장합니다. 해당 API는 결국 사용 중단될 예정입니다. X API를 사용하여 공개 대화를 수집·분석하고, X에서 사용자와 소통하며, 혁신을 이루세요.

이 섹션에서는 엔드포인트와 기능을 설명합니다.

대부분의 X API v2 엔드포인트는 이전 버전의 대체이지만, 여러 새로운 엔드포인트도 도입했습니다. 아래는 v2에서 새로 제공되는 엔드포인트의 예시입니다:

- [Spaces 엔드포인트](/ko/x-api/spaces/introduction): 사용자가 X Spaces를 더 효과적으로 활용하고, 개발자가 오디오 대화의 미래를 설계할 수 있도록 지원합니다.
- [답글 숨기기](/ko/x-api/posts/hide-replies/introduction): 악의적이거나 산만하게 하거나 오해를 불러일으킬 수 있는 답글의 영향을 대규모로 제한하는 도구를 구축할 수 있습니다.
- 새로운 Lists 엔드포인트: [리스트를 고정/해제](/ko/x-api/lists/pinned-lists/introduction)하거나 특정 사용자의 고정된 리스트를 조회할 수 있습니다.
- 새로운 [배치 컴플라이언스 엔드포인트](/ko/x-api/compliance/batch-compliance/introduction): 저장된 사용자 및 트윗 데이터가 컴플라이언스 요구사항을 준수하도록 보장할 수 있습니다.

<div id="discover-new-and-updated-response-objects">
  ### 새롭고 업데이트된 응답 객체 살펴보기
</div>

다음 여섯 가지 데이터 객체를 v2 엔드포인트에서 사용할 수 있습니다:

| Object | Description |
|:--------|:-------------|
| [Tweet](/ko/x-api/fundamentals/data-dictionary#tweet) | Tweet 객체에는 `id`, `text`, `created_at` 등 루트 수준 필드가 다수 포함되어 있습니다. 또한 `user`, `media`, `poll`, `place` 등의 하위 객체를 포함하는 상위 객체이기도 합니다. |
| [User](/ko/x-api/fundamentals/data-dictionary#user) | User 객체에는 참조된 사용자를 설명하는 X 사용자 계정 메타데이터가 포함됩니다. |
| [Spaces](/ko/x-api/fundamentals/data-dictionary#space) | Space 객체는 `state`, `host_id`, `is_ticketed`, `lang` 등의 필드로 구성됩니다. |
| [Lists](/ko/x-api/fundamentals/data-dictionary#list) | List 객체에는 `description`, `member_count`, `owner_id` 등 요청된 리스트의 기본 정보가 포함됩니다. |
| [Media](/ko/x-api/fundamentals/data-dictionary#media) | 트윗에 미디어(예: 이미지)가 포함된 경우 `media.fields` 매개변수로 Media 객체를 요청할 수 있으며, `media_key`, `type`, `url`, `preview_image_url` 등의 필드를 포함합니다. |
| [Poll](/ko/x-api/fundamentals/data-dictionary#poll) | 트윗에 포함된 투표는 어떤 엔드포인트에서도 기본 객체가 아니지만, Tweet 객체에서 확인하고 확장할 수 있습니다. |
| [Place](/ko/x-api/fundamentals/data-dictionary#place) | Place 객체는 `place_id`, `geo` 객체, `country_code` 등과 같은 필드로 구성됩니다. 이 정보는 트윗을 식별하고 위치별로 트윗을 분석하는 데 사용할 수 있습니다. |

[필드와 확장 사용 방법](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)을 자세히 알아보세요.

<div id="flexibility-to-choose-which-objects-and-fields-you-receive">
  ### 수신할 객체와 필드를 선택하는 유연성
</div>

GET 엔드포인트에 요청을 보내면 해당 엔드포인트와 관련된 기본 데이터 객체가 반환되며, 여기에 기본 필드 집합이 포함됩니다. 예를 들어, Tweet 객체는 기본적으로 `id`와 `text` 필드를 제공합니다.

요청에 추가 필드를 포함하려면 [fields](/ko/x-api/fundamentals/fields) 및 [expansions](/ko/x-api/fundamentals/expansions) 매개변수를 사용해야 합니다. expansions 매개변수는 사용자의 고정된 Tweet이나 미디어 객체와 같은 관련 데이터 객체를 가져오도록 하며, fields 연산자는 기본 필드 외에 반환된 객체 내의 특정 필드를 요청할 수 있도록 합니다.

다음은 다양한 X API v2 엔드포인트에서 요청할 수 있는 expansions의 전체 목록입니다:

| 객체 / 리소스 | 사용 가능한 expansions |
|:-------------------|:----------------------|
| Tweets            | `author_id`, `edit_history_tweet_ids`, `entities.mentions.username`, `in_reply_to_user_id`, `referenced_tweets.id`, `referenced_tweets.id.author_id`, `attachments.poll_ids`, `attachments.media_keys`, `geo.place_id` |
| Users             | `pinned_tweet_id` |
| Spaces            | `invited_user_ids`, `speaker_ids`, `creator_id`, `host_ids`, `topic_ids` |

[fields와 expansions를 사용하는 방법](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)을 자세히 알아보세요.

<div id="edit-tweets">
  ## 트윗 편집
</div>

X API v2 엔드포인트는 편집된 트윗의 메타데이터를 제공합니다. _트윗 편집_ 기능은 2022년 9월 1일 X 직원 대상 테스트로 처음 도입되었습니다. 그 날짜부터 편집 대상 트윗은 30분 동안 최대 5회까지 편집할 수 있습니다. [트윗 편집](/ko/x-api/fundamentals/edit-posts)에 대해 자세히 알아보세요.
X API v2를 사용하면 개발자는 다음을 확인할 수 있습니다:

- 트윗이 생성 시점에 편집 대상이었는지 여부. 설문조사가 포함된 트윗이나 예약된 트윗 등은 편집할 수 없습니다.
- 트윗은 30분 동안 편집할 수 있으며, 최대 5회까지 편집할 수 있습니다. 편집 가능한 트윗의 경우 남은 편집 가능 시간과 추가로 가능한 편집 횟수를 확인할 수 있습니다.
- 현재 보고 있는 트윗이 편집된 버전인지 여부(대부분의 경우 API는 특정 과거 버전을 트윗 ID로 요청하지 않는 한 트윗의 최신 버전을 반환합니다).
- 해당 트윗의 전체 편집 이력.
- 트윗의 각 버전에 귀속되는 참여 지표.

<div id="track-threaded-conversations">

<div id="ready-to-migrate">
  ## 마이그레이션 준비 완료
</div>

v2 엔드포인트를 사용하려면 다음이 필요합니다:

- [개발자 계정](https://developer.x.com/en/portal/petition/essential/basic-info)
- [프로젝트](ko/resources/fundamentals/projects) 내에서 생성된 [개발자 앱](https://developer.x.com/en/apps)
- 해당 프로젝트의 개발자 앱에서 발급된 [키와 토큰](ko/resources/fundamentals/authentication)

프로젝트 내 앱에서 발급된 키와 토큰을 사용하는 것이 중요합니다. 프로젝트 외부의 앱에서 발급된 키와 토큰을 사용하는 경우 v2 엔드포인트에 요청할 수 없습니다.

개발자 계정이 있으면 [개발자 포털](https://developer.x.com/en/portal/petition/essential/basic-info)에서 위의 모든 항목을 설정할 수 있습니다.

<div id="authentication">
  ### 인증
</div>

새로운 Twitter API에서는 서로 다른 엔드포인트에 접근하기 위해 OAuth 1.0a User Context와 OAuth 2.0 Bearer Token이라는 두 가지 인증 패턴을 사용합니다. 각 패턴은 엔드포인트에 요청할 때 서로 다른 용도로 사용됩니다:

OAuth 1.0a User Context는 Twitter 사용자를 대신해 요청할 때 필요합니다
OAuth 2.0 Bearer Token은 개발자 앱을 대신해 요청할 때 필요합니다

<div id="tools-and-code">
  ## 도구 및 코드
</div>

새로운 엔드포인트와 기능에 익숙해지고 빠르게 시작할 수 있도록 몇 가지 옵션을 제공합니다:

- Postman 클라이언트를 사용해 개별 엔드포인트에 요청하고 연결할 수 있는 X [Postman 컬렉션](https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400)을 제공합니다. 이는 인증을 검증하고 엔드포인트를 실험해 볼 수 있는 간편한 방법입니다.
- 또한 Ruby, Python, Node, Java 등 다양한 언어에 대해 X가 지원하는 라이브러리와 서드파티 라이브러리 목록을 제공합니다. 자세한 내용은 [도구 및 라이브러리 페이지](/ko/x-api/tools-and-libraries/overview)를 확인하세요.

<div id="migrating-to-the-new-data-format">
  ## 새로운 데이터 형식으로의 마이그레이션
</div>

v1.1 또는 엔터프라이즈에서 v2로 전환할 때, 데이터 제공 형식이 크게 변경되었음을 이해하는 것이 중요합니다. 새로운 필드가 추가되고, 필드 순서가 변경되었으며, 일부 요소는 완전히 제거되었습니다.

이러한 변경 사항을 이해하는 데 도움을 드리기 위해, v2 이전 데이터 형식의 필드를 최신 필드에 매핑하고 새 필드를 요청하는 방법을 설명하는 일련의 가이드를 준비 중입니다.

자세한 내용은 이 마이그레이션 허브의 [데이터 형식 마이그레이션](/ko/x-api/migrate/data-format-migration) 섹션 또는 다음의 개별 데이터 형식 가이드를 방문해 확인하세요:

- [네이티브 형식에서 X API v2로(standard v1.1)](/ko/x-api/migrate/data-format-migration#migrating-from-standard-v1-1s-data-format-to-v2)
- [Native Enriched에서 X API v2로(enterprise)](/ko/x-api/migrate/data-format-migration#migrating-from-native-enriched-data-format-to-v2)
- [Activity Streams에서 X API v2로(enterprise)](/ko/x-api/migrate/data-format-migration#migrating-from-activity-streams-data-format-to-v2)

<div id="whats-next">
  ## 다음 단계는 무엇인가요?
</div>

플랫폼을 오래 사용해 오신 분들은 많은 신규 엔드포인트가 기존 [standard v1.1](https://developer.x.com/en/docs/twitter-api/v1) 및 [enterprise](/ko/x-api/enterprise-gnip-2.0/enterprise-gnip) 엔드포인트와 일치하도록 설계되었음을 알아차리셨을 것입니다. 실제로, 우리는 향후 이들이 세 가지 버전을 모두 대체하도록 할 계획입니다.

[X API 엔드포인트 매핑](/ko/x-api/migrate/x-api-endpoint-map)이 이전 버전과 어떻게 대응되는지 이해하실 수 있도록 표를 준비했습니다.

앞으로 제공될 항목을 확인하시려면 [제품 로드맵](https://trello.com/b/myf7rKwV/twitter-developer-platform-roadmap)을 방문해 주세요.

이미 출시된 사항은 [변경 로그](https://developer.x.com/en/updates/changelog)에서 확인하실 수 있습니다.

<div id="what-should-we-build-next">
  ### 다음으로 무엇을 만들면 좋을까요?
</div>

X API v2의 기능을 확장해 나가면서 지속적으로 여러분의 의견을 듣고자 합니다. 여러분의 [피드백](https://twitterdevfeedback.uservoice.com/)을 환영하며 적극적으로 부탁드립니다.

이미 제출된 아이디어를 확인하고, 필요와 부합하는 항목에 지지를 표시한 뒤 피드백도 남겨 주세요.
