<div id="introduction">
  ## 소개
</div>

X API v2 출시와 함께 새로운 데이터 응답 형식과 객체·필드 요청 방식(통칭 X API v2 형식)을 도입했습니다.

‘일반 차이점’ 섹션에서는 표준 및 엔터프라이즈 사용자에게 관련된 변경 사항을 확인할 수 있습니다. 또한 [standard v1.1 Native format](https://developer.x.com/en/docs/x-api/v1/data-dictionary/overview), 엔터프라이즈용 [Native Enriched format](/ko/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#native-enriched-tweet-object), 엔터프라이즈용 [Activity Streams format](/ko/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#activity-object)에 대한 별도 가이드를 제공하여 필드 매핑을 돕고, 새로운 v2 필드를 요청할 때 사용해야 하는 필드와 확장을 설명합니다.

- [Native format에서 X API v2로 (standard v1.1)](/ko/x-api/migrate/data-format-migration#migrating-from-standard-v1-1s-data-format-to-v2)
- [Native Enriched에서 X API v2로 (enterprise)](/ko/x-api/migrate/data-format-migration#migrating-from-native-enriched-data-format-to-v2)
- [Activity Streams에서 X API v2로 (enterprise)](/ko/x-api/migrate/data-format-migration#migrating-from-activity-streams-data-format-to-v2)

또한 [X API v1.1 데이터 형식](https://developer.x.com/en/docs/x-api/v1/data-dictionary/overview)과 [X API v2 형식](/ko/x-api/introduction)의 차이를 빠르게 비교할 수 있는 [시각적 데이터 형식 마이그레이션 도구](/ko/x-api/migrate/data-format-migration#visual-data-format-migration-tool)도 참고하시기 바랍니다.

<div id="general-differences">
  ### 일반적인 차이
</div>

<div id="requesting-objects-and-fields">
  #### 객체와 필드 요청
</div>

pre-v2 엔드포인트와 v2의 가장 큰 변화 중 하나는, 표준·프리미엄·엔터프라이즈 엔드포인트가 대부분의 필드를 기본으로 제공하던 이전과 달리, v2는 기본적으로 소수의 필드만 반환한다는 점입니다. 새 버전은 [fields](/ko/x-api/fundamentals/fields)와 [expansions](/ko/x-api/fundamentals/expansions) 매개변수를 사용해 기본값을 넘어 추가 데이터를 명시적으로 요청할 수 있게 합니다. 이는 중요하지 않은 필드를 수집하지 않고 필요한 데이터만 요청할 수 있음을 의미합니다.

기본 데이터 객체와 관련된 요청 필드는 기본값과 함께 해당 기본 데이터 객체에 반환됩니다. 반면 expansions 매개변수로 확장 객체를 요청한 경우, 보조 객체는 새로운 includes 객체로 반환됩니다. 두 곳 모두에 포함되는 ID 필드를 사용해 includes의 확장 객체를 기본 객체와 매칭할 수 있습니다.

예를 들어, v2 [Post lookup](/ko/x-api/posts/lookup/introduction) 엔드포인트를 사용하고 요청에 expansions=author\_id 매개변수를 포함하면, 기본 Post 객체 내에서 author\_id 필드를 받으며, 또한 includes 객체에는 각 Post마다 하나의 user 객체가 반환됩니다. 각각에는 user 객체를 Post 객체에 다시 매칭하는 데 사용할 수 있는 기본 id 필드가 포함됩니다. 예시는 다음과 같습니다:

```
{
  "data": [
    {
      "author_id": "2244994945",
      "id": "1397568983931392004",
      "text": "The Twitter Developer Platform. Ooh la la! https://t.co/iGTdPXBfOv https://t.co/Ze8z8EODdg"
    }
  ],
  "includes": {
    "users": [
      {
        "id": "2244994945",
        "name": "Twitter Dev",
        "username": "TwitterDev"
      }
    ]
  }
}
```

<div id="updated-json-design">
  #### 업데이트된 JSON 디자인
</div>

특정 필드를 요청하는 방식의 변경 사항과 함께, X API v2는 API가 반환하는 객체(예: [Post](/ko/x-api/fundamentals/data-dictionary#tweet) 및 [user](/ko/x-api/fundamentals/data-dictionary#user) 객체)에 대해 새로운 JSON 디자인도 도입합니다.

- JSON 루트 수준에서, 기존 표준 엔드포인트는 Post 객체를 **statuses** 배열로 반환하는 반면 X API v2는 **data** 배열로 반환합니다.
- 리트윗 및 인용을 “statuses”로 지칭하던 대신, X API v2 JSON은 리트윗된 트윗과 인용 트윗으로 지칭합니다. **contributors** 및 **user.translator\_type** 같은 많은 레거시 및 사용 중단 필드는 제거됩니다.
- **favorites**(Post 객체)와 **favourites**(user 객체)를 모두 사용하는 대신, X API v2는 용어 like를 사용합니다.
- X는 값이 없는 JSON 값(예: null)을 페이로드에 포함하지 않는 규칙을 채택하고 있습니다. Post 및 user 속성은 null이 아닌 값이 있을 때에만 포함됩니다. 
   

<div id="new-v2-fields">
  #### 새로운 v2 필드
</div>

다음이 포함된 새로운 필드 세트를 [Post 객체](/ko/x-api/fundamentals/data-dictionary#tweet)에 도입했습니다:

- [conversation\_id](/ko/x-api/fundamentals/conversation-id) 필드
- context와 entities를 포함한 두 개의 새로운 [annotations](/ko/x-api/fundamentals/post-annotations) 필드
- 여러 개의 새로운 [metrics](/ko/x-api/fundamentals/metrics) 필드
- 특정 Post에 누가 답글을 달 수 있는지 표시하는 새로운 reply\_setting 필드

<div id="migrating-from-standard-v11s-data-format-to-v2">
  ### 표준 v1.1 데이터 형식에서 v2로 마이그레이션
</div>

아직 확인하지 않았다면, 먼저 [데이터 형식 마이그레이션](/ko/x-api/migrate/data-format-migration) 소개를 읽어보시길 권장합니다. 또한 [시각적 데이터 형식 마이그레이션 도구](/ko/x-api/migrate/data-format-migration#visual-data-format-migration-tool)를 사용해 [X API v1.1 데이터 형식](https://developer.x.com/en/docs/x-api/v1/data-dictionary/overview)과 [X API v2 형식](/ko/x-api/fundamentals/data-dictionary)의 차이를 빠르게 비교해 보실 수 있습니다.

네이티브 형식이라고도 불리는 표준 v1.1 데이터 형식은 [standard v1.1](https://developer.x.com/en/docs/twitter-api/v1) 엔드포인트와 함께 제공되는 기본 형식입니다.

프리미엄 제품을 사용 중이라면 [네이티브 강화 형식 가이드](/ko/x-api/migrate/data-format-migration#migrating-from-native-enriched-data-format-to-v2)를 참고하세요. 엔터프라이즈 클라이언트는 Gnip 콘솔에서의 설정에 따라 네이티브 강화 형식 또는 활동 스트림(activity streams)을 사용하고 있을 수 있습니다.

<div id="migrating-from-native-enriched-data-format-to-v2">
  ### Native Enriched 데이터 형식에서 v2로 마이그레이션
</div>

Native Enriched 데이터 형식은 당사의 [엔터프라이즈](/ko/x-api/enterprise-gnip-2.0/enterprise-gnip) 제품에서 사용됩니다.

Native Enriched 데이터 형식이 편집된 트윗 메타데이터를 제공하도록 업데이트되었습니다. Edit Tweet 메타데이터에 대해 자세히 알아보려면 [Edit Tweets 기본 사항](/ko/x-api/enterprise-gnip-2.0/fundamentals/edit-tweets) 페이지를 확인하세요.

표준 v1.1 엔드포인트를 사용하는 경우 [standard v1.1 to v2 가이드](/ko/x-api/migrate/data-format-migration#migrating-from-standard-v1-1s-data-format-to-v2)를 참고하세요. Activity Streams를 사용하는 엔터프라이즈 제품을 사용 중이라면 [Activity Streams to v2](/ko/x-api/migrate/data-format-migration#migrating-from-activity-streams-data-format-to-v2) 가이드도 참고하세요.

X API v2는 [Tweet](/ko/x-api/fundamentals/data-dictionary#tweet) 및 [user](/ko/x-api/fundamentals/data-dictionary#user) 객체에 대한 새로운 JSON 설계를 도입합니다.

- JSON 루트 수준에서 Native Enriched 형식은 results 배열에 트윗 객체를 반환하는 반면, X API v2는 data 배열을 반환합니다.
- Tweet 객체의 favorites와 user 객체의 favourites를 모두 사용하는 대신, X API v2는 like라는 용어를 사용합니다.
- X는 값이 없는 JSON 항목(예: null)은 페이로드에 작성하지 않는 규칙을 채택하고 있습니다. 트윗 및 사용자 속성은 null이 아닌 값을 가질 때만 포함됩니다.
- v2의 모든 id 필드는 문자열 형식입니다

새로운 JSON 형식 변경 사항과 더불어, 트윗 객체에 다음을 포함한 새로운 필드 세트를 도입했습니다:

- conversation\_id
- reply\_settings
- 미디어의 alt\_text
- context 및 entities를 포함한 두 가지 새로운 [annotations](/ko/x-api/fundamentals/post-annotations) 필드
- 여러 새로운 [metrics](/ko/x-api/fundamentals/metrics) 필드
- 여러 새로운 [polls](/ko/x-api/fundamentals/data-dictionary#poll) 필드

많은 레거시 및 사용 중단 필드가 제거됩니다:

- contributors
- 특정 entities.media 및 extended\_entities.media 필드
- filter\_level
- timestamp\_ms
- truncated

<div id="migrating-from-activity-streams-data-format-to-v2">
  ### Activity Streams 데이터 형식에서 v2로 마이그레이션
</div>

Activity Streams 데이터 형식은 당사의 [엔터프라이즈](/ko/x-api/enterprise-gnip-2.0/enterprise-gnip) 제품에서 제공됩니다.

Activity Streams 데이터 형식은 수정된 트윗 메타데이터를 제공하도록 업데이트되었습니다. 트윗 수정 메타데이터에 대해 자세히 알아보려면 [트윗 수정 기본 사항](/ko/x-api/enterprise-gnip-2.0/fundamentals/edit-tweets) 페이지를 참조하세요.

표준 v1.1 엔드포인트를 사용 중인 경우 [표준 v1.1에서 v2로 가이드](/ko/x-api/migrate/overview)를 참조하세요. 프리미엄 엔드포인트 또는 엔터프라이즈용 Native Enriched 형식을 사용 중인 경우 [Native Enriched에서 v2로 가이드](/ko/x-api/migrate/data-format-migration#migrating-from-native-enriched-data-format-to-v2)를 참조하세요.

X API v2는 [게시물](/ko/x-api/fundamentals/data-dictionary#tweet) 및 [사용자](/ko/x-api/fundamentals/data-dictionary#user) 오브젝트에 대한 새로운 JSON 설계를 도입합니다.

- JSON 루트 레벨에서 Activity Streams 형식은 결과 배열(results)에 트윗 오브젝트를 반환하는 반면, X API v2는 data 배열을 반환합니다.
- 리트윗 및 인용을 “활동(activities)”으로 지칭하는 대신, X API v2 JSON은 리트윗 및 인용 트윗으로 지칭합니다.
- Tweet 오브젝트의 favorites 및 user 오브젝트의 favourites를 사용하는 대신, X API v2는 like라는 용어를 사용합니다.
- Twitter는 값이 없는 JSON 항목(예: null)은 페이로드에 기록하지 않는 규칙을 채택했습니다. 트윗 및 사용자 속성은 null이 아닌 값이 있는 경우에만 포함됩니다.
- v2의 모든 id 필드는 문자열 형식입니다.
   

새로운 JSON 형식 변경에 더해, 트윗 오브젝트에 다음과 같은 새로운 필드 세트를 도입했습니다:

- conversation\_id
- reply\_settings
- 미디어의 alt\_text
- context와 entities를 포함하는 두 개의 새로운 [annotations](/ko/x-api/fundamentals/post-annotations) 필드
- 여러 새로운 [metrics](/ko/x-api/fundamentals/metrics) 필드
- 여러 새로운 [polls](/ko/x-api/fundamentals/data-dictionary#poll) 필드
   

다수의 레거시 및 사용 중단 필드는 제거되거나 대체됩니다:

- display\_text\_range
- generator
- gnip
- link
- objectType
- provider
- twitter\_entities.symbols는 data.entities.cashtags로 대체
- 특정 twitter\_extended\_entities.media 및 twitter\_entities.media 필드
- twitter\_filter\_level
- twitterTimeZone
- verb
