<div id="overview">
  ## 개요
</div>

필터드 스트림 엔드포인트 그룹은 개발자가 공개 게시물 스트림을 필터링할 수 있게 합니다. 이 엔드포인트 그룹에는 규칙을 생성·관리하고, 해당 규칙을 적용해 일치하는 공개 게시물만 반환하도록 스트림을 필터링할 수 있는 여러 엔드포인트가 포함됩니다. 이를 통해 사용자는 특정 주제와 이벤트를 수신하고, 경쟁 관련 대화를 모니터링하며, 트렌드가 어떻게 형성되는지 파악하는 등 다양한 작업을 수행할 수 있습니다.

개발자는 REST [rules 엔드포인트](/ko/x-api/posts/filtered-stream#post-2-tweets-search-stream-rules)를 사용해 정의한 기준과 일치하는 게시물을 걸러내기 위한 규칙을 추가·삭제할 수 있습니다. 이러한 [규칙](/ko/x-api/posts/filtered-stream#building-rules-for-filtered-stream)은 메시지 키워드, 해시태그, URL 등 게시물 속성과 매칭되는 연산자를 사용해 만들 수 있습니다. 연산자와 규칙 절은 불리언 로직과 괄호로 결합해 필터의 매칭 동작을 정교하게 조정할 수 있습니다. 

일련의 규칙을 추가한 후에는 X가 JSON 형식의 [Post objects](/ko/x-api/fundamentals/data-dictionary#tweet)를 전달하기 시작할 웹훅을 등록할 수 있습니다. 웹훅에는 규칙과 일치하는 콘텐츠만 수신됩니다.

이 엔드포인트는 수정된 게시물을 지원합니다. 웹훅은 하나 이상의 필터와 일치하는 수정된 게시물을 해당 게시물 ID 배열을 포함한 수정 이력과 함께 수신합니다. 수정 이력이 없는 게시물의 경우 이 배열에는 단일 ID가 포함됩니다. 수정된 게시물의 경우 이 배열에는 여러 ID가 포함되며, 수정 순서를 반영해 오름차순으로 정렬되고 배열의 마지막에는 최신 버전이 위치합니다. 게시물 수정 방식에 대해 자세히 알아보려면 [게시물 수정 기본 사항](/ko/x-api/fundamentals/edit-posts) 페이지를 참조하세요. 

**참고:** 이 엔드포인트 그룹은 규칙 설정 방식 측면에서 [v2 필터드 스트림 엔드포인트](/ko/x-api/posts/filtered-stream/introduction)와 유사합니다. 유일한 차이는 규칙과 일치하는 게시물의 전달 방식입니다.

- v2 필터드 스트림 엔드포인트의 경우, 지속 연결을 설정하고 규칙과 일치하는 게시물을 수신해야 합니다.
- 이 필터드 스트림 웹훅 엔드포인트의 경우, 웹훅을 등록하면 X가 규칙과 일치하는 게시물을 웹훅으로 전달합니다.

이는 클라이언트 연결 종료나 서비스 정상 재시작으로 인한 서버 측 연결 종료로부터 발생하는 연결 해제를 처리할 필요를 없애줍니다. 초당 수백 건의 이벤트를 전달할 가능성이 높으므로, 웹훅 서버는 이러한 동시 연결 시도를 처리할 수 있어야 합니다.

이 엔드포인트는 현재 Enterprise 개발자에게 제공되며 다음 구성을 지원합니다:

- 프로젝트당 25,000개+ 규칙
- 규칙 구성 시 모든 연산자 사용 가능
- 최대 2048자까지의 규칙 작성 가능
- Enterprise 액세스는 [여기](https://developer.x.com/en/products/x-api/enterprise/enterprise-api-interest-form)에서 신청

필터드 스트림에서 반환된 게시물은 월간 [트윗 한도(게시 한도)](/ko/x-api/fundamentals/post-cap)에 포함됩니다.
