Post는 X의 주요 리소스 중 하나입니다. 가장 단순한 형태에서 Post는 최대 280자의 텍스트를 포함할 수 있으며, 계정 설정에 따라 공개 또는 비공개로 게시될 수 있습니다. 또한 미디어, 장소, 투표, URL 등 다양한 객체를 Post에 첨부할 수 있습니다. 아울러 대부분의 Post는 생성 후 최대 30분 동안 편집할 수 있습니다.

Post에는 다양한 HTTP 조회·선택·전달 방법이 있으나, 이 REST 엔드포인트 그룹은 [Post ID](/ko/resources/fundamentals/x-ids)로 지정된 단일 Post 또는 여러 Post를 반환하는 데 중점을 둡니다. 이 엔드포인트는 Post의 최신 세부 정보를 가져오고, Post의 가용성을 확인하며, 편집 기록을 조회할 수 있습니다. 또한 [컴플라이언스 이벤트](/ko/x-api/enterprise-gnip-2.0/fundamentals/firehouse) 관리를 위해서도 중요합니다.

Post 조회 엔드포인트는 편집된 Post의 메타데이터를 제공합니다. 2022년 9월 29일 이후 생성된 모든 Post에는 실제로 편집되지 않았더라도 편집 메타데이터가 포함됩니다. 각 편집마다 새로운 Post ID가 생성되며, Post의 편집 기록은 원본부터 시작하는 Post ID 배열로 표현됩니다.

이 엔드포인트는 항상 최신 편집본과 그 편집 기록을 함께 반환합니다. 30분 편집 가능 시간 이후에 수집된 Post는 최종 버전을 나타냅니다. Edit Post 메타데이터에 대해 더 알아보려면 [Edit Posts 기본 사항](/ko/x-api/fundamentals/edit-posts) 페이지를 확인하세요.

이 엔드포인트는 GET HTTP 메서드를 사용하며, Post 텍스트, 생성 타임스탬프, 해시태그·멘션·사진의 목록 및 메타데이터 등의 필드를 포함하는 하나 이상의 [Posts 객체](/ko/x-api/fundamentals/data-dictionary)를 반환합니다.

<Note>
  ### 계정 설정

  이 엔드포인트에 액세스하려면 다음이 필요합니다:

  - 승인된 [developer account](https://developer.x.com/en/portal/petition/essential/basic-info).
  - [Project](/ko/resources/fundamentals/projects) 내 [developer App](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용한 인증.

  X API v2 엔드포인트에 대한 액세스 방법은 [시작 가이드](/ko/x-api/getting-started/getting-access)에서 자세히 알아보세요.
</Note>
