<div id="getting-started-with-the-posts-lookup-endpoints">
  ## Posts 조회 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 Postman을 사용해 지정한 필드 세트와 함께 Posts 조회 엔드포인트로 첫 요청을 보내는 방법을 안내합니다.

다른 언어의 샘플 코드를 보려면 [X API v2 샘플 코드](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요.

<Note>
  ### 사전 준비 사항

  이 가이드를 완료하려면 요청 인증에 사용할 [keys and tokens](/ko/resources/fundamentals/authentication)이 필요합니다. 다음 단계를 따라 이러한 keys and tokens를 생성할 수 있습니다.

  - [개발자 계정에 등록](https://developer.x.com/en/portal/petition/essential/basic-info)하고 승인을 받습니다.
  - 개발자 포털에서 [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [developer App](/ko/resources/fundamentals/developer-apps)을 생성합니다.
  - App의 “Keys and tokens” 페이지로 이동해 필요한 자격 증명을 생성합니다. 모든 자격 증명은 안전한 위치에 보관하세요.
</Note>

<div id="steps-to-build-a-get-tweets-request">
  ### GET /tweets 요청을 만드는 단계
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### 1단계: 도구 또는 라이브러리로 시작
</div>

이 엔드포인트에 요청을 보내는 데 사용할 수 있는 도구, 코드 예제, 라이브러리가 여러 가지 있지만, 여기서는 과정을 간소화하기 위해 Postman을 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 로드하려면 아래 버튼을 클릭하세요.

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2를 Postman에 추가
</Button>

Postman에 X API v2 컬렉션을 로드한 후에는 "Post Lookup > Multiple Posts" 요청으로 이동하세요.

<div id="step-two-authenticate-your-request">
  #### 2단계: 요청 인증
</div>

X API에 정상적으로 요청하려면 권한이 있는지 확인해야 합니다. 다음 방법으로 요청을 인증할 수 있습니다:

- [OAuth 2.0 App-Only](ko/resources/fundamentals/authentication)
- [OAuth 2.0 Authorization Code with PKCE](ko/resources/fundamentals/authentication)
- [OAuth 1.0a User Context](ko/resources/fundamentals/authentication)

간단히 진행하기 위해 이 예제에서는 OAuth 2.0 App-Only를 사용하지만, 비공개 [지표](/ko/x-api/fundamentals/metrics)나 게시물을 요청하려면 다른 인증 방법 중 하나를 사용해야 합니다.

OAuth 2.0 App-Only를 사용하려면 Postman에서 오른쪽 상단의 “X API v2” 환경을 선택한 뒤, 키와 토큰(특히 App Access Token, 즉 App-only Bearer Token)을 "initial value" 및 "current value" 필드에 추가하세요(환경 드롭다운 옆 눈 모양 아이콘을 클릭).

올바르게 설정했다면 이러한 변수들이 요청의 authorization 탭에 자동으로 채워집니다.

<div id="step-three-identify-and-specify-which-posts-you-would-like-to-retrieve">
  #### 3단계: 가져올 게시물을 식별하고 지정하기
</div>

요청 본문에서 수신하려는 게시물 또는 게시물 집합을 반드시 지정해야 합니다. X.com에서 게시물을 클릭한 뒤 URL에서 게시물 ID를 확인할 수 있습니다. 예를 들어, 다음 URL의 게시물 ID는 `1228393702244134912`입니다.

`https://x.com/TwitterDev/status/1228393702244134912`

Postman에서 "Params" 탭으로 이동하여 ids 매개변수의 "Value" 열에 이 ID 또는 쉼표로 구분한 여러 게시물 ID를 입력합니다.

| Key  | Value |
| :---- | :----- |
| `ids` | `1228393702244134912,1227640996038684673,1199786642791452673` |

---

<div id="step-four-identify-and-specify-which-fields-you-would-like-to-retrieve">
  #### 4단계: 가져올 필드를 식별하고 지정하기
</div>

3단계를 마친 뒤 "Send" 버튼을 클릭하면 응답에는 기본 [Post 객체 필드](/ko/x-api/fundamentals/data-dictionary)인 `id`, `text`, `edit_history_tweet_ids`가 반환됩니다.

이번 연습에서는 서로 다른 객체에서 다음과 같은 세 가지 추가 필드 세트를 요청합니다:

- 기본 트윗 객체의 추가 필드: `tweet.created_at`
- 반환된 게시물의 작성자에 해당하는 사용자 객체의 기본 필드: `id`, `name`, `username`
- 연관된 사용자 객체의 추가 필드: `user.created_at`

Postman에서 "Params" 탭으로 이동하여 "Query Params" 표에 다음 `key:value` 쌍을 추가하세요:
| Key            | Value               | Returned fields                          |
| :-------------- | :------------------- | :---------------------------------------- |
| `tweet.fields` | `created_at`        | `tweets.created_at`                      |
| `expansions`   | `author_id`         | `includes.users.id`, `includes.users.name`, `includes.users.username` |
| `user.fields`  | `created_at`        | `includes.users.created_at`              |

"Send" 버튼 옆에 다음 URL이 표시되어야 합니다:

```
https://api.x.com/2/tweets?ids=1228393702244134912,1227640996038684673,1199786642791452673&tweet.fields=created_at&expansions=author_id&user.fields=created_at
```
