<div id="helpful-tools">
  ## 유용한 도구
</div>

몇 가지 핵심 개념으로 들어가기 전에 다음 도구에 익숙해지실 것을 권장합니다:

**Postman**\
Postman은 사용 가능한 모든 경로와 본문 매개변수를 포함해 엔드포인트를 테스트하는 데 탁월한 도구입니다. 자세한 내용은 [Postman 시작하기 가이드](/ko/tutorials/postman-getting-started)를 확인하세요.

**코드 샘플**\
선호하는 프로그래밍 언어의 코드 샘플은 [GitHub 페이지](https://github.com/xdevplatform/Twitter-API-v2-sample-code)에서 확인하세요.

**서드파티 라이브러리**\
v2 엔드포인트와 호환되는 커뮤니티 제작 [서드파티 라이브러리](/ko/resources/tools-and-libraries)를 활용하세요.

<div id="authentication">
  ### 인증
</div>

모든 X API v2 엔드포인트는 인증된 요청이 필요합니다. 다음 중 하나로 인증할 수 있습니다:

- API 키, 액세스 토큰, 그리고 [인증 헤더 생성](/ko/resources/fundamentals/authentication#authorizing-a-request)에 필요한 추가 매개변수를 사용하는 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication)
- 요청에 [앱 액세스 토큰](/ko/resources/fundamentals/authentication)을 전달하는 [OAuth 2.0 앱 전용](/ko/resources/fundamentals/authentication#oauth-2-0)
- 앱 범위와 다중 기기 인증을 보다 세밀하게 제어할 수 있는 [OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication)

<Note>
  OAuth 1.0a는 구현이 까다로울 수 있습니다. 익숙하지 않다면 [라이브러리](/ko/resources/tools-and-libraries)를 사용하거나 OAuth 2.0을 고려하세요. 비공개 지표나 게시물에 접근하려면 OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0 Authorization Code with PKCE를 사용하세요.
</Note>

<Tip>
  **참고**

  > 다음 필드를 요청하는 경우 OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0 Authorization Code가 필요합니다:
  >
  > - `tweet.fields.non_public_metrics`
  > - `tweet.fields.promoted_metrics`
  > - `tweet.fields.organic_metrics`
  > - `media.fields.non_public_metrics`
  > - `media.fields.promoted_metrics`
  > - `media.fields.organic_metrics`
</Tip>

<div id="developer-portal-projects-and-apps">
  ### 개발자 포털, 프로젝트와 앱
</div>

X API v2 자격 증명을 얻으려면 다음이 필요합니다.

1. 승인된 [개발자 계정](/ko/resources/fundamentals/developer-portal)
2. 해당 개발자 계정 내의 [프로젝트](/ko/resources/fundamentals/projects)
3. 해당 프로젝트 내의 [개발자 앱](/ko/resources/fundamentals/developer-apps) — 여기에서 키와 토큰을 확인할 수 있습니다.

<div id="rate-limits">
  ### 속도 제한
</div>

요청량을 관리하기 위해 X API 요청에는 [속도 제한](/ko/resources/fundamentals/rate-limits)이 적용됩니다. 제한은 앱 및 사용자 수준 모두에 적용됩니다:

- **앱 수준**: 모든 앱이 일정 기간 내에 수행할 수 있는 요청 수를 제한합니다.
- **사용자 수준**: 인증된 사용자가 개발자 앱 전반에서 게시물 조회를 수행할 수 있는 빈도를 제한합니다.

<div id="fields-and-expansions">
  ### 필드와 확장
</div>

X API v2에서는 `fields`와 `expansions`를 사용해 특정 데이터 필드를 선택할 수 있습니다:

- **확장(expansions)**: 추가로 연관된 객체를 조회할 수 있게 합니다. 지원되는 확장은 다음과 같습니다.
  - `edit_history_tweet_ids`
  - `attachments.poll_ids`
  - `attachments.media_keys`
  - `author_id`
  - `entities.mentions.username`
  - `geo.place_id`
  - `in_reply_to_user_id`
  - `referenced_tweets.id`
  - `referenced_tweets.id.author_id`

- **필드(fields)**: 객체 내에서 반환할 데이터 항목을 지정합니다. Post 객체의 기본 반환 항목은 `id`, `text`, `edit_history_tweet_ids`입니다. `tweet.created_at`, `tweet.entities` 등 다른 항목은 명시적으로 요청해야 합니다.

자세한 내용은 [X API v2 데이터 사전](/ko/x-api/fundamentals/data-dictionary)의 필드와 확장 가이드를 참고하세요.

<div id="post-edits">
  ### 게시물 편집
</div>

자격을 갖춘 게시물은 게시 후 30분 이내에 최대 5회까지 편집할 수 있습니다. Posts 조회 엔드포인트는 항상 최신 게시물 버전을 제공합니다. 거의 실시간에 가까운 사용 사례의 경우 이 시간 범위를 유의하세요. 자세한 내용은 [게시물 편집 기본 사항](/ko/x-api/fundamentals/edit-posts)을 참조하세요.

<div id="edge-cases">
  ### 경계 사례
</div>

- **프로모션 지표**: 프로모션되지 않은 게시물에 대해 프로모션 지표를 요청하면 빈 응답이 반환됩니다.
- **잘린 텍스트**: 리트윗에서는 게시물 텍스트가 잘립니다. 전체 텍스트를 가져오려면 참조된 게시물을 확장하세요.
