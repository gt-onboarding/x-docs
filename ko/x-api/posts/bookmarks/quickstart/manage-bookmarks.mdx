<div id="getting-started-with-the-manage-bookmarks-endpoints">
  ## Bookmarks 관리 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 [Postman](/ko/tutorials/postman-getting-started)을 사용하여 Bookmarks 관리 엔드포인트에 첫 요청을 보내는 방법을 안내합니다.

다양한 언어의 샘플 코드를 보려면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요. 

<Note>
  **사전 준비 사항**

  이 가이드를 완료하려면 요청을 인증할 [keys and tokens](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 통해 키와 토큰을 생성할 수 있습니다:

  - [개발자 계정에 신청](https://developer.x.com/en/apply-for-access)하고 승인을 받습니다.
  - 개발자 포털에서 [Project](/ko/resources/fundamentals/projects)와 연결된 [developer App](/ko/resources/fundamentals/developer-apps)을 생성합니다.
  - App의 “Keys and tokens” 페이지로 이동하여 필요한 자격 증명을 생성합니다. 모든 자격 증명은 안전한 위치에 저장하세요.
</Note>

<div id="steps-to-build-a-manage-bookmarks-request">
  ### Bookmarks 관리 요청 빌드 단계
</div>

**1단계: 도구 또는 라이브러리 시작하기**

이 엔드포인트에 요청을 보내는 데 사용할 수 있는 다양한 도구, 코드 예제, 라이브러리가 있습니다. 여기서는 과정을 단순화하기 위해 Postman을 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 로드하려면 다음 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Postman에 X API v2 추가
</Button>

Postman에 X API v2 컬렉션을 로드한 후 “Bookmarks” 폴더로 이동하여 “Create a Bookmark”를 선택하세요.

**2단계: 요청 인증**

이 엔드포인트에 성공적으로 요청하려면 [OAuth 2.0 Authorization Code Flow with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3)를 사용해야 합니다. Postman 내에서 액세스 토큰을 생성할 수 있습니다.

“Authorization” 탭으로 이동하여 “OAuth 2.0”을 선택하세요.

이 탭에서 다음을 수행하세요:

1. 토큰 이름 지정

2. Grant Type을 Authorization Code (with PKCE)로 선택

3. 매개변수 업데이트:

   **Callback URL** - [https://www.example.com](https://www.example.com/)

   이는 Developer Portal의 인증 설정 페이지에서 설정한 callback URL과 일치해야 합니다.

   **Auth URL** - [https://x.com/i/oauth2/authorize](https://x.com/i/oauth2/authorize)

   **Access Token URL** - [https://api.x.com/2/oauth2/token](https://api.x.com/2/oauth2/token)

   **Client ID** - Developer Portal에서 OAuth 2.0 client ID를 복사해 붙여넣기

   **Client Secret** - Developer Portal에서 OAuth 2.0 client secret을 복사해 붙여넣기. 비밀 클라이언트 유형의 App을 사용하는 경우에만 필요합니다.

4. scopes를 다음 값으로 설정: post.read users.read bookmark.write

5. state 필드에 “State” 입력

6. “Generate Token” 클릭

7. 저장 아이콘을 눌러 폴더 변경 사항 저장

X에 로그인되어 있지 않다는 메시지가 표시될 수 있습니다. 이 오류가 발생하면 Postman 내에서 대리로 게시하려는 X 계정으로 로그인해야 합니다.

**3단계: 사용자 지정**

이 엔드포인트에서는 응답으로 받고자 하는 사용자의 사용자 ID를 지정해야 합니다. 예를 들어 @XDevelopers의 사용자 ID는 `2244994945`입니다.

Postman에서 “Params” 탭으로 이동하여 id 매개변수의 값으로 본인 또는 인증된 사용자의 ID를 입력하세요.

|     |     |     |
| :--- | :--- | :--- |
|     | Key | Value |
|     | id  | `2244994945` |

**4단계: 북마크할 게시물 지정**

“Body” 탭으로 이동하여 북마크에 저장하려는 게시물의 Post ID가 설정되어 있는지 확인하세요. JSON 페이로드는 아래와 유사해야 합니다.

```
      {"tweet_id": "1460323737035677698"}
```

**5단계: 요청 전송 및 응답 확인**

모든 설정을 마쳤다면 "Send" 버튼을 눌러 다음 예시와 유사한 응답을 받게 됩니다:

```
{
   "data": {
       "bookmarked": true
   }
}
```

게시물을 삭제하려면 Postman에 로드된 X API v2 컬렉션의 “Bookmarks” 폴더에서 “Remove a Bookmark” 요청을 선택하세요. 먼저 대리로 요청을 보내는 사용자의 사용자 ID를 “`id`” 열의 값으로 지정해야 합니다. 이 엔드포인트는 또한 삭제하려는 게시물의 ID가 필요합니다. 그런 다음 “Params” 탭에서 삭제하려는 게시물의 ID를 “`tweet_id`” 열의 값으로 입력하세요.

삭제 요청이 성공하면 다음 예시와 유사한 응답을 받게 됩니다:

```
{
   "data": {
       "bookmarked": false
   }
}
```
