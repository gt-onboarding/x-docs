<div id="postman">
  #### Postman
</div>

Postman은 엔드포인트를 시험해 볼 때 유용한 도구입니다. 각 Postman 요청에는 사용할 수 있는 항목을 빠르게 파악할 수 있도록 모든 경로 및 본문 매개변수가 포함되어 있습니다. Postman 컬렉션에 대해 자세히 알아보려면 ["Postman 사용"](/ko/tutorials/postman-getting-started) 페이지를 방문하세요.

<div id="authentication">
  #### 인증
</div>

모든 X API v2 엔드포인트는 키와 토큰으로 불리는 자격 증명 세트를 사용해 요청을 인증해야 합니다.

다음 엔드포인트들은 [OAuth 2.0 Authorization Code Flow with PKCE](/ko/resources/fundamentals/authentication/oauth-2-0/authorization-code)를 반드시 사용해야 합니다. 즉, 요청을 성공적으로 수행하려면 키와 사용자 액세스 토큰이 필요합니다. 액세스 토큰은 여러분이 대신 요청하려는 사용자와 연결되어 있어야 합니다. 다른 사용자의 액세스 토큰을 생성하려면, 해당 사용자가 [Authorize URL](/ko/resources/fundamentals/authentication#authorize-url)을 통해 여러분의 앱을 승인(또는 인증)해야 합니다.

OAuth 2.0은 다소 까다로울 수 있습니다. 이 인증 방식에 익숙하지 않다면, [라이브러리](/ko/x-api/tools-and-libraries/overview)나 Postman 같은 도구를 사용해 요청을 올바르게 인증하는 것을 권장합니다.

<div id="developer-portal-projects-and-developer-apps">
  #### 개발자 포털, 프로젝트와 개발자 앱
</div>

X API v2 엔드포인트에서 사용할 인증 자격 증명을 받으려면 [개발자 계정](/ko/resources/fundamentals/developer-portal)을 보유하고 해당 계정 내에 [프로젝트](/ko/resources/fundamentals/projects)를 설정한 뒤, 그 프로젝트 내에 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성해야 합니다. 이후 개발자 앱에서 키와 토큰을 확인할 수 있습니다. 

<div id="rate-limits">
  #### 요청 한도
</div>

매일 수천 명의 개발자가 X API에 요청을 보냅니다. 이러한 대규모 트래픽을 관리하기 위해 [요청 한도](/ko/resources/fundamentals/rate-limits)가 각 엔드포인트에 적용되며, 앱 또는 인증된 사용자를 대신해 보낼 수 있는 요청 수를 제한합니다.

이 엔드포인트들은 사용자 수준으로 한도가 적용됩니다. 즉, 요청을 위임받은 인증된 사용자는 어떤 개발자 앱을 사용하든 해당 엔드포인트를 호출할 수 있는 횟수가 정해져 있습니다. GET 메서드는 15분당 180건의 사용자 한도가 있습니다. Bookmarks 조회 엔드포인트의 GET 메서드를 사용하면 가장 최근에 북마크한 게시물 800개가 반환됩니다. 또한 POST 및 DELETE 메서드는 15분당 50건의 사용자 한도가 있습니다.
