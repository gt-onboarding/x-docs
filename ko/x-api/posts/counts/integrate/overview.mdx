<div id="postman">
  #### Postman
</div>

Postman은 이러한 엔드포인트를 테스트하는 데 유용한 도구입니다. 각 Postman 요청에는 해당 엔드포인트의 모든 매개변수가 포함되어 있어, 어떤 옵션을 사용할 수 있는지 빠르게 파악하는 데 도움이 됩니다. Postman 컬렉션에 대해 자세히 알아보려면 [Using Postman](/ko/tutorials/postman-getting-started) 페이지를 방문하세요.

<div id="authentication">
  #### 인증
</div>

모든 X API v2 엔드포인트는 키와 토큰으로도 불리는 자격 증명을 사용해 [인증](/ko/resources/fundamentals/authentication)된 요청이어야 합니다. 이 엔드포인트는 [OAuth 2.0 Bearer Token](/ko/resources/fundamentals/authentication#oauth-2-0)을 사용해야 하며, 즉 요청을 성공적으로 처리하려면 [Bearer Token](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)을 포함해 전달해야 합니다. Bearer Token은 개발자 앱에서 직접 생성하거나 [POST oauth2/token](/ko/resources/fundamentals/authentication#post-oauth2-token) 엔드포인트를 사용해 생성할 수 있습니다.

<div id="developer-portal-projects-and-developer-apps">
  #### 개발자 포털, 프로젝트와 개발자 앱
</div>

X API v2의 어떤 엔드포인트를 사용하려면 [개발자 계정](/ko/resources/fundamentals/developer-portal)을 보유하고, 해당 계정 내에 [프로젝트](/ko/resources/fundamentals/projects)를 설정한 다음, 그 프로젝트 안에 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성해야 합니다. 해당 개발자 앱의 키와 토큰은 최근 게시물 수 엔드포인트에서 사용할 수 있습니다. 전체 아카이브 게시물 수 엔드포인트를 사용하거나 고급 연산자와 더 긴 쿼리 길이를 활용하려면 엔터프라이즈 액세스 승인이 필요합니다.

자세한 내용은 엔터프라이즈 액세스 섹션을 방문해 주십시오.

<div id="rate-limits">
  #### 레이트 리밋
</div>

매일 수많은 개발자가 X API에 요청을 보냅니다. 요청량 관리를 위해 각 엔드포인트에는 [레이트 리밋](/ko/x-api/fundamentals/rate-limits)이 적용되어, 앱 또는 인증된 사용자를 대신해 각 개발자가 보낼 수 있는 요청 횟수가 제한됩니다.

이 엔드포인트는 앱 수준 레이트 리밋이 적용됩니다. 즉, 여러분(개발자)은 사용 중인 자격 증명으로 식별되는 어떤 앱에서든, 정해진 기간 동안 이 엔드포인트에 보낼 수 있는 요청 횟수가 제한됩니다. 

<div id="building-queries">
  #### 쿼리 작성
</div>

이 엔드포인트의 핵심 기능은 단일 쿼리를 통해 게시물을 필터링하여 제공할 집계 결과를 산출하는 것입니다. 이러한 쿼리는 메시지 키워드, 해시태그, URL 등 게시물 및 사용자 속성과 일치하는 연산자들로 구성됩니다. 연산자는 불리언 로직과 괄호를 사용해 결합하여 쿼리의 매칭 동작을 더욱 정교하게 조정할 수 있습니다.

[쿼리 작성 방법](/ko/x-api/posts/counts/integrate/build-a-query) 가이드를 참고해 더 알아보세요.

<div id="pagination">
  #### 페이지네이션
</div>

최근 게시물 수의 경우 next\_token이 반환되지 않습니다. 이는 세분화 수준과 관계없이 한 번의 API 호출로 지난 7일간의 게시물 수를 받게 됨을 의미합니다.

전체 아카이브 게시물 수의 경우 지난 30일치 데이터를 받게 됩니다. 30일을 초과하는 데이터에 대해서는 next\_token이 반환되며, 이를 사용해 페이지네이션하여 추가 데이터를 가져올 수 있습니다. 

<div id="building-queries-for-post-counts">
  ### 게시 수(Post counts) 쿼리 작성하기
</div>

Post counts 엔드포인트는 단일 쿼리를 GET 요청으로 받아 해당 쿼리와 일치하는 과거 게시 수 집합을 반환합니다. 쿼리는 다양한 게시 속성과 일치시키는 데 사용되는 연산자로 구성됩니다.
