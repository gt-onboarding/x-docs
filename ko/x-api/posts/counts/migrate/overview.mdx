<div id="comparing-x-apis-post-counts-endpoints">
  ## X API의 게시물 수 엔드포인트 비교
</div>

v2 게시물 수 엔드포인트는 궁극적으로 [엔터프라이즈 Search API의 counts 엔드포인트](/ko/x-api/enterprise-gnip-2.0/fundamentals/search-api#counts-requests-post-count)를 대체할 예정입니다. 이전 버전의 게시물 수 엔드포인트를 사용하는 코드, 앱 또는 도구가 있고 최신 X API v2 엔드포인트로 마이그레이션을 고려 중이라면 이 가이드를 참고하세요.

이 페이지에는 다음과 같은 비교 표가 포함되어 있습니다:

- [최근 게시물 수](#recent)
- [전체 아카이브 게시물 수](#full-archive)
- [필터링 연산자 비교](#operator)

<div id="recent-post-counts-comparison">
  ### 최근 게시 수 비교
</div>

엔터프라이즈 버전의 게시 수 엔드포인트는 30일 또는 전체 아카이브 중 하나의 집계를 조회할 수 있습니다. 따라서 7일 기간을 대상으로 하는 v2 최근 게시 수 엔드포인트는 앞서 언급한 두 엔드포인트의 직접적인 대체재가 아닙니다.

다만 비교를 돕기 위해, v2 최근 게시 수 엔드포인트가 엔터프라이즈 30일 엔드포인트와 어떻게 다른지 살펴보겠습니다.

다음 표는 다양한 유형의 최근 게시 수 엔드포인트를 비교합니다:

| **설명** | **엔터프라이즈** | **X API v2** |
| :--- | :--- | :--- |
| 호스트 도메인 | https://gnip-api.x.com | https://api.x.com |
| 엔드포인트 경로 | /search/30day/accounts/:account\_name/:label/counts.json | /2/tweets/counts/recent |
| [인증](/ko/resources/fundamentals/authentication) | 기본 인증 | OAuth 2.0 베어러 토큰 |
| 타임스탬프 형식 | YYYYMMDDhhmm | YYYY-MM-DDTHH:mm:ssZ  <br />[ISO 8601 / RFC 3339](https://tools.ietf.org/html/rfc3339#section-5.6) |
| 반환되는 게시의 최대 경과 기간 | 31일 | 7일 |
| 지원 HTTP 메서드 | GET | GET |
| 기본 요청 속도 제한 | 1초당 20회 요청, 검색 데이터와 카운트 요청 전반에 대해 집계  <br />분당 한도는 계약에 따라 파트너별로 상이합니다. | 사용자당 15분당 180회 요청  <br />앱당 15분당 450회 요청 |
| [annotations](/ko/x-api/fundamentals/post-annotations)를 사용한 필터링 지원 |     | ✔   |
| [conversation\_id](/ko/x-api/fundamentals/conversation-id)를 사용한 필터링 지원 |     | ✔   |
| 게시 데이터 배열의 JSON 키 이름 | results | data |
| 시간 세분화 수준 | 일, 시간, 분 | 일, 시간, 분 |
| 표준 시간대 | UTC | UTC |
| 기간 선택을 위한 요청 매개변수 | fromDate  <br />toDate | start\_time  <br />end\_time |
| 게시 ID로 탐색하기 위한 요청 매개변수 |     | since\_id  <br />until\_id |
| [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [developer App](/ko/resources/fundamentals/developer-apps)의 자격 증명 필요 |     | ✔   |

| 설명 | 엔터프라이즈 | X API v2 |
| :--- | :--- | :--- |
| 호스트 도메인 | https://gnip-api.x.com | https://api.x.com |
| 엔드포인트 경로 | /search/fullarchive/accounts/:account\_name/:label/counts | /2/tweets/counts/all |

| 반환되는 게시물 수의 기준 시점(최대 과거 범위) | 2006년 3월부터의 전체 아카이브 | 2006년 3월부터의 전체 아카이브 |

| 기본 요청 속도 제한 | 분당 속도 제한은 계약서에 명시된 파트너별로 상이합니다.  <br />초당 20건 요청 | 앱당 15분에 300건 요청  <br />앱당 초당 1건 요청 |
| 세분화 수준 | 일, 시간, 분 | 일, 시간, 분 |
| [annotations](/ko/x-api/fundamentals/post-annotations)를 사용한 필터링 지원 |     | ✔   |
| [conversation\_id](/ko/x-api/fundamentals/conversation-id)를 사용한 필터링 지원 |     | ✔   |
| 게시물 데이터 배열의 JSON 키 이름 | results | data |
| 기간 선택을 위한 요청 매개변수 | fromDate  <br />toDate | start\_time  <br />end\_time |
| 게시물 ID로 탐색하기 위한 요청 매개변수 |     | since\_id  <br />until\_id |
| 페이지네이션을 위한 JSON 키 이름 | next | meta.next\_token |
| 페이지네이션을 위한 요청 매개변수 | next\_token | next\_token 또는 pagination\_token |
| 시간대 | UTC | UTC |
| [Academic Research access](/ko/x-api/getting-started/about-x-api)가 있는 [프로젝트](/ko/resources/fundamentals/projects)에 연결된 [developer App](/ko/resources/fundamentals/developer-apps)의 자격 증명 사용 필요 |     | ✔   |

<div id="filtering-operator-comparison">
  ### 필터링 연산자 비교
</div>

두 가지 버전(Enterprise와 v2)의 Post counts는 사용 가능한 연산자가 다르며, 각 버전 내에서도 연산자 제공 수준이 상이합니다. 아래에서 자세히 설명합니다.

Enterprise

- Enterprise 연산자에는 하위 등급이 없습니다. 모든 Enterprise 사용자는 모든 Enterprise 연산자를 사용할 수 있습니다.

X API v2

- **Core:** 모든 v2 사용자에게 제공됩니다.
- **Advanced:** Academic Research 액세스가 승인된 사용자에게만 제공됩니다.

각 연산자 세트에 대해 더 알아보려면 다음 가이드를 참고하세요.

- [Enterprise operators](/ko/x-api/enterprise-gnip-2.0/fundamentals/search-api#building-search-queries)
- [X API v2 operators](/ko/x-api/posts/search/integrate/build-a-query)
