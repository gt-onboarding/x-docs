<div id="getting-started-with-the-full-archive-post-counts-endpoint">
  ### 전체 아카이브 Post counts 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 HTTP 요청을 보낼 수 있는 그래픽 도구인 Postman을 사용해 전체 아카이브 Post counts 엔드포인트에 첫 요청을 만드는 방법을 안내합니다.

다양한 프로그래밍 언어의 샘플 코드를 보려면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요.

<Note>
  ### 사전 준비 사항

  전체 아카이브 Post counts 엔드포인트는 현재 Pro 또는 Enterprise 액세스 보유자에게만 제공됩니다. 이 엔드포인트를 사용하려면 Pro 또는 [Enterprise 액세스](https://docs.google.com/forms/d/e/1FAIpQLScO3bczKWO2jFHyVZJUSEGfdyfFaqt2MvmOfl_aJp0KxMqtDA/viewform)를 신청해야 합니다.

  액세스 승인을 받는 것 외에도, 요청을 인증하기 위한 [keys and tokens](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따라 이러한 키와 토큰을 생성하세요:

  - 개발자 포털에서 Enterprise 액세스가 있는 [Project](/ko/resources/fundamentals/projects)로 이동해 해당 Project 내에 연결된 [developer App](/ko/resources/fundamentals/developer-apps)이 있는지 확인하세요.

  - App의 “Keys and tokens” 페이지로 이동해 필요한 자격 증명을 생성하세요. 모든 자격 증명은 안전한 위치에 저장하세요.
</Note>

<div id="steps-to-build-a-full-archive-post-counts-request">
  #### 전체 아카이브 Post 카운트 요청을 만드는 단계
</div>

**1단계: 도구 또는 라이브러리 선택**

이 엔드포인트에 요청을 보내는 방법으로 사용할 수 있는 다양한 도구, 코드 예제, 라이브러리가 있습니다. 여기서는 절차를 단순화하기 위해 Postman을 사용합니다.

X API v2 Postman 컬렉션을 환경에 로드하려면 다음 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Postman에 X API v2 추가
</Button>

Postman에 X API v2 컬렉션을 로드한 뒤, Post counts > Full-archive Post counts 요청으로 이동합니다.

**2단계: 요청 인증**

X API에 올바르게 요청하려면 권한이 있어야 합니다. 이 엔드포인트의 경우 [OAuth 2.0 App-Only](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) 방식으로 요청을 인증해야 합니다.

키와 토큰, 특히 [App Access Token](/ko/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token) (App-only Bearer Token)을 Postman에 추가하세요. 이를 위해 Postman 오른쪽 상단에서 “X API v2” 환경을 선택한 후 "initial value" 및 "current value" 필드에 키와 토큰을 입력합니다(환경 드롭다운 옆의 눈 아이콘 클릭).

정상적으로 설정했다면 이 변수는 요청의 authorization 탭에 자동으로 반영됩니다.
 

**3단계: 쿼리 작성**

각 전체 아카이브 Post 카운트 요청에는 단일 [query](/ko/x-api/posts/counts/integrate/build-a-query)가 필요합니다. 이 예에서는 @XDevelopers 계정이 게시한 Post에 일치하는 쿼리를 사용합니다. from 연산자를 사용하고 값을 XDevelopers로 설정합니다(대소문자 무시):

`from:XDevelopers`

Postman에서 "Params" 탭으로 이동하여 이 ID 또는 쉼표로 구분된 Post ID 문자열을 `ids` 매개변수의 "Value" 열에 입력합니다.
 

| Key | Value | Description |
| :--- | :--- | :--- |
| `query` | `from:XDevelopers` | 전체 아카이브 Post 카운트 엔드포인트에 제출할 쿼리 |

4단계(선택 사항): 세분화 수준과 기간 지정

3단계 후 ‘Send’ 버튼을 클릭하면 기본 전체 아카이브 Post 카운트(최근 30일의 시간 단위)가 반환됩니다. 일 단위로 받고 싶다면 granularity 매개변수에 day 값을 추가합니다. 30일 이전 기간의 Post 카운트를 원한다면 start\_time과 end\_time 매개변수를 원하는 값으로 지정합니다. 

Postman에서 "Params" 탭으로 이동하여 "Query Params" 표에 다음 key:value 쌍을 추가합니다:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Description** |
| granularity | day | Post 카운트 결과의 세분화 수준. 가능한 값: day, hour, minute |
| start\_time | 2021-05-01T00:00:00Z | Post가 제공될 가장 이른 UTC 타임스탬프 |
| end\_time | 2021-06-01T00:00:00Z | Post가 제공될 가장 늦은 UTC 타임스탬프 |

이제 "Send" 버튼 옆에 다음 URL이 표시됩니다:

`https://api.x.com/2/tweets/counts/all?query=from%3AXDevelopers&start_time=2021-05-01T00:00:00Z&end_time=2021-06-01T00:00:00Z&granularity=day`

**6단계: 결과 페이지네이션하기**

응답의 meta 객체에 next\_token이 포함되어 있으면, 해당 값을 next\_token 쿼리 매개변수로 전달합니다.

| 키 | 값 | 설명 |
| :--- | :--- | :--- |
| next\_token | 이전 요청의 meta 객체에서 가져온 next\_token 값을 여기에 넣습니다. | 최신 요청에서 남은 결과가 모두 반환되지 않으면 meta 객체에 next\_token이 포함됩니다. 그 필드의 값을 추출해 다음 요청의 next\_token 매개변수 값으로 설정하고, 다른 요청 매개변수는 그대로 유지합니다. |

올바른 next\_token 값을 설정했으면 Send 버튼을 눌러 다음 페이지의 결과를 확인하세요.
