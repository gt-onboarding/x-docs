필터드 스트림 엔드포인트 그룹은 개발자가 공개 게시물 스트림을 필터링할 수 있도록 합니다. 이 엔드포인트 그룹에는 규칙을 생성·관리하고, 해당 규칙을 적용해 일치하는 공개 게시물만 반환하도록 스트림을 필터링할 수 있게 해주는 여러 엔드포인트가 포함됩니다. 이 엔드포인트 그룹을 통해 사용자는 특정 주제와 이벤트를 수신하고, 경쟁 관련 대화를 모니터링하며, 트렌드의 형성을 파악하는 등 다양한 작업을 수행할 수 있습니다.

개발자는 REST [rules endpoint](/ko/x-api/posts/filtered-stream#post-2-tweets-search-stream-rules)를 사용해 연결을 끊지 않고도 지속 스트림 연결에 규칙을 추가·삭제할 수 있습니다. 이러한 [rules](/ko/x-api/posts/filtered-stream#building-rules-for-filtered-stream)는 메시지 키워드, 해시태그, URL 등 게시물 속성과 매칭되는 연산자를 사용해 생성할 수 있습니다. 연산자와 규칙 절은 불리언 논리와 괄호를 조합해 필터의 매칭 동작을 더욱 정교하게 다듬을 수 있습니다.

규칙 세트를 추가한 후에는 [스트리밍 연결을 설정](/ko/x-api/posts/filtered-stream#get-2-tweets-search-stream)하여 지속적인 HTTP 스트리밍 연결을 통해 JSON 형식의 [게시물 객체](/ko/x-api/fundamentals/data-dictionary#tweet)를 수신하기 시작할 수 있습니다. 스트림에 연결된 동안에는 규칙과 일치하는 콘텐츠만 수신합니다.

또는 필터드 스트림을 웹훅에 연결해 웹훅 이벤트로 전달받을 수 있습니다. 필터드 스트림에서 웹훅을 사용하는 방법은 [Filtered Stream Webhooks 소개](/ko/x-api/webhooks/stream/introduction)를 참고하세요.

필터드 검색 엔드포인트는 편집된 게시물을 지원합니다. 이 엔드포인트는 하나 이상의 필터와 일치하는 편집된 게시물과 함께 편집 이력(게시물 ID 배열 포함)을 전달합니다. 편집 이력이 없는 게시물의 경우 이 배열에는 단일 ID가 포함됩니다. 편집된 게시물의 경우 이 배열에는 여러 ID가 포함되며, 편집 순서를 반영해 오름차순으로 정렬되며 배열의 마지막 요소가 최신 버전입니다. 게시물 편집 작동 방식에 대해 자세히 알아보려면 [게시물 편집 기본 사항](/ko/x-api/fundamentals/edit-posts) 페이지를 참조하세요.

필터드 스트림 엔드포인트의 일부 기능은 [액세스 수준](/ko/x-api/getting-started/about-x-api)에 따라 제한됩니다:

**Pro 액세스**

- 프로젝트당 규칙 1,000개
- POST /2/tweets/search/stream/rules 엔드포인트로 규칙을 추가할 때 15분당 100건의 요청
- 규칙 작성 시 모든 연산자 사용 가능
- 최대 1,024자 길이의 규칙 작성 가능

**Enterprise 액세스**

- 프로젝트당 규칙 25,000개 이상
- 규칙 작성 시 모든 연산자 사용 가능
- 최대 2,048자 길이의 규칙 작성 가능
- Enterprise 액세스는 [여기](https://developer.x.com/en/products/x-api/enterprise/enterprise-api-interest-form)에서 신청

필터드 스트림에서 반환된 게시물은 월간 [게시 한도](/ko/x-api/fundamentals/post-cap)에 포함됩니다.

<Note>
  **계정 설정**

  이러한 엔드포인트에 액세스하려면 다음이 필요합니다:

  - 승인된 [개발자 계정](https://developer.x.com/en/portal/petition/essential/basic-info)
  - [프로젝트](/ko/resources/fundamentals/projects) 내에 있는 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용한 인증

  X API v2 엔드포인트 액세스 권한에 대해 자세히 알아보려면 [시작하기 가이드](/ko/x-api/getting-started/getting-access)를 참고하세요.
</Note>
