  ## X API의 필터드 스트림 엔드포인트 비교
</div>

v2 필터드 스트림 엔드포인트 그룹은 [Standard v1.1 statuses/filter](https://developer.x.com/en/docs/x-api/v1/tweets/filter-realtime/api-reference/post-statuses-filter)와 [PowerTrack API](/ko/x-api/enterprise-gnip-2.0/powertrack-api)를 대체합니다. 기존 버전의 필터드 스트림 엔드포인트를 사용하는 코드, 앱, 또는 도구가 있고 최신 X API v2 엔드포인트로의 마이그레이션을 검토 중이라면, 이 비교가 시작하는 데 도움이 됩니다. 

자세한 마이그레이션 가이드는 다음을 참고하세요:

[Standard v1.1에서 X API v2로 마이그레이션](ko/x-api/posts/filtered-stream#standard-v1-1-compared-to-x-api-v2)

[PowerTrack API에서 X API v2로 마이그레이션](/ko/x-api/enterprise-gnip-2.0/powertrack-api#powertrack-operators)

아래 표는 X에서 제공하는 필터드 스트리밍 엔드포인트를 비교합니다:
 

| **설명** | **Standard v1.1** | **PowerTrack API** | **X API v2** |
| :--- | :--- | :--- | :--- |
| 액세스 | X 앱 | 엔터프라이즈 계약 및 계정 필요 | 개발자 계정([가입](https://developer.x.com/en/portal/petition/essential/basic-info))과 [프로젝트](/ko/resources/fundamentals/projects) 내 [X 앱](/ko/resources/fundamentals/developer-apps) 필요 |
| :--- | :--- | :--- | :--- |
| 호스트 도메인 | ******https://stream.x.com****** | ******https://gnip-stream.x.com****** | ******https://api.x.com****** |
| 엔드포인트 경로 | ******1.1/statuses/filter.json****** | ******/stream/powertrack/accounts/{gnip_account_name}/publishers/twitter/{stream_label}.json******<br /><br />******/rules/powertrack/accounts/{gnip_account_name}/publishers/twitter/{stream_label}.json ******<br /><br />******/rules/powertrack/accounts/{gnip_account_name}/publishers/twitter/{stream_label}/validation.json****** | ******/2/tweets/search/stream******<br /><br />******/2/tweets/search/stream/rules****** |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 1.0a 사용자 컨텍스트 | HTTP 기본 인증 | OAuth 2.0 앱 전용 |
| 지원되는 HTTP 메서드 | POST | GET  <br />POST | GET  <br />POST |
| 필수 매개변수 | 연결 시 매개변수로 정의된 규칙, 다음 중 최소 하나:<br /><br />\* ******follow******<br />\* ******track******<br />\* ******locations****** | 스트리밍 연결에는 필수 매개변수 없음, 선택적 백필 매개변수 제공.<br /><br />규칙은 별도로 관리 | 스트리밍 연결에는 필수 매개변수 없음, 응답 형식 정의 및 학술 연구 액세스용 [백필 복구 기능](/ko/x-api/posts/filtered-stream#recovering-missed-data-after-a-disconnection-backfill) 추가를 위한 선택적 매개변수 제공.<br /><br />규칙은 별도로 관리 |
| 전달 유형 | 스트리밍 | 스트리밍<br /><br />REST(규칙 관리용) | 스트리밍<br /><br />REST(규칙 관리용) |
| 기본 요청 속도 제한 | 5분당 연결 시도 5회 | POST와 GET 요청 합산 기준 분당 60건<br /><br />/rules: 특정 스트림의 API(POST 및 GET)에 대한 /rules 엔드포인트 전체 합산 기준 분당 60건 | 엔드포인트 및 [액세스 레벨](/ko/x-api/getting-started/about-x-api)에 따라 다름.<br /><br />[GET /2/tweets/search/stream](/ko/x-api/posts/filtered-stream#get-2-tweets-search-stream):  <br />Pro - 앱당 15분에 50회 요청<br /><br />[GET /2/tweets/search/stream/rules](/ko/x-api/posts/filtered-stream#get-2-tweets-search-stream-rules):  <br />Pro - 앱당 15분에 450회 요청<br /><br />---<br /><br />[POST /2/tweets/search/stream/rules](/ko/x-api/posts/adddelete-rules):  <br />Pro - 앱당 15분에 100회 요청 |
| 허용되는 최대 연결 수 | 인증된 사용자당 동시 2개 | 계약에 따라 결정되는 다중/이중화 연결 지원 | Pro 액세스:  <br />1 |
| [복구 및 이중화 기능](/ko/x-api/posts/filtered-stream#filtered-stream-recovery-and-redundancy-features) | 없음 | 백필, 이중화 연결, Replay API |     |
| [게시 한도](/ko/x-api/fundamentals/post-cap) | 파이어호스의 1%로 제한 | 계약에 따라 결정 | 이 엔드포인트에서 수신한 모든 게시물에 프로젝트 단위의 월별 게시 한도가 적용됩니다:<br /><br />Basic:  <br />게시물 10,000개<br /><br />Pro:  <br />게시물 100만 개 |
| Keep-alive 신호/하트비트 | 최소 20초마다 빈 줄(\r\n 또는 유사) | 10초마다 빈 줄(\r\n 또는 유사) | 최소 20초마다 빈 줄(\r\n 또는 유사) |
| 지연 시간 | 10초 | 2초<br /><br />URL 언와인딩 보강의 경우 최소 10초 | 10초 |
| 허용되는 최대 규칙 수 | 1개 규칙(엔드포인트 연결 요청 내) | 계약에 따라 최대 250,000 | Pro 액세스:  <br />규칙 1000개 |
| 규칙 필터 제한 | 연결당 쿼리 1개, 다음 중 한도 내:<br /><br />- 트랙 키워드 400개<br /><br />- 팔로우 사용자 ID 5,000개<br /><br />- 위치 박스 25개 | 규칙당 최대 2,048자 | Pro Access:   <br />규칙당 1,024자 |
| [게시 JSON 형식](/ko/x-api/fundamentals/data-dictionary) | Standard v1.1 형식 | [Native Enriched](/ko/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#native-enriched-tweet-object) 또는 [Activity Streams]() ( [콘솔](/ko/x-api/enterprise-gnip-2.0/fundamentals/overview)에서 선택) | [X API v2 형식](/ko/x-api/fundamentals/data-dictionary) (요청 매개변수 ******fields****** 및 ******expansions******&#xB85C; 결정되며 v1.1 형식과 하위 호환되지 않음)<br /><br />Standard v1.1 형식에서 X API v2 형식으로 마이그레이션하는 방법은 [데이터 형식 마이그레이션 가이드](/ko/x-api/migrate/data-format-migration)를 참조하세요. Native Enriched 및 Activity Streams에 대한 추가 데이터 형식 마이그레이션 가이드는 곧 제공될 예정입니다. |
| 게시물 편집 이력 및 메타데이터 제공 | ✔   | ✔   | ✔   |
| 고유 기능 | 연결 요청의 쿼리 매개변수로 필터링<br /><br />구성 UI 없음 | 독립 엔드포인트에서 생성한 규칙으로 필터링<br /><br />[Enrichment](/ko/x-api/enterprise-gnip-2.0/enterprise-gnip#enrichments) 기능(계약 포함 시 제공)<br /><br />console.gnip.com UI에서 구성 | 독립 엔드포인트에서 생성한 [규칙](/ko/x-api/posts/filtered-stream/integrate/build-a-rule)으로 필터링<br /><br />[지표](/ko/x-api/fundamentals/metrics) 및 URL 보강 기능 포함<br /><br />요청 매개변수로 객체 [fields](/ko/x-api/fundamentals/fields) 및 [expansions](/ko/x-api/fundamentals/expansions) 지정<br /><br />게시물 [주석](/ko/x-api/fundamentals/post-annotations)<br /><br />[Conversation ID](/ko/x-api/fundamentals/conversation-id) 연산자 및 필드<br /><br />[개발자 포털](/ko/resources/fundamentals/developer-portal)에서 구성 |
