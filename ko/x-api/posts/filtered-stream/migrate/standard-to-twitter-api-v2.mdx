<div id="standard-v11-compared-to-x-api-v2">
  ### 표준 v1.1과 X API v2 비교
</div>

v1.1 [statuses/filter](https://developer.x.com/en/docs/x-api/v1/tweets/filter-realtime/api-reference/post-statuses-filter) 엔드포인트로 작업해 오셨다면, 이 가이드는 표준 엔드포인트와 X API v2 필터드 스트림 엔드포인트의 유사점과 차이점을 이해하는 데 도움이 됩니다.

- **유사점**
  - 요청 매개변수와 연산자
  - 게시물 편집 이력 및 메타데이터 지원
- **차이점**
  - 엔드포인트 URL
  - 앱 및 프로젝트 요건
  - 인증 방식
  - 규칙 볼륨 및 지속형 스트림
  - 응답 데이터 형식
  - 요청 매개변수
  - 복구 및 중복 기능 제공 여부
  - 쿼리 연산자

<div id="similarities">
  #### 유사점
</div>

**요청 매개변수 및 연산자**

표준 v1.1 statuses/filter 엔드포인트에는 스트림을 필터링하기 위해 요청과 함께 전달할 수 있는 몇 가지 매개변수가 있습니다. v2 필터드 스트림에서는 대신 [연산자](/ko/x-api/posts/filtered-stream#building-rules-for-filtered-stream) 집합을 사용하며, 이를 불리언 논리로 연결해 원하는 게시물을 필터링합니다. 사용 가능한 연산자에는 기존 표준 v1.1 매개변수를 직접 대체하는 항목도 포함됩니다.

다음 표준 v1.1 요청 매개변수는 X API v2에서 동등한 연산자가 있습니다:
 

| **Standard** | **X API v2** |
| :--- | :--- |
| follow - 쉼표로 구분된 사용자 ID 목록으로, 스트림에 전달되어야 할 게시물의 작성자를 지정합니다. | 특정 사용자와 관련된 게시물을 찾는 데 도움이 되는 여러 연산자:<br /><br />\* @<br />\* from:<br />\* to:<br />\* 등 |
| track - 스트림에 전달될 게시물을 결정하는 데 사용되는 구문(키워드)들의 쉼표로 구분된 목록입니다. | 특정 키워드와 관련된 게시물을 찾는 데 도움이 되는 여러 연산자:<br /><br />\* keyword<br />\* "정확한 구문 일치"<br />\* #<br />\* 등 |

**게시물 편집 이력 및 메타데이터 지원**

두 버전 모두 편집 이력을 설명하는 메타데이터를 제공합니다. 자세한 내용은 [필터드 스트림 API 참조](/ko/x-api/posts/filtered-stream#api-reference-index)와 [게시물 편집 기본 사항 페이지](/ko/x-api/fundamentals/edit-posts)를 참고하세요. 

<div id="differences">
  #### 차이점
</div>

**엔드포인트 URL**

- 표준 v1.1 엔드포인트:
  - https://stream.x.com/1.1/statuses/filter.json
- X API v2 엔드포인트:
  - https://api.x.com/2/tweets/search/stream
  - https://api.x.com/2/tweets/search/stream/rule

**앱 및 프로젝트 요구 사항**

X API v2 엔드포인트는 요청을 인증할 때 [developer App](/ko/resources/fundamentals/developer-apps)의 자격 증명을 사용해야 하며, 해당 App은 [Project](/ko/resources/fundamentals/projects)와 연결되어 있어야 합니다. 모든 X API v1.1 엔드포인트는 독립 실행형 App 또는 Project와 연결된 App의 자격 증명을 사용할 수 있습니다.

**인증 방식**

표준 엔드포인트는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication)를 지원하는 반면, X API v2 필터드 스트림 엔드포인트는 [OAuth 2.0 App-Only](/ko/resources/fundamentals/authentication#oauth-2-0) (Application-only authentication라고도 함)을 지원합니다. X API v2 버전으로 요청하려면 App Access Token을 사용해 요청을 인증해야 합니다.

개발자 포털에서 Project와 앱을 생성할 때 발급된 App Access Token이 더 이상 없는 경우, 개발자 포털의 앱 “Keys and tokens” 페이지로 이동해 새로 발급받을 수 있습니다. App Access Token을 프로그래밍 방식으로 생성하려면 [OAuth 2.0 App-Only 가이드](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)를 참조하세요. 

**규칙 수량 및 지속 스트림**

표준 v1.1 엔드포인트는 스트리밍 연결을 필터링하기 위한 단일 규칙만 지원합니다. 규칙을 변경하려면 스트림 연결을 종료한 뒤, 매개변수로 수정된 필터링 규칙을 포함해 새 요청을 제출해야 합니다. 

X API v2 필터드 스트림 엔드포인트는 단일 스트림에 여러 규칙을 적용할 수 있으며, 스트림 연결을 유지한 상태로 규칙을 추가·제거할 수 있습니다. 

**응답 데이터 형식**

표준 v1.1과 X API v2 엔드포인트 버전 간 가장 큰 차이 중 하나는 페이로드에 어떤 필드를 반환할지 선택하는 방식입니다.

표준 엔드포인트의 경우 많은 응답 필드가 기본으로 제공되며, 이후 매개변수를 사용해 페이로드에 반환할 필드 또는 필드 집합을 선택할 수 있습니다.

X API v2 버전은 기본적으로 Post id와 text 필드만 제공합니다. 추가 필드나 객체를 요청하려면 [fields](/ko/x-api/fundamentals/fields) 및 [expansions](/ko/x-api/fundamentals/expansions) 매개변수를 사용해야 합니다. 이러한 엔드포인트에서 요청한 Post 필드는 기본 Post 객체로 반환됩니다. 확장된 사용자, 미디어, 설문, 장소 객체와 필드는 응답의 includes 객체로 반환됩니다. 그런 다음 Post와 확장 객체 양쪽에 있는 ID를 매칭하여 확장된 객체를 Post 객체에 다시 연결할 수 있습니다. 

각 가이드에서 이러한 새로운 매개변수에 대해 더 자세히 확인하거나, [fields 및 expansions 사용 방법](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) 가이드를 참고하세요. 

또한 표준 v1.1 필드를 최신 v2 필드에 매핑하는 데 도움이 되는 [데이터 형식 마이그레이션 가이드](/ko/x-api/migrate/data-format-migration#migrating-from-standard-v1-1s-data-format-to-v2)도 제공합니다. 이 가이드는 특정 필드를 반환하기 위해 v2 요청에 함께 전달해야 하는 구체적인 expansion 및 field 매개변수도 안내합니다. 
 

특정 필드 요청 방식의 변경 외에도, X API v2는 API가 반환하는 객체(예: Post 및 [user](/ko/x-api/fundamentals/data-dictionary#user) 객체)에 대해 새로운 JSON 설계를 도입합니다.

- JSON 루트 레벨에서 표준 엔드포인트는 Post 객체를 statuses 배열로 반환하는 반면, X API v2는 data 배열로 반환합니다. 
- 리트윗 및 인용된 "statuses"를 참조하는 대신, X API v2 JSON은 리트윗 및 인용된 Tweets를 참조합니다. contributors 및 user.translator\_type과 같은 많은 레거시/사용 중단 필드는 제거됩니다. 
- Post 객체의 favorites와 user 객체의 favourites를 모두 사용하는 대신, X API v2는 like라는 용어를 사용합니다. 
- X는 값이 없는 JSON 값(예: null)은 페이로드에 기록하지 않는 규칙을 채택하고 있습니다. Post 및 user 속성은 null이 아닌 값이 있을 때에만 포함됩니다. 
   

또한 [Post 객체](/ko/x-api/fundamentals/data-dictionary#tweet)에 다음과 같은 새로운 필드 집합을 도입했습니다:

- [conversation\_id](/ko/x-api/fundamentals/conversation-id) 필드
- 새로운 [annotations](/ko/x-api/fundamentals/post-annotations) 필드 2개(context, entities 포함)
- 새로운 [metrics](/ko/x-api/fundamentals/metrics) 필드 다수
- 특정 게시물에 누가 답글을 달 수 있는지 보여주는 신규 reply\_setting 필드

**요청 매개변수**

X API v2에서는 다음 표준 필터드 스트림 요청 매개변수가 지원되지 않습니다:

| 표준 v1.1 매개변수 | 세부 정보 |
| :--- | :--- |
| locations - 경도,위도 쌍을 쉼표로 구분한 목록으로, 게시물을 필터링할 경계 상자 집합을 지정합니다. | X API v2에서는 위치 기반 연산자를 아직 제공하지 않습니다. |
| Delimited | v1.1 엔드포인트에서는 이를 문자열 길이로 설정하면 스트림에서 상태가 구분되어, 클라이언트가 상태 메시지 끝까지 읽어야 할 바이트 수를 알 수 있습니다.<br /><br />이 기능은 X API v2에서는 제공되지 않습니다. |
| Stall\_warnings | v1.1 엔드포인트에서는 이 매개변수를 true로 설정하면 클라이언트 연결이 끊길 위험이 있을 때 주기적으로 메시지가 전송됩니다. <br /><br />X API v2에서는 일정 간격으로 전송되는 개행과 함께 정체 경고가 기본적으로 전송됩니다. |

**복구 및 중복성 기능의 가용성**

X API v2의 필터드 스트림은 스트리밍 가동 시간을 극대화하고, 5분 이하의 연결 해제로 인해 놓쳤을 수 있는 게시물을 복구하는 데 도움이 되는 복구 및 중복성 기능을 도입합니다.

중복 연결을 사용하면 동일한 스트림에 최대 두 개의 연결을 설정할 수 있어, 한 연결에 문제가 생겨도 항상 스트림 연결을 유지하는 데 도움이 됩니다. 

backfill\_minutes 매개변수를 사용하면 최대 5분치의 누락 데이터를 복구할 수 있습니다. 

이 두 기능은 [Academic Research access](/ko/x-api/getting-started/about-x-api)를 통해서만 사용할 수 있습니다. 이 기능에 대한 자세한 내용은 [recovery and redundancy features](/ko/x-api/posts/filtered-stream#filtered-stream-recovery-and-redundancy-features) 통합 가이드를 참고하세요. 

**새 쿼리 연산자**

X API v2는 두 가지 새로운 기능을 지원하기 위해 새로운 연산자를 도입합니다: 

- **[Conversation IDs](/ko/x-api/fundamentals/conversation-id)** - X에서 대화가 전개되면, 대화에 속한 게시물을 식별하기 위한 conversation ID가 제공됩니다. 대화의 모든 게시물은 대화를 시작한 게시물의 ID로 conversation\_id가 설정됩니다. 
  - conversation\_id:
- **[X Annotations](/ko/x-api/fundamentals/post-annotations)** 는 게시물에 대한 문맥 정보를 제공하며, entity와 context 주석을 포함합니다. 엔터티는 사람, 장소, 제품, 조직으로 구성됩니다. 컨텍스트는 해당 엔터티가 속한 도메인 또는 주제입니다. 예를 들어, 게시물에서 언급된 사람은 운동선수, 배우, 정치인인지 등을 나타내는 컨텍스트를 가질 수 있습니다.  
  - context: - 관심 있는 컨텍스트로 주석이 달린 게시물과 일치합니다. 
  - entity: - 관심 있는 엔터티로 주석이 달린 게시물과 일치합니다.
