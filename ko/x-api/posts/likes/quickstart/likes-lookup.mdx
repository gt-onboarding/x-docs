<div id="getting-started-with-the-likes-lookup-endpoint">
  ### Likes 조회 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 [Postman](/ko/tutorials/postman-getting-started)을 사용해 Likes 조회 엔드포인트에 첫 요청을 보내는 방법을 안내합니다.

다양한 언어의 샘플 코드를 보려면 GitHub의 [X API v2 샘플 코드](https://github.com/xdevplatform/Twitter-API-v2-sample-code) 저장소를 방문하세요.
 

<div id="prerequisites">
  #### 준비 사항
</div>

이 가이드를 완료하려면 [keys and tokens](/ko/resources/fundamentals/authentication) 세트가 필요하며, 다음 단계에 따라 생성할 수 있습니다.

1. [개발자 계정 신청](https://developer.x.com/en/apply-for-access) 후 승인을 받습니다.
2. 개발자 포털에서 [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성합니다.
3. 앱의 “Keys and tokens” 페이지로 이동하여 API Keys, Access Tokens, Bearer Token을 비밀번호 관리자에 저장합니다.

<div id="steps-to-build-a-likes-lookup-request">
  #### Likes 조회 요청을 만드는 단계
</div>

**1단계: 도구 또는 라이브러리로 시작하기**

이 엔드포인트에 요청을 보내는 데 사용할 수 있는 다양한 도구, 코드 예제, 라이브러리가 있지만 여기서는 과정을 단순화하기 위해 Postman을 사용합니다.

X API v2 Postman 컬렉션을 환경에 로드하려면 다음 버튼을 클릭하세요:

[![](https://cdn.cms-twdigitalassets.com/content/dam/developer-twitter/m1_vnext/carat.svg)<br />Postman에 X API v2 추가](https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400)

Postman에서 X API v2 컬렉션을 로드한 후, "Likes" 폴더로 이동해 "Liking users"를 선택하세요. 
 

**2단계: 요청 인증하기**

이 엔드포인트에 성공적으로 요청하려면 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Bearer Token](/ko/resources/fundamentals/authentication#oauth-2-0) 인증을 사용해야 합니다. 이를 위해 "X API v2"라는 환경을 선택한 뒤, 아래 키와 토큰을 Postman에 추가하고 다음 변수를 Initial value 및 Current value 필드에 입력하세요:

- consumer\_key: API Key
- consumer\_secret: API Key Secret
- access\_token: Access Token
- token\_secret: Access Token Secret
   

**3단계: 트윗 지정하기**

이 엔드포인트에서는 좋아요한 사용자를 조회하려는 트윗의 ID를 지정해야 합니다. 트윗으로 이동한 뒤 URL 끝의 숫자 코드를 확인하면 트윗 ID를 찾을 수 있습니다. 예를 들어, 다음 URL의 트윗 ID는 1354143047324299264입니다.

https://x.com/TwitterDev/status/1354143047324299264

Postman에서 "Params" 탭으로 이동해 해당 섹션 하단의 tweet\_id 경로 변수 "Value" 열에 이 트윗 ID를 입력하고, 앞뒤에 공백이 없도록 하세요. 

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 좋아요한 사용자를 조회할 트윗의 ID |

**4단계: 가져올 필드를 식별하고 지정하기**

3단계 후 "Send" 버튼을 클릭하면 응답에서 기본 [user object](/ko/x-api/fundamentals/data-dictionary#user) 필드인 id, name, username을 받게 됩니다.

id, name, username 외에 추가 필드를 받으려면, 요청에 [fields](/ko/x-api/fundamentals/fields) 및/또는 [expansions](/ko/x-api/fundamentals/expansions) 매개변수로 해당 필드를 지정해야 합니다.

이 연습에서는 서로 다른 오브젝트에서 다음 세 가지 추가 필드 세트를 요청합니다:

1. 기본 사용자 오브젝트의 추가 user.created\_at 필드
2. 반환된 사용자에 연결된 고정 트윗 오브젝트의 기본 필드: id 및 text
3. 연결된 트윗 오브젝트의 추가 tweet.created\_at 필드

Postman에서 "Params" 탭으로 이동해 "Query Params" 표에 다음 key:value 쌍을 추가하세요:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| user.fields | created\_at | user.created\_at |
| expansions | pinned\_tweet\_id | tweet.id, tweet.text |
| tweet.fields | created\_at | tweet.created\_at |

이제 "Send" 버튼 옆에 다음 URL이 표시됩니다:

`https://api.x.com/2/tweets/1354143047324299264/liking_users?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=created_at`

**6단계: 사용자의 좋아요한 트윗 가져오기**

사용자가 좋아요한 트윗을 가져오는 요청을 보낼 수도 있습니다. Likes 조회 엔드포인트를 사용하면 사용자가 좋아요한 트윗에 대한 정보를 가져올 수 있습니다. 이를 위해 "Likes" 폴더로 이동해 "Liked Tweets"를 선택하세요.

이 엔드포인트에서는 좋아요한 사용자를 가져오려는 대상의 User ID를 지정해야 합니다. 이 정보는 [user lookup](/ko/x-api/users/lookup/introduction) 엔드포인트로 확인할 수 있습니다.

Postman에서 "Params" 탭으로 이동한 뒤, 섹션 하단의 id 경로 변수의 "Value" 열에 해당 사용자명을 입력하세요. 사용자명 앞뒤에 공백이 없도록 주의합니다.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| id  | 좋아요한 트윗을 가져올 대상의 User ID |
| max\_results | 5   |

이제 "Send" 버튼 옆에서 TwitterDev 대신 여러분의 ID가 들어간 비슷한 URL을 확인할 수 있습니다:

`https://api.x.com/2/users/2244994945/liked_tweets?max_results=5`
