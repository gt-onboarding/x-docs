<div id="getting-started-with-the-manage-likes-endpoints">
  ### 좋아요 관리 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 [Postman](/ko/tutorials/postman-getting-started)을 사용하여 좋아요 관리 엔드포인트에 첫 요청을 보내는 방법을 안내합니다.

다양한 언어의 샘플 코드를 확인하려면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요. 

<div id="prerequisites">
  #### 사전 준비 사항
</div>

이 가이드를 완료하려면 [keys and tokens](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따라 생성하세요:

1. [개발자 계정](https://developer.x.com/en/apply-for-access)을 신청하고 승인을 받습니다.

2. 개발자 포털에서 [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성합니다.

3. 앱의 “Keys and tokens” 페이지로 이동하여 API Keys, Access Tokens, Bearer Token을 비밀번호 관리자에 저장합니다.

<div id="steps-to-build-a-manage-likes-request">
  #### 좋아요 관리 요청을 구성하는 단계
</div>

**1단계: 도구 또는 라이브러리로 시작하기**

이 엔드포인트에 요청을 보내는 데 사용할 수 있는 다양한 도구, 코드 예제, 라이브러리가 있지만, 여기서는 과정을 단순화하기 위해 Postman을 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 로드하려면 다음 버튼을 클릭하세요:

[![](https://cdn.cms-twdigitalassets.com/content/dam/developer-twitter/m1_vnext/carat.svg)<br />X API v2를 Postman에 추가](https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400)

Postman에 X API v2 컬렉션을 로드한 후 “Likes” 폴더로 이동하여 “Like a Post”를 선택하세요.

**2단계: 요청 인증하기**

이 엔드포인트에 성공적으로 요청하려면 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication)를 사용해야 합니다. 이를 위해 “X API v2”라는 환경을 선택하고, 아래 키와 토큰을 초기값과 현재값 필드에 변수로 추가하세요:

- consumer\_key: API Key
- consumer\_secret: API Key Secret
- access\_token: Access Token
- token\_secret: Access Token Secret
   

**3단계: 좋아요를 누를 Post 지정하기**

좋아요 관리 엔드포인트에는 두 개의 ID가 필요합니다. 하나는 사용자(게시물에 좋아요를 누르려는 사용자)용이고, 다른 하나는 사용자가 좋아요 또는 좋아요 취소를 시도하는 Post ID입니다.

사용자 ID는 인증 사용자 ID와 일치해야 하므로, 요청을 인증할 때 해당 사용자 ID와 연결된 [Access Tokens](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)를 전달해야 합니다. 이 경우 자신의 사용자 ID를 지정할 수 있습니다. ID는 다음 두 가지 방법으로 확인할 수 있습니다:

1. [user lookup by username](/ko/x-api/users/user-lookup-by-id) 엔드포인트에서 사용자 이름을 전달하면 id 필드를 받을 수 있습니다.
2. Access Token의 숫자 부분이 사용자 ID입니다.
    

또한 좋아요를 누를 Post를 지정해야 합니다. x.com에서 Post를 클릭한 후 URL에서 Post ID를 확인할 수 있습니다. 예를 들어, 다음 URL의 Post ID는 1228393702244134912입니다.

https://x.com/TwitterDev/status/1228393702244134912

Postman에서 "Params" 탭으로 이동하여 id 경로 변수의 "Value" 열에 자신의 ID를 입력하세요. 이어서 “Body” 탭으로 이동해, 좋아요를 누를 Post의 ID를 tweet\_id 파라미터 값으로 입력하세요. 어떤 ID 앞뒤에도 공백이 없도록 주의하세요.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | (사용자 ID) |
| tweet\_id | (좋아요를 누를 Post의 ID) |

"Send" 버튼을 클릭하면 관계 상태를 포함하는 응답 객체를 받게 됩니다:

- "liked": true라면, 해당 id가 tweet\_id에 성공적으로 좋아요를 누른 것입니다.
   

**4단계: 요청 보내고 응답 검토하기**

모든 설정이 완료되면 "Send" 버튼을 눌러 다음과 같은 응답을 받습니다:

```
{
    "data": {
        "liked": true
    }
}
```

같은 Post의 좋아요를 취소하려면 Postman의 X API v2 컬렉션 내 “Likes” 폴더에 있는 “Unlike a Post” 요청을 사용하세요. id와 tweet\_id는 앞선 예와 동일하게 사용하면 됩니다. 좋아요를 취소할 때는 JSON 본문을 추가할 필요가 없으므로 id와 tweet\_id에 필요한 쿼리 파라미터를 추가했는지 확인하세요.
