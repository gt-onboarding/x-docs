<div id="likes-lookup">
  ### 좋아요 조회
</div>

좋아요 조회 그룹의 엔드포인트를 사용하면 특정 게시물을 좋아요한 계정 목록이나 특정 계정이 좋아요한 게시물 목록을 조회할 수 있습니다. 포함된 엔드포인트는 다음과 같습니다:

- 사용자가 좋아요한 게시물 - GET /2/users/:id/liked\_tweets
- 게시물을 좋아요한 사용자 - GET /2/tweets/:id/liking\_users

이 엔드포인트는 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 베어러 토큰](/ko/resources/fundamentals/authentication#oauth-2-0)으로 인증할 수 있습니다. 좋아요한 게시물 엔드포인트의 경우, 대용량 결과를 순회할 수 있도록 페이지네이션 토큰이 제공됩니다.

liking users 엔드포인트는 모든 기간을 통틀어 게시물당 좋아요한 계정을 총 100개로 제한합니다. 또한 좋아요한 게시물 엔드포인트는 프로젝트 수준에서 적용되는 월간 [트윗 한도(게시 한도)](/ko/x-api/fundamentals/post-cap)의 적용을 받습니다.
 

<div id="manage-likes">
  ### 좋아요 관리
</div>

좋아요 관리 엔드포인트를 사용하면 인증된 계정을 대신해 지정한 게시물에 좋아요를 하거나 취소할 수 있습니다. 이 엔드포인트 그룹에는 POST와 DELETE 두 가지 메서드가 있습니다. POST는 게시물에 좋아요를 추가하고, DELETE는 좋아요를 취소합니다.

사용자 대신 요청하므로 이 엔드포인트는 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication)로 인증해야 하며, [3-legged OAuth 플로우](/ko/resources/fundamentals/authentication#oauth-1-0a-2)/obtaining-user-access-tokens)로 생성한 해당 사용자의 액세스 토큰을 사용해야 합니다. 본인 계정 또는 인증된 사용자의 계정으로 게시물에 좋아요를 할 수 있습니다. 두 엔드포인트 모두 엔드포인트당 15분에 50회로 사용자 기준 레이트 리밋이 적용됩니다. 
 

이 엔드포인트에 액세스하려면 승인된 [개발자 계정](/ko/resources/fundamentals/developer-portal)이 있어야 합니다. 인증 시에는 [프로젝트](/ko/resources/fundamentals/projects) 내에 있는 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용해야 합니다. 

X API v2 엔드포인트에 대한 액세스 방법은 [시작하기](/ko/x-api/getting-started/about-x-api) 페이지에서 자세히 알아보세요.
