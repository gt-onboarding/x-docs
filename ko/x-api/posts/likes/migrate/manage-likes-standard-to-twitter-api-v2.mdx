<div id="manage-likes-standard-v11-compared-to-x-api-v2">
  ### 좋아요 관리: Standard v1.1과 X API v2 비교
</div>

Standard v1.1의 [POST favorites/create](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-create) 및 [POST favorites/destroy](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-destroy) 엔드포인트를 사용해 오셨다면, 이 가이드는 Standard v1.1과 X API v2의 좋아요 관리 엔드포인트 간 유사점과 차이점을 이해하는 데 도움이 되도록 작성되었습니다.

- **유사점**
  - OAuth 1.0a 사용자 컨텍스트
- **차이점**
  - 엔드포인트 URL 및 HTTP 메서드
  - 앱 및 프로젝트 요건
  - 요청 파라미터

<div id="similarities">
  #### 유사점
</div>

**OAuth 1.0a 사용자 컨텍스트 인증 방식**

두 엔드포인트 버전 모두 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication#oauth-1-0a-2)를 지원합니다. 따라서 이전에 표준 v1.1 즐겨찾기 관리 엔드포인트 중 하나를 사용해 왔다면, X API v2로 마이그레이션하더라도 동일한 인증 방식을 계속 사용할 수 있습니다.

<div id="differences">
  #### 차이점
</div>

**엔드포인트 URL과 HTTP 메서드**

- Standard v1.1 엔드포인트:
  - POST https://api.x.com/1.1/favorites/create.json
    (게시물 좋아요)
  - POST https://api.x.com/1.1/favorites/destroy.json
    (게시물 좋아요 취소)
- X API v2 엔드포인트:
  - POST https://api.x.com/2/tweets/:id/likes
    (게시물 좋아요)
  - DELETE https://api.x.com/2/tweets/:id/likes/:tweet\_id
    (게시물 좋아요 취소) 

**App 및 Project 요구사항**

X API v2 엔드포인트는 요청을 인증할 때 [developer App](/ko/resources/fundamentals/developer-apps)의 자격 증명을 사용해야 하며, 해당 App은 [Project](/ko/resources/fundamentals/projects)와 연결되어 있어야 합니다. 모든 X API v1.1 엔드포인트는 독립 실행형 App 또는 Project와 연결된 App의 자격 증명을 사용할 수 있습니다.

**요청 매개변수**

다음 Standard v1.1 요청 매개변수는 X API v2에서의 대응 항목입니다:

| Standard v1.1 | X API v2 |
| :--- | :--- |
| id  | id  |
| includes\_entities | 해당 없음 |

Standard v1.1 매개변수는 쿼리 매개변수로 전달되는 반면, X API v2 매개변수는 POST 엔드포인트에서는 본문 매개변수로, DELETE 엔드포인트에서는 경로 매개변수로 전달됩니다.

또한, Standard v1.1 엔드포인트를 사용할 때는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication)와 함께 전달되는 [Access Tokens](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)에 의해 어떤 사용자가 좋아요/좋아요 취소를 수행하는지가 추론되므로, 게시물에 좋아요를 표시한 사용자의 id는 필요하지 않습니다.
