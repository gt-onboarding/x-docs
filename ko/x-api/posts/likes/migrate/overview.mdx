<div id="comparing-x-apis-likes-endpoints">
  ## X API의 좋아요 엔드포인트 비교
</div>

이 가이드는 다음 내용을 다룹니다:

- **API 요청 매개변수** - X API v2 엔드포인트는 새로운 요청 매개변수 세트를 도입합니다. Labs를 사용해 본 분들께 익숙한 매개변수도 있겠지만, [fields](/ko/x-api/fundamentals/fields) 및 [expansions](/ko/x-api/fundamentals/expansions) 매개변수 도입 등 중요한 차이점이 많습니다.

- **앱 및 프로젝트 요구 사항** - X API v2에 액세스하려면 [프로젝트](/ko/resources/fundamentals/projects)에 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 자격 증명을 사용해야 합니다.

<div id="likes-lookup">
  #### 좋아요 조회
</div>

**게시물에 좋아요한 사용자**

좋아요한 사용자 엔드포인트는 v2의 신규 기능으로, 특정 게시물에 좋아요한 사용자 정보를 조회할 수 있습니다.

| 설명 | X API v2 |
| :--- | :--- |
| 지원 HTTP 메서드 | GET |
| 호스트 도메인 | https://api.x.com |
| 엔드포인트 경로 | /2/tweets/:id/liking\_users |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 2.0 Bearer Token<br /><br />OAuth 1.0a User Context |
| 기본 요청 [요율 제한](/ko/resources/fundamentals/rate-limits) | 15분당 75회(앱 기준)<br /><br />15분당 75회(사용자 기준) |

**사용자가 좋아요한 게시물**

다음 표는 표준 v1.1 [GET favorites/list](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-favorites-list) 엔드포인트와 X API v2의 좋아요한 게시물 엔드포인트를 비교합니다.

| 설명 | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| 지원 HTTP 메서드 | GET | GET |
| 호스트 도메인 | https://api.x.com | https://api.x.com |
| 엔드포인트 경로 | /1.1/favorites/list.json | /2/users/:id/liked\_tweets |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 2.0 Bearer Token<br /><br />OAuth 1.0a User Context |
| 기본 요청 [요율 제한](/ko/resources/fundamentals/rate-limits) | 15분당 75회 | 15분당 75회(앱 기준)<br /><br />15분당 75회(사용자 기준) |
| 데이터 형식 | Standard v1.1 형식 | [X API v2 형식](/ko/x-api/fundamentals/data-dictionary) (fields 및 expansions 요청 매개변수에 따라 결정되며 v1.1 형식과 하위 호환되지 않음)<br /><br />Standard v1.1 형식에서 X API v2 형식으로 마이그레이션하는 방법은 [데이터 형식 마이그레이션 가이드](/ko/x-api/migrate/data-format-migration)를 참조하세요. |

<div id="manage-likes">
  #### 좋아요 관리
</div>

v2 좋아요 관리 엔드포인트는 v1.1의 [POST favorites/create](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-create) 및 [POST favorites/destroy](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-destroy) 엔드포인트를 대체합니다.

다음 표는 표준 v1.1과 X API v2의 좋아요 관리 엔드포인트를 비교합니다.

<div id="like-a-post">
  #### 게시물에 좋아요 표시
</div>

| 설명 | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| 지원 HTTP 메서드 | POST | POST |
| 호스트 도메인 | https://api.x.com | https://api.x.com |
| 엔드포인트 경로 | /1.1/favorites/create.json | /2/users/:id/likes |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 1.0a 사용자 컨텍스트 | OAuth 1.0a 사용자 컨텍스트 |
| 기본 요청 [요율 제한](/ko/resources/fundamentals/rate-limits) | 24시간당 1,000회 요청(사용자당)<br /><br />24시간당 1,000회 요청(앱당) | 15분당 50회 요청(사용자당)<br /><br />24시간당 1,000회 요청(사용자당, DELETE와 공유) |

<div id="unlike-a-post">
  #### 게시물 좋아요 취소
</div>

| 설명 | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| 지원 HTTP 메서드 | POST | DELETE |
| 호스트 도메인 | https://api.x.com | https://api.x.com |
| 엔드포인트 경로 | /1.1/favorites/destroy.json | /2/users/:id/likes/:tweet\_id |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 1.0a 사용자 컨텍스트 | OAuth 1.0a 사용자 컨텍스트 |
| 기본 요청 [요율 제한](/ko/resources/fundamentals/rate-limits) | 24시간당 1000건(사용자 기준)<br /><br />24시간당 1000건(App 기준) | 15분당 50건(사용자 기준)<br /><br />24시간당 1000건(사용자 기준, POST와 공유) |
