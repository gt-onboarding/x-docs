<div id="standard-v11-timelines-to-x-api-v2-timelines">
  ## 표준 v1.1 타임라인에서 X API v2 타임라인으로
</div>

v1.1 타임라인 엔드포인트(statuses/user\_timeline 및 statuses/mentions\_timeline)를 사용해 오셨다면, 이 가이드는 표준 타임라인과 X API v2 타임라인 엔드포인트 간의 공통점과 차이점을 이해하여 현재 통합을 새 버전으로 마이그레이션하는 데 도움을 드리기 위한 것입니다.

- **유사점:**
  - 인증:
    - OAuth 1.0a 사용자 컨텍스트(역순 시간순 홈 타임라인, 사용자 게시 타임라인, 사용자 멘션 타임라인)
    - OAuth 2.0 앱 전용(사용자 게시 타임라인)
  - 과거 데이터 접근 한도: 사용자 타임라인(사용자 게시 타임라인)은 최신 3,200개의 게시물에 접근할 수 있으며, 멘션 타임라인(사용자 멘션 타임라인)은 최신 800개의 멘션에 접근할 수 있습니다.
  - 게시물 수정 이력 및 메타데이터 지원
  - 할당량 제한(사용자 게시 타임라인)
  - 새로 고침 폴링: since\_id 이후의 새 결과를 가져오는 기능
  - 게시물 ID로 타임라인 탐색
  - 결과 사양:
    - 결과 순서: 결과는 최신순(역순 시간순)으로 반환됨
    - 답글 제외 기능(사용자 게시 타임라인에 한함)
    - 리트윗 제외 기능(사용자 게시 타임라인에 한함)
- **차이점**
  - 새로운 인증 기능:
    - OAuth 2.0 앱 전용(사용자 멘션 타임라인)
    - OAuth 2.0 인증 코드 플로우 with PKCE(역순 시간순 홈 타임라인, 사용자 게시 타임라인, 사용자 멘션 타임라인)
  - 접근 요구사항: X API v2 앱 및 프로젝트 요구사항
  - 할당량 제한(사용자 멘션 타임라인 및 역순 시간순 홈 타임라인)
  - 서로 다른 요청 한도 및 처리량 한도
  - 추가 페이지네이션 방식
    - 응답당 서로 다른 max\_results(count)
  - 응답 데이터 형식
  - 요청 매개변수
    - v2 필드 및 확장을 포함한, 요청 매개변수 기반의 맞춤형 데이터 형식
    - 추가로 제공되는 데이터: 지표, 게시물 주석, 투표

<div id="similarities">
  ### 유사점
</div>

**인증**

v1.1 statuses/user\_timeline 및 X API v2 사용자 게시물 타임라인 엔드포인트는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication)와 [OAuth 2.0 App-Only](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)를 모두 지원합니다. 따라서 X API v2로 마이그레이션하더라도 동일한 인증 방식과 권한 토큰을 계속 사용할 수 있습니다.

**과거 데이터 액세스**

v1.1 statuses/user\_timeline 및 X API v2 사용자 게시물 타임라인 엔드포인트는 모두 최신 3,200개의 게시물(리트윗 포함)을 반환합니다.

v1.1 statuses/mentions\_timeline 및 X API v2 사용자 멘션 타임라인 엔드포인트는 최신 800개의 게시물을 반환할 수 있습니다.

**게시물 편집 이력 및 메타데이터 지원**

두 버전 모두 편집 이력을 설명하는 메타데이터를 제공합니다. 자세한 내용은 [필터드 스트림 API 참조](/ko/x-api/posts/filtered-stream#api-reference-index)와 [게시물 편집 기본 사항 페이지](/ko/x-api/fundamentals/edit-posts)를 확인하세요. 

**요청 한도**

|     |     |
| :--- | :--- |
| **Standard v1.1** | **X API v2** |
| user\_timeline:<br /><br />OAuth 1.0a User Context로 15분당 900회 요청<br /><br />OAuth 2.0 App-Only로 15분당 1,500회 요청 | 사용자 게시물 타임라인:<br /><br />OAuth 1.0a User Context로 15분 창 내 900회 요청<br /><br />OAuth 2.0 App-Only로 15분 창 내 1,500회 요청 |

**since\_id를 사용한 갱신 폴링**

두 버전 모두 since\_id로 최신 결과를 폴링할 수 있습니다.

**게시물 ID로 타임라인 탐색**

두 엔드포인트 모두 게시물 ID의 구성 방식에 따른 게시물 ID ‘타임스탬프’를 사용해 타임라인을 탐색할 수 있습니다. 기능은 전반적으로 동일하지만 다음과 같은 차이가 있습니다:

|     |     |
| :--- | :--- |
| **Standard timelines v1.1** | **timelines v2** |
| since\_id(배타적)<br /><br />max\_id(포함적) | since\_id(배타적)<br /><br />until\_id(역시 배타적, 포함적이었던 max\_id와 대비) |

**응답 필터링 매개변수**

|     |     |
| :--- | :--- |
| **Standard timelines v1.1** | **timelines v2** |
| 응답 필터링 매개변수:<br /><br />\* include\_rts<br />\* exclude\_replies | 응답 필터링 매개변수:<br /><br />\* exclude=retweets,replies |
| 예시 <br /><br />https://api.x.com/1.1/statuses/user\_timeline.json?user\_id=2244994945\&include\_rts=0&\&exclude\_replies=1 | 예시:<br /><br />https://api.x.com/2/users/2244994945/tweets?max\_results=100\&exclude=retweets,replies |
| 참고:<br /><br />user\_timeline의 경우:<br /><br />\* include\_rts=0을 사용해도 가능한 과거 게시물 한도인 최신 3,200개는 변하지 않습니다 | 참고:<br /><br />사용자 게시물 타임라인의 경우:<br /><br />\* exclude=retweets를 사용해도 가능한 과거 게시물 한도인 최신 3,200개는 변하지 않습니다 <br />\* exclude=replies를 사용하면 가능한 과거 게시물 한도가 최신 800개 답글로 줄어듭니다 |

<div id="differences">
  ### 차이점
</div>

**인증**

**v1.1 statuses/mentions\_timeline 엔드포인트는 [OAuth 1.0a User Context](https://developer-staging.x.com/resources/fundamentals/authentication)만 지원합니다. X API v2 사용자 멘션 타임라인 엔드포인트는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication), [OAuth 2.0 App-Only](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only), 그리고 [OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authenticationoauth-2-0/authorization-code "이 방법을 사용하면 승인된 앱이 사용자를 대신해, 사용자로서 동작할 수 있습니다. 일반적으로 특정 사용자의 공개 정보에 액세스하거나 게시하는 데 사용되며, 이 엔드포인트가 반환하는 내용과 사용자 간의 관계를 앱이 인지해야 할 때 유용합니다. OAuth 2.0 Authorization Code with PKCE로 인증하는 방법을 보려면 클릭하세요.")를 지원합니다.**

X API v2 사용자 포스트 타임라인 엔드포인트를 통해 비공개 또는 프로모션 지표에 액세스하려면 OAuth 1.0a User Context 또는 OAuth 2.0 Authorization Code with PKCE를 사용하고, 해당 지표에 액세스하려는 포스트를 게시한 사용자와 연결된 사용자 액세스 토큰을 전달해야 합니다.
 

**엔드포인트 URL**

X API v2 타임라인 엔드포인트는 사용자 ID에 대한 경로 매개변수 :id가 필요합니다.

- 표준 v1.1 엔드포인트:
  - https://api.x.com/1.1/statuses/home\_timeline
  - https://api.x.com/1.1/statuses/user\_timeline
  - https://api.x.com/1.1/statuses/mention\_timeline
- X API v2 엔드포인트:
  - https://api.x.com/2/users/:id/timelines/reverse\_chronological
  - https://api.x.com/2/users/:id/tweets
  - https://api.x.com/2/users/:id/mentions

**앱 및 프로젝트 요구사항**

X API v2 엔드포인트는 요청을 인증할 때 [developer App](/ko/resources/fundamentals/developer-apps)의 자격 증명을 사용해야 하며, 해당 앱은 [Project](/ko/resources/fundamentals/projects)에 연결되어 있어야 합니다. 모든 X API v1.1 엔드포인트는 단독 앱 또는 프로젝트에 연결된 앱의 자격 증명을 사용할 수 있습니다. 
 

**레이트 리밋**

|     |     |
| :--- | :--- |
| **mentions\_timeline:**<br /><br />OAuth 1.0a User Context로 15분당 75회 요청 | **user mention timeline:** <br /><br />OAuth 1.0a User Context로 15분당 180회 요청  <br />OAuth 2.0 Bearer Token으로 15분당 450회 요청 |
| **home\_timelime:**<br /><br />15분당 15회 요청  <br /><br />홈 타임라인에서 최대 800개의 포스트 조회 가능 | **reverse chronological home timeline:**<br /><br />15분당 180회 요청<br /><br />지난 7일 동안 타임라인에 생성된 모든 포스트와, 생성 날짜와 관계없이 가장 최근 800개까지 반환할 수 있습니다. |

**요청 한도**

|     |     |
| :--- | :--- |
| **Standard timelines v1.1** | **timelines v2** |
| 일일 요청 한도<br /><br />\* 24시간 동안 100,000회 요청 한도. | 일일 요청 한도 없음, 소비량에 의해서만 제한. |

**소비량 한도**

|     |     |
| :--- | :--- |
| **Standard timelines v1.1** | **timelines v2** |
| 요청 한도로만 소비 제한<br /><br />\* 24시간 동안 100,000회 요청 한도. | [access level](/ko/x-api/getting-started/about-x-api)에 따라 프로젝트 수준 월간 [Post cap](/ko/x-api/fundamentals/post-cap)(여러 v2 엔드포인트에 걸쳐 적용)으로만 소비 제한됨.<br /><br />\* Essential 액세스: 월 500,000개 포스트<br />\* Elevated 액세스: 월 2,000,000개 포스트<br />\* Academic Research 액세스: 월 10,000,000개 포스트<br /><br />역시간순 홈 타임라인은 월간 Post cap의 적용을 받지 않습니다. |

**요청 매개변수**

|     |     |
| :--- | :--- |
| **Standard timelines v1.1** | **timelines v2** |
| 필수: user\_id 또는 screen\_name | 필수: 특정 사용자 ID를 경로(path) 매개변수에 지정 |
| 선택 사항:<br /><br />count - 요청당 반환되는 최대 결과 수 설정<br /><br />exclude\_replies - 결과에서 답글 제거<br /><br />include\_rts - 0으로 설정하면 결과에서 리트윗 제거<br /><br />trim\_user - 결과에서 재수화된 사용자 객체 제거<br /><br />tweet\_mode - 반환 데이터 형식 설정, 140자를 초과하는 게시물의 경우 extended로 설정<br /><br />since\_id - 결과에 포함될 가장 이른 게시물 ID 설정(배타적)<br /><br />max\_id - 결과에 포함될 가장 최신 게시물 ID 설정(포함) | 선택 사항:<br /><br />max\_results - 요청당 반환되는 최대 결과 수 설정<br /><br />exclude=retweets,replies - 결과에서 리트윗 또는 답글 제거<br /><br />tweet.fields - 반환할 게시물(Post) 객체 필드 설정<br /><br />user.fields - 반환할 사용자(User) 객체 필드 설정<br /><br />place.fields - 반환할 장소(place) 객체 필드 설정<br /><br />media.fields - 반환할 미디어(media) 객체 필드 설정<br /><br />poll.fields - 반환할 투표(poll) 객체 필드 설정<br /><br />expansions - 확장할 필드와 반환할 연계 데이터 설정<br /><br />start\_time - 결과의 가장 이른 created\_at 시간 설정<br /><br />end\_time - 결과의 가장 최신 created\_at 시간 설정<br /><br />since\_id - 결과에 포함될 가장 이른 게시물 ID 설정(배타적)<br /><br />until\_id - 결과에 포함될 가장 최신 게시물 ID 설정(배타적) |

**X API v2 JSON 형식**

X API v2는 API가 반환하는 객체(예: [Post](/ko/x-api/fundamentals/data-dictionary#tweet) 및 [user](/ko/x-api/fundamentals/data-dictionary#user))에 대해 새로운 JSON 설계를 도입합니다. X API v2 형식과 필드/확장 사용 방법은 [가이드](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)와 더 폭넓은 [데이터 사전](/ko/x-api/fundamentals/data-dictionary)을 참고하세요.

- JSON 루트 수준에서 standard 엔드포인트는 statuses 배열로 게시물(Post) 객체를 반환하는 반면, X API v2는 data 배열을 반환합니다. 
- 리트윗 및 인용을 “statuses”로 지칭하는 대신, X API v2 JSON은 리트윗 및 인용 트윗으로 지칭합니다. contributors 및 user.translator\_type 같은 많은 레거시/사용 중단 필드는 제거됩니다. 
- 게시물(Post) 객체의 favorites와 사용자(User) 객체의 favorites를 모두 사용하는 대신, X API v2는 like라는 용어를 사용합니다. 
- X는 값이 없는 JSON 값(예: null)은 페이로드에 기록하지 않는 규칙을 채택합니다. 게시물과 사용자 속성은 값이 null이 아닐 때에만 포함됩니다.
   

standard v1.1과 X API v2 엔드포인트 버전 간 가장 큰 차이점 중 하나는 페이로드에 어떤 필드를 반환할지 선택하는 방식입니다. standard 엔드포인트에서는 여러 매개변수로 페이로드에 반환될 필드(또는 필드 집합)를 지정했지만, X API v2는 이러한 매개변수를 [fields](/ko/x-api/fundamentals/fields)와 [expansions](/ko/x-api/fundamentals/expansions)로 단순화합니다. 

- fields: X API v2 엔드포인트는 페이로드에 포함할 필드를 선택할 수 있도록 합니다. 예를 들어 Post, 사용자, 미디어(Media), 장소(Place), 설문조사(Poll) 객체에는 모두 반환 가능(또는 제외 가능)한 필드 목록이 있습니다.

- expansions: Post JSON 페이로드에서 참조되는 보조 객체를 확장하는 데 사용합니다. 예를 들어 모든 리트윗과 답글은 다른 Post를 참조합니다. expansions=referenced\_tweets.id를 설정하면 이러한 다른 Post 객체가 tweet.fields 설정에 따라 확장됩니다. 사용자, 설문조사, 미디어 등의 다른 객체도 확장할 수 있습니다.

- conversation\_id

- 두 가지 새로운 [annotations](/ko/x-api/fundamentals/post-annotations) 필드(context 및 entities 포함)

- 여러 가지 새로운 [metrics](/ko/x-api/fundamentals/metrics) 필드

표준 v1.1 필드를 최신 v2 필드에 매핑하는 데 도움이 되는 [데이터 형식 마이그레이션 가이드](/ko/x-api/migrate/data-format-migration#migrating-from-standard-v1-1s-data-format-to-v2)를 준비했습니다. 이 가이드는 특정 필드를 반환하기 위해 v2 요청에 포함해야 하는 expansion 및 field 매개변수도 제공합니다.
