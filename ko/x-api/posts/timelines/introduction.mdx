이 세 가지 타임라인 엔드포인트는 수정된 게시물을 지원합니다. 이 엔드포인트는 항상 최신 수정본과 수정 이력을 함께 반환합니다. 게시물이 30분 편집 가능 시간 이후에 수집되면 해당 게시물의 최종 버전을 나타냅니다. 편집 메타데이터에는 해당 게시물의 전체 이력에 대한 ID 배열이 포함됩니다. 편집 이력이 없는 게시물의 경우 이 배열에는 단일 ID만 포함됩니다. 편집된 게시물의 경우 이 배열에는 여러 ID가 포함되며, 편집 순서를 반영해 오름차순으로 정렬되고 배열의 마지막 요소가 최신 버전입니다. 게시물 편집 방식에 대해 자세히 알아보려면 [게시물 편집 기본 사항](/ko/x-api/fundamentals/edit-posts) 페이지를 참조하세요.

<div id="reverse-chronological-home-timeline">
  ## 최신순 홈 타임라인
</div>

이 엔드포인트를 사용하면 인증된 사용자와 그 사용자가 팔로우하는 계정이 게시한 최신 게시물, 리트윗, 답글을 조회할 수 있습니다.

사용자 대신 요청하므로, 이 엔드포인트는 [OAuth 2.0 Authorization Code Flow with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) 또는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication)로 인증해야 합니다. 이 엔드포인트의 사용자별 레이트 한도는 15분당 180회 요청입니다. 또한 최근 7일 동안 타임라인에 생성된 모든 게시물과 생성 날짜와 무관하게 가장 최근 800개 게시물을 반환할 수 있습니다.

<div id="user-post-timeline">
  ## 사용자 게시물 타임라인
</div>

사용자 게시물 타임라인 엔드포인트는 특정 X 계정이 게시한 게시물에 대한 액세스를 제공합니다. 사용자의 게시물을 조회하면 사용자 인터페이스에 타임라인을 표시하거나, 사용자의 게시물을 분석해 콘텐츠를 더 잘 이해하거나, 게시물과 프로그램적으로 상호작용하는 워크플로를 구축하는 등의 경험을 만들 수 있습니다. 이 엔드포인트는 사용자의 프로필 타임라인에서 볼 수 있는 것과 유사하게 단일 X 계정의 가장 최근 게시물, 리트윗, 답글, 인용 트윗에 대한 액세스를 제공합니다.

다음은 @XDevelopers의 사용자 타임라인입니다:

사용자 게시물 타임라인 엔드포인트는 원하는 사용자(사용자 ID 기준)를 지정하기 위한 단일 경로 매개변수를 받는 REST 엔드포인트입니다. 이 엔드포인트는 해당 사용자가 게시한 가장 최근의 게시물, 리트윗, 답글, 인용 트윗을 최대 3,200개까지 반환할 수 있습니다.

게시물은 가장 최근 것부터 역순으로 제공됩니다. 결과는 페이지당 최대 100개의 게시물까지 [페이지네이션](/ko/x-api/fundamentals/pagination)됩니다. 대량의 게시물을 탐색할 수 있도록 페이지네이션 토큰이 제공됩니다. 또한 해당 페이지에 포함된 최신 및 최하단(가장 오래된) 게시물의 게시물 ID가 메타데이터로 제공되며, 이는 최근 게시물을 폴링하는 데에도 사용할 수 있습니다. 사용자 게시물 타임라인은 특정 기간 내에 생성된 게시물을 받을 수 있도록 start\_time 및 end\_time 매개변수 지정도 지원합니다. 

사용자 게시물 타임라인 엔드포인트는 [fields](/ko/x-api/fundamentals/fields) 및 [expansions](/ko/x-api/fundamentals/expansions) 매개변수를 지원하며 [새 JSON 데이터 형식](/ko/x-api/fundamentals/data-dictionary)을 반환합니다.

이 엔드포인트에 요청을 성공적으로 보내려면 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication), [OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3), 또는 [OAuth 2.0 App-Only](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) 인증 방식을 사용해 요청에 권한을 부여해야 합니다. 비공개 지표, 프로모션 지표 또는 보호된 사용자의 타임라인을 요청하는 경우에는 OAuth 1.0a User Context 또는 OAuth 2.0 Authorization Code with PKCE를 사용해야 합니다. 

사용자 게시물 타임라인 엔드포인트는 두 가지 일반적인 사용 패턴을 지원하도록 설계되었습니다: 

- "사용자의 과거 게시물 가져오기": 특정 최근 기간 동안 관심 있는 사용자가 작성한 게시물을 시간순으로 받기 위해 사용자 게시물 타임라인에 요청합니다. 기간은 start\_time과 end\_time으로 설정하고 전체 결과를 페이지네이션하여 탐색할 수 있습니다. 경우에 따라 해당 사용자가 계정에서 작성한 게시물이 3,200개 이하인 경우 전체 게시물 이력을 가져올 수 있습니다. 포함되는 게시물은 공개 여부와 요청 시 사용한 인증에 따라 달라집니다.

- "새 게시물 폴링": 특정 사용자가 작성한 새 게시물을 가져오기 위해 사용자 게시물 타임라인에 지속적으로 요청합니다. 마지막으로 수신한 게시물 ID를 이후 요청의 매개변수로 설정해 마지막 게시물 이후의 새 게시물만 조회할 수 있습니다.

<div id="user-mention-timeline">
  ## 사용자 멘션 타임라인
</div>

사용자 멘션 타임라인 엔드포인트를 사용하면 특정 X 사용자를 언급한 게시물을 요청할 수 있습니다. 예를 들어, 어떤 X 계정이 게시물에서 @XDevelopers를 언급한 경우가 이에 해당합니다. 여기에는 요청한 사용자의 게시물에 대한 답글도 포함됩니다. 사용자의 멘션을 조회하면 누가 해당 사용자의 게시물에 답글을 달고 있는지, 언급하고 있는지를 빠르게 파악하거나, 해당 게시물과의 참여 워크플로를 프로그램적으로 구성하는 경험을 만들 수 있습니다. 이 엔드포인트를 사용하면 사용자의 [멘션 알림](https://x.com/notifications/mentions)에서 볼 수 있는 것과 유사하게 단일 사용자의 최신 멘션과 답글을 요청할 수 있습니다.

사용자 멘션 타임라인은 원하는 사용자(사용자 ID 기준)를 지정하기 위해 단일 경로 매개변수를 받는 REST 엔드포인트입니다. 이 엔드포인트는 해당 사용자의 최신 멘션 최대 800개를 반환할 수 있습니다.

게시물은 최신순으로 제공되며, 가장 최근 항목부터 시작됩니다. 결과는 페이지당 최대 100개의 게시물로 [페이지네이션](/ko/x-api/fundamentals/pagination)됩니다. 대량의 게시물을 페이지로 탐색할 수 있도록 페이지네이션 토큰이 제공됩니다. 지정된 페이지에 포함된 가장 최신 및 가장 오래된 게시물의 게시물 ID도 메타데이터로 제공되며, 이는 최신 게시물을 폴링하거나 v1.1의 [mentions\_timeline](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-mentions_timeline) 엔드포인트와 유사하게 타임라인을 탐색하는 데 사용할 수 있습니다. 또한 특정 시간 범위 내에 생성된 게시물을 받을 수 있도록 start\_time 및 end\_time 매개변수를 지정하는 기능도 지원합니다. 

이 엔드포인트로 요청을 성공적으로 보내려면 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication), [OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3), 또는 [OAuth 2.0 App-Only](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) 인증 방법으로 요청을 인증해야 합니다. 비공개 지표, 프로모션 지표 또는 보호된 사용자의 타임라인을 요청하는 경우에는 OAuth 1.0a User Context 또는 OAuth 2.0 Authorization Code with PKCE를 사용해야 합니다.

사용자 멘션 타임라인 엔드포인트는 [fields](/ko/x-api/fundamentals/fields) 및 [expansions](/ko/x-api/fundamentals/expansions) 매개변수를 지원하며 [새 JSON 데이터 형식](/ko/x-api/fundamentals/data-dictionary)을 반환합니다.
