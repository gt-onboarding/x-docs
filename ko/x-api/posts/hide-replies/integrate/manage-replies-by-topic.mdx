<div id="manage-replies-by-topic">
  ### 주제별로 답글 관리하기
</div>

답글 숨기기 엔드포인트를 통해 사람들과 브랜드가 대화를 주제에 맞게 유지하도록 돕는 통합을 구축할 수 있습니다. 이 페이지는 답글 숨기기와 [최근 검색](/ko/x-api/posts/recent-search) 엔드포인트를 사용하여 대화를 관리하는 방법을 보여줍니다.

최근 검색에는 대화와 그 답글을 조회하는 기능이 있으며, Post 페이로드는 언어와 관계없이 각 Post의 맥락과 주제를 이해하는 데 도움이 되는 [Post annotations](/ko/x-api/fundamentals/post-annotations)을 반환합니다.

앱 흐름에는 대화를 표시하고 관리하기 위한 컨트롤이 포함됩니다:

1. 사용자의 Post를 읽고 답글을 관리할 수 있도록 사용자에게 권한을 요청합니다.
2. Post URL에서 최근 대화를 가져와, 해당 대화가 인증된 사용자의 것인지 확인합니다.
3. 대화의 각 Post를 표시하기 위해 최근 검색 엔드포인트를 호출합니다. 요청에는 대화 ID 검색 쿼리와 annotation 확장이 포함되어, X의 해석에 따라 해당 Post가 스포츠 관련인지 여부를 판단합니다.
4. 사용자가 선택하면 답글 숨기기를 호출해 해당 답글을 숨깁니다. 또한 사용자가 항상 제어할 수 있도록 이 작업을 취소할 수 있는 방법도 제공합니다.
5. 더 긴 대화의 경우 [검색 결과 페이지네이션](/ko/x-api/posts/search/integrate/paginate)을 위한 컨트롤을 제공합니다.
    

<div id="optimize-for-the-user-and-for-usage">
  #### 사용자(및 사용) 중심으로 최적화
</div>

사용자가 의도한 모든 작업을 스스로 제어할 수 있도록 플로우를 설계하세요. 이 원칙을 염두에 두면 게시물 소비에 최적화된 통합을 구축하는 데에도 도움이 됩니다.

1. 인증된 사용자는 자신이 시작한 대화만 관리할 수 있으므로, 그렇지 않다면 플로우를 조기에 종료해야 합니다.
   - 먼저 게시물 조회 요청을 수행하세요. 게시물 URL이 유효하지 않거나 대화를 인증된 사용자가 시작하지 않았다면 플로우를 조기에 종료합니다.
   - 이렇게 하면 인증된 사용자가 해당 대화를 중재할 수 없는 경우, 앱이 최근 검색 요청을 수행하지 않아도 됩니다.
2. 별도의 요청을 줄이기 위해 동일한 요청에서 [사용자 및 게시물 필드](/ko/x-api/fundamentals/fields)를 함께 요청하세요. 이 접근 방식은 앱 성능 향상에도 도움이 됩니다.
3. 불필요한 요청을 피하세요. 이 앱은 사용자의 브라우저에 답글의 숨김 상태를 캐시합니다. 이는 대화 규모가 큰 경우 사용자가 나중에 중재를 이어갈 때 유용하며, 앱이 답글 숨김/숨김 해제 요청을 최적화하는 데도 도움이 됩니다.
