이 엔드포인트를 사용하면 정의한 기준에 따라 답글을 프로그래밍적으로 숨기거나 숨김 해제할 수 있습니다. 기본 X 환경과 마찬가지로 답글은 본문 대화에서는 숨겨지지만 별도의 페이지에서 계속 확인할 수 있습니다. 이 엔드포인트를 활용해 사용자가 가치가 없다고 판단하는 답글을 숨기는 데 도움을 주는 앱을 만들 수 있습니다.

hide replies 엔드포인트는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication#oauth-1-0a-2) 또는 [OAuth 2.0 Authorization Code with PKCE](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) 인증을 사용합니다. 성공 시 인증된 사용자가 시작한 게시물 대화에 게시된 단일 답글을 숨깁니다. 각 대화에서는 최대 725개의 게시물을 숨길 수 있습니다.

<Note>
  **계정 설정**

  이 엔드포인트에 액세스하려면 다음이 필요합니다:

  - 승인된 [developer account](https://developer.x.com/en/portal/petition/essential/basic-info).
  - [Project](/ko/resources/fundamentals/projects) 내 [developer App](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용한 인증. 

  X API v2 엔드포인트에 대한 액세스 권한 얻기에 대해 자세히 알아보려면 [시작 가이드](/ko/x-api/getting-started/getting-access)를 참조하세요.
</Note>
