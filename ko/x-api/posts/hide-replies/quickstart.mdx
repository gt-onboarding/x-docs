<div id="getting-started-with-the-hide-replies-endpoint">
  ## 답글 숨기기 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 [Postman](/ko/tutorials/postman-getting-started)을 사용해 답글 숨기기 엔드포인트에 첫 요청을 보내는 방법을 안내합니다.

여러 언어의 코드 스니펫을 보려면 [답글 숨기기 API 레퍼런스 페이지](/ko/x-api/posts/hide-replies#put-2-tweets-id-hidden)를 방문하세요.

<Note>
  ### 사전 준비 사항

  이 가이드를 완료하려면 요청 인증을 위한 [keys and tokens](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따라 이러한 키와 토큰을 생성할 수 있습니다:

  - [개발자 계정에 신청](https://developer.x.com/en/apply-for-access)하고 승인을 받습니다.
  - 개발자 포털에서 [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성합니다.
  - 앱의 “Keys and tokens” 페이지로 이동해 필요한 자격 증명을 생성합니다. 모든 자격 증명은 안전한 위치에 보관하세요.
</Note>

<div id="steps-to-build-a-put-tweetsidhidden-request">
  ### PUT /tweets/:id/hidden 요청을 생성하는 단계
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### 1단계: 도구 또는 라이브러리로 시작하기
</div>

이 엔드포인트에 요청을 보내는 데 사용할 수 있는 다양한 도구, 코드 예제, 라이브러리가 있지만, 여기서는 절차를 간소화하기 위해 Postman을 사용하겠습니다.

X API v2 Postman 컬렉션을 작업 환경에 로드하려면 다음 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Postman에 X API v2 추가
</Button>

Postman에 X API v2 컬렉션을 로드한 후 hide replies 엔드포인트로 이동하세요.

<div id="step-two-authenticate-your-request">
  #### 2단계: 요청 인증
</div>

X API에 정상적으로 요청하려면 권한을 보유하고 있는지 검증해야 합니다. 이 엔드포인트에서는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용해 요청을 인증해야 합니다.

이 예제에서는 OAuth 1.0a User Context를 사용합니다.

Postman에 키와 토큰—즉, API Key, API Secret Key, OAuth 1.0a user Access Token, OAuth 1.0a user Access Token Secret—을 추가해야 합니다. 이를 위해 Postman 우측 상단에서 “X API v2”라는 환경을 선택한 다음, 환경 드롭다운 옆 눈 아이콘을 클릭해 "initial value"와 "current value" 필드에 키와 토큰을 입력합니다.

정상적으로 설정했다면 이러한 변수는 요청의 Authorization 탭에 자동으로 반영됩니다.
 

<div id="step-three-find-a-post-id-to-hide">
  #### 3단계: 숨길 게시물 ID 찾기
</div>

hide replies 엔드포인트는 권한이 부여된 사용자를 대신해 답글을 숨기거나 숨김 해제할 수 있습니다. 이 예제에서는 사용자 프로필에 연결된 Access Token을 사용하므로, 사용자가 시작한 대화에 참여한 사용자의 답글을 숨길 수 있습니다. 마찬가지로 앱을 승인한 다른 사용자에게 속한 Access Token을 사용하는 경우, 해당 계정이 시작한 모든 대화의 답글을 관리할 수 있습니다.

지인에게 게시물에 답글을 달아 달라고 요청하세요(답글 숨김 기능을 테스트 중임을 알려주세요). 또는 테스트 계정으로 본인의 게시물에 답글을 남기세요. 그 답글을 클릭한 뒤 URL에서 숫자 부분만 복사합니다. 이것이 숨길 게시물 ID입니다.

이 경우 다음 게시물을 예로 들며, 해당 ID는 `1232720193182412800`입니다:

`https://x.com/TwitterDev/status/1232720193182412800`

<div id="step-four-hide-the-post">
  #### 4단계: 게시 숨기기
</div>

Postman에서 Hide replies 폴더를 열고 Hide a reply를 선택합니다. Params 탭에서 id 필드 옆에 게시 ID를 붙여넣습니다(URL의 :id를 바꿀 필요는 없습니다). "Send"를 클릭하면 성공 응답이 표시됩니다.

`{"hidden":true}`

<div id="step-five-unhide-the-post">
  #### 5단계: 게시물 숨김 해제
</div>

숨겨진 게시물은 X 앱의 별도 탭으로 이동됩니다. Postman에서 게시물의 숨김을 해제하려면 Hide replies 폴더를 열고 Unhide a reply를 선택하세요. Params 탭에서 이전 단계와 동일한 게시물 ID를 id 필드에 붙여넣습니다. "Send"를 클릭하면 성공 응답이 표시됩니다.

`{"hidden":false}`

hidden 필드는 게시물의 숨김 상태를 나타냅니다. 숨김 상태가 true이면 게시물이 숨겨진 것입니다. 반대로 false이면 게시물이 숨겨져 있지 않음을 의미합니다.
