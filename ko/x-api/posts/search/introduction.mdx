게시물 검색은 특정 주제나 이벤트와 관련된 X 대화를 표면화하는 데 사용되는 중요한 기능입니다. 이 기능은 X에도 있지만, 이 엔드포인트는 게시물을 필터링하고 수집하는 데 더 큰 유연성과 성능을 제공하여 연구에 필요한 데이터를 더 쉽게 찾고, 준실시간 ‘리스닝’ 애플리케이션을 구축하거나, 관심 주제와 관련된 게시물을 탐색·분석·활용할 수 있도록 합니다. 

게시물을 검색할 수 있는 두 가지 엔드포인트를 제공합니다: 최근 검색(Recent search)과 전체 아카이브 검색. 두 REST 엔드포인트는 공통 설계와 기능을 공유하며, 특정 주제를 중심으로 게시물을 필터링하기 위해 단일 검색 쿼리를 사용하는 점이 같습니다. 이러한 검색 쿼리는 메시지 키워드, 해시태그, URL 등 게시물 및 사용자 속성과 일치시키는 연산자 집합으로 구성됩니다. 연산자는 불리언 로직과 괄호로 결합하여 매칭 동작을 정교하게 다듬을 수 있습니다. 

최근 검색과 전체 아카이브 검색 엔드포인트는 모두 편집된 게시물 메타데이터를 제공합니다. 2022년 9월 29일 이후 생성된 모든 게시물 객체에는 해당 게시물이 한 번도 편집되지 않았더라도 게시물 편집 메타데이터가 포함됩니다. 게시물이 편집될 때마다 새로운 게시물 ID가 생성됩니다. 게시물의 편집 이력은 원본 ID부터 시작하는 게시물 ID 배열로 기록됩니다.

이 엔드포인트는 항상 가장 최신 편집본과 모든 편집 이력을 함께 반환합니다. 게시물의 30분 편집 가능 시간 이후에 수집된 게시물은 최종 버전을 나타냅니다. 게시물 편집 메타데이터에 대해 더 알아보려면 [Edit Posts 기본 사항](/ko/x-api/fundamentals/edit-posts) 페이지를 확인하세요.

쿼리를 설정하고 게시물을 수신하기 시작하면, 이 엔드포인트는 시간과 게시물 ID 범위를 기준으로 결과 탐색을 모두 지원합니다. 이는 두 가지 일반적인 사용 사례를 지원하기 위해 설계되었습니다: 

- **히스토리컬 가져오기**: 실시간성에 초점을 두지 않고 관심 기간을 대상으로 요청합니다. 단일 요청을 수행하고, 필요한 경우 페이지네이션으로 모든 일치 데이터를 전달합니다. 이는 Search Posts의 기본 모드입니다.
- **폴링** 또는 **리스닝**: “마지막 요청 이후 새 게시물이 있나요?” 모드로 지속적으로 요청합니다. 일반적으로 관심 게시물에 대한 준실시간 ‘리스닝’ 사용 사례에 초점을 둡니다.

여러 연산자와 쿼리 한도는 [Enterprise access](/ko/x-api/getting-started/about-x-api)에 한해 제공되므로, 추가 기능을 사용하려면 Enterprise access가 부여된 [프로젝트](/ko/resources/fundamentals/projects) 내 앱의 키와 토큰을 사용해야 합니다. 자세한 내용은 아래 엔드포인트 섹션에서 확인할 수 있습니다.

최근 검색과 전체 아카이브 검색 엔드포인트에서 반환된 게시물은 월별 [트윗 한도(게시 한도)](/ko/x-api/fundamentals/post-cap)에 포함됩니다.

**계정 설정**

이 엔드포인트에 액세스하려면 다음이 필요합니다:

- 승인된 [개발자 계정](https://developer.x.com/en/portal/petition/essential/basic-info)
- [프로젝트](/ko/resources/fundamentals/projects) 내 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용한 인증 

X API v2 엔드포인트 액세스 방법은 [시작하기 가이드](/ko/x-api/getting-started/getting-access)에서 자세히 알아보세요.

<div id="recent-search">
  ## 최근 검색
</div>

최근 검색 엔드포인트는 지난 1주일 내에 게시된 공개 게시물 중 필터링된 결과에 프로그래밍 방식으로 접근할 수 있도록 하며, 개발자 계정을 보유하고 [프로젝트](/ko/resources/fundamentals/projects) 내 [앱](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용하는 모든 개발자가 이용할 수 있습니다.

요청 인증은 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication), [OAuth 2.0 앱 전용](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), 또는 [OAuth 2.0 Authorization Code with PKCE](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)로 수행할 수 있습니다. 다만 개인용 지표나 게시물 결과에서 유기적 지표와 프로모션 지표의 구분된 분포를 받으려면, OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0 Authorization Code with PKCE를 사용하고 해당 콘텐츠를 게시한 사용자와 연결된 사용자 액세스 토큰을 전달해야 합니다.

이 엔드포인트는 요청당 최대 100개의 게시물을 최신순으로 반환하며, 일치하는 게시물이 많은 경우 페이징을 위한 [페이지네이션](/ko/x-api/posts/search/integrate/paginate) 토큰이 제공됩니다.

일반 액세스가 적용된 프로젝트에서는 기본 [연산자](/ko/x-api/posts/search/integrate/build-a-query) 세트를 사용할 수 있으며 최대 512자의 쿼리를 만들 수 있습니다. 엔터프라이즈 액세스가 적용된 프로젝트에서는 추가 연산자에 접근할 수 있으며, 최대 4096자의 쿼리를 만들 수 있습니다.

[액세스 수준](/ko/x-api/getting-started/about-x-api)에 대해 자세히 알아보세요.

<div id="full-archive-search">
  ## 전체 아카이브 검색
</div>

v2 전체 아카이브 검색 엔드포인트는 [Pro](/ko/x-api/getting-started/about-x-api "Pro") 액세스 또는 [Enterprise](/ko/x-api/getting-started/about-x-api) 액세스를 보유한 프로젝트에서만 사용할 수 있습니다. 이 엔드포인트는 검색 쿼리에 따라 2006년 3월 첫 게시물까지 포함한 전체 아카이브의 공개 게시물에 프로그램적으로 접근할 수 있도록 합니다.

이 엔드포인트에 대한 요청은 [OAuth 2.0 App-Only](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)로 인증할 수 있으며, [App Access Token](/ko/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)은 Pro 또는 Enterprise 액세스를 가진 프로젝트 내 앱에서 발급되어야 합니다. 또한 이 엔드포인트는 다른 사용자를 대신해 요청하는 방식(OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0 Authorization Code with PKCE)을 지원하지 않으므로, 비공개 [지표](/ko/x-api/fundamentals/metrics)는 조회할 수 없습니다.

이 엔드포인트는 요청당 최대 500개의 게시물을 최신순(내림차순)으로 반환하며, 대량의 일치 결과를 페이징할 수 있도록 [페이지네이션](/ko/x-api/posts/search/integrate/paginate) 토큰을 제공합니다.

**참고:** tweet.fields 매개변수로 [주석](/ko/x-api/fundamentals/post-annotations)을 요청하는 경우, 현재 max\_results 매개변수의 최대값은 100으로 제한됩니다. 추후 변경될 수 있으므로 이 제한을 유의하시기 바랍니다.

이 엔드포인트는 Pro 또는 Enterprise 액세스 승인을 받은 사용자에게만 제공되므로, 전체 검색 [연산자](/ko/x-api/posts/search/integrate/build-a-query) 세트를 사용할 수 있으며 최대 1024자 길이의 쿼리를 작성할 수 있습니다.
