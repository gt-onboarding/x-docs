<div id="enterprise-compared-to-x-api-v2">
  ### Enterprise와 X API v2 비교
</div>

**공통점**

- 페이지네이션
- 표준 시간대
- 게시물 편집 이력 및 메타데이터 지원

**차이점**

- 엔드포인트 URL
- 앱 및 프로젝트 요건
- 사용 가능한 기간
- 응답 데이터 포맷
- HTTP 메서드
- 요청 시간 포맷
- 요청 파라미터
- 필터링 연산자

<div id="similarities">
  #### 유사점
</div>

**페이지네이션**

v2에는 추가적인 페이지네이션 기능이 있습니다(`since_id`와 `until_id`로 게시물 ID를 사용해 탐색할 수 있는 새로운 페이지네이션 매개변수). 그러나 enterprise와 v2 모두 시간 기준(`fromDate`와 `toDate`는 enterprise, `start_time`과 `end_time`은 v2)으로 페이지네이션할 수 있습니다.
 

**시간대**

페이지네이션 섹션에서 언급했듯이, enterprise와 v2 모두 시간으로 서로 다른 데이터 페이지를 탐색할 수 있습니다. 두 경우 모두 이러한 매개변수를 사용할 때 시간대는 UTC를 사용합니다.

**게시물 편집 이력 및 메타데이터 지원**

두 버전 모두 편집 이력을 설명하는 메타데이터를 제공합니다. 자세한 내용은 [Search API 레퍼런스](/ko/x-api/posts/recent-search)와 [게시물 편집 기본 사항](/ko/x-api/fundamentals/edit-posts)을 참조하세요. 

<div id="differences">
  #### 차이점
</div>

**엔드포인트 URL**

- 엔터프라이즈 엔드포인트:
  - 30일 - `http://gnip-api.x.com/search/30day/accounts/:account_name/:label.json`
  - 전체 아카이브 - `http://gnip-api.x.com/search/fullarchive/accounts/:account_name/:label.json`
- X API v2 엔드포인트
  - 최근(7일) - `https://api.x.com/2/tweets/search/recent`
  - 전체 아카이브 - `https://api.x.com/2/tweets/search/all`

**앱 및 프로젝트 요건**

X API v2 엔드포인트는 요청 인증 시 프로젝트의 자격 증명을 사용해야 합니다. X API v1.1의 모든 엔드포인트는 독립형 앱 또는 프로젝트에 연결된 앱의 자격 증명을 사용할 수 있습니다.
 

**사용 가능한 기간**

엔터프라이즈 API와 X API v2 모두 전체 아카이브의 게시물에 대해 필터링된 게시물 데이터를 검색할 수 있는 엔드포인트를 제공합니다.

다만 X API v2에는 엔터프라이즈 API에 있는 30일 기간 엔드포인트가 없습니다. 대신 앞서 언급한 전체 아카이브 또는 7일 기간 엔드포인트를 제공하며, 이는 Native Enriched to v2 및 [Activity Streams to v2](/ko/x-api/migrate/data-format-migration#migrating-from-activity-streams-data-format-to-v2)와 정렬되어 엔터프라이즈 필드를 최신 v2 필드에 매핑하는 데 도움이 됩니다. 이 가이드는 또한 특정 필드를 반환하기 위해 v2 요청에 함께 전달해야 하는 구체적인 expansions 및 fields 매개변수를 제공합니다.
 

**응답 데이터 형식**

[엔터프라이즈 응답 형식](https://developer.x.com/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary)과 [X API v2 형식](/ko/x-api/fundamentals/data-dictionary)의 가장 큰 차이점 중 하나는 페이로드에 어떤 필드를 반환할지 선택하는 방식입니다.

엔터프라이즈 Search API에서는 많은 응답 필드가 기본으로 포함되며, 추가로 어떤 필드 또는 필드 집합을 페이로드에 반환할지 파라미터로 지정할 수 있습니다.

X API v2는 기본적으로 게시물 `id`와 `text` 필드만 반환합니다. 추가 필드나 객체를 요청하려면 [fields](/ko/x-api/fundamentals/fields) 및 [expansions](/ko/x-api/fundamentals/expansions) 파라미터를 사용해야 합니다. 이들 엔드포인트에서 요청한 게시물 필드는 기본 게시물 객체로 반환됩니다. 확장된 사용자, 미디어, 투표, 장소 객체와 필드는 응답의 `includes` 객체에 반환됩니다. 그런 다음 게시물과 확장 객체 모두에 있는 ID를 매칭하여 확장 객체를 게시물 객체에 연결할 수 있습니다.

이러한 새로운 파라미터에 대해서는 각 가이드나 [fields 및 expansions 사용 방법](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) 가이드를 참고하시기 바랍니다.

특정 필드를 요청하는 방식의 변경 외에도, X API v2는 [Post](/ko/x-api/fundamentals/data-dictionary#tweet) 및 [user](/ko/x-api/fundamentals/data-dictionary#user) 객체를 포함하여 API가 반환하는 객체에 대해 새로운 JSON 설계를 도입합니다.

- JSON 루트 레벨에서, 표준 엔드포인트는 게시물 객체를 `statuses` 배열로 반환하는 반면, X API v2는 `data` 배열로 반환합니다.
- 리트윗과 인용을 “statuses”로 지칭하는 대신, X API v2 JSON은 리트윗과 인용 트윗으로 지칭합니다. `contributors`, `user.translator_type` 같은 많은 레거시 및 폐기된 필드는 제거됩니다.
- 게시물 객체의 `favorites`와 사용자 객체의 `favourites`를 모두 사용하는 대신, X API v2는 like라는 용어를 사용합니다.
- X는 값이 없는 JSON 값(예: `null`)은 페이로드에 기록하지 않는 규칙을 채택합니다. 게시물 및 사용자 속성은 null이 아닌 값이 있을 때만 포함됩니다.
   

또한 다음과 같은 새 필드를 게시물 객체에 도입했습니다.

- [conversation\_id](/ko/x-api/fundamentals/conversation-id) 필드
- 컨텍스트와 엔터티를 포함하는 두 개의 새로운 [annotations](/ko/x-api/fundamentals/post-annotations) 필드
- 여러 개의 새로운 [metrics](/ko/x-api/fundamentals/metrics) 필드
- 특정 게시물에 누가 답글을 달 수 있는지를 표시하는 새로운 `reply_setting` 필드

마지막으로 한 가지 더. 프리미엄 응답에는 루트 레벨에 `requestParameters` 객체가 포함되어 있으며, 여기에 요청 시 포함한 파라미터가 담깁니다. v2 버전은 대신 루트 레벨에 `meta` 객체를 포함하며, 추가 결과 페이지가 있는 경우 `newest_id`, `oldest_id`, `result_count`, `next_token`을 포함합니다.

**HTTP 메서드**

API의 엔터프라이즈 버전은 JSON 본문을 포함한 POST HTTP 메서드 또는 쿼리 문자열을 사용하는 GET HTTP 메서드로 요청을 전달할 수 있습니다.

V2는 쿼리 문자열을 사용하는 GET HTTP 메서드만 지원합니다.
 

**요청 시간 형식**

이 엔드포인트의 엔터프라이즈 버전은 페이지네이션 매개변수와 `timePeriod` 응답 필드 모두에 다음 날짜/시간 형식을 사용합니다: `YYYYMMDDHHmm`

v2 엔드포인트는 페이지네이션 매개변수와 `start` 및 `end` 응답 필드 모두에 ISO 8601/RFC 3339 날짜/시간 형식을 사용합니다: `YYYY-MM-DDTHH:mm:ssZ
 `

**요청 매개변수**

다음은 엔터프라이즈와 X API v2의 요청 매개변수 표입니다:

| Enterprise | Search Posts v2 |
| :--- | :--- |
| query | query |
| maxResults | max\_results |
| fromDate (YYMMDDHHmm) | start\_time (YYYY-MM-DDTHH:mm:ssZ) |
| toDate (YYMMDDHHmm) | end\_time (YYYY-MM-DDTHH:mm:ssZ) |
|     | since\_id |
|     | until\_id |
| next | next\_token or pagination\_token |

**필터링 연산자**

엔터프라이즈와 X API v2의 연산자는 대부분 동일하지만, 연산자 제공 범위에 차이가 있으며 X API v2에만 도입된 새로운 연산자도 일부 있습니다.

X API v2, 엔터프라이즈, 그리고 프리미엄 및 스탠다드에서 사용 가능한 연산자의 전체 표는 [Search Posts 마이그레이션 랜딩 페이지](/ko/x-api/posts/search/migrate/overview)에서 확인하세요.
