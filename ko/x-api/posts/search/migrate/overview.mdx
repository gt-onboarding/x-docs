<div id="comparing-x-apis-search-posts-endpoints">
  ## X API의 게시물 검색 엔드포인트 비교
</div>

v2 Search Tweets 엔드포인트는 결국 [standard v1.1 search/posts](/ko/x-api/posts/search/introduction) 엔드포인트와 [enterprise Search API](/ko/x-api/enterprise-gnip-2.0/fundamentals/search-api)를 대체할 예정입니다. 기존 X 검색 엔드포인트를 사용하는 코드, 앱, 또는 도구가 있으며 최신 X API v2 엔드포인트로 마이그레이션을 고려 중이라면 이 가이드를 참고하세요.

이 페이지에는 세 가지 비교 표가 포함되어 있습니다:

- [최근 검색](#recent)
- [전체 아카이브 검색](#full-archive)
- [필터링 연산자 비교](#operators)

| **설명** | **Standard v1.1** | **X API v2** |
| :--- | :--- | :--- |
| 호스트 도메인 | https://api.x.com | https://api.x.com |
| 엔드포인트 경로 | /1.1/search/tweets.json | /2/tweets/search/recent |

| 타임스탬프 형식 | YYYYMMDD | YYYY-MM-DDTHH:mm:ssZ  <br />[ISO 8601 / RFC 3339](https://tools.ietf.org/html/rfc3339#section-5.6) |
| 반환되는 게시물이 이보다 오래되지 않음 | 7일 | 7일 |
| 지원되는 HTTP 메서드 | GET | GET |
| 기본 요청 속도 제한 | OAuth 1.0a 사용자 컨텍스트: 15분당 180회<br /><br />OAuth 2.0 앱 전용: 15분당 450회 | **Basic:**<br /><br />OAuth 2.0 앱 전용: 15분당 60회<br /><br />OAuth 1.0a 사용자 컨텍스트: 15분당 60회<br /><br />OAuth 2.0 Authorization Code with PKCE: 15분당 60회<br /><br />**Pro:**<br /><br />OAuth 2.0 앱 전용: 15분당 450회<br /><br />OAuth 1.0a 사용자 컨텍스트: 15분당 180회<br /><br />OAuth 2.0 Authorization Code with PKCE: 15분당 180회 |
| 완전히 전개된 URL 제공 |     | ✔   |
| 응답당 최대 게시물 수(기본값) | 100 (15) | 100 (10) |
| 게시물 JSON 형식 | Standard v1.1 형식 | [X API v2 형식](/ko/x-api/fundamentals/data-dictionary) (fields 및 expansions 요청 매개변수로 결정되며, v1.1 형식과 하위 호환되지 않음)<br /><br />Standard v1.1 형식에서 X API v2 형식으로 마이그레이션하는 방법은 [데이터 형식 마이그레이션 가이드](/ko/x-api/migrate/data-format-migration)를 참고하세요. |
| 페이로드에서 반환할 [fields](/ko/x-api/fundamentals/fields) 선택 지원 |     | ✔   |
| [annotations](/ko/x-api/fundamentals/post-annotations) 요청 및 수신 지원 |     | ✔   |
| 게시물 객체 내 특정 [지표](/ko/x-api/fundamentals/metrics) 요청 지원 |     | ✔   |
| [conversation\_id](/ko/x-api/fundamentals/conversation-id) 연산자 및 필드 지원 |     | ✔   |
| 게시물 편집 이력 제공 | ✔   | ✔   |
| 게시물 데이터 배열의 JSON 키 이름 | statuses | data |
| 페이지네이션의 JSON 키 이름 | search\_metadata.next\_results | meta.next\_token |
| 시간 범위별 아카이브 탐색 지원 | ✔   | ✔   |
| 시간 기반 요청의 시간 해상도 | day | second |
| 시간대 | UTC | UTC |
| 시간으로 탐색하기 위한 요청 매개변수 | until | start\_time  <br />end\_time |
| 게시물 ID로 탐색하기 위한 요청 매개변수 | since\_id   <br />max\_id | since\_id   <br />until\_id |
| 페이지네이션을 위한 요청 매개변수 | URL 인코딩된 쿼리 제공 | next\_token |
| [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [developer App](/ko/resources/fundamentals/developer-apps)의 자격 증명 사용 필요 |     | ✔   |

**필터링 연산자 비교**

검색 게시물의 버전(standard, enterprise, v2)별로 사용 가능한 연산자가 다르며, 각 버전 내에서도 연산자 제공 수준이 상이합니다. 아래에서 설명합니다. 

Enterprise

- Enterprise 연산자에는 하위 티어가 없습니다

X API v2

- **Free:** 모든 프로젝트에서 사용 가능
- **Basic:** 모든 프로젝트에서 사용 가능
- **Pro:** 프로젝트 사용 시 이용 가능
- **Enterprise:** 프로젝트 사용 시 이용 가능

각 연산자 집합에 대한 자세한 내용은 해당 가이드를 참조하세요:

- [Enterprise 연산자](/ko/x-api/enterprise-gnip-2.0/fundamentals/search-api#available-operators)
- [X API v2 연산자](/ko/x-api/posts/search/integrate/build-a-query#operators)
