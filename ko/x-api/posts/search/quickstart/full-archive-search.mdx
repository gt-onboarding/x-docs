<div id="getting-started-with-the-full-archive-search-endpoint">
  ## 전체 아카이브 검색 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 Postman을 사용해 지정한 필드 세트와 함께 전체 아카이브 검색 엔드포인트로 첫 요청을 보내는 방법을 안내합니다.

다양한 언어의 샘플 코드가 필요하다면 [X API v2 샘플 코드](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요. 

<Note>
  ### 사전 준비 사항

  전체 아카이브 검색 엔드포인트는 현재 Pro 및 Enterprise 액세스에서만 제공됩니다. 이 엔드포인트를 사용하려면 Pro 액세스로 업그레이드하거나 [Enterprise 액세스 등급을 신청](https://docs.google.com/forms/d/e/1FAIpQLScO3bczKWO2jFHyVZJUSEGfdyfFaqt2MvmOfl_aJp0KxMqtDA/viewform)해야 합니다.

  액세스 승인을 받는 것 외에도, 요청 인증을 위한 [키와 토큰](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따라 키와 토큰을 생성할 수 있습니다:

  - 개발자 포털에서 Enterprise 또는 Pro 액세스가 있는 [프로젝트](/ko/resources/fundamentals/projects)로 이동한 뒤, 해당 프로젝트에 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)이 있는지 확인하세요.

  - 앱의 “Keys and tokens” 페이지로 이동해 필요한 자격 증명을 생성하세요. 모든 자격 증명은 안전한 위치에 보관하세요.
</Note>

<div id="steps-to-build-a-full-archive-search-request">
  ### 전체 아카이브 검색 요청을 구성하는 단계
</div>

**1단계: 도구 또는 라이브러리 선택**

이 엔드포인트에 요청을 보내는 데 사용할 수 있는 다양한 도구, 코드 예제, 라이브러리가 있습니다. 여기서는 과정을 단순화하기 위해 Postman을 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 로드하려면 아래 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Postman에 X API v2 추가
</Button>

Postman에 X API v2 컬렉션을 로드한 뒤, Search Posts > Full-archive search request로 이동하세요.

**2단계: 요청 인증**

X API에 올바르게 요청하려면 권한 보유를 확인해야 합니다. 이 엔드포인트의 경우 [OAuth 2.0 App-Only](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) 인증 방식으로 요청을 인증해야 합니다.

키와 토큰, 특히 [App Access Token](/ko/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)(App-only Bearer Token이라고도 함)을 Postman에 추가해야 합니다. Postman 우측 상단에서 “X API v2” 환경을 선택한 뒤, 환경 드롭다운 옆 눈 아이콘을 클릭해 "initial value"와 "current value" 필드에 키와 토큰을 입력하세요.

올바르게 설정했다면 이 변수는 요청의 Authorization 탭에 자동으로 반영됩니다.
 

**3단계: 검색 쿼리 생성**

각 전체 아카이브 검색에는 단일 검색 쿼리가 필요합니다. 이 예시에서는 @XDevelopers 계정이 게시한 게시물에 매칭되는 쿼리를 사용하겠습니다. 이 쿼리에서는 from 연산자를 사용하고 값을 XDevelopers로 설정합니다(대소문자 무시):

`from:XDevelopers`

Postman에서 "Params" 탭으로 이동하여 이 값 또는 쉼표로 구분된 Post ID 문자열을 `ids` 매개변수의 "Value" 열에 입력하세요.
 

| Key | Value | Description |
| :--- | :--- | :--- |
| `query` | `from:XDevelopers` | 전체 아카이브 검색 엔드포인트에 제출할 검색 쿼리 |

4단계: 가져올 필드 식별 및 지정

3단계를 마친 후 "Send" 버튼을 클릭하면, 응답에는 기본 [Post object](/ko/x-api/fundamentals/data-dictionary#tweet) 필드인 `id`, `text`, `edit_history_tweet_ids`가 포함됩니다. 편집 기능 지원 이전의 게시물이 반환되는 경우 `edit_history_tweet_ids` 필드는 제공되지 않습니다. 이 필드에 대한 과거 데이터 백필은 수행되지 않았습니다. 

기본 필드 외 추가 필드를 받고자 한다면, 요청에서 [field](/ko/x-api/fundamentals/fields) 및/또는 [expansion](/ko/x-api/fundamentals/expansions) 매개변수로 해당 필드를 지정해야 합니다.

이 연습에서는 서로 다른 객체에서 네 가지 필드 집합을 요청하겠습니다:

1. 기본 Post 객체 필드
2. 게시물 객체의 추가 tweet.created\_at 필드
3. 반환된 게시물과 연관된 작성자의 [user object](/ko/x-api/fundamentals/data-dictionary#user) 기본 필드
4. 연관된 사용자 객체의 추가 user.description 필드
    

Postman에서 "Params" 탭으로 이동하여 "Query Params" 테이블에 다음 key:value 쌍을 추가하세요:

| Key | Value | Returned fields |
| :--- | :--- | :--- |
| `tweet.fields` | `created_at` | `tweets.created_at` |
| `expansions` | `author_id` | includes.users.id, includes.users.name, includes.users.username |
| user.fields | description | includes.users.description |

이제 "Send" 버튼 옆에 다음 URL이 표시되어야 합니다:

`https://api.x.com/2/tweets/search/all?query=from:XDevelopers&tweet.fields=created_at&expansions=author_id&user.fields=created_at`

<Note>
  **유의사항**

  기본적으로 가장 최근 게시물 10개만 반환됩니다. 요청당 10개 이상의 게시물을 받으려면 max\_results 매개변수를 사용하여 요청당 최대 500개까지 설정할 수 있습니다. 또한 기본적으로 지난 30일의 게시물만 반환됩니다. 30일보다 오래된 게시물을 가져오려면 API 호출에서 start\_time 및 end\_time 매개변수를 사용하세요.
</Note>
