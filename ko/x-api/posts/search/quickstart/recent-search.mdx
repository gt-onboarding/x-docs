<div id="getting-started-with-the-recent-search-endpoint">
  ## 최근 검색 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 Postman을 사용하여 지정된 필드 집합과 함께 최근 검색 엔드포인트에 첫 요청을 보내는 방법을 안내합니다.

다른 언어의 샘플 코드가 필요하다면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 저장소를 방문하세요.

<Note>
  ### **사전 준비 사항**

  이 가이드를 완료하려면 요청 인증에 필요한 [keys and tokens](/ko/resources/fundamentals/authentication) 세트가 있어야 합니다. 다음 단계를 따라 키와 토큰을 생성할 수 있습니다:

  - [개발자 계정에 신청](https://developer.x.com/en/apply-for-access)하고 승인을 받습니다.
  - 개발자 포털에서 [Project](/ko/resources/fundamentals/projects)와 이에 연결된 [developer App](/ko/resources/fundamentals/developer-apps)을 생성합니다.
  - App의 “Keys and tokens” 페이지로 이동해 필요한 자격 증명을 생성합니다. 모든 자격 증명은 안전한 위치에 보관하세요.
</Note>

<div id="steps-to-build-a-recent-search-request">
  ### 최신 검색 요청을 구성하는 단계
</div>

**1단계: 도구 또는 라이브러리 시작하기**

이 엔드포인트에 요청을 보내는 데 사용할 수 있는 다양한 도구, 코드 예제, 라이브러리가 있지만, 여기서는 과정을 단순화하기 위해 Postman을 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 로드하려면 다음 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Postman에 X API v2 추가
</Button>

Postman에 X API v2 컬렉션을 로드했으면, Search Posts > Recent search request로 이동하세요.

**2단계: 요청 인증**

X API에 올바르게 요청하려면 권한이 있는지 확인해야 합니다. 이를 위해 이 엔드포인트에서는 [OAuth 2.0 App-Only](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), [OAuth 2.0 Authorization Code with PKCE](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), 또는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 인증 방식 중 하나로 요청을 인증해야 합니다.

간단히 하기 위해 이 요청에는 OAuth 2.0 App-Only를 사용하겠지만, 비공개 [지표](/ko/x-api/fundamentals/metrics) 또는 게시물을 요청하려면 다른 인증 방식 중 하나를 사용해야 합니다.

OAuth 2.0 App-Only를 사용하려면 키와 토큰, 특히 [App Access Token](/ko/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)(App-only Bearer Token이라고도 함)을 Postman에 추가해야 합니다. Postman 오른쪽 상단에서 “X API v2”라는 이름의 환경을 선택하고, 환경 드롭다운 옆의 눈 아이콘을 클릭해 "initial value" 및 "current value" 필드에 키와 토큰을 추가하세요.

이 변수가 올바르게 설정되었다면 요청의 Authorization 탭에 자동으로 반영됩니다.

**3단계: 검색 쿼리 생성**

각 최신 검색에는 단일 [검색 쿼리](/ko/x-api/posts/search/integrate/build-a-query)가 필요합니다. 이 예시에서는 @XDevelopers 계정이 게시한 게시물과 매칭되는 쿼리를 사용하겠습니다. 이 쿼리에서는 from 연산자를 사용하고 값을 XDevelopers로 설정합니다(대소문자 구분 없음):

`from:XDevelopers`

Postman에서 "Params" 탭으로 이동하여 이 ID 또는 쉼표로 구분된 게시물 ID 문자열을 `ids` 파라미터의 "Value" 열에 입력하세요.

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Description** |
| `query` | from:XDevelopers | 최신 검색 엔드포인트에 제출할 검색 쿼리 |

**4단계: 조회할 필드 식별 및 지정**

3단계 후 "Send" 버튼을 클릭하면, 응답으로 기본 [Post object](/ko/x-api/fundamentals/data-dictionary#tweet) 필드인 id, text, `edit_history_tweet_ids`가 반환됩니다. id, text, `edit_history_tweet_ids` 외의 추가 필드를 받으려면, 요청에서 [field](/ko/x-api/fundamentals/fields) 및/또는 [expansion](/ko/x-api/fundamentals/expansions) 파라미터로 해당 필드를 지정해야 합니다.

이 연습에서는 서로 다른 객체에서 네 가지 필드 세트를 요청하겠습니다:

1. 기본 Post 객체 필드
2. 트윗의 추가 필드 tweet.created\_at
3. 반환된 게시물의 작성자 사용자 객체의 기본 필드
4. 관련 사용자 객체의 추가 필드 user.description

Postman에서 "Params" 탭으로 이동하여 "Query Params" 테이블에 다음 key:value 쌍을 추가하세요:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| `tweet.fields` | `created_at` | `tweets.created_at` |
| `expansions` | `author_id` | `includes.users.id`, `includes.users.name`, `includes.users.username` |
| `user.fields` | `description` | `includes.users.description` |

이제 "Send" 버튼 옆에 다음 URL이 표시되어야 합니다:

`https://api.x.com/2/tweets/search/recent?query=from:XDevelopers&tweet.fields=created_at&expansions=author_id&user.fields=created_at`
