<div id="retweets-lookup">
  ### 리트윗 조회
</div>

리트윗 조회 엔드포인트를 사용하면 특정 게시물을 리트윗한 계정 목록을 가져올 수 있습니다. 이 엔드포인트에서는 최대 100명의 사용자씩 결과를 페이지로 나눠 조회할 수 있도록 페이지네이션 토큰이 제공됩니다.

이 엔드포인트는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication), \[OAuth 2.0 App-Only]\(https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) 또는 [OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3)로 인증할 수 있습니다. 

<div id="retweets-of-me">
  ### 나에 대한 리포스트
</div>

‘나에 대한 리포스트’ 조회를 통해 다른 사용자가 공유한 본인의 게시물을 확인할 수 있습니다.

이는 사용자가 자신의 게시물 중 어떤 것이 공유되고 청중에게 반응을 얻고 있는지 파악하는 데 도움을 주어, 보다 정교한 상호작용과 콘텐츠 전략 수정을 촉진합니다.

이 엔드포인트는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)로 인증할 수 있습니다.

<div id="manage-retweets">
  #### 리트윗 관리
</div>

리트윗 관리 엔드포인트는 인증된 계정을 대신해 지정된 게시물을 리트윗하거나 리트윗을 취소할 수 있게 합니다. 이 엔드포인트 그룹에는 POST와 DELETE 두 가지 메서드가 있습니다. POST 메서드는 게시물을 리트윗할 수 있게 하고, DELETE 메서드는 해당 게시물의 리트윗을 취소할 수 있게 합니다.

사용자를 대신해 요청하므로 이 엔드포인트는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)로 인증해야 하며, 요청 대상 사용자의 액세스 토큰을 사용해야 합니다. 이 사용자 액세스 토큰은 [3-legged OAuth flow](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) 또는 [Authorization Code with PKCE grant flow](/ko/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0)를 통해 생성할 수 있습니다. 본인 계정 또는 인증된 사용자의 계정에서 게시물을 리트윗할 수 있습니다. 두 엔드포인트 모두 사용자 기준 요청 한도는 엔드포인트별 15분당 50회입니다.
