<div id="getting-started-with-the-retweets-lookup-endpoint">
  ### 리트윗 조회 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 [Postman](/ko/tutorials/postman-getting-started)을 사용해 리트윗 조회 엔드포인트로 첫 요청을 보내는 방법을 안내합니다. 다양한 언어의 샘플 코드를 보려면 [X API v2 sample code](https://github.com/xdevplatformTwitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요.

<div id="prerequisites">
  #### 사전 준비 사항
</div>

이 가이드를 완료하려면 요청을 인증하기 위한 [키와 토큰](/ko/resources/fundamentals/authentication)이 필요합니다. 다음 단계를 따라 키와 토큰을 생성할 수 있습니다.

- [개발자 계정에 신청](https://developer.x.com/en/apply-for-access)하고 승인을 받습니다.
- 개발자 포털에서 [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성합니다.
- 앱의 “Keys and tokens” 페이지로 이동하여 필요한 자격 증명을 생성합니다. 모든 자격 증명은 안전한 위치에 보관하세요.

<div id="steps-to-build-a-retweets-lookup-request">
  #### 리트윗 조회 요청을 만드는 단계
</div>

**1단계: 도구나 라이브러리부터 시작하기**

이 엔드포인트에 요청을 보내는 데 사용할 수 있는 도구, 코드 예제, 라이브러리가 여러 가지 있지만, 여기서는 과정을 간소화하기 위해 Postman을 사용하겠습니다.

환경에 X API v2 Postman 컬렉션을 불러오려면 다음 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Postman에 X API v2 추가하기
</Button>

Postman에서 X API v2 컬렉션을 로드한 뒤 "Retweets" 폴더로 이동해 "Retweeted by"를 선택하세요.

**2단계: 요청 인증하기**

X API에 올바르게 요청하려면 권한이 있는지 확인해야 합니다. 이를 위해 이 엔드포인트는 [OAuth 2.0 App-Only](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), [OAuth 2.0 Authorization Code with PKCE](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), 또는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 인증 방식으로 요청에 대한 인증을 수행할 것을 요구합니다.

단순화를 위해 이 요청에는 OAuth 2.0 앱 전용 방식을 사용하겠습니다. 다만 비공개 [지표](/ko/x-api/fundamentals/metrics) 또는 리트윗을 요청하려면 다른 인증 방법을 사용해야 합니다.

OAuth 2.0 App-Only를 사용하려면 키와 토큰, 특히 Postman에 [App Access Token](/ko/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token) (App-only Bearer Token이라고도 함)을 추가해야 합니다. 이를 위해 Postman 우측 상단에서 “X API v2” 환경을 선택한 뒤, 환경 드롭다운 옆의 눈 모양 아이콘을 클릭하여 표시되는 “initial value”와 “current value” 필드에 키와 토큰을 추가하면 됩니다.

올바르게 설정했다면 이러한 변수는 요청의 인증 탭에 자동으로 채워집니다.

**3단계: 게시물 지정**

이 엔드포인트를 사용할 때는 리트윗한 사용자를 조회하려는 게시물 ID를 지정해야 합니다. X에서 해당 게시물로 이동한 뒤 URL 끝의 숫자 코드를 확인하면 게시물 ID를 찾을 수 있습니다. 예를 들어, 다음 URL의 게시물 ID는 1354143047324299264입니다.

https://x.com/TwitterDev/status/1354143047324299264

Postman에서 "Params" 탭으로 이동한 뒤, 해당 섹션의 하단에 있는 id 경로 변수의 "Value" 열에 이 사용자 이름을 입력하세요. 사용자 이름 앞뒤에 공백이 들어가지 않도록 주의하세요.

|     |     |
| :--- | :--- |
| **키** | **값** |
| `id` | 리트윗한 사용자를 가져오려는 게시물 ID |

**4단계: 가져올 필드를 확인하고 지정하기**

3단계가 끝난 뒤 "Send" 버튼을 클릭하면 응답에 기본 [user object](/ko/x-api/fundamentals/data-dictionary#user) 필드인 id, name, username이 포함됩니다.

id, name, username 외의 필드를 추가로 받으려면 요청에서 [fields](/ko/x-api/fundamentals/fields) 및/또는 [expansions](/ko/x-api/fundamentals/expansions) 매개변수로 해당 필드를 지정해야 합니다.

이번 실습에서는 서로 다른 객체에서 추가로 세 가지 필드 세트를 요청하겠습니다:

1. 기본 사용자 객체에 추가된 user.created\_at 필드.
2. 반환된 사용자에 대해 연관된 고정된 Posts 객체의 기본 필드는 id와 text입니다.
3. 연관된 게시물 객체에 추가된 tweet.created\_at 필드.

Postman에서 "Params" 탭으로 이동한 다음 "Query Params" 표에 다음 키:값 쌍을 추가하세요:

|     |     |     |
| :--- | :--- | :--- |
| **키** | **값** | **반환 필드** |
| user.fields | created\_at | user.created\_at |
| expansions | pinned\_tweet\_id | tweet.id, tweet.text |
| tweet.fields | created\_at | tweet.created\_at |

이제 "Send" 버튼 옆에 다음 URL이 표시됩니다:

`https://api.x.com/2/tweets/1354143047324299264/retweeted_by?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=created_at`
