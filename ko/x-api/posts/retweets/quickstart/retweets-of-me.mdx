내 리포스트 조회 기능을 사용하면 다른 사람들이 공유한 내 게시물을 확인할 수 있습니다.

이를 통해 사용자는 어떤 게시물이 공유되어 대상에게 반응을 얻고 있는지에 대한 인사이트를 얻을 수 있으며, 보다 정교한 상호작용과 콘텐츠 전략 조정에 도움이 됩니다.

<div id="authentication">
  #### 인증
</div>

사용자 대신 요청을 보내는 경우, 해당 엔드포인트는 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)로 인증해야 하며, 요청 대상 사용자와 연결된 사용자 액세스 토큰을 사용해야 합니다. 이 사용자 액세스 토큰은 [3-legged OAuth 플로우](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens) (OAuth 1.0a) 또는 [Authorization Code with PKCE 그랜트 플로우](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) (OAuth 2.0)를 사용해 생성할 수 있습니다.

<div id="making-a-request">
  #### 요청 보내기
</div>

사용자가 앱에서 인증을 완료하면, 아래와 같이 해당 사용자를 대신해 Retweets of Me API를 호출할 수 있습니다:

```bash
      curl 'https://api.x.com/2/users/reposts_of_me' --header 'Authorization: ••••••'
```
