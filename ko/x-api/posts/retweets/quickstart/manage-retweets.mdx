<div id="getting-started-with-the-manage-retweets-endpoints">
  ### 리트윗 관리 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 Postman을 사용해 리트윗 관리 엔드포인트에 첫 요청을 보내는 방법을 안내합니다.

다양한 언어의 샘플 코드를 확인하려면 [X API v2 샘플 코드](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요. 

<div id="prerequisites">
  #### 사전 준비 사항
</div>

이 가이드를 완료하려면 요청 인증에 필요한 [키와 토큰](/ko/resources/fundamentals/authentication) 세트가 있어야 합니다. 다음 단계를 따라 키와 토큰을 생성하세요.

- [개발자 계정에 신청](https://developer.x.com/en/apply-for-access)하고 승인을 받습니다.
- 개발자 포털에서 [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성합니다.
- 앱의 “Keys and tokens” 페이지로 이동하여 필요한 자격 증명을 생성합니다. 모든 자격 증명은 안전한 위치에 보관하세요.

<div id="steps-to-build-a-manage-retweets-request">
  #### 리트윗 관리 요청을 구성하는 단계
</div>

**1단계: 도구 또는 라이브러리로 시작하기**

이 엔드포인트에 요청을 보내기 위해 사용할 수 있는 다양한 도구, 코드 예제, 라이브러리가 있습니다. 여기서는 과정을 단순화하기 위해 Postman을 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 로드하려면 다음 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Add X API v2 to Postman
</Button>

Postman에 X API v2 컬렉션을 로드한 후 “Retweets” 폴더로 이동하여 “Retweet a Post”를 선택하세요.
 

2단계: 요청 인증하기

X API에 올바르게 요청하려면 권한이 있는지 확인해야 합니다. 이 엔드포인트의 경우 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)로 요청을 인증해야 합니다.

이 예제에서는 OAuth 1.0a User Context를 사용하겠습니다.

API Key, API Secret Key, OAuth 1.0a user Access Token, OAuth 1.0a user Access Token Secret 등 키와 토큰을 Postman에 추가해야 합니다. Postman 오른쪽 상단에서 “X API v2” 환경을 선택한 뒤, 환경 드롭다운 옆 눈 아이콘을 클릭해 "initial value"와 "current value" 필드에 키와 토큰을 입력하세요.

올바르게 설정했다면 이 변수들은 요청의 Authorization 탭에 자동으로 반영됩니다.

**3단계: 리트윗할 포스트 지정하기**

리트윗 관리 엔드포인트에는 두 개의 ID가 필요합니다. 리트윗하거나 리트윗을 취소하려는 사용자 ID와 포스트의 ID입니다.

사용자 ID는 인증하는 사용자 ID와 같아야 합니다. 즉, 요청을 인증할 때 해당 사용자 ID에 연결된 [Access Tokens](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)를 전달해야 합니다. 이 경우 본인의 사용자 ID를 지정할 수 있습니다. 사용자 ID는 다음 두 가지 방법으로 확인할 수 있습니다:

1. [user lookup by username](/ko/x-api/users/user-lookup-by-username) 엔드포인트에 username을 전달하면 id 필드를 받을 수 있습니다. 
2. Access Token을 보면 숫자 부분이 사용자 ID입니다.
    

또한 리트윗할 포스트를 지정해야 합니다. X.com에서 포스트를 클릭한 뒤 URL에서 포스트 ID를 확인할 수 있습니다. 예를 들어, 다음 URL의 포스트 ID는 1228393702244134912입니다.

https://x.com/TwitterDev/status/1228393702244134912

Postman에서 "Params" 탭으로 이동하여 id 경로 변수의 "Value" 열에 자신의 ID를 입력합니다. 그리고 리트윗하려는 포스트의 ID를 tweet\_id 매개변수의 값으로 입력합니다. 어떤 ID 앞뒤에도 공백이 없도록 주의하세요.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | (your user ID) |
| tweet\_id | (the ID of the Tweet you want to Retweet) |

"Send" 버튼을 클릭하면 관계 상태를 포함한 응답 객체를 받게 됩니다:

- "retweeted": true가 표시되면 해당 id가 tweet\_id를 성공적으로 리트윗한 것입니다.
   

**4단계: 요청 보내고 응답 확인하기**

모든 설정이 완료되면 "Send" 버튼을 누르면 다음과 같은 응답을 받게 됩니다:

```
{
    "data": {
        "retweeted": true
    }
}
```

"retweeted": true가 반환되면 `tweet_id`를 성공적으로 리트윗한 것입니다.

같은 사용자의 리트윗을 취소하려면 “Undo a Retweet”라는 요청을 사용할 수 있으며, 이는 Postman에 로드된 X API v2 컬렉션의 “Retweets” 폴더에도 있습니다. `id`는 사용자 ID여야 하며 `source_tweet_id`는 리트윗을 취소할 포스트 ID여야 합니다. 이를 JSON 본문으로 추가할 필요는 없으므로 id와 `source_tweet_id`에 필요한 쿼리 파라미터를 추가했는지 확인하세요. 

리트윗을 성공적으로 취소하면 다음 예시와 유사한 응답을 받게 됩니다:

```
{
  "data": {
    "retweeted": false
  }
}
```
