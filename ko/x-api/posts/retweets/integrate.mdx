<div id="authentication">
  #### 인증
</div>

모든 X API v2 엔드포인트는 키와 토큰이라고도 하는 자격 증명을 사용해 요청을 인증해야 합니다.

Retweets 조회 엔드포인트에 대한 요청은 OAuth 1.0a 사용자 컨텍스트 또는 OAuth 2.0 베어러 토큰 중 하나로 인증할 수 있습니다.

Retweets 관리 엔드포인트는 OAuth 1.0a 사용자 컨텍스트가 필요합니다. 즉, 요청을 성공적으로 처리하려면 API 키와 사용자 액세스 토큰 세트를 사용해야 합니다. 액세스 토큰은 요청을 대신하는 해당 사용자와 연결되어 있어야 합니다. 다른 사용자의 액세스 토큰 세트를 생성하려면, 해당 사용자가 3-legged OAuth 플로우를 통해 귀하의 앱을 승인(또는 인증)해야 합니다.

OAuth 1.0a는 사용이 까다로울 수 있습니다. 이 인증 방식에 익숙하지 않다면, 요청을 올바르게 인증하기 위해 [라이브러리](/ko/x-api/tools-and-libraries/overview) 사용을 권장합니다.

**유의 사항**

다음 필드를 요청하는 경우 OAuth 1.0a 사용자 컨텍스트가 필요합니다:

- tweet.fields.non\_public\_metrics
- tweet.fields.promoted\_metrics
- tweet.fields.organic\_metrics
- media.fields.non\_public\_metrics
- media.fields.promoted\_metrics
- media.fields.organic\_metrics

<div id="developer-portal-projects-and-developer-apps">
  #### 개발자 포털, 프로젝트와 개발자 앱
</div>

X API v2 엔드포인트에서 사용할 인증 자격 증명 세트를 받으려면 [개발자 계정에 가입](https://developer.x.com/en/portal/petition/essential/basic-info)하고 해당 계정 내에 [프로젝트](/ko/resources/fundamentals/projects)를 설정한 다음, 그 프로젝트 내에 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 만들어야 합니다. 이후 개발자 앱에서 키와 토큰을 확인할 수 있습니다. 
 

<div id="rate-limits">
  #### 요청 한도
</div>

매일 수천 명의 개발자가 X API에 요청을 보냅니다. 이러한 대량의 요청을 관리하기 위해, 각 엔드포인트에는 [요청 한도](/ko/x-api/fundamentals/rate-limits)가 적용되어 앱 또는 인증된 사용자를 대신해 보낼 수 있는 요청 수가 제한됩니다.

리트윗 관리 엔드포인트는 15분당 사용자당 50건으로 제한됩니다. 또한 POST 엔드포인트의 경우 사용자·앱별 3시간 기준 300건으로 제한됩니다.

리트윗 조회 엔드포인트는 15분 기준 75건으로 제한됩니다. 또한 이 엔드포인트는 가장 최근에 리트윗한 사용자 100명만 반환합니다.

<div id="fields-and-expansions">
  #### 필드와 확장
</div>

X API v2는 ‘필드(fields)’와 ‘확장(expansions)’ 도구 세트를 통해 사용자가 API에서 반환받을 데이터를 정확히 선택할 수 있도록 합니다. expansion 매개변수는 페이로드에서 참조되는 객체를 확장해 반환하도록 합니다. 예를 들어, 이 엔드포인트는 다음 [확장](/ko/x-api/fundamentals/expansions)을 가져올 수 있습니다:

- attachments.poll\_ids
- attachments.media\_keys
- author\_id, entities.mentions.username
- geo.place\_id
- in\_reply\_to\_user\_id,
- referenced\_tweets.id,
- referenced\_tweets.id.author\_id

fields 매개변수는 서로 다른 데이터 객체 내에서 수신할 [필드](/ko/x-api/fundamentals/fields)를 정확히 선택할 수 있게 합니다. 이 엔드포인트들은 주로 Post 객체를 반환합니다. 기본적으로 Post 객체는 id와 text 필드를 반환합니다. tweet.created\_at 또는 tweet.entities와 같은 추가 필드를 받으려면 fields 매개변수로 해당 필드를 명시적으로 요청해야 합니다. 통합 시 고려할 만한 중요한 필드로는 투표 데이터, 지표, Post 주석, 대화 ID 등이 있습니다.

[X API v2 데이터 사전](/ko/x-api/fundamentals/data-dictionary)에 [필드와 확장을 함께 사용하는 방법](/ko/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions)에 대한 가이드를 추가했습니다.
