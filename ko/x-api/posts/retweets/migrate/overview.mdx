<div id="comparing-x-apis-retweets-endpoints">
  ## X API의 리트윗 엔드포인트 비교
</div>

**리트윗 조회**

v2 리트윗 조회 엔드포인트는 기존 [v1.1 GET statuses/retweets/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweets-id) 및 [v1.1 GET statuses/retweets/:ids](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweeters-ids) 엔드포인트를 대체합니다.

다음 표는 Standard v1.1과 X API v2 리트윗 엔드포인트를 비교합니다:

| 설명 | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| 지원 HTTP 메서드 | GET | GET |
| 호스트 도메인 | https://api.x.com | https://api.x.com |
| 엔드포인트 경로 | /1.1/retweeters/id.json<br /><br />`/1.1/retweets/ids.json` | /2/users/:id/retweeted\_by |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 1.0a 사용자 컨텍스트 | OAuth 2.0 베어러 토큰<br /><br />OAuth 1.0a 사용자 컨텍스트 |
| 기본 요청 [요율 제한](/ko/resources/fundamentals/rate-limits) | 15분당 75회 | 15분당 75회(앱당)<br /><br />15분당 75회(사용자당) |
| 데이터 형식 | Standard v1.1 형식 | [X API v2 형식](/ko/x-api/fundamentals/data-dictionary) (fields 및 expansions 요청 매개변수로 결정되며 v1.1 형식과 하위 호환되지 않음)<br /><br />Standard v1.1 형식에서 X API v2 형식으로 마이그레이션하는 방법은 [데이터 형식 마이그레이션 가이드](/ko/x-api/migrate/data-format-migration)를 참조하세요. |
| [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 자격 증명 필요 |     | ✔️  |

<div id="manage-retweets">
  ### 리트윗 관리
</div>

다음 표는 표준 v1.1과 X API v2의 리트윗 취소 엔드포인트를 비교합니다:

**게시물 리트윗**

| 설명 | 표준 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 지원 HTTP 메서드 | POST | POST |
| 호스트 도메인 | https://api.x.com | https://api.x.com |
| 엔드포인트 경로 | /1.1/statuses/retweet/:id.json | /2/users/:id/retweets |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 1.0a 사용자 컨텍스트 | OAuth 1.0a 사용자 컨텍스트 |
| 기본 요청 [속도 제한](/ko/resources/fundamentals/rate-limits) | 없음<br /><br />3시간당 300건 요청(사용자·앱 기준). POST Tweet 엔드포인트와 공유됩니다 | 15분당 50건 요청(사용자 기준)<br /><br />3시간당 300건 요청(사용자·앱 기준). 게시물 관리를 위한 POST Tweet 엔드포인트와 공유됩니다. |
| [개발자 앱](/ko/resources/fundamentals/developer-apps)의 자격 증명 사용이 필요하며 해당 앱은 [프로젝트](/ko/resources/fundamentals/projects)와 연결되어 있어야 함 |     | ✔️  |

<div id="undo-a-retweet">
  #### 리트윗 취소
</div>

다음 표는 표준 v1.1과 X API v2의 리트윗 취소 엔드포인트를 비교합니다:

| 설명 | 표준 v1.1 | X API v2 |
| :--- | :--- | :--- |
| 지원 HTTP 메서드 | POST | DELETE |
| 호스트 도메인 | https://api.x.com | https://api.x.com |
| 엔드포인트 경로 | /1.1/statuses/unretweet/:id.json | /2/users/:id/retweets/:source\_tweet\_id |
| [인증](/ko/resources/fundamentals/authentication) | OAuth 1.0a 사용자 컨텍스트 | OAuth 1.0a 사용자 컨텍스트 |
| 기본 요청 [레이트 리밋](/ko/resources/fundamentals/rate-limits) | 없음 | 15분당 50건(사용자당) |
| [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 자격 증명 필요 |     | ✔️  |
