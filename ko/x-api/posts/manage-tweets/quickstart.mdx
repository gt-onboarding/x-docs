## Manage Posts 엔드포인트 시작하기\*\*

이 빠른 시작 가이드는 [Postman](/ko/tutorials/postman-getting-started)을 사용하여 Manage Posts 엔드포인트에 첫 요청을 보내는 방법을 안내합니다.

다양한 언어의 샘플 코드를 확인하려면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 리포지토리를 방문하세요.

<Info>
  **사전 준비 사항**

  이 가이드를 완료하려면 요청 인증을 위한 [keys and tokens](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따라 이러한 키와 토큰을 생성할 수 있습니다.

  - [개발자 계정 신청](https://developer.x.com/en/apply-for-access) 후 승인을 받습니다.
  - 개발자 포털에서 [프로젝트](/ko/resources/fundamentals/projects)와 연결된 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성합니다.
  - 앱의 “Keys and tokens” 페이지로 이동해 필요한 자격 증명을 생성합니다. 모든 자격 증명은 안전한 위치에 보관하세요.
</Info>

<div id="steps-to-building-manage-posts-requests">
  ### Manage Posts 요청을 구성하는 단계
</div>

**1단계: 도구 또는 라이브러리로 시작하기**

이 엔드포인트에 요청을 보내는 방법으로 여러 도구, 코드 예제, 라이브러리를 사용할 수 있지만, 여기서는 과정을 단순화하기 위해 Postman을 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 로드하려면 다음 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Postman에 X API v2 추가
</Button>

Postman에 X API v2 컬렉션을 로드한 후 “Manage Posts” 폴더로 이동하여 “Create a Post”를 선택하세요.
 

**2단계: 요청 인증하기**

X API에 정상적으로 요청하려면 권한을 확인해야 합니다. Manage Posts 엔드포인트에서는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용해 요청을 인증해야 합니다.

이 예에서는 OAuth 1.0a User Context를 사용합니다.

Postman에 키와 토큰(특히 API Key, API Secret Key, OAuth 1.0a 사용자 Access Token, OAuth 1.0a 사용자 Access Token Secret)을 추가해야 합니다. 이를 위해 Postman 오른쪽 상단의 “X API v2” 환경을 선택한 뒤, 환경 드롭다운 옆 눈 아이콘을 클릭하여 "initial value"와 "current value" 필드에 키와 토큰을 입력하세요.

올바르게 설정했다면, 이 변수들은 요청의 Authorization 탭에 자동으로 반영됩니다.

**3단계: 게시물 텍스트 지정하기**

이 엔드포인트로 새 게시물을 생성할 때, 게시물의 텍스트 또는 미디어는 필수 본문 파라미터입니다.

Postman에서 “Body” 탭으로 이동해 `text` 파라미터의 값으로 게시물 텍스트를 입력하세요. 추가로, 투표, 특정 게시물 ID에 대한 답글, 답글 설정 등의 파라미터도 여기에서 추가할 수 있습니다. 사용 가능한 항목은 [API 참조 가이드](/ko/x-api/posts/manage-tweets/introduction)에서 확인하세요.

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Parameter type** |
| `text` | Hello world! | body |

**4단계: 가져올 필드 지정하기**

모든 설정을 완료했다면 "Send" 버튼을 클릭하세요. 그러면 다음 예시와 유사한 응답을 받게 됩니다:

```
{
  "data": {
    "id": "1445880548472328192",
    "text": "Hello world!"
  }
}
```

응답 객체에 `id`와 게시물의 `text`가 포함되어 있다면 게시물이 성공적으로 생성된 것입니다.
 

**5단계: 게시물 삭제하기**

게시물을 삭제하려면, Postman에 로드된 X API v2 컬렉션의 “Manage Posts” 폴더에서 “Delete a Post” 요청을 선택하세요. 이 엔드포인트에는 삭제할 게시물의 ID가 필요합니다. 그런 다음 “Params” 탭에서 삭제할 게시물의 ID를 `id` 열의 값으로 입력하세요. 

삭제 요청이 성공하면 다음 예시와 유사한 응답을 받게 됩니다:

```
{
   "data": {
       "deleted" : true
   }
}
```
