  ## 유용한 도구

<div id="postman">
  ### Postman
</div>

Postman은 엔드포인트를 시험해 볼 수 있는 뛰어난 도구입니다. 각 Postman 요청에는 어떤 기능을 사용할 수 있는지 빠르게 파악할 수 있도록 모든 경로 및 본문 매개변수가 포함되어 있습니다. Postman 컬렉션에 대해 자세히 알아보려면 ["Postman 사용"](/ko/tutorials/postman-getting-started) 페이지를 방문하세요.

<div id="authentication">
  ### 인증
</div>

모든 X API v2 엔드포인트는 키와 토큰으로도 불리는 자격 증명을 사용해 요청을 인증해야 합니다.

해당 엔드포인트는 [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication)를 사용해야 합니다. 즉, 성공적인 요청을 위해 API 키와 사용자 액세스 토큰 세트를 사용해야 합니다. 액세스 토큰은 요청을 위임받은 해당 사용자와 연동되어 있어야 합니다. 다른 사용자의 액세스 토큰 세트를 생성하려면, 그 사용자가 [3-legged OAuth 플로우](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)를 통해 귀하의 앱을 승인(또는 인증)해야 합니다.

OAuth 1.0a는 사용이 까다로울 수 있습니다. 이 인증 방식에 익숙하지 않다면 [라이브러리](/ko/x-api/tools-and-libraries/overview)를 사용하거나 Postman과 같은 도구를 사용하고, 또는 OAuth 2.0으로 요청을 인증할 것을 권장합니다.

[PKCE를 사용하는 OAuth 2.0 권한 부여 코드](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)는 애플리케이션의 범위(scope)에 대한 더 세밀한 제어와 여러 기기에 걸친 권한 부여 플로우를 제공합니다. OAuth 2.0을 사용하면 사용자 대신 특정 권한을 부여하는 세밀한 스코프를 선택할 수 있습니다.

앱에서 OAuth 2.0을 사용하려면 개발자 포털의 앱 설정 섹션에 있는 인증 설정에서 이를 활성화해야 합니다.

<div id="developer-portal-projects-and-developer-apps">
  ### 개발자 포털, 프로젝트, 개발자 앱
</div>

X API v2 엔드포인트에서 사용할 인증 자격 증명 세트를 받으려면 \[개발자 계정]/resources/fundamentals/developer-portal), 해당 계정 내에 [프로젝트](/ko/resources/fundamentals/projects)를 설정하고 그 프로젝트에 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성해야 합니다. 그런 다음 개발자 앱에서 키와 토큰을 확인할 수 있습니다. 
 

<div id="rate-limits">
  ### 사용 한도
</div>

매일 수천 명의 개발자가 X API에 요청을 보냅니다. 이러한 대규모 트래픽을 관리하기 위해 [사용 한도](https://developer.x.com/x-api/fundamentals/rate-limits)가 각 엔드포인트에 적용되며, 이는 앱 또는 인증된 사용자를 대신해 보낼 수 있는 요청 수를 제한합니다.

이 엔드포인트들은 사용자 수준에서 한도가 적용됩니다. 즉, 요청을 위임받은 인증된 사용자는 어떤 개발자 앱을 통해서든 해당 엔드포인트를 호출할 수 있는 횟수가 정해져 있습니다. POST 메서드의 사용자 사용 한도는 15분당 200건입니다. DELETE 메서드의 사용 한도는 15분당 50건입니다. 또한 manage Posts 또는 manage Retweets 권한으로 생성된 게시물을 포함하여 [3시간당 300건의 요청](https://blog.x.com/developer/en_us/topics/tools/2018/new-developer-requirements-to-protect-our-platform) 한도가 있습니다.
 

<div id="source-labels">
  ### 소스 레이블
</div>

앱 이름과 웹사이트 URL은 애플리케이션이 프로그래밍 방식으로 생성한 모든 게시물의 메타데이터에 [소스 레이블](https://help.x.com/en/using-twitter/how-to-tweet#source-labels)로 표시됩니다. X 앱의 사용 사례가 변경되면 [개발자 약관](https://developer.x.com/content/developer-twitter/en/developer-terms/agreement-and-policy) 준수를 위해 이 설정에서 사용 사례를 반드시 업데이트하세요.
 

<div id="profile-settings">
  ### 프로필 설정
</div>

프로필 설정에서 위치 정보(geo)를 활성화한 경우에만 게시물에 위치를 추가할 수 있습니다. 위치 정보가 비활성화되어 있어도 요청 본문에 위치 매개변수를 포함할 수는 있지만, 게시물에는 적용되지 않습니다. 이미지에서 사용자 태그 지정도 마찬가지입니다. 태그하려는 사용자가 사진 태그 지정을 활성화하지 않았다면, 게시물이 성공적으로 생성되더라도 태그된 사용자 목록에 해당 사용자의 이름은 표시되지 않습니다. 
 

<div id="adding-media-to-a-post">
  ### 게시물에 미디어 추가
</div>

현재 X API v2만으로는 미디어를 온전히 업로드할 수 없습니다. 다만, 이전에 업로드한 미디어를 게시물에 첨부할 수 있습니다. 이미 [v1 미디어 엔드포인트](https://developer.x.com/en/docs/twitter-api/v1/media/upload-media/api-reference/post-media-upload)나 [X Media Studio](https://media.x.com/en/articles/products/2018/media-studio)를 통해 업로드된 미디어 ID를 사용할 수 있습니다. 해당 미디어 ID는 본인의 것이거나 인증된 사용자의 것이어야 합니다.
