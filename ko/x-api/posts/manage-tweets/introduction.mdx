X API를 사용해 게시물을 생성·삭제하는 것은 공개 대화에 참여하는 데 필수적입니다. 새로운 게시물 관리 엔드포인트로 이러한 작업은 물론 더 다양한 기능을 수행할 수 있습니다.

게시물 관리를 위한 메서드는 POST와 DELETE 두 가지가 제공됩니다. POST 메서드는 설문 게시, 트윗 인용, 답글 설정 지정, 위치 정보 포함 게시, 미디어 첨부 및 사용자 태그, Super Followers 대상 게시 등 다양한 기능을 제공합니다. DELETE 메서드는 특정 게시물을 삭제할 수 있습니다. POST 메서드에서는 요청을 더 세밀하게 구성할 수 있도록 원하는 [매개변수](/ko/x-api/posts/creation-of-a-post)를 전달할 수 있습니다.

‘게시물 삭제’ 메서드는 편집된 게시물을 지원하도록 업데이트되었습니다. 게시물 편집 체인에 속한 게시물 중 하나를 삭제하면 해당 편집 체인의 모든 게시물이 함께 삭제됩니다. 게시물 편집 메타데이터에 대해 더 알아보려면 [게시물 편집 기본 사항](/ko/x-api/fundamentals/edit-posts) 페이지를 확인하세요.

POST 메서드는 사용자 기준 15분당 200건의 레이트 한도가 있습니다. DELETE 메서드는 15분당 50건의 레이트 한도가 있습니다. 추가로, 게시물 관리 또는 리트윗 관리로 생성된 게시물을 포함하여 3시간당 300건의 요청 한도가 적용됩니다.

게시물 관리 엔드포인트로 사용자를 대신해 요청을 수행하는 경우, [OAuth 1.0a User Context](https://en.docs/authentication/oauth-1-0a) 또는 [OAuth 2.0 Authorization Code with PKCE](https://en.docs/authentication/oauth-2-0/authorization-code)로 인증하고, 귀하의 앱을 승인한 사용자에 연결된 사용자 액세스 토큰을 사용해야 합니다. OAuth 1.0a로 사용자 액세스 토큰을 생성하려면 [3-legged OAuth flow](https://en.docs/authentication/oauth-2-0/bearer-tokens)를 사용할 수 있습니다. OAuth 2.0으로 사용자 액세스 토큰을 생성하려면 [Authorization Code with PKCE grant flow](https://en.docs/authentication/oauth-2-0/user-access-token)를 사용할 수 있습니다.

<Info>
  **계정 설정**

  이 엔드포인트에 액세스하려면 다음이 필요합니다:

  - 승인된 [개발자 계정](https://en/portal/petition/essential/basic-info)
  - [프로젝트](/ko/resources/fundamentals/projects) 내에 있는 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용한 인증

  X API v2 엔드포인트에 대한 액세스 방법은 [시작 가이드](/ko/x-api/getting-started/getting-access)에서 자세히 알아보세요.
</Info>
