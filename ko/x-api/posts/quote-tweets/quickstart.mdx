<div id="getting-started-with-the-quote-tweets-lookup-endpoint">
  ## 인용 트윗 조회 엔드포인트 시작하기
</div>

이 빠른 시작 가이드는 [Postman](/ko/tutorials/postman-getting-started)을 사용하여 지정한 필드 세트와 함께 인용 트윗 조회 엔드포인트에 첫 요청을 보내는 방법을 안내합니다.

다양한 언어의 샘플 코드를 확인하려면 [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) GitHub 저장소를 방문하세요. 

<div id="prerequisites">
  #### 사전 준비 사항
</div>

이 가이드를 완료하려면 요청을 인증할 [키와 토큰](/ko/resources/fundamentals/authentication) 세트가 필요합니다. 다음 단계를 따라 키와 토큰을 생성하세요:

- [개발자 계정을 신청](https://developer.x.com/en/apply-for-access)하고 승인을 받습니다.
- 개발자 포털에서 [프로젝트](/ko/resources/fundamentals/projects)와 연동된 [개발자 앱](/ko/resources/fundamentals/developer-apps)을 생성합니다.
- 앱의 “Keys and tokens” 페이지로 이동하여 필요한 자격 증명을 생성합니다. 모든 자격 증명은 안전한 위치에 보관하세요.

**Quote Tweets 조회 요청을 구축하는 단계**

**1단계: 도구 또는 라이브러리 선택**

이 엔드포인트에 요청을 보내는 데 사용할 수 있는 도구, 코드 예제, 라이브러리가 여러 가지 있지만, 여기서는 과정을 단순화하기 위해 Postman을 사용하겠습니다.

X API v2 Postman 컬렉션을 환경에 로드하려면 아래 버튼을 클릭하세요:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Postman에 X API v2 추가
</Button>

Postman에 X API v2 컬렉션을 로드한 후, timeline 폴더로 이동하여 "Quote Tweets by ID" 요청을 찾으세요.

**2단계: 요청 인증**

X API에 요청하려면 권한이 있는지 확인해야 합니다. 이 엔드포인트의 경우 [OAuth 2.0 App-Only](/ko/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), [OAuth 2.0 Authorization Code with PKCE](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), 또는 [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 인증 방식 중 하나로 요청을 인증해야 합니다.

간단히 하기 위해 이 요청에는 OAuth 2.0 App-Only를 사용하겠지만, 비공개 [지표](/ko/x-api/fundamentals/metrics) 또는 게시물을 요청하려면 다른 인증 방식을 사용해야 합니다.

Postman에 키와 토큰, 특히 [App Access Token](/ko/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token) (App-only Bearer Token이라고도 함)을 추가해야 합니다. 이를 위해 Postman 오른쪽 상단의 “X API v2” 환경을 선택한 뒤, 환경 드롭다운 옆의 눈 아이콘을 클릭하여 "initial value"와 "current value" 필드에 키와 토큰을 입력하세요.

이 변수가 올바르게 설정되었다면 요청의 Authorization 탭에 자동으로 반영됩니다.

**3단계: Quote Tweets를 조회할 게시물 식별 및 지정**

요청에서 Quote Tweets를 조회할 게시물을 게시물 ID로 지정해야 합니다.

이 예시에서 사용할 게시물 ID는 1409931481552543749입니다.

Postman에서 "Params" 탭으로 이동하여 이 게시물 ID를 id 매개변수의 "Value" 열에 입력하세요.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 1409931481552543749 |

**4단계: 조회할 필드 식별 및 지정**

3단계 후 "Send" 버튼을 클릭하면 응답에서 기본 [게시물 객체](/ko/x-api/fundamentals/data-dictionary#tweet) 필드인 id, text, edit\_history\_tweet\_ids를 받게 됩니다.

추가 필드를 받으려면 [fields](/ko/x-api/fundamentals/fields) 및/또는 [expansions](/ko/x-api/fundamentals/expansions) 매개변수로 요청에 해당 필드를 지정해야 합니다.

이 연습에서는 서로 다른 오브젝트에서 다음 세 가지 추가 필드 세트를 요청합니다:

1. 기본 게시물 객체의 추가 tweet.created\_at 필드
2. 반환된 게시물의 작성자 사용자 객체 기본 필드: id, name, username
3. 관련 사용자 객체의 추가 user.created\_at 필드
    

Postman에서 "Params" 탭으로 이동하여 "Query Params" 테이블에 다음 key:value 쌍을 추가하세요:

|     |     |     |     |
| :--- | :--- | :--- | :--- |
| **키** | **값** | **반환 필드** |
| `tweet.fields` | `created_at` | `tweets.created_at` |
| `expansions` | `author_id` | `includes.users.id`, `includes.users.name`, `includes.users.username` |
| `user.fields` | `created_at` | `includes.users.created_at` |     |

이제 "Send" 버튼 옆에 다음 URL이 표시됩니다:

`https://api.x.com/2/tweets/:id/quote_tweets?expansions=author_id&tweet.fields=created_at&user.fields=created_at`

**참고:**

Postman에서는 URL 필드의 경로 매개변수 :id가 `id` 매개변수 필드에 입력한 값으로 자동으로 업데이트되지 않습니다. 따라서 위의 URL에는 1409931481552543749가 아니라 `:id`가 포함되어 있습니다.

<div id="step-six-paginate-through-your-results">
  #### 6단계: 결과 페이지네이션하기
</div>

이전 응답의 하단에는 다음 필드를 포함하는 메타데이터 객체가 있습니다:

- results\_count
- next\_token

4단계에서 `max_results` 파라미터를 지정하지 않았기 때문에 기본적으로 게시물 10개만 받습니다. 추가 페이지의 데이터를 보려면 마지막 결과의 `next_token` 필드 값을 가져와, 그 문자열을 Postman params 페이지의 `pagination_token` 파라미터 값으로 입력하고 나머지 설정은 그대로 둡니다.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `pagination_token` | avdjwk0udyx6 |

모든 설정이 완료되면 "Send"를 다시 클릭하여 다음 페이지의 결과를 받을 수 있습니다.

이 개념을 더 자세히 설명한 [페이지네이션](/ko/x-api/fundamentals/pagination) 가이드를 준비했습니다.
