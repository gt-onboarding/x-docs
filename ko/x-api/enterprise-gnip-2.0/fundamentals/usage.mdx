<div id="usage-api-twitter-api">
  ## 사용량 API | X API
</div>

<div id="overview">
  ### 개요
</div>

`Enterprise`

계정에 속한 다양한 API 전반의 사용량을 모니터링하고 관리하는 데 가장 중요하며 _가장 뛰어난_ 도구입니다.

**중요 고지:**

Usage API에서 반환되는 사용량 집계는 평가판 사용 또는 기타 청구 조정으로 인해 청구서의 수치와 일치하지 않을 수 있습니다. 모든 수치는 해당 날짜(UTC) 내에 소비된 활동에서 중복을 제거한 값을 기준으로 합니다.

<div id="features">
  #### 기능
</div>

- console.gnip.com UI에서 확인 가능한 사용량 데이터를 프로그래밍 방식으로 가져오기
- 스트림 수준 사용량 데이터 - 제품 수준에 더해 스트림 수준(예: dev, prod)의 사용량 데이터 제공
- 세분화되고 설명적인 데이터 - 검색 "요청(requests)"이 전체 아카이브 검색과 30일 검색 제품으로 구분됨
- Historical PowerTrack의 "days" 및 "jobs" 

<div id="supported-apis">
  ### 지원되는 API
</div>

아래는 현재 Usage API에서 지원하는 API 목록입니다:

- PowerTrack API `enterprise`
- 30-Day Search API `enterprise`
- Full-Archive Search API `enterprise`
- Historical PowerTrack `enterprise`

<div id="limitations">
  ### 제한 사항
</div>

- Usage API는 2018년 5월 1일부터의 사용량 데이터에 접근할 수 있습니다. 2019년 7월 1일부터는 **직전 13개 달력월**의 사용량 데이터에 접근할 수 있습니다.

- fromDate와 toDate로 정의되는 **3개월 단위 간격**으로 사용량 데이터에 접근할 수 있습니다.

다음과 같습니다.

<div id="authentication-and-rate-limit">
  #### **인증 및 레이트 리밋**
</div>

<div id="authentication">
  ##### **인증**
</div>

Usage API에 대한 모든 요청에는 HTTP 기본 인증이 필요합니다. console.gnip.com에 로그인하거나 어떤 Gnip 스트림에 연결할 때 사용되는, 계정에 활성화된 이메일/비밀번호 자격 증명 중 하나를 사용하십시오.

<div id="rate-limit">
  ##### **요청 한도**
</div>

Usage API는 분당 최대 2회의 요청만 허용합니다.

<div id="best-practices-limitations">
  #### **모범 사례와 제한 사항**
</div>

<div id="data-availability">
  ##### **데이터 가용성**
</div>

사용량 데이터는 데이터가 처리된 마지막 완전한 기간(UTC)까지 소비된 활동에서 중복을 제거한 기준으로 산정됩니다. 일반적으로 데이터는 분 단위로 처리되어 최신 상태로 업데이트되며, 다만 Gnip이 시스템을 배포하는 경우는 예외입니다.

- Usage API를 통해 2018년 5월 1일부터의 사용량 데이터에 접근할 수 있습니다. 2019년 7월 1일 이후에는 Usage API를 통해 **직전 13개 달력월**의 사용량 데이터에 접근할 수 있습니다.

- fromDate와 toDate로 정의된 **3개월 단위 구간**으로 사용량 데이터에 접근할 수 있습니다

<div id="requesting-and-receiving-data">
  #### **데이터 요청 및 수신**
</div>

Usage API는 계정의 API 엔드포인트에 HTTP BASIC-AUTH 자격 증명을 포함한 HTTP GET 요청을 보내는 방식으로 동작합니다.

<div id="get-request">
  ##### GET 요청:
</div>

사용자 자격 증명과 계정 이름을 사용하여 다음 엔드포인트로 GET 요청을 보내세요:

https://gnip-api.x.com/metrics/usage/accounts/:account\_name.json

**추가 매개변수**

|     |     |
| :--- | :--- |
| bucket | 선택 사항. 사용량 데이터가 제공될 시간 단위입니다. 사용량 데이터는 일별 또는 월별 세분화 수준으로 반환됩니다.  <br />  <br />bucket을 지정하지 않으면 월별 세분화 수준으로 반환됩니다.  <br />  <br />옵션: 'month' 또는 'day' |
| fromDate (YYYYMMDDHHMM) | 선택 사항. 사용량 데이터는 2018년 5월 1일부터 제공됩니다. 사용량 데이터를 제공할 수 있는 가장 이른 UTC 타임스탬프입니다. 타임스탬프는 일 단위 세분화이며 포함입니다(예: 201805010000은 05월 01일을 포함). 시와 분에 '0000'이 아닌 값을 포함하는 요청은 '0000'으로 기본 처리됩니다.  <br />  <br />fromDate나 toDate 없이 요청하면 현재 월의 월별 사용량 데이터와 직전 2개월의 이력 데이터가 반환됩니다.  <br />  <br />**참고:** 2019년 6월 1일부터 과거 13개월의 사용량 데이터에 접근할 수 있습니다. 예를 들어 10월 10일이라면, 전년도 9월 1일까지의 사용량 데이터에 접근할 수 있습니다.  <br />**예:** 201810010000은 2018년 10월 1일부터(10월 1일 포함) 데이터를 반환합니다. |
| toDate (YYYYMMDDHHMM) | 선택 사항. 사용량 데이터를 제공할 최신 UTC 타임스탬프입니다. 타임스탬프는 일 단위 세분화이며 비포함입니다(예: 201703020000은 03월 02일의 데이터를 포함하지 않음). toDate를 현재 날짜나 미래 날짜로 지정한 경우, 사용량 데이터는 마지막으로 완료된 하루(UTC)까지 반환됩니다. 시와 분에 '0000'이 아닌 값을 포함하는 요청은 '0000'으로 기본 처리됩니다.  <br />  <br />toDate가 없는 요청은 다음 버킷으로 기본 처리됩니다(bucket=day이면 다음 날, bucket=month이면 다음 달). fromDate와 toDate 없이 요청하면 bucket=month로 기본 처리되며, 해당 월의 데이터와 직전 두 달의 데이터가 표시됩니다.  <br />  <br />**예:** 201703050000은 2017년 3월 5일까지의 데이터를 반환하되, 3월 5일의 데이터는 포함하지 않습니다. |

**GET 요청 예시**

이 요청은 2017년 3월 1일부터 2017년 3월 5일까지의 월별 세분화 데이터(3월 5일 데이터는 제외)를 반환합니다.

curl -u "https://gnip-api.x.com/metrics/usage/accounts/:account\_name.json?bucket=month\&fromdate=201403010000\&toDate=201403150000"
