---
title: 트윗 편집하기
---

엔터프라이즈 엔드포인트가 수정된 게시물 메타데이터를 제공하도록 업데이트되었습니다. \_게시물 수정\_ 기능은 2022년 9월 1일 X 직원 대상 테스트로 처음 도입되었습니다. 그날부터 자격을 충족하는 게시물은 30분 동안 최대 5회까지 수정할 수 있었습니다. _2022년 9월 29일 이후에 생성된 게시물의 모든 오브젝&#xD2B8;_&#xC5D0;는 게시물이 한 번도 수정되지 않았더라도 게시물 수정 메타데이터가 포함됩니다. 게시물이 수정될 때마다 새로운 게시물 ID가 생성됩니다. 게시물의 수정 이력은 원래 ID부터 시작해 이러한 ID들을 연결하여 추적할 수 있습니다. 또한 수정 체인에 있는 게시물 중 하나라도 삭제되면, 해당 체인의 모든 게시물도 함께 삭제됩니다.

이 메타데이터는 자동으로 포함됩니다. 게시 객체에 제공되는 수정 이력을 포함하려면 별도의 요청 매개변수가 필요하지 않습니다.

이 새로운 메타데이터를 통해 개발자는 다음을 알 수 있습니다:

- 게시물이 생성 시점에 편집 대상 자격이 있었는지 여부입니다. 설문조사가 포함된 게시물이나 예약 게시물 등 일부 게시물은 편집할 수 없습니다.
- 게시물은 30분 동안 편집할 수 있으며 최대 5회까지 편집 가능합니다. 편집 가능한 게시물이라면 남은 편집 시간과 추가 가능한 편집 횟수를 확인할 수 있습니다.
- 게시물의 편집된 버전을 보고 있는 경우. 대부분의 경우 API는 게시물 ID로 특정 과거 버전을 요청하지 않는 한 게시물의 최신 버전을 반환합니다.

루트 수준에 새로운 게시 속성 3개가 추가되었습니다:

- **edit\_history**  - 게시물의 수정 이력에 연결된 모든 게시물 ID를 제공합니다. "initial\_tweet\_id" 속성은 원본 게시물을 나타내며, "edit\_tweet\_ids" 속성은 해당 수정 이력에 연결된 모든 ID를 담은 배열입니다. 게시물이 한 번도 수정되지 않았다면 이 배열에는 단 하나의 ID만 포함됩니다.

```
"edit_history": {
    "initial_tweet_id": "1283764123"
    "edit_tweet_ids": ["1283764123"]
  }
```

- **edit\_controls** - 30분 편집 가능 창의 종료 시점과 남은 편집 가능 횟수를 나타내는 속성을 제공합니다.

```
"edit_controls": {  
     "editable_until_ms": 1660155761384
     "edits_remaining": 3   
  }
```

- **editable** - 게시물이 생성될 때 편집 가능한 상태였는지를 나타냅니다.

"editable": true

대부분의 게시물은 대상에 포함됩니다. 다만 다음 유형의 게시물은 제외됩니다:

- 게시물이 프로모션되었습니다
- 게시물에 투표가 있습니다
- 게시물이 자체 스레드가 아닌 답글입니다
- 게시물이 리포스트입니다(인용 게시물은 편집할 수 있음에 유의)
- 게시물이 널캐스트 상태입니다
- 커뮤니티 포스트
- 슈퍼팔로우 게시물
- 공동 게시

**편집되지 않은 게시물의 예제 속성**

아래 JSON은 게시물 수정 기능이 도입된 이후에 게시된 게시물에 포함되는 수정 메타데이터를 보여줍니다. 이 예시는 수정 이력이 없는 게시물에 대한 것입니다.

참고: "edit\_tweet\_ids" 배열에는 ID가 하나만 포함되어 있습니다.

```
{
  "created_at": "Wed Aug 16 18:29:02 +0000 2022",
  "id": 1557433858676740098,
  "id_str": "1557433858676740098",
  "text": "편집 버튼을 내가 과연 쓸 일이 있을지 궁금하네"
  "edit_history": {
    "initial_tweet_id": "처음 트윗 ID"
    "edit_tweet_ids": ["편집된 트윗 ID"]
  },
  "edit_controls": {
    "editable_until_ms": "편집 가능 기한(ms)"
    "edits_remaining": "남은 편집 횟수"
  },
  "editable": "편집 가능"
}
```

**수정된 게시물의 예제 속성**

아래 JSON은 ‘게시물 수정’ 기능 도입 이후에 게시된 게시물에 포함되는 수정 메타데이터를 보여줍니다. 이 예시는 한 번 수정된 게시물의 사례입니다.

"edit\_tweet\_ids" 배열에는 원본 게시물과 수정된 업데이트에 해당하는 두 개의 ID가 있다는 점에 유의하세요.

```
{
  "created_at": "Wed Aug 16 18:35:42 +0000 2022",
  "id": 1557445923210514432,
  "id_str": "1557445923210514432",
  "text": "언젠가 내가 수정 버튼을 쓰게 될지 궁금하다",
  "edit_history": {
    "initial_tweet_id": "1557433858676740098",
    "edit_tweet_ids": ["1557433858676740098", "1557445923210514432"]
  },
  "edit_controls": {
    "editable_until_ms": 1660155761384,
    "edits_remaining": 4
  },
  "editable": true
}
```

**컴플라이언스(준수) 지원**

[Compliance Firehose](/ko/x-api/enterprise-gnip-2.0/fundamentals/firehouse)와 v2 [배치 컴플라이언스 엔드포인트](/ko/x-api/compliance/batch-compliance/introduction)가 모두 게시물 편집을 지원하도록 업데이트되었습니다:

새로운 "tweet\_edit" 이벤트 유형이 Compliance Firehose에 추가되었습니다.

```
{
  "tweet_edit": {
    "id": <tweetId>,
    "initial_tweet_id": <tweetId>,
    "edit_tweet_ids": [<tweetId1>, <tweetId2>, <tweetId3> ...],
    "timestamp_ms": "<timestampMsStr>"
  }
}
```
