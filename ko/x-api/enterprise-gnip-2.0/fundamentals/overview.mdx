<div id="overview">
  ## 개요
</div>

> 엔터프라이즈 콘솔(console.gnip.com)은 계약된 엔터프라이즈 API 액세스를 보유한 엔터프라이즈 데이터 고객에게 제공됩니다. 현재 엔터프라이즈 데이터 고객이 아니지만 더 자세한 정보가 필요하시면 [엔터프라이즈 데이터](https://developer.x.com/en/products/x-api/enterprise)를 참고하시기 바랍니다.

이 문서는 X의 엔터프라이즈 콘솔 대시보드 개요입니다. 여기에서 다루는 섹션은 제품 상세, 사용량, 일반 계정 관리 등 콘솔의 각 영역에 해당합니다. 특정 계정과 관련해 문의사항이 있으시면 담당 계정 관리자에게 문의하시거나 [엔터프라이즈 제품 기술 문서](/ko/x-api/enterprise-gnip-2.0/enterprise-gnip)를 검토하시기 바랍니다.

엔터프라이즈 체험 및 온보딩 프로세스의 일환으로, 담당 계정 관리자와 함께 진행되는 절차에서 엔터프라이즈 고객은 [console.gnip.com](http://console.gnip.com) 로그인 정보를 받게 됩니다. 이곳은 엔터프라이즈 제품 액세스를 검토하고 구성할 수 있는 사용자 인터페이스입니다. 고객 관리자에 대한 콘솔 최초 액세스는 온보딩 과정에서 X가 설정하며, 이후 관리자는 추가 사용자를 등록할 수 있습니다. 엔터프라이즈 콘솔을 통해 엔터프라이즈 제품 관련 세부 정보를 관리하고 확인할 수 있습니다. 

다음 동영상은 console.gnip.com 대시보드의 다양한 영역을 개괄합니다. 

**고지: 동영상에 표시된 일부 기능(특정 스트림 보강 기능 및 X 외부 데이터 소스 제품 포함)은 더 이상 제공되지 않을 수 있습니다.**

<div id="products-tab">
  ## 제품 탭
</div>

[console.gnip.com](http://console.gnip.com)에 계정으로 로그인하면 대시보드의 제품 탭이 기본으로 표시됩니다. 이 페이지에는 계정에서 현재 사용 가능한 엔터프라이즈 제품의 개요가 포함됩니다.

PowerTrack 또는 Decahose처럼 스트리밍 방식으로 전달되는 제품의 경우, 이 페이지에는 제품 이름과 스트림 레이블, 현재 활성 연결 수, 각 스트림의 활성 규칙 수(해당되는 경우), 그리고 최근 24시간 동안 전달된 활동(예: 게시물)의 원시 개수가 표시됩니다.

Search API처럼 REST 방식으로 전달되는 제품의 경우, 이 페이지에는 제품 이름, 레이블(“streams”로도 표시됨), 해당 엔드포인트를 통해 반환된 현재 활동(예: 게시물), 그리고 엔드포인트별 여러 요청 건수가 표시됩니다.

[Usage API](/ko/x-api/enterprise-gnip-2.0/fundamentals/usage)를 통해 이 데이터의 상당 부분을 프로그래밍 방식으로 제공받을 수 있습니다.

스트림별 상세 정보는 이름 또는 설정 버튼을 클릭하여 확인할 수 있습니다.

<div id="overview">
  ### 개요
</div>

메인 대시보드에서 스트림 중 하나를 클릭하면 해당 스트림의 개요 페이지로 이동합니다.

스트리밍 전달 제품의 경우 이 페이지에는 다음이 포함됩니다.

1. 각 스트림 연결을 통해 전달되는 활동 수를 보여주는 볼륨 차트

2. 스트림에서 현재 활성 상태인 연결의 세부 정보(연결 ID 및 IP 주소)

3. 스트림의 최근 연결, 연결 해제, 규칙 업데이트 이벤트 로그

차트의 축 범위는 오른쪽 상단의 링크로 조정할 수 있습니다. 개별 연결 및 연결 해제 항목의 표시 여부는 차트 바로 아래 범례에서 해당 항목을 클릭하여 토글할 수 있습니다.

<div id="connections">
  ### 연결
</div>

Connections 페이지에서는 스트림의 최근 연결 이벤트에 대한 세부 정보를 제공합니다. 여기에는 각 연결의 시작 및 종료 시간(UTC 24시간 표기), 각 연결의 지속 시간, 연결을 수행한 서버의 IP, 참고용 고유 연결 ID, 그리고 현재 연결 상태가 포함됩니다. 상태는 지정된 연결에 대한 가장 최근 이벤트를 나타냅니다. 즉, Client Connected 또는 연결 해제이며, 연결 해제의 유형이 함께 표시됩니다. 연결 문제 디버깅에 대한 자세한 내용은 [disconnections explained 가이드](/ko/x-api/enterprise-gnip-2.0/powertrack-api#integrating-with-powertrack)를 참고하세요.

<div id="rules">
  ### 규칙
</div>

규칙 탭은 PowerTrack 스트림에서 사용할 수 있으며, 사용자 인터페이스에서 일반 텍스트 규칙을 수동으로 입력해 손쉽게 시작할 수 있습니다. 이 인터페이스는 이 수동 방식으로 최대 1,000개의 규칙 추가만 지원하므로 초기 테스트에만 사용해야 합니다. 프로덕션 환경에서는 [Rules API](/ko/x-api/enterprise-gnip-2.0/powertrack-api#api-reference-index)를 통해 프로그램적으로 규칙을 관리할 것을 권장합니다.

<div id="settings">
  ### 설정
</div>

설정 탭에서는 스트림의 데이터 출력 형식을 변경할 수 있으며, 여러 형식 옵션을 지원합니다. 형식을 변경하려면 각 옵션이 표시된 라디오 버튼을 사용하세요. 변경 사항은 스트림에 다시 연결하면 적용됩니다. 이 설정을 업데이트하면 다음 요청 또는 연결부터 즉시 반영되며, 새 형식으로 인해 파서가 동작하지 않을 수 있습니다. 

**참고:** 가장 많은 데이터를 얻으려면 “데이터를 원래 형식으로 유지”를 권장합니다. 이 옵션은 [여기](/ko/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#post-object)에 설명된 보강된 네이티브 형식으로 데이터를 반환합니다. Activity streams 형식은 [여기](/ko/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#activity-object)에 설명되어 있습니다.

<div id="usage-tab">
  ## 사용량 탭
</div>

사용량 탭에서는 여러 기간에 걸친 스트림 사용 현황을 파악할 수 있습니다. 사용량 정보를 프로그래밍 방식으로 이용하려면 [Usage API](/ko/x-api/enterprise-gnip-2.0/fundamentals/usage)를 참조하세요.

<div id="monthly">
  ### 월간
</div>

월간 사용량 페이지는 제품별로 구분된 스트림 사용량을 표시합니다. 예를 들어, 특정 데이터 소스의 커버리지 제품(예: PowerTrack, Historical PowerTrack, PowerTrack Replay)은 개별 데이터와 통합 합계를 모두 제공하도록 함께 그룹화됩니다. 수치에는 당월 누적, 월말 추정치(현재까지의 당월 사용량과 남은 기간을 기반), 그리고 직전 두 달의 수치가 포함됩니다.

특히, 이 수치는 제품 및 스트림 단위로 중복이 제거됩니다. 동일한 PowerTrack 스트림에 대한 여러 연결을 통해 동일한 게시물을 수신한 경우, 이러한 집계를 위한 계산에서는 해당 게시물이 한 번만 집계됩니다. 수치는 매일 00:00 UTC에 업데이트됩니다.

<div id="daily">
  ### 일별
</div>

Daily Usage 페이지에서는 현재 월의 각 날짜에 대해 제품별로 구분한 중복 제거 일별 수치를 제공합니다. 수치는 매일 00:00 UTC에 24시간 간격으로 업데이트됩니다.

<div id="rules">
  ### 규칙
</div>

규칙 한도는 계약한 PowerTrack 규칙 패키지를 기준으로 하며, 특정 "유형"의 모든 스트림에 동일하게 적용됩니다. 새 규칙이 추가되거나 삭제될 때마다 카운트가 업데이트됩니다.

<div id="my-profile">
  ### 내 프로필
</div>

개별 사용자 프로필 정보를 확인하고 이곳에서 비밀번호를 변경할 수 있습니다. 또한 내 계정에만 적용되는 사용 한도 알림을 설정할 수 있습니다. API 상태 알림은 https://api.twitterstat.us 상태 페이지에서 확인할 수 있습니다.

<div id="account-settings">
  ### 계정 설정
</div>

사용자를 추가, 제거, 편집하고 개별 사용자 및 사용량 임계값 알림에 대한 이메일 알림을 설정할 수 있습니다.

<Frame>
  <img src="/images/account-settings.png.twimg.1920.png" />
</Frame>

세 가지 유형의 사용자가 있습니다: Account Admin, User, Email Only.

- Account Admin은 다른 사용자를 생성/삭제/편집할 수 있으며, 엔터프라이즈 API에 연결하기 위해 사용자 이름(이메일)과 비밀번호를 사용하는 기본 인증을 이용할 수 있습니다.

- User는 다른 사용자를 생성하거나 수정할 수 없지만, 엔터프라이즈 API에 연결하기 위해 사용자 이름(이메일)과 비밀번호를 사용하는 기본 인증을 이용할 수 있습니다.

- Email Only 사용자는 대시보드에 접근할 수 없고, 엔터프라이즈 API에 연결할 권한이 없으며, 알림 섹션에서 수신하도록 설정된 경우에만 알림을 받습니다.
