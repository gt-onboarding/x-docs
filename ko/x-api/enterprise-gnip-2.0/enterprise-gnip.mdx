엔터프라이즈 API는 X 데이터에 의존하는 분들을 위해 최고 수준의 접근성과 신뢰성을 제공합니다. 프리미엄을 넘어 확장하며 보다 안정적인 접근, 맞춤형 패키지, 또는 장기 계약이 필요할 때에 적합합니다. 엔터프라이즈 API 이용에는 전담 계정 매니저와 개인화된 기술 지원이 포함됩니다.

Basic 및 Pro 솔루션만으로는 요구 사항을 충족하기에 충분하지 않다고 판단되면, 아래 링크에서 엔터프라이즈 제품 이용을 신청할 수 있습니다:

<div id="available-endpoints">
  ## 사용 가능한 엔드포인트
</div>

| 실시간 | [PowerTrack API](/ko/x-api/enterprise-gnip-2.0/powertrack-api) | 공개 게시물이 게시되는 즉시 전체 범위를 제공하며, 다양한 연산자와 규칙으로 필터링할 수 있습니다. |
| :--- | :--- | :--- |
| [Decahose API](/ko/x-api/enterprise-gnip-2.0/fundamentals/decahose-api) | 전체 공개 게시물과 해당 좋아요의 10%를 무작위로 샘플링해 스트리밍합니다. |
| :--- | :--- | :--- |
| [Account Activity API](/ko/x-api/enterprise-gnip-2.0/fundamentals/account-activity) | 사용자에 구독하고 게시물, 다이렉트 메시지, 좋아요, 팔로우 등 다양한 활동을 수신합니다. |
| :--- | :--- | :--- |
| 기록 | [Search API](/ko/x-api/enterprise-gnip-2.0/fundamentals/search-api) | 고급 필터링 도구를 사용해 지난 30일 또는 공개 게시물 전체 아카이브를 검색합니다. |
| [Historical PowerTrack API](/ko/x-api/enterprise-gnip-2.0/powertrack-api) | 경제적인 배치 액세스 방식으로 공개 게시물 전체 아카이브를 필터링합니다. |
| 인사이트 | [Engagement API](/ko/x-api/enterprise-gnip-2.0/fundamentals/engagement-api) | 공개 게시물의 노출수와 참여수를 측정합니다. |
| 컴플라이언스 | [Compliance Firehose API](/ko/x-api/enterprise-gnip-2.0/fundamentals/firehouse) | 모든 컴플라이언스 이벤트를 스트리밍합니다. |
| 사용량 | [Usage API](/ko/x-api/enterprise-gnip-2.0/fundamentals/usage) | 엔터프라이즈 제품 전반의 활동 소비량에 대한 프로그래매틱 액세스를 제공합니다. |

<div id="important-concepts">
  ## 핵심 개념
</div>

<div id="gnip-console">
  ### Gnip 콘솔
</div>

[Gnip 콘솔](/ko/x-api/enterprise-gnip-2.0/fundamentals/overview)에서 엔터프라이즈 제품과 계정 접근 권한을 관리할 수 있습니다. 

<div id="enrichments">
  ### 보강 정보
</div>

유료 티어를 사용하는 개발자를 위해 프리미엄 API의 응답 페이로드에 메타데이터를 추가했습니다. 확장 URL, 투표 데이터, 프로필 지오에 대해 더 알아보려면 [보강 정보 섹션](/ko/x-api/enterprise-gnip-2.0/fundamentals/data-enrichments)을 참조하세요.
 

<div id="advanced-filtering-tools">
  ### 고급 필터링 도구
</div>

Premium Search API는 표준 검색 엔드포인트에서는 사용할 수 없는 고급 연산자 세트를 제공합니다. 이들 연산자 중 상당수는 무료 샌드박스 사용자도 이용할 수 있지만, 유료 티어를 사용하면 더 풍부하고 강력한 연산자 세트를 사용할 수 있습니다. 자세한 내용은 [규칙 및 필터 섹션](/ko/x-api/enterprise-gnip-2.0/fundamentals/rules-filtering#getting-started-with-enterprise-rules-and-queries)을 참조하세요.
