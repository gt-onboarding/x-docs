이 가이드는 첫 요청을 보내는 데 따라할 수 있는 몇 가지 단계를 안내합니다. X 계정을 만든 후 활용하기 좋은 자료입니다.

코드 샘플, 보다 기술적인 가이드, 또는 Postman 같은 그래픽 도구를 사용하고 싶다면, 첫 요청을 보내는 데 다음 가이드를 참고하세요:

- [게시 엔드포인트에 첫 요청 보내기 빠른 시작 가이드](/ko/x-api/posts/manage-tweets/quickstart)
- [Postman 시작하기](/ko/tutorials/postman-getting-started) - 초보자에게 권장
- [X API v2 코드 샘플](https://github.com/xdevplatform/Twitter-API-v2-sample-code)
- [게시 조회 API 참고 문서](/ko/x-api/posts/lookup/introduction)

이 가이드는 [API key와 secret](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret), [user Access Token과 Secret](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens), [App Access Token](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)을 확보하여 안전한 위치에 보관했다고 가정합니다. 이에 대한 방법은 [X API 액세스 받기](/ko/x-api/getting-started/getting-access) 가이드의 단계를 따라 배우실 수 있습니다.

<div id="step-1-identify-which-endpoint-you-would-like-to-use">
  ### 1단계. 사용할 엔드포인트 선택
</div>

X API를 사용하면 X 웹사이트나 모바일 애플리케이션에서 할 수 있는 다양한 작업을 코드를 통해 수행할 수 있습니다.

API로 제공되는 엔드포인트의 전체 목록은 [API Reference Index](/ko/x-api/introduction)에 정리되어 있지만, 단순하게 시작하려면 다음 중 하나를 사용하는 것을 권장합니다.

- [Manage Posts > 트윗 게시](/ko/x-api/posts/manage-tweets/introduction)
- [게시물 검색](/ko/x-api/posts/search/introduction)
- [사용자 조회](/ko/x-api/users/lookup/introduction)

<div id="step-2-choose-a-tool-to-make-your-request">
  ### 2단계. 요청을 만들 도구를 선택하세요
</div>

일부 요청은 간단하지만, 다른 요청은 구현이 까다로울 수 있습니다. 그래서 훌륭한 개발자 커뮤니티가 복잡한 부분을 추상화해 주는 도구들을 만들어 왔습니다.

아래는 권장 도구와 그 사용 방법에 대한 상세 정보입니다:

<div id="postman">
  #### Postman
</div>

Postman은 REST 엔드포인트에 요청을 보낼 수 있는 시각적 도구입니다. X API를 통해 제공되는 다양한 엔드포인트를 시작하고 탐색하는 데 도움이 되도록 Postman 관련 유용한 자료를 준비했습니다.

키와 토큰을 추가하고 첫 요청을 보내는 방법을 알아보려면 ["Postman 시작하기" 튜토리얼](/ko/tutorials/postman-getting-started)을 읽어 보시기 바랍니다.

또한 X API v2의 각 엔드포인트별 빠른 시작 가이드를 제공하며, 대부분 Postman을 사용합니다. 이러한 가이드는 각 엔드포인트 섹션에서 확인할 수 있으며, 아래는 일부 링크입니다:

- 빠른 시작: 트윗 게시
- 빠른 시작: 게시물 검색
- 빠른 시작: 사용자 조회

Postman으로는 스트리밍 엔드포인트에 요청할 수 없습니다. 해당 엔드포인트 사용 방법은 [필터드 스트림](/ko/x-api/posts/filtered-stream/quickstart) 또는 [1% 샘플드 스트림](/ko/x-api/posts/sample-stream) 빠른 시작 가이드를 참조하세요.

더 간단한 그래픽 도구를 선호한다면 [Insomnia](https://insomnia.rest) 사용도 고려해 보세요.

<div id="sample-code">
  #### 샘플 코드
</div>

간단한 코드로 요청을 보내고 싶다면, 각 X API v2 엔드포인트별로 다양한 언어의 샘플 코드를 제공합니다.

GitHub 저장소 [X-API-v2-sample-code](https://github.com/xdevplatform/Twitter-API-v2-sample-code)에서 코드 샘플을 확인할 수 있으며, 요청이 올바르게 동작하도록 자격 증명을 설정하는 방법을 안내하는 README 파일도 포함되어 있습니다.

예를 들어, 다음은 사용자 조회 엔드포인트에 대한 cURL 예시입니다. 이 요청을 사용하려면 $ACCESS\_TOKEN과 $USERNAME을 각각 [App Access Token](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)과 X 핸들로 바꾸면 됩니다. 그런 다음 이 코드를 명령줄 도구에 복사해 붙여넣고 Enter(또는 Return) 키를 누르세요.

```bash
curl "https://api.x.com/2/users/by/username/$USERNAME" -H "Authorization: Bearer $ACCESS_TOKEN"
```

<div id="libraries">
  #### 라이브러리
</div>

X Developer 커뮤니티는 다양한 프로그래밍 언어로 X API에 요청을 보낼 수 있는 라이브러리를 구축해 왔습니다.

커뮤니티 라이브러리를 모두 정리한 ["Tools and libraries" 페이지](/ko/x-api/tools-and-libraries/overview)를 제공합니다. 각 라이브러리에는 로컬 환경에서 리포지토리를 설정하고 첫 요청을 보내는 방법을 안내하는 README 파일이 포함되어 있습니다.

> **참고:** 문제가 발생하면 요청하려는 엔드포인트의 개발자 문서를 확인하고, [지원 섹션](https://developer.x.com/en/support)을 참고하거나 포럼의 커뮤니티에 도움을 요청해 주세요. 성공적으로 요청을 완료할 수 있도록 도와드리겠습니다!

<div id="step-3-review-the-response">
  ### 3단계. 응답 검토
</div>

요청이 성공하면 요청과 관련된 메타데이터가 포함된 페이로드를 받게 됩니다.

GET HTTP 메서드를 사용하는 엔드포인트를 사용했다면, 요청한 리소스(게시물, 사용자, 리스트, 스페이스 등)에 대한 메타데이터를 JSON 형식으로 받습니다. 반환된 각 필드를 검토하고, 요청한 정보가 X의 콘텐츠에 올바르게 매핑되는지 확인하세요.

POST, PUT, DELETE HTTP 메서드를 사용하는 엔드포인트를 사용했다면, X에서 어떤 작업을 수행한 것입니다. X.com 또는 모바일 앱에서 해당 작업이 적용되었는지 확인해 보세요.

<div id="step-4-adjust-your-request-using-parameters">
  ### 4단계. 매개변수로 요청 조정하기
</div>

각 엔드포인트는 요청을 변경하는 데 사용할 수 있는 고유한 매개변수 집합을 제공합니다. 예를 들어 GET 엔드포인트에서 `fields` 및 `expansions` 매개변수를 사용해 추가 메타데이터 필드를 요청할 수 있습니다. 또한 [search Posts](/ko/x-api/posts/search/introduction), [Post counts](/ko/x-api/posts/counts/introduction), [filtered stream](/ko/x-api/posts/filtered-stream/introduction) 등의 엔드포인트에서 다양한 필터링 도구를 활용해 수신 데이터를 관심 있는 게시물만으로 좁힐 수 있습니다.

작업 중인 엔드포인트의 API Reference에서 요청 매개변수와 필드의 전체 목록을 확인할 수 있으며, 통합 가이드와 기본 개념 페이지에서 유용한 통합 관련 정보도 폭넓게 제공됩니다.

제공되는 모든 교육 자료는 [Important resources](/ko/x-api/getting-started/important-resources) 가이드에서 자세히 확인할 수 있습니다.
