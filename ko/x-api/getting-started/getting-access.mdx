<div id="step-one-sign-up-for-a-developer-account">
  ### 1단계: 개발자 계정 가입
</div>

개발자 계정 가입은 빠르고 간편합니다! 아래 버튼을 클릭해 몇 가지 질문에 답하면, [Basic access](https://developer.x.com/en/portal/products/basic)로 X API v2를 탐색하고 구축을 시작할 수 있습니다.

다음으로 온보딩 과정에서 프로젝트와 연결된 개발자 앱을 생성하게 되며, 이를 통해 API에 대한 모든 요청을 인증하는 데 사용할 자격 증명 세트를 받습니다.

[가입하기](https://developer.x.com/en/portal/products/basic)

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### 2단계: 앱의 키와 토큰을 저장하고 안전하게 보호하기
</div>

액세스 권한을 받고 프로젝트와 앱을 생성하면, 개발자 앱에서 다음 자격 증명을 확인하거나 생성할 수 있습니다:

- **[API Key 및 Secret](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret):** 본질적으로 앱의 사용자 이름과 비밀번호에 해당합니다. [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication)가 필요한 요청을 인증하거나, 사용자 액세스 토큰 또는 앱 액세스 토큰 등의 다른 토큰을 생성할 때 사용합니다.

- **[Access Token 및 Secret](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens):** 일반적으로 액세스 토큰은 요청을 수행하는 대리 대상 사용자를 나타냅니다. 개발자 포털을 통해 생성하는 토큰은 해당 앱 소유자를 나타냅니다. [OAuth 1.0a 사용자 컨텍스트](/ko/resources/fundamentals/authentication)가 필요한 요청을 인증할 때 사용합니다. 다른 사용자를 대신해 요청하려면 해당 사용자가 승인을 부여할 수 있도록 3-legged OAuth 플로우를 사용해야 합니다.

- **[Client ID 및 Client Secret](/ko/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce):** 이 자격 증명은 OAuth 2.0 인증으로 사용자 액세스 토큰을 얻는 데 사용됩니다. OAuth 1.0a와 유사하게 사용자 액세스 토큰은 비공개 사용자 계정 정보를 제공하거나 다른 계정을 대신해 작업을 수행하는 요청을 인증하는 데 사용되며, 클라이언트 애플리케이션이 사용자에 대해 갖는 액세스 권한을 보다 세밀한 범위로 제어할 수 있습니다.

- **[App only Access Token](/ko/resources/fundamentals/authentication#bearer-token-also-known-as-app-only):** X에서 공개적으로 제공되는 정보로 응답하는 엔드포인트에 요청할 때 이 토큰을 사용합니다.

이 키와 토큰은 재생성하지 않는 한 만료되지 않으므로, 자격 증명을 받는 즉시 비밀번호 관리자 등 안전한 위치에 저장할 것을 권장합니다.

> **참고: 키와 토큰은 [developer portal](/ko/resources/fundamentals/developer-portal)에 한 번만 표시되므로, 생성 즉시 비밀번호 관리 시스템에 저장하는 것이 중요합니다.**
>
> 키와 토큰을 분실하거나 잊은 경우, 재생성이 필요하며 이 과정에서 새 자격 증명이 만들어지고 기존 것은 무효화됩니다. 이는 이전 자격 증명으로 구성된 모든 통합을 업데이트해야 함을 의미합니다.
>
> [인증 모범 사례](/ko/resources/fundamentals/authentication#authentication-best-practices)에 대해 자세히 알아보세요.
