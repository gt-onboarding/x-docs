Direct Messages 조회의 초기 릴리스에는 다음 세 가지 GET 메서드가 포함됩니다:

- **GET /2/dm\_conversations/with/:participant\_id/dm\_events** - 일대일 대화와 연결된 Direct Message 이벤트를 가져옵니다. :participant\_id 경로 매개변수는 이 요청을 수행하는 인증된 사용자와 대화 중인 계정의 사용자 ID입니다.
- **GET /2/dm\_conversations/:dm\_conversation\_id/dm\_events** - :dm\_conversation\_id 경로 매개변수로 지정된 특정 대화 ID와 연결된 Direct Message 이벤트를 가져옵니다.
- **GET /2/dm\_events** - 사용자와 연결된 Direct Message 이벤트를 가져오며, 일대일 및 그룹 대화를 모두 포함합니다. 최대 30일 전 이벤트까지 조회할 수 있습니다.

Direct Message 이벤트 ID는 v1.1과 v2(및 X 앱)에서 공통이므로, 단일 이벤트를 나열하는 v1.1 메서드를 새로운 v2 엔드포인트와 함께 사용할 수 있습니다. 또한 Enterprise 및 Premium Account Activity API는 v2 일대일 메시지를 지원하지만, 아직 그룹 대화는 지원하지 않습니다.

이번 릴리스에서는 세 가지 이벤트 유형이 지원되며, 해당 엔드포인트는 다음 이벤트 유형을 기준으로 필터링할 수 있는 쿼리 매개변수를 지원합니다:

- **MessageCreate** - 메시지가 생성되었습니다.
- **ParticipantsJoin** - 새로운 참가자가 대화에 참여했습니다.
- **ParticipantsLeave** - 참가자가 대화에서 나갔습니다.

GET 메서드에는 15분당 300회까지의 사용자 레이트 리밋이 적용됩니다. 이 레이트 리밋은 해당 GET 엔드포인트 전반에 걸쳐 공유됩니다.

Direct Message v2 엔드포인트로 사용자를 대신해 요청하므로, [OAuth 1.0a User Context](/ko/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3)를 사용해 인증해야 하며, 귀하의 X 앱을 승인한 사용자와 연결된 Access Token을 사용해야 합니다. OAuth 1.0a로 이러한 Access Token을 생성하려면 [3-legged OAuth flow](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)를 사용할 수 있습니다. OAuth 2.0으로 사용자 Access Token을 생성하려면 [Authorization Code with PKCE grant flow](/ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)를 사용할 수 있습니다.

**계정 설정**

이 엔드포인트에 액세스하려면 다음이 필요합니다:

- 승인된 [개발자 계정](https://developer.x.com/en/portal/petition/essential/basic-info)
- [프로젝트](/ko/resources/fundamentals/projects) 내에 있는 [개발자 앱](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용한 인증

[X API v2 엔드포인트에 액세스하는 방법](/ko/x-api/getting-started/getting-access)에 대해 시작 가이드에서 자세히 알아보세요.
