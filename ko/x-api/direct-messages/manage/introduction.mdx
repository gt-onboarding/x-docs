다이렉트 메시지는 X에서 비공개 대화를 가능하게 합니다. 다이렉트 메시지는 X에서 가장 인기 있는 기능 중 하나로, 매우 다양한 사용 사례에 활용됩니다. 사용 사례는 친구 간 그룹 채팅부터 전 세계 브랜드의 고객 지원 운영에 이르기까지 폭넓습니다. 다이렉트 메시지 엔드포인트의 새로운 v2 버전은 단계적으로 도입되며, 첫 단계에는 다이렉트 메시지 생성 및 다이렉트 메시지 대화 이벤트 목록 조회를 위한 기본 엔드포인트가 포함됩니다. X API v2는 처음으로 그룹 대화를 지원합니다.

Manage Direct Messages의 초기 릴리스에는 다이렉트 메시지를 생성하기 위한 세 가지 POST 메서드가 포함됩니다:

- **POST /2/dm\_conversations/with/:participant\_id/messages** - 1:1 다이렉트 메시지를 생성합니다. 이 메서드는 새로운 1:1 대화를 생성하거나 기존 대화를 조회해 해당 대화에 다이렉트 메시지를 추가합니다. :participant\_id 경로 매개변수는 메시지를 받는 계정의 사용자 ID입니다.
- **POST /2/dm\_conversations** - 새로운 그룹 대화를 생성하고 해당 대화에 다이렉트 메시지를 추가합니다. 이 요청에는 대화 참여자 목록이 필요합니다. 동일한 참여자 목록으로 여러 대화를 생성할 수 있습니다. 이 요청은 항상 새로운 대화 ID를 반환합니다.
- **POST /2/dm\_conversations/:dm\_conversation\_id/messages** - 다이렉트 메시지를 생성하여 기존 대화에 추가합니다. :dm\_conversation\_id 경로 매개변수는 메시지가 추가될 대화의 ID입니다.

다이렉트 메시지 이벤트 ID는 v1.1과 v2(및 X 앱) 전반에서 공통이므로, 다이렉트 메시지를 숨기거나 삭제하는 v1.1 메서드를 새로운 v2 엔드포인트와 함께 사용할 수 있습니다. 또한 Enterprise 및 Premium Account Activity API는 v2의 1:1 메시지는 지원하지만, 아직 그룹 대화는 지원하지 않습니다.

POST 메서드에는 사용자 기준 15분당 200건의 레이트 제한이 있습니다. 또한 사용자 기준 24시간당 1,000건의 레이트 제한이 있습니다. 추가로, 24시간당 15,000건의 레이트 제한이 있습니다. 이러한 레이트 제한은 해당 POST 엔드포인트 전체에 걸쳐 공유됩니다.

manage Posts 엔드포인트로 사용자를 대신해 요청을 수행하므로, [OAuth 1.0a User Context](https://developer-staging.x.com/resources/fundamentals/authentication) 또는 [OAuth 2.0 Authorization Code with PKCE](ko/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)로 인증해야 하며, 귀하의 앱을 승인한 사용자와 연결된 사용자 액세스 토큰을 사용해야 합니다. OAuth 1.0a로 이 사용자 액세스 토큰을 생성하려면 [3-legged OAuth flow](/ko/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)를 사용할 수 있습니다. OAuth 2.0으로 사용자 액세스 토큰을 생성하려면 [Authorization Code with PKCE grant flow](/ko/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce))를 사용할 수 있습니다.

<Note>
  **계정 설정**

  이 엔드포인트에 액세스하려면 다음이 필요합니다:

  - 승인된 [developer account](https://developer.x.com/en/portal/petition/essential/basic-info).
  - [Project](/ko/resources/fundamentals/projects) 내에 있는 [developer App](/ko/resources/fundamentals/developer-apps)의 키와 토큰을 사용하여 인증.

  X API v2 엔드포인트 액세스 권한 얻기에 대해 더 알아보려면 [getting started guide](/ko/x-api/getting-started/getting-access)를 참조하세요.
</Note>
