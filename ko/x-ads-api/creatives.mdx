<div id="overview">
  ## 개요
</div>

크리에이티브는 캠페인에서 프로모션할 수 있는 모든 엔터티를 말합니다. 게시물에는 텍스트, 이미지, GIF, 동영상 또는 카드가 포함될 수 있습니다. 카드에는 이미지 또는 동영상이 포함될 수 있습니다.

이미지, GIF, 동영상 크리에이티브는 [POST media/upload](/ko/x-api/media/upload-media#post-media-upload) — 이미지 업로드만 지원하는 간단한 업로드 엔드포인트 — 또는 POST media/upload (chunked) 엔드포인트로 업로드합니다. 업로드한 크리에이티브는 다음에 추가할 수 있습니다.

카드:

- POST accounts/:account\_id/cards
  트윗:
- POST accounts/:account\_id/tweets - 트윗에 카드를 추가하려면 card\_uri 매개변수를 사용하세요.
  예약 트윗:
- POST accounts/:account\_id/scheduled\_tweets

카드에 대한 자세한 내용은 Cards 페이지를 참조하세요. Promoted Video 페이지에서는 동영상을 카드 또는 트윗에 연결하는 방법을 안내합니다.

<div id="cards">
  ### 카드
</div>

X Ads API는 트윗에 사용할 수 있는 여러 카드 유형을 지원하며, 이후 캠페인에서 프로모션할 수 있습니다. 참고: 트윗으로 게시되면 카드 세부 정보는 공개적으로 표시됩니다. 여기에는 카드를 소유한 사용자에 대한 정보가 포함될 수 있습니다.

<div id="image">
  #### 이미지
</div>

다음 이미지 사양은 [Cards](/ko/x-ads-api/creatives#cards-2)에 사용되는 자산에 적용됩니다. 이미지는 3MB 이하여야 하며 너비가 최소 800px 이상이어야 합니다. 또한 다음 가로:세로 비율을 지원합니다.

- Website: 1:1 및 1.91:1
- Image App Download: 1:1 및 1.91:1
- Poll: 1.91:1
- Image Conversation: 1.91:1
- Image Direct Message: 1.91:1

지원하는 이미지 형식: .bmp, .jpeg, .png

<div id="video">
  #### 동영상
</div>

다음 동영상 사양은 [Cards](/ko/x-ads-api/creatives#cards-2)에 사용되는 자산에 적용됩니다. 지원되는 가로:세로 비율은 다음과 같습니다.

- 비디오 웹사이트: 16:9, 1:1
- 비디오 앱 다운로드: 16:9, 1:1
- 투표: 16:9
- 비디오 대화: 16:9
- 비디오 다이렉트 메시지: 16:9

<div id="promoted-video">
  ### 프로모션 동영상
</div>

이 문서는 Ads API를 통해 동영상을 업로드하고 프로모션하는 절차를 간략히 소개합니다.

Ads API는 [트윗](/ko/x-ads-api/creatives#tweets-2)과 다음 카드에서 프로모션 동영상을 지원합니다.

- [Video Website](https://devcommunity.x.com/t/ads-api-version-11/168814)
- [Video App Download](https://devcommunity.x.com/t/ads-api-version-11/168814)
- [Video Conversation](/ko/x-ads-api/creatives#video-conversation-cards)

먼저 [POST media/upload (chunked)](https://developer.x.com/en/docs/media/upload-media/api-reference/post-media-upload-init.html) 엔드포인트를 사용해 동영상을 업로드합니다. 그런 다음 `media_id`를 사용하여 POST accounts/:account\_id/videos 엔드포인트로 해당 동영상을 광고 계정과 연결합니다. 동영상의 `id`는 때때로 `media_key`라고도 하며, 이후 요청에서 사용됩니다. 이 값은 정수로 시작하고 밑줄이 뒤따르며, 긴 숫자 값으로 끝나는 문자열입니다. 예: `13_875943225764098048`.

<div id="promoted-video-in-tweets">
  #### 트윗 내 프로모션 동영상
</div>

트윗을 생성하려면 동영상의 `id`와 함께 [POST accounts/:account\_id/tweet](/ko/x-ads-api/creatives#get-accounts-account-id-tweets) 엔드포인트를 사용하세요. 이 단계에서 동영상 제목, 설명, 콜투액션(CTA)도 제공할 수 있습니다. 이러한 값은 사용자에게 표시됩니다.

<div id="promoted-video-in-cards">
  #### 카드에서 프로모트된 동영상
</div>

Video App Download 및 Video Conversation 카드는 포스터 이미지를 추가할 수 있습니다. 이러한 카드에서 사용할 이미지는 [POST media/upload](https://developer.x.com/en/docs/media/upload-media/api-reference/post-media-upload.html) 엔드포인트로 업로드하세요.

다음 엔드포인트 중 하나를 사용해 카드를 생성하세요:

- [POST accounts/:account\_id/cards/video\_website](https://devcommunity.x.com/t/ads-api-version-11/168814)
- [POST accounts/:account\_id/cards/video\_app\_download](https://devcommunity.x.com/t/ads-api-version-11/168814)
- [POST accounts/:account\_id/cards/video\_conversation](https://devcommunity.x.com/t/ads-api-version-11/168814)

동영상의 `id`와(필요한 경우) 이미지의 `media_id`(포스터 이미지)를 사용합니다.

마지막으로 POST accounts/:account\_id/tweet 엔드포인트로 트윗을 생성하세요. 카드는 `card_uri` 매개변수를 사용해 트윗에 첨부됩니다.

<div id="general-information">
  #### 일반 정보
</div>

API를 통한 동영상 업로드에 대한 자세한 안내는 [Video Upload Guide](https://developer.x.com/en/docs/media/upload-media/uploading-media.html)를 참고하세요.

동영상은 프리롤 자산으로도 프로모션할 수 있습니다. 자세한 내용은 [Video Views Pre-roll Objective Guide](/ko/x-ads-api/campaign-management#video-views-preroll-objective)를 참고하세요.

- (2015-10-22 기준) 프로모션용 콘텐츠에 사용할 동영상을 업로드할 때는 [POST media/upload (chunked)](https://developer.x.com/en/docs/media/upload-media/api-reference/post-media-upload-init.html) 엔드포인트에 대한 모든 `INIT` 명령 요청에서 `media_category` 값을 `amplify_video`로 설정해야 합니다. 이 새 매개변수를 사용하면 동영상이 비동기적으로 사전 처리되어 프로모션용 콘텐츠에 사용할 수 있도록 준비됩니다. 동영상 업로드 후 비동기 처리 완료 여부는 `STATUS` 명령으로 확인할 수 있습니다.
- 현재 허용되는 프로모션 동영상의 최대 길이는 10분이며, 파일 크기는 500MB 이하여야 합니다.
- 업로드하는 동영상은 mp4 또는 mov 형식이어야 합니다.
- 업로드된 동영상은 일반적으로 빠르게 처리되지만, 처리 시간은 동영상 길이와 파일 크기에 따라 달라질 수 있습니다.
- 업로드된 포스터 이미지는 png 또는 jpg 형식이어야 합니다. 가로세로비율이나 크기 요구 사항은 없지만, 포스터 이미지는 동영상 플레이어에 맞게 조정됩니다.

<div id="scheduled-tweets">
  ### 예약 트윗
</div>

<div id="introduction">
  #### 소개
</div>

예약 트윗은 광고주나 사용자가 나중에 게시되도록 트윗을 미리 만들어 예약할 수 있게 합니다. 이러한 트윗을 생성하고 관리하는 것에 더해, API는 트윗이 게시된 후 프로모션할 수 있도록 해당 트윗을 라인 아이템에 연동하는 기능을 제공합니다. 이를 통해 광고주는 네이티브 트윗을 사전에 준비하고 주요 이니셔티브 이전에 캠페인 크리에이티브를 계획할 수 있습니다. 예를 들어, 신제품 발표와 동시에 즉시 게시되도록 트윗 크리에이티브를 준비할 수 있습니다.

Scheduled Tweets API 엔드포인트가 제공하는 전체 기능은 다음과 같습니다:

- 새 예약 트윗 생성, 수정 및 조회
- 예약 트윗을 라인 아이템에 연동
- 기존 예약 트윗 조회 및 관리
- 예약 트윗이 게시된 후 라이브 트윗 `id` 조회

<div id="api-endpoints">
  #### API 엔드포인트
</div>

위 기능과 관련된 엔드포인트의 전체 목록은 다음과 같습니다.

<div id="scheduled-tweet-management">

- [GET accounts/:account\_id/scheduled\_tweets/:scheduled\_tweet\_id](/ko/x-ads-api/creatives#get-accounts-account-id-scheduled-tweets) (`id`로 특정 예약 트윗 조회)
- [POST accounts/:account\_id/scheduled\_tweets](/ko/x-ads-api/creatives#post-accounts-account-id-scheduled-tweets) (새 예약 트윗 생성)
- [PUT accounts/:account\_id/scheduled\_tweets/:scheduled\_tweet\_id](/ko/x-ads-api/creatives#example-request-39) (기존 예약 트윗 수정)

- [GET accounts/:account\_id/scheduled\_tweets/preview/:scheduled\_tweet\_id](/ko/x-ads-api/creatives#scheduled-tweets-2) (기존 예약 트윗 미리보기)

<div id="scheduled-promoted-tweets">
  #### 예약된 프로모션 트윗
</div>

- [GET accounts/:account\_id/scheduled\_promoted\_tweets](/ko/x-ads-api/campaign-management#get-accounts-account-id-scheduled-promoted-tweets) (모든 예약된 프로모션 트윗 목록을 조회)
- [GET accounts/:account\_id/scheduled\_promoted\_tweets/:scheduled\_promoted\_tweet\_id](/ko/x-ads-api/campaign-management#get-accounts-account-id-scheduled-promoted-tweets-scheduled-promoted-tweet-id) (`id`로 예약된 프로모션 트윗 조회)
- [POST accounts/:account\_id/scheduled\_promoted\_tweets](/ko/x-ads-api/campaign-management#post-accounts-account-id-scheduled-promoted-tweets) (새 예약된 프로모션 트윗 생성)
- [DELETE accounts/:account\_id/scheduled\_promoted\_tweets/:scheduled\_promoted\_tweet\_id](/ko/x-ads-api/campaign-management#delete-accounts-account-id-scheduled-promoted-tweets-scheduled-promoted-tweet-id) (`id`로 기존 예약된 프로모션 트윗 삭제)

<div id="scheduled-tweet-view">
  #### 예약된 트윗 보기
</div>

- [GET accounts/:account\_id/scheduled\_tweets/preview/:scheduled\_tweet\_id](/ko/x-ads-api/creatives#scheduled-tweets-2) (기존 예약된 트윗 보기)

예약된 트윗은 “라이브” 트윗과는 별도의 엔터티이므로, 이 트윗을 생성하거나 수정할 때 두 가지 서로 다른 검증이 수행됩니다. 첫 번째 검증 규칙 세트는 예약된 트윗 생성 단계에서, 구체적으로 다음과 같은 방식으로 실행됩니다:

<div id="scheduled-tweet-create">
  #### 예약된 트윗 생성:
</div>

- 인증된 사용자가 특정 @handle에 대해 오가닉 트윗을 생성할 수 있는 권한이 있는지 확인하세요. 프로모션 전용 트윗 생성 권한을 사용하려면, 인증된 사용자가 [Tweet composer permissions](https://developer.x.com/content/developer-twitter/en/docs/ads/campaign-management/api-reference/authenticated-user-access#get-accounts-account-id-authenticated-user-access)을 보유한 계정 사용자여야 합니다.
- scheduled\_at 시간 기준으로 앞뒤 15분 시간대에 생성 예정인 트윗이 30개를 넘지 않는지 확인하세요. SCHEDULED\_TWEET\_LIMIT\_EXCEEDED 오류 메시지는 동일한 미래의 15분 시간대에 예약된 트윗이 너무 많음을 나타냅니다. 광고주는 기존 예약된 트윗을 삭제하거나 scheduled\_at 시간을 앞당기거나 늦춰야 합니다.

<div id="scheduled-tweet-goes-live">
  #### 예약된 트윗이 실제로 게시됨:
</div>

- 이러한 유효성 검사 규칙은 scheduled\_at 시점에 실행되며, API에서 일반 트윗을 생성할 때 적용되는 규칙과 동일합니다. 예를 들어, 예약된 트윗에 이미지와 GIF가 모두 포함되어 있으면 게시되지 않으며, scheduled\_status가 FAILED로 설정됩니다

<div id="workflow">
  #### 워크플로
</div>

**새 예약 트윗 만들기**

새 예약 트윗은 [POST accounts/:account\_id/scheduled\_tweets](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/api-reference/scheduled-tweets#post-accounts-account-id-scheduled-tweets) 엔드포인트로 생성할 수 있습니다. 이 엔드포인트에는 필수 매개변수가 있으며, 트윗에 미디어 엔터티가 포함되지 않은 경우 트윗 `text`와 함께 `scheduled_at` 시간이 필요합니다. 또한 `as_user_id` 매개변수를 통해 다른 @handle을 대신해 예약 트윗을 생성할 수 있으며, 카드(`card_uri`)와 미디어(`media_ids`)를 추가하는 옵션도 제공합니다. 참고로 트윗에는 동일한 유형의 엔터티만 포함될 수 있습니다. 즉, Video, Gif 또는 Image 중 하나만 가능합니다. `nullcast` 매개변수는 트윗이 “Promoted-Only” 트윗인지 여부를 제어합니다. 새로 생성된 모든 예약 트윗은 기본적으로 "Promoted-Only"(`nullcast=true`)입니다. `nullcast=false`이면 오가닉 예약 트윗이 생성됩니다.

예약 트윗이 성공적으로 생성되면 응답에는 예약 트윗의 고유 식별자인 `id` 필드가 포함됩니다. 이외에도 `tweet_id`라는 필드가 반환되는데, 초기에는 `null`이며 트윗이 게시되면 이 필드가 “라이브” 트윗의 ID로 채워집니다.

```
twurl -H 'ads-api.x.com' -X POST "/6/accounts/:account_id/scheduled_tweets" -d 'scheduled_at=2017-12-24T23:59:00Z&text=Happy Holidays!'
```

이는 다음과 같은 예약 트윗을 생성합니다:

```json
{
  "request": {
    "params": {
      "text": "Happy Holidays!",
      "scheduled_at": "2017-12-24T23:59:00Z"
    }
  },
  "data": {
    "completed_at": null,
    "id_str": "917507899668099072",
    "text": "Happy Holidays!",
    "user_id": "3271358660",
    "scheduled_status": "SCHEDULED",
    "id": 917507899668099100,
    "nullcast": true,
    "created_at": "2017-10-09T21:51:44Z",
    "scheduled_at": "2017-12-24T23:59:00Z",
    "card_uri": null,
    "updated_at": "2017-10-09T21:51:44Z",
    "tweet_id": null,
    "media_keys": []
  }
}
```

이 예약 트윗이 게시되면 `tweet_id` 필드가 “라이브” 트윗의 ID로 채워집니다.

**예약 트윗 보기**

[GET accounts/:account\_id/tweet\_previews](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/api-reference/tweet-previews#get-accounts-account-id-tweet-previews) 엔드포인트를 이전 단계에서 받은 예약 트윗 `id`와 함께 사용하여 해당 트윗의 미리보기를 생성할 수 있습니다. API 응답에는 예약 트윗 미리보기를 렌더링하는 데 바로 사용할 수 있는 iframe URL이 포함됩니다. 관련 CSS와 이미지는 X에서 직접 제공됩니다.

```
twurl -H ads-api.x.com -X GET "/6/accounts/18ce54bgxky/tweet_previews?tweet_type=SCHEDULED&tweet_ids=917507899668099072
```

```json
{
  "request": {
    "params": {
      "tweet_type": "SCHEDULED",
      "tweet_ids": [
        "917507899668099072"
      ],
      "account_id": "18ce54bgxky"
    }
  },
  "data": [
```

**예약 트윗을 라인 아이템과 연결하기**

예약 트윗은 오가닉 트윗을 만드는 데 사용할 수 있지만, 파트너가 “프로모션 전용”(‘nullcast=true’) 트윗을 생성하도록 할 수도 있으며, 둘 중 어느 것이든 라인 아이템과 연결할 수 있습니다. 이를 위해 [POST accounts/:account\_id/scheduled\_promoted\_tweets](https://developer.x.com/content/developer-twitter/en/docs/ads/campaign-management/api-reference/scheduled-promoted-tweets#post-accounts-account-id-scheduled-promoted-tweets) 엔드포인트도 제공합니다. 이 엔드포인트는 단일 API 호출에서 하나의 프로모션 예약 트윗만 라인 아이템에 연결할 수 있습니다. 동일한 라인 아이템에 여러 예약 트윗을 연결하려면 여러 번의 API 호출이 필요합니다.

기존 프로모션 예약 트윗은 수정할 수 없습니다.

```
twurl -H 'ads-api.x.com' -X POST "/6/accounts/:account_id/scheduled_promoted_tweets" -d 'line_item_id=a44qc&scheduled_tweet_id=917507899668099072'
```

```json
{
  "data": {
    "line_item_id": "a44qc",
    "id": "26576",
    "created_at": "2017-10-09T22:24:16Z",
    "updated_at": "2017-10-09T22:24:16Z",
    "scheduled_tweet_id": "917507899668099072",
    "tweet_id": null,
    "deleted": false
  },
  "request": {
    "params": {
      "line_item_id": "a44qc",
      "scheduled_tweet_id": 917507899668099100,
      "account_id": "aaaaa"
    }
  }
}
```

이 엔드포인트는 지정된 예약 트윗과 라인 아이템 간의 연결만 생성합니다. 캠페인/라인 아이템의 플라이트 날짜가 현재가 되면 해당 라인 아이템은 해당 “라이브” 트윗을 자동으로 송출하기 시작합니다. 이 단계에서는 예약 트윗이 `SCHEDULED` 상태인지와 지정된 목표에 유효한지만 검증하며, 그 외의 검증은 수행하지 않습니다. 라인 아이템과 예약 트윗에 적용되는 나머지 검증 규칙은 트윗이 “라이브” 상태가 될 때 실행됩니다.

캠페인 송출에 문제가 없도록 하려면 예약 트윗의 `scheduled_at` 시간을 캠페인/라인 아이템의 플라이트 시작일 이전으로 설정하는 것이 좋습니다.

예를 들어, 예약 트윗이 캠페인 시작일 이후에 라이브로 전환되도록 설정되어 있고(하나의 라인 아이템에 하나의 트윗만 연결되어 있다고 가정) 캠페인이 `ACTIVE` 상태가 되더라도, 예약 트윗이 아직 라이브가 아니므로 송출 가능한 크리에이티브가 없게 됩니다.

**예약 트윗 관리**

아래의 나머지 엔드포인트는 API 사용자가 모든 예약 트윗과 프로모션 예약 트윗을 관리할 수 있도록 합니다. 이 API는 모든 예약 트윗 목록을 상태로 선택적으로 필터링하여 반환하거나, 특정 예약 트윗을 `id`로 조회하는 데 사용할 수 있습니다.

<div id="what-happens-when-a-scheduled-tweet-goes-live">
  #### 예약된 트윗이 라이브로 전환되면 어떻게 되나요?
</div>

특정 예약된 트윗이 라이브로 전환될 시점, 즉 `scheduled_at` 시간에 다음과 같은 업데이트가 이루어집니다:

- “라이브” 트윗이 생성되지만 최대 1초의 지연이 발생할 수 있습니다
- `tweet_id`가 다음 엔터티에 추가됩니다:
- 예약된 트윗
- 프로모션용 예약된 트윗
- 새로운 프로모션 트윗 엔터티가 생성됩니다

<div id="best-practices">
  #### 모범 사례
</div>

예약 트윗을 생성하거나 프로모션할 때는 다음 모범 사례를 권장합니다:

- 예약 트윗을 생성할 때 해당 트윗이 유효한지 확인하세요(예: 트윗에는 이미지, 동영상, GIF 중 하나만 포함할 수 있으며, 세 가지를 조합해 포함할 수 없음)
- 캠페인 운영 기간(예: `start_time`, `end_time`)이 해당 예약 트윗의 `scheduled_at` 시간과 일치하도록 설정하세요
- 예약 트윗은 최대 1년(365일) 이후까지로만 예약하세요
- 예약 트윗에 대한 트윗 미리보기는 현재 지원되지 않습니다(생성 전에 예약 트윗을 미리 보는 기능)

<div id="media-library">
  ### 미디어 라이브러리
</div>

<div id="introduction">
  #### 소개
</div>

Media Library 엔드포인트는 X Ads 계정에서 이미지, GIF, 동영상을 관리할 수 있는 기능을 제공합니다. 라이브러리에 있는 미디어 자산은 트윗에 사용하거나 카드를 생성하는 데 활용할 수 있습니다. 또한 여러 [크리에이티브](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/overview)에서 재사용할 수 있어 동일한 자산을 반복 업로드할 필요가 없습니다.

<div id="api-endpoints">
  #### API 엔드포인트
</div>

- [POST media/upload](https://developer.x.com/content/developer-twitter/en/docs/media/upload-media/api-reference/post-media-upload#post-media-upload) 또는 [POST media/upload (chunked)](https://developer.x.com/content/developer-twitter/en/docs/media/upload-media/api-reference/post-media-upload-init#post-media-upload-init) (미디어 업로드)
- [POST accounts/:account\_id/media\_library](https://developer.x.com/x-ads-api/creatives#get-accounts-account-id-media-library#post-accounts-account-id-media-library) (미디어 라이브러리에 미디어 추가)

<div id="adding-to-the-library">
  #### 라이브러리에 추가
</div>

라이브러리에 미디어를 추가하는 과정은 두 단계로 이루어집니다. 먼저 [POST media/upload](https://developer.x.com/content/developer-twitter/en/docs/media/upload-media/api-reference/post-media-upload#post-media-upload) 엔드포인트 또는 [POST media/upload (chunked)](https://developer.x.com/content/developer-twitter/en/docs/media/upload-media/api-reference/post-media-upload-init#post-media-upload-init) 엔드포인트 모음을 사용해 에셋을 업로드합니다. (멀티파트 업로드 절차에 대한 자세한 내용은 [Chunked media upload](https://developer.x.com/content/developer-twitter/en/docs/media/upload-media/uploading-media/chunked-media-upload) 가이드를 참고하세요.)

```
twurl -X POST -H upload.x.com "/1.1/media/upload.json?additional_owners=756201191646691328" --file latte.jpeg --file-field "media"
```

```json
{
   "media_id":966947208837742592,
   "media_id_string":"966947208837742592",
   "size":74194,
   "expires_after_secs":86400,
   "image":{
      "image_type":"image/jpeg",
      "w":800,
      "h":418
   }
}
```

<div id="request-parameters">
  #### 요청 매개변수
</div>

모든 Media Library POST 요청에는 미디어 식별자가 필요합니다. 이 값은 업로드 단계에서 반환됩니다. 위의 예처럼 media\_id를 사용할 때는 media\_category도 지정해야 합니다. 가능한 카테고리 값은 네 가지입니다: AMPLIFY\_VIDEO, TWEET\_GIF, TWEET\_IMAGE, TWEET\_VIDEO.

선택적으로 Media Library의 객체에 name과 file\_name 값을 설정할 수 있습니다. 이러한 속성은 라이브러리에서 미디어 변형을 구분하는 데 도움이 됩니다.

비디오의 경우 title과 description을 설정할 수도 있습니다. 이러한 값은 [POST accounts/:account\_id/tweet](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/api-reference/tweets#post-accounts-account-id-tweet) 엔드포인트의 요청 매개변수인 video\_title 및 video\_description으로 전달하도록 되어 있습니다. 트윗에서는 이 텍스트가 비디오 아래에 표시됩니다.

<div id="attributes">
  #### 속성
</div>

Media Library는 media\_key 개념을 공식적으로 도입합니다. 이는 라이브러리 내 객체의 고유 식별자입니다. 미디어 키는 다음 형식의 문자열 값입니다: 13\_875943225764098048. 이는 모든 카드 엔드포인트에서 완전히 지원됩니다.

또한 Media Library 응답에는 문자열로 표현된 media\_id가 포함됩니다. 이는 현재 미디어 키를 아직 허용하지 않는 리소스에 대해 제공됩니다: [Tweets](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/api-reference/tweets#post-accounts-account-id-tweet)\*, [Tweet preview](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/api-reference/tweets#get-accounts-account-id-tweet-preview)\*, 및 [Scheduled Tweets](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/api-reference/scheduled-tweets#post-accounts-account-id-scheduled-tweets). 당사는 모든 곳에서 미디어 키 지원을 제공하기 위해 작업하고 있습니다.

aspect\_ratio 속성은 GIF와 동영상에 대해 반환됩니다. 이는 특정 종횡비만 허용하는 카드에서 사용할 미디어를 필터링하는 데 활용할 수 있습니다.

\*이들 엔드포인트는 미디어 키인 video\_id 매개변수를 지원합니다.

<div id="identifying-cards">
  ### 카드 식별하기
</div>

<div id="introduction">
  #### 소개
</div>

카드는 미디어를 사용하는 사용자 지정 가능한 광고 형식으로, 웹사이트, 앱 또는 특정 사용자 참여(예: 다이렉트 메시지 시작)를 유도하는 행동 유도 문구(CTA)와 연계할 수 있습니다. 트윗, 예약 트윗 또는 트윗 초안에 첨부할 수 있습니다.

카드는 트윗 객체에서 두 가지 방식으로 참조될 수 있습니다. 카드의 card\_uri로 참조하거나 preview\_url로 참조하는 방법입니다. 각 항목의 예시 값은 아래와 같습니다.

| card\_uri | preview\_url |
| :--- | :--- |
| card://1043282691834048513 | https://cards.x.com/cards/18ce54d4x5t/68w3s |

**참고**: Ads API 버전 3부터 새로 생성된 카드에 대해서는 cards 응답에 card\_uri만 생성되어 반환됩니다.

**참고**: Ads API 버전 5부터 cards 응답에 preview\_url이 더 이상 반환되지 않습니다.

트윗 객체 응답에서 어떤 방식으로 참조되는지는 트윗의 생성 방식에 따라 달라집니다. 즉, card\_uri 요청 매개변수로 트윗을 생성한 경우 응답에 card URI 값이 표시됩니다. 반면 트윗 텍스트에 preview\_url을 포함했다면 응답에는 preview URL이 표시됩니다.

#### preview\_url로 트윗 식별

예약 또는 초안 트윗 객체의 경우, 미리보기 URL은 항상 텍스트 필드에 표시됩니다.

<div id="identifying-media">
  ### 미디어 식별하기
</div>

<div id="introduction">
  #### 소개
</div>

미디어(이미지, GIF, 동영상)는 트윗과 카드에 추가할 수 있습니다. 또한 동영상은 프리롤 자산으로 사용할 수 있고, 이미지는 [X Audience Platform](https://developer.x.com/en/docs/ads/measurement/overview/twitter-audience-platform)에서 프로모션할 수 있습니다. 이 섹션에서는 이러한 엔터티 전반에서 미디어 참조를 찾는 방법을 설명합니다.

미디어 식별자에는 두 가지 유형이 있습니다: ID와 키. 각 유형의 예시 값은 아래와 같습니다.

| **Media ID** | **Media key** |
| :--- | :--- |
| 1029825579531807971 | 13\_1029825579531807971 |

미디어 키는 ID 앞에 숫자 접두사와 밑줄이 붙은 형태입니다.

<div id="image">
  #### 이미지
</div>

다음 표는 각 이미지 관련 리소스의 응답에서 현재 사용 가능한 식별자 유형과 해당 속성 이름을 보여줍니다.

| **리소스** | **식별자** | **속성** |
| :--- | :--- | :--- |
| 이미지 카드 | 없음 |  |
| 트윗 | 둘 다 | `entities["media"]["id_str"]`  `entities["media"]["media_key"]` |
| 예약된 트윗 | 둘 다 | `media_ids` 및 `media_keys` |
| 임시 저장 트윗 | 둘 다 | `media_ids` 및 `media_keys` |
| 계정 미디어 | 없음 |  |
| 미디어 라이브러리 | 둘 다 | `media_id` 및 `media_key` |

이미지 카드와 계정 미디어 이미지에는 어떤 미디어 식별자에 대한 참조도 포함되지 않습니다. 트윗에는 미디어 ID만 포함됩니다. 예약된 트윗과 임시 저장 트윗에는 미디어 ID와 미디어 키가 모두 포함됩니다. 미디어 라이브러리도 둘 다 반환합니다.

트윗의 경우 entities\["media"] 배열 내 객체의 id 및 id\_str 필드는 미디어 ID에 해당합니다. 트윗에 여러 이미지가 포함된 경우, 각 미디어 엔터티에 대한 참조는 extended\_entities\["media"]에서만 찾을 수 있습니다.

식별자 참조 외에도 이미지의 URL에 접근할 수 있는 것이 중요한 경우가 많습니다.

| **리소스** | **속성** | **형식** |
| :--- | :--- | :--- |
| 이미지 카드 | `image` | .jpg |
| 트윗\* | `entities["media"][0]["media_url"]` 또는 `extended_entities["media"][i]["media_url"]` | .jpg |
| 예약된 트윗 | 없음 |  |
| 임시 저장 트윗 | 없음 |  |
| 계정 미디어 | `media_url` | .jpg |
| 미디어 라이브러리 | `media_url` | .jpg |

\* URL 위치는 트윗에 단일 이미지가 있는지, 여러 이미지가 있는지에 따라 달라집니다.

모든 이미지 카드에는 X 이미지 URL을 담는 image 응답 속성이 포함됩니다. (이미지 앱 다운로드 카드의 경우 이름은 wide\_app\_image입니다.)

트윗의 경우 미디어 URL 위치는 미디어 유형과 사용 중인 엔드포인트 모두에 따라 달라집니다. 단일 이미지를 포함하는 트윗에서는 URL을 entities\["media"]\[0]\["media\_url"]에서 찾을 수 있습니다. 이는 Ads API와 Standard API 모두에 해당합니다. 반면 트윗에 여러 이미지가 포함된 경우 URL은 extended\_entities\["media"]\[i]\["media\_url"]에서만 찾을 수 있으며, 이는 Standard API에서만 제공됩니다.

<div id="video">
  #### 동영상
</div>

아래 표는 각 동영상 관련 리소스의 응답에서 현재 사용 가능한 식별자 유형과 해당 속성 이름을 보여줍니다.

| **리소스** | **식별자** | **속성** |
| :--- | :--- | :--- |
| 동영상 카드 | 둘 중 하나 | `video_content_id` |
| 동영상 설문 카드 | 없음 |  |
| 트윗 | 둘 다 | `entities["media"]["id_str"]`  `entities["media"]["media_key"]` |
| 예약된 트윗 | 둘 다 | `media_ids` 및 `media_keys` |
| 임시 저장 트윗 | 둘 다 | `media_ids` 및 `media_keys` |
| 계정 미디어 | 미디어 키 | `video_id` |
| 미디어 라이브러리 | 둘 다 | `media_id` 및 `media_key` |

동영상 카드는(동영상이 포함된 설문 카드는 예외) video\_content\_id 응답 속성을 포함하지만, 반환되는 값의 유형에는 일관성이 없습니다. 어떤 경우에는 미디어 ID이고, 다른 경우에는 미디어 키입니다.

동영상 URL에 접근하는 방법은 아래와 같습니다.

| **리소스** | **속성** | **형식** |
| :--- | :--- | :--- |
| 동영상 카드 | `video_url` 및 `video_hls_url` | .vmap  .m3u8 |
| 동영상이 포함된 트윗 | `extended_entities["media"][i]["video_info"]["variants"][j]["url"]` | .mp4 |
| 예약된 트윗 | 없음 |  |
| 임시 저장 트윗 | 없음 |  |
| 계정 미디어 | 없음 |  |
| 미디어 라이브러리 | `media_url` | .mp4 |

동영상 카드에는 각각 .vmap 및 .m3u8 URL이 있는 `video_url` 및 `video_hls_url` 응답 속성이 포함됩니다.

<div id="media-library">
  #### 미디어 라이브러리
</div>

때로는 미디어 에셋에 대한 추가 정보를 조회해야 할 때가 있습니다. 예를 들어 비디오 카드의 경우 vmap URL 대신 mp4 URL을 가져와야 할 수 있습니다. 이는 미디어 라이브러리에서 가능합니다. 제공되는 정보에 대한 자세한 내용은 [미디어 라이브러리 가이드](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/guides/media-library)를 참조하세요. 광고 계정의 FULL 프로모션 가능 사용자에 속하는 대부분의 에셋은 라이브러리에서 찾을 수 있습니다. 다만 몇 가지 예외가 있습니다.

**미디어 가져오기**

앞서 언급했듯이 이미지 카드는 media ID나 media key에 대한 참조를 포함하지 않습니다. 따라서 미디어 라이브러리를 통해 해당 에셋을 가져올 수 없습니다. 이는 [Account Media](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/api-reference/account-media#account-media) 이미지에도 해당됩니다.

비디오 카드는 생성하기 전에 비디오 에셋이 미디어 라이브러리(또는 이전의 Videos 리소스)에 등록되어 있어야 합니다. 따라서 이러한 에셋은 항상 미디어 라이브러리에서 조회할 수 있습니다. 이는 Account Media PREROLL 에셋에도 해당됩니다.

마지막으로, 트윗에 포함된 미디어는 항상 미디어 라이브러리에 존재함이 보장됩니다.

다음 표는 조회에 사용할 식별자가 리소스 응답에 포함되는지 여부를 고려하여, 미디어 라이브러리에서 어떤 에셋을 조회할 수 있는지 요약합니다.

| **리소스** | **미디어 라이브러리에 포함 여부** |
| :--- | :--- |
| 이미지 카드 | 아니오 |
| 비디오 카드 | 예\* |
| 트윗(모든 미디어)\*\* | 예 |
| 예약된 트윗 | 예 |
| 초안 트윗 | 예 |
| Account media 이미지 | 아니오 |
| Account media 비디오(`PREROLL`) | 예 |

\* `video_content_id`가 media key인 카드의 경우. 값이 media ID인 경우에도 에셋은 미디어 라이브러리에 존재하지만, 가져오려면 숫자 접두사와 밑줄을 앞에 추가해야 합니다.
\*\* 트윗은 media ID만 반환합니다. 에셋이 미디어 라이브러리에 존재하는 것은 보장되지만, 가져오려면 숫자 접두사와 밑줄을 앞에 추가해야 합니다.

**Account Media와의 상호작용**

라이브러리에 추가된 미디어 에셋이 자동으로 Account Media 리소스에 추가되는 경우는 두 가지입니다.

- AMPLIFY\_VIDEO 에셋이 미디어 라이브러리에 추가되면 PREROLL 크리에이티브 유형의 Account Media 에셋으로 자동 추가됩니다.
- 특정 해상도(당사의 [열거 페이지](https://developer.x.com/content/developer-twitter/en/docs/ads/general/overview/enums)의 "Creative Types" 참조)의 이미지를 미디어 라이브러리에 추가하면 자동으로 Account Media 에셋으로 추가됩니다. 크리에이티브 유형(예: INTERSTITIAL)은 이미지 해상도에 따라 달라집니다.

<div id="tweets">
  ### 트윗
</div>

<div id="introduction">
  #### 소개
</div>

X Ads API는 트윗을 게시됨, 예약됨, 초안의 세 가지 유형으로 지원합니다.

<div id="nullcasted-tweets">
  #### Nullcasted 트윗
</div>

트윗은 nullcasted(또는 "Promoted-only")이거나 오가닉일 수 있습니다. Nullcasted 트윗은 게시된 후에도 공개 상태이지만 사용자의 공개 타임라인에는 표시되지 않습니다. 반면 오가닉 트윗은 사용자의 팔로워에게 노출되며 사용자의 공개 타임라인에 표시됩니다.

**트윗 생성**

세 가지 트윗 생성 엔드포인트 각각은 Boolean 형식의 nullcast 파라미터를 지원하여, API 사용자가 nullcasted 또는 오가닉 트윗을 선택해 생성할 수 있습니다. Nullcasted 트윗은 사용자 본인 또는 사용자를 대신해 트윗 생성 권한이 있는 누구나 생성할 수 있습니다. 오가닉 트윗은 [full promotable user](https://developer.x.com/content/developer-twitter/en/docs/tutorials/ads-api-hierarchy-terminology#promotable-users)만 생성할 수 있습니다.

**트윗 업데이트**

예약 및 초안 트윗의 nullcast 속성은 업데이트할 수 있습니다. 예약 트윗의 경우 트윗의 scheduled\_at 시간까지 수정할 수 있습니다. 초안 트윗은 제한 없이 수정할 수 있습니다. 다만 게시된 후에는 트윗을 nullcasted에서 오가닉으로, 또는 그 반대로 변경할 수 없습니다.

<div id="promoting-tweets">
  #### 트윗 프로모션
</div>

게시되었거나 예약된 트윗만 프로모션할 수 있습니다. 둘 다 nullcast 또는 오가닉일 수 있으며 제한은 없습니다. 광고주는 권한을 얻은 경우 자신의 트윗은 물론 다른 사용자의 트윗도 프로모션할 수 있습니다. (자세한 내용은 [다른 사용자의 트윗 프로모션](https://developer.x.com/content/developer-twitter/en/docs/tutorials/promoting-another-users-content)을 참조하세요.)

하나의 캠페인에서 여러 트윗을 프로모션할 수 있습니다. 마찬가지로, 하나의 트윗을 하나 이상의 캠페인에서 프로모션할 수 있습니다.

게시된 트윗을 프로모션하려면 [POST accounts/:account\_id/promoted\_tweets 엔드포인트](https://developer.x.com/content/developer-twitter/en/docs/ads/campaign-management/api-reference/promoted-tweets#post-accounts-account-id-promoted-tweets)를 사용하세요. 게시된 트윗을 라인 아이템과 연결합니다. 예약된 트윗을 프로모션하려면 [POST accounts/:account\_id/scheduled\_promoted\_tweets](https://developer.x.com/content/developer-twitter/en/docs/ads/campaign-management/api-reference/scheduled-promoted-tweets#post-accounts-account-id-scheduled-promoted-tweets) 엔드포인트를 사용하세요.

<div id="tweet-ids">
  #### 트윗 ID
</div>

게시됨, 예약됨, 초안 상태의 트윗 ID는 [숫자형](https://developer.x.com/content/developer-twitter/en/docs/basics/twitter-ids)이며 64비트 부호 없는 정수입니다. 예를 들어, 다음에 나온 게시된 트윗의 ID는 1166476031668015104입니다.

게시되었거나 예약된 트윗을 프로모션하면 해당 프로모션 트윗 엔터티가 생성됩니다. 이 엔터티에는 자체 ID가 있으며, 영숫자로 이루어졌고 base-36으로 인코딩된 값으로 표현됩니다. 예를 들어, 위의 게시된 트윗을 프로모션—즉, 라인 아이템 6c62d에 연결—하면 다음과 같은 API 응답이 반환됩니다.

```
$ twurl -X POST -H ads-api.x.com "/9/accounts/18ce54d4x5t/promoted_tweets?line_item_id=6c62d&tweet_ids=1166476031668015104"
{
  "request": {
    "params": {
      "tweet_ids": [
        1166476031668015104
      ],
      "line_item_id": "6c62d",
      "account_id": "18ce54d4x5t"
    }
  },
  "data": [
    {
      "line_item_id": "6c62d",
      "id": "3qwlq6",
      "entity_status": "ACTIVE",
      "created_at": "2019-09-12T21:39:10Z",
      "updated_at": "2019-09-12T21:39:10Z",
      "approval_status": "ACCEPTED",
      "tweet_id": "1166476031668015104",
      "deleted": false
    }
  ],
  "total_count": 1
}
```

생성 요청에 포함된 트윗 ID와 라인 아이템 ID 외에, 응답에는 값이 3qw1q6인 id 필드도 포함되어 있으며, 이는 프로모션 트윗 ID입니다.

<div id="carousels">
  ### 캐러셀
</div>

<div id="introduction">
  #### 소개
</div>

X Ads API는 동영상 캐러셀과 이미지 캐러셀의 생성 및 조회를 지원합니다. 캐러셀은 2개에서 6개의 미디어 자산을 담을 수 있는 카드 유형입니다. 캐러셀 카드는 사용자를 웹사이트로 유도하거나 모바일 앱 설치를 권장할 수 있습니다. 캐러셀의 개요, 이점, 모범 사례, FAQ는 [Carousel Ads on X](https://business.x.com/en/advertising/carousels.html) 페이지를 참고하세요.

캐러셀은 다른 카드 유형과 마찬가지로 트윗에 사용할 수 있으며, 해당 트윗을 프로모션할 수 있습니다. 워크플로는 이미 익숙하신 것과 동일합니다:

1. 미디어 업로드
2. 카드 생성
3. 트윗 생성
4. 트윗 프로모션

유일한 차이점은 카드 생성 방식입니다. 다른 카드 생성 요청은 쿼리 매개변수를 허용하는 반면, 캐러셀 카드 생성 요청은 JSON POST 본문만 허용합니다.

<div id="endpoints">
  #### 엔드포인트
</div>

X Ads API는 캐러셀 생성과 조회를 지원합니다.

캐러셀을 생성하려면(종류와 관계없이) [POST accounts/:account\_id/cards](https://developer.x.com/en/docs/twitter-ads-api/creatives/api-reference/cards#post-accounts-account-id-cards) 엔드포인트를 사용하세요. 캐러셀을 조회하려면 [GET accounts/:account\_id/cards](https://developer.x.com/en/docs/twitter-ads-api/creatives/api-reference/cards#get-accounts-account-id-cards) 엔드포인트를 사용하세요.

<div id="json-post-body">
  #### JSON POST 본문
</div>

캐러셀은 두 개의 구성요소로 생성합니다. 첫 번째는 사용할 미디어 자산을 지정하고, 두 번째는 웹사이트 또는 앱에 대한 정보를 지정합니다.

구체적으로, 캐러셀 카드는 다음 구성요소를 아래 순서로 사용하여 생성합니다.

- 미디어 키 배열을 받는 `SWIPEABLE_MEDIA` 구성요소 1개
- 다음 중 _하나_:
- 웹사이트 정보를 지정하는 `DETAILS` 구성요소
- 앱 정보를 지정하는 `BUTTON` 구성요소

`SWIPEABLE_MEDIA` 구성요소에는 `media_keys` 배열이 포함되어야 하며, 여기에 2개에서 6개 사이의 이미지 또는 동영상을 지정할 수 있습니다. 전달한 미디어 키의 순서가 렌더링 순서를 결정합니다.

```
{
  "type": "SWIPEABLE_MEDIA",
  "media_keys": [
    "13_1089771253848666112",
    "13_1191948012077092867"
  ]
}
```

참고로, [GET accounts/:account\_id/media\_library](https://developer.x.com/en/docs/twitter-ads-api/creatives/api-reference/media-library#get-accounts-account-id-media-library) 엔드포인트에 요청하여 미디어 키를 얻을 수 있습니다.

두 번째 구성요소 객체의 구성은 사용자를 웹사이트로 이동시키려는지, 아니면 앱 설치를 유도하려는지에 따라 달라집니다. 아래 표는 두 옵션을 요약합니다. (참고: 나열된 모든 키는 필수입니다.)

|  | **Website** | **App** |
| :--- | :--- | :--- |
| 구성요소 유형 지정 | `"type": "DETAILS"` | `"type": "BUTTON"` |
| 제목/레이블 | `"title": "X"` | `"label": { "type": "ENUM", "value": "INSTALL" }` |
| 목적지 | `"destination": { "type": "WEBSITE", "url": "https://www.x.com" }` | `"destination": { "type": "APP", ... }` |

이를 종합하면, 웹사이트 캐러셀 JSON POST 본문 예시는 다음과 같습니다.

```
{
  "name": "website carousel",
  "components": [
    {
      "type": "SWIPEABLE_MEDIA",
      "media_keys": [
        "13_1089771253848666112",
        "13_1191948012077092867"
      ]
    },
    {
      "type": "DETAILS",
      "title": "X",
      "destination": {
        "type": "WEBSITE",
        "url": "https://www.x.com"
      }
    }
  ]
}
```

`BUTTON` 구성요소의 앱 목적지 객체에는 국가 코드와 최소 하나의 앱 식별자가 필요합니다. 선택적으로 딥 링크를 받을 수 있습니다. 해당 필드에 대한 설명은 [참조 문서](https://developer.x.com/en/docs/twitter-ads-api/creatives/api-reference/cards#app-destination)를 참고하세요.

이를 종합하면, 앱 캐러셀 JSON POST 본문 예시는 다음과 같습니다.

```
{
  "name": "app carousel",
  "components": [
    {
      "type": "SWIPEABLE_MEDIA",
      "media_keys": [
        "13_1089771253848666112",
        "13_1191948012077092867"
      ]
    },
    {
      "type": "BUTTON",
      "label": {
        "type": "ENUM",
        "value": "INSTALL"
      },
      "destination": {
        "type": "APP",
        "country_code": "US",
        "googleplay_app_id": "com.twitter.android",
        "iphone_app_id": "333903271"
      }
    }
  ]
}
```

<div id="creative-metadata-tagging">
  ### 크리에이티브 메타데이터 태깅
</div>

<div id="introduction">
  #### 소개
</div>

이 가이드는 크리에이티브 파트너, 에이전시, 크리에이티브 개발자가 X 캠페인에서 사용되는 자산에 태그를 지정하여 개별 자산의 가치와 성과를 더 정확히 파악할 수 있도록 돕기 위한 것입니다.

**참고:** 미디어 자산은 해당 자산을 생성한 파트너 또는 개발자만 태그해야 합니다. 미디어 자산의 사용자가 해당 자산을 직접 생성하지 않았다면 메타데이터 태깅을 적용하지 마십시오.

Creative Metadata Tagging은 자산이 X의 어느 위치에 업로드되었는지, 또는 누가 업로드했는지와 무관하게 크리에이티브 파트너가 제작한 이미지와 동영상에 대한 공로를 귀속합니다. 크리에이티브 자산과 크리에이티브 파트너를 연결하기 위해 [XMP](https://exiftool.org/TagNames/XMP.html) 표준을 사용합니다.

<div id="tagging-creative-assets">
  #### 크리에이티브 에셋 태깅
</div>

다음 표는 각 이미지 관련 리소스의 응답에서 현재 사용 가능한 식별자 유형과 해당 속성 이름을 보여줍니다. 크리에이티브 에셋을 태깅하려면 태깅 도구가 필요합니다. [ExifTool](https://exiftool.org/)은 플랫폼 독립적인 Perl 라이브러리이자 메타정보 읽기/쓰기/편집을 위한 [명령줄 애플리케이션](https://exiftool.org/exiftool_pod.html)으로, 사용을 권장합니다. 지원되는 모든 [파일 형식](https://exiftool.org/#supported)을 확인하세요.

제공된 [ExifTool 설치 지침](https://exiftool.org/install.html)을 따르세요. 또한 [Homebrew](https://formulae.brew.sh/)에서 macOS와 Linux용 [exiftool 설치 명령](https://formulae.brew.sh/formula/exiftool)을 제공하는 패키지가 있어 설치를 더 간편하게 할 수 있습니다. 명령줄에서 exiftool -ver를 입력해 도구의 버전 번호가 반환되는지 확인하여 정상 설치를 확인하세요. ExifTool 명령 파라미터에 대해서는 [ExifTool 문서](https://exiftool.org/)를 참고하세요.

크리에이티브 파트너는 신규 또는 기존 크리에이티브 에셋에 대해 자신의 X app\_id를 Contributor XMP 태그와 Date 태그에 메타데이터로 지정할 수 있습니다. 크리에이티브 에셋은 [미디어 업로드](https://developer.x.com/en/docs/twitter-api/v1/media/upload-media/overview) 시 기존 크기 제한을 따릅니다. 

**참고:** X에서 Contributor XMP 태그를 사용하면 메타데이터가 X에서 진행되는 캠페인에 관한 값만을 캡처하도록 보장합니다.

`exiftool -contributor="<YOUR APP ID>" -creative_file.jpg`

`exiftool -date="<date>" -creative_file.jpg`

app\_id는 [Developer Portal](https://developer.x.com/en/portal/dashboard)의 Projects & Apps에서 확인할 수 있습니다. 예: 16489123

다음 예시는 이미지에 대해 app\_id를 Contributor 태그로, date를 Date 태그로 추가합니다:

```
app_id:858382169
```

```
date:2022-03-13
```

```
creative_file: eiffel_tower.jpg
```

```
exiftool -contributor=858382169 eiffel_tower.jpg
```

```
   1 image files updated
```

```
exiftool -date=2022-03-13 eiffel_tower.jpg
```

```
  1 image files updated
```

이미지가 올바르게 태깅되었는지 확인하세요: 

`exiftool -xmp:all -G1 <filename>`

예시:

`exiftool -xmp:all -G1 eiffel_tower.jpg`

```
[XMP]        XMP Toolkit            : Image::ExifTool 12.30
```

```
[XMP]        Contributor            : 858382169
```

```
[XMP]        Date                   : 2022:03:13
```

<div id="parameters">

| 이름 | 설명 |
| :--- | :--- |
| account\_id   _필수_ | 사용 중인 계정의 식별자입니다. 리소스 경로에 포함되며, [GET accounts](https://developer.x.com/en/docs/ads/campaign-management/api-reference/accounts#get-accounts)를 제외한 대부분의 Advertiser API 요청에서 일반적으로 필요한 매개변수입니다. 지정한 계정은 인증된 사용자와 연결되어 있어야 합니다.  유형: string  예시: `18ce54d4x5t` |
| account\_media\_ids   _선택 사항_ | 쉼표로 구분된 식별자 목록을 지정하여 원하는 계정 미디어로만 응답 범위를 제한합니다. 최대 200개의 ID를 제공할 수 있습니다.  유형: string  예시: `3wpx` |

<div id="components">
  #### 구성 요소
</div>

모든 구성 요소에는 객체의 스키마를 결정하는 `type` 필드가 포함되어야 합니다. Ads API는 다음 구성 요소 유형을 지원하며, 미디어 기반 구성 요소와 설명 기반 구성 요소로 구분됩니다.

- 미디어:
- `MEDIA`: 단일 동영상 또는 이미지
- `SWIPEABLE_MEDIA`: 2~6개의 동영상 또는 이미지
- 설명:
- `DETAILS`
- `BUTTON`

각 구성 요소에는 필수 필드 세트가 있습니다(`type` 키 외). 다음 표에 나열되어 있습니다.

| 구성 요소 `type` | 필드 | 값 유형 |
| :--- | :--- | :--- |
| `MEDIA` | `media_key` | string |
| `SWIPEABLE_MEDIA` | `media_keys` | array of strings |
| `DETAILS` | `title`   `destination` | string   object |
| `BUTTON` | `label`   `destination` | object   object |

다음은 `components` 배열 내 `BUTTON` 구성 요소의 예시입니다(`name` 키는 의도적으로 생략). (줄임표는 추가 정보가 필요한 위치를 나타냅니다.)

```
{
  "components": [
    {
      "type": "BUTTON",
      "label": {
        ...
      },
      "destination": {
        ...
      }
    }
  ]
}
```

구성 요소 객체가 지정된 순서는 렌더링 시 위에서 아래로의 표시 순서를 결정합니다. 카드는 하나의 미디어 기반 구성 요소와 `DETAILS` 또는 `BUTTON` 구성 요소 중 하나를 사용하여 생성해야 합니다. 설명 기반 구성 요소는 미디어 아래에 렌더링되며 URL 또는 모바일 앱과 같은 목적지와 연결됩니다.

**Label**

Label은 버튼에 표시되는 텍스트를 정의하므로 `BUTTON` 구성 요소에만 적용됩니다. Label 객체에는 `type`과 `value` 두 개의 필수 키가 있습니다. `type`은 `ENUM`으로 설정해야 하며, `value`는 `BOOK`, `CONNECT`, `INSTALL`, `OPEN`, `ORDER`, `PLAY`, `SHOP` 중 하나여야 합니다.

이전 예시를 바탕으로, 아래는 `BUTTON` 구성 요소 내 `label` 객체입니다.

```
{
  "components": [
    {
      "type": "BUTTON",
      "label": {
        "type": "ENUM",
        "value": "INSTALL"
      },
      "destination": {
        ...
      }
    }
  ]
}
```

**Destination**

Destination은 광고주가 사용자를 이동시키려는 대상입니다. `DETAILS` 또는 `BUTTON` 구성 요소 내에서 항상 필요합니다. 목적지 유형은 `WEBSITE` 또는 `APP` 두 가지입니다.

**참고**: 웹사이트 목적지는 `DETAILS` 구성 요소에서만 사용할 수 있고, 앱 목적지는 `BUTTON` 구성 요소에서만 사용할 수 있습니다.

**Website Destination**

| 이름 | 설명 |
| :--- | :--- |
| type   _required_ | 목적지 유형으로, 해당 스키마를 결정합니다.  Type: enum  Possible values: `WEBSITE` |
| url   _required_ | 사용자를 리디렉션할 웹사이트의 URL입니다.  Type: string  Example: `https://devcommunity.x.com/c/advertiser-api` |

**App Destination**

| 이름 | 설명 |
| :--- | :--- |
| type   _required_ | 목적지 유형으로, 해당 스키마를 결정합니다.  Type: enum  Possible values: `APP` |
| country\_code   _required_ | 앱이 판매되는 국가의 [ISO 3166-1 alpha-2](https://en.wikipedia.org/wiki/ISO_3166-1_alpha-2) 두 글자 코드.  Type: string  Example: `US` |
| googleplay\_app\_id   _sometimes required_ | Google Play 애플리케이션 패키지 이름.  **참고**: `ios_app_store_identifier` 또는 `googleplay_app_id` 중 하나는 반드시 제공되어야 합니다.  Type: string  Example: `com.twitter.android` |
| ios\_app\_store\_identifier   _sometimes required_ | iOS App Store 식별자.  **참고**: `ios_app_store_identifier` 또는 `googleplay_app_id` 중 하나는 반드시 제공되어야 합니다.  Type: string  Example: `333903271` |
| googleplay\_deep\_link   _optional_ | 홍보 중인 Android 앱으로 연결되는 딥 링크.  **참고**: `googleplay_app_id`가 제공된 경우에만 사용할 수 있습니다.  Type: string |
| ios\_deep\_link   _optional_ | 홍보 중인 iOS 앱으로 연결되는 딥 링크.  **참고**: `ios_app_store_identifier`가 제공된 경우에만 사용할 수 있습니다.  Type: string |

#### GET accounts/:account\_id/preroll\_call\_to\_actions

현재 계정의 라인 아이템과 연동된 프리롤 Call-To-Action(CTA)의 일부 또는 전체에 대한 세부 정보를 조회합니다.

<div id="resource-url">
  #### 리소스 URL
</div>

`https://ads-api.x.com/12/accounts/:account_id/preroll_call_to_actions`

<div id="parameters">
  #### 매개변수
</div>

| 이름 | 설명 |
| :--- | :--- |
| account\_id   _필수_ | 사용 중인 계정의 식별자입니다. 리소스 경로에 포함되며, [GET accounts](https://developer.x.com/en/docs/ads/campaign-management/api-reference/accounts#get-accounts)를 제외한 대부분의 Advertiser API 요청에서 일반적으로 필요한 매개변수입니다. 지정한 계정은 인증된 사용자와 연결되어 있어야 합니다.  유형: string  예시: `18ce54d4x5t` |
| line\_item\_ids   _선택 사항_ | 쉼표로 구분된 식별자 목록을 지정하여 해당 라인 아이템과 연결된 프리롤 CTA로만 응답을 제한합니다. 최대 200개의 ID를 제공할 수 있습니다.  유형: string  예시: `8v53k` |
| count   _선택 사항_ | 각 요청마다 조회할 레코드 수를 지정합니다.  유형: int  기본값: `200`   최소, 최대: `1`, `1000` |
| cursor   _선택 사항_ | 다음 결과 페이지를 가져오기 위한 커서를 지정합니다. 자세한 내용은 [Pagination](https://developer.x.com/x-ads-api/introduction)을 참조하세요.  유형: string  예시: `8x7v00oow` |
| preroll\_call\_to\_action\_ids   _선택 사항_ | 쉼표로 구분된 식별자 목록을 지정하여 원하는 프리롤 CTA로만 응답을 제한합니다. 최대 200개의 ID를 제공할 수 있습니다.  유형: string  예시: `8f0` |
| sort\_by   _선택 사항_ | 지원되는 속성을 기준으로 오름차순 또는 내림차순으로 정렬합니다. 자세한 내용은 [Sorting](https://developer.x.com/x-ads-api/fundamentals/sorting)을 참조하세요.  유형: string  예시: `created_at-asc` |
| with\_deleted   _선택 사항_ | 삭제된 결과를 응답에 포함합니다.  유형: boolean  기본값: `false`   가능한 값: `true`, `false` |
| with\_total\_count   _선택 사항_ | `total_count` 응답 속성을 포함합니다.  참고: 이 매개변수와 `cursor`는 상호 배타적입니다.  참고: `total_count`를 포함한 요청에는 더 낮은 레이트 제한이 적용되며, 현재 15분당 200으로 설정되어 있습니다.  유형: boolean  기본값: `false`   가능한 값: `true`, `false` |

#### GET accounts/:account\_id/tweets

#### POST accounts/:account\_id/tweet

계정의 전체 프로모션 가능 사용자(기본값) 또는 `as_user_id` 매개변수에 지정된 사용자로 트윗을 생성합니다. 널캐스트(기본값)와 오가닉 트윗 생성이 모두 지원됩니다. 널캐스트 트윗은 공개 타임라인에 표시되지 않으며 팔로워에게도 노출되지 않습니다. 두 유형 모두 캠페인에서 사용할 수 있습니다.

인증된 사용자가 이 계정의 `FULL` 프로모션 가능 사용자가 아닌 경우, [GET accounts/:account\_id/authenticated\_user\_access](https://developer.x.com/en/docs/ads/campaign-management/api-reference/authenticated-user-access#get-accounts-account-id-authenticated-user-access) 엔드포인트에 요청을 보내 해당 사용자를 대신해 트윗할 권한이 있는지 확인하세요. `TWEET_COMPOSER` 권한은 사용자가 이 엔드포인트를 사용해 `FULL` [프로모션 가능 사용자](https://developer.x.com/en/docs/ads/campaign-management/api-reference/promotable-users#get-accounts-account-id-promotable-users)를 대신하여 널캐스트 트윗을 생성할 수 있음을 나타냅니다.

미디어에 [upload.x.com 엔드포인트](https://developer.x.com/rest/reference/post/media/upload)를 사용할 때는, 이 엔드포인트에 전달하는 `as_user_id` 값과 동일한 `user_id` 값을 `additional_owners` 매개변수로 전달하세요.
