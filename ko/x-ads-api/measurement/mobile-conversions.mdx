<div id="mact-overview">
  ## MACT 개요
</div>

X [모바일 앱 프로모션](https://biz.x.com/mobile-app-promotion) 측정은 설치 또는 기타 인앱 전환을 유도하도록 설계된 X 광고 캠페인의 성과를 광고주가 추적할 수 있게 합니다. X 모바일 측정 파트너는 광고주가 X에서 프로모션 중인 앱에서 어떤 전환을 추적할지 관리할 수 있도록 해줍니다. 광고주가 추적하려는 이벤트의 유형과 전환 기여 판단을 위한 시간 창을 설정하면, 모바일 측정 파트너는 해당 이벤트를 X로 전송하고, 전환을 유도한 캠페인과 크리에이티브에 사용자가 어떤 방식으로 반응했는지를 나타내는 기여 데이터를 즉시 수신합니다.

<div id="conversion-events">
  ### 전환 이벤트
</div>

[전환 이벤트](https://developer.x.com/en/docs/x-ads-api/measurement/api-reference/conversion-event)를 X에 보고하고 어트리뷰션(attribution) 데이터를 받으려면, 파트너는 모바일 앱 ID, 전환 유형과 타임스탬프, 그리고 광고/디바이스 ID의 해시 값을 전송해야 합니다. 이 데이터를 제공하면 X 플랫폼에 전환 이벤트가 기록되고, 해당되는 경우 어트리뷰션이 반환됩니다. 또한 광고주는 구매 가치, 제품 ID, 가입/등록 방법 등 전환 이벤트에 대한 추가 메타데이터를 선택적으로 제공할 수 있습니다.

#### POST conversion\_event

모바일 측정 전환 이벤트를 기록합니다. 응답에는 X 또는 X Audience Platform(TAP) 귀속이 표시됩니다.

이는 [GET conversion\_attribution](https://developer.x.com/en/docs/x-ads-api/measurement/api-reference/conversion-attribution#get-conversion-attribution) 엔드포인트와 관련됩니다.

응답에서는 X, TAP, 또는 귀속 없음 중 하나가 반환됩니다. X 귀속이 없는 경우 `twitter_attribution` 노드는 항상 존재하며 값이 `null`입니다(아래 예시 응답 참조). TAP 귀속이 있는 경우 `tpn_attribution` 노드가 존재하며 해당 값이 채워집니다. 자세한 내용은 [TAP 개요](https://developer.x.com/en/docs/x-ads-api/measurement/overview/twitter-audience-platform)를 참조하세요.

각 전환 이벤트에 연결할 수 있는 선택적 매개변수가 여러 개 제공됩니다. 이 메타데이터는 귀속 계산에 영향을 주지 않습니다.

<div id="resource-url">
  ##### 리소스 URL
</div>

`https://ads-api.x.com/12/conversion_event`

<div id="parameters">

</div>

| 이름 | 설명 |
| :--- | :--- |

| conversion\_time  <br />_required_ | 밀리초가 포함된 ISO-8601 타임스탬프 형식의 전환 이벤트 시간입니다.<br /><br />Type: string<br /><br />Example: `2014-05-22T02:38:28.103Z` |
| conversion\_type  <br />_required_ | 전환 이벤트의 유형입니다.<br /><br />Type: enum<br /><br />Possible values: `PURCHASE`, `SIGN_UP`, `INSTALL`, `RE_ENGAGE`, `UPDATE`, `TUTORIAL_COMPLETE`, `RESERVATION`, `ADD_TO_CART`, `ADD_TO_WISHLIST`, `LOGIN`, `CHECKOUT_INITIATED`, `SEARCH`, `LEVEL_ACHIEVED`, `ACHIEVEMENT_UNLOCKED`, `CONTENT_VIEW`, `SHARE`, `INVITE`, `ADDED_PAYMENT_INFO`, `SPENT_CREDITS`, `RATED` |
| hashed\_device\_id  <br />_required_ | HMAC\_SHA-256으로 해시된 IDFA 또는 AdID입니다.<br /><br />Type: string<br /><br />Example: `ABCD1234XYZ` |
| os\_type  <br />_required_ | 앱의 OS 유형입니다.<br /><br />Type: enum<br /><br />Possible values: `IOS`, `ANDROID` |
| click\_window  <br />_optional_ | 이 이벤트의 클릭 기여 기간(일 기준)입니다.<br /><br />Type: int<br /><br />**Note**: `click_window`는 view\_through\_window보다 크거나 같아야 합니다.<br /><br />Default: `14`  <br />Possible values: `1`, `7`, `14`, `30` |
| device\_ip\_address  <br />_optional_ | 전환 이벤트 발생 시 기기의 IPv4 또는 IPv6 주소입니다.<br /><br />Type: string<br /><br />Example: `192.133.78.1` |
| level  <br />_optional_ | 이 이벤트와 관련된 레벨입니다.<br /><br />Type: int<br /><br />Example: `2` |

| non\_twitter\_engagement\_type  <br />_optional_ | 전환 이벤트 이전의 비(非)트위터 참여 유형입니다.<br /><br />Type: enum<br /><br />Possible values: `CLICK`, `VIEW` |
| number\_items  <br />_optional_ | 이 이벤트와 관련된 항목 수입니다.<br /><br />Type: int<br /><br />Example: `2` |
| price\_currency  <br />_optional_ | 이 이벤트의 통화를 나타내는 ISO 4217 코드여야 합니다.<br /><br />Type: String<br /><br />Examples: `EUR`,`USD`,`JPY` |
| price\_micro  <br />_optional_ | 이 이벤트와 관련된 가격(마이크로 단위)입니다.<br /><br />Type: int<br /><br />Example: `123450000` |
| user\_payment\_info  <br />_optional_ | 이 이벤트와 관련된 앱에 사용자의 결제 정보가 저장되어 있는지를 나타내는 불리언 값입니다.<br /><br />Type: bool<br /><br />Possible values: `true` or `false` |
| view\_through\_window  <br />_optional_ | 이 이벤트의 뷰스루 기여 기간(일 기준)입니다.<br /><br />**Note**:<br /><br />`click_window`는 view\_through\_window보다 크거나 같아야 합니다.<br /><br />Type: int<br /><br />Default: `1`  <br />Possible values: `0`, `1`, `7`, `14`, `30` |

#### GET accounts/:account\_id/app\_event\_tags

현재 계정과 연동된 일부 또는 모든 앱 이벤트 태그의 세부 정보를 조회합니다.

이는 모바일 앱 전환 추적에서 전환 유형별로 설정된 전환 기간을 정의합니다.

<div id="resource-url">
  ##### 리소스 URL
</div>

`https://ads-api.x.com/12/accounts/:account_id/app_event_tags`

<div id="parameters">
  ##### 매개변수
</div>

| 이름 | 설명 |
| :--- | :--- |
| account\_id  <br />_필수_ | 사용 중인 계정의 식별자입니다. 리소스 경로에 포함되며, [GET accounts](/ko/x-ads-api/campaign-management#accounts)를 제외한 대부분의 Advertiser API 요청에 일반적으로 필수 매개변수입니다. 지정한 계정은 인증된 사용자와 연결되어 있어야 합니다.<br /><br />Type: string<br /><br />Example: `18ce54d4x5t` |
| app\_event\_tag\_ids  <br />_선택_ | 쉼표로 구분한 식별자 목록을 지정하여 원하는 앱 이벤트 태그만 응답에 포함되도록 범위를 제한합니다. 최대 200개의 ID를 제공할 수 있습니다.<br /><br />Type: string<br /><br />Example: `jhp` |
| count  <br />_선택_ | 각 개별 요청당 조회할 레코드 수를 지정합니다.<br /><br />Type: int<br /><br />Default: `200`  <br />Min, Max: `1`, `1000` |
| cursor  <br />_선택_ | 다음 페이지 결과를 가져오기 위한 커서를 지정합니다. 자세한 내용은 [Pagination](/ko/x-ads-api/introduction)을 참조하세요.<br /><br />Type: string<br /><br />Example: `8x7v00oow` |
| sort\_by  <br />_선택_ | 지원되는 속성 기준으로 오름차순 또는 내림차순 정렬합니다. 자세한 내용은 [Sorting](/ko/x-ads-api/fundamentals/sorting)을 참조하세요.<br /><br />Type: string<br /><br />Example: `created_at-asc` |
| with\_deleted  <br />_선택_ | 삭제된 결과를 응답에 포함합니다.<br /><br />Type: boolean<br /><br />Default: `false`  <br />Possible values: `true`, `false` |
| with\_total\_count  <br />_선택_ | `total_count` 응답 속성을 포함합니다.<br /><br />**참고**: 이 매개변수와 `cursor`는 상호 배타적입니다.<br /><br />**참고**: `total_count`를 포함하는 요청에는 더 낮은 속도 제한이 적용되며, 현재 15분당 200으로 설정되어 있습니다.<br /><br />Type: boolean<br /><br />Default: `false`  <br />Possible values: `true`, `false` |

#### GET conversion\_attribution

전환 이벤트를 작성하지 않고 전환 기여도를 확인하도록 X에 쿼리합니다. 응답에는 X 기여도 여부가 표시됩니다.

이는 [POST conversion\_event](https://developer.x.com/en/docs/x-ads-api/measurement/api-reference/conversion-event#post-conversion-event) 엔드포인트와 관련됩니다.

응답에는 X, TAP, 또는 기여도 없음 중 하나가 반환됩니다. `twitter_attribution` 노드는 항상 포함되며 X 기여도가 없으면 값이 `null`이고, 아래 예시 응답처럼 값이 채워질 수 있습니다. TAP 기여도가 확인되면 `tpn_attribution` 노드가 포함되어 해당 값이 채워집니다. 자세한 내용은 [TAP 개요](https://developer.x.com/en/docs/twitter-ads-api/measurement/twitter-audience-platform/overview)를 참조하세요.

<div id="resource-url">
  ##### 리소스 URL
</div>

`https://ads-api.x.com/12/conversion_attribution`
