---
title: Pagination
---

더 많은 캠페인 최대치와 계정에 연관된 모든 엔터티의 효율적 조회를 지원하기 위해, Advertiser API는 이제 여러 GET 엔드포인트에서 페이지네이션을 지원합니다. 이 페이지네이션 메커니즘은 사용하기 쉽고, [컬렉션 탐색을 위한 커서 사용](https://developer.x.com/en/docs/x-api/v1/pagination)에 설명된 REST API의 커서 기반 페이지네이션과 매우 유사합니다.

<div id="getting-started">
  ## 시작하기
</div>

특정 GET 요청에 대해 이제 선택 사항인 `cursor`와 `count` 쿼리 매개변수를 지원합니다. 응답에 `count`보다 많은 엔터티가 포함된 경우, 해당 엔드포인트는 처음 `count`개의 엔터티와 응답 JSON의 `next_cursor` 키를 반환합니다.

`GET https://ads-api.x.com/5/accounts/abc1/campaigns?count=50`

```json
{
  "data": \[...\],
  "next_cursor": "c-3yvu1pzhd3i7",
  "request": {...}
}
```

다음 결과를 받으려면 쿼리 매개변수에 `cursor`를 추가하세요:

`GET https://ads-api.x.com/5/accounts/abc1/campaigns?cursor=c-3yvu1pzhd3i7&count=50`

```json
{
  "data": \[...\],
  "next_cursor": "c-3w3zdyg8ywan",
  "request": {...}
}
```

<div id="going-deeper">
  ## 더 깊이 알아보기
</div>

대부분의 엔드포인트에서 `count`의 최대값은 **1,000**, 최소값은 **1**, 기본값은 **200**입니다.

`next_cursor`로 제공되는 값은 항상 문자열이며, 불투명한 값으로 간주해야 합니다. 구현은 변경될 수 있습니다. 현재 결과 집합의 페이지에서 반환된 엔터티가 `count`보다 적으면 `next_cursor` 값은 `null`이 됩니다.

**참고:** 분석 엔드포인트는 이러한 방식의 페이지 매김을 지원하지 않습니다. 통계 엔드포인트의 페이지 매김은 시간 구간을 지정하여 수행됩니다. 자세한 내용은 해당 개별 엔드포인트의 문서를 참조하세요.
