---
title: 레이트 리밋
---

Advertiser API는 다음 문서에 설명된 대로 REST API v1.1과 유사하게 레이트 리밋이 적용됩니다: [REST API Rate Limiting in v1.1](https://developer.x.com/content/developer-twitter/en/docs/basics/rate-limits). REST API v1.1과 달리 엔드포인트별 제한을 제공하는 프로그래매틱 인덱스는 없습니다. 엔드포인트별 레이트 리밋과 재설정 시간 창은 HTTP 응답 헤더를 통해 전달됩니다. Ads API의 모든 레이트 리밋은 OAuth 1.0A를 사용합니다.

<div id="user-level-and-ad-account-level-limits">
  ## 사용자 수준 및 광고 계정 수준 제한
</div>

요율 제한에는 사용자 토큰 수준과 광고 계정 수준의 두 가지 유형이 있습니다. 일부 엔드포인트는 광고 계정 수준 요율 제한을 사용하도록 설정되어 있습니다. 사용자 토큰은 Ads API에 인증하고 호출할 때 사용하는 [OAuth access token](/ko/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)입니다. 각 사용자 토큰은 하나 이상의 광고 계정에 접근할 수 있습니다.

개발자는 응답 헤더에 광고 계정 수준 요율 제한이 포함되어 반환될 경우 이를 사용하고, 광고 계정 수준 제한이 없을 때에만 사용자 수준 제한을 사용해야 합니다.

사용자 수준 요율 제한은 다음 헤더로 제공됩니다: **x-rate-limit-limit**, **x-rate-limit-remaining**, **x-rate-limit-reset**

광고 계정 수준 요율 제한이 활성화된 엔드포인트의 경우, 제한은 다음 헤더로 제공됩니다: **x-account-rate-limit-limit**, **x-account-rate-limit-remaining**, **x-account-rate-limit-reset**.

광고 계정 수준 요율 제한은 단일 사용자 토큰이 여러 광고 계정에 접근하는 상황에서 애플리케이션이 엔터티 데이터(예: 캠페인 또는 라인 아이템 객체)를 동기화할 수 있도록 GET 엔드포인트에만 제공됩니다. 쓰기 작업은 동일한 광고 계정 수준 요율 제한이 적용된다고 보장되지 않습니다.

광고 계정 수준 요율 제한이 적용된 엔드포인트의 경우, 사용자 수준 요율 제한은 애플리케이션 전체에 대한 전역 할당량을 나타내는 높은 값으로 설정됩니다. 사용할 수 있을 때는 요청량 제어에 광고 계정 수준 요율 제한을 우선 적용해야 합니다.

<div id="best-practices">
  ## 모범 사례
</div>

1. 데이터베이스에 마지막 동기화 시점의 타임스탬프를 저장하고, 해당되는 경우 데이터를 요청할 때 `sort_by=updated_at-desc` 옵션을 사용해 마지막 동기화 타임스탬프보다 이전 데이터에 도달하면 동기화 프로세스를 중지할 수 있도록 하십시오. 이렇게 하면 동일한 데이터를 불필요하게 반복 동기화하는 일을 피할 수 있습니다.

2. 단일 요청에서 여러 엔터티 요청: 일부 엔드포인트는 쉼표로 구분된 값 목록을 지정하여 유사한 데이터를 한 번에 여러 개 조회할 수 있습니다. 이는 전체 호출 횟수를 줄여 레이트 리밋을 더 효율적으로 활용하는 데 도움이 됩니다.

3. 요청 시 최대 "count" 사용: [GET accounts/:account\_id/targeting\_criteria](/ko/x-ads-api/campaign-management#targeting-criteria)와 같은 일부 엔드포인트는 기본값 200 대신 최대 count 값을 사용하여 1000개의 객체를 반환하도록 호출할 것을 강력히 권장합니다.

<div id="faqs">
  ## 자주 묻는 질문
</div>

_특정 광고 계정이나 당사 애플리케이션의 요청 한도를 늘릴 수 있나요?_

일반적으로 요청 한도는 상향 조정해 드릴 수 없으며, 대규모 광고 계정을 지원하도록 설정되어 있습니다. 먼저 이 문서에 제시된 모범 사례를 적용해 보시고, 그럼에도 요청 한도가 확장이나 비즈니스 목표 달성에 여전히 영향을 미친다면, 관련 사용 사례와 요청 사항에 대한 상세 정보를 첨부하여 X Ads API 담당자에게 문의해 주시기 바랍니다.

  ## 레이트 리밋 적용 범위
</div>

<div id="scopes-for-this-doc">
  ### 이 문서의 적용 범위
</div>

- 카테고리: 지정된 카테고리에 속한 모든 엔드포인트는 각 윈도우마다 하나의 공유 한도에서 레이트 리밋이 적용됩니다.
- 엔드포인트: 각 엔드포인트는 윈도우마다 별도의 개별 할당 한도가 있습니다.
