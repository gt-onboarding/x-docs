<div id="overview">
  ## 개요
</div>

매월 X Ads API에 변경 사항을 적용하고 여러 신규 기능을 출시합니다. 이러한 변경은 대부분 하위 호환되지만, 매년 소수의 호환성 깨짐 변경이 발생하기도 합니다. Ads API의 빠른 변경 주기가 새로운 기능 구현, 사용 중단 처리, 변경 사항 테스트 등 개발 사이클에 미치는 어려움에 대해 개발자들로부터 피드백을 받았습니다. Ads 플랫폼에서의 개발자 경험을 개선하기 위해 엔드포인트 버전 관리를 도입했습니다.

아래는 본문에서 사용하는 몇 가지 개념의 정의입니다.

**버전(Version)**: 모든 Ads API 요청의 URL 경로에 표시되는 버전 번호를 의미합니다. 예: GET /{version_number}/accounts. 이 방식의 버전 관리는 URI 버전 관리라고 합니다.

**호환성 깨짐 변경(Breaking Changes)**: 기존 기능을 유지하는 데 개발자 리소스가 필요한 모든 변경을 의미합니다. 여기에는 필요한 변경 사항 조사, 사용 중단될 기능/엔드포인트의 결정, 그리고 이러한 변경의 최종 구현에 투입되는 리소스가 포함됩니다. 호환성 깨짐 변경의 예시는 다음과 같습니다.

- API 요청/응답에서 파라미터 제거
- 파라미터 또는 엔드포인트 이름 변경
- 값 표현 방식 변경(예: preview\_url → card\_uri)
- 엔드포인트 동작 변경(예: 비동기 vs 동기 통계)
- 선택/필수 파라미터 추가 또는 변경(예: 요청에서 name을 필수 필드로 지정)

**사용 중단(Deprecation)**: 사용 중단된 버전이나 제품은 더 이상 지원되지 않으며, 해당 API의 사용 중단을 권장합니다.

**지원 종료(Sunset)**: 제품이나 API가 지원 종료되면, 해당 엔드포인트 집합은 더 이상 API로 접근할 수 없습니다.

<div id="versioning-strategy">
  ## 버전 관리 전략
</div>

이 전략의 핵심 원칙은 다음과 같습니다:

1. 모든 호환성 파괴 변경 사항은 새로운 버전으로 묶어 제공합니다

2. 새로운 버전이 발표될 때 기존 버전의 사용 중단 유예 기간은 6개월입니다

3. 언제든지 API는 두 개의 버전에서 동시 요청을 허용하지만, 둘 중 더 오래된 버전은 지원하지 않습니다

4. 새로운 제품의 신속한 도입을 위해 버전 릴리스 주기와 별개로 지속적으로 출시합니다

5. 모든 API 응답에는 `x-current-api-version`이 포함되며, 이는 현재 API 버전으로 설정됩니다. 또한 사용 중단된 API 엔드포인트를 호출할 때는 `x-api-warn` 헤더가 추가됩니다.

API의 호환성 파괴 변경이 필요한 근본적인 제품 요구 사항 변경(예: 다중 연령 구간 타기팅 사용 중단)이 있는 경우, 당사는 해당 변경을 알리는 90일 사전 공지를 발송하고, 공지 후 최소 90일이 지난 뒤에 해당 변경을 배포합니다

<div id="v9">
  ### v9
</div>

오늘, 2021년 3월 3일부로 X Ads API 버전 9(v9)이 제공됩니다. 이번 릴리스는 기능 동등성을 높이고 캠페인 생성을 단순화하며, Cards 및 Mobile App Promotion 엔드포인트에 대한 핵심 업데이트를 도입하도록 설계되었습니다.

이전 버전과 마찬가지로 v9로의 마이그레이션을 위해 6개월의 전환 기간이 제공됩니다. 2021년 8월 31일부로 기존 Ads API 버전 8(v8)은 더 이상 제공되지 않습니다. 서비스 중단을 피하기 위해 가능한 한 빨리 최신 버전의 Ads API로 마이그레이션할 것을 모든 개발자께 권장합니다.

<Note>
  **참고:** 이번 릴리스 기준으로 Ads API 버전 7(v7)은 지원 종료되어 더 이상 제공되지 않습니다.
</Note>

자세한 내용은 [개발자 포럼의 공지](https://devcommunity.x.com/t/ads-api-version-9/150316)를 참고하세요.

  ### v7
</div>

2020년 3월 20일, [ads.x.com](http://ads.x.com/)과의 기능 동등성을 높이기 위해 설계된 X Ads API 버전 7을 발표합니다.

이전 [버전](https://developer.x.com/en/docs/ads/general/overview/versions)과 마찬가지로 v7로 마이그레이션할 수 있도록 6개월의 전환 기간이 제공됩니다. 2020-09-01부터 Ads API 버전 6은 더 이상 제공되지 않습니다. 서비스 중단을 방지하기 위해 모든 개발자께서 가능한 한 빨리 최신 버전의 API로 마이그레이션하시기를 권장합니다. Ads API 버전 5는 수명이 종료되어 더 이상 제공되지 않습니다.

자세한 내용은 [개발자 포럼의 공지](https://devcommunity.x.com/t/ads-api-version-7/135093)를 참조하세요.

<div id="v6">
  ### v6
</div>

오늘, 2019년 8월 28일, X는 일관성과 개발자 경험 개선에 중점을 둔 Ads API v6를 소개합니다.

이번 릴리스에는 트윗을 조회하기 위한 새로운 엔드포인트, 프로모션 계정에 대한 통계, 이름으로 엔터티를 검색하는 기능, 그리고 현재 처리 중인 비동기 분석 작업의 수에 대한 정보가 포함되어 있습니다. 또한 미디어를 사용하는 엔드포인트와 타기팅 기준 엔드포인트에 일관성 강화를 위한 업데이트를 적용했습니다. 마지막으로 일부 매개변수 이름과 응답 속성을 소폭 수정하고 Scoped Timeline 엔드포인트를 사용 중단합니다.

자세한 내용은 [개발자 포럼의 공지](https://devcommunity.x.com/t/ads-api-version-6/129060)를 참조하세요.

<div id="v5">
  ### v5
</div>

2019년 2월 28일, X는 확장성과 효율성 제고에 초점을 맞춘 업데이트와 함께 Ads API v5를 도입합니다.

이번 릴리스에는 지정된 기간 동안 어떤 엔터티가 활성 상태였는지 파악하는 새로운 엔드포인트, Media Creatives(예: X Audience Platform의 인스트림 동영상 및 이미지)용 통계, 카드 URI로 _여러_ 카드를 가져오는 기능, 타기팅 기준 및 기타 엔터티 조회의 유연성 향상 등이 포함되어 있습니다. 또한 일부 버그를 수정하고 매개변수 이름과 응답 속성을 업데이트했습니다. 마지막으로, 비미디어 앱 카드와 POST `accounts/:account_id/account_media` 엔드포인트는 사용이 중단되었습니다.

이전 버전과 마찬가지로 v5로 마이그레이션할 수 있도록 6개월의 전환 기간이 제공됩니다. **2019-08-28**부터 Ads API 버전 4는 더 이상 제공되지 않습니다. 서비스 중단을 방지하기 위해 모든 파트너가 가능한 한 빨리 최신 API 버전으로 마이그레이션할 것을 권장합니다. _Ads API 버전 3은 지원이 종료되어 더 이상 이용할 수 없습니다._

<div id="new">
  #### 신규
</div>

**활성 엔터티 판별**

[활성 엔터티](/ko/x-ads-api/analytics#active-entities-2) 엔드포인트는 광고 엔터티의 분석 지표에 변경 사항이 있었는지를 나타냅니다. 분석 엔드포인트와 함께 사용하도록 설계되었으며, 활성 엔터티는 엔터티 유형과 날짜 범위(최대 90일)를 지정하면, 귀하의 플랫폼이 분석을 요청해야 하는 엔터티 ID 배열을 반환합니다. 반환된 ID 외의 엔터티는 이후 분석 요청에서 조회하지 마십시오.

이 엔드포인트는 다음 엔터티 유형을 지원합니다: `CAMPAIGN`, `FUNDING_INSTRUMENT`, `LINE_ITEM`, `MEDIA_CREATIVE`, `PROMOTED_TWEET`.

**`MEDIA_CREATIVE` 지표**

Ads API의 분석 엔드포인트가 이제 Media Creative 엔터티에 대한 지표를 제공합니다. Media Creative는 X Audience Platform에서 인스트림 광고 또는 이미지를 프로모션하는 방식입니다. X Ads UI에서는 “In-stream videos” 및 “Display creatives” 탭에서 Media Creative 지표를 확인할 수 있습니다. [동기식](/ko/x-ads-api/analytics#get-stats-accounts-account-id)과 [비동기식](/ko/x-ads-api/analytics#asynchronous-analytics) 분석 엔드포인트 모두 이제 `MEDIA_CREATIVE` 엔터티 enum을 지원합니다.

**여러 카드 가져오기**

카드 URI 값으로 단일 카드를 가져오도록 설계된 엔드포인트의 [v3 릴리스](https://devcommunity.x.com/t/ads-api-version-3/100732)를 개선하여, 이제 [GET accounts/:account\_id/cards/all](/ko/x-ads-api/creatives#cards-fetch) 엔드포인트로 여러 카드를 한 번에 가져올 수 있습니다. 이제 각 카드마다 개별 요청을 보내는 대신, 한 번의 요청으로 최대 200개의 카드를 조회할 수 있습니다.

참고 사항 두 가지:

1. URL 경로는 이제 `accounts/:account_id/cards/all`입니다. (이전 경로는 더 이상 사용되지 않습니다.) 이는 ID로 카드를 조회하는 [엔드포인트](/ko/x-ads-api/creatives#cards-fetch)와의 일관성을 위한 변경입니다.
2. 필수 요청 매개변수 이름이 이제 복수형인 card\_uris입니다.

**검색 유연성 강화**

[GET accounts/:account\_id/targeting\_criteria](/ko/x-ads-api/campaign-management#get-accounts-account-id-targeting-criteria) 엔드포인트가 이제 여러 라인 아이템 ID를 지원합니다. 최대 200개의 ID를 받을 수 있는 line\_item\_ids 매개변수가 필수입니다. 이전에는 단일 라인 아이템만 허용되어 동기화가 어려웠습니다. 이번 변경으로 더 짧은 시간에 더 많은 타게팅을 조회할 수 있습니다.

다음 엔드포인트들도 이제 여러 라인 아이템 ID를 지원하지만, 이들에 대해서는 line\_item\_ids 매개변수가 선택 사항입니다.

- [GET accounts/:account\_id/line\_item\_apps](https://developer.x.com/en/docs/x-ads-api/campaign-management/api-reference/line-item-apps#get-accounts-account-id-line-item-apps)
- [GET accounts/:account\_id/media\_creatives](/ko/x-ads-api/campaign-management#get-accounts-account-id-media-creatives)
- [GET accounts/:account\_id/promoted\_accounts](/ko/x-ads-api/campaign-management#get-accounts-account-id-promoted-accounts)
- [GET accounts/:account\_id/preroll\_call\_to\_actions](/ko/x-ads-api/creatives#preroll-call-to-actions)

<div id="changed">
  #### 변경 사항
</div>

**초안 캠페인 및 라인 아이템 조회**

초안 캠페인과 라인 아이템을 조회하는 방식이 업데이트되었습니다. 이제 with\_draft(boolean) 매개변수를 true로 설정하면 초안 및 비초안 엔터티가 모두 반환됩니다. 이는 삭제된 엔터티를 조회하는 방식(예: with\_deleted 사용)과 일관됩니다. 이전에는 초안과 비초안 엔터티를 모두 가져오려면 최소 두 번의 요청이 필요했지만, 이제 단일 API 호출로 가능합니다.

| **v4** | **v5** |
| :--- | :--- | :--- |
| `draft_only` | `with_draft` |     |

**Network activation duration 타기팅**

Ads API는 Network Activation Duration 타기팅을 추가한 후 응답의 타기팅 타입에 \_IN\_SEC 접미사가 포함되던 표시 문제를 해결했습니다. Network Activation Duration은 항상 개월 수로 표현되므로 초 단위에 대한 참조는 혼란을 야기했습니다. 이번 수정으로 표현이 일관되어 혼란이 줄어듭니다.

| **v4** | **v5** |
| :--- | :--- | :--- |
| `NETWORK_ACTIVATION_DURATION_IN_SEC` | `NETWORK_ACTIVATION_DURATION` |     |

**총계 및 커서**

v5에서는 with\_total\_count와 cursor가 상호 배타적입니다. 요청에서 둘 다 지정하면 EXCLUSIVE\_PARAMETERS 오류 코드가 반환됩니다. v5 이전에는 cursor가 지정되면 with\_total\_count가 무시되었습니다. 이번 변경으로 관계가 명확히 규정되었습니다.

<div id="removed">
  #### 제거됨
</div>

Ads API 응답에서 세 가지 필드가 제거됩니다: preview\_url, account\_id, parent\_ids. 이 세 가지에 대한 엔지니어링 작업 부담은 매우 적습니다.

- v4에서 카드의 preview\_url 응답 매개변수가 항상 null임을 공지했습니다. 이번 마이그레이션의 마지막 단계로 모든 카드 응답에서 preview\_url을 제거합니다.
- 다음 리소스의 account\_id 응답 속성이 제거됩니다. 광고 계정 ID는 이미 URL 및 request.params에 포함되어 있기 때문입니다. (이 목록에서 자금 조달 수단을 제외한 것은 의도적인 결정입니다. 가능한 경우 상위 ID는 응답 객체에 포함되어야 하며, 계정 ID는 자금 조달 수단의 상위 엔터티이기 때문입니다.)
  - Account media
  - App event providers
  - App event tags
  - Campaigns
  - Cards
  - Line items
  - Promotable users
  - Targeting criteria
- [GET accounts/:account\_id/targeting\_criteria](/ko/x-ads-api/campaign-management#get-accounts-account-id-targeting-criteria) 요청의 경우, parent\_ids 필드는 항상 빈 배열이었으므로 더 이상 반환하지 않습니다.

**비미디어 앱 카드**

v5에서는 비미디어 앱 카드를 더 이상 지원하지 않습니다. [이전에](https://devcommunity.x.com/t/deprecation-announcement-app-download-cards/87807) 비미디어 앱 카드를 생성하거나 편집하는 기능이 제거되었습니다. 이제 이 리소스에 대한 남은 엔드포인트가 사용 중단됩니다.

- 참고: 이는 이미지 및 동영상 앱 다운로드 카드에는 영향을 주지 않습니다.

**Account media 생성**

v5에서는 POST accounts/:account\_id/account\_media 엔드포인트를 더 이상 사용할 수 없습니다. 이 리소스의 다른 엔드포인트는 영향을 받지 않습니다. 변경 이유는 [Media Library](/ko/x-ads-api/creatives#media-library)에 미디어를 추가할 때 특정 경우 해당 에셋이 자동으로 Account Media 엔터티로 추가되며, 이미 존재하는 에셋을 Account Media 리소스에 다시 추가하려 하면 오류가 발생하기 때문입니다. 이는 다음과 같은 경우에 발생합니다.

- Media Library에 추가된 `AMPLIFY_VIDEO` 에셋은 `PREROLL` 크리에이티브 유형으로 자동으로 Account Media 에셋으로 추가됩니다.
- 특정 크기의 이미지를 Media Library에 추가하면 자동으로 Account Media 에셋으로 추가됩니다. 크리에이티브 유형(예: `INTERSTITIAL`)은 이미지 크기에 따라 달라집니다. (크기 기준은 [Enumerations](/ko/x-ads-api/introduction) 페이지를 참조하세요.)

<div id="v4">
  ### v4
</div>

Ads API 버전 4가 오늘, 2018년 8월 28일에 출시됩니다.

이번 릴리스에는 보다 견고한 잠재고객 처리 백엔드로 구동되는 새로운 API 인터페이스를 포함해 [Audiences](/ko/x-ads-api/audiences) 제품이 개선되었습니다. 버전 4에는 사용자, 계정, 세금 설정을 관리하기 위한 엔드포인트 세트도 포함됩니다. 또한 accounts/:account\_id/videos 엔드포인트는 사용 중단됩니다. 이 릴리스에는 일부 파라미터와 응답 이름의 소폭 변경도 포함됩니다.

버전 3과 마찬가지로 6개월의 전환 기간을 제공합니다. **2019-02-28**부터 Ads API 버전 3은 더 이상 제공되지 않습니다. 서비스 중단을 피하기 위해 모든 파트너가 가능한 한 빨리 최신 버전의 API로 마이그레이션하시기 바랍니다. 버전 관리 전략에 대한 자세한 내용은 [Versions](/ko/x-ads-api/introduction) 페이지를 참조하세요.

<div id="new">
  #### 신규
</div>

_**Audience API**_

새로운 Audiences API는 향상된 견고성과 신뢰성을 제공하는 새로운 잠재고객 처리 백엔드를 기반으로 구축되었습니다. 이 새 엔드포인트를 통해 파트너는 단일 사용자에 대해 여러 유형의 사용자 식별자를 제공할 수 있으며, 이를 통해 매칭에 추가 신호를 활용할 수 있습니다. 새로운 Audience 엔드포인트에 대한 참고 문서는 [여기](/ko/x-ads-api/audiences)에서 확인할 수 있습니다. 올해 남은 기간 동안 이 제품에 대한 업데이트와 개선을 지속적으로 제공할 예정입니다.

기능 중복으로 인해 다음 엔드포인트는 v4에서 더 이상 제공되지 않습니다(여전히 v3에서는 동작하며, v3 지원이 종료되면 완전히 **중단**됩니다):

- TON 업로드:
  - GET accounts/:account\_id/tailored\_audience\_changes
  - GET accounts/:account\_id/tailored\_audience\_changes/:tailored\_audience\_change\_id
  - POST accounts/:account\_id/tailored\_audience\_changes
  - PUT accounts/:accounti\_d/tailored\_audiences/global\_opt\_out
- 실시간 잠재고객:
  - POST tailored\_audience\_memberships

마지막으로, 버전 4에서는 _모든_ [Tailored Audiences 엔드포인트](/ko/x-ads-api/audiences)의 요청 및 응답에서 `list_type` 매개변수가 제거됩니다.

_**Settings Endpoints**_

이제 계정 관리자가 사용자, 계정, 세금 설정을 구성하고 업데이트할 수 있습니다. [User settings](/ko/x-ads-api/campaign-management#get-accounts-account-id-user-settings-user-id)는 특정 광고 계정에 대한 사용자별 연락처 선호도를 나타냅니다. [PUT accounts/:account\_id](/ko/x-ads-api/campaign-management#put-accounts-account-id) 엔드포인트를 사용하면 광고주가 계정 이름과 업종을 업데이트할 수 있습니다. 마지막으로, [tax settings](/ko/x-ads-api/campaign-management#get-accounts-account-id-tax-settings) 엔드포인트를 통해 부가가치세(VAT)가 부과되는 국가의 광고주는 회사명, 주소, VAT ID, 그리고 계정이 광고주 소유인지 또는 대행사가 광고주를 대신해 보유하는지 여부 등의 정보를 업데이트할 수 있습니다.

- [POST accounts/:account\_id/cards/image\_app\_download](https://devcommunity.x.com/t/ads-api-version-11/168814)
- [PUT accounts/:account\_id/cards/image\_app\_download/:card\_id](https://developer.x.com/content/developer-twitterhttps://developer.x.com/en/docs/ads/creatives/api-reference/image-app-download#put-accounts-account-id-cards-image-app-download-card-id)
- [POST accounts/:account\_id/cards/video\_app\_download](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/api-reference/video-app-download#post-accounts-account-id-cards-video-app-download)
- [PUT accounts/:account\_id/cards/video\_app\_download/:card\_id](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/api-reference/video-app-download#put-accounts-account-id-cards-video-app-download-card-id)

**`preview_url`은 항상 null**

v3 공지에서 약속한 대로 이제 모든 기존 카드에 `card_uri`가 제공됩니다. 그 결과 `preview_url` 값은 항상 `null`입니다.

참고로, `card_uri` 값을 사용해 트윗에 카드를 연결하세요. 다음 예시 요청을 참조하세요.

$ twurl -X POST -H ads-api.x.com "/4/accounts/18ce54d4x5t/tweet?text=Version 4\&card\_uri=card://958225772740714496"

**동영상 엔드포인트**

accounts/:account\_id/videos 엔드포인트는 v4에서 더 이상 제공되지 않습니다. 이 엔드포인트는 [Media Library](/ko/x-ads-api/creatives#get-accounts-account-id-media-library) 엔드포인트 도입으로 대체되었습니다. 아래는 사용 예 비교입니다.

- v3 동영상 엔드포인트: `twurl -H ads-api.x.com "/3/accounts/18ce54d4x5t/videos"`

- v4 동영상용 미디어 라이브러리 엔드포인트: `twurl -H ads-api.x.com "/4/accounts/18ce54d4x5t/media_library?media_type=VIDEO"`

Media Library 엔드포인트는 동영상 엔드포인트와 기능이 완전히 동등하며, 이미지 및 GIF 처리 같은 추가 기능도 지원합니다. 파트너 여러분께서는 모든 미디어 관리에 Media Library만 사용해 주시기 바랍니다.

**Tweet View의 `as_user_id`**

[GET accounts/:account\_id/tweet/preview/:tweet\_id](/ko/x-ads-api/creatives#get-accounts-account-id-tweets) 엔드포인트에서 제공되던 `as_user_id` 매개변수는 더 이상 허용되지 않습니다. 미리보기는 항상 트윗 작성자 기준으로 렌더링됩니다.

<div id="v3">
  ### v3
</div>

X Ads API 버전 3은 2018년 2월 1일에 [출시](https://devcommunity.x.com/t/ads-api-version-3/100732)되었습니다. Ads API 버전 2는 2018년 8월 1일에 지원이 종료됩니다.

이번 릴리스에는 새로운 Audience Intelligence 제품, Media Library 접근, 그리고 개선된 카드 워크플로가 포함됩니다. 또한 [PUT accounts/:account\_id/targeting\_criteria](/ko/x-ads-api/campaign-management#get-accounts-account-id-targeting-criteria) 엔드포인트의 사용 중단을 발표합니다. 마지막으로, 버전 3에는 일부 경미한 매개변수 및 응답 변경과 더 낮은 배치 크기 한도가 포함됩니다.

[버전 2](https://devcommunity.x.com/t/ads-api-version-2/90360)와 마찬가지로 파트너에게 전환 기간을 **6개월** 제공합니다. **2018-08-01**에 Ads API v2는 종료됩니다. 모든 파트너와 개발자께 가능한 한 빨리 v3로 마이그레이션할 것을 권장합니다.

**Audience Intelligence**

Audience Intelligence는 특정 X 오디언스와 가장 관련성 높은 주요 해시태그, @핸들, 이벤트에 대한 실시간 인사이트를 제공합니다. 예를 들어, 미국의 남성 18–34를 입력하면 이 오디언스에서 #nintendoswitch, #cardinal, @ricegum이 트렌드임을 확인할 수 있습니다.

Audience Intelligence [엔드포인트](/ko/x-ads-api/audiences)는 다음 기능을 제공합니다:

- 입력 오디언스를 기준으로 관련성이 높은 상위 해시태그, @핸들, 이벤트를 조회합니다.
- 입력 오디언스를 기준으로 핵심 인구통계 정보(예: 연령, 성별, 가계 소득)를 조회합니다.
- 키워드를 기준으로 트윗 볼륨 시계열을 조회합니다.

**Media Library**

[Media Library](/ko/x-ads-api/creatives#get-accounts-account-id-media-library)는 광고 계정의 이미지, GIF, 동영상을 관리할 수 있는 기능을 제공합니다. 이러한 미디어 객체는 트윗에 사용하거나 카드를 생성하는 데 활용할 수 있습니다. 또한 여러 크리에이티브에서 재사용할 수 있어 동일한 에셋을 여러 번 업로드할 필요가 없습니다.

라이브러리의 객체는 media\_key로 식별됩니다. 미디어 키는 예를 들어 13\_875943225764098048과 같은 형식의 문자열 값입니다. Ads API에서는 모든 미디어에 대해 미디어 키 사용으로 전환하고 있습니다.

**개선된 카드 워크플로**

모든 카드 엔드포인트가 이제 미디어 키를 지원합니다. 이를 통해 Media Library의 객체를 사용하여 카드를 생성하거나 업데이트할 수 있습니다.

또한 [카드 세부 정보 조회](/ko/x-ads-api/creatives#fetching-cards)를 위한 두 개의 새로운 엔드포인트를 도입합니다. 예를 들어, 이 엔드포인트는 트윗 또는 예약 트윗에 사용된 카드를 조회할 때 card\_uri 또는 id를 지정하여 사용할 수 있습니다. 이전에는 가능하지 않았습니다.

<div id="other-changes">
  #### 기타 변경 사항
</div>

이러한 신규 기능 외에도 버전 3에는 다음과 같은 변경 사항이 포함됩니다.

**신규**

- [GET insights/keywords/search](/ko/x-ads-api/audiences) 엔드포인트 응답에 이제 입력 키워드와 관련된 30개의 용어를 담은 related\_keywords 속성이 포함됩니다.

**변경됨**

- 타게팅 기준 배치의 최대 크기가 500으로 상향되었습니다.
- card\_uri 및 preview\_url 응답 속성은 이제 상호 배타적입니다. 카드에 card\_uri가 있으면 preview\_url은 null입니다. 카드에 card\_uri가 없으면 preview\_url만 반환됩니다.
  - 2018-01-29에 생성된 모든 카드에는 card\_uri가 있습니다.
  - 버전 4까지 기존의 모든 카드에 card\_uri가 제공됩니다.
- 5:2 이미지로 카드를 새로 생성할 수 없습니다. 기존 5:2 이미지 기반 카드는 계속 동작하지만, 성능이 더 높은 1.91:1 또는 1:1 종횡비(지원되는 경우)로 전환하는 것을 권장합니다.

**제거됨**

- [PUT accounts/:account\_id/targeting\_criteria](/ko/x-ads-api/campaign-management#get-accounts-account-id-targeting-criteria) 엔드포인트는 더 이상 제공되지 않습니다. 이 엔드포인트의 대체(치환) 동작이 광고주에게 혼란을 야기했고, 한 번에 단일 리소스만 업데이트하는 다른 PUT 엔드포인트들과도 일관되지 않았기 때문입니다. 대신 파트너는 [POST batch/accounts/:account\_id/targeting\_criteria](/ko/x-ads-api/campaign-management#post-batch-accounts-account-id-targeting-criteria) 엔드포인트를 사용해야 하며, 단일 요청에서 타게팅을 추가 및 제거할 수 있는 등 더 큰 유연성을 제공합니다.
- 자금 조달 수단(funding instruments)에 대해 paused 응답 속성은 더 이상 반환되지 않습니다. 대신 해당 엔티티가 일시 중지되었는지 여부는 entity\_status 응답 속성으로 판단하세요. 또한 paused와 cancelled가 동일한 값에 해당하므로 cancelled도 더 이상 응답에 포함되지 않습니다.
- [GET accounts/:account\_id/tweet/preview](/ko/x-ads-api/creatives#get-accounts-account-id-tweets) 엔드포인트에서 card\_id 파라미터가 제거되었습니다.
- 삭제된 예약 트윗은 조회할 수 없으므로 with\_deleted 파라미터는 더 이상 지원되지 않습니다.
- 다음 엔드포인트에서 draft\_only 파라미터가 제거되었습니다. 이러한 엔티티는 초안 상태가 될 수 없기 때문입니다.
  - [GET accounts/:account\_id/targeting\_criteria](/ko/x-ads-api/campaign-management#get-accounts-account-id-targeting-criteria)
  - [GET accounts/:account\_id/promoted\_tweets](/ko/x-ads-api/campaign-management#get-accounts-account-id-promoted-tweets)
  - [GET accounts/:account\_id/promoted\_accounts](/ko/x-ads-api/campaign-management#get-accounts-account-id-promoted-accounts)
  - [GET accounts/:account\_id/media\_creatives](/ko/x-ads-api/campaign-management#get-accounts-account-id-media-creatives)
  - [GET accounts/:account\_id/preroll\_call\_to\_actions](/ko/x-ads-api/creatives#get-accounts-account-id-preroll-call-to-actions)

**참고**

Video Website Card와 예약 트윗은 이제 베타를 종료했습니다. 출시 이후 예약 트윗에 대해 적용된 변경 사항은 [이 스레드](https://devcommunity.x.com/t/announcement-update-to-scheduled-tweets/94869)를 참조하세요. 여기에는 예약 트윗의 HTML [미리보기](https://devcommunity.x.com/t/announcement-tweet-preview-improvements/96577) 생성 기능이 포함됩니다.

  ### v2
</div>

Ads API 버전 2는 2017년 7월 10일 [출시](https://devcommunity.x.com/t/ads-api-version-2/90360)되었습니다. Ads API 버전 1은 2018년 1월 10일에 지원 종료(EOL)됩니다.

**하위 호환성 중단/사용 중단[¶](#breaking-changes-deprecations "Permalink to this headline")**

- `total_count`는 이제 선택적 응답 속성입니다. `with_total_count`가 `true`로 설정된 경우에만 제공됩니다

- `line_items` 및 `campaigns` 요청/응답 객체의 `paused` 및 `draft_only` 필드는 단일 `entity_status` 매개변수로 대체됩니다

- [POST accounts/:account\_id/tweet](/ko/x-ads-api/creatives) 및 [GET accounts/:account\_id/tweet/preview](/ko/x-ads-api/creatives#get-accounts-account-id-tweets) 엔드포인트에서 `status` 매개변수는 `text`로 이름이 변경되었습니다

- [GET targeting\_criteria/locations](/ko/x-ads-api/campaign-management#get-targeting-criteria-locations) 엔드포인트의 `location_type` 열거형은 이제 복수형입니다. `COUNTRY`는 `COUNTRIES`, `REGION`은 `REGIONS` 등으로 변경됩니다. 한 가지 예외로 v2에서는 `CITY`가 ‘도시권(metros)’을 의미하는 DMA(Designated Market Areas)를 올바르게 반영하도록 `METROS`가 되었습니다.

- PUT accounts/:account\_id/promoted\_tweets 엔드포인트의 `display_properties`가 제거됩니다. 이 값은 응답에서도 더 이상 반환되지 않습니다

- 위 변경에 따라 더 이상 promoted\_tweets 엔터티를 업데이트(PUT)할 수 없습니다

- [GET accounts/:account\_id/promoted\_tweets](/ko/x-ads-api/campaign-management#get-accounts-account-id-promoted-tweets) 엔드포인트의 `line_item_id` 매개변수가 제거되었습니다

- v2 엔드포인트에서는 5:2 Website Card를 더 이상 생성할 수 없습니다

- `data_type` 응답 속성은 더 이상 반환되지 않습니다

**초안 캠페인[¶](#draft-campaigns "Permalink to this headline")**

초안 캠페인은 [GET accounts/:account\_id/camapaigns](/ko/x-ads-api/campaign-management#get-accounts-account-id-campaigns) 엔드포인트를 통해 조회할 수 있었습니다. v2부터는 API로 초안 캠페인을 생성/활성화할 수 있습니다.

- [POST accounts/:account\_id/line\_items](/ko/x-ads-api/campaign-management#post-accounts-account-id-line-items) 및 [POST accounts/:account\_id/campaigns](/ko/x-ads-api/campaign-management#post-accounts-account-id-campaigns) 엔드포인트의 `entity_status` 매개변수는 새 초안 캠페인이나 라인 아이템을 생성하기 위해 `DRAFT`로 설정할 수 있습니다.
- 새로 생성되는 초안에 필요한 매개변수 집합:

| 초안 캠페인 | 초안 라인 아이템 |
| :--- | :--- |
| `funding_instrument_id` | `campaign_id` |
| `name` | `objective` |
| `start_time` | `product_type` |
|     | `placements` |

**참고[¶](#notes "Permalink to this headline")**

- 초안 라인 아이템 또는 캠페인은 `entity_status`를 `DRAFT`에서 `PAUSED` 또는 `ACTIVE`로만 전환할 수 있습니다.
- 전체 캠페인(여러 라인 아이템 포함)을 활성화하려면, 해당 캠페인에 속한 각 라인 아이템과 캠페인 자체의 `entity_status`를 모두 `ACTIVE`로 설정해야 합니다.
- 캠페인 또는 라인 아이템의 `entity_status`를 변경하려면 해당 PUT 엔드포인트를 사용하십시오.

Ads API 버전 1은 2016년 3월 31일에 출시되었으며 2018년 1월 10일에 지원이 종료됩니다.

**버전 1의 변경 사항:[¶](#changes-in-version-1 "Permalink to this headline")**

- [버저닝 지원](https://blog.x.com/2016/versioning-is-coming-to-twitter-s-ads-apis)
- `CUSTOM` [목표](/ko/x-ads-api/campaign-management#objective-based-campaigns)는 더 이상 지원되지 않습니다
- [배치 엔드포인트](/ko/x-ads-api/campaign-management#post-batch-accounts-account-id-campaigns)가 이제 일반 공급됩니다
- [도달 추정 변경](https://developer.x.com/en/docs/ads/campaign-management/api-reference/reach-estimate.html#get-accounts-account-id-reach-estimate):
- 더 정확한 도달 추정을 제공하기 위해 이제 엔드포인트가 예산을 인식합니다. 다음 매개변수가 필수입니다:
  - \[신규] `campaign_daily_budget_amount_local_micro`
  - `currency`
  - `bid`
  - `objective`
- 응답 객체가 변경되어 이제 응답 값의 범위를 반환합니다.
- `infinite_count`는 목적에 대한 혼선을 피하기 위해 `infinite_bid_count`로 이름이 변경되었습니다.
- `count` 및 `infinite_bid_count` 외에 다음과 같은 새로운 데이터 포인트도 반환됩니다:
  - `impressions`
  - `engagements`
  - `estimated_daily_spend_local_micro`
- 맞춤 잠재고객 데이터 유형 변경
- 모든 응답에서 맞춤 잠재고객(Tailored Audiences)의 `data_type`이 `tailored_audiences`에서 `tailored_audience`로 변경되었습니다.
- 공유 맞춤 잠재고객은 이제 _API 전용_ 베타로 제공됩니다. 공유 맞춤 잠재고객을 사용하면 하나의 잠재고객을 여러 광고 계정에서 사용할 수 있습니다. 광고 계정 간에 공유하려는 맞춤 잠재고객의 권한을 관리하려면 [POST accounts/:account\_id/tailored\_audiences/:tailored\_audience\_id/permissions](/ko/x-ads-api/audiences) (및 관련) 엔드포인트를 사용하세요.
- 광고주 계정의 성과 [분석](/ko/x-ads-api/analytics) 수집 방식이 크게 개선되었습니다:
- 당사의 [모범 사례](/ko/x-ads-api/analytics#best-practices)에 맞춰, 이제부터 [동기식 통계 엔드포인트](/ko/x-ads-api/analytics#get-stats-accounts-account-id)에서는 최대 **7일**치 데이터만 조회할 수 있습니다.
- 지표 수집을 단순화하기 위해 `metrics` 매개변수를 새 `metric_groups` 매개변수로 교체했습니다. 이제 개발자는 특정 요청에서 반환받을 지표 그룹을 명시해 요청하면 됩니다.
  - 특정 엔터티에 부적합한 지표에 대한 요청은 응답에서 제외되며 `null` 값으로 표시됩니다. _이러한 지표는 분석 비용 한도에 산정되지 않습니다._
- 응답이 대폭 간소화되었으며 이제 UI에서 지표를 노출하는 방식과 보다 밀접하게 일치합니다.
  - 이전에는 각 게재 위치별로 별도의 지표를 제공했습니다(검색의 Promoted Tweets, 타임라인의 Promoted Tweets, 프로필 및 트윗 상세의 Promoted Tweets, X Audience Platform). 이제는 각 위치에 대해 표준화된 지표 세트를 반환하며, 기존의 `promoted_tweet_timeline_impressions`, `promoted_tweet_search_impressions`, `promoted_tweets_profile_impressions`, `promoted_tweets_tpn_impressions` 대신 아래 범주 중 하나로 요청할 때 단일 지표인 `impressions`로 제공됩니다(모든 지표에 동일하게 적용):
  - `ALL_ON_TWITTER`
  - `PUBLISHER_NETWORK`
  - 요청 시 UI의 값과 손쉽게 일치하도록 단일 `impressions` 지표가 반환됩니다.
  - `ALL_ON_TWITTER`와 `PUBLISHER_NETWORK` 데이터를 모두 얻으려면 각각 별도의 두 번 쿼리를 실행해야 하며, 둘을 결합할 수 없습니다.
- 개발자 여러분의 피드백을 반영해 [비동기 분석 엔드포인트](/ko/x-ads-api/analytics#asynchronous-analytics)를 이제 사용할 수 있습니다.
  - 즉시 필요하지 않거나 과거 데이터를 가져올 때 사용할 수 있는, 통계를 비동기적으로 요청하는 새 엔드포인트 세트.
  - 새로운 단일 엔드포인트로 통계 작업을 대기열에 추가하세요. 리소스가 허용하는 범위 내에서 요청한 데이터를 수집합니다.
  - 작업 상태 엔드포인트를 조회해 데이터 이용 가능 여부를 확인할 수 있습니다.
  - 데이터가 준비되면 동기 엔드포인트와 동일한 형식의 JSON 응답을 다운로드할 수 있도록 픽업 ID를 제공합니다.
  - 단일 작업에서 최대 90일치 데이터와 최대 20개 엔터티를 조회할 수 있습니다.
- v0 지표를 v1 지표에 매핑한 내용이 포함된 analytics v1 마이그레이션 가이드를 참조하세요
- Sandbox 개선 사항 \* 이제 Sandbox 환경에서 [여러 테스트 광고 계정](/ko/x-ads-api/campaign-management#post-accounts)을 생성할 수 있습니다. \* 이제 Sandbox 환경에서만 테스트 광고 계정에 여러 결제 수단을 생성할 수 있습니다. 이를 통해 모든 결제 수단 유형을 테스트할 수 있습니다. 이전에는 테스트에 `CREDIT_CARD` 결제 수단만 사용할 수 있었습니다. \* 베타 기능을 테스트하고 싶으신가요? 이제 Sandbox 환경에서 계정의 기능을 켜거나 끄는 토글을 통해 테스트 요구 사항에 맞게 설정할 수 있습니다.
