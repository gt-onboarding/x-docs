<div id="overview">
  ## 개요
</div>

Catalog API는 광고주가 제품 피드를 설정하고, 제품을 세트로 묶고, 카탈로그 제품을 일괄적으로 관리할 수 있게 하는 커머스 솔루션입니다. Catalog API는 프로그래매틱 카탈로그 관리를 지원하는 핵심 도구로, 카탈로그의 수집 및 업데이트 방식을 광고주가 보다 세밀하게 제어할 수 있도록 합니다.

카탈로그에는 사용자의 제품과 제품 세트에 관한 모든 정보가 포함되며, 사용자의 핸들에 연결됩니다. 현재는 **한 사용자가 하나의 카탈로그만 생성할 수 있습니다**. 

Catalog API는 제품을 수집하는 두 가지 방식을 지원합니다:

1.  Scheduled Feed: 사용자가 주기적으로 가져올 피드 URL(CSV, TSV, XML)을 추가할 수 있습니다(X는 현재 최대 8GB 파일을 지원합니다)

2.  Batch Products API: 사용자는 배치(JSON) 요청으로 제품 속성을 조회, 생성, 업데이트 및 삭제할 수 있습니다

X 판매자는 필터 규칙을 기반으로 제품 세트를 생성할 수 있습니다. 이를 통해 제품에 추가 메타데이터를 부여하고, 오가닉 커머스 기능 또는 동적 제품 광고를 통해 해당 제품을 선보이기 위한 특별한 제품 세트를 만들 수 있습니다. Catalog API와 X 쇼핑 매니저는 두 가지 유형의 제품 세트를 지원합니다. 

1.  Manual: 사용자는 최대 50개의 제품을 선택하고 이름과 설명을 지정할 수 있습니다

2.  Filter: 사용자는 최대 30개의 필터를 추가하여 해당 조건을 충족하는 제품으로 세트를 자동 생성할 수 있습니다. 필터 유형에는 가격, Google 제품 카테고리, 제품 유형, 재고, 할인 가격, 사용자 지정 필드가 포함됩니다.

Catalog API에는 카탈로그 관리를 위한 4개의 엔드포인트가 제공됩니다:

* [Product Catalogs](https://developer.x.com/en/docs/x-ads-api/catalog-management/api-reference/product-catalogs) \- 제품 정보 컨테이너 생성
* [Products](https://developer.x.com/en/docs/x-ads-api/catalog-management/api-reference/products) -  카탈로그에서 제품을 동기적으로 추가, 업데이트, 조회 및 제거
* [Product Sets](https://developer.x.com/en/docs/x-ads-api/catalog-management/api-reference/product-sets) -  제품 정보를 기준으로 제품을 그룹으로 분류
* [Schedule Feeds](https://developer.x.com/en/docs/twitter-ads-api/catalog-management/api-reference/scheduled-feeds) \- 파일 기반 제품 데이터를 카탈로그에 비동기적으로 수집하도록 설정

<div id="prerequisites-for-the-catalog-api">
  ### Catalog API 사용을 위한 사전 준비 사항
</div>

Catalog API 엔드포인트는 현재 초기 액세스(early access)로만 제공됩니다. 액세스를 신청하려면 X 담당자에게 문의하세요. 

[X Shopping Manager](https://ads.x.com/shopping_manager)에서 서비스 약관에 동의해야 합니다.

Shopping Manager 및 제품 사양에 대한 자세한 내용은 [제품 사양 가이드](https://business.x.com/en/help/shopping-specs.html)를 참조하세요.

### 레이트 리밋 

다음 표에는 각 엔드포인트의 레이트 리밋이 나열되어 있습니다.
