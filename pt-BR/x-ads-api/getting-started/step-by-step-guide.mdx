<div id="how-to-get-access-to-the-ads-api">
  ## Como obter acesso à API de Anúncios
</div>

1. Inscreva-se para uma [conta de desenvolvedor](https://developer.x.com/en/apply-for-access).
2. Crie um [App de desenvolvedor](https://developer.x.com/app) e mantenha seu token em segurança.
3. Acesse [ads.x.com/help](https://ads.x.com/help) para solicitar acesso à API de Anúncios para cada um dos seus Apps de desenvolvedor.

<Note>**Nota:** Se você já está desenvolvendo na X Developer Platform e possui uma conta de desenvolvedor, vá direto para o passo três.</Note>

<div id="step-one-signup-for-a-developer-account">
  ### Etapa 1: crie uma conta de desenvolvedor
</div>

Para fazer uma requisição a qualquer produto da API do X, primeiro você precisa criar uma conta de desenvolvedor. No portal do desenvolvedor, crie um Projeto e um App de desenvolvedor. Isso fornecerá um conjunto de credenciais que você usará para autenticar todas as requisições à API.

<Button href="https://developer.x.com/en/apply-for-access">
  Crie uma conta de desenvolvedor
</Button>

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### Etapa dois: Salve a chave e os tokens do seu App e mantenha-os seguros
</div>

No seu App de desenvolvedor, você receberá um conjunto de chaves de API (também chamadas de Consumer Keys). Você também poderá gerar um conjunto de tokens de acesso, que podem ser usados para fazer solicitações em nome da sua conta pessoal do X, e um token Bearer, que pode ser usado para autenticar endpoints que exigem OAuth 2.0 Bearer Token. Como essas chaves e tokens não expiram a menos que sejam regenerados, sugerimos criar variáveis de ambiente ou usar um gerenciador de senhas seguro.

Anote também o ID do seu App, que pode ser encontrado na URL da página de detalhes do App. Isso facilitará a solicitação de acesso à API de Anúncios do X na próxima etapa.

<Note>
**Nota:** Suas chaves e tokens serão exibidos apenas uma vez no portal do desenvolvedor, portanto, é importante armazenar essas credenciais no seu gerenciador de senhas assim que forem geradas.

Se você perder ou esquecer as chaves e tokens, será necessário regenerá-los, o que cria novas chaves e tokens e invalida os anteriores. Isso significa que você terá que atualizar quaisquer integrações que tenha configurado com suas credenciais anteriores.

Saiba mais sobre nossas [práticas recomendadas de autenticação](https://developer.x.com/en/docs/authentication/guides/authentication-best-practices).
</Note>

<div id="step-three-apply-for-access-to-the-ads-api">
  ### Terceira etapa: solicite acesso à API de Anúncios
</div>

Neste ponto, você terá [acesso básico à API do X](https://developer.x.com/en/docs/twitter-api/v1), mas não às funcionalidades específicas da API de Anúncios do X. Em seguida, será necessário solicitar e obter aprovação para acesso à API de Anúncios. Visite [ads.x.com/help](https://ads.x.com/help) para enviar uma solicitação de acesso à API de Anúncios para cada um dos seus apps de desenvolvedor.

<div id="access-tiers">
  ### Níveis de acesso
</div>

Como parte do processo de inscrição, você precisará especificar qual nível de acesso precisa. Saiba mais sobre [permissões no nível do App e no nível da Conta de Anúncios](https://developer.x.com/en/docs/twitter-ads-api/obtaining-ads-account-access#levels-of-access).

<div id="conversion-only">
  #### Somente conversão
</div>

Acesso aos endpoints de conversão para dispositivos móveis e web, com permissões de leitura e escrita.

<div id="standard-access">
  #### Acesso padrão
</div>

Acesso aos endpoints de Analytics, Gerenciamento de Campanhas, Criativos, Públicos personalizados e Conversões, com permissões de leitura e escrita.

<Note>**Observação:** Após a aprovação do seu app para acesso à API de Anúncios do X, será necessário [regenerar quaisquer tokens de acesso de usuário](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens) para fazer corretamente [solicitações autenticadas à API de Anúncios](https://developer.x.com/en/docs/twitter-ads-api/making-authenticated-requests).</Note>

<Note>**Observação:** Desenvolvedores da API de Anúncios que solicitaram acesso antes de julho de 2023 podem ter níveis diferentes de acesso e permissões e podem estar limitados a cinco tokens OAuth. Consulte nosso guia sobre [como aumentar o acesso](https://developer.x.com/en/docs/twitter-ads-api/increasing-access) para obter acesso a endpoints adicionais ou remover os limites de tokens de aplicativos existentes.</Note>

<div id="make-your-first-request">
  ## Faça sua primeira requisição
</div>

Para testar seu acesso à API de Anúncios do X, faça uma requisição ao endpoint [GET accounts](https://developer.x.com/en/docs/twitter-ads-api/campaign-management/api-reference/accounts). Esse endpoint retornará as contas de anúncios às quais o usuário atualmente autorizado tem acesso. Use os IDs dessas contas na sequência de requisições à API para ler e gravar dados de uma conta de anúncios específica. Usando o [Twurl](https://developer.x.com/en/docs/tutorials/using-twurl) na linha de comando, a requisição seria semelhante ao seguinte.

**Exemplo de requisição**

```bash
twurl -H ads-api.x.com "/11/accounts"
```
