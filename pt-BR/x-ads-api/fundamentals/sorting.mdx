---
title: Ordenação
---

A ordenação está disponível para a maioria dos endpoints de coleção da API de Anúncios (que retornam uma lista de objetos). Dependendo do endpoint, você encontrará vários parâmetros habilitados para ordenação. A maioria dos campos retornados por esses parâmetros (exceto IDs e ENUMs) poderá ser ordenada.

| Coleção da API | Parâmetros ordenáveis |
| :--- | :--- |
| [Accounts](/pt-BR/x-ads-api/campaign-management#accounts) | `created_at`, `updated_at`, `deleted`, `name` |
| [Funding Instruments](/pt-BR/x-ads-api/campaign-management#funding-instruments) | `created_at`, `updated_at`, `deleted`, `funded_amount_local_micro`, `start_time`, `end_time` |
| [Campaigns](/pt-BR/x-ads-api/campaign-management#campaigns) | `created_at`, `updated_at`, `deleted`, `name`, `start_time`, `end_time`, `daily_budget_amount_local_micro`, `total_budget_amount_local_micro`, `standard_delivery` |
| [Line Items](/pt-BR/x-ads-api/campaign-management#line-items) | `created_at`, `updated_at`, `deleted`, `bid_amount_local_micro` |
| [Cards](https://developer.x.com/en/docs/ads/creatives/api-reference/image-app-download) | `created_at`, `updated_at`, `deleted`, `name` |
| [Promoted Accounts](/pt-BR/x-ads-api/campaign-management#advertiser-api) | `created_at`, `updated_at`, `deleted`, `paused` |
| [Promoted Tweets](/pt-BR/x-ads-api/campaign-management#promoted-tweets) | `created_at`, `updated_at`, `deleted`, `paused` |
| [App Event Tags](/pt-BR/x-ads-api/measurement/mobile-conversions#app-event-tags) | `created_at`, `updated_at`, `deleted`, `post_view_attribution_window`, `post_engagement_attribution_window`, `assisted_conversion`, `provider_app_event_name` |

<div id="getting-started">
  ## Introdução
</div>

Na sua solicitação à API, basta acrescentar `sort_by=[attribute name]-[asc/desc]`, em que o atributo é um campo válido retornado no JSON da sua solicitação GET.

Por exemplo, você pode usar `?name-asc` para ordenar por nome em ordem alfabética ou `?name-desc` para ordenar em ordem inversa.
