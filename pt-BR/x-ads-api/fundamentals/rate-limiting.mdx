---
title: Limitação de taxa
---

A API de Anunciantes tem limitação de taxa semelhante à REST API v1.1, conforme documentado aqui: [Limitação de taxa da REST API v1.1](https://developer.x.com/content/developer-twitter/en/docs/basics/rate-limits). Diferentemente da REST API v1.1, não há um índice programático dos limites por endpoint. Os limites de taxa por endpoint e as janelas de redefinição são informados nos cabeçalhos de resposta HTTP. Toda a limitação de taxa na API de Anúncios utiliza OAuth 1.0A.

<div id="user-level-and-ad-account-level-limits">
  ## Limites no Nível de Usuário e no Nível de Conta de Anúncios
</div>

Existem dois tipos de limites de taxa: no nível do token de usuário e no nível da conta de anúncios. Um subconjunto de endpoints está habilitado para usar limitação de taxa no nível da conta de anúncios. Um token de usuário é o [token de acesso OAuth](/pt-BR/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) que você usa para autenticar e chamar a API de Anúncios do X. Cada token de usuário pode ter acesso a uma ou mais contas de anúncios.

Os desenvolvedores devem usar o limite de taxa no nível da conta de anúncios quando ele for retornado nos cabeçalhos de resposta e usar apenas o limite no nível de usuário quando o limite da conta de anúncios não estiver presente.

Os limites de taxa no nível de usuário são expressos por estes cabeçalhos: **x-rate-limit-limit**, **x-rate-limit-remaining** e **x-rate-limit-reset**.

Para endpoints com limitação de taxa no nível da conta de anúncios habilitada, os limites de taxa são expressos por estes cabeçalhos: **x-account-rate-limit-limit**, **x-account-rate-limit-remaining** e **x-account-rate-limit-reset**.

Os limites de taxa no nível da conta de anúncios são fornecidos apenas para endpoints GET, a fim de permitir que aplicativos sincronizem dados de entidades (como objetos de campanha ou itens de linha) usando um único token de usuário que acessa várias contas de anúncios. Ações de escrita não têm garantia de usar os mesmos limites de taxa no nível da conta de anúncios.

Para endpoints com limitação de taxa no nível da conta de anúncios, o limite de taxa no nível de usuário é definido em um valor alto, representando uma cota global para todo o seu aplicativo. Quando disponível, o limite de taxa no nível da conta de anúncios deve ter precedência no controle do seu volume de solicitações.

<div id="best-practices">
  ## Melhores práticas
</div>

1. Salve no seu banco de dados o timestamp da última sincronização e, ao solicitar dados, quando aplicável, use a opção `sort_by=updated_at-desc` para permitir interromper o processo assim que chegar a dados anteriores ao seu último timestamp. Isso evitará sincronizações redundantes dos mesmos dados.

2. Solicite várias entidades em uma única requisição: alguns endpoints permitem especificar uma lista de valores separada por vírgulas para recuperar vários conjuntos de dados semelhantes. Isso pode reduzir o número total de chamadas e, assim, aproveitar o limite de taxa com mais eficiência.

3. Solicite o valor máximo de “count” em suas requisições: alguns endpoints, como [GET accounts/:account\_id/targeting\_criteria](/pt-BR/x-ads-api/campaign-management#targeting-criteria), devem ser chamados com o valor máximo de count para retornar 1000 objetos em vez do padrão de 200.

<div id="faqs">
  ## FAQs
</div>

_É possível aumentar os limites de taxa para uma conta de anúncios específica ou para nosso aplicativo?_

Em geral, não podemos aumentar os limites de taxa, que são definidos para atender até as maiores contas de anúncios. Implemente as Melhores Práticas listadas neste documento como primeiro passo e, se os limites de taxa ainda estiverem impactando sua capacidade de escalar ou atingir objetivos de negócios, entre em contato com seus representantes da API de Anúncios do X, fornecendo todos os detalhes sobre o caso de uso e as solicitações envolvidas.
