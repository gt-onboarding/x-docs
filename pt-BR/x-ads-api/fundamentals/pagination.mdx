---
title: Paginação
---

Para oferecer suporte a um número maior de campanhas e à recuperação eficiente de todas as entidades associadas a uma conta, a API de Anúncios agora oferece paginação em muitos endpoints GET. O mecanismo de paginação é fácil de usar e muito semelhante à paginação baseada em cursor da REST API, conforme descrito em [Usando cursores para navegar em coleções](https://developer.x.com/en/docs/x-api/v1/pagination).

<div id="getting-started">
  ## Primeiros passos
</div>

Para determinadas solicitações GET, agora aceitamos os parâmetros de consulta `cursor` e `count`, ambos opcionais. Se uma resposta tiver mais entidades do que o valor de `count`, esses endpoints retornarão as primeiras `count` entidades e incluirão uma chave `next_cursor` no JSON de resposta.

`GET https://ads-api.x.com/5/accounts/abc1/campaigns?count=50`

```json
{
  "data": \[...\],
  "next_cursor": "c-3yvu1pzhd3i7",
  "request": {...}
}
```

Para obter a próxima página de resultados, adicione `cursor` aos parâmetros de consulta:

`GET https://ads-api.x.com/5/accounts/abc1/campaigns?cursor=c-3yvu1pzhd3i7&count=50`

```json
{
  "data": \[...\],
  "next_cursor": "c-3w3zdyg8ywan",
  "request": {...}
}
```

<div id="going-deeper">
  ## Aprofundando
</div>

Para a maioria dos endpoints, o valor máximo de `count` é **1.000**, o mínimo é **1** e o padrão é **200**.

O valor fornecido por `next_cursor` é sempre uma string e deve ser tratado como opaco; a implementação pode mudar. Se menos de `count` entidades forem retornadas na página atual do conjunto de resultados, o valor de `next_cursor` será `null`.

**Observação:** os endpoints de analytics não oferecem suporte a esse tipo de paginação. A paginação nos endpoints de estatísticas é feita especificando períodos de tempo. Consulte a documentação desses endpoints individuais para mais informações.
