<div id="timezones-accounts-and-billing">
  ## Fusos horários, contas e faturamento
</div>

Valores de data e hora são sempre retornados em UTC (conforme indicado pelo Z no final do valor). Datas e horas podem ser especificadas em qualquer fuso horário em um comando POST ou PUT usando o formato padrão ISO 8601 para fuso horário. O horário é representado usando um subconjunto do [ISO 8601](http://en.wikipedia.org/wiki/ISO_8601). Mais especificamente, a string strptime para nosso formato de data é `%Y-%m-%dT%l:%M:%S%z`. O fuso horário da conta do anunciante determina o momento exato em que os números oficiais de faturamento são congelados.

Ao consultar a API no nível da conta ([GET accounts](/pt-BR/x-ads-api/campaign-management#accounts)), você obterá informações de fuso horário como estas:

```json
    {
      "request": {
        "params": {
          "account_id": "18ce54d4x5t"
        }
      },
      "data": {
        "name": "API McTestface",
        "business_name": null,
        "timezone": "America/Los_Angeles",
        "timezone_switch_at": "2016-07-21T07:00:00Z",
        "id": "18ce54d4x5t",
        "created_at": "2016-07-21T22:42:09Z",
        "salt": "54cb7b5a34183f77d82dd6d0f4329691",
        "updated_at": "2017-09-09T06:42:14Z",
        "business_id": null,
        "approval_status": "ACCEPTED",
        "deleted": false
      }
    }
```

Isso inclui dois valores de fuso horário: `timezone` (consulte [fusos horários na Wikipedia](https://en.wikipedia.org/wiki/Tz_database#Names_of_time_zones) para uma visão geral) e `timezone_switch_at`. Observe que `timezone_switch_at` é fornecido no fuso horário UTC (+00:00), mas sempre representará meia-noite no fuso horário indicado.

O fuso horário do anunciante não é editável via API. Esse atributo é definido no nível contratual/de faturamento pelo gerente de contas do anunciante no X.

Fique atento ao valor `timezone_switch_at` ao criar relatórios e consultar nossos endpoints de análise, pois haverá uma lacuna no dia em que a conta mudar do fuso horário America/Los\_Angeles para o novo valor local.

<div id="specifying-datetime-values-with-timezone">
  ### Especificando valores de data e hora com fuso horário
</div>

Os valores de data e hora são sempre retornados em UTC (conforme indicado pelo `Z` no final do valor). Datas e horas podem ser especificadas em qualquer fuso horário em um comando POST ou PUT usando o formato de fuso horário do [padrão ISO 8601](http://en.wikipedia.org/wiki/ISO_8601). Por exemplo, `2017-07-10T08:00:00-0800` é um valor de entrada aceitável e será convertido automaticamente para o valor em UTC `2017-07-10T16:00:00Z`.

Ao usar os endpoints de analytics com `granularity` igual a `DAY` ou `TOTAL`, o valor de `start_time` deve ser especificado à meia-noite do dia desejado no fuso horário local do titular da conta. O deslocamento de fuso horário a ser usado será o do dia atual, não o do dia em questão. Por exemplo, para uma conta de anúncios em America/Los\_Angeles durante o horário de verão do Pacífico, o deslocamento em relação ao UTC é -0700. Assim, em uma solicitação de analytics, o horário deve ser especificado como: `start_time=2017-05-21T07:00:00Z` ou `start_time=2017-05-21T00:00:00-0700`. Se a conta de anúncios estivesse em Asia/Tokyo, onde o deslocamento é sempre +09:00, os valores seriam especificados como: `start_time=2017-05-20T15:00:00Z` ou `start_time=2017-05-21T00:00:00+0900`.

<div id="accepted-utc-offset-formats">
  ### Formatos de deslocamento de UTC aceitos
</div>

Consulte [designadores de fuso horário do ISO 8601](http://en.wikipedia.org/wiki/ISO_8601#Time_zone_designators).

Compatíveis: `Z`, `-HHMM`, `+HHMM`, `-HH:MM`, `+HH:MM`, `-HH`, `+HH`
