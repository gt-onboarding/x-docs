<Update label="Aug 3, 2022" description="API do X v2">
  ### Adição de aliases de operadores de filtragem

  Foram adicionados aliases de operadores de filtragem/consulta aos endpoints de busca e Fluxo filtrado, nas versões enterprise, premium e v2. Em muitos casos, o operador se refere a posts em vez de statuses, por exemplo, "in\_reply\_to\_tweet\_id" como um alias para "in\_reply\_to\_status\_id". Em outros casos, o novo alias do operador traz mais clareza à correspondência realizada. Por exemplo, como os posts podem ter apenas um vídeo anexado, um novo alias "has:video\_link" é fornecido para "has:videos". Confira a [tabela de operadores do Fluxo filtrado v2](https://developer.x.com/en/docs/twitter-api/tweets/filtered-stream/integrate/build-a-rule#list) para ver mais exemplos.
</Update>

<Update label="Dec 14, 2021" description="API do X v2: Authentication">
  ### Suporte para autenticação OAuth 2.0

  Hoje, todos os desenvolvedores já podem autenticar usando OAuth 2.0, selecionando OAuth 2.0 como método de autenticação no portal do desenvolvedor. Adicionamos suporte para clientes confidenciais e públicos, e todos os endpoints v2 relevantes podem usar esse método de autenticação como parte deste lançamento.

  OAuth 2.0 é um protocolo de autorização padrão do setor que oferece aos desenvolvedores mais controle sobre os escopos de um aplicativo e melhora os fluxos de autorização em vários dispositivos. Em outras palavras, desenvolvedores que criam aplicativos para pessoas no X terão mais controle sobre as informações que seu app solicita de seus usuários, de modo que você só precise pedir aos seus usuários finais os dados e informações de que precisa.

  Esse protocolo moderno de autorização permitirá apresentar aos seus usuários finais um fluxo de consentimento mais simplificado para autorizar seu app, exibindo apenas os escopos específicos que você solicitou.

  Para saber mais sobre este lançamento, visite nosso anúncio no fórum.
</Update>

<Update label="Nov 15, 2021" description="API do X v2: Acesso antecipado">
  ### Atualizações importantes da plataforma para a API do X v2

  **Atualizações importantes da plataforma: a API do X v2 agora é a versão principal, novos acessos Essential e Elevated e mudanças na política**

  Hoje, estamos anunciando que a API do X v2 agora é a versão principal da API do X. Lançamos endpoints e funcionalidades suficientes na API do X v2 para atender às necessidades de 90% de todos os apps existentes desenvolvidos na API do X.

  Estamos lançando as seguintes mudanças hoje para aprimorar ainda mais a experiência de desenvolvimento com a API do X v2:

  - Acesso Essential rápido e gratuito à API e acesso Elevated gratuito para desenvolvedores que tiveram seus casos de uso aprovados. Há muitos mais detalhes sobre esses níveis de acesso em nossa página About the X API.

  - Estamos removendo trechos em nossa Developer Policy que restringiam como você podia desenvolver com os recursos principais do X e limitavam o número de usuários que seu app poderia atender.

    - Estamos abrindo nossa plataforma para incentivar você a criar ferramentas e produtos que tornem o X melhor, mais saudável e que ampliem a conversa pública. [Veja uma lista de soluções que adoraríamos ver você construir](https://developer.x.com/en/docs/twitter-api/what-to-build).

    - Especificamente, removemos termos que restringiam a replicação da experiência do X, incluindo seus recursos principais, bem como termos que exigiam permissão para ter números elevados de tokens de usuário.

    - Sabemos que criar soluções que ajudam as pessoas no X muitas vezes significa que um desenvolvedor precisa criar (ou replicar) algumas das funcionalidades disponíveis no X. Essas mudanças em nossa Developer Policy têm como objetivo trazer clareza para o ecossistema de desenvolvedores e oferecer uma plataforma de API aberta que facilite para os desenvolvedores construir, inovar e impactar a conversa pública.

  Entramos em muito mais detalhes sobre este lançamento e sobre os lançamentos da API do X que levaram a este momento em nossa [postagem no fórum](https://devcommunity.x.com/t/ushering-in-a-new-era-for-the-twitter-developer-platform-with-the-twitter-api-v2/162087).

  Além de explorar entradas anteriores do changelog, reunimos alguns recursos importantes que você pode revisar para entender tudo o que foi lançado na API do X v2 até o momento:

  - [Mapa de endpoints da API do X](/pt-BR/x-api/migrate/x-api-endpoint-map)

  - [About the X API](/pt-BR/x-api/getting-started/about-x-api)
</Update>

<Update label="Dec 18, 2020" description="Campo de configuração de respostas em posts para a API do X v2">
  ### Adição do campo Reply Settings

  Hoje estamos lançando um novo campo no objeto de post chamado reply\_settings. Este campo está disponível em todos os endpoints da API do X v2 que retornam posts, incluindo post lookup, hide replies, recent search, sampled stream, filtered stream, follows lookup, user post timeline e user mention timeline, ou ao [expandir](/pt-BR/x-api/fundamentals/expansions) o objeto de post em qualquer endpoint.

  Este campo indica como o autor do post permitiu que outros respondessem aos seus posts: todos, apenas as pessoas mencionadas no post ou somente aqueles que o autor segue. Os valores retornados pelo campo `reply_settings` incluem `everyone`, `mentionedUsers` e `following`. Se nenhum controle de conversa for definido para o post, o campo ainda aparecerá no objeto de post com o valor `everyone`.
</Update>

<Update label="May 10, 2018" description="X API: Standard v1.1">
  ### Alterações na renderização do payload de citação

  Hoje, estamos adicionando uma alteração na forma como as URLs serão renderizadas no payload de citação.

  Estamos adicionando uma nova entidade chamada "quoted\_status\_permalink" para garantir que o post citado possa ser referenciado e removeremos o link t.co do campo "text" do post citado.

  Com o novo formato, o link t.co não é mais anexado ao campo "text" do post citado no objeto "quoted\_status".

  Além disso, não colocaremos mais a t.co url, expanded\_url e display\_url do post citado na entidade "urls". Em vez disso, elas serão adicionadas a um novo objeto "quoted\_status\_permalink" com atributos de URL "url", "expanded" e "display". Isso significa que o objeto "quoted\_status\_permalink" ficará reservado para o link t.co de retorno ao post citado, enquanto o array "entities.urls" pode ser usado para identificar quaisquer links compartilhados no post original ou na citação.

  Leia nosso [post no fórum](https://devcommunity.x.com/t/updating-how-urls-are-rendered-in-the-quote-tweet-payload/105473) e a [documentação atualizada](https://devcommunity.x.com/t/upcoming-changes-to-the-developer-platform/104603) para mais detalhes.
</Update>
