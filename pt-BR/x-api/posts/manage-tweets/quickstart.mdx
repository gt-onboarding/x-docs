## Introdução aos endpoints de gerenciamento de Posts\*\*

Este guia de início rápido ajudará você a fazer sua primeira solicitação aos endpoints de gerenciamento de Posts usando o [Postman](/pt-BR/tutorials/postman-getting-started).

Visite nosso repositório no GitHub com [código de exemplo da API do X v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code) se quiser ver exemplos de código em diferentes linguagens.

<Info>
  **Pré-requisitos**

  Para concluir este guia, você precisará de um conjunto de [chaves e tokens](/pt-BR/resources/fundamentals/authentication) para autenticar sua solicitação. Você pode gerar essas chaves e tokens seguindo estas etapas:

  - [Inscreva-se para uma conta de desenvolvedor](https://developer.x.com/en/apply-for-access) e aguarde a aprovação.
  - Crie um [Projeto](/pt-BR/resources/fundamentals/projects) e um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) associado no portal do desenvolvedor.
  - Acesse a página “Keys and tokens” do seu App para gerar as credenciais necessárias. Certifique-se de salvar todas as credenciais em um local seguro.
</Info>

<div id="steps-to-building-manage-posts-requests">
  ### Etapas para criar solicitações de gerenciamento de Posts
</div>

**Primeira etapa: comece com uma ferramenta ou biblioteca**

Há várias ferramentas, exemplos de código e bibliotecas que você pode usar para fazer uma solicitação a este endpoint, mas aqui vamos usar o Postman para simplificar o processo.

Para carregar a coleção da API do X v2 do Postman no seu ambiente, clique no botão a seguir:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Adicionar API do X v2 ao Postman
</Button>

Depois de carregar a coleção da API do X v2 no Postman, navegue até a pasta “Manage Posts” e selecione “Create a Post”.
 

**Segunda etapa: autentique sua solicitação**

Para fazer corretamente uma solicitação à API do X, você precisa verificar se tem permissão para isso. Para fazer isso com os endpoints de gerenciamento de Posts, você deve autenticar sua solicitação usando [OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication) ou [OAuth 2.0 Authorization Code with PKCE](pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

Neste exemplo, vamos usar OAuth 1.0a User Context.

Você deve adicionar suas chaves e tokens (especificamente sua API Key, API Secret Key, OAuth 1.0a user Access Token e OAuth 1.0a user Access Token Secret) ao Postman. Você pode fazer isso selecionando o ambiente chamado “X API v2” (no canto superior direito do Postman) e adicionando suas chaves e tokens nos campos “initial value” e “current value” (clicando no ícone de olho ao lado do menu suspenso de ambientes).

Se você fez isso corretamente, essas variáveis serão automaticamente preenchidas na guia de autorização da solicitação.

**Terceira etapa: especifique o texto do Post**

Ao criar um novo Post com este endpoint, o texto ou a mídia do Post são parâmetros obrigatórios do corpo.

No Postman, navegue até a guia “Body” e insira o texto do Post como valor do parâmetro `text`. Além disso, se desejar adicionar parâmetros para itens como enquetes, responder a um ID de Post ou configurar as opções de resposta, você também pode fazê-lo aqui. Saiba mais sobre o que está disponível em nosso [guia de referência da API](/pt-BR/x-api/posts/manage-tweets/introduction).

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Parameter type** |
| `text` | Hello world! | body |

**Quarta etapa: identifique e especifique quais campos você deseja recuperar**

Quando tudo estiver configurado, clique no botão “Send” e você receberá uma resposta semelhante ao exemplo a seguir:

```
{
  "data": {
    "id": "1445880548472328192",
    "text": "Hello world!"
  }
}
```

Se o objeto de resposta retornado contiver um `id` e o `text` do seu Post, você criou um Post com sucesso.
 

**Quinta etapa: exclua seu Post**

Para excluir um Post, selecione a solicitação “Delete a Post”, também localizada na pasta “Manage Posts” da coleção da API do X v2 carregada no Postman. Este endpoint requer o ID do Post que você deseja excluir. Em seguida, na guia “Params”, insira o ID do Post que deseja excluir como valor da coluna `id`. 

Em caso de exclusão bem-sucedida, você receberá uma resposta semelhante ao exemplo a seguir:

```
{
   "data": {
       "deleted" : true
   }
}
```
