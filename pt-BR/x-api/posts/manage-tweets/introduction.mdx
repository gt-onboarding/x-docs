Criar e excluir Posts usando a API do X é essencial para interagir com a conversa pública. Os novos endpoints de gerenciamento de Posts permitem que você faça exatamente isso e muito mais.

Temos dois métodos disponíveis para gerenciar Posts: POST e DELETE. O método POST permite publicar enquetes, citar Tweets, publicar com configurações de resposta, com geolocalização, com mídia e marcação de usuários, e publicar para Super Followers, além de outros recursos. Da mesma forma, o método DELETE permite excluir um Post específico. No método POST, você pode passar [os parâmetros](/pt-BR/x-api/posts/creation-of-a-post) desejados para personalizar ainda mais sua solicitação.

O método delete Post foi atualizado para oferecer suporte a Posts editados. Quando qualquer Post em uma cadeia de edições é excluído, todos os Posts nessa cadeia de edições também são excluídos. Para saber mais sobre os metadados de edição de Post, consulte a página de [fundamentos de edição de Posts](/pt-BR/x-api/fundamentals/edit-posts).

Há um limite por usuário de 200 solicitações a cada 15 minutos para o método POST. O método DELETE tem um limite de 50 solicitações a cada 15 minutos. Além disso, há um limite de 300 solicitações a cada 3 horas, incluindo Posts criados com manage Posts ou manage Retweets.

Como você está fazendo solicitações em nome de um usuário com os endpoints de manage Posts, deve se autenticar com [OAuth 1.0a User Context](https://en.docs/authentication/oauth-1-0a) ou [OAuth 2.0 Authorization Code with PKCE](https://en.docs/authentication/oauth-2-0/authorization-code) e usar tokens de acesso de usuário associados a um usuário que autorizou seu App. Para gerar esse token de acesso de usuário com OAuth 1.0a, você pode usar o [fluxo OAuth de 3 etapas](https://en.docs/authentication/oauth-2-0/bearer-tokens). Para gerar um token de acesso de usuário com OAuth 2.0, você pode usar o [fluxo de concessão Authorization Code com PKCE](https://en.docs/authentication/oauth-2-0/user-access-token).

  **Configuração da conta**

  - Autenticar-se usando as chaves e tokens de um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) que esteja dentro de um [Project](/pt-BR/resources/fundamentals/projects).

  Saiba mais sobre como obter acesso aos endpoints da API do X v2 em nosso [guia de primeiros passos](/pt-BR/x-api/getting-started/getting-access).
