<div id="standard-v11-compared-to-x-api-v2">
  ## Comparação entre o Standard v1.1 e a API do X v2
</div>

Se você tem trabalhado com os endpoints do Standard v1.1 [POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) e [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id), o objetivo deste guia é ajudar você a entender as semelhanças e diferenças entre os endpoints de gerenciamento de posts do Standard e da API do X v2.

- **Semelhanças**
  - Autenticação
- **Diferenças**
  - URLs de endpoints

  - Requisitos de App e Projeto

  - Parâmetros de requisição

<div id="similarities">
  ### Semelhanças
</div>

**Autenticação**

Tanto os endpoints padrão v1.1 de Posts ([POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) e [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id)) quanto os da API do X v2 usam [OAuth 1.0a User Context](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Portanto, se você já utilizava um dos endpoints padrão v1.1, pode continuar a usar o mesmo método de autenticação ao migrar para a versão na API do X v2. 

<div id="differences">
  ### Diferenças
</div>

**URLs de endpoints**

- Endpoints padrão v1.1:
  - https://api.x.com/1.1/statuses/update.json
    (Cria um post)
  - `https://api.x.com/1.1/statuses/destroy/:id.json`
    (Exclui um post)
- Endpoint da API do X v2:
  - https://api.x.com/2/tweets
    (Cria um post)
  - https://api.x.com/2/tweets/:id
    (Exclui um post específico)

<div id="app-and-project-requirements">
  ### Requisitos de App e Projeto
</div>

Os endpoints da API do X v2 exigem o uso de credenciais de um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) associado a um [Projeto](/pt-BR/resources/fundamentals/developer-apps) ao autenticar suas solicitações. Todos os endpoints da API do X v1.1 podem usar credenciais de Apps independentes ou de Apps associados a um projeto.

<div id="request-parameters">
  ### Parâmetros da solicitação
</div>

Os seguintes parâmetros padrão da v1.1 aceitavam dois parâmetros de consulta (user\_id ou screen\_name). A API do X v2 aceita apenas o ID numérico do Post para o endpoint DELETE, e ele deve ser fornecido como parte do caminho do endpoint.

Para o endpoint POST, parâmetros adicionais precisam ser enviados no corpo JSON da solicitação. Você pode saber mais sobre quais parâmetros estão disponíveis no [guia de referência da API](/pt-BR/x-api/posts/manage-tweets/introduction).
