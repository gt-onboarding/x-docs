<div id="likes-lookup-standard-v11-compared-to-x-api-v2">
  ### Consulta de curtidas: Standard v1.1 em comparação com a API do X v2
</div>

Se você vem trabalhando com o endpoint [GET favorites/list](https://developer.x.com/en/docs/x-api/v1/tweets/post-and-engage/api-reference/get-favorites-list) do Standard v1.1, o objetivo deste guia é ajudar você a entender as semelhanças e diferenças entre os endpoints de consulta de curtidas do Standard v1.1 e da API do X v2.

Com a v2, também apresentamos um novo endpoint de usuários que curtiram, que permite obter informações sobre os usuários que curtiram uma publicação.

- **Semelhanças**
  - Autenticação
  - Limites de taxa
- **Diferenças**
  - URLs de endpoint

  - Limitações de solicitação

  - Requisitos de App e Projeto

  - Parâmetros de solicitação

  - Novo formato JSON

<div id="similarities">
  #### Semelhanças
</div>

**Autenticação**

Tanto os endpoints de consulta de Likes do padrão v1.1 quanto da API do X v2 usam [OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication#oauth-1-0a-2) ou [OAuth 2.0 Bearer Token](/pt-BR/resources/fundamentals/authentication#bearer-token-also-known-as-app-only). Portanto, se você usava anteriormente os endpoints padrão v1.1 [GET favorites/list](https://developer.x.com/en/docs/x-api/v1/tweets/post-and-engage/api-reference/get-favorites-list), pode continuar usando o mesmo método de autenticação ao migrar para a versão da API do X v2, se desejar. 

Dependendo da sua biblioteca/pacote de autenticação preferido, a autenticação com Bearer Token provavelmente é a forma mais simples de começar e pode ser configurada com um cabeçalho de requisição simples. Para saber como gerar um Bearer Token, consulte [este guia de OAuth 2.0 Bearer Token](/pt-BR/resources/fundamentals/authentication#bearer-token-also-known-as-app-only). 
 

**Limites de taxa**

O endpoint padrão v1.1 GET favorites/list tem um limite de 75 requisições por 15 minutos por usuário. O endpoint correspondente de Posts curtidos no v2 também tem esse mesmo limite. No entanto, esse endpoint v2 também possui um limite adicional de 75 requisições por 15 minutos por App.

<div id="differences">
  #### Diferenças
</div>

**URLs de endpoints**

- Endpoints padrão v1.1:
  - GET https://api.x.com/1.1/favorites/list.json
    (lista de Posts do usuário especificado)
  - Não há equivalente na v1.1 para o endpoint da v2 de usuários que curtiram
- Endpoint da API do X v2:
  - GET https://api.x.com/2/users/:id/liked\_tweets
    (lista de Posts curtidos pelo ID de usuário especificado)
  - GET https://api.x.com/2/tweets/:id/liking\_users
    (lista de usuários que curtiram o Post com o ID especificado)

**Limitações de solicitação**

O endpoint da v2 para Posts curtidos permite solicitar de 5 a 100 Posts por solicitação, mas você pode obter todas as curtidas de um Post usando tokens de paginação. O endpoint v1.1 GET favorites/list também permite obter todas as curtidas de Posts, porém é possível recuperar de 20 a 200 Posts por solicitação.

Para o endpoint da v2 de usuários que curtiram, há um limite de 100 usuários que curtiram por Post.
 

**Requisitos de App e Projeto**

Os endpoints da API do X v2 exigem que você use credenciais de um [developer App](/pt-BR/resources/fundamentals/developer-apps) associado a um [Project](/pt-BR/resources/fundamentals/developer-apps) ao autenticar suas solicitações. Todos os endpoints da API do X v1.1 podem usar credenciais de Apps independentes ou Apps associados a um projeto.

**Parâmetros de solicitação**

Os endpoints padrão da v1.1 aceitavam dois parâmetros de query (user\_id ou screen\_name). A API do X v2 aceita apenas o ID numérico do usuário, que deve ser passado como parte do caminho do endpoint.

Uma das maiores diferenças entre as versões de endpoints padrão v1.1 e API do X v2 é como você seleciona quais campos retornam no payload. Nos endpoints padrão, havia vários parâmetros para indicar quais campos ou conjuntos de campos retornariam no payload, enquanto a versão da API do X v2 simplifica esses parâmetros em [fields](/pt-BR/x-api/fundamentals/fields) e [expansions](/pt-BR/x-api/fundamentals/expansions).
 

**Novo formato JSON**

A API do X v2 está introduzindo novos modelos JSON para os objetos retornados pelas APIs, incluindo objetos de [Post](/pt-BR/x-api/fundamentals/data-dictionary#tweet) e de [user](/pt-BR/x-api/fundamentals/data-dictionary#user).

- No nível raiz do JSON, os endpoints padrão retornam objetos de usuário em um array statuses, enquanto a API do X v2 retorna um array data.
- Em vez de se referir a Retweeted e Quoted “statuses”, o JSON da API do X v2 se refere a Retweeted e Quoted Tweets. Muitos campos legados e obsoletos, como contributors e user.translator\_type, estão sendo removidos.
- Em vez de usar tanto favorites (no objeto Post) quanto favourites (no objeto user), a API do X v2 usa o termo like.
- O X está adotando a convenção de que valores JSON sem valor (por exemplo, null) não são escritos no payload. Atributos de Post e user só são incluídos se tiverem valores não nulos.
   

Além das mudanças feitas no novo formato JSON, também introduzimos um novo conjunto de campos no objeto Post, incluindo:

- [conversation\_id](/pt-BR/x-api/fundamentals/conversation-id)
- Dois novos campos de [Post annotations](/pt-BR/x-api/fundamentals/post-annotations), incluindo context e entities
- Vários novos campos de [metrics](/pt-BR/x-api/fundamentals/metrics)
