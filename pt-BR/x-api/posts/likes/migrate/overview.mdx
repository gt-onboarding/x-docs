<div id="comparing-x-apis-likes-endpoints">
  ## Comparando os endpoints de Likes da API do X
</div>

Estes guias abordarão as seguintes áreas:

- **Parâmetros de requisição da API** - O endpoint da API do X v2 introduz um novo conjunto de parâmetros de requisição. Embora alguns parâmetros sejam familiares, especialmente para quem integrou com o Labs, há muitas diferenças importantes, como a introdução dos parâmetros [fields](/pt-BR/x-api/fundamentals/fields) e [expansions](/pt-BR/x-api/fundamentals/expansions).

- **Requisitos de App e Projeto** - Para acessar a API do X v2, você precisará usar credenciais de um [developer App](/pt-BR/resources/fundamentals/developer-apps) associado a um [Project](/pt-BR/resources/fundamentals/projects)

<div id="likes-lookup">
  #### Consulta de curtidas
</div>

**Usuários que curtiram um Post**

O endpoint de usuários que curtiram é uma funcionalidade nova na v2 e permite obter informações sobre os usuários que curtiram um Post.

| Descrição | API do X v2 |
| :--- | :--- |
| Métodos HTTP compatíveis | GET |
| Domínio do host | https://api.x.com |
| Caminho do endpoint | /2/tweets/:id/liking\_users |
| [Autenticação](/pt-BR/resources/fundamentals/authentication) | OAuth 2.0 Bearer Token<br /><br />OAuth 1.0a User Context |
| [Limites de taxa](/pt-BR/resources/fundamentals/rate-limits) padrão de solicitação | 75 solicitações por 15 min (por App)<br /><br />75 solicitações por 15 min (por usuário) |

**Posts curtidos por um usuário**

As tabelas a seguir comparam o endpoint padrão v1.1 [GET favorites/list](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-favorites-list) e os endpoints de Posts curtidos da API do X v2:

| Descrição | Padrão v1.1 | API do X v2 |
| :--- | :--- | :--- |
| Métodos HTTP compatíveis | GET | GET |
| Domínio do host | https://api.x.com | https://api.x.com |
| Caminho do endpoint | /1.1/favorites/list.json | /2/users/:id/liked\_tweets |
| [Autenticação](/pt-BR/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 2.0 Bearer Token<br /><br />OAuth 1.0a User Context |
| [Limites de taxa](/pt-BR/resources/fundamentals/rate-limits) padrão de solicitação | 75 solicitações por 15 min | 75 solicitações por 15 min (por App)<br /><br />75 solicitações por 15 min (por usuário) |
| Formatos de dados | Formato padrão v1.1 | [Formato da API do X v2](/pt-BR/x-api/fundamentals/data-dictionary) (determinado pelos parâmetros de solicitação fields e expansions, não compatível com os formatos da v1.1)<br /><br />Para saber mais sobre como migrar do formato padrão v1.1 para o formato da API do X v2, visite nosso [guia de migração de formatos de dados](/pt-BR/x-api/migrate/data-format-migration). |

<div id="manage-likes">
  #### Gerenciar curtidas
</div>

Os endpoints de gerenciamento de curtidas da v2 substituem os endpoints v1.1 [POST favorites/create](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-create) e [POST favorites/destroy](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-destroy).

As tabelas a seguir comparam os endpoints padrão da v1.1 e da API do X v2 para gerenciamento de curtidas:

<div id="like-a-post">
  #### Curtir um post
</div>

| Descrição | Padrão v1.1 | API do X v2 |
| :--- | :--- | :--- |
| Métodos HTTP compatíveis | POST | POST |
| Domínio do host | https://api.x.com | https://api.x.com |
| Caminho do endpoint | /1.1/favorites/create.json | /2/users/:id/likes |
| [Autenticação](/pt-BR/resources/fundamentals/authentication) | Contexto de usuário OAuth 1.0a | Contexto de usuário OAuth 1.0a |
| [Limites de taxa](/pt-BR/resources/fundamentals/rate-limits) padrão de solicitação | 1000 solicitações por 24 horas (por usuário)<br /><br />1000 solicitações por 24 horas (por app) | 50 solicitações por 15 min (por usuário)<br /><br />1000 solicitações por 24 horas (por usuário, compartilhado com DELETE) |

<div id="unlike-a-post">
  #### Remover curtida de um Post
</div>

| Descrição | Standard v1.1 | API do X v2 |
| :--- | :--- | :--- |
| Métodos HTTP compatíveis | POST | DELETE |
| Domínio do host | https://api.x.com | https://api.x.com |
| Caminho do endpoint | /1.1/favorites/destroy.json | /2/users/:id/likes/:tweet\_id |
| [Autenticação](/pt-BR/resources/fundamentals/authentication) | Contexto de usuário OAuth 1.0a | Contexto de usuário OAuth 1.0a |
| Limites padrão de [taxa de solicitação](/pt-BR/resources/fundamentals/rate-limits) | 1.000 solicitações por 24 horas (por usuário)<br /><br />1.000 solicitações por 24 horas (por App) | 50 solicitações por 15 min (por usuário)<br /><br />1.000 solicitações por 24 horas (por usuário, compartilhado com POST) |
