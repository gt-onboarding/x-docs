<div id="manage-likes-standard-v11-compared-to-x-api-v2">
  ### Gerenciar curtidas: Standard v1.1 em comparação com a API do X v2
</div>

Se você vem trabalhando com os endpoints padrão v1.1 [POST favorites/create](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-create) e [POST favorites/destroy](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-destroy), o objetivo deste guia é ajudar você a entender as semelhanças e diferenças entre os endpoints de gerenciamento de curtidas do padrão v1.1 e da API do X v2.

- **Semelhanças**
  - Contexto de usuário OAuth 1.0a
- **Diferenças**
  - URLs de endpoint e métodos HTTP
  - Requisitos de App e Projeto
  - Parâmetros de requisição

<div id="similarities">
  #### Semelhanças
</div>

**Método de autenticação OAuth 1.0a User Context**

Ambas as versões do endpoint oferecem suporte a [OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication#oauth-1-0a-2). Portanto, se você já utilizava um dos endpoints padrão da v1.1 para gerenciar favoritos, pode continuar usando o mesmo método de autenticação ao migrar para a API do X v2.

<div id="differences">
  #### Diferenças
</div>

**URLs de endpoint e métodos HTTP**

- Endpoints padrão v1.1:
  - POST https://api.x.com/1.1/favorites/create.json
    (curtir um Post)
  - POST https://api.x.com/1.1/favorites/destroy.json
    (remover curtida de um Post)
- Endpoint da API do X v2:
  - POST https://api.x.com/2/tweets/:id/likes
    (curtir um Post)
  - DELETE https://api.x.com/2/tweets/:id/likes/:tweet\_id
    (remover curtida de um Post) 

**Requisitos de App e Projeto**

Os endpoints da API do X v2 exigem o uso de credenciais de um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) associado a um [Projeto](/pt-BR/resources/fundamentals/projects) ao autenticar suas solicitações. Todos os endpoints da API do X v1.1 podem usar credenciais de Apps independentes ou de Apps associados a um Projeto.

**Parâmetros de solicitação**

Os seguintes parâmetros de solicitação do padrão v1.1 têm equivalentes na API do X v2:

| Padrão v1.1 | API do X v2 |
| :--- | :--- |
| id  | id  |
| includes\_entities | Sem equivalente |

Observe que os parâmetros do padrão v1.1 são enviados como parâmetros de consulta, enquanto os da API do X v2 são enviados como parâmetros no corpo para o endpoint POST ou como parâmetros de caminho para o endpoint DELETE.

Além disso, o id do usuário que curte um Post não é necessário ao usar os endpoints do padrão v1.1, pois os [Access Tokens](/pt-BR/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) passados com [OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication) indicam qual usuário está iniciando a ação de curtir/remover curtida.
