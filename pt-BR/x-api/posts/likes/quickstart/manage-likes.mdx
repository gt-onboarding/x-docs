<div id="getting-started-with-the-manage-likes-endpoints">
  ### Introdução aos endpoints de gerenciamento de Curtidas
</div>

Este guia de início rápido ajudará você a fazer sua primeira solicitação aos endpoints de gerenciamento de Curtidas usando o [Postman](/pt-BR/tutorials/postman-getting-started).

Se desejar ver exemplos de código em diferentes linguagens, visite nosso repositório no GitHub: [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code). 

<div id="prerequisites">
  #### Pré-requisitos
</div>

Para concluir este guia, você precisará ter um conjunto de [chaves e tokens](/pt-BR/resources/fundamentals/authentication), que podem ser gerados seguindo estas etapas:

1. [Solicite uma conta de desenvolvedor](https://developer.x.com/en/apply-for-access) e aguarde a aprovação.

2. Crie um [Projeto](/pt-BR/resources/fundamentals/projects) e um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) associado no portal do desenvolvedor.

3. Acesse a página “Keys and tokens” do seu app e salve suas API Keys, Access Tokens e o Bearer Token no seu gerenciador de senhas.

<div id="steps-to-build-a-manage-likes-request">
  #### Etapas para criar uma solicitação de gerenciamento de curtidas
</div>

**Primeira etapa: comece com uma ferramenta ou biblioteca**

Há várias ferramentas, exemplos de código e bibliotecas que você pode usar para fazer uma solicitação para este endpoint, mas aqui vamos usar o Postman para simplificar o processo.

Para carregar a coleção da API do X v2 no seu ambiente do Postman, clique no botão a seguir:

[![](https://cdn.cms-twdigitalassets.com/content/dam/developer-twitter/m1_vnext/carat.svg)<br />Adicionar API do X v2 ao Postman](https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400)

Depois de carregar a coleção da API do X v2 no Postman, navegue até a pasta “Likes” e selecione “Like a Post”.

**Segunda etapa: autentique sua solicitação**

Para fazer uma solicitação bem-sucedida para este endpoint, você precisará usar [OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication). Para isso, selecione o ambiente chamado “X API v2” no Postman e adicione as seguintes variáveis nos campos de valor inicial e valor atual:

- consumer\_key com sua API Key
- consumer\_secret com sua API Key Secret
- access\_token com seu Access Token
- token\_secret com seu Access Token Secret
   

**Terceira etapa: especifique qual Post você vai curtir**

Os endpoints de gerenciamento de curtidas exigem dois IDs: um do usuário (o usuário que deseja curtir um Post) e outro representando o ID do Post que o usuário está tentando curtir ou deixar de curtir.

O ID do usuário deve corresponder ao ID do usuário autenticado, ou seja, você deve passar os [Access Tokens](/pt-BR/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) associados a esse ID ao autenticar sua solicitação. Neste caso, você pode especificar o ID da sua própria conta. Você pode encontrar seu ID de duas formas:

1. Usando o endpoint [user lookup by username](/pt-BR/x-api/users/user-lookup-by-id), você pode informar um nome de usuário e receber o campo id.
2. Observando seu Access Token, você verá que a parte numérica é o seu ID de usuário.
    

Você também deve especificar um Post que deseja curtir. Você pode encontrar o ID do Post acessando x.com, clicando em um Post e olhando a URL. Por exemplo, o ID do Post na URL a seguir é 1228393702244134912.

https://x.com/TwitterDev/status/1228393702244134912

No Postman, vá até a aba "Params" e insira seu ID na coluna "Value" da variável de caminho id. Vá até a aba “Body” e informe o ID do Post que você deseja curtir como valor do parâmetro tweet\_id. Certifique-se de não incluir espaços antes ou depois de qualquer ID.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | (seu ID de usuário) |
| tweet\_id | (o ID do Post que você quer curtir) |

Ao clicar no botão "Send", você receberá um objeto de resposta contendo o status do relacionamento:

- Se você receber "liked": true, então o id curtiu com sucesso o tweet\_id.
   

**Quarta etapa: faça sua solicitação e revise sua resposta**

Depois que tudo estiver configurado, clique no botão "Send" e você receberá a seguinte resposta:

```
{
    "data": {
        "liked": true
    }
}
```

Se desejar deixar de curtir o mesmo Post, use a solicitação intitulada “Unlike a Post”, que também está na pasta “Likes” da coleção da API do X v2 no Postman. O id e o tweet\_id devem ser usados da mesma forma que no exemplo anterior. Para deixar de curtir um Post, você não precisará enviar um corpo JSON; portanto, certifique-se de adicionar os parâmetros de consulta necessários para id e tweet\_id.
