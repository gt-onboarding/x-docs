<div id="retweets-lookup">
  ### Consulta de Retweets
</div>

Com o endpoint de consulta de Retweets, você pode obter a lista de contas que deram Retweet em um Post. Para este endpoint, serão fornecidos tokens de paginação para navegar por grandes conjuntos de resultados em lotes de até 100 usuários.

Você pode autenticar esses endpoints com [OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication), \[OAuth 2.0 App-Only]\(https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), ou [OAuth 2.0 Authorization Code with PKCE](/pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3). 

<div id="retweets-of-me">
  ### Meus retweets
</div>

A consulta Reposts de mim permite ver quais dos seus próprios Posts foram compartilhados por outras pessoas.

Isso oferece aos usuários uma visão sobre quais de seus Posts estão sendo compartilhados e repercutindo com seu público, o que pode ajudar a promover interações mais direcionadas e ajustes na estratégia de conteúdo.

Você pode autenticar esses endpoints com [OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication) ou [OAuth 2.0 Authorization Code with PKCE](/pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

<div id="manage-retweets">
  #### Gerenciar Retweets
</div>

Os endpoints de gerenciamento de Retweets permitem que você faça Retweet ou desfaça um Retweet de um Post específico em nome de uma conta autenticada. Para este grupo de endpoints, há dois métodos disponíveis: POST e DELETE. O método POST permite fazer Retweet de um Post, e o método DELETE permite desfazer o Retweet de um determinado Post.

Como as solicitações são feitas em nome de um usuário, é necessário autenticar esses endpoints com [OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication) ou [OAuth 2.0 Authorization Code with PKCE](pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) e utilizar os tokens de acesso do usuário associados ao usuário em nome de quem você está fazendo a solicitação. Você pode gerar esse token de acesso do usuário usando o [fluxo OAuth de 3 etapas](/pt-BR/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) ou o [fluxo de concessão Authorization Code com PKCE](/pt-BR/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0). Você pode fazer Retweet de um Post a partir da sua conta ou da conta de um usuário autenticado. Em ambos os endpoints, há um limite por usuário de 50 solicitações por 15 minutos por endpoint.
