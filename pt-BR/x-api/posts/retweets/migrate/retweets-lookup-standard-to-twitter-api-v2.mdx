<div id="retweets-lookup-standard-v11-compared-to-x-api-v2">
  ### Consulta de Retweets: padrão v1.1 em comparação com a API do X v2
</div>

Se você vem trabalhando com o padrão v1.1 [v1.1 GET statuses/retweets/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweets-id), [v1.1 GET statuses/retweets/:ids](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweeters-ids), o objetivo deste guia é ajudar você a entender as semelhanças e diferenças entre o padrão v1.1 e os endpoints de consulta de Retweets da API do X v2.

- **Semelhanças**
  - Autenticação
  - Limites de usuários por requisição
- **Diferenças**
  - URLs de endpoints

  - Limitações de requisição

  - Requisitos de Apps e Projetos

  - Formato dos dados da resposta

  - Parâmetros de requisição

<div id="similarities">
  #### Semelhanças
</div>

**Autenticação**

Tanto os endpoints de busca de retweets do padrão v1.1 quanto da API do X v2 ([v1.1 GET statuses/retweets/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweets-id) e [v1.1 GET statuses/retweeters/:ids](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweeters-ids)) usam [OAuth 1.0a User Context](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication) ou OAuth 2.0 Bearer Token.

**Limites de usuários por solicitação**

Para ambos os endpoints GET v1.1 e v2, o número máximo de usuários retornados pelo endpoint de busca de retweets é 100. No endpoint de busca de retweets v2, não há token de paginação; por padrão, retornamos 100 usuários, que é o limite máximo.

<div id="differences">
  #### Diferenças
</div>

**URLs de endpoint**

- Endpoints padrão v1.1:
  - https://api.x.com/1.1/statuses/retweets/:id.json
    (Retorna uma coleção dos 100 Retweets mais recentes do Post especificado pelo parâmetro `id`)
  - `https://api.x.com/1.1/statuses/retweeters/ids.json`
    (Retorna uma coleção de até 100 IDs de usuários que deram Retweet no Post especificado pelo parâmetro `id`)
- Endpoint da API do X v2:
  - https://api.x.com/2/tweets/:id/retweeted\_by
    (Retorna uma lista de contas que deram Retweet em um determinado Post)

**Requisitos de App e Projeto**

Os endpoints da API do X v2 exigem que você use credenciais de um [developer App](/pt-BR/resources/fundamentals/developer-apps) associado a um [Project](/pt-BR/resources/fundamentals/developer-apps) ao autenticar suas solicitações. Todos os endpoints da API do X v1.1 podem usar credenciais de Apps independentes ou de Apps associados a um projeto.

**Formato de dados da resposta**

Uma das maiores diferenças entre os endpoints padrão v1.1 e os da API do X v2 é como você seleciona quais campos serão retornados no payload.

Nos endpoints padrão, muitos campos de resposta são retornados por padrão e, além disso, você pode usar parâmetros para indicar quais campos ou conjuntos de campos devem ser retornados no payload.

A versão da API do X v2 retorna apenas os campos user id, name e username por padrão. Para solicitar campos ou objetos adicionais, você precisará usar os parâmetros [fields](/pt-BR/x-api/fundamentals/fields) e [expansions](/pt-BR/x-api/fundamentals/expansions). Quaisquer campos de usuário solicitados a partir desse endpoint serão retornados no objeto de usuário primário. Qualquer objeto de Post expandido e seus campos serão retornados em um objeto includes dentro da resposta. Em seguida, você pode relacionar quaisquer objetos expandidos ao objeto de usuário fazendo a correspondência dos IDs presentes tanto no usuário quanto no objeto de Post expandido.

Recomendamos que você leia mais sobre esses novos parâmetros em seus respectivos guias ou no nosso guia sobre [como usar fields e expansions](/pt-BR/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions).

Também preparamos um [guia de migração de formato de dados](/pt-BR/x-api/migrate/data-format-migration#migrating-from-standard-v1-1s-data-format-to-v2) que pode ajudar você a mapear os campos padrão v1.1 para os novos campos v2. Esse guia também informará o parâmetro específico de expansão e de campo que você precisará incluir na sua solicitação v2 para retornar campos específicos.

Além das mudanças em como você solicita determinados campos, a API do X v2 também está introduzindo novos modelos JSON para os objetos retornados pelas APIs, incluindo os objetos de [Post](/pt-BR/x-api/fundamentals/data-dictionary#tweet) e de [user](/pt-BR/x-api/fundamentals/data-dictionary#user).

- No nível raiz do JSON, os endpoints padrão retornam objetos de Post em um array statuses, enquanto a API do X v2 retorna um array data.
- Em vez de se referir a “statuses” Retweetados e Citados, o JSON da API do X v2 se refere a Tweets Retweetados e Citados. Muitos campos legados e obsoletos, como contributors e user.translator\_type, estão sendo removidos.
- Em vez de usar tanto favorites (no objeto de Post) quanto favourites (no objeto de usuário), a API do X v2 usa o termo like.
- O X está adotando a convenção de que valores JSON sem valor (por exemplo, null) não são escritos no payload. Atributos de Post e de usuário só são incluídos se tiverem valores não nulos.

Também introduzimos um novo conjunto de campos no [objeto de Post](/pt-BR/x-api/fundamentals/data-dictionary#tweet), incluindo:

- Um campo [conversation\_id](/pt-BR/x-api/fundamentals/conversation-id)
- Dois novos campos de [annotations](/pt-BR/x-api/fundamentals/post-annotations), incluindo context e entities
- Vários novos campos de [metrics](/pt-BR/x-api/fundamentals/metrics)
- Um novo campo reply\_setting, que mostra quem pode responder a um determinado Post

**Parâmetros de solicitação**

Os seguintes parâmetros de solicitação padrão v1.1 aceitam dois parâmetros de consulta (user\_id ou screen\_name). A API do X v2 aceita apenas o ID numérico do usuário, que deve ser passado como parte do caminho do endpoint.
