<div id="manage-retweets-standard-v11-compared-to-x-api-v2">
  ### Gerenciar Retweets: Standard v1.1 em comparação com a API do X v2
</div>

Se você vem trabalhando com os endpoints do standard v1.1 [POST statuses/retweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-retweet-id) e [POST statuses/unretweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-unretweet-id), o objetivo deste guia é ajudar você a entender as semelhanças e diferenças entre os endpoints de Retweets do standard e da API do X v2.

- **Semelhanças**
  - Autenticação
- **Diferenças**
  - URLs de endpoint e métodos HTTP
  - Limitações de requisições
  - Requisitos de App e Projeto
  - Parâmetros de requisição

<div id="similarities">
  #### Semelhanças
</div>

**Autenticação**

Tanto os endpoints de gerenciamento de Retweets do padrão v1.1 ([POST statuses/retweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-retweet-id) e [POST statuses/unretweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-unretweet-id)) quanto os da API do X v2 usam [OAuth 1.0a no contexto do usuário](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Portanto, se você já usava um dos endpoints de consulta de Retweets do padrão v1.1, pode continuar usando o mesmo método de autenticação ao migrar para a versão da API do X v2. 

<div id="differences">
  #### Diferenças
</div>

**URLs de endpoint e métodos HTTP**

- Endpoints Standard v1.1:
  - https://api.x.com/1.1/statuses/retweet/:id.json
    (Faz retweet de um Post. Retorna o Post original com detalhes do retweet incorporados)
  - https://api.x.com/1.1/statuses/unretweet/:id.json
    (Desfaz um retweet. Retorna o Post original com detalhes do retweet incorporados)
- Endpoint da API do X v2:
  - https://api.x.com/2/tweets/:id/retweets
    (Faz retweet de um Post específico)
  - https://api.x.com/2/users/:id/retweets/:source\_tweet\_id
    (Desfaz um retweet de um Post específico) 

**Requisitos de App e Projeto**

Os endpoints da API do X v2 exigem o uso de credenciais de um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) associado a um [Projeto](/pt-BR/resources/fundamentals/projects) ao autenticar suas solicitações. Todos os endpoints da API do X v1.1 podem usar credenciais de Apps independentes ou de Apps associados a um Projeto.

**Parâmetros de solicitação**

Os seguintes endpoints Standard v1.1 aceitavam dois parâmetros de consulta (user\_id ou screen\_name). A API do X v2 aceita apenas o ID numérico do usuário, que deve ser passado como parte do caminho do endpoint.

| Standard v1.1 | API do X v2 |
| :--- | :--- |
| **id** | **id** |
| **includes\_entities** | Não há equivalente |

Observe que, no Standard v1.1, os parâmetros são passados na string de consulta (query), enquanto na API do X v2 os parâmetros são passados no corpo para o endpoint POST ou no caminho para o endpoint DELETE.

Além disso, não é necessário informar o ID do usuário que está fazendo retweet de um Post ao usar os endpoints Standard v1.1, pois os [tokens de acesso](/pt-BR/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) enviados com o [Contexto de Usuário do OAuth 1.0a](/pt-BR/resources/fundamentals/authentication) indicam qual usuário está iniciando ou desfazendo o retweet.
