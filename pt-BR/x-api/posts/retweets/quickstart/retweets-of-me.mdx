A consulta Reposts of Me permite ver quais dos seus próprios Posts foram compartilhados por outras pessoas.

Isso oferece aos usuários uma visão sobre quais de seus Posts estão sendo compartilhados e tendo repercussão junto ao seu público, o que pode ajudar a promover interações mais direcionadas e ajustes na estratégia de conteúdo.

<div id="authentication">
  #### Autenticação
</div>

Como você está fazendo solicitações em nome de um usuário, é necessário autenticar esses endpoints com [OAuth 1.0a (User Context)](/pt-BR/resources/fundamentals/authentication) ou [OAuth 2.0 (Authorization Code com PKCE)](/pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) e usar os tokens de acesso do usuário associados ao usuário em cujo nome você está fazendo a solicitação. Você pode gerar esse token de acesso do usuário usando o [fluxo OAuth de 3 etapas](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens) (OAuth 1.0a) ou o [fluxo de concessão Authorization Code com PKCE](/pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) (OAuth 2.0).

<div id="making-a-request">
  #### Fazendo uma solicitação
</div>

Depois que um usuário autenticar no seu app, você pode chamar a API Retweets of Me em nome dele, conforme mostrado abaixo:

```bash
      curl 'https://api.x.com/2/users/reposts_of_me' --header 'Authorization: ••••••'
```
