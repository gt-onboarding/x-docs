<div id="authentication">
  #### Autenticação
</div>

Todos os endpoints da API do X v2 exigem que você autentique suas solicitações com um conjunto de credenciais, também conhecidas como chaves e tokens.

Você pode usar OAuth 1.0a User Context ou OAuth 2.0 Bearer Token para autenticar suas solicitações ao endpoint de consulta de Retweets.

Os endpoints de gerenciamento de Retweets exigem o uso de OAuth 1.0a User Context, o que significa que você deve usar um conjunto de chaves de API e tokens de acesso do usuário para fazer uma solicitação bem-sucedida. Os tokens de acesso devem estar associados ao usuário em cujo nome você está fazendo a solicitação. Se você deseja gerar um conjunto de tokens de acesso para outro usuário, ele deve autorizar ou autenticar seu App usando o fluxo OAuth de 3 etapas.

Observe que o OAuth 1.0a pode ser complexo de usar. Se você não estiver familiarizado com esse método de autenticação, recomendamos usar uma [biblioteca](/pt-BR/x-api/tools-and-libraries/overview) para autenticar suas solicitações corretamente.

**Observação**

Se você estiver solicitando os seguintes campos, o OAuth 1.0a User Context é obrigatório:

- tweet.fields.non\_public\_metrics
- tweet.fields.promoted\_metrics
- tweet.fields.organic\_metrics
- media.fields.non\_public\_metrics
- media.fields.promoted\_metrics
- media.fields.organic\_metrics

<div id="developer-portal-projects-and-developer-apps">
  #### Portal do desenvolvedor, Projetos e Apps do desenvolvedor
</div>

Para obter um conjunto de credenciais de autenticação que funcione com os endpoints da API do X v2, você deve [criar uma conta de desenvolvedor](https://developer.x.com/en/portal/petition/essential/basic-info), configurar um [Projeto](/pt-BR/resources/fundamentals/projects) nessa conta e criar um [App do desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) nesse Projeto. Em seguida, você poderá encontrar suas chaves e tokens no seu App do desenvolvedor. 
 

<div id="rate-limits">
  #### Limites de taxa
</div>

Todos os dias, milhares de desenvolvedores fazem solicitações à API do X. Para ajudar a gerenciar o grande volume dessas solicitações, [limites de taxa](/pt-BR/x-api/fundamentals/rate-limits) são aplicados a cada endpoint, restringindo o número de solicitações que você pode fazer em nome do seu app ou de um usuário autenticado.

Os endpoints de gerenciamento de Retweets são limitados a 50 solicitações por janela de 15 minutos (por usuário). Além disso, para o endpoint POST, o limite é de 300 solicitações por janela de 3 horas (por usuário, por app).

No endpoint de consulta de Retweets, o limite é de 75 solicitações por janela de 15 minutos. Além disso, apenas os 100 usuários que retuitaram mais recentemente serão retornados por esse endpoint.

<div id="fields-and-expansions">
  #### Campos e expansões
</div>

A API do X v2 permite que os usuários selecionem exatamente quais dados desejam receber da API usando um conjunto de recursos chamados campos e expansões. O parâmetro de expansão permite expandir objetos referenciados no payload. Por exemplo, este endpoint permite obter as seguintes [expansões](/pt-BR/x-api/fundamentals/expansions):

- attachments.poll\_ids
- attachments.media\_keys
- author\_id, entities.mentions.username
- geo.place\_id
- in\_reply\_to\_user\_id,
- referenced\_tweets.id,
- referenced\_tweets.id.author\_id

O parâmetro de campos permite selecionar exatamente quais [campos](/pt-BR/x-api/fundamentals/fields) dentro dos diferentes objetos de dados você deseja receber. Esses endpoints retornam principalmente objetos Post. Por padrão, o objeto Post retorna os campos id e text. Para receber campos adicionais, como tweet.created\_at ou tweet.entities, você precisará solicitá-los explicitamente usando um parâmetro de campos. Alguns campos importantes que você pode considerar usar na sua integração são dados de enquete, métricas, anotações de Post e o ID de conversa.

Adicionamos um guia sobre como [usar campos e expansões](/pt-BR/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) em conjunto ao nosso [dicionário de dados da API do X v2](/pt-BR/x-api/fundamentals/data-dictionary).
