O grupo de endpoints do fluxo filtrado permite que desenvolvedores filtrem um fluxo de Posts públicos. A funcionalidade desse grupo inclui vários endpoints que permitem criar e gerenciar regras e aplicá-las para filtrar um fluxo de Posts que retornará Posts públicos correspondentes. Esse grupo de endpoints permite que os usuários monitorem tópicos e eventos específicos, acompanhem a conversa sobre competições, entendam como as tendências se desenvolvem e muito mais.

Os desenvolvedores podem usar o [endpoint de regras](/pt-BR/x-api/posts/filtered-stream#post-2-tweets-search-stream-rules) REST para adicionar e remover regras em uma conexão de fluxo persistente sem precisar desconectar. Essas [regras](/pt-BR/x-api/posts/filtered-stream#building-rules-for-filtered-stream) podem ser criadas com operadores que fazem correspondência com atributos do Post, como palavras-chave da mensagem, hashtags e URLs. Operadores e cláusulas de regra podem ser combinados com lógica booleana e parênteses para refinar o comportamento de correspondência do filtro.

Depois de adicionar um conjunto de regras, você pode [estabelecer uma conexão de streaming](/pt-BR/x-api/posts/filtered-stream#get-2-tweets-search-stream) que começará a entregar [objetos Post](/pt-BR/x-api/fundamentals/data-dictionary#tweet) em formato JSON por meio de uma conexão HTTP streaming persistente. Você só receberá conteúdo que corresponda às suas regras enquanto estiver conectado ao fluxo.

Como alternativa, você pode vincular o Fluxo filtrado a um webhook para entrega como eventos de webhook. Para mais detalhes sobre o uso de webhooks com o Fluxo filtrado, consulte a [introdução a Webhooks do Fluxo filtrado](/pt-BR/x-api/webhooks/stream/introduction).

O endpoint de busca filtrada oferece suporte a Posts editados. Esse endpoint entregará Posts editados que correspondam a um ou mais dos seus filtros, juntamente com seu histórico de edições, incluindo um array de IDs de Post. Para Posts sem histórico de edições, esse array conterá um único ID. Para Posts que foram editados, esse array contém vários IDs, em ordem ascendente refletindo a sequência das edições, com a versão mais recente na última posição do array. Para saber mais sobre como as edições de Posts funcionam, consulte a página de [fundamentos de edições de Posts](/pt-BR/x-api/fundamentals/edit-posts).

Certos aspectos do endpoint do fluxo filtrado são limitados pelo [nível de acesso](/pt-BR/x-api/getting-started/about-x-api):

**Acesso Pro**

- 1000 regras por projeto
- 100 solicitações por 15 minutos ao usar o endpoint POST /2/tweets/search/stream/rules para adicionar regras
- Pode usar todos os operadores ao criar sua regra
- Pode criar regras com até 1024 caracteres

**Acesso Enterprise**

- 25.000+ regras por projeto
- Pode usar todos os operadores ao criar sua regra
- Pode criar regras com até 2048 caracteres
- Solicite [aqui](https://developer.x.com/en/products/x-api/enterprise/enterprise-api-interest-form) o acesso Enterprise

Os Posts retornados do fluxo filtrado contam para o [limite de posts](/pt-BR/x-api/fundamentals/post-cap) mensal.

  - Autenticar-se usando as chaves e tokens de um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) que esteja localizado dentro de um [Projeto](/pt-BR/resources/fundamentals/projects).

  Saiba mais sobre como obter acesso aos endpoints da API do X v2 em nosso [guia de primeiros passos](/pt-BR/x-api/getting-started/getting-access).
