<div id="authentication">
  ### Autenticação
</div>

Todos os endpoints da API do X v2 exigem solicitações autenticadas. Você pode se autenticar com:

- [OAuth 1.0a no contexto do usuário](/pt-BR/resources/fundamentals/authentication), usando chaves de API, tokens de acesso e parâmetros adicionais para [criar um cabeçalho de autorização](/pt-BR/resources/fundamentals/authentication#authorizing-a-request).
- [OAuth 2.0 somente app](/pt-BR/resources/fundamentals/authentication#oauth-2-0), passando um [token de acesso de app](/pt-BR/resources/fundamentals/authentication) na sua solicitação.
- [OAuth 2.0 com código de autorização e PKCE](/pt-BR/resources/fundamentals/authentication), para maior controle sobre o escopo do app e autorização em vários dispositivos.

<Note>
  Implementar OAuth 1.0a pode ser desafiador. Se não estiver familiarizado, considere usar uma [biblioteca](/pt-BR/resources/tools-and-libraries) ou OAuth 2.0 para suas solicitações. Para métricas privadas ou posts, use OAuth 1.0a no contexto do usuário ou OAuth 2.0 com código de autorização e PKCE.
</Note>

<Tip>
  **Atenção**

  > Se você estiver solicitando os seguintes campos, é necessário usar OAuth 1.0a no contexto do usuário ou OAuth 2.0 com código de autorização:
  >
  > - `tweet.fields.non_public_metrics`
  > - `tweet.fields.promoted_metrics`
  > - `tweet.fields.organic_metrics`
  > - `media.fields.non_public_metrics`
  > - `media.fields.promoted_metrics`
  > - `media.fields.organic_metrics`
</Tip>

<div id="developer-portal-projects-and-apps">
  ### Portal do Desenvolvedor, Projetos e Apps
</div>

Para obter credenciais da API do X v2, você precisa de:

1. Uma [conta de desenvolvedor](/pt-BR/resources/fundamentals/developer-portal) aprovada.
2. Um [Projeto](/pt-BR/resources/fundamentals/projects) na conta de desenvolvedor.
3. Um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) nesse Projeto, onde estão as chaves e os tokens.

<div id="rate-limits">
  ### Limites de taxa
</div>

As solicitações à API do X estão sujeitas a [limites de taxa](/pt-BR/resources/fundamentals/rate-limits) para gerenciar o volume. Os limites se aplicam tanto no nível do App quanto no nível do usuário:

- **Nível do App**: Limita o número de solicitações feitas por período por qualquer app.
- **Nível do usuário**: Limita a frequência com que um usuário autenticado pode realizar consultas de Post em Apps de desenvolvedor.

<div id="fields-and-expansions">
  ### Campos e expansões
</div>

A API do X v2 permite selecionar campos de dados específicos usando `fields` e `expansions`:

- **Expansões**: Permitem recuperar objetos relacionados adicionais. As expansões compatíveis incluem:
  - `edit_history_tweet_ids`
  - `attachments.poll_ids`
  - `attachments.media_keys`
  - `author_id`
  - `entities.mentions.username`
  - `geo.place_id`
  - `in_reply_to_user_id`
  - `referenced_tweets.id`
  - `referenced_tweets.id.author_id`

- **Campos**: Especificam os campos de dados dentro dos objetos para retornar informações adicionais. O objeto Post inclui por padrão `id`, `text` e `edit_history_tweet_ids`. Outras opções, como `tweet.created_at` e `tweet.entities`, devem ser solicitadas explicitamente.

Para mais detalhes, consulte o guia de campos e expansões no [dicionário de dados da API do X v2](/pt-BR/x-api/fundamentals/data-dictionary).

<div id="post-edits">
  ### Edições de posts
</div>

Posts elegíveis podem ser editados até cinco vezes nos 30 minutos após a publicação. O endpoint de consulta de posts sempre retorna a versão mais recente do post. Para casos de uso quase em tempo real, leve em conta essa janela de tempo. Para mais detalhes, consulte [Fundamentos de edição de posts](/pt-BR/x-api/fundamentals/edit-posts).

<div id="edge-cases">
  ### Casos de borda
</div>

- **Métricas promovidas**: Solicitar métricas promovidas para Posts não promovidos retorna uma resposta vazia.
- **Texto truncado**: O texto do Post é truncado em Retweets. Para obter o texto completo, expanda o Post referenciado
