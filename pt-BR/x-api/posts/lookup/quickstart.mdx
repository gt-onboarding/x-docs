<div id="getting-started-with-the-posts-lookup-endpoints">
  ## Introdução aos endpoints de consulta de Posts
</div>

Este guia de início rápido ajudará você a fazer sua primeira requisição aos endpoints de consulta de Posts com um conjunto de campos especificados usando o Postman.

Se quiser ver exemplos de código em diferentes linguagens, visite nosso repositório no GitHub [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code).

<Note>
  ### Pré-requisitos

  Para concluir este guia, você precisará de um conjunto de [chaves e tokens](/pt-BR/resources/fundamentals/authentication) para autenticar sua requisição. Você pode gerar essas chaves e tokens seguindo estas etapas:

  - [Inscreva-se em uma conta de desenvolvedor](https://developer.x.com/en/portal/petition/essential/basic-info) e aguarde a aprovação.
  - Crie um [Projeto](/pt-BR/resources/fundamentals/projects) e um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) associado no portal do desenvolvedor.
  - Acesse a página “Keys and tokens” do seu App para gerar as credenciais necessárias. Certifique-se de salvar todas as credenciais em um local seguro.
</Note>

<div id="steps-to-build-a-get-tweets-request">
  ### Etapas para construir uma solicitação GET /tweets
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### Primeiro passo: comece com uma ferramenta ou biblioteca
</div>

Há várias ferramentas, exemplos de código e bibliotecas que você pode usar para fazer uma requisição a este endpoint, mas aqui vamos usar o Postman para simplificar o processo.

Para carregar a coleção da API do X v2 no seu ambiente do Postman, clique no botão abaixo:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Adicionar API do X v2 ao Postman
</Button>

Depois que a coleção da API do X v2 estiver carregada no Postman, navegue até a requisição "Post Lookup > Multiple Posts".

<div id="step-two-authenticate-your-request">
  #### Etapa dois: Autentique sua solicitação
</div>

Para fazer uma solicitação corretamente à API do X, você precisa confirmar que tem permissão. Você pode autenticar sua solicitação com:

- [OAuth 2.0 App-Only](pt-BR/resources/fundamentals/authentication)
- [OAuth 2.0 Authorization Code com PKCE](pt-BR/resources/fundamentals/authentication)
- [OAuth 1.0a User Context](pt-BR/resources/fundamentals/authentication)

Para simplificar, vamos utilizar OAuth 2.0 App-Only nesta solicitação, mas, se você quiser acessar [métricas](/pt-BR/x-api/fundamentals/metrics) privadas ou Posts, será necessário usar um dos outros métodos de autenticação.

Para utilizar OAuth 2.0 App-Only, adicione suas chaves e tokens (especificamente o App Access Token, também conhecido como App-only Bearer Token) ao Postman selecionando o ambiente chamado “API do X v2” (no canto superior direito do Postman) e inserindo suas chaves e tokens nos campos “initial value” e “current value” (clicando no ícone de olho ao lado do menu suspenso de ambientes).

Se você fez isso corretamente, essas variáveis serão preenchidas automaticamente na guia de autorização da solicitação.

<div id="step-three-identify-and-specify-which-posts-you-would-like-to-retrieve">
  #### Etapa três: Identifique e especifique quais Posts você deseja recuperar
</div>

Você deve informar um Post ou um conjunto de Posts que deseja receber na solicitação. Você pode encontrar o ID do Post acessando X.com, clicando em um Post e, em seguida, verificando a URL. Por exemplo, o ID do Post na seguinte URL é `1228393702244134912`.

`https://x.com/TwitterDev/status/1228393702244134912`

No Postman, vá até a guia "Params" e insira esse ID, ou uma lista de IDs de Post separados por vírgula, na coluna "Value" do parâmetro ids.

| Key  | Value |
| :---- | :----- |
| `ids` | `1228393702244134912,1227640996038684673,1199786642791452673` |

---

<div id="step-four-identify-and-specify-which-fields-you-would-like-to-retrieve">
  #### Etapa quatro: identifique e especifique quais campos você deseja recuperar
</div>

Se você clicar no botão "Send" após a etapa três, receberá os [campos padrão do objeto Post](/pt-BR/x-api/fundamentals/data-dictionary) na sua resposta: `id`, `text` e `edit_history_tweet_ids`.

Para este exercício, solicitaremos três conjuntos adicionais de campos de objetos diferentes:

- O campo adicional `tweet.created_at` nos objetos de post.
- Os campos padrão do objeto de usuário dos autores associados para os Posts retornados: `id`, `name` e `username`
- O campo adicional `user.created_at` nos objetos de usuário associados.

No Postman, vá até a aba "Params" e adicione o seguinte par `key:value` na tabela "Query Params":
| Key            | Value               | Returned fields                          |
| :-------------- | :------------------- | :---------------------------------------- |
| `tweet.fields` | `created_at`        | `tweets.created_at`                      |
| `expansions`   | `author_id`         | `includes.users.id`, `includes.users.name`, `includes.users.username` |
| `user.fields`  | `created_at`        | `includes.users.created_at`              |

Você deverá ver esta URL ao lado do botão "Send":

```
https://api.x.com/2/tweets?ids=1228393702244134912,1227640996038684673,1199786642791452673&tweet.fields=created_at&expansions=author_id&user.fields=created_at
```
