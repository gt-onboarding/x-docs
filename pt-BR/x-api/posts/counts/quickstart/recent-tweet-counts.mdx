<div id="getting-started-with-the-recent-post-counts-endpoint">
  ### Introdução ao endpoint de contagem de posts recentes
</div>

Este guia de início rápido ajudará você a fazer sua primeira solicitação ao endpoint de contagem de posts recentes usando o Postman, uma ferramenta gráfica que permite enviar solicitações HTTP.

Se você quiser ver código de exemplo em diferentes linguagens de programação, visite nosso repositório no GitHub de [códigos de exemplo da API do X v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code).

<Note>
  ### Pré-requisitos

  Para concluir este guia, você precisará de um conjunto de [keys and tokens](/pt-BR/resources/fundamentals/authentication) para autenticar sua solicitação. Você pode gerar essas keys and tokens seguindo estas etapas:

  - [Inscreva-se para uma conta de desenvolvedor](https://developer.x.com/en/apply-for-access) e aguarde a aprovação.
  - Crie um [Project](/pt-BR/resources/fundamentals/projects) e um [developer App](/pt-BR/resources/fundamentals/developer-apps) associado no portal de desenvolvedor.
  - Acesse a página “Keys and tokens” do seu App para gerar as credenciais necessárias. Certifique-se de salvar todas as credenciais em um local seguro.
</Note>

<div id="steps-to-build-a-recent-post-counts-request">
  #### Etapas para criar uma solicitação de contagem de Posts recentes
</div>

**Etapa um: Comece com uma ferramenta ou biblioteca**

Há diversas ferramentas, exemplos de código e bibliotecas que você pode usar para fazer uma solicitação a este endpoint, mas aqui vamos utilizar o Postman para simplificar o processo.

Para carregar a coleção da API do X v2 no seu ambiente do Postman, clique no botão a seguir:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Adicionar API do X v2 ao Postman
</Button>

Depois de carregar a coleção da API do X v2 no Postman, navegue até Post counts > Recent Post counts request.

**Etapa dois: Autentique sua solicitação**

Para fazer corretamente uma solicitação à API do X, você precisa confirmar que tem permissão. Para isso, neste endpoint, autentique sua solicitação com os métodos de autenticação [OAuth 2.0 App-Only](/pt-BR/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token).

Você deve adicionar suas chaves e tokens, especificamente o [App Access Token](/pt-BR/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token) (também conhecido como App-only Bearer Token) ao Postman. Faça isso selecionando o ambiente chamado “API do X v2” no canto superior direito do Postman e adicionando suas chaves e tokens nos campos “initial value” e “current value” (clicando no ícone de olho ao lado do menu suspenso de ambientes).

Essa variável será automaticamente aplicada na guia de autorização da solicitação se você tiver feito isso corretamente.
 

**Etapa três: Crie uma query**

Cada solicitação de contagem de Posts recentes requer uma única [query](/pt-BR/x-api/posts/counts/integrate/build-a-query). Para este exemplo, vamos usar uma query que corresponda a Posts publicados pela conta @XDevelopers. Para essa query, usamos o operador from: e definimos como XDevelopers (não diferencia maiúsculas de minúsculas):

`from:XDevelopers`

No Postman, navegue até a guia "Params" e insira este ID, ou uma sequência de IDs de Post separados por vírgula, na coluna "Value" do parâmetro `ids`.

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Description** |
| `query` | from:XDevelopers | Query a ser enviada para o endpoint de contagem de Posts recentes |

**Etapa quatro (opcional): Especifique a granularidade da solicitação**

Se você clicar no botão "Send" após a etapa três, obterá a contagem de Posts recentes padrão: por hora nos últimos sete dias. Se quiser obter a contagem de Posts recentes por dia, será necessário adicionar o parâmetro granularity com o valor day.

No Postman, navegue até a guia "Params" e digite day na coluna "Value" do parâmetro granularity.

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Description** |
| granularity | day | A granularidade dos resultados de contagem de Posts. Valores possíveis: day, hour ou minute |

Agora você deverá ver a seguinte URL ao lado do botão "Send":

`https://api.x.com/2/tweets/counts/recent?query=from%3AXDevelopers&granularity=day`
