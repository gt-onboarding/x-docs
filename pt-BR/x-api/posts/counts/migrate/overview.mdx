<div id="comparing-x-apis-post-counts-endpoints">
  ## Comparando os endpoints de contagem de posts da API do X
</div>

O endpoint de contagem de posts da v2 substituirá, eventualmente, o [endpoint de contagem da Search API para empresas](/pt-BR/x-api/enterprise-gnip-2.0/fundamentals/search-api#counts-requests-post-count). Se você tem código, apps ou ferramentas que usam uma versão mais antiga de um endpoint de contagem de posts e está considerando migrar para os endpoints mais novos da API do X v2, este guia é para você.

Esta página contém tabelas de comparação:

- [Contagem de posts recentes](#recent)
- [Contagem de posts em todo o arquivo](#full-archive)
- [Comparação de operadores de filtragem](#operator)

<div id="recent-post-counts-comparison">
  ### Comparação de contagens de Posts recentes
</div>

A versão Enterprise dos endpoints de contagem de Posts permite obter contagens para 30 dias ou de toda a busca em todo o arquivo. Portanto, o endpoint de contagem de Posts recentes da v2, que considera um período de 7 dias, não é um substituto direto para nenhum dos endpoints mencionados.

No entanto, para ajudar nas comparações, veremos como o endpoint de contagem de Posts recentes da v2 se compara ao endpoint Enterprise de 30 dias.

A tabela a seguir compara os vários tipos de endpoints de contagem de Posts recentes:

| **Descrição** | **Enterprise** | **API do X v2** |
| :--- | :--- | :--- |
| Domínio do host | https://gnip-api.x.com | https://api.x.com |
| Caminho do endpoint | /search/30day/accounts/:account\_name/:label/counts.json | /2/tweets/counts/recent |
| [Autenticação](/pt-BR/resources/fundamentals/authentication) | Autenticação básica | Token Bearer do OAuth 2.0 |
| Formato de timestamp | YYYYMMDDhhmm | YYYY-MM-DDTHH:mm:ssZ  <br />[ISO 8601 / RFC 3339](https://tools.ietf.org/html/rfc3339#section-5.6) |
| Retorna contagens de Posts com idade máxima de | 31 dias | 7 dias |
| Métodos HTTP compatíveis | GET | GET |
| Limites padrão de taxa de requisições | 20 requisições por 1 s, agregadas entre dados de busca e requisições de contagem  <br />O limite por minuto varia por parceiro, conforme especificado no seu contrato. | 180 requisições por 15 min por usuário  <br />450 requisições por 15 min por App |
| Suporta filtragem usando [annotations](/pt-BR/x-api/fundamentals/post-annotations) |     | ✔   |
| Suporta filtragem usando [conversation\_id](/pt-BR/x-api/fundamentals/conversation-id) |     | ✔   |
| Nome da chave JSON para o array de dados de Post | results | data |
| Granularidade de tempo | Dia, hora ou minuto | Dia, hora ou minuto |
| Fuso horário | UTC | UTC |
| Parâmetros de requisição para selecionar o período | fromDate  <br />toDate | start\_time  <br />end\_time |
| Parâmetros de requisição para navegação por ID de Post |     | since\_id  <br />until\_id |
| Requer o uso de credenciais de um [developer App](/pt-BR/resources/fundamentals/developer-apps) associado a um [project](/pt-BR/resources/fundamentals/projects) |     | ✔   |

<div id="filtering-operator-comparison">
  ### Comparação de operadores de filtragem
</div>

As duas versões (enterprise e v2) de Contagens de posts diferem nos operadores disponíveis e também têm níveis distintos de disponibilidade de operadores dentro de cada versão, conforme explicado abaixo.

Enterprise

- Não há subníveis de operadores enterprise. Todos os operadores enterprise estão disponíveis para todos os usuários enterprise.

API do X v2

- **Principal:** Esses operadores estão disponíveis para qualquer usuário v2.
- **Avançado:** Esses operadores estão disponíveis apenas para usuários aprovados para acesso de Pesquisa Acadêmica.

Você pode saber mais sobre cada um desses conjuntos de operadores em seus respectivos guias:

- [Operadores enterprise](/pt-BR/x-api/enterprise-gnip-2.0/fundamentals/search-api#building-search-queries)
- [Operadores da API do X v2](/pt-BR/x-api/posts/search/integrate/build-a-query)
