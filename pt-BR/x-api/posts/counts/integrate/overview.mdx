</div>

Para saber mais sobre nossas coleções do Postman, visite a página [Usando o Postman](/pt-BR/tutorials/postman-getting-started).
 

<div id="key-concepts">
  ### Conceitos-chave
</div>

<div id="authentication">
  #### Autenticação
</div>

Todos os endpoints da API do X v2 exigem que as solicitações sejam [autenticadas](/pt-BR/resources/fundamentals/authentication) com um conjunto de credenciais, também conhecido como chaves e tokens. Este endpoint específico requer o uso de [OAuth 2.0 Bearer Token](/pt-BR/resources/fundamentals/authentication#oauth-2-0), o que significa que você deve enviar um [Bearer Token](/pt-BR/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) para realizar uma solicitação com sucesso. Você pode gerar um Bearer Token diretamente em um App de desenvolvedor ou gerá-lo usando o endpoint [POST oauth2/token](/pt-BR/resources/fundamentals/authentication#post-oauth2-token).

<div id="developer-portal-projects-and-developer-apps">
  #### Portal do desenvolvedor, Projetos e Apps do desenvolvedor
</div>

Para trabalhar com qualquer endpoint da API do X v2, você deve ter uma [conta de desenvolvedor](/pt-BR/resources/fundamentals/developer-portal), configurar um [Projeto](/pt-BR/resources/fundamentals/projects) nessa conta e criar um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) dentro desse Projeto. As chaves e tokens desse App de desenvolvedor funcionarão para os endpoints de contagem de posts recentes. Se você quiser usar o endpoint de contagem de posts com busca em todo o arquivo ou utilizar operadores avançados e consultas mais longas, será necessário ter aprovação para acesso enterprise.

Visite nossa seção sobre acesso enterprise para saber mais.

<div id="rate-limits">
  #### Limites de taxa
</div>

Todos os dias, milhares de desenvolvedores fazem solicitações à API do X. Para ajudar a gerenciar o volume, [limites de taxa](/pt-BR/x-api/fundamentals/rate-limits) são aplicados a cada endpoint, restringindo o número de solicitações que cada desenvolvedor pode fazer em nome de um app ou de um usuário autenticado.

Este endpoint é limitado no nível do App, o que significa que você, desenvolvedor, só pode fazer um certo número de solicitações a este endpoint dentro de um período específico, a partir de qualquer App (conforme inferido pelas credenciais que estiver usando). 

<div id="building-queries">
  #### Criando consultas
</div>

O principal recurso desses endpoints é o uso de uma única consulta para filtrar os Posts nos totais que são entregues a você. Essas consultas são compostas por operadores que fazem correspondência com atributos de Post e de usuário, como palavras-chave, hashtags e URLs. Os operadores podem ser combinados em consultas com lógica booleana e parênteses para ajudar a refinar o comportamento de correspondência da consulta.

Você pode usar nosso guia sobre [como criar uma consulta](/pt-BR/x-api/posts/counts/integrate/build-a-query) para saber mais.

<div id="pagination">
  #### Paginação
</div>

Para contagens recentes de Posts, não há `next_token` retornado, o que significa que, independentemente da Granularidade, você receberá o volume de Posts dos últimos 7 dias em uma única chamada à API.

Para contagens de Posts da busca em todo o arquivo, você receberá dados dos últimos 30 dias. Para dados com mais de 30 dias, você receberá um `next_token`, que poderá usar para paginar e obter os dados adicionais. 

<div id="building-queries-for-post-counts">
  ### Criando consultas para contagem de Posts
</div>

Os endpoints de contagem de Posts aceitam uma única consulta via solicitação GET e retornam um conjunto de contagens históricas de Posts que atendem à consulta. As consultas são compostas por operadores usados para corresponder a uma variedade de atributos de Posts.
