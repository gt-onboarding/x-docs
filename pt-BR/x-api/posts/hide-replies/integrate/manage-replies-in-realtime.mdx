<div id="manage-replies-in-realtime">
  ### Gerencie respostas em tempo real
</div>

Com o endpoint de ocultar respostas, você pode criar um fluxo de trabalho que ajuda seus usuários a ocultar respostas com probabilidade muito alta de serem irrelevantes.

Apps úteis costumam combinar tecnologias para oferecer valor aos seus usuários. Esta página mostra como ocultar respostas usando a [Perspective API](https://www.perspectiveapi.com/). Essa API é um sistema de inteligência artificial treinado pela [Jigsaw](https://jigsaw.google.com/), uma unidade do Google, para detectar comentários tóxicos. A lógica da aplicação funcionará da seguinte forma:

1.  Solicita a permissão do usuário para ler seus posts e ocultar ou reexibir suas respostas.
2.  Usa a [Account Activity API](https://developer.x.com/en/docs/x-api/v1/accounts-and-users/subscribe-account-activity/overview) para detectar respostas recebidas.
3.  Solicita à Perspective API uma “pontuação” (um número entre 0 e 1) que indica o nível de confiança do algoritmo de que um comentário é semelhante a comentários tóxicos vistos anteriormente. (A Perspective não armazena o texto enviado ao serviço.)
4.  Chama o ocultar respostas se a pontuação do algoritmo for muito alta.

<div id="strive-for-transparency">
  #### Busque transparência
</div>

Como o Perspective não é treinado com Posts reais, certas nuances de linguagem podem levar o app a ocultar uma resposta que o usuário gostaria de manter visível. Independentemente da tecnologia ou da abordagem adotada no design do seu app, sempre faça o possível para garantir que os usuários entendam o que foi ocultado e possam fazer ajustes.

* A melhor opção é sempre confiar no usuário e dar a ele controle total sobre suas decisões. Isso significa que a experiência do usuário deve incluir controles para desfazer qualquer ação executada pelo app em nome do usuário.
* Ao usar inteligência artificial, seu app deve adotar um limiar de confiança muito alto para detectar e ocultar Posts.
* Nem todo mundo usa as mesmas palavras, e seu app deve ser projetado para evitar vieses. Fique atento a palavras ressignificadas e gírias que podem gerar falsos positivos.
* Se estiver treinando uma inteligência artificial, considere adotar um modelo que reflita com precisão a linguagem frequentemente usada no X.
