<div id="manage-replies-by-topic">
  ### Gerencie respostas por tópico
</div>

Por meio do endpoint de ocultar respostas, você pode criar integrações para ajudar pessoas e marcas a manter suas conversas no tema. Esta página mostra como gerenciar uma conversa usando os endpoints de ocultar respostas e [busca recente](/pt-BR/x-api/posts/recent-search).

A busca recente oferece funcionalidade para recuperar uma conversa e suas respostas, e o payload de Post retorna [anotações de Post](/pt-BR/x-api/fundamentals/post-annotations) para ajudar você a entender o contexto e o tema de cada Post, independentemente do idioma.

O fluxo do app terá controles para exibir e gerenciar uma conversa:

1. Solicita a permissão do usuário para ler seus Posts e gerenciar suas respostas.

2. Busca uma conversa recente a partir de uma URL de Post e verifica se a conversa é do usuário autenticado.

3. Chama o endpoint de busca recente para exibir cada Post na conversa. A solicitação incluirá uma consulta de busca pelo ID da conversa e a expansão de anotações para determinar se o Post está relacionado a esportes ou não, de acordo com a interpretação do X sobre o Post.

4. Chama Ocultar respostas para ocultar uma resposta quando o usuário assim optar. Também fornecerá uma forma de desfazer essa ação, para que o usuário esteja sempre no controle.

5. Para conversas mais longas, fornecerá controles para [paginar os resultados da busca](/pt-BR/x-api/posts/search/integrate/paginate).
    

<div id="optimize-for-the-user-and-for-usage">
  #### Otimize para o usuário (e para o uso)
</div>

Você pode projetar um fluxo que coloque o usuário no controle de qualquer ação que deseje realizar. Manter esse princípio em mente também ajuda a criar uma integração que otimize o consumo de Posts.

1. Como o usuário autenticado só pode gerenciar conversas que iniciou, seu fluxo deve ser encerrado antecipadamente quando não for esse o caso.
   - Faça uma consulta inicial ao Post. Encerre o fluxo logo no início se a URL do Post não for válida ou se a conversa não tiver sido iniciada pelo usuário autenticado.
   - Assim, seu app não precisa fazer uma pesquisa recente se a conversa não puder ser moderada pelo usuário autenticado.
2. Solicite [campos de usuário e de Post](/pt-BR/x-api/fundamentals/fields) na mesma solicitação para evitar chamadas separadas. Essa abordagem também pode melhorar o desempenho do seu app.
3. Evite fazer solicitações desnecessárias. Este app armazena em cache, no navegador do usuário, o status de ocultação de uma resposta. Isso é útil em conversas maiores, nas quais o usuário pode querer retomar a moderação mais tarde, e ajuda seu app a otimizar solicitações para ocultar ou reexibir respostas.
