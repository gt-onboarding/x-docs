<div id="getting-started-with-the-hide-replies-endpoint">
  ## Introdução ao endpoint de ocultar respostas
</div>

Este guia de início rápido ajudará você a fazer sua primeira requisição ao endpoint de ocultar respostas usando o [Postman](/pt-BR/tutorials/postman-getting-started).

Se quiser ver alguns trechos de código em diferentes linguagens, visite a [página de referência da API de ocultar respostas](/pt-BR/x-api/posts/hide-replies#put-2-tweets-id-hidden).

<Note>
  ### Pré-requisitos

  Para concluir este guia, você precisará de um conjunto de [chaves e tokens](/pt-BR/resources/fundamentals/authentication) para autenticar sua requisição. Você pode gerar essas chaves e tokens seguindo estas etapas:

  - [Inscreva-se para uma conta de desenvolvedor](https://developer.x.com/en/apply-for-access) e aguarde a aprovação.
  - Crie um [Projeto](/pt-BR/resources/fundamentals/projects) e um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) associado no portal do desenvolvedor.
  - Acesse a página “Keys and tokens” do seu App para gerar as credenciais necessárias. Certifique-se de salvar todas as credenciais em um local seguro.
</Note>

<div id="steps-to-build-a-put-tweetsidhidden-request">
  ### Etapas para criar uma solicitação PUT /tweets/:id/hidden
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### Passo um: comece com uma ferramenta ou biblioteca
</div>

Há várias ferramentas, exemplos de código e bibliotecas que você pode usar para fazer uma requisição a este endpoint, mas aqui vamos usar o Postman para simplificar o processo.

Para adicionar a coleção da API do X v2 do Postman ao seu ambiente, clique no botão a seguir:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Adicionar API do X v2 ao Postman
</Button>

Quando a coleção da API do X v2 estiver carregada no Postman, navegue até o endpoint de ocultar respostas.

<div id="step-two-authenticate-your-request">
  #### Etapa dois: autentique sua solicitação
</div>

Para fazer uma solicitação corretamente à API do X, você precisa confirmar que tem permissão. Para isso, neste endpoint, você deve autenticar sua solicitação usando [OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication) ou [OAuth 2.0 Authorization Code with PKCE](pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

Neste exemplo, usaremos OAuth 1.0a User Context.

Você deve adicionar suas chaves e tokens — especificamente sua API Key, API Secret Key, OAuth 1.0a user Access Token e OAuth 1.0a user Access Token Secret — ao Postman. Você pode fazer isso selecionando o ambiente chamado “API do X v2” no canto superior direito do Postman e adicionando suas chaves e tokens nos campos “initial value” e “current value” (clicando no ícone de olho ao lado da lista suspensa de ambientes).

Essas variáveis serão preenchidas automaticamente na guia de autorização da solicitação se você tiver feito tudo corretamente.
 

<div id="step-three-find-a-post-id-to-hide">
  #### Etapa três: encontre um ID de Post para ocultar
</div>

O endpoint de ocultar respostas pode ocultar ou reexibir respostas em nome de um usuário autorizado. Como estamos usando os tokens de acesso vinculados ao seu perfil de usuário neste exemplo, você poderá ocultar respostas de usuários que participem de uma conversa iniciada por você. Da mesma forma, se você estivesse usando tokens de acesso pertencentes a outro usuário que autorizou seu app, poderia moderar respostas em qualquer conversa iniciada por essa conta.

Peça a um amigo para responder a um Post (avise que você está testando a ocultação de respostas) ou responda a qualquer um dos seus Posts a partir de uma conta de teste. Clique nessa resposta e copie a parte numérica da URL. Esse será o ID do Post que vamos ocultar.

Neste caso, vamos analisar o seguinte Post, que tem o ID `1232720193182412800`:

`https://x.com/TwitterDev/status/1232720193182412800`

<div id="step-four-hide-the-post">
  #### Etapa quatro: ocultar o Post
</div>

No Postman, abra a pasta Hide replies e selecione Hide a reply. Na guia Params, cole o ID do Post ao lado do campo id (você não precisará substituir :id na URL). Clique em "Send" e você verá uma resposta bem-sucedida.

`{"hidden":true}`

<div id="step-five-unhide-the-post">
  #### Etapa cinco: Reexibir o Post
</div>

Posts ocultos são movidos para uma guia separada no app do X. Para reexibir um Post no Postman, abra a pasta Hide replies e selecione Unhide a reply. Na aba Params, cole o mesmo ID do Post usado na etapa anterior no campo id. Clique em "Send" e você verá uma resposta bem-sucedida.

`{"hidden":false}`

O campo hidden representa o status de ocultação do Post. Um status de ocultação igual a true significa que o Post está oculto. Da mesma forma, false significa que o Post não está oculto.
