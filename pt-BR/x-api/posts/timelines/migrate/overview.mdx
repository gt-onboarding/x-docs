<div id="comparing-x-apis-timelines-endpoints">
  ## Comparando os endpoints de timelines da API do X
</div>

Os endpoints de timeline cronológica inversa, timeline de Posts do usuário e timeline de menções do usuário da v2 substituem, respectivamente, os endpoints [v1.1 statuses/home\_timeline](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-home_timeline), [v1.1 statuses/user\_timeline](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-user_timeline.html) e [v1.1 statuses/mentions\_timeline](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-mentions_timeline.html). Se você tem código, apps ou ferramentas que usam uma versão mais antiga desse endpoint e está considerando migrar para o endpoint mais recente da API do X v2, este guia é para você. Para um guia de migração mais detalhado, consulte [Migração do Standard v1.1 para a API do X v2](/pt-BR/x-api/posts/timelines/migrate/standard-to-twitter-api-v2).

Esta página contém três tabelas de comparação:

- Timeline inicial em ordem cronológica inversa
- Timeline de Posts do usuário
- Timeline de menções do usuário

<div id="reverse-chronological-home-timeline">
  ### Timeline inicial em ordem cronológica reversa
</div>

As tabelas a seguir comparam os endpoints padrão da v1.1 e da API do X v2 para a timeline inicial:

|     |     |     |
| :--- | :--- | :--- |
| **Descrição** | **Padrão v1.1** | **API do X v2** |
| Documentação | [Referência da API](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-home_timeline) | [Referência da API](https://developer.x.com/x-api/posts/timelines#user-mention-timeline-3) |
| Métodos HTTP compatíveis | `GET` | `GET` |
| Domínio do host | `https://api.x.com` | `https://api.x.com` |
| Caminhos do endpoint | `/1.1/statuses/home_timeline.json` | `/2/users/:id/timelines/reverse_chronological` |
| Parâmetros obrigatórios | `user_id` ou `screen_name` | ID do usuário definido como parâmetro de caminho :id |
| Autenticação | Contexto de Usuário OAuth 1.0a | Contexto de Usuário OAuth 1.0a<br /><br />[Fluxo de código de autorização OAuth 2.0 com PKCE](/pt-BR/resources/fundamentals/authenticationoauth-2-0/authorization-code) |
| Limites de taxa de requisições/Limites de volume | 15 requisições por janela de 15 minutos com Contexto de Usuário OAuth 1.0a<br /><br />Limite de requisições: 100.000 em um período de 24 horas. | 180 requisições por janela de 15 minutos<br /><br />[Limites de posts](/pt-BR/x-api/fundamentals/post-cap):<br /><br />500.000 com acesso Essential<br /><br />2 milhões com acesso Elevated<br /><br />10 milhões com acesso Academic Research |
| Posts padrão por resposta | 15 | 100 |
| Máximo de Posts por resposta | 800 | Este endpoint retorna todos os Posts criados em uma timeline nos últimos 7 dias, bem como os 800 mais recentes, independentemente da data de criação. |
| Fornece histórico de edição de Post | ✔ | ✔ |
| Posts históricos disponíveis | Os 800 Posts mais recentes, incluindo Retweets | Os 3.200 Posts mais recentes, incluindo Retweets |
| Opções de navegação da timeline | since\_id (exclusivo) usado para polling de atualização<br /><br />`max_id` (inclusive) | `start_time`<br /><br />`end_time`<br /><br />`since_id` (exclusivo) usado para polling de atualização<br /><br />`until_id` (exclusivo) |
| Parâmetros opcionais para refinamento de resultados | `count`<br /><br />`exclude_replies`<br /><br />`include_rts`<br /><br />`trim_user`<br /><br />`tweet_mode`<br /><br />`since_id`<br /><br />`max_id` | `max_results`<br /><br />`exclude` (retweets,replies)<br /><br />`tweet.fields`<br /><br />`user.fields`<br /><br />`place.fields`<br /><br />`media.fields`<br /><br />`poll.fields`<br /><br />`expansions`<br /><br />`start_time`<br /><br />`end_time`<br /><br />`since_id`<br /><br />`until_id` |
| Compatível com solicitação e recebimento de [anotações](/pt-BR/x-api/fundamentals/post-annotations) | N/D | Se as anotações estiverem incluídas em `tweet.fields`, os resultados serão anotados com dados inferidos com base no texto do Post, como “Gênero musical” e “Música folk” ou “Músico” e “Dolly Parton”. |
| Compatível com solicitação e recebimento de [métricas](/pt-BR/x-api/fundamentals/metrics) específicas de Post | N/D | Se as métricas estiverem incluídas em `tweet.fields`, os resultados trarão `public_metrics` por Post, incluindo `retweet_count`, `reply_count`, `quote_count` e `like_count`, e `non_public_metrics`, incluindo `impression_count`, `user_profile_clicks`, `url_link_clicks`.<br /><br />Métricas adicionais de mídia, como `view_count`, e métricas de reprodução de vídeo.<br /><br />`organic_metrics` e `promoted_metrics` adicionais disponíveis com Contexto de Usuário para Posts promovidos. |
| Compatível com solicitação e recebimento de [conversation\_id](/pt-BR/x-api/fundamentals/conversation-id) | N/D | Retorna um campo `conversation_id` cujo valor representa o primeiro Post publicado em um encadeamento de respostas, ajudando a acompanhar conversas. |
| Formato JSON de Post | [Formato de dados Standard v1.1](https://developer.x.com/en/docs/twitter-api/v1/data-dictionary/overview.html) | Formato da [API do X v2](/pt-BR/x-api/fundamentals/data-dictionary) (determinado pelos parâmetros de fields e expansions na solicitação; não é compatível retroativamente com os formatos v1.1)<br /><br />Para saber mais sobre como migrar do formato Standard v1.1 para o formato da API do X v2, visite nosso [guia de migração de formatos de dados](/pt-BR/x-api/migrate/data-format-migration). |
| Ordem dos resultados | Ordem cronológica inversa | Ordem cronológica inversa |
| Paginação dos resultados | N/A — é necessário usar navegação pelo ID do Post | Os resultados podem ser percorridos para frente ou para trás usando um pagination\_token |
| Requer o uso de credenciais de um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) associado a um [Projeto](/pt-BR/resources/fundamentals/projects) |     | ✔   |
