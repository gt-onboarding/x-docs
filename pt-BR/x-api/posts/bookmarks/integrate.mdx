<div id="authentication">
  #### Autenticação
</div>

Todos os endpoints da API do X v2 exigem que você autentique suas solicitações com um conjunto de credenciais, também conhecido como chaves e tokens.

Esses endpoints específicos exigem o uso de [OAuth 2.0 com PKCE (Authorization Code Flow)](/pt-BR/resources/fundamentals/authentication/oauth-2-0/authorization-code), o que significa que você deve usar um conjunto de chaves e tokens de acesso de usuário para fazer uma solicitação bem-sucedida. Os tokens de acesso devem estar associados ao usuário em cujo nome você está fazendo a solicitação. Se você quiser gerar um conjunto de tokens de acesso para outro usuário, ele deverá autorizar ou autenticar seu app usando uma [URL de autorização](/pt-BR/resources/fundamentals/authentication#authorize-url).

Observe que o OAuth 2.0 pode ser complexo de usar. Se você não estiver familiarizado com esse método de autenticação, recomendamos usar uma [biblioteca](/pt-BR/x-api/tools-and-libraries/overview) ou uma ferramenta como o Postman para autenticar suas solicitações corretamente.

<div id="developer-portal-projects-and-developer-apps">
  #### Portal do desenvolvedor, Projetos e Apps de desenvolvedor
</div>

Para obter um conjunto de credenciais de autenticação que funcione com os endpoints da API do X v2, você deve ter uma [conta de desenvolvedor](/pt-BR/resources/fundamentals/developer-portal), configurar um [Projeto](/pt-BR/resources/fundamentals/projects) nessa conta e criar um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) dentro desse Projeto. Em seguida, você encontrará suas chaves e tokens no seu App de desenvolvedor. 

<div id="rate-limits">
  #### Limites de taxa
</div>

Todos os dias, muitos milhares de desenvolvedores fazem solicitações à API do X. Para ajudar a gerenciar o grande volume dessas solicitações, [limites de taxa](/pt-BR/resources/fundamentals/rate-limits) são aplicados a cada endpoint, restringindo o número de solicitações que você pode fazer em nome do seu app ou de um usuário autenticado. 

Esses endpoints têm limite de taxa no nível do usuário, o que significa que o usuário autenticado em nome de quem você está fazendo a solicitação só pode chamar o endpoint um determinado número de vezes em qualquer App de desenvolvedor. Há um limite de taxa por usuário de 180 solicitações por janela de 15 minutos para o método GET. Com o método GET do endpoint de consulta de Bookmarks, você receberá 800 dos seus Posts mais recentes salvos. Além disso, há um limite de taxa por usuário de 50 solicitações por janela de 15 minutos para os métodos POST e DELETE.
