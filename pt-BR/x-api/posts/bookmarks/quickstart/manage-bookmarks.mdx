<div id="getting-started-with-the-manage-bookmarks-endpoints">
  ## Introdução aos endpoints de gerenciamento de Bookmarks
</div>

Este guia rápido ajudará você a fazer sua primeira solicitação aos endpoints de gerenciamento de Bookmarks usando o [Postman](/pt-BR/tutorials/postman-getting-started).

Se você quiser ver códigos de exemplo em diferentes linguagens, visite nosso repositório no GitHub de [código de exemplo da API do X v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code).

<Note>
  **Pré-requisitos**

  Para concluir este guia, você precisará de um conjunto de [chaves e tokens](/pt-BR/resources/fundamentals/authentication) para autenticar sua solicitação. Você pode gerar essas chaves e tokens seguindo estas etapas:

  - [Inscreva-se para uma conta de desenvolvedor](https://developer.x.com/en/apply-for-access) e aguarde a aprovação.
  - Crie um [Projeto](/pt-BR/resources/fundamentals/projects) e um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) associado no portal do desenvolvedor.
  - Acesse a página “Keys and tokens” do seu App para gerar as credenciais necessárias. Certifique-se de salvar todas as credenciais em um local seguro.
</Note>

<div id="steps-to-build-a-manage-bookmarks-request">
  ### Etapas para criar uma solicitação de gerenciamento de Bookmarks
</div>

**Etapa um: Comece com uma ferramenta ou biblioteca**

Há várias ferramentas, exemplos de código e bibliotecas que você pode usar para fazer uma solicitação a este endpoint, mas aqui usaremos o Postman para simplificar o processo.

Para carregar a coleção da API do X v2 no seu ambiente do Postman, clique no botão a seguir:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Adicionar API do X v2 ao Postman
</Button>

Depois que a coleção da API do X v2 estiver carregada no Postman, navegue até a pasta “Bookmarks” e selecione “Create a Bookmark”.

**Etapa dois: Autentique sua solicitação**

Para fazer uma solicitação bem-sucedida a este endpoint, você precisará usar o [OAuth 2.0 Authorization Code Flow with PKCE](/pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3). Você pode gerar um token de acesso no Postman.

Vá até a guia “Authorization” e selecione “OAuth 2.0”.

Nessa guia, siga estes passos:

1. Dê um nome ao token

2. Selecione o Grant Type como Authorization Code (with PKCE)

3. Atualize os parâmetros:

   **Callback URL** - [https://www.example.com](https://www.example.com/)

   Isso deve corresponder ao Callback URL que você definiu na página de configurações de autenticação no Developer Portal.

   **Auth URL** - [https://x.com/i/oauth2/authorize](https://x.com/i/oauth2/authorize)

   **Access Token URL** - [https://api.x.com/2/oauth2/token](https://api.x.com/2/oauth2/token)

   **Client ID** - copie e cole o OAuth 2.0 Client ID do Developer Portal

   **Client Secret** - copie e cole o OAuth 2.0 Client Secret do Developer Portal. Você só precisará disso se estiver usando um tipo de App que seja um cliente confidencial.

4. Atualize os escopos com os seguintes valores: post.read users.read bookmark.write

5. Preencha o campo state com “State”

6. Clique em “Generate Token”

7. Pressione o ícone de salvar para salvar as alterações na pasta.

Você pode receber uma mensagem de que não está conectado ao X. Se isso ocorrer, será necessário entrar na conta do X, dentro do Postman, em nome da qual você está tentando publicar.

**Etapa três: Especifique um usuário**

Com este endpoint, você deve especificar o ID do usuário cujos seguidores você deseja receber na resposta. Por exemplo, o ID de usuário de @XDevelopers é `2244994945`.

No Postman, vá até a guia “Params” e insira o seu ID ou o de um usuário autenticado como valor do parâmetro id.

|     |     |     |
| :--- | :--- | :--- |
|     | Key | Value |
|     | id  | `2244994945` |

**Etapa quatro: Especifique um Post que você deseja salvar nos Bookmarks**

Vá até a guia “Body” e garanta que o Post ID esteja definido para o Post que você deseja salvar nos Bookmarks. O payload JSON deve ser semelhante ao abaixo.

```
      {"tweet_id": "1460323737035677698"}
```

**Etapa cinco: Faça sua solicitação e revise a resposta**

Depois que tudo estiver configurado, clique no botão “Send” e você receberá uma resposta semelhante ao exemplo a seguir:

```
{
   "data": {
       "bookmarked": true
   }
}
```

Para excluir um Post, selecione a solicitação “Remove a Bookmark”, também localizada na pasta “Bookmarks” da coleção da API do X v2 carregada no Postman. Primeiro, especifique o ID do usuário em nome de quem você está fazendo a solicitação como valor da coluna “`id`”. Este endpoint também requer o ID do Post que você deseja excluir. Em seguida, na guia “Params”, insira o ID do Post que você deseja excluir como valor da coluna “`tweet_id`”.

Quando a solicitação de exclusão for bem-sucedida, você receberá uma resposta semelhante ao exemplo a seguir:

```
{
   "data": {
       "bookmarked": false
   }
}
```
