Pesquisar Posts é um recurso importante usado para trazer à tona conversas no X sobre um tópico ou evento específico. Embora essa funcionalidade exista no X, esses endpoints oferecem maior flexibilidade e poder ao filtrar e ingerir Posts, para que você possa encontrar dados relevantes para sua pesquisa com mais facilidade; criar aplicativos de “escuta” quase em tempo real; ou, de modo geral, explorar, analisar e/ou agir sobre Posts relacionados a um tópico de interesse.

Oferecemos dois endpoints que permitem pesquisar Posts: busca recente e busca em todo o arquivo. Ambos esses endpoints REST compartilham um design e recursos em comum, incluindo o uso de uma única consulta de busca para filtrar Posts sobre um tópico específico. Essas consultas são criadas com um conjunto de operadores que fazem correspondência com atributos do Post e do usuário, como palavras-chave da mensagem, hashtags e URLs. Os operadores podem ser combinados em consultas com lógica booleana e parênteses para ajudar a refinar o comportamento de correspondência das consultas.

Tanto os endpoints de busca recente quanto os de busca em todo o arquivo fornecem metadados de edição de Post. Todos os objetos de Posts criados desde 29 de setembro de 2022 incluem metadados de edição, mesmo que o Post nunca tenha sido editado. Cada vez que um Post é editado, um novo ID de Post é criado. O histórico de edições de um Post é documentado por um array de IDs de Post, começando pelo ID original.

Esses endpoints sempre retornam a edição mais recente, juntamente com todo o histórico de edições. Qualquer Post coletado após sua janela de edição de 30 minutos representará sua versão final. Para saber mais sobre metadados de edição de Posts, confira a página [Fundamentos de edição de Posts](/pt-BR/x-api/fundamentals/edit-posts).

Depois que você configurar sua consulta e começar a receber Posts, esses endpoints oferecem suporte à paginação dos resultados tanto por tempo quanto por faixas de ID de Post. Isso foi projetado para atender a dois casos de uso comuns:

- **Obter histórico**: solicitações para um período de interesse, sem foco no caráter em tempo real dos dados. Uma única solicitação é feita, e todos os dados correspondentes são entregues usando paginação conforme necessário. Este é o modo padrão para Search Posts.
- **Polling** ou **escuta**: solicitações feitas no modo “há novos Posts desde minha última solicitação?”. As solicitações são contínuas e, normalmente, há um caso de uso focado em “escutar” quase em tempo real por Posts de interesse.

Muitos operadores e limites de consulta são exclusivos para [Acesso Enterprise](/pt-BR/x-api/getting-started/about-x-api), o que significa que você deve usar chaves e tokens de um App dentro de um [Projeto](/pt-BR/resources/fundamentals/projects) com acesso Enterprise para utilizar a funcionalidade adicional. Você pode saber mais sobre isso nas seções de endpoint abaixo.

Os Posts retornados tanto pelos endpoints de busca recente quanto pelos de busca em todo o arquivo contam para o [Limites de posts](/pt-BR/x-api/fundamentals/post-cap) mensal.

**Configuração de conta**

Para acessar esses endpoints, você precisará de:

- Uma [conta de desenvolvedor](https://developer.x.com/en/portal/petition/essential/basic-info) aprovada.
- Autenticar-se usando as chaves e tokens de um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) que esteja dentro de um [Projeto](/pt-BR/resources/fundamentals/projects).

Saiba mais sobre como obter acesso aos endpoints da API do X v2 em nosso [guia de primeiros passos](/pt-BR/x-api/getting-started/getting-access).

<div id="recent-search">
  ## Busca recente
</div>

O endpoint de busca recente permite acessar programaticamente Posts públicos filtrados publicados na última semana e está disponível para todos os desenvolvedores que têm uma conta de desenvolvedor e estão usando chaves e tokens de um [App](/pt-BR/resources/fundamentals/developer-apps) dentro de um [Project](/pt-BR/resources/fundamentals/projects).

Você pode autenticar suas solicitações com [OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication), [OAuth 2.0 App-Only](/pt-BR/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) ou [OAuth 2.0 Authorization Code com PKCE](pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). No entanto, se quiser receber métricas privadas ou a separação entre métricas orgânicas e promovidas nos resultados do seu Post, será necessário usar OAuth 1.0a User Context ou OAuth 2.0 Authorization Code com PKCE e enviar os Access Tokens do usuário associados ao usuário que publicou o conteúdo em questão.

Esse endpoint pode retornar até 100 Posts por solicitação em ordem cronológica inversa, e tokens de [paginação](/pt-BR/x-api/posts/search/integrate/paginate) são fornecidos para percorrer grandes conjuntos de Posts correspondentes.

Ao usar um Project com acesso regular, você pode usar o conjunto básico de [operadores](/pt-BR/x-api/posts/search/integrate/build-a-query) e fazer consultas com até 512 caracteres. Ao usar um Project com acesso Enterprise, você tem acesso a operadores adicionais. Projects com acesso Enterprise podem fazer consultas com até 4096 caracteres.

Saiba mais sobre [níveis de acesso](/pt-BR/x-api/getting-started/about-x-api).

<div id="full-archive-search">
  ## Busca em todo o arquivo
</div>

O endpoint de busca em todo o arquivo da v2 está disponível apenas para Projetos com acesso [Pro](/pt-BR/x-api/getting-started/about-x-api "Pro") e [Enterprise](/pt-BR/x-api/getting-started/about-x-api). Ele permite acessar programaticamente Posts públicos de todo o arquivo, desde o primeiro Post em março de 2006, com base na sua consulta de busca.

Você pode autenticar suas solicitações a este endpoint usando [OAuth 2.0 App-Only](/pt-BR/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), e o [App Access Token](/pt-BR/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token) deve vir de um App que faça parte de um Projeto com acesso Pro ou Enterprise. Como não é possível fazer solicitações em nome de outros usuários (OAuth 1.0a User Context ou OAuth 2.0 Authorization Code com PKCE) neste endpoint, você não poderá obter [métricas](/pt-BR/x-api/fundamentals/metrics) privadas.

Este endpoint pode retornar até 500 Posts por solicitação em ordem cronológica inversa, e tokens de [paginação](/pt-BR/x-api/posts/search/integrate/paginate) são fornecidos para percorrer grandes conjuntos de Posts correspondentes.

Observação: Se você solicitar [anotações](/pt-BR/x-api/fundamentals/post-annotations) por meio do parâmetro tweet.fields, o parâmetro max\_results está atualmente limitado a um valor máximo de 100. Isso pode mudar no futuro, portanto, fique atento a essa limitação.

Como este endpoint está disponível apenas para quem foi aprovado para acesso Pro e Enterprise, você tem acesso ao conjunto completo de [operadores](/pt-BR/x-api/posts/search/integrate/build-a-query) de busca e pode fazer consultas com até 1024 caracteres.
