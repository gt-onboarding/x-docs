<div id="step-one-sign-up-for-a-developer-account">
  ### Passo um: Crie uma conta de desenvolvedor
</div>

Criar uma conta de desenvolvedor é rápido e fácil! Basta clicar no botão abaixo, responder a algumas perguntas e você já poderá começar a explorar e desenvolver na API do X v2 usando o [Acesso Básico](https://developer.x.com/en/portal/products/basic).

Em seguida, durante o processo de onboarding, você criará um Projeto e um App de desenvolvedor associado, que fornecerão um conjunto de credenciais para autenticar todas as solicitações à API.

[Inscreva-se](https://developer.x.com/en/portal/products/basic)

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### Etapa dois: Salve a chave e os tokens do seu App e mantenha-os em segurança
</div>

Depois que você tiver acesso e criar um Projeto e um App, poderá encontrar ou gerar as seguintes credenciais no seu App de desenvolvedor:

- **[API Key and Secret](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret):** Basicamente o nome de usuário e a senha do seu App. Você usará essas credenciais para autenticar solicitações que exigem [OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication) ou para gerar outros tokens, como tokens de acesso de usuário (user Access Tokens) ou token de acesso do App (App Access Token).

- **[Access Token and Secret](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens):** Em geral, os Access Tokens representam o usuário em nome de quem você está fazendo a solicitação. Os que você pode gerar pelo portal do desenvolvedor representam o usuário proprietário do App. Você usará esses tokens para autenticar solicitações que exigem [OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication). Se quiser fazer solicitações em nome de outro usuário, será necessário usar o fluxo OAuth de 3 etapas (3-legged) para que ele autorize você.

- **[Client ID and Client Secret](/pt-BR/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce):** Essas credenciais são usadas para obter um token de acesso de usuário com autenticação OAuth 2.0. Assim como no OAuth 1.0a, os tokens de acesso de usuário são usados para autenticar solicitações que fornecem informações privadas da conta do usuário ou executam ações em nome de outra conta, porém com escopos granulares para maior controle sobre quais acessos o aplicativo cliente tem ao usuário.

- **[App only Access Token](/pt-BR/resources/fundamentals/authentication#bearer-token-also-known-as-app-only):** Você usará esse token ao fazer solicitações a endpoints que respondem com informações publicamente disponíveis no X.

Como essas chaves e tokens não expiram a menos que sejam regenerados, sugerimos que você os salve em um local seguro, como um gerenciador de senhas, assim que receber suas credenciais.

> **Atenção: suas chaves e tokens serão exibidos apenas uma vez no [developer portal](/pt-BR/resources/fundamentals/developer-portal), portanto, é importante que você armazene essas credenciais no seu gerenciador de senhas assim que as gerar.**
>
> Se você perder ou esquecer as chaves e tokens, será necessário regenerá-los, o que cria novas credenciais e invalida as antigas. Isso significa que você terá que atualizar quaisquer integrações que tenha configurado com suas credenciais anteriores.
>
> Saiba mais sobre nossas [práticas recomendadas de autenticação](/pt-BR/resources/fundamentals/authentication#authentication-best-practices).
