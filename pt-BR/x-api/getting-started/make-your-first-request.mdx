Este guia o conduzirá por algumas etapas que você pode seguir para fazer sua primeira solicitação. É um excelente recurso para usar após criar sua conta no X.

Se você estiver interessado em usar exemplos de código, guias mais técnicos ou uma ferramenta gráfica como o Postman, considere os seguintes recursos para fazer sua primeira solicitação:

- [Guia de início rápido para fazer sua primeira solicitação ao endpoint de posts](/pt-BR/x-api/posts/manage-tweets/quickstart)
- [Introdução ao Postman](/pt-BR/tutorials/postman-getting-started) - ideal para iniciantes
- [Amostras de código da API do X v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code)
- [Referência da API de consulta de posts](/pt-BR/x-api/posts/lookup/introduction)

Este guia pressupõe que você já obteve sua [chave e segredo da API](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret), [Access Token e Secret do usuário](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens), [App Access Token](/pt-BR/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) e os armazenou em um local seguro. Você pode aprender como fazer isso seguindo as etapas do guia [obtendo acesso à API do X](/pt-BR/x-api/getting-started/getting-access).

<div id="step-1-identify-which-endpoint-you-would-like-to-use">
  ### Etapa 1. Identifique qual endpoint você deseja usar
</div>

A API do X permite executar diversas ações por meio de código, semelhantes às que você pode realizar no site do X ou no aplicativo móvel.

Temos uma lista completa dos endpoints disponíveis na API em nosso [Índice de referência da API](/pt-BR/x-api/introduction), mas recomendamos ficar com um dos seguintes, para simplificar:

- [Gerenciar posts > Publicar um Tweet](/pt-BR/x-api/posts/manage-tweets/introduction)
- [Buscar posts](/pt-BR/x-api/posts/search/introduction)
- [Consulta de usuários](/pt-BR/x-api/users/lookup/introduction)

<div id="step-2-choose-a-tool-to-make-your-request">
  ### Etapa 2. Escolha uma ferramenta para fazer sua solicitação
</div>

Embora algumas solicitações sejam diretas, outras podem ser mais difíceis de montar. Por isso, a nossa incrível comunidade de desenvolvedores criou ferramentas que abstraem parte dessa complexidade.

A seguir, confira algumas ferramentas recomendadas e detalhes sobre como usá-las:

<div id="postman">
  #### Postman
</div>

O Postman é uma ferramenta visual que você pode usar para fazer solicitações a endpoints REST. Criamos ótimos materiais sobre o Postman para ajudar você a começar e explorar os diferentes endpoints disponíveis por meio da API do X.

Recomendamos que você leia nosso tutorial ["Introdução ao Postman"](/pt-BR/tutorials/postman-getting-started) para aprender como adicionar suas chaves e tokens e fazer sua primeira solicitação.

Também produzimos um guia de início rápido para cada um dos nossos endpoints da API do X v2, a maioria dos quais usa o Postman. Você pode encontrar esses guias em cada seção de endpoint correspondente; aqui estão alguns links:

- Início rápido: Publicar um post
- Início rápido: Buscar posts
- Início rápido: Consultar um usuário

Observe que não é possível fazer solicitações a endpoints de streaming usando o Postman. Acesse o guia de início rápido do [fluxo filtrado](/pt-BR/x-api/posts/filtered-stream/quickstart) ou do [fluxo amostrado de 1%](/pt-BR/x-api/posts/sample-stream) para aprender como trabalhar com esses endpoints.

Se você preferir uma ferramenta gráfica mais simples, considere também usar o [Insomnia](https://insomnia.rest).

<div id="sample-code">
  #### Código de exemplo
</div>

Se você quiser usar um código simples para fazer sua solicitação, reunimos exemplos em várias linguagens para cada um dos nossos endpoints da API do X v2.

Você pode encontrar os exemplos de código no nosso repositório no GitHub, [X-API-v2-sample-code](https://github.com/xdevplatform/Twitter-API-v2-sample-code), que também inclui um arquivo README com instruções sobre como configurar suas credenciais para que as solicitações funcionem corretamente.

Por exemplo, aqui está um exemplo de cURL para o endpoint de consulta de usuário. Tudo o que você precisa fazer para usar essa solicitação é substituir $ACCESS\_TOKEN e $USERNAME pelo seu [Token de Acesso do App](/pt-BR/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) e pelo @ do X. Em seguida, copie e cole este código na sua ferramenta de linha de comando e pressione Enter.

```bash
curl "https://api.x.com/2/users/by/username/$USERNAME" -H "Authorization: Bearer $ACCESS_TOKEN"
```

<div id="libraries">
  #### Bibliotecas
</div>

A incrível comunidade de desenvolvedores do X também criou bibliotecas em diversas linguagens de programação que podem ser usadas para fazer requisições à API do X.

Criamos uma página de ["Ferramentas e bibliotecas"](/pt-BR/x-api/tools-and-libraries/overview) que lista todas as bibliotecas da comunidade das quais temos conhecimento. Cada biblioteca deve incluir um arquivo README com instruções para configurar o repositório na sua máquina e fazer sua primeira requisição.

> **Observação:** Se você tiver algum problema, consulte a documentação do endpoint ao qual está fazendo a requisição, nossa [seção de suporte](https://developer.x.com/en/support) ou peça ajuda à comunidade em nossos fóruns. Nós o ajudaremos a realizar uma requisição bem-sucedida!

<div id="step-3-review-the-response">
  ### Etapa 3. Revise a resposta
</div>

Depois de fazer uma solicitação bem-sucedida, você receberá um payload com metadados relacionados à solicitação.

Se você usou um endpoint que utiliza o método HTTP GET, receberá metadados relacionados ao recurso (post, usuário, lista, Space, etc.) ao qual fez a solicitação, em formato JSON. Revise os diferentes campos retornados e veja se consegue mapear as informações solicitadas ao conteúdo no X.

Se você usou um endpoint que utiliza os métodos HTTP POST, PUT ou DELETE, você executou uma ação no X. Acesse X.com ou o app móvel e verifique se consegue localizar essa ação.

<div id="step-4-adjust-your-request-using-parameters">
  ### Etapa 4. Ajuste sua solicitação usando parâmetros
</div>

Cada endpoint tem um conjunto diferente de parâmetros que você pode usar para ajustar sua solicitação. Por exemplo, ao usar endpoints GET, você pode solicitar campos de metadados adicionais com os parâmetros fields e expansions. Você também pode experimentar diversas ferramentas de filtragem em endpoints como [search Posts](/pt-BR/x-api/posts/search/introduction), [Post counts](/pt-BR/x-api/posts/counts/introduction) e [filtered stream](/pt-BR/x-api/posts/filtered-stream/introduction) para restringir os dados recebidos apenas aos Posts do seu interesse.

Você encontra a lista completa de parâmetros e campos de solicitação na referência da API do endpoint com o qual está trabalhando, além de várias outras informações úteis de integração em nossos guias de integração e páginas de fundamentos.

Saiba mais sobre todos os materiais educacionais que disponibilizamos no nosso guia de [Important resources](/pt-BR/x-api/getting-started/important-resources).
