<div id="authentication">
  #### Autenticação
</div>

Todos os endpoints da API do X v2 exigem que você autentique suas solicitações com um conjunto de credenciais, também conhecido como chaves e tokens. Você pode usar OAuth 1.0a User Context, App only ou OAuth 2.0 Authorization Code com PKCE para autenticar suas solicitações a esses endpoints.

[OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication#oauth-1-0a-user-context), o que significa que você deve usar um conjunto de API Keys e Access Tokens de usuário para fazer uma solicitação bem-sucedida. Os access tokens devem estar associados ao usuário em nome de quem você está fazendo a solicitação. Se você quiser gerar um conjunto de Access Tokens para outro usuário, ele deve autorizar seu App usando o [fluxo OAuth de 3 etapas](/pt-BR/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow).

Observe que o OAuth 1.0a pode ser difícil de usar. Se você não estiver familiarizado com esse método de autenticação, recomendamos usar uma [biblioteca](/pt-BR/x-api/tools-and-libraries/overview), uma ferramenta como o Postman ou o OAuth 2.0 para autenticar suas solicitações.

[OAuth 2.0 Authorization Code com PKCE](/pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) permite maior controle sobre o escopo de um aplicativo e fluxos de autorização em vários dispositivos. O OAuth 2.0 permite escolher escopos específicos e granulares que concedem permissões específicas em nome de um usuário.

Para habilitar o OAuth 2.0 no seu App, você deve ativá-lo nas configurações de autenticação do seu App, encontradas na seção de configurações do App no portal do desenvolvedor.

[App only](/pt-BR/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) exige apenas que você envie um [App only Access Token](/pt-BR/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) com sua solicitação. Você pode gerar um token App only diretamente em um App de desenvolvedor ou gerar um usando o endpoint [POST oauth2/token](/pt-BR/resources/fundamentals/authentication#post-oauth2-token).

<div id="developer-portal-projects-and-developer-apps">
  #### Portal do desenvolvedor, Projetos e Apps de desenvolvedor
</div>

Para obter um conjunto de credenciais de autenticação compatível com os endpoints da API do X v2, você deve [se inscrever para uma conta de desenvolvedor](https://developer.x.com/en/portal/petition/essential/basic-info), configurar um [Projeto](/pt-BR/resources/fundamentals/projects) nessa conta e criar um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) nesse Projeto. Em seguida, você encontrará suas chaves e tokens no seu App de desenvolvedor.
 

<div id="rate-limits">
  #### Limites de taxa
</div>

Todos os dias, muitos milhares de desenvolvedores fazem solicitações à API do X. Para ajudar a gerenciar o grande volume dessas solicitações, [limites de taxa](/pt-BR/x-api/fundamentals/rate-limits) são aplicados a cada endpoint, limitando o número de solicitações que você pode fazer em nome do seu app ou de um usuário autenticado. 

Esses endpoints têm limites de taxa tanto no nível do app quanto no nível do usuário. O limite de taxa do app significa que você, como desenvolvedor, só pode fazer um certo número de solicitações a esse endpoint em um determinado período a partir de qualquer app (usando a API Key e a API Secret Key, ou o App-only Access Token). O limite de taxa do usuário significa que o usuário autenticado em cujo nome você está fazendo a solicitação só pode realizar uma consulta de Lista um certo número de vezes em qualquer app de desenvolvedor.

A tabela abaixo mostra os limites de taxa para cada endpoint.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Método HTTP** | **Limite de taxa** |
| /2/lists/:id | GET | 75 solicitações por 15 minutos |
| /2/users/:id/owned\_lists | GET | 15 solicitações por 15 minutos |

Campos e expansões

O endpoint GET da API do X v2 permite que os usuários selecionem exatamente quais dados desejam obter da API usando um conjunto de recursos chamados `fields` e `expansions`. O parâmetro `expansions` permite expandir objetos referenciados no payload. Por exemplo, consultar uma Lista por ID permite obter as seguintes [expansions](/pt-BR/x-api/fundamentals/expansions):

- `owner_id`

O parâmetro `fields` permite selecionar exatamente quais [fields](/pt-BR/x-api/fundamentals/fields) dentro dos diferentes objetos de dados você deseja receber. Esses endpoints fornecem principalmente objetos de Lista. Por padrão, o objeto de Lista retorna os campos `id` e `name`. Para receber campos adicionais, como `list.created_at` ou `list.follower_count`, você precisará solicitá-los especificamente usando o parâmetro list.fields. 

Adicionamos um guia sobre como usar [fields e expansions](/pt-BR/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) em conjunto ao nosso [dicionário de dados da API do X v2](/pt-BR/x-api/fundamentals/data-dictionary).

A tabela abaixo mostra os campos e expansões disponíveis para este grupo de endpoints:

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Fields** | **Expansions** |
| /2/lists/:id | `list.fields`<br /><br />`user.fields` | `owner_id` |
| /2/users/:id/owned\_lists | `list.fields`<br /><br />`user.fields` | `owner_id` |

<div id="pagination">
  #### Paginação
</div>

Consultar listas de um usuário pode retornar muitos dados. Para garantir resultados consistentes e com alto desempenho a qualquer momento, usamos paginação. A paginação é um recurso dos endpoints da API do X v2 que retornam mais resultados do que cabem em uma única resposta. Quando isso acontece, os dados são retornados em uma série de “páginas”. Saiba mais sobre como [paginar resultados.](/pt-BR/x-api/fundamentals/pagination)
