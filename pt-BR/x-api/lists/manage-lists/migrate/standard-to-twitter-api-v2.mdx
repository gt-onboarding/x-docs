<div id="manage-lists-standard-v11-compared-to-x-api-v2">
  ### Gerenciar listas: Standard v1.1 em comparação com a API do X v2
</div>

Se você vem trabalhando com os endpoints padrão v1.1 [POST lists/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-create), [POST lists/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-destroy) e [POST lists/update](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-update), o objetivo deste guia é ajudar você a entender as semelhanças e diferenças entre os endpoints de gerenciamento de listas do Standard v1.1 e da API do X v2.

- **Semelhanças**
  - Autenticação
- **Diferenças**
  - URLs de endpoint
  - Requisitos de App e Projeto
  - Métodos HTTP
  - Limites de taxa
  - Parâmetros de solicitação

<div id="similarities">
  #### Semelhanças
</div>

**Autenticação**

Ambas as versões do endpoint oferecem suporte a [OAuth 1.0a User Context](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Portanto, se você já usava um dos endpoints padrão v1.1 para gerenciamento de listas, poderá continuar utilizando o mesmo método de autenticação ao migrar para a versão da API do X v2.

<div id="differences">
  #### Diferenças
</div>

**URLs de endpoints**

- Endpoints do Standard v1.1:
  - POST https://api.x.com/1.1/lists/create.json
    (Cria uma lista)
  - POST https://api.x.com/1.1/lists/destroy.json
    (Exclui uma lista)
  - POST https://api.x.com/1.1/lists/update.json
    (Atualiza uma lista)
- Endpoint da API do X v2:
  - POST https://api.x.com/2/lists
    (Cria uma lista)

  - DELETE https://api.x.com/2/lists/:id
    (Exclui uma lista)

  - PUT https://api.x.com/2/lists/:id
    (Atualiza uma lista)

**Limites de uso**

| **Standard v1.1** | **API do X v2** |
| :--- | :--- |
| /1.1/lists/create.json<br /><br />nenhum | /2/lists<br /><br />300 solicitações por janela de 15 minutos com OAuth 1.0a (User Context) |
| /1.1/lists/destroy.json<br /><br />nenhum | /2/lists/:id<br /><br />300 solicitações por janela de 15 minutos com OAuth 1.0a (User Context) |
| /1.1/lists/update.json<br /><br />nenhum | /2/lists/:id<br /><br />300 solicitações por janela de 15 minutos com OAuth 1.0a (User Context) |

**Requisitos de App e Projeto**

Os endpoints da API do X v2 exigem o uso de credenciais de um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) associado a um [Projeto](/pt-BR/resources/fundamentals/projects) ao autenticar suas solicitações. Todos os endpoints da API do X v1.1 podem usar credenciais de Apps independentes ou de Apps vinculados a um projeto.

**Parâmetros da solicitação**

Os seguintes parâmetros de solicitação do Standard v1.1 têm equivalentes na API do X v2:

**Criar uma lista**

| **Standard** | **API do X v2** |
| :--- | :--- |
| name | name |
| mode | private |
| description | description |

**Excluir/Atualizar uma lista**

| **Standard** | **API do X v2** |
| :--- | :--- |
| owner\_screen\_name | Sem equivalente |
| owner\_id | Sem equivalente |
| list\_id | id  |
| slug | Sem equivalente |

<Note>
  **Atenção:** os parâmetros do Standard v1.1 são enviados como parâmetros de consulta (query), enquanto os parâmetros da API do X v2 são enviados no corpo da requisição (para o endpoint POST) ou no caminho (path) da URL (para os endpoints DELETE e PUT).
</Note>
