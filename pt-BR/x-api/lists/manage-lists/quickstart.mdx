<div id="getting-started-with-the-manage-lists-endpoint-group">
  ## Introdução ao grupo de endpoints para gerenciar Listas
</div>

Esta visão geral rápida ajudará você a fazer sua primeira solicitação aos endpoints de gerenciamento de Listas usando o [Postman](/pt-BR/tutorials/postman-getting-started).

Se você quiser ver códigos de exemplo em diferentes linguagens, visite nosso repositório no GitHub com [código de exemplo da API do X v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code).

**Observação:** Neste exemplo, faremos uma solicitação ao endpoint _Create a List_, mas você pode aplicar o que aprender neste guia rápido a outras solicitações de gerenciamento.

<Note>
  **Pré-requisitos**

  Para concluir este guia, você precisará de um conjunto de [chaves e tokens](/pt-BR/resources/fundamentals/authentication) para autenticar sua solicitação. Você pode gerar essas chaves e tokens seguindo estas etapas:

  - [Inscreva-se para uma conta de desenvolvedor](https://developer.x.com/en/apply-for-access) e aguarde a aprovação.
  - Crie um [Projeto](/pt-BR/resources/fundamentals/projects) e um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) associado no portal do desenvolvedor.
  - Acesse a página “Keys and tokens” do seu App para gerar as credenciais necessárias. Certifique-se de salvar todas as credenciais em um local seguro.
</Note>

<div id="steps-to-build-a-manage-list-request">
  ### Etapas para criar uma solicitação de gerenciamento de Lista
</div>

**Primeira etapa: comece com uma ferramenta ou biblioteca**

Há várias ferramentas, exemplos de código e bibliotecas que você pode usar para fazer uma solicitação a este endpoint, mas aqui usaremos o Postman para simplificar o processo.

Para carregar a coleção da API do X v2 no seu ambiente do Postman, clique no botão a seguir:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Adicionar API do X v2 ao Postman
</Button>

Depois que a coleção da API do X v2 estiver carregada no Postman, navegue até a pasta “List”, selecione a pasta “Manage List” e, em seguida, escolha "Create a List".

**Segunda etapa: autentique sua solicitação**

Para fazer uma solicitação à API do X corretamente, você precisa verificar se tem permissão para isso. Para fazer isso com os endpoints de gerenciamento de Posts, autentique sua solicitação usando [OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication) ou [OAuth 2.0 Authorization Code with PKCE](pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

Neste exemplo, usaremos OAuth 1.0a User Context.

Você deve adicionar suas chaves e tokens (especificamente sua API Key, API Secret Key, OAuth 1.0a User Access Token e OAuth 1.0a User Access Token Secret) ao Postman. Faça isso selecionando o ambiente chamado “X API v2” (no canto superior direito do Postman) e adicionando suas chaves e tokens nos campos "initial value" e "current value" (clicando no ícone de olho ao lado da lista de ambientes).

Se você fez isso corretamente, essas variáveis serão automaticamente inseridas na guia de autorização da solicitação.
 
**Terceira etapa: especifique o nome da nova Lista**

Ao criar uma nova Lista com este endpoint, o nome da Lista é um parâmetro obrigatório no corpo. Opcionalmente, você pode fornecer uma descrição e especificar se a Lista é privada.

No Postman, acesse a guia “Body” e insira o nome da Lista como valor do parâmetro name. Além disso, se desejar adicionar uma descrição para a Lista, adicione uma nova chave chamada description da mesma forma que name, seguida da descrição da Lista como valor. Tornar uma Lista privada seguirá o mesmo padrão, mas apenas os valores true ou false são aceitos para esse parâmetro. 

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Parameter type** |
| `name` | Name of the list (required) | body |
| description | Description for the list (optional) | body |
| private | true or false (optional) | body |

Agora você deverá ver uma URL semelhante ao lado do botão "Send":

`https://api.x.com/2/lists`

Quarta etapa: envie sua solicitação e revise sua resposta

Depois que tudo estiver configurado, clique no botão "Send" e você receberá uma resposta semelhante ao exemplo a seguir:

```
{
  "data": {
    "id": "1441162269824405510",
    "name": "New list created from Postman"
  }
}
```

Se o objeto de resposta retornado contiver um id e o nome da sua Lista, você criou a Lista com sucesso. 

Para excluir uma Lista, selecione a solicitação “Delete a List” também encontrada na pasta “Lists” da coleção da API do X v2 carregada no Postman. Este endpoint requer o ID da Lista que você deseja excluir. Na guia “Params”, insira o ID da Lista que você deseja excluir como valor da coluna id. 

Em uma solicitação de exclusão bem-sucedida, você receberá uma resposta semelhante ao exemplo a seguir: 

```
{
  "data": {
    "deleted": true
  }
}
```
