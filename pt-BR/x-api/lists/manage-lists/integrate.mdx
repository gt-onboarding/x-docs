<div id="authentication">
  #### Autenticação
</div>

Todos os endpoints da API do X v2 exigem que você autentique suas requisições com um conjunto de credenciais, também conhecidas como chaves e tokens.

Esses endpoints específicos exigem o uso de [OAuth 1.0a em contexto de usuário](/pt-BR/resources/fundamentals/authentication), o que significa que você deve usar um conjunto de chaves de API e tokens de acesso do usuário para fazer uma requisição bem-sucedida. Os tokens de acesso devem estar associados ao usuário em nome de quem você está fazendo a requisição. Se você quiser gerar um conjunto de tokens de acesso para outro usuário, essa pessoa deve autorizar ou autenticar seu app usando o [fluxo OAuth de 3 etapas (3-legged)](/pt-BR/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow).

Observe que o OAuth 1.0a pode ser complexo de usar. Se você não estiver familiarizado com esse método de autenticação, recomendamos usar uma [biblioteca](/pt-BR/x-api/tools-and-libraries/overview) ou uma ferramenta como o Postman para autenticar suas requisições corretamente.
 

<div id="developer-portal-projects-and-developer-apps">
  #### Portal do desenvolvedor, Projetos e Apps de desenvolvedor
</div>

Para obter um conjunto de credenciais de autenticação que funcione com os endpoints da API do X v2, você deve [se inscrever em uma conta de desenvolvedor](https://developer.x.com/en/portal/petition/essential/basic-info), configurar um [Projeto](/pt-BR/resources/fundamentals/projects) nessa conta e criar um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) nesse Projeto. Em seguida, você poderá encontrar suas chaves e tokens no seu App de desenvolvedor.
 

<div id="rate-limits">
  #### Limites de taxa
</div>

Todos os dias, muitos milhares de desenvolvedores fazem solicitações à API do X. Para ajudar a gerenciar o grande volume dessas solicitações, [limites de taxa](/pt-BR/x-api/fundamentals/rate-limits) são aplicados a cada endpoint, limitando o número de solicitações que você pode fazer em nome do seu app ou de um usuário autenticado.

Esses endpoints têm limite de taxa no nível do usuário, o que significa que o usuário autenticado em nome de quem você está fazendo a solicitação só pode chamar o endpoint um certo número de vezes em qualquer App de desenvolvedor.

A tabela abaixo mostra os limites de taxa para cada endpoint.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Método HTTP** | **Limite de taxa** |
| /2/lists | POST | 300 solicitações por 15 minutos |
| /2/lists/:id | DELETE / PUT | 300 solicitações por 15 minutos |
