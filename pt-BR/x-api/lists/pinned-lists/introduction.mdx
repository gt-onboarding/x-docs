Fixar listas permite que os usuários do X acessem com mais facilidade as listas às quais se inscreveram ou que eles mesmos criaram.

Como você está fazendo solicitações em nome de um usuário com esses endpoints, é necessário autenticá-lo com [OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication) ou [OAuth 2.0 Authorization Code with PKCE](pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) e usar os tokens de acesso associados a um usuário que autorizou seu app, os quais podem ser gerados usando o [fluxo OAuth de 3 etapas](/pt-BR/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) ou o [fluxo de concessão Authorization Code com PKCE](/pt-BR/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0).

<div id="pinned-list-lookup">
  ### Consulta de Lista fixada
</div>

A consulta de Lista fixada possui um endpoint que permite recuperar as Listas fixadas de um usuário autenticado. Há um limite de 15 solicitações a cada 15 minutos para esse endpoint.
 

<div id="manage-pinned-lists">
  ### Gerenciar listas fixadas
</div>

Os endpoints para gerenciar listas fixadas permitem fixar e desafixar uma lista em nome de um usuário autenticado. Para esses endpoints, há dois métodos disponíveis: POST e DELETE. O método POST permite fixar uma lista, e o método DELETE permite desafixar uma lista. Há um limite de 50 solicitações por usuário a cada 15 minutos para ambos os endpoints.

<Note>
  **Configuração de conta**

  Para acessar esses endpoints, você precisará de:

  - Uma [conta de desenvolvedor](https://developer.x.com/en/portal/petition/essential/basic-info) aprovada.
  - Autenticar-se usando as chaves e tokens de um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) que está localizado dentro de um [Projeto](/pt-BR/resources/fundamentals/projects).

  Saiba mais sobre como obter acesso aos endpoints da API do X v2 em nosso [guia de primeiros passos](/pt-BR/x-api/getting-started/getting-access).
</Note>
