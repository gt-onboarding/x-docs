<div id="steps-to-build-a-pinned-list-lookup-request">
  ### Etapas para criar uma solicitação de consulta de Lista fixada
</div>

**Primeiro passo: escolha a coleção de endpoints de Lista no Postman**

Depois de carregar a coleção da API do X v2 no Postman, navegue até a pasta “List”, selecione a subpasta “Pinned Lists” e, em seguida, escolha "User's pinned Lists".
 

**Segundo passo: identifique e especifique o usuário**

Para recuperar a Lista fixada de um usuário, você deve especificar o ID desse usuário na solicitação. O ID do usuário deve corresponder ao ID do usuário autenticado, o que significa que você deve enviar os Tokens de Acesso associados a esse ID ao autenticar sua solicitação. Neste exemplo, você pode especificar o ID pertencente à sua própria conta. Você pode encontrar seu ID de duas maneiras:

1. Usando o endpoint [users lookup](/pt-BR/x-api/users/lookup/introduction) por nome de usuário, você pode informar um nome de usuário e receber o campo id.
2. No seu Token de Acesso, a parte numérica corresponde ao seu ID de usuário.

No Postman, vá até a guia "Params" e insira esse ID de usuário na coluna "Value" do parâmetro `id`.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 2244994945 (user ID) |

**Terceiro passo: identifique e especifique quais campos deseja recuperar**

Se você clicar no botão "Send" após o terceiro passo, receberá os campos padrão do [List object](/pt-BR/x-api/fundamentals/data-dictionary#list) na resposta: `id` e `name`.

Se quiser receber campos adicionais além de `id` e `name`, será necessário especificá-los na sua solicitação com os parâmetros [`field`](/pt-BR/x-api/fundamentals/fields) e/ou [`expansion`](/pt-BR/x-api/fundamentals/expansions).

Para este exercício, solicitaremos três conjuntos adicionais de campos de objetos diferentes:

- O campo adicional `follower_count` no objeto List principal.
- O [user object](/pt-BR/x-api/fundamentals/data-dictionary#user) completo usando o parâmetro de expansão.
- O campo adicional `tweet.created_at` no objeto de usuário associado.

No Postman, acesse a guia "Params" e adicione o seguinte par chave:valor à tabela "Query Params":

|     |     |     |
| :--- | :--- | :--- |
| Key | Value | Returned fields |
| list.fields | follower\_count | `follower_count` |
| expansions | owner\_id | `includes.users.id,  <br/>includes.users.name,  <br/>includes.users.username` |
| user.fields | created\_at | `includes.users.created_at` |

Agora você deverá ver uma URL semelhante ao lado do botão “Send”:

`https://api.x.com/2/users/2244994945/pinned_lists?expansions=owner_id&list.fields=follower_count&user.fields=created_at`

**Quarto passo: faça sua solicitação e revise a resposta**

Depois que tudo estiver configurado, clique no botão "Send" e você receberá uma resposta semelhante ao exemplo a seguir:

```
{
  "data": [
    {
      "follower_count": 0,
      "id": "1454155907651158017",
      "name": "test list",
      "owner_id": "2244994945"
    }
  ],
  "includes": {
    "users": [
      {
        "username": "TwitterDev",
        "id": "2244994945",
        "created_at": "2013-12-14T04:35:55.000Z",
        "name": "Twitter Dev"
      }
    ]
  },
  "meta": {
    "result_count": 1
  }
}
```
