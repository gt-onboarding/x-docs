<div id="getting-started-with-the-pinned-list-endpoint-group">
  ## Introdução ao grupo de endpoints de Listas fixadas
</div>

Esta visão geral rápida ajudará você a fazer sua primeira solicitação aos endpoints de Listas fixadas usando o [Postman](/pt-BR/tutorials/postman-getting-started).

Visite nosso repositório no GitHub com [código de exemplo da API do X v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code) se quiser ver exemplos em diferentes linguagens.

<Note>
  ### Pré-requisitos

  Para concluir este guia, você precisará de um conjunto de [chaves e tokens](/pt-BR/resources/fundamentals/authentication) para autenticar sua solicitação. Você pode gerar essas chaves e tokens seguindo estas etapas:

  - [Inscreva-se para uma conta de desenvolvedor](https://developer.x.com/en/apply-for-access) e aguarde a aprovação.
  - Crie um [Projeto](/pt-BR/resources/fundamentals/projects) e um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) associado no portal do desenvolvedor.
  - Acesse a página “Keys and tokens” do seu App para gerar as credenciais necessárias. Certifique-se de salvar todas as credenciais em um local seguro.
</Note>

<div id="authenticate-your-request">
  #### Autentique sua solicitação
</div>

Para fazer uma solicitação bem-sucedida aos endpoints de **lookup**, você pode usar [OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication), [OAuth 2.0 App-Only](/pt-BR/resources/fundamentals/authentication#oauth-2-0) ou [OAuth 2.0 Authorization Code with PKCE](pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). No entanto, com endpoints de **manage**, você só pode se autenticar com OAuth 1.0a User Context ou OAuth 2.0 Authorization Code with PKCE.

De qualquer forma, ao usar o Postman, as chaves e os tokens de autenticação padrão serão preenchidos automaticamente em suas solicitações, desde que você tenha configurado seu ambiente corretamente.

Você pode fazer isso selecionando o ambiente chamado “X API v2” (no canto superior direito do Postman) e adicionando suas chaves e tokens nos campos “initial value” e “current value” (clicando no ícone de olho ao lado do menu suspenso de ambientes). Essas chaves incluem o seguinte:

- API Key (também conhecida como Consumer Key)
- API Secret Key (também conhecida como Consumer Secret)
- OAuth 1.0a User Access Token
- OAuth 1.0a User Access Token Secret
- OAuth 2.0 App Access Token
- OAuth 2.0 Client Key (disponível somente se você configurou as definições de OAuth 2.0 User Authentication nas configurações do seu App)
- OAuth 2.0 Client Secret (disponível somente se você configurou as definições de OAuth 2.0 User Authentication nas configurações do seu App)
