<div id="authentication">
  #### Autenticação
</div>

Todos os endpoints da API do X v2 exigem que você autentique suas solicitações com um conjunto de credenciais, também conhecido como chaves e tokens. Você pode usar OAuth 1.0a no contexto do usuário para autenticar suas solicitações a este endpoint.

[OAuth 1.0a no contexto do usuário](/pt-BR/resources/fundamentals/authentication), o que significa que você deve usar um conjunto de chaves de API e tokens de acesso do usuário para fazer uma solicitação bem-sucedida. Os tokens de acesso devem estar associados ao usuário em nome de quem você está fazendo a solicitação. Se você quiser gerar um conjunto de tokens de acesso para outro usuário, ele deverá autorizar seu app usando o [fluxo OAuth de 3 etapas](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens).

Observe que o OAuth 1.0a pode ser difícil de usar. Se você não estiver familiarizado com esse método de autenticação, recomendamos usar uma [biblioteca](/pt-BR/x-api/tools-and-libraries/overview) ou uma ferramenta como o Postman.

<div id="developer-portal-projects-and-developer-apps">
  #### Portal do desenvolvedor, Projetos e Apps de desenvolvedor
</div>

Para obter um conjunto de credenciais de autenticação que funcione com os endpoints da API do X v2, você deve [criar uma conta de desenvolvedor](https://developer.x.com/en/portal/petition/essential/basic-info), configurar um [Projeto](/pt-BR/resources/fundamentals/projects) nessa conta e criar um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) nesse Projeto. Em seguida, você poderá encontrar suas chaves e tokens no seu App de desenvolvedor.
 

<div id="rate-limits">
  #### Limites de taxa
</div>

Todos os dias, muitos milhares de desenvolvedores fazem solicitações à API do X. Para ajudar a gerenciar o grande volume dessas solicitações, [limites de taxa](/pt-BR/x-api/fundamentals/rate-limits) são aplicados a cada endpoint, limitando o número de solicitações que você pode fazer em nome do seu app ou de um usuário autenticado.

Esses endpoints têm limites de taxa no nível do usuário, o que significa que o usuário autenticado em cujo nome você está fazendo a solicitação só pode chamar o endpoint um determinado número de vezes em qualquer App de desenvolvedor.

A tabela abaixo mostra os limites de taxa para cada endpoint.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Método HTTP** | **Limite de taxa** |
| /2/users/:id/pinned\_lists | POST | 50 solicitações a cada 15 minutos |
| /2/users/:id/pinned\_lists/:list\_id | DELETE | 50 solicitações a cada 15 minutos |
| /2/users/:id/pinned\_lists | GET | 15 solicitações a cada 15 minutos |

<div id="fields-and-expansions">
  #### Campos e expansões
</div>

O endpoint GET da API do X v2 permite selecionar exatamente quais dados retornar da API usando um conjunto de ferramentas chamadas `fields` e `expansions`. O parâmetro `expansions` permite expandir objetos referenciados no payload. Por exemplo, consultar Listas fixadas permite obter as seguintes [expansions](/pt-BR/x-api/fundamentals/expansions):

- `owner_id`

O parâmetro `fields` permite selecionar exatamente quais [fields](/pt-BR/x-api/fundamentals/fields) dentro dos diferentes objetos de dados você deseja receber. Este endpoint retorna principalmente objetos de usuário. Por padrão, o objeto List retorna os campos `id` e `name`. Para receber campos adicionais, como `list.created_at` ou `list.description`, você precisará solicitá-los especificamente usando um parâmetro de fields.

Adicionamos um guia sobre como usar [fields e expansions](/pt-BR/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) em conjunto em nosso [dicionário de dados da API do X v2](/pt-BR/x-api/fundamentals/data-dictionary).

O quadro abaixo mostra os fields e expansions disponíveis para o endpoint de consulta:

| :--- | :--- | :--- |
| **Endpoint** | **Fields** | **Expansions** |
| /2/users/:id/pinned\_lists | `list.fields`<br /><br />`user.fields` | `owner_id` |
