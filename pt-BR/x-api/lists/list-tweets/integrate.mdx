<div id="authentication">
  #### Autenticação
</div>

Todos os endpoints da API do X v2 exigem que você autentique suas solicitações com um conjunto de credenciais, também conhecido como chaves e tokens. Você pode usar OAuth 1.0a (User Context), apenas App ou OAuth 2.0 Authorization Code com PKCE para autenticar suas solicitações a este endpoint.

[OAuth 1.0a (User Context)](/pt-BR/resources/fundamentals/authentication#oauth-1-0a-2), o que significa que você deve usar um conjunto de API Keys e Access Tokens de usuário para fazer uma solicitação com êxito. Os access tokens devem estar associados ao usuário em cujo nome você está fazendo a solicitação. Se você quiser gerar um conjunto de Access Tokens para outro usuário, ele deverá autorizar seu App usando o [fluxo OAuth de 3 partes (3-legged)](/pt-BR/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow).

Observe que o OAuth 1.0a pode ser difícil de usar. Se você não estiver familiarizado com esse método de autenticação, recomendamos usar uma [biblioteca](/pt-BR/x-api/tools-and-libraries/overview), uma ferramenta como o Postman ou então OAuth 2.0 ou apenas App para autenticar suas solicitações.

[OAuth 2.0 Authorization Code com PKCE](/pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) permite maior controle sobre o escopo de uma aplicação e fluxos de autorização em vários dispositivos. O OAuth 2.0 permite escolher escopos específicos e granulares que concedem permissões precisas em nome de um usuário.

Para habilitar o OAuth 2.0 no seu App, ative-o nas configurações de autenticação do seu App, localizadas na seção de configurações do App no portal do desenvolvedor.

[Somente App](/pt-BR/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) exige apenas que você envie um [Access Token Somente App](/pt-BR/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) com sua solicitação. Você pode gerar um Access Token Somente App diretamente em um App de desenvolvedor ou gerar um usando o endpoint [POST oauth2/token](/pt-BR/resources/fundamentals/authentication#post-oauth2-token).

<div id="developer-portal-projects-and-developer-apps">
  #### Portal do desenvolvedor, Projetos e Apps de desenvolvedor
</div>

Para obter um conjunto de credenciais de autenticação que funcione com os endpoints da API do X v2, você deve [inscrever-se em uma conta de desenvolvedor](https://developer.x.com/en/portal/petition/essential/basic-info), configurar um [Projeto](/pt-BR/resources/fundamentals/projects) nessa conta e criar um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) nesse Projeto. Em seguida, você poderá encontrar suas chaves e tokens no seu App de desenvolvedor.
 

<div id="rate-limits">
  #### Limites de taxa
</div>

Todos os dias, muitos milhares de desenvolvedores fazem solicitações à API do X. Para ajudar a gerenciar o grande volume dessas solicitações, [limites de taxa](https://developer.x.com/x-api/fundamentals/rate-limits) são aplicados a cada endpoint, limitando o número de solicitações que você pode fazer em nome do seu app ou de um usuário autenticado.

Este endpoint tem limite de taxa tanto no nível do app quanto no nível do usuário. O limite de taxa do app significa que você, desenvolvedor, só pode fazer um certo número de solicitações a este endpoint em um determinado período, a partir de qualquer app (com base no uso de API Key e API Secret Key ou do Bearer Token). O limite de taxa do usuário significa que o usuário autenticado em cujo nome você está fazendo a solicitação só pode realizar uma consulta de Posts da Lista um certo número de vezes, em qualquer app de desenvolvedor.

A tabela abaixo mostra os limites de taxa para cada endpoint.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Método HTTP** | **Limite de taxa** |
| /2/lists/:id/tweets | GET | 900 solicitações por 15 minutos |

Fields and expansions

O endpoint GET da API do X v2 permite que os usuários selecionem exatamente quais dados desejam obter da API usando um conjunto de recursos chamados `fields` e `expansions`. O parâmetro `expansions` permite expandir objetos referenciados no payload. Por exemplo, consultar Posts da Lista permite extrair as seguintes [expansions](/pt-BR/x-api/fundamentals/expansions):

- `author_id`

O parâmetro `fields` permite selecionar exatamente quais [fields](/pt-BR/x-api/fundamentals/fields) dentro dos diferentes objetos de dados você deseja receber. Este endpoint retorna principalmente objetos Post. Por padrão, o objeto Post retorna os campos `id` e `text`. Para receber campos adicionais, como `tweet.created_at` ou `tweet.lang`, você deve solicitá-los especificamente usando um parâmetro fields.

Adicionamos um guia sobre como usar [fields e expansions](/pt-BR/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) em conjunto no nosso [dicionário de dados da API do X v2](/pt-BR/x-api/fundamentals/data-dictionary).

A tabela abaixo mostra os fields e expansions disponíveis para o endpoint de consulta:

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Fields** | **Expansions** |
| /2/lists/:id/tweets | `tweet.fields`<br /><br />`user.fields` | `author_id` |

Pagination

Consultar Posts da Lista pode retornar muitos dados. Para garantir resultados consistentes e de alto desempenho a qualquer momento, usamos paginação. Paginação é um recurso em endpoints da API do X v2 que retornam mais resultados do que cabem em uma única resposta. Quando isso acontece, os dados são retornados em uma série de “páginas”. Saiba mais sobre como [paginar pelos resultados.](/pt-BR/x-api/fundamentals/pagination)
