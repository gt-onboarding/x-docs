<div id="list-posts-lookup-standard-v11-compared-to-x-api-v2">
  ### Consulta de Posts de Lista: v1.1 padrão em comparação com a API do X v2
</div>

Se você vem trabalhando com o endpoint v1.1 padrão [GET lists/statuses](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-statuses), o objetivo deste guia é ajudar você a entender as semelhanças e diferenças entre os endpoints v1.1 padrão e API do X v2.

- **Semelhanças**
  - Métodos de autenticação
  - Limites de requisição
- **Diferenças**
  - URLs dos endpoints
  - Requisitos de Apps e Projetos
  - Limites de objetos de dados por requisição
  - Formatos de dados de resposta
  - Parâmetros de requisição

<div id="similarities">
  #### Semelhanças
</div>

**Autenticação**

Ambas as versões do endpoint oferecem suporte a [OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication#oauth-2-0). Portanto, se você usava anteriormente um dos endpoints padrão de consulta de posts de Lista da v1.1, pode continuar usando o mesmo método de autenticação ao migrar para a API do X v2.

Dependendo da sua biblioteca/pacote de autenticação preferido, a autenticação somente com App provavelmente é a maneira mais simples de começar e pode ser configurada com um cabeçalho de requisição. Para saber como gerar um token de acesso de App only, consulte [este guia de App only](/pt-BR/resources/fundamentals/authentication#bearer-token-also-known-as-app-only).

**Limites de taxa**

|     |     |
| :--- | :--- |
| **Standard v1.1** | **API do X v2** |
| /1.1/lists/statuses.json<br /><br />900 solicitações por janela de 15 minutos com OAuth 1.0a User Context<br /><br />900 solicitações por janela de 15 minutos com App only | /2/lists/:id/tweets<br /><br />900 solicitações por janela de 15 minutos com OAuth 1.0a User Context<br /><br />900 solicitações por janela de 15 minutos com OAuth 2.0 Authorization Code with PKCE<br /><br />900 solicitações por janela de 15 minutos com App only |

<div id="differences">
  #### Diferenças
</div>

**URLs de endpoints**

- Endpoints padrão v1.1:
  - GET https://api.x.com/1.1/lists/statuses.json
    (Buscar posts de uma Lista especificada)
- Endpoint da API do X v2:
  - GET https://api.x.com/2/lists/:id/tweets
    (Buscar posts de uma Lista especificada)

**Requisitos de App e Projeto**

Os endpoints da API do X v2 exigem que você use credenciais de um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) associado a um [Projeto](/pt-BR/resources/fundamentals/projects) ao autenticar suas solicitações. Todos os endpoints da API do X v1.1 podem usar credenciais de Apps independentes ou de Apps associados a um projeto.

**Limites de objetos de dados por solicitação**

O endpoint padrão v1.1 /lists/statuses permite retornar até 5000 posts por solicitação. Os novos endpoints v2 permitem retornar até 100 posts por solicitação. Por padrão, 100 objetos de usuário serão retornados; para alterar o número de resultados, você precisará passar o parâmetro de consulta max\_results= com um número entre 1 e 100; em seguida, você pode passar o next\_token retornado no payload da resposta para o parâmetro de consulta pagination\_token na sua próxima solicitação.

**Formato de dados da resposta**

Uma das maiores diferenças entre as versões de endpoint padrão v1.1 e da API do X v2 é como você seleciona quais campos retornar no seu payload.

Nos endpoints padrão, muitos campos de resposta são retornados por padrão e, além disso, você tem a opção de usar parâmetros para indicar quais campos ou conjuntos de campos adicionais devem retornar no payload.

A versão da API do X v2 retorna apenas os campos id e text do post por padrão. Para solicitar quaisquer campos ou objetos adicionais, você precisará usar os parâmetros [fields](/pt-BR/x-api/fundamentals/fields) e [expansions](/pt-BR/x-api/fundamentals/expansions). Quaisquer campos de post solicitados deste endpoint serão retornados no objeto principal de post. Quaisquer campos de objetos expandidos serão retornados em um objeto includes dentro da sua resposta. Você pode então associar quaisquer objetos expandidos ao objeto principal de post combinando os IDs do objeto principal e dos objetos expandidos. 

Aqui estão exemplos de possíveis campos e expansões de post:

- attachments

- author\_id

- context\_annotations

- created\_at

- geo

- lang

|     |     |
| :--- | :--- |
| **Endpoint** | **Expansão** |
| /2/lists/:id/tweets | author\_id |

Recomendamos que você leia mais sobre esses novos parâmetros em seus respectivos guias ou em nosso guia sobre [como usar fields e expansions](/pt-BR/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions). 

Também preparamos um [guia de migração de formato de dados](/pt-BR/x-api/migrate/data-format-migration) que pode ajudar você a mapear campos padrão v1.1 para os campos mais novos da v2. Esse guia também fornecerá o parâmetro específico de expansão e de campo que você precisará passar com sua solicitação v2 para retornar campos específicos. 

Além das mudanças em como você solicita certos campos, a API do X v2 também está introduzindo novos designs de JSON para os objetos retornados pelas APIs, incluindo objetos de [post](/pt-BR/x-api/fundamentals/data-dictionary#tweet) e de [usuário](/pt-BR/x-api/fundamentals/data-dictionary#user).

- No nível raiz do JSON, os endpoints padrão retornam objetos de post em um array **statuses**, enquanto a API do X v2 retorna um array **data**. 

- Em vez de se referir a Retweeted e Quoted "statuses", o JSON da API do X v2 se refere a Retweeted e Quoted Tweets. Muitos campos legados e descontinuados, como **contributors** e **user.translator\_type**, estão sendo removidos. 

- Em vez de usar tanto **favorites** (no objeto de post) quanto **favourites** (no objeto de usuário), a API do X v2 usa o termo **like**.

- O X está adotando a convenção de que valores JSON sem valor (por exemplo, **null**) não são escritos no payload. Atributos de post e de usuário só são incluídos se tiverem valores não nulos.
   

**Parâmetros de solicitação**

Os seguintes parâmetros de solicitação padrão v1.1 têm equivalentes na API do X v2:

|     |     |
| :--- | :--- |
| Padrão v1.1 | API do X v2 |
| list\_id | id  |
| slug | Sem equivalente |
| owner\_screen\_name | Sem equivalente |
| owner\_id | Solicitado com o parâmetro expansions com valor author\_id |
| since\_id | Sem equivalente |
| max\_id | Sem equivalente |
| include\_entities | Solicitado com o parâmetro tweet.fields com valor entities |
| include\_rts | Sem equivalente |
| count | max\_results |
