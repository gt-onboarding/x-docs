<div id="steps-to-build-a-list-members-lookup-request">
  ### Etapas para criar uma solicitação de busca de membros de uma Lista
</div>

**Etapa um: Escolha a coleção de endpoints de Lista no Postman**

Depois de carregar a coleção da API do X v2 no Postman, navegue até a pasta “List”, selecione a pasta “List members” e, em seguida, escolha “Members lookup”.
 

**Etapa dois: Identifique e especifique de qual Lista você deseja recuperar membros**

Você deve especificar a Lista da qual deseja obter os membros. Você pode encontrar o ID da Lista acessando x.com, clicando em uma Lista e olhando a URL. Por exemplo, o ID da Lista da URL a seguir é 84839422.

https://x.com/i/lists/84839422

O ID de destino pode ser qualquer ID de Lista válido. No Postman, vá até a guia “Params” e insira seu ID na coluna “Value” da variável de caminho id. Certifique-se de não incluir espaços antes ou depois do ID.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 84839422 (List ID) |

**Etapa três: Identifique e especifique quais campos você deseja recuperar**

Se você clicar no botão “Send” após a etapa três, receberá na resposta os campos padrão do [objeto de usuário](/pt-BR/x-api/fundamentals/data-dictionary#user): id, name e username.

Se quiser receber campos adicionais além de id, name e username, será necessário especificá-los na solicitação usando os parâmetros [field](/pt-BR/x-api/fundamentals/fields) e/ou [expansion](/pt-BR/x-api/fundamentals/expansions).

Para este exercício, solicitaremos três conjuntos adicionais de campos de objetos diferentes:

- O campo adicional user.created\_at nos objetos de usuário primários.
- O [objeto de Post](/pt-BR/x-api/fundamentals/data-dictionary#tweet) completo usando o parâmetro expansion.
- O campo adicional tweet.created\_at nos objetos de Post associados.

No Postman, navegue até a guia “Params” e adicione o seguinte par key:value à tabela “Query Params”:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| user.fields | created\_at | `created_at` |
| expansions | pinned\_tweet\_id | `includes.tweets.id,  <br/>includes.tweets.text  <br/>` |
| tweet.fields | created\_at | `includes.tweets.created_at` |

Agora você deve ver uma URL semelhante ao lado do botão “Send”:

`https://api.x.com/2/lists/84839422/members?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=created_at`

**Etapa quatro: Faça sua solicitação e revise sua resposta**

Depois que tudo estiver configurado, clique no botão “Send” e você receberá uma resposta semelhante ao exemplo a seguir:
