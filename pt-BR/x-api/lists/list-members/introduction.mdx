<div id="list-members-lookup">
  ### Consulta de membros de Lista
</div>

O grupo de consulta de membros possui dois endpoints disponíveis. Você pode obter detalhes sobre os membros de uma Lista específica e ver de quais Listas um usuário faz parte. Esses endpoints podem ser usados para ajudar as pessoas a criar e organizar novas Listas com base nas informações de participação.

Há um limite de 900 solicitações a cada 15 minutos ao consultar detalhes de membros e um limite de 75 solicitações a cada 15 minutos ao consultar as participações de usuários.

Você pode usar [OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication), [App only](https://developer.x.com/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) ou [OAuth 2.0 Authorization Code with PKCE](pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) para autenticar suas solicitações a este endpoint. 
 

<div id="manage-list-members">
  ### Gerenciar membros da Lista
</div>

Os endpoints para gerenciar membros de uma Lista permitem adicionar e remover membros de uma Lista em nome de um usuário autenticado. Para esses endpoints, há dois métodos disponíveis: POST e DELETE. O método POST permite adicionar um membro a uma Lista, e o método DELETE permite remover um membro de uma Lista. Há um limite por usuário de 300 solicitações a cada 15 minutos para ambos os endpoints.

Observe que as Listas não podem ter mais de 5.000 membros.

Como você fará solicitações em nome de um usuário com esses endpoints, é necessário autenticá-lo com [OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication) ou [OAuth 2.0 Authorization Code with PKCE](pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) e usar os tokens de acesso do usuário associados a um usuário que autorizou seu App, os quais podem ser gerados usando o [fluxo OAuth de 3 etapas](/pt-BR/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) ou o [fluxo de concessão Authorization Code with PKCE](/pt-BR/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0).

<Note>
