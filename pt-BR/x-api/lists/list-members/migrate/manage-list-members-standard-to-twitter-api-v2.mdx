<div id="manage-list-members-standard-v11-compared-to-x-api-v2">
  ### Gerenciar membros de listas: padrão v1.1 comparado à API do X v2
</div>

Se você vem trabalhando com os endpoints padrão v1.1 [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create) e [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy), o objetivo deste guia é ajudar você a entender as semelhanças e diferenças entre os endpoints de gerenciamento de membros de listas do padrão v1.1 e da API do X v2.

- **Semelhanças**
  - Autenticação
- **Diferenças**
  - URLs de endpoint
  - Requisitos de App e Projeto
  - Métodos HTTP
  - Limites de requisições
  - Parâmetros de requisição

<div id="similarities">
  #### Semelhanças
</div>

**Autenticação**

Ambas as versões do endpoint oferecem suporte a [OAuth 1.0a User Context](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Portanto, se você anteriormente utilizava um dos endpoints padrão v1.1 para gerenciar membros de listas, poderá continuar usando o mesmo método de autenticação ao migrar para a versão da API do X v2.

<div id="differences">
  #### Diferenças
</div>

**URLs de endpoints**

- Endpoints do Standard v1.1:
  - POST https://api.x.com/1.1/lists/members/create.json
    (Adiciona um membro a uma Lista especificada)
  - POST https://api.x.com/1.1/lists/members/destroy.json
    (Remove um membro de uma Lista especificada)
- Endpoint da API do X v2:
  - POST https://api.x.com/2/lists/:id/members
    (Adiciona um membro a uma Lista especificada)

  - DELETE https://api.x.com/2/lists/:id/members/:user\_id
    (Remove um membro de uma Lista especificada)

**Limites de taxa**

| **Standard v1.1** | **API do X v2** |
| :--- | :--- |
| /1.1/lists/members/create.json<br /><br />nenhum | /2/lists/:id/members<br /><br />300 solicitações por janela de 15 minutos com OAuth 1.0a (User Context)<br /><br />300 solicitações por janela de 15 minutos com OAuth 2.0 (Authorization Code com PKCE) |
| /1.1/lists/members/destroy.json<br /><br />nenhum | /2/lists/:id/members/:user\_id<br /><br />300 solicitações por janela de 15 minutos com OAuth 1.0a (User Context)<br /><br />300 solicitações por janela de 15 minutos com OAuth 2.0 (Authorization Code com PKCE) |

**Requisitos de App e Projeto**

Os endpoints da API do X v2 exigem o uso de credenciais de um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) associado a um [Projeto](/pt-BR/resources/fundamentals/projects) ao autenticar suas solicitações. Todos os endpoints da API do X v1.1 podem usar credenciais de Apps independentes ou Apps associados a um projeto.

**Parâmetros de solicitação**

Os seguintes parâmetros de solicitação do Standard v1.1 têm equivalentes na API do X v2:

| **Standard v1.1** | **API do X v2** |
| :--- | :--- |
| list\_id | id  |
| slug | Sem equivalente |
| screen\_name | Sem equivalente |
| owner\_screen\_name | Sem equivalente |
| owner\_id | Sem equivalente |
