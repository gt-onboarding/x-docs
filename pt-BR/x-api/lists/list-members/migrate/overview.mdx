<div id="comparing-x-apis-list-members-endpoints">
  ## Comparando os endpoints de membros de listas da API do X
</div>

O conjunto de endpoints de membros de listas da v2 substituirá os endpoints padrão da v1.1 [GET lists/members](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-members), [GET lists/memberships](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-memberships), [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create) e [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy). Se você tem código, apps ou ferramentas que usam uma dessas versões dos endpoints de membros de listas e está considerando migrar para o endpoint mais recente da API do X v2, este conjunto de guias é para você. 

<div id="list-members-lookup">
  ### Pesquisa de membros de Lista
</div>

Os endpoints v2 de pesquisa de membros de Lista substituirão os endpoints padrão [GET lists/members](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-members) e [GET lists/memberships](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-memberships).

As tabelas a seguir comparam os endpoints padrão v1.1 e da API do X v2 para Listas:

**Pesquisa de membro de Lista**

|     |     |     |
| :--- | :--- | :--- |
| Descrição | Padrão v1.1 | API do X v2 |
| Métodos HTTP compatíveis | `GET` | `GET` |
| Domínio do host | `https://api.x.com` | `https://api.x.com` |
| Caminho do endpoint | `/1.1/lists/members.json` | `/2/lists/:id/members` |
| [Authentication](/pt-BR/resources/fundamentals/authentication) | OAuth 1.0a Contexto do Usuário<br /><br />Somente app | OAuth 1.0a Contexto do Usuário<br /><br />OAuth 2.0 Authorization Code com PKCE<br /><br />Somente app |
| [Limites de taxa](/pt-BR/resources/fundamentals/rate-limits) padrão de requisição | 900 solicitações por 15 min com OAuth 1.0a<br /><br />75 solicitações por 15 min com Somente app | 900 solicitações por 15 min com OAuth 1.0a<br /><br />900 solicitações por 15 min com OAuth 2.0<br /><br />900 solicitações por 15 min com Somente app |

**Pesquisa de associações de Lista**

|     |     |     |
| :--- | :--- | :--- |
| Descrição | Padrão v1.1 | API do X v2 |
| Métodos HTTP compatíveis | `GET` | `GET` |
| Domínio do host | `https://api.x.com` | `https://api.x.com` |
| Caminho do endpoint | `/1.1/lists/memberships.json` | `/2/users/:id/list_memberships` |
| [Authentication](/pt-BR/resources/fundamentals/authentication) | OAuth 1.0a Contexto do Usuário<br /><br />Somente app | OAuth 1.0a Contexto do Usuário<br /><br />OAuth 2.0 Authorization Code com PKCE<br /><br />Somente app |
| [Limites de taxa](/pt-BR/resources/fundamentals/rate-limits) padrão de requisição | 75 solicitações por 15 min com OAuth 1.0a<br /><br />75 solicitações por 15 min com Somente app | 75 solicitações por 15 min com OAuth 1.0a<br /><br />75 solicitações por 15 min com OAuth 2.0<br /><br />75 solicitações por 15 min com Somente app |

<div id="manage-list-members">
  ### Gerenciar membros da lista
</div>

Os endpoints v2 para gerenciar membros da lista substituirão os endpoints padrão [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create) e [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy).

As tabelas a seguir comparam os endpoints de listas do v1.1 padrão e da API do X v2:

**Adicionar membro**

| Descrição | v1.1 padrão | API do X v2 |
| :--- | :--- | :--- |
| Métodos HTTP compatíveis | POST | POST |
| Domínio do host | https://api.x.com | https://api.x.com |
| Caminho do endpoint | /1.1/lists/members/create.json | /2/lists/:id/members |
| [Autenticação](/pt-BR/resources/fundamentals/authentication) | Contexto do usuário OAuth 1.0a | Contexto do usuário OAuth 1.0a<br /><br />Código de Autorização OAuth 2.0 com PKCE |
| [Limites de taxa](/pt-BR/resources/fundamentals/rate-limits) padrão da solicitação | Nenhum | 300 solicitações por 15 min (por usuário) |

**Remover membro**

| Descrição | v1.1 padrão | API do X v2 |
| :--- | :--- | :--- |
| Métodos HTTP compatíveis | POST | DELETE |
| Domínio do host | https://api.x.com | https://api.x.com |
| Caminho do endpoint | /1.1/lists/members/destroy.json | /2/lists/:id/:user\_id |
| [Autenticação](/pt-BR/resources/fundamentals/authentication) | Contexto do usuário OAuth 1.0a | Contexto do usuário OAuth 1.0a<br /><br />Código de Autorização OAuth 2.0 com PKCE |
| [Limites de taxa](/pt-BR/resources/fundamentals/rate-limits) padrão da solicitação | Nenhum | 300 solicitações por 15 min (por usuário) |

Para acessar os endpoints da API do X v2, você deve [se inscrever para uma conta de desenvolvedor](https://developer.x.com/en/portal/petition/essential/basic-info). Ao autenticar, use as chaves e os tokens de um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) que esteja dentro de um [Projeto](/pt-BR/resources/fundamentals/projects).

Saiba mais sobre como obter acesso aos endpoints da API do X v2 na nossa página de [introdução](/pt-BR/x-api/getting-started/getting-access).
