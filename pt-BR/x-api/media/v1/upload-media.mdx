title: Fazer upload de mídia

<div id="overview">
  ## Visão geral
</div>

Um objeto de mídia representa uma única foto, vídeo ou GIF animado. Objetos de mídia são usados por muitos endpoints na API do X e podem ser incluídos em posts, Mensagens Diretas, perfis de usuário, peças criativas de publicidade e outros contextos. Cada objeto de mídia pode ter várias variantes de exibição ou reprodução, com resoluções ou formatos diferentes.

<div id="media-types-size-restrictions">
  ## Tipos de mídia e restrições de tamanho
</div>

Restrições de tamanho para upload via API

- Imagem: 5 MB
- GIF: 15 MB
- Vídeo: 512 MB (ao usar media_category=amplify)

<div id="creation">
  ## Criação
</div>

Objetos como Tweets, Mensagens Diretas, fotos de perfil de usuário, cards de Anúncios hospedados etc. podem conter um ou mais objetos de mídia. Esses objetos de nível superior são coletivamente conhecidos como entidades. A API relevante de criação de entidades (por exemplo, POST statuses/update) pode receber um ou mais objetos de mídia usando um media_id exclusivo.

Uma entidade que contém objeto(s) de mídia pode ser criada seguindo estas etapas:

1. Envie o(s) arquivo(s) de mídia usando o upload segmentado recomendado (imagens/GIF/vídeo) ou o upload simples mais antigo (apenas imagens).
2. Receba um media_id da etapa 1. Esta etapa pode ser repetida várias vezes com mídias diferentes se a entidade permitir que vários parâmetros media_id sejam passados.
3. Crie a entidade chamando o endpoint apropriado, incluindo o media_id e outros parâmetros obrigatórios. Por exemplo, anexe um media_id a um Tweet usando o endpoint POST statuses/update.

<div id="retrieving">
  ## Recuperando
</div>

Consulte o objeto de mídia no dicionário de dados de Tweets.
