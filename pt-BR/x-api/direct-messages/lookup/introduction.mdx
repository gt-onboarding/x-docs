Este lançamento inicial da pesquisa de Mensagens Diretas inclui três métodos GET:

- **GET /2/dm\_conversations/with/:participant\_id/dm\_events** - Recupera eventos de Mensagem Direta associados a uma conversa um para um. O parâmetro de caminho :participant\_id é o ID de usuário da conta que está conversando com o usuário autenticado que faz esta solicitação.
- **GET /2/dm\_conversations/:dm\_conversation\_id/dm\_events** - Recupera eventos de Mensagem Direta associados a um ID de conversa específico, conforme indicado pelo parâmetro de caminho :dm\_conversation\_id.
- **GET /2/dm\_events** - Recupera eventos de Mensagem Direta associados a um usuário, incluindo conversas um para um e em grupo. Eventos de até 30 dias atrás estão disponíveis.

Observe que os IDs de eventos de Mensagem Direta são comuns entre as versões v1.1 e v2 (assim como no X App), portanto, o método v1.1 para listar um único evento pode ser usado junto com esses novos endpoints v2. Observe também que as APIs Enterprise e Premium Account Activity têm suporte a mensagens um para um da v2, mas ainda não oferecem suporte a conversas em grupo.

Com este lançamento, três tipos de evento são compatíveis, e esses endpoints aceitam parâmetros de consulta para filtrá-los:

- **MessageCreate** - Uma mensagem foi criada.
- **ParticipantsJoin** - Um novo participante entrou em uma conversa.
- **ParticipantsLeave** - Um participante saiu de uma conversa.

Há um limite por usuário de 300 solicitações a cada 15 minutos para os métodos GET. Esse limite é compartilhado entre esses endpoints GET.

Como você está fazendo solicitações em nome de um usuário com os endpoints de Mensagens Diretas v2, é necessário autenticar com [OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication) ou [OAuth 2.0 Authorization Code with PKCE](/pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3), usando tokens de acesso associados a usuários que autorizaram seu X App. Para gerar esses tokens de acesso com OAuth 1.0a, você pode usar o [fluxo OAuth de 3 etapas](/pt-BR/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow). Para gerar tokens de acesso de usuário com OAuth 2.0, você pode usar o [fluxo de concessão Authorization Code com PKCE](/pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

**Configuração da conta**

Para acessar esses endpoints, você precisará de:

- Uma [conta de desenvolvedor](https://developer.x.com/en/portal/petition/essential/basic-info) aprovada.
- Autenticar usando as chaves e os tokens de um [developer App](/pt-BR/resources/fundamentals/developer-apps) que esteja dentro de um [Project](/pt-BR/resources/fundamentals/projects).

Saiba mais sobre como obter acesso aos endpoints da API do X v2 em nosso [guia de primeiros passos](/pt-BR/x-api/getting-started/getting-access).
