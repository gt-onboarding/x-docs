Os endpoints para gerenciar bloqueios de MD permitem bloquear ou desbloquear uma conta específica em nome de um usuário autenticado. Para esses endpoints, há dois métodos POST disponíveis:

- **/2/users/:id/dm/block**: Permite bloquear uma conta
- **/2/users/:id/dm/unblock**: Permite desbloquear uma conta

<div id="getting-started">
  ### Primeiros passos
</div>

<div id="authentication">
  ### Autenticação
</div>

Como você está fazendo solicitações em nome de um usuário, é necessário autenticar esses endpoints com [OAuth 1.0a (User Context)](/pt-BR/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) ou [OAuth 2.0 Authorization Code com PKCE](/pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) e usar os tokens de acesso do usuário associados ao usuário em cujo nome você está fazendo a solicitação. Você pode gerar esse token de acesso do usuário usando o [fluxo OAuth de 3 etapas](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens) (OAuth 1.0a) ou o [fluxo de concessão Authorization Code com PKCE](/pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) (OAuth 2.0).

<div id="making-a-request">
  ### Fazendo uma solicitação
</div>

Bloquear

Depois que um usuário autenticar no seu app, você pode chamar o endpoint Block em nome desse usuário, conforme mostrado abaixo:

```
      curl --request POST 'https://api.x.com/2/users/:id/dm/block' --header 'Authorization: ••••••'
```

Se a solicitação for bem-sucedida, você verá a resposta JSON mostrada abaixo:

```
{
   "data": {
       "blocked": true
   }
}
```

**Desbloquear**

Depois que um usuário autenticar no seu app, você pode chamar o endpoint Unblock em nome desse usuário, conforme mostrado abaixo:

```
      curl --request POST 'https://api.x.com/2/users/:id/dm/unblock' --header 'Authorization: ••••••'
```

Se a solicitação for bem-sucedida, você verá a resposta JSON mostrada abaixo:

```
{
   "data": {
       "blocked": false
   }
}
```
