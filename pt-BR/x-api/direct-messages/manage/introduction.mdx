Mensagens Diretas permitem conversas privadas no X. Elas estão entre os recursos mais populares do X, com uma ampla variedade de casos de uso — de bate‑papos em grupo entre amigos ao suporte ao cliente para marcas no mundo todo. Novas versões v2 dos endpoints de Mensagens Diretas serão lançadas em etapas, e esta primeira inclui endpoints fundamentais para criar Mensagens Diretas e listar eventos de conversas. Pela primeira vez, a API do X v2 oferece suporte a conversas em grupo.

Este lançamento inicial de Gerenciar Mensagens Diretas inclui três métodos POST para criar Mensagens Diretas:

- **POST /2/dm\_conversations/with/:participant\_id/messages** - Cria uma Mensagem Direta individual. Esse método cria uma nova conversa 1‑1 ou recupera a conversa atual e adiciona a Mensagem Direta a ela. O parâmetro de caminho :participant\_id é o ID do usuário da conta que receberá a mensagem.
- **POST /2/dm\_conversations** - Cria uma nova conversa em grupo e adiciona uma Mensagem Direta a ela. Essas solicitações exigem uma lista de participantes da conversa. Observe que é possível criar várias conversas com a mesma lista de participantes. Essas solicitações sempre retornarão um novo ID de conversa.
- **POST /2/dm\_conversations/:dm\_conversation\_id/messages** - Cria uma Mensagem Direta e a adiciona a uma conversa existente. O parâmetro de caminho :dm\_conversation\_id é o ID da conversa à qual a mensagem será adicionada.

Observe que os IDs de eventos de Mensagens Diretas são comuns entre as versões v1.1 e v2 (bem como no app do X), portanto, os métodos v1.1 para ocultar/excluir Mensagens Diretas podem ser usados juntamente com este novo endpoint v2. Observe também que as APIs Enterprise e Premium Account Activity oferecem suporte a mensagens 1‑1 da v2, mas ainda não dão suporte a conversas em grupo.

Há um limite de 200 solicitações por 15 minutos por usuário para o método POST. Há também um limite de 1.000 solicitações por 24 horas por usuário. Além disso, há um limite de 15.000 solicitações por 24 horas. Esses limites são compartilhados entre esses endpoints POST.

Como você está fazendo solicitações em nome de um usuário com os endpoints de gerenciamento de Posts, é necessário autenticar usando [OAuth 1.0a User Context](https://developer-staging.x.com/resources/fundamentals/authentication) ou [OAuth 2.0 Authorization Code with PKCE](pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) e usar tokens de acesso de um usuário que autorizou seu App. Para gerar esse token de acesso de usuário com OAuth 1.0a, use o [fluxo OAuth de 3 etapas](/pt-BR/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token). Para gerar um token de acesso de usuário com OAuth 2.0, use o [fluxo de concessão Authorization Code with PKCE](/pt-BR/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)).

<Note>
  **Configuração da conta**

  Para acessar esses endpoints, você precisará de:

  - Uma [conta de desenvolvedor](https://developer.x.com/en/portal/petition/essential/basic-info) aprovada.
  - Autenticar usando as chaves e tokens de um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) que esteja localizado dentro de um [Projeto](/pt-BR/resources/fundamentals/projects).

  Saiba mais sobre como obter acesso aos endpoints da API do X v2 em nosso [guia de primeiros passos](/pt-BR/x-api/getting-started/getting-access).
</Note>
