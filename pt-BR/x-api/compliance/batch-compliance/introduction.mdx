X está comprometido com nossa comunidade de desenvolvedores que constroem com a API do X. Como parte desse compromisso, buscamos tornar nossa API aberta e justa para desenvolvedores, segura para as pessoas no X e benéfica para a plataforma X como um todo. É crucial que qualquer desenvolvedor que armazene conteúdo do X offline garanta que os dados reflitam a intenção do usuário e o estado atual do conteúdo no X. Por exemplo, quando alguém no X exclui um Post ou sua conta, protege seus Posts ou remove as informações de geolocalização de seus Posts, é essencial tanto para o X quanto para nossos desenvolvedores respeitar as expectativas e a intenção dessa pessoa. Os endpoints de conformidade em lote fornecem aos desenvolvedores uma ferramenta simples para ajudar a manter os dados do X em conformidade com o [Acordo e Política do Desenvolvedor do X](https://developer.x.com/en/developer-terms/policy). 

Esses endpoints de conformidade em lote permitem que você envie grandes conjuntos de IDs de Post ou de usuários para recuperar seu status de conformidade, a fim de determinar quais dados exigem ação para colocar seus conjuntos de dados em conformidade. Observe que o uso dos endpoints de conformidade em lote é restrito aos casos de uso mencionados anteriormente, e qualquer outro propósito é proibido e pode resultar em ação de fiscalização.

Normalmente, há 4 etapas envolvidas ao trabalhar com este endpoint:

1. **Criar uma tarefa de conformidade**
   Você pode especificar o tipo de tarefa (com o valor tweets ou users) para indicar se o conjunto de dados que deseja enviar contém IDs de Post ou IDs de usuário. Você pode ter uma tarefa concorrente por tipo de tarefa a qualquer momento.
2. **Fazer upload do seu conjunto de dados para o upload\_url**
   Em seguida, você faz upload do seu conjunto de dados como um arquivo de texto simples para o upload\_url fornecido, com cada linha do arquivo contendo um único ID de Post ou ID de usuário. O upload\_url expira após 15 minutos.
3. **(Opcional) Verificar o status da tarefa**
   Você pode verificar o status da sua tarefa de conformidade para ver se ela está created, in\_progress, failed ou complete.
4. **Baixar os resultados**
   Quando sua tarefa for concluída, você poderá baixar os resultados usando o download\_url. O download\_url expira após uma semana (a partir da criação da tarefa).

   Esse resultado conterá um conjunto de objetos JSON (um objeto por linha). Cada objeto conterá um ID do Post, a data de criação do Post (útil para localizar Posts organizados por data), a ação necessária, o motivo da ação de conformidade e a data em que o usuário foi suspenso.

Você receberá os seguintes tipos de eventos de conformidade em seus resultados:

- deleted - indica que o Post ou a conta do Usuário foi excluída
- deactivated - indica que o Post ou a conta do Usuário foi desativada
- scrub\_geo - indica que as informações de geolocalização associadas ao Post ou ao Usuário foram removidas
- protected - indica que a conta que fez o Post se tornou privada
- suspended - indica que a conta que fez o Post foi suspensa

**Configuração de conta**

Para acessar esses endpoints, você precisará:

- Uma [conta de desenvolvedor](https://developer.x.com/en/portal/petition/essential/basic-info) aprovada.
- Autenticar-se usando as chaves e tokens de um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) que está localizado dentro de um [Projeto](/pt-BR/resources/fundamentals/projects). 

Saiba mais sobre como obter acesso aos endpoints da API do X v2 em nosso [guia de introdução](/pt-BR/x-api/getting-started/getting-access).
