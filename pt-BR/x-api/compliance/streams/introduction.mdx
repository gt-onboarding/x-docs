X está comprometido com nossa comunidade de desenvolvedores que constroem com a API do X. Como parte desse compromisso, buscamos tornar nossa API aberta e justa para desenvolvedores, segura para as pessoas no X e benéfica para a plataforma X como um todo. É crucial que qualquer desenvolvedor que armazene conteúdo do X offline garanta que os dados reflitam a intenção do usuário e o estado atual do conteúdo no X. Por exemplo, quando alguém no X exclui um Post ou sua conta, protege seus Posts ou edita um Post, é fundamental tanto para o X quanto para nossos desenvolvedores respeitar as expectativas e a intenção dessa pessoa.

Fluxos em tempo real de eventos de conformidade fornecem aos desenvolvedores as ferramentas para manter os dados do X em conformidade com o [Acordo e Política do Desenvolvedor do X](https://developer.x.com/en/developer-terms/policy).

Existem dois fluxos de eventos de conformidade, um para eventos de conformidade de Post e outro para eventos de conformidade de Usuário. Esses fluxos estão disponíveis com acesso Enterprise e foram projetados para ajudar parceiros que ingerem grandes volumes de dados a “ouvir” eventos de conformidade, como eventos de edição de Post.

Esses fluxos fornecem os seguintes eventos:

**Fluxo de conformidade de Post:**

- delete - indica que o Post foi excluído.
- tweet\_edit - indica que um Post foi editado e fornece o ID do Post atualizado.
- withheld - indica que o Post foi retido em um ou mais países.
- drop - indica que o Post deve ser removido da visualização pública.
- undrop - indica que o Post pode ser exibido novamente e tratado como público.

**Fluxo de conformidade de Usuário:**

- **user\_delete** - indica que a conta do Usuário foi excluída
- **user\_undelete** - indica que a conta do Usuário foi restaurada
- **user\_protect** - indica que a conta do Usuário se tornou privada
- **user\_unprotect** - indica que a conta do Usuário se tornou pública
- **user\_withheld** - indica que a conta do Usuário foi retida em um ou mais países.
- **user\_suspend** - indica que a conta do Usuário foi suspensa
- **user\_unsuspend** - indica que a conta do Usuário foi reativada
- **user\_profile\_modification** - indica que o perfil do Usuário foi atualizado. Isso inclui uma descrição, nome, localização e URL atualizados.

**Configuração da conta**

Para acessar esses endpoints, você precisará de:

- Uma [conta de desenvolvedor](https://developer.x.com/en/portal/petition/essential/basic-info) aprovada.
- Autenticar usando as chaves e tokens de um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) que esteja localizado dentro de um [Projeto](/pt-BR/resources/fundamentals/projects).

Saiba mais sobre como obter acesso aos endpoints da API do X v2 em nosso [guia de primeiros passos](/pt-BR/x-api/getting-started/getting-access).
