<p>Junto com a API do X v2, lançamos uma nova estratégia de versionamento que permite aos desenvolvedores entender melhor quando esperar mudanças nas APIs públicas do X e quando precisarão migrar para novas versões. </p>
<p>Os desenvolvedores serão notificados sobre desativações, encerramentos, alterações e adições à API do X por meio de nossos <a href="https://developer.x.com/en/updates/stay-informed">canais de comunicação</a>, para que possam planejar adequadamente como incorporar essas mudanças em seu roadmap de desenvolvimento. Todas as alterações na API serão registradas no changelog.</p>
<p>Atualmente, a API do X possui três versões diferentes. Recomendamos fortemente que os usuários usem a API do X v2 ao planejar sua integração, a menos que alguma funcionalidade necessária ao seu caso de uso ainda não tenha sido lançada na v2. </p>

<div id="versioning-strategy">
  ## Estratégia de versionamento
</div>

<p>O versionamento da API do X será representado por números de versão declarados no caminho da rota dos nossos endpoints:</p>
<p>`https://api.x.com/2/tweets`</p>
<p>Nosso objetivo é lançar versões principais da API do X conforme necessário, mas não com uma frequência maior do que a cada 12 meses. Uma versão principal será lançada quando alterações incompatíveis forem introduzidas na API. Publicaremos guias de migração ao lançar uma nova versão principal para ajudar os desenvolvedores a migrarem para a nova versão. </p>
<p>Uma alteração incompatível exige que os desenvolvedores modifiquem seu código para manter a funcionalidade existente no app. Alterações compatíveis serão aditivas e implementadas na versão mais recente quando estiverem prontas, não exigindo trabalho do desenvolvedor, a menos que ele queira aproveitar a nova funcionalidade.</p>
<p>Se uma alteração incompatível precisar ser implementada no meio do ciclo (por motivos de segurança ou privacidade), essa alteração será feita na versão mais recente.</p>

<div id="breaking-changes">
  ### Alterações incompatíveis
</div>

<p>Essas alterações exigem que os desenvolvedores modifiquem seu código para manter a funcionalidade existente do aplicativo.</p>

<ul>
  <li>Inclusão de um novo parâmetro obrigatório</li>
  <li>Remoção de um endpoint existente</li>
  <li>Remoção de qualquer campo na resposta (obrigatório ou opcional)</li>
  <li>Remoção de um parâmetro de consulta</li>
  <li>Reestruturação do formato de entrada ou saída (por exemplo, transformar um campo de nível superior em um subcampo ou alterar a localização dos erros para que fiquem inline)</li>
  <li>Alteração do nome ou do tipo de dados de um parâmetro de entrada ou valor de saída existente</li>
  <li>Alteração do nome de um campo</li>
  <li>Alteração do nome do recurso</li>
  <li>Alteração de um código de resposta</li>
  <li>Alteração dos tipos de erro</li>
  <li>Alterações nos escopos de autorização existentes</li>
</ul>

<div id="non-breaking-changes">
  ### Alterações que não quebram compatibilidade
</div>

<ul>
  <li>Adição de um novo endpoint</li>
  <li>Adição de um novo parâmetro opcional</li>
  <li>Adição de um novo campo na resposta</li>
  <li>Alteração do texto em mensagens de erro</li>
  <li>Disponibilidade de novos escopos</li>
  <li>“Nulificação” de campos (definir o valor de um campo como null por motivos de privacidade/segurança, como alternativa à remoção completa)</li>
</ul>

<div id="deprecation-and-retirement">
  ### Descontinuação e desativação
</div>

<p>Antes de tudo, segue nossa definição do que descontinuação e desativação significam para a API do X:</p>

<ul>
  <li><b>Descontinuação:</b> O recurso deixa de ser mantido pela equipe. Não haverá novas funcionalidades relacionadas a esse recurso e, se surgirem bugs ou problemas no produto, a probabilidade de implementarmos uma correção é extremamente baixa. </li>
  <li><b>Desativado:</b> O recurso não ficará mais acessível.</li>
</ul>

<p>Na maioria dos casos, assim que uma nova versão for lançada, a versão anterior será marcada como descontinuada. As versões permanecerão nesse estado por um período e, depois, serão desativadas. </p>
<p><a href="https://developer.x.com/en/updates/stay-informed">Mantenha-se informado</a> para saber mais sobre futuras descontinuações e desativações.</p>
