<div id="introduction">
  ## Introdução
</div>

Paginação é um recurso em endpoints da API do X v2 que retornam mais resultados do que cabem em uma única resposta. Quando isso ocorre, os dados são retornados em uma série de “páginas”. Paginação refere-se aos métodos para solicitar programaticamente todas as páginas a fim de recuperar todo o conjunto de resultados. Nem todos os endpoints da API oferecem suporte ou exigem paginação, mas ela é frequentemente utilizada quando os conjuntos de resultados são grandes.

<div id="use-cases-for-pagination">
  ### Casos de uso para paginação
</div>

**Para recuperar todos os resultados de uma requisição:** Use paginação para receber todos os dados relevantes relacionados a uma requisição e seus parâmetros. A paginação é necessária quando há mais resultados correspondentes do que o `max_results` de uma requisição. Repetir requisições com tokens de paginação permite recuperar todos os resultados. Quando uma resposta vier sem um `next_token`, pode-se presumir que todos os resultados já foram paginados. A paginação não deve ser usada para fins de polling. Para obter os resultados mais recentes desde uma requisição anterior, consulte a seção de polling com `since_id`.

**Para percorrer os resultados de uma requisição:** A paginação oferece opções direcionais para navegar pelos resultados de uma requisição, usando os valores `next_token` e `previous_token` das respostas. Esses tokens podem ser definidos como `pagination_token` na requisição seguinte para ir para a próxima ou a página anterior de resultados.

<div id="pagination-token-definitions">
  ### Definições de tokens de paginação
</div>

- `next_token` - String opaca retornada dentro do objeto meta na resposta de endpoints que oferecem suporte à paginação. Indica que há mais resultados disponíveis e pode ser usada como o parâmetro `pagination_token` na próxima solicitação para retornar a próxima página de resultados. A última página de resultados não terá um `next_token`.
  - `previous_token` - String opaca retornada dentro do objeto meta na resposta de endpoints que oferecem suporte à paginação. Indica que há uma página anterior de resultados disponível e pode ser usada como o parâmetro `pagination_token` na próxima solicitação para retornar a página anterior de resultados.
  - `pagination_token` - Parâmetro usado em solicitações de paginação. Defina com o valor de `next_token` para obter a próxima página de resultados. Defina com o valor de `previous_token` para obter a página anterior de resultados.

<div id="fundamentals-of-pagination">
  ### Fundamentos da paginação
</div>

- Endpoints que usam paginação responderão a uma solicitação inicial com a primeira página de resultados e fornecerão um `next_token` dentro do objeto meta na resposta JSON se houver páginas de resultados adicionais disponíveis. Para receber todos os resultados, repita esse processo até que nenhum `next_token` seja incluído na resposta.
  - Os resultados são retornados em ordem cronológica inversa. Isso vale dentro de cada página individual e também entre múltiplas páginas:
    - O primeiro Post na primeira resposta será o mais recente.
    - O último Post na última resposta será o mais antigo.
  - O parâmetro de solicitação `max_results` permite configurar o número de Posts retornados por página. Existem valores padrão e máximo para `max_results`.
  - Toda implementação de paginação envolve extrair tokens do payload da resposta, que podem ser usados em solicitações subsequentes.
  - Em algumas circunstâncias, você pode receber menos do que o `max_results` por página. Não presuma que os resultados por página sempre serão iguais ao valor do parâmetro `max_results`.
  - As melhores maneiras de usar paginação para obter resultados completos são utilizando lógica de loop no código de integração ou usando uma [biblioteca](/pt-BR/x-api/tools-and-libraries/overview) que ofereça suporte à API do X v2.
