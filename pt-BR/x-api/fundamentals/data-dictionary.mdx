<div id="tweet">
  ### Tweet
</div>

Tweets são os blocos básicos de tudo no Twitter. O objeto Tweet tem uma longa lista de campos de “nível raiz”, como `id`, `text` e `created_at`. Objetos Tweet também são o objeto “pai” de vários objetos filhos, incluindo `user`, `media`, `poll` e `place`. Use o parâmetro de campo `tweet.fields` ao solicitar esses campos de nível raiz no objeto Tweet.

O objeto Tweet pode ser encontrado e expandido no recurso de usuário. Tweets adicionais relacionados ao Tweet solicitado também podem ser encontrados e expandidos no recurso Tweet. O objeto está disponível para expansão com `?expansions=pinned_tweet_id` no recurso de usuário ou `?expansions=referenced_tweets.id` no recurso Tweet para obter o objeto apenas com os campos padrão. Use essa expansão junto com o parâmetro de campo `tweet.fields` ao solicitar campos adicionais para completar o objeto.

| Field Value           | Type   | Description                                                                                                                           | How it Can Be Used                                                  |
|:-----------------------|:--------|:---------------------------------------------------------------------------------------------------------------------------------------|:----------------------------------------------------------------------|
| **id (default)**      | string | O identificador exclusivo do Tweet solicitado.                                                                                        | Use isto para recuperar programaticamente um Tweet específico.       |
| **text (default)**    | string | O texto em UTF-8 do Tweet. Consulte o [twitter-text](https://github.com/twitter/twitter-text/) para detalhes sobre caracteres válidos. | Extração de palavras-chave e análise/classificação de sentimento.    |
| **edit\_history\_tweet\_ids (default)** | object | Identificadores exclusivos que indicam todas as versões de um Tweet. Para Tweets sem edições, haverá um ID. Para Tweets com histórico de edições, haverá vários IDs. | Use estas informações para encontrar o histórico de edições de um Tweet. |
| **attachments**       | object | Especifica o tipo de anexos (se houver) presentes neste Tweet.                                                                        | Compreender os objetos retornados para expansões solicitadas.        |
| **author\_id**         | string | O identificador exclusivo do usuário que publicou este Tweet.                                                                         | Hidratar o objeto User; compartilhar conjunto de dados para revisão por pares. |
| **context\_annotations** | array | Contém anotações de contexto do Tweet.                                                                                                | Reconhecimento/extração de entidades; análise de tópicos.            |
| **conversation\_id**   | string | O ID do Tweet original da conversa (inclui respostas diretas e encadeadas).                                                           | Use isto para reconstruir a conversa a partir de um Tweet.           |
| **created\_at**        | date (ISO 8601) | Data e hora de criação do Tweet.                                                                                                    | Útil para análise de séries temporais e para entender quando um Tweet foi criado. |
| **edit\_controls**     | object | Indica por quanto tempo o Tweet ainda pode ser editado e o número de edições restantes.                                              | Use isto para determinar se um Tweet é elegível para edição.         |
| **entities**          | object | Entidades extraídas do texto do Tweet. Veja entidades em Twitter Objects.                                                             | Fornece informações adicionais sobre hashtags, URLs, menções etc.    |
| **in\_reply\_to\_user\_id** | string | Se o Tweet representado for uma resposta, este campo conterá o ID do autor do Tweet original.                                       | Determinar se um Tweet foi resposta a outro Tweet.                   |
| **lang**              | string | Idioma do Tweet, se detectado pelo Twitter.                                                                                           | Classificar Tweets por idioma.                                       |
| **non\_public\_metrics** | object | Métricas de engajamento não públicas do Tweet no momento da solicitação. Requer autenticação com contexto de usuário.                | Determinar o total de impressões geradas pelo Tweet.                 |
| **organic\_metrics**   | object | Métricas de engajamento, acompanhadas em um contexto orgânico, do Tweet no momento da solicitação. Requer autenticação com contexto de usuário. | Medir o engajamento orgânico do Tweet.                               |
| **possibly\_sensitive** | boolean | Indica se o conteúdo pode ser reconhecido como sensível.                                                                              | Estudar a circulação de certos tipos de conteúdo.                    |
| **promoted\_metrics**  | object | Métricas de engajamento, acompanhadas em um contexto promovido, do Tweet no momento da solicitação. Requer autenticação com contexto de usuário. | Medir o engajamento do Tweet quando ele foi promovido.               |
| **public\_metrics**    | object | Métricas de engajamento públicas do Tweet no momento da solicitação.                                                                  | Medir o engajamento do Tweet.                                        |
| **referenced\_tweets** | array  | Uma lista de Tweets aos quais este Tweet faz referência, como Retweets, Tweets com citação ou respostas.                              | Entender aspectos conversacionais de retweets etc.                   |
| **reply\_settings**    | string | Mostra quem pode responder a um determinado Tweet. As opções são "everyone", "mentioned\_users" e "followers".                         | Determinar as configurações de resposta do Tweet.                    |
| **withheld**          | object | Contém detalhes de retenção para [conteúdo retido](https://help.x.com/en/rules-and-policies/tweet-withheld-by-country).             |                                                                     |
| **media\_metadata**          | array | Contém metadados dos anexos de mídia do Tweet.     | Obter metadados adicionais, como o `alt_text` do anexo de mídia de um Tweet. |

**Recuperando um objeto Tweet**

**Exemplo de solicitação**

Na solicitação a seguir, estamos requisitando campos do Tweet no endpoint [Tweets lookup](/pt-BR/resources/fundamentals/rate-limits). Certifique-se de substituir `$BEARER_TOKEN` pelo seu próprio [Bearer Token](/pt-BR/resources/fundamentals/authentication) gerado.

```bash
curl --request GET 'https://api.x.com/2/tweets?ids=1212092628029698048&tweet.fields=attachments,author_id,context_annotations,created_at,entities,geo,id,in_reply_to_user_id,lang,possibly_sensitive,public_metrics,referenced_tweets,text,withheld&expansions=referenced_tweets.id' --header 'Authorization: Bearer $BEARER_TOKEN'
```

<div id="user">
  ### Usuário
</div>

O objeto de usuário contém metadados da conta de usuário do Twitter que descrevem o usuário referenciado. Ele é o principal objeto retornado no endpoint [users lookup](/pt-BR/x-api/users/lookup/introduction). Ao solicitar campos de usuário adicionais nesse endpoint, use o parâmetro `user.fields`.

O objeto de usuário também pode aparecer como objeto filho e ser expandido dentro do objeto Tweet. Ele pode ser expandido com `?expansions=author_id` ou `?expansions=in_reply_to_user_id` para obter a versão condensada, contendo apenas os campos padrão. Use essas expansões em conjunto com o parâmetro de campos `user.fields` ao solicitar campos adicionais para completar o objeto.

<div id="space">
  ### Espaço
</div>

Spaces permitem a expressão e a interação por meio de conversas de áudio ao vivo. O dicionário de dados de um Space contém metadados relevantes sobre o Space; todos os detalhes são atualizados em tempo real.

Os objetos de usuário podem ser localizados e expandidos no recurso de usuário. Eles ficam disponíveis para expansão ao adicionar pelo menos um entre `host_ids`, `creator_id`, `speaker_ids` e `mentioned_user_ids` ao parâmetro de consulta `expansions`.

Ao contrário dos Posts, os Espaços são efêmeros e ficam indisponíveis após terminarem ou quando são cancelados por seu criador. <div id="direct-message-events">
  ### Eventos de Mensagem Direta
</div>

As conversas de Mensagem Direta (DM) são compostas por eventos. A API do X v2 atualmente oferece suporte a três tipos de evento: MessageCreate, ParticipantsJoin e ParticipantsLeave.

Os objetos de evento de DM são retornados pelos endpoints de [consulta de Mensagens Diretas](/pt-BR/x-api/direct-messages/lookup/introduction), e um evento MessageCreate é criado quando Mensagens Diretas são criadas com sucesso pelos endpoints de [Gerenciar Mensagens Diretas](/pt-BR/x-api/direct-messages/lookup/introduction).

Ao solicitar eventos de DM, três atributos padrão do objeto de evento (ou campos) são incluídos: id, event\_type e text. Para receber campos adicionais, use o parâmetro [fields](/pt-BR/x-api/fundamentals/fields) dm\_event.fields para selecionar outros. Outros campos de evento disponíveis incluem: dm\_conversation\_id, created\_at, sender\_id, attachments, participant\_ids e referenced\_tweets.

Vários desses campos fornecem os IDs de outros objetos do X relacionados ao evento de Mensagem Direta:

- sender\_id - O ID da conta que enviou a mensagem ou que convidou um participante para uma conversa em grupo
- participant\_ids - Uma matriz de IDs de conta. Para eventos ParticipantsJoin e ParticipantsLeave, essa matriz conterá um único ID da conta que criou o evento
- attachments - Fornece IDs de mídia para conteúdo que foi enviado ao Twitter pelo remetente
- referenced\_tweets - Se uma URL de Tweet for encontrada no campo text, o ID desse Tweet será incluído na resposta

As [expansions](/pt-BR/x-api/fundamentals/expansions) de sender\_id, participant\_ids, referenced\_tweets.id e attachments.media\_keys estão disponíveis para expandir esses IDs de objetos do Twitter.

| :--- | :--- | :--- | :--- |

**Recuperando um objeto de evento de Mensagem Direta**

**Exemplo de solicitação**

Neste exemplo, criaremos uma solicitação que recupera eventos associados a uma conversa um a um. Essa solicitação retornará campos fundamentais do evento de Mensagem Direta, juntamente com campos adicionais para Tweets referenciados e seus autores. Vamos criar uma consulta que peça:

- Atributos fundamentais do evento, como quando foi criado e de qual conversa ele faz parte (dm\_conversation).
- O ID da conta e a descrição de quem enviou a Mensagem Direta.
- O texto de qualquer Tweet referenciado e quando ele foi publicado.
- O ID da conta e a descrição de qualquer autor de Tweet referenciado.

Para retornar esses atributos, sua consulta incluiria o seguinte:

`?dm_event.fields=id,sender_id,text,created_at,dm_conversation_id&expansions=sender_id,referenced_tweets.id&tweet.fields=created_at,text,author_id&user.fields=description`

```bash
      curl --request GET 'https://api.x.com/2/dm_conversations/with/:participant_id/dm_events?tweet.fields=created_at,text,author_id&user.fields=description&expansions=sender_id,participant_ids,referenced_tweets.id&dm_event.fields=id,sender_id,text,participant_ids,created_at,'
    --header 'Authorization: Bearer $BEARER_TOKEN'
```

<div id="how-to-use-fields-and-expansions">
  ## Como usar fields e expansions
</div>

Por padrão, os objetos de dados da API do X v2 incluem um pequeno conjunto de campos quando você faz uma solicitação sem usar os parâmetros [fields](/pt-BR/x-api/fundamentals/fields) ou [expansions](/pt-BR/x-api/fundamentals/expansions). Este guia mostrará como usar os parâmetros de consulta `fields` e `expansions` na sua solicitação para receber objetos e campos adicionais na resposta.

Neste guia, solicitaremos vários campos com base na seguinte captura de tela de um Post.
 

![Esta imagem inclui uma captura de tela de um Post publicado por @X. Você pode ver o texto do Post, o nome de usuário, a data e hora de publicação, a origem (source) e as métricas públicas. Também inclui um vídeo. ](https://cdn.cms-twdigitalassets.com/content/dam/developer-twitter/ScreenShotDataDictionaryGuide.png.twimg.1920.png)

Como você pode ver na captura de tela, há várias informações visíveis relacionadas ao Post, incluindo o autor do Post, as métricas do Post, o carimbo de data e hora de criação, o vídeo e a contagem de visualizações do vídeo. Há também diversos dados que não são visíveis na captura de tela, mas que ainda podem ser solicitados.

Ao fazer uma solicitação à API, a resposta padrão é simples, contendo apenas os campos padrão do Post (id e text). Você também receberá apenas o objeto principal retornado pelo endpoint que estiver usando, e não quaisquer objetos de dados associados que possam se relacionar ao objeto principal.

Essa simplicidade, junto com os parâmetros fields e expansions, permite solicitar apenas os campos necessários, conforme o seu caso de uso.
