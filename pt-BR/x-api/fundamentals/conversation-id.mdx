<div id="conversation-threads-using-the-x-api">
  ### Tópicos de conversa usando a API do X
</div>

Se você observar como as conversas evoluem no X, um Post pode gerar vários tópicos de conversa, cada um dos quais pode crescer em extensão e complexidade à medida que mais pessoas entram na conversa. Identificar relações entre Posts e entender tópicos de conversa é um recurso do payload e das funcionalidades de busca da API do X v2. Quando Posts são publicados em resposta a um Post (conhecido como resposta) ou em resposta a uma resposta, há um `conversation_id` definido em cada resposta, que corresponde ao ID do Post original que iniciou a conversa.

Respostas a um determinado Post, bem como respostas a essas respostas, estão todas incluídas na conversa derivada do Post original. Independentemente de quantos tópicos de resposta surjam, todos compartilharão um `conversation_id` comum ao Post original que deu início à conversa. Usando a API do X v2, você pode recuperar e reconstruir um tópico de conversa inteiro, permitindo entender o que está sendo dito e como as conversas e ideias evoluem.

Recuperando `conversation_id` como um parâmetro de `tweet.fields`
Para solicitar o `conversation_id` para todos os Posts retornados em um endpoint v2, adicione o campo tweet.fields=conversation\_id aos parâmetros da requisição. O campo conversation\_id é sempre o ID do Post original no thread de respostas da conversa. Todos os Posts dentro do mesmo thread de respostas, incluindo threads de respostas criados a partir de threads anteriores, exibirão o mesmo conversation\_id.

<div id="request-with-conversation_id-parameter">
  ### Requisição com o parâmetro `conversation_id`
</div>

```
curl --request GET \
  --url 'https://api.x.com/2/tweets?ids=1225917697675886593&tweet.fields=author_id,conversation_id,created_at,in_reply_to_user_id,referenced_tweets&expansions=author_id,in_reply_to_user_id,referenced_tweets.id&user.fields=name,username' \
  --header 'Authorization: Bearer $ACCESS_TOKEN'
```

<div id="using-conversation_id-as-a-filter-operator">
  ### Usando `conversation_id` como operador de filtro
</div>

O `conversation_id` pode ser usado como parâmetro de consulta na busca recente ou como operador em uma regra para o [fluxo filtrado](/pt-BR/x-api/posts/filtered-stream/introduction). Isso retornará toda a conversa de Posts em tempo real por meio do fluxo filtrado ou em ordem cronológica inversa a partir de [search Tweets](/pt-BR/x-api/posts/search/introduction). Você também pode obter a contagem de Posts em uma conversa usando esse operador com [Posts counts](/pt-BR/x-api/posts/counts/introduction).

<div id="request-to-query-by-conversation_id">
  ## Solicitação para consultar por `conversation_id`
</div>

```
curl --request GET \
  --url 'https://api.x.com/2/tweets/search/recent?query=conversation_id:1279940000004973111&tweet.fields=in_reply_to_user_id,author_id,created_at,conversation_id' \
  --header 'Authorization: Bearer $ACCESS_TOKEN'
```
