<div id="what-is-the-x-api-v2">
  ## O que é a API do X v2?
</div>

A API do X v2 é atualmente a API principal do X, onde estão concentrados os investimentos e a inovação de produto. Trabalhamos em parceria com desenvolvedores para construir a próxima geração da API do X e atender melhor nossa diversa comunidade de desenvolvedores. Com base no feedback da comunidade, reestruturamos a API para cobrir um conjunto mais amplo de necessidades, introduzimos novos recursos e endpoints e aprimoramos a experiência do desenvolvedor.

A API do X v2 é atualmente a API principal do X, onde estão concentrados os investimentos e a inovação de produto. Ao longo dos últimos anos, trabalhamos com desenvolvedores e reestruturamos a API para atender a um conjunto mais amplo de necessidades, introduzir novos recursos e endpoints e aprimorar a experiência do desenvolvedor. Estamos comprometidos em continuar construindo uma plataforma aberta para desenvolvedores e estamos entusiasmados para ver o que você criará com a API do X v2.

<div id="why-migrate">
  ## Por que migrar?
</div>

A API do X v2 foi desenvolvida sobre uma base moderna e mais sustentável e inclui endpoints de substituição aprimorados para a v1.1 padrão e produtos corporativos, além de funcionalidades totalmente novas. Recomendamos fortemente que clientes das APIs legadas (v1.1 e corporativa) comecem a migrar para a v2, pois pretendemos descontinuá-las eventualmente. Use a API do X para acompanhar e analisar a conversa pública, interagir com pessoas no X e inovar.

Nesta seção, discutiremos os endpoints e os recursos.

<div id="discover-new-and-updated-response-objects">
  ### Descubra novos e atualizados objetos de resposta
</div>

Os seis objetos de dados a seguir estão disponíveis nos endpoints da versão v2:

| Objeto | Descrição |
|:--------|:-------------|
| [Tweet](/pt-BR/x-api/fundamentals/data-dictionary#tweet) | O objeto Tweet possui uma extensa lista de campos no nível raiz, como `id`, `text` e `created_at`. Objetos Tweet também são o objeto pai de vários objetos filhos, incluindo `user`, `media`, `poll` e `place`. |
| [User](/pt-BR/x-api/fundamentals/data-dictionary#user) | O objeto user contém metadados da conta do X que descrevem o usuário referenciado. |
| [Spaces](/pt-BR/x-api/fundamentals/data-dictionary#space) | O objeto Space é composto por campos como `state`, `host_id`, `is_ticketed` e `lang`. |
| [Lists](/pt-BR/x-api/fundamentals/data-dictionary#list) | O objeto List contém informações básicas sobre a lista solicitada, incluindo `description`, `member_count` e `owner_id`. |
| [Media](/pt-BR/x-api/fundamentals/data-dictionary#media) | Se um Tweet contiver mídia (como imagens), o objeto media pode ser solicitado usando o parâmetro `media.fields` e inclui campos como `media_key`, `type`, `url`, `preview_image_url` e outros. |
| [Poll](/pt-BR/x-api/fundamentals/data-dictionary#poll) | Uma enquete incluída em um Tweet não é um objeto primário em nenhum endpoint, mas pode ser localizada e expandida dentro do objeto Tweet. |
| [Place](/pt-BR/x-api/fundamentals/data-dictionary#place) | O objeto place é composto por campos como `place_id`, o objeto `geo`, `country_code` e outros. Essas informações podem ser usadas para identificar e analisar Tweets por localização. |

Saiba mais sobre [como usar fields e expansions](/pt-BR/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions).

<div id="flexibility-to-choose-which-objects-and-fields-you-receive">
  ### Flexibilidade para escolher quais objetos e campos você receberá
</div>

Ao fazer uma solicitação a um endpoint GET, você receberá o objeto de dados principal relacionado a esse endpoint, que incluirá um conjunto de campos padrão. Por exemplo, o objeto Tweet retorna os campos `id` e `text` por padrão.

Se você quiser recuperar campos adicionais na sua solicitação, será necessário usar os parâmetros [fields](/pt-BR/x-api/fundamentals/fields) e [expansions](/pt-BR/x-api/fundamentals/expansions). O parâmetro expansions permite recuperar objetos de dados relacionados, como o Tweet fixado de um usuário ou um objeto de mídia, enquanto os operadores de campo permitem solicitar campos específicos dentro dos objetos retornados além dos padrões.

Aqui está uma lista completa de expansions que você pode solicitar com os diferentes endpoints da API do X v2:

| Object / Resource | Available Expansions |
|:-------------------|:----------------------|
| Tweets            | `author_id`, `edit_history_tweet_ids`, `entities.mentions.username`, `in_reply_to_user_id`, `referenced_tweets.id`, `referenced_tweets.id.author_id`, `attachments.poll_ids`, `attachments.media_keys`, `geo.place_id` |
| Users             | `pinned_tweet_id` |
| Spaces            | `invited_user_ids`, `speaker_ids`, `creator_id`, `host_ids`, `topic_ids` |

Saiba mais sobre [como usar fields e expansions](/pt-BR/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions).

<div id="edit-tweets">
  ## Editar posts
</div>

Os endpoints da API do X v2 fornecem metadados de posts editados. O recurso _Editar post_ foi introduzido inicialmente para testes entre funcionários do X em 1º de setembro de 2022. A partir dessa data, posts elegíveis podem ser editados por 30 minutos e até 5 vezes. Saiba mais sobre [Editar posts](/pt-BR/x-api/fundamentals/edit-posts).
Usando a API do X v2, um desenvolvedor pode verificar:

- Se um post era elegível para edição no momento da criação. Alguns posts, como os com enquetes ou agendados, não podem ser editados.
- Que posts são editáveis por 30 minutos e podem ser editados até 5 vezes. Para posts editáveis, é possível ver se ainda há tempo para editar e quantas edições adicionais são possíveis.
- Se você está visualizando uma versão editada de um post (na maioria dos casos, a API retornará a versão mais recente de um post, a menos que uma versão anterior específica seja solicitada pelo ID do post).
- Todo o histórico de edições do post.
- O engajamento atribuído a cada versão do post.

</div>

Um novo campo de Tweet ajuda a identificar a qual conversa um Tweet pertence. O ID da conversa é o ID do Tweet que iniciou a conversa. Saiba mais sobre [rastreamento de conversas](/pt-BR/x-api/fundamentals/conversation-id).

<div id="ready-to-migrate">
  ## Pronto para migrar
</div>

Para usar os endpoints da API do X v2, você precisará do seguinte:

- [Uma conta de desenvolvedor](https://developer.x.com/en/portal/petition/essential/basic-info)
- [Um App de desenvolvedor](https://developer.x.com/en/apps) criado em um [Projeto](pt-BR/resources/fundamentals/projects)
- [Chaves e tokens](pt-BR/resources/fundamentals/authentication) do App de desenvolvedor desse Projeto

É fundamental usar chaves e tokens de um App dentro de um Projeto. Se você estiver usando chaves e tokens de um App fora de um Projeto, não será possível fazer solicitações aos endpoints da API do X v2.

Depois que você tiver uma conta de desenvolvedor, poderá configurar todos os itens acima no [portal do desenvolvedor](https://developer.x.com/en/portal/petition/essential/basic-info).

<div id="authentication">
  ### Autenticação
</div>

Com a nova API do Twitter, você usará dois padrões de autenticação diferentes, OAuth 1.0a User Context e OAuth 2.0 Bearer Token, para acessar diferentes endpoints. Cada um tem uma finalidade diferente ao fazer solicitações aos endpoints:

OAuth 1.0a User Context é necessário ao fazer uma solicitação em nome de um usuário do Twitter
OAuth 2.0 Bearer Token é necessário para fazer solicitações em nome do seu app de desenvolvedor

<div id="migrating-to-the-new-data-format">
  ## Migrando para o novo formato de dados
</div>

Ao migrar do v1.1 ou enterprise para o v2, é importante entender que o formato de entrega dos dados mudou significativamente. Adicionamos novos campos, alteramos a ordem dos campos e, em alguns casos, removemos elementos por completo.

Para saber mais sobre essas mudanças, estamos elaborando uma série de guias que ajudarão você a mapear os campos do formato de dados anterior ao v2 para os campos mais recentes e explicarão como solicitar esses novos campos.

Você pode saber mais visitando a seção de [migração de formatos de dados](/pt-BR/x-api/migrate/data-format-migration) neste hub de migração ou acessando nossos guias específicos de formato de dados:

- [Formato nativo para API do X v2 (standard v1.1)](/pt-BR/x-api/migrate/data-format-migration#migrating-from-standard-v1-1s-data-format-to-v2)
- [Native Enriched para API do X v2 (enterprise)](/pt-BR/x-api/migrate/data-format-migration#migrating-from-native-enriched-data-format-to-v2)
