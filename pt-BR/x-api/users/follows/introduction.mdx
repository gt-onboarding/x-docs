Oferecemos dois conjuntos de grupos de endpoints para ajudar você a consultar, criar e excluir relações de seguimento: consulta de seguidores e gerenciamento de seguidores.

<div id="follows-lookup">
  ### Consulta de seguidores/seguidos
</div>

Disponível apenas para quem tem [acesso Enterprise](/pt-BR/x-api/getting-started/about-x-api)

Os endpoints de consulta de seguidores/seguidos permitem explorar e analisar os relacionamentos entre usuários, prática às vezes chamada de análise de redes. Especificamente, há dois endpoints REST que retornam [objetos de usuário](/pt-BR/x-api/fundamentals/data-dictionary#user) representando quem um usuário especificado segue ou quem segue esse usuário.

Você pode autenticar este endpoint com [OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication), \[App only]\(https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), ou [OAuth 2.0 Authorization Code with PKCE](pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). É possível solicitar até 1.000 usuários por requisição, e tokens de paginação serão fornecidos para percorrer grandes volumes de resultados.
 

<div id="manage-follows">
  ### Gerenciar follows
</div>

Os endpoints de gerenciamento de follows permitem seguir ou deixar de seguir usuários.

Como você está fazendo solicitações em nome de um usuário, deve autenticar esses endpoints com [OAuth 1.0a (Contexto de Usuário)](/pt-BR/resources/fundamentals/authentication) ou [OAuth 2.0 com PKCE](pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) e utilizar os tokens de acesso do usuário associados ao usuário em cujo nome você está fazendo a solicitação. Você pode gerar esse token de acesso do usuário usando o [fluxo OAuth de 3 etapas](/pt-BR/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) ou o [fluxo de concessão Authorization Code com PKCE](/pt-BR/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0).

Você está limitado a 400 ações de follow por dia em nome de cada usuário autenticado e a 1.000 ações por dia por App considerando todos os seus usuários autenticados. Por exemplo, se você tiver cinco usuários autenticados, poderá seguir 400 usuários por dia (limite por usuário) com dois desses usuários, totalizando 800 ações, e terá que dividir as 200 ações restantes (por App) entre os três usuários restantes. Esse limite não se aplica ao endpoint de unfollow, que possui um limite separado de 500 ações por dia (por App).

<Note>
  **Configuração da conta**

  Para acessar esses endpoints, você precisará de:

  - Uma [conta de desenvolvedor](https://developer.x.com/en/portal/petition/essential/basic-info) aprovada.
  - Autenticar usando as chaves e tokens de um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) que esteja dentro de um [Projeto](/pt-BR/resources/fundamentals/projects).
</Note>

Saiba mais sobre como obter acesso aos endpoints da API do X v2 em nosso [guia de primeiros passos](/pt-BR/x-api/getting-started/getting-access).
