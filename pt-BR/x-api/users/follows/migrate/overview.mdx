  ## Comparando os endpoints de seguidores da API do X
</div>

<div id="follows-lookup">
  ### Consulta de follows
</div>

Os endpoints de consulta de follows da v2 substituirão os endpoints padrão da v1.1 [followers/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-followers-ids), v1.1 [followers/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-followers-list), v1.1 [friends/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-friends-ids) e v1.1 [friends/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-friends-list).

As tabelas a seguir comparam os vários tipos de endpoints de consulta de follows:

|     |     |     |
| :--- | :--- | :--- |
| **Descrição** | **Padrão v1.1** | **API do X v2** |
| Métodos HTTP compatíveis | `GET` | `GET` |
| Domínio do host | `https://api.x.com` | `https://api.x.com` |
| Caminho do endpoint | /1.1/friends/ids.json<br /><br />/1.1/friends/list.json<br /><br />/1.1/followers/ids.json<br /><br />/1.1/followers/list.json | /2/users/:id/following<br /><br />/2/users/:id/followers |
| [Autenticação](/pt-BR/resources/fundamentals/authentication) | OAuth 1.0a User Context<br /><br />Somente app | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code com PKCE<br /><br />Somente app |
| [Limites de taxa](/pt-BR/resources/fundamentals/rate-limits) padrão por solicitação | 15 solicitações por 15 min (por usuário)<br /><br />15 solicitações por 15 min (por app) | 15 solicitações por 15 min (por usuário)<br /><br />15 solicitações por 15 min (por app) |
| Máximo de usuários por resposta | GET friends/ids e GET followers/ids retornam no máximo 5.000 IDs de usuários por página.<br /><br />  <br /><br />GET friends/list e GET followers/list retornam no máximo 200 objetos de usuário por página. | 1.000 objetos de usuário por página |
| Paginação | O token é retornado em um campo next\_cursor, que pode ser passado como valor para o parâmetro cursor para retornar a próxima página de resultados. | O token é retornado em um campo next\_token, que pode ser passado como valor para o parâmetro token para retornar a próxima página de resultados.<br /><br />A carga útil da v2 também fornece um campo previous\_token, que pode ser passado com o parâmetro pagination\_token para retornar a página anterior de resultados. |
| Formato JSON | Formato padrão v1.1 | [Formato da API do X v2](/pt-BR/x-api/fundamentals/data-dictionary) (determinado pelos parâmetros de solicitação fields e expansions, não retrocompatível com os formatos v1.1)<br /><br />Para saber mais sobre como migrar do formato padrão v1.1 para o formato da API do X v2, visite nosso [guia de migração de formatos de dados](/pt-BR/x-api/migrate/data-format-migration). |
| Suporta selecionar quais [fields](/pt-BR/x-api/fundamentals/data-dictionary) retornar na carga |     | ✔   |
| Suporta os campos de [anotações de post](/pt-BR/x-api/fundamentals/post-annotations) |     | ✔   |
| Suporta solicitar novos campos de [métricas](/pt-BR/x-api/fundamentals/metrics) |     | ✔   |
| Suporta o campo [conversation\_id](/pt-BR/x-api/fundamentals/conversation-id) |     | ✔   |
| Requer o uso de credenciais de um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) associado a um [projeto](/pt-BR/resources/fundamentals/projects) |     | ✔   |

<div id="manage-follows">
  #### Gerenciar follows
</div>

Os endpoints v2 de gerenciamento de follows substituirão os endpoints padrão v1.1 [POST friendships/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-create) e [POST friendships/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-destroy).

As tabelas a seguir comparam os endpoints padrão v1.1 e da API do X v2 para criar follow:

<div id="follow-a-user">
  #### Seguir um usuário
</div>

| Descrição | Padrão v1.1 | API do X v2 |
| :--- | :--- | :--- |
| Métodos HTTP compatíveis | POST | POST |
| Domínio do host | https://api.x.com | https://api.x.com |
| Caminho do endpoint | /1.1/friendships/create.json | /2/users/:id/following |
| [Autenticação](/pt-BR/resources/fundamentals/authentication) | Contexto de Usuário OAuth 1.0a | Contexto de Usuário OAuth 1.0a<br /><br />OAuth 2.0 com PKCE (Código de Autorização) |
| [Limites de taxa](/pt-BR/resources/fundamentals/rate-limits) padrão da solicitação | 50 solicitações por 15 min | 50 solicitações por 15 min |
| Operações diárias máximas por usuário | 400 | 400 |
| Operações diárias máximas por app | 1000 | 1000 |
| Exige o uso de credenciais de um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) associado a um [Projeto](/pt-BR/resources/fundamentals/projects) |     | ✔️  |

| Descrição | Padrão v1.1 | API do X v2 |
