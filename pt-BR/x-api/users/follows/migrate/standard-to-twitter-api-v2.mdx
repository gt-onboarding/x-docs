<div id="manage-follows-standard-v11-compared-to-x-api-v2">
  #### Gerenciar follows: v1.1 padrão em comparação com a API do X v2
</div>

Se você tem trabalhado com os endpoints padrão v1.1 [POST friendships/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-create) e [POST friendships/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-destroy), o objetivo deste guia é ajudar você a entender as semelhanças e diferenças entre os endpoints de gerenciamento de follows da versão padrão e da API do X v2.

- **Semelhanças**
  - Contexto de usuário OAuth 1.0a
- **Diferenças**
  - URLs de endpoint
  - Requisitos de Apps e Projetos
  - Métodos HTTP
  - Parâmetros da solicitação

     

<div id="similarities">
  #### Semelhanças
</div>

**Método de autenticação OAuth 1.0a User Context**

Ambas as versões do endpoint oferecem suporte a [OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication#oauth-1-0a-2). Portanto, se você vinha usando um dos endpoints padrão v1.1 para gerenciar follows, pode continuar usando o mesmo método de autenticação ao migrar para a versão da API do X v2. 

<div id="differences">
  #### Diferenças
</div>

**URLs de endpoint**

- Endpoints Standard v1.1:
  - POST https://api.x.com/1.1/friendships/create.json
    (seguir um usuário)
  - POST https://api.x.com/1.1/friendships/destroy.json
    (deixar de seguir um usuário)
- Endpoint da API do X v2:
  - POST https://api.x.com/2/users/:id/following
    (seguir um usuário)
  - DELETE https://api.x.com/2/users/:source\_user\_id/following/:target\_user\_id
    (deixar de seguir um usuário)

**Requisitos de App e Projeto**

Os endpoints da API do X v2 exigem que você use credenciais de um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) associado a um [Projeto](/pt-BR/resources/fundamentals/projects) ao autenticar suas solicitações. Todos os endpoints da API do X v1.1 podem usar credenciais de Apps independentes ou Apps associados a um projeto.

**Parâmetros de solicitação**

Os seguintes parâmetros de solicitação do Standard v1.1 têm equivalentes na API do X v2:

| Standard v1.1 | API do X v2 |
| :--- | :--- |
| Sem equivalente | id (POST), source\_user\_id (DELETE) |
| user\_id | target\_user\_id |
| screen\_name | Sem equivalente |

Observe que os parâmetros do Standard v1.1 são passados como parâmetros de consulta, enquanto os parâmetros da API do X v2 são passados como parâmetros no corpo (para o endpoint POST) ou como parâmetros de caminho (para o endpoint DELETE).

Além disso, os parâmetros id e source\_user\_id da v2 não são necessários ao usar os endpoints do Standard v1.1, pois os tokens de acesso enviados com OAuth 1.0a User Context permitiam inferir qual usuário estava iniciando a ação de seguir/deixar de seguir.
