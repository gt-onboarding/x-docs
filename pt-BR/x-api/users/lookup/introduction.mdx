Você pode autenticar sua solicitação para todos os endpoints de busca de usuários, exceto o de busca do usuário autenticado, com [OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication), [App only](/pt-BR/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), ou [OAuth 2.0 Authorization code with PKCE](pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). No entanto, se quiser acessar métricas privadas ou dados de usuários privados, será necessário utilizar OAuth 1.0a User Context ou OAuth 2.0 Authorization Code com PKCE e enviar o Access Token do usuário autenticado junto com a solicitação.    

Este endpoint é comumente usado para obter detalhes atualizados de um usuário, verificar se um usuário existe ou atualizar os dados armazenados após um evento de conformidade.

<div id="authenticated-user-lookup">
  ### Consulta de usuário autenticado
</div>

Se estiver autenticando em nome de outros usuários, é fundamental saber em nome de quem você pode fazer uma solicitação.

Este endpoint requer o uso de [OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication) ou [OAuth 2.0 Authorization Code com PKCE](pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). Ele retorna informações sobre o usuário autorizado, ou seja, o usuário associado aos tokens de acesso de usuário que você envia com a solicitação.

Você pode acessar este endpoint pelo método GET. Há um limite por usuário de 75 solicitações a cada 15 minutos para este endpoint.

<Note>
  **Configuração da conta**

  Para acessar estes endpoints, você precisará de:

  - Uma [conta de desenvolvedor](https://developer.x.com/en/portal/petition/essential/basic-info) aprovada.
  - Autenticar-se usando as chaves e tokens de um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) que esteja dentro de um [Projeto](/pt-BR/resources/fundamentals/projects).

  Saiba mais sobre como obter acesso aos endpoints da API do X v2 em nosso [guia de primeiros passos](/pt-BR/x-api/getting-started/getting-access).
</Note>
