  ## Comparando os endpoints de consulta de usuários da API do X
</div>

Os endpoints de consulta de usuários da v2 substituirão os endpoints padrão da v1.1 [GET users/lookup](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-users-lookup.html) e [GET users/show](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-users-show.html). Se você tem código, apps ou ferramentas que usam uma dessas versões dos endpoints de consulta de usuários e está considerando migrar para o endpoint mais recente da API do X v2, este conjunto de guias é para você. 
 

As tabelas a seguir comparam os vários tipos de endpoints de consulta de usuários:
 

|     |     |     |
| :--- | :--- | :--- |
| **Descrição** | **Padrão v1.1** | **API do X v2** |
| Métodos HTTP compatíveis | `GET` | `GET` |
| Domínio do host | `https://api.x.com` | `https://api.x.com` |
| Caminho do endpoint | `/1.1/users/show.json` `/1.1/users/lookup.json` | `/2/users`<br /><br />`/2/users/:id`<br /><br />`/2/users/by`<br /><br />`/2/users/by/:username` |
| [Autenticação](/pt-BR/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />Somente app<br /><br />OAuth 2.0 Authorization Code with PKCE |
| Limites de [taxa de requisições](/pt-BR/resources/fundamentals/rate-limits) padrão | 900 requisições por 15 min (por usuário)<br /><br />/show - 900 requisições por 15 min (por app)  <br />/lookup - 300 requisições por 15 min (por app) | 900 requisições por 15 min (por usuário)<br /><br />300 requisições por 15 min (por app) |
| Máximo de usuários por resposta | /show - 1<br /><br />/lookup - 100 | 100 |
| Formato do objeto de resposta JSON | Formato padrão v1.1 | [Formato da API do X v2](/pt-BR/x-api/fundamentals/data-dictionary) (determinado pelos parâmetros de requisição fields e expansions, não compatível retroativamente com os formatos v1.1)<br /><br />Para saber mais sobre como migrar do formato padrão v1.1 para o formato da API do X v2, visite nosso [guia de migração de formatos de dados](/pt-BR/x-api/migrate/data-format-migration). |
| Permite selecionar quais [fields](/pt-BR/x-api/fundamentals/data-dictionary) retornar no payload |     | ✔   |
| Oferece suporte aos campos [annotations](/pt-BR/x-api/fundamentals/post-annotations) (no Post fixado) |     | ✔   |
| Oferece suporte à solicitação de novos campos de [metrics](/pt-BR/x-api/fundamentals/metrics) (no Post fixado) |     | ✔   |
| Oferece suporte ao campo [conversation\_id](/pt-BR/x-api/fundamentals/conversation-id) (no Post fixado) |     | ✔   |
| Exige o uso de credenciais de um [developer App](/pt-BR/resources/fundamentals/developer-apps) associado a um [project](/pt-BR/resources/fundamentals/projects) |     | ✔   |
