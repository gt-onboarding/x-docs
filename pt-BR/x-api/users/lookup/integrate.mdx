<div id="authentication">
  #### Autenticação
</div>

Todos os endpoints da API do X v2 exigem que as solicitações sejam [autenticadas](/pt-BR/resources/fundamentals/authentication) com um conjunto de credenciais, também conhecido como chaves e tokens. Você pode usar OAuth 1.0a User Context, App only ou OAuth 2.0 Authorization Code com PKCE para autenticar solicitações a esses endpoints.

[OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication#oauth-1-0a-2) exige que você utilize suas API Keys, user Access Tokens e alguns outros parâmetros para [criar um cabeçalho de autorização](/pt-BR/resources/fundamentals/authentication#authorizing-a-request), que você então enviará com sua solicitação. Os Access Tokens devem estar associados ao usuário em nome de quem você está fazendo a solicitação. Se você deseja gerar um conjunto de Access Tokens para outro usuário, ele deverá autorizar seu App usando o [fluxo OAuth de 3 etapas](https://developer-staging.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens).

Observe que o OAuth 1.0a pode ser difícil de usar. Se você não está familiarizado com esse método de autenticação, recomendamos que utilize uma [biblioteca](/pt-BR/x-api/tools-and-libraries/overview), use uma ferramenta como o Postman ou use OAuth 2.0 para autenticar suas solicitações. Se você deseja recuperar um Post ou métricas privadas desses endpoints, será necessário usar OAuth 1.0a User Context ou OAuth 2.0 Authorization Code com PKCE, o que garantirá que você tenha as permissões adequadas do usuário proprietário desse conteúdo.

[App only](/pt-BR/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) exige apenas que você envie um [App only Access Token](/pt-BR/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) com sua solicitação. Você pode gerar um App only Access Token diretamente em um App de desenvolvedor ou gerá-lo usando o endpoint [POST oauth2/token](/pt-BR/resources/fundamentals/authentication#post-oauth2-token).

[OAuth 2.0 Authorization Code com PKCE](/pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) permite maior controle sobre o escopo de uma aplicação e fluxos de autorização em vários dispositivos. O OAuth 2.0 permite escolher escopos específicos e granulares que concedem permissões específicas em nome de um usuário.

Para habilitar o OAuth 2.0 no seu App, você deve ativá-lo nas configurações de autenticação do seu App, encontradas na seção de configurações do App no portal do desenvolvedor.

**Observação**

Se você estiver solicitando os seguintes campos, OAuth 1.0a User Context ou OAuth 2.0 Authorization Code é obrigatório:

- tweet.fields.non\_public\_metrics
- tweet.fields.promoted\_metrics
- tweet.fields.organic\_metrics

<div id="developer-portal-projects-and-developer-apps">
  #### Portal do desenvolvedor, Projetos e Apps de desenvolvedor
</div>

Para obter um conjunto de credenciais de autenticação compatível com os endpoints da API do X v2, você deve [se inscrever para uma conta de desenvolvedor](https://developer.x.com/en/portal/petition/essential/basic-info), configurar um [Projeto](/pt-BR/resources/fundamentals/projects) nessa conta e criar um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) nesse Projeto. Em seguida, você poderá encontrar suas chaves e tokens no seu App de desenvolvedor. 
 

<div id="rate-limits">
  #### Limites de taxa
</div>

Todos os dias, muitos milhares de desenvolvedores fazem solicitações à API do X. Para ajudar a gerenciar o grande volume dessas solicitações, [limites de taxa](/pt-BR/x-api/fundamentals/rate-limits) são aplicados a cada endpoint, restringindo o número de solicitações que você pode fazer em nome do seu app ou de um usuário autenticado.

Os endpoints de consulta de usuários têm limites de taxa tanto no nível do app quanto no nível do usuário. No entanto, o endpoint de consulta do usuário autenticado tem limite de taxa no nível do usuário.

O limite de taxa no nível do app significa que você, desenvolvedor, só pode fazer um certo número de solicitações a esse endpoint em um período de tempo definido a partir de qualquer App (conforme identificado pelas chaves e tokens que você está usando). O limite de taxa no nível do usuário significa que o usuário autenticado em cujo nome você está fazendo a solicitação só pode realizar um certo número de solicitações em qualquer App de desenvolvedor.

A tabela abaixo mostra os limites de taxa para cada endpoint.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Método HTTP** | **Limite de taxa / Nível** |
| /2/users | GET | 900 solicitações por 15 minutos / App e Usuário |
| /2/users/:id | GET | 900 solicitações por 15 minutos / App e Usuário |
| /2/users/by | GET | 900 solicitações por 15 minutos / App e Usuário |
| /2/users/by/username/:username | GET | 900 solicitações por 15 minutos / App e Usuário |
| /2/users/me | GET | 75 solicitações por 15 minutos / Usuário |

<div id="fields-and-expansions">
  #### Campos e expansões
</div>

A API do X v2 permite selecionar exatamente quais dados você deseja obter da API usando um conjunto de ferramentas chamado campos e expansões. O parâmetro expansions permite expandir objetos referenciados no payload. Por exemplo, este endpoint permite usar a expansão pinned\_tweet\_id.

O parâmetro fields permite escolher exatamente quais [campos](/pt-BR/x-api/fundamentals/fields) dentro dos diferentes objetos de dados você deseja receber. Esses endpoints entregam principalmente objetos de usuário. Por padrão, o objeto de usuário retorna os campos id, name e username. Para receber campos adicionais, como user.created\_at ou user.location, você precisará solicitar especificamente esses campos usando o parâmetro fields. Alguns campos importantes que você pode considerar na sua integração são os dados de enquete de Post, métricas, anotações e o identificador de conversa.

Incluímos um guia sobre como [usar campos e expansões](/pt-BR/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) em nosso [dicionário de dados da API do X v2](/pt-BR/x-api/fundamentals/data-dictionary).
