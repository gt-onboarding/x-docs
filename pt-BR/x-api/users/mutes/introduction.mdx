Silenciar uma conta permite remover os Posts dessa conta da sua timeline sem deixar de segui-la ou bloqueá-la. As contas silenciadas não saberão que foram silenciadas e você pode reverter o silêncio a qualquer momento. Com os endpoints de gerenciamento de silenciamentos, desenvolvedores podem criar experiências mais seguras para as pessoas no X. Um exemplo de como usar o gerenciamento de silenciamentos é um aplicativo que permite silenciar contas que possam postar sobre tópicos específicos por um período definido. Com o endpoint de consulta de silenciamentos, você pode ver quem você ou um usuário autenticado silenciou. Isso pode ser útil para entender como você interage com as contas silenciadas.

Como você está fazendo solicitações de informações privadas com a consulta de silenciamentos e agindo em nome de um usuário com o gerenciamento de silenciamentos, você deve autenticar esses endpoints com [OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication) ou [OAuth 2.0 Authorization Code with PKCE](pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), e usar os tokens de acesso do usuário associados a um usuário que autorizou seu app, os quais podem ser gerados usando o [fluxo OAuth de 3 etapas](/pt-BR/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) ou o [fluxo de concessão Authorization Code com PKCE](/pt-BR/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0).

<div id="mutes-lookup">
  ### Consulta de silenciamentos
</div>

O endpoint de consulta de silenciamentos permite ver quais contas o usuário autenticado silenciou. Este endpoint tem um limite de 15 solicitações por usuário a cada 15 minutos.

<div id="manage-mutes">
  ### Gerenciar silenciamentos
</div>

Os endpoints de gerenciamento de silenciamentos permitem silenciar ou deixar de silenciar uma conta específica em nome de um usuário autenticado. Para esses endpoints, há dois métodos disponíveis: POST e DELETE. O método POST permite silenciar uma conta, e o método DELETE permite deixar de silenciar uma conta. Há um limite de 50 solicitações por usuário a cada 15 minutos para os endpoints POST e DELETE.

**Observação:** Se um usuário silenciar pelo [X](https://x.com/), há um limite de 200 solicitações a cada 15 minutos.

**Configuração da conta**

Para acessar esses endpoints, você precisará de:

- Uma [conta de desenvolvedor](https://developer.x.com/en/portal/petition/essential/basic-info) aprovada.
- Autenticar usando as chaves e tokens de um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) que esteja dentro de um [Projeto](/pt-BR/resources/fundamentals/projects).

Saiba mais sobre como obter acesso aos endpoints da API do X v2 em nosso [guia de primeiros passos](/pt-BR/x-api/getting-started/getting-access).
