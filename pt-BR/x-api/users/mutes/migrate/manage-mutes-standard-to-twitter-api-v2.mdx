<div id="manage-mutes-standard-v11-compared-to-x-api-v2">
  ### Gerenciar silenciamentos: Standard v1.1 em comparação com a API do X v2
</div>

Se você vem trabalhando com os endpoints padrão v1.1 [POST mutes/users/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-create) e [POST mutes/users/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-destroy), o objetivo deste guia é ajudar você a entender as semelhanças e diferenças entre os endpoints de gerenciamento de silenciamentos do Standard v1.1 e da API do X v2.

- **Semelhanças**
  - Contexto de Usuário OAuth 1.0a
- **Diferenças**
  - URLs dos endpoints
  - Requisitos de Apps e Projetos
  - Métodos HTTP
  - Parâmetros da requisição
     

<div id="similarities">
  #### Semelhanças
</div>

**Método de autenticação OAuth 1.0a User Context**

Ambas as versões do endpoint oferecem suporte ao [OAuth 1.0a User Context](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Portanto, se você já utilizava um dos endpoints padrão v1.1 para gerenciar silenciamentos, poderá continuar usando o mesmo método de autenticação ao migrar para a versão da API do X v2.

<div id="differences">
  #### Diferenças
</div>

**URLs de endpoint**

- Endpoints Standard v1.1:
  - POST https://api.x.com/1.1/mutes/users/create.json
    (silenciar um usuário)
  - POST https://api.x.com/1.1/mutes/users/destroy.json
    (remover o silêncio de um usuário)
- Endpoint da API do X v2:
  - POST https://api.x.com/2/users/:id/muting
    (silenciar um usuário)
  - DELETE https://api.x.com/2/users/:source\_user\_id/muting/:target\_user\_id
    (remover o silêncio de um usuário)

**Requisitos de App e Projeto**

Os endpoints da API do X v2 exigem o uso de credenciais de um [developer App](/pt-BR/resources/fundamentals/developer-apps) associado a um [Project](/pt-BR/resources/fundamentals/projects) ao autenticar suas solicitações. Todos os endpoints da API do X v1.1 podem usar credenciais de Apps independentes ou de Apps associados a um projeto.

**Parâmetros de solicitação**

Os seguintes parâmetros de solicitação do Standard v1.1 têm equivalentes na API do X v2:

| Standard v1.1 | API do X v2 |
| :--- | :--- |
| user\_id | target\_user\_id |
| screen\_name | Sem equivalente |

Observe que os parâmetros do Standard v1.1 são enviados como parâmetros de consulta, enquanto os parâmetros da API do X v2 são enviados como parâmetros de corpo (para o endpoint POST) ou de caminho (para o endpoint DELETE).

Além disso, não é necessário informar o id do usuário que está silenciando um usuário de destino ao usar os endpoints do Standard v1.1, pois os tokens de acesso enviados com [OAuth 1.0a User Context](/pt-BR/resources/fundamentals/authentication) indicavam qual usuário estava iniciando a ação de silenciar/remover silêncio.
