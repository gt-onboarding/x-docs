  ## Comparando os endpoints de silenciamento da API do X
</div>

<div id="mutes-lookup">
  #### Consulta de silenciamentos
</div>

O endpoint de consulta de silenciamentos da v2 substituirá os endpoints padrão [v1.1 GET mutes/users/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/get-mutes-users-ids) e [GET mutes/users/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/get-mutes-users-list).

As tabelas a seguir comparam os endpoints de silenciamentos padrão v1.1 e da API do X v2:
 

| Descrição | Padrão v1.1 | API do X v2 |
| :--- | :--- | :--- |
| Métodos HTTP compatíveis | ******GET****** | ******GET****** |
| Domínio do host | ******https://api.x.com****** | ******https://api.x.com****** |
| Caminho do endpoint | ******/1.1/mutes/users/ids.json******<br /><br />/1.1/mutes/users/list.json | ******/2/users/:id/muting****** |
| [Autenticação](/pt-BR/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code with PKCE |
| Limites padrão de [taxa de solicitação](/pt-BR/resources/fundamentals/rate-limits) | 15 solicitações por 15 min (por usuário) | 15 solicitações por 15 min (por usuário) |
| Formatos de dados | Formato padrão v1.1 | [Formato da API do X v2](/pt-BR/x-api/fundamentals/data-dictionary) (determinado pelos parâmetros de solicitação fields e expansions; não é compatível com os formatos do v1.1)<br /><br />Para saber mais sobre como migrar do formato padrão v1.1 para o formato da API do X v2, visite nosso [guia de migração de formatos de dados](/pt-BR/x-api/migrate/data-format-migration). |
| Exige o uso de credenciais de um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) associado a um [Projeto](/pt-BR/resources/fundamentals/projects) |     | ✔️  |

<div id="manage-mutes">
  #### Gerenciar silenciamentos
</div>

Os endpoints v2 de gerenciamento de silenciamentos substituirão os endpoints padrão v1.1 [POST mutes/users/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-create) e [POST mutes/users/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-destroy).

As tabelas a seguir comparam os endpoints de silenciamento do padrão v1.1 e da API do X v2:

**Silenciar um usuário**

| Descrição | Padrão v1.1 | API do X v2 |
| :--- | :--- | :--- |
| Métodos HTTP compatíveis | POST | POST |
| Domínio do host | https://api.x.com | https://api.x.com |
| Caminho do endpoint | /1.1/mutes/users/create.json | /2/users/:id/muting |
| [Autenticação](/pt-BR/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code with PKCE |
| [Limites de taxa](/pt-BR/resources/fundamentals/rate-limits) padrão de solicitação | 50 solicitações por 15 min | 50 solicitações por 15 min |
| Exige o uso de credenciais de um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) associado a um [Projeto](/pt-BR/resources/fundamentals/projects) |     | ✔️  |

| [Limites de taxa](/pt-BR/resources/fundamentals/rate-limits) padrão de solicitação | 50 solicitações por 15 min | 50 solicitações por 15 min |
| Exige o uso de credenciais de um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) associado a um [Projeto](/pt-BR/resources/fundamentals/projects) |     | ✔️  |
