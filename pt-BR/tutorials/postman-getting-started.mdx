<div id="introduction">
  ## Introdução
</div>

O Postman é um aplicativo para desktop e web que permite fazer requisições a uma API por meio de uma interface gráfica. Recomendamos usar o Postman com a API do X, a API de Anúncios do X e os endpoints do Labs ao explorar as funcionalidades das APIs, bem como ao solucionar problemas no seu aplicativo.

Atualmente, temos duas coleções do Postman disponíveis:

<div id="prerequisites">
  ### Pré-requisitos
</div>

Antes de começar a usar as coleções do Postman do X, você precisará garantir que tem o acesso e as credenciais adequados para a ferramenta da plataforma de desenvolvedores do X que escolher. Você pode saber mais sobre como obter acesso na nossa [página “Introdução”](/pt-BR/resources/platform-overview).

Depois que você tiver uma [conta de desenvolvedor](https://developer.x.com/en/portal/petition/essential/basic-info), tiver configurado um [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps) e um conjunto de chaves e tokens de [autenticação](/pt-BR/resources/fundamentals/authentication), e tiver configurado corretamente seu ambiente para fazer solicitações à API específica que planeja usar, você pode seguir as etapas abaixo para começar.
 

<div id="getting-started-with-xs-postman-collections">
  ## Introdução às coleções do Postman da X
</div>

<div id="step-one-add-one-of-the-x-postman-collections-to-your-account">
  ### Etapa 1: Adicione uma das coleções do Postman do X à sua conta
</div>

Embora você possa configurar manualmente os endpoints específicos que deseja usar no Postman, nós já fizemos esse trabalho por você e criamos uma coleção pronta para uso com as APIs relevantes. Basta clicar em um dos links na seção anterior “Coleções do Postman” e uma coleção com todos os endpoints associados à API selecionada será adicionada ao seu app do Postman. Essas coleções também estão disponíveis na [Postman API Network](https://explore.postman.com/). Cada endpoint inclui automaticamente os parâmetros disponíveis, exemplos de resposta e o tipo de autenticação já configurado; assim, você só precisa adicionar suas credenciais e valores de parâmetros para começar a explorar a funcionalidade.

Neste exemplo, vamos trabalhar com a [coleção da API do X v2](https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400). 
 

<div id="step-two-add-your-keys-and-tokens-as-environmental-variables">
  ### Etapa dois: Adicione suas chaves e tokens como variáveis de ambiente
</div>

Quando você adiciona a coleção à sua instância do Postman, é criado automaticamente um ambiente chamado “API do X v2”, no qual será necessário incluir suas chaves e tokens. Nesta etapa, vamos guiá-lo pelo processo de adicionar suas chaves e tokens do seu App de desenvolvedor ao ambiente “API do X v2”.

Para adicionar suas chaves e tokens ao ambiente “API do X v2”, clique no botão “manage environments” no canto superior direito do Postman. Em seguida, clique no botão de configurações no canto superior direito.

<Frame>
  <img src="/images/using-postman-1.png.twimg.1920.png" alt="Esta imagem mostra o botão &#x22;manage environments&#x22; destacado no console do Postman." />
</Frame>

Na lista de ambientes, clique em “API do X v2”.

Em seguida, adicione uma variável para cada uma das chaves e tokens que você gerou pelo painel de Apps na tabela. Aqui está um exemplo de como sua tabela deve ficar:

| VARIABLE | INITIAL VALUE | CURRENT VALUE |
| :--- | :--- | :--- |
| consumer\_key | `QAktM6W6DF6F7XXXXXX` | `QAktM6W6DF6F7XXXXXX` |
| consumer\_secret | `AJX560A2Omgwyjr6Mml2esedujnZLHXXXXXX` | `AJX560A2Omgwyjr6Mml2esedujnZLHXXXXXX` |
| access\_token | `1995XXXXX-0NGqVhk3s96IX6SgT3H2bbjOPjcyQXXXXXXX` | `1995XXXXX-0NGqVhk3s96IX6SgT3H2bbjOPjcyQXXXXXXX` |
| token\_secret | `rHVuh7dgDuJCOGeoe4tndtjKwWiDjBZHLaZXXXXXX` | `rHVuh7dgDuJCOGeoe4tndtjKwWiDjBZHLaZXXXXXX` |
| bearer\_token | `AAAAAAAAAAAAAAAAAAAAAL9v6AAAAAAA99t03huuqRYg0mpYAAFRbPR3XXXXXXX` | `AAAAAAAAAAAAAAAAAAAAAL9v6AAAAAAA99t03huuqRYg0mpYAAFRbPR3XXXXXXX` |

Observe que as chaves e tokens usados na tabela acima são fictícios e não funcionarão se usados em uma requisição.

Depois de adicionar suas credenciais como variáveis e se certificar de que o ambiente API do X v2 está selecionado, você estará pronto para fazer requisições à coleção da API do X v2. Isso porque cada uma das abas de autorização dos endpoints está configurada para herdar automaticamente as variáveis desse ambiente.

Para usar o Postman com tokens de acesso de usuário, pule XXXX para ver detalhes adicionais sobre como proceder.

### Terceira etapa: selecione um endpoint 

A próxima etapa é escolher um endpoint da coleção e começar a montar sua requisição. Você pode selecionar um endpoint na navegação à direita. Veja como fica:

<Frame>
  <img src="/images/using-postman-2.png.twimg.1920.png" alt="Esta imagem mostra a requisição “Single Posts” selecionada no menu suspenso “Post Lookup” na seção “API do X v2”." />
</Frame>

Neste exemplo, vamos usar o endpoint API do X v2 > Post Lookup > Single Post. 

<div id="step-four-add-values-to-the-params-tab">
  #### Etapa quatro: Adicionar valores na aba Params
</div>

Na próxima etapa, navegue até a aba Params. Você deverá ver um conjunto de parâmetros inativos com descrições que explicam o que cada parâmetro faz e uma lista de todos os valores possíveis que podem ser enviados com a sua solicitação.

Neste exemplo, vamos ativar os parâmetros de consulta expansions e tweet.fields e adicionar os seguintes valores:

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `tweet.fields` | `created_at,attachments` |
| expansions | author\_id |

Além de adicionar os parâmetros de consulta, precisamos incluir a Path Variable obrigatória, id. Como este endpoint retorna Posts, precisamos adicionar um ID de Post válido como valor.

Você pode encontrar o ID do Post acessando x.com, clicando em um Post e, em seguida, observando a URL. Por exemplo, o ID do Post na URL a seguir é `1228393702244134912`:

`https://x.com/XDevelopers/status/1228393702244134912`

Na aba Params, role além de todos os parâmetros de consulta até a seção “Path Variables”. Vamos adicionar o ID do Post que você deseja usar como valor para a chave id.

Se você preencheu tudo nesta etapa corretamente, a aba Params deverá estar como no exemplo a seguir:

<div id="step-five-send-your-request-and-review-your-response">
  #### Passo cinco: Envie sua solicitação e analise a resposta
</div>

Agora que tudo está configurado na sua solicitação, você pode clicar no botão "Send".

Se tudo estiver configurado corretamente, você deverá receber o seguinte payload:

```json
{
    "data": {
        "author_id": "2244994945",
        "text": "What did the developer write in their Valentine’s card?\n  \nwhile(true) {\n    I = Love(You);  \n}",
        "id": "1228393702244134912",
        "created_at": "2020-02-14T19:00:55.000Z"
    },
    "includes": {
        "users": [
            {
                "username": "XDevelopers",
                "name": "Developers",
                "id": "2244994945"
            }
        ]
    }
}
```

<div id="generating-a-user-access-token-with-postman">
  ### Gerando um token de acesso de usuário com o Postman:
</div>

<div id="using-oauth-10a-to-generate-a-user-access-token">
  #### Usando OAuth 1.0a para gerar um token de acesso do usuário
</div>

Confira o processo em três etapas usado na [coleção de teste do fluxo OAuth 1.0a](https://www.postman.com/xapidevelopers/workspace/twitter-s-public-workspace/collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400?ctx=documentation).

<div id="using-oauth-20-to-generate-a-user-access-token">
  #### Usando OAuth 2.0 para gerar um token de acesso de usuário
</div>

Se você quiser gerar um token de acesso de usuário OAuth 2.0 no Postman, pode gerar tokens de acesso OAuth 2.0 para usar em conjunto com a [coleção do Postman da API do X v2](https://www.postman.com/xapidevelopers/workspace/twitter-s-public-workspace/collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400?ctx=documentation).

Clique na coleção no seu workspace, abra a guia “Auth” e selecione o tipo “OAuth 2.0”. Em seguida, em “Configure New Token”, vá até “Configuration Options”. Atualize o “Grant Type” para “Authorization Code (With PKCE)”.

Atualize sua Callback URL para corresponder à URL de retorno (callback) associada ao seu aplicativo. Além disso, atualize os seguintes parâmetros:

- Auth URL -  https://x.com/i/oauth2/authorize
- Access Token URL -  https://api.x.com/2/oauth2/token
- Client ID - Seu Client ID do OAuth 2.0 no Dev Portal
- Client Secret - Caso esteja usando um cliente confidencial
- Scope - Escopos que correspondam aos endpoints aos quais você deseja se conectar. Exemplo: “tweet.read users.read”
- Sua Callback URL (também conhecida como Redirect URL). Ela deve corresponder ao que está nas configurações de autenticação do seu App.
- State - state

Quando estiver pronto, clique em “Get New Access Token” para gerar um token de acesso. Se aparecer uma caixa de diálogo informando que algo deu errado, talvez seja necessário pressionar o botão voltar para fazer login. Você precisará autorizar seu app a acessar sua conta clicando em “Authorize app” na caixa de diálogo.

Depois de autorizar seu app, você será redirecionado ao Postman, onde poderá ver seu token e clicar no botão “Use Token” para começar a fazer solicitações em nome de um usuário autorizado.

Agora você está pronto para usar a coleção do Postman.
