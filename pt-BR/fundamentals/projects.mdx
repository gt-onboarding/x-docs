<div id="introduction">
  ## Introdução
</div>

Projetos permitem organizar seu trabalho com base em como você pretende usar a API do X, para que você possa gerenciar de forma eficaz seu acesso à API e monitorar seu uso. Cada Projeto pode conter um ou vários [Apps](/pt-BR/resources/fundamentals/developer-apps), dependendo do seu nível de acesso (descrito mais adiante nesta página). Você usará esses Apps para gerar credenciais de [autenticação](/pt-BR/resources/fundamentals/authentication), como [API Keys e Secrets](/pt-BR/resources/fundamentals/authentication#api-key-and-secret), [Access Tokens](/pt-BR/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) de usuário e [App Access Token](/pt-BR/resources/fundamentals/authentication#bearer-token-also-known-as-app-only).

Embora você possa usar essas chaves e tokens de qualquer App para acessar a API do X ou a API de Anúncios do X (é necessário [solicitar acesso adicional](/pt-BR/x-ads-api/getting-started/step-by-step-guide) para usar a API de Anúncios), você deve usar chaves e tokens de um App associado a um Projeto para poder usar os endpoints da [API do X v2](/pt-BR/x-api/introduction).

Se você tem uma conta de desenvolvedor, pode visualizar e gerenciar seus Projetos na página [Projects & Apps](https://developer.x.com/en/portal/projects-and-apps) no portal do desenvolvedor. [Cadastre-se](https://developer.x.com/en/portal/petition/essential/basic-info) para obter uma conta se ainda não tiver uma.

<div id="projects-and-x-api-access-levels">
  ### Projetos e níveis de acesso à API do X
</div>

No momento, há 4 níveis diferentes de acesso aplicados no nível de Projeto:

- Gratuito
- Básico
- Pro
- Enterprise

Para saber mais sobre o que cada um desses níveis de acesso oferece, visite a página [sobre a API do X](/pt-BR/x-api/getting-started/about-x-api).

Você só pode ter um único Projeto nos níveis Gratuito, Básico ou Pro.

A seguir, descreveremos algumas diferenças específicas de Projeto:

<div id="free">
  #### Gratuito
</div>

Este nível de acesso é disponibilizado a qualquer pessoa que [se inscreveu](https://developer.twitter.com/en/portal/petition/essential/basic-info) para uma conta de desenvolvedor.

Número de Apps neste Projeto: 1
[Limites de posts](/pt-BR/x-api/fundamentals/post-cap): 1.500 Tweets por mês

Métodos de autenticação:

- [OAuth 2.0 com PKCE](/pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
- Somente app

<div id="basic">
  #### Basic
</div>

Este nível de acesso é fornecido a qualquer pessoa que assinou o acesso Basic por meio do portal do desenvolvedor.

Número de Apps neste Projeto: 2
[Limites de posts](/pt-BR/x-api/fundamentals/post-cap): 10.000 Tweets por mês
Métodos de autenticação:

- [OAuth 2.0 com PKCE](/pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
- [OAuth 1.0a](/pt-BR/resources/fundamentals/authentication#oauth-1-0a-2)
- Somente app

<div id="pro">
  #### Pro
</div>

Este nível de acesso é disponibilizado a quem assinou o plano Pro pelo portal do desenvolvedor.

Número de Apps neste Projeto: 3
[Limites de posts](/pt-BR/x-api/fundamentals/post-cap): 1 milhão de tweets por mês

Métodos de autenticação:

- [OAuth 2.0 com PKCE](/pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
- [OAuth 1.0a](/pt-BR/resources/fundamentals/authentication#oauth-1-0a-2)
- Somente app

  #### Enterprise
</div>

Este nível de acesso é disponibilizado a quem solicitou o Enterprise, foi aprovado por nossa equipe e assinou um contrato mensal com renovação automática por meio de um gerente de contas do X. [Solicite aqui](https://docs.google.com/forms/d/e/1FAIpQLScO3bczKWO2jFHyVZJUSEGfdyfFaqt2MvmOfl_aJp0KxMqtDA/viewform).

Métodos de autenticação:

- [OAuth 2.0 com PKCE](/pt-BR/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)

<div id="tweet-caps-and-projects">
  ### Limites de posts e Projetos
</div>

Os limites de consumo de posts se aplicam no nível do Projeto, restringindo o volume de posts que você pode recuperar de determinados endpoints da API do X v2 em um determinado mês.

Saiba mais sobre [limites de posts](/pt-BR/x-api/fundamentals/post-cap).

<div id="configuring-projects">
  ### Configuração de Projetos
</div>

<div id="creating-a-project">
  #### Criando um Projeto
</div>

Para criar um Projeto, clique em “New Project” no seu [dashboard](https://developer.twitter.com/en/portal/dashboard) ou na página [Projects & Apps](https://developer.twitter.com/en/portal/projects-and-apps) no portal do desenvolvedor. Você só verá essa opção se ainda não tiver criado um Projeto. Será solicitado que você informe um nome para o Projeto, uma descrição e o caso de uso. Também será solicitado que você crie um novo App ou conecte um App autônomo existente.

<div id="creating-or-connecting-an-app-for-your-project">
  #### Criando ou conectando um App ao seu Projeto
</div>

Se o seu Projeto não tem um App, você pode adicionar um clicando no nome do Projeto no painel. A partir daí, é possível criar um novo App ou selecionar um App independente existente para conectá-lo ao seu Projeto. No App, você pode gerar as chaves e os tokens de autenticação mencionados no início deste guia.

<div id="editing-a-project">
  #### Editando um Projeto
</div>

Para editar um Projeto, clique no nome do seu Projeto no painel ou na página Projetos e Apps, dentro do portal do desenvolvedor. Em seguida, você verá os detalhes do seu Projeto e poderá selecionar “editar” para fazer alterações.

<div id="standalone-apps">
  ### Apps independentes
</div>

Apps independentes são Apps que existem fora da estrutura de Projeto. As credenciais de autenticação associadas a esses Apps independentes podem fazer solicitações com êxito às APIs padrão v1.1, premium v1.1, enterprise do X ou à API de Anúncios do X. Apps independentes não conseguirão fazer solicitações aos endpoints da API do X v2, a menos que você os conecte a um Projeto.

Se você criou um App antes de agosto de 2020, ele ficará visível na seção "Apps independentes" do portal do desenvolvedor em [Projects & Apps](https://developer.twitter.com/en/portal/projects-and-apps). Você estará limitado a um total de dez Apps, incluindo aqueles que estão conectados ao seu Projeto.

Se você faz parte de uma conta de equipe, verá os Apps que você possui em Apps independentes. Se um colega possuir um App que faça parte de um Projeto, você poderá ver o nome do App e as informações do proprietário, mas não poderá alterar suas configurações, acessar suas chaves e tokens ou regenerá-los. Você deve entrar em contato com o proprietário do App para fazer quaisquer alterações no App dele.
