### POST oauth/request\_token

Permite que um aplicativo consumidor obtenha um OAuth Request Token para solicitar autorização do usuário. Este método cumpre a [Seção 6.1](https://oauth.net/core/1.0/#auth_step1) do [fluxo de autenticação OAuth 1.0](http://oauth.net/core/1.0/#anchor9).

**É obrigatório usar HTTPS em todas as etapas de autorização OAuth.**

**Nota de uso:** Apenas valores ASCII são aceitos para o `oauth_nonce`.

**Resource URL[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth/request_token`

**Resource Information[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Formatos de resposta | JSON |
| Requer autenticação? | Não |
| Sujeito a rate limit? | Sim |

**Parameters[](#parameters "Permalink to this headline")**

| Nome | Obrigatório | Descrição | Exemplo |
| :--- | :--- | :--- | :--- |
| oauth\_callback | obrigatório | Para conformidade com OAuth 1.0a, este parâmetro é **obrigatório**. O valor especificado aqui será usado como a URL para a qual o usuário será redirecionado caso aprove o acesso do seu aplicativo à conta. Defina como `oob` para o modo de PIN fora de banda. Esta também é a forma de especificar callbacks personalizados para uso em aplicativos desktop/dispositivos móveis. Sempre envie um `oauth_callback` nesta etapa, independentemente de haver um callback pré-registrado.<br /><br />É obrigatório que qualquer URL de callback usada com este endpoint esteja configurada nas configurações do App em developer.x.com\* | `http://themattharris.local/auth.php` `twitterclient://callback` |
| x\_auth\_access\_type | opcional | Substitui o nível de acesso que um aplicativo solicita à conta de um usuário. Os valores aceitos são `read` ou `write`. Este parâmetro permite que um desenvolvedor registre um aplicativo de leitura/gravação, mas solicite acesso somente leitura quando apropriado. |     |

Saiba mais sobre como aprovar suas URLs de callback nesta [página](/pt-BR/resources/fundamentals/developer-apps#callback-urls).

**Atenção** - Você pode visualizar e editar seus [apps do X](/pt-BR/resources/fundamentals/developer-apps) existentes pelo [painel de apps do X](https://developer.x.com/en/apps) se estiver conectado à sua conta do X em developer.x.com.

**Example request[](#example-request "Permalink to this headline")**

URL da solicitação: `POST https://api.x.com/oauth/request_token`

Corpo POST da solicitação: _N/D_

Cabeçalho de autorização: `OAuth oauth_nonce="K7ny27JTpKVsTgdyLdDfmQQWVLERj2zAK5BslRsqyw", oauth_callback="http%3A%2F%2Fmyapp.com%3A3005%2Ftwitter%2Fprocess_callback", oauth_signature_method="HMAC-SHA1", oauth_timestamp="1300228849", oauth_consumer_key="OqEqJeafRSF11jBMStrZz", oauth_signature="Pc%2BMLdv028fxCErFyi8KXFM%2BddU%3D", oauth_version="1.0"`

Resposta: `oauth_token=Z6eEdO8MOmk394WozF5oKyuAv855l4Mlqo7hhlSLik&oauth_token_secret=Kd75W4OQfb2oJTV0vzGzeXftVAwgMnEK9MumzYcM&oauth_callback_confirmed=true`

<div id="get-oauthauthorize">
  ### GET oauth/authorize
</div>

Permite que um aplicativo Consumer use um OAuth Request Token para solicitar a autorização do usuário. Este método cumpre a [Seção 6.2](http://oauth.net/core/1.0/#auth_step2) do [fluxo de autenticação OAuth 1.0](http://oauth.net/core/1.0/#anchor9). Aplicativos desktop devem usar este método (e não podem usar [GET oauth / authenticate](/pt-BR/resources/fundamentals/authentication/api-reference#get-oauth-authenticate)).

Observação de uso: um `oauth_callback` nunca é enviado para este método; em vez disso, forneça-o em [POST oauth / request\_token](/pt-BR/resources/fundamentals/authentication/api-reference#post-oauth-request-token).

**Resource URL[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth/authorize`

**Resource Information[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Response formats | JSON |
| Requires authentication? | Yes |
| Rate limited? | Yes |

**Parameters[](#parameters "Permalink to this headline")**

|     |     |     |     |     |
| :--- | :--- | :--- | :--- | :--- |
| Name | Required | Description | Default Value | Example |
| force\_login | optional | Obriga o usuário a inserir suas credenciais para garantir que a conta correta seja autorizada. |     |     |
| screen\_name | optional | Preenche o campo de nome de usuário na tela de login do OAuth com o valor fornecido. |     |     |

**Example request[](#example-request "Permalink to this headline")**

Redirecione o usuário para a etapa `oauth/authorize` em um navegador, incluindo o parâmetro oauth\_token:
`https://api.x.com/oauth/authorize?oauth_token=Z6eEdO8MOmk394WozF5oKyuAv855l4Mlqo7hhlSLik`

<div id="get-oauthauthenticate">
  ### GET oauth/authenticate
</div>

Permite que um aplicativo Consumer use um `request_token` do OAuth para solicitar autorização do usuário.

Este método substitui a [Seção 6.2](http://oauth.net/core/1.0/#auth_step2) do [fluxo de autenticação OAuth 1.0](http://oauth.net/core/1.0/#anchor9) para aplicativos que usam o fluxo de autenticação com callback. O método usará o usuário atualmente conectado como a conta para autorização de acesso, a menos que o parâmetro `force_login` esteja definido como `true`.

Este método difere de [GET oauth/authorize](/pt-BR/resources/fundamentals/authentication/api-reference#get-oauth-authorize) porque, se o usuário já concedeu permissão ao aplicativo, o redirecionamento ocorrerá sem que ele precise aprovar o aplicativo novamente. Para habilitar esse comportamento, você deve ativar a configuração _Use Sign in with X_ no seu [registro do aplicativo](https://developer.x.com/apps).

**Resource URL[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth/authenticate`

**Resource Information[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Response formats | JSON |
| Requires authentication? | Sim |
| Rate limited? | Sim |

**Parameters[](#parameters "Permalink to this headline")**

|     |     |     |     |     |
| :--- | :--- | :--- | :--- | :--- |
| Name | Required | Description | Default Value | Example |
| force\_login | opcional | Obriga o usuário a inserir suas credenciais para garantir que a conta correta seja autorizada. |     | _true_ |
| screen\_name | opcional | Pré-preenche o campo de nome de usuário na tela de login do OAuth com o valor fornecido. |     |     |

**Example request[](#example-request "Permalink to this headline")**

Envie o usuário para a etapa `oauth/authenticate` em um navegador, incluindo um parâmetro oauth\_token:
`https://api.x.com/oauth/authenticate?oauth_token=Z6eEdO8MOmk394WozF5oKyuAv855l4Mlqo7hhlSLik`

### POST oauth/access\_token

Permite que um aplicativo consumidor troque o OAuth Request Token por um OAuth Access Token. Este método atende à [Seção 6.3](http://oauth.net/core/1.0/#auth_step3) do [fluxo de autenticação OAuth 1.0](http://oauth.net/core/1.0/#anchor9).

**URL do recurso[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth/access_token`

**Informações do recurso[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Formatos de resposta | JSON |
| Requer autenticação? | Sim |
| Possui limite de taxa? | Sim |

**Parâmetros[](#parameters "Permalink to this headline")**

|     |     |     |     |     |
| :--- | :--- | :--- | :--- | :--- |
| Nome | Obrigatório | Descrição | Valor padrão | Exemplo |
| oauth\_token | obrigatório | O `oauth_token` aqui deve ser o mesmo `oauth_token` retornado na etapa `request_token`. |     |     |
| oauth\_verifier | obrigatório | Se estiver usando o fluxo web do OAuth, defina este parâmetro como o valor de `oauth_verifier` retornado na URL de callback. Se estiver usando OAuth out-of-band, defina este valor como o PIN. Para conformidade com o OAuth 1.0a, este parâmetro é **obrigatório**. O OAuth 1.0a é rigidamente aplicado e aplicativos que não usarem o `oauth_verifier` não conseguirão concluir o fluxo do OAuth. |     |     |

**Exemplo de solicitação[](#example-request "Permalink to this headline")**

`POST https://api.x.com/oauth/access_token?oauth_token=qLBVyoAAAAAAx72QAAATZxQWU6P&oauth_verifier=ghLM8lYmAxDbaqL912RZSRjCCEXKDIzx`

A partir de PIN: `POST https://api.x.com/oauth/access_token?oauth_token=9Npq8AAAAAAAx72QBRABZ4DAfY9&oauth_verifier=4868795`

**Exemplo de resposta[](#example-response "Permalink to this headline")**

`oauth_token=6253282-eWudHldSbIaelX7swmsiHImEL4KinwaGloHANdrY&oauth_token_secret=2EEfA6BG5ly3sR3XjE0IBSnlQu4ZrUzPiYTmrkVU&user_id=6253282&screen_name=xapi`

### POST oauth/invalidate\_token

Permite que um aplicativo registrado revogue um OAuth `access_token` emitido, apresentando suas credenciais de cliente. Após a invalidação de um `access_token`, novas tentativas de criação gerarão um token de acesso diferente e o uso do token invalidado não será mais permitido.

**Resource URL[](#resource-url "Permalink to this headline")**

`https://api.x.com/1.1/oauth/invalidate_token`

**Resource Information[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Formatos de resposta | JSON |
| Requer autenticação? | Sim — contexto do usuário com os tokens de acesso que você deseja invalidar |
| Possui limite de taxa? | Sim |

**Example request[](#example-request "Permalink to this headline")**

```bash
        curl --request POST
          --url 'https://api.x.com/1.1/oauth/invalidate_token.json'
          --header 'authorization: OAuth oauth_consumer_key="CLIENT_KEY",
         oauth_nonce="AUTO_GENERATED_NONCE", oauth_signature="AUTO_GENERATED_SIGNATURE",
         oauth_signature_method="HMAC-SHA1", oauth_timestamp="AUTO_GENERATED_TIMESTAMP",
         oauth_token="ACCESS_TOKEN", oauth_version="1.0"'
```

**Example response[](#example-response "Permalink to this headline")**

```bash
        HTTP/1.1 200 OK
        Content-Type: application/json; charset=utf-8
        Content-Length: 127
        ...

        {"access_token":"ACCESS_TOKEN"}
```

**Example error response after token has been invalidated[](#example-error-response-after-token-has-been-invalidated "Permalink to this headline")**

```bash
        HTTP/1.1 401 Authorization Required
        ...

        {"errors": [{
          "code": 89,
          "message": "Invalid or expired token."}
        ]}
```

<div id="post-oauth2token">
  ### POST oauth2/token
</div>

Permite que um aplicativo registrado obtenha um token Bearer OAuth 2, que pode ser usado para fazer solicitações à API em nome do próprio aplicativo, sem contexto de usuário. Isso é chamado de [autenticação somente de aplicativo](/pt-BR/resources/fundamentals/authentication/oauth-2-0/application-only).

Um token Bearer pode ser invalidado usando oauth2/invalidate\_token. Depois que um token Bearer for invalidado, novas tentativas de criação gerarão um token Bearer diferente e o uso do token anterior não será mais permitido.

Apenas um token Bearer pode existir para um aplicativo; solicitações repetidas a este método retornarão o mesmo token já existente até que ele seja invalidado.

Respostas bem-sucedidas incluem uma estrutura JSON descrevendo o token Bearer concedido.

Os tokens recebidos por este método devem ser armazenados em cache. Se tentadas com muita frequência, as solicitações serão rejeitadas com HTTP 403 e código 99.

**Resource URL[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth2/token`

**Resource Information[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Response formats | JSON |
| Requires authentication? | Sim — Basic auth com sua chave de API como nome de usuário e o segredo da chave de API como senha |
| Rate limited? | Sim |

**Parameters[](#parameters "Permalink to this headline")**

|     |     |     |     |     |
| :--- | :--- | :--- | :--- | :--- |
| Name | Required | Description | Default Value | Example |
| grant\_type | required | Especifica o tipo de concessão solicitado pelo aplicativo. No momento, apenas _client\_credentials_ é permitido. Consulte [autenticação somente de aplicativo](/pt-BR/resources/fundamentals/authentication/oauth-2-0/application-only) para mais informações. |     | _client\_credentials_ |

**Example request[](#example-request "Permalink to this headline")**

```bash
    POST /oauth2/token HTTP/1.1
    Host: api.x.com
    User-Agent: My X App v1.0.23
    Authorization: Basic eHZ6MWV2R ... o4OERSZHlPZw==
    Content-Type: application/x-www-form-urlencoded;charset=UTF-8
    Content-Length: 29
    Accept-Encoding: gzip

    grant_type=client_credentials
```

**Example response:**

```bash
    HTTP/1.1 200 OK
    Status: 200 OK
    Content-Type: application/json; charset=utf-8
    ...
    Content-Encoding: gzip
    Content-Length: 140

    {"token_type":"bearer","access_token":"AAAA%2FAAA%3DAAAAAAAA"}
```

### POST oauth2/invalidate\_token

Permite que um aplicativo registrado revogue um Bearer Token do OAuth 2.0 emitido, apresentando suas credenciais de cliente. Depois que um Bearer Token for invalidado, novas tentativas de criação gerarão um Bearer Token diferente e o uso do token invalidado não será mais permitido.

Respostas bem-sucedidas incluem uma estrutura JSON descrevendo o Bearer Token revogado.

**URL do recurso[](#resource-url "Permalink to this headline")**

`https://api.x.com/oauth2/invalidate_token`

**Informações do recurso[](#resource-information "Permalink to this headline")**

|     |     |
| :--- | :--- |
| Formatos de resposta | JSON |
| Requer autenticação? | Sim — [OAuth 1.0a](/pt-BR/resources/fundamentals/authentication/oauth-1-0a) com as chaves de API do consumidor do aplicativo e o access token e access token secret do proprietário do aplicativo |
| Sujeito a limite de taxa? | Sim |

**Parâmetros[](#parameters "Permalink to this headline")**

| Nome | Obrigatório | Descrição |
| :--- | :--- | :--- |
| access\_token | obrigatório | O valor do bearer token que você deseja invalidar |

**Exemplo de solicitação[](#example-request "Permalink to this headline")**

```
        curl --request POST
          --url 'https://api.x.com/oauth2/invalidate_token?access_token=AAAA%2FAAA%3DAAAAAAAA'
          --header 'authorization: OAuth oauth_consumer_key="CLIENT_KEY",
         oauth_nonce="AUTO_GENERATED_NONCE", oauth_signature="AUTO_GENERATED_SIGNATURE",
         oauth_signature_method="HMAC-SHA1", oauth_timestamp="AUTO_GENERATED_TIMESTAMP",
         oauth_token="ACCESS_TOKEN", oauth_version="1.0"'
```

**Exemplo de resposta[](#example-response "Permalink to this headline")**

```
         Status: 200 OK
         Content-Type: application/json; charset=utf-8
         Content-Length: 135
         ...
       {
        "access_token": "AAAA%2FAAA%3DAAAAAAAA"
        }
```
