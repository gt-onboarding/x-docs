<div id="general">
  ## Geral
</div>

<AccordionGroup>
  <Accordion title="O que é OAuth?">
    OAuth é um protocolo de autenticação que permite que usuários autorizem um aplicativo a agir em seu nome sem compartilhar a senha. Mais informações estão disponíveis em [oauth.net](http://oauth.net/).
  </Accordion>

  <Accordion title="Como gero tokens de acesso?">
    Você precisa ter um [app do X](/pt-BR/resources/fundamentals/developer-apps) para gerar tokens de acesso. Saiba mais sobre tokens de acesso [aqui](/pt-BR/resources/fundamentals/authentication#oauth-1-0a-2).
  </Accordion>

  <Accordion title="Como crio um app do X?">
    Você precisa ter uma [conta de desenvolvedor](/pt-BR/resources/fundamentals/developer-portal) para criar um [app do X](/pt-BR/resources/fundamentals/developer-apps). Você pode se inscrever [aqui](https://developer.x.com/en/portal/petition/essential/basic-info).
  </Accordion>

  <Accordion title="Se eu já tenho um app do X, como visualizo e edito esse app?">
    Você pode visualizar e editar seu app no [painel do app do X](https://developer.x.com/content/developer-twitter/en/apps) se estiver conectado à sua conta do X em developer.x.com.
  </Accordion>
</AccordionGroup>

<div id="technical">
  ## Técnico
</div>

<AccordionGroup>
  <Accordion title="Por quanto tempo um token de acesso permanece válido?">
    Tokens de acesso não expiram explicitamente. Um token de acesso será invalidado se o usuário revogar explicitamente um aplicativo nas configurações da conta do X, ou se o X suspender um aplicativo. Se um aplicativo for suspenso, haverá uma nota no painel do [app do X](/pt-BR/resources/fundamentals/developer-apps) informando que ele foi suspenso.
  </Accordion>

  <Accordion title="E se um token de acesso se tornar inválido?">
    Considere que o token de acesso de um usuário _pode_ se tornar inválido a qualquer momento. Se isso acontecer, solicite que o usuário autorize o aplicativo novamente. Garantir que essa situação seja tratada de forma adequada é importante para uma boa experiência do usuário.
  </Accordion>

  <Accordion title="A página de registro do aplicativo pergunta sobre acesso de leitura/gravação. O que é considerado gravação?">
    Muitos usuários confiam em um aplicativo para ler suas informações, mas não necessariamente para alterar informações do perfil ou publicar novos status. Atualizar informações pela API do X — seja nome, localização ou adicionar um novo status — requer um HTTP POST. Qualquer método da API que exija um HTTP POST é considerado um método de gravação e requer acesso de leitura e gravação.
  </Accordion>
</AccordionGroup>
