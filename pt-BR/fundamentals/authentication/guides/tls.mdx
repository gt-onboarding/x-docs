Conexões TLS são obrigatórias para acessar os endpoints da API do X. A comunicação via TLS preserva a privacidade e a segurança do usuário ao proteger as informações entre o usuário e a API do X enquanto elas trafegam pela Internet pública. As conexões com a API do X exigem TLS na versão 1.2.


<div id="verification">
  ## Verificação
</div>

<div id="use-an-up-to-date-root-store">
  ### Use um repositório de raízes atualizado
</div>

É importante que seu aplicativo ou biblioteca use um repositório de raízes confiável e atualizado ao verificar o certificado do X. Quando possível, usar o repositório de raízes fornecido pelo sistema operacional pode ser a abordagem mais simples. Alternativamente, o [repositório de raízes da Mozilla (NSS)](https://www.mozilla.org/en-US/about/governance/policies/security-group/certs/) é mantido de forma pública e transparente. O curl também fornece [uma versão desse repositório em formato PEM](https://curl.haxx.se/docs/caextract.html).

Atualmente, o X emite a maior parte de nossos certificados a partir da [DigiCert High Assurance EV Root CA](https://www.digicert.com/digicert-root-certificates.htm), mas isso não é verdade para 100% dos certificados relacionados ao X e pode não permanecer assim para sempre; portanto, confiar apenas nas raízes da DigiCert atualmente usadas pode causar problemas com seu app no futuro.

<div id="check-crls-and-the-ocsp-status">
  ### Verifique as CRLs e o status do OCSP[](#check-crls-and-the-ocsp-status "Permalink to this headline")
</div>

Muitos aplicativos não verificam a Lista de Revogação de Certificados dos certificados recebidos ou dependem do sistema operacional para fazê-lo. Certifique-se de que seu aplicativo ou biblioteca TLS esteja configurado para exigir a verificação de CRL e OCSP (Online Certificate Status Protocol) antes de aceitar o certificado do X.

<div id="cdns">
  ### CDNs[](#cdns "Link permanente para este título")
</div>

Ao exibir Tweets que contêm mídia, use o atributo `media_url_https` para obter as URLs HTTPS ao mostrar imagens. No futuro, todas as URLs retornadas pelos endpoints da API fornecerão caminhos HTTPS.

<div id="provide-an-indication-of-security-status">
  ## Indique o status de segurança
</div>

Sempre que possível, mostre o status atual da conexão entre seu aplicativo e o X. Alguns navegadores da web indicam isso exibindo um ícone de cadeado, enquanto outros mostram o estado da conexão com mensagens descritivas.
