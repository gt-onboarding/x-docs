<div id="bearer-token-also-known-as-app-only">
  ### Token Bearer (também conhecido como somente app)
</div>

O Token Bearer do OAuth 2.0 autentica solicitações em nome do seu [App de desenvolvedor](/pt-BR/resources/fundamentals/developer-apps). Como esse método é específico do App, ele não envolve usuários. Esse método é normalmente usado por desenvolvedores que precisam de acesso somente leitura a informações públicas.

Esse método de autenticação requer que você envie um Token Bearer com sua solicitação, o qual pode ser gerado na seção Keys and tokens dos seus Apps de desenvolvedor. Veja um exemplo de como fica uma solicitação com um Token Bearer fictício:

```json
  curl "https://api.x.com/2/tweets?ids=1261326399320715264,1278347468690915330" \
  -H "Authorization: Bearer AAAAAAAAAAAAAAAAAAAAAFnz2wAAAAAAxTmQbp%2BIHDtAhTBbyNJon%2BA72K4%3DeIaigY0QBrv6Rp8KZQQLOTpo9ubw5Jt?WRE8avbi"
```

Chamadas à API usando autenticação somente com app são [limitadas por taxa](/pt-BR/resources/fundamentals/rate-limits) por endpoint no nível do App.

Para usar esse método, você precisará de um Token Bearer, que pode ser gerado passando sua API Key e Secret pelo endpoint [POST oauth2/token](/pt-BR/resources/fundamentals/authentication/api-reference#post-oauth2-token), ou gerando-o na seção "keys and tokens" das configurações do seu App no [portal do desenvolvedor](/pt-BR/resources/fundamentals/developer-portal).

Se você quiser revogar um Token Bearer, pode usar o endpoint [POST oauth2/invalidate\_token](/pt-BR/resources/fundamentals/authentication/api-reference#post-oauth2-invalidate-token), ou clicar em "revoke" ao lado do Token Bearer na seção "keys and tokens" das configurações do seu App.

<div id="oauth-20-authorization-code-flow-with-pkce">
  ### Fluxo de Código de Autorização OAuth 2.0 com PKCE
</div>

O Fluxo de Código de Autorização OAuth 2.0 com PKCE permite autenticar em nome de outro usuário, oferecendo mais controle sobre os escopos do aplicativo e melhorando os fluxos de autorização em vários dispositivos. Em outras palavras, desenvolvedores que criam aplicativos para pessoas no X terão mais controle sobre as informações que seus apps solicitam de seus usuários, para que você só precise pedir aos seus usuários finais os dados e informações de que realmente precisa.

Esse protocolo moderno de autorização permite apresentar aos seus usuários finais um fluxo de consentimento mais enxuto para autorizar seu app, exibindo apenas os escopos específicos que você solicitou. Isso não apenas reduz sua carga de dados, como também pode aumentar a confiança dos usuários finais.
