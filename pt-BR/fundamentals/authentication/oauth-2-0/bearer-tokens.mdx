<div id="using-and-generating-an-app-only-bearer-token">
  ### Usando e gerando um Bearer Token somente para app
</div>

Um Bearer Token permite que desenvolvedores tenham um ponto de entrada mais seguro para usar as APIs do X e é um dos recursos centrais do OAuth 2.0.

A autenticação que usa um Bearer Token também é conhecida como autenticação somente de aplicativo. Um Bearer Token é um vetor de bytes de formato não especificado que você gera usando um script, como um comando curl. Você também pode obter um Bearer Token no portal do desenvolvedor, na seção de chaves e tokens das configurações do seu App. Mais informações sobre esse recurso podem ser encontradas na [documentação oficial do OAuth](https://oauth.net/2/bearer-tokens/).

<div id="when-are-they-used">
  #### Quando eles são usados?
</div>

Os produtos que exigem o uso de um token Bearer são os seguintes:

- [Engagement API](/pt-BR/x-api/enterprise-gnip-2.0/fundamentals/engagement-api)
- [Account Activity API](/pt-BR/x-api/enterprise-gnip-2.0/fundamentals/account-activity)
- Outras APIs que usam autenticação OAuth 2.0 com token Bearer, como os endpoints v2 e Labs.

<div id="prerequisites">
  #### Pré-requisitos
</div>

Você precisará [criar uma conta de desenvolvedor](https://developer.x.com/en/portal/petition/essential/basic-info) e ter criado um [X App](/pt-BR/resources/fundamentals/developer-apps). Depois disso, você também precisará obter as chaves de API disponíveis no [portal do desenvolvedor](/pt-BR/resources/fundamentals/developer-portal). Siga as etapas abaixo:

1. Acesse sua conta do X em developer.x.com.
2. Vá até o [painel do X App](https://developer.x.com/content/developer-twitter/en/apps) e abra o X App para o qual você deseja gerar tokens de acesso.
3. Acesse a página “keys and tokens”.
4. Nessa página, você encontrará as chaves de API, os tokens de acesso do usuário e o Bearer Token.

<div id="how-to-generate-a-bearer-token">
  ### Como gerar um Bearer Token
</div>

Você pode encontrar o Bearer Token do seu App junto com as demais “Keys and Tokens”.

Copie a solicitação cURL abaixo para o seu terminal após inserir as seguintes chaves de API de consumidor previamente obtidas no seu [X App](/pt-BR/resources/fundamentals/developer-apps). Observe que as chaves de API de consumidor usadas nesta página foram descontinuadas e não funcionarão em solicitações reais.

- **API key** `<API key>`, por exemplo: `xvz1evFS4wEEPTGEFPHBog`
- **API secret key** `<API secret key>`, por exemplo: `L8qq9PZyRg6ieKGEKhZolGC0vJWLw8iEJ88DRdyOg`

  ```bash
  curl -u "$API_KEY:$API_SECRET_KEY" \
  --data 'grant_type=client_credentials' \
  'https://api.x.com/oauth2/token'
  ```

Veja um exemplo de como a solicitação cURL deve ficar com suas chaves de API inseridas:

```bash
   curl -u 'xvz1evFS4wEEPTGEFPHBog:L8qq9PZyRg6ieKGEKhZolGC0vJWLw8iEJ88DRdyOg' \
  --data 'grant_type=client_credentials' \
  'https://api.x.com/oauth2/token'
```

A resposta seria semelhante a esta. Observe que este Bearer Token foi descontinuado:

`{"token_type":"bearer","access_token":"AAAAAAAAAAAAAAAAAAAAAMLheAAAAAAA0%2BuSeid%2BULvsea4JtiGRiSDSJSI%3DEUifiRBkKG5E2XzMDjRfl76ZC9Ub0wnz4XsNiRVBChTYbJcE3F"}`

O Bearer Token usado para autenticar recursos com OAuth 2.0 seria:

`AAAAAAAAAAAAAAAAAAAAAMLheAAAAAAA0%2BuSeid%2BULvsea4JtiGRiSDSJSI%3DEUifiRBkKG5E2XzMDjRfl76ZC9Ub0wnz4XsNiRVBChTYbJcE3F`
