As APIs do X lidam com enormes volumes de dados. Garantimos a segurança desses dados para desenvolvedores e usuários por meio de autenticação. Existem alguns métodos de autenticação, listados abaixo.

A maioria dos desenvolvedores não precisará lidar com as complexidades da autenticação, pois as bibliotecas cliente tratam automaticamente dessas questões.

Você pode encontrar uma lista de bibliotecas cliente disponíveis na nossa página de [Ferramentas e bibliotecas](/pt-BR/resources/tools-and-libraries).

<div id="authentication-methods">
  ## Métodos de autenticação
</div>

<CardGroup cols={2}>
  <Card title="Contexto de usuário OAuth 1.0a" href="/pt-BR/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret">
    O OAuth 1.0a permite que um app de desenvolvedor do X autorizado acesse informações privadas de uma conta ou execute uma ação no X em nome de uma conta do X.

    <br />

    [**Saiba mais**](/pt-BR/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret)
  </Card>

  <Card title="Somente app" href="/pt-BR/resources/fundamentals/authentication/oauth-2-0/overview">
    O App-only Access Token permite que um app de desenvolvedor do X acesse informações publicamente disponíveis no X.

    <br />

    [**Saiba mais**](/pt-BR/resources/fundamentals/authentication/oauth-2-0/overview)
  </Card>

  <Card title="Autenticação básica" href="/pt-BR/resources/fundamentals/authentication/basic-auth">
    Muitas APIs empresariais do X exigem o uso de autenticação básica HTTP.

    <br />

    [**Saiba mais**](/pt-BR/resources/fundamentals/authentication/basic-auth)
  </Card>

  <Card title="Fluxo de código de autorização OAuth 2.0 com PKCE" href="/pt-BR/resources/fundamentals/authentication/oauth-2-0/authorization-code">
    O contexto de usuário do OAuth 2.0 permite autenticar em nome de outra conta, com maior controle sobre o escopo do aplicativo e fluxos de autorização em vários dispositivos.

    <br />

    [**Saiba mais**](/pt-BR/resources/fundamentals/authentication/oauth-2-0/authorization-code)
  </Card>
</CardGroup>

<Note>
  **Observação:**
  As chaves de API do seu app e o App-only Access Token, assim como seu Access Token pessoal e o Access Token Secret, podem ser obtidos na seção [Apps de desenvolvedor do X](/pt-BR/resources/fundamentals/developer-apps) no [portal de desenvolvedores](/pt-BR/resources/fundamentals/developer-portal).

  **Se você deseja fazer solicitações em nome de outro usuário**, será necessário gerar um conjunto separado de tokens de acesso para esse usuário usando o [fluxo OAuth de 3 etapas](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens) e enviar os tokens desse usuário junto com suas solicitações no contexto de usuário OAuth 1.0a ou no contexto de usuário OAuth 2.0.
</Note>
