<div id="api-key-and-secret">
  ### Chave e segredo da API
</div>

A chave e o segredo da API (também conhecidos como Consumer Key e Consumer Secret) são as credenciais mais básicas necessárias para acessar a API do X. Essas credenciais funcionam como o nome de usuário e a senha do seu app do X e são usadas pela API do X para identificar de qual app as solicitações se originam.

Essas credenciais podem ser usadas pelos [endpoints de autenticação](/pt-BR/resources/fundamentals/authentication/api-reference) para gerar credenciais adicionais, como [tokens e segredos de acesso do usuário](/pt-BR/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret) e [tokens Bearer](/pt-BR/resources/fundamentals/authentication/oauth-2-0/bearer-tokens). Você também precisa usar essas credenciais juntamente com tokens de acesso e outros parâmetros de autorização para [autorizar solicitações](/pt-BR/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request) que exigem autenticação OAuth 1.0a com contexto de usuário. 

<div id="how-to-acquire-an-api-key-and-secret">
  #### Como obter uma API Key e Secret
</div>

Para obter uma X API Key e Secret, siga estas etapas:

1. [Crie uma conta de desenvolvedor do X](https://developer.x.com/en/apply-for-access)
2. Crie um [X App](/pt-BR/resources/fundamentals/developer-apps) no [developer portal](/pt-BR/resources/fundamentals/developer-portal). Observe que, se você quiser usar a [API do X v2](/pt-BR/x-api/introduction), deverá adicionar seu X App a um [Project](/pt-BR/resources/fundamentals/projects).
    

Ao criar seu X App, você verá sua API Key e Secret, juntamente com um Bearer Token. Observe que exibimos essas credenciais apenas uma vez; portanto, certifique-se de salvá-las no seu gerenciador de senhas ou em um local seguro.

Temos mais recomendações sobre como lidar com suas keys e tokens na nossa página de [melhores práticas de autenticação](/pt-BR/resources/fundamentals/authentication/guides/authentication-best-practices), incluindo detalhes sobre o que fazer caso suas credenciais tenham sido comprometidas.
 

<div id="how-to-find-and-regenerate-your-api-key-and-secret-after-app-creation">
  #### Como localizar e regenerar sua API Key e Secret após criar o App
</div>

Se você já criou um App e precisa localizar ou regenerar sua API Key e Secret, siga estas etapas:

1. Acesse o portal do desenvolvedor
2. Expanda o menu suspenso “Projetos e Apps” na barra lateral
3. Abra o App associado à API Key e Secret que você deseja localizar ou regenerar
4. Vá até a guia Keys and tokens

A partir daí, você verá todas as credenciais associadas ao seu App. 
 

<div id="how-to-use-your-api-key-and-secret">
  #### Como usar sua API Key e Secret
</div>

Se você está apenas explorando a X Developer Platform, recomendamos usar uma [ferramenta ou biblioteca](/pt-BR/resources/tools-and-libraries) para ver o que está disponível na plataforma. Essas ferramentas tratam a autenticação de forma eficiente e podem economizar muito tempo e frustração. Recomendamos especificamente [começar com o Postman](/pt-BR/tutorials/postman-getting-started) ou o [Insomnia](https://insomnia.rest/) para desenvolvedores iniciantes.

Se você estiver interessado em criar uma requisição do zero, leia nosso guia sobre [como autorizar uma requisição OAuth 1.0a](/pt-BR/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request).
