<div id="oauth-10a">
  ## OAuth 1.0a
</div>

Muitos endpoints na plataforma de desenvolvedores do X usam o método OAuth 1.0a para atuar, ou fazer solicitações à API, em nome de uma conta do X. Por exemplo, se você tem um app de desenvolvedor do X, pode fazer solicitações à API em nome de qualquer conta do X, desde que esse usuário autorize seu app.

Observação: se você não estiver familiarizado com conceitos como HMAC-SHA1 e percent-encoding, recomendamos consultar a seção “ferramentas úteis” abaixo, que lista alguns clients de API que simplificam bastante o processo de autenticação.

<div id="key-concepts">
  ### Conceitos-chave
</div>

<div id="signing-a-request-with-keys-and-tokens">
  #### Assinando uma solicitação com chaves e tokens
</div>

Você deve assinar cada solicitação à API, passando várias chaves e tokens gerados no cabeçalho de autorização. Para começar, você pode gerar várias chaves e tokens na página de detalhes do seu [app de desenvolvedor do X](/pt-BR/resources/fundamentals/developer-apps), incluindo:

|     |     |
| :--- | :--- |
| Chave e segredo da API:<br /><br />`oauth_consumer_key`<br /><br />`oauth_consumer_secret` | Pense neles como o nome de usuário e a senha que representam seu app de desenvolvedor do X ao fazer solicitações à API. |
| Token de acesso e segredo:<br /><br />`oauth_token`<br /><br />`oauth_token_secret` | Um token de acesso e um segredo de token de acesso são credenciais específicas do usuário usadas para autenticar solicitações à API com OAuth 1.0a. Eles especificam a conta do X em nome da qual a solicitação é feita.<br /><br />Você pode gerar seu próprio token de acesso e segredo de token se quiser que seu app faça solicitações em nome da mesma conta do X associada à sua conta de desenvolvedor na página de detalhes do seu app de desenvolvedor do X.<br /><br />Se quiser gerar tokens de acesso para um usuário diferente, consulte “Fazendo solicitações em nome de usuários” abaixo. |

<div id="making-requests-on-behalf-of-users">
  #### Fazendo solicitações em nome de usuários
</div>

Ao criar uma assinatura, você precisa de um conjunto de tokens de acesso que representem o usuário em cujo nome você fará a solicitação.

Você pode gerar um conjunto de tokens de acesso que representa a conta do X proprietária do app de desenvolvedor do X na [página de detalhes do app,](https://developer.x.com/content/developer-twitter/en/apps) mas, se quiser fazer uma solicitação em nome de outra conta do X, o proprietário dessa conta deve conceder acesso a você, fazendo login na conta como parte do [fluxo OAuth em 3 etapas](/pt-BR/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow). O resultado desse processo é um conjunto de tokens de acesso (oauth\_token e oauth\_token\_secret) que pode ser usado para fazer uma solicitação OAuth 1.0a.

Depois que você tiver essas chaves e tokens, pode [criar uma assinatura](/pt-BR/resources/fundamentals/authentication#creating-a-signature) do zero. Não recomendamos isso a menos que você saiba exatamente o que está fazendo, a não ser que esteja usando uma das ferramentas mencionadas abaixo para fazer uma solicitação a um endpoint que exija OAuth 1.0a.

Para referência, segue um exemplo de uma solicitação cURL assinada com todos os tokens gerados incluídos no cabeçalho de autorização:

```bash
curl --request POST \
  --url 'https://api.x.com/1.1/statuses/update.json?status=Hello%20world' \
  --header 'authorization: OAuth oauth_consumer_key="CONSUMER_API_KEY", oauth_nonce="OAUTH_NONCE", oauth_signature="OAUTH_SIGNATURE", oauth_signature_method="HMAC-SHA1", oauth_timestamp="OAUTH_TIMESTAMP", oauth_token="ACCESS_TOKEN", oauth_version="1.0"' \
```

<Note>
  **Observação:** Tokens de acesso de usuário são confidenciais e devem ser protegidos com muito cuidado. Quando tokens de acesso são gerados, o usuário que eles representam está confiando que seu aplicativo os manterá seguros. Se a segurança tanto das chaves de API quanto dos tokens de acesso do usuário for comprometida, seu aplicativo poderá expor acesso a informações privadas e funcionalidades da conta. Recomendamos que você saiba mais sobre [proteger chaves e tokens de acesso](/pt-BR/resources/fundamentals/developer-apps).
</Note>

<div id="useful-tools">
  ### Ferramentas úteis
</div>

O processo de [assinar uma solicitação](/pt-BR/resources/fundamentals/authentication#creating-a-signature) é complexo. Recomendamos usar uma biblioteca cliente de API que gere automaticamente grande parte do token de autenticação:

|     |     |
| :--- | :--- |
| [Postman](https://www.getpostman.com/) | Um cliente de API que permite criar e enviar solicitações REST. Leia nosso “[Introdução ao Postman](/pt-BR/tutorials/postman-getting-started)” tutorial para saber mais sobre esta ferramenta. |
| [Insomnia](https://insomnia.rest/) | O Insomnia é um cliente de API REST com gerenciamento de cookies, variáveis de ambiente, geração de código e autenticação para Mac, Windows e Linux. |
