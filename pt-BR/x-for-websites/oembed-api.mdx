Você pode usar a API oEmbed para retornar, de forma programática, conteúdo incorporado, como [posts](https://developer.x.com/en/docs/twitter-for-websites/embedded-tweets/overview) e [linhas do tempo](https://developer.x.com/en/docs/twitter-for-websites/timelines/overview). 

A resposta da API oEmbed retorna um trecho de HTML que será reconhecido automaticamente quando [o JavaScript de widgets do X estiver incluído na página](https://developer.x.com/web/javascript/loading).

Observe que a API é recomendada para execução de tarefas em lote; para incorporar conteúdo, recomendamos usar nossa ferramenta robusta [publish.x.com](https://publish.x.com/#).

<Tabs>
  <Tab title="Embedded timelines">
    O trecho de HTML retornado será reconhecido automaticamente como uma [linha do tempo incorporada](https://developer.x.com/en/docs/twitter-for-websites/timelines/overview) quando [o JavaScript de widget do X estiver incluído na página](https://developer.x.com/web/javascript/loading).

    O endpoint oEmbed permite personalizar a aparência final de uma linha do tempo incorporada definindo as propriedades correspondentes na marcação HTML, que serão interpretadas pelo JavaScript do X incluído por padrão com a resposta HTML. O formato da marcação retornada pode mudar ao longo do tempo, à medida que o X adiciona novos recursos ou ajusta sua representação da linha do tempo.

    Para uma linha do tempo do X especificada pela URL da linha do tempo, em um formato JSON compatível com [oEmbed](https://oembed.com/). Linhas do tempo de usuário e de lista são compatíveis. A marcação da linha do tempo deve ser armazenada em cache nos seus servidores por até o tempo de vida sugerido especificado pela propriedade cache\_age.

    ## Resource URL

    _https://publish.x.com/oembed_

    ## Resource Information

    |     |     |
    | :--- | :--- |
    | Response formats | JSON |
    | Requires authentication? | No  |
    | Rate limited | No  |

    ## Parameters

    | Name | Description | Example |
    | :--- | :--- | :--- |
    | **url** | A URL da linha do tempo do X a ser incorporada | \*   https://x.com/TwitterDev<br /> _https://x.com/TwitterDev/lists/national-parks
    | limit | Exibir até N itens, em que N é um valor entre 1 e 20 inclusive | 6   |
    | maxwidth | Define a largura máxima do widget. Deve estar entre 180 e 1200 inclusive | 300 |
    | maxheight | Define a altura máxima do widget. Deve ser maior que 200 | 400 |
    | omit\_script | Não incluir um elemento script na resposta | 1   |
    | lang | Um [código de idioma](/pt-BR/en/docs/twitter-for-websites/twitter-for-websites-supported-languages/overview "Twitter language code") do X compatível | es  |
    | theme | Quando definido como dark, a linha do tempo é exibida com texto claro sobre um fundo escuro | dark |
    | chrome | Remove um componente de exibição da linha do tempo com tokens separados por espaço<br /><br />_   noheader - oculta o cabeçalho<br />\*   nofooter - oculta o rodapé, se visível<br />\*   noborders - remove todas as bordas: ao redor do widget, entre Tweets e dentro de um Tweet<br />\*   noscrollbar - corta e oculta a barra de rolagem da linha do tempo, se visível<br />\*   transparent - remove a cor de fundo | noheader%20nofooter |
    | aria\_polite | Define um valor assertivo de [polidez de região ao vivo ARIA](https://www.w3.org/TR/wai-aria/states_and_properties#aria-live) para Tweets adicionados a uma linha do tempo | assertive |
    | dnt | Quando definido como true, a linha do tempo e sua página incorporada no seu site não são usadas para finalidades que incluem [sugestões personalizadas](https://support.x.com/articles/20169421) e [anúncios personalizados](https://support.x.com/articles/20170405) | true |

    ## Example Requests

    ```bash
    curl --request GET --url 'https://publish.x.com/oembed?url=https%3A%2F%2Ftwitter.com%2FInterior%2Fstatus%2F507185938620219395'
    twurl -H publish.x.com "/oembed?url=https://x.com/Interior/status/463440424141459456"
    ```

    ```json
    }
    ```

  <Tab title="Embedded Tweets">
    O trecho de HTML retornado será reconhecido automaticamente como um [Tweet incorporado](https://developer.x.com/web/embedded-tweets) quando [o JavaScript de widgets do X estiver incluído na página](https://developer.x.com/web/javascript/loading).

    O endpoint oEmbed permite personalizar a aparência final de um Tweet incorporado definindo as propriedades correspondentes na marcação HTML, que serão interpretadas pelo JavaScript do X incluído na resposta HTML por padrão. O formato da marcação retornada pode mudar ao longo do tempo, à medida que o X adiciona novos recursos ou ajusta sua representação de Tweets.

    A marcação de fallback de Tweet deve ser mantida em cache nos seus servidores por até o tempo de vida de cache recomendado, conforme especificado pela propriedade `cache_age`.

    ## URL do recurso

    _https://publish.x.com/oembed_

    ## Informações do recurso

    |     |     |
    | :--- | :--- |
    | Formatos de resposta | JSON |
    | Exige autenticação? | Não  |
    | Sujeito a limite de taxa | Não  |

    ## Parâmetros

    | Nome | Padrão | Descrição |
    | :--- | :--- | :--- |
    | `url` obrigatório  <br />String |     | A URL do Post a ser incorporado |
    | `maxwidth`  <br />Int `[220..550]` | `325` | A largura máxima de um Post renderizado em pixels inteiros. Um valor informado abaixo ou acima da faixa permitida será ajustado para a largura mínima ou máxima compatível, respectivamente; o valor de largura redefinido será refletido na propriedade `width` retornada. Observe que o X não oferece suporte ao parâmetro `maxheight` do oEmbed. Posts são essencialmente texto e, portanto, têm altura imprevisível que não pode ser escalonada como uma imagem ou vídeo. Da mesma forma, a resposta do oEmbed não fornecerá um valor para `height`. Implementações que precisam de alturas consistentes para Posts devem consultar os parâmetros `hide_thread` e `hide_media` abaixo. |
    | `hide_media`  <br />Boolean, String ou Int | `false` | Quando definido como `true`, `"t"` ou `1`, links em um Post não são expandidos para prévias de foto, vídeo ou de link. |
    | `hide_thread`  <br />Boolean, String ou Int | `false` | Quando definido como `true`, `"t"` ou `1`, uma versão recolhida do Post anterior em um fio de conversa não será exibida quando o Post solicitado for uma resposta a outro Post. |
    | `omit_script`  <br />Boolean, String ou Int | `false` | Quando definido como `true`, `"t"` ou `1`, o `<script>` responsável por carregar `widgets.js` não será retornado. Suas páginas da web devem incluir sua própria referência a `widgets.js` para uso em todos os widgets do X, incluindo [Posts incorporados](https://developer.x.com/web/embedded-tweets). |
    | `align`  <br />Enum `{left,right,center,none}` | `none` | Especifica se o Post incorporado deve ser alinhado à esquerda, à direita ou ao centro na página em relação ao elemento pai. |
    | `lang`  <br />Enum([Idiomas](https://developer.x.com/en/docs/twitter-for-websites/twitter-for-websites-supported-languages/overview)) | `en` | Solicita HTML e um Post renderizado no [idioma do X compatível com Posts incorporados](https://developer.x.com/web/overview/languages) especificado. |
    | `theme`  <br />Enum `{light, dark}` | `light` | Quando definido como `dark`, o Post é exibido com texto claro sobre fundo escuro. |
    | `dnt`  <br />Boolean | `false` | Quando definido como `true`, o Post e sua página incorporada no seu site não são usados para finalidades que incluem [sugestões personalizadas](https://support.x.com/articles/20169421) e [anúncios personalizados](https://support.x.com/articles/20170405). |
