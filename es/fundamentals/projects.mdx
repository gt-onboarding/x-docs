<div id="introduction">
  ## Introducción
</div>

Los Proyectos te permiten organizar tu trabajo según cómo planees usar la API de X, para gestionar eficazmente tu acceso a la API y supervisar tu uso. Cada Proyecto puede contener una o varias [Apps](/es/resources/fundamentals/developer-apps), según tu nivel de acceso (descrito más adelante en esta página). Usarás estas Apps para generar credenciales de [autenticación](/es/resources/fundamentals/authentication), como [Claves y Secretos de API](/es/resources/fundamentals/authentication#api-key-and-secret), [Tokens de acceso](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) de usuario y [Token de acceso de App](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only).

Si bien puedes usar estas claves y tokens de cualquier App para acceder a la API de X o a la API de Anuncios de X (debes [solicitar acceso adicional](/es/x-ads-api/getting-started/step-by-step-guide) para usar la API de Anuncios), debes utilizar claves y tokens de una App asociada a un Proyecto para poder usar los endpoints de la [API de X v2](/es/x-api/introduction).

Si tienes una cuenta de desarrollador, puedes ver y administrar tus Proyectos en la página [Projects & Apps](https://developer.x.com/en/portal/projects-and-apps) dentro del portal para desarrolladores. [Regístrate](https://developer.x.com/en/portal/petition/essential/basic-info) para obtener una cuenta si aún no tienes una.

<div id="projects-and-x-api-access-levels">
  ### Proyectos y niveles de acceso de la API de X
</div>

En este momento, hay 4 niveles de acceso diferentes que se aplican a nivel de proyecto:

- Free
- Basic
- Pro
- Enterprise

Para obtener más información sobre lo que ofrece cada uno de estos niveles de acceso, visite la página [sobre la API de X](/es/x-api/getting-started/about-x-api).

Solo puede tener un proyecto con Free, Basic o Pro.

A continuación, describiremos algunas diferencias específicas del proyecto:

<div id="free">
  #### Gratis
</div>

Este nivel de acceso se ofrece a cualquier persona que se haya [registrado](https://developer.twitter.com/en/portal/petition/essential/basic-info) para obtener una cuenta de desarrollador.

Número de Apps dentro de ese Proyecto: 1
[Límite de consumo de tuits](/es/x-api/fundamentals/post-cap): 1,500 tuits por mes

Métodos de autenticación:

- [OAuth 2.0 con PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
- Solo App

<div id="basic">
  #### Básico
</div>

Este nivel de acceso está disponible para cualquiera que se haya suscrito al acceso Básico a través del portal para desarrolladores.

Número de Apps dentro de ese Proyecto: 2
[Límite de consumo de tuits](/es/x-api/fundamentals/post-cap): 10.000 tuits por mes
Métodos de autenticación:

- [OAuth 2.0 con PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
- [OAuth 1.0a](/es/resources/fundamentals/authentication#oauth-1-0a-2)
- Solo app

<div id="pro">
  #### Pro
</div>

Este nivel de acceso se ofrece a cualquier persona suscrita al plan Pro a través del portal para desarrolladores.

Número de Apps dentro de ese Proyecto: 3
[Límite de consumo de tuits](/es/x-api/fundamentals/post-cap): 1 millón de tuits por mes

Métodos de autenticación:

- [OAuth 2.0 con PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
- [OAuth 1.0a](/es/resources/fundamentals/authentication#oauth-1-0a-2)
- Solo App

<div id="enterprise">
  #### Enterprise
</div>

Este nivel de acceso se proporciona a cualquier persona que haya solicitado Enterprise, haya sido aprobada por nuestro equipo y haya firmado un contrato de renovación automática mensual a través de un account manager de X. [Solicita aquí](https://docs.google.com/forms/d/e/1FAIpQLScO3bczKWO2jFHyVZJUSEGfdyfFaqt2MvmOfl_aJp0KxMqtDA/viewform).

Número de Apps dentro de ese Proyecto: 3+
[Límite de consumo de tuits](/es/x-api/fundamentals/post-cap): más de 50 millones de tuits por mes

Métodos de autenticación:

- [OAuth 2.0 con PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
- [OAuth 1.0a](/es/resources/fundamentals/authentication#oauth-1-0a-2)
- Solo App

<div id="tweet-caps-and-projects">
  ### Límites de tuits y Proyectos
</div>

Los límites de consumo de tuits se aplican a nivel de Proyecto y restringen el volumen de tuits que puede recuperar desde ciertos endpoints de la API de X v2 en un mes determinado.

Obtenga más información sobre los [límites de tuits](/es/x-api/fundamentals/post-cap).

<div id="configuring-projects">
  ### Configuración de proyectos
</div>

<div id="creating-a-project">
  #### Creación de un Proyecto
</div>

Para crear un Proyecto, haz clic en “New Project” en tu [panel](https://developer.twitter.com/en/portal/dashboard) o en la página de [Projects & Apps](https://developer.twitter.com/en/portal/projects-and-apps) dentro del portal para desarrolladores. Solo verás esta opción si aún no has creado un Proyecto. Se te pedirá que indiques un nombre de Proyecto, una descripción y un caso de uso. También se te solicitará crear una nueva App o conectar una App independiente existente.

<div id="creating-or-connecting-an-app-for-your-project">
  #### Crear o conectar una App para su Proyecto
</div>

Si su Proyecto no incluye una App, puede añadir una haciendo clic en el nombre del Proyecto en el panel. Desde allí, puede crear una App nueva o seleccionar una App independiente existente para conectarla a su Proyecto. En la App es donde puede generar las claves y los tokens de autenticación mencionados al inicio de esta guía.

<div id="editing-a-project">
  #### Editar un proyecto
</div>

Para editar un proyecto, haz clic en el nombre de tu proyecto desde el panel o la página de Proyectos y Apps dentro del portal para desarrolladores. Allí verás los detalles de tu proyecto y podrás seleccionar “Editar” para realizar cambios.

<div id="standalone-apps">
  ### Apps independientes
</div>

Las Apps independientes son Apps que existen fuera de la estructura de Proyecto. Las credenciales de autenticación asociadas a estas Apps independientes pueden realizar solicitudes correctamente a la API estándar v1.1 de X, la premium v1.1, la enterprise o a la API de Anuncios de X. Las Apps independientes fallarán al intentar realizar solicitudes a los endpoints de la API de X v2, a menos que las conectes a un Proyecto.

Si creaste una App antes de agosto de 2020, será visible en la sección “Standalone Apps” del portal para desarrolladores, en [Projects & Apps](https://developer.twitter.com/en/portal/projects-and-apps). Estarás limitado a diez Apps en total, incluidas las que estén conectadas a tu Proyecto.

Si formas parte de una cuenta de equipo, verás las Apps de las que eres propietario en Apps independientes. Si un compañero es propietario de una App que forma parte de un Proyecto, podrás ver el nombre de la App y la información del propietario, pero no podrás cambiar su configuración, acceder a sus claves y tokens, ni regenerarlos. Debes contactar al propietario de la App para realizar cualquier cambio en su App.
