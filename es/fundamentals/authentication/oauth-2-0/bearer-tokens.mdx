<div id="using-and-generating-an-app-only-bearer-token">
  ### Uso y generación de un Bearer Token de solo aplicación
</div>

Un Bearer Token permite a los desarrolladores contar con un punto de acceso más seguro para usar las APIs de X y es una de las funciones fundamentales de OAuth 2.0.

La autenticación que utiliza un Bearer Token también se conoce como autenticación de solo aplicación. Un Bearer Token es un arreglo de bytes de formato no especificado que puedes generar con un script, por ejemplo, un comando curl. También puedes obtener un Bearer Token en el portal para desarrolladores, dentro de la sección de claves y tokens en la configuración de tu App. Puedes encontrar más información sobre esta función en la [documentación oficial de OAuth](https://oauth.net/2/bearer-tokens/).

<div id="when-are-they-used">
  #### ¿Cuándo se utilizan?
</div>

Los productos que requieren el uso de un token Bearer son los siguientes:

- [Engagement API](/es/x-api/enterprise-gnip-2.0/fundamentals/engagement-api)
- [Account Activity API](/es/x-api/enterprise-gnip-2.0/fundamentals/account-activity)
- Otras API que utilizan autenticación OAuth 2.0 con token Bearer, como los endpoints de v2 y de Labs.

<div id="prerequisites">
  #### Requisitos previos
</div>

Deberás [registrarte para obtener una cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) y haber creado una [X App](/es/resources/fundamentals/developer-apps). Una vez que tengas eso, también deberás obtener las claves de API que se encuentran en el [portal para desarrolladores](/es/resources/fundamentals/developer-portal). Sigue los pasos a continuación:

1. Inicia sesión en tu cuenta de X en developer.x.com.
2. Ve al [panel de X App](https://developer.x.com/content/developer-twitter/en/apps) y abre la X App para la que deseas generar tokens de acceso.
3. Ve a la página “keys and tokens”.
4. En esta página encontrarás las claves de API, los Access Tokens de usuario y el Bearer Token.

<div id="how-to-generate-a-bearer-token">
  ### Cómo generar un token Bearer
</div>

Puedes encontrar el token Bearer de tu App junto con el resto de tus “Keys and Tokens”.

Copia la siguiente solicitud cURL en tu línea de comandos después de sustituir las claves de API de consumidor que obtuviste previamente de tu [X App](/es/resources/fundamentals/developer-apps). Ten en cuenta que las claves de API de consumidor usadas en esta página han sido descontinuadas y no funcionarán para solicitudes reales.

- **API key** `<API key>` p. ej., `xvz1evFS4wEEPTGEFPHBog`
- **API secret key** `<API secret key>` p. ej., `L8qq9PZyRg6ieKGEKhZolGC0vJWLw8iEJ88DRdyOg`

  ```bash
  curl -u "$API_KEY:$API_SECRET_KEY" \
  --data 'grant_type=client_credentials' \
  'https://api.x.com/oauth2/token'
  ```

Aquí tienes un ejemplo de cómo debería verse la solicitud curl con tus claves de API ingresadas:

```bash
   curl -u 'xvz1evFS4wEEPTGEFPHBog:L8qq9PZyRg6ieKGEKhZolGC0vJWLw8iEJ88DRdyOg' \
  --data 'grant_type=client_credentials' \
  'https://api.x.com/oauth2/token'
```

Así se vería la respuesta. Ten en cuenta que este es un token Bearer descontinuado:

`{"token_type":"bearer","access_token":"AAAAAAAAAAAAAAAAAAAAAMLheAAAAAAA0%2BuSeid%2BULvsea4JtiGRiSDSJSI%3DEUifiRBkKG5E2XzMDjRfl76ZC9Ub0wnz4XsNiRVBChTYbJcE3F"}`

Nuestro token Bearer usado para autenticarnos en recursos con OAuth 2.0 sería:

`AAAAAAAAAAAAAAAAAAAAAMLheAAAAAAA0%2BuSeid%2BULvsea4JtiGRiSDSJSI%3DEUifiRBkKG5E2XzMDjRfl76ZC9Ub0wnz4XsNiRVBChTYbJcE3F`
