Se requieren conexiones TLS para acceder a los endpoints de la API de X. La comunicación mediante TLS preserva la privacidad y la seguridad del usuario al proteger la información entre el usuario y la API de X mientras viaja por Internet pública. Las conexiones a la API de X requieren TLS versión 1.2.


<div id="verification">
  ## Verificación
</div>

<div id="use-an-up-to-date-root-store">
  ### Use un almacén de confianza actualizado
</div>

Es importante que su aplicación o biblioteca use un almacén de confianza (root store) fiable y actualizado al verificar el certificado de X. Cuando sea posible, usar el almacén de confianza proporcionado por su sistema operativo puede ser el enfoque más sencillo. Como alternativa, el [almacén de confianza de Mozilla (NSS)](https://www.mozilla.org/en-US/about/governance/policies/security-group/certs/) se mantiene de forma pública y transparente. Curl también proporciona [una versión de este almacén en formato PEM](https://curl.haxx.se/docs/caextract.html).

Actualmente, X emite la mayoría de nuestros certificados desde la [DigiCert High Assurance EV Root CA](https://www.digicert.com/digicert-root-certificates.htm), pero esto no aplica al 100% de los certificados relacionados con X y podría cambiar en el futuro; por lo tanto, confiar únicamente en las raíces de DigiCert actualmente en uso puede provocar problemas con su aplicación más adelante.

<div id="check-crls-and-the-ocsp-status">
  ### Verifique las CRL y el estado de OCSP[](#check-crls-and-the-ocsp-status "Permalink to this headline")
</div>

Muchas aplicaciones no comprueban la Lista de Revocación de Certificados para los certificados devueltos o dependen del sistema operativo para hacerlo. Asegúrese de que su aplicación o biblioteca TLS esté configurada para exigir la verificación de CRL y OCSP (Online Certificate Status Protocol) antes de aceptar el certificado de X.

<div id="cdns">
  ### CDNs[](#cdns "Permalink to this headline")
</div>

Al mostrar Tweets que contienen contenido multimedia, utiliza el atributo `media_url_https` para las URL HTTPS que se deben usar al mostrar imágenes. En el futuro, todas las URL entregadas desde endpoints de la API proporcionarán rutas HTTPS.

<div id="provide-an-indication-of-security-status">
  ## Indica el estado de seguridad
</div>

Si es posible, muestra el estado actual de la conexión entre tu aplicación y X. Algunos navegadores web lo indican con un icono de candado, mientras que otros muestran el estado de la conexión con mensajes descriptivos.
