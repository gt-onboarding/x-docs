<div id="general">
  ## General
</div>

<AccordionGroup>
  <Accordion title="¿Qué es OAuth?">
    OAuth es un protocolo de autenticación que permite a los usuarios autorizar que una aplicación actúe en su nombre sin compartir su contraseña. Puede encontrar más información en [oauth.net](http://oauth.net/).
  </Accordion>

  <Accordion title="¿Cómo genero tokens de acceso?">
    Debe tener una [app de X](/es/resources/fundamentals/developer-apps) para generar tokens de acceso. Obtenga más información sobre los tokens de acceso [aquí](/es/resources/fundamentals/authentication#oauth-1-0a-2). 
  </Accordion>

  <Accordion title="¿Cómo creo una app de X?">
    Debe tener una [cuenta de desarrollador](/es/resources/fundamentals/developer-portal) para crear una [app de X](/es/resources/fundamentals/developer-apps). Puede registrarse [aquí](https://developer.x.com/en/portal/petition/essential/basic-info).
  </Accordion>

  <Accordion title="Si ya tengo una app de X, ¿cómo veo y edito esa app?">
    Puede ver y editar su app desde el [panel de apps de X](https://developer.x.com/content/developer-twitter/en/apps) si inició sesión en su cuenta de X en developer.x.com.
  </Accordion>
</AccordionGroup>

<div id="technical">
  ## Técnico
</div>

<AccordionGroup>
  <Accordion title="¿Cuánto dura un token de acceso?">
    Los tokens de acceso no caducan explícitamente. Un token de acceso se invalidará si un usuario revoca explícitamente una aplicación en la configuración de su cuenta de X, o si X suspende una aplicación. Si una aplicación está suspendida, habrá una nota en el panel de [apps de X](/es/resources/fundamentals/developer-apps) indicando que ha sido suspendida.
  </Accordion>

  <Accordion title="¿Qué ocurre si un token de acceso se vuelve inválido?">
    Suponga que el token de acceso de un usuario _puede_ volverse inválido en cualquier momento. Si esto ocurre, solicite al usuario que vuelva a autorizar la aplicación. Garantizar que esta situación se maneje de manera adecuada es importante para ofrecer una buena experiencia de usuario.
  </Accordion>

  <Accordion title="La página de registro de la aplicación pregunta sobre acceso de lectura/escritura. ¿Qué se considera escritura?">
    Muchos usuarios confían en que una aplicación lea su información, pero no necesariamente en que cambie la información de su perfil o publique nuevos estados. Actualizar información a través de la API de X —ya sea el nombre, la ubicación o añadir un nuevo estado— requiere una solicitud HTTP POST. Cualquier método de la API que requiera una solicitud HTTP POST se considera un método de escritura y requiere acceso de lectura y escritura.
  </Accordion>
</AccordionGroup>
