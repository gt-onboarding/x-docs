<div id="api-key-and-secret">
  ### Clave y secreto de API
</div>

La clave y el secreto de API (también conocidos como clave y secreto de consumidor) son las credenciales más básicas necesarias para acceder a la API de X. Estas credenciales actúan como el nombre de usuario y la contraseña de tu app de X, y las utiliza la API de X para identificar de qué app provienen las solicitudes.

Estas credenciales pueden ser usadas por los [endpoints de autenticación](/es/resources/fundamentals/authentication/api-reference) para generar credenciales adicionales, como [tokens y secretos de acceso de usuario](/es/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret) y [tokens Bearer](/es/resources/fundamentals/authentication/oauth-2-0/bearer-tokens). También debes usar estas credenciales junto con tokens de acceso y otros parámetros de autorización para [autorizar solicitudes](/es/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request) que requieren autenticación con contexto de usuario de OAuth 1.0a. 

<div id="how-to-acquire-an-api-key-and-secret">
  #### Cómo obtener una API Key y Secret
</div>

Para obtener una X API Key y Secret, siga estos pasos:

1. [Regístrese para obtener una cuenta de desarrollador de X](https://developer.x.com/en/apply-for-access)
2. Cree una [X App](/es/resources/fundamentals/developer-apps) dentro del [developer portal](/es/resources/fundamentals/developer-portal). Tenga en cuenta que, si desea usar la [API de X v2](/es/x-api/introduction), debe agregar su X App a un [Project](/es/resources/fundamentals/projects).
    

Cuando cree su X App, se le mostrarán su API Key y Secret, junto con un Bearer Token. Tenga en cuenta que solo mostramos estas credenciales una vez, así que asegúrese de guardarlas en su gestor de contraseñas o en un lugar seguro.

Encontrará más recomendaciones sobre cómo manejar sus keys y tokens en nuestra página de [authentication best practices](/es/resources/fundamentals/authentication/guides/authentication-best-practices), incluido qué hacer si sus credenciales se han visto comprometidas.
 

<div id="how-to-find-and-regenerate-your-api-key-and-secret-after-app-creation">
  #### Cómo encontrar y regenerar tu API Key y Secret después de crear una App
</div>

Si ya creaste una App y necesitas encontrar o regenerar tu API Key y Secret, sigue estos pasos:

1. Ve al portal para desarrolladores
2. Despliega el menú 'Proyectos y Apps' en la barra lateral
3. Abre la App asociada con la API Key y el Secret que deseas encontrar o regenerar
4. Ve a la pestaña Keys and tokens

Ahí encontrarás todas las credenciales asociadas con tu App. 
 

<div id="how-to-use-your-api-key-and-secret">
  #### Cómo usar tu API Key y Secret
</div>

Si solo está explorando la X Developer Platform, le recomendamos usar una [herramienta o biblioteca](/es/resources/tools-and-libraries) para ver qué hay disponible en la plataforma. Estas herramientas gestionan la autenticación de forma eficaz y pueden ahorrarle mucho tiempo y frustración. En particular, recomendamos [empezar con Postman](/es/tutorials/postman-getting-started) o [Insomnia](https://insomnia.rest/) para desarrolladores principiantes.

Si le interesa crear una solicitud desde cero, lea nuestra guía sobre [autorizar una solicitud OAuth 1.0a](/es/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request).
