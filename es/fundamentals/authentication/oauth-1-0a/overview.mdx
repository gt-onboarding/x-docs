<div id="oauth-10a">
  ## OAuth 1.0a
</div>

Muchos endpoints de la plataforma para desarrolladores de X usan el método OAuth 1.0a para actuar, o realizar solicitudes a la API, en nombre de una cuenta de X. Por ejemplo, si tienes una app de desarrollador de X, puedes realizar solicitudes a la API en nombre de cualquier cuenta de X siempre que ese usuario autentique tu app.

Ten en cuenta: si no estás familiarizado con conceptos como HMAC-SHA1 y codificación por porcentaje, te recomendamos consultar la sección “herramientas útiles” a continuación, que enumera algunos clientes de API que simplifican enormemente el proceso de autenticación.

<div id="key-concepts">
  ### Conceptos clave
</div>

<div id="signing-a-request-with-keys-and-tokens">
  #### Firmar una solicitud con claves y tokens
</div>

Debes firmar cada solicitud a la API pasando varias claves y tokens generados en un encabezado de autorización. Para comenzar, puedes generar varias claves y tokens en la página de detalles de tu [app de desarrollador de X](/es/resources/fundamentals/developer-apps), incluidos los siguientes:

|     |     |
| :--- | :--- |
| Clave y secreto de API:<br /><br />`oauth_consumer_key`<br /><br />`oauth_consumer_secret` | Piénsalos como el nombre de usuario y la contraseña que representan tu app de desarrollador de X al realizar solicitudes a la API. |
| Token de acceso y secreto:<br /><br />`oauth_token`<br /><br />`oauth_token_secret` | Un token de acceso y su secreto son credenciales específicas del usuario que se utilizan para autenticar solicitudes a la API con OAuth 1.0a. Indican la cuenta de X en cuyo nombre se realiza la solicitud.<br /><br />Puedes generar tu propio token de acceso y su secreto si deseas que tu app realice solicitudes en nombre de la misma cuenta de X asociada a tu cuenta de desarrollador en la página de detalles de la app de desarrollador de X.<br /><br />Si deseas generar tokens de acceso para otro usuario, consulta “Realizar solicitudes en nombre de usuarios” a continuación. |

<div id="making-requests-on-behalf-of-users">
  #### Realizar solicitudes en nombre de usuarios
</div>

Al crear una firma, necesitas un conjunto de tokens de acceso que representen al usuario en cuyo nombre vas a realizar una solicitud.

Puedes generar un conjunto de tokens de acceso que represente la cuenta de X que posee la app de desarrollador de X desde la [página de detalles de la app](https://developer.x.com/content/developer-twitter/en/apps), pero si deseas realizar una solicitud en nombre de una cuenta de X distinta, el propietario de esa cuenta debe concederte acceso iniciando sesión en su cuenta como parte del [flujo OAuth de 3 partes](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow). El resultado de este proceso es un conjunto de tokens de acceso (oauth\_token y oauth\_token\_secret) que puede usarse para realizar una solicitud con OAuth 1.0a.

Una vez que tengas estas claves y tokens, puedes [crear una firma](/es/resources/fundamentals/authentication#creating-a-signature) desde cero. No recomendamos esto a menos que sepas lo que haces, o si estás utilizando una de las herramientas mencionadas a continuación para realizar una solicitud a un endpoint que requiere OAuth 1.0a.

Como referencia, aquí tienes un ejemplo de una solicitud cURL firmada con todos los tokens generados incluidos en un encabezado de autorización:

```bash
curl --request POST \
  --url 'https://api.x.com/1.1/statuses/update.json?status=Hello%20world' \
  --header 'authorization: OAuth oauth_consumer_key="CONSUMER_API_KEY", oauth_nonce="OAUTH_NONCE", oauth_signature="OAUTH_SIGNATURE", oauth_signature_method="HMAC-SHA1", oauth_timestamp="OAUTH_TIMESTAMP", oauth_token="ACCESS_TOKEN", oauth_version="1.0"' \
```

<Note>
  **Nota:** Los tokens de acceso de usuario son sensibles y deben protegerse con mucho cuidado. Cuando se generan tokens de acceso, el usuario al que representan confía en que tu aplicación los mantendrá seguros. Si se compromete la seguridad de las claves de API y de los tokens de acceso de usuario, tu aplicación podría exponer potencialmente acceso a información privada y a la funcionalidad de la cuenta. Te recomendamos obtener más información sobre [cómo proteger claves y tokens de acceso](/es/resources/fundamentals/developer-apps).
</Note>

<div id="useful-tools">
  ### Herramientas útiles
</div>

El proceso de [firmar una solicitud](/es/resources/fundamentals/authentication#creating-a-signature) es complejo. Recomendamos usar una biblioteca cliente de API que genere automáticamente gran parte del token de autenticación:

|     |     |
| :--- | :--- |
| [Postman](https://www.getpostman.com/) | Un cliente de API que le permite crear y enviar solicitudes a APIs REST. Lea nuestro tutorial “[Primeros pasos con Postman](/es/tutorials/postman-getting-started)” para obtener más información sobre esta herramienta. |
| [Insomnia](https://insomnia.rest/) | Insomnia es un cliente de API REST con gestión de cookies, variables de entorno, generación de código y autenticación para Mac, Windows y Linux. |
