<Note>
  **Configuración de la cuenta**

  Para acceder a estos endpoints, necesitará:

  - Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
  - Autenticarse con las claves y los tokens de una [App de desarrollador](/es/resources/fundamentals/developer-apps) que se encuentra dentro de un [Proyecto](/es/resources/fundamentals/projects).

  Obtenga más información sobre cómo acceder a los endpoints de la API de X v2 en nuestra [guía de primeros pasos](/es/x-api/getting-started/getting-access).
</Note>

**Primeros pasos**

<div id="authentication">
  ### Autenticación
</div>

Puedes autenticar este endpoint con [OAuth 1.0a en contexto de usuario](/es/resources/fundamentals/authentication), \[OAuth 2.0 App-Only]\(https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), o [OAuth 2.0 con código de autorización y PKCE](es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).
 

<div id="making-a-request">
  ### Realizar una solicitud
</div>

Puedes llamar al endpoint Communities Lookup proporcionando un ID como se muestra a continuación (usando el ID `1758747817642700922`):

```
      curl --location 'https://api.x.com/2/communities/1758747817642700922' --header 'Authorization: ••••••'
```

Si la solicitud se realiza correctamente, deberías ver la respuesta JSON como se muestra a continuación:

```
{
    "data": {
        "access": "Public",
        "description": "Welcome to the Anime Community! Where anime fans gather to share their favorite shows and discuss everything anime-related.",
        "join_policy": "Open",
        "name": "Anime Community"
    }
}
```

<div id="community-fields">
  ### Campos de la comunidad
</div>

Puede obtener datos adicionales asociados a una comunidad especificando Campos de la comunidad.

Están disponibles los siguientes Campos de la comunidad:

| Valor del campo | Tipo | Descripción |     |
| :--- | :--- | :--- | :--- |
| created\_at | date (ISO 8601) | Fecha de creación de la comunidad. |     |
| id  | string | El identificador único de la comunidad. |     |
| name | string | El nombre de la comunidad. |     |
| description | string | El texto de la descripción de la comunidad, si se proporciona. |     |
| access | string | El nivel de acceso de la comunidad.  <br />  <br />Puede ser uno de:<br /><br />\* `Public`<br />\* `Closed` |     |
| join\_policy | string | La política de incorporación de la comunidad.  <br />  <br />Puede ser una de:<br /><br />\* `Open`<br />\* `RestrictedJoinRequestsDisabled`<br />\* `RestrictedJoinRequestsRequireAdminApproval`<br />\* `RestrictedJoinRequestsRequireModeratorApproval`<br />\* `SuperFollowRequired` |     |
| member\_count | integer | El número de miembros que se han unido a la comunidad. |     |

<div id="example-request">
  #### Ejemplo de solicitud
</div>

Puede invocar el endpoint Communities Lookup con campos adicionales de la comunidad, como se muestra a continuación (usando el ID `1758747817642700922`):

```
curl --location 'https://api.x.com/2/communities/1758747817642700922?community.fields=access,created_at,description,id,join_policy,member_count,name' --header 'Authorization: ••••••'
```

**Ejemplo de respuesta**

Si la solicitud se completa correctamente, verá una respuesta JSON como la siguiente:

```
{
    "data": {
        "description": "Bienvenido a la comunidad de anime, donde los aficionados se reúnen para compartir sus series favoritas y hablar de todo lo relacionado con el anime.",
        "access": "Pública",
        "member_count": 40369,
        "created_at": "2024-02-17T06:58:50.000Z",
        "join_policy": "Abierta",
        "name": "Comunidad de anime",
        "id": "Q29tbXVuaXR5OjE3NTg3NDc4MTc2NDI3MDA5MjI="
    }
}
```
