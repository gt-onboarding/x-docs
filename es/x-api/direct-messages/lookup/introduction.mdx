Esta versión inicial de la consulta de Mensajes Directos incluye tres métodos GET:

- **GET /2/dm\_conversations/with/:participant\_id/dm\_events** - Recupera eventos de Mensajes Directos asociados a una conversación individual (uno a uno). El parámetro de ruta :participant\_id es el ID de usuario de la cuenta que mantiene la conversación con el usuario autenticado que realiza esta solicitud.
- **GET /2/dm\_conversations/:dm\_conversation\_id/dm\_events** - Recupera eventos de Mensajes Directos asociados a un ID de conversación específico, según lo indica el parámetro de ruta :dm\_conversation\_id.
- **GET /2/dm\_events** - Recupera eventos de Mensajes Directos asociados a un usuario, incluidas conversaciones uno a uno y de grupo. Hay eventos disponibles de hasta 30 días atrás.

Tenga en cuenta que los ID de eventos de Mensajes Directos son comunes entre v1.1 y v2 (así como en la app de X), por lo que el método de v1.1 para listar un único evento puede usarse junto con estos nuevos endpoints de v2. También tenga en cuenta que las Enterprise y Premium Account Activity APIs admiten mensajes uno a uno de v2, pero todavía no admiten conversaciones de grupo.

Con este lanzamiento se admiten tres tipos de eventos, y estos endpoints aceptan parámetros de consulta para filtrarlos:

- **MessageCreate** - Se ha creado un mensaje.
- **ParticipantsJoin** - Un nuevo participante se ha unido a una conversación.
- **ParticipantsLeave** - Un participante ha abandonado una conversación.

Existe un límite por usuario de 300 solicitudes cada 15 minutos para los métodos GET. Este límite se comparte entre estos endpoints GET.

Dado que está realizando solicitudes en nombre de un usuario con los endpoints de Mensajes Directos de v2, debe autenticarse con [OAuth 1.0a User Context](/es/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3), usando tokens de acceso asociados a usuarios que hayan autorizado su app de X. Para generar estos tokens de acceso con OAuth 1.0a, puede usar el [flujo OAuth de 3 patas](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow). Para generar tokens de acceso de usuario con OAuth 2.0, puede usar el [flujo de concesión Authorization Code with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

**Configuración de la cuenta**

Para acceder a estos endpoints, necesitará:

- Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
- Autenticarse usando las claves y tokens de una [App de desarrollador](/es/resources/fundamentals/developer-apps) que se encuentre dentro de un [Proyecto](/es/resources/fundamentals/projects).

Obtenga más información sobre cómo acceder a los endpoints de la API de X v2 en nuestra [guía de introducción](/es/x-api/getting-started/getting-access).
