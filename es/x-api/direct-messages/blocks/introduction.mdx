Los endpoints para gestionar bloqueos de MD permiten bloquear o desbloquear una cuenta específica en nombre de un usuario autenticado. Para estos endpoints, hay dos métodos POST disponibles:

- **/2/users/:id/dm/block**: Permite bloquear una cuenta
- **/2/users/:id/dm/unblock**: Permite desbloquear una cuenta

<div id="getting-started">
  ### Comenzar
</div>

<div id="authentication">
  ### Autenticación
</div>

Dado que realiza solicitudes en nombre de un usuario, debe autenticar estos endpoints con [OAuth 1.0a User Context](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) o [OAuth 2.0 Authorization Code with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), y utilizar los tokens de acceso del usuario asociados con el usuario en cuyo nombre realiza la solicitud. Puede generar este token de acceso de usuario mediante el [flujo de OAuth de tres patas](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens) (OAuth 1.0a) o mediante el [flujo de concesión Authorization Code with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) (OAuth 2.0).

<div id="making-a-request">
  ### Realizar una solicitud
</div>

Bloquear

Una vez que un usuario se haya autenticado con tu app, puedes llamar al endpoint Block en nombre del usuario como se muestra a continuación:

```
      curl --request POST 'https://api.x.com/2/users/:id/dm/block' --header 'Authorization: ••••••'
```

Si la solicitud se completa correctamente, verás una respuesta JSON como la siguiente:

```
{
   "data": {
       "blocked": true
   }
}
```

**Desbloquear**

Una vez que un usuario se haya autenticado con tu app, puedes llamar al endpoint Unblock en nombre del usuario como se muestra a continuación:

```
      curl --request POST 'https://api.x.com/2/users/:id/dm/unblock' --header 'Authorization: ••••••'
```

Si la solicitud se completa correctamente, verás una respuesta JSON como la siguiente:

```
{
   "data": {
       "blocked": false
   }
}
```
