<div id="authentication">
  ### Autenticación
</div>

Todos los endpoints de la API de X v2 requieren solicitudes autenticadas. Puedes autenticarte mediante:

- [OAuth 1.0a en contexto de usuario](/es/resources/fundamentals/authentication) usando claves de API, tokens de acceso y parámetros adicionales para [crear un encabezado de autorización](/es/resources/fundamentals/authentication#authorizing-a-request).
- [OAuth 2.0 solo para apps](/es/resources/fundamentals/authentication#oauth-2-0) incluyendo un [token de acceso de la app](/es/resources/fundamentals/authentication) en tu solicitud.
- [Código de autorización de OAuth 2.0 con PKCE](/es/resources/fundamentals/authentication) para un control más preciso del alcance de la app y la autorización multidispositivo.

<Note>
  OAuth 1.0a puede resultar complejo de implementar. Si no estás familiarizado, considera usar una [biblioteca](/es/resources/tools-and-libraries) u OAuth 2.0 para las solicitudes. Para métricas privadas o publicaciones, usa OAuth 1.0a en contexto de usuario o el código de autorización de OAuth 2.0 con PKCE.
</Note>

<Tip>
  **Ten en cuenta**

  > Si solicitas los siguientes campos, se requiere OAuth 1.0a en contexto de usuario o el código de autorización de OAuth 2.0:
  >
  > - `tweet.fields.non_public_metrics`
  > - `tweet.fields.promoted_metrics`
  > - `tweet.fields.organic_metrics`
  > - `media.fields.non_public_metrics`
  > - `media.fields.promoted_metrics`
  > - `media.fields.organic_metrics`
</Tip>

<div id="developer-portal-projects-and-apps">
  ### Portal para desarrolladores, Proyectos y Apps
</div>

Para obtener credenciales para la API de X v2, necesitas:

1. Una [cuenta de desarrollador](/es/resources/fundamentals/developer-portal) aprobada.
2. Un [Proyecto](/es/resources/fundamentals/projects) dentro de la cuenta de desarrollador.
3. Una [App de desarrollador](/es/resources/fundamentals/developer-apps) dentro de ese Proyecto, donde encontrarás las claves y los tokens.

<div id="rate-limits">
  ### Límites de uso
</div>

Las solicitudes a la API de X están sujetas a [límites de uso](/es/resources/fundamentals/rate-limits) para gestionar el volumen. Los límites se aplican tanto a nivel de App como de usuario:

- **Nivel de App**: Limita la cantidad de solicitudes que cualquier App puede realizar por período.
- **Nivel de usuario**: Limita la frecuencia con la que un usuario autenticado puede realizar consultas de publicaciones en todas las Apps de desarrollador.

<div id="fields-and-expansions">
  ### Campos y expansiones
</div>

La API de X v2 permite seleccionar campos de datos específicos mediante `fields` y `expansions`:

- **Expansions**: Permiten recuperar objetos relacionados adicionales. Las expansiones admitidas incluyen:
  - `edit_history_tweet_ids`
  - `attachments.poll_ids`
  - `attachments.media_keys`
  - `author_id`
  - `entities.mentions.username`
  - `geo.place_id`
  - `in_reply_to_user_id`
  - `referenced_tweets.id`
  - `referenced_tweets.id.author_id`

- **Fields**: Especifican los campos de datos dentro de los objetos para devolver información adicional. El objeto Post, de forma predeterminada, incluye `id`, `text` y `edit_history_tweet_ids`. Otras opciones, como `tweet.created_at` y `tweet.entities`, deben solicitarse explícitamente.

Para más información, consulta la guía de campos y expansiones en el [diccionario de datos de la API de X v2](/es/x-api/fundamentals/data-dictionary).

<div id="post-edits">
  ### Ediciones de publicaciones
</div>

Las publicaciones elegibles se pueden editar hasta cinco veces dentro de los 30 minutos posteriores a su publicación. El endpoint de búsqueda de publicaciones siempre devuelve la versión más reciente de la publicación. Para casos de uso casi en tiempo real, tenga en cuenta esta ventana temporal. Para más detalles, consulte [Fundamentos de edición de publicaciones](/es/x-api/fundamentals/edit-posts).

<div id="edge-cases">
  ### Casos extremos
</div>

- **Métricas promocionadas**: Solicitar métricas promocionadas para publicaciones no promocionadas devuelve una respuesta vacía.
- **Texto truncado**: El texto de la publicación se trunca en los Retweets. Para obtener el texto completo, expanda la publicación referenciada.
