<div id="comparing-x-apis-posts-lookup-endpoints">
  ## Comparación de los endpoints de consulta de publicaciones de la API de X
</div>

Los endpoints de consulta de publicaciones de la versión v2 sustituyen a los endpoints estándar de v1.1 [GET statuses/lookup](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-lookup) y [GET statuses/show](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-show-id). Esta guía está dirigida a desarrolladores que migran de esas versiones anteriores a la API de X v2.

<div id="standard-v11-compared-to-x-api-v2">
  ## Estándar v1.1 comparado con la API de X v2
</div>

Si ha estado trabajando con los endpoints estándar v1.1 GET statuses/show y GET statuses/lookup, esta guía le ayudará a comprender las similitudes y diferencias entre los endpoints de consulta de publicaciones del estándar y la API de X v2.

También puede estar interesado en nuestra [herramienta visual de migración de formato de datos](/es/x-api/migrate/data-format-migration), que le ayudará a ver rápidamente las diferencias entre el [formato de datos de la API de X v1.1](/es/x-api/fundamentals/data-dictionary) y el [formato de la API de X v2](/es/x-api/fundamentals/data-dictionary).

- **Similitudes**
  - Contexto de usuario OAuth 1.0a
  - Límites de publicaciones por solicitud
  - Compatibilidad con el historial de ediciones y metadatos de publicaciones

- **Diferencias**
  - URL de los endpoints
  - Requisitos de Apps y Proyectos
  - Formato de datos de la respuesta
  - Parámetros de la solicitud

<div id="similarities">
  ### Similitudes
</div>

<div id="oauth-10a-user-context-authentication-method">
  #### Método de autenticación de Contexto de usuario OAuth 1.0a
</div>

El endpoint estándar admite [Contexto de usuario OAuth 1.0a](/es/resources/fundamentals/authentication), mientras que el nuevo endpoint de consulta de publicaciones de la API de X v2 admite tanto el Contexto de usuario OAuth 1.0a como [OAuth 2.0 solo para Apps](/es/resources/fundamentals/authentication). Por lo tanto, si antes usabas uno de los endpoints estándar v1.1 de consulta de publicaciones, puedes seguir utilizando el mismo método de autenticación si migras a la versión de la API de X v2.

La autenticación solo para Apps probablemente sea la forma más sencilla de empezar. Para aprender a generar un App Access Token, consulta [esta guía de OAuth 2.0 solo para Apps](/es/resources/fundamentals/authentication).

<div id="posts-per-request-limits">
  #### Límite de publicaciones por solicitud
</div>

El endpoint v1.1 [GET statuses/lookup](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-lookup) permite especificar hasta 100 publicaciones por solicitud. Esto también aplica al endpoint GET /tweets. Para indicar 100 publicaciones completas, use el parámetro `ids` como parámetro de consulta con una lista de [IDs de publicaciones](/es/resources/fundamentals/x-ids) separada por comas.

**Compatibilidad con el historial de edición y metadatos de publicaciones**

Ambas versiones proporcionan metadatos que describen cualquier historial de edición. Consulte las referencias de la API de búsqueda de publicaciones y la [página de fundamentos sobre edición de publicaciones](/es/x-api/fundamentals/edit-posts) para obtener más detalles.

<div id="differences">
  ### Diferencias
</div>

<div id="endpoint-urls">
  #### URLs de endpoints
</div>

- **Endpoints estándar v1.1:**
  - `https://api.x.com/1.1/statuses/show`
  - `https://api.x.com/1.1/statuses/lookup`

- **Endpoint de la API de X v2:**
  - `https://api.x.com/2/tweets`
  - `https://api.x.com/2/tweets/:id`

<div id="app-and-project-requirements">
  #### Requisitos de Apps y Proyectos
</div>

Los endpoints de la API de X v2 requieren credenciales de una [App de desarrollador](/es/resources/fundamentals/developer-apps) asociada a un [Proyecto](/es/resources/fundamentals/projects) para la autenticación. Los endpoints de la API de X v1.1 pueden usar credenciales de Apps independientes o de Apps asociadas a un Proyecto.

<div id="response-data-format">
  #### Formato de datos de la respuesta
</div>

Una diferencia significativa entre las versiones de endpoints estándar v1.1 y API de X v2 es cómo se seleccionan los campos en la carga.

En los endpoints estándar, muchos campos de la respuesta se incluyen de forma predeterminada, con la opción de usar parámetros para especificar campos adicionales.

Sin embargo, la API de X v2 solo entrega por defecto los campos `id` y `text` de la Publicación. Los campos y objetos adicionales requieren el uso de los parámetros [fields](/es/x-api/fundamentals/fields) y [expansions](/es/x-api/fundamentals/expansions). Los campos ampliados se devuelven en un objeto `includes` dentro de la respuesta, que se puede vincular al objeto principal de la Publicación emparejando los ID.

Para más información sobre el uso de fields y expansions, consulta la [guía sobre cómo usar fields y expansions](/es/x-api/fundamentals/data-dictionary). Una [guía de migración de formato de datos](/es/x-api/fundamentals/fields) también mapea los campos estándar de v1.1 a los campos más recientes de v2.

Además, la API de X v2 introduce nuevos diseños JSON para objetos, incluida la Publicación y los objetos de [user](/es/x-api/fundamentals/fields):

- Los endpoints estándar devuelven objetos de Publicación en un array `statuses`, mientras que la API de X v2 usa un array `data`.
- En la API de X v2, los Retweets y Tweets citados reemplazan la terminología de “statuses”.
- Nueva terminología como `like` reemplaza términos como `favorites` y `favourites`.
- Los atributos sin valor (p. ej., `null`) no se incluyen en las cargas de la API de X v2.

El objeto de Publicación en la API de X v2 incluye nuevos campos, como:

- `conversation_id`
- Dos nuevos campos de [annotations](/es/x-api/fundamentals/post-annotations) (`context` y `entities`)
- Nuevos campos de [metrics](/es/x-api/fundamentals/metrics)
- Campo `reply_setting` que indica quién puede responder a una Publicación determinada

<div id="request-parameters">
  #### Parámetros de solicitud
</div>

Los siguientes parámetros estándar de v1.1 tienen equivalentes en la API de X v2:

| Estándar | API de X v2 |
|:----------|:----------|
| `id`     | `ids`    |

Ciertos parámetros estándar de v1.1 **no** son compatibles en la API de X v2:

| Estándar            | Comentario                                                                                                                   |
|:---------------------|:------------------------------------------------------------------------------------------------------------------------------|
| `tweet_mode`        | Sustituido por la funcionalidad de fields y expansions.                                                                      |
| `trim_user`         | Sustituido por fields y expansions. Use la expansión `author_id` y `user.fields` para los datos de usuario.                 |
| `include_my_retweet`| Proporciona el ID de la publicación de origen para publicaciones retuiteadas por el usuario autenticado.                    |
| `include_entities`  | Use fields y expansions para controlar las entidades en el payload.                                                          |
| `include_ext_alt_text` | Agrega el campo `ext_alt_text` en la entidad de media si hay texto alternativo.                                           |
| `include_card_uri`  | Agrega `card_uri` cuando se adjunta una tarjeta de anuncios.                                                                 |
| `map`               | Devuelve el ID de la publicación y el mensaje de error para publicaciones no disponibles en la API de X v2, en lugar de campos anulados en v1.1. |

<div id="curl-requests">
  ### Solicitudes cURL
</div>

Las siguientes solicitudes cURL muestran endpoints estándar de v1.1 y sus equivalentes en v2. Reemplace `ACCESS_TOKEN` en el encabezado por el token de acceso de su app. Para los endpoints de v2, el token debe pertenecer a una [App de desarrollador](/es/resources/fundamentals/developer-apps/overview) dentro de un [Proyecto](/es/resources/fundamentals/projects).

Las cargas de respuesta de v1.1 serán diferentes a las de v2. En v2, puede solicitar distintos campos con los parámetros [fields](/es/x-api/fundamentals/fields) y [expansions](/es/x-api/fundamentals/expansions).

**Endpoints estándar de v1.1 `GET statuses/lookup` y de v2 `GET /tweets`**

```bash
curl --request GET \
  --url 'https://api.x.com/1.1/statuses/lookup.json?id=1460323737035677698%2C1460323743339741184' \
  --header 'Authorization: Bearer $ACCESS_TOKEN'
```

```
  curl --request GET \
  --url 'https://api.x.com/2/tweets?ids=1460323737035677698%2C1460323743339741184&tweet.fields=created_at&expansions=author_id&user.fields=created_at' \
  --header 'Authorization: Bearer $ACCESS_TOKEN'
```

**Endpoints estándar de v1.1 `GET statuses/show/:id` y de v2 `GET /tweets/:id`**

```
curl --request GET \
  --url 'https://api.x.com/1.1/statuses/show.json?id=1460323737035677698' \
  --header 'Authorization: Bearer $ACCESS_TOKEN'
```

```
curl --request GET \
  --url 'https://api.x.com/2/tweets/1460323737035677698?tweet.fields=created_at&expansions=author_id&user.fields=created_at' \
  --header 'Authorization: Bearer $ACCESS_TOKEN'
```
