La búsqueda de Publicaciones es una función importante que se utiliza para mostrar conversaciones en X sobre un tema o evento específico. Aunque esta funcionalidad existe en X, estos endpoints ofrecen mayor flexibilidad y potencia al filtrar e ingerir Publicaciones, para que pueda encontrar datos relevantes para su investigación con mayor facilidad; crear aplicaciones de “escucha” casi en tiempo real; o, en general, explorar, analizar y/o actuar sobre Publicaciones relacionadas con un tema de interés.

Ofrecemos dos endpoints que le permiten buscar Publicaciones: búsqueda reciente y búsqueda de archivo completo. Ambos endpoints REST comparten un diseño y características comunes, incluido el uso de una única consulta de búsqueda para filtrar Publicaciones sobre un tema específico. Estas consultas se crean con un conjunto de operadores que hacen coincidencia con atributos de la Publicación y del usuario, como palabras clave del mensaje, hashtags y URL. Los operadores pueden combinarse en consultas con lógica booleana y paréntesis para ayudar a refinar el comportamiento de coincidencia de las consultas.

Tanto los endpoints de búsqueda reciente como los de búsqueda de archivo completo proporcionan metadatos de edición de la Publicación. Todos los objetos de Publicaciones creadas desde el 29 de septiembre de 2022 incluyen metadatos de edición de la Publicación, incluso si la Publicación nunca se editó. Cada vez que se edita una Publicación, se crea un nuevo ID de Publicación. El historial de edición de una Publicación se documenta mediante un arreglo de IDs de Publicación, comenzando con el ID original.

Estos endpoints siempre devolverán la edición más reciente, junto con cualquier historial de edición. Cualquier Publicación recopilada después de su ventana de edición de 30 minutos representará su versión final. Para obtener más información sobre los metadatos de edición de Publicaciones, consulte la página [Conceptos fundamentales de edición de Publicaciones](/es/x-api/fundamentals/edit-posts).

Una vez que haya configurado su consulta y comience a recibir Publicaciones, estos endpoints admiten la navegación por los resultados tanto por tiempo como por rangos de ID de Publicación. Esto está diseñado para admitir dos casos de uso comunes:

- **Obtener histórico**: las solicitudes abarcan un período de interés, sin centrarse en la naturaleza en tiempo real de los datos. Se realiza una sola solicitud y todos los datos coincidentes se entregan usando paginación según sea necesario. Este es el modo predeterminado para Buscar Publicaciones.
- **Sondeo** o **escucha**: las solicitudes se realizan en modo “¿hay nuevas Publicaciones desde mi última solicitud?”. Las solicitudes se realizan de forma continua y, por lo general, hay un caso de uso centrado en la “escucha” casi en tiempo real de Publicaciones de interés.

Muchos operadores y límites de consulta son exclusivos del [acceso Enterprise](/es/x-api/getting-started/about-x-api), lo que significa que debe usar claves y tokens de una App dentro de un [Proyecto](/es/resources/fundamentals/projects) con acceso Enterprise para aprovechar la funcionalidad adicional. Puede obtener más información sobre esto en las secciones de endpoints a continuación.

Las Publicaciones devueltas tanto por los endpoints de búsqueda reciente como por los de búsqueda de archivo completo contribuyen al [límite de publicaciones](/es/x-api/fundamentals/post-cap) mensual.

**Configuración de la cuenta**

Para acceder a estos endpoints, necesitará:

- Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
- Autenticarse utilizando las claves y tokens de una [App de desarrollador](/es/resources/fundamentals/developer-apps) que se encuentra dentro de un [Proyecto](/es/resources/fundamentals/projects).

Obtenga más información sobre cómo acceder a los endpoints de la API de X v2 en nuestra [guía de primeros pasos](/es/x-api/getting-started/getting-access).

<div id="recent-search">
  ## Búsqueda reciente
</div>

El endpoint de búsqueda reciente le permite acceder de forma programática a publicaciones públicas filtradas publicadas durante la última semana y está disponible para todos los desarrolladores que tengan una cuenta de desarrollador y estén usando claves y tokens de una [App](/es/resources/fundamentals/developer-apps) dentro de un [Project](/es/resources/fundamentals/projects).

Puede autenticar sus solicitudes con [OAuth 1.0a User Context](/es/resources/fundamentals/authentication), [OAuth 2.0 App-Only](/es/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) o [OAuth 2.0 Authorization Code with PKCE](es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). Sin embargo, si desea recibir métricas privadas o un desglose de métricas orgánicas y promocionadas dentro de sus resultados de publicaciones, tendrá que usar OAuth 1.0a User Context o OAuth 2.0 Authorization Code with PKCE, y pasar tokens de acceso de usuario asociados con el usuario que publicó el contenido en cuestión.

Este endpoint puede entregar hasta 100 publicaciones por solicitud en orden cronológico inverso, y se proporcionan tokens de [paginación](/es/x-api/posts/search/integrate/paginate) para recorrer grandes conjuntos de publicaciones coincidentes.

Al usar un Project con acceso regular, puede usar el conjunto básico de [operadores](/es/x-api/posts/search/integrate/build-a-query) y realizar consultas de hasta 512 caracteres. Al usar un Project con acceso Enterprise, tiene acceso a operadores adicionales. Los Projects con acceso Enterprise pueden realizar consultas de hasta 4096 caracteres.

Obtenga más información sobre [niveles de acceso](/es/x-api/getting-started/about-x-api).

<div id="full-archive-search">
  ## Búsqueda de archivo completo
</div>

El endpoint de búsqueda de archivo completo de v2 solo está disponible para Proyectos con acceso [Pro](/es/x-api/getting-started/about-x-api "Pro") y [Enterprise](/es/x-api/getting-started/about-x-api). Este endpoint permite acceder de forma programática a publicaciones públicas del archivo completo, que se remonta a la primera publicación de marzo de 2006, según tu consulta de búsqueda.

Puedes autenticar tus solicitudes a este endpoint usando [OAuth 2.0 App-Only](/es/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), y el [App Access Token](/es/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token) debe provenir de una App que esté dentro de un Proyecto con acceso Pro o Enterprise. Dado que no puedes realizar solicitudes en nombre de otros usuarios (OAuth 1.0a User Context o OAuth 2.0 Authorization Code con PKCE) con este endpoint, no podrás obtener [métricas](/es/x-api/fundamentals/metrics) privadas.

Este endpoint puede devolver hasta 500 publicaciones por solicitud en orden cronológico inverso, y se proporcionan tokens de [paginación](/es/x-api/posts/search/integrate/paginate) para recorrer conjuntos grandes de publicaciones coincidentes.

**Nota:** Si solicitas [anotaciones](/es/x-api/fundamentals/post-annotations) mediante el parámetro tweet.fields, el parámetro max\_results está actualmente limitado a un valor máximo de 100. Esto puede cambiar en el futuro; ten en cuenta esta limitación.

Dado que este endpoint solo está disponible para quienes han sido aprobados para acceso Pro y Enterprise, tienes acceso al conjunto completo de [operadores](/es/x-api/posts/search/integrate/build-a-query) de búsqueda y puedes realizar consultas de hasta 1024 caracteres.
