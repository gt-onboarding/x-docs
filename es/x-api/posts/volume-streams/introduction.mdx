- Flujo muestreado al 1%.
- Flujo muestreado al 10%. Comúnmente llamado “Decahose”.

Se los denomina “flujos de volumen” porque ambos entregan grandes volúmenes de datos. Incluso el flujo del 1% puede emitir varias decenas de publicaciones cada segundo. Con estos flujos, puedes identificar y seguir tendencias, monitorear el sentimiento general, monitorear eventos globales y mucho más. 

Estos endpoints de streaming entregan [objetos de publicación](/es/x-api/fundamentals/data-dictionary#tweet) mediante una conexión HTTP GET persistente y usan autenticación [OAuth 2.0 App-Only](/es/resources/fundamentals/authentication#oauth-2-0). Con el acceso Essential, puedes tener una conexión a la vez. Con todos los niveles de acceso, se pueden realizar solicitudes de conexión hasta 50 veces por ventana de 15 minutos.

Estos endpoints de flujos de volumen admiten publicaciones editadas. Entregarán publicaciones editadas junto con su historial de ediciones, incluido un arreglo de IDs de publicación. Para las publicaciones sin historial de ediciones, este arreglo contendrá un único ID. Para las publicaciones que han sido editadas, este arreglo contiene múltiples IDs, ordenados de forma ascendente que reflejan el orden de las ediciones, con la versión más reciente en la última posición del arreglo. Para obtener más información sobre cómo funcionan las ediciones de publicación, consulta la página de [fundamentos de ediciones de publicaciones](/es/x-api/fundamentals/edit-posts). 

_Para usar estas API, primero debes configurar una cuenta con nuestro equipo de ventas para empresas._

<Note>
  **Configuración de cuenta**

  Para acceder a estos endpoints, necesitarás:

  - Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
  - Autenticarse con las claves y los tokens de una [App de desarrollador](/es/resources/fundamentals/developer-apps) que esté dentro de un [Proyecto](/es/resources/fundamentals/projects). 

  Obtén más información sobre cómo acceder a los endpoints de la API de X v2 en nuestra [guía de primeros pasos](/es/x-api/getting-started/getting-access).
</Note>
