El grupo de endpoints de flujo filtrado permite a los desarrolladores filtrar un flujo de publicaciones públicas. La funcionalidad de este grupo incluye varios endpoints que le permiten crear y administrar reglas, y aplicar esas reglas para filtrar un flujo de publicaciones que devolverá publicaciones públicas coincidentes. Este grupo de endpoints permite escuchar temas y eventos específicos, monitorear la conversación en torno a competiciones, comprender cómo se desarrollan las tendencias y mucho más.

Los desarrolladores pueden usar el [endpoint de reglas](/es/x-api/posts/filtered-stream#post-2-tweets-search-stream-rules) de REST para agregar y eliminar reglas en una conexión de flujo persistente sin necesidad de desconectarse. Estas [reglas](/es/x-api/posts/filtered-stream#building-rules-for-filtered-stream) se pueden crear con operadores que coinciden con atributos de la publicación, como palabras clave, hashtags y URL. Los operadores y las cláusulas de reglas se pueden combinar con lógica booleana y paréntesis para refinar el comportamiento de coincidencia del filtro.

Una vez que haya agregado un conjunto de reglas, puede [establecer una conexión de streaming](/es/x-api/posts/filtered-stream#get-2-tweets-search-stream) que comenzará a entregar [objetos de publicación](/es/x-api/fundamentals/data-dictionary#tweet) en formato JSON a través de una conexión HTTP Streaming persistente. Solo recibirá contenido que coincida con sus reglas mientras esté conectado al flujo.

Como alternativa, puede vincular el flujo filtrado a un webhook para la entrega como eventos de webhook. Para más detalles sobre el uso de webhooks con flujo filtrado, consulte la [introducción a webhooks de flujo filtrado](/es/x-api/webhooks/stream/introduction).

El endpoint de búsqueda filtrada admite publicaciones editadas. Este endpoint entregará publicaciones editadas que coincidan con uno o más de sus filtros, junto con su historial de ediciones, incluida una matriz de ID de publicación. Para las publicaciones sin historial de ediciones, esta matriz contendrá un único ID. Para las publicaciones que han sido editadas, esta matriz contendrá múltiples ID, dispuestos en orden ascendente que refleja el orden de las ediciones, con la versión más reciente en la última posición de la matriz. Para obtener más información sobre cómo funcionan las ediciones de publicaciones, consulte la página [Fundamentos de ediciones de publicaciones](/es/x-api/fundamentals/edit-posts).

Ciertos aspectos del endpoint de flujo filtrado están limitados por el [nivel de acceso](/es/x-api/getting-started/about-x-api):

**Acceso Pro**

- 1000 reglas por proyecto
- 100 solicitudes por 15 minutos cuando se usa el endpoint POST /2/tweets/search/stream/rules para agregar reglas
- Puede usar todos los operadores al crear su regla
- Puede crear reglas de hasta 1024 caracteres de longitud

**Acceso Enterprise**

- 25 000+ reglas por proyecto
- Puede usar todos los operadores al crear su regla
- Puede crear reglas de hasta 2048 caracteres de longitud
- Solicite [aquí](https://developer.x.com/en/products/x-api/enterprise/enterprise-api-interest-form) el acceso Enterprise

Las publicaciones devueltas del flujo filtrado cuentan para el [límite de publicaciones](/es/x-api/fundamentals/post-cap) mensual.

<Note>
  **Configuración de la cuenta**

  Para acceder a estos endpoints, necesitará:

  - Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
  - Autenticarse usando las claves y los tokens de una [App de desarrollador](/es/resources/fundamentals/developer-apps) que esté ubicada dentro de un [Proyecto](/es/resources/fundamentals/projects).

  Obtenga más información sobre cómo obtener acceso a los endpoints de la API de X v2 en nuestra [guía de introducción](/es/x-api/getting-started/getting-access).
</Note>
