<div id="authentication">
  ### Autenticación
</div>

Todos los endpoints de la API de X v2 requieren que autentiques tus solicitudes con un conjunto de credenciales, también conocidas como claves y tokens.

Estos endpoints específicos requieren el uso de [OAuth 1.0a en contexto de usuario](/es/resources/fundamentals/authentication), lo que significa que debes usar un conjunto de claves de API y tokens de acceso de usuario para realizar una solicitud correcta. Los tokens de acceso deben estar asociados con el usuario en cuyo nombre haces la solicitud. Si deseas generar un conjunto de tokens de acceso para otro usuario, ese usuario debe autorizar o autenticar tu App mediante el [flujo OAuth de 3 pasos](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow).

Ten en cuenta que OAuth 1.0a puede resultar complicado de usar. Si no estás familiarizado con este método de autenticación, te recomendamos usar una [biblioteca](/es/x-api/tools-and-libraries/overview), una herramienta como Postman o OAuth 2.0 para autenticar tus solicitudes.

[OAuth 2.0 Authorization Code con PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) permite un mayor control sobre el alcance de una aplicación y los flujos de autorización en múltiples dispositivos. OAuth 2.0 te permite elegir alcances específicos y granulares que te otorgan permisos precisos en nombre de un usuario.

Para habilitar OAuth 2.0 en tu App, debes activarlo en la configuración de autenticación de tu App, ubicada en la sección de configuración de la App en el portal para desarrolladores.

<div id="developer-portal-projects-and-developer-apps">
  ### Límites de tasa
</div>

Cada día, miles de desarrolladores realizan solicitudes a la API de X. Para ayudar a gestionar el gran volumen de estas solicitudes, se aplican [límites de tasa](https://developer.x.com/x-api/fundamentals/rate-limits) a cada endpoint que restringen la cantidad de solicitudes que puedes realizar en nombre de tu app o de un usuario autenticado.

Estos endpoints tienen límites de tasa a nivel de usuario, lo que significa que el usuario autenticado en cuyo nombre realizas la solicitud solo puede llamar al endpoint un número determinado de veces en cualquier App de desarrollador. Existe un límite de 200 solicitudes por 15 minutos para el método POST. El método DELETE tiene un límite de 50 solicitudes por 15 minutos. Además, existe un límite de [300 solicitudes por 3 horas](https://blog.x.com/developer/en_us/topics/tools/2018/new-developer-requirements-to-protect-our-platform), incluidas las publicaciones creadas con los permisos manage Posts o manage Retweets.
 

<div id="rate-limits">
  ### Etiquetas de origen
</div>

El nombre de tu App y la URL de tu sitio web se mostrarán como la [etiqueta de origen](https://help.x.com/en/using-twitter/how-to-tweet#source-labels) en los metadatos de cualquier publicación creada de forma programática por tu aplicación. Si cambias el caso de uso de una App de X, asegúrate de actualizarlo en esta configuración para garantizar el cumplimiento de los [Términos para Desarrolladores](https://developer.x.com/content/developer-twitter/en/developer-terms/agreement-and-policy).
 

<div id="source-labels">
  ### Configuración del perfil
</div>

Solo puedes agregar una ubicación a las publicaciones si tienes la geolocalización habilitada en la configuración de tu perfil. Si no la tienes habilitada, puedes incluir un parámetro de ubicación en el cuerpo de la solicitud, pero no se adjuntará a la publicación. Lo mismo aplica para etiquetar usuarios en imágenes. Si el usuario al que etiquetas no tiene habilitado el etiquetado de fotos, su nombre no aparecerá en la lista de usuarios etiquetados aunque la publicación se cree correctamente. 
 

<div id="profile-settings">
  ### Agregar multimedia a una publicación
</div>

Actualmente, no hay manera de cargar por completo contenido multimedia usando la API de X v2. Sin embargo, puedes adjuntar multimedia previamente cargada a una publicación. Puedes usar IDs de multimedia que ya hayan sido [cargados mediante el endpoint de multimedia v1](https://developer.x.com/en/docs/twitter-api/v1/media/upload-media/api-reference/post-media-upload) o [X Media Studio](https://media.x.com/en/articles/products/2018/media-studio). Estos IDs de multimedia deben ser tuyos o de un usuario autenticado.
