## Introducción a los endpoints de gestión de publicaciones\*\*

Esta guía de inicio rápido le ayudará a realizar su primera solicitud a los endpoints de gestión de publicaciones usando [Postman](/es/tutorials/postman-getting-started).

Visite nuestro repositorio de GitHub [código de ejemplo de la API de X v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code) si desea ver ejemplos de código en distintos lenguajes.

<Info>
  **Requisitos previos**

  Para completar esta guía, necesitará un conjunto de [claves y tokens](/es/resources/fundamentals/authentication) para autenticar su solicitud. Puede generar estas claves y tokens siguiendo estos pasos:

  - [Solicite una cuenta de desarrollador](https://developer.x.com/en/apply-for-access) y espere la aprobación.
  - Cree un [Proyecto](/es/resources/fundamentals/projects) y una [App de desarrollador](/es/resources/fundamentals/developer-apps) asociada en el portal para desarrolladores.
  - Vaya a la página “Keys and tokens” de su App para generar las credenciales necesarias. Asegúrese de guardar todas las credenciales en un lugar seguro.
</Info>

<div id="steps-to-building-manage-posts-requests">
  ### Pasos para crear solicitudes para administrar Publicaciones
</div>

**Paso uno: Comience con una herramienta o biblioteca**

Existen varias herramientas, ejemplos de código y bibliotecas que puede usar para realizar una solicitud a este endpoint, pero aquí usaremos la herramienta Postman para simplificar el proceso.

Para cargar la colección de Postman de la API de X v2 en su entorno, haga clic en el siguiente botón:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Agregar API de X v2 a Postman
</Button>

Una vez que tenga la colección de la API de X v2 cargada en Postman, navegue a la carpeta “Manage Posts” y seleccione “Create a Post”.
 

**Paso dos: Autentique su solicitud**

Para realizar correctamente una solicitud a la API de X, debe verificar que tiene permiso para hacerlo. Para hacerlo con los endpoints de administración de Publicaciones, debe autenticar su solicitud utilizando [OAuth 1.0a User Context](/es/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code with PKCE](es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

En este ejemplo, usaremos OAuth 1.0a User Context.

Debe agregar sus claves y tokens (específicamente su API Key, API Secret Key, OAuth 1.0a user Access Token y OAuth 1.0a user Access Token Secret) a Postman. Puede hacerlo seleccionando el entorno llamado “X API v2” (en la esquina superior derecha de Postman) y agregando sus claves y tokens en los campos “initial value” y “current value” (haciendo clic en el ícono de ojo junto al menú desplegable de entornos).

Si ha hecho esto correctamente, estas variables se incorporarán automáticamente en la pestaña de autorización de la solicitud.

**Paso tres: Especifique el texto de la Publicación**

Al crear una nueva Publicación con este endpoint, el texto o los medios de la Publicación son los parámetros obligatorios del cuerpo.

En Postman, navegue a la pestaña “Body” e ingrese el texto de la Publicación como el valor del parámetro `text`. Además, si desea agregar parámetros para elementos como encuestas, responder a un ID de Publicación o agregar configuraciones de respuesta, también puede hacerlo aquí. Puede obtener más información sobre lo disponible en nuestra [guía de referencia de la API](/es/x-api/posts/manage-tweets/introduction).

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Parameter type** |
| `text` | Hello world! | body |

**Paso cuatro: Identifique y especifique qué campos desea recuperar**

Una vez que tenga todo configurado, presione el botón “Send” y recibirá una respuesta similar al siguiente ejemplo:

```
{
  "data": {
    "id": "1445880548472328192",
    "text": "Hello world!"
  }
}
```

Si el objeto de respuesta devuelto contiene un `id` y el `text` de su Publicación, habrá creado una Publicación correctamente.
 

**Paso cinco: Elimine su Publicación**

Para eliminar una Publicación, seleccione la solicitud “Delete a Post” que también se encuentra en la carpeta “Manage Posts” de la colección de la API de X v2 cargada en Postman. Este endpoint requiere el ID de la Publicación que desea eliminar. Luego, en la pestaña “Params”, ingrese el ID de la Publicación que desea eliminar como valor de la columna `id`. 

Si la solicitud de eliminación se realiza correctamente, recibirá una respuesta similar al siguiente ejemplo:

```
{
   "data": {
       "deleted" : true
   }
}
```
