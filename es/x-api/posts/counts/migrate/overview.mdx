<div id="comparing-x-apis-post-counts-endpoints">
  ## Comparación de los endpoints de conteo de publicaciones de la API de X
</div>

El endpoint de conteo de publicaciones de v2 reemplazará eventualmente el [endpoint de conteo de la Search API para empresas](/es/x-api/enterprise-gnip-2.0/fundamentals/search-api#counts-requests-post-count). Si tienes código, apps o herramientas que usan una versión anterior de un endpoint de conteo de publicaciones y estás considerando migrar a los endpoints más recientes de la API de X v2, esta guía es para ti.

Esta página contiene dos tablas de comparación:

- [Conteos de publicaciones recientes](#recent)
- [Conteos de publicaciones del archivo completo](#full-archive)
- [Comparación de operadores de filtrado](#operator)

<div id="recent-post-counts-comparison">
  ### Comparación de recuentos de publicaciones recientes
</div>

La versión Enterprise de los endpoints de recuento de publicaciones permite obtener recuentos ya sea para 30 días o desde el archivo completo. Por lo tanto, el endpoint de recuento de publicaciones recientes de la API v2, que abarca un período de 7 días, no sustituye directamente a ninguno de los endpoints antes mencionados.

No obstante, para facilitar las comparaciones, analizaremos cómo se compara el endpoint de recuento de publicaciones recientes de la API v2 con el endpoint Enterprise de 30 días.

La siguiente tabla compara los distintos tipos de endpoints de recuento de publicaciones recientes:

| **Descripción** | **Enterprise** | **API de X v2** |
| :--- | :--- | :--- |
| Dominio del host | https://gnip-api.x.com | https://api.x.com |
| Ruta del endpoint | /search/30day/accounts/:account\_name/:label/counts.json | /2/tweets/counts/recent |
| [Autenticación](/es/resources/fundamentals/authentication) | Autenticación básica | Token Bearer de OAuth 2.0 |
| Formato de marca de tiempo | YYYYMMDDhhmm | YYYY-MM-DDTHH:mm:ssZ  <br />[ISO 8601 / RFC 3339](https://tools.ietf.org/html/rfc3339#section-5.6) |
| Devuelve recuentos de publicaciones con una antigüedad no mayor a | 31 días | 7 días |
| Métodos HTTP admitidos | GET | GET |
| Límites de tasa de solicitud predeterminados | 20 solicitudes por 1 s, agregadas entre datos de búsqueda y solicitudes de recuentos  <br />El límite por minuto variará según el socio, según lo especificado en tu contrato. | 180 solicitudes por 15 min por usuario  <br />450 solicitudes por 15 min por App |
| Admite filtrado mediante [annotations](/es/x-api/fundamentals/post-annotations) |     | ✔   |
| Admite filtrado mediante [conversation\_id](/es/x-api/fundamentals/conversation-id) |     | ✔   |
| Nombre de la clave JSON para el arreglo de datos de publicaciones | results | data |
| Granularidad temporal | Día, hora o minuto | Día, hora o minuto |
| Zona horaria | UTC | UTC |
| Parámetros de solicitud para seleccionar el período de tiempo | fromDate  <br />toDate | start\_time  <br />end\_time |
| Parámetros de solicitud para navegar por ID de publicación |     | since\_id  <br />until\_id |
| Requiere el uso de credenciales de una [App de desarrollador](/es/resources/fundamentals/developer-apps) asociada con un [proyecto](/es/resources/fundamentals/projects) |     | ✔   |

<div id="filtering-operator-comparison">
  ### Comparación de operadores de filtrado
</div>

Las dos versiones (enterprise y v2) de Recuentos de publicaciones difieren en qué operadores están disponibles y también presentan distintos niveles de disponibilidad de operadores dentro de cada versión, lo cual se explica a continuación.

Enterprise

- No existen subniveles de operadores en enterprise. Todos los operadores de enterprise están disponibles para todos los usuarios de enterprise.

API de X v2

- **Core:** Estos operadores están disponibles para cualquier usuario de v2.
- **Advanced:** Estos operadores solo están disponibles para usuarios aprobados para el acceso de Academic Research.

Puedes obtener más información sobre cada uno de estos conjuntos de operadores en sus respectivas guías:

- [Enterprise operators](/es/x-api/enterprise-gnip-2.0/fundamentals/search-api#building-search-queries)
- [X API v2 operators](/es/x-api/posts/search/integrate/build-a-query)
