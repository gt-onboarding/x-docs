Los endpoints de conteo de Publicaciones de la v2 permiten a los desarrolladores comprender y recuperar el volumen de datos para una consulta determinada. Esto puede ser útil por varias razones, entre ellas:

- Entender el volumen de Publicaciones para una palabra clave a fin de crear visualizaciones, como líneas de tendencia.

- Identificar el período en el que ocurrió un evento o conversación para asegurarte de que tu consulta capture los datos relevantes.

- Estimar cuántas Publicaciones devolverá una consulta de búsqueda para refinarla antes de usar los endpoints de búsqueda reciente o de búsqueda de archivo completo.
  **Ten en cuenta:** Los conteos no siempre coincidirán con los resultados que devuelven los endpoints de búsqueda porque estos pasan por controles de cumplimiento adicionales por los que los endpoints de conteo no pasan.

- Dimensionar el tamaño de la conversación en torno a un tema, sin tener que extraer los datos sin procesar ni consumir tu [límite de publicaciones](/es/x-api/fundamentals/post-cap) mensual.

Al desarrollar una consulta, estarás limitado por una longitud máxima y por operadores específicos según tu [nivel de acceso](/es/x-api/getting-started/about-x-api).

- Si estás usando un Proyecto con acceso Pro, puedes usar todos los operadores disponibles y consultas de hasta 1024 caracteres.

- Si estás usando un Proyecto con acceso Enterprise, puedes usar todos los operadores disponibles y consultas de hasta 4096 caracteres.

También puedes especificar la Granularidad (que puede ser día, hora o minuto), así como el período para el que necesitas los conteos de Publicaciones (usando los parámetros start\_time y end\_time). La granularidad temporal predeterminada de este endpoint es hora; es decir, si no especificas el parámetro de granularidad, el endpoint devolverá los conteos de Publicaciones por hora de los últimos 7 días.

<Note>
  **Configuración de la cuenta**

  Para acceder a estos endpoints, necesitarás:

  - Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
  - Autenticación usando las claves y los tokens de una [App de desarrollador](/es/resources/fundamentals/developer-apps) que esté dentro de un [Proyecto](/es/resources/fundamentals/projects).

  Obtén más información sobre cómo obtener acceso a los endpoints de la API de X v2 en nuestra [guía de introducción](/es/x-api/getting-started/getting-access).
</Note>

<div id="recent-post-counts">
  ### Recuentos recientes de publicaciones
</div>

El endpoint de recuentos recientes de publicaciones permite recuperar de forma programática el número de publicaciones que coinciden con una consulta durante los últimos siete días. Este endpoint está disponible para cualquier persona que use claves y tokens asociados a una [App](/es/resources/fundamentals/developer-apps) dentro de un [Project](/es/resources/fundamentals/projects) y utiliza [OAuth 2.0 App-Only](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) para la autenticación.

<div id="full-archive-post-counts">
  ### Recuentos de publicaciones del archivo completo
</div>

El endpoint de recuentos de publicaciones del archivo completo te permite obtener de forma programática el número de Publicaciones que devuelve una consulta, a partir de todo el archivo de Publicaciones públicas. Actualmente, este endpoint solo está disponible para quienes han sido aprobados para [Investigación académica o acceso Enterprise](/es/x-api/getting-started/about-x-api) y usan [OAuth 2.0 App-Only](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) para la autenticación.

Un ejemplo: puedes usar el endpoint de recuentos de publicaciones del archivo completo para ver la cantidad de Publicaciones con el hashtag #SOSHurricaneHarvey por día entre agosto y septiembre de 2017.

**Ten en cuenta:** El endpoint de recuentos pagina en intervalos de 31 días por respuesta. Por ejemplo, establecer una granularidad de día devolverá el recuento de resultados por día durante 31 días por página. Establecer una granularidad de hora devolverá el recuento de resultados por hora durante 744 horas (31 días × 24 horas) por página. Si no especificas la granularidad y el período de tiempo, este endpoint te dará los recuentos de Publicaciones por hora para una consulta, correspondientes a los últimos 30 días.
