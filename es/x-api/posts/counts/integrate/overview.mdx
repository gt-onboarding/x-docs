</div>

Antes de comenzar a explorar algunos conceptos clave, le recomendamos usar una de las siguientes herramientas o ejemplos de código para empezar a probar la funcionalidad de estos endpoints.

¿Le interesa configurar estos endpoints con código en su lenguaje de programación preferido? Contamos con varios ejemplos de código que puede usar como punto de partida en nuestra [página de GitHub](https://github.com/xdevplatform/Twitter-API-v2-sample-code), incluido un [cliente de Python](https://github.com/xdevplatform/search-tweets-python).

<div id="postman">

</div>

Postman es una excelente herramienta que puede utilizar para probar estos endpoints. Cada solicitud de Postman incluye todos los parámetros del endpoint correspondiente para ayudarle a comprender rápidamente qué tiene a su disposición. Para obtener más información sobre nuestras colecciones de Postman, visite nuestra página [Uso de Postman](/es/tutorials/postman-getting-started).
 

<div id="key-concepts">
  ### Conceptos clave
</div>

<div id="authentication">
  #### Autenticación
</div>

Todos los endpoints de la API de X v2 requieren que las solicitudes estén [autenticadas](/es/resources/fundamentals/authentication) con un conjunto de credenciales, también conocidas como claves y tokens. Este endpoint en particular requiere el uso de [OAuth 2.0 Bearer Token](/es/resources/fundamentals/authentication#oauth-2-0), lo que significa que debe incluir un [Bearer Token](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) para realizar una solicitud correctamente. Puede generar un Bearer Token directamente en una App de desarrollador o generarlo mediante el endpoint [POST oauth2/token](/es/resources/fundamentals/authentication#post-oauth2-token).

<div id="developer-portal-projects-and-developer-apps">
  #### Portal para desarrolladores, Proyectos y Apps de desarrollador
</div>

Para trabajar con cualquier endpoint de la API de X v2, debe tener una [cuenta de desarrollador](/es/resources/fundamentals/developer-portal), configurar un [Proyecto](/es/resources/fundamentals/projects) dentro de esa cuenta y crear una [App de desarrollador](/es/resources/fundamentals/developer-apps) dentro de ese Proyecto. Sus claves y tokens de esa App de desarrollador funcionarán con los endpoints de recuento de publicaciones recientes. Si desea usar el endpoint de recuento de publicaciones del archivo completo o utilizar operadores avanzados y consultas más largas, deberá contar con aprobación para el acceso empresarial.

Visite nuestra sección sobre acceso empresarial para obtener más información.

<div id="rate-limits">
  #### Límites de uso
</div>

Cada día, miles de desarrolladores realizan solicitudes a la API de X. Para ayudar a gestionar el volumen, se aplican [límites de uso](/es/x-api/fundamentals/rate-limits) a cada endpoint, que restringen la cantidad de solicitudes que cada desarrollador puede realizar en nombre de una App o de un usuario autenticado.

Este endpoint tiene límites de uso a nivel de App, lo que significa que usted, como desarrollador, solo puede realizar una cierta cantidad de solicitudes a este endpoint durante un período de tiempo determinado desde cualquier App (según las credenciales que esté utilizando). 

<div id="building-queries">
  #### Creación de consultas
</div>

La característica principal de estos endpoints es que utilizan una única consulta para filtrar las Publicaciones y obtener los conteos que se te entregan. Estas consultas se componen de operadores que hacen match con atributos de la Publicación y del usuario, como palabras clave del mensaje, hashtags y URL. Los operadores pueden combinarse en consultas mediante lógica booleana y paréntesis para refinar el comportamiento de coincidencia de la consulta.

Puedes consultar nuestra guía sobre [cómo crear una consulta](/es/x-api/posts/counts/integrate/build-a-query) para obtener más información.

<div id="pagination">
  #### Paginación
</div>

Para los recuentos de publicaciones recientes, no se devuelve ningún next\_token, lo que significa que, independientemente de la granularidad, obtendrás el volumen de publicaciones de los últimos 7 días en una sola llamada a la API.

Para los recuentos de publicaciones del archivo completo, obtendrás datos de los últimos 30 días. Para datos de más de 30 días, recibirás un next\_token que luego puedes usar para paginar y obtener los datos adicionales. 

<div id="building-queries-for-post-counts">
  ### Creación de consultas para recuentos de publicaciones
</div>

Los endpoints de recuentos de publicaciones aceptan una única consulta mediante una solicitud GET y devuelven un conjunto de recuentos históricos de publicaciones que coinciden con la consulta. Las consultas se componen de operadores que se utilizan para hacer coincidir diversos atributos de las publicaciones.
