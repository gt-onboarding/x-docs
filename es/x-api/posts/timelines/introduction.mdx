Estos tres endpoints de cronología admiten publicaciones editadas. Estos endpoints siempre devolverán la edición más reciente, junto con el historial de ediciones. Cualquier publicación recopilada después de su ventana de edición de 30 minutos representará su versión final. Los metadatos de edición incluyen un arreglo de ID de todas las publicaciones en su historial. Para las publicaciones sin historial de ediciones, este arreglo contendrá un único ID. Para las publicaciones que han sido editadas, este arreglo contiene múltiples ID, ordenados de forma ascendente según el orden de las ediciones, con la versión más reciente en la última posición del arreglo. Para obtener más información sobre cómo funcionan las ediciones de publicaciones, consulta la página [Fundamentos de edición de publicaciones](/es/x-api/fundamentals/edit-posts).

<div id="reverse-chronological-home-timeline">
  ## Cronología de inicio en orden inverso
</div>

Este endpoint te permite obtener las Publicaciones, Retuits y respuestas más recientes del usuario autenticado y de las cuentas que sigue.

Como realizas solicitudes en nombre de un usuario, debes autenticar estos endpoints mediante el [flujo de código de autorización de OAuth 2.0 con PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) o [contexto de usuario de OAuth 1.0a](/es/resources/fundamentals/authentication). Este endpoint tiene un límite por usuario de 180 solicitudes por ventana de 15 minutos. Puede devolver todas las Publicaciones creadas en una cronología durante los últimos 7 días, así como las 800 más recientes, independientemente de la fecha de creación.

<div id="user-post-timeline">
  ## Cronología de publicaciones del usuario
</div>

El endpoint de la cronología de publicaciones del usuario proporciona acceso a las publicaciones realizadas por una cuenta específica de X. Recuperar las publicaciones de un usuario le permite crear experiencias como mostrar una cronología en una interfaz de usuario, analizar las publicaciones de un usuario para comprender mejor su contenido o crear flujos de trabajo de interacción con sus publicaciones de forma programática. Este endpoint le da acceso a las publicaciones, retuits, respuestas y citas de tuits más recientes de una sola cuenta de X, similar a lo que puede verse en la cronología del perfil de un usuario.

Esta es una cronología de usuario para @XDevelopers:

El endpoint de la cronología de publicaciones del usuario es un endpoint REST que recibe un único parámetro de ruta para indicar el usuario deseado (por ID de usuario). El endpoint puede devolver hasta las 3.200 publicaciones, retuits, respuestas y citas de tuits más recientes publicadas por el usuario.

Las publicaciones se entregan en orden cronológico inverso, comenzando por la más reciente. Los resultados están [paginados](/es/x-api/fundamentals/pagination) hasta 100 publicaciones por página. Se proporcionan tokens de paginación para recorrer conjuntos grandes de publicaciones. Los IDs de publicación de las publicaciones más nueva y más antigua incluidas en la página también se proporcionan como metadatos, lo que puede utilizarse para sondear cronologías en busca de publicaciones recientes. La cronología de publicaciones del usuario también admite los parámetros start\_time y end\_time para recibir publicaciones creadas dentro de una ventana de tiempo determinada.

El endpoint de la cronología de publicaciones del usuario admite los parámetros de [campos](/es/x-api/fundamentals/fields) y [expansiones](/es/x-api/fundamentals/expansions), y devuelve el [nuevo formato de datos JSON](/es/x-api/fundamentals/data-dictionary).

Para realizar correctamente una solicitud a este endpoint, deberá autorizarla con los métodos de autenticación [OAuth 1.0a User Context](/es/resources/fundamentals/authentication), [OAuth 2.0 Authorization Code with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) o [OAuth 2.0 App-Only](/es/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token). Debe usar OAuth 1.0a User Context u OAuth 2.0 Authorization Code with PKCE cuando solicite métricas no públicas, métricas promocionadas o la cronología de un usuario protegido.

El endpoint de la cronología de publicaciones del usuario está diseñado para admitir dos patrones de uso comunes:

- "Obtener las publicaciones históricas de un usuario": Solicitudes realizadas a la cronología de publicaciones del usuario para recibir las publicaciones del usuario de interés en orden cronológico dentro de un periodo reciente específico. El periodo puede definirse usando start\_time y end\_time y paginando por todos los resultados. En algunos casos, se puede recuperar todo el historial de publicaciones de un usuario si solo ha publicado hasta 3.200 publicaciones en su cuenta. Las publicaciones incluidas dependerán de la disponibilidad pública y de la autenticación utilizada para las solicitudes.

- "Sondear nuevas publicaciones": Solicitudes realizadas a la cronología de publicaciones del usuario de forma continua para recuperar nuevas publicaciones creadas por un usuario específico. El último ID de publicación recibido puede establecerse como parámetro para cualquier nueva solicitud desde la última publicación.

<div id="user-mention-timeline">
  ## Cronología de menciones de usuario
</div>

El endpoint de cronología de menciones de usuario le permite solicitar publicaciones que mencionen a un usuario específico de X; por ejemplo, si una cuenta de X mencionó a @XDevelopers en una publicación. Esto también incluirá las respuestas a las publicaciones del usuario solicitado. Recuperar las menciones de un usuario le permite crear experiencias como descubrir rápidamente quién responde a las publicaciones de un usuario o lo menciona, o crear flujos de trabajo de interacción con sus publicaciones de forma programática. El endpoint permite solicitar las menciones y respuestas más recientes de un único usuario, similar a lo que puede verse en las [notificaciones de menciones](https://x.com/notifications/mentions) de un usuario en X.

La cronología de menciones de usuario es un endpoint REST que recibe un único parámetro de ruta para indicar el usuario deseado (por ID de usuario). El endpoint puede devolver las 800 menciones más recientes de ese usuario.

Las publicaciones se entregan en orden cronológico inverso, comenzando por la más reciente. Los resultados se [paginan](/es/x-api/fundamentals/pagination) en hasta 100 publicaciones por página. Se proporcionan tokens de paginación para recorrer conjuntos grandes de publicaciones. Los ID de publicación de las publicaciones más nueva y más antigua incluidas en la página también se proporcionan como metadatos, lo que puede usarse para sondear cronologías en busca de publicaciones recientes o para navegar por la cronología de forma similar al endpoint v1.1 [mentions\_timeline](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-mentions_timeline). El endpoint también permite especificar los parámetros start\_time y end\_time para recibir publicaciones creadas dentro de una ventana de tiempo determinada. 

Para realizar correctamente una solicitud a este endpoint, deberá autorizarla con los métodos de autenticación [OAuth 1.0a User Context](/es/resources/fundamentals/authentication), [OAuth 2.0 Authorization Code with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) o [OAuth 2.0 App-Only](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only). Debe usar OAuth 1.0a User Context u OAuth 2.0 Authorization Code with PKCE cuando solicite métricas no públicas, métricas promocionadas o la cronología de un usuario protegido.

El endpoint de cronología de menciones de usuario admite los parámetros de [fields](/es/x-api/fundamentals/fields) y [expansions](/es/x-api/fundamentals/expansions), y devuelve el [nuevo formato de datos JSON](/es/x-api/fundamentals/data-dictionary).
