<div id="comparing-x-apis-timelines-endpoints">
  ## Comparación de los endpoints de cronologías de la API de X
</div>

Los endpoints de cronología inversa (del más reciente al más antiguo), cronología de publicaciones del usuario y cronología de menciones del usuario de la v2 reemplazan, respectivamente, los endpoints [v1.1 statuses/home\_timeine,](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-home_timeline) [v1.1 statuses/user\_timeline](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-user_timeline.html) y [v1.1 statuses/mentions\_timeline](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-mentions_timeline.html). Si tiene código, aplicaciones o herramientas que usan una versión anterior de este endpoint y está considerando migrar al endpoint más reciente de la API de X v2, esta guía es para usted. Para una guía de migración más detallada, consulte [Migración de Standard v1.1 a la API de X v2](/es/x-api/posts/timelines/migrate/standard-to-twitter-api-v2).

Esta página contiene tres tablas comparativas:

- Cronología de inicio inversa
- Cronología de publicaciones del usuario
- Cronología de menciones del usuario

<div id="reverse-chronological-home-timeline">
  ### Cronología inversa de la cronología de inicio
</div>

Las siguientes tablas comparan los endpoints de la cronología de inicio estándar de la v1.1 y de la API de X v2:

|     |     |     |
| :--- | :--- | :--- |
| **Descripción** | **Estándar v1.1** | **API de X v2** |
| Documentación | [Referencia de la API](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-home_timeline) | [Referencia de la API](https://developer.x.com/x-api/posts/timelines#user-mention-timeline-3) |
| Métodos HTTP compatibles | `GET` | `GET` |
| Dominio del host | `https://api.x.com` | `https://api.x.com` |
| Rutas del endpoint | `/1.1/statuses/home_timeline.json` | `/2/users/:id/timelines/reverse_chronological` |
| Parámetros requeridos | `user_id` o `screen_name` | ID de usuario establecido como parámetro de ruta :id |
| Autenticación | Contexto de usuario de OAuth 1.0a | Contexto de usuario de OAuth 1.0a<br /><br />[Flujo de código de autorización de OAuth 2.0 con PKCE](/es/resources/fundamentals/authenticationoauth-2-0/authorization-code) |
| Límites de frecuencia/Límites de volumen | 15 solicitudes por ventana de 15 minutos con contexto de usuario de OAuth 1.0a<br /><br />Límite de solicitudes: 100.000 en un período de 24 horas. | 180 solicitudes por ventana de 15 minutos <br /><br />[Límite de publicaciones](/es/x-api/fundamentals/post-cap):<br /><br />500.000 con acceso Essential<br /><br />2 millones con acceso Elevated<br /><br />10 millones con acceso Academic Research |
| Publicaciones predeterminadas por respuesta | 15  | 100 |
| Máximo de publicaciones por respuesta | 800 | Este endpoint devuelve todas las publicaciones creadas en una cronología durante los últimos 7 días, así como las 800 más recientes independientemente de la fecha de creación. |
| Proporciona historial de edición de la publicación | ✔   | ✔   |
| Publicaciones históricas disponibles | Las 800 publicaciones más recientes, incluidos los Retweets | Las 3.200 publicaciones más recientes, incluidos los Retweets |
| Opciones de navegación de la cronología | since\_id (exclusivo) usado para sondeo de actualizaciones<br /><br />`max_id` (inclusivo) | `start_time`<br /><br />`end_time`<br /><br />`since_id` (exclusivo) usado para sondeo de actualizaciones <br /><br />`until_id` (exclusivo) |
| Parámetros opcionales para refinar resultados | `count`<br /><br />`exclude_replies`<br /><br />`include_rts`<br /><br />`trim_user`<br /><br />`tweet_mode`<br /><br />`since_id`<br /><br />`max_id` | `max_results`<br /><br />`exclude` (retweets,replies)<br /><br />`tweet.fields`<br /><br />`user.fields`<br /><br />`place.fields`<br /><br />`media.fields`<br /><br />`poll.fields`<br /><br />`expansions`<br /><br />`start_time`<br /><br />`end_time`<br /><br />`since_id`<br /><br />`until_id` |
| Admite solicitar y recibir [anotaciones](/es/x-api/fundamentals/post-annotations) | N/D | Si las anotaciones están incluidas en `tweet.fields`, los resultados se anotarán con datos de anotación inferidos basados en el texto de la publicación, como «Género musical» y «Música folk» o «Músico» y «Dolly Parton». |
| Admite solicitar y recibir [métricas](/es/x-api/fundamentals/metrics) específicas de la publicación | N/D | Si las métricas están incluidas en `tweet.fields`, los resultados incluirán `public_metrics` por publicación, como `retweet_count`, `reply_count`, `quote_count` y `like_count`, y `non_public_metrics`, como `impression_count`, `user_profile_clicks`, `url_link_clicks`.<br /><br />Métricas de medios adicionales como `view_count` y métricas de reproducción de video.<br /><br />Métricas adicionales `organic_metrics` y `promoted_metrics` disponibles con contexto de usuario para publicaciones promocionadas. |
| Admite solicitar y recibir [conversation\_id](/es/x-api/fundamentals/conversation-id) | N/D | Devuelve un campo `conversation_id` cuyo valor representa la primera publicación publicada en un hilo de respuestas para ayudar a rastrear conversaciones. |
| Formato JSON de la publicación | [Formato de datos estándar v1.1](https://developer.x.com/en/docs/twitter-api/v1/data-dictionary/overview.html) | Formato de la API de X v2 (determinado por los parámetros de solicitud fields y expansions; no es compatible con los formatos de v1.1)<br /><br />Para obtener más información sobre cómo migrar del formato estándar v1.1 al formato de la API de X v2, visite nuestra [guía de migración de formatos de datos](/es/x-api/migrate/data-format-migration). |
| Orden de resultados | Cronológico inverso | Cronológico inverso |
| Paginación de resultados | N/D; debe usar navegación por ID de publicación | Los resultados se pueden recorrer hacia adelante o hacia atrás usando un pagination\_token |
| Requiere el uso de credenciales de una [App de desarrollador](/es/resources/fundamentals/developer-apps) asociada a un [Proyecto](/es/resources/fundamentals/projects) |     | ✔   |
