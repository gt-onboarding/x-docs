<div id="authentication">
  #### Autenticación
</div>

Todos los endpoints de la API de X v2 requieren que autentiques tus solicitudes con un conjunto de credenciales, también conocidas como claves y tokens.

Estos endpoints específicos requieren el uso de [OAuth 2.0 con PKCE (flujo de código de autorización)](/es/resources/fundamentals/authentication/oauth-2-0/authorization-code), lo que significa que debes usar un conjunto de claves y tokens de acceso de usuario para realizar una solicitud correctamente. Los tokens de acceso deben estar asociados con el usuario en cuyo nombre haces la solicitud. Si quieres generar un conjunto de tokens de acceso para otro usuario, ese usuario debe autorizar o autenticar tu App usando una [URL de autorización](/es/resources/fundamentals/authentication#authorize-url).

Ten en cuenta que OAuth 2.0 puede resultar complejo de usar. Si no estás familiarizado con este método de autenticación, te recomendamos usar una [biblioteca](/es/x-api/tools-and-libraries/overview) o una herramienta como Postman para autenticar correctamente tus solicitudes.

<div id="developer-portal-projects-and-developer-apps">
  #### Portal para desarrolladores, Proyectos y Apps de desarrollador
</div>

Para obtener un conjunto de credenciales de autenticación que funcione con los endpoints de la API de X v2, debes tener una [cuenta de desarrollador](/es/resources/fundamentals/developer-portal), configurar un [Proyecto](/es/resources/fundamentals/projects) dentro de esa cuenta y crear una [App de desarrollador](/es/resources/fundamentals/developer-apps) dentro de ese Proyecto. Luego podrás encontrar tus claves y tokens en tu App de desarrollador. 

<div id="rate-limits">
  #### Límites de solicitudes
</div>

Cada día, muchos miles de desarrolladores realizan solicitudes a la API de X. Para ayudar a gestionar el volumen de estas solicitudes, se aplican [límites de solicitudes](/es/resources/fundamentals/rate-limits) a cada endpoint que restringen la cantidad de solicitudes que puedes realizar en nombre de tu app o de un usuario autenticado.

Estos endpoints tienen límites a nivel de usuario, lo que significa que el usuario autenticado en cuyo nombre realizas la solicitud solo puede invocar el endpoint un número determinado de veces en cualquier App de desarrollador. Existe un límite de usuario de 180 solicitudes por ventana de 15 minutos para el método GET. Con el método GET del endpoint de consulta de marcadores, recibirás hasta 800 de tus publicaciones marcadas más recientes. Además, existe un límite de usuario de 50 solicitudes por cada 15 minutos para los métodos POST y DELETE.
