<div id="manage-bookmarks">
  ### Administrar marcadores
</div>

Disponemos de dos métodos para administrar marcadores: POST y DELETE. El método POST permite crear marcadores. Del mismo modo, el método DELETE permite eliminar un marcador específico.

Hay un límite por usuario de 50 solicitudes cada 15 minutos para los métodos POST y DELETE.

Dado que haces solicitudes en nombre de un usuario con los endpoints de administración de marcadores, debes autenticarte generando un token de acceso de usuario con OAuth 2.0. Puedes usar el [flujo de concesión Authorization Code with PKCE](/es/resources/fundamentals/authentication/oauth-2-0/user-access-token) para hacerlo. Para usar este endpoint debes incluir los alcances `tweet.read`, `users.read` y `bookmark.write`.

<div id="bookmarks-lookup">
  ### Consulta de marcadores
</div>

El endpoint de consulta de marcadores tiene un único método disponible: GET. Este método te permite obtener los marcadores tuyos o de una cuenta autenticada. Se proporcionarán tokens de paginación para recorrer conjuntos grandes de resultados en este endpoint.

Existe un límite por usuario de 180 solicitudes por ventana de 15 minutos para el método GET. Con este endpoint, recibirás hasta 800 de tus publicaciones marcadas más recientes.

Como estás realizando solicitudes en nombre de un usuario con los endpoints de consulta de marcadores, debes autenticarte generando un token de acceso de usuario con OAuth 2.0. Puedes usar el [flujo de concesión de Código de autorización con PKCE](/es/resources/fundamentals/authentication/oauth-2-0/user-access-token) para hacerlo. Para usar este endpoint debes incluir los scopes `tweet.read`, `users.read` y `bookmark.read`.

**Configuración de la cuenta**

Para acceder a estos endpoints, necesitarás:

- Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
- Autenticarte usando las claves y los tokens de una [App de desarrollador](/es/resources/fundamentals/developer-apps) que esté dentro de un [Proyecto](/es/resources/fundamentals/projects).

Obtén más información sobre cómo acceder a los endpoints de la API de X v2 en nuestra [guía de introducción](/es/x-api/getting-started/getting-access).
