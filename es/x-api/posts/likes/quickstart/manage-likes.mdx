<div id="getting-started-with-the-manage-likes-endpoints">
  ### Introducción a los endpoints para gestionar los Me gusta
</div>

Esta guía de inicio rápido le ayudará a realizar su primera solicitud a los endpoints para gestionar los Me gusta usando [Postman](/es/tutorials/postman-getting-started).

Si desea ver código de ejemplo en diferentes lenguajes, visite nuestro repositorio de GitHub con [código de ejemplo de la API de X v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code). 

<div id="prerequisites">
  #### Requisitos previos
</div>

Para completar esta guía, necesitarás un conjunto de [claves y tokens](/es/resources/fundamentals/authentication), que puedes generar siguiendo estos pasos:

1. [Solicita una cuenta de desarrollador](https://developer.x.com/en/apply-for-access) y espera la aprobación.

2. Crea un [Proyecto](/es/resources/fundamentals/projects) y una [App de desarrollador](/es/resources/fundamentals/developer-apps) asociada en el portal para desarrolladores.

3. Ve a la página “Keys and tokens” de tu app y guarda tus API Keys, Access Tokens y el Bearer Token en tu gestor de contraseñas.

<div id="steps-to-build-a-manage-likes-request">
  #### Pasos para crear una solicitud para gestionar Me gusta
</div>

**Paso uno: comience con una herramienta o biblioteca**

Existen varias herramientas, ejemplos de código y bibliotecas que puede usar para realizar una solicitud a este endpoint, pero aquí usaremos la herramienta Postman para simplificar el proceso.

Para cargar la colección de Postman de la API de X v2 en su entorno, haga clic en el siguiente botón:

[![](https://cdn.cms-twdigitalassets.com/content/dam/developer-twitter/m1_vnext/carat.svg)<br />Agregar API de X v2 a Postman](https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400)

Una vez que tenga la colección de la API de X v2 cargada en Postman, navegue a la carpeta “Likes” y seleccione “Like a Post”.

**Paso dos: autentique su solicitud**

Para realizar una solicitud correcta a este endpoint, deberá usar [OAuth 1.0a User Context](/es/resources/fundamentals/authentication). Para hacerlo, debe agregar las siguientes claves y tokens en Postman seleccionando el entorno llamado “X API v2” y agregando las siguientes variables en los campos de valor inicial y valor actual:

- consumer\_key con su API Key
- consumer\_secret con su API Key Secret
- access\_token con su Access Token
- token\_secret con su Access Token Secret
   

**Paso tres: especifique qué Publicación va a indicar que le gusta**

Los endpoints para gestionar Me gusta requieren dos ID: uno para el usuario (el usuario que desea indicar que le gusta una Publicación) y otro que representa el ID de la Publicación que el usuario intenta indicar que le gusta o quitar el Me gusta.

El ID del usuario debe corresponder al ID del usuario autenticado, lo que significa que debe pasar los [Access Tokens](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) asociados con ese ID de usuario al autenticar su solicitud. En este caso, puede especificar el ID que pertenece a su propio usuario. Puede encontrar su ID de dos maneras:

1. Usando el endpoint [user lookup by username](/es/x-api/users/user-lookup-by-id), puede pasar un nombre de usuario y recibir el campo id.
2. Al revisar su Access Token, verá que la parte numérica es su ID de usuario.
    

También debe especificar una Publicación a la que quiera indicar que le gusta. Puede encontrar el ID de la Publicación navegando a x.com, haciendo clic en una Publicación y luego consultando la URL. Por ejemplo, el ID de la Publicación de la siguiente URL es 1228393702244134912.

https://x.com/TwitterDev/status/1228393702244134912

En Postman, navegue a la pestaña "Params" e ingrese su ID en la columna "Value" de la variable de ruta id. Navegue a la pestaña “Body” y proporcione el ID de la Publicación que desea indicar que le gusta como valor del parámetro tweet\_id. Asegúrese de no incluir espacios antes o después de ningún ID.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | (su ID de usuario) |
| tweet\_id | (el ID de la Publicación a la que quiere indicar que le gusta) |

Si hace clic en el botón "Send", recibirá un objeto de respuesta que contiene el estado de la relación:

- Si recibe "liked": true, entonces el id está indicando correctamente que le gusta el tweet\_id.
   

**Paso cuatro: realice su solicitud y revise su respuesta**

Una vez que tenga todo configurado, presione el botón "Send" y recibirá la siguiente respuesta:

```
{
    "data": {
        "liked": true
    }
}
```

Si desea quitar el Me gusta de la misma Publicación, puede usar la solicitud titulada “Unlike a Post”, que también se encuentra en la carpeta “Likes” de la colección de la API de X v2 en Postman. El id y el tweet\_id deben usarse de la misma manera que en el ejemplo anterior. Para quitar el Me gusta de una Publicación, no tendrá que agregar esto como un cuerpo JSON, por lo que deberá asegurarse de agregar los parámetros de consulta requeridos para id y tweet\_id.
