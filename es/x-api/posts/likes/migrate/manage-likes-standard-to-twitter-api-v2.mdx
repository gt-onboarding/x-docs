<div id="manage-likes-standard-v11-compared-to-x-api-v2">
  ### Gestionar Me gusta: estándar v1.1 frente a la API de X v2
</div>

Si ha estado trabajando con los endpoints estándar v1.1 [POST favorites/create](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-create) y [POST favorites/destroy](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-destroy), el objetivo de esta guía es ayudarle a comprender las similitudes y diferencias entre los endpoints para gestionar Me gusta del estándar v1.1 y de la API de X v2.

- **Similitudes**
  - Contexto de usuario OAuth 1.0a
- **Diferencias**
  - URL de endpoints y métodos HTTP
  - Requisitos de App y Proyecto
  - Parámetros de la solicitud

<div id="similarities">
  #### Similitudes
</div>

**Método de autenticación OAuth 1.0a en contexto de usuario**

Ambas versiones del endpoint admiten [OAuth 1.0a User Context](/es/resources/fundamentals/authentication#oauth-1-0a-2). Por lo tanto, si anteriormente utilizabas uno de los endpoints estándar de v1.1 para gestionar favoritos, puedes seguir usando el mismo método de autenticación si migras a la versión de la API de X v2.

<div id="differences">
  #### Diferencias
</div>

**URLs de endpoints y métodos HTTP**

- Endpoints de Standard v1.1:
  - POST https://api.x.com/1.1/favorites/create.json
    (indicar “me gusta” a una Publicación)
  - POST https://api.x.com/1.1/favorites/destroy.json
    (quitar “me gusta” a una Publicación)
- Endpoint de la API de X v2:
  - POST https://api.x.com/2/tweets/:id/likes
    (indicar “me gusta” a una Publicación)
  - DELETE https://api.x.com/2/tweets/:id/likes/:tweet\_id
    (quitar “me gusta” a una Publicación) 

**Requisitos de App y Proyecto**

Los endpoints de la API de X v2 requieren que uses credenciales de una [developer App](/es/resources/fundamentals/developer-apps) asociada a un [Project](/es/resources/fundamentals/projects) al autenticar tus solicitudes. Todos los endpoints de X API v1.1 pueden usar credenciales de Apps independientes o de Apps asociadas a un Project.

**Parámetros de solicitud**

Los siguientes parámetros de solicitud de Standard v1.1 tienen equivalentes en la API de X v2:

| Standard v1.1 | API de X v2 |
| :--- | :--- |
| id  | id  |
| includes\_entities | Sin equivalente |

Ten en cuenta que los parámetros de Standard v1.1 se envían como parámetros de consulta, mientras que los parámetros de la API de X v2 se envían como parámetros en el cuerpo para el endpoint POST o como parámetros de ruta para el endpoint DELETE.

Además, no se requiere el id del usuario que indica “me gusta” a una Publicación cuando se usan los endpoints de Standard v1.1, ya que los [Access Tokens](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) enviados con [OAuth 1.0a User Context](/es/resources/fundamentals/authentication) permiten inferir qué usuario inicia la acción de indicar o quitar “me gusta”.
