<div id="likes-lookup-standard-v11-compared-to-x-api-v2">
  ### Búsqueda de Me gusta: estándar v1.1 frente a la API de X v2
</div>

Si ha estado trabajando con el endpoint estándar v1.1 [GET favorites/list](https://developer.x.com/en/docs/x-api/v1/tweets/post-and-engage/api-reference/get-favorites-list), el objetivo de esta guía es ayudarle a comprender las similitudes y diferencias entre los endpoints de búsqueda de Me gusta del estándar v1.1 y la API de X v2.

Con v2, también hemos incorporado un nuevo endpoint de usuarios que dieron Me gusta, que le permite obtener información sobre los usuarios a los que les gustó una Publicación.

- **Similitudes**
  - Autenticación
  - Límites de tasa
- **Diferencias**
  - URL de endpoints

  - Limitaciones de la solicitud

  - Requisitos de App y Proyecto

  - Parámetros de la solicitud

  - Nuevo formato JSON

<div id="similarities">
  #### Similitudes
</div>

**Autenticación**

Tanto los endpoints de consulta de Me gusta de la versión estándar v1.1 como de la API de X v2 utilizan [OAuth 1.0a User Context](/es/resources/fundamentals/authentication#oauth-1-0a-2) o [OAuth 2.0 Bearer Token](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only). Por lo tanto, si antes utilizaba los endpoints estándar v1.1 [GET favorites/list endpoints](https://developer.x.com/en/docs/x-api/v1/tweets/post-and-engage/api-reference/get-favorites-list), puede seguir usando el mismo método de autenticación si decide migrar a la versión de la API de X v2.

Según la biblioteca o paquete de autenticación que elija, la autenticación con Bearer Token probablemente sea la forma más sencilla de empezar y puede configurarse con un encabezado de solicitud simple. Para aprender a generar un Bearer Token, consulte [esta guía de OAuth 2.0 Bearer Token](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only).
 

**Límites de uso**

El endpoint estándar v1.1 GET favorites/list tiene un límite de 75 solicitudes por 15 minutos por usuario. El endpoint correspondiente de publicaciones con Me gusta en v2 también tiene este mismo límite. Sin embargo, este endpoint de v2 también tiene un límite adicional de 75 solicitudes por 15 minutos por App.

<div id="differences">
  #### Diferencias
</div>

**URLs de endpoints**

- Endpoints estándar v1.1:
  - GET https://api.x.com/1.1/favorites/list.json
    (lista de Publicaciones del usuario especificado)
  - No existe un equivalente en v1.1 para el endpoint de usuarios que indican Me gusta en v2
- Endpoint de la API de X v2:
  - GET https://api.x.com/2/users/:id/liked\_tweets
    (lista de Publicaciones a las que indicó Me gusta el usuario con el ID especificado)
  - GET https://api.x.com/2/tweets/:id/liking\_users
    (lista de usuarios que indicaron Me gusta a la Publicación con el ID especificado)

**Limitaciones de solicitud**

El endpoint de v2 para Publicaciones con Me gusta permite solicitar de 5 a 100 Publicaciones por solicitud, pero puede obtener todos los Me gusta de una Publicación usando tokens de paginación. El endpoint v1.1 GET favorites/list también permite extraer todos los Me gusta de Publicaciones, pero puede extraer de 20 a 200 Publicaciones por solicitud.

Para el endpoint de usuarios que indican Me gusta en v2, está limitado a 100 usuarios que indicaron Me gusta por Publicación. 
 

**Requisitos de App y Proyecto**

Los endpoints de la API de X v2 requieren usar credenciales de una [App de desarrollador](/es/resources/fundamentals/developer-apps) asociada con un [Proyecto](/es/resources/fundamentals/developer-apps) al autenticar sus solicitudes. Todos los endpoints de la API de X v1.1 pueden usar credenciales de Apps independientes o Apps asociadas con un proyecto.

**Parámetros de solicitud**

Los endpoints estándar v1.1 aceptaban dos parámetros de consulta (user\_id o screen\_name). La API de X v2 solo acepta el ID numérico de usuario, y debe pasarse como parte de la ruta del endpoint.

Una de las mayores diferencias entre los endpoints estándar v1.1 y la API de X v2 es cómo se seleccionan los campos que se devuelven en la carga. En los endpoints estándar, había varios parámetros para indicar qué campos o conjuntos de campos se incluían en la carga, mientras que la versión de la API de X v2 simplifica estos parámetros en [campos](/es/x-api/fundamentals/fields) y [expansiones](/es/x-api/fundamentals/expansions). 
 

**Nuevo formato JSON**

La API de X v2 introduce nuevos diseños JSON para los objetos que devuelven las APIs, incluidos los objetos de [Publicación](/es/x-api/fundamentals/data-dictionary#tweet) y [usuario](/es/x-api/fundamentals/data-dictionary#user).

- En el nivel raíz de JSON, los endpoints estándar devuelven objetos de usuario en un arreglo statuses, mientras que la API de X v2 devuelve un arreglo data. 
- En lugar de referirse a “statuses” Retweeted y Quoted, el JSON de la API de X v2 se refiere a Publicaciones retuiteadas y citadas. Muchos campos heredados y obsoletos, como contributors y user.translator\_type, se están eliminando. 
- En lugar de usar tanto favorites (en el objeto de Publicación) como favourites (en el objeto de usuario), la API de X v2 usa el término like. 
- X adopta la convención de que los valores JSON sin valor (por ejemplo, null) no se escriben en la carga. Los atributos de Publicación y usuario solo se incluyen si tienen valores no nulos.
   

Además de los cambios en el nuevo formato JSON, también introdujimos un nuevo conjunto de campos en el objeto de Publicación, que incluye lo siguiente:

- [conversation\_id](/es/x-api/fundamentals/conversation-id)
- Dos nuevos campos de [anotaciones de Publicación](/es/x-api/fundamentals/post-annotations), incluidos context y entities
- Varios campos nuevos de [métricas](/es/x-api/fundamentals/metrics)
