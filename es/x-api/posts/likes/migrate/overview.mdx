<div id="comparing-x-apis-likes-endpoints">
  ## Comparación de los endpoints de Me gusta de la API de X
</div>

Estas guías se enfocarán en las siguientes áreas:

- **Parámetros de solicitud de la API** - El endpoint de la API de X v2 introduce un nuevo conjunto de parámetros de solicitud. Si bien algunos parámetros serán familiares, especialmente para quienes se integran con Labs, hay muchas diferencias importantes, como la inclusión de los parámetros [fields](/es/x-api/fundamentals/fields) y [expansions](/es/x-api/fundamentals/expansions).

- **Requisitos de Apps y Proyectos** - Para acceder a la API de X v2, deberá usar credenciales de una [App de desarrollador](/es/resources/fundamentals/developer-apps) asociada a un [Proyecto](/es/resources/fundamentals/projects)

<div id="likes-lookup">
  #### Búsqueda de Me gusta
</div>

**Usuarios que han indicado Me gusta en una publicación**

El endpoint de usuarios que han indicado Me gusta es una funcionalidad nueva de v2 que te permite obtener información sobre los usuarios a los que les gustó una publicación.

| Descripción | API de X v2 |
| :--- | :--- |
| Métodos HTTP compatibles | GET |
| Dominio del host | https://api.x.com |
| Ruta del endpoint | /2/tweets/:id/liking\_users |
| [Autenticación](/es/resources/fundamentals/authentication) | Token de portador OAuth 2.0<br /><br />Contexto de usuario OAuth 1.0a |
| [Límites de velocidad](/es/resources/fundamentals/rate-limits) de solicitud predeterminados | 75 solicitudes por 15 min (por App)<br /><br />75 solicitudes por 15 min (por usuario) |

**Publicaciones que le han gustado a un usuario**

Las siguientes tablas comparan el endpoint estándar v1.1 [GET favorites/list](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-favorites-list) y los endpoints de publicaciones con Me gusta en la API de X v2:

| Descripción | Estándar v1.1 | API de X v2 |
| :--- | :--- | :--- |
| Métodos HTTP compatibles | GET | GET |
| Dominio del host | https://api.x.com | https://api.x.com |
| Ruta del endpoint | /1.1/favorites/list.json | /2/users/:id/liked\_tweets |
| [Autenticación](/es/resources/fundamentals/authentication) | Contexto de usuario OAuth 1.0a | Token de portador OAuth 2.0<br /><br />Contexto de usuario OAuth 1.0a |
| [Límites de velocidad](/es/resources/fundamentals/rate-limits) de solicitud predeterminados | 75 solicitudes por 15 min | 75 solicitudes por 15 min (por App)<br /><br />75 solicitudes por 15 min (por usuario) |
| Formatos de datos | Formato estándar v1.1 | [Formato de la API de X v2](/es/x-api/fundamentals/data-dictionary) (determinado por los parámetros de solicitud fields y expansions; no es compatible con versiones anteriores v1.1)<br /><br />Para obtener más información sobre cómo migrar del formato estándar v1.1 al formato de la API de X v2, visita nuestra [guía de migración de formatos de datos](/es/x-api/migrate/data-format-migration). |

<div id="manage-likes">
  #### Administrar Me gusta
</div>

Los endpoints de la v2 para administrar Me gusta sustituyen a los endpoints de la v1.1 [POST favorites/create](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-create) y [POST favorites/destroy](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-destroy).

Las siguientes tablas comparan los endpoints estándar de la v1.1 y de la API de X v2 para administrar Me gusta:

<div id="like-a-post">
  #### Indicar que te gusta una publicación
</div>

| Descripción | v1.1 estándar | API de X v2 |
| :--- | :--- | :--- |
| Métodos HTTP compatibles | POST | POST |
| Dominio del host | https://api.x.com | https://api.x.com |
| Ruta del endpoint | /1.1/favorites/create.json | /2/users/:id/likes |
| [Autenticación](/es/resources/fundamentals/authentication) | Contexto de usuario OAuth 1.0a | Contexto de usuario OAuth 1.0a |
| [Límites de solicitudes](/es/resources/fundamentals/rate-limits) predeterminados | 1000 solicitudes cada 24 horas (por usuario)<br /><br />1000 solicitudes cada 24 horas (por app) | 50 solicitudes cada 15 min (por usuario)<br /><br />1000 solicitudes cada 24 horas (por usuario, compartidas con DELETE) |
