<div id="getting-started-with-the-manage-retweets-endpoints">
  ### Introducción a los endpoints para gestionar Retweets
</div>

Esta guía de inicio rápido le ayudará a realizar su primera solicitud a los endpoints para gestionar Retweets usando Postman.

Si desea ver código de ejemplo en distintos lenguajes, visite nuestro repositorio de GitHub [Código de ejemplo de la API de X v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code). 

<div id="prerequisites">
  #### Requisitos previos
</div>

Para completar esta guía, necesitarás un conjunto de [claves y tokens](/es/resources/fundamentals/authentication) para autenticar tu solicitud. Puedes generar estas claves y tokens siguiendo estos pasos:

- [Solicita una cuenta de desarrollador](https://developer.x.com/en/apply-for-access) y obtén la aprobación.
- Crea un [Proyecto](/es/resources/fundamentals/projects) y una [App de desarrollador](/es/resources/fundamentals/developer-apps) asociada en el portal para desarrolladores.
- Ve a la página “Keys and tokens” de tu App para generar las credenciales necesarias. Asegúrate de guardar todas las credenciales en un lugar seguro.

<div id="steps-to-build-a-manage-retweets-request">
  #### Pasos para crear una solicitud para gestionar Retweets
</div>

**Paso uno: comience con una herramienta o biblioteca**

Existen varias herramientas, ejemplos de código y bibliotecas que puede usar para realizar una solicitud a este endpoint, pero aquí usaremos la herramienta Postman para simplificar el proceso.

Para cargar la colección de Postman de la API de X v2 en su entorno, haga clic en el siguiente botón:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Agregar API de X v2 a Postman
</Button>

Una vez que tenga la colección de la API de X v2 cargada en Postman, navegue a la carpeta “Retweets” y seleccione “Retweet a Post”.
 

Paso dos: autentique su solicitud

Para realizar correctamente una solicitud a la API de X, debe verificar que tiene permiso. Para hacerlo con este endpoint, debe autenticar su solicitud usando [OAuth 1.0a User Context](/es/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code with PKCE](es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

En este ejemplo, usaremos OAuth 1.0a User Context.

Debe agregar sus claves y tokens —específicamente su API Key, API Secret Key, OAuth 1.0a user Access Token y OAuth 1.0a user Access Token Secret— a Postman. Puede hacerlo seleccionando el entorno llamado “X API v2” en la esquina superior derecha de Postman y agregando sus claves y tokens en los campos "initial value" y "current value" (haciendo clic en el ícono de ojo junto al menú desplegable de entornos).

Estas variables se incorporarán automáticamente en la pestaña de autorización de la solicitud si lo ha hecho correctamente.

**Paso tres: especifique qué publicación va a retuitear**

Los endpoints para gestionar Retweets requieren dos IDs: uno para el usuario que desea retuitear o deshacer un Retweet y el id de la publicación.

El ID del usuario debe corresponder al ID del usuario que autentica, lo que significa que debe pasar los [Access Tokens](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) asociados con ese ID de usuario al autenticar su solicitud. En este caso, puede especificar el ID que pertenece a su propio usuario. Puede encontrar su ID de dos maneras:

1. Usando el endpoint [user lookup by username](/es/x-api/users/user-lookup-by-username), puede pasar un nombre de usuario y recibir el campo id.
2. Revisando su Access Token, verá que la parte numérica es su ID de usuario.
    

También debe especificar una publicación que desea retuitear. Puede encontrar el ID de la publicación navegando a X.com, haciendo clic en una publicación y luego mirando la URL. Por ejemplo, el ID de la publicación de la siguiente URL es 1228393702244134912.

https://x.com/TwitterDev/status/1228393702244134912

En Postman, navegue a la pestaña "Params" e ingrese su ID en la columna "Value" de la variable de ruta id, e ingrese el ID de la publicación que desea retuitear como valor del parámetro tweet\_id. Asegúrese de no incluir espacios antes o después de ningún ID.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | (su ID de usuario) |
| tweet\_id | (el ID del Tweet que desea retuitear) |

Si hace clic en el botón "Send", recibirá un objeto de respuesta que contiene el estado de la relación:

- Si recibe "retweeted": true, entonces el id está retuiteando correctamente el tweet\_id.
   

**Paso cuatro: realice su solicitud y revise su respuesta**

Una vez que tenga todo configurado, presione el botón "Send" y recibirá la siguiente respuesta:

```
{
    "data": {
        "retweeted": true
    }
}
```

Si recibe `"retweeted": true`, se ha retuiteado correctamente el `tweet_id`.

Si desea deshacer un Retweet del mismo usuario, puede usar la solicitud titulada “Undo a Retweet”, que también se encuentra en la carpeta “Retweets” de la colección de la API de X v2 cargada en Postman. El `id` debe ser su ID de usuario y `source_tweet_id` debe ser el ID de la publicación cuyo Retweet desea deshacer. No tendrá que agregar esto como un cuerpo JSON, así que asegúrese de incluir los parámetros de consulta requeridos para `id` y `source_tweet_id`.

Al deshacer correctamente un Retweet, recibirá una respuesta similar al siguiente ejemplo:

```
{
  "data": {
    "retweeted": false
  }
}
```
