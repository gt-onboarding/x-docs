<Note>
  **Configuración de la cuenta**

  Para acceder a estos endpoints, necesitará:

  - Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
  - Autenticarse con las claves y los tokens de una [App de desarrollador](/es/resources/fundamentals/developer-apps) que esté dentro de un [Proyecto](/es/resources/fundamentals/projects).

<div id="retweets-lookup">
  ### Búsqueda de Retweets
</div>

Con el endpoint de búsqueda de Retweets, puedes obtener una lista de cuentas que han retweeteado una publicación. Para este endpoint, se proporcionarán tokens de paginación para navegar por grandes conjuntos de resultados en lotes de hasta 100 usuarios.

Puedes autenticar estos endpoints con [OAuth 1.0a User Context](/es/resources/fundamentals/authentication), \[OAuth 2.0 App-Only]\(https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), o [OAuth 2.0 con PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3). 

<div id="retweets-of-me">
  ### Reposts de mí
</div>

La consulta Reposts of Me te permite ver cuáles de tus propias publicaciones han sido compartidas por otros.

Esto ofrece a los usuarios información sobre qué publicaciones se comparten y cuáles resuenan con su audiencia, lo que puede ayudar a promover interacciones más dirigidas y a ajustar la estrategia de contenido.

Puedes autenticar estos endpoints con [OAuth 1.0a User Context](/es/resources/fundamentals/authentication) o con [OAuth 2.0 Authorization Code with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

<div id="manage-retweets">
  #### Administrar retuits
</div>

Los endpoints para administrar retuits te permiten retuitear o deshacer el retuit de una Publicación específica en nombre de una cuenta autenticada. Para este grupo de endpoints, hay dos métodos disponibles: POST y DELETE. El método POST te permite retuitear una Publicación y el método DELETE te permite deshacer el retuit de una Publicación determinada.

Como estás realizando solicitudes en nombre de un usuario, debes autenticar estos endpoints con [OAuth 1.0a User Context](/es/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code with PKCE](es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), y utilizar los tokens de acceso del usuario asociados con el usuario en cuyo nombre realizas la solicitud. Puedes generar este token de acceso del usuario usando el [flujo de OAuth de 3 patas](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) o usando el [flujo de concesión Authorization Code con PKCE](/es/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0). Puedes retuitear una Publicación desde tu cuenta o desde la cuenta de un usuario autenticado. Con ambos endpoints, hay un límite por usuario de 50 solicitudes cada 15 minutos por endpoint.
