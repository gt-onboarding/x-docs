<div id="comparing-x-apis-retweets-endpoints">
  ## Comparación de los endpoints de Retweets de la API de X
</div>

**Consulta de Retweets**

El endpoint de consulta de Retweets de v2 reemplazará los endpoints estándar [v1.1 GET statuses/retweets/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweets-id) y [v1.1 GET statuses/retweets/:ids](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-statuses-retweeters-ids).

Las siguientes tablas comparan los endpoints de Retweets de la v1.1 estándar y de la API de X v2:

| Descripción | v1.1 estándar | API de X v2 |
| :--- | :--- | :--- |
| Métodos HTTP compatibles | GET | GET |
| Dominio del host | https://api.x.com | https://api.x.com |
| Ruta del endpoint | /1.1/retweeters/id.json<br /><br />`/1.1/retweets/ids.json` | /2/users/:id/retweeted\_by |
| [Autenticación](/es/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 2.0 Bearer Token<br /><br />OAuth 1.0a User Context |
| [Límites de frecuencia](/es/resources/fundamentals/rate-limits) de solicitud predeterminados | 75 solicitudes por 15 min | 75 solicitudes por 15 min (por App)<br /><br />75 solicitudes por 15 min (por usuario) |
| Formato de datos | Formato estándar v1.1 | [Formato de la API de X v2](/es/x-api/fundamentals/data-dictionary) (determinado por los parámetros de solicitud fields y expansions; no es retrocompatible con los formatos de v1.1)<br /><br />Para obtener más información sobre cómo migrar del formato estándar v1.1 al formato de la API de X v2, visita nuestra [guía de migración de formatos de datos](/es/x-api/migrate/data-format-migration). |
| Requiere el uso de credenciales de una [App de desarrollador](/es/resources/fundamentals/developer-apps) asociada a un [Proyecto](/es/resources/fundamentals/projects) |     | ✔️  |

<div id="manage-retweets">
  ### Gestionar los Retweets
</div>

Las siguientes tablas comparan el endpoint para deshacer un Retweet en la versión estándar v1.1 y en la API de X v2:

**Retuitear una publicación**

| Descripción | Estándar v1.1 | API de X v2 |
| :--- | :--- | :--- |
| Métodos HTTP compatibles | POST | POST |
| Dominio del host | https://api.x.com | https://api.x.com |
| Ruta del endpoint | /1.1/statuses/retweet/:id.json | /2/users/:id/retweets |
| [Autenticación](/es/resources/fundamentals/authentication) | Contexto de usuario de OAuth 1.0a | Contexto de usuario de OAuth 1.0a |
| [Límites de frecuencia](/es/resources/fundamentals/rate-limits) predeterminados de la solicitud | Ninguno<br /><br />300 solicitudes por ventana de 3 horas (por usuario, por app). Se comparte con el endpoint POST Tweet | 50 solicitudes por cada 15 min (por usuario)<br /><br />300 solicitudes por ventana de 3 horas (por usuario, por app). Se comparte con el endpoint POST Tweet para gestionar publicaciones. |
| Requiere el uso de credenciales de una [App de desarrollador](/es/resources/fundamentals/developer-apps) asociada a un [Proyecto](/es/resources/fundamentals/projects) |     | ✔️  |
