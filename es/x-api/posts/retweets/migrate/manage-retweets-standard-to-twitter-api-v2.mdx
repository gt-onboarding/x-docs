<div id="manage-retweets-standard-v11-compared-to-x-api-v2">
  ### Gestionar Retweets: v1.1 estándar frente a API de X v2
</div>

Si ha estado trabajando con los endpoints estándar v1.1 [POST statuses/retweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-retweet-id) y [POST statuses/unretweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-unretweet-id), el objetivo de esta guía es ayudarle a comprender las similitudes y diferencias entre los endpoints de Retweets estándar y de la API de X v2.

- **Similitudes**
  - Autenticación
- **Diferencias**
  - URLs de endpoints y métodos HTTP
  - Limitaciones de la solicitud
  - Requisitos de App y Proyecto
  - Parámetros de la solicitud

<div id="similarities">
  #### Similitudes
</div>

**Autenticación**

Tanto los endpoints de gestión de retuits de la versión estándar v1.1 ([POST statuses/retweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-retweet-id) y [POST statuses/unretweet/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-unretweet-id)) como los de la API de X v2 usan [OAuth 1.0a User Context](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Por lo tanto, si anteriormente se utilizaba uno de los endpoints de consulta de retuits de la versión estándar v1.1, se puede seguir utilizando el mismo método de autenticación al migrar a la versión de la API de X v2. 

<div id="differences">
  #### Diferencias
</div>

**URLs de endpoints y métodos HTTP**

- Endpoints de Standard v1.1:
  - https://api.x.com/1.1/statuses/retweet/:id.json
    (Hace Retweet de una publicación. Devuelve la publicación original con los detalles del Retweet incorporados)
  - https://api.x.com/1.1/statuses/unretweet/:id.json
    (Deshace un Retweet. Devuelve la publicación original con los detalles del Retweet incorporados)
- Endpoint de la API de X v2:
  - https://api.x.com/2/tweets/:id/retweets
    (Hace Retweet de una publicación específica)
  - https://api.x.com/2/users/:id/retweets/:source\_tweet\_id
    (Deshace el Retweet de una publicación específica) 

**Requisitos de App y Proyecto**

Los endpoints de la API de X v2 requieren que uses credenciales de una [App de desarrollador](/es/resources/fundamentals/developer-apps) asociada a un [Proyecto](/es/resources/fundamentals/projects) al autenticar tus solicitudes. Todos los endpoints de la API de X v1.1 pueden usar credenciales de Apps independientes o Apps asociadas a un Proyecto.

**Parámetros de solicitud**

Los siguientes endpoints de Standard v1.1 aceptaban dos parámetros de consulta (user\_id o screen\_name). La API de X v2 solo acepta el ID numérico del usuario y debe pasarse como parte de la ruta del endpoint.

| Standard v1.1 | API de X v2 |
| :--- | :--- |
| **id** | **id** |
| **includes\_entities** | Sin equivalente |

Ten en cuenta que los parámetros de Standard v1.1 se pasan como parámetros de consulta, mientras que en la API de X v2 se pasan como parámetros del cuerpo para el endpoint POST o como parámetros de ruta para el endpoint DELETE.

Además, no se requiere el id del usuario que hace Retweet de una publicación cuando se usan los endpoints de Standard v1.1, ya que los [tokens de acceso](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) enviados con [OAuth 1.0a User Context](/es/resources/fundamentals/authentication) permiten inferir qué usuario inicia el Retweet o deshace un Retweet.
