<div id="authentication">
  #### Autenticación
</div>

Todos los endpoints de la API de X v2 requieren que autentique sus solicitudes con un conjunto de credenciales, también conocidas como claves y tokens.

Puede usar OAuth 1.0a User Context o OAuth 2.0 Bearer Token para autenticar sus solicitudes al endpoint de consulta de Retweets.

Los endpoints para gestionar Retweets requieren el uso de OAuth 1.0a User Context, lo que significa que debe usar un conjunto de claves de API y tokens de acceso de usuario para efectuar una solicitud correcta. Los tokens de acceso deben estar asociados con el usuario en cuyo nombre está realizando la solicitud. Si desea generar un conjunto de tokens de acceso para otro usuario, este debe autorizar o autenticar su App utilizando el flujo de OAuth de 3 etapas (3-legged OAuth).

Tenga en cuenta que OAuth 1.0a puede resultar complejo de usar. Si no está familiarizado con este método de autenticación, recomendamos usar una [biblioteca](/es/x-api/tools-and-libraries/overview) para autenticar correctamente sus solicitudes.

**Tenga en cuenta**

Si solicita los siguientes campos, se requiere OAuth 1.0a User Context:

- tweet.fields.non\_public\_metrics
- tweet.fields.promoted\_metrics
- tweet.fields.organic\_metrics
- media.fields.non\_public\_metrics
- media.fields.promoted\_metrics
- media.fields.organic\_metrics

<div id="developer-portal-projects-and-developer-apps">
  #### Portal para desarrolladores, Proyectos y Apps de desarrollador
</div>

Para obtener un conjunto de credenciales de autenticación que funcionen con los endpoints de la API de X v2, debe [registrarse para obtener una cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info), configurar un [Proyecto](/es/resources/fundamentals/projects) dentro de esa cuenta y crear una [App de desarrollador](/es/resources/fundamentals/developer-apps) dentro de ese Proyecto. Luego podrá encontrar sus claves y tokens dentro de su App de desarrollador. 
 

<div id="rate-limits">
  #### Límites de tasa
</div>

Cada día, miles de desarrolladores realizan solicitudes a la API de X. Para gestionar el gran volumen de estas solicitudes, se aplican [límites de tasa](/es/x-api/fundamentals/rate-limits) a cada endpoint que restringen la cantidad de solicitudes que puedes hacer en nombre de tu app o de un usuario autenticado.

Los endpoints para gestionar Retweets están limitados a 50 solicitudes por cada ventana de 15 minutos (por usuario). Además, para el endpoint POST, el límite es de 300 solicitudes por ventana de 3 horas (por usuario, por app).

Con el endpoint de consulta de Retweets, el límite es de 75 solicitudes por ventana de 15 minutos. Además, este endpoint solo devolverá a los 100 usuarios que hayan hecho Retweet más recientemente.

<div id="fields-and-expansions">
  #### Campos y expansiones
</div>

La API de X v2 permite seleccionar exactamente qué datos se desean obtener de la API mediante un conjunto de herramientas denominadas campos y expansiones. El parámetro expansion permite ampliar los objetos referenciados en la carga útil. Por ejemplo, este endpoint permite extraer las siguientes [expansiones](/es/x-api/fundamentals/expansions):

- attachments.poll\_ids
- attachments.media\_keys
- author\_id, entities.mentions.username
- geo.place\_id
- in\_reply\_to\_user\_id,
- referenced\_tweets.id,
- referenced\_tweets.id.author\_id

El parámetro fields permite seleccionar exactamente qué [campos](/es/x-api/fundamentals/fields) dentro de los distintos objetos de datos se desea recibir. Estos endpoints proporcionan principalmente objetos Post. De forma predeterminada, el objeto Post devuelve los campos id y text. Para recibir campos adicionales como tweet.created\_at o tweet.entities, será necesario solicitarlos explícitamente usando un parámetro fields. Algunos campos importantes que quizá se desee considerar en la integración son los datos de encuestas, las métricas, las anotaciones de la publicación y los campos de ID de conversación.

Hemos añadido una guía sobre cómo [usar campos y expansiones](/es/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) conjuntamente en nuestro [diccionario de datos de la API de X v2](/es/x-api/fundamentals/data-dictionary).
