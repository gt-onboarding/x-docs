<div id="manage-replies-by-topic">
  ### Gestiona las respuestas por tema
</div>

Con el endpoint de ocultar respuestas, puedes crear integraciones que ayuden a personas y marcas a mantener sus conversaciones en el tema. Esta página muestra cómo gestionar una conversación usando los endpoints de ocultar respuestas y [búsqueda reciente](/es/x-api/posts/recent-search).

Búsqueda reciente ofrece funcionalidad para recuperar una conversación y sus respuestas, y la carga útil de la Publicación devuelve [anotaciones de la Publicación](/es/x-api/fundamentals/post-annotations) que te ayudan a entender el contexto y el tema de cada Publicación, sin importar el idioma.

El flujo de la app incluirá controles para mostrar y gestionar una conversación:

1. Solicita el permiso del usuario para leer sus Publicaciones y gestionar sus respuestas.

2. Recupera una conversación reciente a partir de la URL de una Publicación y verifica que la conversación pertenezca al usuario autenticado.

3. Llamará al endpoint de búsqueda reciente para mostrar cada Publicación de la conversación. La solicitud incluirá una consulta por ID de conversación y la expansión de anotaciones para determinar si la Publicación está relacionada con deportes o no, según la interpretación de X de la Publicación.

4. Llama a Ocultar respuestas para ocultar una respuesta cuando el usuario así lo decida. También ofrecerá una forma de deshacer esta acción, para que el usuario siempre tenga el control.

5. Para conversaciones más largas, ofrecerá controles para [paginar los resultados de búsqueda](/es/x-api/posts/search/integrate/paginate).
    

<div id="optimize-for-the-user-and-for-usage">
  #### Optimiza para el usuario (y para el uso)
</div>

Puedes diseñar un flujo que dé al usuario el control de cualquier acción que quiera realizar. Mantener este principio en mente también te ayuda a crear una integración que optimice el consumo de Publicaciones.

1. Dado que el usuario autenticado solo puede gestionar conversaciones que inició, tu flujo debería finalizar de forma anticipada cuando no sea el caso.
   - Realiza una consulta inicial de la Publicación. Finaliza el flujo de forma anticipada si la URL de la Publicación no es válida o la conversación no fue iniciada por el usuario que se está autenticando.
   - Así, tu app no tendrá que realizar una búsqueda reciente si la conversación no puede ser moderada por el usuario autenticado.
2. Solicita [campos de usuario y de Publicación](/es/x-api/fundamentals/fields) en la misma solicitud para evitar solicitudes separadas. Este enfoque también puede mejorar el rendimiento de tu app.
3. Evita realizar solicitudes cuando no sea necesario. Esta app almacena en caché el estado de oculto de una respuesta en el navegador del usuario. Esto es útil en conversaciones más extensas, donde el usuario puede querer retomar la moderación más adelante, y ayuda a tu app a optimizar las solicitudes para ocultar o dejar de ocultar respuestas.
