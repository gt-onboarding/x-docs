<div id="getting-started-with-the-hide-replies-endpoint">
  ## Introducción al endpoint para ocultar respuestas
</div>

Esta guía de inicio rápido le ayudará a realizar su primera solicitud al endpoint para ocultar respuestas con [Postman](/es/tutorials/postman-getting-started).

Si desea ver fragmentos de código en distintos lenguajes, visite la [página de referencia del API para ocultar respuestas](/es/x-api/posts/hide-replies#put-2-tweets-id-hidden).

<Note>
  ### Requisitos previos

  Para completar esta guía, necesitará un conjunto de [claves y tokens](/es/resources/fundamentals/authentication) para autenticar su solicitud. Puede generar estas claves y tokens siguiendo estos pasos:

  - [Regístrese para obtener una cuenta de desarrollador](https://developer.x.com/en/apply-for-access) y reciba la aprobación.
  - Cree un [Proyecto](/es/resources/fundamentals/projects) y una [App de desarrollador](/es/resources/fundamentals/developer-apps) asociada en el portal para desarrolladores.
  - Vaya a la página “Keys and tokens” de su App para generar las credenciales necesarias. Asegúrese de guardar todas las credenciales en un lugar seguro.
</Note>

<div id="steps-to-build-a-put-tweetsidhidden-request">
  ### Pasos para construir una solicitud PUT /tweets/:id/hidden
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### Primer paso: Comience con una herramienta o biblioteca
</div>

Existen varias herramientas, ejemplos de código y bibliotecas que puede usar para realizar una solicitud a este endpoint, pero aquí usaremos Postman para simplificar el proceso.

Para cargar la colección de Postman de la API de X v2 en su entorno, haga clic en el siguiente botón:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Agregar API de X v2 a Postman
</Button>

Una vez que tenga cargada en Postman la colección de la API de X v2, navegue al endpoint para ocultar respuestas.

<div id="step-two-authenticate-your-request">
  #### Paso dos: Autentique su solicitud
</div>

Para realizar correctamente una solicitud a la API de X, debe verificar que tiene permiso. Para hacerlo con este endpoint, debe autenticar su solicitud usando [OAuth 1.0a User Context](/es/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code with PKCE](es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

En este ejemplo, usaremos OAuth 1.0a User Context.

Debe agregar sus claves y tokens —específicamente su API Key, API Secret Key, OAuth 1.0a user Access Token y OAuth 1.0a user Access Token Secret— a Postman. Puede hacerlo seleccionando el entorno llamado “API de X v2” en la esquina superior derecha de Postman y agregando sus claves y tokens en los campos “initial value” y “current value” (haciendo clic en el ícono de ojo junto al menú desplegable de entornos).

Estas variables se incorporarán automáticamente en la pestaña de autorización de la solicitud si lo ha hecho correctamente.
 

<div id="step-three-find-a-post-id-to-hide">
  #### Paso tres: Encontrar un ID de Publicación para ocultar
</div>

El endpoint para ocultar respuestas puede ocultar o mostrar respuestas en nombre de un usuario autorizado. Como en este ejemplo estamos usando los tokens de acceso asociados a tu perfil de usuario, podrás ocultar respuestas de usuarios que participen en una conversación iniciada por ti. De manera similar, si usaras tokens de acceso que pertenecen a otro usuario que autorizó tu app, podrías moderar respuestas en cualquier conversación iniciada por esa cuenta.

Pídele a un amigo que responda a una Publicación (avísale que estás probando la función de ocultar respuestas) o responde a cualquiera de tus Publicaciones desde una cuenta de prueba. Haz clic en esa respuesta y luego copia la parte numérica de su URL. Ese será el ID de la Publicación que ocultaremos.

En este caso, veremos la siguiente Publicación, que tiene el ID `1232720193182412800`:

`https://x.com/TwitterDev/status/1232720193182412800`

<div id="step-four-hide-the-post">
  #### Paso cuatro: Ocultar la publicación
</div>

En Postman, abre la carpeta Hide replies y selecciona Hide a reply. En la pestaña Params, pega el ID de la publicación junto al campo id (no necesitas reemplazar :id en la URL). Haz clic en "Send" y verás una respuesta satisfactoria.

`{"hidden":true}`

<div id="step-five-unhide-the-post">
  #### Paso cinco: Dejar de ocultar la Publicación
</div>

Las Publicaciones ocultas se mueven a una pestaña separada en la app de X. Para dejar de ocultar una Publicación en Postman, abre la carpeta Hide replies y selecciona Unhide a reply. En la pestaña Params, pega el mismo ID de la Publicación usado en el paso anterior en el campo id. Haz clic en "Send" y verás una respuesta satisfactoria.

`{"hidden":false}`

El campo hidden representa el estado de ocultación de la Publicación. Un estado hidden en true significa que la Publicación está oculta. Del mismo modo, false significa que la Publicación no está oculta.
