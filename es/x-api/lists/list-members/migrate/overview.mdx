<div id="comparing-x-apis-list-members-endpoints">
  ## Comparación de los endpoints de miembros de Listas de la API de X
</div>

El grupo de endpoints de miembros de Listas de v2 reemplazará los endpoints estándar de v1.1 [GET lists/members](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-members), [GET lists/memberships](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-memberships), [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create) y [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy). Si cuenta con código, apps o herramientas que utilizan alguna de estas versiones de los endpoints de miembros de Listas y está considerando migrar al endpoint más reciente de la API de X v2, este conjunto de guías es para usted. 

<div id="list-members-lookup">
  ### Búsqueda de miembros de listas
</div>

Los endpoints de búsqueda de miembros de listas de v2 reemplazarán los endpoints estándar [GET lists/members](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-members), [GET lists/memberships](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-memberships).

Las siguientes tablas comparan los endpoints de listas estándar de v1.1 y de la API de X v2:

**Búsqueda de miembros de listas**

|     |     |     |
| :--- | :--- | :--- |
| Descripción | Estándar v1.1 | API de X v2 |
| Métodos HTTP admitidos | `GET` | `GET` |
| Dominio host | `https://api.x.com` | `https://api.x.com` |
| Ruta del endpoint | `/1.1/lists/members.json` | `/2/lists/:id/members` |
| [Autenticación](/es/resources/fundamentals/authentication) | Contexto de usuario de OAuth 1.0a<br /><br />Solo app | Contexto de usuario de OAuth 1.0a<br /><br />OAuth 2.0 Código de autorización con PKCE<br /><br />Solo app |
| [Límites de velocidad](/es/resources/fundamentals/rate-limits) predeterminados de solicitud | 900 solicitudes por 15 min con OAuth 1.0a<br /><br />75 solicitudes por 15 min con solo app | 900 solicitudes por 15 min con OAuth 1.0a<br /><br />900 solicitudes por 15 min con OAuth 2.0<br /><br />900 solicitudes por 15 min con solo app |

**Búsqueda de membresías de listas**

|     |     |     |
| :--- | :--- | :--- |
| Descripción | Estándar v1.1 | API de X v2 |
| Métodos HTTP admitidos | `GET` | `GET` |
| Dominio host | `https://api.x.com` | `https://api.x.com` |
| Ruta del endpoint | `/1.1/lists/memberships.json` | `/2/users/:id/list_memberships` |
| [Autenticación](/es/resources/fundamentals/authentication) | Contexto de usuario de OAuth 1.0a<br /><br />Solo app | Contexto de usuario de OAuth 1.0a<br /><br />OAuth 2.0 Código de autorización con PKCE<br /><br />Solo app |
| [Límites de velocidad](/es/resources/fundamentals/rate-limits) predeterminados de solicitud | 75 solicitudes por 15 min con OAuth 1.0a<br /><br />75 solicitudes por 15 min con solo app | 75 solicitudes por 15 min con OAuth 1.0a<br /><br />75 solicitudes por 15 min con OAuth 2.0<br /><br />75 solicitudes por 15 min con solo app |

<div id="manage-list-members">
  ### Administrar miembros de listas
</div>

Los endpoints de la v2 para administrar miembros de listas reemplazarán los endpoints estándar [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create) y [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy).

Las siguientes tablas comparan los endpoints de listas estándar de v1.1 y de la API de X v2:

**Agregar miembro**

| Descripción | Estándar v1.1 | API de X v2 |
| :--- | :--- | :--- |
| Métodos HTTP compatibles | POST | POST |
| Dominio host | https://api.x.com | https://api.x.com |
| Ruta del endpoint | /1.1/lists/members/create.json | /2/lists/:id/members |
| [Autenticación](/es/resources/fundamentals/authentication) | Contexto de usuario OAuth 1.0a | Contexto de usuario OAuth 1.0a<br /><br />Código de autorización OAuth 2.0 con PKCE |
| [Límites de velocidad](/es/resources/fundamentals/rate-limits) predeterminados de la solicitud | Ninguno | 300 solicitudes por 15 min (por usuario) |

**Eliminar miembro**

| Descripción | Estándar v1.1 | API de X v2 |
| :--- | :--- | :--- |
| Métodos HTTP compatibles | POST | DELETE |
| Dominio host | https://api.x.com | https://api.x.com |
| Ruta del endpoint | /1.1/lists/members/destroy.json | /2/lists/:id/:user\_id |
| [Autenticación](/es/resources/fundamentals/authentication) | Contexto de usuario OAuth 1.0a | Contexto de usuario OAuth 1.0a<br /><br />Código de autorización OAuth 2.0 con PKCE |
| [Límites de velocidad](/es/resources/fundamentals/rate-limits) predeterminados de la solicitud | Ninguno | 300 solicitudes por 15 min (por usuario) |

Para acceder a los endpoints de la API de X v2, debe [registrarse para obtener una cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info). Al autenticarse, debe usar claves y tokens de una [App de desarrollador](/es/resources/fundamentals/developer-apps) que se encuentre dentro de un [Proyecto](/es/resources/fundamentals/projects).

Obtenga más información sobre cómo acceder a los endpoints de la API de X v2 en nuestra página de [inicio](/es/x-api/getting-started/getting-access).
