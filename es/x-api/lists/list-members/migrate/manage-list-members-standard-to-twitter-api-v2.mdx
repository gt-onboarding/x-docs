<div id="manage-list-members-standard-v11-compared-to-x-api-v2">
  ### Administrar miembros de listas: estándar v1.1 vs. API de X v2
</div>

Si ha estado trabajando con los endpoints estándar v1.1 [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create) y [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy), el objetivo de esta guía es ayudarle a comprender las similitudes y diferencias entre los endpoints de administración de miembros de listas en el estándar v1.1 y en la API de X v2.

- **Similitudes**
  - Autenticación
- **Diferencias**
  - URL de los endpoints
  - Requisitos de Apps y Proyectos
  - Métodos HTTP
  - Límites de uso
  - Parámetros de solicitud

<div id="similarities">
  #### Similitudes
</div>

**Autenticación**

Ambas versiones del endpoint admiten [OAuth 1.0a User Context](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Por lo tanto, si antes usaba uno de los endpoints estándar de la v1.1 para gestionar miembros de listas, puede continuar utilizando el mismo método de autenticación si migra a la API de X v2.

<div id="differences">
  #### Diferencias
</div>

**URLs de endpoints**

- Endpoints estándar v1.1:
  - POST https://api.x.com/1.1/lists/members/create.json
    (Agrega un miembro a una Lista especificada)
  - POST https://api.x.com/1.1/lists/members/destroy.json
    (Elimina a un miembro de una Lista especificada)
- Endpoint de la API de X v2:
  - POST https://api.x.com/2/lists/:id/members
    (Agrega un miembro a una Lista especificada)

  - DELETE https://api.x.com/2/lists/:id/members/:user\_id
    (Elimina a un miembro de una Lista especificada)

**Límites de frecuencia**

| **Estándar v1.1** | **API de X v2** |
| :--- | :--- |
| /1.1/lists/members/create.json<br /><br />ninguno | /2/lists/:id/members<br /><br />300 solicitudes por ventana de 15 minutos con OAuth 1.0a (User Context)<br /><br />300 solicitudes por ventana de 15 minutos con OAuth 2.0 (Authorization Code con PKCE) |
| /1.1/lists/members/destroy.json<br /><br />ninguno | /2/lists/:id/members/:user\_id<br /><br />300 solicitudes por ventana de 15 minutos con OAuth 1.0a (User Context)<br /><br />300 solicitudes por ventana de 15 minutos con OAuth 2.0 (Authorization Code con PKCE) |

**Requisitos de App y Proyecto**

Los endpoints de la API de X v2 requieren usar credenciales de una [App de desarrollador](/es/resources/fundamentals/developer-apps) asociada a un [Proyecto](/es/resources/fundamentals/projects) al autenticar tus solicitudes. Todos los endpoints de la API de X v1.1 pueden usar credenciales de Apps independientes o Apps asociadas a un proyecto.

**Parámetros de solicitud**

Los siguientes parámetros de solicitud estándar v1.1 tienen equivalentes en la API de X v2:

| **Estándar v1.1** | **API de X v2** |
| :--- | :--- |
| list\_id | id  |
| slug | Sin equivalente |
| screen\_name | Sin equivalente |
| owner\_screen\_name | Sin equivalente |
| owner\_id | Sin equivalente |
