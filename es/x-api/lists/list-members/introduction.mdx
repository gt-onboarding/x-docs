<div id="list-members-lookup">
  ### Consulta de miembros de Listas
</div>

El grupo de consulta de miembros tiene dos endpoints disponibles. Puede obtener detalles de los miembros de una Lista específica y ver en qué Listas participa un usuario. Estos endpoints pueden usarse para permitir que las personas creen y organicen nuevas Listas basándose en la información de membresía.

Hay un límite de 900 solicitudes por 15 minutos al consultar detalles de miembros y un límite de 75 solicitudes por 15 minutos al consultar membresías de usuario.

Puede usar [OAuth 1.0a User Context](/es/resources/fundamentals/authentication), [App only](https://developer.x.com/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) o [OAuth 2.0 Authorization Code with PKCE](es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) para autenticar sus solicitudes a este endpoint. 
 

<div id="manage-list-members">
  ### Administrar miembros de Listas
</div>

Los endpoints para administrar miembros de Listas le permiten agregar y eliminar miembros de una Lista en nombre de un usuario autenticado. Para estos endpoints hay dos métodos disponibles: POST y DELETE. El método POST le permite agregar un miembro a una Lista y el método DELETE le permite eliminar un miembro de una Lista. Existe un límite de tasa por usuario de 300 solicitudes cada 15 minutos para ambos endpoints.

Tenga en cuenta que las Listas no pueden tener más de 5.000 miembros.

Como está realizando solicitudes en nombre de un usuario con estos endpoints, debe autenticarlo con [OAuth 1.0a User Context](/es/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code with PKCE](es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), y usar los tokens de acceso de usuario asociados con un usuario que haya autorizado su App, los cuales se pueden generar usando el [flujo de OAuth de 3 patas](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) o el [flujo de concesión Authorization Code with PKCE](/es/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0).

<Note>
  **Configuración de la cuenta**

  Para acceder a estos endpoints, necesitará:

  - Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
  - Autenticarse utilizando las claves y tokens de una [App de desarrollador](/es/resources/fundamentals/developer-apps) que se encuentre dentro de un [Project](/es/resources/fundamentals/projects).

  Obtenga más información sobre cómo acceder a los endpoints de la API de X v2 en nuestra [guía de introducción](/es/x-api/getting-started/getting-access).
</Note>
