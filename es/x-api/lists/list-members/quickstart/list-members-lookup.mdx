<div id="steps-to-build-a-list-members-lookup-request">
  ### Pasos para crear una solicitud de búsqueda de miembros de una Lista
</div>

**Paso uno: Elija la colección de endpoints de Lista en Postman**

Una vez que tenga cargada la colección de la API de X v2 en Postman, vaya a la carpeta “List”, seleccione la subcarpeta “List members” y luego elija "Members lookup".
 

**Paso dos: Identifique y especifique de qué Lista desea recuperar miembros**

Debe especificar una Lista de la cual desea obtener miembros. Puede encontrar el ID de la Lista navegando a x.com, haciendo clic en una Lista y luego mirando la URL. Por ejemplo, el ID de la Lista de la siguiente URL es 84839422.

https://x.com/i/lists/84839422

El ID de destino puede ser cualquier ID de Lista válido. En Postman, vaya a la pestaña "Params" e introduzca su ID en la columna "Value" de la variable de ruta id. Asegúrese de no incluir espacios antes o después de ningún ID.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 84839422 (List ID) |

**Paso tres: Identifique y especifique qué campos desea recuperar**

Si hace clic en el botón "Send" después del paso tres, recibirá en su respuesta los campos predeterminados del [user object](/es/x-api/fundamentals/data-dictionary#user): id, name y username.

Si desea recibir campos adicionales más allá de id, name y username, deberá especificarlos en su solicitud con los parámetros [field](/es/x-api/fundamentals/fields) y/o [expansion](/es/x-api/fundamentals/expansions).

Para este ejercicio, solicitaremos tres conjuntos adicionales de campos de diferentes objetos:

- El campo adicional user.created\_at en los objetos de usuario principales.
- El [Post object](/es/x-api/fundamentals/data-dictionary#tweet) completo usando el parámetro expansion.
- El campo adicional tweet.created\_at en los objetos de Publicación asociados.

En Postman, vaya a la pestaña "Params" y agregue el siguiente par clave:valor a la tabla "Query Params":

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| user.fields | created\_at | `created_at` |
| expansions | pinned\_tweet\_id | `includes.tweets.id,  <br/>includes.tweets.text  <br/>` |
| tweet.fields | created\_at | `includes.tweets.created_at` |

Ahora debería ver una URL similar junto al botón “Send”:

`https://api.x.com/2/lists/84839422/members?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=created_at`
