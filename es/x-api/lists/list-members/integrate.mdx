<div id="authentication">
  #### Autenticación
</div>

Todos los endpoints de la API de X v2 requieren que autentiques tus solicitudes con un conjunto de credenciales, también conocidas como claves y tokens. Puedes usar OAuth 1.0a User Context, OAuth 2.0 Authorization Code con PKCE o App only para autenticar tus solicitudes en los endpoints de Listas de **lookup**. Sin embargo, debes autenticarte con OAuth 1.0a User Context u OAuth 2.0 para los endpoints de Listas de **manage**.

[OAuth 1.0a User Context](/es/resources/fundamentals/authentication) significa que debes usar un conjunto de API Keys y Access Tokens de usuario para realizar una solicitud correcta. Los access tokens deben estar asociados con el usuario en cuyo nombre realizas la solicitud. Si deseas generar un conjunto de access tokens para otro usuario, ese usuario debe autorizar tu App usando el [flujo OAuth de 3 pasos](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow).

Ten en cuenta que OAuth 1.0a puede ser difícil de usar. Si no estás familiarizado con este método de autenticación, te recomendamos usar una [biblioteca](/es/x-api/tools-and-libraries/overview), una herramienta como Postman, o usar OAuth 2.0 o App only para autenticar tus solicitudes.

[OAuth 2.0 Authorization Code with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) permite un mayor control sobre el alcance (scope) de una aplicación y flujos de autorización en múltiples dispositivos. OAuth 2.0 te permite elegir ámbitos detallados (scopes granulares) que te otorgan permisos específicos en nombre de un usuario.

Para habilitar OAuth 2.0 en tu App, debes activarlo en la configuración de autenticación de tu App, ubicada en la sección de configuración de la App del portal para desarrolladores.

[App only](/es/resources/fundamentals/authentication#oauth-2-0) solo requiere que incluyas un [App only Access Token](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) en tu solicitud. Puedes generar un App only Access Token directamente en una App de desarrollador o generarlo usando el endpoint [POST oauth2/token](/es/resources/fundamentals/authentication#post-oauth2-token).

[App only](/es/resources/fundamentals/authentication#oauth-2-0) solo requiere que incluyas un [App only Access Token](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) en tu solicitud. Puedes generar un App only Access Token directamente en una App de desarrollador o generarlo usando el endpoint [POST oauth2/token](/es/resources/fundamentals/authentication#post-oauth2-token).

<div id="developer-portal-projects-and-developer-apps">
  #### Portal para desarrolladores, Proyectos y Apps de desarrollador
</div>

Para obtener un conjunto de credenciales de autenticación que funcione con los endpoints de la API de X v2, debe [registrarse para una cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info), configurar un [Proyecto](/es/resources/fundamentals/projects) en esa cuenta y crear una [App de desarrollador](/es/resources/fundamentals/developer-apps) dentro de ese Proyecto. Luego podrá encontrar sus claves y tokens en su App de desarrollador.
 

<div id="rate-limits">
  #### Límites de uso
</div>

Cada día, muchos miles de desarrolladores realizan solicitudes a la API de X. Para ayudar a gestionar el gran volumen de estas solicitudes, se aplican [límites de uso](/es/x-api/fundamentals/rate-limits) a cada endpoint que restringen la cantidad de solicitudes que puedes realizar en nombre de tu App o de un usuario autenticado.

Los endpoints de consulta (GET) tienen límites de uso tanto a nivel de App como a nivel de usuario, mientras que los endpoints de gestión (POST/DELETE) están limitados a nivel de usuario. El límite de uso de la App significa que tú, como desarrollador, solo puedes realizar una cierta cantidad de solicitudes a este endpoint durante un período determinado desde cualquier App (ya sea usando la API Key y la API Secret Key, o el App-only Access Token). El límite de uso a nivel de usuario significa que el usuario autenticado en cuyo nombre haces la solicitud solo puede realizar una consulta de Lista un número determinado de veces en cualquier App de desarrollador.

El siguiente cuadro muestra los límites de uso para cada endpoint.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Método HTTP** | **Límite de uso** |
| /2/lists/:id/members | GET | 900 solicitudes por 15 minutos |
| /2/users/:id/list\_memberships | GET | 75 solicitudes por 15 minutos |
| /2/lists/:id/members | POST | 300 solicitudes por 15 minutos |
| /2/lists/:id/members/:user\_id | DELETE | 300 solicitudes por 15 minutos |

<div id="fields-and-expansions">
  #### Campos y expansiones
</div>

El endpoint GET de la API de X v2 permite seleccionar con precisión qué datos devolver desde la API usando un conjunto de herramientas llamadas `fields` y `expansions`. El parámetro `expansions` permite ampliar los objetos referenciados en el payload. Por ejemplo, consultar los miembros de una Lista permite obtener las siguientes [expansions](/es/x-api/fundamentals/expansions):

- `pinned_tweet_id`

El parámetro `fields` permite seleccionar exactamente qué [fields](/es/x-api/fundamentals/fields) dentro de los distintos objetos de datos se desean recibir. La consulta de miembros de Lista entrega principalmente objetos de usuario. De forma predeterminada, el objeto de usuario devuelve los campos id, name y username. Para recibir campos adicionales como `user.created_at` o `user.description`, será necesario solicitarlos explícitamente usando el parámetro user.fields. 

Hemos añadido una guía sobre el uso de [fields and expansions](/es/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions).

La tabla a continuación muestra los fields y expansions disponibles para cada endpoint de consulta:

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Fields** | **Expansions** |
| /2/lists/:id/members | `user.fields`<br /><br />`tweet.fields` | `pinned_tweet_id` |
| /2/users/:id/list\_memberships | list.fields<br /><br />user.fields | owner\_id |

<div id="pagination">
  #### Paginación
</div>

Consultar membresías/miembros puede devolver muchos datos. Para garantizar resultados consistentes y de alto rendimiento en todo momento, usamos paginación. La paginación es una funcionalidad en los endpoints de la API de X v2 que devuelve más resultados de los que pueden incluirse en una sola respuesta. Cuando eso sucede, los datos se devuelven en una serie de “páginas”. Obtén más información sobre cómo [paginar los resultados.](/es/x-api/fundamentals/pagination)
