<div id="authentication">
  #### Autenticación
</div>

Todos los endpoints de la API de X v2 requieren que autentique sus solicitudes con un conjunto de credenciales, también conocidas como claves y tokens. Puede usar OAuth 1.0a en contexto de usuario, solo app u OAuth 2.0 con Código de autorización y PKCE para autenticar sus solicitudes a este endpoint. 

[OAuth 1.0a User Context](/es/resources/fundamentals/authentication#oauth-1-0a-2), lo que significa que debe usar un conjunto de claves de API y tokens de acceso de usuario para realizar una solicitud correcta. Los tokens de acceso deben estar asociados con el usuario en cuyo nombre realiza la solicitud. Si desea generar un conjunto de tokens de acceso para otro usuario, ese usuario debe autorizar su app usando el [flujo OAuth de 3 patas](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow).

Tenga en cuenta que OAuth 1.0a puede ser difícil de usar. Si no está familiarizado con este método de autenticación, le recomendamos que use una [biblioteca](/es/x-api/tools-and-libraries/overview), una herramienta como Postman, o que utilice OAuth 2.0 o solo app para autenticar sus solicitudes.

[OAuth 2.0 Authorization Code with PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) permite un mayor control sobre el alcance (scope) de una aplicación y flujos de autorización en múltiples dispositivos. OAuth 2.0 le permite elegir scopes granulares que le otorgan permisos específicos en nombre de un usuario. 

Para habilitar OAuth 2.0 en su app, debe activarlo en la configuración de autenticación de su app, en la sección de configuración del portal para desarrolladores.

[App only](/es/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) solo requiere que envíe un [App only Access Token](/es/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) con su solicitud. Puede generar un App only Access Token directamente en una app de desarrollador o generarlo usando el endpoint [POST oauth2/token](/es/resources/fundamentals/authentication#post-oauth2-token).

<div id="developer-portal-projects-and-developer-apps">
  #### Portal para desarrolladores, Proyectos y Apps de desarrollador
</div>

Para obtener un conjunto de credenciales de autenticación que funcione con los endpoints de la API de X v2, debe [registrarse para obtener una cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info), configurar un [Proyecto](/es/resources/fundamentals/projects) dentro de esa cuenta y crear una [App de desarrollador](/es/resources/fundamentals/developer-apps) dentro de ese Proyecto. Luego podrá encontrar sus claves y tokens dentro de su App de desarrollador.
 

<div id="rate-limits">
  #### Límites de tasa
</div>

Cada día, muchos miles de desarrolladores realizan solicitudes a la API de X. Para ayudar a gestionar el gran volumen de estas solicitudes, se aplican [límites de tasa](https://developer.x.com/x-api/fundamentals/rate-limits) a cada endpoint que restringen la cantidad de solicitudes que puedes hacer en nombre de tu app o de un usuario autenticado. 

Este endpoint está sujeto a límites de tasa tanto a nivel de App como a nivel de usuario. El límite de la app significa que tú, como desarrollador, solo puedes hacer una cierta cantidad de solicitudes a este endpoint durante un período determinado desde cualquier App (mediante la API Key y la API Secret Key, o el Bearer Token). El límite de usuario significa que el usuario autenticado en cuyo nombre realizas la solicitud solo puede efectuar una consulta de publicaciones de lista un número determinado de veces, independientemente de la App de desarrollador.

Campos y expansiones

El endpoint GET de la API de X v2 permite a los usuarios seleccionar exactamente qué datos quieren obtener de la API usando un conjunto de herramientas llamadas `fields` y `expansions`. El parámetro `expansions` permite expandir objetos referenciados en el payload. Por ejemplo, consultar publicaciones de lista permite extraer las siguientes [expansions](/es/x-api/fundamentals/expansions):

- `author_id`

El parámetro `fields` permite seleccionar exactamente qué [fields](/es/x-api/fundamentals/fields) dentro de los diferentes objetos de datos deseas recibir. Este endpoint entrega principalmente objetos Post. De forma predeterminada, el objeto Post devuelve los campos `id` y `text`. Para recibir campos adicionales como `tweet.created_at` o `tweet.lang`, debes solicitarlos específicamente usando un parámetro fields. 

Hemos añadido una guía sobre cómo usar [fields y expansions](/es/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) juntos en nuestro [diccionario de datos de la API de X v2](/es/x-api/fundamentals/data-dictionary).

Paginación

Consultar publicaciones de lista puede devolver muchos datos. Para garantizar resultados consistentes y de alto rendimiento en todo momento, utilizamos paginación. La paginación es una característica de los endpoints de la API de X v2 que devuelve más resultados de los que pueden incluirse en una sola respuesta. Cuando eso sucede, los datos se devuelven en una serie de “páginas”. Obtén más información sobre cómo [paginar resultados.](/es/x-api/fundamentals/pagination)
