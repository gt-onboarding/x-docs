  ## Comparación de los endpoints de consulta de publicaciones de listas de la API de X
</div>

El endpoint de consulta de publicaciones de listas de v2 reemplazará el estándar v1.1 [GET lists/statuses](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-statuses). Si tienes código, apps o herramientas que usan esta versión del endpoint y estás considerando migrar al endpoint más reciente de la API de X v2, este conjunto de guías es para ti.

Las siguientes tablas comparan los endpoints estándar v1.1 y de la API de X v2 para Listas:

**Búsqueda de lista por ID**

|     |     |     |
| :--- | :--- | :--- |
| Descripción | v1.1 estándar | API de X v2 |
| Métodos HTTP compatibles | `GET` | `GET` |
| Dominio host | `https://api.x.com` | `https://api.x.com` |
| Ruta del endpoint | `/1.1/lists/statuses.json` | `/2/lists/:id/tweets` |
| [Autenticación](/es/resources/fundamentals/authentication) | Contexto de usuario de OAuth 1.0a<br /><br />Solo app | Contexto de usuario de OAuth 1.0a<br /><br />Código de autorización de OAuth 2.0 con PKCE<br /><br />Solo app |
| [Límites de frecuencia](/es/resources/fundamentals/rate-limits) predeterminados de la solicitud | 900 solicitudes por 15 min con OAuth 1.0a<br /><br />900 solicitudes por 15 min con solo app | 900 solicitudes por 15 min con OAuth 1.0a<br /><br />900 solicitudes por 15 min con OAuth 2.0<br /><br />900 solicitudes por 15 min con solo app |

Para acceder a los endpoints de la API de X v2, debes [registrarte para obtener una cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info). Al autenticarte, debes usar claves y tokens de una [App de desarrollador](/es/resources/fundamentals/developer-apps) que se encuentra dentro de un [Proyecto](/es/resources/fundamentals/projects).

Obtén más información sobre cómo obtener acceso a los endpoints de la API de X v2 en nuestra página de [introducción](/es/x-api/getting-started/getting-access).
