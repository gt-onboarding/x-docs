<div id="getting-started-with-the-pinned-list-endpoint-group">
  ## Introducción al grupo de endpoints de Listas fijadas
</div>

Este breve resumen le ayudará a realizar su primera solicitud a los endpoints de Listas fijadas usando [Postman](/es/tutorials/postman-getting-started).

Visite nuestro repositorio de GitHub de [código de ejemplo de la API de X v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code) si desea ver ejemplos de código en distintos lenguajes.

<Note>
  ### Requisitos previos

  Para completar esta guía, necesitará contar con un conjunto de [claves y tokens](/es/resources/fundamentals/authentication) para autenticar su solicitud. Puede generar estas claves y tokens siguiendo estos pasos:

  - [Regístrese para obtener una cuenta de desarrollador](https://developer.x.com/en/apply-for-access) y espere la aprobación.
  - Cree un [Proyecto](/es/resources/fundamentals/projects) y una [App de desarrollador](/es/resources/fundamentals/developer-apps) asociada en el portal para desarrolladores.
  - Vaya a la página “Keys and tokens” de su App para generar las credenciales necesarias. Asegúrese de guardar todas las credenciales en una ubicación segura.
</Note>

Cargue la colección de Postman de la API de X v2

Para cargar la colección de Postman de la API de X v2 en su espacio de trabajo, haga clic en el siguiente botón:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Agregar API de X v2 a Postman
</Button>

 

<div id="authenticate-your-request">
  #### Autentica tu solicitud
</div>

Para realizar correctamente una solicitud a los endpoints de **lookup**, puedes usar [OAuth 1.0a User Context](/es/resources/fundamentals/authentication), [OAuth 2.0 App-Only](/es/resources/fundamentals/authentication#oauth-2-0) o [OAuth 2.0 Authorization Code with PKCE](es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). Sin embargo, con los endpoints de **manage** solo puedes autenticarte con OAuth 1.0a User Context o OAuth 2.0 Authorization Code with PKCE.

En cualquier caso, cuando uses Postman, las claves y los tokens de autenticación predeterminados se completarán automáticamente en tus solicitudes, siempre que hayas configurado correctamente tu entorno.

Puedes hacerlo seleccionando el entorno llamado “API de X v2” (en la esquina superior derecha de Postman) y agregando tus claves y tokens en los campos “initial value” y “current value” (haciendo clic en el ícono del ojo junto al menú desplegable del entorno). Estas claves incluyen lo siguiente:

- API Key (también conocida como Consumer Key)
- API Secret Key (también conocida como Consumer Secret)
- OAuth 1.0a user Access Token
- OAuth 1.0a user Access Token Secret
- OAuth 2.0 App Access Token
- OAuth 2.0 Client Key (solo disponible si has configurado la autenticación de usuario OAuth 2.0 en la configuración de tu App)
- OAuth 2.0 Client Secret (solo disponible si has configurado la autenticación de usuario OAuth 2.0 en la configuración de tu App)
