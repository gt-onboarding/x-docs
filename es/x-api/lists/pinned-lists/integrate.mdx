<div id="authentication">
  #### Autenticación
</div>

Todos los endpoints de la API de X v2 requieren autenticar tus solicitudes con un conjunto de credenciales, también conocidas como claves y tokens. Puedes usar OAuth 1.0a en contexto de usuario (User Context) para autenticar tus solicitudes a este endpoint.

[OAuth 1.0a User Context](/es/resources/fundamentals/authentication), lo que significa que debes usar un conjunto de claves de API (API Keys) y tokens de acceso de usuario (Access Tokens) para realizar una solicitud correctamente. Los tokens de acceso deben estar asociados con el usuario en cuyo nombre realizas la solicitud. Si deseas generar un conjunto de tokens de acceso para otro usuario, ese usuario debe autorizar tu App mediante el [flujo OAuth de 3 factores (3-legged OAuth)](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens).

Ten en cuenta que OAuth 1.0a puede ser difícil de usar. Si no estás familiarizado con este método de autenticación, te recomendamos usar una [biblioteca](/es/x-api/tools-and-libraries/overview) o una herramienta como Postman.

<div id="developer-portal-projects-and-developer-apps">
  #### Portal para desarrolladores, Proyectos y Apps de desarrollador
</div>

Para obtener un conjunto de credenciales de autenticación que funcionen con los endpoints de la API de X v2, debes [registrarte para obtener una cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info), configurar un [Proyecto](/es/resources/fundamentals/projects) dentro de esa cuenta y crear una [App de desarrollador](/es/resources/fundamentals/developer-apps) dentro de ese Proyecto. Luego podrás encontrar tus claves y tokens dentro de tu App de desarrollador.
 

<div id="rate-limits">
  #### Límites de uso
</div>

Cada día, miles de desarrolladores realizan solicitudes a la API de X. Para ayudar a gestionar el gran volumen de estas solicitudes, se aplican [límites de uso](/es/x-api/fundamentals/rate-limits) a cada endpoint que restringen la cantidad de solicitudes que puedes realizar en nombre de tu app o de un usuario autenticado. 

Estos endpoints tienen límites de uso a nivel de usuario, lo que significa que el usuario autenticado en cuyo nombre haces la solicitud solo puede invocar el endpoint un número determinado de veces desde cualquier App de desarrollador. 

La siguiente tabla muestra los límites de uso para cada endpoint.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Método HTTP** | **Límite de uso** |
| /2/users/:id/pinned\_lists | POST | 50 solicitudes por 15 minutos |
| /2/users/:id/pinned\_lists/:list\_id | DELETE | 50 solicitudes por 15 minutos |
| /2/users/:id/pinned\_lists | GET | 15 solicitudes por 15 minutos |

<div id="fields-and-expansions">
  #### Campos y expansiones
</div>

El endpoint GET de la API de X v2 permite seleccionar exactamente qué datos quieres obtener de la API mediante un conjunto de parámetros llamados `fields` y `expansions`. El parámetro `expansions` permite expandir objetos referenciados en la carga útil. Por ejemplo, consultar listas fijadas permite obtener las siguientes [expansions](/es/x-api/fundamentals/expansions):

- `owner_id`

El parámetro `fields` permite seleccionar exactamente qué [fields](/es/x-api/fundamentals/fields) dentro de los distintos objetos de datos deseas recibir. Este endpoint entrega principalmente objetos de usuario. De forma predeterminada, el objeto List devuelve los campos `id` y `name`. Para recibir campos adicionales como `list.created_at` o `list.description`, debes solicitarlos explícitamente usando el parámetro fields.

Hemos añadido una guía sobre cómo usar [fields and expansions](/es/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) conjuntamente en nuestro [diccionario de datos de la API de X v2](/es/x-api/fundamentals/data-dictionary).
