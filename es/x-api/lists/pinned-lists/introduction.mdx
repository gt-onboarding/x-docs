Fijar listas permite a los usuarios de X acceder más fácilmente a las listas a las que se han suscrito o que han creado ellos mismos.

Dado que estás realizando solicitudes en nombre de un usuario con estos endpoints, debes autenticarlo con [OAuth 1.0a en contexto de usuario](/es/resources/fundamentals/authentication) o [OAuth 2.0 con código de autorización y PKCE](es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), y usar los tokens de acceso de usuario asociados con un usuario que haya autorizado tu App, los cuales pueden generarse mediante el [flujo OAuth de 3 patas](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) o el [flujo de concesión de código de autorización con PKCE](/es/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0).

<div id="pinned-list-lookup">
  ### Consulta de Listas fijadas
</div>

La consulta de Listas fijadas cuenta con un único endpoint que permite obtener las Listas fijadas de un usuario autenticado. Este endpoint tiene un límite de 15 solicitudes cada 15 minutos.
 

<div id="manage-pinned-lists">
  ### Administrar listas fijadas
</div>

Los endpoints para administrar listas fijadas te permiten fijar y desfijar una lista en nombre de un usuario autenticado. Para estos endpoints hay dos métodos disponibles: POST y DELETE. El método POST permite fijar una lista y el método DELETE permite desfijarla. Existe un límite por usuario de 50 solicitudes cada 15 minutos para ambos endpoints.

<Note>
  **Configuración de la cuenta**

  Para acceder a estos endpoints, necesitarás:

  - Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info)) aprobada.
  - Autenticación con las claves y los tokens de una [App de desarrollador](/es/resources/fundamentals/developer-apps) ubicada dentro de un [Proyecto](/es/resources/fundamentals/projects).

  Obtén más información sobre cómo acceder a los endpoints de la API de X v2 en nuestra [guía de introducción](/es/x-api/getting-started/getting-access).
</Note>
