<div id="authentication">
  #### Autenticación
</div>

Todos los endpoints de la API de X v2 requieren que autentiques tus solicitudes con un conjunto de credenciales, también conocidas como claves y tokens. Puedes usar OAuth 1.0a User Context, App only o OAuth 2.0 Authorization Code con PKCE para autenticar tus solicitudes a estos endpoints.

[OAuth 1.0a User Context](/es/resources/fundamentals/authentication#oauth-1-0a-user-context) implica que debes usar un conjunto de API Keys y Access Tokens de usuario para realizar una solicitud correcta. Los access tokens deben estar asociados con el usuario en cuyo nombre realizas la solicitud. Si deseas generar un conjunto de Access Tokens para otro usuario, ese usuario debe autorizar tu App usando el [flujo de OAuth de 3 patas](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow).

Ten en cuenta que OAuth 1.0a puede ser difícil de usar. Si no estás familiarizado con este método de autenticación, te recomendamos que uses una [biblioteca](/es/x-api/tools-and-libraries/overview), una herramienta como Postman, o que uses OAuth 2.0 para autenticar tus solicitudes.

[OAuth 2.0 Authorization Code con PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) permite un mayor control sobre el alcance (scope) de una aplicación y flujos de autorización a través de múltiples dispositivos. OAuth 2.0 te permite elegir scopes específicos y granulares que te otorgan permisos concretos en nombre de un usuario.

Para habilitar OAuth 2.0 en tu App, debes activarlo en la configuración de autenticación de tu App, ubicada en la sección de configuración de la App en el portal para desarrolladores.

[App only](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) solo requiere que envíes un [App only Access Token](/es/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) con tu solicitud. Puedes generar un token App only directamente dentro de una App de desarrollador, o generarlo usando el endpoint [POST oauth2/token](/es/resources/fundamentals/authentication#post-oauth2-token).

<div id="developer-portal-projects-and-developer-apps">
  #### Portal para desarrolladores, Proyectos y Apps de desarrollador
</div>

Para obtener un conjunto de credenciales de autenticación que funcione con los endpoints de la API de X v2, debes [registrarte para obtener una cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info), configurar un [Proyecto](/es/resources/fundamentals/projects) dentro de esa cuenta y crear una [App de desarrollador](/es/resources/fundamentals/developer-apps) dentro de ese Proyecto. Luego podrás encontrar tus claves y tokens dentro de tu App de desarrollador.
 

<div id="rate-limits">
  #### Límites de frecuencia
</div>

Cada día, muchos miles de desarrolladores realizan solicitudes a la API de X. Para ayudar a gestionar el gran volumen de estas solicitudes, se aplican [límites de frecuencia](/es/x-api/fundamentals/rate-limits) a cada endpoint que restringen la cantidad de solicitudes que puedes realizar en nombre de tu app o de un usuario autenticado.

Estos endpoints tienen límites de frecuencia tanto a nivel de app como a nivel de usuario. El límite de app significa que tú, como desarrollador, solo puedes realizar cierta cantidad de solicitudes a este endpoint durante un periodo determinado desde cualquier app (usando la API Key y la API Secret Key, o el App-only Access Token). El límite de usuario significa que el usuario autenticado en cuyo nombre realizas la solicitud solo puede efectuar búsquedas de Listas un número determinado de veces a través de cualquier app de desarrollador.

La siguiente tabla muestra los límites de frecuencia para cada endpoint.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Método HTTP** | **Límite de frecuencia** |
| /2/lists/:id | GET | 75 solicitudes cada 15 minutos |
| /2/users/:id/owned\_lists | GET | 15 solicitudes cada 15 minutos |

Campos y expansiones

El endpoint GET de la API de X v2 permite seleccionar exactamente qué datos devolver usando un conjunto de herramientas llamadas `fields` y `expansions`. El parámetro `expansions` permite expandir objetos referenciados en el payload. Por ejemplo, consultar una Lista por ID te permite solicitar las siguientes [expansions](/es/x-api/fundamentals/expansions):

- `owner_id`

El parámetro `fields` te permite seleccionar exactamente qué [fields](/es/x-api/fundamentals/fields) dentro de los distintos objetos de datos deseas recibir. Estos endpoints entregan principalmente objetos de Lista. De forma predeterminada, el objeto Lista devuelve los campos `id` y `name`. Para recibir campos adicionales como `list.created_at` o `list.follower_count`, tendrás que solicitarlos explícitamente usando el parámetro list.fields.

Hemos añadido una guía sobre cómo usar [fields y expansions](/es/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) juntos en nuestro [diccionario de datos de la API de X v2](/es/x-api/fundamentals/data-dictionary).

La siguiente tabla muestra los fields y expansions disponibles para este grupo de endpoints:

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Fields** | **Expansions** |
| /2/lists/:id | `list.fields`<br /><br />`user.fields` | `owner_id` |
| /2/users/:id/owned\_lists | `list.fields`<br /><br />`user.fields` | `owner_id` |

<div id="pagination">
  #### Paginación
</div>

Consultar las Listas propiedad de un usuario puede devolver muchos datos. Para garantizar resultados consistentes y de alto rendimiento en todo momento, utilizamos paginación. La paginación es una función de los endpoints de la API de X v2 que devuelve más resultados de los que pueden incluirse en una sola respuesta. Cuando eso sucede, los datos se devuelven en una serie de “páginas”. Obtén más información sobre cómo [paginar los resultados.](/es/x-api/fundamentals/pagination)
