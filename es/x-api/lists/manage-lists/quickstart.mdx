<div id="getting-started-with-the-manage-lists-endpoint-group">
  ## Introducción al grupo de endpoints para gestionar Listas
</div>

Este breve resumen le ayudará a realizar su primera solicitud a los endpoints para gestionar Listas usando [Postman](/es/tutorials/postman-getting-started).

Si desea ver código de ejemplo en distintos lenguajes, visite nuestro repositorio de GitHub de [código de ejemplo de la API de X v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code).

**Nota:** Para este ejemplo, haremos una solicitud al endpoint _Create a List_, pero también puede aplicar lo aprendido en este inicio rápido a otras solicitudes de gestión.

<Note>
  **Requisitos previos**

  Para completar esta guía, necesitará un conjunto de [keys and tokens](/es/resources/fundamentals/authentication) para autenticar su solicitud. Puede generar estas keys and tokens siguiendo estos pasos:

  - [Regístrese para obtener una cuenta de desarrollador](https://developer.x.com/en/apply-for-access) y reciba la aprobación.
  - Cree un [Project](/es/resources/fundamentals/projects) y una [developer App](/es/resources/fundamentals/developer-apps) asociada en el portal para desarrolladores.
  - Vaya a la página “Keys and tokens” de su App para generar las credenciales requeridas. Asegúrese de guardar todas las credenciales en una ubicación segura.
</Note>

<div id="steps-to-build-a-manage-list-request">
  ### Pasos para crear una solicitud para administrar listas
</div>

**Paso uno: comience con una herramienta o biblioteca**

Existen varias herramientas, ejemplos de código y bibliotecas que puede usar para realizar una solicitud a este endpoint, pero aquí usaremos Postman para simplificar el proceso.

Para cargar en su entorno la colección de Postman de la API de X v2, haga clic en el siguiente botón:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Agregar API de X v2 a Postman
</Button>

Una vez que tenga cargada en Postman la colección de la API de X v2, navegue a la carpeta “List”, seleccione la subcarpeta “Manage List” y luego elija "Create a List".

**Paso dos: autentique su solicitud**

Para realizar correctamente una solicitud a la API de X, debe verificar que tiene permiso para hacerlo. Para los endpoints de administración de publicaciones, debe autenticar su solicitud usando [OAuth 1.0a User Context](/es/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code with PKCE](es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

En este ejemplo, usaremos OAuth 1.0a User Context.

Debe agregar sus claves y tokens (específicamente su API Key, API Secret Key, OAuth 1.0a User Access Token y OAuth 1.0a User Access Token Secret) a Postman. Puede hacerlo seleccionando el entorno llamado “X API v2” (en la esquina superior derecha de Postman) y agregando sus claves y tokens en los campos "initial value" y "current value" (haciendo clic en el ícono de ojo junto al menú desplegable del entorno).

Si lo ha hecho correctamente, estas variables se completarán automáticamente en la pestaña de autorización de la solicitud.
 
**Paso tres: especifique el nombre de la nueva lista**

Al crear una nueva lista con este endpoint, el nombre de la lista es un parámetro de cuerpo obligatorio. Opcionalmente, puede proporcionar una descripción y especificar si la lista es privada.

En Postman, vaya a la pestaña “Body” e ingrese el nombre de la lista como valor del parámetro name. Además, si desea agregar una descripción para la lista, simplemente agregue una nueva clave llamada description del mismo modo que name, seguida de la descripción de la lista como valor. Hacer que una lista sea privada sigue el mismo patrón, pero solo se aceptan los valores true o false para este parámetro. 

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Parameter type** |
| `name` | Nombre de la lista (obligatorio) | body |
| description | Descripción de la lista (opcional) | body |
| private | true o false (opcional) | body |

Ahora debería ver una URL similar junto al botón "Send":

`https://api.x.com/2/lists`

Paso cuatro: realice su solicitud y revise la respuesta

Una vez que tenga todo configurado, haga clic en el botón "Send" y recibirá una respuesta similar al siguiente ejemplo:

```
{
  "data": {
    "id": "1441162269824405510",
    "name": "New list created from Postman"
  }
}
```

Si el objeto de respuesta contiene un id y el nombre de su lista, habrá creado la lista correctamente. 

Para eliminar una lista, seleccione la solicitud “Delete a List”, que también se encuentra en la carpeta “Lists” de la colección de la API de X v2 cargada en Postman. Este endpoint requiere el ID de la lista que desea eliminar. En la pestaña “Params”, ingrese el ID de la lista que desea eliminar como valor de la columna id. 

Si la solicitud de eliminación se realiza correctamente, recibirá una respuesta similar al siguiente ejemplo: 

```
{
  "data": {
    "deleted": true
  }
}
```
