[Las Listas de X](https://help.x.com/en/using-twitter/twitter-lists) permiten a los usuarios personalizar, organizar y priorizar las publicaciones que ven en su cronología. Con los endpoints de Listas, puedes crear soluciones que permitan a las personas curar y organizar publicaciones según sus preferencias, intereses, grupos o temas.

Dado que estás realizando solicitudes en nombre de un usuario con los endpoints de administración de Listas, debes autenticar estos endpoints con [OAuth 1.0a en contexto de usuario](/es/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code con PKCE](es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), y usar los tokens de acceso de usuario asociados a un usuario que haya autorizado tu App, los cuales pueden generarse usando el [flujo OAuth de 3 patas](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) o el [flujo de concesión Authorization Code con PKCE](/es/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0).

<div id="manage-lists">
  ### Administrar listas
</div>

Los endpoints para administrar listas permiten crear, eliminar y actualizar listas en nombre de un usuario autenticado. Estos endpoints admiten tres métodos: POST, DELETE y PUT. El método POST permite crear una lista, DELETE permite eliminarla y PUT permite actualizar sus metadatos. El límite es de 300 solicitudes por usuario cada 15 minutos para los tres endpoints.

Ten en cuenta que puedes crear hasta 1000 listas por cuenta.

**Configuración de la cuenta**

Para acceder a estos endpoints, necesitarás:

- Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
- Autenticación con las claves y los tokens de una [App de desarrollador](/es/resources/fundamentals/developer-apps) que esté dentro de un [Proyecto](/es/resources/fundamentals/projects).

Obtén más información sobre cómo acceder a los endpoints de la API de X v2 en nuestra [guía de primeros pasos](/es/x-api/getting-started/getting-access).
