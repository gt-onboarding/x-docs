<div id="manage-lists-standard-v11-compared-to-x-api-v2">
  ### Administrar listas: estándar v1.1 frente a la API de X v2
</div>

Si ha estado trabajando con los endpoints estándar v1.1 [POST lists/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-create), [POST lists/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-destroy) y [POST lists/update](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-update), el objetivo de esta guía es ayudarle a comprender las similitudes y diferencias entre los endpoints para administrar listas del estándar v1.1 y de la API de X v2.

- **Similitudes**
  - Autenticación
- **Diferencias**
  - URL de endpoints
  - Requisitos de App y Proyecto
  - Métodos HTTP
  - Límites de solicitud
  - Parámetros de solicitud

<div id="similarities">
  #### Similitudes
</div>

**Autenticación**

Ambas versiones del endpoint admiten [OAuth 1.0a User Context](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Por lo tanto, si antes utilizabas uno de los endpoints estándar de v1.1 para gestionar listas, puedes seguir usando el mismo método de autenticación si migras a la versión de la API de X v2.

<div id="differences">
  #### Diferencias
</div>

**URLs de endpoints**

- Endpoints estándar v1.1:
  - POST https://api.x.com/1.1/lists/create.json
    (Crea una lista)
  - POST https://api.x.com/1.1/lists/destroy.json
    (Elimina una lista)
  - POST https://api.x.com/1.1/lists/update.json
    (Actualiza una lista)
- Endpoint de la API de X v2:
  - POST https://api.x.com/2/lists
    (Crea una lista)

  - DELETE https://api.x.com/2/lists/:id
    (Elimina una lista)

  - PUT https://api.x.com/2/lists/:id
    (Actualiza una lista)

**Límites de frecuencia**

| **Estándar v1.1** | **API de X v2** |
| :--- | :--- |
| /1.1/lists/create.json<br /><br />ninguno | /2/lists<br /><br />300 solicitudes por ventana de 15 minutos con OAuth 1.0a en contexto de usuario |
| /1.1/lists/destroy.json<br /><br />ninguno | /2/lists/:id<br /><br />300 solicitudes por ventana de 15 minutos con OAuth 1.0a en contexto de usuario |
| /1.1/lists/update.json<br /><br />ninguno | /2/lists/:id<br /><br />300 solicitudes por ventana de 15 minutos con OAuth 1.0a en contexto de usuario |

**Requisitos de App y Proyecto**

Los endpoints de la API de X v2 requieren que uses credenciales de una [App de desarrollador](/es/resources/fundamentals/developer-apps) asociada a un [Proyecto](/es/resources/fundamentals/projects) al autenticar tus solicitudes. Todos los endpoints de la API de X v1.1 pueden usar credenciales de Apps independientes o Apps relacionadas con un proyecto.

**Parámetros de la solicitud**

Los siguientes parámetros de solicitud estándar v1.1 tienen equivalentes en la API de X v2:

**Crear una lista**

| **Estándar** | **API de X v2** |
| :--- | :--- |
| name | name |
| mode | private |
| description | description |

**Eliminar/Actualizar una lista**

| **Estándar** | **API de X v2** |
| :--- | :--- |
| owner\_screen\_name | Sin equivalente |
| owner\_id | Sin equivalente |
| list\_id | id  |
| slug | Sin equivalente |

<Note>
  **Ten en cuenta:** los parámetros estándar v1.1 se pasan como parámetros de consulta, mientras que los de la API de X v2 se pasan como parámetros del cuerpo (para el endpoint POST) o como parámetros de ruta (para los endpoints DELETE y PUT).
</Note>
