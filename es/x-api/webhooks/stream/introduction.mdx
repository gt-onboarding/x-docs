<div id="overview">
  ## Descripción general
</div>

El grupo de endpoints de flujo filtrado permite a los desarrolladores filtrar un flujo de Publicaciones públicas. La funcionalidad de este grupo de endpoints incluye varios endpoints que permiten crear y administrar reglas, y aplicarlas para filtrar un flujo de Publicaciones que devolverá las Publicaciones públicas coincidentes. Este grupo de endpoints permite escuchar temas y eventos específicos, monitorear la conversación en torno a competiciones, comprender cómo se desarrollan las tendencias y mucho más.

Los desarrolladores pueden usar el [endpoint de reglas](/es/x-api/posts/filtered-stream#post-2-tweets-search-stream-rules) de REST para agregar y eliminar reglas a fin de filtrar Publicaciones que coincidan con un criterio definido. Estas [reglas](/es/x-api/posts/filtered-stream#building-rules-for-filtered-stream) se pueden crear con operadores que hacen coincidir atributos de la Publicación, como palabras clave del mensaje, hashtags y URL. Los operadores y las cláusulas de reglas se pueden combinar con lógica booleana y paréntesis para refinar el comportamiento de coincidencia del filtro.

Una vez que hayas agregado un conjunto de reglas, puedes registrar tu webhook, donde X comenzará a entregar [objetos de Publicación](/es/x-api/fundamentals/data-dictionary#tweet) en formato JSON. Solo recibirás en tu webhook contenido que coincida con tus reglas.

Este endpoint admite Publicaciones editadas. Tu webhook recibirá Publicaciones editadas que coincidan con uno o más de tus filtros, junto con su historial de ediciones, incluido un arreglo de ID de Publicación. Para Publicaciones sin historial de edición, este arreglo contendrá un único ID. Para Publicaciones que han sido editadas, este arreglo contiene múltiples ID, ordenados de forma ascendente según el orden de las ediciones, con la versión más reciente en la última posición del arreglo. Para obtener más información sobre cómo funcionan las ediciones de Publicaciones, consulta la página [Fundamentos de ediciones de Publicaciones](/es/x-api/fundamentals/edit-posts).

**Nota:** Este grupo de endpoints es similar al [endpoint de flujo filtrado de la v2](/es/x-api/posts/filtered-stream/introduction) para configurar tus reglas. La única diferencia son los mecanismos de entrega de las Publicaciones que coinciden con tus reglas.

- En el caso del endpoint de flujo filtrado de la v2, debes establecer una conexión persistente y escuchar las Publicaciones que coincidan con tus reglas.
- En el caso de este endpoint de webhook de flujo filtrado, registras tu webhook y X entrega a tu webhook las Publicaciones que coincidan con tus reglas.

Esto elimina la necesidad de manejar desconexiones derivadas de caídas del cliente o del lado del servidor por reinicios normales de nuestro servicio. Dado que probablemente entregaremos cientos de eventos por segundo, es necesario que tu servidor de webhook pueda manejar estos intentos de conexión simultáneos.

Este endpoint está disponible actualmente para desarrolladores Enterprise y admite la siguiente configuración:

- Más de 25,000 reglas por proyecto
- Puedes usar todos los operadores al crear tu regla
- Puedes crear reglas de hasta 2048 caracteres
- Solicita acceso para Enterprise [aquí](https://developer.x.com/en/products/x-api/enterprise/enterprise-api-interest-form)

Las Publicaciones devueltas por el flujo filtrado cuentan para el [límite de publicaciones](/es/x-api/fundamentals/post-cap) mensual.

  - Autenticarte con las claves y tokens de una [App de desarrollador](/es/resources/fundamentals/developer-apps) que se encuentre dentro de un [Proyecto](/es/resources/fundamentals/projects).
