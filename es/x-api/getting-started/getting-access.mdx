<div id="step-one-sign-up-for-a-developer-account">
  ### Paso uno: Regístrate para obtener una cuenta de desarrollador
</div>

Registrarte para obtener una cuenta de desarrollador es rápido y sencillo. Solo haz clic en el botón de abajo, responde algunas preguntas y podrás empezar a explorar y crear con la API de X v2 usando el [Acceso básico](https://developer.x.com/en/portal/products/basic).

A continuación, crearás un Proyecto y una App de desarrollador asociada durante el proceso de incorporación, lo que te proporcionará un conjunto de credenciales que usarás para autenticar todas las solicitudes a la API.

[Regístrate](https://developer.x.com/en/portal/products/basic)

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### Paso dos: Guarda la clave y los tokens de tu App y mantenlos seguros
</div>

Una vez que tengas acceso y hayas creado un Proyecto y una App, podrás encontrar o generar las siguientes credenciales dentro de tu App de desarrollador:

- **[API Key y Secret](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret):** Básicamente, el nombre de usuario y la contraseña de tu App. Las usarás para autenticar solicitudes que requieren [OAuth 1.0a User Context](/es/resources/fundamentals/authentication), o para generar otros tokens como el Access Token de usuario o el App Access Token.

- **[Access Token y Secret](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens):** En general, los Access Tokens representan al usuario en cuyo nombre realizas la solicitud. Los que puedes generar a través del portal de desarrolladores representan al propietario de la App. Los usarás para autenticar solicitudes que requieren [OAuth 1.0a User Context](/es/resources/fundamentals/authentication). Si deseas realizar solicitudes en nombre de otro usuario, deberás usar el flujo OAuth de 3 patas para que te autorice.

- **[Client ID y Client Secret](/es/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce):** Estas credenciales se usan para obtener un Access Token de usuario con autenticación OAuth 2.0. De forma similar a OAuth 1.0a, los Access Tokens de usuario se usan para autenticar solicitudes que proporcionan información privada de una cuenta de usuario o realizan acciones en nombre de otra cuenta, pero con un alcance de permisos más granular para tener mayor control sobre a qué puede acceder la aplicación cliente en el usuario.

- **[App only Access Token](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only):** Usarás este token al realizar solicitudes a endpoints que responden con información disponible públicamente en X.

Dado que estas claves y tokens no caducan a menos que se regeneren, te sugerimos guardarlos en un lugar seguro, como un gestor de contraseñas, en cuanto recibas tus credenciales.

> **Ten en cuenta: tus claves y tokens solo se mostrarán una vez en el [portal de desarrolladores](/es/resources/fundamentals/developer-portal), por lo que es importante que almacenes estas credenciales en tu gestor de contraseñas tan pronto como las generes.**
>
> Si extravías u olvidas las claves y los tokens, deberás regenerarlos, lo que crea credenciales nuevas e invalida las anteriores. Esto significa que tendrás que actualizar cualquier integración que hayas configurado con tus credenciales previas.
>
> Obtén más información sobre nuestras [mejores prácticas de autenticación](/es/resources/fundamentals/authentication#authentication-best-practices).
