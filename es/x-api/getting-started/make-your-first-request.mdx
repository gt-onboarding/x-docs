Esta guía le mostrará algunos pasos que puede seguir para realizar su primera solicitud. Es un excelente recurso para ayudarle una vez que haya creado una cuenta de X.

Si le interesa usar ejemplos de código, guías más técnicas o una herramienta gráfica como Postman, considere usar las siguientes guías para realizar su primera solicitud:

- [Guía de inicio rápido para realizar su primera solicitud al endpoint de publicaciones](/es/x-api/posts/manage-tweets/quickstart)
- [Primeros pasos con Postman](/es/tutorials/postman-getting-started) - ideal para principiantes
- [Ejemplos de código de la API de X v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code)
- [Referencia de la API de consulta de publicaciones](/es/x-api/posts/lookup/introduction)

Esta guía asume que ya ha obtenido su [clave y secreto de API](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret), [token de acceso de usuario y secreto](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens), [token de acceso de la app](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) y que los ha almacenado en un lugar seguro. Puede aprender cómo hacerlo siguiendo los pasos de la guía [Cómo obtener acceso a la API de X](/es/x-api/getting-started/getting-access).

<div id="step-1-identify-which-endpoint-you-would-like-to-use">
  ### Paso 1. Identifique qué endpoint desea usar
</div>

La API de X le permite realizar, mediante código, una variedad de acciones que también podría realizar en el sitio web o en la aplicación móvil de X.

Contamos con una lista completa de los endpoints disponibles a través de la API en nuestro [Índice de referencia de la API](/es/x-api/introduction), pero recomendamos ceñirse a uno de los siguientes por simplicidad:

- [Administrar publicaciones > Publicar un tuit](/es/x-api/posts/manage-tweets/introduction)
- [Buscar publicaciones](/es/x-api/posts/search/introduction)
- [Búsqueda de usuarios](/es/x-api/users/lookup/introduction)

<div id="step-2-choose-a-tool-to-make-your-request">
  ### Paso 2. Elige una herramienta para realizar tu solicitud
</div>

Si bien algunas solicitudes pueden ser sencillas, otras pueden resultar más complejas de implementar. Por ello, la excelente comunidad de desarrolladores ha creado herramientas que abstraen parte de esa complejidad.

A continuación, se presentan algunas herramientas recomendadas y detalles sobre cómo utilizarlas:

<div id="postman">
  #### Postman
</div>

Postman es una herramienta visual que puedes usar para enviar solicitudes a endpoints REST. Hemos creado recursos muy útiles sobre Postman para ayudarte a empezar y a explorar los diferentes endpoints disponibles a través de la API de X.

Te recomendamos leer nuestro tutorial ["Introducción a Postman"](/es/tutorials/postman-getting-started) para aprender a agregar tus claves y tokens y enviar tu primera solicitud.

También hemos preparado una guía de inicio rápido para cada uno de nuestros endpoints de la API de X v2, la mayoría de los cuales usan Postman. Puedes encontrar estas guías en la sección del endpoint correspondiente, pero aquí tienes enlaces a algunas:

- Inicio rápido: Publicar un tuit
- Inicio rápido: Buscar publicaciones
- Inicio rápido: Consultar un usuario

Ten en cuenta que no puedes enviar solicitudes a endpoints de streaming usando Postman. Visita la guía de inicio rápido de [flujo filtrado](/es/x-api/posts/filtered-stream/quickstart) o [flujo muestreado del 1%](/es/x-api/posts/sample-stream) para aprender a trabajar con esos endpoints.

Si prefieres una herramienta gráfica más sencilla, también puedes considerar usar [Insomnia](https://insomnia.rest).

<div id="sample-code">
  #### Código de ejemplo
</div>

Si le interesa usar código sencillo para realizar su solicitud, hemos preparado ejemplos en varios lenguajes para cada uno de nuestros endpoints de la API de X v2.

Puede encontrar los ejemplos de código en nuestro repositorio de GitHub, [X-API-v2-sample-code](https://github.com/xdevplatform/Twitter-API-v2-sample-code), que también contiene un archivo README que puede usar para aprender a configurar sus credenciales y que funcionen correctamente con las solicitudes.

Por ejemplo, aquí tiene un ejemplo de cURL para el endpoint de búsqueda de usuarios. Todo lo que debe hacer para usar esta solicitud es reemplazar $ACCESS\_TOKEN y $USERNAME por su [App Access Token](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) y su handle de X. Luego, copie y pegue este código en su herramienta de línea de comandos y presione “Return” o “Enter”.

```bash
curl "https://api.x.com/2/users/by/username/$USERNAME" -H "Authorization: Bearer $ACCESS_TOKEN"
```

<div id="libraries">
  #### Bibliotecas
</div>

La comunidad de desarrolladores de X también ha creado bibliotecas en diversos lenguajes de programación que pueden utilizarse para realizar solicitudes a la API de X.

Hemos creado una página de ["Herramientas y bibliotecas"](/es/x-api/tools-and-libraries/overview) que enumera todas las bibliotecas de la comunidad que conocemos. Cada biblioteca debería incluir un archivo README con instrucciones para configurar el repositorio en tu equipo y realizar tu primera solicitud.

> **Ten en cuenta:** Si encuentras algún problema, consulta la documentación para desarrolladores del endpoint al que estás enviando la solicitud, nuestra [sección de soporte](https://developer.x.com/en/support) o ponte en contacto con la comunidad en nuestros foros para obtener ayuda. ¡Te ayudaremos a completar una solicitud con éxito!

<div id="step-3-review-the-response">
  ### Paso 3. Revisa la respuesta
</div>

Una vez que hayas realizado una solicitud correctamente, recibirás un payload con metadatos relacionados con la solicitud.

Si usaste un endpoint que emplea el método HTTP GET, recibirás metadatos del recurso (publicación, usuario, lista, Space, etc.) al que hiciste la solicitud, en formato JSON. Revisa los distintos campos devueltos y verifica si puedes mapear la información solicitada con el contenido en X.

Si usaste un endpoint que emplea un método HTTP POST, PUT o DELETE, realizaste una acción en X. Ve a X.com o a la app móvil y comprueba si puedes encontrar esa acción.

<div id="step-4-adjust-your-request-using-parameters">
  ### Paso 4. Ajusta tu solicitud con parámetros
</div>

Cada endpoint tiene un conjunto distinto de parámetros que puedes usar para modificar tu solicitud. Por ejemplo, puedes solicitar campos de metadatos adicionales cuando uses endpoints GET con los parámetros fields y expansions. También puedes probar distintas herramientas de filtrado con endpoints como [search Posts](/es/x-api/posts/search/introduction), [Post counts](/es/x-api/posts/counts/introduction) y [filtered stream](/es/x-api/posts/filtered-stream/introduction) para acotar los datos que recibes únicamente a las Publicaciones que te interesan.

Puedes encontrar la lista completa de parámetros y campos de la solicitud en la Referencia de la API del endpoint con el que estés trabajando, además de otra información útil para la integración en nuestras guías de integración y páginas de fundamentos.

Puedes obtener más información sobre todos los materiales educativos que hemos puesto a tu disposición en nuestra guía de [Important resources](/es/x-api/getting-started/important-resources).
