<div id="manage-follows-standard-v11-compared-to-x-api-v2">
  #### Administrar seguimientos: estándar v1.1 frente a la API de X v2
</div>

Si ha estado trabajando con los endpoints estándar v1.1 [POST friendships/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-create) y [POST friendships/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-destroy), el objetivo de esta guía es ayudarle a comprender las similitudes y diferencias entre los endpoints de administración de seguimientos del estándar y de la API de X v2.

- **Similitudes**
  - Contexto de usuario OAuth 1.0a
- **Diferencias**
  - URL de los endpoints
  - Requisitos de Apps y Proyectos
  - Métodos HTTP
  - Parámetros de la solicitud

     

<div id="similarities">
  #### Similitudes
</div>

**Método de autenticación OAuth 1.0a User Context**

Ambas versiones del endpoint admiten [OAuth 1.0a User Context](/es/resources/fundamentals/authentication#oauth-1-0a-2). Por lo tanto, si antes usabas uno de los endpoints estándar de v1.1 para administrar seguimientos, puedes seguir usando el mismo método de autenticación si migras a la versión de la API de X v2. 

<div id="differences">
  #### Diferencias
</div>

**URLs de endpoints**

- Endpoints estándar v1.1:
  - POST https://api.x.com/1.1/friendships/create.json
    (seguir a un usuario)
  - POST https://api.x.com/1.1/friendships/destroy.json
    (dejar de seguir a un usuario)
- Endpoint de la API de X v2:
  - POST https://api.x.com/2/users/:id/following
    (seguir a un usuario)
  - DELETE https://api.x.com/2/users/:source\_user\_id/following/:target\_user\_id
    (dejar de seguir a un usuario)

**Requisitos de App y Proyecto**

Los endpoints de la API de X v2 requieren que uses credenciales de una [developer App](/es/resources/fundamentals/developer-apps) asociada a un [Project](/es/resources/fundamentals/projects) al autenticar tus solicitudes. Todos los endpoints de la API de X v1.1 pueden usar credenciales de Apps independientes o Apps asociadas a un proyecto.

**Parámetros de solicitud**

Los siguientes parámetros de solicitud estándar v1.1 tienen equivalentes en la API de X v2:

| Estándar v1.1 | API de X v2 |
| :--- | :--- |
| Sin equivalente | id (POST), source\_user\_id (DELETE) |
| user\_id | target\_user\_id |
| screen\_name | Sin equivalente |

Ten en cuenta que los parámetros de la estándar v1.1 se pasan como parámetros de consulta, mientras que los de la API de X v2 se pasan como parámetros del cuerpo (para el endpoint POST) o parámetros de ruta (para el endpoint DELETE).

Además, en v2 id y source\_user\_id no son obligatorios cuando se usan los endpoints estándar v1.1, ya que los Access Tokens enviados con OAuth 1.0a User Context inferían qué usuario iniciaba la acción de seguir/dejar de seguir.
