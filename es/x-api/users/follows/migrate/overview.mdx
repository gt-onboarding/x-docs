<div id="comparing-x-apis-follows-endpoints">
  ## Comparación de los endpoints de seguimientos de la API de X
</div>

<div id="follows-lookup">
  ### Consulta de seguimientos
</div>

Los endpoints de consulta de seguimientos de v2 reemplazarán los endpoints estándar v1.1 [followers/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-followers-ids), v1.1 [followers/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-followers-list), v1.1 [friends/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-friends-ids) y v1.1 [friends/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-friends-list).

Las siguientes tablas comparan los distintos tipos de endpoints de consulta de seguimientos:

|     |     |     |
| :--- | :--- | :--- |
| **Descripción** | **Standard v1.1** | **API de X v2** |
| Métodos HTTP compatibles | `GET` | `GET` |
| Dominio del host | `https://api.x.com` | `https://api.x.com` |
| Ruta del endpoint | /1.1/friends/ids.json<br /><br />/1.1/friends/list.json<br /><br />/1.1/followers/ids.json<br /><br />/1.1/followers/list.json | /2/users/:id/following<br /><br />/2/users/:id/followers |
| [Autenticación](/es/resources/fundamentals/authentication) | OAuth 1.0a User Context<br /><br />Solo app | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code with PKCE<br /><br />Solo app |
| [Límites de frecuencia](/es/resources/fundamentals/rate-limits) de solicitud predeterminados | 15 solicitudes por 15 min (por usuario)<br /><br />15 solicitudes por 15 min (por app) | 15 solicitudes por 15 min (por usuario)<br /><br />15 solicitudes por 15 min (por app) |
| Máximo de usuarios por respuesta | GET friends/ids y GET followers/ids devuelven un máximo de 5000 IDs de usuario por página.<br /><br />  <br /><br />GET friends/list y GET followers/list devuelven un máximo de 200 objetos de usuario por página. | 1000 objetos de usuario por página |
| Paginación | El token se devuelve en el campo next\_cursor, que luego puede pasarse como valor al parámetro cursor para obtener la siguiente página de resultados. | El token se devuelve en el campo next\_token, que luego puede pasarse como valor al parámetro token para obtener la siguiente página de resultados.<br /><br />La carga útil de v2 también incluye un campo previous\_token, que puede pasarse con el parámetro pagination\_token para obtener la página anterior de resultados. |
| Formato JSON | Formato Standard v1.1 | [Formato de la API de X v2](/es/x-api/fundamentals/data-dictionary) (determinado por los parámetros de solicitud fields y expansions; no es compatible con los formatos de v1.1)<br /><br />Para obtener más información sobre cómo migrar del formato Standard v1.1 al formato de la API de X v2, visita nuestra [guía de migración de formatos de datos](/es/x-api/migrate/data-format-migration). |
| Permite seleccionar qué [fields](/es/x-api/fundamentals/data-dictionary) se devuelven en la carga útil |     | ✔   |
| Permite los campos de [anotaciones de publicaciones](/es/x-api/fundamentals/post-annotations) |     | ✔   |
| Permite solicitar nuevos campos de [métricas](/es/x-api/fundamentals/metrics) |     | ✔   |
| Permite el campo [conversation\_id](/es/x-api/fundamentals/conversation-id) |     | ✔   |
| Requiere el uso de credenciales de una [App de desarrollador](/es/resources/fundamentals/developer-apps) asociada con un [proyecto](/es/resources/fundamentals/projects) |     | ✔   |

<div id="manage-follows">
  #### Gestionar seguimientos
</div>

Los endpoints de gestión de seguimientos de la v2 sustituirán a los endpoints estándar de la v1.1 [POST friendships/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-create) y [POST friendships/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-destroy).

Las siguientes tablas comparan los endpoints estándar de la v1.1 y de la API de X v2 para crear seguimientos:

<div id="follow-a-user">
  #### Seguir a un usuario
</div>

| Descripción | Estándar v1.1 | API de X v2 |
| :--- | :--- | :--- |
| Métodos HTTP admitidos | POST | POST |
| Dominio del host | https://api.x.com | https://api.x.com |
| Ruta del endpoint | /1.1/friendships/create.json | /2/users/:id/following |
| [Autenticación](/es/resources/fundamentals/authentication) | Contexto de usuario de OAuth 1.0a | Contexto de usuario de OAuth 1.0a<br /><br />OAuth 2.0 con PKCE (Código de autorización) |
| [Límites de frecuencia](/es/resources/fundamentals/rate-limits) predeterminados por solicitud | 50 solicitudes por 15 min | 50 solicitudes por 15 min |
| Máximo de operaciones diarias por usuario | 400 | 400 |
| Máximo de operaciones diarias por app | 1000 | 1000 |
| Requiere usar credenciales de una [App de desarrollador](/es/resources/fundamentals/developer-apps) asociada a un [Proyecto](/es/resources/fundamentals/projects) |     | ✔️  |

<div id="unfollow-a-user">
  #### Dejar de seguir a un usuario
</div>

La siguiente tabla compara los endpoints para eliminar un seguimiento en Standard v1.1 y la API de X v2:

| Descripción | Standard v1.1 | API de X v2 |
| :--- | :--- | :--- |
| Métodos HTTP compatibles | POST | DELETE |
| Dominio del host | https://api.x.com | https://api.x.com |
| Ruta del endpoint | /1.1/friendships/destroy.json | /2/users/:source\_user\_id/following/:target\_user\_id |
| [Autenticación](/es/resources/fundamentals/authentication) | Contexto de usuario de OAuth 1.0a | Contexto de usuario de OAuth 1.0a<br /><br />Código de autorización de OAuth 2.0 con PKCE |
| [Límites de frecuencia](/es/resources/fundamentals/rate-limits) de solicitud predeterminados | 15 solicitudes por 15 min (por usuario) | 50 solicitudes por 15 min (por usuario) |
| Operaciones diarias máximas por app | Ninguna | 500 |
| Requiere el uso de credenciales de una [App de desarrollador](/es/resources/fundamentals/developer-apps) asociada a un [Proyecto](/es/resources/fundamentals/projects) |     | ✔️  |
