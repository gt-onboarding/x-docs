Ofrecemos dos conjuntos de grupos de endpoints para ayudarte a consultar, crear y eliminar relaciones de seguimiento: búsqueda de seguimientos y gestión de seguimientos.

<div id="follows-lookup">
  ### Búsqueda de seguimientos
</div>

Solo disponible para aquellos con [acceso Enterprise](/es/x-api/getting-started/about-x-api)

Los endpoints de búsqueda de seguimientos te permiten explorar y analizar las relaciones entre usuarios, lo que a veces se denomina análisis de redes. Específicamente, hay dos endpoints REST que devuelven [objetos de usuario](/es/x-api/fundamentals/data-dictionary#user) que representan a quién sigue un usuario específico, o quién sigue a un usuario específico.

Puedes autenticar este endpoint con [Contexto de Usuario OAuth 1.0a](/es/resources/fundamentals/authentication), \[Solo App]\(https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), o [OAuth 2.0 con PKCE](es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). Puedes solicitar hasta 1000 usuarios por solicitud, y se proporcionarán tokens de paginación para navegar por grandes conjuntos de resultados.
 

<div id="manage-follows">
  ### Gestionar seguimientos
</div>

Los endpoints de gestión de seguimientos te permiten seguir o dejar de seguir a usuarios.

Dado que realizas solicitudes en nombre de un usuario, debes autenticar estos endpoints con [OAuth 1.0a User Context](/es/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code with PKCE](es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), y utilizar los tokens de acceso de usuario asociados con el usuario en cuyo nombre haces la solicitud. Puedes generar este token de acceso de usuario usando el [flujo OAuth de 3 patas](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) o el [flujo de concesión Authorization Code with PKCE](/es/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0).

Tienes un límite de 400 acciones de seguimiento por día en nombre de cada usuario autenticado y un límite de 1,000 acciones por día por App en total entre todos tus usuarios autenticados. Por ejemplo, si tienes cinco usuarios autenticados, puedes seguir a 400 usuarios por día (límite por usuario) con dos de esos usuarios para un total de 800 acciones, y tendrás que repartir las 200 acciones restantes (por App) entre los otros tres usuarios. Este límite no se aplica al endpoint de dejar de seguir, que tiene un límite independiente de 500 acciones por día (por App).

<Note>
  **Configuración de la cuenta**

  Para acceder a estos endpoints, necesitarás:

  - Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
  - Autenticarte usando las claves y los tokens de una [App de desarrollador](/es/resources/fundamentals/developer-apps) que se encuentre dentro de un [Project](/es/resources/fundamentals/projects).
</Note>

Obtén más información sobre cómo conseguir acceso a los endpoints de la API de X v2 en nuestra [guía de introducción](/es/x-api/getting-started/getting-access).
