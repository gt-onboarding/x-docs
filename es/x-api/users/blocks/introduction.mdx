<div id="blocks-lookup">
  ### Consulta de bloqueos
</div>

El endpoint GET de consulta de bloqueos permite ver qué cuentas has bloqueado en nombre de un usuario autorizado. Este endpoint tiene un límite de 15 solicitudes cada 15 minutos por usuario.

Dado que estás realizando solicitudes de información privada con la consulta de bloqueos y en nombre de un usuario con permiso para gestionar bloqueos, debes autenticar estos endpoints con [OAuth 1.0a User Context](/es/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code with PKCE](es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), y usar los tokens de acceso de usuario asociados con un usuario que haya autorizado tu App, los cuales pueden generarse mediante el [flujo OAuth de 3 patas](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) o el [flujo de concesión Authorization Code con PKCE](/es/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0).

<Note>
  **Configuración de la cuenta**

  Para acceder a estos endpoints, necesitarás:

  - Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
  - Autenticación usando las claves y tokens de una [App de desarrollador](/es/resources/fundamentals/developer-apps) que esté dentro de un [Project](/es/resources/fundamentals/projects).

  Obtén más información sobre cómo acceder a los endpoints de la API de X v2 en nuestra [guía de inicio](/es/x-api/getting-started/getting-access).
</Note>
