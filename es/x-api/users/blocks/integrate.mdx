<div id="authentication">
  #### Autenticación
</div>

Todos los endpoints de la API de X v2 requieren que autentiques tus solicitudes con un conjunto de credenciales, también conocidas como claves y tokens. Puedes usar OAuth 1.0a User Context o OAuth 2.0 Authorization Code con PKCE para autenticar tus solicitudes a estos endpoints.

[OAuth 1.0a User Context](/es/resources/fundamentals/authentication) requiere que utilices tus API Keys, Access Tokens de usuario y otros parámetros para [crear un encabezado de autorización](/es/resources/fundamentals/authentication#authorizing-a-request), que luego enviarás con tu solicitud. Los Access Tokens deben estar asociados con el usuario en cuyo nombre realizas la solicitud. Si deseas generar un conjunto de Access Tokens para otro usuario, ese usuario debe autorizar tu App mediante el [flujo OAuth de 3 pasos](https://developer-staging.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens).

Ten en cuenta que OAuth 1.0a puede ser difícil de usar. Si no estás familiarizado con este método de autenticación, te recomendamos usar una [biblioteca](/es/x-api/tools-and-libraries/overview), una herramienta como Postman o OAuth 2.0 para autenticar tus solicitudes.

[OAuth 2.0 Authorization Code con PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3) permite un mayor control sobre el alcance (scope) de una aplicación y flujos de autorización en múltiples dispositivos. OAuth 2.0 te permite elegir scopes específicos y granulares que otorgan permisos concretos en nombre de un usuario.

Para habilitar OAuth 2.0 en tu App, debes activarlo en la configuración de autenticación de tu App, ubicada en la sección de configuración del portal para desarrolladores.

<div id="developer-portal-projects-and-developer-apps">
  #### Portal para desarrolladores, Proyectos y Apps de desarrollador
</div>

Para obtener un conjunto de credenciales de autenticación que funcionen con los endpoints de la API de X v2, debes [registrarte para una cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info), configurar un [Proyecto](/es/resources/fundamentals/projects) dentro de esa cuenta y crear una [App de desarrollador](/es/resources/fundamentals/developer-apps) dentro de ese Proyecto. Luego podrás encontrar tus claves y tokens dentro de tu App de desarrollador.
 

<div id="rate-limits">
  #### Límites de frecuencia
</div>

Cada día, miles de desarrolladores realizan solicitudes a la API de X. Para gestionar el gran volumen de estas solicitudes, se aplican [límites de frecuencia](/es/x-api/fundamentals/rate-limits) a cada endpoint que restringen la cantidad de solicitudes que puede realizar en nombre de su App o de un usuario autenticado.

Estos endpoints tienen límites de frecuencia a nivel de usuario, lo que significa que el usuario autenticado en cuyo nombre realiza la solicitud solo puede invocar el endpoint un número determinado de veces en cualquier App de desarrollador. El límite de frecuencia por usuario es de 50 solicitudes por cada 15 minutos por endpoint para los métodos POST y DELETE. Sin embargo, para el método GET, el límite es de solo 15 solicitudes por cada 15 minutos.
 

<div id="fields-and-expansions">
  #### Campos y expansiones
</div>

Los endpoints GET de la API de X v2 permiten seleccionar con precisión qué datos desea obtener de la API mediante un conjunto de herramientas denominadas campos y expansiones. El parámetro expansions le permite expandir los objetos referenciados en la carga. Por ejemplo, este endpoint permite extraer las siguientes [expansions](/es/x-api/fundamentals/expansions):

- pinned\_tweet\_id

El parámetro **fields** le permite seleccionar exactamente qué [fields](/es/x-api/fundamentals/fields) dentro de los distintos objetos de datos desea recibir. Estos endpoints entregan principalmente objetos Post. De forma predeterminada, el objeto Post devuelve los campos **id** y **text**. Para recibir campos adicionales como **tweet.created\_at** o **tweet.entities**, deberá solicitarlos explícitamente usando el parámetro **fields**. Algunos campos importantes que quizá desee considerar en su integración son los datos de encuestas, las métricas, las anotaciones de publicaciones y los campos de ID de conversación.

Hemos añadido una guía sobre cómo [usar fields y expansions](/es/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) en conjunto en nuestro [diccionario de datos de la API de X v2](/es/x-api/fundamentals/data-dictionary).
