  ## Comparación de los endpoints de consulta de usuarios de la API de X
</div>

Los endpoints de consulta de usuarios de la v2 reemplazarán los endpoints estándar de v1.1 [GET users/lookup](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-users-lookup.html) y [GET users/show](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-users-show.html). Si cuentas con código, apps o herramientas que usan alguna de estas versiones de los endpoints de consulta de usuarios y estás considerando migrar al endpoint más reciente de la API de X v2, este conjunto de guías es para ti. 
 

Las siguientes tablas comparan los distintos tipos de endpoints de consulta de usuarios:
 

|     |     |     |
| :--- | :--- | :--- |
| **Descripción** | **Estándar v1.1** | **API de X v2** |
| Métodos HTTP compatibles | `GET` | `GET` |
| Dominio del host | `https://api.x.com` | `https://api.x.com` |
| Ruta del endpoint | `/1.1/users/show.json` `/1.1/users/lookup.json` | `/2/users`<br /><br />`/2/users/:id`<br /><br />`/2/users/by`<br /><br />`/2/users/by/:username` |
| [Autenticación](/es/resources/fundamentals/authentication) | OAuth 1.0a en contexto de usuario | OAuth 1.0a en contexto de usuario<br /><br />Sólo app<br /><br />OAuth 2.0 con PKCE |
| [Límites de frecuencia](/es/resources/fundamentals/rate-limits) predeterminados de la solicitud | 900 solicitudes por 15 min (por usuario)<br /><br />/show: 900 solicitudes por 15 min (por app)  <br />/lookup: 300 solicitudes por 15 min (por app) | 900 solicitudes por 15 min (por usuario)<br /><br />300 solicitudes por 15 min (por app) |
| Máximo de usuarios por respuesta | /show: 1<br /><br />/lookup: 100 | 100 |
| Formato del objeto de respuesta JSON | Formato estándar v1.1 | [Formato de la API de X v2](/es/x-api/fundamentals/data-dictionary) (determinado por los parámetros de solicitud fields y expansions; no es compatible con los formatos de v1.1)<br /><br />Para obtener más información sobre cómo migrar del formato estándar v1.1 al formato de la API de X v2, visita nuestra [guía de migración de formatos de datos](/es/x-api/migrate/data-format-migration). |
| Permite seleccionar qué [campos](/es/x-api/fundamentals/data-dictionary) se devuelven en el payload |     | ✔   |
| Permite los campos de [annotations](/es/x-api/fundamentals/post-annotations) (en la publicación fijada) |     | ✔   |
| Permite solicitar nuevos campos de [metrics](/es/x-api/fundamentals/metrics) (en la publicación fijada) |     | ✔   |
| Permite el campo [conversation\_id](/es/x-api/fundamentals/conversation-id) (en la publicación fijada) |     | ✔   |
| Requiere el uso de credenciales de una [developer App](/es/resources/fundamentals/developer-apps) asociada a un [project](/es/resources/fundamentals/projects) |     | ✔   |
