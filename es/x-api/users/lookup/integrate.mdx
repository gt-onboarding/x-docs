<div id="authentication">
  #### Autenticación
</div>

Todos los endpoints de la API de X v2 requieren que las solicitudes estén [autenticadas](/es/resources/fundamentals/authentication) con un conjunto de credenciales, también conocidas como claves y tokens. Puede usar OAuth 1.0a User Context, App only u OAuth 2.0 Authorization Code con PKCE para autenticar solicitudes a estos endpoints.

[OAuth 1.0a User Context](/es/resources/fundamentals/authentication#oauth-1-0a-2) requiere que utilice sus API Keys, Access Tokens de usuario y algunos otros parámetros para [crear un encabezado de autorización](/es/resources/fundamentals/authentication#authorizing-a-request), que luego enviará con su solicitud. Los Access Tokens deben estar asociados con el usuario en cuyo nombre realiza la solicitud. Si desea generar un conjunto de Access Tokens para otro usuario, ese usuario debe autorizar su App utilizando el [flujo OAuth de 3 pasos](https://developer-staging.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens).

Tenga en cuenta que OAuth 1.0a puede ser difícil de usar. Si no está familiarizado con este método de autenticación, le recomendamos usar una [biblioteca](/es/x-api/tools-and-libraries/overview), una herramienta como Postman o usar OAuth 2.0 para autenticar sus solicitudes. Si desea solicitar una publicación o métricas privadas desde estos endpoints, deberá usar OAuth 1.0a User Context u OAuth 2.0 Authorization Code con PKCE, lo que garantizará que tenga los permisos adecuados del usuario propietario de ese contenido.

[App only](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) solo requiere que envíe un [App only Access Token](/es/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) con su solicitud. Puede generar un App only Access Token directamente dentro de una App de desarrollador o generarlo usando el endpoint [POST oauth2/token](/es/resources/fundamentals/authentication#post-oauth2-token).

[OAuth 2.0 Authorization Code con PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) permite un mayor control sobre el alcance de una aplicación y flujos de autorización en múltiples dispositivos. OAuth 2.0 le permite elegir ámbitos de permisos granulares que le otorgan autorizaciones específicas en nombre de un usuario.

Para habilitar OAuth 2.0 en su App, debe activarlo en la configuración de autenticación de su App, ubicada en la sección de configuración de la App en el portal de desarrolladores.

**Tenga en cuenta**

Si solicita los siguientes campos, se requiere OAuth 1.0a User Context u OAuth 2.0 Authorization Code:

- tweet.fields.non\_public\_metrics
- tweet.fields.promoted\_metrics
- tweet.fields.organic\_metrics

<div id="developer-portal-projects-and-developer-apps">
  #### Portal para desarrolladores, Proyectos y Apps de desarrollador
</div>

Para obtener un conjunto de credenciales de autenticación que funcionen con los endpoints de la API de X v2, debes [registrarte para obtener una cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info), configurar un [Proyecto](/es/resources/fundamentals/projects) dentro de esa cuenta y crear una [App de desarrollador](/es/resources/fundamentals/developer-apps) dentro de ese Proyecto. Luego podrás encontrar tus claves y tokens dentro de tu App de desarrollador. 
 

<div id="rate-limits">
  #### Límites de uso
</div>

Cada día, muchos miles de desarrolladores realizan solicitudes a la API de X. Para ayudar a gestionar el gran volumen de estas solicitudes, se aplican [límites de uso](/es/x-api/fundamentals/rate-limits) a cada endpoint que restringen la cantidad de solicitudes que puedes realizar en nombre de tu app o de un usuario autenticado.

Los endpoints de consulta de usuarios tienen límites de uso tanto a nivel de app como a nivel de usuario. Sin embargo, el endpoint de consulta del usuario autenticado tiene límites de uso a nivel de usuario.

El límite de uso a nivel de app significa que tú, como desarrollador, solo puedes realizar una cierta cantidad de solicitudes a este endpoint durante un periodo determinado desde cualquier App (según las claves y tokens que estés utilizando). El límite de uso a nivel de usuario significa que el usuario autenticado en cuyo nombre realizas la solicitud solo puede efectuarla un número determinado de veces en cualquier App de desarrollador.

La siguiente tabla muestra los límites de uso para cada endpoint.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Método HTTP** | **Límite de uso / Nivel** |
| /2/users | GET | 900 solicitudes por 15 minutos / App y Usuario |
| /2/users/:id | GET | 900 solicitudes por 15 minutos / App y Usuario |
| /2/users/by | GET | 900 solicitudes por 15 minutos / App y Usuario |
| /2/users/by/username/:username | GET | 900 solicitudes por 15 minutos / App y Usuario |
| /2/users/me | GET | 75 solicitudes por 15 minutos / Usuario |

<div id="fields-and-expansions">
  #### Campos y expansiones
</div>

La API de X v2 permite seleccionar exactamente qué datos deseas obtener de la API mediante un conjunto de herramientas llamadas campos y expansiones. El parámetro de expansión permite ampliar los objetos referenciados en la carga útil. Por ejemplo, este endpoint permite usar la expansión pinned\_tweet\_id.

El parámetro de campos permite seleccionar exactamente qué [campos](/es/x-api/fundamentals/fields) dentro de los distintos objetos de datos deseas recibir. Estos endpoints entregan principalmente objetos de usuario. De forma predeterminada, el objeto de usuario devuelve los campos id, name y username. Para recibir campos adicionales como user.created\_at o user.location, tendrás que solicitarlos explícitamente mediante un parámetro de campos. Algunos campos importantes que podrías considerar para tu integración son los de datos de encuestas de publicaciones, métricas, anotaciones y el ID de conversación.

Hemos añadido una guía sobre cómo [usar campos y expansiones](/es/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) en nuestro [diccionario de datos de la API de X v2](/es/x-api/fundamentals/data-dictionary).
