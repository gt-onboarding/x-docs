Puede autenticar su solicitud a todos los endpoints de consulta de usuarios, excepto el de consulta del usuario autenticado, con [OAuth 1.0a User Context](/es/resources/fundamentals/authentication), [App only](/es/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) o [OAuth 2.0 Authorization code with PKCE](es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). Sin embargo, si desea acceder a métricas privadas o a datos de usuarios privados, deberá utilizar OAuth 1.0a User Context u OAuth 2.0 Authorization Code with PKCE y enviar el Access Token de los usuarios autenticados con su solicitud.

Este endpoint se utiliza comúnmente para obtener detalles actualizados de un usuario, verificar que un usuario exista o actualizar sus datos almacenados tras un evento de cumplimiento.

<div id="authenticated-user-lookup">
  ### Consulta de usuario autenticado
</div>

Si se autentica en nombre de otros usuarios, es fundamental poder ver por quién puede realizar solicitudes.

Este endpoint requiere usar [OAuth 1.0a User Context](/es/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code with PKCE](es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). Devuelve información sobre el usuario autorizado, es decir, el usuario asociado con los tokens de acceso de usuario que envía con la solicitud.

Puede acceder a este endpoint mediante el método GET. Existe un límite por usuario de 75 solicitudes cada 15 minutos para este endpoint.

<Note>
  **Configuración de la cuenta**

  Para acceder a estos endpoints, necesitará:

  - Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
  - Autenticarse con las claves y tokens de una [App de desarrollador](/es/resources/fundamentals/developer-apps) que esté dentro de un [Proyecto](/es/resources/fundamentals/projects).

  Obtenga más información sobre cómo acceder a los endpoints de la API de X v2 en nuestra [guía de inicio](/es/x-api/getting-started/getting-access).
</Note>
