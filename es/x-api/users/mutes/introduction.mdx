Silenciar una cuenta te permite dejar de ver las publicaciones de esa cuenta en tu cronología sin dejar de seguirla ni bloquearla. Las cuentas silenciadas no sabrán que las has silenciado y puedes quitar el silencio en cualquier momento. Con los endpoints para gestionar silencios, los desarrolladores pueden crear experiencias más seguras para las personas en X. Un ejemplo de cómo crear con la gestión de silencios es una aplicación que te permite silenciar cuentas que podrían publicar sobre temas específicos durante un período determinado. Con el endpoint de consulta de silencios, puedes ver a quién has silenciado tú o un usuario autenticado. Esto puede ser útil para determinar cómo interactúas con las cuentas silenciadas.

Dado que estás realizando solicitudes de información privada con la consulta de silencios y, en nombre de un usuario, con la gestión de silencios, debes autenticar estos endpoints con [OAuth 1.0a User Context](/es/resources/fundamentals/authentication) o [OAuth 2.0 Authorization Code with PKCE](es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), y usar los tokens de acceso de usuario asociados con un usuario que haya autorizado tu app, los cuales se pueden generar usando el [flujo OAuth de 3 pasos](/es/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) o el [flujo de concesión Authorization Code with PKCE](/es/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0).

<div id="mutes-lookup">
  ### Consulta de silenciamientos
</div>

El endpoint de consulta de silenciamientos permite ver qué cuentas ha silenciado el usuario autenticado. Este endpoint tiene un límite de 15 solicitudes cada 15 minutos por usuario.

<div id="manage-mutes">
  ### Administrar silencios
</div>

Los endpoints de administración de silencios te permiten silenciar o desilenciar una cuenta específica en nombre de un usuario autenticado. Para estos endpoints hay dos métodos disponibles: POST y DELETE. El método POST te permite silenciar una cuenta y el método DELETE te permite desilenciarla. Existe un límite por usuario de 50 solicitudes cada 15 minutos para los endpoints POST y DELETE.

**Ten en cuenta:** Si un usuario silencia desde [X](https://x.com/), hay un límite de 200 solicitudes cada 15 minutos.

**Configuración de la cuenta**

Para acceder a estos endpoints, necesitarás:

- Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
- Autenticarte usando las claves y los tokens de una [App de desarrollador](/es/resources/fundamentals/developer-apps) que se encuentra dentro de un [Proyecto](/es/resources/fundamentals/projects).

Obtén más información sobre cómo obtener acceso a los endpoints de la API de X v2 en nuestra [guía de inicio](/es/x-api/getting-started/getting-access).
