<div id="comparing-x-apis-mutes-endpoints">
  ## Comparación de los endpoints de silenciamiento de la API de X
</div>

<div id="mutes-lookup">
  #### Búsqueda de cuentas silenciadas
</div>

El endpoint de consulta de silencios de la API de X v2 reemplazará los endpoints estándar [v1.1 GET mutes/users/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/get-mutes-users-ids) y [GET mutes/users/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/get-mutes-users-list).

Las siguientes tablas comparan los endpoints de silencios de la versión estándar v1.1 y de la API de X v2:
 

| Descripción | Estándar v1.1 | API de X v2 |
| :--- | :--- | :--- |
| Métodos HTTP admitidos | ******GET****** | ******GET****** |
| Dominio del host | ******https://api.x.com****** | ******https://api.x.com****** |
| Ruta del endpoint | ******/1.1/mutes/users/ids.json******<br /><br />/1.1/mutes/users/list.json | ******/2/users/:id/muting****** |
| [Autenticación](/es/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code with PKCE |
| [Límites de frecuencia](/es/resources/fundamentals/rate-limits) de solicitud predeterminados | 15 solicitudes por 15 min (por usuario) | 15 solicitudes por 15 min (por usuario) |
| Formatos de datos | Formato estándar v1.1 | [Formato de la API de X v2](/es/x-api/fundamentals/data-dictionary) (determinado por los parámetros de solicitud fields y expansions; no es retrocompatible con los formatos de v1.1)<br /><br />Para obtener más información sobre cómo migrar del formato estándar v1.1 al formato de la API de X v2, consulta nuestra [guía de migración de formatos de datos](/es/x-api/migrate/data-format-migration). |
| Requiere el uso de credenciales de una [App de desarrollador](/es/resources/fundamentals/developer-apps) asociada a un [Proyecto](/es/resources/fundamentals/projects) |     | ✔️  |

<div id="manage-mutes">
  #### Administrar silencios
</div>

Los endpoints de administración de silencios de v2 reemplazarán los endpoints estándar de v1.1 [POST mutes/users/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-create) y [POST mutes/users/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/mute-block-report-users/api-reference/post-mutes-users-destroy).

Las siguientes tablas comparan los endpoints de silencios estándar de v1.1 y de la API de X v2:

**Silenciar a un usuario**

| Descripción | v1.1 estándar | API de X v2 |
| :--- | :--- | :--- |
| Métodos HTTP compatibles | POST | POST |
| Dominio host | https://api.x.com | https://api.x.com |
| Ruta del endpoint | /1.1/mutes/users/create.json | /2/users/:id/muting |
| [Autenticación](/es/resources/fundamentals/authentication) | Contexto de usuario OAuth 1.0a | Contexto de usuario OAuth 1.0a<br /><br />Código de autorización OAuth 2.0 con PKCE |
| [Límites de velocidad](/es/resources/fundamentals/rate-limits) de solicitud predeterminados | 50 solicitudes por 15 min | 50 solicitudes por 15 min |
| Requiere el uso de credenciales de una [App de desarrollador](/es/resources/fundamentals/developer-apps) asociada a un [Proyecto](/es/resources/fundamentals/projects) |     | ✔️  |

**Quitar el silencio a un usuario**
