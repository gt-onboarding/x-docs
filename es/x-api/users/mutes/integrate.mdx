<div id="authentication">
  #### Autenticación
</div>

Todos los endpoints de la API de X v2 requieren que autentiques tus solicitudes con un conjunto de credenciales, también conocidas como claves y tokens. Puedes usar OAuth 1.0a en contexto de usuario o el flujo de Código de Autorización de OAuth 2.0 con PKCE para autenticar tus solicitudes a estos endpoints.

[OAuth 1.0a en contexto de usuario](/es/resources/fundamentals/authentication) requiere que utilices tus API Keys, tokens de acceso de usuario y algunos otros parámetros para [crear un encabezado de autorización](/es/resources/fundamentals/authentication#authorizing-a-request), que luego enviarás con tu solicitud. Los tokens de acceso deben estar asociados con el usuario en cuyo nombre haces la solicitud. Si deseas generar un conjunto de tokens de acceso para otro usuario, ese usuario debe autorizar tu App usando el [flujo OAuth de 3 patas](https://developer-staging.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens).

Ten en cuenta que OAuth 1.0a puede resultar difícil de usar. Si no estás familiarizado con este método de autenticación, te recomendamos usar una [biblioteca](/es/x-api/tools-and-libraries/overview), una herramienta como Postman o OAuth 2.0 para autenticar tus solicitudes.

[OAuth 2.0 con Código de Autorización y PKCE](/es/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) permite un mayor control sobre el alcance (scope) de una aplicación y flujos de autorización en múltiples dispositivos. OAuth 2.0 te permite elegir scopes específicos y granulares que otorgan permisos concretos en nombre de un usuario.

Para habilitar OAuth 2.0 en tu App, debes activarlo en la configuración de autenticación de tu App, ubicada en la sección de configuración del portal para desarrolladores.

<div id="developer-portal-projects-and-developer-apps">
  #### Portal para desarrolladores, Proyectos y Apps de desarrollador
</div>

Para obtener un conjunto de credenciales de autenticación que funcionen con los endpoints de la API de X v2, debes [registrarte para una cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info), configurar un [Proyecto](/es/resources/fundamentals/projects) dentro de esa cuenta y crear una [App de desarrollador](/es/resources/fundamentals/developer-apps) dentro de ese Proyecto. Luego podrás encontrar tus claves y tokens en tu App de desarrollador.
 

<div id="rate-limits">
  #### Límites de frecuencia
</div>

Cada día, muchos miles de desarrolladores realizan solicitudes a la API de X. Para ayudar a gestionar el gran volumen de estas solicitudes, se aplican [límites de frecuencia](/es/x-api/fundamentals/rate-limits) a cada endpoint que restringen la cantidad de solicitudes que puedes realizar en nombre de tu app o de un usuario autenticado.

Estos endpoints tienen límites de frecuencia a nivel de usuario, lo que significa que el usuario autenticado en cuyo nombre haces la solicitud solo puede invocar el endpoint un número determinado de veces en cualquier App de desarrollador.

Existe un límite de frecuencia por usuario de 50 solicitudes por 15 minutos y por endpoint con los métodos POST y DELETE. Sin embargo, con el método GET, el límite es de solo 15 solicitudes por 15 minutos.
 

<div id="fields-and-expansions">
  #### Campos y expansiones
</div>

El endpoint GET de la API de X v2 permite seleccionar con precisión qué datos desea obtener de la API mediante un conjunto de herramientas denominadas campos y expansiones. El parámetro expansions permite ampliar los objetos referenciados en el payload. Por ejemplo, este endpoint permite obtener las siguientes [expansiones](/es/x-api/fundamentals/expansions):

- pinned\_tweet\_id

El parámetro **fields** permite seleccionar exactamente qué [campos](/es/x-api/fundamentals/fields) de los distintos objetos de datos desea recibir. Este endpoint entrega principalmente objetos User. De forma predeterminada, el objeto User devuelve los campos **id**, **name** y **username**. Para recibir campos adicionales como **user.created\_at** o **user.entities**, deberá solicitarlos explícitamente usando el parámetro **fields**.

Hemos añadido una guía sobre cómo usar [campos y expansiones](/es/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions) conjuntamente en nuestro [diccionario de datos de la API de X v2](/es/x-api/fundamentals/data-dictionary).

<div id="pagination">
  #### Paginación
</div>

La búsqueda de silenciamientos puede devolver muchos datos. Para garantizar resultados coherentes y de alto rendimiento en todo momento, usamos paginación. La paginación es una función de los endpoints de la API de X v2 que devuelven más resultados de los que se pueden incluir en una sola respuesta. Cuando ocurre esto, los datos se devuelven en una serie de “páginas”. Obtén más información sobre cómo [paginar resultados.](/es/x-api/fundamentals/pagination)

<Note>
  **Ten en cuenta:** Si un usuario silencia desde [X](https://x.com/), hay un límite de 200 solicitudes por cada 15 minutos.
</Note>
