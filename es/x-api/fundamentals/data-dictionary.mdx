<div id="tweet">
  ### Tuit
</div>

Los tuits son el componente básico de todo en Twitter. El objeto Tuit tiene una extensa lista de campos de “nivel raíz”, como `id`, `text` y `created_at`. Los objetos Tuit también son el objeto “principal” de varios objetos secundarios, incluidos `user`, `media`, `poll` y `place`. Use el parámetro de campo `tweet.fields` cuando solicite estos campos de nivel raíz en el objeto Tuit.

El objeto Tuit puede encontrarse y expandirse en el recurso de usuario. También pueden encontrarse y expandirse tuits adicionales relacionados con el tuit solicitado en el recurso Tuit. El objeto está disponible para expansión con `?expansions=pinned_tweet_id` en el recurso de usuario o con `?expansions=referenced_tweets.id` en el recurso Tuit para obtener el objeto solo con los campos predeterminados. Use la expansión junto con el parámetro de campo `tweet.fields` cuando solicite campos adicionales para completar el objeto.

| Valor del campo       | Tipo   | Descripción                                                                                                                           | Cómo puede usarse                                                   |
|:-----------------------|:--------|:---------------------------------------------------------------------------------------------------------------------------------------|:----------------------------------------------------------------------|
| **id (predeterminado)**      | string | El identificador único del Tweet solicitado.                                                                                        | Úselo para recuperar programáticamente un Tweet específico.          |
| **text (predeterminado)**    | string | El texto UTF-8 del Tweet. Consulte [twitter-text](https://github.com/twitter/twitter-text/) para obtener detalles sobre caracteres válidos.   | Extracción de palabras clave y análisis/clasificación de sentimiento. |
| **edit\_history\_tweet\_ids (predeterminado)** | object | Identificadores únicos que indican todas las versiones de un Tweet. Para Tweets sin ediciones habrá un ID. Para Tweets con historial de ediciones habrá varios IDs. | Use esta información para encontrar el historial de ediciones de un Tweet. |
| **attachments**       | object | Especifica el tipo de archivos adjuntos (si los hay) presentes en este Tweet.                                                                    | Comprender los objetos devueltos para expansiones solicitadas.       |
| **author\_id**         | string | El identificador único del usuario que publicó este Tweet.                                                                             | Completar el objeto User; compartir conjuntos de datos para revisión por pares. |
| **context\_annotations** | array | Contiene anotaciones de contexto del Tweet.                                                                                          | Reconocimiento/extracción de entidades; análisis temático.           |
| **conversation\_id**   | string | El ID del Tweet original de la conversación (incluye respuestas directas y respuestas a respuestas).                         | Úselo para reconstruir la conversación a partir de un Tweet.         |
| **created\_at**        | date (ISO 8601) | Fecha y hora de creación del Tweet.                                                                                                      | Útil para análisis de series temporales y para entender cuándo se creó un Tweet. |
| **edit\_controls**     | object | Indica cuánto tiempo más puede editarse el Tweet y la cantidad de ediciones restantes.                                                | Úselo para determinar si un Tweet es elegible para edición.          |
| **entities**          | object | Entidades extraídas del texto del Tweet. Consulte entities en Twitter Objects.                                       | Proporciona información adicional sobre hashtags, URL, menciones, etc. |
| **in\_reply\_to\_user\_id** | string | Si el Tweet representado es una respuesta, este campo contendrá el ID del autor del Tweet original.                                       | Determinar si un Tweet fue una respuesta a otro Tweet.               |
| **lang**              | string | Idioma del Tweet, si es detectado por Twitter.                                                                                      | Clasificar Tweets por idioma.                                        |
| **non\_public\_metrics** | object | Métricas de interacción no públicas del Tweet en el momento de la solicitud. Requiere autenticación con contexto de usuario.                       | Determinar el total de impresiones generadas por el Tweet.           |
| **organic\_metrics**   | object | Métricas de interacción, registradas en un contexto orgánico, del Tweet en el momento de la solicitud. Requiere autenticación con contexto de usuario.  | Medir la interacción orgánica del Tweet.                             |
| **possibly\_sensitive** | boolean | Indica si el contenido puede considerarse sensible.                                                                           | Estudiar la circulación de ciertos tipos de contenido.               |
| **promoted\_metrics**  | object | Métricas de interacción, registradas en un contexto promocionado, del Tweet en el momento de la solicitud. Requiere autenticación con contexto de usuario. | Medir la interacción del Tweet cuando fue promocionado.              |
| **public\_metrics**    | object | Métricas de interacción públicas del Tweet en el momento de la solicitud.                                                                 | Medir la interacción del Tweet.                                      |
| **referenced\_tweets** | array  | Lista de Tweets a los que este Tweet hace referencia, como Retweets, Tweets citados o respuestas.                                                | Entender aspectos conversacionales de los retweets, etc.             |
| **reply\_settings**    | string | Muestra quién puede responder a un Tweet. Las opciones son "everyone", "mentioned\_users" y "followers".                                  | Determinar la configuración de respuestas del Tweet.                 |
| **withheld**          | object | Contiene detalles de retención para [contenido retenido](https://help.x.com/en/rules-and-policies/tweet-withheld-by-country).     |                                                                     |
| **media\_metadata**          | array | Contiene metadatos de los archivos multimedia adjuntos del Tweet.     | Obtener metadatos adicionales como el `alt_text` del archivo multimedia adjunto de un Tweet. |

**Recuperación de un objeto Tweet**

**Solicitud de ejemplo**

En la siguiente solicitud, se piden campos del Tweet en el endpoint de [Tweets lookup](/es/resources/fundamentals/rate-limits). Asegúrese de reemplazar `$BEARER_TOKEN` por su propio [Bearer Token](/es/resources/fundamentals/authentication) generado.

```bash
curl --request GET 'https://api.x.com/2/tweets?ids=1212092628029698048&tweet.fields=attachments,author_id,context_annotations,created_at,entities,geo,id,in_reply_to_user_id,lang,possibly_sensitive,public_metrics,referenced_tweets,text,withheld&expansions=referenced_tweets.id' --header 'Authorization: Bearer $BEARER_TOKEN'
```

<div id="user">
  ### Usuario
</div>

El objeto de usuario contiene metadatos de la cuenta de usuario de Twitter que describen al usuario de referencia. El objeto de usuario es el objeto principal devuelto en el endpoint [users lookup](/es/x-api/users/lookup/introduction). Al solicitar campos de usuario adicionales en este endpoint, simplemente use el parámetro de campos `user.fields`.

El objeto de usuario también puede aparecer como objeto secundario y expandirse dentro del objeto Tweet. Está disponible para expansión con `?expansions=author_id` o `?expansions=in_reply_to_user_id` para obtener el objeto condensado con solo los campos predeterminados. Use la expansión junto con el parámetro de campos `user.fields` al solicitar campos adicionales para completar el objeto.

<div id="space">
  ### Espacio
</div>

Los Spaces permiten expresarse e interactuar mediante conversaciones de audio en vivo. El diccionario de datos de un Space contiene metadatos relevantes sobre ese Space; todos los detalles se actualizan en tiempo real.

Los objetos de usuario se pueden encontrar y expandir en el recurso de usuario. Estos objetos están disponibles para expansión agregando al menos uno de `host_ids`, `creator_id`, `speaker_ids` o `mentioned_user_ids` al parámetro de consulta `expansions`.

A diferencia de los Tweets, los Spaces son efímeros y dejan de estar disponibles cuando finalizan o cuando su creador los cancela. Cuando tu app gestione datos de Spaces, eres responsable de proporcionar la información más actualizada y debes eliminar los datos que ya no estén disponibles en la plataforma. Los [endpoints de consulta de Spaces](/es/x-api/spaces/lookup/introduction) pueden ayudarte a garantizar que respetas las expectativas y la intención de los usuarios.

<div id="direct-message-events">
  ### Eventos de Mensaje Directo
</div>

Las conversaciones de Mensaje Directo (DM) se componen de eventos. La API de X v2 actualmente admite tres tipos de eventos: MessageCreate, ParticipantsJoin y ParticipantsLeave.

Los objetos de eventos de DM los devuelven los endpoints de [consulta de Mensajes Directos](/es/x-api/direct-messages/lookup/introduction), y se crea un evento MessageCreate cuando los Mensajes Directos se crean correctamente con los endpoints de [gestión de Mensajes Directos](/es/x-api/direct-messages/lookup/introduction).

Al solicitar eventos de DM, se incluyen tres atributos predeterminados del objeto de evento, o campos: id, event\_type y text. Para recibir campos de evento adicionales, use el parámetro [fields](/es/x-api/fundamentals/fields) dm\_event.fields para seleccionar otros. Otros campos de evento disponibles incluyen lo siguiente: dm\_conversation\_id, created\_at, sender\_id, attachments, participant\_ids y referenced\_tweets.

Varios de estos campos proporcionan los ID de otros objetos de X relacionados con el evento de Mensaje Directo:

- sender\_id - El ID de la cuenta que envió el mensaje o que invitó a un participante a una conversación grupal
- participant\_ids - Una matriz de ID de cuenta. Para los eventos ParticipantsJoin y ParticipantsLeave, esta matriz contendrá un único ID de la cuenta que creó el evento
- attachments - Proporciona ID de medios para contenido que el remitente ha subido a Twitter
- referenced\_tweets - Si se encuentra una URL de Tweet en el campo text, el ID de ese Tweet se incluye en la respuesta

Las [expansions](/es/x-api/fundamentals/expansions) sender\_id, participant\_ids, referenced\_tweets.id y attachments.media\_keys están disponibles para ampliar estos ID de objetos de Twitter.

<div id="how-to-use-fields-and-expansions">
  ## Cómo usar fields y expansions
</div>

De forma predeterminada, los objetos de datos de la API de X v2 incluyen una pequeña cantidad de campos predeterminados cuando se realiza una solicitud sin usar los parámetros [fields](/es/x-api/fundamentals/fields) o [expansions](/es/x-api/fundamentals/expansions). Esta guía le mostrará cómo usar los parámetros de consulta `fields` y `expansions` en su solicitud para recibir objetos y campos adicionales en la respuesta.

En esta guía, solicitaremos varios campos tomando como referencia la siguiente captura de pantalla de un Tweet.
 

![Esta imagen incluye una captura de pantalla de un Tweet publicado por @X. Puede ver el texto del Tweet, el nombre de usuario, la fecha y hora de publicación, la fuente y las métricas públicas. También incluye un video. ](https://cdn.cms-twdigitalassets.com/content/dam/developer-twitter/ScreenShotDataDictionaryGuide.png.twimg.1920.png)

Como puede ver en la captura, hay varios elementos de información visibles relacionados con el Tweet, incluido el autor, las métricas, la marca de tiempo de creación, el video y el conteo de visualizaciones del video. También hay varios datos que no son visibles en la captura, pero que igualmente se pueden solicitar.

Al realizar una solicitud a la API, la respuesta predeterminada es simple y contiene solo los campos predeterminados del Tweet (id y text). También recibirá únicamente el objeto principal que devuelve el endpoint que esté utilizando, y no los objetos de datos asociados que puedan estar relacionados con el objeto principal.

Esta simplicidad, junto con los parámetros fields y expansions, le permite solicitar solo los campos que necesita, según su caso de uso.
