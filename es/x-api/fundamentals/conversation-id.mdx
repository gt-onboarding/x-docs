<div id="conversation-threads-using-the-x-api">
  ### Hilos de conversación con la API de X
</div>

Si observas cómo evolucionan las conversaciones en X, una publicación puede generar varios hilos, cada uno de los cuales puede crecer en longitud y complejidad a medida que más personas intervienen. Identificar las relaciones entre publicaciones y comprender los hilos de conversación es una función del payload y de las capacidades de búsqueda de la API de X v2. Cuando se publica una respuesta a una publicación (o una respuesta a otra respuesta), cada respuesta incluye ahora un `conversation_id` definido, que coincide con el ID de la publicación original que inició la conversación.

Las respuestas a una publicación determinada, así como las respuestas a esas respuestas, forman parte de la conversación que se origina en esa publicación original. Independientemente de cuántos hilos de respuesta se generen, todos compartirán el mismo `conversation_id` correspondiente a la publicación original que desencadenó la conversación. Con la API de X v2, puedes recuperar y reconstruir un hilo de conversación completo, lo que te permite comprender qué se dice y cómo evolucionan las conversaciones e ideas.

Recuperar `conversation_id` como parámetro `tweet.fields`
Para solicitar el `conversation_id` de todas las publicaciones que devuelve un endpoint de la v2, agrega el campo tweet.fields=conversation\_id a los parámetros de la solicitud. El campo conversation\_id siempre es el ID de la publicación original en el hilo de respuestas de la conversación. Todas las publicaciones dentro del mismo hilo de respuestas, incluidas las que se crean a partir de hilos de respuestas anteriores, mostrarán el mismo conversation\_id.

<div id="request-with-conversation_id-parameter">
  ### Solicitud con el parámetro `conversation_id`
</div>

```
curl --request GET \
  --url 'https://api.x.com/2/tweets?ids=1225917697675886593&tweet.fields=author_id,conversation_id,created_at,in_reply_to_user_id,referenced_tweets&expansions=author_id,in_reply_to_user_id,referenced_tweets.id&user.fields=name,username' \
  --header 'Authorization: Bearer $ACCESS_TOKEN'
```

<div id="using-conversation_id-as-a-filter-operator">
  ### Uso de `conversation_id` como operador de filtro
</div>

El `conversation_id` puede utilizarse como parámetro de consulta en la búsqueda reciente o como operador dentro de una regla para el [flujo filtrado](/es/x-api/posts/filtered-stream/introduction). Esto devolverá todo el hilo de conversación de las publicaciones, ya sea en tiempo real a través del flujo filtrado o en orden cronológico inverso desde [search Tweets](/es/x-api/posts/search/introduction). También puedes obtener un conteo de las publicaciones en una conversación usando este operador con [Posts counts](/es/x-api/posts/counts/introduction).

<div id="request-to-query-by-conversation_id">
  ## Solicitud para consultar por `conversation_id`
</div>

```
curl --request GET \
  --url 'https://api.x.com/2/tweets/search/recent?query=conversation_id:1279940000004973111&tweet.fields=in_reply_to_user_id,author_id,created_at,conversation_id' \
  --header 'Authorization: Bearer $ACCESS_TOKEN'
```
