<div id="introduction">
  ## Introducción
</div>

La paginación es una funcionalidad de los endpoints de la API de X v2 que devuelve más resultados de los que pueden incluirse en una única respuesta. Cuando eso sucede, los datos se devuelven en una serie de “páginas”. La paginación hace referencia a los métodos para solicitar de forma programática todas las páginas y recuperar el conjunto completo de resultados. No todos los endpoints de la API admiten o requieren paginación, pero suele utilizarse cuando los conjuntos de resultados son grandes.

<div id="use-cases-for-pagination">
  ### Casos de uso de la paginación
</div>

**Para obtener todos los resultados de una solicitud:** Usa la paginación para recibir todos los datos relevantes asociados a una solicitud y sus parámetros. La paginación es necesaria cuando hay más resultados coincidentes que el valor de `max_results` de la solicitud. Repetir solicitudes con tokens de paginación te permite obtener todos los resultados. Cuando la respuesta no incluya un `next_token`, puede asumirse que ya se recorrieron todas las páginas de resultados. No utilices la paginación para sondeo. Para obtener los resultados más recientes desde una solicitud anterior, consulta el sondeo con `since_id`.

**Para desplazarte por los resultados de una solicitud:** La paginación ofrece opciones direccionales para navegar por los resultados de una solicitud, usando los valores `next_token` y `previous_token` de las respuestas. Estos tokens pueden establecerse como `pagination_token` en la siguiente solicitud para ir a la página siguiente o anterior de resultados.

<div id="pagination-token-definitions">
  ### Definiciones de tokens de paginación
</div>

- `next_token` - Cadena opaca devuelta dentro del objeto meta en las respuestas de endpoints que admiten paginación. Indica que hay más resultados disponibles y puede usarse como parámetro `pagination_token` en la siguiente solicitud para devolver la página siguiente de resultados. La última página de resultados no incluirá un `next_token`.
  - `previous_token` - Cadena opaca devuelta dentro del objeto meta en las respuestas de endpoints que admiten paginación. Indica que hay una página anterior de resultados disponible y puede establecerse como parámetro `pagination_token` en la siguiente solicitud para devolver la página anterior de resultados.
  - `pagination_token` - Parámetro utilizado en las solicitudes de paginación. Establézcalo con el valor de `next_token` para la siguiente página de resultados. Establézcalo con el valor de `previous_token` para la página anterior de resultados.

<div id="fundamentals-of-pagination">
  ### Fundamentos de la paginación
</div>

- Los endpoints que usan paginación responderán a una solicitud inicial con la primera página de resultados y proporcionarán un `next_token` dentro del objeto meta en la respuesta JSON si hay páginas adicionales disponibles. Para recibir todos los resultados, repita este proceso hasta que la respuesta ya no incluya un `next_token`.
  - Los resultados se devuelven en orden cronológico inverso. Esto se cumple dentro de cada página y también entre páginas:
    - La primera publicación en la primera respuesta será la más reciente.
    - La última publicación en la última respuesta será la más antigua.
  - El parámetro de solicitud `max_results` le permite configurar la cantidad de publicaciones devueltas por página de respuesta. Existe un valor predeterminado y un valor máximo para `max_results`.
  - Toda implementación de paginación implicará extraer tokens del payload de la respuesta, que pueden usarse en solicitudes posteriores.
  - En algunas circunstancias, puede recibir menos elementos que el valor de `max_results` por página. No suponga que los resultados por página siempre serán iguales al valor del parámetro `max_results`.
  - Las mejores formas de usar la paginación para obtener resultados completos son mediante lógica de bucle en el código de integración o utilizando una [biblioteca](/es/x-api/tools-and-libraries/overview) compatible con la API de X v2.
