---
title: Subir contenido multimedia
---

<div id="overview">
  ## Descripción general
</div>

Un objeto multimedia representa una sola foto, un video o un GIF animado. Los objetos multimedia se utilizan en muchos endpoints de la API de X y pueden incluirse en publicaciones, Mensajes Directos, perfiles de usuario, creatividades publicitarias y otros lugares. Cada objeto multimedia puede tener múltiples variantes de visualización o reproducción, con diferentes resoluciones o formatos.

<div id="media-types-size-restrictions">
  ## Tipos de medios y restricciones de tamaño
</div>

Restricciones de tamaño para subir mediante la API

- Imagen: 5 MB
- GIF: 15 MB
- Video: 512 MB (cuando se usa media_category=amplify)

<div id="creation">
  ## Creación
</div>

Objetos como Tweets, Mensajes Directos, fotos de perfil de usuario, tarjetas de anuncios alojadas, etc., pueden contener uno o más objetos multimedia. Estos objetos de nivel superior se conocen colectivamente como entidades. A la API de creación de entidades correspondiente (p. ej., POST statuses/update) se le pueden pasar uno o más objetos multimedia usando un media_id único.

Una entidad que contiene objeto(s) multimedia puede crearse siguiendo estos pasos:

1. Cargue el/los archivo(s) multimedia utilizando la carga fragmentada recomendada (imágenes/GIF/video) o la carga simple anterior (solo imágenes).
2. Reciba un media_id del paso 1. Este paso puede repetirse varias veces con diferentes medios si la entidad permite pasar múltiples parámetros media_id.
3. Cree la entidad llamando al endpoint correspondiente, incluyendo el media_id y otros parámetros requeridos. Por ejemplo, adjunte un media_id a un Tweet utilizando el endpoint POST statuses/update.

<div id="retrieving">
  ## Recuperación
</div>

Consulte el objeto de medios en el diccionario de datos de Tweets.
