Un objeto multimedia representa una única foto, video o GIF animado. Los objetos multimedia son utilizados por muchos endpoints dentro de la API de X y pueden incluirse en publicaciones, Mensajes Directos, perfiles de usuario, creatividades publicitarias y otros lugares. Cada objeto multimedia puede tener múltiples variantes de visualización o reproducción, con diferentes resoluciones o formatos.

<div id="media-types-size-restrictions">
  ## Tipos de medios y restricciones de tamaño
</div>

Restricciones de tamaño para cargar mediante la API

- **Imagen**: `5 MB`
- **GIF**: `15 MB`
- **Video**: `512 MB` (cuando se utiliza `media_category=amplify_video`)

<div id="creation">
  ## Creación
</div>

Objetos como publicaciones, Mensajes Directos, fotos de perfil de usuario, tarjetas de anuncios alojadas, etc., pueden contener uno o más objetos multimedia. Estos objetos de nivel superior se conocen colectivamente como entidades. A la API de creación de la entidad correspondiente (p. ej., [`POST /2/tweets`](/es/x-api/posts/creation-of-a-post)) se le pueden pasar uno o más objetos multimedia utilizando un `media_id` único.

Se puede crear una entidad que contenga objetos multimedia siguiendo estos pasos:

1. Cargue el/los archivo(s) multimedia usando la carga [fragmentada](/es/x-api-/media/quickstart/media-upload-chunked) recomendada (imágenes/GIF/video) o la carga [simple](/es/x-api/media-upload/media-upload) anterior (solo imágenes).
2. Reciba un `media_id` del paso 1. Este paso puede repetirse varias veces con diferentes medios si la entidad permite pasar varios parámetros `media_id`.
3. Cree la entidad llamando al endpoint correspondiente, incluyendo el `media_id` y otros parámetros requeridos. Por ejemplo, adjunte un `media_id` a una publicación usando el endpoint [`POST /2/tweets`](/es/x-api/posts/creation-of-a-post).
