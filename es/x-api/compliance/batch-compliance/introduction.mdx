X está comprometido con nuestra comunidad de desarrolladores que crean con la API de X. Como parte de este compromiso, nuestro objetivo es que nuestra API sea abierta y justa para los desarrolladores, segura para las personas en X y beneficiosa para la plataforma de X en su conjunto. Es crucial que cualquier desarrollador que almacene contenido de X sin conexión se asegure de que los datos reflejen la intención del usuario y el estado actual del contenido en X. Por ejemplo, cuando alguien en X elimina una publicación o su cuenta, protege sus publicaciones o elimina la información de geolocalización de sus publicaciones, es fundamental tanto para X como para nuestros desarrolladores respetar las expectativas e intención de esa persona. Los endpoints de cumplimiento por lotes proporcionan a los desarrolladores una herramienta sencilla para ayudar a mantener los datos de X en cumplimiento con el [Acuerdo y Política para Desarrolladores de X](https://developer.x.com/en/developer-terms/policy). 

Estos endpoints de cumplimiento por lotes te permiten cargar grandes conjuntos de datos de IDs de publicaciones o de usuarios para recuperar su estado de cumplimiento, a fin de determinar qué datos requieren acción para poner tus conjuntos de datos en cumplimiento. Ten en cuenta que el uso de los endpoints de cumplimiento por lotes está restringido a los casos de uso antes mencionados, y cualquier otro propósito está prohibido y puede dar lugar a medidas de aplicación.

Por lo general, hay 4 pasos al trabajar con este endpoint:

1. **Crear un trabajo de cumplimiento**
   Puedes especificar el tipo de trabajo (con el valor tweets o users) para indicar si el conjunto de datos que deseas cargar contiene IDs de publicaciones o IDs de usuarios. Puedes tener un trabajo concurrente por tipo de trabajo en cualquier momento.
2. **Cargar tu conjunto de datos en el upload\_url**
   A continuación, carga tu conjunto de datos como un archivo de texto plano en el upload\_url proporcionado, con cada línea del archivo conteniendo un único ID de publicación o ID de usuario. El upload\_url expira a los 15 minutos.
3. **(Opcional) Consultar el estado del trabajo**
   Puedes consultar el estado de tu trabajo de cumplimiento para ver si está en created, in\_progress, failed o complete.
4. **Descargar los resultados**
   Cuando tu trabajo esté completo, podrás descargar los resultados usando el download\_url. El download\_url expira después de una semana (desde la creación del trabajo).

   Este resultado contendrá un conjunto de objetos JSON (un objeto por línea). Cada objeto contendrá un ID de publicación, la fecha de creación de la publicación (útil para localizar publicaciones organizadas por fecha), la acción requerida, el motivo de la acción de cumplimiento y la fecha en que el usuario fue suspendido.

Recibirás los siguientes tipos de eventos de cumplimiento en tus resultados:

- deleted: indica que la publicación o la cuenta de usuario fue eliminada
- deactivated: indica que la publicación o la cuenta de usuario ha sido desactivada
- scrub\_geo: indica que se eliminó la información de geolocalización asociada con la publicación o el usuario
- protected: indica que la cuenta que realizó la publicación se volvió privada
- suspended: indica que la cuenta que realizó la publicación fue suspendida

**Configuración de la cuenta**

Para acceder a estos endpoints, necesitarás:

- Una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info) aprobada.
- Autenticarte usando las claves y los tokens de una [App de desarrollador](/es/resources/fundamentals/developer-apps) que se encuentra dentro de un [Proyecto](/es/resources/fundamentals/projects). 

Obtén más información sobre cómo obtener acceso a los endpoints de la API de X v2 en nuestra [guía de introducción](/es/x-api/getting-started/getting-access).
