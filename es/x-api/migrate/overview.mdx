<div id="what-is-the-x-api-v2">
  ## ¿Qué es la API de X v2?
</div>

La API de X v2 es ahora la API principal de X y es donde se concentran la inversión y la innovación del producto. Nos hemos aliado con desarrolladores para crear la próxima generación de la API de X y servir mejor a nuestra diversa comunidad de desarrolladores. Con base en sus comentarios, hemos rediseñado la API para responder a un conjunto más amplio de necesidades, incorporar nuevas funciones y endpoints, y mejorar la experiencia para desarrolladores.

La API de X v2 es ahora la API principal de X y es donde se concentran la inversión y la innovación del producto. En los últimos años, nos aliamos con desarrolladores y rediseñamos la API para responder a un conjunto más amplio de necesidades, incorporar nuevas funciones y endpoints, y mejorar la experiencia para desarrolladores. Seguimos comprometidos con la construcción de una plataforma abierta para desarrolladores y nos entusiasma ver lo que crearás con la API de X v2.

<div id="why-migrate">
  ## ¿Por qué migrar?
</div>

La API de X v2 se basa en una infraestructura moderna y más sostenible e incluye endpoints de reemplazo mejorados para la versión estándar v1.1 y los productos empresariales, además de incorporar funcionalidades totalmente nuevas. Recomendamos encarecidamente a los clientes de las API heredadas (v1.1 y enterprise) que comiencen a migrar a v2, ya que tenemos la intención de retirarlas eventualmente. Use la API de X para escuchar y analizar la conversación pública, interactuar con las personas en X e innovar.

En esta sección, analizaremos los endpoints y su funcionalidad.

<div id="discover-new-and-updated-response-objects">
  ### Descubra objetos de respuesta nuevos y actualizados
</div>

Los siguientes seis objetos de datos están disponibles con los endpoints de la v2:

| Objeto | Descripción |
|:--------|:-------------|
| [Tweet](/es/x-api/fundamentals/data-dictionary#tweet) | El objeto Tweet tiene una extensa lista de campos a nivel raíz, como `id`, `text` y `created_at`. Los objetos Tweet también son el objeto padre de varios objetos secundarios, incluidos `user`, `media`, `poll` y `place`. |
| [User](/es/x-api/fundamentals/data-dictionary#user) | El objeto de usuario contiene metadatos de la cuenta de X que describen al usuario de referencia. |
| [Spaces](/es/x-api/fundamentals/data-dictionary#space) | El objeto Space consta de campos como `state`, `host_id`, `is_ticketed` e incluso `lang`. |
| [Lists](/es/x-api/fundamentals/data-dictionary#list) | El objeto List contiene información básica sobre la lista solicitada, incluidos `description`, `member_count` y `owner_id`. |
| [Media](/es/x-api/fundamentals/data-dictionary#media) | Si un Tweet contiene contenido multimedia (por ejemplo, imágenes), puede solicitarse el objeto media usando el parámetro `media.fields`; incluye campos como `media_key`, `type`, `url`, `preview_image_url`, entre otros. |
| [Poll](/es/x-api/fundamentals/data-dictionary#poll) | Una encuesta incluida en un Tweet no es un objeto primario en ningún endpoint, pero puede encontrarse y expandirse dentro del objeto Tweet. |
| [Place](/es/x-api/fundamentals/data-dictionary#place) | El objeto place consta de campos como `place_id`, el objeto `geo`, `country_code` y más. Esta información puede usarse para identificar Tweets y analizarlos por ubicación. |

Obtenga más información sobre [cómo usar campos y expansiones](/es/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions).

<div id="flexibility-to-choose-which-objects-and-fields-you-receive">
  ### Flexibilidad para elegir qué objetos y campos recibes
</div>

Al realizar una solicitud a un endpoint GET, recibirás el objeto de datos principal correspondiente a ese endpoint, que incluirá un conjunto de campos predeterminados. Por ejemplo, el objeto Tweet entrega los campos `id` y `text` como valores predeterminados.

Si deseas recuperar campos adicionales con tu solicitud, deberás usar los parámetros [fields](/es/x-api/fundamentals/fields) y [expansions](/es/x-api/fundamentals/expansions). El parámetro expansions te permite recuperar objetos de datos relacionados, como el Tweet fijado de un usuario o un objeto de medios, mientras que los operadores de campos te permiten solicitar campos específicos dentro de los objetos devueltos más allá de los predeterminados.

A continuación se muestra una lista completa de expansiones que puedes solicitar con los diferentes endpoints de la API de X v2:

| Object / Resource | Available Expansions |
|:-------------------|:----------------------|
| Tweets            | `author_id`, `edit_history_tweet_ids`, `entities.mentions.username`, `in_reply_to_user_id`, `referenced_tweets.id`, `referenced_tweets.id.author_id`, `attachments.poll_ids`, `attachments.media_keys`, `geo.place_id` |
| Users             | `pinned_tweet_id` |
| Spaces            | `invited_user_ids`, `speaker_ids`, `creator_id`, `host_ids`, `topic_ids` |

Obtén más información sobre [cómo usar fields y expansions](/es/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions).

<div id="edit-tweets">
  ## Editar tuits
</div>

Los endpoints de la API de X v2 proporcionan metadatos de tuits editados. La función _Editar tuit_ se presentó por primera vez para pruebas entre empleados de X el 1 de septiembre de 2022. A partir de esa fecha, los tuits aptos pueden editarse durante 30 minutos y hasta 5 veces. Obtén más información sobre [Editar tuits](/es/x-api/fundamentals/edit-posts).
Con la API de X v2, un desarrollador puede determinar:

- Si un tuit era apto para edición en el momento de su creación. Algunos tuits, como los que tienen encuestas o los programados, no se pueden editar.
- Los tuits pueden editarse durante 30 minutos y hasta 5 veces. En los tuits editables puedes ver si aún queda tiempo para editar y cuántas ediciones más son posibles.
- Si estás viendo una versión editada de un tuit (en la mayoría de los casos, la API devolverá la versión más reciente de un tuit, a menos que se solicite una versión anterior específica por ID del tuit).
- Todo el historial de ediciones del tuit.
- La interacción atribuida a cada versión del tuit.

<div id="track-threaded-conversations">
  ## Seguimiento de conversaciones encadenadas
</div>

Un nuevo campo de tuit ayuda a identificar a qué hilo de conversación pertenece un tuit. Un ID de conversación es el ID del tuit que inició la conversación. Obtén más información sobre el [seguimiento de conversaciones](/es/x-api/fundamentals/conversation-id).

<div id="ready-to-migrate">
  ## Listo para migrar
</div>

Para usar los endpoints de la API de X v2, necesitará lo siguiente:

- [Una cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info)
- [Una App de desarrollador](https://developer.x.com/en/apps) creada dentro de un [Proyecto](es/resources/fundamentals/projects)
- [Claves y tokens](es/resources/fundamentals/authentication) de la App de desarrollador de ese Proyecto

Tenga en cuenta la importancia de usar claves y tokens de una App dentro de un Proyecto. Si usa claves y tokens de una App fuera de un Proyecto, no podrá realizar solicitudes a los endpoints de la API de X v2.

Una vez que tenga una cuenta de desarrollador, puede configurar todo lo anterior en el [portal para desarrolladores](https://developer.x.com/en/portal/petition/essential/basic-info).

<div id="authentication">
  ### Autenticación
</div>

Con la nueva API de Twitter, usarás dos patrones de autenticación diferentes, OAuth 1.0a User Context y OAuth 2.0 Bearer Token, para acceder a distintos endpoints. Cada uno cumple un propósito diferente al realizar solicitudes a los endpoints:

OAuth 1.0a User Context es necesario cuando haces una solicitud en nombre de un usuario de Twitter
OAuth 2.0 Bearer Token es necesario para realizar solicitudes en nombre de tu App de desarrollador

<div id="migrating-to-the-new-data-format">
  ## Migración al nuevo formato de datos
</div>

A medida que migre de v1.1 o enterprise a v2, es importante comprender que el formato en el que se entregan los datos ha cambiado significativamente. Hemos agregado campos nuevos, modificado el orden de los campos y, en algunos casos, eliminado elementos por completo.

Para obtener más información sobre estos cambios, estamos desarrollando una serie de guías que le ayudarán a mapear los campos del formato de datos previo a v2 con los campos más recientes y a describir cómo solicitar estos nuevos campos.

Puede obtener más información visitando la sección de [migración de formatos de datos](/es/x-api/migrate/data-format-migration) de este centro de migración o consultando nuestras guías específicas de formato de datos:

- [Formato nativo a API de X v2 (v1.1 estándar)](/es/x-api/migrate/data-format-migration#migrating-from-standard-v1-1s-data-format-to-v2)
- [Native Enriched a API de X v2 (enterprise)](/es/x-api/migrate/data-format-migration#migrating-from-native-enriched-data-format-to-v2)
