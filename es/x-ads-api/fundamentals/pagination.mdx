---
title: Paginación
---

Para admitir un mayor número máximo de campañas y la recuperación eficiente de todas las entidades asociadas a una cuenta, la API de Anunciantes ahora admite paginación en muchos endpoints GET. El mecanismo de paginación es fácil de usar y muy similar a la paginación basada en cursores de la API REST, como se describe en [Uso de cursores para navegar por colecciones](https://developer.x.com/en/docs/x-api/v1/pagination).

<div id="getting-started">
  ## Introducción
</div>

Para determinadas solicitudes GET, ahora se aceptan los parámetros de consulta `cursor` y `count`, ambos opcionales. Si una respuesta contiene más de `count` entidades, estos endpoints devolverán las primeras `count` entidades y una clave `next_cursor` en el JSON de la respuesta.

`GET https://ads-api.x.com/5/accounts/abc1/campaigns?count=50`

```json
{
  "data": \[...\],
  "next_cursor": "c-3yvu1pzhd3i7",
  "request": {...}
}
```

Para obtener el siguiente conjunto de resultados, agregue `cursor` a los parámetros de consulta:

`GET https://ads-api.x.com/5/accounts/abc1/campaigns?cursor=c-3yvu1pzhd3i7&count=50`

```json
{
  "data": \[...\],
  "next_cursor": "c-3w3zdyg8ywan",
  "request": {...}
}
```

<div id="going-deeper">
  ## Profundización
</div>

Para la mayoría de los endpoints, el valor máximo de `count` es **1.000**, el mínimo es **1** y el valor predeterminado es **200**.

El valor proporcionado por `next_cursor` siempre es una cadena y debe considerarse opaco; la implementación está sujeta a cambios. Si en la página actual del conjunto de resultados se devuelven menos de `count` entidades, el valor de `next_cursor` será `null`.

**Nota:** los endpoints de analítica no admiten este estilo de paginación. La paginación en los endpoints de estadísticas se realiza especificando períodos de tiempo. Consulte la documentación de esos endpoints individuales para obtener más información.
