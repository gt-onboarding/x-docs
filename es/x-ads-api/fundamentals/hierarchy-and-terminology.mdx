title: "Jerarquía y terminología de la API de Anuncios"

Para tener éxito con la API de Anuncios de X, es importante comprender cómo se relacionan entre sí las entidades en X Ads. Este tutorial repasa los fundamentos de la jerarquía de objetos de X Ads.

![Jerarquía de la API de Anuncios de X](/images/adsapi-heirarchy.png.twimg.1920.png)

<div id="ad-group-line-items">
  ## Grupo de anuncios (line items)
</div>

Los grupos de anuncios, conocidos como [line items](https://developer.x.com/content/developer-twitter/en/docs/tutorials/ads-api-hierarchy-terminology#line-items) en la API de Anuncios de X, se encuentran dentro de las campañas y se utilizan para la segmentación y la puja frente a un conjunto de usuarios de X. Los anunciantes promocionan publicaciones o contenido multimedia (p. ej., videos que se promocionan como anuncios In-stream) al asociarlos con un line item. Véase también Grupo de anuncios.

<div id="account-media">
  ## Medios de la cuenta
</div>

Una colección de recursos creativos que se han subido a una cuenta de anuncios como [medios de la cuenta](/es/x-ads-api/creatives#account-media), que pueden incluir [recursos de prerroll](/es/x-ads-api/campaign-management#video-views-preroll-objective) de video promocionado —también conocidos como anuncios in‑stream— e imágenes que pueden promocionarse en la [X Audience Platform](https://developer.x.com/content/developer-twitter/en/docs/ads/measurement/overview/twitter-audience-platform).

<div id="ads-accounts">
  ## Cuentas de anuncios
</div>

En la parte superior de la jerarquía de la API de Anuncios de X se encuentran las cuentas de anuncios. Una cuenta de anuncios puede ser gestionada por varios usuarios de X. @Furni, nuestra tienda de muebles de demostración, sería un ejemplo de un usuario que gestiona una cuenta de anuncios. Identificamos las cuentas de anuncios mediante un account\_id, que es una cadena alfanumérica en [base36](https://en.wikipedia.org/wiki/Base36).

En cada cuenta, almacenamos varios tipos de entidades que pueden reutilizarse en distintas campañas, como audiencias personalizadas que pueden usarse para la segmentación, medios de la cuenta (videos o imágenes) y tarjetas, que son creatividades enriquecidas que puedes agregar a una publicación.

<div id="amplify-video">
  ## Amplify Video
</div>

Al cargar un recurso de video, la entidad que lo sube puede definir una categoría para el video (p. ej., Tweet Video, Tweet GIF, Tweet Image y Amplify Video). La categoría Amplify Video es obligatoria cuando se intenta cargar un video que se publicará como un anuncio in-stream. Estos videos requieren un procesamiento especial para garantizar que se puedan publicar y, por lo tanto, se requiere la categoría de medio Amplify Video.

<div id="analytics">
  ## Analítica
</div>

Las métricas de analítica ayudan a socios y anunciantes a comprender el rendimiento del contenido que promocionan en X. Esto incluye información como impresiones, clics, reproducciones de video y gasto. Además, los socios y anunciantes pueden obtener métricas detalladas para distintos segmentos de las audiencias a las que llegan.

La API de Anuncios de X admite dos formas de recuperar métricas detalladas de rendimiento de campaña: de manera sincrónica y asincrónica.

Con las solicitudes de [analítica sincrónica](/es/x-ads-api/analytics#synchronous-analytics), las métricas solicitadas se devuelven en la respuesta. El endpoint sincrónico admite rangos de tiempo cortos y es ideal para consultar el rendimiento de campañas en tiempo real.

Con los endpoints de [analítica asincrónica](/es/x-ads-api/analytics#asynchronous-analytics), las métricas solicitadas están disponibles en un archivo de resultados descargable después de que el trabajo asociado haya terminado de procesarse. Los endpoints asincrónicos permiten solicitar métricas durante un período mucho más amplio, devolver datos segmentados y están diseñados para obtener volúmenes mucho mayores de datos, lo que es ideal para generar informes o completar históricos.

<div id="campaigns">
  ## Campañas
</div>

Cada campaña que ejecute el anunciante debe crearse bajo un instrumento de financiación existente. Una campaña permite definir elementos como el nombre, el calendario y el presupuesto de una campaña publicitaria. Siguiendo con el ejemplo de Furni, podríamos crear una campaña llamada “Furni home decor”.

Cabe señalar que una cuenta de anuncios está limitada a un máximo de 200 campañas activas; este límite puede aumentarlo el gerente de cuenta del anunciante. Una campaña se considera activa hasta que se alcanza su hora de finalización o se agota su presupuesto. Una campaña en pausa sigue considerándose una campaña activa a efectos de los límites de campaña.

<div id="cards">
  ## Tarjetas
</div>

Las tarjetas permiten ampliar visualmente un tuit para incluir una imagen, un video o elementos de interfaz de usuario. La API de Anuncios de X admite varios tipos de tarjetas que pueden usarse en tuits y luego promocionarse en campañas.

<div id="creatives">
  ## Creatividades
</div>

Los anunciantes pueden promocionar publicaciones en campañas que incluyen creatividades, como imágenes, GIF y video. Estas creatividades pueden asociarse con tarjetas de X que permiten adjuntar fotos, videos y experiencias multimedia enriquecidas a las publicaciones. Las creatividades también pueden adjuntarse directamente a las publicaciones. Por ejemplo, puedes incluir texto y un video (o hasta 3 imágenes) y crear una publicación promocionada.

Cada elemento de línea debe tener al menos una creatividad asociada para poder ejecutarse. Hay varias opciones de creatividades disponibles, incluidas Cuentas promocionadas para ganar nuevos seguidores, Creatividades de medios como una imagen o un video para elementos de línea en toda la Plataforma de Audiencia de X, o Publicaciones promocionadas.

<div id="funding-instruments">
  ## Instrumentos de financiación
</div>

También dentro de la cuenta de anuncios se encuentran los instrumentos de financiación, que son las distintas fuentes de fondos que el anunciante ha configurado para usar con los Anuncios de X. Estos van desde órdenes de inserción firmadas con X hasta una tarjeta de crédito vinculada a la cuenta.

<div id="line-items">
  ## Partidas
</div>

Dentro de una campaña, puedes tener uno o más grupos de anuncios, también conocidos como partidas en la API de Anuncios de X. Las partidas pueden considerarse como diferentes segmentos de una campaña, cada uno con su propio conjunto de segmentación y su propia pieza creativa. Las campañas que contienen múltiples partidas a menudo se denominan “campañas de grupos de anuncios”.

Uno de los beneficios de tener múltiples partidas dentro de una sola campaña es que puedes crear una mayor variedad de combinaciones de segmentación y creatividades.

En nuestra campaña “Furni home decor”, podríamos querer crear dos partidas: una podría ser “Compradores de vivienda en San Francisco” y la otra “Compradores de vivienda en Londres”.

Se permiten hasta 100 partidas por campaña, y una cuenta de anuncios está limitada a 4.000 partidas activas en total en todas las campañas.

<div id="mobile-app-promotion-map">
  ## Promoción de aplicaciones móviles (MAP)
</div>

La Promoción de aplicaciones móviles, o MAP, es un conjunto de productos que permite a los anunciantes promocionar apps móviles en X. Esto incluye instalaciones de apps, conversiones y reenganche.

<div id="mobile-app-conversion-tracking-mact">
  ## Seguimiento de conversiones en apps móviles (MACT)
</div>

La medición de la promoción de apps móviles en X permite a los anunciantes evaluar el rendimiento de las campañas publicitarias en X diseñadas para impulsar instalaciones u otras conversiones dentro de la app. Un socio de medición móvil de X permite al anunciante definir qué conversiones desea medir de las apps que promociona en X. <div id="media-library">
  ## Biblioteca de medios
</div>

Los endpoints de la Biblioteca de medios permiten gestionar imágenes, GIF y videos para cuentas de la API de Anuncios de X. Los recursos multimedia de la biblioteca pueden usarse en publicaciones y para crear cards. También pueden reutilizarse en múltiples creativos, lo que elimina la necesidad de cargar el mismo recurso varias veces.

<div id="promoted-accounts">
  ## Cuentas promocionadas
</div>

Las Cuentas promocionadas (campañas de seguidores) forman parte de Who to follow, que sugiere cuentas que las personas aún no siguen y que podrían resultarles interesantes. Las Cuentas promocionadas ayudan a presentar una variedad aún más amplia de cuentas que podrían gustarles a los usuarios.

Las Cuentas promocionadas permiten a los anunciantes promocionar su cuenta y hacer crecer su audiencia en X. Las campañas de seguidores se muestran en múltiples ubicaciones de la plataforma de X —incluidos la cronología de Inicio, Who to follow y los resultados de búsqueda—, y esta sugerencia aparece con la etiqueta Promocionado para distinguirla de otras cuentas recomendadas.

<div id="promotable-users">
  ## Usuarios promocionables
</div>

Cada cuenta de la API de Anuncios de X puede tener uno o más usuarios promocionables, es decir, usuarios que se pueden promocionar a través de la cuenta de anuncios. El usuario @Furni de Furni sería un ejemplo de un usuario promocionable COMPLETO en la cuenta de anuncios de Furni, lo que significa que podemos ejecutar todo tipo de campañas para @Furni y sus tuits. Solo puede haber un usuario promocionable COMPLETO por cuenta de anuncios.

Tenga en cuenta que también puede haber otros usuarios promocionables en una cuenta de anuncios. Por ejemplo, si quisiéramos promocionar los tuits de @Jack sobre productos de Furni, él se convertiría en un usuario promocionable solo para retuits en la cuenta, lo cual se indica con RETWEETS\_ONLY. Los anunciantes deben ponerse en contacto con su gerente de cuenta de X para obtener información sobre cómo agregar usuarios de retuit a su cuenta de anuncios.

<div id="promoted-video">
  ## Video promocionado
</div>

Los videos promocionados son videos dentro de Tweets promocionados pagados por anunciantes. Aparecen en la cronología de Inicio, en la parte superior de los resultados de búsqueda en X y en otras ubicaciones de la plataforma, y están claramente marcados como “Promocionado”.

<div id="scheduled-tweets">
  ## Tweets programados
</div>

Los Tweets programados permiten que un anunciante o usuario cree un Tweet para publicarlo en una fecha posterior. Además de poder crear y administrar estos Tweets, la API permite asociarlos a un elemento de línea para promocionarlos una vez que el Tweet se publique. Esto permite a los anunciantes preparar Tweets nativos y planificar las creatividades de sus campañas con antelación a cualquier iniciativa clave. Por ejemplo, preparar una creatividad de Tweet para que se publique inmediatamente tras el anuncio de un nuevo producto.

<div id="segmentation">
  ## Segmentación
</div>

La segmentación permite a los partners y a los anunciantes obtener métricas desglosadas por valores de segmentación específicos (como edad, dispositivos o idiomas) y solo está disponible a través de nuestros endpoints de analítica asíncrona. Para solicitar métricas segmentadas, use el parámetro de solicitud segmentation\_type. Para más detalles sobre las opciones de segmentación, consulte Métricas y segmentación.

<div id="tailored-audience-list">
  ## Audiencia personalizada - Lista
</div>

Las audiencias personalizadas a partir de listas son uno de los tres tipos de Audiencias personalizadas. Se crean al cargar un archivo que contiene sus propios datos de usuarios y clientes. Sus listas se hacen coincidir con usuarios en X para que pueda orientarlos directamente en sus campañas. Revise estas restricciones.

<div id="tailored-audience-web">
  ## Audiencia personalizada - Web
</div>

La Audiencia personalizada desde web, una de las tres clases de Audiencias personalizadas, es un conjunto de usuarios de X que han visitado tu sitio web. Para usar Audiencias personalizadas desde web, debes tener instalado nuestro píxel de X en tu sitio o utilizar uno de nuestros socios aceptados de gestión de etiquetas. Revisa estas restricciones.

<div id="targeting-criteria">
  ## Criterios de segmentación
</div>

Como mencionamos anteriormente, los criterios de segmentación están asociados a una línea de pedido. Ofrecemos una variedad de opciones de segmentación que puedes usar, desde ubicación, palabras clave y género hasta atributos más granulares, como comportamientos e intereses de los usuarios. Para la línea de pedido “Home Buyers in San Francisco”, podríamos segmentar por el comportamiento “home buying” junto con la ubicación “San Francisco”.

Para obtener información sobre los diferentes tipos de segmentación disponibles, visita la página Targeting Overview en nuestro sitio para desarrolladores.

<div id="twitter-object-nest-ton">
  ## Twitter Object Nest (TON)
</div>

Twitter Object Nest (TON) aloja contenido multimedia y diversos recursos cargados a través de los endpoints de Media Upload para usarlos como creatividades.
