<div id="how-to-get-access-to-the-ads-api">
  ## Cómo obtener acceso a la API de Anuncios
</div>

1. Regístrate para obtener una [cuenta de desarrollador](https://developer.x.com/en/apply-for-access).
2. Crea una [App de desarrollador](https://developer.x.com/app) y protege tu token.
3. Visita [ads.x.com/help](https://ads.x.com/help) para enviar una solicitud de acceso a la API de Anuncios para cada una de tus Apps de desarrollador.

<Note>**Nota:** Si ya estás desarrollando en la Plataforma para Desarrolladores de X y tienes una cuenta de desarrollador, pasa al paso tres.</Note>

<div id="step-one-signup-for-a-developer-account">
  ### Paso uno: Regístrese para obtener una cuenta de desarrollador
</div>

Para realizar solicitudes a cualquiera de los productos de la API de X, primero debe registrarse para obtener una cuenta de desarrollador. En el portal para desarrolladores, cree un Proyecto y una App de desarrollador. Esto le proporcionará un conjunto de credenciales que usará para autenticar todas las solicitudes a la API.

<Button href="https://developer.x.com/en/apply-for-access">
  Regístrese para obtener una cuenta de desarrollador
</Button>

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### Paso dos: Guarda la clave y los tokens de tu App y mantenlos seguros
</div>

Dentro de tu App de desarrollador, se te proporcionará un conjunto de API Keys (también conocidas como Consumer Keys). También tendrás la posibilidad de generar un conjunto de Access Tokens que pueden usarse para realizar solicitudes en nombre de tu cuenta personal de X, y un Bearer Token que puede usarse para autenticar endpoints que requieren el Bearer Token de OAuth 2.0. Dado que estas claves y tokens no expiran a menos que se regeneren, recomendamos crear variables de entorno o usar un gestor de contraseñas seguro.

Toma nota también de tu App ID, que se encuentra en la URL de los detalles de tu App. Esto facilitará solicitar acceso a la API de Anuncios de X en el siguiente paso.

<Note>
**Nota:** Tus claves y tokens solo se mostrarán una vez en el portal para desarrolladores, por lo que es importante que almacenes estas credenciales en tu sistema de gestión de contraseñas tan pronto como las generes.

Si extravías u olvidas las claves y tokens, tendrás que regenerarlos, lo cual crea nuevas claves y tokens e invalida las anteriores. Esto significa que tendrás que actualizar cualquier integración que hayas configurado con tus credenciales previas.

Obtén más información sobre nuestras [mejores prácticas de autenticación](https://developer.x.com/en/docs/authentication/guides/authentication-best-practices).
</Note>

<div id="step-three-apply-for-access-to-the-ads-api">
  ### Paso tres: Solicitar acceso a la API de Anuncios
</div>

En este punto, tendrá [acceso básico a la API de X](https://developer.x.com/en/docs/twitter-api/v1), pero no a la funcionalidad específica de la API de Anuncios de X. A continuación, deberá solicitar acceso y ser aprobado para la API de Anuncios. Visite [ads.x.com/help](https://ads.x.com/help) para enviar una solicitud de acceso a la API de Anuncios para cada una de sus apps de desarrollador.

<div id="access-tiers">
  ### Niveles de acceso
</div>

Como parte del proceso de solicitud, deberá especificar qué nivel de acceso requiere. Obtenga más información sobre los [permisos a nivel de App y a nivel de cuenta publicitaria](https://developer.x.com/en/docs/twitter-ads-api/obtaining-ads-account-access#levels-of-access).

<div id="conversion-only">
  #### Solo conversiones
</div>

Acceso a los endpoints de conversiones para móvil y web, con permisos de lectura y escritura.

<div id="standard-access">
  #### Acceso estándar
</div>

Acceso a los endpoints de Analytics, Gestión de campañas, Creatividades, Audiencias personalizadas y Conversiones, con permisos de lectura y escritura.

<Note>**Nota:** Después de que tu app sea aprobada para el acceso a la API de Anuncios de X, deberás [regenerar los tokens de acceso de usuario](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens) para realizar correctamente [solicitudes autenticadas a la API de Anuncios de X](https://developer.x.com/en/docs/twitter-ads-api/making-authenticated-requests).</Note>

<Note>**Nota:** Los desarrolladores de la API de Anuncios de X que solicitaron acceso antes de julio de 2023 pueden tener distintos niveles de acceso y permisos, y pueden estar limitados a cinco tokens OAuth. Consulta nuestra guía sobre [cómo aumentar el acceso](https://developer.x.com/en/docs/twitter-ads-api/increasing-access) para habilitar el acceso a endpoints adicionales o eliminar los límites de tokens en aplicaciones existentes.</Note>

<div id="make-your-first-request">
  ## Realiza tu primera solicitud
</div>

Para probar tu acceso a la API de Anuncios, haz una solicitud al endpoint [GET accounts](https://developer.x.com/en/docs/twitter-ads-api/campaign-management/api-reference/accounts). Este endpoint devolverá las cuentas publicitarias a las que el usuario autorizado actualmente tiene acceso. Usa los ID de cuenta publicitaria de esta respuesta en solicitudes posteriores a la API para leer y escribir datos de una cuenta publicitaria específica. Con [Twurl](https://developer.x.com/en/docs/tutorials/using-twurl) en la línea de comandos, la solicitud sería la siguiente.

**Solicitud de ejemplo**

```bash
twurl -H ads-api.x.com "/11/accounts"
```
