<div id="requesting-additional-app-level-permissions">
  ### Solicitar permisos adicionales a nivel de App
</div>

Si tu aplicación está limitada a cualquiera de los siguientes permisos a nivel de App y deseas acceder a endpoints adicionales, comunícate con tu representante de X para solicitar **Acceso estándar**.

Los desarrolladores que solicitaron acceso a la API de Anuncios de X antes de julio de 2023 pudieron haber solicitado cualquiera de los siguientes permisos a nivel de App, que ahora están incluidos en
el nivel de Acceso estándar:

- **Analytics** (solo lectura): El usuario tiene acceso a los endpoints de [Analytics](https://developer.x.com/en/docs/twitter-ads-api/analytics/overview) con acceso de [solo lectura](https://developer.x.com/en/docs/apps/app-permissions). El usuario no puede acceder a los endpoints de Campaign Management, Creative o Custom Audiences, pero podrá obtener métricas pasando entidades creativas en los parámetros de la solicitud de los endpoints de Analytics. Para obtener la granularidad de media_id o media_name, la aplicación del usuario necesitaría tener acceso de escritura a la API de Anuncios de X.

- **Campaign Management & Creative** (lectura y escritura): El usuario tiene acceso a los endpoints de [Campaign Management](https://developer.x.com/en/docs/twitter-ads-api/campaign-management/overview) y [Creative](https://developer.x.com/en/docs/twitter-ads-api/creatives/overview) con acceso de [lectura y escritura](https://developer.x.com/en/docs/apps/app-permissions). El usuario también tiene acceso a los endpoints de Analytics.

- **Custom Audiences** (lectura y escritura): El usuario tiene acceso a los endpoints de [Custom Audiences](https://developer.x.com/en/docs/twitter-ads-api/audiences/overview)
con acceso de [lectura y escritura](https://developer.x.com/en/docs/apps/app-permissions). El usuario también tiene acceso a los endpoints de Campaign Management, Creative y Analytics.

- **Conversion** (lectura y escritura): El usuario tiene acceso a los endpoints de [Mobile](https://developer.x.com/en/docs/twitter-ads-api/measurement/mobile-conversions/overview) y [Web Conversion](https://developer.x.com/en/docs/twitter-ads-api/measurement/web-conversions/overview) con acceso de [lectura y escritura](https://developer.x.com/en/docs/apps/app-permissions). Es posible que el usuario tenga o no acceso a otros endpoints de la API de Anuncios de X. 

<div id="requesting-an-increase-to-user-oauth-token-limits">
  ### Solicitar un aumento de los límites de tokens OAuth de usuario
</div>

Los desarrolladores que solicitaron acceso a la API de Anuncios de X antes de julio de 2023 pueden tener un límite de 5 tokens OAuth de usuario en su aplicación. Si necesita aumentar el límite de tokens de su App de desarrollador, comuníquese con su representante de X.

<Note>**Nota:** No aumentamos los [límites de solicitudes](https://developer.x.com/en/docs/twitter-ads-api/rate-limiting) de ningún endpoint de la API de Anuncios de X. Consulte nuestras [mejores prácticas](https://developer.x.com/en/docs/twitter-ads-api/rate-limiting) para saber cómo mantenerse dentro de esos límites.</Note>
