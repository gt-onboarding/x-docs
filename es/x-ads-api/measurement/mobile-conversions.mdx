<div id="mact-overview">
  ## Descripción general de MACT
</div>

La medición de la [promoción de aplicaciones móviles](https://biz.x.com/mobile-app-promotion) en X permite a los anunciantes evaluar el éxito de las campañas publicitarias en X diseñadas para impulsar instalaciones u otras conversiones dentro de la aplicación. Un socio de medición móvil de X permite al anunciante gestionar qué conversiones desea medir de las aplicaciones que promociona en X. Una vez que el anunciante ha configurado el tipo de eventos y las ventanas temporales que desea medir, el socio de medición móvil enviará todos esos eventos a X y recibirá de inmediato datos de atribución que indican la campaña y el elemento creativo con el que interactuó el usuario y que impulsó la conversión.

<div id="conversion-events">
  ### Eventos de conversión
</div>

Para reportar un [evento de conversión](https://developer.x.com/en/docs/x-ads-api/measurement/api-reference/conversion-event) a X y obtener datos de atribución, el socio debe enviar el ID de la app móvil, el tipo y la marca de tiempo de la conversión, así como un valor con hash del ID publicitario/del dispositivo. Al proporcionar estos datos, se registrará el evento de conversión en la plataforma de X y se devolverá la atribución cuando corresponda. Además, el anunciante puede optar por proporcionar metadatos adicionales sobre el evento de conversión, como el valor de una compra, el ID de un producto o el método de registro.

#### POST conversion\_event

Registra un evento de conversión de medición móvil. La respuesta indicará la atribución de X o de X Audience Platform (TAP).

Esto está relacionado con el endpoint [GET conversion\_attribution](https://developer.x.com/en/docs/x-ads-api/measurement/api-reference/conversion-attribution#get-conversion-attribution).

En la respuesta se indicará atribución de X, de TAP o ninguna. El nodo `twitter_attribution` siempre estará presente y tendrá el valor `null` cuando no haya atribución de X (vea el ejemplo de respuesta más abajo). Si se reclama la atribución de TAP, se incluirá un nodo `tpn_attribution` con los datos correspondientes. Consulte la [descripción general de TAP](https://developer.x.com/en/docs/x-ads-api/measurement/overview/twitter-audience-platform) para obtener más información.

Hay varios parámetros opcionales disponibles para establecer metadatos asociados a cada evento de conversión. Estos metadatos no afectan los cálculos de atribución.

<div id="resource-url">
  ##### URL del recurso
</div>

`https://ads-api.x.com/12/conversion_event`

<div id="parameters">
  ##### Parámetros
</div>

| Nombre | Descripción |
| :--- | :--- |
| app\_id  <br />_obligatorio_ | El identificador único en la tienda de aplicaciones correspondiente.<br /><br />Tipo: int, string<br /><br />Ejemplo: `333903271`, `com.vine.android` |
| conversion\_time  <br />_obligatorio_ | La marca de tiempo del evento de conversión en formato ISO-8601, con milisegundos incluidos.<br /><br />Tipo: string<br /><br />Ejemplo: `2014-05-22T02:38:28.103Z` |
| conversion\_type  <br />_obligatorio_ | El tipo de evento de conversión.<br /><br />Tipo: enum<br /><br />Valores posibles: `PURCHASE`, `SIGN_UP`, `INSTALL`, `RE_ENGAGE`, `UPDATE`, `TUTORIAL_COMPLETE`, `RESERVATION`, `ADD_TO_CART`, `ADD_TO_WISHLIST`, `LOGIN`, `CHECKOUT_INITIATED`, `SEARCH`, `LEVEL_ACHIEVED`, `ACHIEVEMENT_UNLOCKED`, `CONTENT_VIEW`, `SHARE`, `INVITE`, `ADDED_PAYMENT_INFO`, `SPENT_CREDITS`, `RATED` |
| hashed\_device\_id  <br />_obligatorio_ | El IDFA o AdID con hash HMAC\_SHA-256.<br /><br />Tipo: string<br /><br />Ejemplo: `ABCD1234XYZ` |
| os\_type  <br />_obligatorio_ | El tipo de sistema operativo de la app.<br /><br />Tipo: enum<br /><br />Valores posibles: `IOS`, `ANDROID` |
| click\_window  <br />_opcional_ | La ventana de clic para este evento, en días.<br /><br />Tipo: int<br /><br />**Nota**: `click_window` debe ser mayor o igual que view\_through\_window.<br /><br />Predeterminado: `14`  <br />Valores posibles: `1`, `7`, `14`, `30` |
| device\_ip\_address  <br />_opcional_ | Dirección IPv4 o IPv6 del dispositivo cuando ocurrió el evento de conversión.<br /><br />Tipo: string<br /><br />Ejemplo: `192.133.78.1` |
| level  <br />_opcional_ | Un nivel asociado con este evento.<br /><br />Tipo: int<br /><br />Ejemplo: `2` |
| non\_twitter\_engagement\_time  <br />_opcional_ | La marca de tiempo de la última interacción fuera de Twitter previa a la conversión.<br /><br />Tipo: string<br /><br />Ejemplo: `2014-05-22T02:38:28.103Z` |
| non\_twitter\_engagement\_type  <br />_opcional_ | El tipo de interacción fuera de Twitter previa al evento de conversión.<br /><br />Tipo: enum<br /><br />Valores posibles: `CLICK`, `VIEW` |
| number\_items  <br />_opcional_ | Número de artículos asociados con este evento.<br /><br />Tipo: int<br /><br />Ejemplo: `2` |
| price\_currency  <br />_opcional_ | Código ISO 4217 que indica la moneda asociada con este evento.<br /><br />Tipo: string<br /><br />Ejemplos: `EUR`, `USD`, `JPY` |
| price\_micro  <br />_opcional_ | Importe de precio asociado a este evento en micro-unidades.<br /><br />Tipo: int<br /><br />Ejemplo: `123450000` |
| user\_payment\_info  <br />_opcional_ | Valor booleano que indica si la información de pago del usuario está almacenada en la app asociada con este evento.<br /><br />Tipo: bool<br /><br />Valores posibles: `true` o `false` |
| view\_through\_window  <br />_opcional_ | La ventana de visualización para este evento, en días.<br /><br />**Nota**:<br /><br />`click_window` debe ser mayor o igual que view\_through\_window.<br /><br />Tipo: int<br /><br />Predeterminado: `1`  <br />Valores posibles: `0`, `1`, `7`, `14`, `30` |

#### GET accounts/:account\_id/app\_event\_tags

Recupera los detalles de algunas o todas las etiquetas de eventos de la app asociadas con la cuenta actual.

Estas etiquetas definen las ventanas de conversión configuradas para cada tipo de conversión en el seguimiento de conversiones de aplicaciones móviles.

<div id="resource-url">

</div>

#### GET conversion\_attribution

Consulta a X para verificar la atribución de conversiones sin registrar un evento de conversión. La respuesta indicará la atribución de X.

Esto está relacionado con el endpoint [POST conversion\_event](https://developer.x.com/en/docs/x-ads-api/measurement/api-reference/conversion-event#post-conversion-event).

En la respuesta se informará atribución de X, de TAP o ninguna. El nodo `twitter_attribution` siempre estará presente y tendrá el valor `null` cuando no haya atribución de X; en caso contrario, se poblará como en el ejemplo de respuesta siguiente. Si se declara atribución de TAP, habrá un nodo `tpn_attribution` presente y se poblará en consecuencia. Consulta la [introducción a TAP](https://developer.x.com/en/docs/twitter-ads-api/measurement/twitter-audience-platform/overview) para más información.
