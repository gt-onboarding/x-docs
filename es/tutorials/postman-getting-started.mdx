<div id="introduction">
  ## Introducción
</div>

Postman es una aplicación de escritorio y web que permite realizar solicitudes a una API desde una interfaz gráfica. Recomendamos usar Postman con la API de X, la API de Anuncios de X y los endpoints de Labs al explorar la funcionalidad de las API, así como al solucionar problemas en su aplicación.

Actualmente, hay dos colecciones de Postman disponibles:

<div id="prerequisites">
  ### Requisitos previos
</div>

Antes de comenzar a usar las colecciones de Postman de X, asegúrese de contar con el acceso y las credenciales adecuados para la herramienta de la plataforma de desarrolladores de X que elija. Puede obtener más información sobre cómo obtener acceso en nuestra [página “Introducción”](/es/resources/platform-overview).

Una vez que tenga una [cuenta de desarrollador](https://developer.x.com/en/portal/petition/essential/basic-info), haya configurado una [App de desarrollador](/es/resources/fundamentals/developer-apps) y disponga de un conjunto de claves y tokens de [autenticación](/es/resources/fundamentals/authentication), y haya configurado correctamente su entorno para realizar solicitudes a la API específica que planea usar, puede seguir los pasos a continuación para comenzar.
 

<div id="getting-started-with-xs-postman-collections">
  ## Introducción a las colecciones de Postman de X
</div>

<div id="step-one-add-one-of-the-x-postman-collections-to-your-account">
  ### Paso uno: Añade una de las colecciones de Postman de X a tu cuenta
</div>

Aunque podrías configurar manualmente los endpoints específicos que quieras usar en Postman, nosotros ya hicimos ese trabajo por ti y preparamos una colección lista para usar con las APIs relevantes. Solo haz clic en uno de los enlaces de la sección anterior “Colecciones de Postman” y se añadirá a tu app de Postman una colección con todos los endpoints asociados a la API seleccionada. Estas colecciones también están disponibles en la [red de API de Postman](https://explore.postman.com/). Cada endpoint incluirá automáticamente los parámetros disponibles, respuestas de ejemplo y el tipo de autenticación correspondiente, por lo que solo necesitas añadir tus credenciales y los valores de los parámetros para empezar a explorar la funcionalidad.

En este ejemplo, vamos a trabajar con la [colección de la API de X v2](https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400). 
 

<div id="step-two-add-your-keys-and-tokens-as-environmental-variables">
  ### Paso dos: Agrega tus claves y tokens como variables de entorno
</div>

Cuando agregues la colección a tu instancia de Postman, se añadirá automáticamente un entorno llamado “X API v2”, en el que tendrás que añadir tus claves y tokens. En este paso, te guiaremos por el proceso de añadir tus claves y tokens desde tu App de desarrollador al entorno “X API v2”.

Para añadir tus claves y tokens al entorno “X API v2”, haz clic en el botón “manage environments” en la esquina superior derecha de Postman. También puedes hacer clic en el botón de configuración en la esquina superior derecha.

<Frame>
  <img src="/images/using-postman-1.png.twimg.1920.png" alt="Esta imagen muestra el botón «manage environments» resaltado en la consola de Postman." />
</Frame>

En la lista de entornos, haz clic en “X API v2”.

A continuación, agrega una variable para cada una de las diferentes claves y tokens que has generado en el panel de Apps, dentro de la tabla. Aquí tienes un ejemplo de cómo debería verse tu tabla:

| VARIABLE | VALOR INICIAL | VALOR ACTUAL |
| :--- | :--- | :--- |
| consumer\_key | `QAktM6W6DF6F7XXXXXX` | `QAktM6W6DF6F7XXXXXX` |
| consumer\_secret | `AJX560A2Omgwyjr6Mml2esedujnZLHXXXXXX` | `AJX560A2Omgwyjr6Mml2esedujnZLHXXXXXX` |
| access\_token | `1995XXXXX-0NGqVhk3s96IX6SgT3H2bbjOPjcyQXXXXXXX` | `1995XXXXX-0NGqVhk3s96IX6SgT3H2bbjOPjcyQXXXXXXX` |
| token\_secret | `rHVuh7dgDuJCOGeoe4tndtjKwWiDjBZHLaZXXXXXX` | `rHVuh7dgDuJCOGeoe4tndtjKwWiDjBZHLaZXXXXXX` |
| bearer\_token | `AAAAAAAAAAAAAAAAAAAAAL9v6AAAAAAA99t03huuqRYg0mpYAAFRbPR3XXXXXXX` | `AAAAAAAAAAAAAAAAAAAAAL9v6AAAAAAA99t03huuqRYg0mpYAAFRbPR3XXXXXXX` |

Ten en cuenta que las claves y tokens usados en la tabla anterior son inventados y no funcionarán si se usan en una solicitud.

Una vez que hayas agregado tus credenciales como variables y te hayas asegurado de que el entorno “X API v2” esté seleccionado, estarás listo para realizar solicitudes a la colección de la API de X v2. Esto se debe a que cada una de las pestañas de autorización de los endpoints está configurada para heredar automáticamente las variables de este entorno.

Para usar Postman con tokens de acceso de usuario, salta a XXXX para ver detalles adicionales sobre cómo hacerlo.

### Paso tres: Seleccionar un endpoint 

El siguiente paso es elegir un endpoint de la colección y empezar a construir su solicitud. Puede seleccionar un endpoint desde la navegación del lado derecho. Así es como se ve:

<Frame>
  <img src="/images/using-postman-2.png.twimg.1920.png" alt="Esta imagen muestra la solicitud «Single Posts» seleccionada en el menú desplegable «Post Lookup» de la sección «API de X v2»." />
</Frame>

Para este ejemplo, usaremos el endpoint API de X v2 > Post Lookup > Single Post. 

<div id="step-four-add-values-to-the-params-tab">
  #### Paso cuatro: agregar valores en la pestaña Params
</div>

El siguiente paso es ir a la pestaña Params. Deberías ver un conjunto de parámetros inactivos con descripciones que explican qué hace cada parámetro y una lista de todos los valores posibles que puedes pasar en tu solicitud. 

En este ejemplo, vamos a activar los parámetros de consulta expansions y tweet.fields y a agregar los siguientes valores:

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `tweet.fields` | `created_at,attachments` |
| expansions | author\_id |

Además de agregar los parámetros de consulta, debemos añadir la variable de ruta obligatoria, id. Como este endpoint devuelve publicaciones, debemos agregar un ID de publicación válido como valor.

Puedes encontrar el ID de la publicación navegando a x.com, haciendo clic en una publicación y luego mirando la URL. Por ejemplo, el ID de la publicación en la siguiente URL es `1228393702244134912`:

`https://x.com/XDevelopers/status/1228393702244134912`

En la pestaña Params, desplázate más allá de todos los parámetros de consulta hasta la sección “Path Variables”. Agregaremos el ID de la publicación que te gustaría usar como valor de la clave id.

Si ingresaste correctamente todo lo de este paso, la pestaña Params debería verse como lo siguiente:

<div id="step-five-send-your-request-and-review-your-response">
  #### Paso cinco: Envía tu solicitud y revisa la respuesta
</div>

Ahora que todo está configurado en tu solicitud, puedes hacer clic en el botón “Send”.

Si todo se configuró correctamente, deberías recibir la siguiente carga útil:


<div id="generating-a-user-access-token-with-postman">
  ### Generación de un token de acceso de usuario con Postman:
</div>

<div id="using-oauth-10a-to-generate-a-user-access-token">
  #### Uso de OAuth 1.0a para generar un token de acceso de usuario
</div>

Revise el proceso de tres pasos que se utiliza en la [colección de prueba del flujo OAuth 1.0a](https://www.postman.com/xapidevelopers/workspace/twitter-s-public-workspace/collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400?ctx=documentation).

<div id="using-oauth-20-to-generate-a-user-access-token">
  #### Uso de OAuth 2.0 para generar un token de acceso de usuario
</div>

Si desea generar un token de acceso de usuario de OAuth 2.0 en Postman, puede generar tokens de acceso de OAuth 2.0 para usarlos junto con la [colección de Postman de la API de X v2](https://www.postman.com/xapidevelopers/workspace/twitter-s-public-workspace/collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400?ctx=documentation).

Haga clic en la colección en su espacio de trabajo, vaya a la pestaña “Auth” y seleccione el tipo “OAuth 2.0”. Luego, en el encabezado “Configure New Token”, vaya a “Configuration Options”. Puede actualizar el “Grant Type” a “Authorization Code (With PKCE)”.

Actualice su Callback URL para que coincida con la URL de retorno asociada con su aplicación. Además, actualice los siguientes parámetros:

- Auth URL - https://x.com/i/oauth2/authorize
- Access Token URL - https://api.x.com/2/oauth2/token
- Client ID - Su client ID de OAuth 2.0 del Dev Portal
- Client Secret - Si utiliza un cliente confidencial
- Update Scope - Ámbitos que coincidan con los endpoints a los que desea conectarse. Ejemplo: “tweet.read users.read”
- Su callback URL (también conocida como redirect URL). Debe coincidir con lo que tiene en la configuración de autenticación de su App.
- State - state

Cuando esté listo, haga clic en “Get New Access Token” para generar un token de acceso. Si ve un cuadro de diálogo que indica que algo salió mal, tal vez deba presionar el botón Atrás para iniciar sesión. Deberá autorizar que su app tenga acceso a su cuenta haciendo clic en “Authorize app” en el cuadro de diálogo.

Después de autorizar su app, será redirigido a Postman, donde podrá ver su token y seleccionar el botón “Use Token” para comenzar a realizar solicitudes en nombre de un usuario autorizado.

Ahora está listo para usar la colección de Postman.
