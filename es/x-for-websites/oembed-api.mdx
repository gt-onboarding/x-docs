Puede usar la API oEmbed para obtener contenido insertado de forma programática, como [Posts](https://developer.x.com/en/docs/twitter-for-websites/embedded-tweets/overview) y [cronologías](https://developer.x.com/en/docs/twitter-for-websites/timelines/overview). 

La respuesta de la API oEmbed devolverá un fragmento de HTML que se reconocerá automáticamente cuando [el JavaScript del widget de X esté incluido en la página](https://developer.x.com/web/javascript/loading).

Tenga en cuenta que se recomienda usar la API para ejecutar tareas por lotes, y le aconsejamos utilizar nuestra sólida herramienta [publish.x.com](https://publish.x.com/#) para insertar contenido.

<Tabs>
  <Tab title="Embedded timelines">
    El fragmento de HTML devuelto se reconocerá automáticamente como una [línea de tiempo incrustada](https://developer.x.com/en/docs/twitter-for-websites/timelines/overview) cuando [se incluya en la página el JavaScript del widget de X](https://developer.x.com/web/javascript/loading).

    El endpoint oEmbed permite personalizar la apariencia final de una línea de tiempo incrustada estableciendo las propiedades correspondientes en el marcado HTML, que serán interpretadas por el JavaScript de X incluido de forma predeterminada con la respuesta HTML. El formato del marcado devuelto puede cambiar con el tiempo a medida que X agrega nuevas funciones o ajusta su representación de la línea de tiempo.

    Para una línea de tiempo de X especificada por la URL de la línea de tiempo, en un formato JSON compatible con [oEmbed](https://oembed.com/). Se admiten líneas de tiempo de usuario y de listas. El marcado de la línea de tiempo está pensado para almacenarse en caché en sus servidores durante un máximo del tiempo de vida sugerido especificado por la propiedad cache\_age.

    ## Resource URL

    _https://publish.x.com/oembed_

    ## Resource Information

    |     |     |
    | :--- | :--- |
    | Formatos de respuesta | JSON |
    | ¿Requiere autenticación? | No  |
    | Límite de tasa | No  |

    ## Parameters

    | Nombre | Descripción | Ejemplo |
    | :--- | :--- | :--- |
    | **url** | La URL de la línea de tiempo de X que se va a incrustar | \*   https://x.com/TwitterDev<br /> _https://x.com/TwitterDev/lists/national-parks
    | limit | Mostrar hasta N elementos, donde N es un valor entre 1 y 20 inclusive | 6   |
    | maxwidth | Establece el ancho máximo del widget. Debe estar entre 180 y 1200 inclusive | 300 |
    | maxheight | Establece la altura máxima del widget. Debe ser mayor que 200 | 400 |
    | omit\_script | No incluir un elemento script en la respuesta | 1   |
    | lang | Un [código de idioma](/es/en/docs/twitter-for-websites/twitter-for-websites-supported-languages/overview "Twitter language code") de X compatible | es  |
    | theme | Cuando se establece en dark, la línea de tiempo se muestra con texto claro sobre un fondo oscuro | dark |
    | chrome | Quitar un componente de visualización de la línea de tiempo con tokens separados por espacios<br /><br />_   noheader - oculta el encabezado<br />\*   nofooter - oculta el pie de página, si es visible<br />\*   noborders - elimina todos los bordes: alrededor del widget, entre los tuits y dentro de un tuit<br />\*   noscrollbar - recorta y oculta la barra de desplazamiento de la línea de tiempo, si es visible<br />\*   transparent - elimina el color de fondo | noheader%20nofooter |
    | aria\_polite | Establecer un valor asertivo de [cortesía de región viva ARIA](https://www.w3.org/TR/wai-aria/states_and_properties#aria-live) para los tuits agregados a una línea de tiempo | assertive |
    | dnt | Cuando se establece en true, la línea de tiempo y su página incrustada en su sitio no se utilizan para fines que incluyen [sugerencias personalizadas](https://support.x.com/articles/20169421) y [anuncios personalizados](https://support.x.com/articles/20170405) | true |

    ## Example Requests

    ```bash
    curl --request GET --url 'https://publish.x.com/oembed?url=https%3A%2F%2Ftwitter.com%2FInterior%2Fstatus%2F507185938620219395'
    twurl -H publish.x.com "/oembed?url=https://x.com/Interior/status/463440424141459456"
    ```

      "url": "https://x.com/TwitterDev",

      "provider_url": "https://x.com",

    }
    ```

  <Tab title="Embedded Tweets">
    El fragmento de HTML devuelto se reconocerá automáticamente como un [Tweet incrustado](https://developer.x.com/web/embedded-tweets) cuando [el JavaScript de widgets de X esté incluido en la página](https://developer.x.com/web/javascript/loading).

    El endpoint oEmbed permite personalizar la apariencia final de un Tuit insertado configurando las propiedades correspondientes en el marcado HTML, que el JavaScript de X incluido en la respuesta HTML interpretará de forma predeterminada. El formato del marcado devuelto puede cambiar con el tiempo a medida que X incorpora nuevas funciones o ajusta la representación del Tuit.

    El marcado de respaldo del tuit está diseñado para almacenarse en caché en sus servidores por un período de hasta la vida útil de caché recomendada, especificada por la propiedad `cache_age`.

    ## URL del recurso

    _https://publish.x.com/oembed_

    ## Información del recurso

    |     |     |
    | :--- | :--- |
    | Formatos de respuesta | JSON |
    | ¿Requiere autenticación? | No  |
    | ¿Tiene limitación de tasa? | No  |

    ## Parámetros

    | Nombre | Predeterminado | Descripción |
    | :--- | :--- | :--- |
    | `url` obligatorio  <br />String |     | La URL del Tweet que se va a incrustar |
    | `maxwidth`  <br />Int `[220..550]` | `325` | El ancho máximo de un Tweet renderizado en píxeles enteros. Si se proporciona un valor por debajo o por encima del rango permitido, se devolverá el ancho mínimo o máximo admitido, respectivamente; el valor de ancho reajustado se reflejará en la propiedad `width` de la respuesta. Tenga en cuenta que X no admite el parámetro oEmbed `maxheight`. Los Tweets son, fundamentalmente, texto y, por lo tanto, tienen una altura impredecible que no puede escalarse como una imagen o un video. En relación con esto, la respuesta de oEmbed no proporcionará un valor para `height`. Las implementaciones que necesiten alturas uniformes para los Tweets deben consultar los parámetros `hide_thread` y `hide_media` a continuación. |
    | `hide_media`  <br />Boolean, String o Int | `false` | Cuando se establece en `true`, `"t"` o `1`, los enlaces en un Tweet no se expanden a vistas previas de fotos, videos o enlaces. |
    | `hide_thread`  <br />Boolean, String o Int | `false` | Cuando se establece en `true`, `"t"` o `1`, no se mostrará una versión contraída del Tweet anterior en un hilo de conversación cuando el Tweet solicitado sea una respuesta a otro Tweet. |
    | `omit_script`  <br />Boolean, String o Int | `false` | Cuando se establece en `true`, `"t"` o `1`, no se devolverá la etiqueta `<script>` responsable de cargar `widgets.js`. Sus páginas web deben incluir su propia referencia a `widgets.js` para su uso en todos los widgets de X, incluidos los [Tweets incrustados](https://developer.x.com/web/embedded-tweets). |
    | `align`  <br />Enum `{left,right,center,none}` | `none` | Especifica si el Tweet incrustado debe flotar a la izquierda, a la derecha o centrarse en la página con respecto al elemento padre. |
    | `lang`  <br />Enum([Language](https://developer.x.com/en/docs/twitter-for-websites/twitter-for-websites-supported-languages/overview)) | `en` | Solicita el HTML de respuesta y un Tweet renderizado en el [idioma de X admitido por los Tweets incrustados](https://developer.x.com/web/overview/languages) especificado. |
    | `theme`  <br />Enum `{light, dark}` | `light` | Cuando se establece en `dark`, el Tweet se muestra con texto claro sobre un fondo oscuro. |
    | `dnt`  <br />Boolean | `false` | Cuando se establece en `true`, el Tweet y su página incrustada en su sitio no se utilizan para fines que incluyen [sugerencias personalizadas](https://support.x.com/articles/20169421) y [anuncios personalizados](https://support.x.com/articles/20170405). |

    ```
