<Update label="Aug 3, 2022" description="API de X v2">
  ### Adición de alias de operadores de filtrado

  Se añadieron alias de operadores de filtrado/consulta a los endpoints de búsqueda y flujo filtrado, en enterprise, premium y v2. En muchos casos, el operador se refiere a publicaciones en lugar de statuses, p. ej., "in\_reply\_to\_tweet\_id" como alias de "in\_reply\_to\_status\_id". En otros casos, el nuevo alias de operador aporta más claridad a la coincidencia realizada. Por ejemplo, dado que las publicaciones solo pueden tener un video adjunto, se proporciona un nuevo alias "has:video\_link" para "has:videos". Consulta la [tabla de operadores del flujo filtrado v2](https://developer.x.com/en/docs/twitter-api/tweets/filtered-stream/integrate/build-a-rule#list) para ver más ejemplos.
</Update>

<Update label="Dec 14, 2021" description="API de X v2: Authentication">
  ### Compatibilidad con autenticación OAuth 2.0

  A partir de hoy, todos los desarrolladores pueden autenticarse usando OAuth 2.0 seleccionando OAuth 2.0 como método de autenticación en el portal de desarrolladores. Hemos agregado compatibilidad para clientes confidenciales y públicos, y todos los endpoints relevantes de v2 pueden usar este método de autenticación como parte de este lanzamiento.

  OAuth 2.0 es un protocolo de autorización estándar del sector que brinda a los desarrolladores mayor control sobre los alcances de una aplicación y mejora los flujos de autorización en múltiples dispositivos. En otras palabras, los desarrolladores que crean aplicaciones para personas en X tendrán más control sobre la información que su app solicita a sus usuarios, de modo que solo tengas que pedir a tus usuarios finales los datos y la información que necesitas.

  Este protocolo de autorización moderno te permitirá presentar a tus usuarios finales un flujo de consentimiento más simplificado para autorizar tu app, que solo muestra los alcances específicos que les has solicitado.

  Para obtener más información sobre este lanzamiento, visita nuestro anuncio en el foro.
</Update>

<Update label="Nov 15, 2021" description="X API v2: Early Access">
  ### Actualizaciones importantes de la plataforma para la API de X v2

  **Actualizaciones importantes de la plataforma: la API de X v2 ahora es la versión principal, nuevos accesos Essential y Elevated, y cambios en la Política**

  Hoy anunciamos que la API de X v2 es ahora la versión principal de la API de X. Hemos lanzado suficientes endpoints y funcionalidades en la API de X v2 para satisfacer las necesidades del 90% de todas las aplicaciones existentes creadas sobre la API de X.

  Hoy lanzamos los siguientes cambios para seguir mejorando la experiencia de desarrollo con la API de X v2:

  - Acceso Essential rápido y gratuito a la API, y acceso Elevated gratuito para los desarrolladores cuyos casos de uso hayan sido aprobados. Tenemos muchos más detalles sobre estos niveles de acceso disponibles en nuestra página About the X API.

  - Eliminamos lenguaje en nuestra Developer Policy que restringía cómo construir con las funciones principales de X y limitaba la cantidad de usuarios que puedes admitir a través de tu aplicación.

    - Abrimos nuestra plataforma para animarte a crear herramientas y productos que hagan que X sea mejor, más saludable y que amplíen la conversación pública. [Consulta una lista de soluciones que nos encantaría que construyeras](https://developer.x.com/en/docs/twitter-api/what-to-build).

    - Específicamente, hemos eliminado términos que restringían la replicación de la experiencia de X, incluidas las funciones principales de X, así como términos que requerían permiso para tener un número alto de tokens de usuario.

    - Sabemos que crear soluciones que ayuden a las personas en X a menudo significa que un desarrollador tenga que construir (o replicar) algunas de las cosas que están disponibles en X. Estos cambios en nuestra Developer Policy están destinados a brindar claridad al ecosistema de desarrolladores y proporcionar una plataforma de API abierta que facilite a los desarrolladores crear, innovar y tener un impacto en la conversación pública.

  Entramos en mucho más detalle sobre este lanzamiento y los lanzamientos de la API de X que condujeron a este momento en nuestra [publicación en el foro](https://devcommunity.x.com/t/ushering-in-a-new-era-for-the-twitter-developer-platform-with-the-twitter-api-v2/162087).

  Además de explorar entradas anteriores del registro de cambios, hemos reunido algunos recursos clave que puedes revisar para entender todo lo que se ha lanzado en la API de X v2 hasta la fecha:

  - [Mapa de endpoints de la X API](/es/x-api/migrate/x-api-endpoint-map)

  - [Acerca de la X API](/es/x-api/getting-started/about-x-api)
</Update>

<Update label="May 10, 2018" description="X API: Standard v1.1">
  ### Cambios en la representación del payload de citas

  Hoy incorporamos un cambio en cómo se representarán las URL en el payload de la cita.

  Estamos añadiendo una nueva entidad llamada "quoted\_status\_permalink" para garantizar que se pueda hacer referencia a la publicación citada y eliminaremos el enlace t.co del campo "text" de la publicación citada.

  Con el nuevo formato, el enlace t.co ya no se agrega al campo "text" de la publicación citada en el objeto "quoted\_status".

  Además, ya no incluiremos en la entidad "urls" la url, expanded\_url y display\_url de t.co de la publicación citada. En su lugar, se añadirán a un nuevo objeto "quoted\_status\_permalink" con los atributos de URL "url", "expanded" y "display". Esto significa que el objeto "quoted\_status\_permalink" se reservará para el enlace t.co que redirige a la publicación citada, mientras que el array "entities.urls" puede usarse para identificar cualquier enlace compartido dentro de la publicación original o la cita.

  Lee nuestra [publicación en el foro](https://devcommunity.x.com/t/updating-how-urls-are-rendered-in-the-quote-tweet-payload/105473) y la [documentación actualizada](https://devcommunity.x.com/t/upcoming-changes-to-the-developer-platform/104603) para más detalles.

<Update label="Mar 9, 2017" description="X API: Standard v1.1">
  ### Retiro de la compatibilidad con video MPEG-DASH

  Las URL de MPEG-DASH (.mpd) se han eliminado del payload de una publicación que contiene video. Lee más sobre esta actualización [en nuestro foro](https://devcommunity.x.com/t/retiring-mpeg-dash-video-support-on-march-9th-2017/82761).
</Update>

<Update label="Feb 22, 2017" description="X API: Enterprise v1.1">
  ### Nuevos enriquecimientos y cambios en el payload

  ¡Nuevo enriquecimiento! Los metadatos de encuestas ahora están disponibles a través de nuestras API empresariales. Consulta la [documentación](/es/x-api/enterprise-gnip-2.0/fundamentals/account-activity#migration-introduction) para más detalles.

  Nuevo campo de payload: el campo de texto alternativo de imagen (description) puede estar presente en la sección entities.media\[] y extended\_entities.media\[] del payload si un usuario elige la opción "Add description" al agregar una foto a una publicación. Esto está disponible solo en formato nativo enriquecido.

  Formato predeterminado de payload: los flujos creados recientemente en productos de Gnip ahora tendrán como predeterminado el formato JSON original.
</Update>
