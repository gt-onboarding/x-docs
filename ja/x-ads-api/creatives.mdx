<div id="overview">
  ## 概要
</div>

クリエイティブとは、キャンペーンでプロモーションできるあらゆるエンティティを指します。投稿にはテキスト、画像、GIF、動画、またはカードを含めることができます。カードには画像または動画を含めることができます。

画像、GIF、または動画のクリエイティブは、[POST media/upload](/ja/x-api/media/upload-media#post-media-upload)（画像のみをサポートするシンプルなアップロードエンドポイント）または POST media/upload（chunked）エンドポイントのいずれかでアップロードできます。これらは次の対象に追加できます。

cards:

- POST accounts/:account\_id/cards
  Tweets:
- POST accounts/:account\_id/tweets - ツイートにカードを追加するには、card\_uri パラメータを使用します。
  Scheduled Tweets:
- POST accounts/:account\_id/scheduled\_tweets

カードの詳細は Cards ページをご覧ください。Promoted Video ページでは、動画をカードまたはツイートに関連付ける方法の詳細を説明しています。

<div id="cards">
  ### カード
</div>

X 広告 API では、キャンペーンでプロモーション可能なポストに使用できる複数のカードタイプをサポートしています。**注意**：一度ポストすると、カードの詳細は公開されます。カードの所有者であるユーザーに関する情報が含まれる場合があります。

<div id="image">
  #### 画像
</div>

以下の画像仕様は、[Cards](/ja/x-ads-api/creatives#cards-2) で使用されるアセットに適用されます。画像は 3MB 以下、かつ幅が 800px 以上である必要があります。さらに、以下の幅:高さのアスペクト比に対応しています。

- Website: 1:1 および 1.91:1
- Image App Download: 1:1 および 1.91:1
- Poll: 1.91:1
- Image Conversation: 1.91:1
- Image Direct Message: 1.91:1

対応している画像形式は .bmp、.jpeg、.png です。

<div id="video">
  #### 動画
</div>

以下の動画仕様は、[Cards](/ja/x-ads-api/creatives#cards-2) で使用するアセットに適用されます。サポートされるアスペクト比（幅:高さ）は次のとおりです。

- Video Website: 16:9 および 1:1
- Video App Download: 16:9 および 1:1
- Poll: 16:9
- Video Conversation: 16:9
- Video Direct Message: 16:9

<div id="promoted-video">
  ### プロモーテッドビデオ
</div>

このドキュメントでは、X 広告 API を用いて動画をアップロードし、配信（プロモーション）する手順の概要を簡潔に説明します。

X 広告 API は、[ツイート](/ja/x-ads-api/creatives#tweets-2)内および以下のカードでのプロモーテッドビデオをサポートします:

- [Video Website](https://devcommunity.x.com/t/ads-api-version-11/168814)
- [Video App Download](https://devcommunity.x.com/t/ads-api-version-11/168814)
- [Video Conversation](/ja/x-ads-api/creatives#video-conversation-cards)

まず、[POST media/upload (chunked)](https://developer.x.com/en/docs/media/upload-media/api-reference/post-media-upload-init.html) エンドポイントを使用して動画をアップロードします。続いて、`media_id` を使い、POST accounts/:account\_id/videos エンドポイントで動画を広告アカウントに関連付けます。動画の `id`（`media_key` と呼ばれることもあります）は後続のリクエストで使用します。これは先頭に整数、続いてアンダースコア、最後に長い値が続く文字列です。例: `13_875943225764098048`。

<div id="promoted-video-in-tweets">
  #### ツイート内のプロモーテッド動画
</div>

ツイートを作成するには、動画の `id` とともに [POST accounts/:account\_id/tweet](/ja/x-ads-api/creatives#get-accounts-account-id-tweets) エンドポイントを使用します。このステップでは、動画のタイトル、説明、CTA（コールトゥアクション）も指定できます。これらの値はユーザーに表示されます。

<div id="promoted-video-in-cards">
  #### カード内のプロモ動画
</div>

Video App Download と Video Conversation のカードでは、ポスター画像を追加できます。これらのカードで使用する画像は、[POST media/upload](https://developer.x.com/en/docs/media/upload-media/api-reference/post-media-upload.html) エンドポイントでアップロードします。

次のいずれかのエンドポイントを使用してカードを作成します:

- [POST accounts/:account\_id/cards/video\_website](https://devcommunity.x.com/t/ads-api-version-11/168814)
- [POST accounts/:account\_id/cards/video\_app\_download](https://devcommunity.x.com/t/ads-api-version-11/168814)
- [POST accounts/:account\_id/cards/video\_conversation](https://devcommunity.x.com/t/ads-api-version-11/168814)

動画の `id` と、必要に応じて画像の `media_id`（ポスター画像用）を指定します。

最後に、POST accounts/:account\_id/tweet エンドポイントでツイートを作成します。カードは `card_uri` パラメータを使用してツイートに添付します。

<div id="general-information">
  #### 一般情報
</div>

API 経由での動画アップロードの詳細な手順については、[Video Upload Guide](https://developer.x.com/en/docs/media/upload-media/uploading-media.html) を参照してください。

動画はプレロールアセットとして配信することも可能です。詳しくは [Video Views Pre-roll Objective Guide](/ja/x-ads-api/campaign-management#video-views-preroll-objective) を参照してください。

- （2015-10-22 時点）プロモーション用コンテンツで使用する動画をアップロードする際は、[POST media/upload (chunked)](https://developer.x.com/en/docs/media/upload-media/api-reference/post-media-upload-init.html) エンドポイントへのすべての `INIT` コマンドリクエストで、`media_category` パラメータに `amplify_video` を設定する必要があります。この新規パラメータを使用すると、動画が非同期に前処理され、プロモーション用コンテンツで使用できる状態に準備されます。アップロード後の非同期処理の完了確認には `STATUS` コマンドを使用できます。
- 現在許可されているプロモーション用動画の最大長は 10 分、ファイルサイズは 500MB 以下です。
- アップロードする動画は mp4 または mov 形式である必要があります。
- アップロードした動画は一般的に迅速に処理されますが、処理時間は動画の長さやファイルサイズによって変動します。
- アップロードするポスター画像は png または jpg 形式にしてください。アスペクト比やサイズの要件はありませんが、ポスター画像は動画プレーヤーに合わせて調整されます。

<div id="scheduled-tweets">
  ### 予約投稿
</div>

<div id="introduction">
  #### 概要
</div>

Scheduled Tweets は、広告主またはユーザーが、後日に公開するようスケジュールできる Tweet を作成できる機能です。これらの Tweet を作成・管理できるだけでなく、API により、公開後にプロモーションできるようラインアイテムに関連付けることも可能です。これにより、広告主はネイティブ Tweet を事前に用意し、重要な施策に先立ってキャンペーンクリエイティブを計画できます。たとえば、新製品の発表と同時に即時公開されるよう Tweet クリエイティブをステージングすることが可能です。

Scheduled Tweets API エンドポイントで提供される機能は以下のとおりです。

- 新規にスケジュールした Tweet の作成、変更、閲覧
- スケジュール済み Tweet をラインアイテムに関連付け
- 既存のスケジュール済み Tweet の照会および管理
- スケジュール済み Tweet が公開されたら、公開された Tweet の `id` を取得

</div>

  #### 予約投稿の表示
</div>

- [GET accounts/:account\_id/scheduled\_tweets/preview/:scheduled\_tweet\_id](/ja/x-ads-api/creatives#scheduled-tweets-2)（既存の予約投稿を表示）

予約投稿は「ライブ」投稿とは別個のエンティティであるため、これらの投稿を作成または編集する際には、2 種類の検証が行われます。最初の検証ルールは、予約投稿の作成時に実行されます。具体的には次のとおりです。

<div id="scheduled-tweet-create">
  #### 予約ツイートの作成:
</div>

- 認証済みユーザーが、指定された @handle に対してオーガニックツイートを作成できる権限を持っていることを検証します。プロモーション専用ツイートの作成権限には、認証済みユーザーが[ツイートコンポーザーの権限](https://developer.x.com/content/developer-twitter/en/docs/ads/campaign-management/api-reference/authenticated-user-access#get-accounts-account-id-authenticated-user-access)を持つアカウントユーザーであることが必要です。
- scheduled\_at で指定された時刻の前後15分のウィンドウ内に、作成予定のツイートが30件を超えないことを検証します。SCHEDULED\_TWEET\_LIMIT\_EXCEEDED エラーメッセージは、同一の将来の15分間の時間枠に予約ツイートが多すぎることを示します。広告主は既存の予約ツイートを削除するか、scheduled\_at の時刻を前倒しまたは後ろ倒しにする必要があります。

<div id="scheduled-tweet-goes-live">
  #### 予約投稿が「公開」されるタイミング:
</div>

- これらの検証ルールは scheduled\_at の時刻に実行され、API での通常の投稿作成時に適用されるものと同一です。たとえば、予約投稿に画像と GIF の両方が含まれている場合、公開は行われず、scheduled\_status は FAILED に設定されます

<div id="workflow">
  #### ワークフロー
</div>

**新しい予約投稿の作成**

新しい予約投稿は、[POST accounts/:account\_id/scheduled\_tweets](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/api-reference/scheduled-tweets#post-accounts-account-id-scheduled-tweets) エンドポイントで作成できます。メディアエンティティを含まない場合、必須パラメータは `scheduled_at`（予約時刻）と投稿本文の `text` です。加えて、このエンドポイントでは、`as_user_id` パラメータで別の @handle として予約投稿を作成したり、カード（`card_uri`）やメディア（`media_ids`）を追加したりできます。なお、投稿に含められるエンティティは同一タイプに限られます（Video、Gif、Image のいずれか）。`nullcast` パラメータは、その投稿が「Promoted-Only」かどうかを制御します。新規に作成される予約投稿は、デフォルトで「Promoted-Only」（`nullcast=true`）です。`nullcast=false` の場合は、オーガニックの予約投稿が作成されます。

予約投稿の作成に成功すると、レスポンスには予約投稿自体の一意の識別子である `id` フィールドが含まれます。これに加えて `tweet_id` フィールドも返されます。このフィールドは当初 `null` ですが、投稿が公開されると「ライブ」投稿の ID が設定されます。

```
twurl -H 'ads-api.x.com' -X POST "/6/accounts/:account_id/scheduled_tweets" -d 'scheduled_at=2017-12-24T23:59:00Z&text=Happy Holidays!'
```

次の予約投稿が作成されます:

```json
{
  "request": {
    "params": {
      "text": "Happy Holidays!",
      "scheduled_at": "2017-12-24T23:59:00Z"
    }
  },
  "data": {
    "completed_at": null,
    "id_str": "917507899668099072",
    "text": "Happy Holidays!",
    "user_id": "3271358660",
    "scheduled_status": "SCHEDULED",
    "id": 917507899668099100,
    "nullcast": true,
    "created_at": "2017-10-09T21:51:44Z",
    "scheduled_at": "2017-12-24T23:59:00Z",
    "card_uri": null,
    "updated_at": "2017-10-09T21:51:44Z",
    "tweet_id": null,
    "media_keys": []
  }
}
```

この予約投稿が公開されると、`tweet_id` フィールドに「ライブ」投稿の ID が設定されます。

**Scheduled Tweet をラインアイテムに関連付ける**

Scheduled Tweet はオーガニックツイートの作成にも使用できますが、パートナーは「Promoted-Only」（`nullcast=true`）のツイートを作成することもでき、いずれもラインアイテムに関連付けられます。これを可能にするために、[POST accounts/:account\_id/scheduled\_promoted\_tweets](https://developer.x.com/content/developer-twitter/en/docs/ads/campaign-management/api-reference/scheduled-promoted-tweets#post-accounts-account-id-scheduled-promoted-tweets) エンドポイントも提供しています。このエンドポイントでは、1 回の API 呼び出しで 1 件の Promoted Scheduled Tweet のみをラインアイテムに関連付けられます。同一のラインアイテムに複数の Scheduled Tweet を関連付けるには、複数回の API 呼び出しが必要です。

既存の Scheduled Promoted Tweet を変更することはできませんのでご注意ください。

```
twurl -H 'ads-api.x.com' -X POST "/6/accounts/:account_id/scheduled_promoted_tweets" -d 'line_item_id=a44qc&scheduled_tweet_id=917507899668099072'
```

```json
{
  "data": {
    "line_item_id": "a44qc",
    "id": "26576",
    "created_at": "2017-10-09T22:24:16Z",
    "updated_at": "2017-10-09T22:24:16Z",
    "scheduled_tweet_id": "917507899668099072",
    "tweet_id": null,
    "deleted": false
  },
  "request": {
    "params": {
      "line_item_id": "a44qc",
      "scheduled_tweet_id": 917507899668099100,
      "account_id": "aaaaa"
    }
  }
}
```

このエンドポイントは、指定した Scheduled Tweet とラインアイテムとの関連付けのみを作成します。キャンペーン／ラインアイテムのフライト期間が現在になると、そのラインアイテムは対応する「ライブ」ツイートの配信を自動的に開始します。このステップでは、Scheduled Tweet が `SCHEDULED` 状態であること、また指定の目的に対して有効であることは検証しますが、それ以外の検証は行いません。ラインアイテムおよび Scheduled Tweet に適用される残りの検証ルールは、ツイートが「ライブ」になる時点で実行されます。

キャンペーン配信に問題が生じないよう、Scheduled Tweet はキャンペーン／ラインアイテムのフライト期間より前の時刻に `scheduled_at` されていることを推奨します。

たとえば、Scheduled Tweet がキャンペーン開始日より後にライブになるよう設定されている（かつ 1 つのラインアイテムに 1 つのツイートのみが関連付けられている）場合、キャンペーンは `ACTIVE` になりますが、Scheduled Tweet がまだライブでないため、配信可能なクリエイティブは存在しません。

**Scheduled Tweet の管理**

残りの一連のエンドポイントにより、API 利用者はすべての Scheduled Tweet および Scheduled Promoted Tweet を管理できます。これらの API は、任意の状態でフィルタリング可能なすべての Scheduled Tweet の一覧を返すほか、`id` で特定の Scheduled Tweet を検索するためにも使用できます。

<div id="what-happens-when-a-scheduled-tweet-goes-live">
  #### 予約済みツイートが公開されるとどうなりますか？
</div>

特定の予約済みツイートが公開される直前、つまり `scheduled_at` の時刻になると、次の更新が行われます。

- 「公開中」のツイートが作成されますが、最大で1秒の遅延が発生する場合があります
- `tweet_id` が次のエンティティに追加されます:
- 予約済みツイート
- プロモーション用予約済みツイート
- 新しいプロモーションツイートのエンティティが作成されます

<div id="best-practices">
  #### ベストプラクティス
</div>

Scheduled Tweet を作成またはプロモーションする際は、次のベストプラクティスを推奨します。

- Scheduled Tweet を作成する際は、Tweet が有効であることを確認してください（例：Tweet には画像、動画、または GIF のいずれかのみを含めることができ、これらを組み合わせることはできません）
- キャンペーンのフライト日程（`start_time` と `end_time`）が Scheduled Tweet の `scheduled_at` に合致していることを確認してください
- Scheduled Tweet のスケジュールは将来日付で 1 年（365 日）を超えないようにしてください
- Scheduled Tweet の Tweet プレビューは現在サポートされていません（作成前に Scheduled Tweet をプレビューする機能）

  ### メディアライブラリ
</div>

<div id="introduction">

</div>

Media Library エンドポイントは、X 広告アカウント向けに画像、GIF、動画の管理機能を提供します。ライブラリ内のメディアアセットは、投稿（ツイート）やカードの作成に利用できます。さらに、同一の[クリエイティブ](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/overview)で繰り返し再利用できるため、同じアセットを何度もアップロードする必要がありません。

<div id="api-endpoints">

</div>

- [POST media/upload](https://developer.x.com/content/developer-twitter/en/docs/media/upload-media/api-reference/post-media-upload#post-media-upload) または [POST media/upload（分割アップロード）](https://developer.x.com/content/developer-twitter/en/docs/media/upload-media/api-reference/post-media-upload-init#post-media-upload-init)（メディアのアップロード）

<div id="adding-to-the-library">

<div id="request-parameters">

</div>

すべての Media Library の POST リクエストにはメディア識別子が必要です。この値はアップロード手順の際に返されます。上記の例のように media\_id を使用する場合は、media\_category も指定する必要があります。カテゴリー値は次の4種類です: AMPLIFY\_VIDEO、TWEET\_GIF、TWEET\_IMAGE、TWEET\_VIDEO。

任意で、Media Library 内のオブジェクトに対して name および file\_name の値を設定できます。これらの属性は、ライブラリ内のメディアのバリアントを見分けるのに役立ちます。

動画の場合は、title と description を設定することもできます。これらの値は、[POST accounts/:account\_id/tweet](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/api-reference/tweets#post-accounts-account-id-tweet) エンドポイントで video\_title と video\_description のリクエストパラメータとして渡すことを想定しています。Tweet では、このテキストは動画の下に表示されます。

<div id="attributes">
  #### 属性
</div>

Media Library は、media\_key の概念を正式に導入します。これはライブラリ内のオブジェクトを一意に識別する ID です。media\_key は 13\_875943225764098048 のような形式の文字列で、当社のすべてのカード用エンドポイントで完全にサポートされています。

また、Media Library のレスポンスには文字列として表される media\_id も含まれます。これは現在 media\_key を受け付けないリソース向けに提供されます: [Tweets](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/api-reference/tweets#post-accounts-account-id-tweet)\*、[Tweet preview](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/api-reference/tweets#get-accounts-account-id-tweet-preview)\*、および [Scheduled Tweets](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/api-reference/scheduled-tweets#post-accounts-account-id-scheduled-tweets)。当社は、すべての箇所で media\_key をサポートできるよう取り組んでいます。

aspect\_ratio 属性は GIF および動画で返されます。これは、特定のアスペクト比のみを受け付けるカードで使用するメディアをフィルタリングするために利用できます。

\*これらのエンドポイントは、media\_key である video\_id パラメータをサポートしています。

  ### カードの判別
</div>

<div id="introduction">
  #### はじめに
</div>

Cards はメディアを用いるカスタマイズ可能な広告フォーマットで、ウェブサイト、アプリ、または特定のユーザーエンゲージメント（例：ダイレクトメッセージの開始）を促すコールトゥアクションに関連付けられます。Tweet、Scheduled Tweet、Draft Tweet に付加できます。

Cards は Tweet オブジェクト内で、card の card\_uri か preview\_url のいずれかで参照されます。各値の例は以下のとおりです。

| card\_uri | preview\_url |
| :--- | :--- |
| card://1043282691834048513 | https://cards.x.com/cards/18ce54d4x5t/68w3s |

**注記**: Ads API バージョン 5 以降、cards レスポンスで preview\_url は返されません。

Tweet オブジェクトのレスポンスにおける参照の種別は、Tweet の作成方法に依存します。つまり、card\_uri リクエストパラメータを使用して Tweet が作成された場合は、レスポンスに card URI の値が含まれます。一方、Tweet テキストの一部として preview\_url が含まれていた場合は、レスポンスに preview URL が含まれます。

<div id="identifying-media">
  ### メディアの特定
</div>

<div id="introduction">
  #### はじめに
</div>

メディア（画像、GIF、動画）は、投稿やカードに追加できます。さらに、動画はプレロール用アセットとして使用でき、画像は[X Audience Platform](https://developer.x.com/en/docs/ads/measurement/overview/twitter-audience-platform)でプロモーションできます。本セクションでは、これらのエンティティ全体にわたってメディア参照を見つける方法を説明します。

メディア識別子には2種類あります：IDとキーです。各種の例は以下のとおりです。

| **Media ID** | **Media key** |
| :--- | :--- |
| 1029825579531807971 | 13\_1029825579531807971 |

メディアキーは、IDの前に数値のプレフィックスとアンダースコアを付けたものです。

<div id="image">
  #### 画像
</div>

以下の表は、各画像関連リソースのレスポンスで現在利用可能な識別子の種類と、それに対応する属性名を示します。

| **Resource** | **Identifier** | **Attribute(s)** |
| :--- | :--- | :--- |
| Image cards | None |  |
| Tweet | Both | `entities["media"]["id_str"]`  `entities["media"]["media_key"]` |
| Scheduled Tweet | Both | `media_ids` and `media_keys` |
| Draft Tweet | Both | `media_ids` and `media_keys` |
| Account Media | None |  |
| Media Library | Both | `media_id` and `media_key` |

Image cards と Account Media の画像には、いずれのメディア識別子への参照も含まれません。Tweets にはメディア ID のみが含まれます。Scheduled および Draft の Tweets には、メディア ID とメディアキーの両方が含まれます。Media Library も同様に両方を返します。

Tweets の場合、entities\["media"] 配列内のオブジェクトにある id および id\_str フィールドはメディア ID に対応します。Tweet に複数の画像が含まれる場合、各メディアエンティティへの参照は extended\_entities\["media"] にのみ見つかります。

識別子への参照に加えて、画像の URL へのアクセスが重要となることもよくあります。

| **Resource** | **Attribute(s)** | **Format** |
| :--- | :--- | :--- |
| Image cards | `image` | .jpg |
| Tweet\* | `entities["media"][0]["media_url"]` or `extended_entities["media"][i]["media_url"]` | .jpg |
| Scheduled Tweet | None |  |
| Draft Tweet | None |  |
| Account Media | `media_url` | .jpg |
| Media Library | `media_url` | .jpg |

\* この URL の場所は、Tweet に単一画像が含まれるか複数画像が含まれるかによって異なります。

すべての Image cards には、X の画像 URL を含む image レスポンス属性が含まれます（image app download cards の場合、属性名は wide\_app\_image です）。

Tweets の場合、メディア URL の場所はメディアの種類と使用しているエンドポイントの両方に依存します。単一画像の Tweets では、URL は entities\["media"]\[0]\["media\_url"] にあります。これは X 広告 API と Standard API の両方で当てはまります。一方、複数画像を含む Tweets では、URL は extended\_entities\["media"]\[i]\["media\_url"] にのみ存在します。これは Standard API でのみ利用可能です。

<div id="video">
  #### 動画
</div>

以下の表は、各動画関連リソースのレスポンスで現在利用可能な識別子の種類と、それに対応する属性名を示しています。

| **リソース** | **識別子** | **属性** |
| :--- | :--- | :--- |
| Video cards | いずれか | `video_content_id` |
| Video poll cards | なし |  |
| Tweet | 両方 | `entities["media"]["id_str"]`  `entities["media"]["media_key"]` |
| Scheduled Tweet | 両方 | `media_ids` と `media_keys` |
| Draft Tweet | 両方 | `media_ids` と `media_keys` |
| Account Media | Media key | `video_id` |
| Media Library | 両方 | `media_id` と `media_key` |

Video cards（動画付きの投票カードを除く）には `video_content_id` レスポンス属性が含まれますが、返される値の型に一貫性がありません。場合によっては media ID、別の場合は media key になります。

動画のURLへのアクセス方法は以下のとおりです。

| **リソース** | **属性** | **形式** |
| :--- | :--- | :--- |
| Video cards | `video_url` と `video_hls_url` | .vmap  .m3u8 |
| Tweet with video | `extended_entities["media"][i]["video_info"]["variants"][j]["url"]` | .mp4 |
| Scheduled Tweet | なし |  |
| Draft Tweet | なし |  |
| Account Media | なし |  |
| Media Library | `media_url` | .mp4 |

Video cards には、.vmap と .m3u8 のURLをそれぞれ指す `video_url` および `video_hls_url` レスポンス属性が含まれます。

<div id="media-library">
  #### メディアライブラリ
</div>

メディアアセットに関する追加情報が必要になることがあります。ビデオカードのユースケースの一つとして、vmap ではなく mp4 の URL を取得することが挙げられます。これはメディアライブラリで取得できます。利用可能な情報の詳細は、[Media Library Guide](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/guides/media-library)を参照してください。広告アカウントの FULL プロモータブルユーザーに属するアセットの大半はライブラリで見つかりますが、いくつか例外があります。

**メディアの取得**

前述のとおり、イメージカードには media ID や media key への参照が含まれていません。したがって、メディアライブラリ経由でそれらのアセットを取得することはできません。これは[Account Media](https://developer.x.com/content/developer-twitter/en/docs/ads/creatives/api-reference/account-media#account-media)の画像でも同様です。

ビデオカードは、作成前にビデオアセットがメディアライブラリ（または以前の Videos リソース）に登録されている必要があります。したがって、これらのアセットは常にメディアライブラリから取得できます。これは Account Media の PREROLL アセットにも当てはまります。

最後に、ポスト内のメディアは常にメディアライブラリに存在することが保証されています。

次の表は、参照に使用できる識別子がリソースのレスポンスに含まれるかどうかを考慮しつつ、メディアライブラリから取得可能なアセットをまとめたものです。

| **Resource** | **In the Media Library** |
| :--- | :--- |
| Image cards | No |
| Video cards | Yes\* |
| Tweets (any media)\*\* | Yes |
| Scheduled Tweets | Yes |
| Draft Tweets | Yes |
| Account media images | No |
| Account media videos (`PREROLL`) | Yes |

\* `video_content_id` がメディアキーであるカードの場合。値がメディア ID の場合でもアセットはメディアライブラリに存在しますが、取得するには先頭に数値のプレフィックスとアンダースコアを付与する必要があります。
\*\* ポストはメディア ID のみを返します。アセットはメディアライブラリに存在することが保証されていますが、取得には先頭に数値のプレフィックスとアンダースコアを付与する必要があります。

**Account Media との相互作用**

ライブラリに追加されたメディアアセットが自動的に Account Media リソースにも追加されるケースが2つあります。

- AMPLIFY\_VIDEO アセットがメディアライブラリに追加されると、PREROLL のクリエイティブタイプとして Account Media アセットに自動的に追加されます。
- 特定の寸法（[enumerations page](https://developer.x.com/content/developer-twitter/en/docs/ads/general/overview/enums)の「Creative Types」を参照）を持つ画像がメディアライブラリに追加されると、自動的に Account Media アセットとして追加されます。クリエイティブタイプ（例：INTERSTITIAL）は画像の寸法によって決まります。

<div id="tweets">
  ### 投稿
</div>

<div id="introduction">
  #### はじめに
</div>

X 広告 API は、公開済み、予約投稿、下書きという 3 種類のツイートをサポートしています。

<div id="nullcasted-tweets">
  #### Nullcasted Tweet（ヌルキャストされたツイート）
</div>

Tweet は「ヌルキャスト（別名「Promoted-only」）」またはオーガニックのいずれかです。ヌルキャストされた Tweet は公開後、ユーザーの公開タイムラインには表示されませんが、公開状態ではあります。一方、オーガニックな Tweet はユーザーのフォロワーに配信され、ユーザーの公開タイムラインにも表示されます。

**Tweet の作成**

3 つの Tweet 作成エンドポイントにはいずれも、ヌルキャストまたはオーガニックの Tweet を作成するかを選択できる Boolean 型の `nullcast` パラメータがあります。ヌルキャストされた Tweet は、ユーザー本人、またはユーザーの代理で Tweet を作成する権限を持つ任意のユーザーが作成できます。オーガニックな Tweet は [full promotable user](https://developer.x.com/content/developer-twitter/en/docs/tutorials/ads-api-hierarchy-terminology#promotable-users) のみが作成できます。

**Tweet の更新**

予約投稿および下書きの Tweet については、`nullcast` プロパティを更新できます。予約投稿の場合、Tweet の `scheduled_at` 時刻まで編集が可能です。下書きの Tweet は無期限に編集できます。ただし、一度公開されると、Tweet をヌルキャストからオーガニック、またはその逆に変更することはできません。

<div id="promoting-tweets">
  #### ツイートのプロモーション
</div>

プロモーションできるのは、公開済みまたは予約投稿のツイートのみです。これらは nullcast でもオーガニックでも問題ありません。制限はありません。広告主は、自身のツイートだけでなく、許可を得ていれば他のユーザーのツイートもプロモーションできます。（詳細は「[他のユーザーのツイートをプロモーションする](https://developer.x.com/content/developer-twitter/en/docs/tutorials/promoting-another-users-content)」を参照してください。）

1 つのキャンペーンで複数のツイートをプロモーションできます。同様に、1 つのツイートを 1 つ以上のキャンペーンでプロモーションすることも可能です。

公開済みツイートをプロモーションするには、[POST accounts/:account\_id/promoted\_tweets エンドポイント](https://developer.x.com/content/developer-twitter/en/docs/ads/campaign-management/api-reference/promoted-tweets#post-accounts-account-id-promoted-tweets)を使用します。これにより、公開済みツイートがラインアイテムに関連付けられます。予約投稿ツイートをプロモーションするには、[POST accounts/:account\_id/scheduled\_promoted\_tweets エンドポイント](https://developer.x.com/content/developer-twitter/en/docs/ads/campaign-management/api-reference/scheduled-promoted-tweets#post-accounts-account-id-scheduled-promoted-tweets)を使用します。

<div id="tweet-ids">
  #### ツイート ID
</div>

公開済み・予約済み・下書きのツイート ID は[数値](https://developer.x.com/content/developer-twitter/en/docs/basics/twitter-ids)で、64 ビットの符号なし整数です。例えば、次の公開済みツイートの ID は 1166476031668015104 です。

公開または予約済みのツイートをプロモーションすると、対応するプロモーションツイートのエンティティが作成されます。これらのエンティティには固有の ID があり、英数字で、base-36 でエンコードされた値として表されます。例えば、上記の公開済みツイートをプロモーションする（つまり、ラインアイテム 6c62d に関連付ける）と、次の API レスポンスが返されます。

    }
  },
  "data": [
    {
      "line_item_id": "6c62d",
      "id": "3qwlq6",
      "entity_status": "ACTIVE",

<div id="carousels">
  ### カルーセル
</div>

<div id="introduction">
  #### はじめに
</div>

X 広告 API は、動画カルーセルおよび画像カルーセルの作成と取得をサポートします。カルーセルは、2～6 個のメディアアセットを含められるカードタイプです。カルーセルカードは、ユーザーをウェブサイトへ誘導したり、モバイルアプリのインストールを促したりできます。カルーセルの概要、利点、ベストプラクティス、FAQ については、[Carousel Ads on X](https://business.x.com/en/advertising/carousels.html) を参照してください。

カルーセルは他のカードタイプと同様に Tweet で使用でき、その Tweet をプロモートできます。ワークフローはこれまでと同様です。

1. メディアをアップロード
2. カードを作成
3. Tweet を作成
4. Tweet をプロモート

唯一の違いはカードの作成方法です。他のカード作成リクエストがクエリパラメータを受け付けるのに対し、カルーセルカードの作成リクエストは JSON の POST ボディのみを受け付けます。

<div id="endpoints">
  #### エンドポイント
</div>

X 広告 API では、カルーセルの作成および取得が可能です。

任意のタイプのカルーセルを作成するには、[POST accounts/:account\_id/cards](https://developer.x.com/en/docs/twitter-ads-api/creatives/api-reference/cards#post-accounts-account-id-cards) エンドポイントを使用します。カルーセルを取得するには、[GET accounts/:account\_id/cards](https://developer.x.com/en/docs/twitter-ads-api/creatives/api-reference/cards#get-accounts-account-id-cards) エンドポイントを使用します。

<div id="json-post-body">
  #### JSON POST 本文
</div>

カルーセルは2つのコンポーネントを使用して作成します。1つ目は使用するメディアアセットを指定します。2つ目はウェブサイトまたはアプリのいずれかに関する情報を指定します。

具体的には、カルーセルカードは次のコンポーネントをこの順序で使用して作成します。

- 配列のメディアキーを受け取る `SWIPEABLE_MEDIA` コンポーネントを1つ
- 次のうち「いずれか」を1つ:
- ウェブサイト情報を指定する `DETAILS` コンポーネント
- アプリ情報を指定する `BUTTON` コンポーネント

`SWIPEABLE_MEDIA` コンポーネントには、2～6個の画像または動画を指定できる `media_keys` 配列を含める必要があります。渡したメディアキーの順序が、そのままレンダリング順になります。

```
{
  "type": "SWIPEABLE_MEDIA",
  "media_keys": [
    "13_1089771253848666112",
    "13_1191948012077092867"
  ]
}
```

参考までに、メディアキーは [GET accounts/:account\_id/media\_library](https://developer.x.com/en/docs/twitter-ads-api/creatives/api-reference/media-library#get-accounts-account-id-media-library) エンドポイントへのリクエストで取得できます。

2つ目のコンポーネントオブジェクトの構成は、ユーザーをウェブサイトに誘導するか、アプリのインストールを促すかによって異なります。以下の表は、この2つのオプションの概要です。（注: 記載されているキーはすべて必須です。）

|  | **ウェブサイト** | **アプリ** |
| :--- | :--- | :--- |
| コンポーネントタイプの指定 | `"type": "DETAILS"` | `"type": "BUTTON"` |
| タイトル／ラベル | `"title": "X"` | `"label": { "type": "ENUM", "value": "INSTALL" }` |
| 遷移先 | `"destination": { "type": "WEBSITE", "url": "https://www.x.com" }` | `"destination": { "type": "APP", ... }` |

これらを組み合わせると、ウェブサイトカルーセルの JSON POST 本文の例は以下のとおりです。

```
{
  "name": "website carousel",
  "components": [
    {
      "type": "SWIPEABLE_MEDIA",
      "media_keys": [
        "13_1089771253848666112",
        "13_1191948012077092867"
      ]
    },
    {
      "type": "DETAILS",
      "title": "X",
      "destination": {
        "type": "WEBSITE",
        "url": "https://www.x.com"
      }
    }
  ]
}
```

`BUTTON` コンポーネント内のアプリ遷移先オブジェクトには、国コードと少なくとも1つのアプリ識別子が必要です。任意でディープリンクを指定できます。これらのフィールドの説明については、[リファレンスドキュメント](https://developer.x.com/en/docs/twitter-ads-api/creatives/api-reference/cards#app-destination)を参照してください。

これらを組み合わせると、アプリカルーセルの JSON POST 本文の例は以下のとおりです。

```
{
  "name": "app carousel",
  "components": [
    {
      "type": "SWIPEABLE_MEDIA",
      "media_keys": [
        "13_1089771253848666112",
        "13_1191948012077092867"
      ]
    },
    {
      "type": "BUTTON",
      "label": {
        "type": "ENUM",
        "value": "INSTALL"
      },
      "destination": {
        "type": "APP",
        "country_code": "US",
        "googleplay_app_id": "com.twitter.android",
        "iphone_app_id": "333903271"
      }
    }
  ]
}
```

<div id="creative-metadata-tagging">
  ### クリエイティブメタデータのタグ付け
</div>

<div id="introduction">
  #### はじめに
</div>

本ガイドは、クリエイティブパートナー、代理店、クリエイティブ開発者が、X キャンペーンで使用するアセットにタグ付けを行い、個々のアセットの価値とパフォーマンスをより深く把握するためのものです。

**注:** メディアアセットへのタグ付けは、当該メディアアセットを作成したパートナーまたは開発者のみが実施してください。メディアアセットの利用者がそのアセットを作成していない場合は、メタデータのタグ付けを実装しないでください。

Creative Metadata Tagging は、アセットが X のどこにアップロードされるか、またアップロードを行う主体が誰かに関わらず、クリエイティブパートナーが作成した画像や動画の帰属情報を提供します。クリエイティブアセットとクリエイティブパートナーを関連付けるために、[XMP](https://exiftool.org/TagNames/XMP.html) 標準を使用します。

<div id="tagging-creative-assets">
  #### クリエイティブアセットへのタグ付け
</div>

次の表は、各画像関連リソースのレスポンスで現在利用可能な識別子タイプと、それに対応する属性名を示しています。クリエイティブアセットにタグを付与するには、タグ付けツールが必要です。[ExifTool](https://exiftool.org/) は、メタ情報の読み取り・書き込み・編集が可能なプラットフォーム非依存の Perl ライブラリおよび[コマンドラインアプリケーション](https://exiftool.org/exiftool_pod.html)で、推奨されます。サポートされているすべての[ファイルタイプ](https://exiftool.org/#supported)をご確認ください。

提供されている [ExifTool のインストール手順](https://exiftool.org/install.html)に従ってください。[Homebrew](https://formulae.brew.sh/) でも、macOS と Linux 向けに [exiftool のインストールコマンド](https://formulae.brew.sh/formula/exiftool)を提供しており、インストールをさらに簡素化できます。コマンドラインで exiftool -ver を実行して、ツールが正しくインストールされていることを確認し、バージョン番号を表示します。ExifTool のコマンドパラメータについては [ExifTool ドキュメント](https://exiftool.org/)を参照してください。

クリエイティブパートナーは、新規または既存のクリエイティブアセットに対して、X の app\_id を contributor XMP タグおよび date タグとしてメタデータに付与できます。クリエイティブアセットは、[Uploading Media](https://developer.x.com/en/docs/twitter-api/v1/media/upload-media/overview) の既存のサイズ制限に従います。 

**注:** X における contributor XMP タグの使用により、メタデータは X 上のキャンペーン専用の値のみを確実に保持します。

`exiftool -contributor="<YOUR APP ID>" -creative_file.jpg`

`exiftool -date="<date>" -creative_file.jpg`

app\_id は [Developer Portal](https://developer.x.com/en/portal/dashboard) の Projects & Apps 内で確認できます。例: 16489123

次の例では、画像に対して contributor タグとして app\_id を、date タグとして date を追加します:

```
app_id:858382169
```

```
date:2022-03-13
```

```
creative_file: eiffel_tower.jpg
```

```
exiftool -contributor=858382169 eiffel_tower.jpg
```

```
   1 image files updated
```

```
exiftool -date=2022-03-13 eiffel_tower.jpg
```

```
  1 image files updated
```

画像に正しくタグが付与されていることを確認します: 

`exiftool -xmp:all -G1 <filename>`

例:

`exiftool -xmp:all -G1 eiffel_tower.jpg`

```
[XMP]        XMP Toolkit            : Image::ExifTool 12.30
```

```
[XMP]        Contributor            : 858382169
```

```
[XMP]        Date                   : 2022:03:13
```

#### POST accounts/:account\_id/cards

指定したアカウントに紐づく新規カードを作成します。

カード作成リクエストは JSON の POST 本文のみを受け付けます。`Content-Type` は `application/json` に設定する必要があります。

詳しい使用例は [Carousels Guide](https://developer.x.com/en/docs/twitter-ads-api/creatives/guides/carousels) をご覧ください。

<div id="resource-url">
  #### リソースURL
</div>

`https://ads-api.x.com/12/accounts/:account_id/cards`

<div id="parameters">
  #### パラメーター
</div>

JSON の POST ボディには、カードの `name` と `components` 配列を含める必要があります。コンポーネントはオブジェクトで表され、広告主向けのカード属性を記述します。

次の例は、ペイロードの一般的な構造を示します（実際には動作しない値が含まれています）。

```
{
  "name": "some name",
  "components": [
    {
      "type": "TYPE_ENUM",
      "key": "value"
    }
  ]
}
```

コンポーネントに関する追加情報は以下のとおりです。

| 名前 | 説明 |
| :--- | :--- |
| account\_id   _必須_ | 利用するアカウントの識別子。リソースパス内に含まれ、[GET accounts](https://developer.x.com/en/docs/ads/campaign-management/api-reference/accounts#get-accounts) を除くすべての Advertiser API リクエストで一般に必須のパラメーターです。指定したアカウントは認証済みユーザーに関連付けられている必要があります。  種別: string  例: `18ce54d4x5t` |
| name   _必須_ | カードの名前。最大 80 文字。  種別: string  例: `component based card` |
| components   _場合により必須_ | カードの作成に使用するコンポーネントを記述します。詳細は以下を参照。  `slides` と同時には指定できません。  **注**: コンポーネントの順序は重要です。  種別: オブジェクトの配列 |
| slides   _場合により必須_ | Multi-Destination Carousel を作成するための配列の配列です。各カードをコンポーネントのグループとして記述します。各スライドはカードの完全な表現である必要があります。  `components` と同時には指定できません。  **注**: 各スライドの順序は重要です。  種別: 配列の配列 |

<div id="components">
  #### コンポーネント
</div>

すべてのコンポーネントには、オブジェクトのスキーマを決定する `type` フィールドを含める必要があります。X 広告 API は、以下のコンポーネントタイプをサポートしており、メディア系と説明系のコンポーネントに分類されます。

- メディア:
- `MEDIA`: 単一の動画または画像
- `SWIPEABLE_MEDIA`: 2〜6 個の動画または画像
- 説明:
- `DETAILS`
- `BUTTON`

各コンポーネントには（`type` キーに加えて）必須フィールドのセットがあります。次の表に示します。

| コンポーネント `type` | フィールド | 値の型 |
| :--- | :--- | :--- |
| `MEDIA` | `media_key` | string |
| `SWIPEABLE_MEDIA` | `media_keys` | array of strings |
| `DETAILS` | `title`   `destination` | string   object |
| `BUTTON` | `label`   `destination` | object   object |

以下は、`components` 配列内における `BUTTON` コンポーネントの例です（意図的に `name` キーを省略）。（省略記号は、追加の情報を指定する必要がある箇所を示します。）

```
{
  "components": [
    {
      "type": "BUTTON",
      "label": {
        ...
      },
      "destination": {
        ...
      }
    }
  ]
}
```

コンポーネントオブジェクトの指定順序が、レンダリング時の上から下への表示順序を決定します。カードは、1 つのメディア系コンポーネントと、`DETAILS` か `BUTTON` のいずれかのコンポーネントを組み合わせて作成する必要があります。説明系コンポーネントはメディアの下にレンダリングされ、URL またはモバイルアプリのいずれかを遷移先として関連付けます。

**Label**

Label はボタンに表示されるテキストを定義するため、`BUTTON` コンポーネントにのみ適用されます。Label オブジェクトには必須キーが 2 つあります：`type` と `value`。`type` は `ENUM` に設定する必要があり、`value` は `BOOK`、`CONNECT`、`INSTALL`、`OPEN`、`ORDER`、`PLAY`、`SHOP` のいずれかです。

前の例に基づき、以下は `BUTTON` コンポーネント内の `label` オブジェクトを示しています。

```
{
  "components": [
    {
      "type": "BUTTON",
      "label": {
        "type": "ENUM",
        "value": "INSTALL"
      },
      "destination": {
        ...
      }
    }
  ]
}
```

**Destination**

Destination は広告主がユーザーを誘導したい場所です。`DETAILS` または `BUTTON` コンポーネント内で常に必須です。Destination のタイプは `WEBSITE` または `APP` の 2 種類です。

**注**: Website の Destination は `DETAILS` コンポーネントでのみ使用でき、App の Destination は `BUTTON` コンポーネントでのみ使用できます。

**Website Destination**

| 名前 | 説明 |
| :--- | :--- |
| type   _必須_ | Destination のタイプ。スキーマを決定します。  型: enum  可能な値: `WEBSITE` |
| url   _必須_ | ユーザーをリダイレクトするウェブサイトの URL。  型: string  例: `https://devcommunity.x.com/c/advertiser-api` |

**App Destination**

| 名前 | 説明 |
| :--- | :--- |
| type   _必須_ | Destination のタイプ。スキーマを決定します。  型: enum  可能な値: `APP` |
| country\_code   _必須_ | アプリが販売されている国の [ISO 3166-1 alpha-2](https://en.wikipedia.org/wiki/ISO_3166-1_alpha-2) 2 文字コード。  型: string  例: `US` |
| googleplay\_app\_id   _場合により必須_ | Google Play のアプリケーションパッケージ名。  **注**: `ios_app_store_identifier` または `googleplay_app_id` のいずれか一方は必須です。  型: string  例: `com.twitter.android` |
| ios\_app\_store\_identifier   _場合により必須_ | iOS App Store の識別子。  **注**: `ios_app_store_identifier` または `googleplay_app_id` のいずれか一方は必須です。  型: string  例: `333903271` |
| googleplay\_deep\_link   _オプション_ | プロモーション対象の Android アプリへのディープリンク。  **注**: `googleplay_app_id` が指定されている場合にのみ使用可能です。  型: string |
| ios\_deep\_link   _オプション_ | プロモーション対象の iOS アプリへのディープリンク。  **注**: `ios_app_store_identifier` が指定されている場合にのみ使用可能です。  型: string |

#### POST accounts/:account\_id/tweet

アカウントのフルのプロモータブルユーザー（デフォルト）または `as_user_id` パラメータで指定したユーザーとして投稿（Tweet）を作成します。ヌルキャスト（デフォルト）とオーガニックの両方の投稿作成をサポートします。ヌルキャスト投稿はパブリックタイムラインに表示されず、フォロワーにも配信されません。どちらのタイプもキャンペーンで利用できます。

認証済みユーザーがこのアカウントの `FULL` プロモータブルユーザーでない場合は、[GET accounts/:account\_id/authenticated\_user\_access](https://developer.x.com/en/docs/ads/campaign-management/api-reference/authenticated-user-access#get-accounts-account-id-authenticated-user-access) エンドポイントにリクエストし、そのユーザーの代わりに投稿できる権限があるかを確認してください。`TWEET_COMPOSER` 権限がある場合、そのユーザーはこのエンドポイントを使用して、`FULL` の[プロモータブルユーザー](https://developer.x.com/en/docs/ads/campaign-management/api-reference/promotable-users#get-accounts-account-id-promotable-users)の代わりにヌルキャスト投稿を作成できます。

メディアに [upload.x.com エンドポイント](https://developer.x.com/rest/reference/post/media/upload) を使用する場合は、このエンドポイントに渡す `as_user_id` の値と同じ `user_id` を `additional_owners` パラメータに指定してください。
