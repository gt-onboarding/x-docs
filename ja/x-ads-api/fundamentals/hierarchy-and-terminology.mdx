---
title: "X 広告 API の階層と用語"
---

X 広告 API で成果を上げるには、X 広告におけるエンティティ同士の関係を理解することが重要です。本チュートリアルでは、X 広告のオブジェクト階層の基本を解説します。

![X 広告 API の階層](/images/adsapi-heirarchy.png.twimg.1920.png)

<div id="ad-group-line-items">
  ## 広告グループ（line items）
</div>

広告グループは、X 広告 API では [line items](https://developer.x.com/content/developer-twitter/en/docs/tutorials/ads-api-hierarchy-terminology#line-items) と呼ばれ、キャンペーンの配下に位置し、特定の X ユーザー集合に対するターゲティングや入札に使用されます。広告主は、line item に紐づけることで、投稿やメディア（例：インストリーム広告として配信される動画）をプロモーションします。Ad Group も参照してください。

<div id="account-media">
  ## アカウントメディア
</div>

[アカウントメディア](/ja/x-ads-api/creatives#account-media)として広告アカウントにアップロードされたクリエイティブのコレクションで、プロモーション可能な動画の[プレロールアセット](/ja/x-ads-api/campaign-management#video-views-preroll-objective)（インストリーム広告とも呼ばれます）や、[X Audience Platform](https://developer.x.com/content/developer-twitter/en/docs/ads/measurement/overview/twitter-audience-platform)で配信可能な画像を含むことがあります。

<div id="ads-accounts">
  ## 広告アカウント
</div>

X 広告の階層の最上位には広告アカウントがあります。広告アカウントは複数の X ユーザーが管理できます。@Furni（当社のデモ用家具店）は、広告アカウントを管理するユーザーの一例です。広告アカウントは、[base36](https://en.wikipedia.org/wiki/Base36) の英数字文字列である account\_id で識別されます。

各アカウントには、キャンペーン間で再利用できる複数のエンティティタイプが保存されます。たとえば、ターゲティングに使用できるテーラードオーディエンス、動画や画像であるアカウントメディア、投稿に追加できる拡張クリエイティブであるカードなどがあります。

<div id="amplify-video">
  ## Amplify Video
</div>

動画アセットをアップロードする際、アップロード主体はその動画のカテゴリ（例：Tweet Video、Tweet Gif、Tweet Image、Amplify Video）を指定できます。インストリーム広告として配信する動画をアップロードする場合は、Amplify Video カテゴリが必須です。これらの動画は配信可能性を担保するための特別な処理が必要であるため、メディアカテゴリとして Amplify Video を指定する必要があります。

<div id="analytics">
  ## アナリティクス
</div>

アナリティクス指標は、パートナーや広告主がX上でプロモーションしたコンテンツの成果を把握するのに役立ちます。これには、インプレッション、クリック、動画再生数、広告費などの情報が含まれます。さらに、パートナーや広告主は、リーチしたオーディエンスのさまざまなセグメントに対する詳細な指標を取得できます。

X 広告 API では、キャンペーンの詳細なパフォーマンス指標を取得する方法として、同期と非同期の2通りをサポートしています。

[同期アナリティクス](/ja/x-ads-api/analytics#synchronous-analytics)のリクエストでは、要求した指標がレスポンスで返されます。同期エンドポイントは短い期間を対象としており、リアルタイムのキャンペーンパフォーマンスの照会に最適です。

[非同期アナリティクス](/ja/x-ads-api/analytics#asynchronous-analytics)のエンドポイントでは、関連する「ジョブ」の処理が完了した後、ダウンロード可能な結果ファイルで要求した指標を取得できます。非同期エンドポイントは、はるかに長い期間にわたる指標の取得をサポートし、セグメント化データの返却にも対応しており、より大規模なデータの取得を目的としています。レポート作成や履歴データのバックフィルに最適です。

<div id="campaigns">
  ## キャンペーン
</div>

広告主が実施する各キャンペーンは、既存の資金管理手段の配下で作成する必要があります。キャンペーンでは、広告キャンペーンの名称、スケジュール、予算などを設定できます。Furni の例に沿うと、「Furni home decor」という名前のキャンペーンを作成するかもしれません。

広告アカウントには、有効なキャンペーンは最大 200 件までという上限がある点に留意してください。この上限は、広告主のアカウントマネージャーによって引き上げられる場合があります。キャンペーンは、終了時刻に達するか予算を使い切るまで有効と見なされます。一時停止中のキャンペーンも、上限の算定においては有効なキャンペーンとして扱われます。

<div id="cards">
  ## カード
</div>

カードは、画像、動画、または UI 要素を含めてツイートを視覚的に拡張する手段です。X 広告 API は、ツイートで使用できる複数のカードタイプをサポートしており、これらはキャンペーンで配信（プロモーション）できます。

<div id="creatives">
  ## クリエイティブ
</div>

広告主は、画像、GIF、動画などのクリエイティブを含むキャンペーンで投稿をプロモーションできます。これらのクリエイティブは、投稿にリッチな写真、動画、メディア体験を添付できるXカードに関連付けることができます。クリエイティブは投稿に直接添付することも可能です。たとえば、テキストと動画（または最大3枚の画像）を含めて、プロモーション投稿を作成できます。

各ラインアイテムは、配信するために少なくとも1つのクリエイティブを関連付ける必要があります。新規フォロワー獲得のためのPromoted Accounts、TX Audience Platform全体のラインアイテム向けの画像や動画などのMedia Creatives、またはPromoted Postsなど、クリエイティブには複数の選択肢があります。

<div id="funding-instruments">
  ## 資金手段
</div>

広告アカウントには「資金手段」も含まれます。これは、広告主がX 広告で利用するために設定したさまざまな支払い元を指し、X と締結したインサーションオーダーからアカウントに紐づけたクレジットカードまでを含みます。

<div id="line-items">
  ## ラインアイテム
</div>

キャンペーン内では、1つ以上の広告グループ（X 広告 API ではラインアイテムとも呼ばれます）を設定できます。ラインアイテムはキャンペーン内の異なるセグメントと捉えることができ、それぞれに固有のターゲティング設定とクリエイティブを持ちます。複数のラインアイテムを含むキャンペーンは、しばしば「広告グループキャンペーン」と呼ばれます。

1つのキャンペーン内に複数のラインアイテムを設定する利点の1つは、ターゲティングとクリエイティブの組み合わせの幅を広げられることです。

「Furni home decor」というキャンペーンの下で、2つのラインアイテムを作成するとします。1つは「サンフランシスコの住宅購入者」、もう1つは「ロンドンの住宅購入者」とすることができます。

1つのキャンペーンあたり最大100件のラインアイテムが許可されており、広告アカウント全体では、すべてのキャンペーンを合算して有効なラインアイテムは合計4,000件までに制限されています。

<div id="mobile-app-promotion-map">
  ## Mobile App Promotion（MAP）
</div>

Mobile App Promotion（以下、MAP）は、広告主がX上でモバイルアプリを宣伝できる製品スイートです。アプリのインストール、コンバージョン、リエンゲージメントを対象とします。

<div id="media-library">
  ## メディアライブラリ
</div>

メディアライブラリのエンドポイントは、X 広告アカウントで画像、GIF、動画を管理する機能を提供します。ライブラリ内のメディアアセットは、投稿やカードの作成に使用できます。複数のクリエイティブで再利用できるため、同じアセットを繰り返しアップロードする必要がありません。

<div id="promoted-accounts">
  ## プロモアカウント
</div>

プロモアカウント（フォロワーキャンペーン）は「おすすめユーザー」の一部で、現在フォローしていないが興味を持つ可能性のあるアカウントを提案します。プロモアカウントは、ユーザーが楽しめるアカウントの幅をさらに広げるのに役立ちます。

プロモアカウントを利用すると、広告主は自社アカウントを宣伝し、X 上でオーディエンスを拡大できます。フォロワーキャンペーンは、ホームタイムライン、「おすすめユーザー」、検索結果など、X プラットフォーム内の複数の場所に表示され、他のおすすめアカウントと区別するために「プロモーション」として表示されます。

<div id="promotable-users">
  ## プロモート可能ユーザー
</div>

各 X 広告アカウントには、広告アカウントを通じてプロモーションできるユーザー（プロモート可能ユーザー）を 1 人以上設定できます。Furni の広告アカウントにおける @Furni は FULL のプロモート可能ユーザーの例であり、@Furni およびその投稿に対してあらゆる種類のキャンペーンを実施できることを意味します。1 つの広告アカウントにつき、FULL のプロモート可能ユーザーは 1 人のみです。

また、広告アカウントには他のプロモート可能ユーザーを設定することもできます。たとえば、Furni 製品に関する @Jack の投稿をプロモーションしたい場合、@Jack は RETWEETS\_ONLY で示されるリツイート専用のプロモート可能ユーザーになります。広告主は、リツイートユーザーを広告アカウントに追加する方法について、X のアカウントマネージャーまでお問い合わせください。

<div id="promoted-video">
  ## プロモ動画
</div>

プロモ動画は、広告主が費用を支払って掲載するプロモツイート内の動画です。ホームタイムライン、X の検索結果上部、プラットフォーム内のその他の場所に表示され、明確に「Promoted」と表示されます。

[X Audience Platform](/ja/x-ads-api/audiences) を参照してください。

<div id="scheduled-tweets">
  ## 予約投稿
</div>

予約投稿は、広告主やユーザーが、後日公開する投稿（Tweet）をあらかじめ作成・スケジュールできる機能です。これらの投稿を作成・管理できるだけでなく、API を使って、公開時にプロモーションできるよう投稿をラインアイテムに関連付けることも可能です。これにより、広告主はネイティブ投稿を事前に用意し、重要な取り組みに先立ってキャンペーンのクリエイティブを計画できます。たとえば、新製品の発表と同時に公開されるよう、投稿クリエイティブをあらかじめ用意しておく、といった使い方が可能です。

<div id="segmentation">
  ## セグメンテーション
</div>

セグメンテーションを使用すると、パートナーや広告主は年齢、デバイス、言語などの特定のターゲティング値ごとに分割されたメトリクスを取得できます。これは当社の非同期アナリティクスエンドポイントでのみ利用可能です。セグメント化されたメトリクスを取得するには、`segmentation_type` リクエストパラメータを使用してください。セグメンテーションの選択肢の詳細は「Metrics and Segmentation」を参照してください。

<div id="tailored-audience-list">
  ## テーラードオーディエンス - リスト
</div>

リスト由来のテーラードオーディエンスは、テーラードオーディエンスの3種類のうちの1つで、独自のユーザー／顧客データを含むファイルをアップロードして作成します。リストはX上のユーザーと照合され、キャンペーンでそのユーザーを直接ターゲティングできます。これらの制限事項をご確認ください。

<div id="tailored-audience-web">
  ## テーラードオーディエンス - Web
</div>

Web 由来のテーラードオーディエンスは、3 種類のテーラードオーディエンスの一つで、あなたのウェブサイトを訪れた X のユーザーをまとめたものです。Web 由来のテーラードオーディエンスを利用するには、ウェブサイトに X ピクセルを実装するか、当社が承認するタグマネージャーパートナーを利用する必要があります。これらの制限事項をご確認ください。

<div id="targeting-criteria">
  ## ターゲティング条件
</div>

前述のとおり、ターゲティング条件はラインアイテムに紐づきます。ロケーション、キーワード、性別から、ユーザーの行動や興味・関心といった、より細かな属性まで、さまざまなターゲティングが利用できます。「Home Buyers in San Francisco」というラインアイテムでは、「home buying」の行動と「San Francisco」のロケーションを組み合わせてターゲティングすることが想定されます。

利用可能なターゲティングの種類については、開発者サイトの Targeting Overview ページをご覧ください。

<div id="twitter-object-nest-ton">
  ## Twitter Object Nest (TON)
</div>

Twitter Object Nest（TON）は、Media Upload エンドポイント経由でアップロードされたメディアや各種アセットを格納し、クリエイティブとして利用するためのストレージです。

<div id="creatives">
  ## クリエイティブ
</div>

各ラインアイテムは、配信するには少なくとも1つのクリエイティブを紐づける必要があります。クリエイティブにはいくつかの種類があり、新規フォロワー獲得のための Promoted Accounts、X Audience Platform 全体のラインアイテムで使用できる画像や動画などの Media Creatives、あるいは Promoted Posts などがあります。以下は、@Furni がプロモーションしてクリエイティブとして使用できる投稿の例です。
