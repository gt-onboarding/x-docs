---
title: ページネーション
---

より多くのキャンペーンの最大数に対応し、アカウントに関連付けられたすべてのエンティティを効率的に取得できるようにするため、Advertiser API は複数の GET エンドポイントでページネーションをサポートするようになりました。このページング機構は使いやすく、[Using cursors to navigate collections](https://developer.x.com/en/docs/x-api/v1/pagination) で説明されている REST API のカーソル方式のページネーションと非常によく似ています。

<div id="getting-started">
  ## はじめに
</div>

特定の GET リクエストで、任意指定のクエリパラメータ `cursor` と `count` を受け付けるようになりました。レスポンスのエンティティ数が `count` を超える場合、該当エンドポイントは最初の `count` 件のエンティティと、レスポンス JSON に `next_cursor` キーを返します。

`GET https://ads-api.x.com/5/accounts/abc1/campaigns?count=50`

```json
{
  "data": \[...\],
  "next_cursor": "c-3yvu1pzhd3i7",
  "request": {...}
}
```

次のページの結果を取得するには、クエリパラメータに `cursor` を追加します:

`GET https://ads-api.x.com/5/accounts/abc1/campaigns?cursor=c-3yvu1pzhd3i7&count=50`

```json
{
  "data": \[...\],
  "next_cursor": "c-3w3zdyg8ywan",
  "request": {...}
}
```

<div id="going-deeper">
  ## さらに踏み込む
</div>

ほとんどのエンドポイントでは、`count` の最大値は **1,000**、最小値は **1**、デフォルトは **200** です。

`next_cursor` で提供される値は常に文字列であり、不透明な値として扱う必要があります。実装は変更される可能性があります。結果セットの現在のページで返されるエンティティが `count` 未満の場合、`next_cursor` の値は `null` になります。

**注:** アナリティクス系のエンドポイントはこの方式のページネーションをサポートしていません。統計系エンドポイントでのページングは、時間期間を指定することで行います。詳しくは各エンドポイントのドキュメントをご参照ください。
