---
title: レート制限
---

Advertiser API には、こちらに記載のとおり REST API v1.1 と同様のレート制限が適用されます: [REST API Rate Limiting in v1.1](https://developer.x.com/content/developer-twitter/en/docs/basics/rate-limits)。REST API v1.1 と異なり、エンドポイントごとの制限についてのプログラム的な一覧は提供されていません。エンドポイント単位のレート制限とリセットウィンドウは、HTTP レスポンスヘッダーを通じて通知されます。Ads API におけるすべてのレート制限は、OAuth 1.0A を使用します。

<div id="user-level-and-ad-account-level-limits">
  ## ユーザーレベルと広告アカウントレベルの制限
</div>

レート制限には、ユーザートークンレベルと広告アカウントレベルの2種類があります。エンドポイントの一部は、広告アカウントレベルのレート制限に対応しています。ユーザートークンは、Ads API の認証と呼び出しに使用する[OAuth access token](/ja/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)です。各ユーザートークンは、1つ以上の広告アカウントにアクセスできます。

開発者は、レスポンスヘッダーで返される場合は広告アカウントレベルのレート制限を使用し、広告アカウントの制限が見つからない場合にのみユーザーレベルの制限を使用してください。

ユーザーレベルのレート制限は、次のヘッダーで示されます：**x-rate-limit-limit**、**x-rate-limit-remaining**、**x-rate-limit-reset**

広告アカウントレベルのレート制限が有効なエンドポイントでは、レート制限は次のヘッダーで示されます：**x-account-rate-limit-limit**、**x-account-rate-limit-remaining**、**x-account-rate-limit-reset**。

広告アカウントレベルのレート制限は、単一のユーザートークンが複数の広告アカウントにアクセスしてエンティティデータ（キャンペーンやラインアイテムのオブジェクトなど）を同期できるように、GET エンドポイントに対してのみ提供されます。書き込みアクションが同じ広告アカウントレベルのレート制限を使用することは保証されていません。

広告アカウントレベルのレート制限が適用されるエンドポイントでは、ユーザーレベルのレート制限は、アプリケーション全体のグローバルクォータを表す高い値に設定されています。利用可能な場合は、リクエスト量の制御において広告アカウントレベルのレート制限を優先してください。

<div id="best-practices">
  ## ベストプラクティス
</div>

1. 最終同期時刻をデータベースに保存し、該当する場合はデータ取得時に `sort_by=updated_at-desc` オプションを指定して、最後に同期したタイムスタンプより古いデータに達した時点で同期処理を停止できるようにします。これにより、同じデータを重複して同期することを防げます。

2. 1回のリクエストで複数のエンティティを取得する: 一部のエンドポイントでは、カンマ区切りの値リストを指定することで、類似のデータを一度に複数取得できます。これにより、呼び出し回数を削減し、レート制限をより効率的に活用できます。

3. リクエストでは可能な限り最大の「count」を指定する: [GET accounts/:account\_id/targeting\_criteria](/ja/x-ads-api/campaign-management#targeting-criteria) など一部のエンドポイントは、デフォルトの200ではなく1000件のオブジェクトを返すよう、最大の count 値で呼び出すことが強く推奨されています。

<div id="faqs">
  ## よくある質問（FAQ）
</div>

_特定の広告アカウントまたは自社アプリケーションのレート制限を引き上げることは可能ですか？_

一般的にレート制限の引き上げはできず、最大規模の広告アカウントをサポートできるよう設定されています。まずは本ドキュメントに記載のベストプラクティスを実装してください。それでもレート制限がスケールやビジネス目標の達成に影響する場合は、ユースケースおよび関連リクエストの詳細を添えて、X 広告 API の担当窓口までご連絡ください。

<div id="scopes-of-rate-limiting">
  ## レート制限の適用範囲
</div>

<div id="scopes-for-this-doc">
  ### このドキュメントにおけるスコープ
</div>

- Category: 指定カテゴリに属するすべてのエンドポイントは、各ウィンドウで共有の割り当て上限に基づいてレート制限されます。
- Endpoint: 各エンドポイントは、各ウィンドウごとに個別の割り当て上限が適用されます。
