<div id="timezones-accounts-and-billing">
  ## タイムゾーン、アカウント、および課金
</div>

日時値は常に UTC（日時値の末尾の Z が示す）で返されます。日時は、POST または PUT コマンドで、ISO 8601 準拠のタイムゾーン表記を用いて任意のタイムゾーンで指定できます。時刻表現は [ISO-8601](http://en.wikipedia.org/wiki/ISO_8601) のサブセットに従います。具体的には、当社の日付形式に対する strptime 文字列は `%Y-%m-%dT%l:%M:%S%z` です。広告主アカウントのタイムゾーンが、公式な課金数値が確定される実際の時刻を決定します。

アカウントレベルで API にクエリする際（[GET accounts](/ja/x-ads-api/campaign-management#accounts)）、次のようなタイムゾーン情報が得られます:

```json
    {
      "request": {
        "params": {
          "account_id": "18ce54d4x5t"
        }
      },
      "data": {
        "name": "API McTestface",
        "business_name": null,
        "timezone": "America/Los_Angeles",
        "timezone_switch_at": "2016-07-21T07:00:00Z",
        "id": "18ce54d4x5t",
        "created_at": "2016-07-21T22:42:09Z",
        "salt": "54cb7b5a34183f77d82dd6d0f4329691",
        "updated_at": "2017-09-09T06:42:14Z",
        "business_id": null,
        "approval_status": "ACCEPTED",
        "deleted": false
      }
    }
```

ここには 2 つのタイムゾーン関連の値が含まれます：`timezone`（概要は[Wikipedia のタイムゾーン](https://en.wikipedia.org/wiki/Tz_database#Names_of_time_zones)を参照）と `timezone_switch_at`。`timezone_switch_at` は UTC（+00:00）で示されますが、常に指定されたタイムゾーンの午前0時を表します。

広告主のタイムゾーンは API では変更できません。この属性は、X の広告主担当アカウントマネージャーにより契約／課金レベルで設定されます。

アカウントが America/Los\_Angeles のタイムゾーンから新しいローカル値へ切り替わる当日にはギャップが生じるため、レポート作成やアナリティクスエンドポイントのクエリ時は `timezone_switch_at` の値に留意してください。

<div id="specifying-datetime-values-with-timezone">
  ### タイムゾーン付き日時値の指定
</div>

日時値は常に UTC（日時値の末尾の `Z` で示されます）で返されます。日時は、タイムゾーンの表記として [ISO 8601 standard](http://en.wikipedia.org/wiki/ISO_8601) 形式を用い、POST または PUT コマンドで任意のタイムゾーンで指定できます。たとえば、`2017-07-10T08:00:00-0800` は有効な入力値であり、自動的に UTC の値 `2017-07-10T16:00:00Z` に変換されます。

`granularity` が `DAY` または `TOTAL` のアナリティクスエンドポイントを使用する場合、`start_time` の値は、アカウント保有者のローカルタイムゾーンにおける対象日の午前0時（00:00）で指定する必要があります。使用されるタイムゾーンオフセットは、対象日ではなく「現在日」のオフセットになります。たとえば、America/Los\_Angeles の広告アカウントで夏時間（Pacific Daylight Time）期間中は、UTC オフセットは -0700 です。したがって、アナリティクスリクエストでは、`start_time=2017-05-21T07:00:00Z` または `start_time=2017-05-21T00:00:00-0700` のように指定します。広告アカウントが Asia/Tokyo にあり、常に +09:00 のオフセットである場合、`start_time=2017-05-20T15:00:00Z` または `start_time=2017-05-21T00:00:00+0900` のように指定します。

<div id="accepted-utc-offset-formats">
  ### 受け付け可能な UTC オフセットの形式
</div>

[ISO 8601 のタイムゾーン指定子](http://en.wikipedia.org/wiki/ISO_8601#Time_zone_designators)を参照してください。

サポートされている形式: `Z`, `-HHMM`, `+HHMM`, `-HH:MM`, `+HH:MM`, `-HH`, `+HH`
