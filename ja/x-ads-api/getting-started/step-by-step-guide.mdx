<div id="how-to-get-access-to-the-ads-api">
  ## X 広告 API へのアクセス方法
</div>

1. [開発者アカウント](https://developer.x.com/en/apply-for-access)に登録します。
2. [開発者アプリ](https://developer.x.com/app)を作成し、トークンを安全に管理します。
3. [ads.x.com/help](https://ads.x.com/help) にアクセスし、各開発者アプリごとに X 広告 API へのアクセス申請を行います。

<Note>**注意:** すでに X Developer Platform で開発しており、開発者アカウントをお持ちの場合は、手順3に進んでください。</Note>

<div id="step-one-signup-for-a-developer-account">
  ### ステップ1：開発者アカウントに登録する
</div>

X の各種 API を利用するには、まず開発者アカウントに登録してください。開発者ポータルで、プロジェクトと開発者アプリを作成します。これにより、API へのすべてのリクエストで使用する認証用のクレデンシャル一式が発行されます。

<Button href="https://developer.x.com/en/apply-for-access">
  開発者アカウントに登録
</Button>

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### ステップ2：アプリのキーとトークンを保存し、安全に保管する
</div>

開発者用アプリには、API Keys（Consumer Keys とも呼ばれます）が発行されます。加えて、個人の X アカウントの代理でリクエストを送るために使用できる Access Tokens と、OAuth 2.0 Bearer Token を必要とするエンドポイントの認証に使用できる Bearer Token を生成できます。これらのキーとトークンは再生成しない限り有効期限がないため、環境変数の利用や、安全なパスワードマネージャーの使用を推奨します。

また、App の詳細ページの URL に含まれる App ID も控えておいてください。これは次のステップで X 広告 API へのアクセスを申請する際に役立ちます。

<Note>
**注:** キーとトークンは開発者ポータル上で一度しか表示されません。生成したらすぐに、これらの認証情報をパスワード管理システムに保管することが重要です。

キーやトークンを紛失または失念した場合は再生成が必要となり、新しいキーとトークンが作成され、既存のものは無効化されます。つまり、以前の認証情報で設定していた統合はすべて更新する必要があります。

[認証のベストプラクティス](https://developer.x.com/en/docs/authentication/guides/authentication-best-practices)の詳細をご覧ください。
</Note>

<div id="step-three-apply-for-access-to-the-ads-api">
  ### ステップ3：X 広告 API へのアクセスを申請する
</div>

この時点では[基本的な X API へのアクセス](https://developer.x.com/en/docs/twitter-api/v1)はありますが、X 広告 API の機能にはまだアクセスできません。次に、広告 API へのアクセスを申請し、承認を受ける必要があります。各開発者アプリごとに広告 API アクセスの申請を行うには、[ads.x.com/help](https://ads.x.com/help) にアクセスしてください。

<div id="access-tiers">
  ### アクセスレベル
</div>

申請手続きの一環として、必要とするアクセスレベルを指定する必要があります。詳しくは、[アプリレベルおよび広告アカウントレベルの権限](https://developer.x.com/en/docs/twitter-ads-api/obtaining-ads-account-access#levels-of-access)をご覧ください。

<div id="conversion-only">
  #### Conversion Only
</div>

モバイルおよびウェブのコンバージョン向けエンドポイントへの読み取り・書き込み権限。

<div id="standard-access">
  #### スタンダードアクセス
</div>

Analytics、キャンペーン管理、クリエイティブ、カスタムオーディエンス、Conversion の各エンドポイントに対する読み取り・書き込みアクセス。

<Note>**注:** アプリが X 広告 API のアクセス承認を受けた後、[認証済みの Ads API リクエスト](https://developer.x.com/en/docs/twitter-ads-api/making-authenticated-requests)を正しく行うために、[ユーザーアクセストークンを再生成](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)する必要があります。</Note>

<Note>**注:** 2023 年 7 月以前にアクセスを申請した Ads API 開発者は、アクセスレベルや権限が異なる場合があり、OAuth トークンが 5 個に制限されている可能性があります。追加エンドポイントへのアクセスや既存アプリのトークン上限解除については、[アクセス拡張](https://developer.x.com/en/docs/twitter-ads-api/increasing-access)ガイドをご覧ください。</Note>

<div id="make-your-first-request">
  ## 最初のリクエストを実行する
</div>

X 広告 API へのアクセスをテストするには、[GET accounts](https://developer.x.com/en/docs/twitter-ads-api/campaign-management/api-reference/accounts) エンドポイントにリクエストを送信します。このエンドポイントは、現在の認可ユーザーがアクセス可能な広告アカウントを返します。以降の API リクエストでは、このレスポンスで取得した広告アカウント ID を使用して、特定の広告アカウントのデータを読み書きします。コマンドラインで [Twurl](https://developer.x.com/en/docs/tutorials/using-twurl) を使用する場合、リクエストは次のようになります。

**リクエスト例**

```bash
twurl -H ads-api.x.com "/11/accounts"
```

**レスポンス例**

```json
{
  "request": {
    "params": {}
  },
  "data": [
   {
    "name": "Furni",
    "business_name": null,
    "timezone": "America/Los_Angeles",
    "timezone_switch_at": "2016-04-06T07:00:00Z",
    "id": "18ce54ayf0z",
    "created_at": "2016-04-07T14:40:15Z",

    "business_id": null,

    "deleted": false
     }

```
