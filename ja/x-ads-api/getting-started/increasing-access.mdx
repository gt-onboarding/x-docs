<div id="requesting-additional-app-level-permissions">
  ### 追加のアプリレベル権限のリクエスト
</div>

アプリケーションが以下のいずれかのアプリレベル権限に限定されており、追加のエンドポイントへアクセスする必要がある場合は、**Standard Access** の申請について X の担当者までご連絡ください。

2023年7月以前に Ads API へのアクセスを申請した開発者は、以下のいずれかのアプリレベル権限を個別に申請していた可能性があります。現在はこれらが Standard Access レベルに含まれます。

- **Analytics**（読み取りのみ）: ユーザーは [Analytics](https://developer.x.com/en/docs/twitter-ads-api/analytics/overview) エンドポイントに [読み取り専用](https://developer.x.com/en/docs/apps/app-permissions) でアクセスできます。ユーザーは Campaign Management、Creative、Custom Audiences の各エンドポイントにはアクセスできませんが、Analytics エンドポイントのリクエストパラメータでクリエイティブエンティティを指定することでメトリクスを取得できます。media_id または media_name の粒度で取得するには、ユーザーのアプリケーションに Ads API の書き込み権限が必要です。

- **Campaign Management & Creative**（読み取りと書き込み）: ユーザーは [Campaign Management](https://developer.x.com/en/docs/twitter-ads-api/campaign-management/overview) および [Creative](https://developer.x.com/en/docs/twitter-ads-api/creatives/overview) エンドポイントに [読み取りと書き込み](https://developer.x.com/en/docs/apps/app-permissions) でアクセスできます。ユーザーは Analytics エンドポイントにもアクセスできます。

- **Custom Audiences**（読み取りと書き込み）: ユーザーは [Custom Audiences](https://developer.x.com/en/docs/twitter-ads-api/audiences/overview) エンドポイントに
[読み取りと書き込み](https://developer.x.com/en/docs/apps/app-permissions) でアクセスできます。ユーザーは Campaign Management、Creative、Analytics の各エンドポイントにもアクセスできます。

- **Conversion**（読み取りと書き込み）: ユーザーは [Mobile](https://developer.x.com/en/docs/twitter-ads-api/measurement/mobile-conversions/overview) および [Web Conversion](https://developer.x.com/en/docs/twitter-ads-api/measurement/web-conversions/overview) エンドポイントに [読み取りと書き込み](https://developer.x.com/en/docs/apps/app-permissions) でアクセスできます。ユーザーが他の Ads API エンドポイントへアクセスできるかどうかは場合により異なります。 

<div id="requesting-an-increase-to-user-oauth-token-limits">
  ### ユーザー OAuth トークン上限の引き上げをリクエストする
</div>

2023年7月以前に X 広告 API へのアクセスを申請した開発者のアプリケーションでは、ユーザー OAuth トークンの上限が 5 に設定されている場合があります。開発者向けアプリのトークン上限の引き上げが必要な場合は、担当の X 窓口までご連絡ください。

<Note>**注:** いずれの X 広告 API エンドポイントに対しても、[レート制限](https://developer.x.com/en/docs/twitter-ads-api/rate-limiting)の引き上げは行っていません。レート制限内に収める方法については、[ベストプラクティス](https://developer.x.com/en/docs/twitter-ads-api/rate-limiting)をご確認ください。</Note>
