<div id="mact-overview">
  ## MACT の概要
</div>

X の[モバイルアプリプロモーション](https://biz.x.com/mobile-app-promotion)の計測機能により、X 上でインストールやその他のアプリ内コンバージョンの促進を目的として設計された広告キャンペーンの成果を、広告主が追跡できます。X のモバイル計測パートナーは、広告主が X でプロモーションしているアプリにおいて、どのコンバージョンを追跡するかを管理できるようにします。広告主が追跡対象のイベントの種類と計測ウィンドウを設定すると、モバイル計測パートナーはそれらのイベントを X に送信し、コンバージョンに至るまでにユーザーが関与したキャンペーンやクリエイティブを示すアトリビューションデータを即時に受領します。

<div id="conversion-events">
  ### コンバージョンイベント
</div>

[コンバージョンイベント](https://developer.x.com/en/docs/x-ads-api/measurement/api-reference/conversion-event)をXに報告してアトリビューションデータを取得するには、パートナーはモバイルアプリID、コンバージョンの種類とタイムスタンプに加え、広告ID／デバイスIDのハッシュ化値を送信する必要があります。これらのデータを提供すると、X のプラットフォーム上でコンバージョンイベントが記録され、該当する場合にはアトリビューション情報が返されます。さらに、広告主は購入額、商品のID、登録方法など、コンバージョンイベントに関する追加のメタデータを任意で提供できます。

#### POST conversion\_event

モバイル計測のコンバージョンイベントを記録します。レスポンスには、X または X Audience Platform（TAP）のアトリビューションが示されます。

これは [GET conversion\_attribution](https://developer.x.com/en/docs/x-ads-api/measurement/api-reference/conversion-attribution#get-conversion-attribution) エンドポイントに関連します。

レスポンスでは、X、TAP、またはアトリビューションなしのいずれかが返されます。X のアトリビューションがない場合、`twitter_attribution` ノードは常に存在し、その値は `null` になります（下記のレスポンス例を参照）。TAP のアトリビューションがある場合は、`tpn_attribution` ノードが存在し、適切に設定されます。詳細は [TAP overview](https://developer.x.com/en/docs/x-ads-api/measurement/overview/twitter-audience-platform) を参照してください。

各コンバージョンイベントに関連付けるメタデータを設定できる任意のパラメータがいくつかあります。これらのメタデータはアトリビューションの計算には影響しません。

<div id="resource-url">
  ##### リソース URL
</div>

`https://ads-api.x.com/12/conversion_event`

<div id="parameters">
  ##### パラメータ
</div>

| Name | Description |
| :--- | :--- |
| app\_id  <br />_必須_ | 対応するアプリストアにおける一意の識別子。<br /><br />Type: int, string<br /><br />Example: `333903271`, `com.vine.android` |
| conversion\_time  <br />_必須_ | ミリ秒付きの ISO-8601 タイムスタンプ形式によるコンバージョンイベントの時刻。<br /><br />Type: string<br /><br />Example: `2014-05-22T02:38:28.103Z` |
| conversion\_type  <br />_必須_ | コンバージョンイベントの種類。<br /><br />Type: enum<br /><br />Possible values: `PURCHASE`, `SIGN_UP`, `INSTALL`, `RE_ENGAGE`, `UPDATE`, `TUTORIAL_COMPLETE`, `RESERVATION`, `ADD_TO_CART`, `ADD_TO_WISHLIST`, `LOGIN`, `CHECKOUT_INITIATED`, `SEARCH`, `LEVEL_ACHIEVED`, `ACHIEVEMENT_UNLOCKED`, `CONTENT_VIEW`, `SHARE`, `INVITE`, `ADDED_PAYMENT_INFO`, `SPENT_CREDITS`, `RATED` |
| hashed\_device\_id  <br />_必須_ | HMAC\_SHA-256 でハッシュ化された IDFA または AdID。<br /><br />Type: string<br /><br />Example: `ABCD1234XYZ` |
| os\_type  <br />_必須_ | アプリの OS 種別。<br /><br />Type: enum<br /><br />Possible values: `IOS`, `ANDROID` |
| click\_window  <br />_任意_ | このイベントのクリック計測期間（日）。<br /><br />Type: int<br /><br />Note: `click_window` は view\_through\_window 以上である必要があります。<br /><br />Default: `14`  <br />Possible values: `1`, `7`, `14`, `30` |
| device\_ip\_address  <br />_任意_ | コンバージョンイベント発生時のデバイスの IPv4 または IPv6 アドレス。<br /><br />Type: string<br /><br />Example: `192.133.78.1` |
| level  <br />_任意_ | このイベントに関連するレベル。<br /><br />Type: int<br /><br />Example: `2` |
| non\_twitter\_engagement\_time  <br />_任意_ | コンバージョン前の直近の非 Twitter エンゲージメントの時刻。<br /><br />Type: string<br /><br />Example: `2014-05-22T02:38:28.103Z` |
| non\_twitter\_engagement\_type  <br />_任意_ | コンバージョンイベント前の非 Twitter エンゲージメントの種類。<br /><br />Type: enum<br /><br />Possible values: `CLICK`, `VIEW` |
| number\_items  <br />_任意_ | このイベントに関連するアイテム数。<br /><br />Type: int<br /><br />Example: `2` |
| price\_currency  <br />_任意_ | このイベントに関連する通貨を示す ISO 4217 コード。<br /><br />Type: string<br /><br />Examples: `EUR`, `USD`, `JPY` |
| price\_micro  <br />_任意_ | このイベントに関連する価格（マイクロ単位）。<br /><br />Type: int<br /><br />Example: `123450000` |
| user\_payment\_info  <br />_任意_ | このイベントに関連するアプリにユーザーの支払い情報が保存されているかを示すブール値。<br /><br />Type: bool<br /><br />Possible values: `true` or `false` |
| view\_through\_window  <br />_任意_ | このイベントのビュー経由計測期間（日）。<br /><br />Note:<br /><br />`click_window` は view\_through\_window 以上である必要があります。<br /><br />Type: int<br /><br />Default: `1`  <br />Possible values: `0`, `1`, `7`, `14`, `30` |

#### GET accounts/:account\_id/app\_event\_tags

現在のアカウントに関連付けられている一部またはすべてのアプリイベントタグの詳細を取得します。

これらは、Mobile App Conversion Tracking で各コンバージョンタイプごとに設定されるコンバージョンウィンドウを定義します。

<div id="resource-url">
  ##### リソース URL
</div>

`https://ads-api.x.com/12/accounts/:account_id/app_event_tags`

<div id="parameters">
  ##### パラメータ
</div>

| Name | Description |
| :--- | :--- |
| account\_id  <br />_required_ | 利用するアカウントの識別子。リソースのパス内に含まれ、[GET accounts](/ja/x-ads-api/campaign-management#accounts) を除くすべての Advertiser API リクエストで通常必須のパラメータです。指定したアカウントは認証済みユーザーに関連付けられている必要があります。<br /><br />Type: string<br /><br />Example: `18ce54d4x5t` |
| app\_event\_tag\_ids  <br />_optional_ | コンマ区切りの識別子リストを指定して、必要なアプリイベントタグのみにレスポンスを絞り込みます。最大 200 個の ID を指定できます。<br /><br />Type: string<br /><br />Example: `jhp` |
| count  <br />_optional_ | リクエストごとに取得を試みるレコード数を指定します。<br /><br />Type: int<br /><br />Default: `200`  <br />Min, Max: `1`, `1000` |
| cursor  <br />_optional_ | 次ページの結果を取得するためのカーソルを指定します。詳細は [Pagination](/ja/x-ads-api/introduction) を参照してください。<br /><br />Type: string<br /><br />Example: `8x7v00oow` |
| sort\_by  <br />_optional_ | サポートされる属性で昇順または降順にソートします。詳細は [Sorting](/ja/x-ads-api/fundamentals/sorting) を参照してください。<br /><br />Type: string<br /><br />Example: `created_at-asc` |
| with\_deleted  <br />_optional_ | 削除済みの結果を含めます。<br /><br />Type: boolean<br /><br />Default: `false`  <br />Possible values: `true`, `false` |
| with\_total\_count  <br />_optional_ | レスポンス属性 `total_count` を含めます。<br /><br />**Note**: このパラメータと `cursor` は排他です。<br /><br />**Note**: `total_count` を含むリクエストはレート制限が低く、現在は 15 分あたり 200 に設定されています。<br /><br />Type: boolean<br /><br />Default: `false`  <br />Possible values: `true`, `false` |

#### GET conversion\_attribution

レスポンスでは、X、TAP、またはアトリビューションなしのいずれかが示されます。`twitter_attribution` ノードは常に含まれ、X のアトリビューションがない場合は値が `null`、ある場合は以下の例のとおり値が設定されます。TAP のアトリビューションがある場合は、`tpn_attribution` ノードが含まれ、適切な値が設定されます。詳細は [TAP overview](https://developer.x.com/en/docs/twitter-ads-api/measurement/twitter-audience-platform/overview) を参照してください。
