<div id="overview">
  ## 概要
</div>

Catalog API は、広告主が商品フィードの設定、商品をセットへのグルーピング、カタログ商品の一元管理を行えるコマースソリューションです。Catalog API は、プログラムによるカタログ管理を可能にする中核ツールであり、カタログの取り込みと更新方法に対して広告主により大きなコントロールを提供します。

カタログは、ユーザーの商品の情報および商品セットに関するすべてを保持し、ユーザーのハンドルに関連付けられます。現在、**1 人のユーザーが作成できるカタログは 1 つのみです**。 

Catalog API は、商品の取り込み方法として次の 2 つをサポートします。

1.  スケジュール済みフィード: ユーザーはフィード URL（CSV、TSV、XML）を追加でき、定期的に取得されます（X では現在、最大 8GB のファイルをサポート）

2.  Batch Products API: ユーザーはバッチ（JSON）リクエストで商品の属性の閲覧、作成、更新、削除が可能

X のマーチャントは、フィルタールールに基づいて商品セットを作成できます。これにより、商品に追加のメタデータを付与し、オーガニックなコマース機能や動的商品広告でそれらを紹介するための特別な商品セットを作成できます。Catalog API と X Shopping Manager は 2 種類の商品セットをサポートしています。 

1.  手動: ユーザーは最大 50 点の商品を選択し、名前と説明を設定できる

2.  フィルター: ユーザーは最大 30 個のフィルターを追加して、これらのフィルターを満たす商品で構成されるセットを自動生成できる。フィルターの種類には、価格、Google 商品カテゴリ、商品タイプ、在庫、セール価格、カスタムフィールドが含まれる。

Catalog API には、カタログを管理するための 4 つのエンドポイントがあります:

* [Product Catalogs](https://developer.x.com/en/docs/x-ads-api/catalog-management/api-reference/product-catalogs) \- 商品情報のコンテナを作成
* [Products](https://developer.x.com/en/docs/x-ads-api/catalog-management/api-reference/products) -  カタログへの商品の同期的な追加、更新、閲覧、削除
* [Product Sets](https://developer.x.com/en/docs/x-ads-api/catalog-management/api-reference/product-sets) -  商品情報に基づいて商品をグループに分類
* [Scheduled Feeds](https://developer.x.com/en/docs/twitter-ads-api/catalog-management/api-reference/scheduled-feeds) \- ファイルベースの商品データをカタログへ非同期取り込みする設定

<div id="prerequisites-for-the-catalog-api">
  ### Catalog API の前提条件
</div>

Catalog API のエンドポイントは現在、アーリーアクセスでのみご利用いただけます。アクセスを申請する場合は、担当の X の窓口までご連絡ください。

[X Shopping Manager](https://ads.x.com/shopping_manager) 上で、利用規約に同意していただく必要があります。

Shopping Manager および製品仕様の詳細は、[Product specifications guide](https://business.x.com/en/help/shopping-specs.html) をご参照ください。
