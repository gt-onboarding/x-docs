<div id="situation">
  ## 状況
</div>

これまでデジタルマーケターは自社データを一元的に把握できていませんでした。オーガニックのデータはあるシステム、ペイドのデータは別のシステム、会話はさらに別の場所から、という具合です。しかし、クライアントやステークホルダーから「昨日のインプレッション数は？」「オーガニックとペイドの内訳は？」と矢継ぎ早に問われる中で、こうしたデータを効率的に集約・解釈する手段の必要性はこれまで以上に高まっています。

この課題を解決するため、X Developer Platform と10年以上にわたり連携してきたソーシャルアナリティクスプラットフォームの ListenFirst は、単一の信頼できる情報源からオーガニックおよびペイドの X データを収集できるツールの構築を必要としていました。従来、これらのデータは異なるプラットフォーム、API、エンドポイントにまたがる別々のサイロに分断されており、集約・合算・ブレンドされることは想定されていませんでした。

<div id="solution">
  ## 解決策
</div>

クライアント向けのインサイト創出をより効率化するため、ListenFirst は X の v2 エンドポイントを採用し、X 上でユーザーが話題にしている内容を自社プラットフォームへ変換・反映するプロセスを合理化しました。ListenFirst のチームは「よりクリーンなデータを作る」という明確な目標を掲げて取り組みを開始しました。

> *「私たちにとって v2 の活用は、統合の最適化にほかなりません。自社サイトで組み合わせる部品が少なくなれば、運用は容易になり、より多くのブランド、より多くのコンテンツ、そしてより優れたインサイトへと提供を拡大できます。」*
> 
> - ListenFirst 最高製品責任者 Jonathan Farb

X の v2 API、特に Tweets lookup エンドポイントの採用を優先した理由は、エンゲージメント数とインプレッション数の正確性にありました。このエンドポイントを活用することで、ListenFirst は、他の場所で顧客が目にするネイティブの X アナリティクスと整合する、より信頼性の高いメトリクスを生成できました。

<div id="impact">
  ## 影響
</div>

まだ黎明期ではあるものの、v2 の採用は ListenFirst の業務効率に好影響をもたらしています。従来、ListenFirst は複数の API 群からデータを寄せ集める必要があり、数値はネイティブのアナリティクスと整合していませんでした。v2 の活用により、ListenFirst の数値は X のネイティブアナリティクスに近づき、精度と顧客からの信頼が向上しました。現在、ListenFirst は、ブランドが X 上でマーケティング戦略を完成させるのに資する、説明可能で洗練されたインサイトを提供できます。ListenFirst は、v2 のさらなる採用が、精度と透明性の向上を通じてクライアントのメトリクスに対する信頼を高めると考えています。
