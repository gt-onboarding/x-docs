<div id="overview">
  ## 概要
</div>

フィルタードストリームのエンドポイントグループは、開発者が公開投稿のストリームをフィルタリングできるようにします。このエンドポイントグループには、ルールの作成・管理を行い、それらのルールを適用して一致する公開投稿のみを返すための複数のエンドポイントが含まれます。これにより、特定のトピックやイベントの監視、競合に関する会話のモニタリング、トレンド形成の把握などが可能になります。

開発者は REST の[rules endpoint](/ja/x-api/posts/filtered-stream#post-2-tweets-search-stream-rules)を使用して、定義した条件に一致する投稿をフィルタリングするルールを追加・削除できます。これらの[ルール](/ja/x-api/posts/filtered-stream#building-rules-for-filtered-stream)は、メッセージのキーワード、ハッシュタグ、URL などの投稿属性にマッチするオペレーターで作成できます。オペレーターやルール句は、ブールロジックや括弧と組み合わせて、フィルタのマッチング動作を精緻化できます。 

一連のルールを追加したら、Webhook を登録すると、X から JSON 形式の[Post objects](/ja/x-api/fundamentals/data-dictionary#tweet)の配信が開始されます。Webhook には、ルールに一致するコンテンツのみが配信されます。

このエンドポイントは編集済みの投稿をサポートします。Webhook には、1 つ以上のフィルタに一致する編集済みの投稿が、その編集履歴（投稿 ID の配列を含む）とともに配信されます。編集履歴がない投稿では、この配列には単一の ID が入ります。編集が行われた投稿では、この配列には複数の ID が入り、編集の順序を反映して昇順に並び、配列の最後に最新バージョンが配置されます。投稿の編集の仕組みについては、[Posts edits fundamentals](/ja/x-api/fundamentals/edit-posts)をご覧ください。 

**注:** このエンドポイントグループは、ルールの設定方法は[v2 filtered stream endpoint](/ja/x-api/posts/filtered-stream/introduction)と同様です。違いは、ルールに一致した投稿の配信メカニズムのみです。

- v2 filtered stream endpoint の場合は、永続的な接続を確立し、ルールに一致する投稿をリッスンする必要があります。
- この filtered stream webhook endpoint の場合は、Webhook を登録すると、X がルールに一致する投稿をあなたの Webhook に配信します。

これにより、クライアント側の切断や、当社サービスの通常の再起動に起因するサーバー側の切断への対応が不要になります。毎秒数百件のイベントを配信する可能性が高いため、Webhook サーバーはこれらの同時接続試行を処理できる必要があります。

このエンドポイントは現在 Enterprise 開発者が利用可能で、以下の設定をサポートします:

- プロジェクトあたり 25,000 件以上のルール
- ルール作成時にすべてのオペレーターを使用可能
- 最大 2048 文字のルールを作成可能
- Enterprise アクセスは[こちら](https://developer.x.com/en/products/x-api/enterprise/enterprise-api-interest-form)から申請

フィルタードストリームから返される投稿は、月次の[投稿上限](/ja/x-api/fundamentals/post-cap)にカウントされます。

<Note>
  **アカウント設定**

  これらのエンドポイントにアクセスするには、以下が必要です:

  - 承認済みの[開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)。
  - [プロジェクト](/ja/resources/fundamentals/projects)内の[開発者アプリ](/ja/resources/fundamentals/developer-apps)のキーとトークンを使用して認証すること。 

  X API v2 のエンドポイントへのアクセスについては、[はじめにガイド](/ja/x-api/getting-started/getting-access)をご覧ください。
</Note>
