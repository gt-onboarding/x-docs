[X Lists](https://help.x.com/en/using-twitter/twitter-lists) は、ユーザーがタイムラインに表示される投稿をカスタマイズ、整理、優先順位付けできる機能です。Lists エンドポイントを使用すると、嗜好、関心、グループ、トピックに基づいて投稿をキュレーションおよび整理できるソリューションを構築できます。

manage List エンドポイントを用いてユーザーに代わりリクエストを行う場合、これらのエンドポイントは [OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかで認可し、あなたのアプリを承認したユーザーに紐づくユーザー Access Token を使用する必要があります。これらのトークンは、[3-legged OAuth flow](/ja/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）または [Authorization Code with PKCE grant flow](/ja/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)（OAuth 2.0）で生成できます。

<div id="manage-lists">
  ### リストの管理
</div>

リスト管理エンドポイントを使用すると、認証済みユーザーに代わってリストを作成、削除、更新できます。これらのエンドポイントでは POST、DELETE、PUT の3つのメソッドが利用可能です。POST はリストの作成、DELETE はリストの削除、PUT はリストのメタデータの更新を行います。これら3つのエンドポイントには、ユーザーあたり15分間に300リクエストのレート制限が適用されます。

アカウントごとに最大1000件のリストを作成できます。

**アカウントの設定**

これらのエンドポイントにアクセスするには、以下が必要です:

- 承認済みの[開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)。
- [プロジェクト](/ja/resources/fundamentals/projects)内の[開発者アプリ](/ja/resources/fundamentals/developer-apps)のキーとトークンを使用した認証。 

X API v2 エンドポイントへのアクセス方法の詳細は、[はじめにガイド](/ja/x-api/getting-started/getting-access)をご覧ください。
