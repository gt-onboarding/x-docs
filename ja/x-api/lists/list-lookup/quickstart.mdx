<div id="getting-started-with-the-list-lookup-endpoint">
  ## List lookup エンドポイントの使い方を始める
</div>

このクイックスタートガイドでは、Postman を使用して List lookup エンドポイントへ初めてのリクエストを送る手順を説明します。

さまざまな言語のサンプルコードを確認したい場合は、[X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) の GitHub リポジトリをご覧ください。

**注:** この例では、_List lookup by ID_ エンドポイントにリクエストを送信しますが、このクイックスタートで学んだ内容は他の lookup リクエストにも適用できます。

<Note>
  ### 前提条件

  このガイドを完了するには、リクエストを認証するための[キーとトークン](/ja/resources/fundamentals/authentication)が必要です。以下の手順でキーとトークンを生成できます。

  - [開発者アカウントに登録](https://developer.x.com/en/apply-for-access)し、承認を受けます。
  - 開発者ポータルで[プロジェクト](/ja/resources/fundamentals/projects)と、それに関連付けた[開発者アプリ](/ja/resources/fundamentals/developer-apps)を作成します。
  - アプリの「Keys and tokens」ページに移動して、必要な認証情報を生成します。すべての認証情報は安全な場所に保存してください。
</Note>

<div id="steps-to-build-a-list-lookup-request">
  ### List lookup リクエストを構築する手順
</div>

**手順1: ツールまたはライブラリから始める**

このエンドポイントにリクエストする方法として、いくつかのツール、コード例、ライブラリがありますが、ここでは手順を簡潔にするため Postman を使用します。

X API v2 の Postman コレクションを環境に読み込むには、次のボタンをクリックしてください:

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  Add X API v2 to Postman
</Button>

Postman に X API v2 コレクションを読み込んだら、「List」フォルダーに移動し、「List lookup」フォルダーを選択してから「List by ID」を選びます。
 

**手順2: リクエストを認証する**

X API に適切にリクエストするには、権限があることを確認する必要があります。このエンドポイントでは、[App only](/ja/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)、[OAuth 2.0 Authorization Code with PKCE](ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)、または [OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) のいずれかで認証してください。

簡便化のため本手順では App only を使用しますが、非公開の[メトリクス](/ja/x-api/fundamentals/metrics)や List にアクセスする場合は、他のいずれかの認証方法が必要です。 

App only を使うには、Postman で「X API v2」という環境（Postman 右上）を選択し、キーとトークン（特に[App only Access Token](/ja/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)）を「initial value」と「current value」フィールドに追加します（環境ドロップダウン横の目のアイコンをクリック）。

正しく設定されていれば、これらの変数は自動的にリクエストの Authorization タブに反映されます。
 

**手順3: 取得したい List を特定して指定する**

リクエスト内で取得する List を指定する必要があります。List ID は x.com で対象の List を開き、URL を確認すると見つかります。たとえば、次の URL の List ID は 84839422 です。

https://x.com/i/lists/84839422

対象 ID は有効な List ID であれば任意で構いません。Postman で「Params」タブに移動し、id パス変数の「Value」列に ID を入力します。ID の前後にスペースを入れないよう注意してください。

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| id  | 84839422（この List の ID） |

**手順4: 取得したいフィールドを特定して指定する**

手順3の後に「Send」ボタンをクリックすると、レスポンスには既定の [List オブジェクト](/ja/x-api/fundamentals/data-dictionary#list)のフィールド（id, name）が返されます。

追加のフィールドを取得したい場合は、list.fields および／または [expansions](/ja/x-api/fundamentals/expansions) パラメータでそれらのフィールドを指定します。

この演習では、異なるオブジェクトから次の 3 つの追加フィールドセットを要求します:

- プライマリの Lists オブジェクトにおける追加フィールド created\_at

- expansions パラメータを使用した完全な [user オブジェクト](/ja/x-api/fundamentals/data-dictionary#user)

- 関連する user オブジェクトにおける追加フィールド user.created\_at

Postman で「Params」タブに移動し、「Query Params」テーブルに次の key:value ペアを追加します:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| list.fields | created\_at | created\_at |
| expansions | owner\_id | includes.users.id,  <br />includes.users.name,  <br />includes.users.username |
| user.fields | created\_at | includes.users.created\_at |

これで「Send」ボタンの横に次のような URL が表示されるはずです:

`https://api.x.com/2/lists/84839422?list.fields=owner_id&expansions=owner_id&user.fields=created_at`

**手順5: リクエストを送信し、レスポンスを確認する**

すべての設定が完了したら、「Send」ボタンを押すと、次の例と同様のレスポンスが返されます:

```
{
  "data": {
    "id": "84839422",
    "name": "Official Twitter Accounts",
    "owner_id": "783214"
  },
  "includes": {
    "users": [
      {
        "name": "Twitter",
        "created_at": "2007-02-20T14:35:54.000Z",
        "username": "Twitter",
        "id": "783214"
      }
    ]
  }
}
```
