リストをピン留めすると、X のユーザーは自分が購読しているリストや自身で作成したリストに、より簡単にアクセスできるようになります。

これらのエンドポイントでユーザーに代わってリクエストを行う場合は、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) で当該ユーザーを認証し、あなたのアプリを承認したユーザーに紐づくユーザー Access Token を使用する必要があります。これらのトークンは、[3-legged OAuth flow](/ja/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）または [Authorization Code with PKCE grant flow](/ja/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)（OAuth 2.0）で生成できます。

<div id="pinned-list-lookup">
  ### ピン留めリストのルックアップ
</div>

ピン留めリストのルックアップには、認証済みユーザーのピン留めリストを取得できるエンドポイントが1つあります。このエンドポイントのレート制限は15分あたり15リクエストです。
 

<div id="manage-pinned-lists">
  ### ピン留めリストを管理する
</div>

ピン留めリスト管理エンドポイントでは、認証済みユーザーに代わってリストをピン留め／解除できます。これらのエンドポイントには POST と DELETE の2つのメソッドがあります。POST はリストをピン留めし、DELETE はピン留めを解除します。いずれのエンドポイントも、ユーザーあたり15分間に50リクエストのレート制限が適用されます。

<Note>
  **アカウントのセットアップ**

  これらのエンドポイントにアクセスするには、次が必要です。

  - 承認済みの [developer account](https://developer.x.com/en/portal/petition/essential/basic-info)。
  - [Project](/ja/resources/fundamentals/projects) 内の [developer App](/ja/resources/fundamentals/developer-apps) のキーとトークンを使用した認証。 

  X API v2 エンドポイントへのアクセス方法の詳細は、[getting started guide](/ja/x-api/getting-started/getting-access) を参照してください。
</Note>
