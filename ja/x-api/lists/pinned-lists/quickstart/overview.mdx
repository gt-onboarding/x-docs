<div id="getting-started-with-the-pinned-list-endpoint-group">
  ## ピン留めリストのエンドポイントグループのはじめ方
</div>

このクイックガイドでは、[Postman](/ja/tutorials/postman-getting-started) を使用してピン留めリストのエンドポイントへの最初のリクエストを行う方法を説明します。

各種言語のサンプルコードを確認したい場合は、[X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) の GitHub リポジトリをご覧ください。

<Note>
  ### 前提条件

  このガイドを完了するには、リクエストを認証するための[キーとトークン](/ja/resources/fundamentals/authentication)が必要です。以下の手順でキーとトークンを生成できます。

  - [開発者アカウントに登録](https://developer.x.com/en/apply-for-access)し、承認を受けます。
  - 開発者ポータルで[プロジェクト](/ja/resources/fundamentals/projects)と関連付けられた[開発者アプリ](/ja/resources/fundamentals/developer-apps)を作成します。
  - アプリの「Keys and tokens」ページに移動し、必要な認証情報を生成します。すべての認証情報は安全な場所に保管してください。
</Note>

X API v2 の Postman コレクションを読み込む

X API v2 の Postman コレクションをワークスペースに読み込むには、次のボタンをクリックしてください。

</Button>

 

<div id="authenticate-your-request">
  #### リクエストの認証
</div>

**lookup** エンドポイントに正常にリクエストするには、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication)、[OAuth 2.0 App-Only](/ja/resources/fundamentals/authentication#oauth-2-0)、または [OAuth 2.0 Authorization Code with PKCE](ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかを使用できます。ただし、**manage** エンドポイントでは、OAuth 1.0a User Context か OAuth 2.0 Authorization Code with PKCE のみが使用可能です。

いずれの場合でも、Postman を使用する際に環境が正しく設定されていれば、既定の認証キーとトークンがリクエストに自動入力されます。

これは、Postman 右上の “X API v2” という環境を選択し、環境ドロップダウン横の目のアイコンをクリックして、「initial value」と「current value」フィールドにキーとトークンを入力することで行えます。これらのキーには次が含まれます:

- API Key（Consumer Key とも呼ばれます）
- API Secret Key（Consumer Secret とも呼ばれます）
- OAuth 1.0a User Access Token
- OAuth 1.0a User Access Token Secret
- OAuth 2.0 App Access Token
- OAuth 2.0 Client Key（アプリの設定で OAuth 2.0 User Authentication を有効化している場合のみ）
- OAuth 2.0 Client Secret（アプリの設定で OAuth 2.0 User Authentication を有効化している場合のみ）
