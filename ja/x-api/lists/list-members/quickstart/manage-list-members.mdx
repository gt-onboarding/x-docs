<div id="steps-to-build-a-manage-list-member-request">
  ### リストメンバー管理リクエストを作成する手順
</div>

<div id="step-one-choose-the-list-endpoint-collection-from-postman">
  #### ステップ1：Postman で List エンドポイントコレクションを選択する
</div>

Postman に X API v2 コレクションを読み込んだら、「List」フォルダーに移動し、「List members」フォルダーを開いてから、「Add a member」を選択します。
 

<div id="step-two-specify-the-user-to-add">
  #### ステップ2：追加するユーザーを指定する
</div>

List メンバー管理エンドポイントには 2 つの ID が必要です：List の ID と、追加するユーザーの ID です。[users lookup](/ja/x-api/users/lookup/introduction) を使ってユーザーの ID を取得できます。ユーザー名を指定すると、id フィールドが返ってきます。

ターゲット ID は有効なユーザー ID であれば任意です。Postman で「Params」タブに移動し、id パス変数の「Value」列にあなたの ID を入力します。次に「Body」タブに移動し、追加したいユーザーの ID を user\_id パラメータの値として入力します。いずれの ID の前後にもスペースを含めないでください。

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Parameter type** |
| `id` | List の ID | path |
| user\_id | メンバーとして追加したい対象ユーザーの ID | body |

「Send」ボタンの横に、次のような URL が表示されるはずです:

`https://api.x.com/2/lists/1441162269824405510/members`

ステップ3：リクエストを送信し、レスポンスを確認する

すべての設定が完了したら「Send」ボタンを押すと、次の例のレスポンスに類似した結果が返ってきます:

```
{
  "data": {
    "is_member": true
  }
}
```

返されたレスポンスオブジェクトで is\_member キーの値が true であれば、ユーザーを List のメンバーとして正常に追加できています。

List からメンバーを削除するには、Postman に読み込まれた X API v2 コレクションの「Lists」フォルダ内にある「Remove a member」リクエストを選択します。このエンドポイントには、List の ID と、削除したいメンバーのユーザー ID が必要です。「Params」タブで、id 列の値として List の ID を、user\_id 列の値として削除対象ユーザーの ID を入力します。

削除リクエストが成功すると、次の例のようなレスポンスが返ってきます:

```
{
  "data": {
    "is_member": false
  }
}
```
