<div id="steps-to-build-a-list-members-lookup-request">
  ### リストメンバー照会リクエストを作成する手順
</div>

**ステップ1：Postman で List エンドポイントコレクションを選択**

Postman に X API v2 コレクションを読み込んだら、「List」フォルダーに移動し、続いて「List members」フォルダーを選択し、最後に「Members lookup」を選びます。
 

**ステップ2：メンバーを取得する対象のリストを特定して指定**

メンバーを取得したいリストを指定する必要があります。x.com を開き、任意のリストをクリックして URL を確認すると、List ID を確認できます。たとえば、次の URL の List ID は 84839422 です。

https://x.com/i/lists/84839422

対象 ID は有効な List ID であれば任意で構いません。Postman で「Params」タブに移動し、id パス変数の「Value」列に対象の ID を入力します。ID の前後に空白を入れないようご注意ください。

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 84839422（List ID） |

**ステップ3：取得したいフィールドを特定して指定**

このステップの設定後に「Send」ボタンをクリックすると、レスポンスにはデフォルトの [user object](/ja/x-api/fundamentals/data-dictionary#user) のフィールド（id、name、username）が返されます。

id、name、username 以外の追加フィールドを受け取りたい場合は、リクエストで [field](/ja/x-api/fundamentals/fields) および／または [expansion](/ja/x-api/fundamentals/expansions) パラメータを指定してください。

この演習では、異なるオブジェクトから次の 3 つの追加フィールドセットをリクエストします。

- 主要なユーザーオブジェクトの追加フィールド user.created\_at
- expansion パラメータを使用して取得する完全な [Post object](/ja/x-api/fundamentals/data-dictionary#tweet)
- 関連する投稿オブジェクトの追加フィールド tweet.created\_at

Postman で「Params」タブに移動し、「Query Params」テーブルに以下の key:value を追加します。

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| user.fields | created\_at | `created_at` |
| expansions | pinned\_tweet\_id | `includes.tweets.id,  <br/>includes.tweets.text  <br/>` |
| tweet.fields | created\_at | `includes.tweets.created_at` |

「Send」ボタンの横に次のような URL が表示されるはずです。

`https://api.x.com/2/lists/84839422/members?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=created_at`
