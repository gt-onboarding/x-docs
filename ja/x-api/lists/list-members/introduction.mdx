<div id="list-members-lookup">
  ### メンバーのルックアップ
</div>

メンバーのルックアップには、利用可能なエンドポイントが2つあります。指定したListのメンバーの詳細を取得したり、特定のユーザーがどのListのメンバーかを確認できます。これらのエンドポイントを使用すると、メンバーシップ情報に基づいて新しいListを作成・整理できます。

レート制限は、メンバーの詳細のルックアップが15分あたり900リクエスト、ユーザーのメンバーシップのルックアップが15分あたり75リクエストです。

このエンドポイントへのリクエストの認証には、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication)、[App only](https://developer.x.com/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)、または[OAuth 2.0 Authorization Code with PKCE](ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)を使用できます。 
 

<div id="manage-list-members">
  ### リストメンバーの管理
</div>

リストメンバー管理エンドポイントを使用すると、認証済みユーザーに代わってリストのメンバーを追加および削除できます。これらのエンドポイントでは、POST と DELETE の 2 つのメソッドが利用可能です。POST はリストにメンバーを追加し、DELETE はリストからメンバーを削除します。どちらのエンドポイントも、ユーザー単位のレート制限は 15 分あたり 300 リクエストです。

リストのメンバー数は 5,000 人を超えることはできません。

これらのエンドポイントでユーザーに代わってリクエストを行うためには、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかで当該ユーザーを認証し、あなたのアプリを承認したユーザーに紐づくユーザーアクセストークンを使用する必要があります。これらのトークンは、[3-legged OAuth flow](/ja/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）または [Authorization Code with PKCE grant flow](/ja/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)（OAuth 2.0）で生成できます。

<Note>
  **アカウント設定**

  これらのエンドポイントにアクセスするには、以下が必要です。

  - 承認済みの [developer account](https://developer.x.com/en/portal/petition/essential/basic-info)。
  - [Project](/ja/resources/fundamentals/projects) 内にある [developer App](/ja/resources/fundamentals/developer-apps) のキーとトークンを用いた認証。 

  X API v2 エンドポイントへのアクセス方法の詳細は、[getting started guide](/ja/x-api/getting-started/getting-access)をご覧ください。
</Note>
