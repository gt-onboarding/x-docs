<div id="comparing-x-apis-list-members-endpoints">
  ## X API のリストメンバーエンドポイントの比較
</div>

v2 のリストメンバーエンドポイント群は、従来の v1.1 の [GET lists/members](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-members)、[GET lists/memberships](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-memberships)、[POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create)、および [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy) エンドポイントを置き換えます。これらのリストメンバーエンドポイントを利用しているコード、アプリ、またはツールがあり、より新しい X API v2 エンドポイントへの移行を検討している場合は、本ガイド一式をご活用ください。 

<div id="list-members-lookup">
  ### リストメンバー参照
</div>

v2 のリストメンバー参照エンドポイントは、標準の [GET lists/members](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-members)、[GET lists/memberships](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-memberships) エンドポイントを置き換えます。

次の表では、標準の v1.1 と X API v2 のリスト系エンドポイントを比較します。

**リストメンバー参照**

|     |     |     |
| :--- | :--- | :--- |
| 説明 | 標準 v1.1 | X API v2 |
| サポートされる HTTP メソッド | `GET` | `GET` |
| ホストドメイン | `https://api.x.com` | `https://api.x.com` |
| エンドポイントパス | `/1.1/lists/members.json` | `/2/lists/:id/members` |
| [Authentication](/ja/resources/fundamentals/authentication) | OAuth 1.0a User Context<br /><br />App only | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code with PKCE<br /><br />App only |
| 既定のリクエスト[レート制限](/ja/resources/fundamentals/rate-limits) | OAuth 1.0a: 15 分あたり 900 リクエスト<br /><br />App only: 15 分あたり 75 リクエスト | OAuth 1.0a: 15 分あたり 900 リクエスト<br /><br />OAuth 2.0: 15 分あたり 900 リクエスト<br /><br />App only: 15 分あたり 900 リクエスト |

**リストメンバーシップ参照**

|     |     |     |
| :--- | :--- | :--- |
| 説明 | 標準 v1.1 | X API v2 |
| サポートされる HTTP メソッド | `GET` | `GET` |
| ホストドメイン | `https://api.x.com` | `https://api.x.com` |
| エンドポイントパス | `/1.1/lists/memberships.json` | `/2/users/:id/list_memberships` |
| [Authentication](/ja/resources/fundamentals/authentication) | OAuth 1.0a User Context<br /><br />App only | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code with PKCE<br /><br />App only |
| 既定のリクエスト[レート制限](/ja/resources/fundamentals/rate-limits) | OAuth 1.0a: 15 分あたり 75 リクエスト<br /><br />App only: 15 分あたり 75 リクエスト | OAuth 1.0a: 15 分あたり 75 リクエスト<br /><br />OAuth 2.0: 15 分あたり 75 リクエスト<br /><br />App only: 15 分あたり 75 リクエスト |

<div id="manage-list-members">
  ### リストメンバーの管理
</div>

v2 のリストメンバー管理エンドポイントは、従来の [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create)、[POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy) エンドポイントを置き換えます。

次の表は、従来の v1.1 と X API v2 のリストエンドポイントの比較です。

**メンバーを追加**

| 説明 | 従来の v1.1 | X API v2 |
| :--- | :--- | :--- |
| サポートされる HTTP メソッド | POST | POST |
| ホストドメイン | https://api.x.com | https://api.x.com |
| エンドポイントパス | /1.1/lists/members/create.json | /2/lists/:id/members |
| [認証](/ja/resources/fundamentals/authentication) | OAuth 1.0a ユーザーコンテキスト | OAuth 1.0a ユーザーコンテキスト<br /><br />OAuth 2.0 認可コード（PKCE） |
| 既定のリクエスト[レート制限](/ja/resources/fundamentals/rate-limits) | なし | 15 分あたり 300 リクエスト（ユーザー単位） |

**メンバーを削除**

| 説明 | 従来の v1.1 | X API v2 |
| :--- | :--- | :--- |
| サポートされる HTTP メソッド | POST | DELETE |
| ホストドメイン | https://api.x.com | https://api.x.com |
| エンドポイントパス | /1.1/lists/members/destroy.json | /2/lists/:id/:user\_id |
| [認証](/ja/resources/fundamentals/authentication) | OAuth 1.0a ユーザーコンテキスト | OAuth 1.0a ユーザーコンテキスト<br /><br />OAuth 2.0 認可コード（PKCE） |
| 既定のリクエスト[レート制限](/ja/resources/fundamentals/rate-limits) | なし | 15 分あたり 300 リクエスト（ユーザー単位） |

X API v2 のエンドポイントにアクセスするには、[開発者アカウントに登録](https://developer.x.com/en/portal/petition/essential/basic-info)する必要があります。認証時は、[プロジェクト](/ja/resources/fundamentals/projects)内の[開発者アプリ](/ja/resources/fundamentals/developer-apps)で発行されたキーとトークンを使用してください。

X API v2 エンドポイントへのアクセス方法の詳細は、[はじめに](/ja/x-api/getting-started/getting-access)ページをご覧ください。
