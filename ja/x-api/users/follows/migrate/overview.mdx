<div id="comparing-x-apis-follows-endpoints">
  ## X API のフォロー関連エンドポイントの比較
</div>

<div id="follows-lookup">
  ### フォロー検索
</div>

v2フォロー検索エンドポイントは、標準v1.1の[followers/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-followers-ids)、v1.1の[followers/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-followers-list)、v1.1の[friends/ids](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-friends-ids)、およびv1.1の[friends/list](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-friends-list)エンドポイントに代わるものです。

以下の表は、各種フォロー検索エンドポイントを比較したものです：

|     |     |     |
| :--- | :--- | :--- |
| **説明** | **標準v1.1** | **X API v2** |
| サポートされるHTTPメソッド | `GET` | `GET` |
| ホストドメイン | `https://api.x.com` | `https://api.x.com` |
| エンドポイントパス | /1.1/friends/ids.json<br /><br />/1.1/friends/list.json<br /><br />/1.1/followers/ids.json<br /><br />/1.1/followers/list.json | /2/users/:id/following<br /><br />/2/users/:id/followers |
| [認証](/ja/resources/fundamentals/authentication) | OAuth 1.0aユーザーコンテキスト<br /><br />アプリのみ | OAuth 1.0aユーザーコンテキスト<br /><br />OAuth 2.0（PKCE）<br /><br />アプリのみ |
| デフォルトリクエスト[レート制限](/ja/resources/fundamentals/rate-limits) | 15分間に15リクエスト（ユーザーあたり）<br /><br />15分間に15リクエスト（アプリあたり） | 15分間に15リクエスト（ユーザーあたり）<br /><br />15分間に15リクエスト（アプリあたり） |
| レスポンスあたりの最大ユーザー数 | GET friends/idとGET followers/idは、1ページあたり最大5,000のユーザーIDを返します。<br /><br />  <br /><br />GET friends/listとGET followers/listは、1ページあたり最大200のユーザーオブジェクトを返します。 | 1ページあたり1,000のユーザーオブジェクト |
| ページネーション | トークンはnext\_cursorフィールドで返され、cursorパラメータの値として渡すことで次のページの結果を取得できます。 | トークンはnext\_tokenフィールドで返され、tokenパラメータの値として渡すことで次のページの結果を取得できます。<br /><br />v2ペイロードはprevious\_tokenフィールドも提供し、pagination\_tokenパラメータと一緒に渡すことで前のページの結果を取得できます。 |
| JSON形式 | 標準v1.1形式 | [X API v2形式](/ja/x-api/fundamentals/data-dictionary)（fieldsおよびexpansionsリクエストパラメータによって決定され、v1.1形式との後方互換性はありません）<br /><br />標準v1.1形式からX API v2形式への移行方法について詳しくは、[データ形式移行ガイド](/ja/x-api/migrate/data-format-migration)をご覧ください。 |
| ペイロードで返される[フィールド](/ja/x-api/fundamentals/data-dictionary)の選択をサポート |     | ✔   |
| 投稿[アノテーション](/ja/x-api/fundamentals/post-annotations)フィールドをサポート |     | ✔   |
| 新しい[メトリクス](/ja/x-api/fundamentals/metrics)フィールドのリクエストをサポート |     | ✔   |
| [conversation\_id](/ja/x-api/fundamentals/conversation-id)フィールドをサポート |     | ✔   |
| [プロジェクト](/ja/resources/fundamentals/projects)に関連付けられた[開発者アプリ](/ja/resources/fundamentals/developer-apps)からの認証情報の使用が必要 |     | ✔   |

<div id="manage-follows">
  #### フォローの管理
</div>

v2 のフォロー管理エンドポイントは、標準の v1.1 の [POST friendships/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-create) および [POST friendships/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-destroy) エンドポイントを置き換えます。

以下の表では、標準の v1.1 と X API v2 のフォロー作成エンドポイントを比較します。

<div id="follow-a-user">
  #### ユーザーをフォローする
</div>

| 説明 | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| サポートされている HTTP メソッド | POST | POST |
| ホストドメイン | https://api.x.com | https://api.x.com |
| エンドポイントパス | /1.1/friendships/create.json | /2/users/:id/following |
| [認証](/ja/resources/fundamentals/authentication) | OAuth 1.0a ユーザーコンテキスト | OAuth 1.0a ユーザーコンテキスト<br /><br />OAuth 2.0（PKCE） |
| 既定の[レート制限](/ja/resources/fundamentals/rate-limits) | 15分あたり50リクエスト | 15分あたり50リクエスト |
| ユーザーごとの1日あたりの最大操作回数 | 400 | 400 |
| アプリごとの1日あたりの最大操作回数 | 1000 | 1000 |
| [プロジェクト](/ja/resources/fundamentals/projects)に関連付けられた[開発者用アプリ](/ja/resources/fundamentals/developer-apps)の認証情報が必要 |     | ✔️  |

<div id="unfollow-a-user">
  #### ユーザーのフォロー解除
</div>

次の表は、Standard v1.1 と X API v2 のフォロー解除エンドポイントの比較です。

| 説明 | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| サポートされる HTTP メソッド | POST | DELETE |
| ホストドメイン | https://api.x.com | https://api.x.com |
| エンドポイントパス | /1.1/friendships/destroy.json | /2/users/:source\_user\_id/following/:target\_user\_id |
| [認証](/ja/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code with PKCE |
| 既定のリクエスト[レート制限](/ja/resources/fundamentals/rate-limits) | 15 分あたり 15 リクエスト（ユーザーごと） | 15 分あたり 50 リクエスト（ユーザーごと） |
| アプリごとの 1 日あたりの最大操作数 | なし | 500 |
| [プロジェクト](/ja/resources/fundamentals/projects) に関連付けられた [developer App](/ja/resources/fundamentals/developer-apps) の認証情報の使用が必要 |     | ✔️  |
