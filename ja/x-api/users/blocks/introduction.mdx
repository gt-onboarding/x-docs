<div id="blocks-lookup">
  ### ブロックの照会
</div>

blocks lookup の GET エンドポイントを使用すると、認可済みユーザーに代わって、そのユーザーがブロックしているアカウントを確認できます。このエンドポイントには、ユーザーごとに15分あたり15リクエストのレート制限があります。

blocks lookup では非公開情報を扱い、かつ manage blocks 権限を持つユーザーに代わってリクエストを行うため、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかで認証し、あなたのアプリを認可したユーザーに紐づくユーザーアクセストークンを使用する必要があります。これらのトークンは、[3-legged OAuth flow](/ja/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）または [Authorization Code with PKCE grant flow](/ja/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)（OAuth 2.0）で生成できます。

<Note>
  **アカウントのセットアップ**

  これらのエンドポイントにアクセスするには、以下が必要です:

  - 承認済みの [developer account](https://developer.x.com/en/portal/petition/essential/basic-info)。
  - [Project](/ja/resources/fundamentals/projects) 内の [developer App](/ja/resources/fundamentals/developer-apps) のキーとトークンを使用して認証すること。

  X API v2 のエンドポイントへのアクセス方法については、[getting started guide](/ja/x-api/getting-started/getting-access) を参照してください。
</Note>
