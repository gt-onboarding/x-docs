アカウントをミュートすると、そのアカウントをフォロー解除やブロックをせずに、あなたのタイムラインからそのアカウントの投稿を非表示にできます。ミュートされたアカウントに通知は届かず、ミュートはいつでも解除できます。manage mutes エンドポイントを使うことで、開発者は X 上でより安全な体験を提供できます。たとえば、特定の期間だけ、特定のトピックについて投稿する可能性のあるアカウントをミュートできるアプリケーションを構築できます。mutes lookup エンドポイントでは、あなた自身または認証済みユーザーが誰をミュートしているかを確認できます。これは、ミュートしたアカウントとの関わり方を把握するのに役立ちます。 

mute lookup では非公開情報へのリクエストを行い、manage mutes ではユーザーに代わって操作を行うため、これらのエンドポイントは [OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかで認証し、あなたのアプリを承認したユーザーに紐づくユーザーアクセストークンを使用する必要があります。これらのトークンは、[3-legged OAuth flow](/ja/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）または [Authorization Code with PKCE grant flow](/ja/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)（OAuth 2.0）を使用して生成できます。

<div id="mutes-lookup">
  ### ミュートのルックアップ
</div>

mutes lookup エンドポイントを使用すると、認証済みユーザーがミュートしているアカウントを確認できます。このエンドポイントには、ユーザーごとに15分あたり15件のリクエストというレート制限が適用されます。

<div id="manage-mutes">
  ### ミュートの管理
</div>

ミュート管理エンドポイントを使用すると、認証済みユーザーに代わって指定したアカウントをミュートまたはミュート解除できます。これらのエンドポイントでは、POST と DELETE の 2 つのメソッドが利用可能です。POST はアカウントをミュートし、DELETE はミュートを解除します。POST と DELETE の両エンドポイントには、ユーザー単位で15分あたり50リクエストのレート制限が適用されます。

**注意:** ユーザーが [X](https://x.com/) 上でミュート操作を行う場合は、15分あたり200リクエストの制限が適用されます。

**アカウントのセットアップ**

これらのエンドポイントにアクセスするには、以下が必要です。

- 承認済みの[開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)
- [プロジェクト](/ja/resources/fundamentals/projects)内の[開発者アプリ](/ja/resources/fundamentals/developer-apps)で発行されたキーとトークンを用いた認証

X API v2 エンドポイントへのアクセス方法の詳細は、[はじめに](/ja/x-api/getting-started/getting-access)をご覧ください。
