認証済みユーザーのルックアップ以外のすべてのユーザー・ルックアップエンドポイントへのリクエストは、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication)、[App only](/ja/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)、または [OAuth 2.0 Authorization Code with PKCE](ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) で認証できます。ただし、非公開メトリクスや非公開ユーザーのデータにアクセスする場合は、OAuth 1.0a User Context か OAuth 2.0 Authorization Code with PKCE を使用し、リクエストに認証済みユーザーのアクセス・トークンを渡す必要があります。    

このエンドポイントは、ユーザーの最新情報の取得、ユーザーの存在確認、またはコンプライアンスイベントに応じた保存情報の更新によく使用されます。

<div id="authenticated-user-lookup">
  ### 認証済みユーザーの参照
</div>

他のユーザーの代理で認証する場合、誰の代理でリクエストできるかを把握することが重要です。

このエンドポイントを使用するには、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0（PKCE）](ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) を利用する必要があります。リクエストに付与するユーザーアクセストークンに関連付けられた、認可済みユーザーに関する情報を返します。

このエンドポイントには GET メソッドでアクセスできます。ユーザー単位のレート制限は 15 分あたり 75 リクエストです。

<Note>
  **アカウントのセットアップ**

  これらのエンドポイントにアクセスするには、以下が必要です:

  - 承認された [開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)
  - [プロジェクト](/ja/resources/fundamentals/projects) 内の [開発者アプリ](/ja/resources/fundamentals/developer-apps) のキーとトークンを用いた認証

  X API v2 エンドポイントへのアクセス方法の詳細は、[はじめにガイド](/ja/x-api/getting-started/getting-access)をご覧ください。
</Note>
