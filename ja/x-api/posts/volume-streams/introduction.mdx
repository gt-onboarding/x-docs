- 1% サンプリングストリーム。
- 10% サンプリングストリーム。一般に「Decahose（デカホース）」と呼ばれます。

どちらも大容量のデータを配信するため、「ボリュームストリーム」と呼ばれます。1% ストリームでも、毎秒数十件の投稿が流れる場合があります。これらのストリームを使うことで、トレンドの特定・追跡、全体的なセンチメントの把握、世界的な出来事のモニタリングなどが可能です。

これらのストリーミングエンドポイントは、永続的な HTTP GET 接続で[Post objects](/ja/x-api/fundamentals/data-dictionary#tweet)を配信し、[OAuth 2.0 App-Only](/ja/resources/fundamentals/authentication#oauth-2-0) 認証を使用します。Essential アクセスでは同時接続は 1 件のみ可能です。すべてのアクセスレベルで、接続リクエストは 15 分あたり最大 50 回まで行えます。

これらのボリュームストリームエンドポイントは投稿の編集に対応しています。編集履歴（投稿 ID の配列を含む）とともに編集済みの投稿を配信します。編集履歴がない投稿では、この配列には単一の ID が入ります。編集がある投稿では、編集順を反映した昇順で複数の ID が格納され、配列の最後の要素が最新バージョンになります。投稿の編集の仕組みについては、[Post edits fundamentals](/ja/x-api/fundamentals/edit-posts) を参照してください。

_これらの API を使用するには、まずエンタープライズセールスチームとのアカウント設定が必要です。_

<Note>
  **アカウント設定**

  これらのエンドポイントにアクセスするには、以下が必要です:

  - 承認済みの[developer account](https://developer.x.com/en/portal/petition/essential/basic-info)
  - [Project](/ja/resources/fundamentals/projects) 内の[developer App](/ja/resources/fundamentals/developer-apps)で発行されたキーとトークンを用いた認証

  X API v2 エンドポイントへのアクセス方法は、[getting started guide](/ja/x-api/getting-started/getting-access)をご覧ください。
</Note>
