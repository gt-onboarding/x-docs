<div id="likes-lookup">
  ### いいねのルックアップ
</div>

Likes lookup グループのエンドポイントを使用すると、特定のポストに「いいね」したアカウントの一覧、または特定のアカウントが「いいね」したポストの一覧を取得できます。これらのエンドポイントには次が含まれます：

- ユーザーが「いいね」したポスト - GET /2/users/:id/liked\_tweets
- ポストに「いいね」したユーザー - GET /2/tweets/:id/liking\_users

これらのエンドポイントは、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Bearer Token](/ja/resources/fundamentals/authentication#oauth-2-0) で認証できます。liked Posts エンドポイントでは、大量の結果をページングするためのページネーショントークンが提供されます。

liking users エンドポイントでは、全期間を通じて1つのポストあたり「いいね」したアカウントは合計100件に制限されます。さらに、liked Posts エンドポイントには、プロジェクトレベルで適用される毎月の[投稿上限](/ja/x-api/fundamentals/post-cap)も適用されます。
 

<div id="manage-likes">
  ### いいねを管理する
</div>

いいね管理エンドポイントを使用すると、認証済みアカウントに代わって指定した投稿に「いいね」または「いいね解除」を行えます。このエンドポイントグループには、POST と DELETE の 2 つのメソッドがあります。POST メソッドは投稿に「いいね」するためのもので、DELETE メソッドは「いいね解除」を行うためのものです。

ユーザーに代わってリクエストを行うため、これらのエンドポイントは [OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) で認証し、ユーザーに紐づくアクセス・トークンを使用する必要があります。アクセス・トークンは [3-legged OAuth フロー](/ja/resources/fundamentals/authentication#oauth-1-0a-2)（obtaining-user-access-tokens）で生成できます。自分のアカウント、または認証済みユーザーのアカウントから投稿に「いいね」できます。いずれのエンドポイントにも、ユーザー単位でエンドポイントごとに 15 分あたり 50 リクエストのレート制限が適用されます。 
 

これらのエンドポイントにアクセスするには、承認済みの [developer account](/ja/resources/fundamentals/developer-portal) が必要です。認証時には、[Project](/ja/resources/fundamentals/projects) 内の [developer App](/ja/resources/fundamentals/developer-apps) のキーとトークンを使用する必要があります。 

X API v2 のエンドポイントへのアクセス方法については、[はじめに](/ja/x-api/getting-started/about-x-api) ページをご覧ください。
