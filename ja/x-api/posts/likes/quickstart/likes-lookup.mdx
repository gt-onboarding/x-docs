<div id="getting-started-with-the-likes-lookup-endpoint">
  ### Likes ルックアップエンドポイントの始め方
</div>

このクイックスタートガイドでは、[Postman](/ja/tutorials/postman-getting-started) を使用して、Likes ルックアップエンドポイントへの最初のリクエストを送信する手順を説明します。

各種言語のサンプルコードは、GitHub の [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) リポジトリをご覧ください。
 

<div id="prerequisites">
  #### 前提条件
</div>

このガイドを完了するには、[Keys and tokens](/ja/resources/fundamentals/authentication) のセットが必要です。次の手順で生成できます。

1. [開発者アカウントに申請](https://developer.x.com/en/apply-for-access)し、承認を受けます。

2. 開発者ポータルで [Project](/ja/resources/fundamentals/projects) と、それに関連付けられた [Developer App](/ja/resources/fundamentals/developer-apps) を作成します。

3. アプリの「Keys and tokens」ページに移動し、API Key、Access Token、Bearer Token をパスワードマネージャーに保存します。

<div id="steps-to-build-a-likes-lookup-request">
  #### いいね参照リクエストを作成する手順
</div>

**ステップ1: ツールまたはライブラリから始める**

このエンドポイントにリクエストを送るために使用できるツール、コード例、ライブラリはいくつかありますが、ここでは手順を簡素化するために Postman を使用します。

X API v2 の Postman コレクションを環境に読み込むには、次のボタンをクリックしてください:

[![](https://cdn.cms-twdigitalassets.com/content/dam/developer-twitter/m1_vnext/carat.svg)<br />Add X API v2 to Postman](https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400)

Postman に X API v2 コレクションを読み込んだら、「Likes」フォルダに移動して「Liking users」を選択します。 
 

**ステップ2: リクエストを認証する**

このエンドポイントへのリクエストを成功させるには、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Bearer Token](/ja/resources/fundamentals/authentication#oauth-2-0) 認証を使用する必要があります。これを行うには、「X API v2」という名前の環境を選択し、Initial value と Current value の各フィールドに次の変数を追加して、以下のキーとトークンを Postman に設定します:

- consumer\_key に API Key
- consumer\_secret に API Key Secret
- access\_token に Access Token
- token\_secret に Access Token Secret
   

**ステップ3: ツイートを指定する**

このエンドポイントでは、いいねしたユーザーを取得したいツイートのツイート ID を指定する必要があります。ツイートの ID は、X 上で該当ツイートを開き、URL の末尾にある数値を確認することで見つけられます。例えば、次の URL のツイート ID は 1354143047324299264 です。

https://x.com/TwitterDev/status/1354143047324299264

Postman で「Params」タブに移動し、セクションの一番下にある tweet\_id パス変数の「Value」列にこの ID を入力し、前後にスペースが入らないようにしてください。 

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | いいねしたユーザーを取得したいツイートのツイート ID |

**ステップ4: 取得したいフィールドを特定して指定する**

ステップ3の後に「Send」ボタンをクリックすると、レスポンスにはデフォルトの [user object](/ja/x-api/fundamentals/data-dictionary#user) のフィールド id、name、username が含まれます。

id、name、username 以外のフィールドも取得したい場合は、[fields](/ja/x-api/fundamentals/fields) および／または [expansions](/ja/x-api/fundamentals/expansions) パラメータで、それらのフィールドをリクエストに指定する必要があります。

この演習では、異なるオブジェクトから次の3つの追加フィールドセットをリクエストします:

1. プライマリのユーザーオブジェクトにおける追加の user.created\_at フィールド。
2. 返されたユーザーに関連するピン留めツイートオブジェクトのデフォルトフィールド: id と text。
3. 関連ツイートオブジェクトにおける追加の tweet.created\_at フィールド。

Postman で「Params」タブに移動し、「Query Params」テーブルに次のキー:値のペアを追加します:

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| user.fields | created\_at | user.created\_at |
| expansions | pinned\_tweet\_id | tweet.id, tweet.text |
| tweet.fields | created\_at | tweet.created\_at |

「Send」ボタンの横に次の URL が表示されるはずです:

`https://api.x.com/2/tweets/1354143047324299264/liking_users?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=created_at`

**ステップ6：ユーザーが「いいね」したツイートを取得する**

ユーザーが「いいね」したツイートを取得するリクエストも行うとよいでしょう。Likes ルックアップエンドポイントを使用すると、ユーザーが「いいね」したツイートの情報を取得できます。実行するには、"Likes" フォルダーに移動し、"Liked Tweets" を選択します。

このエンドポイントでは、「いいね」したツイートを取得したいユーザーの User ID を指定する必要があります。この情報は、[user lookup](/ja/x-api/users/lookup/introduction) エンドポイントで取得できます。

Postman で "Params" タブに移動し、このユーザー名をセクション下部の id パス変数の "Value" 列に入力します。ユーザー名の前後にスペースが入らないようにしてください。

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| id  | 「いいね」したツイートを取得したいユーザーの User ID |
| max\_results | 5 |

"Send" ボタンの横に、TwitterDev ではなく自分の ID が入った同様の URL が表示されます。

`https://api.x.com/2/users/2244994945/liked_tweets?max_results=5`
