<div id="comparing-x-apis-likes-endpoints">
  ## X API の「いいね」エンドポイントの比較
</div>

このガイドでは、次の点に焦点を当てます。

- **API リクエストパラメータ** - X API v2 のエンドポイントでは、新しいリクエストパラメータが導入されています。特に Labs を利用している方には馴染みのあるパラメータもありますが、[fields](/ja/x-api/fundamentals/fields) や [expansions](/ja/x-api/fundamentals/expansions) パラメータの導入など、重要な相違点が多数あります。

- **アプリおよびプロジェクトの要件** - X API v2 にアクセスするには、[プロジェクト](/ja/resources/fundamentals/projects) に関連付けられた [開発者向けアプリ](/ja/resources/fundamentals/developer-apps) の認証情報を使用する必要があります。

<div id="likes-lookup">
  #### いいねのルックアップ
</div>

**投稿に「いいね」したユーザー**

liked users エンドポイントは v2 の新機能で、特定の投稿に「いいね」したユーザーに関する情報を取得できます。

| 説明 | X API v2 |
| :--- | :--- |
| サポートされる HTTP メソッド | GET |
| ホストドメイン | https://api.x.com |
| エンドポイントパス | /2/tweets/:id/liking\_users |
| [認証](/ja/resources/fundamentals/authentication) | OAuth 2.0 Bearer Token<br /><br />OAuth 1.0a User Context |
| 既定のリクエスト[レート制限](/ja/resources/fundamentals/rate-limits) | 15分あたり75リクエスト（アプリごと）<br /><br />15分あたり75リクエスト（ユーザーごと） |

**ユーザーが「いいね」した投稿**

以下の表は、Standard v1.1 の [GET favorites/list](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-favorites-list) エンドポイントと X API v2 の「いいね」済み投稿エンドポイントを比較したものです。

| 説明 | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| サポートされる HTTP メソッド | GET | GET |
| ホストドメイン | https://api.x.com | https://api.x.com |
| エンドポイントパス | /1.1/favorites/list.json | /2/users/:id/liked\_tweets |
| [認証](/ja/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 2.0 Bearer Token<br /><br />OAuth 1.0a User Context |
| 既定のリクエスト[レート制限](/ja/resources/fundamentals/rate-limits) | 15分あたり75リクエスト | 15分あたり75リクエスト（アプリごと）<br /><br />15分あたり75リクエスト（ユーザーごと） |
| データフォーマット | Standard v1.1 フォーマット | [X API v2 フォーマット](/ja/x-api/fundamentals/data-dictionary)（fields と expansions のリクエストパラメータによって決まり、v1.1 のフォーマットとは後方互換性がありません）<br /><br />Standard v1.1 フォーマットから X API v2 フォーマットへの移行方法の詳細は、[データフォーマット移行ガイド](/ja/x-api/migrate/data-format-migration)をご覧ください。 |

<div id="manage-likes">
  #### いいねの管理
</div>

v2 のいいね管理エンドポイントは、v1.1 の [POST favorites/create](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-create) および [POST favorites/destroy](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-destroy) エンドポイントに代わるものです。

以下の表では、標準の v1.1 と X API v2 のいいね管理エンドポイントを比較します。

<div id="like-a-post">
  #### ポストに「いいね」する
</div>

| 説明 | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| サポートされる HTTP メソッド | POST | POST |
| ホストドメイン | https://api.x.com | https://api.x.com |
| エンドポイントパス | /1.1/favorites/create.json | /2/users/:id/likes |
| [認証](/ja/resources/fundamentals/authentication) | OAuth 1.0a ユーザーコンテキスト | OAuth 1.0a ユーザーコンテキスト |
| 既定のリクエスト[レート制限](/ja/resources/fundamentals/rate-limits) | 24 時間あたり 1000 リクエスト（ユーザーごと）<br /><br />24 時間あたり 1000 リクエスト（アプリごと） | 15 分あたり 50 リクエスト（ユーザーごと）<br /><br />24 時間あたり 1000 リクエスト（ユーザーごと、DELETE と共有） |

<div id="unlike-a-post">
  #### 投稿とは異なる点
</div>

| 説明 | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| サポートされる HTTP メソッド | POST | DELETE |
| ホストドメイン | https://api.x.com | https://api.x.com |
| エンドポイントパス | /1.1/favorites/destroy.json | /2/users/:id/likes/:tweet\_id |
| [認証](/ja/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context |
| 既定のリクエスト[レート制限](/ja/resources/fundamentals/rate-limits) | 24時間あたり1000リクエスト（ユーザーごと）<br /><br />24時間あたり1000リクエスト（アプリごと） | 15分あたり50リクエスト（ユーザーごと）<br /><br />24時間あたり1000リクエスト（ユーザーごと、POSTと共有） |
