<div id="getting-started-with-the-hide-replies-endpoint">
  ## 返信を非表示にするエンドポイントのはじめ方
</div>

このクイックスタートでは、[Postman](/ja/tutorials/postman-getting-started) を使用して「返信を非表示にする」エンドポイントへの最初のリクエストを送信する方法を説明します。

各種言語のコードスニペットは、[返信を非表示にする API リファレンス](/ja/x-api/posts/hide-replies#put-2-tweets-id-hidden) を参照してください。

<Note>
  ### 前提条件

  このガイドを完了するには、リクエストを認証するための [Keys and tokens](/ja/resources/fundamentals/authentication) 一式が必要です。これらの Keys と tokens は、次の手順で生成できます。

  - [開発者アカウントに登録](https://developer.x.com/en/apply-for-access) し、承認を受ける。
  - 開発者ポータルで、[プロジェクト](/ja/resources/fundamentals/projects) とそれに関連付けられた [開発者アプリ](/ja/resources/fundamentals/developer-apps) を作成する。
  - アプリの「Keys and tokens」ページに移動し、必要なクレデンシャルを生成する。すべてのクレデンシャルは安全な場所に保存すること。
</Note>

<div id="steps-to-build-a-put-tweetsidhidden-request">
  ### PUT /tweets/:id/hidden リクエスト構築の手順
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### ステップ1：ツールまたはライブラリから始める
</div>

このエンドポイントにリクエストする方法はいくつかあります。各種ツール、コード例、ライブラリを利用できますが、ここでは手順を簡潔にするため Postman を使用します。

X API v2 の Postman コレクションを環境に読み込むには、次のボタンをクリックしてください。

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2 を Postman に追加
</Button>

Postman に X API v2 コレクションを読み込んだら、返信を非表示にするエンドポイントへ移動します。

<div id="step-two-authenticate-your-request">
  #### 手順2: リクエストの認証
</div>

X API に正しくリクエストするには、権限を確認する必要があります。 このエンドポイントでは、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかでリクエストを認証してください。

この例では OAuth 1.0a User Context を使用します。

API Key、API Secret Key、OAuth 1.0a user Access Token、OAuth 1.0a user Access Token Secret を Postman に追加する必要があります。 追加するには、Postman の右上で “X API v2” という名前の environment を選択し、"initial value" と "current value" の各フィールドにキーとトークンを入力します（environment のドロップダウン横の目のアイコンをクリック）。

これらの変数は、正しく設定していればリクエストの authorization タブに自動的に反映されます。
 

<div id="step-three-find-a-post-id-to-hide">
  #### ステップ3：非表示にする投稿 ID を探す
</div>

hide replies エンドポイントは、認可済みユーザーに代わって返信を非表示または再表示できます。この例では、あなたのユーザープロフィールに紐づく Access Tokens を使用しているため、あなたが開始した会話に参加しているユーザーの返信を非表示にできます。同様に、あなたのアプリを認可した別のユーザーの Access Tokens を使用している場合は、そのアカウントが開始したあらゆる会話の返信をモデレートできます。

友人に投稿へ返信してもらう（hide replies のテストであることを知らせてください）か、テスト用アカウントであなたの投稿に返信します。その返信を開き、URL の数値部分をコピーしてください。これが非表示にする投稿 ID です。

この例では、次の投稿を参照します。ID は `1232720193182412800` です。

`https://x.com/TwitterDev/status/1232720193182412800`

<div id="step-four-hide-the-post">
  #### ステップ4：投稿を非表示にする
</div>

Postman で Hide replies フォルダーを開き、Hide a reply を選択します。Params タブで、id フィールドの横に投稿 ID を貼り付けます（URL の :id を置き換える必要はありません）。「Send」をクリックすると、成功レスポンスが表示されます。

`{"hidden":true}`

<div id="step-five-unhide-the-post">
  #### ステップ5：投稿の非表示を解除する
</div>

非表示の投稿は X アプリ内の別タブに移動されます。Postman で投稿の非表示を解除するには、Hide replies フォルダーを開き、Unhide a reply を選択します。Params タブで、前のステップと同じ投稿 ID を id フィールドに貼り付けます。「Send」をクリックすると、成功のレスポンスが表示されます。

`{"hidden":false}`

hidden フィールドは投稿の非表示ステータスを表します。true の場合は投稿が非表示であることを意味し、false の場合は非表示ではないことを意味します。
