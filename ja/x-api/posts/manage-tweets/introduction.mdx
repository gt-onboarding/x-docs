X API を使用して投稿を作成・削除することは、公開の会話に参加するうえで不可欠です。新しい manage Posts エンドポイントにより、それらに加えてさらに多様な操作が可能になります。

投稿を管理するためのメソッドは POST と DELETE の 2 種類です。POST メソッドでは、投票の投稿、引用ツイート、返信設定付きの投稿、位置情報付きの投稿、メディア付きおよびユーザーへのタグ付け、Super Followers への投稿などに対応しています。DELETE メソッドでは、特定の投稿を削除できます。POST メソッドでは、リクエストをさらにカスタマイズできるように、目的に応じた[パラメータ](/ja/x-api/posts/creation-of-a-post)を指定できます。

「投稿の削除」メソッドは編集済みの投稿をサポートするよう更新されました。編集チェーン内のいずれかの投稿が削除されると、その編集チェーン内のすべての投稿も削除されます。編集投稿のメタデータの詳細は、[Edit Posts の基礎](/ja/x-api/fundamentals/edit-posts)をご覧ください。

POST メソッドには、ユーザー単位で 15 分あたり 200 リクエストのレート制限があります。DELETE メソッドには、15 分あたり 50 リクエストのレート制限があります。さらに、manage Posts または manage Retweets のいずれかで作成された投稿を含め、3 時間あたり 300 リクエストの上限があります。

manage Posts エンドポイントでユーザーに代わってリクエストを行うため、[OAuth 1.0a User Context](https://en.docs/authentication/oauth-1-0a) または [OAuth 2.0 Authorization Code with PKCE](https://en.docs/authentication/oauth-2-0/authorization-code) で認証し、あなたのアプリを承認したユーザーに紐づくユーザーアクセス トークンを使用する必要があります。OAuth 1.0a でこのユーザーアクセス トークンを生成するには、[3-legged OAuth フロー](https://en.docs/authentication/oauth-2-0/bearer-tokens)を使用できます。OAuth 2.0 でユーザーアクセス トークンを生成するには、[Authorization Code with PKCE フロー](https://en.docs/authentication/oauth-2-0/user-access-token)を使用できます。

<Info>
  **アカウントのセットアップ**

  これらのエンドポイントにアクセスするには、以下が必要です。

  - 承認済みの[開発者アカウント](https://en/portal/petition/essential/basic-info)
  - [プロジェクト](/ja/resources/fundamentals/projects)内にある[開発者アプリ](/ja/resources/fundamentals/developer-apps)のキーとトークンを使用した認証

  X API v2 エンドポイントへのアクセス方法の詳細は、[はじめに](/ja/x-api/getting-started/getting-access)をご覧ください。
</Info>
