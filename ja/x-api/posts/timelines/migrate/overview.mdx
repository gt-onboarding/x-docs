<div id="comparing-x-apis-timelines-endpoints">
  ## X API のタイムラインエンドポイントの比較
</div>

v2 の逆時系列タイムライン、ユーザーポストタイムライン、ユーザーメンションタイムラインの各エンドポイントは、それぞれ [v1.1 statuses/home\_timeine](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-home_timeline)、[v1.1 statuses/user\_timeline](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-user_timeline.html)、[v1.1 statuses/mentions\_timeline](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-mentions_timeline.html) の各エンドポイントを置き換えます。既存のコード、アプリ、またはツールでこのエンドポイントの旧バージョンを使用しており、新しい X API v2 エンドポイントへの移行を検討している場合は、本ガイドをご利用ください。より詳細な移行ガイドについては、[Standard v1.1 から X API v2 への移行](/ja/x-api/posts/timelines/migrate/standard-to-twitter-api-v2)を参照してください。

このページには 3 つの比較表が含まれます。

- 逆時系列のホームタイムライン
- ユーザーポストタイムライン
- ユーザーメンションタイムライン

<div id="reverse-chronological-home-timeline">
  ### 逆時系列のホームタイムライン
</div>

以下の表は、標準の v1.1 と X API v2 のホームタイムライン用エンドポイントを比較しています。

|     |     |     |
| :--- | :--- | :--- |
| **説明** | **Standard v1.1** | **X API v2** |
| ドキュメント | [API リファレンス](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-home_timeline) | [API リファレンス](https://developer.x.com/x-api/posts/timelines#user-mention-timeline-3) |
| 対応 HTTP メソッド | `GET` | `GET` |
| ホストドメイン | `https://api.x.com` | `https://api.x.com` |
| エンドポイントパス | `/1.1/statuses/home_timeline.json` | `/2/users/:id/timelines/reverse_chronological` |
| 必須パラメータ | `user_id` または `screen_name` | パスパラメータ :id に設定されたユーザー ID |
| 認証 | OAuth 1.0a ユーザーコンテキスト | OAuth 1.0a ユーザーコンテキスト<br /><br />[OAuth 2.0（PKCE）認可コードフロー](/ja/resources/fundamentals/authenticationoauth-2-0/authorization-code) |
| リクエストレート制限／ボリューム制限 | OAuth 1.0a ユーザーコンテキストで 15 分あたり 15 リクエスト<br /><br />リクエスト上限: 24 時間で 100,000 件 | 15 分ウィンドウあたり 180 リクエスト <br /><br />[投稿上限](/ja/x-api/fundamentals/post-cap):<br /><br />Essential アクセス: 500,000 件<br /><br />Elevated アクセス: 200 万件<br /><br />Academic Research アクセス: 1,000 万件 |
| レスポンスあたりの既定の投稿数 | 15 | 100 |
| レスポンスあたりの最大投稿数 | 800 | このエンドポイントは、直近 7 日間にタイムライン上で作成されたすべての投稿に加え、作成日時に関わらず直近の 800 件も返します。 |
| 投稿の編集履歴を提供 | ✔ | ✔ |
| 取得可能な過去の投稿 | 直近 800 件の投稿（リツイートを含む） | 直近 3,200 件の投稿（リツイートを含む） |
| タイムラインのナビゲーションオプション | 更新ポーリング用に `since_id`（排他的）を使用<br /><br />`max_id`（包括的） | `start_time`<br /><br />`end_time`<br /><br />更新ポーリング用に `since_id`（排他的） <br /><br />`until_id`（排他的） |
| 結果を絞り込むための任意パラメータ | `count`<br /><br />`exclude_replies`<br /><br />`include_rts`<br /><br />`trim_user`<br /><br />`tweet_mode`<br /><br />`since_id`<br /><br />`max_id` | `max_results`<br /><br />`exclude`(retweets,replies)<br /><br />`tweet.fields`<br /><br />`user.fields`<br /><br />`place.fields`<br /><br />`media.fields`<br /><br />`poll.fields`<br /><br />`expansions`<br /><br />`start_time`<br /><br />`end_time`<br /><br />`since_id`<br /><br />`until_id` |
| [アノテーション](/ja/x-api/fundamentals/post-annotations)のリクエストと受信に対応 | 該当なし | `tweet.fields` にアノテーションが含まれている場合、結果は投稿テキストに基づく推論アノテーションデータ（例: 「音楽ジャンル」「フォークミュージック」や「ミュージシャン」「Dolly Parton」など）で注釈付けされます。 |
| 特定の投稿[メトリクス](/ja/x-api/fundamentals/metrics)のリクエストと受信に対応 | 該当なし | `tweet.fields` にメトリクスが含まれている場合、結果には各投稿の `public_metrics`（`retweet_count`、`reply_count`、`quote_count`、`like_count`）および `non_public_metrics`（`impression_count`、`user_profile_clicks`、`url_link_clicks`）が含まれます。<br /><br />`view_count` や動画再生メトリクスなどの追加メディアメトリクス。<br /><br />プロモーション投稿については、ユーザーコンテキストで追加の `organic_metrics` および `promoted_metrics` が利用可能です。 |
| [conversation\_id](/ja/x-api/fundamentals/conversation-id) のリクエストと受信に対応 | 該当なし | 返信スレッドの最初に公開された投稿を表す値を持つ `conversation_id` フィールドを返し、会話の追跡に役立ちます。 |
| 投稿のJSON形式 | [Standard v1.1 のデータ形式](https://developer.x.com/en/docs/twitter-api/v1/data-dictionary/overview.html) | [X API v2](/ja/x-api/fundamentals/data-dictionary) の形式（fields と expansions のリクエストパラメータで決定。v1.1 の形式とは後方互換性なし）<br /><br />Standard v1.1 形式から X API v2 形式への移行方法については、[データ形式の移行ガイド](/ja/x-api/migrate/data-format-migration)をご覧ください。 |
| 結果の並び順 | 新しい順（逆時系列） | 新しい順（逆時系列） |
| 結果のページネーション | 該当なし。投稿IDによるナビゲーションが必要 | pagination\_token を使用して前後に移動して確認可能 |
| [プロジェクト](/ja/resources/fundamentals/projects) に関連付けられた[開発者アプリ](/ja/resources/fundamentals/developer-apps)の認証情報が必要 |     | ✔   |

<div id="user-post-timeline">
  ### ユーザー投稿タイムライン
</div>

以下の表では、標準の v1.1 と X API v2 のユーザー投稿タイムライン用エンドポイントを比較します。

|     |     |     |
| :--- | :--- | :--- |
| **説明** | **Standard v1.1** | **X API v2** |
| ドキュメント | [API リファレンス](https://developer.x.com/en/docs/twitter-api/v1/tweets/timelines/api-reference/get-statuses-user_timeline.html) | [API リファレンス](/ja/x-api/posts/user-posts-timeline-by-user-id) |
| 対応 HTTP メソッド | GET | GET |
| ホストドメイン | https://api.x.com | https://api.x.com |
| エンドポイントパス | /1.1/statuses/user\_timeline.json | /2/users/:id/tweets |
| 必須パラメータ | user\_id または screen\_name | パスパラメータ :id として指定するユーザー ID |
| 認証 | OAuth 1.0a ユーザーコンテキスト<br /><br />OAuth 2.0 アプリ専用 | OAuth 1.0a ユーザーコンテキスト<br /><br />OAuth 2.0 アプリ専用<br /><br />OAuth 2.0 認可コード（PKCE） |
| リクエストレート制限／ボリューム制限 | OAuth 1.0a ユーザーコンテキストで 15 分あたり 900 リクエスト<br /><br />OAuth 2.0 アプリ専用で 15 分あたり 1500 リクエスト<br /><br />リクエスト上限：24 時間で 100,000 | OAuth 1.0a ユーザーコンテキストで 15 分ウィンドウあたり 900 リクエスト<br /><br />OAuth 2.0 アプリ専用で 15 分ウィンドウあたり 1500 リクエスト<br /><br />[投稿上限](/ja/x-api/fundamentals/post-cap)：  <br />Essential アクセス時 500,000  <br />Elevated アクセス時 200 万  <br />Academic Research アクセス時 1,000 万 |
| 応答あたりのデフォルト投稿数 | 15  | 10  |
| 応答あたりの最大投稿数 | 200 | 100 |
| 取得可能な過去の投稿 | 直近の 3,200 件の投稿（リポストを含む） | 直近の 3,200 件の投稿（リポストを含む） |
| タイムラインのナビゲーションオプション | since\_id（排他的）—更新のポーリングに使用<br /><br />max\_id（包括的） | start\_time<br /><br />end\_time<br /><br />since\_id（排他的）—更新のポーリングに使用 <br /><br />until\_id（排他的） |
| 結果を絞り込むためのオプションパラメータ | count  <br />exclude\_replies  <br />include\_rts  <br />trim\_user  <br />tweet\_mode  <br />since\_id  <br />max\_id | max\_results  <br />exclude(retweets,replies)  <br />tweet.fields  <br />user.fields  <br />place.fields  <br />media.fields  <br />poll.fields  <br />expansions  <br />start\_time  <br />end\_time  <br />since\_id  <br />until\_id |
| [アノテーション](/ja/x-api/fundamentals/post-annotations)の要求と受信のサポート | 該当なし | 投稿テキストに基づいて推定されたアノテーションデータ（例：「音楽ジャンル」「フォークミュージック」や「ミュージシャン」「Dolly Parton」）を含む投稿結果を返します |
| 特定の投稿[メトリクス](/ja/x-api/fundamentals/metrics)の要求と受信のサポート | 該当なし | 各投稿の public\_metrics（retweet\_count、reply\_count、quote\_count、like\_count など）を含む投稿結果を返します。  <br />  <br />OAuth 1.0a ユーザーコンテキストで利用可能：<br />impression\_count、user\_profile\_clicks、url\_link\_clicks などの追加の non\_public\_metrics。<br /><br />view\_count や動画再生メトリクスなどの追加のメディアメトリクス。<br /><br />プロモーション投稿向けには、OAuth 1.0a ユーザーコンテキストで追加の organic\_metrics と promoted\_metrics。 |
| [conversation\_id](/ja/x-api/fundamentals/conversation-id)の要求と受信のサポート | 該当なし | 会話の追跡に役立つよう、返信スレッドで最初に公開された投稿を表す値をもつ conversation\_id フィールドを返します。 |
| 投稿のJSON形式 | [Standard v1.1 のデータ形式](/ja/x-api/fundamentals/data-dictionary) | [X API v2](/ja/x-api/fundamentals/data-dictionary) の形式（fields と expansions のリクエストパラメータで決定。v1.1 形式とは後方互換性がありません）<br /><br />Standard v1.1 形式から X API v2 形式への移行については、[データ形式移行ガイド](/ja/x-api/migrate/data-format-migration)をご覧ください。 |
| 結果の並び順 | 逆時系列（新しい順） | 逆時系列（新しい順） |
| 結果のページネーション | 該当なし。投稿IDでのナビゲーションが必要 | pagination\_token を使用して前方・後方に結果を辿れます |
| [プロジェクト](/ja/resources/fundamentals/projects) に関連付けられた[開発者アプリ](/ja/resources/fundamentals/developer-apps)の認証情報が必要 |     | ✔   |
| 投稿の編集履歴を提供 | ✔   | ✔   |
