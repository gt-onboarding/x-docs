<div id="comparing-x-apis-search-posts-endpoints">
  ## X API の検索投稿エンドポイントの比較
</div>

v2 の Search Tweets エンドポイントは、最終的に [standard v1.1 search/posts](/ja/x-api/posts/search/introduction) エンドポイントおよび [enterprise Search API](/ja/x-api/enterprise-gnip-2.0/fundamentals/search-api) を置き換える予定です。X の古い検索エンドポイントを使用しているコード、アプリ、またはツールがあり、より新しい X API v2 の各エンドポイントへの移行を検討している場合は、このガイドをご参照ください。

このページには次の3つの比較表があります:

- [Recent search](#recent)
- [Full-archive search](#full-archive)
- [Filtering operator comparison](#operators)

**フィルタリングオペレーターの比較**

検索の4つのバージョン（standard、enterprise、v2）は、利用可能なオペレーターが異なり、さらに各バージョン内でもオペレーターの提供レベルに差があります。以下で説明します。

Enterprise

- enterprise オペレーターにサブティアはありません

X API v2

- **Free:** いずれのプロジェクトでも利用可能
- **Basic:** いずれのプロジェクトでも利用可能
- **Pro:** プロジェクトで利用可能
- **Enterprise:** プロジェクトで利用可能

これら各オペレーター群の詳細は、それぞれのガイドをご参照ください。

- [Enterprise operators](/ja/x-api/enterprise-gnip-2.0/fundamentals/search-api#available-operators)
- [X API v2 operators](/ja/x-api/posts/search/integrate/build-a-query#operators)

X API v2 内のオペレーターレベルの違いを把握したところで、検索におけるオペレーターの提供状況をマッピングした表を以下に示します（セルが空白の場合、そのオペレーターは利用できません）。
