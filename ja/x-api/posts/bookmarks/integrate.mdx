  ### 便利なツール
</div>

このエンドポイントを統合するうえで役立つ主要な概念に入る前に、次の内容に慣れておくことをおすすめします。

<div id="postman">
  #### Postman
</div>

Postman は、エンドポイントのテストに使える優れたツールです。各 Postman リクエストには、利用可能な項目をすばやく把握できるよう、すべてのパスおよびボディパラメータが含まれています。Postman コレクションの詳細は「[Using Postman](/ja/tutorials/postman-getting-started)」ページをご覧ください。 

<div id="code-samples">
  #### コードサンプル
</div>

お好みのプログラミング言語でこのエンドポイントを使い始めたい方へ。出発点として利用できるコードサンプルを複数、[GitHub のページ](https://github.com/xdevplatform/Twitter-API-v2-sample-code)で提供しています。

<div id="third-party-libraries">
  #### サードパーティ製ライブラリ
</div>

コミュニティが提供する[サードパーティ製ライブラリ](/ja/x-api/tools-and-libraries/overview)を活用して、スムーズに開始しましょう。適切なバージョンタグを確認すれば、v2 エンドポイントに対応するライブラリを見つけられます。

  ### 重要な基本概念
</div>

<div id="authentication">
  #### 認証
</div>

すべての X API v2 エンドポイントでは、キーやトークンとも呼ばれる認証情報でリクエストに認証を付与する必要があります。

これらの特定のエンドポイントでは、[OAuth 2.0（PKCE）による認可コードフロー](/ja/resources/fundamentals/authentication/oauth-2-0/authorization-code)の使用が必要です。つまり、リクエストを成功させるには、キーのセットとユーザーのアクセス トークンを使用する必要があります。アクセス トークンは、あなたが代理でリクエストしているユーザーに紐づいている必要があります。別のユーザーのアクセス トークンを生成したい場合は、そのユーザーが [Authorize URL](/ja/resources/fundamentals/authentication#authorize-url) を使用してあなたのアプリを承認または認証する必要があります。

なお、OAuth 2.0 の利用は難しい場合があります。この認証方式に不慣れな場合は、[ライブラリ](/ja/x-api/tools-and-libraries/overview)や Postman のようなツールを使って、リクエストを正しく認証することをおすすめします。

<div id="developer-portal-projects-and-developer-apps">
  #### デベロッパーポータル、プロジェクトとデベロッパーアプリ
</div>

X API v2 のエンドポイントで利用できる認証情報一式を取得するには、[デベロッパーアカウント](/ja/resources/fundamentals/developer-portal)を作成し、そのアカウント内に[プロジェクト](/ja/resources/fundamentals/projects)を設定し、そのプロジェクト内に[デベロッパーアプリ](/ja/resources/fundamentals/developer-apps)を作成する必要があります。これらの手順を完了すると、デベロッパーアプリ内でキーとトークンを確認できます。 

<div id="rate-limits">
  #### レート制限
</div>

毎日、数多くの開発者が X API にリクエストを送信しています。これらの膨大なリクエスト量を管理するため、各エンドポイントには[レート制限](/ja/resources/fundamentals/rate-limits)が設けられており、アプリケーションまたは認証済みユーザーに代わって行えるリクエスト数が制限されています。

これらのエンドポイントはユーザーレベルで制限されており、代理でリクエストを行う認証済みユーザーは、どの開発者向け App からであっても、そのエンドポイントを一定回数までしか呼び出せません。GET メソッドのユーザーレート制限は 15 分あたり 180 リクエストです。Bookmarks ルックアップエンドポイントの GET メソッドでは、直近のブックマーク済み投稿を最大 800 件取得できます。さらに、POST および DELETE メソッドのユーザーレート制限は 15 分あたり 50 リクエストです。
