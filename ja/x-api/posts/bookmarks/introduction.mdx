<div id="manage-bookmarks">
  ### ブックマークの管理
</div>

ブックマークを管理する方法は POST と DELETE の 2 種類があります。POST メソッドではブックマークを作成できます。同様に、DELETE メソッドでは特定のブックマークを削除できます。

POST と DELETE の各メソッドには、ユーザーごとに 15 分あたり 50 件のリクエストというレート制限があります。

manage Bookmarks エンドポイントでユーザーに代わってリクエストを送信するため、OAuth 2.0 を使用してユーザーアクセストークンを生成し、認証する必要があります。そのためには、[Authorization Code with PKCE grant flow](/ja/resources/fundamentals/authentication/oauth-2-0/user-access-token) を使用できます。このエンドポイントを使用するには、`tweet.read`、`users.read`、`bookmark.write` のスコープを付与して渡す必要があります。

<div id="bookmarks-lookup">
  ### ブックマークの参照
</div>

Bookmarks lookup エンドポイントで利用可能なメソッドは GET のみです。このメソッドにより、自分自身または認証済みアカウントのブックマークを取得できます。大量の結果をページングするためのページネーショントークンが提供されます。

GET メソッドには、ユーザーごとに 15 分あたり 180 リクエストのレート制限があります。このエンドポイントでは、直近でブックマークしたポストを最大 800 件取得できます。

Bookmarks lookup エンドポイントでユーザーに代わってリクエストを行うため、OAuth 2.0 を用いてユーザーアクセストークンを生成し、認証する必要があります。その際は [Authorization Code with PKCE grant flow](/ja/resources/fundamentals/authentication/oauth-2-0/user-access-token) を使用できます。このエンドポイントを使用するには、`tweet.read`、`users.read`、`bookmark.read` のスコープを指定する必要があります。

**アカウントの設定**

これらのエンドポイントにアクセスするには、以下が必要です。

- 承認済みの[開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)
- [プロジェクト](/ja/resources/fundamentals/projects) 内にある[開発者アプリ](/ja/resources/fundamentals/developer-apps)のキーとトークンを使用した認証

X API v2 エンドポイントへのアクセス方法の詳細は、[はじめにガイド](/ja/x-api/getting-started/getting-access)をご覧ください。
