  ## X API のフィルタードストリームエンドポイントの比較
</div>

v2 のフィルタードストリームエンドポイント群は、[Standard v1.1 の statuses/filter](https://developer.x.com/en/docs/x-api/v1/tweets/filter-realtime/api-reference/post-statuses-filter) と [PowerTrack API](/ja/x-api/enterprise-gnip-2.0/powertrack-api) を置き換えるものです。古いバージョンのフィルタードストリームエンドポイントを利用しているコード、アプリ、またはツールがあり、より新しい X API v2 のエンドポイントへの移行を検討している場合、この比較が移行の出発点として役立ちます。

より詳しい移行ガイドもご覧ください:

[Standard v1.1 から X API v2 への移行比較](ja/x-api/posts/filtered-stream#standard-v1-1-compared-to-x-api-v2)

[PowerTrack API から X API v2 への移行](/ja/x-api/enterprise-gnip-2.0/powertrack-api#powertrack-operators)

以下の表では、X が提供するフィルタードストリームエンドポイントを比較します:
 

| **説明** | **Standard v1.1** | **PowerTrack API** | **X API v2** |
| :--- | :--- | :--- | :--- |
| アクセス | X アプリ | エンタープライズ契約とアカウントが必要 | 開発者アカウント（[登録](https://developer.x.com/en/portal/petition/essential/basic-info)）と、[プロジェクト](/ja/resources/fundamentals/projects)内の[X アプリ](/ja/resources/fundamentals/developer-apps)が必要 |
| :--- | :--- | :--- | :--- |
| ホストドメイン | ******https://stream.x.com****** | ******https://gnip-stream.x.com****** | ******https://api.x.com****** |
| エンドポイントパス | ******1.1/statuses/filter.json****** | ******/stream/powertrack/accounts/{gnip_account_name}/publishers/twitter/{stream_label}.json******<br /><br />******/rules/powertrack/accounts/{gnip_account_name}/publishers/twitter/{stream_label}.json******<br /><br />******/rules/powertrack/accounts/{gnip_account_name}/publishers/twitter/{stream_label}/validation.json****** | ******/2/tweets/search/stream******<br /><br />******/2/tweets/search/stream/rules****** |
| [認証](/ja/resources/fundamentals/authentication) | OAuth 1.0a ユーザーコンテキスト | HTTP Basic 認証 | OAuth 2.0 アプリ専用 |
| サポートされる HTTP メソッド | POST | GET  <br />POST | GET  <br />POST |
| 必須パラメータ | 接続時のパラメータとして定義されたルール。少なくとも次のいずれか1つ：<br /><br />\* ******follow******<br />\* ******track******<br />\* ******locations****** | ストリーミング接続に必須パラメータはなし。任意でバックフィル用パラメータあり。<br /><br />ルールは別途管理 | ストリーミング接続に必須パラメータはなし。レスポンス形式の定義や、学術研究アクセス向けの[バックフィル復旧機能](/ja/x-api/posts/filtered-stream#recovering-missed-data-after-a-disconnection-backfill)の追加に関する任意パラメータあり。<br /><br />ルールは別途管理 |
| 配信タイプ | ストリーミング | ストリーミング<br /><br />REST（ルール管理） | ストリーミング<br /><br />REST（ルール管理） |
| 既定のリクエストレート制限 | 5分あたり5回の接続試行 | POST と GET を合算して1分あたり60リクエスト<br /><br />/rules: 特定ストリームの API に対する /rules エンドポイント（POST・GET）の全リクエスト合算で1分あたり60リクエスト | エンドポイントおよび[アクセスレベル](/ja/x-api/getting-started/about-x-api)による。<br /><br />[GET /2/tweets/search/stream](/ja/x-api/posts/filtered-stream#get-2-tweets-search-stream):  <br />Pro - アプリあたり15分で50リクエスト<br /><br />[GET /2/tweets/search/stream/rules](/ja/x-api/posts/filtered-stream#get-2-tweets-search-stream-rules):  <br />Pro - アプリあたり15分で450リクエスト<br /><br />---<br /><br />[POST /2/tweets/search/stream/rules](/ja/x-api/posts/adddelete-rules):  <br />Pro - アプリあたり15分で100リクエスト |
| 最大許可接続数 | 認可ユーザーあたり同時2接続 | 契約により決定（複数／冗長接続をサポート） | Pro アクセス:  <br />1 |
| [復旧および冗長化機能](/ja/x-api/posts/filtered-stream#filtered-stream-recovery-and-redundancy-features) | なし | バックフィル、冗長接続、Replay API |     |
| [投稿上限](/ja/x-api/fundamentals/post-cap) | ファイアホースの1%に制限 | 契約により決定 | このエンドポイントから受信するすべての投稿に対して、プロジェクト単位の月次の投稿上限が適用されます：<br /><br />Basic:  <br />1万件の投稿<br /><br />Pro:  <br />100万件の投稿 |
| キープアライブ信号／ハートビート | 少なくとも20秒ごとに空行（\r\n など） | 10秒ごとに空行（\r\n など） | 少なくとも20秒ごとに空行（\r\n など） |
| レイテンシ | 10秒 | 2秒<br /><br />URL 展開エンリッチメントは最低10秒 | 10秒 |
| 最大許可ルール数 | 1ルール（エンドポイント接続リクエスト内） | 契約により決定（最大250,000） | Pro アクセス:  <br />1000ルール |
| ルールフィルターの制限 | 1 接続あたり 1 クエリ。上限は次のいずれか：<br /><br />- トラック対象キーワード 400 個<br /><br />- フォロー対象ユーザー ID 5,000 個<br /><br />- ロケーションボックス 25 個 | 1 ルールあたり 2,048 文字まで | Pro アクセス：   <br />1 ルールあたり 1,024 文字 |
| [投稿の JSON 形式](/ja/x-api/fundamentals/data-dictionary) | Standard v1.1 形式 | [Native Enriched](/ja/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#native-enriched-tweet-object) または [Activity Streams]()（[コンソール](/ja/x-api/enterprise-gnip-2.0/fundamentals/overview)で選択） | [X API v2 形式](/ja/x-api/fundamentals/data-dictionary)（リクエストパラメータの ******fields****** と ******expansions****** により決定。v1.1 形式との後方互換性はありません）<br /><br />Standard v1.1 形式から X API v2 形式への移行方法は、[データ形式移行ガイド](/ja/x-api/migrate/data-format-migration)をご参照ください。Native Enriched と Activity Streams 向けの追加のデータ形式移行ガイドも近日公開予定です。 |
| 投稿の編集履歴とメタデータの提供 | ✔   | ✔   | ✔   |
| 固有の機能 | 接続リクエスト時のクエリパラメータによるフィルタリング<br /><br />設定用 UI なし | 独立したエンドポイントで作成するルールによるフィルタリング<br /><br />契約により [Enrichment](/ja/x-api/enterprise-gnip-2.0/enterprise-gnip#enrichments) 機能が利用可能<br /><br />console.gnip.com の UI で設定 | 独立したエンドポイントで作成する[ルール](/ja/x-api/posts/filtered-stream/integrate/build-a-rule)によるフィルタリング<br /><br />[メトリクス](/ja/x-api/fundamentals/metrics) と URL エンリッチメント機能を含む<br /><br />リクエストパラメータでオブジェクトの[fields](/ja/x-api/fundamentals/fields) と [expansions](/ja/x-api/fundamentals/expansions) を指定<br /><br />投稿の[Annotations](/ja/x-api/fundamentals/post-annotations)<br /><br />[Conversation ID](/ja/x-api/fundamentals/conversation-id) オペレーターおよびフィールド<br /><br />[開発者ポータル](/ja/resources/fundamentals/developer-portal) から設定 |
