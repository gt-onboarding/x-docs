フィルタードストリームのエンドポイントグループは、開発者が公開投稿のストリームをフィルタリングできるようにします。このエンドポイントグループには、ルールの作成・管理や、そのルールを適用して一致する公開投稿のみを返すストリームをフィルタリングできる複数のエンドポイントが含まれます。このエンドポイントグループにより、特定のトピックやイベントの監視、競技イベント周辺の会話のモニタリング、トレンド形成の把握などが可能になります。

開発者は、接続を切断することなく永続的なストリーム接続にルールを追加・削除するために、REST の[rules endpoint](/ja/x-api/posts/filtered-stream#post-2-tweets-search-stream-rules)を使用できます。これらの[ルール](/ja/x-api/posts/filtered-stream#building-rules-for-filtered-stream)は、メッセージのキーワード、ハッシュタグ、URL などの投稿属性に一致させるオペレーターを用いて作成できます。オペレーターやルール節は、ブール論理や括弧と組み合わせて、フィルタの一致挙動をより精緻にできます。

一連のルールを追加したら、[ストリーミング接続を確立](/ja/x-api/posts/filtered-stream#get-2-tweets-search-stream)できます。これにより、永続的な HTTP ストリーミング接続を通じて、JSON 形式の[投稿オブジェクト](/ja/x-api/fundamentals/data-dictionary#tweet)の配信が開始されます。ストリームに接続している間は、ルールに一致するコンテンツのみを受信します。

また、フィルタードストリームを webhook に連携し、webhook イベントとして配信することもできます。フィルタードストリームでの webhook の利用について詳しくは、[Filtered Stream Webhooks introduction](/ja/x-api/webhooks/stream/introduction)を参照してください。

フィルタード検索エンドポイントは編集済みの投稿をサポートします。このエンドポイントは、1 つ以上のフィルターに一致する編集済みの投稿を、その編集履歴（投稿 ID の配列を含む）とともに配信します。編集履歴がない投稿では、この配列には単一の ID が入ります。編集済みの投稿では、この配列には複数の ID が含まれ、編集順を反映した昇順で並び、最新バージョンは配列の最後になります。投稿の編集の仕組みの詳細は、[Posts edits fundamentals](/ja/x-api/fundamentals/edit-posts)ページをご覧ください。

フィルタードストリームエンドポイントの一部の機能は、[アクセスレベル](/ja/x-api/getting-started/about-x-api)によって制限されます。

**Pro アクセス**

- プロジェクトあたり 1,000 個のルール
- POST /2/tweets/search/stream/rules エンドポイントでルールを追加する場合、15 分あたり 100 リクエスト
- ルールの作成時にすべてのオペレーターを使用可能
- 最大 1,024 文字のルールを作成可能

**Enterprise アクセス**

- プロジェクトあたり 25,000 個超のルール
- ルールの作成時にすべてのオペレーターを使用可能
- 最大 2,048 文字のルールを作成可能
- Enterprise アクセスの申請は[こちら](https://developer.x.com/en/products/x-api/enterprise/enterprise-api-interest-form)

フィルタードストリームから返される投稿は、月間の[投稿上限](/ja/x-api/fundamentals/post-cap)にカウントされます。

<Note>
  **アカウントのセットアップ**

  これらのエンドポイントにアクセスするには、以下が必要です。

  - 承認済みの[開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)
  - [プロジェクト](/ja/resources/fundamentals/projects)内にある[開発者アプリ](/ja/resources/fundamentals/developer-apps)のキーとトークンを用いた認証

  X API v2 エンドポイントへのアクセスについては、[はじめにガイド](/ja/x-api/getting-started/getting-access)をご覧ください。
</Note>
