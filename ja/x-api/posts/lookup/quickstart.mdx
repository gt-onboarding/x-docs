<div id="getting-started-with-the-posts-lookup-endpoints">
  ## Posts ルックアップエンドポイントの利用開始
</div>

このクイックスタートガイドでは、Postman を使用して、指定したフィールド群とともに Posts ルックアップエンドポイントへ初回リクエストを送信する方法を説明します。

各種言語のサンプルコードは、GitHub リポジトリの [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) をご参照ください。

<Note>
  ### 前提条件

  このガイドを完了するには、リクエスト認証に必要な [keys and tokens](/ja/resources/fundamentals/authentication) が必要です。以下の手順で keys と tokens を生成できます。

  - [開発者アカウントに登録](https://developer.x.com/en/portal/petition/essential/basic-info)し、承認を受ける。
  - 開発者ポータルで [プロジェクト](/ja/resources/fundamentals/projects) と、それに関連付けられた [開発者アプリ](/ja/resources/fundamentals/developer-apps) を作成する。
  - アプリの「Keys and tokens」ページに移動し、必要なクレデンシャルを生成する。すべてのクレデンシャルは安全な場所に保存すること。
</Note>

<div id="steps-to-build-a-get-tweets-request">
  ### GET /tweets リクエストを作成する手順
</div>

<div id="step-one-start-with-a-tool-or-library">
  #### ステップ1：ツールまたはライブラリから始める
</div>

このエンドポイントにリクエストする方法はいくつかあり、ツール、コード例、ライブラリが利用できますが、ここでは手順を簡素化するために Postman を使用します。

X API v2 の Postman コレクションを環境に読み込むには、以下のボタンをクリックしてください。

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2 を Postman に追加
</Button>

Postman に X API v2 コレクションを読み込んだら、「Post Lookup > Multiple Posts」リクエストに移動します。

<div id="step-two-authenticate-your-request">
  #### ステップ2：リクエストの認証
</div>

X API に正しくリクエストするには、権限を確認する必要があります。リクエストの認証方法は次のとおりです。

- [OAuth 2.0 App-Only](ja/resources/fundamentals/authentication)
- [OAuth 2.0 Authorization Code with PKCE](ja/resources/fundamentals/authentication)
- [OAuth 1.0a User Context](ja/resources/fundamentals/authentication)

ここでは簡潔さを優先して OAuth 2.0 App-Only を使用しますが、非公開の[メトリクス](/ja/x-api/fundamentals/metrics)や投稿を取得する場合は、他のいずれかの認証方法を使用する必要があります。

OAuth 2.0 App-Only を使用するには、Postman の右上にある環境ドロップダウンで「X API v2」という環境を選択し、「initial value」と「current value」フィールド（環境ドロップダウン横の目のアイコンをクリック）にキーとトークン（特に App Access Token、別名 App-only Bearer Token）を追加してください。

正しく設定できていれば、これらの変数は自動的にリクエストの Authorization タブに反映されます。

<div id="step-three-identify-and-specify-which-posts-you-would-like-to-retrieve">
  #### ステップ3：取得する投稿を特定して指定する
</div>

リクエスト内で、取得したい投稿（単体または複数）を必ず指定してください。Post ID は、X.com で該当の投稿を開き、URL 内を確認することで見つけられます。たとえば、次の URL の Post ID は `1228393702244134912` です。

`https://x.com/TwitterDev/status/1228393702244134912`

Postman では「Params」タブを開き、この ID、またはカンマ区切りの複数の Post ID を、ids パラメータの「Value」列に入力します。

| Key  | Value |
| :---- | :----- |
| `ids` | `1228393702244134912,1227640996038684673,1199786642791452673` |

---

<div id="step-four-identify-and-specify-which-fields-you-would-like-to-retrieve">
  #### ステップ4：取得するフィールドの特定と指定
</div>

ステップ3の後に「Send」ボタンをクリックすると、レスポンスにはデフォルトの[投稿オブジェクトのフィールド](/ja/x-api/fundamentals/data-dictionary)が含まれます：`id`、`text`、`edit_history_tweet_ids`。

この演習では、異なるオブジェクトから追加で3つのフィールドセットを要求します。

- 主要な投稿オブジェクトにおける追加の `tweet.created_at` フィールド
- 返された投稿の作成者に対応するユーザーオブジェクトのデフォルトフィールド：`id`、`name`、`username`
- 関連するユーザーオブジェクトにおける追加の `user.created_at` フィールド

Postmanで「Params」タブに移動し、「Query Params」テーブルに次の `key:value` ペアを追加します。
| Key            | Value               | Returned fields                          |
| :-------------- | :------------------- | :---------------------------------------- |
| `tweet.fields` | `created_at`        | `tweets.created_at`                      |
| `expansions`   | `author_id`         | `includes.users.id`, `includes.users.name`, `includes.users.username` |
| `user.fields`  | `created_at`        | `includes.users.created_at`              |

「Send」ボタンの横に次のURLが表示されるはずです。

```
https://api.x.com/2/tweets?ids=1228393702244134912,1227640996038684673,1199786642791452673&tweet.fields=created_at&expansions=author_id&user.fields=created_at
```
