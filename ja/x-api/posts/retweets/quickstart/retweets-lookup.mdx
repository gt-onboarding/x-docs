<div id="getting-started-with-the-retweets-lookup-endpoint">
  ### Retweets ルックアップエンドポイントの始め方
</div>

このクイックスタートガイドでは、[Postman](/ja/tutorials/postman-getting-started) を使用して Retweets ルックアップエンドポイントに最初のリクエストを送信する方法を説明します。各種言語のサンプルコードは、GitHub の [X API v2 sample code](https://github.com/xdevplatformTwitter-API-v2-sample-code) リポジトリをご覧ください。

<div id="prerequisites">
  #### 前提条件
</div>

このガイドを完了するには、リクエストを認証するための一連の[keys and tokens](/ja/resources/fundamentals/authentication)が必要です。以下の手順に従ってキーとトークンを生成できます。

- [開発者アカウントに登録](https://developer.x.com/en/apply-for-access)し、承認を受ける。
- 開発者ポータルで[プロジェクト](/ja/resources/fundamentals/projects)と関連する[開発者アプリ](/ja/resources/fundamentals/developer-apps)を作成する。
- アプリの「Keys and tokens」ページに移動し、必要な認証情報を生成する。すべての認証情報は安全な場所に保存すること。

<div id="steps-to-build-a-retweets-lookup-request">
  #### リツイート照会リクエストの作成手順
</div>

**ステップ1：ツールまたはライブラリから始めましょう**

このエンドポイントにリクエストする方法はいくつかあり、ツールやコード例、ライブラリが用意されていますが、ここでは手順を簡素化するために Postman を使用します。

環境に X API v2 の Postman コレクションを読み込むには、以下のボタンをクリックしてください。

<Button href="https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400">
  X API v2 を Postman に追加
</Button>

Postman に X API v2 コレクションを読み込んだら、「Retweets」フォルダに移動し、「Retweeted by」を選択します。

**ステップ2：リクエストを認可する**

X API に正しくリクエストするには、適切な権限を持っていることを確認する必要があります。そのため、このエンドポイントでは、[OAuth 2.0 App-Only](/ja/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)、[OAuth 2.0 Authorization Code with PKCE](ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)、または [OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) のいずれかの認証方式でリクエストを認証する必要があります。

簡単のため、このリクエストでは OAuth 2.0 のアプリ専用認証を使用しますが、非公開の[メトリクス](/ja/x-api/fundamentals/metrics)やリツイートを取得したい場合は、別の認証方法を使用する必要があります。

OAuth 2.0 の App-Only を利用するには、キーとトークン、特に Postman に[アプリアクセストークン](/ja/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token)（App-Only ベアラートークンとも呼ばれます）を追加する必要があります。手順としては、Postman 右上の環境で「X API v2」を選択し、環境ドロップダウン横の目のアイコンをクリックして表示される「initial value」と「current value」フィールドにキーとトークンを追加します。

これらの変数は、正しく設定できていれば、リクエストの認証タブに自動的に反映されます。

**ステップ3：投稿を指定**

このエンドポイントでは、リツイートしたユーザーを取得したい投稿の投稿 ID を指定する必要があります。X 上で該当の投稿を開き、URL の末尾にある数値を取得することで、その投稿の ID を確認できます。たとえば、次の URL の投稿 ID は 1354143047324299264 です。

https://x.com/TwitterDev/status/1354143047324299264

Postman で「Params」タブに移動し、セクション最下部にある id パス変数の「Value」列にこのユーザー名を入力します。ユーザー名の前後に空白が入らないようにしてください。

|     |     |
| :--- | :--- |
| **キー** | **値** |
| `id` | リポストしたユーザーを取得したい投稿のID |

**ステップ4：取得するフィールドを特定して指定する**

手順3の後に「送信」ボタンをクリックすると、レスポンスにはデフォルトの［ユーザーオブジェクト］(/x-api/fundamentals/data-dictionary#user)のフィールド（id、name、username）が返されます。

id、name、username 以外のフィールドも取得したい場合は、リクエストで [fields](/ja/x-api/fundamentals/fields) および／または [expansions](/ja/x-api/fundamentals/expansions) パラメータを指定する必要があります。

この演習では、別々のオブジェクトから追加で3つのフィールドセットを要求します。

1. 主要ユーザーオブジェクト内の追加フィールド user.created\_at。
2. 返されるユーザーについて、関連付けられた固定投稿オブジェクトのデフォルトフィールドは id と text です。
3. 関連する Post オブジェクトの追加フィールド tweet.created\_at。

Postman で「Params」タブに移動し、「Query Params」テーブルに次のキーと値のペアを追加します。

|     |     |     |
| :--- | :--- | :--- |
| **キー** | **値** | **返されるフィールド** |
| user.fields | created\_at | user.created\_at |
| expansions | pinned\_tweet\_id | tweet.id, tweet.text |
| tweet.fields | created\_at | tweet.created\_at |

［送信］ボタンの横に、次のURLが表示されているはずです。

`https://api.x.com/2/tweets/1354143047324299264/retweeted_by?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=created_at`

**ステップ5：リクエストを送信し、レスポンスを確認する**
