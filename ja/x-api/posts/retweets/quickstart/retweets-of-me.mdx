「Reposts of Me」ルックアップを使用すると、自分の投稿のうち、他のユーザーにリポストされたものを確認できます。

これにより、ユーザーはどの投稿がリポストされ、オーディエンスに響いているかを把握でき、より的確なコミュニケーションやコンテンツ戦略の見直しに役立ちます。

<div id="authentication">
  #### 認証
</div>

ユーザーに代わってリクエストを行う場合、これらのエンドポイントは [OAuth 1.0a ユーザーコンテキスト](/ja/resources/fundamentals/authentication) または [OAuth 2.0（PKCE）による認可コードフロー](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかで認証し、対象ユーザーに紐づくユーザーアクセストークンを使用する必要があります。ユーザーアクセストークンは、[3-legged OAuth フロー](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)（OAuth 1.0a）または [認可コード（PKCE）グラントフロー](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3)（OAuth 2.0）で生成できます。

<div id="making-a-request">
  #### リクエストの送信
</div>

ユーザーがあなたのアプリで認証を完了したら、以下のとおりユーザーに代わって Retweets of Me API を呼び出せます。

```bash
      curl 'https://api.x.com/2/users/reposts_of_me' --header 'Authorization: ••••••'
```
