<div id="retweets-lookup">
  ### リツイート検索
</div>

リツイート検索エンドポイントを使用すると、投稿をリツイートしたアカウントのリストを取得できます。このエンドポイントでは、最大100ユーザーずつのバッチで大量の結果セットをページングするためのページネーショントークンが提供されます。

これらのエンドポイントは、[OAuth 1.0a ユーザーコンテキスト](/ja/resources/fundamentals/authentication)、\[OAuth 2.0 アプリのみ]\(https://developer.x.com(/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token)、または[OAuth 2.0（PKCE）](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3)のいずれかで認証できます。 

<div id="retweets-of-me">
  ### 自分の投稿のリポスト
</div>

「Reposts of Me」ルックアップを使用すると、自分の投稿のうち、他のユーザーに再共有されたものを確認できます。

これにより、どの投稿が共有され、オーディエンスに響いているかを把握でき、より的確なコミュニケーションやコンテンツ戦略の見直しに役立ちます。

これらのエンドポイントは、[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかで認証できます。

<div id="manage-retweets">
  #### リツイートの管理
</div>

Manage Retweets エンドポイントを使用すると、認証済みアカウントに代わって、指定した投稿をリツイートしたり、リツイートを取り消したりできます。このエンドポイントグループには、POST と DELETE の 2 つのメソッドがあります。POST メソッドは投稿のリツイート、DELETE メソッドは指定した投稿のリツイートの取り消しを行います。

ユーザーに代わってリクエストを行うため、これらのエンドポイントは [OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) または [OAuth 2.0 Authorization Code with PKCE](ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) で認証し、リクエスト対象ユーザーに紐づくユーザーアクセストークンを使用する必要があります。ユーザーアクセストークンは、[3-legged OAuth flow](/ja/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)（OAuth 1.0a）または [Authorization Code with PKCE grant flow](/ja/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)（OAuth 2.0）で生成できます。自分のアカウント、または認証済みユーザーのアカウントから投稿をリツイートできます。どちらのエンドポイントも、ユーザーあたりのレート制限は各エンドポイントにつき 15 分あたり 50 リクエストです。
