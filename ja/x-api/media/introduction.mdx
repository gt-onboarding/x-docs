メディアオブジェクトは、1つの写真、動画、またはアニメーションGIFを表します。メディアオブジェクトは X API の多くのエンドポイントで使用され、投稿、ダイレクトメッセージ、ユーザープロフィール、広告クリエイティブなどに含まれる場合があります。各メディアオブジェクトには、解像度や形式の異なる複数の表示または再生バリアントが存在することがあります。

<div id="media-types-size-restrictions">
  ## メディアタイプとサイズ制限
</div>

API 経由でアップロードする際のサイズ制限

- **画像**: `5 MB`
- **GIF**: `15 MB`
- **動画**: `512 MB`（`media_category=amplify_video` を使用する場合）

<div id="creation">
  ## 作成
</div>

Post、Direct Message、ユーザープロフィール画像、ホストされた広告カードなどのオブジェクトには、1 つ以上のメディアオブジェクトを含められます。これらのトップレベルオブジェクトは総称してエンティティと呼びます。該当するエンティティ作成 API（例: [`POST /2/tweets`](/ja/x-api/posts/creation-of-a-post)）には、一意の `media_id` を用いて 1 つ以上のメディアオブジェクトを渡せます。

メディアオブジェクトを含むエンティティは、次の手順で作成します。

1. 推奨の[分割（チャンク）](/ja/x-api-/media/quickstart/media-upload-chunked)アップロード（画像／GIF／動画）または旧式の[シンプル](/ja/x-api/media-upload/media-upload)アップロード（画像のみ）を使用して、メディアファイルをアップロードします。
2. 手順 1 の結果として `media_id` を受け取ります。エンティティが複数の `media_id` パラメータを受け付ける場合は、異なるメディアについてこの手順を複数回繰り返せます。
3. `media_id` とその他の必須パラメータを含めて適切なエンドポイントを呼び出し、エンティティを作成します。たとえば、[`POST /2/tweets`](/ja/x-api/posts/creation-of-a-post) エンドポイントを使用して、Post に `media_id` を添付します。
