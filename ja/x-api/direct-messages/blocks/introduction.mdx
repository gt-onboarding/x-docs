manage DM blocks エンドポイントは、認証済みユーザーに代わって指定したアカウントをブロックまたはブロック解除するための機能を提供します。これらのエンドポイントには、次の 2 つの POST メソッドがあります。

- **/2/users/:id/dm/block**: アカウントをブロックします
- **/2/users/:id/dm/unblock**: アカウントのブロックを解除します

  ### はじめに
</div>

<div id="authentication">
  ### 認証
</div>

ユーザーを代表してリクエストを行うため、これらのエンドポイントは [OAuth 1.0a User Context](/ja/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) または [OAuth 2.0 Authorization Code with PKCE](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) のいずれかで認証し、リクエスト対象のユーザーに紐づくユーザーアクセス・トークンを使用する必要があります。ユーザーアクセス・トークンは、[3-legged OAuth フロー](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)（OAuth 1.0a）または [Authorization Code with PKCE グラント・フロー](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)（OAuth 2.0）を用いて生成できます。

<div id="making-a-request">
  ### リクエストの送信
</div>

ブロック

ユーザーがアプリで認証を済ませたら、以下のようにユーザーに代わって Block エンドポイントを呼び出せます。

```
      curl --request POST 'https://api.x.com/2/users/:id/dm/block' --header 'Authorization: ••••••'
```

リクエストが成功すると、以下のような JSON レスポンスが返されます。

```
{
   "data": {
       "blocked": true
   }
}
```

**ブロック解除**

ユーザーがアプリで認証を済ませたら、以下のようにユーザーに代わって Unblock エンドポイントを呼び出せます。

```
      curl --request POST 'https://api.x.com/2/users/:id/dm/unblock' --header 'Authorization: ••••••'
```

リクエストが成功すると、以下のような JSON レスポンスが返されます。

```
{
   "data": {
       "blocked": false
   }
}
```
