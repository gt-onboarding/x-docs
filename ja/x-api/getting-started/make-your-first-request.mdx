このガイドでは、初回リクエストを行うための手順を順に説明します。X アカウントに登録した直後に役立つリソースです。

コードサンプル、より技術的なガイド、または Postman のようなグラフィカルツールを使いたい場合は、初回リクエストに向けて次のガイドもご確認ください。

- [投稿エンドポイントへ初回リクエストを行うためのクイックスタートガイド](/ja/x-api/posts/manage-tweets/quickstart)
- [Postman のはじめ方](/ja/tutorials/postman-getting-started) - 初心者に最適
- [X API v2 のコードサンプル](https://github.com/xdevplatform/Twitter-API-v2-sample-code)
- [投稿ルックアップ API リファレンス](/ja/x-api/posts/lookup/introduction)

このガイドは、[API key と secret](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret)、[ユーザー Access Token と Secret](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)、[App Access Token](/ja/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) を取得し、安全な場所に保存済みであることを前提としています。手順の詳細は、[X API へのアクセスの取得](/ja/x-api/getting-started/getting-access)ガイドをご参照ください。

<div id="step-1-identify-which-endpoint-you-would-like-to-use">
  ### ステップ 1. 使用するエンドポイントを選定する
</div>

X API を使うと、X のウェブサイトやモバイルアプリで行える操作の多くを、コードで実行できます。

API で利用可能なエンドポイントの一覧は [API Reference Index](/ja/x-api/introduction) に掲載していますが、まずは次のいずれかから始めることをお勧めします。

- [Manage Posts > Post a Tweet](/ja/x-api/posts/manage-tweets/introduction)
- [Search Posts](/ja/x-api/posts/search/introduction)
- [User lookup](/ja/x-api/users/lookup/introduction)

<div id="step-2-choose-a-tool-to-make-your-request">
  ### ステップ 2. リクエスト作成に使うツールを選ぶ
</div>

一部のリクエストは簡単に作成できますが、なかには実装が難しいものもあります。そこで、開発者コミュニティが複雑さを軽減するためのツールを用意しています。

以下に推奨ツールと、その使い方の詳細を示します。

<div id="postman">
  #### Postman
</div>

Postman は、REST エンドポイントにリクエストを送信できるビジュアルツールです。X API で利用可能な各種エンドポイントを使い始めたり探索したりするのに役立つ、Postman に関する資料を用意しています。

キーとトークンの追加方法や最初のリクエストの送信方法を学ぶには、["Getting started with Postman" チュートリアル](/ja/tutorials/postman-getting-started)をご覧になることをおすすめします。

また、X API v2 の各エンドポイント向けに、主に Postman を用いたクイックスタートガイドも提供しています。各エンドポイントのセクションで参照できますが、以下にいくつかリンクを示します。

- クイックスタート: 投稿を作成する
- クイックスタート: 投稿を検索する
- クイックスタート: ユーザーを取得する

Postman ではストリーミングエンドポイントにリクエストを送信できない点にご注意ください。これらのエンドポイントの扱い方については、[フィルタードストリーム](/ja/x-api/posts/filtered-stream/quickstart) または [1% サンプリングストリーム](/ja/x-api/posts/sample-stream) のクイックスタートガイドをご覧ください。

よりシンプルなグラフィカルツールをご希望の場合は、[Insomnia](https://insomnia.rest) の利用もご検討ください。

<div id="sample-code">
  #### サンプルコード
</div>

シンプルなコードでリクエストを実行したい場合は、各 X API v2 エンドポイント向けに複数言語のサンプルコードを用意しています。

コードサンプルは GitHub リポジトリ [X-API-v2-sample-code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) で公開しています。README も含まれており、リクエストを正しく実行するための認証情報の設定方法を学べます。

例として、ユーザー照会エンドポイントの cURL 例を示します。このリクエストを使用するには、$ACCESS\_TOKEN と $USERNAME をご自身の [App Access Token](/ja/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) と X のハンドルに置き換えてください。あとは、このコードをコマンドラインツールにコピー＆ペーストして「Return」または「Enter」を押します。

```bash
curl "https://api.x.com/2/users/by/username/$USERNAME" -H "Authorization: Bearer $ACCESS_TOKEN"
```

<div id="libraries">
  #### ライブラリ
</div>

優れた X Developer コミュニティは、さまざまなプログラミング言語でライブラリを提供しており、X API へのリクエストに利用できます。

私たちは、把握しているコミュニティ製ライブラリを一覧化した「[ツールとライブラリ](/ja/x-api/tools-and-libraries/overview)」ページを用意しています。各ライブラリには、ローカル環境でリポジトリをセットアップし、最初のリクエストを送信する方法を学べる README が用意されているはずです。

> 注意: 問題が発生した場合は、リクエスト先のエンドポイントに関する開発者ドキュメント、[サポートセクション](https://developer.x.com/en/support)、またはフォーラムでコミュニティにお問い合わせください。成功するリクエストの実行までサポートします。

<div id="step-3-review-the-response">
  ### ステップ 3. レスポンスを確認する
</div>

リクエストが成功すると、そのリクエストに関連するメタデータを含むペイロードが返されます。

GET HTTP メソッドを利用するエンドポイントを使用した場合、リクエスト先のリソース（投稿、ユーザー、リスト、スペースなど）に関するメタデータが JSON 形式で返されます。返却された各フィールドを確認し、要求した情報を X 上のコンテンツに対応付けられるか確認してください。

POST、PUT、または DELETE の HTTP メソッドを利用するエンドポイントを使用した場合は、X 上でアクションを実行したことになります。X.com またはモバイルアプリにアクセスして、そのアクションを確認できるか追跡してください。

<div id="step-4-adjust-your-request-using-parameters">
  ### ステップ 4. パラメータを使ってリクエストを調整する
</div>

各エンドポイントには、リクエスト内容を調整するために利用できるパラメータが用意されています。たとえば、GET エンドポイントで `fields` と `expansions` を指定すると、追加のメタデータフィールドを取得できます。さらに、[search Posts](/ja/x-api/posts/search/introduction)、[Post counts](/ja/x-api/posts/counts/introduction)、[filtered stream](/ja/x-api/posts/filtered-stream/introduction) などのエンドポイントで多様なフィルタリング手法を試し、取得するデータを関心のある投稿に絞り込めます。

対象エンドポイントのリクエストパラメータとフィールドの一覧は、そのエンドポイントの API Reference に記載されています。統合ガイドやファンダメンタルのページには、その他の有用な統合関連情報も掲載しています。

利用可能な学習資料の詳細は、[Important resources](/ja/x-api/getting-started/important-resources) ガイドをご覧ください。
