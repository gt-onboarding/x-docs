<div id="step-one-sign-up-for-a-developer-account">
  ### ステップ1：開発者アカウントに登録
</div>

開発者アカウントの登録は簡単です。以下のボタンをクリックしていくつかの質問に答えるだけで、[Basic access](https://developer.x.com/en/portal/products/basic) を利用して X API v2 の探索と開発を開始できます。

続いて、オンボーディング中にプロジェクトと関連する開発者アプリを作成します。これにより、API へのすべてのリクエストの認証に使用する認証情報一式が発行されます。

[登録](https://developer.x.com/en/portal/products/basic)

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### ステップ2：アプリのキーとトークンを保存し、安全に管理する
</div>

アクセス権を取得してプロジェクトとアプリを作成すると、開発者用アプリ内で次の認証情報を確認または生成できるようになります。

- **[API Key と Secret](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret)：** 実質的にアプリのユーザー名とパスワードに相当します。[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) を要するリクエストの認証や、ユーザー Access Token や App Access Token など他のトークンの生成に使用します。

- **[Access Token と Secret](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)：** 一般に、Access Token はリクエストを代行されるユーザーを表します。開発者ポータルで生成できるものは、そのアプリの所有者であるユーザーを表します。[OAuth 1.0a User Context](/ja/resources/fundamentals/authentication) を要するリクエストの認証に使用します。別のユーザーに代わってリクエストを行う場合は、3-legged OAuth フローを用いて、そのユーザーから認可を得る必要があります。

- **[Client ID と Client Secret](/ja/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)：** これらの認証情報は、OAuth 2.0 認証でユーザー Access Token を取得するために使用します。OAuth 1.0a と同様に、ユーザー Access Token は非公開のユーザーアカウント情報を提供する、または他アカウントの代わりに操作を行うリクエストを認証するために使用されますが、クライアントアプリケーションがユーザーに対して持つアクセス権限をより細かく制御できるスコープが提供されます。

- **[App only Access Token](/ja/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)：** X 上で公開されている情報を返すエンドポイントにリクエストする際に使用します。

これらのキーとトークンは再生成しない限り有効期限がないため、認証情報を受領後はパスワードマネージャーなどの安全な場所に保存することを推奨します。

> **注意：キーとトークンは [developer portal](/ja/resources/fundamentals/developer-portal) に一度しか表示されません。生成後ただちにパスワード管理システムへ保存してください。**
>
> キーやトークンを紛失・失念した場合は再生成が必要となり、新しい認証情報が作成され、既存のものは無効化されます。これにより、以前の認証情報で構成していた統合を更新する必要があります。
>
> [認証におけるベストプラクティス](/ja/resources/fundamentals/authentication#authentication-best-practices)の詳細をご覧ください。
