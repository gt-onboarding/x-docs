<div id="overview">
  ## 概要
</div>

> エンタープライズコンソール（console.gnip.com）は、契約に基づくエンタープライズ API アクセス権を有するエンタープライズデータのお客様がご利用いただけます。現在エンタープライズデータのお客様でない場合は、詳細について[エンタープライズデータ](https://developer.x.com/en/products/x-api/enterprise)をご覧ください。

本ドキュメントは、X のエンタープライズコンソール ダッシュボードの概要です。製品の詳細、利用状況、一般的なアカウント管理に関わるコンソール内の各領域について説明します。ご自身のアカウントに関するご質問は、担当のアカウントマネージャーまでお問い合わせいただくか、[エンタープライズ製品の技術ドキュメント](/ja/x-api/enterprise-gnip-2.0/enterprise-gnip)をご確認ください。

アカウントマネージャーとのエンタープライズ向けトライアルおよびオンボーディングの一環として、エンタープライズのお客様には [console.gnip.com](http://console.gnip.com) のログイン情報が付与されます。これは、エンタープライズ製品へのアクセス状況の確認や設定が行えるユーザーインターフェースです。管理者（カスタマー管理者）向けの初回コンソールアクセスは、オンボーディングプロセスの一環として X によって設定され、その後、管理者は追加ユーザーを登録できます。エンタープライズコンソールでは、エンタープライズ製品に関する詳細の管理および参照が可能です。 

以下の動画では、console.gnip.com ダッシュボードの各セクションの概要を紹介します。 

**免責事項：動画に表示されている一部の機能（特定のストリームのエンリッチメントや X 以外のデータソース製品を含む）は、現在ご利用いただけない場合があります。**

<div id="products-tab">
  ## Products タブ
</div>

[console.gnip.com](http://console.gnip.com) にアカウントでログインすると、ダッシュボードの Products タブが表示されます。このページには、現在お使いのアカウントで利用可能なエンタープライズ製品の概要が示されます。

PowerTrack や Decahose のようにストリーミング配信を利用する製品では、製品名とストリームラベル、現在のアクティブ接続数、（該当する場合）アクティブなルール数、直近24時間に配信されたアクティビティ（例：投稿）の件数（生のカウント）が一覧表示されます。

Search API のように REST 配信の製品では、製品名、ラベル（「streams」としても表示）、これらのエンドポイントを通じて返された現在のアクティビティ（例：投稿）、およびエンドポイントごとの各種リクエスト件数が表示されます。

[Usage API](/ja/x-api/enterprise-gnip-2.0/fundamentals/usage) を使用すると、これらのデータの多くをプログラムから取得できます。

各ストリームの詳細は、名前または設定ボタンをクリックすると確認できます。

<div id="overview">
  ### 概要
</div>

メインダッシュボードで任意のストリームをクリックすると、そのストリームの概要ページに移動します。

ストリーミング配信プロダクトの場合、このページには以下が含まれます。

1. 各ストリーム接続経由で配信されるアクティビティ数のボリュームチャート
2. ストリームで現在アクティブな接続の詳細（接続 ID と IP アドレス）
3. そのストリームにおける最近の接続・切断・ルール更新イベントのログ

チャートの尺度は右上のリンクで調整できます。個々の接続や切断の表示は、チャート直下の凡例で該当するキーをクリックして切り替えられます。

<div id="connections">
  ### 接続
</div>

Connections ページでは、ストリームで最近発生した接続イベントの詳細を表示します。各接続の開始・終了時刻（UTC の24時間表記）、接続の継続時間、接続元サーバーの IP、参照用の一意の接続 ID、現在の接続ステータスが含まれます。ステータスは指定された接続の最新イベントに対応し、たとえば「Client Connected」や切断（切断の種類付き）を示します。接続のデバッグの詳細は、[Disconnections Explained ガイド](/ja/x-api/enterprise-gnip-2.0/powertrack-api#integrating-with-powertrack)を参照してください。

  ### API ヘルプ
</div>

API ヘルプページには、ストリーム向けの API エンドポイント URL と、該当する場合はそのストリームの Rules API エンドポイントが記載されています。加えて、curl のサンプルコマンド、ストリームエンドポイントへの接続手順、さらにストリームの Rules API エンドポイントを使用してルールをプログラムから追加・削除・一覧取得する方法が含まれます。

<div id="rules">
  ### ルール
</div>

Rules タブは PowerTrack ストリームで使用でき、ユーザーインターフェースからプレーンテキストのルールを手動入力して、すばやく始められる簡便な方法を提供します。なお、このインターフェースで手動追加できるルールは最大 1000 件までで、初期テストのみに使用してください。本番環境では、[Rules API](/ja/x-api/enterprise-gnip-2.0/powertrack-api#api-reference-index) を使用してルールをプログラムで管理することを推奨します。

<div id="settings">
  ### 設定
</div>

「設定」タブでは、ストリーム内データの出力形式を切り替えられ、複数の形式オプションが利用できます。形式を切り替えるには、各オプションを示すラジオボタンを使用します。変更はストリームへ再接続すると反映されます。なお、この設定を更新すると次回のリクエストまたは接続時に即時適用され、新しい形式によってはパーサーが動作しなくなる可能性があります。 

**注意:** 最も多くのデータを取得するための推奨設定は「データを元の形式のままにする」です。これにより、[こちら](/ja/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#post-object)で説明している拡張済みのネイティブ形式でデータが返されます。Activity Streams 形式については[こちら](/ja/x-api/enterprise-gnip-2.0/fundamentals/data-dictionary#activity-object)をご参照ください。

<div id="usage-tab">
  ## Usage タブ
</div>

Usage タブでは、さまざまな期間にわたるストリームの利用状況に関するインサイトを提供します。利用状況情報にプログラムからアクセスするには、[Usage API](/ja/x-api/enterprise-gnip-2.0/fundamentals/usage)を参照してください。

<div id="monthly">
  ### 月次
</div>

月次使用状況ページでは、製品別に分解されたストリームの使用量が表示されます。たとえば、特定のデータソースに対するカバレッジ製品（例：PowerTrack、Historical PowerTrack、PowerTrack Replay）は、個別のデータに加えて合算のロールアップ件数を提供するため、ひとまとめに表示されます。件数には、当月累計、推定月末値（今月これまでの使用量と残り期間に基づく）、および過去2か月分の件数が含まれます。

なお、これらの件数は製品およびストリームごとに重複排除されています。同じ PowerTrack ストリームに複数の接続で同一の投稿を受信した場合でも、本用途ではその投稿は1回としてカウントされます。件数は協定世界時 00:00 に、24時間ごとに更新されます。

<div id="daily">
  ### 日次
</div>

Daily Usage ページでは、当月の各日について、プロダクト別に集計した日次の重複排除済み件数を提供します。件数は24時間ごと、UTCの00:00に更新されます。

<div id="rules">
  ### ルール
</div>

ルールの上限は、契約している PowerTrack ルールパッケージに基づき、同一の「タイプ」に属するすべてのストリームに適用されます。カウントは、新しいルールが追加または削除されるたびに更新されます。

<div id="account-tab">
  ## アカウント タブ
</div>

<div id="my-profile">
  ### マイプロフィール
</div>

個人のユーザープロフィールの詳細を表示し、ここでパスワードを変更できます。あわせて、アカウント固有の利用閾値通知を設定できます。API のステータス通知は https://api.twitterstat.us のステータスページで確認できます。

<div id="account-settings">
  ### アカウント設定
</div>

ユーザーの追加・削除・編集に加え、個々のユーザー向けのメール通知や利用量しきい値アラートを設定できます。

<Frame>
  <img src="/images/account-settings.png.twimg.1920.png" />
</Frame>

ユーザーには3種類があります：Account Admin、User、Email Only。

- Account Admin は他のユーザーの作成／削除／編集が可能で、エンタープライズ API に接続する際にユーザー名（メールアドレス）とパスワードによるベーシック認証を使用できます

- User は他のユーザーの作成や変更はできませんが、エンタープライズ API に接続する際にユーザー名（メールアドレス）とパスワードによるベーシック認証を使用できます

- Email Only ユーザーはダッシュボードにアクセスできず、エンタープライズ API への接続権限もありません。通知セクションで受信するように設定されている場合にのみ通知を受け取ります。

<div id="usage-thresholds">
  ### 使用量のしきい値
</div>

製品ごとに使用量のしきい値を設定します。プロフィールで通知を有効にしているユーザーには、各製品の警告しきい値と重大しきい値の両方に対してメールアラートが送信されます。

<Frame>
  <img src="/images/usage-thresholds.png.twimg.1920.png" />
</Frame>

これらのしきい値は毎日 19:30（UTC）に1回のみ評価され、リアルタイムでは評価されませんのでご注意ください。
