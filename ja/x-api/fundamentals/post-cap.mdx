---
title= "投稿上限"
---

X API v2 には、月ごとに特定のエンドポイントから取得できる投稿数を制限する投稿消費上限があります。投稿上限は[プロジェクト](/ja/resources/fundamentals/projects)単位で適用されます。つまり、そのプロジェクト内の[開発者向けアプリ](/ja/resources/fundamentals/developer-apps)のキーとトークンを使用して以下に記載のエンドポイントへ行うあらゆるリクエストは、そのプロジェクトの投稿上限にカウントされます。

<div id="limited-v2-endpoints">
  ## 制限対象の v2 エンドポイント
</div>

以下のいずれかのエンドポイントから取得した投稿は、月間の投稿上限にカウントされます。

- [投稿のルックアップ](/ja/x-api/posts/lookup/introduction)
- [最近の検索](/ja/x-api/posts/counts/introduction)
- [全アーカイブ検索](/ja/x-api/posts/counts/introduction)
- [フィルタードストリーム](/ja/x-api/posts/filtered-stream)
- [ユーザー投稿のタイムライン](/ja/x-api/posts/timelines)
- [ユーザーのメンションタイムライン](/ja/x-api/posts/timelines)
- [「いいね」のルックアップ - ユーザーが「いいね」した投稿](/ja/x-api/posts/likes)
- [ブックマークのルックアップ - ユーザーがブックマークした投稿](/ja/x-api/posts/bookmarks/introduction)
- [リスト投稿のルックアップ](/ja/x-api/lists/list-tweets/introduction)
- [Spaces のルックアップ](/ja/x-api/spaces/lookup/introduction)

<Note>
  **注:** 同一の投稿が同日に複数のクエリで返された場合でも、投稿上限へのカウントは1回のみです。投稿は日次で重複排除されます。
</Note>

<div id="post-cap-volumes">
  ## 投稿上限のボリューム
</div>

投稿上限のボリュームは、アクセスレベルによって異なります。

| アクセスティア | 月間の投稿上限                    |
|:---------------|:---------------------------------|
| **Free**      | 月あたり 1,500 件の投稿           |
| **Basic**     | 月あたり 10,000 件の投稿          |
| **Pro**       | 月あたり 1,000,000 件の投稿       |
| **Enterprise**| 月あたり 5,000 万件以上の投稿     |

[developer portal](/ja/resources/fundamentals/developer-portal) の[メインダッシュボードページ](https://developer.x.com/content/developer-twitter/en/portal/dashboard)を表示すると、月間の投稿上限に対する利用状況を確認できます。プロジェクト名の下に、当月の利用状況を上限に対する比率で示すステータスバーが表示され、取得済みの投稿総数、使用済みの割合、利用状況のリセット日が確認できます。
