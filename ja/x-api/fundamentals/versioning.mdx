---
title: "バージョニング"
---

<p>X API v2 のリリースに合わせて、開発者が X のパブリック API にいつ変更が入るのか、またいつ新しいバージョンへ移行する必要があるのかをより正確に把握できる新しいバージョニング戦略を導入しました。</p>
<p>開発者は、変更への対応を開発ロードマップに適切に反映できるよう、<a href="https://developer.x.com/en/updates/stay-informed">コミュニケーションチャネル</a>を通じて、非推奨、廃止、変更、追加についての通知を受け取ります。API へのすべての変更は変更履歴に記載されます。</p>
<p>現在、X API には 3 つのバージョンがあります。ユースケースで必要な機能が v2 にまだ提供されていない場合を除き、統合の計画時には X API v2 の利用を強く推奨します。</p>
<p>各バージョンの詳細は、以下のページをご参照ください。</p>

<ul>
  <li><a href="/ja/x-api/introduction">X API v2</a></li>
  <li>Gnip 2.0（<a href="/ja/x-api/enterprise-gnip-2.0/enterprise-gnip">エンタープライズ</a>）</li>
</ul>

<div id="versioning-strategy">
  ## バージョニング戦略
</div>

<p>X API のバージョニングは、エンドポイントのルートパスに宣言されたバージョン番号で表します。</p>
<p>`https://api.x.com/2/tweets`</p>
<p>X API のメジャーバージョンは必要に応じてリリースしますが、12か月より短い間隔では行いません。API に破壊的変更を導入する際にメジャーバージョンをリリースします。新しいメジャーバージョンを公開する際には、開発者が新バージョンへ移行できるよう移行ガイドを提供します。</p>
<p>破壊的変更とは、アプリの既存機能を維持するために開発者がコードを変更する必要がある変更を指します。非破壊的変更は追加型で、準備が整い次第、最新バージョンにロールアウトされます。新機能を活用したい場合を除き、開発者側の対応は不要です。</p>
<p>サイクルの途中で（セキュリティまたはプライバシー上の理由により）破壊的変更を展開する必要がある場合は、その変更は最新バージョンに適用されます。</p>

<div id="breaking-changes">
  ### 破壊的変更
</div>

<p>これらの変更により、アプリケーションの既存の機能を維持するためには、開発者はコードを更新する必要があります。</p>

<ul>
  <li>新たな必須パラメータの追加</li>
  <li>既存のエンドポイントの削除</li>
  <li>レスポンス内のいずれかのフィールドの削除（必須・任意を問わず）</li>
  <li>クエリパラメータの削除</li>
  <li>入力または出力フォーマットの再構成（例：トップレベルのフィールドをサブフィールドにする、エラーの位置をインラインに変更する など）</li>
  <li>既存の入力パラメータまたは出力値の名称またはデータ型の変更</li>
  <li>フィールド名の変更</li>
  <li>リソース名の変更</li>
  <li>レスポンスコードの変更</li>
  <li>エラー種別の変更</li>
  <li>既存の認可スコープの変更</li>
</ul>

<div id="non-breaking-changes">
  ### 互換性を損なわない変更
</div>

<ul>
  <li>新規エンドポイントの追加</li>
  <li>新規オプションパラメータの追加</li>
  <li>新規レスポンスフィールドの追加</li>
  <li>エラーメッセージの文言変更</li>
  <li>新しいスコープの利用可能化</li>
  <li>フィールドの「null 化」（完全に削除する代替として、プライバシー／セキュリティ上の理由で値を null に変更）</li>
</ul>

<div id="deprecation-and-retirement">
  ### 廃止と提供終了
</div>

<p>はじめに、X API における「廃止」と「提供終了」の定義を示します。</p>

<ul>
  <li><b>廃止（Deprecation）：</b> この機能はチームによるサポート対象外となります。該当機能に関する新機能の追加は行われず、製品に不具合や問題があっても修正に対応する可能性は極めて低くなります。 </li>
  <li><b>提供終了（Retired）：</b> この機能には今後アクセスできなくなります。</li>
</ul>

<p>多くの場合、新しいバージョンがリリースされると同時に、以前のバージョンは廃止としてマークされます。バージョンは一定期間、廃止状態のまま維持され、その後に提供終了となります。 </p>
<p>将来の廃止や提供終了に関する情報は、<a href="https://developer.x.com/en/updates/stay-informed">最新情報</a>をご確認ください。</p>
