<div id="conversation-threads-using-the-x-api">
  ### X API を使用した会話スレッド
</div>

X における会話の展開を見てみると、1つの投稿が複数の会話スレッドを生み出し、参加者が増えるにつれてそれぞれが長く、複雑になることがあります。投稿間の関係を特定し、会話スレッドを把握する機能は、X API v2 のペイロードおよび検索機能に含まれています。投稿に対する投稿（返信）や、返信への再返信が行われると、各返信には、その会話を開始した元の投稿の投稿 ID と一致する `conversation_id` が設定されます。

特定の投稿への返信、およびそれらの返信への返信はすべて、単一の元の投稿から派生する同一の会話に含まれます。どれだけ多くの返信スレッドが派生しても、会話のきっかけとなった元の投稿に共通の `conversation_id` を共有します。X API v2 を使用すれば、会話スレッド全体を取得して再構築でき、何が語られているのか、会話やアイデアがどのように展開していくのかを把握できます。

`tweet.fields` パラメータとしての `conversation_id` の取得
v2 エンドポイントで返されるすべての投稿に対して `conversation_id` を取得するには、リクエストパラメータに tweet.fields=conversation\_id を追加します。conversation\_id は、会話の返信スレッドにおける元の投稿の投稿 ID です。以前の返信スレッドから派生したものも含め、同じ返信スレッド内のすべての投稿は同一の conversation\_id を示します。

<div id="request-with-conversation_id-parameter">
  ### `conversation_id` パラメータを使ったリクエスト
</div>

```
curl --request GET \
  --url 'https://api.x.com/2/tweets?ids=1225917697675886593&tweet.fields=author_id,conversation_id,created_at,in_reply_to_user_id,referenced_tweets&expansions=author_id,in_reply_to_user_id,referenced_tweets.id&user.fields=name,username' \
  --header 'Authorization: Bearer $ACCESS_TOKEN'
```

<div id="using-conversation_id-as-a-filter-operator">
  ### フィルター演算子としての `conversation_id` の使用
</div>

`conversation_id` は、最新検索での検索クエリパラメータとして、または [フィルタードストリーム](/ja/x-api/posts/filtered-stream/introduction) のルール内の演算子として使用できます。これにより、フィルタードストリームによるリアルタイム取得、または [投稿検索](/ja/x-api/posts/search/introduction) による新しい順で、投稿の会話スレッド全体が返されます。また、この演算子を [投稿数](/ja/x-api/posts/counts/introduction) と組み合わせることで、会話内の投稿数を取得することもできます。

<div id="request-to-query-by-conversation_id">
  ## `conversation_id` で検索するリクエスト
</div>

```
curl --request GET \
  --url 'https://api.x.com/2/tweets/search/recent?query=conversation_id:1279940000004973111&tweet.fields=in_reply_to_user_id,author_id,created_at,conversation_id' \
  --header 'Authorization: Bearer $ACCESS_TOKEN'
```
