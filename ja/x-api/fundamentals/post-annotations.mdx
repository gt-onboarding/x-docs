<div id="overview">
  ## 概要
</div>

すべての v2 エンドポイントで返される Post オブジェクトに注釈が追加されました。Post の注釈は、Post 自体に関する文脈情報を把握する手段を提供します。全件の Post がレビューされますが、テキストの内容により、注釈が付与されるのは一部に限られます。

1. **エンティティ注釈（NER）：** エンティティには人物、場所、製品、組織などが含まれ、`entity` ペイロードセクションで提供されます。Post テキストで明示的に言及された内容（固有表現認識）に基づき、プログラムによって付与されます。

2. **コンテキスト注釈：** Post のテキスト分析から導出され、ドメインとエンティティの組み合わせとして提供されます。これにより、これまで発見が難しかったトピックの Post を見つけやすくなります。現在、Post を分類するために 80 以上のドメインを使用しています。利用可能なコンテキスト注釈エンティティの CSV ファイルは、[GitHub リポジトリ](https://github.com/xdevplatform/twitter-context-annotations)で入手できます。

<div id="post-annotation-types">
  ## 投稿のアノテーションの種類
</div>

<div id="entities">
  ### エンティティ
</div>

エンティティ注釈は、`entities` フィールド内でプログラムによって定義され、ペイロードでは注釈として表現されます。各注釈には信頼度スコアが付与され、投稿テキスト内のどこでエンティティが認識されたか（`start` および `end` フィールド）を示します。

エンティティ注釈の種類には次が含まれます:

- **Person** - 例: Barack Obama、Daniel、George W. Bush
- **Place** - 例: Detroit、Cali、San Francisco
- **Product** - 例: Mountain Dew、Mozilla Firefox
- **Organization** - 例: Chicago White Sox、IBM
- **Other** - 例: Diabetes、Super Bowl 50

<div id="context-annotations">
  ### コンテキスト注釈
</div>

以下の質問は、投稿注釈の「コンテキスト注釈」要素に特化した内容です。詳細は [概要](/ja/x-api/fundamentals/post-annotations) ページをご参照ください。

<AccordionGroup>
  <Accordion title="How does Twitter context annotations work?">
    X ではツイートを意味的に分類します。つまり、特定のトピックに関連するキーワード、ハッシュタグ、@ハンドルのリストを作成・管理しています。ツイートに指定したテキストが含まれていれば、適切なラベルが付与されます。これは、モデルを訓練してテキスト（この場合はツイート）を分類し、出力／分類と併せて確率スコアを算出する機械学習アプローチとは異なります。
  </Accordion>

  <Accordion title="How do I know that your data is complete and trustworthy?">
    Twitter のアノテーションは、数年にわたり磨き上げられたリサーチおよび QA のプロセスに基づき、ドメインの専門家が整備しています。このプロセスは、優れた適合率と再現率を維持できる範囲でデータ追跡を拡張するためのカスタムツールによって支えられています。さらに、当社のデータは内部チームによる定期的な監査を受けており、直近の数四半期にわたり適合率スコアは約 80% を記録しています。
  </Accordion>

  <Accordion title="How do you ensure precision?">
    私たちのチームは、高い適合率と再現率を維持するため、毎日エンティティの品質確認（QA）を実施しています。さらに、四半期ごとに社内チームが監査を行い、すべてのドメインにわたって合計1万件のTweetを手作業でレビューして適合率スコアを算出します。
  </Accordion>

  <Accordion title="How do you decide what to track?">
    スポーツやテレビなど一部のドメインでは、グラフの構築に自動取り込みを用いています。News ドメインでは、Twitter Moments チームが公開したストーリーに関するデータを追跡しています。その他の領域では、プラットフォーム上で高い話題性を集めるトピックを追跡対象として特定するために、チームがさまざまな調査手法を用います。
  </Accordion>

  <Accordion title="What historical support is available with Tweet Annotations?">
    エンティティが公開された時点からデータの追跡が始まります。したがって、特定のエンティティが追跡対象になる前に公開された投稿には注釈を付与しません。たとえば、新興ブランド／企業がタクソノミーに追加された場合でも、その注釈が追加される以前にそのブランドについて投稿されたものを遡って注釈することはありません。
  </Accordion>

  <Accordion title="What underlying 'semantics' does Twitter rely on to annotate a Tweet?">
    ツイートのアノテーションは、ツイートに注釈を付与するため、次の意味要素で構成されます。

    - アカウント — 特定のハンドルからのツイート、またはそのハンドルへの言及を含むツイートに注釈を付与できます
    - ハッシュタグ
    - キーワード／フレーズ

    PowerTrack などのフィルタードストリーミング API に精通しているお客様には、アノテーションで用いる意味要素は、ツイートのストリームをフィルタリングするために定義されるブール式のルールと原理的に類似しています。ツイートが基礎となる意味条件に合致した場合、対応するタグが付与されます。
  </Accordion>

  <Accordion title="Why do some Tweets have entities associated with them while others do not?">
    目標は可能な限り多くの投稿に注釈を付与することですが、次のような理由で注釈が付かない場合があります。

    - 一部の投稿は意味的な情報量が不十分で、現行の注釈ルールではラベル付けできません
    - 一部の投稿はトピック性がありません
    - 非常に一過性の話題についての投稿で、当社のグラフに存在しない
    - 対応する言語／市場をカバーしていない
    - 言語／市場はカバーしているが、既に追跡しているトピックに関連するトピックや特定の用語／アカウント／ハッシュタグが不足している
  </Accordion>

  <Accordion title="When there are multiple domains (for example, [3,30]), does the Entity ID remain the same?">
    あるエンティティは複数のドメインに属することがあります。ドメイン ID は変わりますが、エンティティ ID は変わりません。Donald Glover は人物（ドメイン 10）、俳優（ドメイン 56）、ミュージシャン（ドメイン 54）ですが、彼のエンティティ ID は 875072662527029248 のままです。
  </Accordion>

  <Accordion title="Do you have an established timeline for show/movie tracking? In other words, how long is a show/movie tracked before/after release?">
    追跡はリリースの1か月前から開始します。Marvelの映画のような人気の大作であれば、公開を予告し始めた時点で追跡を開始できます。
  </Accordion>

  <Accordion title="Do movies have a locale filter similar to the one for TV shows?">
    いいえ、違います。
  </Accordion>
