<div id="tweet">
  ### Tweet
</div>

Tweet は、Twitter に関するあらゆる要素の基本的な構成単位です。Tweet オブジェクトには、`id`、`text`、`created_at` などの「ルートレベル」フィールドが多数あります。Tweet オブジェクトは、`user`、`media`、`poll`、`place` など複数の子オブジェクトの「親」オブジェクトでもあります。Tweet オブジェクトのこれらのルートレベルフィールドをリクエストする際は、フィールドパラメータ `tweet.fields` を使用してください。

Tweet オブジェクトは user リソースで検索および展開できます。リクエストした Tweet に関連する追加の Tweet も、Tweet リソースで検索および展開できます。オブジェクトは、user リソースでは `?expansions=pinned_tweet_id`、Tweet リソースでは `?expansions=referenced_tweets.id` を指定して、デフォルトフィールドのみを持つオブジェクトとして展開可能です。オブジェクトを補完するために追加フィールドをリクエストする場合は、拡張に加えてフィールドパラメータ `tweet.fields` を使用してください。

| フィールド値           | 型   | 説明                                                                                                                           | 使用方法                                                  |
|:-----------------------|:--------|:---------------------------------------------------------------------------------------------------------------------------------------|:----------------------------------------------------------------------|
| **id (default)**      | string | リクエストされたツイートの一意の識別子。                                                                                        | これを使用して特定のツイートをプログラムで取得します。              |
| **text (default)**    | string | ツイートの実際の UTF-8 テキスト。有効な文字の詳細は [twitter-text](https://github.com/twitter/twitter-text/) を参照してください。   | キーワード抽出および感情分析／分類。            |
| **edit\_history\_tweet\_ids (default)** | object | ツイートのすべてのバージョンを示す一意の識別子。編集がないツイートには ID が 1 つ、編集履歴があるツイートには複数の ID があります。 | この情報を使用してツイートの編集履歴を取得します。            |
| **attachments**       | object | このツイートに存在する添付（ある場合）の種類を指定します。                                                                    | リクエストしたエキスパンションで返されるオブジェクトの把握に使用。         |
| **author\_id**         | string | このツイートを投稿したユーザーの一意の識別子。                                                                             | User オブジェクトのハイドレーション、ピアレビュー用のデータセット共有。              |
| **context\_annotations** | array | ツイートのコンテキスト注釈を含みます。                                                                                          | エンティティ認識／抽出、トピック分析。                     |
| **conversation\_id**   | string | 会話の元ツイート（直接の返信、返信への返信を含む）のツイート ID。                         | これを使用してツイートから会話を再構築します。               |
| **created\_at**        | date (ISO 8601) | ツイートの作成時刻。                                                                                                      | 時系列分析やツイートが作成されたタイミングの把握に有用。 |
| **edit\_controls**     | object | ツイートをあとどれくらい編集できるか、残りの編集回数を示します。                                                | ツイートが編集可能かどうかを判定するために使用します。            |
| **entities**          | object | ツイートのテキストから抽出・解析されたエンティティ。Twitter Objects の entities を参照してください。                                       | ハッシュタグ、URL、メンションなどに関する追加情報を提供。 |
| **in\_reply\_to\_user\_id** | string | 表示されているツイートが返信である場合、このフィールドには元のツイートの投稿者 ID が含まれます。                                       | ツイートが別のツイートへの返信であったかを判定。                  |
| **lang**              | string | X により検出された場合のツイートの言語。                                                                                      | 使用言語ごとにツイートを分類。                                  |
| **non\_public\_metrics** | object | リクエスト時点のツイートに関する非公開のエンゲージメント指標。ユーザーコンテキスト認証が必要です。                       | ツイートで生成された総インプレッション数を判定。                 |
| **organic\_metrics**   | object | リクエスト時点のツイートに関するオーガニック文脈で追跡されたエンゲージメント指標。ユーザーコンテキスト認証が必要です。  | ツイートのオーガニックエンゲージメントを測定。                            |
| **possibly\_sensitive** | boolean | コンテンツがセンシティブと認識される可能性があるかを示します。                                                                           | 特定タイプのコンテンツの流通を調査。                       |
| **promoted\_metrics**  | object | リクエスト時点のツイートに関するプロモーション文脈で追跡されたエンゲージメント指標。ユーザーコンテキスト認証が必要です。 | プロモーション時のツイートのエンゲージメントを測定。               |
| **public\_metrics**    | object | リクエスト時点のツイートに関する公開エンゲージメント指標。                                                                 | ツイートのエンゲージメントを測定。                                            |
| **referenced\_tweets** | array  | このツイートが参照しているツイートの一覧（リツイート、引用ツイート、返信など）。                                                | リツイート等の会話的側面を把握。                  |
| **reply\_settings**    | string | 指定されたツイートに誰が返信できるかを示します。オプションは「everyone」「mentioned\_users」「followers」です。                                  | ツイートの返信設定を判定。                 |
| **withheld**          | object | [withheld content](https://help.x.com/en/rules-and-policies/tweet-withheld-by-country) に関する保留情報を含みます。     |                                                                     |
| **media\_metadata**          | array | ツイートのメディア添付のメタデータを含みます。     | ツイートのメディア添付の `alt_text` などの追加メタデータを取得。 |

**ツイートオブジェクトの取得**

**サンプルリクエスト**

次のリクエストでは、[Tweets lookup](/ja/resources/fundamentals/rate-limits) エンドポイントでツイートのフィールドをリクエストしています。必ず `$BEARER_TOKEN` を自身が生成した [Bearer Token](/ja/resources/fundamentals/authentication) に置き換えてください。

```bash
curl --request GET 'https://api.x.com/2/tweets?ids=1212092628029698048&tweet.fields=attachments,author_id,context_annotations,created_at,entities,geo,id,in_reply_to_user_id,lang,possibly_sensitive,public_metrics,referenced_tweets,text,withheld&expansions=referenced_tweets.id' --header 'Authorization: Bearer $BEARER_TOKEN'
```

<div id="user">
  ### User
</div>

user オブジェクトには、参照先ユーザーを記述する X のユーザーアカウントのメタデータが含まれます。user オブジェクトは、[users lookup](/ja/x-api/users/lookup/introduction) エンドポイントで返される主要なオブジェクトです。このエンドポイントで追加のユーザーフィールドを取得するには、フィールドパラメーター `user.fields` を使用します。

user オブジェクトは、Tweet オブジェクト内の子オブジェクトとしても含まれ、展開できます。デフォルトフィールドのみを含む簡易オブジェクトを取得するには、`?expansions=author_id` または `?expansions=in_reply_to_user_id` による展開が利用可能です。オブジェクトを完全な形にするために追加フィールドを要求する場合は、フィールドパラメーター `user.fields` と併用して展開を行ってください。

<div id="space">
  ### スペース
</div>

Spacesは、ライブ音声での会話を通じて表現と交流を可能にします。Spaceのデータディクショナリには、そのSpaceに関する関連メタデータが含まれており、すべての詳細はリアルタイムで更新されます。

ユーザーオブジェクトは user リソースで参照および展開できます。これらのオブジェクトは、`expansions` クエリパラメータに `host_ids`、`creator_id`、`speaker_ids`、`mentioned_user_ids` のうち少なくとも1つを追加することで展開できます。

ツイートとは異なり、Spaces はエフェメラル（短命）であり、終了後や作成者によりキャンセルされた場合は利用できなくなります。アプリで Spaces のデータを扱う際は、常に最新の情報を返す責任があり、プラットフォーム上で利用できなくなったデータは削除する必要があります。[Spaces ルックアップエンドポイント](/ja/x-api/spaces/lookup/introduction)を利用すれば、ユーザーの期待や意図を適切に尊重できます。

| フィールド値          | 型              | 説明                                                                                                         | 使用方法                                                 |
| :-------------------- | :--------------- | :----------------------------------------------------------------------------------------------------------- | :------------------------------------------------------- |
| id (デフォルト)       | string          | リクエストしたSpaceの一意の識別子。<br />`"id": "1zqKVXPQhvZJB"`                                              | レスポンスで返されたSpaceを一意に識別します。           |
| state (デフォルト)    | string          | Spaceが開始済みか、開始予定か、終了済みかを示します。<br />`"state": "live"`                                  | ライブまたは予約済みのSpaceをフィルタリングします。      |
| created\_at           | date (ISO 8601) | このSpaceの作成時刻。<br />`"created_at": "2021-07-04T23:12:08.000Z"`                                          | Spaceの作成時点を把握し、時刻でソートします。           |
| ended\_at             | date (ISO 8601) | 該当する場合、Spaceが終了した時刻。<br />`"ended_at": "2021-07-04T00:11:44.000Z"`                             | ライブSpaceの終了時刻から実行時間を算出します。         |
| host\_ids             | array           | Spaceのホストの一意の識別子。<br />`"host_ids": ["2244994945", "6253282"]`                                     | Userオブジェクトを展開し、エンゲージメントを把握します。 |
| lang                 | string          | 検出された場合のSpaceの言語。<br />`"lang": "en"`                                                              | 言語別にSpaceを分類します。                              |

| invited\_user\_ids     | array           | スピーカーとして招待されたユーザーIDの一覧。<br />`"invited_user_ids": ["2244994945", "6253282"]`              | Userオブジェクトを展開し、エンゲージメントを把握します。 |
| participant\_count    | integer         | ホストとスピーカーを含む、Space内のユーザー数。<br />`"participant_count": 420`                               | エンゲージメントを把握し、レポートを作成します。         |

| scheduled\_start      | date (ISO 8601) | Spaceの予定開始時刻。<br />`"scheduled_start": "2021-07-14T08:00:00.000Z"`                                    | カレンダー通知と連携します。                             |
| speaker\_ids          | array           | いずれかの時点で発言したユーザーの一覧。<br />`"speaker_ids": ["2244994945", "6253282"]`                       | Userオブジェクトを展開し、エンゲージメントを把握します。 |
| started\_at           | date (ISO 8601) | Spaceの実際の開始時刻。<br />`"started_at": "2021-07-14T08:00:12.000Z"`                                       | Spaceの開始時刻を特定します。                           |
| title                | string          | Spaceのタイトル。<br />`"title": "Say hello to the Space data object!"`                                       | キーワード、ハッシュタグ、メンションを把握します。       |
| topic\_ids            | array           | Spaceの作成者が選択したトピックのID。<br />`"topic_ids": ["2244994945", "6253282"]`                            | キーワード、ハッシュタグ、メンションを把握します。       |
| updated\_at           | date (ISO 8601) | Spaceのメタデータの最終更新時刻。<br />`"updated_at": "2021-07-11T14:44:44.000Z"`                              | 情報を最新の状態に保ちます。                             |

<div id="list">
  ### リスト
</div>

List オブジェクトには、参照されているリストを説明する [Twitter Lists](https://help.x.com/en/using-twitter/twitter-lists) のメタデータが含まれます。List オブジェクトは、List ルックアップエンドポイントで返される主要なオブジェクトです。このエンドポイントで追加の List フィールドを要求する場合は、フィールドパラメータ `list.fields` を使用します。

List オブジェクトは、他のデータオブジェクトの子要素としては返されません。ただし、ユーザーオブジェクトはユーザーリソースで取得・展開できます。これらのオブジェクトは、`expansions` クエリパラメータに `owner_id` を追加することで展開可能です。主要な List オブジェクトを補完するために追加フィールドを要求する際は、この展開を `list.fields` フィールドパラメータと併用し、展開オブジェクトを補完するために `user.fields` を使用します。

| フィールド値   | 型              | 説明                                                                                                                                                             | 利用方法                                                               |
|:---------------|:-----------------|:------------------------------------------------------------------------------------------------------------------------------------------------------------------|:-------------------------------------------------------------------------|
| id (default)  | string          | このリストの一意の識別子。<br />`"id": "2244994945"`                                                                                                              | 特定のリストに関する情報をプログラムで取得するために使用します。        |
| name (default)| string          | 作成時に定義されたリスト名。<br />`"name": "Twitter Lists"`                                                                                                        |                                                                         |
| created\_at    | date (ISO 8601) | リストが作成された UTC の日時。<br />`"created_at": "2013-12-14T04:35:55.000Z"`                                                                                     | リストが X 上でどれくらい存在しているかを判断します。                   |
| description   | string          | リストについてユーザーに伝えるための簡潔な説明。<br />`"description": "People that are active members of the Bay area cycling community on Twitter."`             |                                                                         |
| follower\_count| integer         | このリストをフォローしているユーザー数。<br />`"follower_count": 198`                                                                                               |                                                                         |
| member\_count  | integer         | このリストのメンバー数。<br />`"member_count": 60`                                                                                                                  |                                                                         |
| private       | boolean         | リストが非公開かどうか。<br />`"private": false`                                                                                                                    |                                                                         |
| owner\_id      | string          | このリストの所有者の一意の識別子。<br />`"owner_id": "1255542774432063488"`                                                                                        | このユーザーが他のリストを所有しているかの確認や、User オブジェクトの展開に使用できます。 |

<div id="direct-message-events">
  ### ダイレクトメッセージイベント
</div>

ダイレクトメッセージ（DM）の会話はイベントで構成されます。X API v2 は現在、MessageCreate、ParticipantsJoin、ParticipantsLeave の3種類のイベントタイプをサポートしています。

DM イベントオブジェクトは [ダイレクトメッセージのルックアップ](/ja/x-api/direct-messages/lookup/introduction) エンドポイントで返され、[ダイレクトメッセージの管理](/ja/x-api/direct-messages/lookup/introduction) エンドポイントでダイレクトメッセージが正常に作成されると、MessageCreate イベントが作成されます。

DM イベントをリクエストする場合、デフォルトで含まれるイベントオブジェクトの属性（フィールド）は id、event\_type、text の3つです。追加のイベントフィールドを受け取るには、[fields](/ja/x-api/fundamentals/fields) パラメータ dm\_event.fields を使用して選択してください。選択可能なその他のイベントフィールドには、dm\_conversation\_id、created\_at、sender\_id、attachments、participant\_ids、referenced\_tweets などがあります。

これらのフィールドのいくつかは、ダイレクトメッセージイベントに関連する他の X オブジェクトの ID を示します。

- sender\_id - メッセージを送信した、またはグループ会話に参加者を招待したアカウントの ID
- participant\_ids - アカウント ID の配列。ParticipantsJoin と ParticipantsLeave イベントでは、この配列にはイベントを作成したアカウントの単一の ID が含まれます
- attachments - 送信者が X にアップロードしたコンテンツのメディア ID を提供します
- referenced\_tweets - text フィールド内でツイートの URL が見つかった場合、そのツイートの ID がレスポンスに含まれます

sender\_id、participant\_ids、referenced\_tweets.id、attachments.media\_keys の [expansions](/ja/x-api/fundamentals/expansions) を使用して、これらの X オブジェクト ID を展開できます。

|     |     |     |     |
| :--- | :--- | :--- | :--- |
| **フィールド値** | **型** | **説明** | **使用方法** |
| id (default) | string | イベントの一意の識別子。<br /><br />"id": "1050118621198921728" | これを使用して、特定の会話イベントをプログラムで取得します（v1.1 エンドポイントで利用可能）。 |
| event\_type (default) | string | イベントの種類を示します。現在サポートされているのは次の3種類です：<br /><br />\* MessageCreate<br />    <br />\* ParticipantsJoin<br />    <br />\* ParticipantsLeave<br />    <br /><br />"event\_type": "MessageCreate" | 会話履歴を取得する際に、メッセージが作成された時点、グループ会話では参加者が参加・退出した時点を把握できます。すべての GET メソッドは、event\_type= クエリパラメータで特定のイベントタイプによるフィルタリングをサポートします。 |
| text (default) | string | ダイレクトメッセージの UTF-8 テキスト本体。<br /><br />"text": "Hello, just you!" | チャットボットでは、メッセージ内容の分析や自動応答の決定に利用できます。会話検索機能の構築にも使えます。 |
| sender\_id | string | イベントを作成したユーザーの ID。レスポンスでこのオブジェクトを展開するには、sender\_id を expansions に含め、user.fields クエリパラメータで関心のある User オブジェクト属性を指定します。<br /><br />"sender\_id": "906948460078698496" | MessageCreate または ParticipantsJoin イベントを作成したユーザーの User オブジェクトを取得します。 |
| participant\_ids | array (of strings) | グループ会話に参加・退出する参加者の ID。新しいグループ会話の作成時にも使用されます。レスポンスでこのオブジェクトを展開するには、participant\_ids を expansions に含め、user.fields クエリパラメータで関心のある User オブジェクト属性を指定します。<br /><br />"participant\_ids": \[<br /><br />     "906948460078698496"<br /><br />] | グループ会話に参加・退出する参加者の User オブジェクトを取得するために使用します。 |
| dm\_conversation\_id | string | イベントが属する会話の一意の識別子。<br /><br />"dm\_conversation\_id": "1584988213961031680" | これを使用して、会話からイベントをプログラムで取得したり、ダイレクトメッセージを追加したりできます。 |
| created\_at | date (ISO 8601) | ツイートの作成時刻（UTC）。<br /><br />"created\_at": "2019-06-04T23:12:08.000Z" | ダイレクトメッセージが作成された時点や、会話の参加者が参加・退出した時点を把握するのに使用できます。 |
| referenced\_tweets | array | ダイレクトメッセージ文中で言及されたツイートの ID。レスポンスでこのオブジェクトを展開するには、referenced\_tweets.id を expansions に含め、tweet.fields クエリパラメータで関心のある Tweet オブジェクト属性を指定します。<br /><br />"referenced\_tweets": \[<br /><br />    <br /><br />"id": "1578868150510456833"<br /><br />    <br /><br />] | ダイレクトメッセージがツイートを参照している場合、これらの ID を使用してツイートの詳細を取得できます。 |
| attachments | object | メディアが添付されたダイレクトメッセージについて、アップロード済みコンテンツ（写真、動画、GIF）のメディアキーを提供します。レスポンスでこのオブジェクトを展開するには、attachments.media\_keys を expansions に含め、media.fields クエリパラメータで関心のある media オブジェクト属性を指定します。現在、添付は 1 件のみサポートされています。<br /><br />"attachments": <br /><br />    "media\_keys": \[<br /><br />        "3\_1136048009270239232"<br /><br />    ]<br /><br /> | ダイレクトメッセージに添付されたメディアオブジェクトの把握。 |

<div id="how-to-use-fields-and-expansions">
  ## フィールドとエクスパンションの使い方
</div>

デフォルトでは、[fields](/ja/x-api/fundamentals/fields) や [expansions](/ja/x-api/fundamentals/expansions) パラメータを指定せずにリクエストすると、X API v2 のデータオブジェクトには一部のデフォルトフィールドのみが含まれます。本ガイドでは、レスポンスで追加のオブジェクトやフィールドを受け取るために、リクエストで `fields` および `expansions` クエリパラメータを使う方法を説明します。

本ガイドでは、以下のポストのスクリーンショットに対して複数のフィールドを要求します。
 

![この画像には、@X が投稿したポストのスクリーンショットが含まれています。本文、ユーザー名、公開日時、ソース、公開メトリクスが確認できます。動画も含まれています。](https://cdn.cms-twdigitalassets.com/content/dam/developer-twitter/ScreenShotDataDictionaryGuide.png.twimg.1920.png)

スクリーンショットのとおり、ポストには投稿者、ポストのメトリクス、作成タイムスタンプ、動画、動画の再生回数など、画面上で確認できる情報がいくつもあります。スクリーンショットには表示されていないものの、リクエスト可能なデータも複数存在します。 

API にリクエストする場合、デフォルトのレスポンスはシンプルで、デフォルトのポストフィールド（id と text）のみが含まれます。また、利用しているエンドポイントが返す主要オブジェクトのみが取得され、主要オブジェクトに関連するデータオブジェクトは含まれません。

このシンプルさと fields および expansions パラメータを組み合わせることで、ユースケースに応じて必要なフィールドだけをリクエストできます。
