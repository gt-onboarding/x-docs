X は、X API を用いて構築する開発者コミュニティへの支援に取り組んでいます。この取り組みの一環として、当社は API を開発者に対してオープンで公正に、X 上の人々にとって安全に、そして X プラットフォーム全体にとって有益なものにすることを目指しています。X のコンテンツをオフラインで保存するすべての開発者は、データがユーザーの意図と X 上のコンテンツの最新状態を反映していることを確実にすることが極めて重要です。たとえば、X 上の誰かが投稿やアカウントを削除したり、投稿を非公開化したり、投稿から位置情報を削除した場合、X と当社の開発者の双方にとって、その人の期待と意図を尊重することが不可欠です。バッチコンプライアンスエンドポイントは、開発者が [X Developer Agreement and Policy](https://developer.x.com/en/developer-terms/policy) に準拠した X データの維持を支援するための簡便なツールを提供します。 

これらのバッチコンプライアンスエンドポイントを使用すると、投稿 ID またはユーザー ID の大規模なデータセットをアップロードし、コンプライアンスステータスを取得して、準拠のためにアクションが必要なデータを特定できます。なお、バッチコンプライアンスエンドポイントの使用は前述のユースケースに限定されており、それ以外の目的での使用は禁じられており、違反した場合は措置の対象となることがあります。

通常、このエンドポイントの利用には 4 つのステップがあります。

1. **コンプライアンスジョブを作成する**
   ジョブタイプを指定できます（tweets または users の値を使用して、アップロードするデータセットに投稿 ID かユーザー ID のどちらが含まれるかを示します）。任意の時点で、ジョブタイプごとに同時実行できるジョブは 1 つまでです。
2. **upload\_url にデータセットをアップロードする**
   次に、提供された upload\_url にプレーンテキストファイルとしてデータセットをアップロードします。ファイルの各行には単一の投稿 ID またはユーザー ID を含めます。upload\_url は 15 分で有効期限が切れます。
3. **（任意）ジョブステータスを確認する**
   コンプライアンスジョブのステータス（created、in\_progress、failed、complete）を確認できます。
4. **結果をダウンロードする**
   ジョブが完了したら、download\_url を使用して結果をダウンロードできます。download\_url は（ジョブ作成時点から）1 週間で有効期限が切れます。

   結果には、JSON オブジェクトの集合（1 行につき 1 オブジェクト）が含まれます。各オブジェクトには、投稿 ID、投稿の作成日（日時で投稿を整理するのに有用）、必要なアクション、コンプライアンスアクションの理由、ユーザーが凍結された日付が含まれます。

結果には次のコンプライアイベントタイプが含まれます。

- deleted - 投稿またはユーザーアカウントが削除されたことを示します
- deactivated - 投稿またはユーザーアカウントが無効化されたことを示します
- scrub\_geo - 投稿またはユーザーに関連付けられた位置情報が削除されたことを示します
- protected - 投稿を行ったアカウントが非公開になったことを示します
- suspended - 投稿を行ったアカウントが凍結されたことを示します

**アカウントのセットアップ**

これらのエンドポイントにアクセスするには、以下が必要です。

- 承認済みの [developer account](https://developer.x.com/en/portal/petition/essential/basic-info)
- [Project](/ja/resources/fundamentals/projects) 内にある [developer App](/ja/resources/fundamentals/developer-apps) のキーとトークンを使用して認証すること

X API v2 エンドポイントへのアクセス方法の詳細は、[getting started guide](/ja/x-api/getting-started/getting-access) をご覧ください。
