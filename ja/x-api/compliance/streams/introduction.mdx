X は、X API を活用して開発する開発者コミュニティを支援することに尽力しています。その一環として、API を開発者に対してオープンかつ公正に保ち、X 上の人々にとって安全であり、プラットフォーム全体にとって有益なものとすることを目指しています。X のコンテンツをオフラインで保存する開発者は、データがユーザーの意図および X 上のコンテンツの最新の状態を反映していることを確実にすることが極めて重要です。たとえば、X 上のユーザーがポストやアカウントを削除したり、ポストを非公開にしたり、ポストを編集した場合、その人の期待と意図を尊重することは、X と開発者の双方にとって不可欠です。

コンプライアンスイベントのリアルタイムストリームは、開発者が [X Developer Agreement and Policy](https://developer.x.com/en/developer-terms/policy) に準拠して X データを維持するためのツールを提供します。 

コンプライアンスイベントのストリームは 2 種類あり、1 つはポストのコンプライアンスイベント用、もう 1 つはユーザーのコンプライアンスイベント用です。これらのストリームは Enterprise アクセスで利用可能で、大量のデータを取り込むパートナーが、ポストの編集イベントなどのコンプライアンスイベントをリッスンできるよう設計されています。

これらのストリームでは、次のイベントが提供されます。 

**ポストコンプライアンスストリーム:** 

- delete - ポストが削除されたことを示します。
- tweet\_edit - ポストが編集され、更新後のポストの ID を提供します。 
- withheld - ポストが 1 つ以上の国で保留されたことを示します。 
- drop - ポストを公開表示から削除すべきであることを示します。
- undrop - ポストを再び表示でき、公開として扱えることを示します。

**ユーザーコンプライアンスストリーム:** 

- **user\_delete** - ユーザーアカウントが削除されたことを示します
- **user\_undelete** - ユーザーアカウントの削除が取り消されたことを示します
- **user\_protect** - ユーザーアカウントが非公開になったことを示します
- **user\_unprotect** - ユーザーアカウントが公開になったことを示します
- **user\_withheld** - ユーザーアカウントが 1 つ以上の国で保留されたことを示します。 
- **user\_suspend** - ユーザーアカウントが凍結されたことを示します
- **user\_unsuspend** - ユーザーアカウントの凍結が解除されたことを示します
- **user\_profile\_modification** - ユーザープロフィールが更新されたことを示します（説明、名前、場所、URL の更新を含みます）。

**アカウントのセットアップ**

これらのエンドポイントにアクセスするには、次が必要です:

- 承認済みの [developer account](https://developer.x.com/en/portal/petition/essential/basic-info)
- [Project](/ja/resources/fundamentals/projects) 内の [developer App](/ja/resources/fundamentals/developer-apps) のキーとトークンを使用して認証すること 

X API v2 のエンドポイントへのアクセス方法の詳細は、[getting started guide](/ja/x-api/getting-started/getting-access) をご覧ください。
