<Update label="Aug 3, 2022" description="X API v2">
  ### フィルタリングオペレーターのエイリアスの追加

  フィルタリング／クエリオペレーターの「エイリアス」が、エンタープライズ、プレミアム、および v2 を通じて search と フィルタ済みストリーム エンドポイントに追加されました。多くの場合、オペレーターは statuses ではなく posts を参照します。例：「in\_reply\_to\_status\_id」のエイリアスとして「in\_reply\_to\_tweet\_id」。別のケースでは、新しいオペレーター エイリアスにより照合の明確性が高まります。例えば、ポストには動画の添付は1つしかできないため、「has:videos」に対して新しい「has:video\_link」エイリアスが提供されています。さらに多くの例は [v2 filtered stream operator table](https://developer.x.com/en/docs/twitter-api/tweets/filtered-stream/integrate/build-a-rule#list) をご確認ください。
</Update>

<Update label="Dec 14, 2021" description="X API v2: Authentication">
  ### OAuth 2.0 認可のサポート

  本日より、すべての開発者はデベロッパーポータルで認証方法として OAuth 2.0 を選択することで、OAuth 2.0 を使用して認証できるようになりました。今回のリリースでは、機密クライアントおよびパブリッククライアントが追加され、関連するすべての v2 エンドポイントでこの認証方法を利用できます。

  OAuth 2.0 は業界標準の認可プロトコルであり、アプリケーションのスコープをより細かく制御でき、複数デバイスにまたがる認可フローを改善します。言い換えると、X 上のユーザー向けにアプリケーションを構築する開発者は、アプリがユーザーに要求する情報をより適切に制御できるため、必要なデータや情報のみをエンドユーザーに求めることができます。

  この最新の認可プロトコルにより、リクエストした特定のスコープのみを表示する、より簡潔で洗練された同意フローを提示して、エンドユーザーによるアプリの承認を得られるようになります。

  このリリースの詳細は、フォーラムでのお知らせをご覧ください。
</Update>

<Update label="Nov 15, 2021" description="X API v2: Early Access">
  ### X API v2 への主要なプラットフォーム更新

  **主要なプラットフォーム更新：X API v2 が主要バージョンに、新たな Essential および Elevated アクセス、ならびにポリシー変更**

  本日、X API v2 が X API の主要バージョンになったことを発表します。X API v2 には、既存の X API 上で構築されたアプリの 90% のニーズを満たすのに十分な数のエンドポイントと機能を提供しました。

  X API v2 の開発者エクスペリエンスをさらに向上させるため、以下の変更を本日リリースします。

  - API への高速かつ無料の Essential アクセス、そしてユースケースが承認された開発者向けの無料の Elevated アクセス。これらのアクセスレベルの詳細は、About the X API ページでご確認いただけます。

  - 当社の Developer Policy において、X のコア機能を用いた構築方法を制限し、アプリでサポートできるユーザー数を制限していた文言を削除します。

    - プラットフォームを開放し、X をより良く、健全にし、公共の会話を拡張するツールや製品の構築を促進します。[当社が皆さんに構築してほしいソリューションの一覧をご覧ください](https://developer.x.com/en/docs/twitter-api/what-to-build)。

    - 具体的には、X 体験（X のコア機能を含む）の複製を制限していた条項や、多数のユーザートークンの保有に許可を要していた条項を削除しました。

    - X 上の人々を支援するソリューションの構築には、開発者が X で提供されている機能の一部を構築（または複製）する必要が生じることがよくあります。今回の Developer Policy の変更は、開発者エコシステムに明確性をもたらし、開発者が構築・イノベーションを行い、公共の会話に影響を与えやすくするオープンな API プラットフォームを提供することを目的としています。

  このリリースおよびこの段階に至るまでの X API のリリースについては、[フォーラム投稿](https://devcommunity.x.com/t/ushering-in-a-new-era-for-the-twitter-developer-platform-with-the-twitter-api-v2/162087)で詳しくご紹介しています。

<Update label="Feb 13, 2019" description="X API: Enterprise v1.1">
  ### is:reply 演算子の追加

  本日、演算子 is:reply が [enterprise search APIs](/ja/x-api/enterprise-gnip-2.0/fundamentals/search-api) および有料版の [premium search endpoints](https://developer.x.com/content/developer-twitter/en/docs/tweets/search/api-reference/premium-search) で利用可能になりました。

  is:reply ブール演算子は、ポストへのすべての返信を抽出する、またはルールに一致する明示的な返信を除外するために使用できます。この演算子は is:quote 演算子と同様に機能します。

  Search API では、すべての「is:」および「has:」演算子は、キーワードやハッシュタグなどの単独で使用可能な演算子と併用する必要がある点にご留意ください。

  この演算子については[ドキュメント](/ja/x-api/enterprise-gnip-2.0/fundamentals/rules-filtering)で詳しくご確認いただけます。
</Update>

<Update label="May 10, 2018" description="X API: Standard v1.1">
  ### 引用ペイロードのレンダリング変更

  本日、引用ペイロード内での URL のレンダリング方法を変更します。

  引用ポストを参照できるようにするため、新たに "quoted\_status\_permalink" エンティティを追加し、引用ポストの "text" フィールドから t.co リンクを削除します。

  新しい形式では、"quoted\_status" オブジェクト内の引用ポストの "text" フィールドに t.co リンクは付加されません。

  さらに、引用ポストの t.co url、expanded\_url、display\_url は ‘urls’ エンティティには含めません。代わりに、これらは "url"、"expanded"、"display" の URL 属性を持つ新しい "quoted\_status\_permalink" オブジェクトに追加されます。これは、"quoted\_status\_permalink" オブジェクトが引用ポストへの t.co リンク専用となり、一方で "entities.urls" 配列は元のポストまたは引用内で共有されたリンクの識別に使用できることを意味します。

  詳細は[フォーラムの投稿](https://devcommunity.x.com/t/updating-how-urls-are-rendered-in-the-quote-tweet-payload/105473)および[更新されたドキュメント](https://devcommunity.x.com/t/upcoming-changes-to-the-developer-platform/104603)をご覧ください。
</Update>

<Update label="Nov 14, 2017" description="X API: Premium v1.1">
  ### Premium API のローンチ

  Premium API をリリースしました。

  これらの新しい API は当社のエンタープライズデータプラットフォームの品質と安定性を基盤としており、段階的な料金モデルによってより広くご利用いただけます。標準エンドポイントでの7日分という制限に代わり、初めて30日分のポストを横断検索できるようになりました。オプションで、プロフィールの位置情報、URL 展開、投票データといったプレミアム拡張にもアクセス可能です。新機能の一部は次のとおりです。

  - リクエストあたりに取得できるポスト数の増加（Sandbox + Premium）

  - より複雑なクエリを可能にする、はるかに表現力の高いクエリ言語（Sandbox + Premium）

  - ポストの時系列カウントを返す counts エンドポイント（Premium）

  - 展開された URL や Profile Geo などのメタデータ拡張（Premium）

  このリリースの詳細は[当社のブログ](https://blog.x.com/developer/en_us/topics/tools/2017/introducing-twitter-premium-apis.html)をご覧ください。
</Update>

<Update label="May 5, 2017" description="X API: Standard v1.1">
  ### DM コマンドのオプションパラメーター

  statuses/update のオプションパラメーター `enable_dm_commands` を使用すると、移行期間終了前に DM コマンドのサポートを早期に無効化できます。デフォルト値は true（すなわち現在のレガシー動作）ですが、false に設定すると 11 月 1 日以降の新しい動作になります。statuses/update のオプションパラメーター `fail_dm_commands` を true に設定すると、API は DM コマンドに対して HTTP 403（エラーコード 151）を返します。デフォルト値は false です。詳細は[フォーラム](https://devcommunity.x.com/t/retiring-legacy-dm-commands-on-the-standard-tweet-api/86653)をご覧ください。
</Update>
