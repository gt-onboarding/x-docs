  ## 概要

<div id="introduction">
  ### はじめに
</div>

X デベロッパーポータルには、開発者が X API および X 広告 API へのアクセスを自主管理できるセルフサービスのツールが用意されています。

このポータルでは、次のことが可能です。

- X の[プロジェクト](/ja/resources/fundamentals/projects)と[アプリ](/ja/resources/fundamentals/developer-apps)（それらが提供する認証キーおよびトークンを含む）の作成と管理
- X API standard v1.1 および v2 エンドポイントに対するアクセスレベルと統合の管理
- 利用可能な各種エンドポイントと機能についての学習

<div id="ready-to-get-access">
  ### アクセスの準備はできましたか？
</div>

アカウントに登録すると、X API の利用を開始できます。

追加機能やより高い[投稿上限](/ja/x-api/fundamentals/post-cap)が必要な場合は、開発者ポータルで Basic または Pro を購入できます。Enterprise に関心がある方は、[こちら](https://docs.google.com/forms/d/e/1FAIpQLScO3bczKWO2jFHyVZJUSEGfdyfFaqt2MvmOfl_aJp0KxMqtDA/viewform)からお申し込みください。

  ## 開発者ポータルでできること
</div>

<div id="onboarding">
  ### オンボーディング
</div>

オンボーディングウィザードは、最初の[プロジェクト](/ja/resources/fundamentals/projects)と[アプリ](/ja/resources/fundamentals/developer-apps)のセットアップを順を追って案内します。API リクエストの認証に必要なクレデンシャルを受け取るには、プロジェクトとアプリを作成する必要があります。開発者アカウントに初めてアクセスする場合、ウィザードが表示されます。

このプロセスを通じて、認証用のキーとトークン一式が発行されます。詳細は[アプリの概要ページ](/ja/resources/fundamentals/developer-apps#overview)をご覧ください。X API での認証に必要な内容については、[認証セクション](/ja/resources/fundamentals/authentication)を参照してください。

**ご注意:** 後で参照できるように、[キーとトークンは安全な場所に保管](/ja/resources/fundamentals/authentication/guides/authentication-best-practices)してください。再生成せずにこれらのクレデンシャルを再表示することはできません。

<div id="project-and-app-management">
  ### プロジェクトとアプリの管理
</div>

開発者アカウントの主な役割の1つは、[プロジェクトとアプリを管理](/ja/resources/fundamentals/developer-apps#app-management)できるようにすることです。開発者は、開発者ポータルのダッシュボードから X のプロジェクトとアプリを作成および管理できます。ここでは、アプリIDを確認し、アプリの設定、権限、コールバックURLを編集し、キーとトークンを生成および失効（取り消し）できます。

<div id="learn-more-about-whats-available-with-x-api-v2">
  ### X API v2 で利用可能な内容の詳細を学ぶ
</div>

開発者ポータルには、X API の各バージョンやアクセスレベルについて詳しく学べる製品セクションがあります。

X API v2 の製品セクションには、Free、Basic、Pro、Enterprise の各アクセス階層に関する重要な情報が含まれています。このページでは、プロジェクトとアプリの制限、投稿上限、料金に加え、エンドポイント固有のレート制限や特別な属性の詳細を確認できます。異なるアクセスレベルを比較し、利用可能な場合は追加アクセスを申請することも可能です。

[すべての v2 アクセスレベル](/ja/x-api/introduction#access-levels)を確認してください。

<div id="team-management">
  ## チーム管理
</div>

<Note>
  **Enterprise v2 アカウントでのみご利用いただけます。**
</Note>

**[開発者ポータルの「チーム」ページを表示](https://developer.twitter.com/en/portal/teams)。**

<div id="why-use-team-functionality">
  ### チーム機能を使う理由
</div>

チーム機能は、X Developer Platform 内でのプロジェクトとアプリの共同開発を円滑にします。多くの場合、チームにはアクセス制御や請求／支払いを担当するメンバーがそれぞれいますが、この機能により、そのようなメンバーをプロジェクトに招待して共同作業に参加してもらえます。

<div id="inviting-team-members">
  ### チームメンバーの招待
</div>

チームにメンバーを招待するには、管理者がそのユーザーのXハンドルあてに招待を送信します。相手はメールで通知を受け取り、そのメールから招待を承諾できます。承諾後、Developer Agreement & Policy に同意する必要があり、その後メインアカウントのチームページにアクセスできるようになります。

**注意:** 現在、チーム管理では [App](/ja/resources/fundamentals/developer-apps) のクレデンシャルに基づいて API アクセスを付与または制限することはできません。チーム内で App の管理を共有することもできません。アプリ（キー／トークン）は、所有者以外が編集、作成、または削除できません。

<div id="team-dashboard">
  ### チームダッシュボード
</div>

チームダッシュボードの「members」タブでは、すべてのメンバーとそのロールを確認できます。チームの管理者であれば、開発者アクセスを管理し、各メンバーのロールを編集できます。

管理者はチームダッシュボードの「pending」タブにもアクセスできます。ここでは、送信済みの各招待の詳細を確認し、管理できます。

<div id="team-roles">
  ### チームの役割
</div>

**管理者ロール**

- チームのプロジェクトとアプリを管理する権限
- すべてのアプリ環境を管理する権限
- サブスクリプションの選択・アップグレードを行う権限
- 請求・支払い方法を更新する権限
- チームメンバーを追加・削除する権限
- チームメンバーのロールを編集する権限

**開発者ロール**

- 自分のプロジェクトとアプリを管理する権限
- チームのプロジェクトとアプリへの読み取り専用アクセス権
- チームを退出する権限
