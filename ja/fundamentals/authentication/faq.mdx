<div id="general">
  ## 一般
</div>

<AccordionGroup>
  <Accordion title="OAuth とは何ですか？">
    OAuth は、ユーザーがパスワードを共有せずに、自分に代わってアプリケーションの操作を承認できる認可プロトコルです。詳細は [oauth.net](http://oauth.net/) をご覧ください。
  </Accordion>

  <Accordion title="アクセストークンはどのように生成しますか？">
    アクセストークンを生成するには、[X アプリ](/ja/resources/fundamentals/developer-apps) が必要です。アクセストークンの詳細は[こちら](/ja/resources/fundamentals/authentication#oauth-1-0a-2)をご確認ください。
  </Accordion>

  <Accordion title="X アプリはどのように作成しますか？">
    [X アプリ](/ja/resources/fundamentals/developer-apps) を作成するには、[開発者アカウント](/ja/resources/fundamentals/developer-portal) が必要です。サインアップは[こちら](https://developer.x.com/en/portal/petition/essential/basic-info)から行えます。
  </Accordion>

  <Accordion title="すでに X アプリを持っている場合、そのアプリをどのように表示・編集できますか？">
    developer.x.com で X アカウントにログインしていれば、[X アプリのダッシュボード](https://developer.x.com/content/developer-twitter/en/apps) からアプリを表示および編集できます。
  </Accordion>
</AccordionGroup>

<div id="technical">
  ## 技術情報
</div>

<AccordionGroup>
  <Accordion title="アクセストークンはどのくらいの期間有効ですか？">
    アクセストークンには明示的な有効期限はありません。ユーザーがXのアカウント設定でアプリケーションのアクセスを明示的に取り消した場合、またはXがアプリケーションを停止した場合、アクセストークンは無効になります。アプリケーションが停止された場合は、[X app](/ja/resources/fundamentals/developer-apps) ダッシュボードに停止の注記が表示されます。
  </Accordion>

  <Accordion title="アクセストークンが無効になった場合はどうすればよいですか？">
    ユーザーのアクセストークンはいつ無効になってもおかしくないと想定してください。発生した場合は、ユーザーにアプリケーションの再認可を促してください。これを適切に処理することは、良好なユーザー体験のために重要です。
  </Accordion>

  <Accordion title="アプリケーション登録ページで読み取り/書き込みアクセスについて聞かれます。書き込みとは何を指しますか？">
    多くのユーザーは、アプリケーションが自分の情報を読み取ることには信頼を寄せますが、プロフィール情報の変更や新しい投稿の作成までは必ずしも許可しません。X API を通じて情報を更新する場合（名前、所在地の変更や新規投稿の追加など）は、HTTP POST が必要です。HTTP POST を要するすべての API メソッドは書き込みメソッドと見なされ、読み取りおよび書き込みアクセスが必要です。
  </Accordion>
</AccordionGroup>
