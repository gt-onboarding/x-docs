<div id="api-key-and-secret">
  ### API Key と Secret
</div>

API Key と Secret（Consumer Key と Secret とも呼ばれます）は、X API へのアクセスに必須となる最も基本的なクレデンシャルです。これらは X アプリのユーザー名とパスワードに相当し、どのアプリからのリクエストかを X API が識別するために使用されます。

これらのクレデンシャルは、[認証エンドポイント](/ja/resources/fundamentals/authentication/api-reference)で、[ユーザー Access Token と Secret](/ja/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret)や[Bearer Token](/ja/resources/fundamentals/authentication/oauth-2-0/bearer-tokens)などの追加クレデンシャルを生成するために使用できます。また、OAuth 1.0a の User Context 認証を要するリクエストを[認可](/ja/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request)するために、Access Token やその他の認可パラメータと併せてこれらのクレデンシャルを使用する必要があります。 

<div id="how-to-acquire-an-api-key-and-secret">
  #### API Key と Secret を取得する方法
</div>

X の API Key と Secret を取得するには、次の手順に従ってください。

1. [X の開発者アカウントに登録](https://developer.x.com/en/apply-for-access)
2. [developer portal](/ja/resources/fundamentals/developer-portal) 内で [X App](/ja/resources/fundamentals/developer-apps) を作成します。なお、[X API v2](/ja/x-api/introduction) を利用する場合は、X App を [Project](/ja/resources/fundamentals/projects) に追加する必要があります。
    

X App を作成すると、API Key と Secret に加えて Bearer Token が表示されます。これらの認証情報は一度しか表示されないため、パスワードマネージャーなどの安全な場所に必ず保存してください。

キーやトークンの取り扱いに関する推奨事項は、認証情報の侵害時に取るべき対応の詳細も含め、[authentication best practices](/ja/resources/fundamentals/authentication/guides/authentication-best-practices) のページをご参照ください。
 

<div id="how-to-find-and-regenerate-your-api-key-and-secret-after-app-creation">
  #### アプリ作成後に API Key と Secret を確認・再生成する方法
</div>

すでにアプリを作成済みで、API Key と Secret を確認する、または再生成する必要がある場合は、次の手順に従ってください。

1. developer portal に移動します
2. サイドナビの「Projects and Apps」ドロップダウンを展開します
3. 該当の API Key と Secret が関連付けられているアプリを開きます
4. 「Keys and tokens」タブに移動します

ここで、アプリに関連付けられているすべての認証情報を確認できます。 
 

<div id="how-to-use-your-api-key-and-secret">
  #### API Key と Secret の使い方
</div>

X Developer Platform を探索している段階であれば、プラットフォームで利用可能な機能を確認するために[ツールやライブラリ](/ja/resources/tools-and-libraries)の利用をお勧めします。これらのツールは認証を適切に処理し、時間と手間を大幅に省けます。特に、初心者の開発者には[Postman ではじめる](/ja/tutorials/postman-getting-started)または[Insomnia](https://insomnia.rest/)を推奨します。

ゼロからリクエストを作成したい場合は、[OAuth 1.0a リクエストの認可](/ja/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request)に関するガイドをご覧ください。
