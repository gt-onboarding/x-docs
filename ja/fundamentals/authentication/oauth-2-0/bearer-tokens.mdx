<div id="using-and-generating-an-app-only-bearer-token">
  ### アプリ専用の Bearer Token の使用と生成
</div>

Bearer Token は、開発者が X API を利用する際のより安全なエントリポイントを提供するもので、OAuth 2.0 の中核機能の一つです。

Bearer Token を用いる認証はアプリケーション専用認証（application-only authentication）とも呼ばれます。Bearer Token は、curl コマンドのようなスクリプトで生成する、形式が特定されていないバイト配列です。App の設定内にある Keys and tokens セクションの開発者ポータルから取得することもできます。この機能の詳細は [OAuth の公式ドキュメント](https://oauth.net/2/bearer-tokens/)をご覧ください。

<div id="when-are-they-used">
  #### いつ使用しますか？
</div>

Bearer Token の使用が必要なプロダクトは次のとおりです。

- [Engagement API](/ja/x-api/enterprise-gnip-2.0/fundamentals/engagement-api)
- [Account Activity API](/ja/x-api/enterprise-gnip-2.0/fundamentals/account-activity)
- v2 や Labs のエンドポイントなど、OAuth 2.0 の Bearer Token 認証を利用するその他の API

<div id="prerequisites">
  #### 前提条件
</div>

[開発者アカウントに登録](https://developer.x.com/en/portal/petition/essential/basic-info)し、[X App](/ja/resources/fundamentals/developer-apps)を作成しておく必要があります。準備が整ったら、[developer portal](/ja/resources/fundamentals/developer-portal)で API キーを取得してください。以下の手順に従ってください。

1. developer.x.com で X アカウントにログインします。
2. [X App dashboard](https://developer.x.com/content/developer-twitter/en/apps) に移動し、アクセス・トークンを生成したい X App を開きます。
3. 「keys and tokens」ページに移動します。
4. このページで API keys、ユーザー Access Tokens、Bearer Token を確認できます。

<div id="how-to-generate-a-bearer-token">
  ### Bearer Token の生成方法
</div>

アプリの Bearer Token は「Keys and Tokens」に他の情報とともにあります。

[X アプリ](/ja/resources/fundamentals/developer-apps) で事前に取得した以下のコンシューマー API キーを設定したうえで、次の cURL リクエストをコマンドラインにコピーしてください。なお、このページに記載のコンシューマー API キーは提供終了しており、実際のリクエストでは使用できません。

- **API key** `<API key>` 例: `xvz1evFS4wEEPTGEFPHBog`
- **API secret key** `<API secret key>` 例: `L8qq9PZyRg6ieKGEKhZolGC0vJWLw8iEJ88DRdyOg`

  ```bash
  curl -u "$API_KEY:$API_SECRET_KEY" \
  --data 'grant_type=client_credentials' \
  'https://api.x.com/oauth2/token'
  ```

以下は、API キーを入力した場合の curl リクエストの例です。

```bash
   curl -u 'xvz1evFS4wEEPTGEFPHBog:L8qq9PZyRg6ieKGEKhZolGC0vJWLw8iEJ88DRdyOg' \
  --data 'grant_type=client_credentials' \
  'https://api.x.com/oauth2/token'
```

レスポンスは次のようになります。これは提供終了した Bearer Token である点にご注意ください。

`{"token_type":"bearer","access_token":"AAAAAAAAAAAAAAAAAAAAAMLheAAAAAAA0%2BuSeid%2BULvsea4JtiGRiSDSJSI%3DEUifiRBkKG5E2XzMDjRfl76ZC9Ub0wnz4XsNiRVBChTYbJcE3F"}`

OAuth 2.0 でリソースにアクセスする際に使用する Bearer Token は次のとおりです。

`AAAAAAAAAAAAAAAAAAAAAMLheAAAAAAA0%2BuSeid%2BULvsea4JtiGRiSDSJSI%3DEUifiRBkKG5E2XzMDjRfl76ZC9Ub0wnz4XsNiRVBChTYbJcE3F`
