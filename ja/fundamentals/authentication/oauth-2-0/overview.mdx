<div id="bearer-token-also-known-as-app-only">
  ### ベアラートークン（アプリ専用とも呼ばれる）
</div>

OAuth 2.0 ベアラートークンは、[developer App](/ja/resources/fundamentals/developer-apps) に代わってリクエストを認証します。この方式はアプリに固有で、ユーザーは関与しません。主に公開情報へ読み取り専用でアクセスする必要がある開発者向けです。

この認証方式では、リクエストにベアラートークンを含める必要があります。トークンは developer Apps の「Keys and tokens」セクションで生成できます。以下は、偽のベアラートークンを使用したリクエスト例です:

```json
  curl "https://api.x.com/2/tweets?ids=1261326399320715264,1278347468690915330" \
  -H "Authorization: Bearer AAAAAAAAAAAAAAAAAAAAAFnz2wAAAAAAxTmQbp%2BIHDtAhTBbyNJon%2BA72K4%3DeIaigY0QBrv6Rp8KZQQLOTpo9ubw5Jt?WRE8avbi"
```

アプリ専用認証を使用する API 呼び出しは、アプリレベルでエンドポイントごとに[レート制限](/ja/resources/fundamentals/rate-limits)されます。

この方式を使用するにはベアラートークンが必要です。これは、[POST oauth2/token](/ja/resources/fundamentals/authentication/api-reference#post-oauth2-token) エンドポイントで API Key と Secret を送信して生成するか、[developer portal](/ja/resources/fundamentals/developer-portal) の App 設定内の「Keys and tokens」セクションで生成できます。

ベアラートークンを失効させたい場合は、[POST oauth2/invalidate\_token](/ja/resources/fundamentals/authentication/api-reference#post-oauth2-invalidate-token) エンドポイントを使用するか、App 設定内の「Keys and tokens」セクションでベアラートークンの横にある「Revoke」をクリックします。

<div id="oauth-20-authorization-code-flow-with-pkce">
  ### OAuth 2.0 認可コードフロー（PKCE）
</div>

OAuth 2.0 認可コードフロー（PKCE）は、別のユーザーに代わって認証を行う際にアプリケーションのスコープをより細かく制御でき、複数デバイス間での認可フローを改善します。言い換えると、X 上の利用者向けにアプリケーションを構築する開発者は、自身のアプリがユーザーに要求する情報をより厳密に管理できるようになり、エンドユーザーには必要なデータと情報だけを求めればよくなります。

この最新の認可プロトコルにより、エンドユーザーに対して、要求した特定のスコープのみを表示する、より洗練された同意フローを提示できます。これはデータ負荷を軽減するだけでなく、エンドユーザーからの信頼向上にもつながる可能性があります。
