<div id="basic-authentication">
  ## ベーシック認証
</div>

X のエンタープライズ API の多くは HTTP ベーシック認証を使用します。ベーシック認証が必要な API に正常にリクエストするには、各リクエストの認証ヘッダーに有効なメールアドレスとパスワードの組み合わせを渡す必要があります。メールアドレスとパスワードは[enterprise API console](https://console.gnip.com/) にログインする際のものと同じで、このコンソール内で編集できます。

ベーシック認証でリクエストを作成する際は、HTTPS 上でエンコードした認証情報を含む Authentication: Basic の HTTP ヘッダーを必ず追加してください。

次の cURL リクエスト例では、送信前に `<email_address>` と `<password>` を自身の認証情報に置き換えてください。

```bash
curl -v --compressed -u<email_address>:<password>
    "https://gnip-api.x.com/search/30day/accounts/<account-name>/prod/counts.json?query=from%3Axdevelopers"
```

**ベーシック認証が必要な API:**

- [PowerTrack API](/ja/x-api/enterprise-gnip-2.0/powertrack-api) enterprise
- [Decahose stream API](http://localhost:3000/x-api/enterprise-gnip-2.0/fundamentals/decahose-api) enterprise
- [30-Day Search API](/ja/x-api/enterprise-gnip-2.0/fundamentals/search-api) enterprise
- [Full-Archive Search API](/ja/x-api/enterprise-gnip-2.0/fundamentals/search-api) enterprise
- [Usage API](/ja/x-api/enterprise-gnip-2.0/fundamentals/usage) enterprise
