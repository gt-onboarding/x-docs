X API のエンドポイントにアクセスするには、TLS 接続が必須です。TLS 上で通信することで、公開インターネットを経由する間、ユーザーと X API 間の情報が保護され、ユーザーのプライバシーとセキュリティが維持されます。X API への接続には TLS 1.2 が必要です。


<div id="verification">
  ## 検証
</div>

<div id="use-an-up-to-date-root-store">
  ### 最新のルートストアを使用する
</div>

X の証明書を検証する際は、アプリケーションまたはライブラリで、信頼できる最新のルートストアを使用することが重要です。可能であれば、OS が提供するルートストアを使用するのが最も簡便な方法です。あるいは、[Mozilla（NSS）ルートストア](https://www.mozilla.org/en-US/about/governance/policies/security-group/certs/)は公開かつ透明性の高い形で適切に保守されています。curl でも、[このストアの PEM 形式版](https://curl.haxx.se/docs/caextract.html)が提供されています。

X は現在、証明書の大半を [DigiCert High Assurance EV Root CA](https://www.digicert.com/digicert-root-certificates.htm) から発行していますが、これは X 関連の証明書すべてに当てはまるわけではなく、今後も常に当てはまるとは限りません。したがって、現在使用されている DigiCert のルートのみを信頼対象とすると、将来的にアプリで問題が発生する可能性があります。

<div id="check-crls-and-the-ocsp-status">
  ### CRL と OCSP ステータスの確認[](#check-crls-and-the-ocsp-status "Permalink to this headline")
</div>

多くのアプリケーションは、返された証明書に対して証明書失効リスト（CRL）を確認しないか、確認をオペレーティングシステムに任せています。アプリケーションまたは TLS ライブラリが、X の証明書を受け入れる前に CRL および OCSP（Online Certificate Status Protocol）の検証を必須とするように設定されていることを確認してください。

<div id="cdns">
  ### CDN[](#cdns "この見出しへのパーマリンク")
</div>

メディアを含むツイートを表示する場合は、画像表示に使用する HTTPS URL として `media_url_https` 属性を使用してください。将来的には、API エンドポイントから提供されるすべての URL が HTTPS のパスを提供するようになります。

<div id="provide-an-indication-of-security-status">
  ## セキュリティ状態の表示を提供する
</div>

可能であれば、アプリケーションとXの間の現在の状態を示す表示を行ってください。ウェブブラウザによっては、錠前アイコンで示すものもあれば、現在の接続状態を説明的なメッセージで示すものもあります。
