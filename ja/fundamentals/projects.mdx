<div id="introduction">
  ## はじめに
</div>

プロジェクトは、X API の利用目的に基づいて作業を整理し、API へのアクセスを効果的に管理して使用状況を監視できるようにする仕組みです。各プロジェクトには、アクセスレベル（本ページ後半で説明）に応じて、1 つ以上の[アプリ](/ja/resources/fundamentals/developer-apps)を含められます。これらのアプリを使用して、[API Key と Secret](/ja/resources/fundamentals/authentication#api-key-and-secret)、ユーザーの[アクセス トークン](/ja/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow)、および[アプリ アクセス トークン](/ja/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) などの[認証](/ja/resources/fundamentals/authentication)用クレデンシャルを生成します。

これらのキーとトークンは、どのアプリのものでも X API または X 広告 API へのアクセスに使用できます（X 広告 API を利用するには[追加アクセスの申請](/ja/x-ads-api/getting-started/step-by-step-guide)が必要です）。ただし、[X API v2](/ja/x-api/introduction) のエンドポイントを使用するには、プロジェクトに関連付けられたアプリのキーとトークンを使用する必要があります。

開発者アカウントをお持ちの場合は、開発者ポータルの[Projects & Apps](https://developer.x.com/en/portal/projects-and-apps)ページでプロジェクトを表示・管理できます。お持ちでない場合は、[Sign up](https://developer.x.com/en/portal/petition/essential/basic-info)からアカウントを作成してください。

<div id="projects-and-x-api-access-levels">
  ### プロジェクトと X API のアクセスレベル
</div>

現時点では、プロジェクト単位で適用されるアクセスレベルは 4 種類あります。

- Free
- Basic
- Pro
- Enterprise

各アクセスレベルで提供される内容の詳細は、[about X API](/ja/x-api/getting-started/about-x-api) ページをご覧ください。

Free、Basic、Pro のいずれかのアクセスレベルを持つプロジェクトは、1 つのみ作成できます。

ここでは、プロジェクト固有の違いについていくつか説明します。

<div id="free">
  #### Free
</div>

このアクセスティアは、開発者アカウントに[登録](https://developer.twitter.com/en/portal/petition/essential/basic-info)したすべての方に提供されます。

そのプロジェクト内のアプリ数: 1
[投稿上限](/ja/x-api/fundamentals/post-cap): 月あたり1,500件の投稿

認証方法:

- [OAuth 2.0（PKCE）](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
- アプリ専用

<div id="basic">
  #### Basic
</div>

このアクセスティアは、開発者ポータルで Basic アクセスを購読したすべての方に提供されます。

そのプロジェクト内のアプリ数: 2
[投稿上限](/ja/x-api/fundamentals/post-cap): 月間 10,000 件の投稿
認証方法:

- [OAuth 2.0（PKCE）](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
- [OAuth 1.0a](/ja/resources/fundamentals/authentication#oauth-1-0a-2)
- アプリのみ

<div id="pro">
  #### Pro
</div>

このアクセス層は、開発者ポータルで Pro アクセスに加入したすべての方に提供されます。

そのプロジェクト内のアプリ数: 3
[Tweet consumption cap](/ja/x-api/fundamentals/post-cap): 月間 100 万件のツイート

認証方法:

- [OAuth 2.0（PKCE）](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
- [OAuth 1.0a](/ja/resources/fundamentals/authentication#oauth-1-0a-2)
- アプリのみ

<div id="enterprise">
  #### Enterprise
</div>

このアクセス階層は、Enterprise に申請し、当社チームの承認を受け、X のアカウントマネージャー経由で月次の自動更新契約に署名された方に提供されます。[こちらから申請](https://docs.google.com/forms/d/e/1FAIpQLScO3bczKWO2jFHyVZJUSEGfdyfFaqt2MvmOfl_aJp0KxMqtDA/viewform)。

そのプロジェクト内のアプリ数: 3+
[投稿上限](/ja/x-api/fundamentals/post-cap): 月あたり 5,000 万件以上の投稿

認証方法:

- [OAuth 2.0（PKCE）](/ja/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2)
- [OAuth 1.0a](/ja/resources/fundamentals/authentication#oauth-1-0a-2)
- アプリのみ

<div id="tweet-caps-and-projects">
  ### 投稿上限とプロジェクト
</div>

投稿の消費上限はプロジェクト単位で適用され、各月において、特定の X API v2 エンドポイントから取得できる投稿数を実質的に制限します。

[投稿上限](/ja/x-api/fundamentals/post-cap)の詳細をご確認ください。

<div id="configuring-projects">
  ### プロジェクトの構成
</div>

<div id="creating-a-project">
  #### プロジェクトの作成
</div>

プロジェクトを作成するには、[dashboard](https://developer.twitter.com/en/portal/dashboard) または開発者ポータル内の [Projects & Apps](https://developer.twitter.com/en/portal/projects-and-apps) ページで「New Project」をクリックします。すでにプロジェクトを作成している場合のみ、このオプションは表示されません。プロジェクト名、説明、ユースケースの入力を求められます。さらに、新しい App を作成するか、既存のスタンドアロンの App を接続するよう求められます。

<div id="creating-or-connecting-an-app-for-your-project">
  #### プロジェクト用のアプリの作成または接続
</div>

プロジェクトにアプリが含まれていない場合は、ダッシュボードでプロジェクト名をクリックして追加できます。そこから、新しいアプリを作成するか、既存のスタンドアロンアプリを選択してプロジェクトに接続できます。アプリでは、このガイドの冒頭に記載の認証キーおよびトークンを生成できます。

<div id="editing-a-project">
  #### プロジェクトの編集
</div>

プロジェクトを編集するには、開発者ポータルのダッシュボードまたは「Projects & Apps」ページで、編集したいプロジェクト名をクリックします。プロジェクトの詳細ページが表示されるので、「Edit」を選択して変更を行います。

<div id="standalone-apps">
  ### スタンドアロンアプリ
</div>

スタンドアロンアプリは、プロジェクト構造の外に存在するアプリです。これらのスタンドアロンアプリに関連付けられた認証情報は、X の API の standard v1.1、premium v1.1、enterprise、または X 広告 API へのリクエストを正常に実行できます。スタンドアロンアプリは、プロジェクトに接続しない限り、X API v2 のエンドポイントへのリクエストは失敗します。

2020 年 8 月以前にアプリを作成している場合は、開発者ポータルの [Projects & Apps](https://developer.twitter.com/en/portal/projects-and-apps) の「Standalone Apps」セクションに表示されます。プロジェクトに接続されているアプリを含め、合計で最大 10 個のアプリに制限されます。

チームアカウントのメンバーである場合、スタンドアロンアプリの下には自分が所有するアプリが表示されます。チームメイトがプロジェクトの一部であるアプリを所有している場合は、アプリ名と所有者情報は確認できますが、設定の変更、キーやトークンへのアクセス、あるいはそれらの再生成はできません。アプリの変更が必要な場合は、アプリ所有者に連絡してください。
