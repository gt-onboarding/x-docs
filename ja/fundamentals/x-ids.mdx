---
title: X IDs
icon: id-card-clip
---

X 内の各オブジェクト（投稿、ダイレクトメッセージ、ユーザー、リストなど）には、一意の ID があります。

プラットフォーム初期には、これらの ID は十分に小さい数値で、連番で生成できました。成長に対応するため、ID は 32 ビットから 64 ビットへ移行しました。現在、X の ID は一意の 64 ビット符号なし整数で、連番ではなく時刻に基づいています。ID はタイムスタンプ、ワーカーナンバー、シーケンス番号で構成されます。X は、これらの ID を安定して生成するために「Snowflake」と呼ばれる内部サービスを開発しました（[詳細は X のブログをご覧ください](https://blog.x.com/engineering/en_us/a/2010/announcing-snowflake.html)）。

64 ビットのように大きな数値は、64 ビット未満で整数を表すプログラミング言語で問題を引き起こす可能性があります。たとえば JavaScript では、整数は 53 ビットまでに制限されています。これへの対処として、X API の初期設計（v1/1.1）では、ID 値は整数と文字列の 2 つの形式で返されていました。

```json
{"id": 10765432100123456789, "id_str": "10765432100123456789"}
```

ブラウザの JavaScript コンソールで `(10765432100123456789).toString()` を実行すると、結果は `"10765432100123458000"` になります。64 ビット整数はこの変換により精度を失います（これは「munging」と呼ばれることがあり、データが破壊的に変化することを指します）。

X の API v1.1 までは、精度低下を避けるため、常に数値の文字列表現を使用してください。

新しいバージョンの API では、大きな整数値はすべてデフォルトで文字列として表現されます。
