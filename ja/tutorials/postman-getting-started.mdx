<div id="introduction">
  ## はじめに
</div>

Postman は、グラフィカルユーザーインターフェースから API にリクエストを送信できるデスクトップ／ウェブアプリケーションです。API の機能を調査・検証する場合や、アプリケーションの問題をトラブルシューティングする場合は、X API、X 広告 API、ならびに Labs の各エンドポイントで Postman を使用することを推奨します。

現在、Postman コレクションを 2 種類提供しています：

<div id="prerequisites">
  ### 前提条件
</div>

X の Postman コレクションを使い始める前に、選択した X 開発者プラットフォームのツールで適切なアクセス権と認証情報を用意していることを確認してください。アクセスの取得については、[「はじめに」ページ](/ja/resources/platform-overview)をご覧ください。 

[開発者アカウント](https://developer.x.com/en/portal/petition/essential/basic-info)を取得し、[開発者アプリ](/ja/resources/fundamentals/developer-apps)を設定し、[認証](/ja/resources/fundamentals/authentication)用のキーとトークン一式を用意し、使用予定の特定の API にリクエストを送るための環境を適切にセットアップできたら、以下の手順に従って開始できます。
 

<div id="getting-started-with-xs-postman-collections">
  ## X の Postman コレクションを使い始める
</div>

<div id="step-one-add-one-of-the-x-postman-collections-to-your-account">
  ### ステップ1：X の Postman コレクションのいずれかをアカウントに追加する
</div>

Postman 内で使いたい特定のエンドポイントを自分で揃えることもできますが、こちらで準備済みの関連 API コレクションをご用意しています。前述の「Postman コレクション」セクションのいずれかのリンクをクリックすると、選択した API に対応するすべてのエンドポイントを含むコレクションが Postman アプリに追加されます。これらのコレクションは [Postman API Network](https://explore.postman.com/) でも利用可能です。各エンドポイントには、利用可能なパラメータ、サンプルレスポンス、認証方式があらかじめ設定されているため、認証情報とパラメータ値を追加するだけで機能の検証を始められます。

この例では、X の [API v2 コレクション](https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400) を使用します。 
 

<div id="step-two-add-your-keys-and-tokens-as-environmental-variables">
  ### ステップ2：キーとトークンを環境変数として追加する
</div>

コレクションを Postman のインスタンスに追加すると、自動的に「X API v2」という環境が作成されます。ここにキーとトークンを追加する必要があります。本ステップでは、開発者向けアプリから取得したキーとトークンを「X API v2」環境へ追加する手順を説明します。 

「X API v2」環境にキーとトークンを追加するには、Postman 画面右上の「manage environments」ボタンをクリックします。 
 

<Frame>
  <img src="/images/using-postman-1.png.twimg.1920.png" alt="この画像は、Postman コンソールで強調表示された「manage environments」ボタンを示しています。" />
</Frame>

環境の一覧から「X API v2」をクリックします。 

次に、Apps ダッシュボードで生成した各種キーとトークンに対応する変数をテーブルに追加します。テーブルの例は次のとおりです。

| VARIABLE | INITIAL VALUE | CURRENT VALUE |
| :--- | :--- | :--- |
| consumer\_key | `QAktM6W6DF6F7XXXXXX` | `QAktM6W6DF6F7XXXXXX` |
| consumer\_secret | `AJX560A2Omgwyjr6Mml2esedujnZLHXXXXXX` | `AJX560A2Omgwyjr6Mml2esedujnZLHXXXXXX` |
| access\_token | `1995XXXXX-0NGqVhk3s96IX6SgT3H2bbjOPjcyQXXXXXXX` | `1995XXXXX-0NGqVhk3s96IX6SgT3H2bbjOPjcyQXXXXXXX` |
| token\_secret | `rHVuh7dgDuJCOGeoe4tndtjKwWiDjBZHLaZXXXXXX` | `rHVuh7dgDuJCOGeoe4tndtjKwWiDjBZHLaZXXXXXX` |
| bearer\_token | `AAAAAAAAAAAAAAAAAAAAAL9v6AAAAAAA99t03huuqRYg0mpYAAFRbPR3XXXXXXX` | `AAAAAAAAAAAAAAAAAAAAAL9v6AAAAAAA99t03huuqRYg0mpYAAFRbPR3XXXXXXX` |

上記のテーブルで使用しているキーとトークンはダミーであり、リクエストに使用しても動作しません。 

認証情報を変数として追加し、「X API v2」環境が選択されていることを確認したら、X API v2 コレクションにリクエストを送信する準備は完了です。各エンドポイントの Authorization タブは、この環境から変数を自動的に継承するよう設定されています。 

Postman で User Access Tokens を使用する場合は、XXXX をスキップして、詳細な手順をご確認ください。

### ステップ3：エンドポイントを選択する 

次は、コレクションからエンドポイントを選び、リクエストの作成を始めます。右側のナビゲーションからエンドポイントを選択できます。表示イメージは次のとおりです。

<Frame>
  <img src="/images/using-postman-2.png.twimg.1920.png" alt="この画像は、“X API v2” セクションの “Post Lookup” ドロップダウン内で “Single Posts” リクエストが選択されている様子を示しています。" />
</Frame>

この例では、X API v2 > Post Lookup > Single Post エンドポイントを使用します。 

<div id="step-four-add-values-to-the-params-tab">
  #### ステップ4：Params タブに値を追加する
</div>

次のステップでは、Params タブに移動します。ここでは、各パラメータの説明（そのパラメータの役割）と、リクエストで指定できる可能な値の一覧が付いた、未アクティブのパラメータが表示されます。

この例では、`expansions` と `tweet.fields` のクエリパラメータを有効にし、次の値を追加します。

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `tweet.fields` | `created_at,attachments` |
| expansions | author\_id |

クエリパラメータの追加に加えて、必須の Path Variable である `id` も追加する必要があります。このエンドポイントは投稿（Posts）を返すため、値として有効な Post ID を指定してください。

Post ID は、x.com で対象の投稿（Post）を開き、URL を確認すると見つかります。たとえば、次の URL の Post ID は `1228393702244134912` です。

`https://x.com/XDevelopers/status/1228393702244134912`

Params タブで、すべてのクエリパラメータの下までスクロールし、「Path Variables」セクションを表示します。使用したい Post ID を `id` キーの値として追加します。

このステップの入力がすべて正しければ、Params タブは次のようになります。

  <img src="/images/using-postman-3.png.twimg.1920.png" alt="この画像は、ページ前半の手順に基づいて入力された「Params」テーブルを示しています。" />
</Frame>

<div id="step-five-send-your-request-and-review-your-response">
  #### ステップ5：リクエストを送信し、レスポンスを確認する
</div>

リクエストの設定が完了したら、「Send」ボタンをクリックします。

すべて正しく設定されていれば、次のペイロードが返ってきます。

```json
{
    "data": {
        "author_id": "2244994945",
        "text": "What did the developer write in their Valentine’s card?\n  \nwhile(true) {\n    I = Love(You);  \n}",
        "id": "1228393702244134912",
        "created_at": "2020-02-14T19:00:55.000Z"
    },
    "includes": {
        "users": [
            {
                "username": "XDevelopers",
                "name": "Developers",
                "id": "2244994945"
            }
        ]
    }
}
```

<div id="generating-a-user-access-token-with-postman">
  ### Postman を使ってユーザーアクセス・トークンを生成する:
</div>

<div id="using-oauth-10a-to-generate-a-user-access-token">
  #### OAuth 1.0a を使用してユーザーアクセス トークンを生成する
</div>

[OAuth 1.0a フローテストコレクション](https://www.postman.com/xapidevelopers/workspace/twitter-s-public-workspace/collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400?ctx=documentation)で採用されている 3 段階のプロセスを確認してください。

<div id="using-oauth-20-to-generate-a-user-access-token">
  #### OAuth 2.0 を使用してユーザーアクセストークンを生成する
</div>

Postman で OAuth 2.0 のユーザーアクセストークンを生成する場合は、X の[API v2 Postman コレクション](https://www.postman.com/xapidevelopers/workspace/twitter-s-public-workspace/collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400?ctx=documentation)と併用するための OAuth 2.0 アクセストークンを作成できます。

ワークスペース内のコレクションをクリックし、「Auth」タブを開いて、タイプを「OAuth 2.0」に選択します。続いて、「Configure New Token」という見出しの下にある「Configuration Options」に進みます。「Grant Type」を「Authorization Code (With PKCE)」に設定できます。

Callback URL を、アプリケーションに関連付けられているコールバック URL と一致するように更新してください。あわせて、以下のパラメータも更新してください:

- Auth URL - https://x.com/i/oauth2/authorize
- Access Token URL - https://api.x.com/2/oauth2/token
- Client ID - Dev Portal の OAuth 2.0 クライアント ID
- Client Secret - 機密クライアントを使用している場合
- Scope - 接続したいエンドポイントに合わせたスコープ（例: “tweet.read users.read”）
- Callback URL（リダイレクト URL とも呼ばれます）。App の認証設定にある値と一致している必要があります。
- State - state

準備ができたら「Get New Access Token」をクリックしてアクセストークンを生成します。「問題が発生しました」というダイアログが表示された場合は、ログイン画面に戻るために戻るボタンを押す必要がある場合があります。ダイアログ内の「Authorize app」をクリックして、アプリにアカウントへのアクセスを許可してください。

アプリを承認すると、Postman にリダイレクトされ、トークンが表示されます。「Use Token」ボタンを選択して、承認済みユーザーとしてリクエストの送信を開始できます。

これで Postman コレクションを使用する準備が整いました。

  ## 次のステップ
</div>

Postman の「Code」ボタンをクリックすると、先ほど作成したリクエストを Python、Node、Ruby などお好みの言語に変換でき、スムーズに始められます。Postman には役立つ[優れたドキュメント](https://learning.getpostman.com/)があります。エンドポイントへの統合をより迅速に進められるよう、[GitHub 上のサンプルコード](https://github.com/xdevplatform)も用意しています。
