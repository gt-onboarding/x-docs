oEmbed API を使用すると、[ポスト](https://developer.x.com/en/docs/twitter-for-websites/embedded-tweets/overview)や[タイムライン](https://developer.x.com/en/docs/twitter-for-websites/timelines/overview)などの埋め込みコンテンツをプログラムで取得できます。

oEmbed API のレスポンスは、[X のウィジェット用 JavaScript をページに読み込む](https://developer.x.com/web/javascript/loading)と自動的に認識される HTML スニペットを返します。

この API は一括処理での利用を推奨しています。コンテンツの埋め込みには高機能な [publish.x.com](https://publish.x.com/#) ツールのご利用をお勧めします。

<Tabs>
  <Tab title="Embedded timelines">
    返される HTML スニペットは、[X のウィジェット JavaScript をページに読み込んだ場合](https://developer.x.com/web/javascript/loading)、自動的に[埋め込みタイムライン](https://developer.x.com/en/docs/twitter-for-websites/timelines/overview)として認識されます。

    oEmbed エンドポイントでは、HTML レスポンスに同梱された X の JavaScript が解釈する対応プロパティを HTML マークアップ内で設定することで、埋め込みタイムラインの最終的な見た目をカスタマイズできます。返されるマークアップの形式は、X が新機能を追加したりタイムラインの表現を調整したりするに伴い、将来的に変更される場合があります。

    タイムライン URL で指定された X タイムラインについて、[oEmbed](https://oembed.com/) 互換の JSON 形式を返します。ユーザーおよびリストのタイムラインをサポートします。タイムラインのマークアップは、cache\_age プロパティで指定された推奨キャッシュ存続期間の範囲内で、あなたのサーバー上にキャッシュしておくことを意図しています。

    ## リソース URL

    _https://publish.x.com/oembed_

    ## リソース情報

    |     |     |
    | :--- | :--- |
    | レスポンス形式 | JSON |
    | 認証が必要か？ | いいえ  |
    | レート制限 | いいえ  |

    ## パラメータ

    | 名前 | 説明 | 例 |
    | :--- | :--- | :--- |
    | **url** | 埋め込む X タイムラインの URL | \*   https://x.com/TwitterDev<br /> _https://x.com/TwitterDev/lists/national-parks
    | limit | 表示する項目数を最大 N 件にします。N は 1 から 20（両端含む） | 6   |
    | maxwidth | ウィジェットの最大幅を設定します。180 から 1200（両端含む）である必要があります | 300 |
    | maxheight | ウィジェットの最大高さを設定します。200 より大きい必要があります | 400 |
    | omit\_script | レスポンスに script 要素を含めない | 1   |
    | lang | サポートされている X の[言語コード](/ja/en/docs/twitter-for-websites/twitter-for-websites-supported-languages/overview "Twitter language code") | es  |
    | theme | dark に設定すると、暗い背景に明るいテキストでタイムラインが表示されます | dark |
    | chrome | スペース区切りのトークンでタイムライン表示コンポーネントを削除<br /><br />_   noheader - ヘッダーを非表示<br />\*   nofooter - 表示されている場合はフッターを非表示<br />\*   noborders - すべてのボーダーを削除：ウィジェットの周囲、ツイート間、およびツイート内<br />\*   noscrollbar - 表示されている場合はタイムラインのスクロールバーを切り抜いて非表示<br />\*   transparent - 背景色を削除 | noheader%20nofooter |
    | aria\_polite | タイムラインに追加されるツイートに対して、[ARIA ライブリージョンの丁寧さ](https://www.w3.org/TR/wai-aria/states_and_properties#aria-live)を assertive に設定 | assertive |
    | dnt | true に設定すると、タイムラインおよびあなたのサイト上に埋め込まれたそのページは、[パーソナライズされたおすすめ](https://support.x.com/articles/20169421)や[パーソナライズ広告](https://support.x.com/articles/20170405)などの目的に使用されません | true |

    ## リクエスト例

    ```bash
    curl --request GET --url 'https://publish.x.com/oembed?url=https%3A%2F%2Ftwitter.com%2FInterior%2Fstatus%2F507185938620219395'
    twurl -H publish.x.com "/oembed?url=https://x.com/Interior/status/463440424141459456"
    ```

  <Tab title="Embedded Tweets">
    返された HTML スニペットは、[X のウィジェット JavaScript をページに読み込んでいる場合](https://developer.x.com/web/javascript/loading)、自動的に[埋め込みツイート](https://developer.x.com/web/embedded-tweets)として認識されます。

    oEmbed エンドポイントでは、HTML レスポンスに同梱された X の JavaScript によって解釈される対応プロパティを HTML マークアップで指定することで、埋め込み投稿の最終的な外観をカスタマイズできます。返されるマークアップの形式は、X が新機能を追加したり投稿の表現を調整したりするに伴い、時間の経過とともに変更される場合があります。

    Tweet のフォールバックマークアップは、`cache_age` プロパティで指定された推奨キャッシュ期間に従い、その上限までサーバー側でキャッシュすることを想定しています。

    ## パラメータ

    | 名前 | 既定値 | 説明 |
    | :--- | :--- | :--- |
    | `url`必須  <br />文字列 |     | 埋め込む投稿の URL |
    | `maxwidth`  <br />整数 `[220..550]` | `325` | 描画される投稿の最大幅（ピクセル単位、整数）。許容範囲を下回る／上回る値が指定された場合は、最小または最大のサポート幅として返され、そのリセット後の幅は返される `width` プロパティに反映されます。なお、X は oEmbed の `maxheight` パラメータをサポートしていません。投稿は本質的にテキストであり、高さは予測不能で、画像や動画のようにスケーリングできません。同様に、oEmbed レスポンスは `height` の値を提供しません。投稿の高さを一定にする必要がある場合は、以下の `hide_thread` と `hide_media` パラメータを参照してください。 |
    | `hide_media`  <br />Boolean、String、または Int | `false` | `true`、`"t"`、または `1` を指定すると、投稿内のリンクは写真・動画・リンクのプレビューに展開されません。 |
    | `hide_thread`  <br />Boolean、String、または Int | `false` | `true`、`"t"`、または `1` を指定すると、要求された投稿が別の投稿への返信である場合、会話スレッド内の直前の投稿の折りたたみ表示は行われません。 |
    | `omit_script`  <br />Boolean、String、または Int | `false` | `true`、`"t"`、または `1` を指定すると、`widgets.js` の読み込みを行う `<script>` は返されません。ウェブページ側で、[埋め込み投稿](https://developer.x.com/web/embedded-tweets) を含むすべての X ウィジェットで利用できるよう、`widgets.js` への参照を自身で含めてください。 |
    | `align`  <br />Enum `{left,right,center,none}` | `none` | 埋め込み投稿を、親要素に対して左寄せ・右寄せ・中央寄せで回り込ませるかどうかを指定します。 |
    | `lang`  <br />Enum([言語](https://developer.x.com/en/docs/twitter-for-websites/twitter-for-websites-supported-languages/overview)) | `en` | 指定した [埋め込み投稿でサポートされる X の言語](https://developer.x.com/web/overview/languages) で、返される HTML と描画される投稿を要求します。 |
    | `theme`  <br />Enum `{light, dark}` | `light` | `dark` を指定すると、暗い背景に明るい文字で表示されます。 |
    | `dnt`  <br />Boolean | `false` | `true` を指定すると、あなたのサイト上の当該投稿およびその埋め込みページは、[パーソナライズされたおすすめ](https://support.x.com/articles/20169421) や [パーソナライズド広告](https://support.x.com/articles/20170405) などの目的には使用されません。 |
